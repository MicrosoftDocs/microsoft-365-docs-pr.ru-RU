# <a name="configurable-settings-reference---microsoft-managed-desktop"></a>Справочные сведения о настраиваемых параметрах — управляемый Майкрософт Настольный компьютер

В этом разделе перечислены категории параметров, которые пользователи могут настроить с помощью управляемого рабочего стола Майкрософт. Каждая категория параметров включает сведения о требованиях, рекомендациях и способах настройки категории параметров. 

## <a name="desktop-background-picture"></a>Фоновый рисунок рабочего стола
Вы можете настроить фоновый рисунок рабочего стола для устройств, управляемых Майкрософт, в Организации. Это можно использовать для применения фирменной символики компании или маркетинговых материалов. 

### <a name="requirements"></a>Требования

Для фонового рисунка на рабочем столе должны быть выполнены следующие требования:
- Формат файла изображений — JPG, JPEG или PNG
- Расположение файлов — узел в надежном расположении Secure HTTP (HTTPS). 
- Не разрешено: HTTP и расположения файлов общего доступа (UNC) не поддерживаются. 

### <a name="customize-and-deploy-desktop-background-picture"></a>Настройка и развертывание фонового рисунка рабочего стола

**Добавление пользовательского фонового рисунка рабочего стола**
1. Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2. В разделе **Параметры**выберите пункт **Настраиваемая**.
3. В **настраиваемой** рабочей области выберите пункт **фоновый рисунок на рабочем столе**. 
4. Введите путь к рисунку, который вы хотите использовать. 
5. Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе. 

## <a name="browser-start-pages"></a>Начальные страницы браузера
Начальная страница браузера открывается на отдельных вкладках, когда пользователи запускают Microsoft Edge. Чтобы пользователи могли легко открыть набор часто используемых сайтов, добавьте начальную страницу браузера для каждого сайта. 

### <a name="requirements"></a>Требования

Для начальных страниц браузера необходимо указать полное доменное имя (FQDN) для веб-сайтов интрасети или Интернета. Если настроены внутренние сайты, сообщите пользователям о том, что доступ к этим сайтам разрешен только при подключении к внутренней сети в офисе или при подключении к VPN-подключению. 

### <a name="customize-and-deploy-browser-start-pages"></a>Настройка и развертывание начальных страниц браузера

**Добавление начальной страницы браузера**
1. Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2. В разделе **Параметры**выберите пункт **Настраиваемая**.
3. В **настраиваемой** рабочей области выберите **начальные страницы браузера**. 
4. Выберите **Добавить начальную страницу**.
5. На **странице "Запуск браузера**" введите URL-адрес сайта, который необходимо использовать, а затем нажмите кнопку **Добавить начальную страницу**. 
6. Повторите шаги 1-5 для дополнительных начальных страниц браузера. 
7. Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе.

## <a name="enterprise-mode-site-list-location"></a>Расположение списка сайтов в режиме предприятия

Если у вас есть определенные веб-сайты и приложения, у которых есть проблемы совместимости с Microsoft EDGE, вы можете использовать список сайтов режима предприятия, чтобы веб-сайты автоматически открывались с помощью Internet Explorer 11. Кроме того, если вы знаете, что сайты интрасети не будут правильно работать с Microsoft EDGE, вы можете настроить все сайты интрасети для автоматического открытия с помощью Internet Explorer 11. Использование режима предприятия означает, что вы можете продолжать использовать Microsoft EDGE в качестве используемого по умолчанию браузера, а также убедиться, что ваши приложения продолжают работать в Internet Explorer 11. Дополнительные сведения о списках сайтов в корпоративном режиме см. в разделе [Корпоративный режим и список сайтов в режиме](https://docs.microsoft.com/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode)предприятия. 

Можно указать расположение https://или расположение для внутреннего общего ресурса, в котором вы размещаете список сайтов в режиме предприятия. 

### <a name="requirements"></a>Требования

Эти требования должны выполняться для файла списка сайтов в режиме предприятия:
- Формат файла XML-файл, соответствующий [требованиям к файлам](https://docs.microsoft.com/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode#site-list-xml-file)
- Location файл — файл узла во внутреннем расположении HTTPS. 
- Не разрешается размещение на внутреннем файловом ресурсе, например *//шаренаме*, запрещено

### <a name="best-practices"></a>Рекомендации

Эти рекомендации помогут клиентам принимать решения, модернизация их ИТ – инфраструктуру.
- **Выберите ограниченное число сайтов** — управляемый корпорацией Майкрософт настольный браузер использует Microsoft EDGE в качестве предпочтительного браузера для оптимизации общей безопасности Организации и удобства использования для пользователей. Большинство сайтов в этом списке предназначены для устаревших веб-приложений, которым необходима более старая версия браузера, не включающая в себя все функции безопасности. 
- **Рассмотрите альтернативу** – рассмотрите другой сайт или веб-приложение, для которого не требуется более старая версия браузера. Или обновите сайт, чтобы он мог использовать новые браузеры. Новые браузеры используют новейшие технологии и помогают повысить безопасность.

### <a name="customize-and-deploy-enterprise-site-mode-list-location"></a>Настройка и развертывание расположения списка в режиме корпоративного сайта

**Добавление расположения списка в режиме корпоративного сайта**

1.  Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2.  В разделе **Параметры**выберите пункт **Настраиваемая**.
3.  В **настраиваемой** рабочей области выберите **Расположение списка сайтов в режиме предприятия**. 
4.  Введите расположение HTTPS для списка сайтов. 
5.  Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе.

## <a name="trusted-sites"></a>Надежные сайты

Надежные сайты позволяют настраивать зоны безопасности или сайты, которые можно использовать для разных сайтов. К зонам безопасности относятся: 
- Зона 1 — зона местной интрасети
- Зона 2 — зона надежных сайтов
- Зона 3 — зона Интернета
- Зона 4 — Зона ограниченных сайтов

### <a name="requirements"></a>Требования

Укажите полное доменное имя (FQDN) для сайтов интрасети или Интернета для каждого надежного сайта. 

### <a name="customize-and-deploy-trusted-sites"></a>Настройка и развертывание надежных сайтов

**Добавление надежного сайта**

1. Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2. В разделе **Параметры**выберите пункт **Настраиваемая**.
3. В **настраиваемой** рабочей области выберите **Надежные сайты**, а затем нажмите **Добавить доверенный сайт**. 
4. На **странице Добавление надежного сайта**введите URL-адрес, выберите зону безопасности, а затем нажмите **Добавить доверенный сайт**. 
5. Повторите шаги 1-4 для каждого надежного сайта, который нужно добавить. 
6. Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе.

**Удаление надежного сайта**

1. Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2. В разделе **Параметры**выберите пункт **Настраиваемая**.
3. В **настраиваемой** рабочей области выберите **Надежные сайты**. 
4. Выберите сайт, который нужно удалить, а затем нажмите кнопку **Удалить**. 
5. Повторите шаги 1-4 для каждого надежного сайта, который требуется удалить. 
6. Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе.

## <a name="proxy"></a>Сервера
Для организации можно управлять параметрами прокси-сервера сети. Добавьте прокси-сервер и номер порта, а затем добавьте исключения прокси-сайта. Рабочий стол, управляемый Майкрософт, включает набор исключений прокси-серверов по умолчанию, необходимых для работы службы. Список исключений по умолчанию может быть изменен только службой "управляемые настольные компьютеры Майкрософт".  Дополнительные сведения можно найти в разделе [Настройка сети для управляемого рабочего стола Майкрософт](../get-ready/network.md). 

Исключения прокси-сайта, которые вы добавляете на портале для настольных компьютеров Майкрософт, добавляются к исключениям прокси-сервера по умолчанию, включенным в службу управляемых компьютеров Майкрософт. 

> [!NOTE]
> Обновление списка исключений прокси-сервера по умолчанию всегда назначается по приоритету при развертывании клиентов. Это означает, что поэтапное развертывание будет приостановлено при наличии развертывания для списка исключений прокси-сервера по умолчанию.  

### <a name="requirements"></a>Требования

Эти требования должны выполняться для исключений прокси-сервера и прокси-сервера.
- Должен быть допустимым адресом сервера и номером порта
- URL-адреса должны быть допустимым сайтом http 

### <a name="customize-and-deploy-proxies"></a>Настройка и развертывание прокси-серверов

**Добавление исключения для отдельного прокси-сайта**

1. Вход на [портал администрирования рабочих столов с управляемыми Майкрософт](https://aka.ms/mwaasportal)
2. В разделе **Параметры**выберите пункт **Настраиваемая**.
3. В **настраиваемой** рабочей области выберите **прокси-сервер**. 
4. Введите **адрес** и **номер порта** для прокси-сервера, а затем выберите **Добавить исключение прокси-** сервера. 
5. Введите URL-адрес действительного сайта http, а затем выберите **Добавить исключение прокси-сервера**. 
6. Повторите шаги 1-5 для каждого надежного сайта, который нужно добавить. 
7. Выберите **развертывание стадии** , чтобы сохранить изменения и развернуть их в тестовой группе.
