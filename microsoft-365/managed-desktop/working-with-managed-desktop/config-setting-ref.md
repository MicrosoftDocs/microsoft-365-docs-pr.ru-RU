# <a name="configurable-settings-reference---microsoft-managed-desktop"></a>Ссылка на настраиваемые параметры — компьютеры, управляемые Майкрософт

В этом разделе перечислены категории параметров, которые клиенты могут настроить с помощью компьютеры, управляемые Майкрософт. В каждой категории параметров содержатся сведения о требованиях, передовой практике и настройке категории параметров. 

## <a name="desktop-background-picture"></a>Фоновое изображение рабочего стола
Вы можете настроить фоновое изображение рабочего стола для компьютеры, управляемые Майкрософт устройств в организации. С помощью этого можно применить бренд компании или маркетинговый материал. 

### <a name="requirements"></a>Требования

Эти требования должны быть выполнены для фоновой картинки рабочего стола:
- Формат файла изображений — .jpg, jpeg или .png
- Расположение файла . Размещение на надежном безопасном http (https) расположении. 
- Не разрешено . Расположения http и share file (unc) не поддерживаются. 

### <a name="customize-and-deploy-desktop-background-picture"></a>Настройка и развертывание фоновой картинки рабочего стола

**Добавление настраиваемой фоновой картинки рабочего стола**
1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области выберите **фоновое изображение рабочего стола.** 
4. Введите расположение изображения, который вы хотите использовать. 
5. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test. 

## <a name="browser-start-pages"></a>Страницы запуска браузера
Страницы запуска браузера открываются на отдельных вкладок, когда пользователи начинают Microsoft Edge. Чтобы пользователям было проще открывать часто используемый набор сайтов, добавьте страницу запуска браузера для каждого сайта. 

### <a name="requirements"></a>Требования

Вы должны предоставить полное доменное имя (FQDN) для интрасети или веб-сайтов для страниц запуска браузера. Если настроены внутренние сайты, дайте пользователям знать, что доступ к этим сайтам разрешен только при под подключении к внутренней сети, когда в офисе или при подключении к VPN-подключению. 

### <a name="customize-and-deploy-browser-start-pages"></a>Настройка и развертывание страниц запуска браузера

**Добавление страницы запуска браузера**
1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области выберите **страницы запуска браузера.** 
4. Выберите **страницу Добавить начните**.
5. На **странице Добавить страницу** запуска браузера введите URL-адрес сайта, который вы хотите использовать, а затем выберите **страницу Добавить начните**. 
6. Повторите шаги 1-5 для дополнительных страниц запуска браузера. 
7. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test.

## <a name="enterprise-mode-site-list-location"></a>Enterprise списка сайтов режима

Если у вас есть определенные веб-сайты и приложения, которые, как вы знаете, имеют проблемы с совместимостью Microsoft Edge, вы можете использовать список сайтов Enterprise Mode, чтобы веб-сайты автоматически открывали с помощью Internet Explorer 11. Кроме того, если вы знаете, что ваши сайты интрасети не будут работать правильно с Microsoft Edge, вы можете настроить все сайты интрасети для открытия с помощью Internet Explorer 11 автоматически. Использование Enterprise Mode означает, что вы можете продолжать использовать Microsoft Edge в качестве браузера по умолчанию, а также обеспечить продолжение работы приложений в Internet Explorer 11. Дополнительные сведения о списках сайтов в режиме предприятия см. [в Enterprise Mode и Enterprise Mode Site Lists.](/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode) 

Вы можете указать https:// или расположение внутренней доли, в которой был опубликован список веб-сайтов режима предприятия. 

### <a name="requirements"></a>Требования

Эти требования должны быть выполнены для файла списка сайтов корпоративного режима:
- Формат файла — XML-файл, отвечающий [требованиям к файлам](/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode#site-list-xml-file)
- Расположение файла — хост-файл на внутреннем расположении https. 
- Запрещено размещение на внутренней папке файлов, например *//sharename,*

### <a name="best-practices"></a>Рекомендации

Эти лучшие практики предлагаются для принятия клиентами решений по модернизации ИТ-инфраструктуры:
- **Выберите ограниченное число** сайтов , компьютеры, управляемые Майкрософт использует Microsoft Edge в качестве предпочтительного браузера для повышения общей безопасности для организации и использования для пользователей. Большинство сайтов в этом списке относятся к устаревшим веб-приложениям, которые нуждаются в более старой версии браузера, которая не будет включать столько функций безопасности. 
- **Рассмотрим альтернативный** вариант — рассмотрим другой сайт или веб-приложение, которое не требует более старого браузера. Или подумайте об обновлении сайта, чтобы он можно было использовать более новые браузеры. Новые браузеры используют новейшие технологии и помогают повысить безопасность.

### <a name="customize-and-deploy-enterprise-site-mode-list-location"></a>Настройка и развертывание Enterprise списка сайтов

**Добавление расположения списка режимов корпоративного сайта**

1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области **выберите расположение списка Enterprise режима.** 
4. Введите расположение https для списка сайтов. 
5. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test.

## <a name="trusted-sites"></a>Надежные сайты

Доверенные сайты позволяют настраивать зоны безопасности или использовать веб-сайт для различных сайтов. Зоны безопасности включают: 
- Зона 1 — локализованная зона интрасети
- Зона 2 — зона надежных сайтов
- Зона 3 — зона Интернета
- Зона 4 — зона ограниченных сайтов

### <a name="requirements"></a>Требования

Предоставление полностью квалифицированного доменного имени (FQDN) для интрасети или сайтов в Интернете для каждого надежного сайта. 

### <a name="customize-and-deploy-trusted-sites"></a>Настройка и развертывание надежных сайтов

**Добавление доверенного сайта**

1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области выберите **доверенные сайты,** а затем выберите **Добавить надежный сайт**. 
4. На **добавлении доверенного** сайта введите URL-адрес, выберите зону безопасности, а затем выберите **добавить надежный сайт.** 
5. Повторите шаги по 1-4 для каждого доверенного сайта, который необходимо добавить. 
6. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test.

**Удаление доверенного сайта**

1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области выберите **доверенные сайты.** 
4. Выберите сайт, который необходимо удалить, а затем выберите **Удалить**. 
5. Повторите шаги по 1-4 для каждого надежного сайта, который необходимо удалить. 
6. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test.

## <a name="proxy"></a>Прокси
Вы можете управлять настройками сетевых прокси-серверов для вашей организации. Добавьте прокси-сервер и номер порта, а затем добавьте исключения прокси-сайта. компьютеры, управляемые Майкрософт включает набор исключений прокси по умолчанию, которые необходимы для работы службы. Список исключений по умолчанию может быть изменен только службой компьютеры, управляемые Майкрософт.  Дополнительные сведения см. в [сетевой конфигурации для компьютеры, управляемые Майкрософт.](../get-ready/network.md) 

Исключения прокси-сайта, которые вы добавляете на портале компьютеры, управляемые Майкрософт, добавляются к исключениям прокси по умолчанию, включенным в компьютеры, управляемые Майкрософт службу. 

> [!NOTE]
> Обновление списка исключений прокси по умолчанию всегда имеет приоритет над развертыванием клиентов. Это означает, что поэтапное развертывание будет приостановлено при развертывании списка исключений прокси по умолчанию.  

### <a name="requirements"></a>Требования

Эти требования должны быть выполнены для исключений прокси-сервера и прокси-сайта:
- Должен быть допустимый адрес сервера и номер порта
- URL-адреса должны быть допустимым веб-сайтом http 

### <a name="customize-and-deploy-proxies"></a>Настройка и развертывание прокси

**Добавление исключения отдельного прокси-сайта**

1. Войдите в [Microsoft Endpoint Manager](https://endpoint.microsoft.com/) и перейдите в меню **Devices**
2. Посмотрите раздел компьютеры, управляемые Майкрософт, выберите **Параметры**.
3. В **Параметры** рабочей области выберите **Прокси.** 
4. Введите **номер адрес** и **порт** для прокси-сервера, а затем выберите исключение **Добавить прокси.** 
5. Введите URL-адрес допустимого сайта http и выберите **исключение Добавить прокси.** 
6. Повторите шаги по 1-5 для каждого доверенного сайта, который необходимо добавить. 
7. Выберите **развертывание Stage,** чтобы сохранить изменения и развернуть их в группе Test.
