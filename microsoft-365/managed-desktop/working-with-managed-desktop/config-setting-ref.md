# <a name="configurable-settings-reference---microsoft-managed-desktop"></a>Справочник по настраиваемым настройкам — рабочий стол, управляемый Майкрософт

В этом разделе перечислены категории параметров, которые клиенты могут настраивать с помощью компьютеров, управляемых Майкрософт. Каждая категория параметров включает сведения о требованиях, практических методиках и настройке категории параметров. 

## <a name="desktop-background-picture"></a>Фоновое изображение рабочего стола
Вы можете настроить фоновый рисунок рабочего стола для настольных устройств, управляемых Майкрософт, в вашей организации. Его можно использовать для применения фирменой марки или маркетинговых материалов. 

### <a name="requirements"></a>Requirements

Эти требования должны быть выполнены для фонового рисунка рабочего стола:
- Формат файла рисунка — JPG, JPEG или PNG
- Расположение файла — размещение в надежном безопасном расположении http (https). 
- Не разрешено — расположения Http и файлового файла (unc) не поддерживаются. 

### <a name="customize-and-deploy-desktop-background-picture"></a>Настройка и развертывание фонового рисунка рабочего стола

**Добавление фонового рисунка пользовательского рабочего стола**
1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите **фоновое изображение рабочего стола.** 
4. Введите расположение изображения, который вы хотите использовать. 
5. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования. 

## <a name="browser-start-pages"></a>Начальные страницы браузера
При запуске Microsoft Edge пользователи открывают страницы запуска браузера на отдельных вкладок. Если вы хотите, чтобы пользователям было проще открывать часто используемый набор сайтов, добавьте начните страницу браузера для каждого сайта. 

### <a name="requirements"></a>Requirements

Необходимо предоставить полное доменное имя (FQDN) для сайтов интрасети или Интернета для страниц браузера. Если настроены внутренние сайты, дайте пользователям знать, что доступ к этим сайтам разрешен только при подсети к внутренней сети в офисе или при подсети VPN. 

### <a name="customize-and-deploy-browser-start-pages"></a>Настройка и развертывание страниц запуска браузера

**Добавление страницы запуска браузера**
1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите **"Начните страницу браузера".** 
4. Выберите **"Добавить начните страницу".**
5. На **странице "Добавление запуска браузера"** введите URL-адрес сайта, который вы хотите использовать, а затем выберите **"Добавить начните страницу".** 
6. Повторите шаги 1–5 для дополнительных страниц браузера. 
7. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования.

## <a name="enterprise-mode-site-list-location"></a>Расположение списка сайтов в режиме предприятия

Если у вас есть определенные веб-сайты и приложения, которые, как вы знаете, имеют проблемы совместимости с Microsoft Edge, вы можете использовать список сайтов в режиме предприятия, чтобы веб-сайты автоматически открывался с помощью Internet Explorer 11. Кроме того, если вы знаете, что сайты интрасети не будут правильно работать с Microsoft Edge, вы можете настроить автоматическое открытие всех сайтов интрасети с помощью Internet Explorer 11. Использование режима предприятия означает, что вы можете продолжать использовать Microsoft Edge в качестве браузера по умолчанию, а также обеспечить работу ваших приложений в Internet Explorer 11. Дополнительные сведения о списках сайтов в режиме предприятия см. в списках сайтов в режиме предприятия и [в режиме предприятия.](https://docs.microsoft.com/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode) 

Вы можете указать https:// или расположение для внутренней совместной работы, в которой был расположен список сайтов в режиме предприятия. 

### <a name="requirements"></a>Requirements

Эти требования должны быть выполнены для файла списка сайтов в режиме предприятия:
- Формат файла — XML-файл, который соответствует [требованиям к файлам](https://docs.microsoft.com/internet-explorer/ie11-deploy-guide/what-is-enterprise-mode#site-list-xml-file)
- Расположение файла — файл хоста во внутреннем расположении HTTPS. 
- Не разрешено — размещение во внутренней файловой папке, например *//sharename,* запрещено

### <a name="best-practices"></a>Рекомендации

Эти методики помогут клиентам принять решения по модернизации ИТ-инфраструктуры:
- **Выберите ограниченное количество** сайтов — на компьютере, управляемом Майкрософт, в качестве предпочтительного браузера используется Microsoft Edge, чтобы повысить общую безопасность вашей организации и возможности использования для пользователей. Большинство сайтов в этом списке относятся к устаревшим веб-приложениям, которые нуждаются в более старой версии браузера, включаемом не так много функций безопасности. 
- **Рассмотрим альтернативный** вариант— рассмотрим другой сайт или веб-приложение, для работы с чем-то не требуется старый браузер. Кроме того, можно обновить сайт, чтобы он можно было использовать более новые браузеры. Новые браузеры используют новейшие технологии и помогают повысить безопасность.

### <a name="customize-and-deploy-enterprise-site-mode-list-location"></a>Настройка и развертывание расположения списка в режиме корпоративного сайта

**Добавление расположения списка в режиме предприятия**

1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите расположение списка сайтов **в режиме предприятия.** 
4. Введите расположение https для списка сайтов. 
5. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования.

## <a name="trusted-sites"></a>Надежные сайты

Надежные сайты позволяют настраивать зоны безопасности или использовать сайты для разных сайтов. Зоны безопасности включают: 
- Зона 1 — зона локальной интрасети
- Зона 2 — зона надежных сайтов
- Зона 3 — зона Интернета
- Зона 4 — зона ограниченных сайтов

### <a name="requirements"></a>Requirements

Удать полное доменное имя (FQDN) для сайтов интрасети или Интернета для каждого доверенного сайта. 

### <a name="customize-and-deploy-trusted-sites"></a>Настройка и развертывание надежных сайтов

**Добавление доверенного сайта**

1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите **"Надежные сайты",** а затем выберите **"Добавить доверенный сайт".** 
4. На **сайте "Добавление надежного** сайта" введите URL-адрес, выберите зону безопасности, а затем выберите **"Добавить доверенный сайт".** 
5. Повторите шаги 1–4 для каждого доверенного сайта, который нужно добавить. 
6. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования.

**Удаление доверенного сайта**

1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите **надежные сайты.** 
4. Выберите сайт, который нужно удалить, а затем выберите **"Удалить".** 
5. Повторите шаги 1–4 для каждого надежного сайта, который нужно удалить. 
6. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования.

## <a name="proxy"></a>Прокси-сервер
Вы можете управлять настройками прокси-сервера сети в организации. Добавьте прокси-сервер и номер порта, а затем добавьте исключения прокси-сайта. Управляемый Майкрософт рабочий стол включает набор исключений прокси-сервера по умолчанию, необходимых для работы службы. Список исключений по умолчанию может быть изменен только службой компьютеров, управляемых Майкрософт.  Дополнительные сведения см. в [конфигурации сети для компьютеров, управляемых Майкрософт.](../get-ready/network.md) 

Исключения прокси-сайта, добавляемая на портале microsoft Managed Desktop, добавляются к исключениям прокси-сервера по умолчанию, включенным в службу управляемых компьютеров Майкрософт. 

> [!NOTE]
> Обновление списка исключений прокси-сервера по умолчанию всегда имеет приоритет над развертываниями клиентов. Это означает, что поэтапное развертывание будет приостановлено, если имеется развертывание для списка исключений прокси-сервера по умолчанию.  

### <a name="requirements"></a>Requirements

Эти требования должны быть выполнены для исключений прокси-сервера и прокси-сайта:
- Должен быть допустимым адресом сервера и номером порта
- URL-адреса должны быть допустимым http-сайтом 

### <a name="customize-and-deploy-proxies"></a>Настройка и развертывание прокси-прокси

**Добавление исключения отдельного прокси-сайта**

1. Войдите в [Microsoft Endpoint Manager и](https://endpoint.microsoft.com/) перейдите в меню **"Устройства"**
2. Наберем раздел "Рабочий стол, управляемый Майкрософт", выберите **"Параметры".**
3. В **рабочей области "Параметры"** выберите "Прокси".  
4. Введите **адрес и** номер порта **для** прокси-сервера и выберите исключение **"Добавить прокси-сервер".** 
5. Введите URL-адрес допустимого http-сайта и выберите исключение **"Добавить прокси-сервер".** 
6. Повторите шаги 1–5 для каждого доверенного сайта, который нужно добавить. 
7. Выберите **стадию развертывания,** чтобы сохранить изменения и развернуть их в группе тестирования.
