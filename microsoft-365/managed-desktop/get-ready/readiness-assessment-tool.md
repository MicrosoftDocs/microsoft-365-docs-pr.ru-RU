# <a name="readiness-assessment-tools"></a>Средства оценки готовности

Для максимально плавной регистрации в Microsoft Managed Desktop существуют параметры и другие параметры, которые необходимо установить заранее, а также определенные требования к устройству и сети для удовлетворения. Один из средств, доступный через портал администрирования управляемых настольных компьютеров Майкрософт, проверяет параметры, связанные с управлением. Другой инструмент, который можно загрузить, проверяет требования к отдельным устройствам и сетевые параметры. Эти средства можно использовать для проверки этих параметров и получения подробных действий по исправлению ошибок.

## <a name="downloadable-readiness-assessment-checker-for-devices-and-network"></a>Проверка готовности к загрузке для устройств и сети

Подробные сведения об использовании проверяемой проверки готовности для скачивания см. в публикации [Downloadable readiness assessment checker.](readiness-assessment-downloadable.md)

## <a name="online-readiness-assessment-tool-for-management-settings"></a>Средство оценки готовности в Интернете для параметров управления

[Он-лайн](https://aka.ms/mmdart) инструмент проверяет параметры в Microsoft Endpoint Manager (в частности, Microsoft Intune), Azure Active Directory (Azure AD) и Microsoft 365, чтобы убедиться, что они будут работать с Microsoft Managed Desktop. Управляемый рабочий стол Майкрософт сохраняет данные, связанные с этими проверками, в течение 12 месяцев после последнего запуска проверки в организации Azure AD (клиент). Через 12 месяцев мы сохраняем его в деидентифицированной форме. Вы можете удалить собранные данные.

Любой человек с ролью глобального читателя или администратора Intune сможет запустить этот инструмент, [](readiness-assessment-fix.md#multifactor-authentication) но для двух проверок[(политики](readiness-assessment-fix.md#conditional-access-policies) условного доступа и многофакторная проверка подлинности требуются дополнительные разрешения.
 
Средство оценки проверяет эти элементы:

## <a name="microsoft-intune-settings"></a>Параметры Microsoft Intune

|Чек  |Описание  |
|---------|---------|
|Профиль развертывания автопилота     | Проверяет, что назначение профиля развертывания автопилота не распространяется на  все устройства (профиль не должен быть назначен на любые устройства Microsoft Managed Desktop.)       |
|Соединители сертификатов     | Проверяет состояние соединители сертификатов, чтобы убедиться, что они активны   |
|Условный доступ     | Проверяет, что политики условного доступа не назначены всем пользователям (политики  условного доступа не должны быть назначены учетным записям службы microsoft Managed Desktop.)    |
|Политики соответствия требованиям устройств     | Проверяет, что политики соответствия intune не назначены всем  пользователям (политики не должны быть назначены любым устройствам Microsoft Managed Desktop.)    |
|Профили конфигурации устройств     | Подтверждает, что профили конфигурации не назначены всем пользователям или  всем устройствам (профили конфигурации не должны быть назначены любым устройствам Microsoft Managed Desktop.)     |
|Ограничения типа устройства     | Проверка того, что устройствам Windows 10 в организации разрешена регистрация в Intune        |
|Страница Состояние регистрации     | Подтверждает, что страница состояние регистрации не включена      |
|Регистрация intune     | Проверяет, что устройства Windows 10 в вашей организации Azure AD автоматически регистрируются в Intune         |
|Microsoft Store для бизнеса     | Подтверждает, что Microsoft Store для бизнеса включена и синхронизирована с Intune        |
|Многофакторная проверка подлинности | Проверяет, что многофакторная проверка подлинности не применяется к учетным записям службы microsoft Managed Desktop.
|Сценарии PowerShell     | Проверки Windows PowerShell сценариев не  назначены таким образом, чтобы они были нацелены на управляемые настольные устройства Майкрософт    |
|Region     | Проверяет, поддерживает ли ваш регион управляемый рабочий стол Майкрософт        |
|Базовые показатели безопасности     | Проверяет, что базовый профиль безопасности не ориентирован на всех пользователей  или все устройства (базовые политики безопасности не должны быть нацелены на любые устройства Microsoft Managed Desktop.)       |
|Приложения Для Windows     | Просмотрите, какие приложения необходимо назначить устройствам Microsoft Managed Desktop      |
|Windows Hello для бизнеса     | Проверка включения Windows Hello для бизнеса        |
|Кольцо обновления Windows 10     | Проверяет, что политика Intune "Кольцо обновления Windows 10" не ориентирована на  всех пользователей или все устройства (политика не должна быть ориентирована на любые управляемые настольные устройства Microsoft.)     |


## <a name="azure-active-directory-settings"></a>Параметры Azure Active Directory

|Чек  |Описание  |
|---------|---------|
|Подписки на "Ad hoc" для корпоративного государственного роуминга     | Советы по проверке параметра, который (если установлено на "false") может запретить правильно работать в корпоративном государственном роуминге  |
|Enterprise State Roaming     | Советы о том, как проверить, включен ли корпоративный государственный роуминг       |
|Лицензии     | Проверка получения необходимых [лицензий](prerequisites.md#more-about-licenses)         |
|Многофакторная проверка подлинности     | Проверяет, что многофакторная проверка подлинности не применяется для всех пользователей (многофакторная проверка подлинности не должна случайно применяться к учетным записям службы microsoft Managed Desktop.)|
|Имена учетных записей безопасности   | Проверка того, что имена пользователей не конфликтуются с именами, которые Microsoft Managed Desktop резервировать для собственного использования        |
|Роли администратора безопасности     | Подтверждает, что пользователям с ролями "Считыватель безопасности", "Оператор безопасности" или "Глобальный читатель" назначены эти роли в Microsoft Defender for Endpoint.         |
|Параметры безопасности по умолчанию | Проверяет, включена ли в вашей организации Azure AD по умолчанию безопасность в Azure Active Directory |
|Самостоятельный сброс пароля     | Подтверждает, что перезагрузка пароля самообслуживляется        |
|Стандартная роль пользователя     | Проверяет, что пользователи являются стандартными пользователями и не имеют прав локального администратора         |


## <a name="microsoft-365-apps-for-enterprise-settings"></a>Приложения Microsoft 365 для корпоративных параметров

|Чек  |Описание  |
|---------|---------|
|OneDrive для бизнеса     | Проверяет, использует ли OneDrive для бизнеса неподтверченные параметры.        |


Для каждой проверки средство будет сообщать об одном из четырех возможных результатов:


|Результат  |Смысл  |
|---------|---------|
|Ready     | До завершения регистрации не требуется никаких действий.        |
|Консультационные услуги    | Следуйте шагам в инструменте для наилучшего опыта с регистрацией и для пользователей. Вы *можете* завершить регистрацию, но перед развертыванием первого устройства необходимо устранить эти проблемы.        |
|Не готов | *Если вы* не исправите эти проблемы, регистрация не состоится. Выполните действия в инструменте для их устранения.        |
|Error | Используемая вами роль Active Director (AD) Azure не имеет достаточного разрешения для запуска этой проверки. |

После завершения регистрации в Microsoft Managed Desktop не забудьте вернуться и настроить определенные параметры Intune и Azure AD. Подробные сведения см. в [материале Adjust settings after enrollment.](../get-started/conditional-access.md)
