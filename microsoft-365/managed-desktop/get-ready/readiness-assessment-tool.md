# <a name="readiness-assessment-tool"></a>Средство оценки готовности

Для максимально плавной регистрации на компьютере, управляемом Майкрософт, существуют важные параметры и другие параметры, которые необходимо установить заранее. С помощью этого средства можно проверить эти параметры и получить подробные инструкции по исправлению неправильного.

Средство проверяет параметры в Microsoft Endpoint Manager (в частности, Microsoft Intune), Azure Active Directory (Azure AD) и Microsoft 365, чтобы убедиться, что они будут работать с компьютерами, управляемыми Майкрософт. На компьютере, управляемом Майкрософт, данные, связанные с этими проверками, сохраняются в течение 12 месяцев после последнего запуска проверки в организации (клиенте) Azure AD. По идентифицированной форме мы сохраняем ее через 12 месяцев.  Вы можете удалить собираемые данные.

Любой, у кого есть хотя бы роль администратора Intune, сможет запустить это средство, но две проверки[(политики](readiness-assessment-fix.md#conditional-access-policies) условного доступа и многофакторная проверка подлинности требуют дополнительных разрешений. [](readiness-assessment-fix.md#multifactor-authentication)
 
Средство оценки проверяет эти элементы:

## <a name="microsoft-intune-settings"></a>Параметры Microsoft Intune

|Чек  |Описание  |
|---------|---------|
|Профиль развертывания Autopilot     | Проверяет, применимо ли назначение профиля развертывания Autopilot к всем  устройствам (профиль не должен быть назначен на любые устройства, управляемые Майкрософт).)       |
|Соединители сертификатов     | Проверяет состояние соединители сертификатов, чтобы убедиться, что они активны   |
|условный доступ;     | Проверяет, назначены ли политики условного доступа не всем пользователям (политики  условного доступа не должны быть назначены учетным записям службы управляемых компьютеров Майкрософт).)    |
|Политики соответствия устройств требованиям     | Проверяет, назначены ли политики соответствия intune не всем  пользователям (политики не должны быть назначены ни для каких устройств, управляемых Майкрософт на настольных компьютерах).)    |
|Профили конфигурации устройств     | Подтверждает, что профили конфигурации не назначены всем пользователям или  всем устройствам (профили конфигурации не должны быть назначены ни для каких устройств, управляемых Майкрософт).)     |
|Ограничения типа устройства     | Проверяет, разрешено ли устройствам с Windows 10 в организации зарегистрироваться в Intune        |
|Страница состояния регистрации     | Подтверждает, что страница состояния регистрации не включена      |
|Регистрация в Intune     | Проверяет, зарегистрированы ли устройства с Windows 10 в вашей организации Azure AD автоматически в Intune         |
|Microsoft Store для бизнеса     | Подтверждает, что Microsoft Store для бизнеса включен и синхронизирован с Intune        |
|Многофакторная проверка подлинности | Проверяет, не применяется ли многофакторная проверка подлинности к учетным записям службы microsoft Managed Desktop.
|Сценарии PowerShell     | Проверяет, Windows PowerShell сценарии не  назначены для настольных устройств, управляемых Майкрософт    |
|Region     | Проверяет, поддерживается ли ваш регион на компьютерах, управляемых Майкрософт        |
|Базовые параметры безопасности     | Проверяет, не нацелен ли базовый профиль безопасности на всех пользователей  или все устройства (базовые политики безопасности не должны быть нацелены на любые настольные устройства, управляемые Майкрософт.)       |
|Приложения для Windows     | Просмотр приложений, которые вы хотите назначить устройствам, управляемым Майкрософт      |
|Windows Hello для бизнеса     | Проверка включения Windows Hello для бизнеса        |
|Круг обновления Windows 10     | Проверка того, что политика Intune "Круг обновления Windows 10" не ориентирована  на всех пользователей или все устройства (политика не должна быть нацелена на любые настольные устройства, управляемые Майкрософт).)     |


## <a name="azure-active-directory-settings"></a>Параметры Azure Active Directory

|Чек  |Описание  |
|---------|---------|
|"Ad hoc" subscriptions for Enterprise State Roaming     | Рекомендации по проверке параметра, который (если установлено в "false") может препятствовать правильной работе роуминга состояния предприятия  |
|Enterprise State Roaming     | Рекомендации по проверке включения enterprise State Roaming       |
|Лицензии     | Проверка получения необходимых [лицензий](prerequisites.md#more-about-licenses)         |
|Многофакторная проверка подлинности     | Проверяет, применяется ли многофакторная проверка подлинности не для всех пользователей (многофакторная проверка подлинности не должна случайно применяться к учетным записям службы управляемых компьютеров Майкрософт.)|
|Имена учетных записей безопасности   | Проверка того, что имена пользователей не конфликтуют с именами пользователей, которые microsoft Managed Desktop резервировать для собственного использования        |
|Роли администраторов безопасности     | Подтверждает, что пользователям с ролями "Читатель безопасности", "Оператор безопасности" или "Глобальный читатель" назначены эти роли в Microsoft Defender для конечной точки         |
|Параметры безопасности по умолчанию | Проверяет, включены ли в вашей организации Azure AD значения по умолчанию для обеспечения безопасности в Azure Active Directory |
|Самостоятельный сброс пароля     | Подтверждение включения самостоятельного сброса пароля        |
|Стандартная роль пользователя     | Проверяет, являются ли пользователи стандартными пользователями и не имеют прав локального администратора         |


## <a name="microsoft-365-apps-for-enterprise-settings"></a>Параметры приложений Microsoft 365 для предприятий

|Чек  |Описание  |
|---------|---------|
|OneDrive для бизнеса     | Проверяет, использует ли OneDrive для бизнеса неподтверченные параметры.        |


Для каждой проверки средство сообщает один из четырех возможных результатов:


|Результат  |Смысл  |
|---------|---------|
|Ready     | Перед завершением регистрации никаких действий не требуется.        |
|Консультации    | Следуйте шагам в средстве для наилучшего опыта регистрации и для пользователей. Вы *можете* завершить регистрацию, но перед развертыванием первого устройства необходимо устранить эти проблемы.        |
|Не готово | *Если не устранить* эти проблемы, регистрация не удастся. Выполните действия в средстве, чтобы устранить их.        |
|Error | Используемая роль Azure Active Director (AD) не имеет достаточных разрешений для запуска этой проверки. |

## <a name="after-enrollment"></a>После регистрации

Завершив регистрацию на компьютере, управляемом Майкрософт, не забудьте вернуться и настроить определенные параметры Intune и Azure AD. Подробные сведения [см. в подстройки параметров после регистрации.](../get-started/conditional-access.md)
