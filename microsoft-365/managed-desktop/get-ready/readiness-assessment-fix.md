# <a name="fix-issues-found-by-the-readiness-assessment-tool"></a>Исправление проблем, обнаруженных средством проверки готовности

Для каждой проверки средство сообщает один из четырех возможных результатов:


|Результат  |Смысл  |
|---------|---------|
|Ready     | Перед регистрацией никаких действий не требуется.        |
|Консультации    | Следуйте шагам в средстве или в этой статье, чтобы наилучшим образом работать с регистрацией и для пользователей. Вы *можете* завершить регистрацию, но перед развертыванием первого устройства необходимо устранить эти проблемы.        |
|Не готово | *Если не устранить эти проблемы, регистрация не удаться.* Выполните действия, которые необходимо предпринять в средстве или в этой статье, чтобы устранить их.        |
|Error | Используемая роль Azure Active Director (AD) не имеет достаточных разрешений для запуска этой проверки. |

> [!NOTE]
> Результаты, полученные с помощью этого средства, отражают состояние параметров только на определенный момент времени его работы. Если позднее вы внести какие-либо изменения в политики в Microsoft Intune, Azure Active Directory или Microsoft 365, элементы, которые были готовы, могут стать "Не готовы". Чтобы избежать проблем с операциями microsoft Managed Desktop, проверьте параметры, описанные в этой статье, прежде чем изменять политики.

## <a name="microsoft-intune-settings"></a>Параметры Microsoft Intune

Доступ к настройкам Intune можно получить в Центре администрирования Microsoft Endpoint [Manager.](https://endpoint.microsoft.com)

### <a name="autopilot-deployment-profile"></a>Профиль развертывания Autopilot

У вас не должно быть существующих профилей Autopilot, которые нацелены на соответствующие или динамические группы, используемые компьютерами, управляемыми Майкрософт. Компьютеры, управляемые Майкрософт, используют Autopilot для предоставления новых устройств.

**Не готово**

У вас есть профиль Autopilot, который назначен всем устройствам. Действия см. в [подразделе "Регистрация устройств с Windows в Intune с помощью Windows Autopilot".](https://docs.microsoft.com/mem/autopilot/enrollment-autopilot) После регистрации на компьютере, управляемом Майкрософт, задайте политику Autopilot, чтобы исключить группу **"Современные** рабочие устройства — все устройства Azure AD".

**Консультации**

Убедитесь, что ваши профили Autopilot нацелены на назначенную или динамическую группу Azure AD, которая не включает управляемые Майкрософт настольные устройства, которые будут созданы при регистрации. Действия см. в [подразделе "Регистрация устройств с Windows в Intune с помощью Windows Autopilot".](https://docs.microsoft.com/mem/autopilot/enrollment-autopilot) После регистрации на компьютере, управляемом Майкрософт, задайте политику Autopilot, чтобы исключить группу **"Современные** рабочие устройства — все устройства Azure AD".


### <a name="certificate-connectors"></a>Соединители сертификатов

Если у вас есть соединители сертификатов, используемые устройствами, которые вы хотите зарегистрировать на компьютерах, управляемых Майкрософт, они не могут иметь ошибок. В вашей ситуации будет применяться только одно из следующих консультантов, поэтому внимательно проверьте их.

**Консультации**

Соединители сертификатов не присутствуют. Возможно, вам не нужны соединительные устройства, но следует оценить, нужны ли вам какие-либо соединительные устройства для сетевого подключения к устройствам, управляемым Майкрософт. Дополнительные сведения см. в [подготовьте сертификаты и сетевые профили для компьютеров, управляемых Майкрософт.](certs-wifi-lan.md)

**Консультации**

Ошибка произошла по крайней мере у одного соединитела сертификата. Если вам нужен этот соединительную связь для подключения к настольным устройствам, управляемым Майкрософт, необходимо устранить ошибку. Дополнительные сведения см. в [подготовьте сертификаты и сетевые профили для компьютеров, управляемых Майкрософт.](certs-wifi-lan.md)


**Консультации**

У вас есть по крайней мере один соединители сертификатов, и об ошибках не сообщается. Однако может потребоваться создать профиль для повторного использования соединительная система для настольных устройств, управляемых Майкрософт. Дополнительные сведения см. в [подготовьте сертификаты и сетевые профили для компьютеров, управляемых Майкрософт.](certs-wifi-lan.md)


### <a name="conditional-access-policies"></a>Политики условного доступа

Политики условного доступа в организации Azure AD не должны быть нацелены на пользователей Microsoft Manage Desktop.

**Не готово**

У вас есть по крайней мере одна политика условного доступа, которая ориентирована на всех пользователей. Сброс политики для определенной группы Azure AD, не включаемой в группу Azure AD учетных записей службы управляемых компьютеров Майкрософт, которые будут созданы при регистрации. По шагам см. [условный доступ: пользователи и группы.](https://docs.microsoft.com/azure/active-directory/conditional-access/concept-conditional-access-users-groups)

**Консультации**

Убедитесь, что все политики условного доступа, которые вы исключили, исключают группу Azure AD "Учетные записи **современных** рабочих служб". Действия см. в [подстройки условного доступа.](https://docs.microsoft.com/microsoft-365/managed-desktop/get-started/conditional-access) Группа Azure AD **"Учетные** записи современных рабочих мест" — это динамическая группа, которую мы создаем для службы при регистрации. Вам придется вернуться, чтобы исключить эту группу после регистрации. Дополнительные информацию об этих учетных записях служб см. в [стандартных операционных процедурах.](../service-description/operations-and-monitoring.md#standard-operating-procedures)

**Error**

Роль администратора Intune не имеет достаточных разрешений для этой проверки. Вам также потребуется любая из этих ролей Azure AD, назначенная для запуска этой проверки:

- Читатель сведений о безопасности
- Администратор безопасности
- Администратор условного доступа
- Глобальный читатель
- Администратор устройств

[в подстройке "Создание политики соответствия требованиям" в Microsoft Intune.](https://docs.microsoft.com/mem/intune/protect/create-compliance-policy)

**Консультации**

Убедитесь, что все политики соответствия требованиям не включают пользователей компьютеров, управляемых Майкрософт. Дополнительные действия см. [в подстройке "Создание политики соответствия требованиям" в Microsoft Intune.](https://docs.microsoft.com/mem/intune/protect/create-compliance-policy)

Дополнительные сведения см. в подсети ["Настройка страницы состояния регистрации".](https://docs.microsoft.com/mem/intune/enrollment/windows-enrollment-status)

### <a name="multifactor-authentication"></a>Многофакторная проверка подлинности

Многофакторная проверка подлинности не должна случайно применяться к учетным записям службы microsoft Managed Desktop.


**Не готово**

Некоторые политики многофакторной проверки подлинности (MFA) задаются как "необходимые" для политик условного доступа, которые назначены всем пользователям. Измените политику, чтобы использовать назначение, которое ориентировано на определенную группу Azure AD, которая не включает какие-либо устройства с управляемыми Майкрософт настольными компьютерами. Дополнительные сведения [см.](#conditional-access-policies) в политиках условного доступа и [условном доступе: "Требовать многофайловую многофайловую аудию для всех пользователей".](https://docs.microsoft.com/azure/active-directory/conditional-access/howto-conditional-access-policy-all-users-mfa)

**Консультации**

Убедитесь, что все политики условного доступа, которые требуют MFA, исключают группу **Modern Workplace -All** Azure AD. Дополнительные сведения [см.](#conditional-access-policies) в политиках условного доступа и [условном доступе: "Требовать многофайловую многофайловую аудию для всех пользователей".](https://docs.microsoft.com/azure/active-directory/conditional-access/howto-conditional-access-policy-all-users-mfa) Группа **"Современное** рабочее место — все Azure AD" — это динамическая группа, которую мы создадим при регистрации на компьютере, управляемом Майкрософт, поэтому вам придется вернуться, чтобы исключить эту группу после регистрации.

**Error**

Роль администратора Intune не имеет достаточных разрешений для этой проверки. Вам также потребуется любая из этих ролей Azure AD, назначенная для запуска этой проверки:

- Читатель сведений о безопасности
- Администратор безопасности
- Администратор условного доступа
- Глобальный читатель
- Администратор устройств

**Консультации**

Убедитесь, что все политики кругов обновления, которые у вас есть, исключают группу **"Современные** рабочие устройства — все устройства Azure AD". Если вы написали этим политикам группу пользователей Azure AD, убедитесь, что все политики кругов обновления также исключали группу **Modern Workplace -All** Azure AD, которая включает пользователей компьютеров, управляемых Майкрософт. По шагам [см. управление обновлениями программного обеспечения Windows 10 в Intune.](https://docs.microsoft.com/mem/intune/protect/windows-update-for-business-configure) Как современные рабочие устройства **— все,** так и современные рабочие места — все группы **Azure** AD — это группы, которые мы создаем при регистрации на компьютере, управляемом Майкрософт, поэтому вам придется вернуться, чтобы исключить эту группу после регистрации.


## <a name="azure-active-directory-settings"></a>Параметры Azure Active Directory

Доступ к настройкам Azure Active Directory можно получить на [портале Azure.](https://portal.azure.com)

### <a name="ad-hoc-subscriptions"></a>Ad hoc subscriptions

Рекомендации по проверке параметра, который может препятствовать правильной работе роуминга в корпоративном состоянии (если установлено в "false").

**Консультации**

Убедитесь, **что для allowAdHocSubscriptions** установлено **true.** В противном случае может не работать роуминг состояния предприятия. Дополнительные сведения [см. в подсети Set-MsolCompanySettings.](https://docs.microsoft.com/powershell/module/msonline/set-msolcompanysettings?view=azureadps-1.0)


### <a name="enterprise-state-roaming"></a>Enterprise State Roaming

Должен быть включен роуминг состояния предприятия.

**Консультации**

Убедитесь, что для всех  или для выбранных групп включен роуминг состояния **предприятия.** Дополнительные сведения см. в подсети ["Включить роуминг состояния предприятия в Azure Active Directory".](https://docs.microsoft.com/azure/active-directory/devices/enterprise-state-roaming-enable)

### <a name="licenses"></a>Лицензии

Для использования компьютеров, управляемых Майкрософт, требуется несколько лицензий.

**Не готово**

У вас нет всех лицензий, необходимых для использования компьютеров, управляемых Майкрософт. Дополнительные сведения см. в технологиях [microsoft Managed Desktop и](../intro/technologies.md) [дополнительных сведениях о лицензиях.](prerequisites.md#more-about-licenses)


### <a name="security-account-names"></a>Имена учетных записей безопасности

Некоторые имена учетных записей безопасности могут конфликтовть с именами, созданными на компьютере, управляемом Майкрософт.

**Не готово**

У вас есть хотя бы одно имя учетной записи, конфликтуя с учетной записью, созданной на компьютере, управляемом Майкрософт. Вместе с представителем учетной записи Майкрософт исключите эти имена учетных записей.


### <a name="security-administrator-roles"></a>Роли администраторов безопасности

Пользователи с определенными ролями безопасности должны иметь те, которые назначены в Microsoft Defender для конечной точки.

**Консультации**

Если в вашей организации Azure AD есть пользователи, которые назначены какой-либо из этих ролей, убедитесь, что им также назначены эти роли в Microsoft Defender для конечной точки. В противном случае администраторы с этими ролями не смогут получить доступ к порталу администрирования.

- Оператор безопасности
- Глобальный читатель

Дополнительные сведения см. в сведениях о создании ролей и управлении ими [для управления доступом на основе ролей.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/user-roles)


### <a name="security-default"></a>Безопасность по умолчанию

Значения по умолчанию для системы безопасности в Azure Active Directory не смогут управлять вашими устройствами на компьютерах, управляемых Майкрософт.

**Не готово**

Вы включили настройки безопасности по умолчанию. Отключите значения по умолчанию для системы безопасности и настроите политики условного доступа. Дополнительные сведения см. в общих [политиках условного доступа.](https://docs.microsoft.com/azure/active-directory/conditional-access/concept-conditional-access-policy-common)

### <a name="self-service-password-reset"></a>Самостоятельный сброс пароля

Самостоятельный сброс пароля (SSPR) должен быть включен для всех пользователей компьютеров, управляемых Майкрософт, за исключением учетных записей службы управляемых компьютеров Майкрософт. Дополнительные сведения см. в руководстве. В этом руководстве пользователи могут разблокировать свою учетную запись или сбросить пароли с помощью самостоятельного сброса [пароля Azure Active Directory.](https://docs.microsoft.com/azure/active-directory/authentication/tutorial-enable-sspr)

**Консультации**

Убедитесь, что параметр SSPR **Selected** содержит управляемые Майкрософт настольные устройства, но исключает учетные записи службы управляемых компьютеров Майкрософт. Учетные записи службы управляемых компьютеров Майкрософт не могут работать ожидаемым, если включенА SSPR.  


### <a name="standard-user-role"></a>Стандартная роль пользователя

Кроме пользователей, которым назначены роли глобального администратора и администратора устройств в Azure AD, пользователи компьютеров, управляемых Майкрософт, будут стандартными пользователями без прав локального администратора. Всем другим пользователям будет назначена стандартная роль пользователя при запуске устройства microsoft Managed Desktop.

**Консультации**

Пользователи компьютеров, управляемых Майкрософт, не будут иметь права локального администратора на своих устройствах, управляемых Майкрософт, после регистрации.

## <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

### <a name="onedrive"></a>OneDrive

Разрешение **синхронизации только на компьютерах,** которые присоединились к определенному параметру домена, конфликтует с компьютером, управляемым Майкрософт. Доступ к настройкам OneDrive можно получить в Центре [администрирования OneDrive.](https://admin.onedrive.com)

**Консультации**

Вы используете параметр "Разрешить синхронизацию только на ПК, которые **присоединились к определенному параметру** домена". Этот параметр не будет работать с компьютером, управляемым Майкрософт. Отключать этот параметр и настраивать OneDrive для использования политики условного доступа. См. [справку по планированию развертывания условного](https://docs.microsoft.com/azure/active-directory/conditional-access/plan-conditional-access) доступа.
