# <a name="fix-issues-found-by-the-readiness-assessment-tool"></a>Устранение проблем, обнаруженных средством оценки готовности

Для каждой проверки средство сообщит один из четырех возможных результатов:


|Результат  |Смысл  |
|---------|---------|
|Ready     | Перед регистрацией не требуются никакие действия.        |
|Рекомендация    | Выполните действия, описанные в этом инструменте, или в этой статье, чтобы получить лучшие сведения о возможностях регистрации и для пользователей. Вы *можете* выполнить регистрацию, но вам необходимо устранить эти проблемы перед развертыванием первого устройства.        |
|Не готов | *Если вы не устраните эти проблемы, регистрация завершится с ошибками.* Выполните действия, описанные в этом инструменте, или в этой статье, чтобы устранить их.        |
|Error | Используемая роль Azure Active Directory (AD) не обладает разрешениями, достаточными для запуска этой проверки. |

## <a name="microsoft-intune-settings"></a>Параметры Microsoft Intune

### <a name="autopilot-deployment-profile"></a>Профиль развертывания для автопилота

Вы не должны иметь существующие профили автопилота, которые нацелены на назначенные или динамические группы, используемые Microsoft Managed Desktop. На рабочем столе Майкрософт используется Автопилот для подготовки новых устройств.

**Не готов**

У вас есть профиль автопилота, назначаемый всем устройствам. Для получения инструкций обратитесь к разделу [Регистрация устройств Windows в Intune с помощью Windows автопилота](https://docs.microsoft.com/mem/autopilot/enrollment-autopilot). После регистрации на рабочем столе, управляемой Майкрософт, настройте политику автопилота, чтобы исключить **все современные устройства — все** группы Azure AD.

**Рекомендация**

Убедитесь, что профили автопилота нацелены на назначенную или динамическую группу Azure AD, которая не включает устройства, управляемые Майкрософт для настольных компьютеров, которые будут созданы при регистрации. Для получения инструкций обратитесь к разделу [Регистрация устройств Windows в Intune с помощью Windows автопилота](https://docs.microsoft.com/mem/autopilot/enrollment-autopilot). После регистрации на рабочем столе, управляемой Майкрософт, настройте политику автопилота, чтобы исключить **все современные устройства — все** группы Azure AD.


### <a name="certificate-connectors"></a>Соединители сертификатов

При наличии соединителей сертификатов, используемых устройствами, которые вы хотите зарегистрировать на компьютере, управляемом Майкрософт, соединители не могут содержать ошибок. В вашу ситуацию будет применен только один из следующих рекомендаций, поэтому следует тщательно проверять их.

**Рекомендация**

Нет соединителей сертификатов. Возможно, вам не потребуются соединители, но необходимо оценить, требуются ли для сетевого подключения к управляемым устройствам Майкрософт для настольных ПК. Дополнительные сведения можно найти в разделе [Prepare Certificate and Network Profiles for Microsoft Managed Desktop](certs-wifi-lan.md).

**Рекомендация**

По крайней мере один соединитель сертификатов содержит ошибку. Если этот соединитель необходим для подключения к настольным устройствам, управляемым корпорацией Майкрософт, необходимо устранить эту ошибку. Дополнительные сведения можно найти в разделе [Prepare Certificate and Network Profiles for Microsoft Managed Desktop](certs-wifi-lan.md).


**Рекомендация**

У вас есть по крайней мере один соединитель сертификатов, и при этом не сообщается об ошибках. Тем не менее, может потребоваться создать профиль для повторного использования соединителя для настольных устройств, управляемых Майкрософт. Дополнительные сведения можно найти в разделе [Prepare Certificate and Network Profiles for Microsoft Managed Desktop](certs-wifi-lan.md).


### <a name="conditional-access-policies"></a>Политики условного доступа

Политики условного доступа в Организации Azure AD не должны быть нацелены на управление пользователями настольных систем Майкрософт.

**Не готов**

У вас есть по крайней мере одна политика условного доступа, предназначенная для всех пользователей. Сбросьте политику, чтобы она была ориентирована на определенную группу Azure AD, которая не включает группу Azure AD управляемых учетных записей служб рабочего стола Майкрософт, которые будут создаваться при регистрации. Для получения инструкций обратитесь [к разделу условный доступ: пользователи и группы](https://docs.microsoft.com/azure/active-directory/conditional-access/concept-conditional-access-users-groups).

**Рекомендация**

Убедитесь, что все политики условного доступа исключены из группы Azure AD " **учетные записи служб современного рабочего места** ". Инструкции по [настройке условного доступа](https://docs.microsoft.com/microsoft-365/managed-desktop/get-started/conditional-access). Группа Azure AD " **учетные записи служб современного рабочего места** " — это динамическая группа, которая создается для службы при регистрации. После регистрации вам потребуется вернуться к исключению этой группы. Для получения дополнительных сведений об этих учетных записях служб ознакомьтесь со статьями [стандартные рабочие процедуры](../service-description/operations-and-monitoring.md#standard-operating-procedures).

**Error**

Роль администратора Intune не имеет достаточных разрешений для этой проверки. Кроме того, для выполнения этой проверки потребуются все назначенные роли Azure AD.

- Читатель сведений о безопасности
- Администратор безопасности
- Администратор условного доступа
- Глобальный читатель
- Администратор устройств

Показывает, как проверить, что параметр, для которого задано значение "false"), может препятствовать правильной работе перемещения корпоративных состояний.

**Рекомендация**

Убедитесь, что для параметра **алловадхоксубскриптионс** задано **значение true**. В противном случае роуминг состояния предприятия может не работать. Дополнительные сведения см. в статье [Set – мсолкомпанисеттингс](https://docs.microsoft.com/powershell/module/msonline/set-msolcompanysettings?view=azureadps-1.0).


### <a name="enterprise-state-roaming"></a>Enterprise State Roaming

Роуминг состояния предприятия должен быть включен.

**Рекомендация**

Убедитесь, что роуминг состояния предприятия включен для **всех** или для **выбранных** групп. Дополнительные сведения см в статье [Enable Enterprise State роуминг в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/devices/enterprise-state-roaming-enable).

### <a name="licenses"></a>Лицензии

Для использования настольного компьютера, управляемого Майкрософт, требуется несколько лицензий.

**Не готов**

У вас нет лицензий, необходимых для использования компьютера, управляемого корпорацией Майкрософт. Дополнительные сведения можно найти в [статье управляемые технологии для настольных ПК Майкрософт](../intro/technologies.md) и [Дополнительные сведения о лицензиях](prerequisites.md#more-about-licenses).


### <a name="security-account-names"></a>Имена учетных записей безопасности

Некоторые имена учетных записей безопасности могут конфликтовать с настольными компьютерами, созданными корпорацией Майкрософт.

**Не готов**

У вас есть по крайней мере одно имя учетной записи, которое будет конфликтовать с рабочим столом, созданным корпорацией Майкрософт. Работайте с представителем учетной записи Майкрософт, чтобы исключить эти имена учетных записей.


### <a name="security-administrator-roles"></a>Роли администраторов безопасности

Пользователям с определенными ролями безопасности должны быть назначены в защитнике Майкрософт для конечной точки.

**Рекомендация**

Если в вашей организации Azure AD назначена любая из этих ролей, убедитесь, что они также назначены в защитнике Майкрософт для конечной точки. В противном случае администраторы с этими ролями не смогут получить доступ к порталу администрирования.

- Читатель сведений о безопасности
- Оператор безопасности
- Глобальный читатель

Дополнительные сведения см. в статье [Создание ролей и управление ими для управления доступом на основе ролей](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/user-roles).


### <a name="security-default"></a>По умолчанию для безопасности

Параметры безопасности по умолчанию в Azure Active Directory не допускают управления устройствами с помощью Microsoft Managed Desktop.

**Не готов**

Параметры безопасности по умолчанию включены. Отключите параметры безопасности по умолчанию и настройте политики условного доступа. Более подробную информацию можно узнать в статье [Общие политики условного доступа](https://docs.microsoft.com/azure/active-directory/conditional-access/concept-conditional-access-policy-common).

### <a name="self-service-password-reset"></a>Самостоятельный сброс пароля

Самостоятельный сброс паролей (SSPR) должен быть включен для всех пользователей, исключая учетные записи службы управляемых рабочих столов Майкрософт. Дополнительные сведения см. [в разделе Tutorial: разрешить пользователям разблокировать свою учетную запись или сбросить пароли с помощью функции самообслуживания Azure Active Directory для самостоятельного сброса пароля](https://docs.microsoft.com/azure/active-directory/authentication/tutorial-enable-sspr).

**Рекомендация**

Убедитесь, что **выбранный** параметр SSPR включает в себя системные устройства, управляемые корпорацией Майкрософт, но не включает учетные записи служб, управляемые корпорацией Майкрософт. Управляемые учетные записи служб рабочего стола Майкрософт не работают должным образом, если включен SSPR.  


### <a name="standard-user-role"></a>Роль обычного пользователя

Пользователи настольных компьютеров, которым назначены роли Azure AD для глобальных администраторов и администраторов устройств, будут иметь стандартные пользователи без прав локального администратора. Всем остальным пользователям будет назначена роль стандартного пользователя при запуске своего устройства, управляемого Майкрософт.

**Рекомендация**

Пользователи настольных систем, управляемых Майкрософт, не будут иметь права локального администратора на своих устройствах, управляемых Майкрософт, после регистрации.

## <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

### <a name="onedrive-for-business"></a>OneDrive для бизнеса

Параметр **Разрешить синхронизацию только на компьютерах, присоединенных к определенным доменам** , будет конфликтовать с управляемым рабочим столом Майкрософт.

**Рекомендация**

Вы используете параметр **Разрешить синхронизацию только на компьютерах, присоединенных к определенным доменам** . Этот параметр не будет работать с настольным компьютером, управляемым Майкрософт. Отключите этот параметр, а затем настройте OneDrive для бизнеса для использования политики условного доступа. Ознакомьтесь с разкрывающимся [планом развертывания условного доступа](https://docs.microsoft.com/azure/active-directory/conditional-access/plan-conditional-access) для получения справки.
