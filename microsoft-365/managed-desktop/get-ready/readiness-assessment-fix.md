# <a name="fix-issues-found-by-the-readiness-assessment-tool"></a>Исправление проблем, обнаруженных средством проверки готовности

Для каждой проверки средство будет сообщать об одном из четырех возможных результатов:


|Результат  |Смысл  |
|---------|---------|
|Ready     | До завершения регистрации не требуется никаких действий.        |
|Предупреждение    | Следуйте шагам в инструменте или этой статье для наилучшего опыта с регистрацией и для пользователей. Вы *можете* завершить регистрацию, но перед развертыванием первого устройства необходимо устранить эти проблемы.        |
|Не готов | *Если вы не исправите эти проблемы, регистрация не состоится.* Выполните действия в инструменте или этой статье для их устранения.        |
|Error | Используемая вами роль Azure Active Directory (AD) не имеет достаточного разрешения для запуска этой проверки. |

> [!NOTE]
> Результаты, полученные с помощью этого средства, отражают состояние параметров только в определенный момент времени, когда вы его запустили. Если позднее внести изменения в политики в Microsoft Intune, Azure Active Directory или Microsoft 365, элементы, которые были "Готовы", могут стать "Не готовы". Чтобы избежать проблем с операциями Microsoft Managed Desktop, ознакомьтесь с определенными настройками, описанными в этой статье, прежде чем менять какие-либо политики.

### <a name="autopilot-deployment-profile"></a>Профиль развертывания автопилота

Не следует иметь существующие профили автопилота, которые должны быть нацелены на задаваемую или динамическую группу с помощью устройств Microsoft Managed Desktop. Microsoft Managed Desktop использует автопилот для предоставления новых устройств.

**Не готов**

У вас есть профиль Автопилота, который назначен всем устройствам. Действия см. в [записи устройств Windows в Intune с помощью Автопилота Windows.](/mem/autopilot/enrollment-autopilot) После регистрации microsoft Managed Desktop установите политику автопилота, чтобы исключить группу Современных устройств на рабочем месте **- All** Azure AD.

**Предупреждение**

Убедитесь, что профили автопилота ориентированы на назначенную или динамичную группу Azure AD, которая не включает устройства Microsoft Managed Desktop. Действия см. в [записи устройств Windows в Intune с помощью Автопилота Windows.](/mem/autopilot/enrollment-autopilot) После регистрации microsoft Managed Desktop установите профили автопилота, чтобы исключить группу Современные устройства на рабочем месте **-All** Azure AD.


### <a name="certificate-connectors"></a>Соединители сертификатов

Если у вас есть соединители сертификата, которые будут использоваться устройствами, которые вы хотите зарегистрироваться в Microsoft Managed Desktop, соединители не должны иметь никаких ошибок. Только один из следующих консультантов будет применяться к вашей ситуации, поэтому проверьте их тщательно.

**Предупреждение**

Соединители сертификатов не присутствуют. Возможно, вам не нужны соединительные устройства, но следует оценить, может ли вам понадобиться подключение к сети на устройствах Microsoft Managed Desktop. Дополнительные сведения см. в [странице Подготовка сертификатов и сетевых профилей для microsoft Managed Desktop.](certs-wifi-lan.md)

**Предупреждение**

Ошибка имеется по крайней мере в одном соединители сертификата. Если вам нужен этот соединительная система для предоставления сертификатов устройствам Microsoft Managed Desktop, необходимо устранить ошибку. Дополнительные сведения см. в [странице Подготовка сертификатов и сетевых профилей для microsoft Managed Desktop.](certs-wifi-lan.md)


**Предупреждение**

У вас есть по крайней мере один соединитатель сертификата, и об ошибках не сообщается. Однако при подготовке к развертыванию может потребоваться создать профиль для повторного использования соединительная система для устройств Microsoft Managed Desktop. Дополнительные сведения см. в [странице Подготовка сертификатов и сетевых профилей для microsoft Managed Desktop.](certs-wifi-lan.md)


### <a name="conditional-access-policies"></a>Политики условного доступа

Политики условного доступа не должны препятствовать управлению организацией Azure AD (клиентом) в Intune и Azure AD.

**Не готов**

У вас есть по крайней мере одна политика условного доступа, которая ориентирована на всех пользователей. Во время регистрации мы исключаем учетные записи службы управляемых настольных компьютеров Майкрософт из соответствующих политик условного доступа и применяем новые политики условного доступа, чтобы ограничить доступ к этим учетным записям. После регистрации вы можете просмотреть политику условного доступа microsoft Managed Desktop в Microsoft Endpoint Manager. Дополнительные данные об этих учетных записях служб см. в [стандартных операционных процедурах.](../service-description/operations-and-monitoring.md#standard-operating-procedures)

**Предупреждение**

У вас есть политики условного доступа, которые могут запретить Microsoft Managed Desktop управлять службой управляемых настольных компьютеров Майкрософт. Во время регистрации мы исключаем учетные записи службы управляемых настольных компьютеров Майкрософт из соответствующих политик условного доступа и применяем новые политики условного доступа, чтобы ограничить доступ к этим учетным записям. Дополнительные данные об этих учетных записях служб см. в [стандартных операционных процедурах.](../service-description/operations-and-monitoring.md#standard-operating-procedures)

**Ошибка**

Роль администратора Intune не имеет достаточных разрешений для этой проверки. Чтобы выполнить эту проверку, вам также понадобится любая из этих ролей Azure AD.

- Читатель сведений о безопасности
- Администратор безопасности
- Администратор условного доступа
- Глобальный читатель
- Администратор устройств

Измените политику, чтобы она была ориентирована на конкретную группу Azure AD, которая не включает пользователей или устройств Microsoft Managed Desktop. Дополнительные действия см. [в сайте Create a compliance policy in Microsoft Intune.](/mem/intune/protect/create-compliance-policy)

**Предупреждение**

Убедитесь, что любые политики соответствия требованиям не ориентированы на пользователей microsoft Managed Desktop. Дополнительные действия см. [в сайте Create a compliance policy in Microsoft Intune.](/mem/intune/protect/create-compliance-policy)

**Предупреждение**

### <a name="intune-enrollment"></a>Регистрация intune

Устройства Windows 10 в вашей организации Azure AD должны иметь возможность автоматически зарегистрироваться в Intune.

**Предупреждение**

Убедитесь, **что область пользователя MDM задарена** как **"Некоторые"** или **"Все",** а не **"Нет".** Если вы выбираете **Некоторые,** возвращайся после регистрации и выберите  группу **Modern Workplace -All** Azure AD для групп или эквивалентную группу, ориентированную на всех пользователей Microsoft Managed Desktop.  Настройка регистрации для устройств Windows с [помощью Microsoft Intune.](/mem/intune/enrollment/windows-enroll#enable-windows-10-automatic-enrollment)


### <a name="ad-hoc-subscriptions"></a>Ad hoc subscriptions

Советы по проверке параметра, который (если установлено на "false") может запретить правильно работать в корпоративном государственном роуминге.

**Предупреждение**

Убедитесь, **что allowAdHocSubscriptions** задайте **true**. В противном случае роуминг состояния предприятия может не работать. Дополнительные сведения см. [в рублях Set-MsolCompanySettings.](/powershell/module/msonline/set-msolcompanysettings?view=azureadps-1.0)


### <a name="enterprise-state-roaming"></a>Enterprise State Roaming

Должен быть включен корпоративный государственный роуминг.

**Предупреждение**

Убедитесь, что роуминг состояния предприятия включен для **всех** или для **выбранных** групп. Дополнительные сведения см. в [видеоролике Enable Enterprise State Roaming in Azure Active Directory.](/azure/active-directory/devices/enterprise-state-roaming-enable)

### <a name="licenses"></a>Лицензии

Для использования Microsoft Managed Desktop требуется несколько лицензий.

**Не готово**

У вас нет всех лицензий, необходимых для использования Microsoft Managed Desktop. Дополнительные сведения см. [в дополнительных сведениях о](../intro/technologies.md) технологиях Microsoft Managed Desktop и [дополнительных сведениях о лицензиях.](prerequisites.md#more-about-licenses)


### <a name="microsoft-managed-desktop-service-accounts"></a>Учетные записи служб microsoft Managed Desktop

Некоторые имена учетных записей могут противоречить именам учетных записей, созданным Microsoft Managed Desktop для управления службой microsoft Managed Desktop.

**Не готов**

У вас есть по крайней мере одно имя учетной записи, которое будет противоречить именам учетных записей, созданным Microsoft Managed Desktop. Работайте с представителем учетной записи Майкрософт, чтобы исключить эти имена учетных записей. Мы не перечисляем имена учетных записей публично, чтобы свести к минимуму риск безопасности. 


### <a name="security-administrator-roles"></a>Роли администратора безопасности

Пользователям с определенными ролями безопасности должны быть назначены роли в Microsoft Defender для конечной точки.

**Предупреждение**

Если вам назначены какие-либо из этих ролей в вашей организации Azure AD, убедитесь, что им также назначены эти роли в Microsoft Defender для конечной точки. В противном случае администраторы с этими ролями не смогут получить доступ к порталу Admin.

- Оператор безопасности
- Глобальный читатель

Дополнительные сведения см. в дополнительных сведениях о создании и [управлении ролями для](/windows/security/threat-protection/microsoft-defender-atp/user-roles)управления доступом на основе ролей.


### <a name="security-default"></a>По умолчанию безопасности

По умолчанию безопасность в Azure Active Directory не позволит Microsoft Managed Desktop управлять устройствами.

**Не готов**

Включены по умолчанию системы безопасности. Отключите по умолчанию безопасность и установите политики условного доступа. Дополнительные сведения см. [в общих политиках условного доступа.](/azure/active-directory/conditional-access/concept-conditional-access-policy-common)

### <a name="self-service-password-reset"></a>Сброс пароля самообслуживаемой

Сбросить пароль самообслуживной службы (SSPR) можно для всех пользователей microsoft Managed Desktop, за исключением учетных записей службы microsoft Managed Desktop. Дополнительные сведения см. в руководстве. Включить пользователям возможность разблокировать учетную запись или сбросить пароли с помощью сброса пароля самообслуживщика [Azure Active Directory.](/azure/active-directory/authentication/tutorial-enable-sspr)

**Предупреждение**

Убедитесь, что параметр SSPR **Selected** включает пользователей Microsoft Managed Desktop, но исключает учетные записи службы microsoft Managed Desktop. Учетные записи службы microsoft Managed Desktop не могут работать так, как ожидалось при включенной службе SSPR.  


### <a name="standard-user-role"></a>Стандартная роль пользователя

Кроме тех пользователей, которым назначены роли Azure AD глобального администратора и администратора устройств, пользователи Microsoft Managed Desktop будут стандартными пользователями без привилегий локального администратора. Всем другим пользователям будет назначена стандартная роль пользователя при запуске устройства Microsoft Managed Desktop.

**Предупреждение**

Пользователи управляемых настольных компьютеров Майкрософт не будут иметь локальные привилегии администратора на своих управляемых настольных устройствах Майкрософт после регистрации.

### <a name="onedrive"></a>OneDrive

Возможность **синхронизации только на компьютерах, присоединившись** к определенным доменам, будет противоречить microsoft Managed Desktop. Вы можете получить доступ к настройкам OneDrive в центре [администрирования](https://admin.onedrive.com)OneDrive.

**Предупреждение**

Вы используете функцию Разрешить синхронизацию только на **ПК, присоединившись к определенным доменам.** Этот параметр не будет работать с microsoft Managed Desktop. Отключить этот параметр и вместо этого настроить OneDrive для использования политики условного доступа. Подробнее [о развертывании условного доступа см.](/azure/active-directory/conditional-access/plan-conditional-access) в справке.
