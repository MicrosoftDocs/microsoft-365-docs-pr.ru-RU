# <a name="privacy-and-personal-data"></a>Конфиденциальность и личные данные

Пользователи могут получать, передавать и хранить данные на устройствах, управляемых Microsoft Managed Desktop. Они доверяют, что конфиденциальность данных защищена и используется только таким образом, чтобы соответствовать их ожиданиям. В этой статье рассказывается о том, как Microsoft Managed Desktop собирает, сохраняет, сохраняет, обрабатывает, обеспечивает, разделяет, аудитирует и экспортирует персональные данные. Вы также узнаете, как администратор может просматривать, исправлять и удалять персональные данные.

Microsoft Managed Desktop не использует личные данные, собранные в рамках предоставления службы для профилирования, рекламы или маркетинга.

## <a name="data-collection-of-microsoft-managed-desktop"></a>Коллекция данных управляемого рабочего стола Майкрософт

При регистрации корпоративных устройств в Microsoft Managed Desktop сбор данных обрабатывается на техническом уровне с помощью Windows и Microsoft Intune. Эти источники собирают персональные данные о устройствах пользователей, например имена устройств для Microsoft Managed Desktop, чтобы иметь возможность идентифицировать устройство, которое будет управляться и предоставляться с помощью microsoft Managed Desktop.

Microsoft Managed Desktop не собирает данные самостоятельно для предоставления своей службы (за исключением контактных данных [ИТ-администратора.](#it-admin-contact-information) Вместо этого Microsoft Managed Desktop повторно передает данные, собранные другими источниками, такими как Windows и Microsoft Intune. Microsoft Managed Desktop использует данные, которые эти службы собирают с зарегистрированных устройств:

- Диагностические данные Windows с устройств, управляемых Microsoft Managed Desktop, отправляются в хранилища диагностических данных Для Windows.
- Microsoft Managed Desktop использует [современное управление](/learn/modules/introduction-to-modern-management-in-microsoft-365/) для управления зарегистрированными устройствами. В рамках "современного управления" устройства должны быть зарегистрированы в Azure Active Directory клиента.
- Для распространения высоко оптимизированной и безопасной конфигурации на зарегистрированных устройствах Microsoft Managed Desktop использует Microsoft Intune.
- Microsoft Managed Desktop использует данные разведки безопасности от Microsoft Defender Advanced Thread Protection для тех клиентов, которые используют эту службу.

## <a name="data-storage-and-sources-in-microsoft-managed-desktop"></a>Хранение данных и источники в Microsoft Managed Desktop

После того как Microsoft Managed Desktop получает данные, она должна предоставить свою службу, хранение и обработку этих данных следующим образом:

### <a name="storing-data-storage-location-and-data-retention"></a>Хранение данных, расположение хранилища и хранение данных

Microsoft Managed Desktop хранит свои данные в одной или более из следующих служб хранения Майкрософт:

- Azure SQL
- Хранилище Azure
- Dynamics 365

Microsoft Managed Desktop хранит свои данные в США. Персональные данные сохраняются microsoft Managed Desktop не более 30 дней, за исключением данных оповещений для устройств Microsoft Managed Desktop, собранных Microsoft Defender для конечной точки. Фактические данные оповещений (которые могут включать персональные данные) хранятся в течение 180 дней. Оповещение об удалении персональных данных хранится до двух лет. В соответствии с Общим регламентом защиты данных (GDPR) и Законом о конфиденциальности потребителей в Калифорнии (CCPA) Microsoft Managed Desktop чтим права субъекта данных для любых персональных данных, хранимых в данных оповещения.

Команды microsoft Managed Desktop Operations and Security Operations расположены в США и Индии.

## <a name="data-subject-requests-for-the-gdpr-and-ccpa"></a>Запросы субъектов данных, определенные в GDPR и CCPA

Правило общей защиты данных Европейского союза [(GDPR)](https://ec.europa.eu/justice/data-protection/reform/index_en.htm) предоставляет людям (известным в регламенте как субъекты данных) права на управление персональными данными, собранными работодателем или другим типом агентства или организации (известного как контроллер данных или просто контроллер). Персональные данные в GDPR в очень широком смысле определяются как любые данные, имеющие какое-либо отношение к определенному или определяемому физическому лицу. GDPR предоставляет субъектам данных определенные права на их персональные данные, в том числе следующие: получение копий персональных данных, запрос на внесение в них исправлений, ограничение возможностей их обработки, удаление или получение данных в электронном виде для передачи другому управляющему. Официальный запрос от субъекта данных к управляющему данными, требующий выполнения некоторых действий с персональными данными этого субъекта, называется  Запросом субъектов данных или DSR.

Кроме того, CCPA предоставляет права и обязательства по конфиденциальности для потребителей Калифорнии, включая права, аналогичные правам субъекта данных GDPR, таким как право на удаление, доступ и получение (переносимость) их персональных данных. В CCPA также предусмотрены определенные требования к раскрытию информации, защите от дискриминации при избрании прав на осуществление, а также требования к "отказу от передачи данных, классифицируются как "продажи". Широкое определение продаж включает обмен данными для встречного удовлетворения. Дополнительные сведения о CCPA см. в статьях [Закон Калифорнии о конфиденциальности данных](/compliance/regulatory/offering-ccpa?view=o365-worldwide) и [Вопросы и ответы о законе Калифорнии о конфиденциальности данных](/compliance/regulatory/ccpa-faq?view=o365-worldwide).

## <a name="microsoft-defender-for-endpoint-alert-data"></a>Microsoft Defender для данных оповещений конечной точки

Администраторы безопасности могут запрашивать извлечение или удаление персональных данных, связанных с оповещениями Microsoft Defender для конечных точек на управляемом устройстве Microsoft Managed Desktop в своей среде. Администратор безопасности должен войти на портал [](https://aka.ms/memadmin) администрирования управляемых настольных компьютеров Майкрософт и отправить запрос на поддержку. Выберите **тип** запроса на  поддержку запроса  на **изменение,** категорию безопасности и подкатегорию **Других,** а затем указайте соответствующие имена устройств в описании вместе с запросом на извлечение или удаление данных.

### <a name="user-related-personal-data"></a>Персональные данные, связанные с пользователем

Кроме того, Microsoft Managed Desktop не собирает персональные данные самостоятельно. Вместо этого он использует персональные данные, собранные другими службами Microsoft Enterprise Online. ИТ-администраторы, которые должны отвечать на запросы пользователей, чтобы просматривать, исправлять и удалять свои персональные данные, могут использовать соответствующие функциональные возможности служб, от которых зависит Microsoft Managed Desktop. Если вы заинтересованы в просмотре или удалении персональных данных, используемых этими службами, сначала см. в статье [GDPR](/compliance/regulatory/gdpr-dsr-Azure) "Запросы субъекта данных Azure".
