# <a name="privacy-and-personal-data"></a>Конфиденциальность и персональные данные

Пользователи могут получать, передавать и хранить данные на устройствах, управляемых Microsoft Desktop. Они доверяют тому, что конфиденциальность данных защищена и используется только в соответствии с их ожиданиями. В этой статье объясняется, как компьютеры, управляемые Майкрософт, собирают, сохраняют, сохраняют, процессы, безопасность, обетовыв, аудит и экспортируют персональные данные. Вы также узнаете, как администратор может просматривать, исправлять и удалять персональные данные.

Компьютеры, управляемые Майкрософт, не используют персональные данные, собранные в рамках предоставления службы для профилирования, рекламы или маркетинга.

## <a name="data-collection-of-microsoft-managed-desktop"></a>Сбор данных компьютеров, управляемых Майкрософт

Когда пользователи регистрют корпоративные устройства на компьютере, управляемом Майкрософт, сбор данных (на техническом уровне) обрабатывается с помощью Windows и Microsoft Intune. Эти источники собирают персональные данные об устройствах пользователей, такие как имена устройств для компьютеров, управляемых Майкрософт, чтобы иметь возможность идентифицировать устройство, которым необходимо управлять, и предоставляться с помощью средств microsoft Managed Desktop.

Компьютеры, управляемые Майкрософт, не собирают данные самостоятельно для предоставления службы (за исключением контактных данных [ИТ-администратора).](#it-admin-contact-information) Вместо этого на компьютере, управляемом Майкрософт, повторное использование данных, уже собранных другими источниками, такими как Windows и Microsoft Intune. Компьютеры, управляемые Майкрософт, используют данные, которые эти службы собирают с зарегистрированных устройств:

- Диагностические данные Windows с устройств, управляемых Microsoft Managed Desktop, отправляются в хранилища диагностических данных Windows корпорации Майкрософт.
- Компьютеры, управляемые [Майкрософт, используют современное](https://docs.microsoft.com/learn/modules/introduction-to-modern-management-in-microsoft-365/) управление для управления зарегистрированными устройствами. В рамках "современного управления" устройства должны быть зарегистрированы в Azure Active Directory клиента.
- Для распространения своей конфигурации с высокой оптимизацией и безопасностью на зарегистрированных устройствах на компьютере, управляемом Майкрософт, используется Microsoft Intune.
- Компьютеры, управляемые Майкрософт, используют данные аналитики безопасности из Advanced Thread Protection в Microsoft Defender для тех клиентов, которые используют эту службу.

## <a name="data-subject-requests-for-the-gdpr-and-ccpa"></a>Запросы субъектов данных, определенные в GDPR и CCPA

Общий регламент по защите данных Европейского Союза [(GDPR)](https://ec.europa.eu/justice/data-protection/reform/index_en.htm) предоставляет людям (известным в регламенте как субъекты данных) права на управление персональными данными, собранными нанимателем или другим типом агентства или организации (известного как контролер данных или просто контролер). Персональные данные в GDPR в очень широком смысле определяются как любые данные, имеющие какое-либо отношение к определенному или определяемому физическому лицу. GDPR предоставляет субъектам данных определенные права на их персональные данные, в том числе следующие: получение копий персональных данных, запрос на внесение в них исправлений, ограничение возможностей их обработки, удаление или получение данных в электронном виде для передачи другому управляющему. Официальный запрос от субъекта данных к управляющему данными, требующий выполнения некоторых действий с персональными данными этого субъекта, называется  Запросом субъектов данных или DSR.

Аналогичным образом Закон Калифорнии о конфиденциальности потребителей (CCPA) предоставляет права и обязательства в области конфиденциальности для потребителей в Калифорнии, включая права, аналогичные правам субъектов данных GDPR, такие как право на удаление, доступ и получение (переносимость) их персональных данных. В CCPA также предусмотрены определенные требования к раскрытию, защите от нарушения прав на осуществление, а также требования "отказаться или отказаться" для определенных передач данных, классифицированных как "продажи". Широкое определение продаж включает обмен данными для встречного удовлетворения. Дополнительные сведения о CCPA см. в статьях [Закон Калифорнии о конфиденциальности данных](https://docs.microsoft.com/microsoft-365/compliance/offering-ccpa?view=o365-worldwide) и [Вопросы и ответы о законе Калифорнии о конфиденциальности данных](https://docs.microsoft.com/microsoft-365/compliance/ccpa-faq?view=o365-worldwide).
