## <a name="change-management"></a>Управление изменениями

В случае служб баланс ответственности за такие задачи, как обслуживание оборудования и установка обновлений для системы безопасности, смещается от клиента (вас) к поставщику службы (Майкрософт). Тем не менее, необходимо убедиться, что стороннее и пользовательское программное обеспечение продолжают работать должным образом при развертывании обновлений.

Для локальных продуктов ваша организация принимает ответственность за управление изменениями.

### <a name="change-types"></a>Типы изменений

Не все изменения одинаково влияют на пользователей, и не все предусматривают действие. Некоторые из них запланированы, а некоторые незапланированные (обновления для системы безопасности и обновления для системы безопасности обычно не планируются). ## <a name="device-wipe-with-factory-reset"></a>Очистка устройства с помощью перезапуска фабрики

Команда "управляемые операции на настольных компьютерах Майкрософт" может выполнить фабричный сброс устройств, зарегистрированных в службе, когда это необходимо. Это полезно, если вам нужно назначить устройство другому сотруднику или если сотрудник покидает свою компанию. 

Существует несколько требований:

- Глобальный администратор должен отправить запрос на обслуживание.
- Включение имени компьютера устройства в запрос.
- Перед сбросом устройства учетная запись пользователя должна быть в Azure AD.

Управляемые операции с настольными компьютерами выполняет следующие действия:

- Поиск имени устройства в Intune
- Отправка команды заводского сброса на устройство

>[!NOTE]
>Не удаляйте учетную запись пользователя из Azure AD перед сбросом устройства. Если пользователь не входит в Azure AD, Intune не может отправить на устройство команду фабричной перегрузки. 

Устройство будет загружаться с "неработающий", а все предустановленные приложения и параметры будут применены снова. Пользователю устройства необходимо снова предоставить начальную информацию о настройке. 

Когда устройство будет сброшено, вы можете назначить его другому пользователю в Организации. На устройстве не будут ни данные предыдущего пользователя, ни корпоративные данные. Следующий пользователь пройдет тот же процесс, что и предыдущий пользователь, с новым устройством, управляемым Майкрософт.

BitLocker — ключевой компонент безопасности данных в этом процессе. При шифровании BitLocker на настольных устройствах, управляемых Майкрософт, данные на диске остаются безопасными даже после завершения заводского перезапуска устройства. Все данные, которые находились на диске, будут недоступны следующему пользователю устройства. Более подробную информацию можно узнать в статье [Обзор BitLocker](https://docs.microsoft.com/windows/security/information-protection/bitlocker/bitlocker-overview).

Дополнительные [сведения см.](https://docs.microsoft.com/intune/remote-actions/devices-wipe#factory-reset-a-device)
