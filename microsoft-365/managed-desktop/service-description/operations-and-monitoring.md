## <a name="change-management"></a>Управление изменениями

В случае служб баланс ответственности за такие задачи, как обслуживание оборудования и установка обновлений для системы безопасности, смещается от клиента (вас) к поставщику службы (Майкрософт). Тем не менее, вам по-прежнему необходимо убедиться, что не microsoft и настраиваемые программы продолжают функционировать, как ожидалось, когда обновления будут свернуты.

Для локальной продукции организация берет на себя всю ответственность за управление изменениями.

### <a name="change-types"></a>Типы изменений

Не все изменения одинаково влияют на пользователей, и не все предусматривают действие. Некоторые из них запланированы, а некоторые незапланированы по своему характеру (обновления и обновления безопасности, как правило, не планируется). ## <a name="device-wipe-with-factory-reset"></a>Стирка устройства с сбросом заводской

Группа компьютеры, управляемые Майкрософт операций может при необходимости выполнить заводской сброс устройств, зарегистрированных в службе. Сброс полезно, если необходимо предоставить устройство другому сотруднику или если сотрудник покинет вашу компанию.

Существует несколько требований:

- Глобальный администратор должен отправить запрос на обслуживание.
- Включай в запрос имя компьютера устройства.
- Учетная запись пользователя должна быть в Azure AD перед сбросом устройства.

Команда управляемых операций на рабочем столе будет делать следующее:

- Посмотрите имя устройства в Intune
- Отправка команды сброса фабрики на устройство

> [!NOTE]
> Не удалять учетную запись пользователя из Azure AD до сброса устройства. Если пользователя нет в Azure AD, Intune не может отправить команду сброса фабрики на устройство.

Устройство будет загрузиться в "вне окна", и все предустановленные приложения и параметры будут применены снова. Пользователю устройства необходимо снова предоставить начальные сведения о настройке. 

После сброса устройства вы можете предоставить его другому лицу в организации. Ни один из данных предыдущего пользователя или корпоративных данных не будет на устройстве. Следующий пользователь будет проходить тот же процесс, что и предыдущий пользователь с новым компьютеры, управляемые Майкрософт устройством.

BitLocker является ключевым компонентом безопасности данных в этом процессе. С помощью шифрования BitLocker на компьютеры, управляемые Майкрософт устройствах данные на диске остаются безопасными даже после сброса устройства. Все данные, которые были на диске, не будут доступны следующему пользователю устройства. Дополнительные сведения см. в [обзоре BitLocker.](/windows/security/information-protection/bitlocker/bitlocker-overview)

Дополнительные сведения см. в [перезагрузах устройства Factory.](/intune/remote-actions/devices-wipe#factory-reset-a-device)
