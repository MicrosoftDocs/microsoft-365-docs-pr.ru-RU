## <a name="change-management"></a>Управление изменениями

В случае служб баланс ответственности за такие задачи, как обслуживание оборудования и установка обновлений для системы безопасности, смещается от клиента (вас) к поставщику службы (Майкрософт). Однако вам по-прежнему необходимо убедиться, что программное обеспечение, не внося в корпорацию Майкрософт и пользовательское программное обеспечение, продолжает работать должным образом при обновлении.

Организация берет на себя всю ответственность за управление изменениями в локальной организации.

### <a name="change-types"></a>Типы изменений

Существует несколько типов изменений, которые мы регулярно внося в службу. Канал связи для этих изменений и действия, за которые вы несете ответственность, зависит от того, за какие действия вы несете ответственность.

Не все изменения одинаково влияют на пользователей, и не все предусматривают действие. Некоторые из них планируются, а другие незапланированы по своей природе (обновления, не внося в безопасность и обновления для системы безопасности, обычно не планируются). ## <a name="device-wipe-with-factory-reset"></a>Стирка устройства с сбросом заводских насайт

При необходимости группа microsoft Managed Desktop Operations может выполнить сброс заводских устройств, зарегистрированных в службе. Сброс полезен, если требуется предоставить устройство другому сотруднику или сотрудник уедет из вашей компании. 

Существует несколько требований:

- Глобальный администратор должен отправить запрос на обслуживание.
- Включаем имя компьютера устройства в запрос.
- Учетная запись пользователя должна быть в Azure AD перед сбросом устройства.

Группа операций с управляемым рабочим столом будет делать следующее:

- Искать имя устройства в Intune
- Отправка команды сброса заводских насайтов на устройство

>[!NOTE]
>Не удалять учетную запись пользователя из Azure AD до сброса устройства. Если пользователь не работает в Azure AD, Intune не сможет отправить на устройство команду сброса заводских насытов. 

Устройство будет загрузиться в "стандартное", и все предустановленные приложения и параметры будут применены снова. Пользователю устройства необходимо снова предоставить начальные сведения об установке. 

После сброса устройства его можно предоставить другому человеку в организации. Никакие данные предыдущего пользователя или корпоративные данные не будут на устройстве. Следующий пользователь будет проходить тот же процесс, что и предыдущий пользователь с новым настольным устройством, управляемым Майкрософт.

BitLocker — это ключевой компонент безопасности данных в этом процессе. С помощью шифрования BitLocker на настольных устройствах, управляемых Майкрософт, данные на диске остаются безопасными даже после сброса заводских насайлов на устройстве. Любые данные, которые были на диске, будут недоступны следующему пользователю устройства. Дополнительные сведения см. в [обзоре BitLocker.](https://docs.microsoft.com/windows/security/information-protection/bitlocker/bitlocker-overview)

Дополнительные сведения [см. в подстановлении устройства заводом.](https://docs.microsoft.com/intune/remote-actions/devices-wipe#factory-reset-a-device)
