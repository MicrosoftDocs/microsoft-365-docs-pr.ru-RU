# <a name="first-run-experience-with-autopilot-and-the-enrollment-status-page"></a>Первый запуск с помощью Autopilot и страницы состояния регистрации

На компьютере, управляемом Майкрософт, используется страница состояния регистрации [Windows Autopilot](https://docs.microsoft.com/windows/deployment/windows-autopilot/windows-autopilot) и страница состояния регистрации Microsoft Intune [(ESP),](https://docs.microsoft.com/windows/deployment/windows-autopilot/enrollment-status) чтобы обеспечить пользователям максимально возможные возможности первого запуска.

Страница состояния регистрации в настоящее время находится в открытой предварительной версии.

## <a name="initial-deployment"></a>Начальное развертывание

Чтобы обеспечить поддержку ESP, необходимо зарегистрировать устройства в службе компьютеров, управляемых Майкрософт. Подробнее о регистрации см. в под вопросе "Регистрация [новых устройств](../get-started/register-devices-self.md) самостоятельно" или "Действия [для партнеров по регистрации устройств".](../get-started/register-devices-partner.md)

После регистрации устройств в службе вы можете включить ESP для настольных устройств, управляемых Майкрософт, подав заявку в службу поддержки на [портале администрирования.](https://portal.azure.com/) Сначала мы развернем конфигурацию ESP в тестовой группе при подаче заявки. Он развертывается в других последующих группах развертывания (First, Fast и Broad) каждые 24 часа. Чтобы приостановить развертывание, подайте еще один запрос на удержание operations.

## <a name="autopilot-profile-settings"></a>Параметры профиля Autopilot

Компьютеры, управляемые Майкрософт, используют эти параметры в профиле Autopilot, используемом для устройств пользователей:


|Setting  |Значение  |
|---------|---------|
|Режим развертывания |  Управляемый пользователем       |
|Присоединитесь к Azure AD как     |  Присоединилось к Azure AD       |
|Язык (регион)     | Операционная система по умолчанию        |
|Автоматическое настройка клавиатуры     | Нет        |
|Условия лицензионного соглашения на программное обеспечение корпорации Майкрософт     |  Скрыть       |
|Параметры конфиденциальности     | Скрыть        |
|Скрытие параметров учетной записи изменения     | Показать        |
|Тип учетной записи пользователя     |  Стандартный       |
|Разрешить OOBE для белых перчаток     |  Да       |
|Применение шаблона имени устройства     | Да        |
|Введите имя     | MMD-%RAND:11%        |

> [!NOTE]
> Подготовка "белых перьев" включена только для клиентов с включенным ESP, но в настоящее время она не поддерживается на компьютерах, управляемых Майкрософт.

## <a name="enrollment-status-page-settings"></a>Параметры страницы состояния регистрации

Компьютеры, управляемые Майкрософт, используют эти параметры для страницы состояния регистрации:


|Setting  |Значение  |
|---------|---------|
|Показать ход настройки приложения и профиля     | Да        |
|Показывать ошибку, когда установка занимает больше указанного количества минут     |  60       |
|Показывать настраиваемые сообщения, когда возникает ошибка ограничения по времени     |  Да       |
|Сообщение об ошибке     | Да, настройка устройства проходит немного дольше, чем ожидалось. Щелкните ниже, чтобы начать работу, и мы завершим настройку в фоновом режиме        |
|Разрешить пользователям собирать журналы об ошибках установки     |  Да       |
|Показывать только страницу на устройствах, которые были собъендуем при приоюзовых устройствах (OOBE)     | Да        |
|Блокировать использование устройств до тех пор, пока не будут установлены все приложения и профили     |  Да       |
|Разрешить пользователям сбрасывать устройство при ошибке установки     |  Да       |
|Разрешить пользователям использовать устройство в случае ошибки установки     |  Да       |
|Блокировать использование устройств до тех пор, пока эти необходимые приложения не будут установлены, если они назначены пользователю или устройству     |  Современное рабочее место — исправление времени       |



Страница состояния регистрации происходит в три этапа. Дополнительные сведения [см. в сведениях об отслеживании страницы состояния регистрации.](https://docs.microsoft.com/mem/intune/enrollment/windows-enrollment-status#enrollment-status-page-tracking-information)

Этот опыт продолжается следующим образом:

1. Запускается интерфейс Autopilot, и пользователь вводит свои учетные данные.
2. Устройство открывает страницу состояния регистрации и переходит на этапы подготовки устройства и установки устройства. Третий этап (настройка учетной записи) в настоящее время пропускается в конфигурации компьютера, управляемого Майкрософт, так как пользователь ESP отключен.  Устройство перезагружается.
3. После перезагрузки устройство открывает страницу для входов в Windows с другим **пользователем.**
4. Пользователи снова введите свои учетные данные и откроется рабочий стол.

> [!NOTE]
> Приложения Win32 развертываются во время ESP, только если версия Windows 10 — 1903 или более поздней.

## <a name="white-glove-provisioning"></a>Подготовка белых перо

Microsoft Managed Desktop в настоящее время не поддерживает функцию "белых перо" Windows Autopilot.

## <a name="change-to-autopilot-and-enrollment-status-page-settings"></a>Изменение параметров страницы состояния Autopilot и регистрации

Если настройка, используемая компьютером под управлением Майкрософт, не соответствует вашим потребностям, вы можете подать запрос в службу поддержки на [портале администрирования.](https://portal.azure.com/) Вот несколько примеров типов конфигурации, которые могут потребоваться.

### <a name="autopilot-settings-change"></a>Изменение параметров Autopilot

Может потребоваться запросить другой шаблон имени устройства. Однако нельзя изменить режим развертывания, присоединиться к Azure AD As, параметрам конфиденциальности или типу учетной записи пользователя.

### <a name="enrollment-status-page-settings-change"></a>Изменение параметров страницы состояния регистрации

- Больше минут для параметра "Показать ошибку, когда установка занимает больше указанного количества минут".
- Отобразилось сообщение об ошибке
- Добавление или удаление приложений в параметре "Блокировать использование устройства до тех пор, пока эти необходимые приложения не будут установлены, если они назначены пользователю или устройству".

## <a name="required-applications"></a>Необходимые приложения

- Необходимо нацелить приложения в группах устройств "Современное рабочее *место"* "Тестирование", "Первый", "Быстрый" и "Широкий". Приложения должны устанавливаться в контексте "Система". Обязательно выполните тестирование ESP в группе "Тестирование", прежде чем назначать их всем группам.
- Никакие приложения не должны требовать перезапуска устройства. При сборке пакета приложения рекомендуется установить для приложений "Ничего не делать", если для них требуется перезагрузка.
- Ограничив необходимые приложения только основными приложениями, которые пользователь должен сразу же получить при входе на устройство.
- Общий размер всех приложений в совокупности должен быть меньше 1 ГБ, чтобы избежать периодов времени в течение этапа установки приложения.
- В идеале приложения не должны иметь зависимостей. Если у вас есть *приложения,* которые должны иметь зависимости, убедитесь, что вы настроите, протестировали и проверьте их в рамках оценки ESP.
- В общедоступный предварительный просмотр ESP не могут быть включены приложения, для работы с которые требуется контекст пользователя (например, Teams).
