# <a name="first-run-experience-with-autopilot-and-the-enrollment-status-page"></a>Первый запуск с помощью Autopilot и страницы состояния регистрации

компьютеры, управляемые Майкрософт использует как [Windows](/windows/deployment/windows-autopilot/windows-autopilot) автопилот, так и страницу состояния Microsoft Intune регистрации [(ESP),](/windows/deployment/windows-autopilot/enrollment-status) чтобы предоставить пользователям максимально возможный первый запуск.

Страница состояние регистрации в настоящее время находится в режиме предварительного просмотра.

## <a name="initial-deployment"></a>Начальное развертывание

Чтобы обеспечить службу ESP, необходимо зарегистрировать устройства в компьютеры, управляемые Майкрософт службе. Дополнительные информацию о регистрации см. в перерегистрации новых [устройств самостоятельно](../get-started/register-devices-self.md) или Действия [для партнеров для регистрации устройств.](../get-started/register-devices-partner.md)

После регистрации устройств в службе можно включить ESP для компьютеры, управляемые Майкрософт устройств, подав билет на поддержку через [портал администрирования.](https://portal.azure.com/) Сначала мы развернем конфигурацию ESP в группе Test при подаче билета. Он развертывается в других последующих группах развертывания (First, Fast и Broad) каждые 24 часа. Чтобы приостановить развертывание, задайте другой билет с просьбой о проведении операций.

компьютеры, управляемые Майкрософт использует эти параметры для работы страницы состояния регистрации:

<br>

****

|Параметр|Значение|

|Показать прогресс конфигурации приложений и профилей|Да|

|Разрешить пользователям собирать журналы об ошибках установки|Да|

Опыт страницы состояния регистрации происходит в три этапа. Дополнительные сведения [см. в странице отслеживания состояния регистрации.](/mem/intune/enrollment/windows-enrollment-status#enrollment-status-page-tracking-information)

Этот опыт будет выполняться следующим образом:

1. Начинается опыт автопилота, и пользователь вводит свои учетные данные.
2. Устройство открывает страницу состояния регистрации и переходит на этапы подготовки и установки устройств. Третий шаг (установка учетной записи) в настоящее время пропускается в конфигурации компьютеры, управляемые Майкрософт, так как пользовательский ESP отключен.  Устройство перезапуска.
3. После перезапуска устройство открывает страницу Windows регистрации с другим **пользователем.**
4. Пользователи снова введите учетные данные и откроется рабочий стол.

> [!NOTE]
> Приложения Win32 развертываются только во время ESP, если Windows 10 версии 1903 или более поздней версии.

## <a name="autopilot-for-pre-provisioned-deployment"></a>Автопилот для предварительного развертывания

> [!NOTE]
> Автопилот для предварительного развертывания в компьютеры, управляемые Майкрософт в настоящее время находится в режиме предварительного просмотра.

## <a name="additional-prerequisites-for-autopilot-for-pre-provisioned-deployment"></a>Дополнительные предпосылки для автопилота для предварительного развертывания

- Необходимо включить страницу состояния регистрации (ESP). Дополнительные сведения см. в [начальном развертывании.](#initial-deployment)
- Устройство должно иметь проводное сетевое подключение.
- Если у вас есть устройства, зарегистрированные на портале компьютеры, управляемые Майкрософт до августа 2020 г., отрегистрируйте и зарегистрируйте их снова.
- Устройства должны иметь фабричных изображений, которые включает в себя накопительное обновление [19H1/19H2 2020.11C](https://support.microsoft.com/topic/november-19-2020-kb4586819-os-builds-18362-1237-and-18363-1237-preview-25cbb849-74af-b8b8-29b8-68aa925e8cc3) или [20H1 2020.11C,](https://support.microsoft.com/topic/november-30-2020-kb4586853-os-builds-19041-662-and-19042-662-preview-8fb07fb8-a7dd-ea62-d65e-3305da09f92e) как это необходимо установить или должны быть повторно с последним компьютеры, управляемые Майкрософт изображения.
- Физические устройства должны поддерживать проверку TPM 2.0 и устройства. Виртуальные машины не поддерживаются. В процессе подготовки используется Windows автопилота, поэтому требуется TPM 2.0. Процесс проверки TPM также требует доступа к набору URL-адресов HTTPS, которые являются уникальными для каждого поставщика TPM. Дополнительные сведения см. в записи для режима самостоятельного развертывания автопилота и предварительного развертывания автопилота в Windows сетей [автопилота.](/mem/autopilot/networking-requirements#tpm)

## <a name="sequence-of-events-in-autopilot-for-pre-provisioned-deployment"></a>Последовательность событий в автопилоте для предварительного развертывания

1. ИТ-администратор повторно или сбросит устройство при необходимости.
2. ИТ-администратор сапогирует устройство, выходит на выход из окна и пять раз Windows клавишу.
3. ИТ-администратор выбирает Windows автопилот подготовка, а затем выбирает **Продолжить**. На экране Windows автопилота будут отображаться сведения об устройстве.
4. ИТ-администратор **выбирает Положение** для запуска процесса подготовка.
5. Устройство запускает ESP и проходит этапы подготовки и установки устройства. На этапе установки устройства отображается установка **приложения x x** (в зависимости от точной конфигурации профиля ESP).
6. Шаг настройки учетной записи в настоящее время пропущен в конфигурации компьютеры, управляемые Майкрософт, так как мы отключаем пользовательский ESP.
7. Устройство перезапуска.

После перезапуска устройство будет показывать зеленый экран состояния с кнопкой **Reseal.**

> [!IMPORTANT]
> Известные проблемы
>
> - ESP не запустится после автопилота для предварительной функции повторного развертывания.
> - Устройство не переименовываются автопилотом для предварительного развертывания. Устройство будет переименовано только после того, как будет проходить пользовательский поток ESP.

## <a name="change-to-autopilot-and-enrollment-status-page-settings"></a>Изменение параметров состояния страницы автопилота и регистрации

Если установка, используемая компьютеры, управляемые Майкрософт, не соответствует вашим потребностям, вы можете подать билет на поддержку через [портал администрирования.](https://portal.azure.com/) Вот несколько примеров типов конфигурации, которые могут потребоваться:

### <a name="autopilot-settings-change"></a>Изменение параметров автопилота

Может потребоваться запросить другой шаблон имени устройства. Однако изменить режим развертывания, присоединиться к Azure AD As, privacy Параметры или типу учетной записи пользователя нельзя.

### <a name="enrollment-status-page-settings-change"></a>Изменение параметров состояния страницы регистрации

- Большее количество минут для параметра "Показать ошибку, когда установка занимает больше времени, чем указанное количество минут".
- Отображается сообщение об ошибке
- Добавление или удаление приложений в параметре "Блокировка использования устройств до установки этих необходимых приложений, если они назначены пользователю или устройству".

## <a name="required-applications"></a>Необходимые приложения

- Необходимо нацелить приложения в группах устройств Modern *Workplace* Test, First, Fast и Broad. Приложения должны устанавливаться в контексте "Система". Убедитесь, что перед назначением их всем группам необходимо завершить тестирование с помощью ESP в группе Test.
- Никакие приложения не должны требовать перезапуска устройства. При создании пакета приложений рекомендуется задать приложениям "Ничего не делать", если для них потребуется перезапуск.
- Ограничить необходимые приложения только основными приложениями, которые нужны пользователю сразу при входе на устройство.
- Сохраняй общий размер всех приложений в совокупности до 1 ГБ, чтобы избежать периодов времени во время этапа установки приложения.
- В идеале приложения не должны иметь зависимостей. Если у вас есть *приложения,* которые должны иметь зависимости, убедитесь, что вы настраивали, тестировали и проверяли их в рамках оценки ESP.
- Приложения, которые требуют "пользовательского" контекста (например, Teams), не могут быть включены в общедоступный предварительный просмотр ESP.
