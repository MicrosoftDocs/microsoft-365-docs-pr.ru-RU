# <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

## <a name="initial-deployment"></a>Начальное развертывание

компьютеры, управляемые Майкрософт обеспечивает установку Приложения Microsoft 365 для предприятий (64-bit) в составе изображения на всех [устройствах программы.](../service-description/device-list.md) Все следующие приложения должны присутствовать на устройстве при доставке:

- Word
- Excel
- PowerPoint
- Outlook
- Publisher
- Access
- Skype для бизнеса
- OneNote

Этот подход минимизирует влияние сети и гарантирует, что пользователи смогут работать продуктивно сразу после получения устройства. Затем мы развертываем дополнительные политики на управляемых устройствах, чтобы настроить приложения для использования.

> [!NOTE]
> Microsoft Teams развертывается отдельно от Приложения Microsoft 365 для предприятий и не входит в базовое изображение. 

### <a name="available-deployment-to-users"></a>Доступное развертывание для пользователей

Если у пользователя нет Приложения Microsoft 365 по какой-либо причине, вы можете использовать пакет, чтобы вернуть устройство в ожидаемое состояние. Добавьте пользователя в группу **Modern Workplace-Office-Office365_Install** и приложения станут доступны для них в Корпоративный портал.

### <a name="microsoft-365-apps-for-enterprise-32-bit"></a>Приложения Microsoft 365 для предприятий (32-битный)

компьютеры, управляемые Майкрософт не поддерживает развертывание 32-битной версии M365 Apps для предприятия.

## <a name="updates-to-microsoft-365-apps"></a>Обновления для Приложения Microsoft 365

Приложения Microsoft 365 для обновления на канале [Monthly Enterprise.](/deployoffice/overview-update-channels#monthly-enterprise-channel-overview) Эта практика предоставляет пользователям новые Office каждый месяц, но они будут получать только одно обновление в месяц в предсказуемом графике выпуска. Обновления выпускаются во второй вторник месяца; эти обновления могут включать обновления функций, безопасности и качества. Эти обновления происходят автоматически и извлекаются непосредственно из Office CDN для этого конкретного канала.

компьютеры, управляемые Майкрософт каждый выпуск определяет потенциальные проблемы в среде. Мы завершаем выкатку через 28 дней после выпуска из группы Microsoft 365 приложений. компьютеры, управляемые Майкрософт выпуски обновлений для разных групп, чтобы дать время для проверки и тестирования следующим образом: 

- Тест: ноль дней
- Первый: ноль дней
- Быстрый: 3 дня
- Broad: 7 дней

компьютеры, управляемые Майкрософт устанавливает семидневный срок [обновления](/deployoffice/configure-update-settings-microsoft-365-apps) для устройств. После того как обновление доступно, оно должно быть установлено в течение семи дней. Пользователи получают [уведомления](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) о том, что обновления требуются в нескольких расположениях: приложении в подносе системы за 12 часов до крайнего срока, и они получают предупреждение за 15 минут до крайнего срока. Все Приложения Microsoft 365 должны быть закрыты для завершения обновления.

### <a name="pausing-or-rolling-back-an-update"></a>Прио пауза или откат обновления

Если по какой-либо причине необходимо приостановить или откат [](../working-with-managed-desktop/admin-support.md) Microsoft 365 обновления приложения, задай запрос на поддержку администратора на компьютеры, управляемые Майкрософт портале.

Во время выпуска компьютеры, управляемые Майкрософт отслеживает показатели ошибок всех Приложения Microsoft 365. Если мы заметим существенную разницу в качестве между новым выпуском и его предшественником, мы можем связаться с вами через портал администрирования компьютеры, управляемые Майкрософт администратора. В зависимости от серьезности, мы будем спрашивать, хотите ли вы приостановить выпуск или сообщить вам, что мы приняли меры для смягчения проблемы. 

### <a name="delivery-optimization"></a>Оптимизация доставки

Оптимизация доставки — это одноранговая технология распространения, доступная в Windows 10. Это позволяет устройствам обмениваться контентом, например обновлениями, которые устройства скачали из Microsoft через Интернет. С его помощью можно уменьшить пропускную способность сети, так как устройство может получать части обновления с другого устройства в локальной сети вместо того, чтобы полностью скачать обновление из Microsoft.

[Оптимизация доставки](/deployoffice/delivery-optimization) включена по умолчанию на устройствах с Windows 10 Корпоративная или Windows 10 для образовательных учреждений выпусками. 

## <a name="settings-managed-by-microsoft-managed-desktop"></a>Параметры управляется компьютеры, управляемые Майкрософт

Корпорация Майкрософт управляет некоторыми настройками в рамках службы. компьютеры, управляемые Майкрософт не управляет базовым Office безопасности, но вы можете установить его самостоятельно, следуя указаниям в разделе Параметры [управления.](#settings-you-manage)

### <a name="update-settings"></a>Обновление параметров

компьютеры, управляемые Майкрософт поддерживает все параметры [обновления](/deployoffice/configure-update-settings-microsoft-365-apps) для управляемых устройств, и эти параметры следует изменить.

### <a name="set-updates-to-occur-automatically"></a>Настроить автоматические обновления

**Значение по умолчанию:** включено

Эта политика настроена для обеспечения того, чтобы все Office устройства могли быть в курсе облачных технологий. 

### <a name="set-a-deadline-when-updates-have-to-be-applied"></a>Установите крайний срок, когда необходимо применять обновления

**Значение по умолчанию:** 7 дней

Политика **UpdateDeadline** используется для настройки льготного периода, который пользователи имеют до принудительного обновления на устройстве. Эта политика крайних сроков также вызывает [уведомления](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) для пользователя, чтобы сообщить им об изменениях, необходимых на их устройстве.  

### <a name="defer-updates-on-a-device-for-a-period"></a>Отсрочка обновлений на устройстве на период

Эта политика настраивается по-разному для каждой группы устройств управления обновлениями и требуется для компьютеры, управляемые Майкрософт для удовлетворения целей обновления:  

- Тест: ноль дней
- Первый: ноль дней
- Быстрый 7 дней
- Broad: 21 дней

### <a name="update-notifications-settings"></a>Обновление параметров уведомлений

**Значение по умолчанию:** False

Параметр "скрыть уведомления об обновлении" задается **параметру False** на компьютеры, управляемые Майкрософт устройствах, чтобы предоставить пользователям наилучшие сведения об обновлении, уведомив их о необходимости обновления. [](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps)

### <a name="specify-a-location-to-look-for-updates"></a>Указать расположение для поиска обновлений

**Значение по умолчанию:** ежемесячный Enterprise канал

Сочетание политик **UpdatePath** и **UpdateChannel** используется по мере необходимости для достижения расписания обновлений. Эти политики должны гарантировать, что все Office устройства получают обновления непосредственно из CDN для ежемесячного Enterprise Channel.

### <a name="specify-the-target-version-of-microsoft-365-apps"></a>Укажите целевую версию Приложения Microsoft 365

Политика target Version иногда используется компьютеры, управляемые Майкрософт для отката или закрепления определенной версии Office. 


### <a name="hide-the-option-to-enable-or-disable-office-automatic-updates"></a>Скрыть параметр, чтобы включить или отключить Office автоматические обновления

**Значение по умолчанию:** включено

Этот параметр необходим для компьютеры, управляемые Майкрософт для удовлетворения целей обновления для Microsoft 365 приложений. 

### <a name="first-run-settings"></a>Параметры первого запуска 

Существует несколько параметров, влияющих на поведение при первом Office запуска.

### <a name="accept-the-license-terms-on-behalf-of-the-end-user"></a>Принятие условий лицензии от имени конечного пользователя

**Значение по умолчанию:** отключено

При первом открываемом приложении Microsoft 365 пользователю будет предложено принять условия лицензии. Если вы хотите принять условия лицензии от имени пользователей, подайте запрос на обслуживание в команду компьютеры, управляемые Майкрософт operations с просьбой включить этот параметр. 

### <a name="suppress-outlook-mobile-check-box"></a>Подавление Outlook мобильного окна

**Значение по умолчанию:** отключено

При первом открываемом Outlook пользователю будет предложено установить Outlook Mobile. Если вы не хотите, чтобы пользователи видели этот контрольный ящик, отдайте запрос на обслуживание группе компьютеры, управляемые Майкрософт операций с просьбой включить этот параметр для устройств. 

## <a name="other-settings"></a>Другие параметры

Есть другие параметры Microsoft 365, которые компьютеры, управляемые Майкрософт можно дополнительно настроить от вашего имени. 

### <a name="disable-personal-onedrive"></a>Отключение персональных OneDrive

Некоторые организации обеспокоены тем, что пользователи имеют доступ как к корпоративным, так и к личным файлам на своих устройствах. Вы можете подать запрос на службу в компьютеры, управляемые Майкрософт operations с просьбой включить этот параметр. 

## <a name="settings-you-manage"></a>Параметры управлять

Существует множество других политик, которые компьютеры, управляемые Майкрософт еще не установлены в качестве части нашей службы. Эти политики можно настроить с помощью Microsoft Intune, которая использует [службу Office облачной](/DeployOffice/overview-office-cloud-policy-service#how-the-policy-configuration-is-applied) политики. Чтобы установить эти политики, выполните следующие действия:

1.  Во входе в Microsoft Endpoint Manager центр администрирования.
2.  Выбор **политик > приложений для Office приложений > Create**
3.  На странице **Создание конфигурации** политики сделайте следующее:
    - Введите имя.
    - Предоставление описания (необязательно).
    - В **заданиях** выберите, применима ли эта политика ко всем пользователям Приложения Microsoft 365 для предприятий или просто к пользователям, которые анонимно используют документы Office веб-страницы.
    - Выберите группу безопасности на основе AAD, назначенную конфигурации политики. Каждая конфигурация политик может быть назначена только одной группе, а каждой группе может быть назначена только одна конфигурация политики.
    - Настройка параметров политики, которые будут включены в конфигурацию политики. Вы можете найти имя параметра политики, чтобы найти параметр политики, который необходимо настроить. Вы также можете фильтровать приложение, является ли политика рекомендуемой базой безопасности и настроена ли политика. Столбец платформы указывает, применяется ли политика к Приложения Microsoft 365 для предприятий для Windows устройств, Office веб-сайтов или для всех.
4.  После выбора выберите **Create**.
