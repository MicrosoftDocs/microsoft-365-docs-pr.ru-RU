# <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

## <a name="initial-deployment"></a>Начальное развертывание

Microsoft Managed Desktop гарантирует, что приложения Microsoft 365 для предприятия (64-битные) устанавливаются в составе изображения на всех [устройствах программы.](../service-description/device-list.md) Все следующие приложения должны присутствовать на устройстве при доставке:

- Word
- Excel
- PowerPoint
- Outlook
- Publisher
- Access
- Skype для бизнеса
- OneNote

Этот подход минимизирует влияние сети и гарантирует, что пользователи смогут работать продуктивно сразу после получения устройства. Затем мы развертываем дополнительные политики на управляемых устройствах, чтобы настроить приложения для использования.

> [!NOTE]
> Microsoft Teams развертывается отдельно от Microsoft 365 Apps для предприятия и не входит в базовое изображение. 

### <a name="available-deployment-to-users"></a>Доступное развертывание для пользователей

Если у пользователя нет приложений Microsoft 365 на устройстве по какой-либо причине, вы можете использовать пакет, чтобы вернуть устройство в ожидаемое состояние. Добавьте пользователя в группу **Modern Workplace-Office-Office365_Install** и приложения станут доступны для них на портале компании.

### <a name="microsoft-365-apps-for-enterprise-32-bit"></a>Приложения Microsoft 365 для предприятия (32-битные)

Microsoft Managed Desktop не поддерживает развертывание 32-битной версии M365 Apps для предприятия.

## <a name="updates-to-microsoft-365-apps"></a>Обновления приложений Microsoft 365

Приложения Microsoft 365 будут обновляться на ежемесячном [корпоративном канале.](/deployoffice/overview-update-channels#monthly-enterprise-channel-overview) Эта практика предоставляет пользователям новые функции Office каждый месяц, но в предсказуемом расписании выпуска они будут получать только одно обновление в месяц. Обновления выпускаются во второй вторник месяца; эти обновления могут включать обновления функций, безопасности и качества. Эти обновления происходят автоматически и извлекаются непосредственно из CDN Office для этого конкретного канала.

Microsoft Managed Desktop задавляет каждый выпуск, чтобы определить возможные проблемы в вашей среде. Мы завершаем выпуск через 28 дней после выпуска из группы продуктов Microsoft 365 App. Microsoft Managed Desktop запланирует выпуски обновлений для различных групп, чтобы дать время для проверки и тестирования следующим образом: 

- Тест: ноль дней
- Первый: ноль дней
- Быстрый: 3 дня
- Broad: 7 дней

Microsoft Managed Desktop задает семидневный срок [обновления](/deployoffice/configure-update-settings-microsoft-365-apps) для устройств. После того как обновление доступно, оно должно быть установлено в течение семи дней. Пользователи получают [уведомления](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) о том, что обновления требуются в нескольких расположениях: приложении в подносе системы за 12 часов до крайнего срока, и они получают предупреждение за 15 минут до крайнего срока. Все приложения Microsoft 365 должны быть закрыты для завершения обновления.

### <a name="pausing-or-rolling-back-an-update"></a>Прио пауза или откат обновления

Если по какой-либо причине необходимо приостановить или откатать [](../working-with-managed-desktop/admin-support.md) обновление приложения Microsoft 365, подаст запрос на поддержку администратора на портале Microsoft Managed Desktop.

Во время выпуска microsoft Managed Desktop отслеживает показатели ошибок всех приложений Microsoft 365. Если мы заметим существенную разницу в качестве между новым выпуском и его предшественником, мы можем связаться с вами через портал администрирования управляемых настольных компьютеров Майкрософт. В зависимости от серьезности, мы будем спрашивать, хотите ли вы приостановить выпуск или сообщить вам, что мы приняли меры для смягчения проблемы. 

### <a name="delivery-optimization"></a>Оптимизация доставки

Оптимизация доставки — это одноранговая технология распространения, доступная в Windows 10. Это позволяет устройствам обмениваться контентом, например обновлениями, которые устройства скачали из Microsoft через Интернет. С его помощью можно уменьшить пропускную способность сети, так как устройство может получать части обновления с другого устройства в локальной сети вместо того, чтобы полностью скачать обновление из Microsoft.

[Оптимизация доставки](/deployoffice/delivery-optimization) включена по умолчанию на устройствах, работающих в выпусках Windows 10 Enterprise или Windows 10 Education. 

## <a name="settings-managed-by-microsoft-managed-desktop"></a>Параметры, управляемые Microsoft Managed Desktop

Корпорация Майкрософт управляет некоторыми настройками в рамках службы. Microsoft Managed Desktop не управляет базовым уровнем безопасности Office, но вы можете установить его самостоятельно, следуя указаниям в разделе [Параметры, которыми вы управляете.](#settings-you-manage)

### <a name="update-settings"></a>Обновление параметров

Microsoft Managed Desktop поддерживает все [параметры обновления](/deployoffice/configure-update-settings-microsoft-365-apps) для управляемых устройств, и эти параметры следует изменить.

### <a name="set-updates-to-occur-automatically"></a>Настроить автоматические обновления

**Значение по умолчанию:** включено

Эта политика настроена для обеспечения того, чтобы все устройства Office могли быть в курсе облачного времени. 

### <a name="set-a-deadline-when-updates-have-to-be-applied"></a>Установите крайний срок, когда необходимо применять обновления

**Значение по умолчанию:** 7 дней

Политика **UpdateDeadline** используется для настройки льготного периода, который пользователи имеют до принудительного обновления на устройстве. Эта политика крайних сроков также вызывает [уведомления](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) для пользователя, чтобы сообщить им об изменениях, необходимых на их устройстве.  

### <a name="defer-updates-on-a-device-for-a-period"></a>Отсрочка обновлений на устройстве на период

Эта политика настраивается по-разному для каждой группы устройств управления обновлениями и требуется для Microsoft Managed Desktop для удовлетворения своих целей обновления:  

- Тест: ноль дней
- Первый: ноль дней
- Быстрый 7 дней
- Broad: 21 дней

### <a name="update-notifications-settings"></a>Обновление параметров уведомлений

**Значение по умолчанию:** False

Параметр "Скрыть уведомления об обновлении" задается **параметру False** на управляемых [](/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) настольных устройствах Microsoft, чтобы предоставить пользователям наилучшие сведения об обновлении, уведомив их о необходимости обновления.

### <a name="specify-a-location-to-look-for-updates"></a>Указать расположение для поиска обновлений

**Значение по умолчанию:** Ежемесячный корпоративный канал

Сочетание политик **UpdatePath** и **UpdateChannel** используется по мере необходимости для достижения расписания обновлений. Эти политики должны гарантировать, что все устройства Office получают обновления непосредственно из CDN для ежемесячного корпоративного канала.

### <a name="specify-the-target-version-of-microsoft-365-apps"></a>Укажите целевую версию приложений Microsoft 365

Политика целевой версии иногда используется Microsoft Managed Desktop для отката или закрепления определенной версии Office. 


### <a name="hide-the-option-to-enable-or-disable-office-automatic-updates"></a>Скрыть возможность включить или отключить автоматические обновления Office

**Значение по умолчанию:** включено

Этот параметр необходим для Microsoft Managed Desktop, чтобы соответствовать его целям обновления для приложений Microsoft 365. 

### <a name="first-run-settings"></a>Параметры первого запуска 

Существует несколько параметров, влияющих на поведение при первом запуске Office.

### <a name="accept-the-license-terms-on-behalf-of-the-end-user"></a>Принятие условий лицензии от имени конечного пользователя

**Значение по умолчанию:** отключено

При первом открываемом пользователем приложении Microsoft 365 им будет предложено принять условия лицензии. Если вы хотите принять условия лицензии от имени пользователей, подайте запрос на обслуживание в команду операций microsoft Managed Desktop Operations с просьбой включить этот параметр. 

### <a name="suppress-outlook-mobile-check-box"></a>Подавление мобильного окна Outlook

**Значение по умолчанию:** отключено

При первом открываемом пользователем Outlook приложении Outlook им будет предложено установить Outlook Mobile. Если вы не хотите, чтобы пользователи видели этот контрольный ящик, задайте запрос на обслуживание в группе операций microsoft Managed Desktop Operations с просьбой включить этот параметр для устройств. 

## <a name="other-settings"></a>Другие параметры

Существуют и другие параметры приложения Microsoft 365, которые Microsoft Managed Desktop может дополнительно настроить от вашего имени. 

### <a name="disable-personal-onedrive"></a>Отключение личной OneDrive

**Значение по умолчанию:** отключено

Некоторые организации обеспокоены тем, что пользователи имеют доступ как к корпоративным, так и к личным файлам на своих устройствах. Вы можете подать запрос на службу в команду операций microsoft Managed Desktop Operations с просьбой включить этот параметр. 

## <a name="settings-you-manage"></a>Параметры, которые вы управляете

Существует множество других политик, которые Microsoft Managed Desktop еще не за набором в составе нашей службы. Эти политики можно настроить с помощью Microsoft Intune, которая использует [службу облачной политики Office.](/DeployOffice/overview-office-cloud-policy-service#how-the-policy-configuration-is-applied) Чтобы установить эти политики, выполните следующие действия:

1.  Во входе в центр администрирования Microsoft Endpoint Manager.
2.  Выбор **политик > приложений Office > Create**
3.  На странице **Создание конфигурации** политики сделайте следующее:
    - Введите имя.
    - Предоставление описания (необязательно).
    - В **заданиях** выберите, применяется ли эта политика ко всем пользователям приложений Microsoft 365 для предприятия или только к пользователям, которые анонимно имеют доступ к документам с помощью Office для веб-сайта.
    - Выберите группу безопасности на основе AAD, назначенную конфигурации политики. Каждая конфигурация политик может быть назначена только одной группе, а каждой группе может быть назначена только одна конфигурация политики.
    - Настройка параметров политики, которые будут включены в конфигурацию политики. Вы можете найти имя параметра политики, чтобы найти параметр политики, который необходимо настроить. Вы также можете фильтровать приложение, является ли политика рекомендуемой базой безопасности и настроена ли политика. Столбец платформы указывает, применяется ли политика к приложениям Microsoft 365 для корпоративных устройств Windows, Office для Интернета или всем.
4.  После выбора выберите **Create**.

> [!NOTE]
> Политики конфигурации Office поддерживают развертывание только на основе пользователя
