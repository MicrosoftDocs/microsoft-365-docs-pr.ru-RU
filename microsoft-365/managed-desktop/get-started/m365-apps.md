# <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

## <a name="initial-deployment"></a>Начальное развертывание

Компьютеры, управляемые Майкрософт, обеспечивают установку приложений Microsoft 365 для предприятий (64-битных версий) в составе образа на всех [программных устройствах.](../service-description/device-list.md) Все следующие приложения должны присутствовать на устройстве при его доставке:

- Word
- Excel
- PowerPoint
- Outlook
- Publisher
- Access
- Skype для бизнеса
- OneNote

Этот подход сводит к минимуму влияние на сеть и гарантирует, что пользователи смогут работать эффективно сразу после получения устройства. Затем мы развертываем дополнительные политики на управляемых устройствах, чтобы настроить приложения для использования.

> [!NOTE]
> Microsoft Teams развертывается отдельно от приложений Microsoft 365 для предприятий и не включается в базовый образ. 

### <a name="available-deployment-to-users"></a>Доступное развертывание для пользователей

Если у пользователя нет приложений Microsoft 365 на своем устройстве по какой-либо причине, вы можете использовать пакет, чтобы вернуть устройство в ожидаемое состояние. Добавьте пользователя в группу **Modern Workplace-Office-Office365_Install,** и приложения станут доступны им на портале компании.

### <a name="microsoft-365-apps-for-enterprise-32-bit"></a>Приложения Microsoft 365 для предприятий (32-битная)

Компьютеры, управляемые Майкрософт, не поддерживают развертывание 32-битной версии приложений M365 для предприятий.

## <a name="updates-to-microsoft-365-apps"></a>Обновления приложений Microsoft 365

Приложения Microsoft 365 обновляются в [monthly Enterprise Channel.](https://docs.microsoft.com/deployoffice/overview-update-channels#monthly-enterprise-channel-overview) Эта практика предоставляет пользователям новые функции Office каждый месяц, но они будут получать только одно обновление в месяц при прогнозируемом расписании выпусков. Обновления выпускаются во второй вторник месяца; эти обновления могут включать обновления функций, безопасности и качества. Эти обновления происходят автоматически и извлекаются непосредственно из CDN Office для этого канала.

На компьютерах, управляемых Майкрософт, каждый выпуск определяет потенциальные проблемы в вашей среде. Мы завершаем выпуск через 28 дней после выпуска из группы продуктов приложения Microsoft 365. Компьютеры, управляемые Майкрософт, запланируют выпуски обновлений для разных групп, чтобы дать время на проверку и тестирование следующим образом: 

- Тест: ноль дней
- Первый: ноль дней
- Быстрое: 7 дней
- Широкая: 21 день

На компьютерах, управляемых Майкрософт, устанавливается семидневный [крайний](https://docs.microsoft.com/deployoffice/configure-update-settings-microsoft-365-apps) срок обновления для устройств. После того как обновление станет доступным, оно должно быть установлено в течение семи дней. Пользователи [](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) получают уведомления о том, что обновления требуются в нескольких местах: приложение в системной области за 12 часов до крайнего срока и получает 15-минутное предупреждение перед крайним сроком. Для завершения обновления все приложения Microsoft 365 должны быть закрыты.

### <a name="pausing-or-rolling-back-an-update"></a>Приоренка или откат обновления

Если по какой-либо причине вам нужно приостановить или откатить обновление приложения Microsoft 365, поделайте запрос в службу поддержки администратора на портале Microsoft Managed Desktop. [](../working-with-managed-desktop/admin-support.md)

Во время выпуска компьютеры, управляемые Майкрософт, отслеживают количество ошибок во всех приложениях Microsoft 365. Если мы отметим существенное различие в качестве между новым выпуском и его предшественником, мы можем связаться с вами на портале администрирования управляемых компьютеров Майкрософт. В зависимости от серьезности мы запросим, хотите ли вы приостановить выпуск, или проинформировать вас о том, что мы приняли меры для решения проблемы. 

### <a name="delivery-optimization"></a>Оптимизация доставки

Оптимизация доставки — это технология одноранговой рассылки, доступная в Windows 10. Это позволяет устройствам обмениваться содержимым, например обновлениями, которые устройства скачали из Майкрософт через Интернет. Использование этого решения может помочь уменьшить пропускную способность сети, так как устройство может получать части обновления с другого устройства в локальной сети вместо того, чтобы полностью загружать обновление от Майкрософт.

[Оптимизация доставки](https://docs.microsoft.com/deployoffice/delivery-optimization) включена по умолчанию на устройствах под управлением выпусков Windows 10 Корпоративная или Windows 10 для образовательных образования. 

## <a name="settings-managed-by-microsoft-managed-desktop"></a>Параметры, управляемые компьютером, управляемым Майкрософт

Корпорация Майкрософт управляет некоторыми настройками в рамках службы. Компьютеры, управляемые Майкрософт, не управляют базовыми показателями безопасности Office, но их можно настроить самостоятельно, следуя указаниям в разделе ["Параметры".](#settings-you-manage)

### <a name="update-settings"></a>Обновление параметров

На компьютере, управляемом Майкрософт, поддерживаются все параметры [обновления](https://docs.microsoft.com/deployoffice/configure-update-settings-microsoft-365-apps) для управляемых устройств, и эти параметры следует изменить.

### <a name="set-updates-to-occur-automatically"></a>Настроить автоматические обновления

**Значение по умолчанию:** включено

Эта политика настроена для обеспечения того, чтобы все устройства Office могли быть в курсе всех данных из облака. 

### <a name="set-a-deadline-when-updates-have-to-be-applied"></a>Установите крайний срок, когда необходимо применить обновления

**Значение по умолчанию:** 7 дней

Политика **UpdateDeadline** используется для настройки льготного периода, который пользователи имеют до принудительного обновления на устройстве. Эта политика крайнего [](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) срока также инициирует уведомления пользователя об изменениях, необходимых для его устройства.  

### <a name="defer-updates-on-a-device-for-a-period"></a>Отсрочка обновлений на устройстве на период

Эта политика настраивается по-разному для каждой группы устройств управления обновлениями и необходима для того, чтобы компьютеры, управляемые Майкрософт, отвечали целевым объектам обновления:  

- Тест: ноль дней
- Первый: ноль дней
- Быстрое 7 дней
- Широкая: 21 день

### <a name="update-notifications-settings"></a>Обновление параметров уведомлений

**Значение по умолчанию:** False

Для параметра "скрыть уведомления об обновлениях" на настольных устройствах, управляемых Майкрософт, задайте для них параметр **False,** чтобы обеспечить наилучшее обновление для пользователей, уведомляя их о необходимости обновления. [](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps)

### <a name="specify-a-location-to-look-for-updates"></a>Указать расположение для поиска обновлений

**Значение по умолчанию:** Monthly Enterprise Channel

Для достижения расписания обновления используется сочетание политик **UpdatePath** и **UpdateChannel.** Эти политики гарантируют, что все устройства Office будут получать обновления непосредственно из CDN для Monthly Enterprise Channel.

### <a name="specify-the-target-version-of-microsoft-365-apps"></a>Указание целевой версии приложений Microsoft 365

Политика целевой версии иногда используется компьютером, управляемым Майкрософт, для отката или закрепления определенной версии Office. 


### <a name="hide-the-option-to-enable-or-disable-office-automatic-updates"></a>Скрытие параметра для включаемого или отключения автоматических обновлений Office

**Значение по умолчанию:** включено

Этот параметр необходим для того, чтобы компьютеры, управляемые Майкрософт, соответствуют целям обновления для приложений Microsoft 365. 

### <a name="first-run-settings"></a>Параметры первого запуска 

Существует несколько параметров, влияющих на поведение при первом запуске Office.

### <a name="accept-the-license-terms-on-behalf-of-the-end-user"></a>Принятие условий лицензии от имени конечного пользователя

**Значение по умолчанию:** Отключено

Когда пользователь впервые открывает приложение Microsoft 365, пользователю будет предложено принять условия лицензионного соглашения. Если вы хотите принять условия лицензионного соглашения от имени пользователей, подайте запрос на обслуживание в команду Microsoft Managed Desktop Operations с запросом включения этого параметра. 

### <a name="suppress-outlook-mobile-check-box"></a>Подавление мобильного outlook

**Значение по умолчанию:** Отключено

Когда пользователь впервые открывает Outlook, пользователю будет предложено установить Outlook Mobile. Если вы не хотите, чтобы пользователи видели этот параметр, подайте запрос на обслуживание в команду Microsoft Managed Desktop Operations, включив этот параметр для ваших устройств. 

## <a name="other-settings"></a>Другие параметры

Существуют другие параметры приложения Microsoft 365, которые можно дополнительно настроить от вашего имени. 

### <a name="disable-personal-onedrive"></a>Отключение личного oneDrive

**Значение по умолчанию:** Отключено

В некоторых организациях пользователи имеют доступ как к корпоративным, так и к личным файлам на своих устройствах. Вы можете подать запрос на обслуживание в команду Microsoft Managed Desktop Operations с запросом включения этого параметра. 

## <a name="settings-you-manage"></a>Параметры, которые вы управляете

Существует множество других политик, которые компьютеры, управляемые Майкрософт, еще не настроены как часть нашей службы. Вы можете настроить эти политики с помощью Microsoft Intune, который использует службу [политики Облака Office.](https://docs.microsoft.com/DeployOffice/overview-office-cloud-policy-service#how-the-policy-configuration-is-applied) Чтобы настроить эти политики, выполните следующие действия:

1.  Во входе в Центр администрирования Microsoft Endpoint Manager.
2.  Select **Apps > Policies for Office apps > Create**
3.  На странице **"Создание конфигурации** политики" сделайте следующее:
    - Введите имя.
    - В качестве описания (необязательно).
    - В **заданиях** выберите, применяется ли эта политика ко всем пользователям приложений Microsoft 365 для предприятий или только к пользователям, которые анонимно используют Office для Интернета.
    - Выберите группу безопасности на основе AAD, назначенную конфигурации политики. Каждая конфигурация политики может быть назначена только одной группе, а каждой группе может быть назначена только одна конфигурация политики.
    - Настройте параметры политики, которые необходимо включить в конфигурацию политики. Чтобы найти параметр политики, который необходимо настроить, можно найти имя параметра политики. Вы также можете отфильтровать приложение, определить, является ли политика рекомендуемой базой безопасности и настроена ли политика. Столбец платформы указывает, применяется ли политика к приложениям Microsoft 365 для предприятий для устройств с Windows, Office в Интернете или всем.
4.  После выбора выберите **"Создать".**

> [!NOTE]
> Политики конфигурации Office поддерживают развертывание только на основе пользователей
