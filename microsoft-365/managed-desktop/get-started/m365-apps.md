# <a name="microsoft-365-apps-for-enterprise"></a>Приложения Microsoft 365 для предприятий

## <a name="initial-deployment"></a>Начальное развертывание

Настольные компьютеры, управляемые Майкрософт, гарантируют, что приложения Microsoft 365 для Enterprise (64-бит) установлены как часть изображения на всех [устройствах программ](../service-description/device-list.md). Все указанные ниже приложения должны присутствовать на устройстве при его доставке.

- Word
- Excel
- PowerPoint
- Outlook
- Publisher
- Access
- Skype для бизнеса
- OneNote

Такой подход сводит к минимуму влияние на сеть и гарантирует, что пользователи могут работать сразу же после получения своего устройства. Затем мы развертываем дополнительные политики на управляемых устройствах, чтобы настроить приложения для использования.

> [!NOTE]
> Microsoft Teams разворачивается отдельно от Microsoft 365 приложений для предприятия и не включается в базовый образ. 

### <a name="available-deployment-to-users"></a>Доступное развертывание для пользователей

Если у пользователя нет приложений Microsoft 365 на своем устройстве по какой либо причине, можно использовать пакет, чтобы вернуть устройство к ожидаемому состоянию. Добавьте пользователя на **современном рабочем месте — Office Office365_Install** группу, и приложения станут доступны для них на портале компании.

### <a name="microsoft-365-apps-for-enterprise-32-bit"></a>Приложения Microsoft 365 для предприятий (32-разрядная версия)

Настольные компьютеры, управляемые корпорацией Майкрософт, не поддерживают развертывание 32 – разрядной версии приложений M365 для предприятия.

## <a name="updates-to-microsoft-365-apps"></a>Обновления для приложений Microsoft 365

Приложения Microsoft 365 настроены на обновление на [месячном корпоративном канале](https://docs.microsoft.com/deployoffice/overview-update-channels#monthly-enterprise-channel-overview). Это практическое руководство предоставляет пользователям новые функции Office каждый месяц, но они получают только одно обновление в месяц на предсказуемое расписание выпусков. Обновления выпускаются во второй вторник месяца; Эти обновления могут включать в себя функции, безопасность и обновления качества. Эти обновления происходят автоматически и извлекаются непосредственно из CDN Office для соответствующего канала.

Все выпуски, управляемые корпорацией Майкрософт, задаются для определения потенциальных проблем в вашей среде. Мы завершаем развертывание за 28 дней после выпуска из группы продуктов Microsoft 365. Для настольных компьютеров под управлением Майкрософт обновления выпусков для разных групп разрешаются для проверки и тестирования следующим образом: 

- Тест: 0 дней
- Первый: 0 дней
- Высокая скорость: 7 дней
- Общие действия: 21 день

На рабочем столе Майкрософт устанавливаются семь дней [обновления](https://docs.microsoft.com/deployoffice/configure-update-settings-microsoft-365-apps) для устройств. Когда обновление будет доступно, оно должно быть установлено в течение семи дней. Пользователи получают [уведомление](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) о том, что эти обновления необходимы в нескольких расположениях: приложение в панели задач 12 часов до крайнего срока и оно получает 15 минут перед крайним сроком. Все приложения Microsoft 365 должны быть закрыты для завершения обновления.

### <a name="pausing-or-rolling-back-an-update"></a>Приостановка или откат обновления

Если вам нужно приостановить или откатить обновление Microsoft 365 по любой причине, файл должен быть [запросом администратора](../working-with-managed-desktop/admin-support.md) через портал для настольных компьютеров Майкрософт.

Во время выпуска на рабочем столе Майкрософт контролируется частота ошибок всех приложений Microsoft 365. Если у вас есть существенная разница между новым выпуском и его предшественником, мы можем связаться с вами через портал администрирования настольных компьютеров, управляемый Майкрософт. В зависимости от серьезности мы попросим вас приостановить выпуск или сообщить о том, что мы предприняли меры по устранению проблемы. 

### <a name="delivery-optimization"></a>Оптимизация доставки

Оптимизация доставки — это технология однорангового распределения, доступная в Windows 10. Он позволяет устройствам обмениваться контентом, например обновлением, которые были загружены из Майкрософт через Интернет. Это может способствовать снижению пропускной способности сети, так как устройство может получать части обновления с другого устройства в локальной сети, а не загружать обновление полностью с Майкрософт.

[Оптимизация доставки](https://docs.microsoft.com/deployoffice/delivery-optimization) по умолчанию включена на устройствах под управлением выпусков Windows 10 Корпоративная или Windows 10 для образовательных учреждений. 

## <a name="settings-managed-by-microsoft-managed-desktop"></a>Параметры, управляемые настольным компьютером, управляемым Майкрософт

Майкрософт управляет некоторыми параметрами как частью службы. Рабочий стол, управляемый Майкрософт, не управляет базовым планом безопасности Office, но вы можете задать его самостоятельно, следуя указаниям в разделе [Параметры, которыми вы управляете](#settings-you-manage) .

### <a name="update-settings"></a>Обновление параметров

На рабочем столе Майкрософт поддерживаются все [Параметры обновления](https://docs.microsoft.com/deployoffice/configure-update-settings-microsoft-365-apps) для управляемых устройств, и эти параметры следует изменить.

### <a name="set-updates-to-occur-automatically"></a>Настроить автоматические обновления

**Значение по умолчанию**: включено

Эта политика настроена, чтобы гарантировать, что все устройства Office можно хранить в облаке. 

### <a name="set-a-deadline-when-updates-have-to-be-applied"></a>Задать крайний срок, когда требуется применить обновления

**Значение по умолчанию**: 7 дней

Политика **упдатедеадлине** используется для настройки льготного периода, который пользователи имеют до применения обновления на устройстве. Эта политика крайних сроков также инициирует уведомления для пользователя [о](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) том, какие изменения необходимы на их устройствах.  

### <a name="defer-updates-on-a-device-for-a-period"></a>Откладывание обновления устройства в течение периода

Эта политика настроена по-разному для каждой группы устройств управления обновлениями и является обязательной для компьютеров, управляемых Майкрософт, для соответствия целевым целям обновления:  

- Тест: 0 дней
- Первый: 0 дней
- Быстро 7 дней
- Общие действия: 21 день

### <a name="update-notifications-settings"></a>Обновление параметров уведомлений

**Значение по умолчанию**: false

Параметр "скрыть уведомления об обновлениях" имеет значение **false** на компьютерах, управляемых Майкрософт, чтобы обеспечить оптимальное взаимодействие с пользователями, [уведомляя](https://docs.microsoft.com/deployoffice/end-user-update-notifications-microsoft-365-apps#notifications-your-users-see-when-you-set-an-update-deadline-for-microsoft-365-apps) их, когда требуется обновление.

### <a name="specify-a-location-to-look-for-updates"></a>Указать расположение для поиска обновлений

**Значение по умолчанию**: месячный корпоративный канал

При необходимости для достижения расписания обновления используется комбинация политик **UpdatePath** и **UpdateChannel** . Эти политики настроены так, чтобы все устройства Office получали обновления непосредственно из сети CDN для ежемесячного корпоративного канала.

### <a name="specify-the-target-version-of-microsoft-365-apps"></a>Указание целевой версии приложений Microsoft 365

Целевая политика версий иногда используется на настольном компьютере, управляемом Майкрософт, для отката или закрепления определенной версии Office. 


### <a name="hide-the-option-to-enable-or-disable-office-automatic-updates"></a>Скрытие параметра для включения и отключения автоматического обновления Office

**Значение по умолчанию**: включено

Этот параметр необходим для компьютеров, управляемых корпорацией Майкрософт, для соответствия целевым целям обновления для приложений Microsoft 365. 

### <a name="first-run-settings"></a>Параметры первого запуска 

Существует несколько параметров, влияющих на поведение при первом запуске Office.

### <a name="accept-the-license-terms-on-behalf-of-the-end-user"></a>Принятие условий лицензионного соглашения от имени конечного пользователя

**Значение по умолчанию**: отключено

При первом открытии приложения Microsoft 365 пользователю предлагается принять условия лицензионного соглашения. Если вы хотите принять условия лицензии от имени ваших пользователей, запросите запрос на обслуживание с помощью группы операций с управляемыми рабочими столами Майкрософт, чтобы включить этот параметр. 

### <a name="suppress-outlook-mobile-check-box"></a>Флажок "запретить Outlook Mobile"

**Значение по умолчанию**: отключено

При первом открытии Outlook пользователю предлагается установить Outlook Mobile. Если вы не хотите, чтобы пользователи видели этот флажок, запросите запрос на обслуживание с помощью группы операций с управляемыми рабочими столами Майкрософт, чтобы включить этот параметр для устройств. 

## <a name="other-settings"></a>Другие параметры

Существуют другие параметры приложений Microsoft 365, которые могут быть настроены на компьютере, управляемом Майкрософт, при необходимости. 

### <a name="disable-personal-onedrive"></a>Отключить личный OneDrive

**Значение по умолчанию**: отключено

Некоторые организации интересуют пользователей, имеющих доступ к корпоративным и личным файлам на своих устройствах. Вы можете создать запрос на обслуживание с помощью группы операций с управляемыми рабочими столами Майкрософт, чтобы включить этот параметр. 

## <a name="settings-you-manage"></a>Параметры, которыми вы управляете

Существует множество других политик, которые не задаются с помощью Microsoft Managed Desktop в нашей службе. Их можно настроить с помощью Microsoft Intune, который использует службу [политики облака Office](https://docs.microsoft.com/DeployOffice/overview-office-cloud-policy-service#how-the-policy-configuration-is-applied) . Для этого выполните следующие действия:

1.  Войдите в центр администрирования Microsoft Endpoint Manager.
2.  Выбор **приложений > политик для приложений Office > создание**
3.  На странице " **Создание конфигурации политики** " выполните следующие действия:
    - Введите имя.
    - Укажите описание (необязательно).
    - В разделе **назначения**выберите, применяется ли эта политика ко всем пользователям приложений Microsoft 365 для предприятий или только к пользователям, которые анонимно обращаются к документам с помощью Office для Интернета.
    - Выберите группу безопасности на основе AAD, назначенную конфигурации политики. Каждая конфигурация политики может быть назначена только одной группе, и каждой группе может быть назначена только одна конфигурация политики.
    - Настройте параметры политики для включения в конфигурацию политики. Вы можете выполнить поиск по имени параметра политики, чтобы найти параметр политики, который необходимо настроить. Вы также можете отфильтровать приложение, в зависимости от того, является ли политика рекомендуемой для безопасности, а также в зависимости от того, настроена ли политика. Столбец Platform указывает, применяется ли политика к приложениям Microsoft 365 для устройств с Windows, Office в Интернете или всем.
4.  Выполнив выбор, нажмите кнопку **создать**.

> [!NOTE]
> Политики конфигурации Office поддерживают только развертывание на основе пользователей
