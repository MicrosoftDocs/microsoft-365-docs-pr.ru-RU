# <a name="during-and-after-your-data-move"></a>Во время и после перемещения данных

Перемещение данных — это тыловая операция с минимальным влиянием на конечных пользователей. Пока корпорация Майкрософт перемещает каждую службу и связанные данные клиента в новый геоцентр данных, никаких действий не требуется. Передача и проверка данных происходят заранее в фоновом режиме с минимальным влиянием на пользователей.
  
> [!NOTE]
> Перемещения происходят в разное время для каждой службы. В результате вы увидите описанные функции для каждой службы в разное время. 
  
Просмотрите Центр сообщений Microsoft 365 для подтверждения завершения перемещения для каждой службы чата Exchange Online, SharePoint Online и Teams. ## <a name="exchange-online"></a>Exchange Online

Так как перемещение каждого пользователя в новый геоцентр данных для одного клиента занимает некоторое время, некоторые пользователи будут по-прежнему в старом географическом месте центра обработки данных во время перемещения, а другие будут в новом географическом месте центра данных. Это означает, что некоторые функции, которые включают доступ к нескольким почтовым ящикам, могут не полностью работать в течение периода перемещения, который может длиться недели. Эти функции описаны в следующих разделах.
  
### <a name="open-shared-folder-in-outlook-web-access"></a>Открытие "Общей папки" в Outlook Web Access

Некоторые пользователи открывают общую папку почты из другого почтового ящика (в который пользователь имеет разрешения на чтение или записи) в Outlook Web Access с помощью функции "Общая папка". В следующей таблице описано, как работает доступ к общим папок во время перемещения почтового ящика. Обратите внимание, что пользователи с полными разрешениями на доступ к общему почтовому ящику могут открывать почтовый ящик с помощью Outlook Web Access во время перемещения. 
  
|**Конфигурация**|**Описание**|
|:-----|:-----|
|Пользователь имеет разрешение папки почтового ящика на другой почтовый ящик  <br/> |Потенциально ограниченная.  <br/> Если пользователь А и почтовый ящик B не имеют одного географического географического интерфейса во время перемещения клиента, пользователь А не сможет открыть папку почтового ящика B в Outlook Web Access, если пользователь А имеет разрешение только на определенную папку в почтовом ящике B.  <br/> Чтобы добавить общую папку, щелкните правой кнопкой мыши имя пользователя на левой панели навигации и выберите "Добавить **общую папку".**  <br/> |
|Пользователь с полным разрешением на доступ к другому почтовому ящику  <br/> |Полностью поддерживается.  <br/> Если у пользователя А есть разрешение "Полный доступ" к почтовому ящику B, пользователь А может щелкнуть общую папку на левой панели навигации в Outlook Web Access, чтобы открыть окно с почтовым ящиком B.  Пользователь может открыть общий почтовый ящик с помощью Outlook Web Access во время перемещения без каких-либо отрицательных последствий. Ограничение применяется только к совместному доступу на уровне папок в почтовом ящике.           |
  
## <a name="sharepoint-online"></a>SharePoint Online

При перемещении SharePoint Online также перемещаются данные для следующих служб:
  
- OneDrive для бизнеса
    
- Видеослужб Microsoft 365
    
- Office в браузере
    
- Приложения Microsoft 365 для предприятий
    
- Visio Pro для Microsoft 365
    
После завершения перемещения данных SharePoint Online вы можете увидеть некоторые из следующих эффектов.
  
### <a name="microsoft-365-video-services"></a>Видеослужб Microsoft 365

- Перемещение данных для видео занимает больше времени, чем перемещение остального содержимого в SharePoint Online.
    
- После того как содержимое SharePoint Online будет перемещено, будет период времени, в течение которых не удастся отыграть видео.
    
- Мы удаляем перекодированные копии из предыдущего центра обработки данных и повторно перекодировали их в новом.
    
### <a name="search"></a>Поиск

В процессе перемещения данных SharePoint Online мы переносим ваш индекс поиска и параметры поиска в новое расположение. Пока мы не **завершим** перемещение данных SharePoint Online, мы по-прежнему будем обслуживать пользователей из индекса в исходном расположении. В новом расположении поиск автоматически начинает обход контента после завершения перемещения данных SharePoint Online. С этого момента и далее мы обслуживаем пользователей из перенесенного индекса. Изменения содержимого, произошедшие после миграции, не включаются в индекс переноса, пока обход не заберет их. Большинство клиентов не замечают, что результаты стали менее новыми сразу после завершения переноса данных SharePoint Online, но у некоторых клиентов может снизиться их свежесть в течение первых 24–48 часов. 
  
Затронуты следующие функции поиска:
  
- Результаты поиска и веб-части: результаты не включают изменения, которые произошли после миграции, пока обход не заберет их. 
    
- Delve: Delve не включает изменения, которые произошли после миграции, пока обход не заберет их.
    
- Отчеты о популярности и поиске для сайта: количество отчетов Excel в новом расположении включает только перенесенные подсчеты и подсчеты из отчетов об использовании, которые были выполнены после завершения перемещения данных SharePoint Online. Все подсчеты за промежуточный период теряются и не могут быть восстановлены. Этот период обычно составляет пару дней. У некоторых клиентов могут быть более короткие или более длительные убытки.
    
- Видеопорт портал: количество просмотров и статистика видеопортла зависят от статистики для отчетов Excel, поэтому количество просмотров и статистика видеопортла теряются за тот же период времени, что и отчеты Excel.
    
- EDiscovery: элементы, измененные во время миграции, не показываются, пока обход не отберет изменения.
    
- Защита от потери данных (DLP): политики не применяются к элементов, которые изменяются, пока обход не захотеет внести изменения.

## <a name="microsoft-teams"></a>Microsoft Teams

Помимо Exchange Online, SharePoint Online и OneDrive для бизнеса, корпорация Майкрософт будет перенести данные службы чата Teams в локальный центр обработки данных.

- Сообщения чата Teams, включая личные сообщения и сообщения каналов.
- Изображения Teams, используемые в чатах.

Файлы Teams хранятся в SharePoint Online, а файлы чата Teams — в OneDrive для бизнеса. Голосовая почта, календарь, история чата и контакты хранятся в Exchange Online. Во многих случаях Exchange Online, SharePoint Online и OneDrive для бизнеса уже используются клиентом в локальном географическом центрах обработки данных и также являются частью программы миграции Microsoft 365 для соответствующих стран пользователей.
