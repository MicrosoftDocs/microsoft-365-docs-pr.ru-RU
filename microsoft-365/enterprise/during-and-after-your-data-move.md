# <a name="during-and-after-your-data-move"></a>Во время и после перемещения данных

Перемещение данных — это фоновая операция с минимальным влиянием на конечных пользователей. Пока корпорация Майкрософт переместит каждую службу и связанные данные для клиента в новый Geo центра обработки данных, никаких действий не требуется. Передача и проверка данных выполняются в фоновом режиме заранее с минимальным влиянием на пользователей.
  
> [!NOTE]
> Перемещения происходят в разное время для каждой службы. В результате вы увидите описание ограниченной функциональности для каждой службы в другое время. 
  
Просмотрите центр сообщений Microsoft 365 на наличие подтверждения при перемещении для каждого из Exchange Online, SharePoint Online, Teams и Skype для бизнеса завершены. Как показано в приведенной ниже таблице, может потребоваться до 24 месяцев после завершения срока подачи заявки, чтобы все запрошенные данные перемещались для всех клиентов в определенном географическом формате. Если после перемещения вы видите проблемы с клиентом, обратитесь в [службу поддержки](https://go.microsoft.com/fwlink/p/?LinkID=522459) , чтобы получить помощь.

## <a name="exchange-online"></a>Exchange Online

Так как требуется время для перемещения каждого пользователя в новый центр обработки данных для одного клиента, некоторые пользователи по-прежнему будут находиться в старом центре обработки данных во время перемещения, а другие — в новом географическом центре центра обработки данных. Это означает, что некоторые функции, включающие доступ к нескольким почтовым ящикам, могут не полностью работать в течение периода перемещения, который может длиться последние недели. Эти функции описаны в следующих разделах.
  
### <a name="open-shared-folder-in-outlook-web-access"></a>Открытие "общей папки" в Outlook Web Access

Некоторые пользователи открывают общую папку почты из другого почтового ящика (у пользователя есть разрешения на чтение или запись) в Outlook Web Access с помощью функции "Общая папка". В следующей таблице описано, как доступ к общим папкам выполняется при перемещении почтового ящика. Обратите внимание, что пользователи с полными разрешениями на доступ к общему почтовому ящику могут открыть этот почтовый ящик с помощью Outlook Web Access во время перемещения. 
  
|**Конфигурация**|**Описание**|
|:-----|:-----|
|Для пользователя есть разрешение на доступ к папке почтового ящика  <br/> |Может быть ограниченным.  <br/> Если пользователь а и почтовый ящик B находятся в разных регионах во время перемещения клиента, пользователь а не может открыть папку "B" в Outlook Web Access, если пользователь а имеет разрешение только на определенную папку в почтовом ящике B.  <br/> Чтобы добавить общую папку, щелкните правой кнопкой мыши имя пользователя в левой панели навигации и выберите команду **Добавить общую папку**.  <br/> |
|Пользователь с разрешением полного почтового ящика для другого почтового ящика  <br/> |Полностью поддерживается.  <br/> Если пользователь а имеет разрешение "полный доступ" для почтового ящика B, то пользователь а может щелкнуть общую папку на левой панели навигации в Outlook Web Access, чтобы открыть окно, в котором отображается почтовый ящик B.  Пользователь может открыть общий почтовый ящик с помощью Outlook Web Access во время перемещения без каких-либо неблагоприятных последствий. Ограничение применяется только к общему доступу на уровне папок в почтовом ящике.           |
  
## <a name="sharepoint-online"></a>SharePoint Online

При перемещении SharePoint Online также перемещается данные для следующих служб:
  
- OneDrive для бизнеса
    
- Project Online
    
- Project для Microsoft 365
    
- Службы Microsoft 365 Video Services
    
- Office в браузере s
    
- Приложения Microsoft 365 для предприятий
    
- Visio Pro для Microsoft 365
    
После переноса данных SharePoint Online могут отображаться некоторые из следующих эффектов.
  
### <a name="microsoft-365-video-services"></a>Службы Microsoft 365 Video Services

- Перемещение данных для видеоролика занимает больше времени, чем перемещение содержимого в SharePoint Online.
    
- После перемещения содержимого SharePoint Online будет использоваться период времени, в течение которого невозможно воспроизводить видео.
    
- Мы удаляем посторонние копии из предыдущего центра обработки данных и перекодируете их в новом центре обработки данных.
    
### <a name="search"></a>Поиск

В процессе перемещения данных из SharePoint Online мы перенесем поисковые индексы и параметры поиска в новое расположение. Пока мы не **перейдем к данным** SharePoint Online, мы по-прежнему предоставляем пользователям индекс в исходном расположении. В новом расположении Поиск автоматически начинает обход контента после завершения перемещения данных SharePoint Online. С этого момента и далее мы предоставляем пользователям из перенесенного индекса. Изменения содержимого, произошедшие после переноса, не включаются в перенесенный индекс до тех пор, пока обход контента не будет выполнен. Большинство клиентов не завнимание, что результаты являются менее свежими после завершения перемещения данных SharePoint Online, но некоторые клиенты могут испытывать снижение актуальности в течение первых 24-48 часов. 
  
Затронуты следующие функции поиска:
  
- Результаты поиска и веб-части поиска: результаты не включают изменения, произошедшие после миграции, пока их обход не будет выполнен. 
    
- Delve: Delve не включает изменений, произошедших после миграции, пока обход контента не будет выполнен.
    
- Отчеты о популярности и поиске для сайта: счетчики отчетов Excel в новом расположении включают в себя только перенесенные счетчики и счетчики из отчетов об использовании, выполненных после завершения перемещения данных SharePoint Online. Все счетчики из промежуточного периода теряются и не могут быть восстановлены. Этот период обычно составляет несколько дней. Некоторые клиенты могут испытывать меньше или больше потерь.
    
- Видеопортал: Просмотр счетчиков и статистики для видеопортала зависит от статистики для отчетов Excel, поэтому счетчики просмотра и статистика для видеопортала будут утрачены за один и тот же период времени, что и для отчетов Excel.
    
- Обнаружение электронных данных: элементы, которые были изменены во время миграции, не отображаются до тех пор, пока обход контента не изменится.
    
- Защита от потери данных (DLP): политики не применяются к элементам, которые меняются до тех пор, пока обход контента не изменится.

## <a name="microsoft-teams"></a>Microsoft Teams

Помимо Exchange Online, SharePoint Online и OneDrive для бизнеса, корпорация Майкрософт будет переносить данные Teams в локальный центр обработки данных.

- Сообщения в Microsoft Teams, включая конфиденциальные сообщения и сообщения канала.
- Изображения Teams, используемые в беседах.

Файлы Teams хранятся в SharePoint Online и файлы чата в Teams хранятся в OneDrive для бизнеса. Голосовая почта, календарь, журнал чата и контакты хранятся в Exchange Online. Во многих случаях Exchange Online, SharePoint Online и OneDrive для бизнеса уже используются клиентом в географическом центре обработки данных и являются частью программы миграции Microsoft 365 для соответствующих стран клиентов.

## <a name="skype-for-business"></a>Skype для бизнеса

Перемещение в Skype для бизнеса доступно для Австралии, Японии, Индии, Канада, Великобритании и Южной Кореи.

Все пользователи будут выходить из клиентского программного обеспечения Skype для бизнеса во время вырезания. Автоматический вход приведет к повторному подключению пользователей в течение двух минут.
  
|**Функции, которые работают во время всего перемещения**|**Функции, которые могут быть ограничены во время части перемещения**|
|:-----|:-----|
| Обмен мгновенными сообщениями и голосовые вызовы  <br/>  Пользователи могут добавлять контакты, добавлять группы контактов, добавлять собрания, задавать их расположение и изменять "что происходит сегодня".  <br/>  Параметры поставщика аудио-и видеоконференций (ACP) копируются в целевой объект центра обработки данных. Если поставщик ACP присутствует в целевом центре обработки данных, он будет работать. В противном случае он не будет.  <br/> | Администратор клиента TRPS (Remote Remote PowerShell) не будет доступен администраторам для создания сеансов.  <br/>  Администратор клиента LAC будет недоступен для входа и изменения параметров пользователя.  <br/> |
   
|**После перемещения**|
|:-----|
| Данные о собрании (отправленные презентации и т. д.) не будут перемещены и их потребуется отправить повторно.  <br/>  Более ранние клиенты Lync, такие как клиент Lync 2010 и клиент Lync для Mac 2011, задают кэширование информации DNS для службы, которая вызывает проблемы с входом. Очистка кэша DNS может потребоваться, если пользователь не входит в последнюю версию клиента Skype для бизнеса для Windows. Ознакомьтесь [с разрешениями проблем с конфигурацией DNS в Skype для бизнеса Online в Office 365](https://docs.microsoft.com/skypeforbusiness/troubleshoot/online-configuration/dns-configuration-issue). Пользователи клиента Lync для Mac должны выполнить указанные ниже [инструкции](https://support.microsoft.com/kb/2629861).  <br/> |
   
### <a name="skype-for-business-moves-that-involve-a-third-party-audio-conferencing-provider"></a>В Skype для бизнеса перемещаются сведения, связанные с поставщиком услуг голосовой связи стороннего производителя
Сторонние надстройки поставщика услуг аудио-и видеоконференций для Skype для бизнеса недоступны для пользователей, размещенных в новых географических центрах обработки данных.  Существующие клиенты, использующие сторонние службы сторонних поставщиков конференций, не должны запрашивать перемещение в новый центр обработки данных.  Новые клиенты, развернутые в новых географических центрах обработки данных, должны запросить перемещение в региональный центр обработки данных, чтобы использовать стороннего поставщика конференций с аудио-и видеоконференциями.
