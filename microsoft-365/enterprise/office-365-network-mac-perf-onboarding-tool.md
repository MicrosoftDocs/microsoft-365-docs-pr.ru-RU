# <a name="microsoft-365-network-connectivity-test-tool-preview"></a>Средство проверки сетевого подключения Microsoft 365 (предварительная версия)

Средство проверки сетевого подключения Microsoft 365 находится по адресу <https://connectivity.office.com> . Это средство для оценки сети и сведений о сети, доступных в Центре администрирования Microsoft 365 в центре | **Меню подключения.**

> [!IMPORTANT]
> Важно войти в клиент Microsoft 365, так как все тестовые отчеты будут общими для администратора и загружены в клиент во время входов.

![Средство проверки подключения](../media/m365-mac-perf/m365-mac-perf-test-tool-page.png)

>[!NOTE]
>Средство проверки сетевого подключения поддерживает клиентов в WW Commercial и Germany, но не на GCC Moderate, GCC High, DoD или Китае.

Данные о сети в Центре администрирования Microsoft 365 основаны на регулярных измерениях продукта для клиента Microsoft 365, которые объединяются каждый день. Для сравнения, данные о сети из теста сетевого подключения Microsoft 365 запускаются локально и один раз в средстве. Тестирование, которое может быть сделано в продукте, ограничено, и путем локального тестирования для пользователя можно собрать дополнительные данные, что дает более глубокие сведения. Затем следует учитывать, что данные о сети в Центре администрирования Microsoft 365 показывают, что существует проблема с сетью для использования Microsoft 365 в определенном офисе. Тест подключения Microsoft 365 может помочь определить первопричину этой проблемы, что ведет к рекомендуемой меры по повышению производительности сети.

Рекомендуется использовать их вместе, если состояние качества сети можно оценить для каждого расположения офиса в Центре администрирования Microsoft 365, а более конкретные данные можно найти после развертывания тестирования на основе теста подключения Microsoft 365.

>[!IMPORTANT]
>Анализ сети, рекомендации по производительности и оценки в Центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, которые зарегистрированы в программе предварительного просмотра функций.

## <a name="what-happens-at-each-test-step"></a>Что происходит на каждом тестовом этапе

### <a name="office-location-identification"></a>Идентификация расположения Office

При нажатии кнопки "Запустить тест" мы откажемся от запущенной тестовой страницы и определите расположение офиса. Вы можете ввести свое местоположение по городу, штату и стране или определить его в веб-браузере. Если вы обнаружите его, мы запросим широту и долготу из веб-браузера и ограничим точность до 300 м на 300 м перед использованием. Это необходимо, так как нет необходимости определять расположение точнее, чем здание для производительности сети. 

### <a name="javascript-tests"></a>Тесты JavaScript

После идентификации местоположения офиса мы запускаем тест задержки TCP в JavaScript и запрашиваем у службы данные о используемом и рекомендуемых серверах office 365 service front door. По их завершению мы покажем их на карте и на вкладке сведений, где их можно просмотреть до следующего шага.

### <a name="download-the-advanced-tests-client-application"></a>Скачивание клиентского приложения расширенных тестов

Далее мы начнем скачивание клиентского приложения расширенных тестов. Мы надеемся, что пользователь запустит клиентские приложения, и у них также должен быть установлен .NET Core.

Проверка сетевого подключения Microsoft 365 имеет две части. веб-сайт и загружаемые клиентские приложения Windows, которые <https://connectivity.office.com> тестирует расширенные сетевые подключения. Большинство тестов требуют запуска приложения. При этом результаты будут заполнены на веб-странице по мере ее работы.

Вам будет предложено скачать приложение advanced client test с веб-сайта после завершения тестов веб-браузера. Откройте и запустите файл при запросе.

![Клиентские приложения расширенных тестов](../media/m365-mac-perf/m365-mac-perf-open-run-file.png)

### <a name="start-the-advanced-tests-client-application"></a>Запуск клиентского приложения расширенных тестов

После запуска клиентского приложения веб-страница обновят, чтобы отдемонстрировать ее, и тестовые данные начнут быть получены на веб-страницу. Он обновляется при каждом получении новых данных, и вы можете просмотреть их по мере их поступления.

### <a name="advanced-tests-completed-and-test-report-upload"></a>Расширенные тесты завершены и отправка отчета

После завершения тестов веб-страница и клиент расширенных тестов будет указывать на это, и если пользователь вошел в тестовый отчет, он будет загружен в клиент клиента.

### <a name="your-location-information"></a>Сведения о расположении

В этом разделе показаны результаты тестирования, связанные с вашим расположением.

#### <a name="your-location"></a>Расположение

Расположение пользователя определяется из веб-браузера пользователей или может ввестися по выбору пользователя. Он используется для определения расстояния между сетями и определенными частями периметра корпоративной сети. В отчете сохраняются только город от этого местоположения и расстояние до других точек сети.

Расположение офиса пользователя отображается в представлении карты.

#### <a name="network-egress-location-the-location-where-your-network-connects-to-your-isp"></a>Сетевой эксгрисс (расположение, в котором сеть подключается к вашему интернет-президенту)

Мы определяем IP-адрес сетевого экскурса на стороне сервера. Базы данных расположений используются для поиска приблизительного расположения для сетевого экскурса. Эти базы данных обычно имеют точность около 90 % IP-адресов. Если расположение с IP-адреса сетевого адреса не является точным, это приведет к ложному результату этого теста. Чтобы проверить, произошла ли эта ошибка для определенного IP-адреса, можно использовать общедоступные веб-сайты расположения IP-адресов сети для сравнения с фактическим расположением.

#### <a name="your-distance-from-the-network-egress-location"></a>Расстояние от расположения сетевого экскурса

Мы определяем расстояние от этого расположения до расположения офиса. Это показывается как сетевое представление, если расстояние превышает **500** метров (800 метров), так как это, скорее всего, увеличит задержку TCP более чем на 25 мс и может повлиять на работу пользователей.

Расположение сетевого экскурса отображается в представлении карты и подключено к расположению офиса пользователя, указывающее внутренний маршрут сети в корпоративной сети WAN.

Для подключения к сети Microsoft 365 рекомендуется реализовать локальный и прямой сетевой доступ из офисов пользователей в Интернет. Улучшение локального и прямого выходов — лучший способ решения этой сетевой проблемы.

#### <a name="proxy-server-information"></a>Сведения о прокси-сервере

Мы определяем прокси-серверы, настроенные на локальном компьютере. Мы определяем, настроены ли какие-либо из них в сетевом пути для оптимизации сетевого трафика Microsoft 365 категории. Мы определяем расстояние от расположения офиса пользователя до прокси-серверов. Сначала тестировать расстояние с помощью проверки связи ICMP, а в случае неудачи мы тестим TCP-проверку и, наконец, в случае сбойного поиска IP-адреса прокси-сервера в базе данных расположений IP-адресов. Мы покажем сетевую информацию, если прокси-сервер находится на расстоянии **более 500** метров (800 метров) от расположения офиса пользователя.

#### <a name="virtual-private-network-vpn-you-use-to-connect-to-your-organization"></a>Виртуальная частная сеть (VPN), используемая для подключения к организации

Это определяет, используете ли вы VPN для подключения к Office 365. Если у вас нет VPN или у вас есть VPN с рекомендуемой конфигурацией раздельного туннеля для Office 365, то будет выявить результат передачи.

#### <a name="vpn-split-tunnel"></a>VPN Split Tunnel

Каждый оптимизированый маршрут категорий для Exchange Online, SharePoint Online и Microsoft Teams тестирован на туннелирован по VPN или нет. Разделенная рабочая нагрузка полностью избегает VPN. Туннелная рабочая нагрузка отправляется по VPN. Выборочная туннелемная рабочая нагрузка имеет некоторые маршруты, переданные по VPN, а некоторые разделены. В результате передачи будет видно, разделены ли все рабочие нагрузки или выборочный туннель.

#### <a name="customers-in-your-metropolitan-area-with-better-performance"></a>Клиенты в вашем регионе с более производительностью

Задержка TCP в сетевом расположении офиса пользователя перед входной записью службы Exchange Online сравнивается с другими клиентами Microsoft 365 в той же области. Если 10 % или более клиентов в одной области имеют более 10 % клиентов в одной области, отображается представление о сети. Это означает, что их пользователи будут работать лучше в пользовательском интерфейсе Microsoft 365.

Эта информация о сети создается на основе того, что все пользователи в городе имеют доступ к одной телекоммуникационной инфраструктуре и одинаковой близости к интернет-схемам и сети Майкрософт.

#### <a name="time-to-make-a-dns-request-on-your-network"></a>Время для запроса DNS в сети

Здесь показан DNS-сервер, настроенный на клиентский компьютер, на который были даны тесты. Это может быть DNS-сервер рекурсивного разрешения, но это редко. С большей вероятностью это сервер DNS-сервера, который кэшет результаты DNS и перенаночет любые некваченные DNS-запросы на другой DNS-сервер.

Эта информация предоставляется только для сведений и не способствует анализу сети.

#### <a name="your-distance-from-andor-time-to-connect-to-a-dns-recursive-resolver"></a>Расстояние от рекурсивного разрешения DNS и/или время для подключения к рекурсивной разрешанию DNS

In-use DNS Recursive Resolver is identified by making a specific DNS request and then asking the DNS Name Server for the IP Address that it received the same request from. Этот IP-адрес является рекурсивным разрешением DNS и будет искаться в базах данных расположений IP-адресов для поиска расположения. Затем вычисляется расстояние от расположения офиса пользователя к расположению сервера рекурсивного разрешения DNS. Это показывается как сетевое представление, если расстояние превышает **500 метров** (800 метров).

Возможно, расположение с IP-адреса сетевого источна, и это приведет к ложному результату этого теста. Чтобы проверить, произошла ли эта ошибка для определенного IP-адреса, можно использовать общедоступные веб-сайты расположения IP-адресов сети.

Эта сетевая информация будет специально влиять на выбор front door службы Exchange Online. Чтобы устранить эту ситуацию, локальный и прямой сетевой эксгресс должен быть предварительным условием, а затем рекурсивный разрешительный DNS-адрес должен быть расположен близко к этому сетевому выпадаю.

### <a name="exchange-online"></a>Exchange Online

В этом разделе показаны результаты тестирования, связанные с Exchange Online.

#### <a name="exchange-service-front-door-location"></a>Расположение переднего входа службы Exchange

The in-use Exchange service front door is identified in the same way that Outlook does this and we measure the network TCP latency from the user location to it. Показана задержка TCP, а используемая в службе Exchange передняя часть сравнивается со списком лучших переднего входа службы для текущего расположения. Это показывается как сетевое представление, если не используется одна из лучших входных то или иных служб Exchange.

Если не использовать одну из лучших входных ток служб Exchange, это может быть вызвано выходом сети из сети перед выходом корпоративной сети, в этом случае рекомендуется локальный и прямой выход из сети. Это также может быть вызвано использованием удаленного сервера рекурсивного разрешения DNS, в этом случае рекомендуется привести сервер рекурсивного разрешения DNS в соответствие с сетевым перегресом.

Мы вычисляем потенциальное улучшение задержки TCP (мс) до входной открыть службу Exchange. Это делается путем проверки задержки сети расположения офиса пользователя и вычитания задержки сети из текущего расположения в переднюю линию службы Exchange. Разница представляет потенциальную возможность для улучшения.

### <a name="sharepoint-online"></a>SharePoint Online

В этом разделе показаны результаты тестирования, связанные с SharePoint Online и OneDrive.

#### <a name="the-service-front-door-location"></a>Расположение службы переднего входа

Используемая клиентская служба SharePoint идентифицирована так же, как и клиент OneDrive, и мы измеряем задержку TCP в сети от расположения офиса пользователя к ней.

#### <a name="download-speed"></a>Скорость скачивания

Мы измеряем скорость скачивания файла размером 15 МБ из переднего входа службы SharePoint. Результат отображается в мегабайтах в секунду, чтобы указать размер файла в мегабайтах, который можно загрузить из SharePoint или OneDrive за **одну секунду.** Это значение должно быть примерно таким же, как одна десятая часть минимальной пропускной способности канала в мегабитах в секунду. Например, если у вас есть подключение к Интернету со 100 Мбит/с, можно ожидать 10 мегабайт в секунду (10 Мбит/с).

#### <a name="buffer-bloat"></a>Буферное ветвь

Во время скачивания размером 15 МБ мы измеряем задержку TCP до входной точки службы SharePoint. Это задержка при загрузке, которая сравнивается с задержкой, когда она не загружена. Увеличение задержки при загрузке под нагрузкой часто происходит из-за загрузки буферов сетевых устройств потребителей (или их раздува). Для любого 1000 или более в сети отображается представление о сети.

#### <a name="service-front-door-recorded-in-the-client-dns"></a>Входная запись службы в клиентской службе DNS

Здесь показано DNS-имя и IP-адрес сервера переднего входа службы SharePoint, на который вы были направлены. Она предоставляется только для сведений и не имеет связанной сетевой информации.

### <a name="microsoft-teams"></a>Microsoft Teams

#### <a name="media-connectivity-audio-video-and-application-sharing"></a>Возможность подключения к мультимедиа (аудио, видео и общий доступ к приложениям)

#### <a name="connectivity"></a>Подключение

Мы проверяем возможность подключения HTTP из расположения офиса пользователя во все необходимые конечные точки сети Microsoft 365. Они опубликованы на сайте [https://aka.ms/o365ip](https://aka.ms/o365ip) . Отображается представление о сети для всех необходимых конечных точек сети, к которым невозможно подключаться.

Подключение может быть заблокировано прокси-сервером, брандмауэром или другим устройством сетевой безопасности в периметре корпоративной сети. Подключение к TCP-порту 80 тестироваться с помощью HTTP-запроса, а подключение к TCP-порту 443 — с помощью HTTPS-запроса. Если ответа нет, то FQDN помечается как сбой. Если имеется код http-ответа 407, то FQDN помечается как сбой. Если имеется код http-ответа 403, мы проверяем атрибут Server отклика и, если он кажется прокси-сервером, мы пометим это как сбой. Вы можете имитировать тесты, выполняемые с помощью средства командной строки Windows curl.exe.

Мы тестируем SSL-сертификат на каждой необходимой конечной точке сети Microsoft 365, которая находится в категории оптимизации или разрешить, как определено в [https://aka.ms/o365ip](https://aka.ms/o365ip) . Если какие-либо тесты не находят сертификат Microsoft SSL, зашифрованная сеть должна быть перехвачена промежуточным сетевым устройством. Анализ сети отображается на всех перехваченных зашифрованных конечных точках сети.

При обнаружении SSL-сертификата, не предоставленного корпорацией Майкрософт, мы откажемся в качестве FQDN для теста и используемого владельца SSL-сертификата. Этот владелец SSL-сертификата может быть поставщиком прокси-сервера или самозаверять корпоративный сертификат.

## <a name="faq"></a>Вопросы и ответы

Ниже даны ответы на некоторые из наших часто задамого вопросов.

### <a name="is-this-tool-released-and-supported-by-microsoft"></a>Выпущено ли это средство и поддерживается корпорацией Майкрософт?

В настоящее время это предварительная версия, и мы планируем регулярно предоставлять обновления, пока не достигнете общего состояния выпуска доступности при поддержке корпорации Майкрософт. Пожалуйста, сообщите нам об этом. Мы планируем опубликовать более подробное руководство по подсети Office 365 в рамках этого средства, которое настраивается для организации по результатам тестирования.

### <a name="what-is-required-to-run-the-advanced-test-client"></a>Что требуется для запуска клиента advanced test?

Для клиента расширенных тестов требуется .NET Core 3.1 Desktop Runtime. Если запустить расширенный тестовый клиент без этого установленного, вы будете перенаправляем на страницу установщика [.NET Core 3.1.](https://dotnet.microsoft.com/download/dotnet-core/3.1) Убедитесь, что на странице устанавливается не SDK, а ASP.NET windows Core Runtime. Для установки .NET Core требуются разрешения администратора на компьютере.

### <a name="what-is-microsoft-365-service-front-door"></a>Что такое вход в службу Microsoft 365?

Входная точка службы Microsoft 365 — это точка входа в глобальную сеть Майкрософт, в которой клиенты и службы Office прерывают сетевое подключение. Для оптимального сетевого подключения к Microsoft 365 рекомендуется прерывать сетевое подключение до ближайшего входа Microsoft 365 в вашем городе или в другом городе.

Примечание. Клиентский доступ к службе Microsoft 365 не имеет прямой связи с продуктом **Azure Front Door Service,** доступным в Azure Marketplace.

### <a name="what-is-the-best-microsoft-365-service-front-door"></a>Каков лучший вход в службу Microsoft 365?

Лучший вход в службу Microsoft 365 (прежнее название — оптимальная входная точка службы) — это тот, который находится ближе всего к выходу вашей сети, как правило, в вашем городе или области. Используйте средство обеспечения производительности сети Microsoft 365, чтобы определить расположение используемой службы Microsoft 365 переднего входа и самых лучших входных то есть входных данных службы. Если средство определяет, что используемая у вас передняя часть является одной из лучших, следует ожидать отличного подключения к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение для интернет-эксгримации?

Расположение выхода из Интернета — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это расположение также определено как расположение, в котором имеется устройство NAT и обычно используется подключение к поставщику услуг Интернета (ISP). Если между вашим расположением и расположением интернет-игреса достаточно далеко, это может идентифицировать значительную часть заднего пути WAN.
