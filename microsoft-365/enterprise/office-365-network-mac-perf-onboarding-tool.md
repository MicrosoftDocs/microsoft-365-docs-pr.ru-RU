# <a name="microsoft-365-network-connectivity-test-tool-preview"></a>Средство проверки сетевого подключения Microsoft 365 (Предварительная версия)

Средство тестирования сетевых подключений (Майкрософт) 365 находится по адресу <https://connectivity.office.com> . Это средство аджункт для оценки сети и сведений о сети, доступных в центре администрирования Microsoft 365 в области **работоспособность | Меню подключения** .

![Средство проверки подключения](../media/m365-mac-perf/m365-mac-perf-test-tool-page.png)

>[!NOTE]
>Средство проверки сетевого подключения поддерживает клиентов в WW и Германии, но не на GCC умеренный, GCC High, DoD или Китае.

Сведения о сети в центре администрирования Microsoft 365 основаны на регулярных измерениях для вашего клиента Microsoft 365, которые ежемесячно собраны. В сравнении с проверкой сетевых подключений Microsoft 365 выполняется локально и один раз в средстве. Тестирование, которое можно выполнить в продукте, ограничено, и при выполнении тестов, локальных для пользователя, могут быть собраны сведения, полученные в результате более глубокого анализа. Обратите внимание на то, что в центре администрирования Microsoft 365 есть проблема с сетью для использования Microsoft 365 в определенном расположении Office. Проверка возможности подключения к Microsoft 365 поможет определить основную причину проблемы, которая привела к рекомендованному действию по улучшению производительности сети.

Рекомендуется использовать их вместе с тем, где состояние качества сети можно оценить для каждого расположения Office в центре администрирования Microsoft 365, а дополнительные сведения можно найти после развертывания тестирования на основе проверки подключения Microsoft 365.

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, зарегистрированных в программе предварительного просмотра компонентов.

## <a name="what-happens-at-each-test-step"></a>Что происходит на каждом шаге теста

### <a name="office-location-identification"></a>Идентификация местоположения Office

Когда вы нажимайте кнопку Run Test (выполнить тест), на которой отображается выполняемая тестовая страница, и определите расположение Office. Вы можете ввести свое местоположение по городу, региону и стране или в веб-браузере. Если вы обнаружите его, вы запрашиваете широту и долготу в веб-браузере и ограничиваете точность 300m до 300m перед использованием. Это делается потому, что нет необходимости определять расположение более точно, чем здание для производительности сети. 

### <a name="javascript-tests"></a>Проверки JavaScript

После идентификации местоположения Office мы выполняем тест задержки TCP в JavaScript и запросили данные из службы об используемых и рекомендуемых серверах передней дверцы службы Office 365. По завершении они отображаются на карте и на вкладке сведения, где их можно просмотреть перед следующим шагом.

### <a name="download-the-advanced-tests-client-application"></a>Скачайте клиентское приложение расширенных тестов

Далее мы начнем загрузку клиентского приложения с расширенными тестами. Мы используем пользователь для запуска клиентского приложения, а также должны быть установлены компоненты .NET Core.

Проверка сетевого подключения Microsoft 365 состоит из двух частей; веб-сайт <https://connectivity.office.com> и загружаемое клиентское приложение Windows, в котором выполняются расширенные проверки сетевого подключения. Большинство тестов требуют запуска приложения. Они заполняют результаты обратно на веб-странице в ходе ее выполнения.

После завершения тестирования веб-браузера вам будет предложено скачать тестовое приложение расширенного клиента с веб-сайта. Откройте и запустите файл при появлении соответствующего запроса.

![Клиентское приложение расширенных тестов](../media/m365-mac-perf/m365-mac-perf-open-run-file.png)

### <a name="start-the-advanced-tests-client-application"></a>Запуск клиентского приложения расширенных тестов

После запуска клиентского приложения веб-страница будет обновлена, чтобы показать, что эти и тестовые данные будут запущены на веб-странице. Он обновляется каждый раз, когда получаются новые данные, и вы можете просмотреть данные по мере их поступления.

### <a name="advanced-tests-completed-and-test-report-upload"></a>Расширенные тесты завершены и тест отправки отчета

После завершения тестирования веб-страница и клиент расширенных тестов будут указывать этот параметр и, если пользователь вошел в тестовый отчет, будут отправлены в клиент клиентов.

### <a name="your-location-information"></a>Сведения о расположении

В этом разделе показаны результаты тестирования, связанные с вашим расположением.

#### <a name="your-location"></a>Ваше расположение

Расположение пользователя определяется в веб-браузере пользователей, или его можно ввести в варианте пользователи. Он используется для идентификации расстояний по сети для определенных частей периметра сети предприятия. В отчете сохраняются только город из этого определения расположения и расстояние до других точек сети.

Расположение пользователя в офисе отображается в представлении "карта".

#### <a name="network-egress-location-the-location-where-your-network-connects-to-your-isp"></a>Расположение сетевого выхода (расположение, в которое сеть подключается к поставщику услуг Интернета)

Мы идентифицирующее IP-адрес выхода сети на стороне сервера. Базы данных расположений используются для поиска приближенного места для выхода в сеть. Как правило, эти базы данных имеют точность около 90% IP-адресов. Если расположение с IP-адресом выхода из сети неточнее, это приведет к ложному результату проверки. Чтобы проверить, возникает ли эта ошибка по определенному IP-адресу, вы можете использовать общедоступные веб-сайты местоположения сетевых IP-адресов для сравнения с фактическим расположением.

#### <a name="your-distance-from-the-network-egress-location"></a>Ваше расстояние от расположения сетевого выхода

Мы определим расстояние от этого расположения до расположения Office. Это отображается как анализ сети, если расстояние больше **500 миль** (800 километров), так как это может привести к увеличению задержки TCP на более чем 25ms и повлиять на взаимодействие с пользователем.

Расположение сетевого выхода отображается в представлении карты и подключается к расположению пользователя, которое указывает, что в корпоративной глобальной сети выполняется обратной связи.

Реализация локального и прямого сетевого выхода из мест в офисах пользователей в Интернете рекомендуется для подключения к сети Microsoft 365. Улучшения локального и прямого выхода — лучший способ устранить эту информацию об этой сети.

#### <a name="proxy-server-information"></a>Сведения о прокси-сервере

Мы выявление прокси-серверов, настроенных на локальном компьютере. Мы вычислим, какие из них настроены в сетевом пути для категории оптимизировать сетевой трафик Microsoft 365. Мы определяем расстояние от местонахождения пользователя в офисе до прокси-серверов. Расстояние проверяется сначала с помощью ICMP ping и в случае неудачного протеста с помощью протокола TCP ping и, если это не удается выполнить поиск IP-адреса прокси-сервера в базе данных расположения IP-адресов. Мы видим, что если прокси-сервер больше **500 миль** (800 километров) из расположения для пользователя, мы видим, что в сети есть информация о сети.

#### <a name="virtual-private-network-vpn-you-use-to-connect-to-your-organization"></a>Виртуальная частная сеть (VPN), используемая для подключения к Организации

Это определяет, используется ли VPN для подключения к Office 365. Передаваемый результат будет отображаться, если у вас нет VPN или у вас есть VPN с рекомендуемой конфигурацией расщепленных туннелей для Office 365.

#### <a name="vpn-split-tunnel"></a>Разделенный туннель VPN

Каждый процесс оптимизации маршрута категорий для Exchange Online, SharePoint Online и Microsoft Teams тестируется, чтобы проверить, не является ли он туннельным по отношению к VPN. При разделении рабочей нагрузки виртуальную сеть полностью обходится. Туннельная нагрузка передается по сети VPN. Для выборочной нисходящей нагрузки некоторые маршруты передаются по сети VPN и частично разбиваются. Передаваемый результат будет отображаться, если все рабочие нагрузки будут разбиты или выборочно туннелны.

#### <a name="customers-in-your-metropolitan-area-with-better-performance"></a>Клиенты в районе городов с более высокой производительностью

Задержка TCP в сети для местонахождения пользователя в передней дверце службы Exchange Online сравнивается с другими клиентами Microsoft 365 в той же области Metro. Обзор сети отображается, если 10% или более клиентов в одной области Metro имеют лучшую производительность. Это означает, что их пользователи имеют лучшую производительность в пользовательском интерфейсе Microsoft 365.

Эта информация о сети создается на основе того, что все пользователи в городе имеют доступ к одной и той же инфраструктуре связи, а также к Интернет-цепям и сети корпорации Майкрософт.

#### <a name="time-to-make-a-dns-request-on-your-network"></a>Время на создание DNS-запроса в сети

В этом примере показан DNS-сервер, настроенный на клиентском компьютере, на котором выполнялись тесты. Это может быть рекурсивный сервер распознавателя DNS, но это нераспространенный. Скорее всего, это сервер пересылки DNS, который кэширует результаты DNS и пересылает все некэшированные DNS-запросы на другой DNS-сервер.

Он предоставляется только для информации и не участвует в недоступности сети.

#### <a name="your-distance-from-andor-time-to-connect-to-a-dns-recursive-resolver"></a>Ваше расстояние от или до времени для подключения к рекурсивному распознавателю DNS

Используемый рекурсивный арбитр DNS идентифицируется путем создания определенного DNS-запроса и запроса DNS-сервера DNS для IP-адреса, полученного от этого запроса. Этот IP-адрес является рекурсивным распознавателем DNS, и он будет искаться в базах данных расположения IP-адресов, чтобы найти расположение. После этого рассчитывается расстояние от местонахождения пользователя в офисе до сервера DNS с рекурсивным распознавательм. Это отображается в виде анализа сети, если расстояние больше **500 миль** (800 километров).

Адрес, проходящий из IP-адреса выхода сети, может быть неточным, и это может привести к ложному результату проверки. Чтобы проверить, возникает ли эта ошибка по определенному IP-адресу, вы можете использовать общедоступные веб-сайты расположения сетевых IP-адресов.

Эта информация о сети будет оказывать влияние на выбор передней дверцы службы Exchange Online. Чтобы устранить эту проблему, необходимо быть предварительным условием, а затем рекурсивным распознавателем DNS, который должен находиться ближе к этой сети.

### <a name="exchange-online"></a>Exchange Online

В этом разделе показаны результаты тестирования, связанные с Exchange Online.

#### <a name="exchange-service-front-door-location"></a>Расположение передней дверцы службы Exchange

Используемая Передняя дверца службы Exchange идентифицируется так же, как и в Outlook, и мы измеряем задержку TCP в сети от расположения пользователя до этого. Задержка TCP отображается, а используемая в ней Передняя дверца службы Exchange сравнивается со списком лучших дверей для текущего расположения. Это отображается как анализ сети, если одна из лучших дверей (-ов) службы Exchange не используется.

Использование одной из лучших дверей (s) службы Exchange может быть вызвано обратной связи сети до выхода корпоративной сети, в которой мы рекомендуем использовать локальную и прямую сеть. Кроме того, это может быть вызвано использованием удаленного сервера рекурсивного разрешения DNS, в результате чего мы рекомендуем выравнивать рекурсивный сервер сопоставителя DNS с выходом в сеть.

Мы вычисляем возможное улучшение задержки TCP (МС) до передней дверцы службы Exchange. Для этого необходимо протестировать сетевое расположение протестированного пользователя в сети и прочесть задержку сети из текущего расположения в переднюю дверцу службы Exchange. Разница представляет потенциальную возможность для улучшения.

#### <a name="best-exchange-service-front-doors-for-your-location"></a>Лучшие передние двери службы Exchange для вашего расположения

В этом списке представлены наиболее подходящие расположения передней дверцы службы Exchange по городу для своего расположения.

#### <a name="service-front-door-recorded-in-the-client-dns"></a>Передняя дверца службы, записанная в DNS-клиенте

Здесь отображается DNS-имя и IP-адрес сервера переднего плана службы Exchange, на который вы нанаправлялись. Он предоставляется только для информации и не имеет связанного анализа сети.

### <a name="sharepoint-online"></a>SharePoint Online

В этом разделе показаны результаты тестирования, связанные с SharePoint Online и OneDrive.

#### <a name="the-service-front-door-location"></a>Положение передней дверцы службы

Передняя дверца службы SharePoint используется так же, как и клиент OneDrive, и она измеряет задержку TCP в сети от расположения пользователя в офисе.

#### <a name="download-speed"></a>Скорость скачивания

Мы измеряем скорость загрузки файла 15Mb из передней дверцы службы SharePoint. Результат отображается в мегабайтах в секунду, чтобы указать, какой размер файла в мегабайтах можно скачать из SharePoint или OneDrive, за **одну секунду**. Число должно быть приблизительно равно одной десятой минимальной пропускной способности в мегабит в секунду. Например, если у вас есть 100 Мбит/с, то вы можете ожидать 10 мегабайт в секунду (10 Мбит/с).

#### <a name="buffer-bloat"></a>Чрезмерное помещает буфер

Во время загрузки 15Mb мы измеряем задержку TCP на переднюю дверцу службы SharePoint. Это задержка при нагрузке, и она сравнивается с задержкой при загрузке. Частое увеличение задержек при загрузке при загрузке часто соотнесено на загружаемые и загруженные буферы сетевых устройств потребителя. Обзор сети отображается для любого чрезмерного числа 1 000 или более.

## <a name="faq"></a>Вопросы и ответы

Здесь приведены ответы на некоторые часто задаваемые вопросы.

### <a name="is-this-tool-released-and-supported-by-microsoft"></a>Это средство выпущено и поддерживается корпорацией Майкрософт?

В настоящее время это предварительная версия, и мы планируем обеспечить регулярное предоставление обновлений, пока не будет достигнут общий статус выпуска от Майкрософт. Отправьте свои отзывы, чтобы помочь нам улучшить. Мы планируем опубликовать более подробное руководство по подключению сети Office 365 в рамках этого средства, которое настроено для Организации по результатам тестирования.

### <a name="what-is-required-to-run-the-advanced-test-client"></a>Что требуется для запуска клиента расширенной проверки?

Клиент расширенной проверки требует среды выполнения .NET Core 3,1 для настольных ПК. Если вы запускаете расширенный тестовый клиент без установки, вы будете перенаправлены на [страницу установщика .NET Core 3,1](https://dotnet.microsoft.com/download/dotnet-core/3.1). Обязательно установите среду выполнения для настольных ПК, а не пакет SDK или основную среду выполнения ASP.NET, которая выше на странице. Для установки .NET Core требуется разрешение администратора на компьютере.

### <a name="what-is-microsoft-365-service-front-door"></a>Что такое Передняя дверца службы Microsoft 365?

Передняя дверца службы Microsoft 365 — это точка входа в глобальной сети Майкрософт, в которой клиенты и службы Office завершают свое сетевое подключение. Для оптимального сетевого подключения к Microsoft 365 рекомендуется завершить сетевое подключение к ближайшему передней дверце Microsoft 365 в городе или Metro.

Note: у передней дверцы службы Microsoft 365 нет прямого отношения к продукту **службы Front дверь Azure** , доступному в Azure Marketplace.

### <a name="what-is-the-best-microsoft-365-service-front-door"></a>Какова лучшая Передняя дверца службы Microsoft 365?

Лучшей дверце службы Microsoft 365 (прежнее название — оптимальная дверца службы) — это наиболее близкий к выходу из сети, как правило, в области город или Metro. Используйте средство Microsoft 365 Network Performance Tool, чтобы определить расположение вашей передней дверцы службы Microsoft 365 и лучших дверей (s). Если вы определили, что используемая Передняя дверца является одной из лучших, то вы должны ожидать, что в глобальную сеть корпорации Майкрософт вы будете иметь доступ к глобальной сети.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение для выхода в Интернет?

Расположение для выхода в Интернет — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это также определяется как расположение, в котором у вас есть устройство преобразования сетевых адресов (NAT) и который обычно используется для подключения к поставщику услуг Интернета. Если вы видите большое расстояние между местоположением и расположением для выхода в Интернет, это может привести к значительной обратной связи глобальной сети.
