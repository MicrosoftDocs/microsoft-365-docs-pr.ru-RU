# <a name="deploy-sharepoint-and-onedrive-for-microsoft-365-enterprise"></a>Развертывание SharePoint и OneDrive для Microsoft 365 корпоративный

*Эта рабочая нагрузка включена в планы E3 и E5 Microsoft 365 корпоративный.*

SharePoint и Microsoft Teams служат для хранения файлов и предоставления общего доступа к ним, управления содержимым и взаимодействия. Они являются основными элементами Microsoft 365 корпоративный для командной работы. 

В SharePoint предусмотрены расширенные возможности для обеспечения безопасности, включая управление доступом с разрешениями и шифрованием хранящихся и передаваемых данных. Система безопасности SharePoint является основной составляющей Microsoft 365 корпоративный для обеспечения интеллектуальной безопасности.

Если вы не знакомы с SharePoint, см. статьи [SharePoint](https://products.office.com/sharepoint/collaboration) и [Начало работы с SharePoint](https://support.office.com/article/video-what-is-sharepoint-online-c17b6824-cc22-478f-8757-497cc6b57121).

Пройдя описанные ниже этапы и выполнив указанные шаги, вы определите роль SharePoint в организации, подключите организацию путем проведения поэтапных развертываний и предоставите возможность использовать SharePoint. Прежде чем начать, настройте правильные этапы [базовой инфраструктуры](deploy-workloads.md#foundation-infrastructure-prerequisites), чтобы на сайтах SharePoint были доступны нужные функции безопасности. 

Инструкции по развертыванию OneDrive для Microsoft 365 корпоративный см. в [руководстве по OneDrive для предприятий](https://docs.microsoft.com/onedrive/plan-onedrive-enterprise).

## <a name="phase-1-envision"></a>Этап 1. Определение
На этом этапе вы соберете партнеров организации для развертывания SharePoint и определите, как ваша организация будет использовать SharePoint для удовлетворения бизнес-потребностей.

### <a name="step-1-gather-your-sharepoint-deployment-members"></a>Шаг 1. Выбор участников развертывания SharePoint

Для успешного развертывания SharePoint в [базовой инфраструктуре Microsoft 365 корпоративный](deploy-foundation-infrastructure.md) вам необходимо правильно подобрать сотрудников для его проведения и получения обратной связи. Среди таких сотрудников должны быть лица, принимающие бизнес-решения, ИТ-специалисты (например, архитекторы и исполнители действий), а также представители пользователей. 

Эти три группы позволят учесть при развертывании бизнес-потребности, технические аспекты переноса и защиты папок и документов, а также потребности типичных пользователей.

#### <a name="result"></a>Результат

Список людей, позволяющий учесть бизнес-потребности, технические особенности, а также интересы пользователей в организации.

### <a name="step-2-determine-and-prioritize-your-sharepoint-business-scenarios"></a>Шаг 2. Определение бизнес-сценариев и приоритетов для SharePoint

SharePoint можно использовать для разных целей. Вам нужно определить, что подходит для вашей компании. Обеспечьте возможность хранения документов и предоставления доступа к ним, управления содержимым, и совместной работы с помощью SharePoint для команд, отделов или всей организации. 

См. список сценариев и возможностей в статье [SharePoint](https://products.office.com/sharepoint/collaboration ).

Указанные ниже бизнес-принципы помогут удовлетворить потребности организации.

|||
|:-----|:-----|
| Общий доступ и совместная работа | Использование сайтов групп, информационных сайтов и синхронизации. |
| Оповещение и привлечение | Данные, поступающие в дальнейшем. |
| Преобразование | Использование Flow для создания автоматических рабочих процессов между приложениями и службами. |
| Использование коллективных знаний | Использование службы поиска для предоставления нужных результатов в организации. |
| Защита | Обеспечение защиты данных и соответствия требованиям в организации. |
|||

Сведения о настройке SharePoint под ваши потребности см. в статье [Администратор SharePoint](https://docs.microsoft.com/sharepoint/sharepoint-online).

Один из способов оценить преимущества SharePoint — понаблюдать за взаимодействием на уровне отдельных пользователей, команд, отделов или всей организации, а затем выбрать сценарий, обеспечивающий более простые способы хранения файлов и предоставления общего доступа к ним. Учтите, что [Microsoft Teams](teams-workload.md) может больше подходить для некоторых сценариев.

#### <a name="result"></a>Результат
Список сценариев SharePoint, в которых учтены потребности вашей организации в отношении хранения документов, предоставления общего доступа к ним, управления содержимым, совместной работы и безопасности.

## <a name="phase-2-onboard"></a>Этап 2. Адаптация

На этом этапе вы спланируете технические аспекты развертывания SharePoint и начнете их развертывание для отдельных групп пользователей.

### <a name="prerequisites-identity-and-device-access-configuration"></a>Необходимые условия: конфигурация доступа для удостоверений и устройств

Чтобы защитить доступ к сайтам SharePoint, убедитесь, что вы настроили [политики доступа для удостоверений и устройств](identity-access-policies.md), а также [рекомендуемые политики доступа к SharePoint](sharepoint-file-access-policies.md).

### <a name="step-1-complete-your-technical-planning"></a>Шаг 1. Завершение технического планирования

Прежде чем начать техническое планирование, решите, требуется ли вам FastTrack. Если в вашей организации более 50 рабочих мест и используется [поддерживаемый план](https://docs.microsoft.com/fasttrack/O365-fasttrack-benefit-for-office-365), вам доступны бесплатные преимущества FastTrack. Они помогут выполнить планирование, миграцию, развертывание и внедрение служб. Вы также можете решить эти задачи самостоятельно, используя наши мастера по подключению FastTrack. Чтобы получить их, войдите на сайт [FastTrack](https://docs.microsoft.com/fasttrack/m365-fasttrack-benefit-overview) под своей учетной записью Microsoft 365.

Если вы самостоятельно выполняете планирование (или совместно с FastTrack), вам потребуется определить готовность вашей сети и организации к использованию SharePoint. Очень важно выполнение [условий готовности для сети](networking-exit-criteria.md) в базовой инфраструктуре, при котором особое внимание уделяется полосе пропускания для Интернета и задержкам трафика. Это необходимо, чтобы достичь максимальной производительности дополнительного трафика для документов SharePoint.

Чтобы подготовиться к развертыванию, см. статью [Миграция в SharePoint](https://docs.microsoft.com/sharepointmigration/migrate-to-sharepoint-online). 

Ознакомьтесь со следующими ресурсами, чтобы лучше понять принципы обеспечения безопасности в SharePoint:

-     [Как SharePoint и OneDrive защищают облачные данные](https://docs.microsoft.com/sharepoint/safeguarding-your-data)
-     [Шифрование данных в OneDrive и SharePoint](https://docs.microsoft.com/microsoft-365/compliance/data-encryption-in-odb-and-spo)

#### <a name="result"></a>Результат

Вы понимаете, что требуется для локальных папок, переноса документов, обеспечения безопасности, сайтов SharePoint. Вы готовы начать развертывание SharePoint для выбранных групп в организации.

### <a name="step-2-run-an-it-pilot"></a>Шаг 2. Запуск пилотного ИТ-развертывания

В большинстве средних и крупных организаций следует запустить пилотный ИТ-проект развертывания для заинтересованных лиц, упомянутых на этапе 1, а также для ранних последователей и технических энтузиастов. В процессе такого развертывания выполните указанные ниже действия.

- Выберите бизнес-сценарий SharePoint, на котором могут практиковаться участники пилотного ИТ-проекта развертывания.
- Сообщите пилотным участникам, как проверять совместную работу, хранение документов и общий доступ к ним, а также другие возможности SharePoint.
- Определите свою стратегию управления изменениями и создайте материалы для внедрения SharePoint во всей организации. Материалы по управлению изменениями могут включать текст уведомлений по электронной почте, внутренние планы обучения, плакаты в коридорах и презентации. Эти материалы служат для информирования сотрудников организации о SharePoint и его преимуществах. Ваша цель — повысить осведомленность и стимулировать использование. Чтобы начать работу, ознакомьтесь с сайтом [Ресурсы для внедрения SharePoint](https://resources.techcommunity.microsoft.com/resources/SharePoint-adoption/).
- Попросите участников пилотного ИТ-развертывания изучить материалы о стратегии управления изменениями и оценить их на основе своего опыта. Они могут дать рекомендации о том, как лучше всего описать преимущества SharePoint и как его использовать.

#### <a name="result"></a>Результат

Завершен пилотный ИТ-проект развертывания SharePoint, а также разработаны, проверены и уточнены первоначальные материалы по управлению изменениями.

### <a name="step-3-roll-out-to-a-business-group"></a>Шаг 3. Развертывание в бизнес-группе

Завершив пилотный ИТ-проект, выполните развертывание SharePoint для бизнес-группы или отдела вашей организации. Такое развертывание должно включать указанные ниже элементы.

- Идентификация ключевых бизнес-сценариев для SharePoint в бизнес-группе.
- Выпуск объявлений, информирующих отделы, рабочие или проектные группы об ожиданиях и временных шкалах, которые касаются использования SharePoint.
- Перенос локальных папок и документов участников бизнес-групп в SharePoint.
- Предоставление учебных материалов и ссылок на ресурсы, помогающие ознакомиться с SharePoint и узнать, как его использовать. См. обучающие видео по [SharePoint](https://support.office.com/article/sharepoint-online-video-training-cb8ef501-84db-4427-ac77-ec2009fb8e23).
- Обеспечение механизма обратной связи. Например, создание центральной группы Microsoft Teams, в которую входят все пользователи бизнес-группы, для сбора комментариев и реагирования на сообщения о проблемах от пользователей в бизнес-группе.
 
В процессе развертывания вы можете уточнить свои материалы по управлению изменениями при подготовке к развертыванию в масштабах всей организации.

#### <a name="result"></a>Результат

Создана и работает бизнес-группа, использующая SharePoint. Материалы по управлению изменениями протестированы и уточнены.

## <a name="phase-3-drive-value"></a>Этап 3. Обеспечение использования

На этом этапе завершается развертывание SharePoint, и вы помогаете пользователям оценить его преимущества.

### <a name="step-1-roll-out-to-the-rest-of-your-organization"></a>Шаг 1. Развертывание для остальных сотрудников организации

Развертывание для остальных сотрудников организации должно включать указанные ниже элементы.

- Идентификация ключевых бизнес-сценариев для SharePoint в разных бизнес-группах.
- Использование уточненных материалов по управлению изменениями для выпуска объявлений, информирующих сотрудников в организации об ожиданиях и временных шкалах.
- Перенос папок и документов для остальных сотрудников организации в SharePoint.
- Предоставление учебных материалов и ссылок на ресурсы, помогающие ознакомиться с SharePoint и узнать, как его использовать.
- Обеспечение механизма обратной связи. Например, создание центральной группы, в которую входят все, для сбора комментариев и реагирования на сообщения о проблемах от пользователей в организации. Если в вашей организации меньше 2500 сотрудников, воспользуйтесь общедоступным каналом в Teams. В противном случае используйте общедоступную группу в Yammer.

#### <a name="result"></a>Результат
Ваша организация создана и работает, и вы используете стратегию управления изменениями для информирования пользователей и их обучения работе с SharePoint.

### <a name="step-2-measure-usage-manage-satisfaction-and-drive-adoption"></a>Шаг 2. Измерение показателей использования, управление удовлетворенностью и стимулирование внедрения

После развертывания для всей организации следует продолжить использовать стратегию управления изменениями для указанных ниже целей.

- Руководству следует преподнести SharePoint в качестве основного средства для хранения документов и предоставления общего доступа к ним.
- Стимулируйте сотрудников использовать эту службу для хранения документов и обмена ими в бизнес-группах, отделах, рабочих и проектных группах.

Ниже приведены некоторые рекомендуемые действия.

- Лучшие методики внедрения облачных служб см. в документе [Факторы успеха Microsoft 365](https://aka.ms/successfactors). 
- Сведения об использовании службы в организации см. в статье [Отчеты Microsoft 365 в Центре администрирования](https://docs.microsoft.com/office365/admin/activity-reports/activity-reports). Если вы не являетесь глобальным администратором в своей организации, попросите глобального администратора предоставить вашей учетной записи пользователя разрешения для чтения отчетов, чтобы вы могли получать доступ к отчетам о действиях.
- Отслеживайте канал обратной связи (общедоступный канал в центральной команде Teams или Yammer), чтобы своевременно получать отзывы и сообщения о проблемах от пользователей SharePoint. Как можно быстрее отвечайте на вопросы и помогайте решать проблемы, чтобы не допустить разочарования, и демонстрируйте поддержку развертывания.
- Определите и воспитайте лидеров в каждой бизнес-группе и обращайте внимание на их достижения и рекомендации по использованию SharePoint. Освещайте их успехи в организации, чтобы показать успех проекта и его внедрение. Одобрение со стороны технических лидеров в бизнес-группах может оказать сильное влияние на руководителей и других сотрудников.

#### <a name="result"></a>Результат

Ваша организация внедрила SharePoint в Microsoft 365 корпоративный для хранения документов и совместной работы.
