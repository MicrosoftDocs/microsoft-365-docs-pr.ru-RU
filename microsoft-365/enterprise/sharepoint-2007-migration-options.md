# <a name="sharepoint-2007-migration-options-to-consider"></a>Параметры переноса SharePoint 2007 для рассмотрения

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft SharePoint 2007 и SharePoint Server 2007 достигли конца поддержки. Пришло время обновиться! В этой статье данная статья содержит сведения о вариантах миграции.
  
## <a name="common-upgrade-strategies-for-sharepoint"></a>Общие стратегии обновления для SharePoint

Существует несколько методов обновления среды SharePoint Server. Если у вас есть Microsoft Office SharePoint Server 2007, вот несколько примеров методов обновления:
  
- Обновление с присоединением баз данных
    
- Одностороннее обновление
    
- Обновление на месте
    
- Гибридное обновление (на месте с отсоединяемой базой данных или отдельным присоединением к базе данных)
    
- Гибриды SharePoint (подключение через Интернет к локальной sharePoint)
    
- Вручную перемещать данные между коллекциями сайтов или библиотеками
    
- Обновление мастера FastTrack до Microsoft 365 (советник по развертыванию[SharePoint Online)](https://aka.ms/spoguidance)
    
- API миграции в SharePoint Online (SPO) в Microsoft 365
    
Что лучше всего работает для вас?
  
Ваши знания о том, что ваша ферма делает и используется для является тактической силы, когда дело доходит до обновления. Способ использования фермы SharePoint поможет вам выбрать один из вариантов.
  
> [!TIP]
> Microsoft Office SharePoint Server 2007 также имеет постепенное обновление, не охваченные здесь. Чтобы увидеть список отдельных статей обновления, см. в статье [SharePoint Server 2007 end of support Roadmap.](sharepoint-2007-end-of-support.md) 
  
Не забудьте проверить [жизненный](https://support.microsoft.com/lifecycle/search) цикл продукта и требования к системе для любой версии SharePoint, в который вы обновляете. Это значит, что вы будете знать, когда будет необходимо следующее обновление (например, если вы приостановите на устаревшем продукте, например SharePoint Server 2010, чтобы планировать дополнительные обновления, убедитесь, что вы знаете его конец даты поддержки), и убедитесь, что у вас есть оборудование, которое поддерживает ваш план. 
  
Если вы планируете перейти некоторые или все сайты SharePoint на Microsoft 365 в облаке, настало время закладки ссылки на описания служб [Microsoft 365 и Office 365.](/office365/servicedescriptions/office-365-service-descriptions-technet-library) Вам потребуется описание службы, чтобы узнать о свойствах SharePoint Online и о том, как они могут отличаться от локального SharePoint Server. Обновление функциональных Microsoft Office ферм SharePoint Server 2007. Если в вашей установке есть сайты с ошибками, исправьте их перед обновлением.
  
## <a name="a-note-about-managing-risk"></a>Примечание об управлении рисками

Методы, такие как "бок о бок", важны в схеме логики обновления. При обновлении рядом вы поддерживаете ферму Microsoft Office SharePoint Server 2007, но создайте ферму в следующей версии (SharePoint Server 2010) на новом оборудовании. Это помогает тремя способами:
  
1. У вас есть место для резервного копирования баз данных Microsoft Office SharePoint Server 2007 для их отдельного обновления с помощью приложения базы данных.
    
2. Если вы выясните, что в ферме Microsoft Office SharePoint Server 2007 используется только небольшое количество критически важных библиотек документов и других сведений, можно вручную перенести данные из Microsoft Office SharePoint Server 2007 в SharePoint Server 2010 или перенести в следующую версию только определенные сайты и веб-сайты (что может упростить работу).
    
3. Чем меньше вы делаете для фермы Microsoft Office SharePoint Server 2007, тем безопаснее данные фермы при обновлении.
    
Такие методы, как In-Place обновления, будут действовать непосредственно на ферме SharePoint Server 2007 Microsoft Office SharePoint Server 2007, что даст вам меньше простых вариантов, чтобы отказаться от пути и начать снова с нетронутой средой. Насколько это возможно, создайте некоторые меры безопасности (например, принимая и проверяя резервные копии исходной среды). Например, если ферма Microsoft Office SharePoint Server 2007 является виртуальной и дублируется в целях резервного копирования и восстановления, резервное копирование и восстановление наиболее актуальных баз данных до окна службы для обновления. Зная, что у вас есть возможность восстановления резервных копий баз данных не только даст вам сбой, это может дать вам спокойствие.

## <a name="make-your-plan"></a>Сделайте свой план

Если вам нужно обновить, вам нужен план, и один размер не подходит всем в этих случаях. Ваш план может быть таким же простым, как "Создайте подписку Microsoft 365 с SharePoint Online, зарегистрируйте домен и перенаправляйте людей, чтобы сохранить там свои файлы". И это может быть не так. Это решение за вами, и оно до того, что вам и вашим пользователям действительно нужно.
  
> [!NOTE]
> Рискованно работать с программным обеспечением, жизненный цикл которого завершился. Продукты, которые не поддерживаются, больше не исправлены при обнаружении проблем. Это также означает, что в случае возникновения новых угроз безопасности не будет исправлений или исправлений, так как продукты жизненного цикла не поддерживаются. Пожалуйста, избегайте этой ситуации! 
  
### <a name="first-know-your-farm"></a>Во-первых, знать свою ферму

При обновлении процесс принятия решений должен основываться на том, что ваша ферма делает для организации. Какие потребности он удовлетворяет? Какова его роль? Каждая ферма в вашей компании может иметь свою роль. Некоторые фермы SharePoint могут иметь критическое  *значение,*  некоторые — архивы файлов — для безопасного сохраняемого. Или, если ферма заполняет сразу несколько ролей, может потребоваться знать, какие коллекции сайтов, веб-сайты или даже библиотеки документов делают, какие-либо настройки и насколько они важны. Анализ данных на этом уровне может показаться большим объемом работы, но это экономит время и усилия для освоения домена перед обновлением или миграцией. После того как вы узнаете все движущиеся части и наиболее важные биты, вы также узнаете, что вы переросли и можете оставить позади. Эти знания только прибьют вам идти вперед. 
  
Итак, что, по словам пользователей, наиболее важно для фермы SharePoint Server?
  
- Встроенные функции SharePoint
    
- Большой корпус данных (например, архив файлов)
    
- Доступность
    
- Критически важные приложения, веб-части или документы в ферме (критическая ферма миссии)
    
- Стандарты соответствия требованиям
    
- Настройки
    
Если вы запустите что-то важное для бизнеса из фермы SharePoint, скажем, он действует как большой каталог критически важных данных о требованиях к обслуживанию клиентов, вы можете поставить галочку рядом с "Критическими приложениями", но и "Доступность", то есть, если вы не сможете использовать SharePoint некоторое время. Кроме того, можно проверить параметры "Настройки", так как критически важные службы, которые предлагает ферма, основаны на настраиваемом коде, определениях сайтов или ряде настроек, которые работают вместе.
  
Если SharePoint выполнил эти потребности без необходимости ничего делать за пределами использования встроенного программного обеспечения, и вы обычно обновляете его и проводите нормальное администрирование и обслуживание, возможно, вы выбрали встроенный SharePoint — это также может быть вашей причиной для сидения на более старой версии SharePoint. Другими словами, он уже делает то, что вам нужно, и вам не нужно обновляться до сих пор, в конце Microsoft Office SharePoint Server 2007.
  
При перечне этих вещей создаются критерии для обновления. Другими словами, любое обновление должно соответствовать этому планку. Это позволяет исключить методы, которые в настоящее время не соответствуют вашим потребностям.
  
### <a name="a-simple-sample-plan"></a>Простой пример плана

Возможно, потребуется более широкий консенсус с руководством и другими администраторами по пути, по пути, который будет проходить обновление SharePoint. Администраторы SharePoint Server часто сотрудничают с Microsoft SQL Server администраторами, работают с группами networking и Security и другими. Если заинтересованных сторон много, вам может потребоваться создать соглашение или настроить план обновления и миграции. Например, если вы переносите данные, чтобы часть вашей компании использовала SharePoint Online в Microsoft 365, в сети, скорее всего, потребуется настройка производительности или тестирование. Затронутые группы должны быть проинформированы заранее.
  
В моем простом примере я покажу предложение администратора SharePoint, а затем перечислить план, согласованный всеми заинтересованными сторонами. Чтобы получить ясность, закадрите свои соглашения и решения.
  
План начинается после углубленного анализа фермы и пытается определить роль фермы, точки боли и другие важные сведения, которые приведут к сужжингу некоторых параметров обновления.
