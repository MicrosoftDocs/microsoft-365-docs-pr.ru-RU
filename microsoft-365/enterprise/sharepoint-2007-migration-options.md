# <a name="sharepoint-2007-migration-options-to-consider"></a>Варианты миграции SharePoint 2007, которые следует учитывать

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft SharePoint 2007 и SharePoint Server 2007 достигают конца поддержки. Время на обновление! В этой статье приводятся сведения о параметрах миграции.
  
## <a name="common-upgrade-strategies-for-sharepoint"></a>Общие стратегии обновления для SharePoint

Существует несколько способов обновления среды SharePoint Server. Если у вас есть ферма Microsoft Office SharePoint Server 2007, вот несколько примеров способов обновления:
  
- Обновление с присоединением баз данных
    
- Параллельное обновление
    
- Обновление на месте
    
- Гибридное обновление (на месте с отсоединенными базами данных/отдельное присоединение базы данных)
    
- Гибридные среды SharePoint (подключение к сети в локальной среде SharePoint)
    
- Вручную переместить данные между семействами веб-сайтов или библиотеками
    
- Мастер FastTrack: обновление до Microsoft 365 ([советник по развертыванию SharePoint Online](https://aka.ms/spoguidance))
    
- API миграции в SharePoint Online (SPO) в Microsoft 365
    
Что вам лучше всего подходит вам?
  
Сведения о том, что делает ферма и где она используется, — это сила тактикал при обновлении. Способ использования фермы SharePoint поможет вам выбрать нужные параметры.
  
> [!TIP]
> В Microsoft Office SharePoint Server 2007 также используется постепенное обновление, не описанное здесь. Список статей, посвященных обновлению, можно найти в статье [SharePoint Server 2007 End of support](sharepoint-2007-end-of-support.md). 
  
Не забудьте проверить [жизненный цикл продукта](https://support.microsoft.com/lifecycle/search) и требования к системе для любой версии SharePoint, до которой выполняется обновление. Поэтому вы будете знать, когда потребуется следующее обновление (например, если вы приостановитесь в устаревшем продукте, например SharePoint Server 2010, чтобы спланировать дополнительные обновления, убедитесь, что Дата окончания срока службы поддержки) и что у вас есть оборудование, поддерживающее ваш план. 
  
Если вы планируете перенести некоторые или все сайты SharePoint в Microsoft 365 в облаке, это время Закладка ссылки на [Описание службы microsoft 365 и Office 365](https://docs.microsoft.com/office365/servicedescriptions/office-365-service-descriptions-technet-library). Чтобы узнать о возможностях SharePoint Online и о том, как они могут отличаться от локального SharePoint Server, вам потребуются описания служб. Обновление функциональных ферм Microsoft Office SharePoint Server 2007. Если у установки есть неработающие сайты, устраните их перед обновлением.
  
## <a name="a-note-about-managing-risk"></a>Примечание об управлении риском

Методы, такие как "параллельные", важны в схеме логики обновления. При параллельном обновлении вы сохраняете ферму Microsoft Office SharePoint Server 2007, но создаете ферму в следующей версии (SharePoint Server 2010) на новом оборудовании. Это помогает тремя способами:
  
1. Вы можете создать резервные копии баз данных Microsoft Office SharePoint Server 2007, чтобы обновить их по отдельности с помощью присоединения базы данных.
    
2. Если вы выберете, что в ферме Microsoft Office SharePoint Server 2007 используется только небольшое количество критических библиотек документов и другие сведения, вы можете вручную переместить данные из Microsoft Office SharePoint Server 2007 в SharePoint Server 2010 или только определенные сайты и веб-сайты в следующую версию (что может облегчить выполнение задачи).
    
3. Чем меньше вы перемещаетесь на ферму серверов Microsoft Office SharePoint Server 2007, вы можете напрямую повысить безопасность данных, содержащихся в ферме.
    
Такие методы, как обновление на месте, будут работать непосредственно в ферме Microsoft Office SharePoint Server 2007, что позволит вам уменьшить количество простых способов отказаться от пути и снова начать работу с вашей средой Пристине. Как можно больше, создайте некоторые меры безопасности (например, создание и тестирование резервных копий исходной среды). Например, если ваша ферма Microsoft Office SharePoint Server 2007 является виртуальной и дублируется в целях резервного копирования и восстановления, создайте резервную копию и восстановите самые актуальные базы данных перед обновлением. Если у вас есть возможность восстановления резервных копий баз данных, вы можете не только переработать ее, но и предоставить вам все заботы.
  
> [!TIP]
> Документы с рекомендациями по обновлению существуют для [Microsoft Office SharePoint Server 2007](https://technet.microsoft.com/library/cc261992%28v=office.12%29.aspx), [SharePoint Server 2010](https://technet.microsoft.com/library/cc261992%28v=office.14%29.aspx), [SharePoint server 2013](https://technet.microsoft.com/library/cc261992%28v=office.15%29.aspx)и [SharePoint Server 2016](https://technet.microsoft.com/library/cc261992%28v=office.16%29.aspx). Вы также можете искать [партнеров Майкрософт](https://partnercenter.microsoft.com/pcv/search) , у которых есть опыт обновления или миграции Microsoft 365. 
  
## <a name="make-your-plan"></a>Сделайте план

Если вам нужно выполнить обновление, вам потребуется план, и один из них не подходит для всех этих случаев. Ваш план может быть простым при создании подписки на Microsoft 365 с помощью SharePoint Online, регистрации домена и перенаправлении пользователей для сохранения файлов. И он может быть недоступен. Это ваше решение, которое вам и вашим пользователям действительно необходимо.
  
> [!NOTE]
> Это рискованно запускать на программном обеспечении, жизненный цикл которого закончился. Продукты, которые не поддерживаются, не поддаются исправлению при обнаружении проблем. Это также означает, что при возникновении новых угроз безопасности не будут выдаваться исправления или исправления системы безопасности, так как эти продукты больше не поддерживаются. Не используйте эту ситуацию. 
  
### <a name="first-know-your-farm"></a>Сначала необходимо знать ферму

При обновлении ваше решение должно основываться на том, что делает ваша ферма в вашей организации. Что необходимо для этого? Что такое роль? Каждая ферма в вашей организации может иметь другую роль. Некоторые фермы SharePoint могут быть  *критическими*  , некоторые из них могут быть архивами файлов для безопасного хранения. Если ферма заполняет несколько ролей одновременно, вам может понадобиться знать, какие семейства веб-сайтов, веб-сайты или даже библиотеки документов выполняют, какие настройки и как важны. Анализ данных на этом уровне может показаться значительным объемом работы, но экономит время и усилия для основного домена перед обновлением или миграцией. Когда вы узнаете обо всех движущихся частях и самых важных битах, вы также узнаете, что вы увеличили и можете оставить позади. Этот набор знаний будет выгоден только при пересылке. 
  
Итак, что говорят пользователи, наиболее важные сведения о ферме SharePoint Server?
  
- Встроенные функции SharePoint
    
- Собрании данных большого размера (например, Архив файлов).
    
- Доступность
    
- Критические приложения, веб-части или документы в ферме (критическая ферма)
    
- Соблюдение стандартов соответствия требованиям
    
- Настройки
    
Если вы запустили что-то важное для вашего бизнеса в вашей ферме SharePoint, скажите, что он действует как большой каталог критически важных данных о требованиях к клиентской службе, вы можете поместить рядом с ключевыми словами "критические приложения", но также "доступность" — это означает, что ваша организация будет оказывать влияние на работу с SharePoint. Аналогично, вы можете проверить "настройки", так как важные службы, которые предлагает ферма, основаны на настраиваемом коде, определениях сайтов или в нескольких пользовательских настройках.
  
Если SharePoint соответствует этим потребностям без необходимости выполнять какие-либо действия в рамках встроенного программного обеспечения, и вы обычно обновляете и выполняете обычное администрирование и обслуживание, возможно, вы выбрали "Встроенная поддержка SharePoint" — это может быть также вызвано тем, что вы не используете более раннюю версию SharePoint. Другими словами, он уже выполняет необходимые действия, и вам не требовалось его обновлять до этого момента в Microsoft Office SharePoint Server 2007.
  
Когда вы маркированный список этих вещей, вы создадите условия для обновления. Другими словами, любое обновление должно соответствовать этой строке. Это позволяет исключить методы, которые в настоящее время не соответствуют вашим потребностям.
  
### <a name="a-simple-sample-plan"></a>Простой пример плана

Может потребоваться более широкое согласование с лидерами и другими администраторами по пути, которое займет обновление SharePoint. Администраторы SharePoint Server часто взаимодействуют с администраторами Microsoft SQL Server, работая с сетями и группами безопасности, а также с другими. Если у вас много заинтересованных лиц, вам может потребоваться создать соглашение по обновлению и миграции. Например, если вы переносите данные таким образом, что часть вашей компании использует SharePoint Online в Microsoft 365, то, скорее всего, потребуется настройка производительности или тестирование в вашей сети. Затронутые команды должны быть заранее уведомлены об этом.
  
В моем простом примере я покажу предложение администратора SharePoint, а затем запланировать, что все заинтересованные стороны согласованы. Для ясности Задокументируйте свои соглашения и решения.
  
План начинается после углубленного анализа фермы и пытается определить роль фермы, проблемные баллы и другие важные сведения, которые могут привести к сужению некоторых параметров обновления. После этого администратор SharePoint принимает предложение по обновлению, а заинтересованные стороны договариваются о плане действий.
  
Список "самый важный" маркированный список:
  
- Доступность, компоненты, встроенные в SharePoint и стандарты соответствия требованиям.
    
- Большая часть данных состоит из трех семейств веб-сайтов с одной рабочей областью для собраний, используемой группой разработчиков, особенно важной и интенсивно используемой в нескольких часовых поясах мира.
    
- Существует Севентин других широко используемых сайтов.
    
- Две библиотеки документов (Рабочая область для собраний и документы в корневом семействе веб-сайтов) имеют наибольшее значение (свыше 8000 документов каждый). У нас есть большое количество архивных документов и списков с вложениями электронных таблиц.
    
- Существуют четырнадцати списки библиотек, которые содержат конфиденциальные данные, которые должны оставаться в соответствии с требованиями.
    
- Мы должны иметь возможность выполнять удержание и электронное обнаружение везде, где мы проходили.
    
- Некоторые из этих данных должны оставаться на локальном уровне из-за правил ИНФОСЕК.
