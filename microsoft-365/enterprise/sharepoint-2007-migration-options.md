# <a name="sharepoint-2007-migration-options-to-consider"></a>Варианты миграции SharePoint 2007, которые необходимо рассмотреть

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Поддержка Microsoft SharePoint 2007 и SharePoint Server 2007 заканчивается. Пора обновиться! В этой статье данная статья содержит сведения о вариантах миграции.
  
## <a name="common-upgrade-strategies-for-sharepoint"></a>Распространенные стратегии обновления для SharePoint

Существует несколько методов обновления среды SharePoint Server. Если у вас есть Microsoft Office SharePoint Server 2007, вот несколько примеров методов обновления:
  
- Обновление с присоединением баз данных
    
- Рядом с обновлением
    
- Обновление на месте
    
- Гибридное обновление (на месте с отключенными базами данных или отдельным присоединением баз данных)
    
- Гибриды SharePoint (подключение через Интернет к локальной сети SharePoint)
    
- Перемещение данных между коллекциями веб-сайтов или библиотеками вручную
    
- Обновление мастера FastTrack до Microsoft 365[(помощник по развертыванию SharePoint Online)](https://aka.ms/spoguidance)
    
- API миграции в SharePoint Online (SPO) в Microsoft 365
    
Что вам лучше всего работает?
  
Ваши знания о том, для чего используется ферма, — это тактическая сила при обновлении. Способ использования фермы SharePoint поможет вам выбрать один из вариантов.
  
> [!TIP]
> Microsoft Office SharePoint Server 2007 также имеет постепенное обновление, не освещается здесь. Список статей о конкретных шагах обновления см. в статье "План действий по окончании поддержки [SharePoint Server 2007".](sharepoint-2007-end-of-support.md) 
  
Не забудьте проверить [жизненный цикл](https://support.microsoft.com/lifecycle/search) продукта и системные требования для всех версий SharePoint, до которые вы обновляете. Это необходимо, чтобы вы знали, когда потребуется следующее обновление (например, если вы приостановите на устаревшем продукте, например SharePoint Server 2010, для планирования дополнительных обновлений, убедитесь, что вы знаете дату окончания поддержки) и убедитесь, что у вас есть оборудование, которое поддерживает ваш план. 
  
Если вы планируете перейти с некоторых или всех сайтов SharePoint на Microsoft 365 в облаке, это время для закладки ссылки на описания служб [Microsoft 365 и Office 365.](https://docs.microsoft.com/office365/servicedescriptions/office-365-service-descriptions-technet-library) Вам потребуется описание служб, чтобы узнать о функции SharePoint Online и их отличиях от локальной службы SharePoint Server. Обновление функциональных Microsoft Office Фермы SharePoint Server 2007. Если в установке имеются сайты, которые не были с ошибками, исправьте их перед обновлением.
  
## <a name="a-note-about-managing-risk"></a>Примечание об управлении рисками

Такие методы, как "рядом", важны в схеме логики обновления. При обновлении рядом ферма Microsoft Office SharePoint Server 2007, но создается ферма в следующей версии (SharePoint Server 2010) на новом оборудовании. Это помогает в трех способах:
  
1. Есть место для отдельного обновления баз данных Microsoft Office SharePoint Server 2007 с помощью подключений баз данных.
    
2. Если вы выясните, что в ферме Microsoft Office SharePoint Server 2007 используется только небольшое количество критически важных библиотек документов и других сведений, можно вручную переместить данные из Microsoft Office SharePoint Server 2007 в SharePoint Server 2010 или перенести только определенные сайты и веб-сайты в следующую версию (что упростит работу).
    
3. Чем меньше вы делаете с фермой Microsoft Office SharePoint Server 2007, тем более безопасными будут данные, которые содержатся в ферме при обновлении.
    
Такие методы, как In-Place обновления, будут действовать непосредственно на вашей ферме Microsoft Office SharePoint Server 2007, что позволит вам с меньшего количество простых вариантов отказаться от пути и начать заново со своей среды. По мере возможности выстройте некоторые меры безопасности (например, создание и тестирование резервных копий исходной среды). Например, если ферма share Microsoft Office SharePoint Server 2007 является виртуальной и дублируется в целях резервного копирования и восстановления, то перед обновлением скопируйте и восстановите последние базы данных до окна службы. Зная, что имеется возможность восстановления резервных копий баз данных, вы не только не сможете восстановить резервную копию базы данных, но и не сможете ее использовать.
  
> [!TIP]
> Microsoft Office Для [sharePoint Server 2007,](https://technet.microsoft.com/library/cc261992%28v=office.12%29.aspx) [SharePoint Server 2010,](https://technet.microsoft.com/library/cc261992%28v=office.14%29.aspx) [SharePoint Server 2013](https://technet.microsoft.com/library/cc261992%28v=office.15%29.aspx)и [SharePoint Server 2016](https://technet.microsoft.com/library/cc261992%28v=office.16%29.aspx)существуют документы с практическими методиками обновления. Вы также можете искать партнеров [Майкрософт,](https://partnercenter.microsoft.com/pcv/search) у которых есть опыт обновления или миграции Microsoft 365. 
  
## <a name="make-your-plan"></a>Планирование

Если требуется обновление, вам нужен план, и в таких случаях один размер подходит не всем. Ваш план может быть таким же простым, как "Создание подписки на Microsoft 365 с помощью SharePoint Online, регистрация домена и перенаправление людей для сохранения файлов в ней". И это может быть не так. Это решение является вашим решением, и вам и вашим пользователям действительно нужно.
  
> [!NOTE]
> На программном обеспечении, жизненный цикл которого закончился, существует риск. Продукты, которые не поддерживаются, больше не будут исправлены при обнаружении проблем. Это также означает, что при новых угрозах безопасности не будет исправлений или исправлений для системы безопасности, так как продукты на конец жизненного цикла больше не поддерживаются. Избегайте такой ситуации! 
  
### <a name="first-know-your-farm"></a>Сначала нужно знать свою ферму

При обновлении процесс принятия решений должен основываться на том, что ваша ферма делает для вашей организации. Что необходимо для удовлетворения потребностей? Какова его роль? Каждая ферма в вашей компании может иметь свою роль. Некоторые фермы SharePoint могут быть критически  *важными,*  некоторые из них могут быть файловые архивы для безопасного сохраняемого. Или, если ферма одновременно заполняет множество ролей, может потребоваться знать, какие коллекции веб-сайтов, веб-сайты или даже библиотеки документов, какие настройки и насколько они важны. Анализ данных на этом уровне может показаться большим объемом работы, но он экономит время и усилия, чтобы управлять доменом перед обновлением или миграцией. После того как вы узнаете все движущиеся части и самые важные биты, вы также узнаете, что вы загромощены и можете оставить после себя. Эти знания будут только вам помочь в будущем. 
  
Итак, что пользователи говорят о том, что самое важное для фермы SharePoint Server?
  
- Встроенные функции SharePoint
    
- Большой объем данных (например, архив файлов)
    
- Доступность
    
- Критически важные приложения, веб-части или документы в ферме (критически важная ферма)
    
- Соблюдение стандартов соответствия требованиям
    
- Настройки
    
Если вы запустите что-то важное для своей компании из фермы SharePoint, скажем, это будет большой каталог критически важных данных о требованиях к клиентской службе, вы можете поместить рядом с "Критическими приложениями" так же, как и "Доступность", то есть если вы не сможете использовать SharePoint некоторое время, это повлияет на ваш бизнес. Кроме того, можно проверить "Настройки", так как критические службы фермы основаны на пользовательском коде, определениях сайтов или ряде настроек, которые работают вместе.
  
Если SharePoint выполнил эти потребности, не делайте ничего, кроме встроенного программного обеспечения, и вы обычно обновляете его и проводите обычное администрирование и обслуживание, возможно, вы выбрали "Встроенный SharePoint". Это также может быть вашей причиной для работы с более старой версией SharePoint. Другими словами, он уже делает то, что вам нужно, и вам не нужно было обновлять до этого момента, Microsoft Office SharePoint Server 2007.
  
Когда вы перечисляете эти данные, вы создаете условия для обновления. Другими словами, любое обновление должно соответствовать этой панели, чтобы считаться. Это позволяет исключить методы, которые в настоящее время не соответствуют вашим потребностям.
  
### <a name="a-simple-sample-plan"></a>Простой пример плана

Возможно, потребуется более широкое согласование с руководством и другими администраторами того, как будет проходить обновление SharePoint. Администраторы SharePoint Server часто взаимодействуют с Microsoft SQL Server администраторами, работают с группами сети и безопасности и другими вопросами. Если имеется большое количество заинтересованных лиц, может потребоваться создать соглашение или скорректировать план обновления и миграции. Например, если перенести данные таким образом, чтобы часть вашей компании использовала SharePoint Online в Microsoft 365, скорее всего, потребуется настроить или тестировать производительность в вашей сети. Затронутые команды следует уведомить заранее.
  
В моем простом примере покажите предложение администратора SharePoint, а затем перечислите план, согласованный всеми заинтересованными сторонами. Для ясности документировать свои соглашения и решения.
  
План начинается после подробного анализа фермы и пытается определить роль фермы, точки сбоя и другую важную информацию, которая может привести к сужжингу некоторых параметров обновления. После этого администратор SharePoint предлагает обновление, и заинтересованные стороны согласовыют план действий.
  
Мой "самый важный" список:
  
- Доступность, встроенные функции SharePoint и стандарты соответствия требованиям.
    
- Большая часть данных находится в трех коллекциях веб-сайтов, причем одна рабочее пространство для собраний, используемая командой разработчиков, особенно важна и интенсивно используется в нескольких часовых поясах по всему миру.
    
- Существуют другие сайты, которые широко используются.
    
- Две библиотеки документов (рабочее пространство для собраний и документы в корневом наборе веб-сайтов) имеют наибольшее количество (более 8000 документов в каждой). У нас есть большое количество архивных документов и списков с вложениями в электронные таблицы.
    
- Существует несколько списков библиотек с конфиденциальными данными, которые ДОЛЖНЫ оставаться в соответствии с нормативными требованиям.
    
- У нас должна быть возможность делать удерживая и обнаруживать электронные возможности, куда бы мы ни перейти.
    
- Некоторые из этих данных ДОЛЖНЫ оставаться в локальной организации из-за правил InfoSec.
