# <a name="additional-device-information-for-the-migration-from-microsoft-cloud-deutschland"></a>Дополнительные сведения об устройстве для миграции из Microsoft Cloud Deutschland

Подключенные и зарегистрированные устройства Azure AD, подключенные к Microsoft Cloud Deutschland, должны быть перенесены после 9-го этапа и до 10-го этапа. Миграция устройства зависит от типа устройств, операционной системы и отношения Azure AD.

## <a name="azure-ad-joined-windows-10-devices"></a>Устройства Azure AD Windows 10
Если Windows 10 подключено Azure AD, оно должно быть отключено от Azure AD и должно быть подключено снова.

[![Устройство Azure AD ](../media/ms-cloud-germany-migration-opt-in/AAD-ReJoin-flow.png) Re-Join Flow](../media/ms-cloud-germany-migration-opt-in/AAD-ReJoin-flow.png#lightbox)


Если пользователь является администратором на устройстве Windows 10, пользователь может оторегистрить устройство из Azure AD и повторно присоединиться к нему в течение трех шагов.

### <a name="step-1-determine-if-the-device-is-azure-id-joined"></a>Шаг 1. Определите, присоединилось ли устройство к Azure ID

1. Войдите в службу с помощью рабочей учетной записи.
2. Перейдите **в Параметры**  >  **Учетные**  >  **записи Доступ к работе или школе**.
3. Найми учетную запись в списке с **подключением к [...]' s Azure AD**.
4. Если подключенная учетная запись существует, приступить к шагу 2.

### <a name="step-2-disconnect-the-device-from-azure-ad"></a>Шаг 2. Отключение устройства от Azure AD

1. Щелкните **Отключение** на подключенной работе или учетной записи школы.
2. Дважды подтвердите отключение.
3. Введите имя пользователя и пароль местного администратора. Устройство отключено.
4. Перезапустите устройство.

### <a name="step-3-join-the-device-to-azure-ad"></a>Шаг 3. Присоединиться к устройству в Azure AD

1. Вопишитесь с учетными данными местного администратора.
2. Перейдите **в Параметры**  >  **Учетные**  >  **записи Доступ к работе или школе**.
3. Нажмите кнопку **Подключить**.
4. **ВАЖНО:** **Нажмите кнопку Присоединиться к Azure AD**.
5. Введите адрес электронной почты и пароль вашей учетной записи. Устройство подключено.
6. Перезапустите устройство.
7. Войте вход с адресом электронной почты и паролем вашей учетной записи.

Если пользователь не является администратором устройства, глобальный администратор Azure AD может создать учетную запись локального администратора на устройстве по этому пути настройки и отсоединить устройство:

*Параметры > учетные записи > другие учетные записи > учетные данные неизвестные > Добавить пользователя без Microsoft-Account*

Для повторного присоединения на этом шаге можно использовать учетные данные любой учетной записи вашей организации.

Обратите внимание на то, что учетная запись, используемая для работы с устройством, будет автоматически повышена в качестве администратора устройства.
Любая другая учетная запись в организации может войти на устройство, но не имеет прав администратора.

## <a name="azure-ad-registered-workplace-joined-windows-10-devices"></a>Зарегистрированные устройства Azure AD (с Windows 10 рабочими устройствами)

Если Windows 10 зарегистрирован Azure AD, его необходимо отключить от Azure AD и снова подключать.

[![Устройство Azure AD ](../media/ms-cloud-germany-migration-opt-in/AAD-ReRegistration-flow.png) Re-Registration Flow](../media/ms-cloud-germany-migration-opt-in/AAD-ReJoin-flow.png#lightbox)

### <a name="step-1-determine-if-the-device-is-azure-id-registered"></a>Шаг 1. Определите, зарегистрировано ли устройство в Azure ID

1. Вход с пользователем.
2. Перейдите **в Параметры**  >  **Учетные**  >  **записи Доступ к работе или школе**.
3. Откройте свою учетную запись в списке и проверьте, подключена ли она **к [...]» s Azure AD**.

    Если ваша учетная запись находится в списке, но не подключена к Azure AD, приступить к шагу 2.

    В противном случае устройство является устройством, присоединимой к Azure AD, и вам необходимо обратиться к устройствам [Azure AD Windows 10.](#azure-ad-joined-windows-10-devices)

### <a name="step-2-disconnect-the-device-from-azure-ad"></a>Шаг 2. Отключение устройства от Azure AD

1. Щелкните свою учетную запись. Отображаются кнопки *Info* и *Disconnect.*
2. Щелкните **Отключение**.
3. Подтверждение удаления учетной записи с устройства, нажав **кнопку Да**.

### <a name="step-3-connect-the-device-to-azure-ad"></a>Шаг 3. Подключение устройство в Azure AD

1. Нажмите кнопку **Подключить**.
2. Введите адрес электронной почты вашей учетной записи и нажмите **кнопку Далее**.
3. Введите пароль вашей учетной записи и нажмите **кнопку Вход**.
4. Подтвердив, нажав **кнопку Готово**. Ваша учетная запись снова указана.

## <a name="android"></a>Android

Для Android пользователям потребуется оторегистрируйте и перерегистрируйте свои устройства. Это можно сделать с помощью Microsoft Authenticator или Корпоративный портал приложения.

- В приложении Microsoft Authenticator пользователи могут перейти к Параметры > **регистрации устройств.** Оттуда пользователи могут отрегистрируйте и перерегистрируйте свое устройство.

- С Корпоративный портал пользователи могут перейти на вкладку **Devices** и удалить устройство. После этого повторно зарегистрив устройство с помощью Корпоративный портал.

- Пользователи также могут отрегистрируйтесь и перерегистрируйтесь, удалив учетную запись со страницы параметров учетной записи, а затем повторно добавив учетную запись.

Для регистрации и перерегистрации устройства на Android с помощью Microsoft Authenticator приложения:

1. Откройте приложение Microsoft Authenticator и перейдите **в Параметры**.
2. Выберите **регистрацию устройства.**
3. Незарегистрированное устройство, выбрав **Unregister**.
4. Для **регистрации устройства** перерегистрируйте устройство, введя адрес электронной почты, а затем выберите **Register**.

Для регистрации и перерегистрации устройства Android на странице Android Параметры:

1. Откройте **устройство Параметры** и перейдите к **учетным записям.**
2. Выберите учетную запись работы, которую необходимо перерегистрируйте, и выберите **учетную запись Remove.**
3. После удаления учетной записи со страницы **Учетные** записи выберите учетную запись **Add Account > work account.**
4. Для **workplace Join** введите адрес электронной почты и выберите **Присоединиться,** чтобы завершить регистрацию устройства.

Для регистрации и перерегистрации устройства на Android с Корпоративный портал:

1. Запустите Корпоративный портал и перейдите на **вкладку Devices.**
2. Выберите устройство, чтобы увидеть сведения об устройстве.
3. Из меню эллипсов (три точки) выберите **Remove Device** и выполните удаление, подтвердив в диалоговом окантовке.
4. Теперь вы должны войти в систему из Корпоративный портал приложения. Выберите **вход для** повторной регистрации устройства.

Дополнительные сведения о любых действиях, необходимых на этапе миграции этой рабочей нагрузки или воздействии на администрирование или использование, см. в Azure Active Directory (Azure AD) в дополнительных сведениях Azure AD для миграции из [Microsoft Cloud Deutschland](ms-cloud-germany-transition-azure-ad.md).

## <a name="ios"></a>iOS

На устройствах iOS пользователю необходимо вручную удалить все кэшированные учетные записи из Microsoft Authenticator, отрегистрить устройство и выйти из любых родных приложений на устройстве.

### <a name="step-1-if-present-remove-the-account-from-the-microsoft-authenticator-app"></a>Шаг 1. Если она присутствует, удалите учетную запись из Microsoft Authenticator приложения

1. Нажмите на учетную запись в Microsoft Authenticator приложении.
2. Нажмите **значок Параметры** в правом верхнем углу. Если вы не видите значок **Параметры,** возможно, вы не используете последнюю версию Microsoft Authenticator.
3. Нажмите **кнопку Удалить учетную запись.**
4. Нажмите **кнопку Все приложения на этом устройстве**.

### <a name="step-2-unregister-the-device-from-the-microsoft-authenticator-app"></a>Шаг 2. Незарегистрированное устройство из Microsoft Authenticator приложения

1. Нажмите значок меню в правом верхнем углу.
2. Нажмите **кнопку Параметры,** а затем **регистрация устройств.**
3. Если учетная запись отображается, нажмите **кнопку Незарегистрированное устройство** **и продолжайте** в диалоговом окте. После этого вы не должны видеть учетную запись.

### <a name="step-3-sign-out-from-individual-apps-if-necessary"></a>Шаг 3. При необходимости выпишитесь из отдельных приложений

Пользователи могут перейти к отдельным приложениям, Outlook, Teams и OneDrive, а также удалить учетные записи из этих приложений.

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Как узнать, затронута ли моя организация?**

Администраторы должны проверить, есть ли у них зарегистрированные `https://portal.microsoftazure.de` устройства Azure AD или Azure AD. Если в вашей организации зарегистрированы устройства Azure AD или к Azure AD присоединились, организация должна следовать инструкциям на этой странице.

**Когда пользователи перерегистрируют свои устройства?**

Важно, чтобы после завершения [9-го](ms-cloud-germany-transition-phases.md#phase-9--10-azure-ad-finalization) этапа вы регистрируете и перерегистрируете устройства. Перед началом 10-го этапа необходимо завершить перерегистрацию, в противном случае вы можете потерять доступ к устройству.

**Как узнать, что все мои устройства зарегистрированы в общедоступных облаках?**

Чтобы проверить, зарегистрированы ли устройства в общеобязуемом облаке, необходимо экспортировать и загружать список устройств с портала Azure AD в Excel таблицу. Затем фильтруйте устройства, зарегистрированные (с помощью столбца _registeredTime)_ после даты, когда ваша организация прошла [этап 9 процесса миграции.](ms-cloud-germany-transition-phases.md#phase-9--10-azure-ad-finalization)

## <a name="additional-considerations"></a>Дополнительные рекомендации

> [!IMPORTANT]
> Руководитель службы Intune будет включен после [3-го](ms-cloud-germany-transition-phases.md#phase-3-subscription-transfer)этапа процесса миграции, что подразумевает активацию регистрации устройств Azure AD. Если вы заблокировали регистрацию устройств Azure AD перед миграцией, необходимо отключить главу службы Intune с PowerShell, чтобы снова отключить регистрацию устройств Azure AD на портале Azure AD. Вы можете отключить главу службы Intune с этой командой в Azure Active Directory PowerShell для Graph модуля.

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```
