# <a name="additional-device-information-for-the-migration-from-microsoft-cloud-deutschland"></a>Дополнительные сведения об устройстве для миграции с Microsoft Cloud записей

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Как узнать, подвержена ли моя организация?**

Администраторы должны проверить `https://portal.microsoftazure.de` , есть ли у них зарегистрированные устройства. Если в вашей организации есть зарегистрированные устройства, это повлияет на работу.

**Каково воздействие на пользователей?**

Пользователи из зарегистрированного устройства больше не смогут войти в систему после перехода на этап [заключительного завершения миграции Azure AD](ms-cloud-germany-transition.md#how-is-the-migration-organized) .  

Убедитесь, что все устройства зарегистрированы в глобальной конечной точке, прежде чем ваша организация будет отключена от Microsoft Cloud записей.
  
**Когда мои пользователи повторно регистрируют свои устройства?**

Вам очень важно успешно выполнить регистрацию и повторно зарегистрировать ваши устройства на [отдельном этапе записей миграции Microsoft Cloud](ms-cloud-germany-transition.md#how-is-the-migration-organized) .

**Как восстановить состояние устройства после миграции?**

Для гибридных и присоединенных к домену устройств с Windows, которые зарегистрированы в Azure AD, администраторы могут управлять переносом этих устройств через удаленно запущенные рабочие процессы, которые будут отменять регистрацию старых состояний устройств.
  
Для всех остальных устройств, включая персональные устройства Windows, зарегистрированные в Azure AD, конечный пользователь должен выполнить эти действия вручную. Для устройств, присоединенных к Azure AD, пользователям необходимо иметь учетную запись локального администратора для отмены регистрации и повторной регистрации своих устройств.

Корпорация Майкрософт будет публиковать инструкции о том, как успешно восстановить состояние устройства. 
 
**Как узнать, что все мои устройства зарегистрированы в общедоступном облаке?**

Чтобы проверить, зарегистрированы ли ваши устройства в общедоступном облаке, необходимо экспортировать и скачать список устройств с портала Azure AD в электронную таблицу Excel. Затем отфильтруйте устройства, которые зарегистрированы (с помощью столбца _регистередтиме_ ), после фазы миграции [Microsoft Cloud записей](ms-cloud-germany-transition.md#how-is-the-migration-organized) .

Регистрация устройства отключена после миграции клиента и не может быть включена или отключена. Если Intune не используется, войдите в свою подписку и выполните следующую команду для повторной активации параметра:

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

## <a name="windows-hybrid-azure-ad-join"></a>Присоединение к гибридной службе Windows Azure AD

### <a name="windows-down-level"></a>Windows Down

_Устройства Windows нижнего уровня_ — это устройства Windows, которые в настоящее время работают под управлением более ранних версий Windows (например, Windows 8,1 или Windows 7) или работают под управлением Windows Server версий более ранних, чем 2019 и 2016. Если такие устройства были зарегистрированы ранее, вам потребуется отменить регистрацию и повторно зарегистрировать эти устройства. 

Чтобы определить, было ли устройство нижнего уровня Windows было присоединено к Azure AD, выполните следующую команду на устройстве:

```console
%programfiles%\Microsoft Workplace Join\autoworkplace /status
```

Если устройство было ранее присоединено к Azure AD, и если устройство имеет сетевое подключение к глобальным конечным точкам Azure AD, вы увидите следующий результат:

```console
+----------------------------------------------------------------------+
| Device Details                                                       |
+----------------------------------------------------------------------+
         DeviceId : AEE2B956-DA62-48D0-BB47-046DD992A110
       Thumbprint : 00fdfa2de5c32feae57489873a13aa6a3ff7433b
             User : user1@<tenantname>.de
Private key state : Okay
     Device state : Unknown
```

На соответствующих устройствах будет указано "состояние устройства" со значением "неизвестно". Если выходные данные имеют значение «устройство не присоединено» или значение "состояние устройства" имеет значение "хорошо", проигнорируйте приведенные ниже рекомендации.

Только для устройств, которые показывают, что устройство присоединено (в соответствии с идентификатором deviceId, отпечатком и т. д.), а значение Device State — "Unknown", администраторы должны выполнить следующую команду в контексте входа пользователя домена на таком устройстве:

```console
"%programfiles%\Microsoft Workplace Join\autoworkplace /leave"
```

Предыдущая команда должна выполняться только один раз для входа пользователя домена на устройстве Windows нижнего уровня. Эту команду следует выполнить в контексте входа пользователя в домен. 

Необходимо принять меры, чтобы не выполнять эту команду при последующем входе пользователя. При выполнении предыдущей команды будет снято состояние объединения локального гибридного гибридного компьютера с присоединением Azure AD — для пользователя, выполнившего вход. Если же компьютер по-прежнему настроен для гибридной службы Azure AD — присоединяется к клиенту, он будет пытаться присоединиться при повторном входе пользователя.

### <a name="windows-current"></a>Windows Current

#### <a name="unjoin"></a>Не присоединение

Чтобы определить, было ли устройство Windows 10 ранее присоединено к Azure AD, выполните следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство является гибридной службой Azure AD — присоединено к сети, администратор увидит следующие выходные данные:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : YES
```

Если результатом является "Азуреаджоинед: No", проигнорируйте приведенные ниже рекомендации.

Только для устройств, которые показывают, что устройство присоединено к Azure AD, выполните следующую команду в качестве администратора, чтобы удалить состояние объединения устройства.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущая команда должна выполняться один раз в административном контексте на устройстве с Windows.

#### <a name="hybrid-ad-joinre-registration"></a>Гибридная служба AD Жоин\ре-регистратион

Устройство автоматически присоединяется к Azure AD без вмешательства пользователя или администратора, если устройство имеет сетевое подключение к глобальным конечным точкам Azure AD. 


## <a name="windows-azure-ad-join"></a>Присоединение к Windows Azure AD

**Важно!** Субъект службы Intune будет включен после миграции в Commerce, что означает активацию регистрации устройств Azure AD. Если вы заблокировали регистрацию устройств Azure AD перед миграцией, необходимо отключить субъект службы Intune с PowerShell, чтобы отключить регистрацию устройств Azure AD с помощью портала Azure AD. Вы можете отключить субъект службы Intune с помощью этой команды в модуле PowerShell Azure Active Directory PowerShell для Graph.

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

### <a name="unjoin"></a>Не присоединение

Чтобы определить, было ли устройство Windows 10 было ранее присоединено к Azure AD, пользователь или администратор может выполнить следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство присоединено к Azure AD, пользователь или администратор увидит следующий результат:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : NO
```

Если результатом является "Азуреаджоинед: NO", проигнорируйте приведенные ниже рекомендации.

Пользователь: Если устройство подключено к Azure AD, пользователь может отсоединить устройство от параметров. Убедитесь, что на устройстве есть учетная запись локального администратора, прежде чем отсоединить устройство от Azure AD. Для обратного входа в устройство требуется учетная запись локального администратора.

Администратор: Если администратору организации требуется отсоединить устройства пользователей, которые входят в состав Azure AD — это можно сделать, выполнив следующую команду на каждом устройстве, используя такой же механизм, как групповая политика. Администратор должен проверить наличие учетной записи локального администратора на устройстве перед отсоединением устройства от Azure AD. Учетная запись локального администратора необходима для обратного входа в устройство.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущая команда должна выполняться один раз в административном контексте на устройстве с Windows. 

### <a name="azure-ad-joinre-registration"></a>Присоединение и повторная регистрация Azure AD

Пользователь может присоединить устройство к Azure AD из параметров Windows: **параметры > учетных записей > доступ к рабочим или учебным > подключиться**.
 

## <a name="windows-azure-ad-registered-company-owned"></a>Зарегистрированный Windows Azure AD (владелец компании)

Чтобы определить, зарегистрировано ли устройство Windows 10 в Azure AD, выполните следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство зарегистрировано в Azure AD, вы увидите следующий результат:

```console
+----------------------------------------------------------------------+
| User State                                                           |
+----------------------------------------------------------------------+
           WorkplaceJoined : YES
          WamDefaultSet : NO
          WamDefaultAuthority : organizations
```

Чтобы удалить существующую учетную запись, зарегистрированную в Azure AD, на устройстве:

- Чтобы удалить учетную запись, зарегистрированную в Azure AD — на устройстве, используйте Клеанупвпж, средство, которое можно скачать отсюда: [CleanupWPJ.zip](https://download.microsoft.com/download/8/e/f/8ef13ae0-6aa8-48a2-8697-5b1711134730/WPJCleanUp.zip).

- Извлеките ZIP-файл и запустите **впжклеануп. cmd**. Это средство запустит правильный исполняемый файл в соответствии с версией Windows на устройстве.

- С помощью такого механизма, как групповая политика, администратор может выполнить команду на устройстве в контексте любого пользователя, выполнившего вход на устройстве.

Чтобы отключить диспетчер учетных записей веб-записей, чтобы зарегистрировать устройство в Azure AD, добавьте следующее значение реестра: 

- Расположение: Хклм\софтваре\полиЦиес\микрософт\виндовс\воркплацежоин
- Тип: DWORD (32 бит)
- Name: Блоккаадворкплацежоин
- Данные значения: 1

Наличие этого значения реестра должно заблокировать присоединение к рабочему месту и предотвратить отображение приглашений на присоединение к устройству.

## <a name="android"></a>Android

Для Android пользователям потребуется отменить регистрацию и повторно зарегистрировать их устройства. Это можно сделать с помощью приложения Microsoft Authenticator или приложения корпоративного портала. 

- В приложении Microsoft Authenticator пользователи могут перейти к разделу **параметры > Device Registration**. Пользователи могут отменить регистрацию и повторно зарегистрировать их устройства.
 
- На портале компании пользователи могут перейти на вкладку **устройства** и удалить устройство. После этого повторно зарегистрируйте устройство с помощью корпоративного портала.
 
- Кроме того, пользователи могут отменить регистрацию и повторно зарегистрироваться, удалив учетную запись на странице параметров учетной записи, а затем повторно добавляя рабочую учетную запись.

Отмена регистрации и повторная регистрация устройства на Android с помощью приложения проверки подлинности (Майкрософт):

1.  Откройте приложение Microsoft Authenticator и перейдите к разделу **Параметры**.
2.  Выберите **Device Registration (регистрация устройства**).
3.  Отмените регистрацию устройства, выбрав **отменить регистрацию**.
4.  Для **регистрации устройства** повторно зарегистрируйте его, введя свой адрес электронной почты, а затем нажмите кнопку **зарегистрировать**.

Чтобы отменить регистрацию и повторно зарегистрировать устройство Android на странице параметров Android, выполните следующие действия:

1.  Откройте **Параметры устройства** и перейдите к разделу **учетные записи**.
2.  Выберите рабочую учетную запись, которую необходимо повторно зарегистрировать, и выберите пункт **удалить учетную запись**.
3.  После удаления учетной записи на странице " **учетные записи** " выберите **Добавить учетную запись > рабочий учетной записи**.
4.  Для параметра **Присоединение к рабочему месту** введите свой адрес электронной почты и нажмите кнопку **присоединиться** , чтобы завершить регистрацию устройства.

Отмена регистрации и повторная регистрация устройства на Android с корпоративного портала:

1.  Запустите корпоративный портал и перейдите на вкладку **устройства** .
2.  Выберите устройство, чтобы просмотреть сведения об устройстве.
3.  В меню многоточия (три точки) выберите пункт **удалить устройство** и завершите удаление, выполнив подтверждение в диалоговом окне.
4.  Теперь необходимо выйти из приложения корпоративного портала. Нажмите кнопку **Вход** , чтобы повторно зарегистрировать устройство.

Для получения дополнительных сведений о действиях, которые необходимо выполнить на этапе миграции данной рабочей нагрузки, или влиянии на администрирование или использование, ознакомьтесь со сведениями об Azure Active Directory в разделе [Дополнительные общие сведения о миграции с Microsoft Cloud записей](ms-cloud-germany-transition-add-general.md#azure-active-directory).

## <a name="ios"></a>iOS

На устройствах с iOS пользователь должен вручную удалить все кэшированные учетные записи с помощью средства проверки подлинности (Майкрософт), отменить регистрацию устройства и выйти из любых собственных приложений на устройстве.

### <a name="step-1-if-present-remove-the-account-from-the-microsoft-authenticator-app"></a>Шаг 1: Если этот параметр указан, удалите учетную запись из приложения для проверки подлинности (Майкрософт)

1. Коснитесь учетной записи в приложении Microsoft Authenticator.
2. Коснитесь значка **Параметры** в правом верхнем углу. Если значок **Параметры** не отображается, возможно, вы не используете последнюю версию средства проверки подлинности (Майкрософт).
3. Нажмите кнопку **удалить учетную запись** .
4. Коснитесь пункта **все приложения на этом устройстве**.
 
### <a name="step-2-unregister-the-device-from-the-microsoft-authenticator-app"></a>Шаг 2: Отмена регистрации устройства в приложении Microsoft Authenticator

1. Коснитесь значка меню в правом верхнем углу.
2. Нажмите **Параметры** , а затем **Device Registration**.
4. Если ваша учетная запись отображается, нажмите кнопку **Отмена регистрации устройства** и **продолжите работу** в диалоговом окне. После этого не должна отображаться учетная запись.
 
### <a name="step-3-sign-out-from-individual-apps-if-necessary"></a>Шаг 3: Выйдите из отдельных приложений, если это необходимо

Пользователи могут перейти к отдельным приложениям, например Outlook, Teams и OneDrive, и удалить учетные записи из этих приложений.
