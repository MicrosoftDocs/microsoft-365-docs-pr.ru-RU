# <a name="additional-device-information-for-the-migration-from-microsoft-cloud-deutschland"></a>Дополнительные сведения об устройстве для миграции из Microsoft Cloud Deutschland

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Как узнать, затронута ли моя организация?**

Администраторы должны проверить, есть ли `https://portal.microsoftazure.de` у них зарегистрированные устройства. Если в организации зарегистрированы устройства, это влияет на вас.

**Какое влияние оказывает на пользователей?**

Пользователи с зарегистрированного устройства больше не смогут войти после того, как миграция вступает в этап переноса [Azure AD.](ms-cloud-germany-transition.md#how-is-the-migration-organized)  

Убедитесь, что все ваши устройства зарегистрированы в конечной точке во всем мире до отключения организации от Microsoft Cloud Deutschland.
  
**Когда пользователи перерегистрируют свои устройства?**

Для вашего успеха важно, чтобы вы только отрегистрируете и перерегистрируете устройства на этапе миграции Отдельно от [Microsoft Cloud Deutschland.](ms-cloud-germany-transition.md#how-is-the-migration-organized)

**Как восстановить состояние устройства после миграции?**

Для гибридных устройств Azure AD и Windows, зарегистрированных в Azure AD, администраторы смогут управлять миграцией этих устройств с помощью удаленных процессов, которые будут регистрировать старые состояния устройств.
  
Для всех других устройств, включая Windows устройства, зарегистрированные в Azure AD, конечный пользователь должен выполнять эти действия вручную. Для устройств, присоединив Azure AD, пользователям необходимо иметь учетную запись локального администратора, чтобы отозарегистрировать их, а затем перерегистрируйте свои устройства.

Корпорация Майкрософт опубликует инструкции по успешному восстановлению состояния устройства. 
 
**Как узнать, что все мои устройства зарегистрированы в общедоступных облаках?**

Чтобы проверить, зарегистрированы ли устройства в общеобязуемом облаке, необходимо экспортировать и загружать список устройств с портала Azure AD в Excel таблицу. Затем фильтруем устройства, зарегистрированные (с помощью столбца _registeredTime)_ после этапа миграции Отдельно от [Microsoft Cloud Deutschland.](ms-cloud-germany-transition.md#how-is-the-migration-organized)

Регистрация устройства отключается после миграции клиента и не может быть включена или отключена. Если Intune не используется, включите подписку и запустите эту команду, чтобы повторно активировать параметр:

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

## <a name="hybrid-azure-ad-join"></a>Гибридное присоединение к Azure AD

### <a name="windows-down-level"></a>Windows уровня

_Windows_ — это устройства Windows, которые в настоящее время запускают более ранние версии Windows (например, Windows 8.1 или Windows 7), или которые запускаются в Windows Server раньше 2019 и 2016 годов. Если такие устройства были зарегистрированы ранее, необходимо оторегистрируйте и перерегистрируйте эти устройства. 

Чтобы определить, было ли Windows ранее соединялось с Azure AD, используйте следующую команду на устройстве:

```console
%programfiles%\Microsoft Workplace Join\autoworkplace /status
```

Если устройство ранее было подключено к Azure AD и если устройство подключено к глобальным конечным точкам Azure AD, вы увидите следующий вывод:

```console
+----------------------------------------------------------------------+
| Device Details                                                       |
+----------------------------------------------------------------------+
         DeviceId : AEE2B956-DA62-48D0-BB47-046DD992A110
       Thumbprint : 00fdfa2de5c32feae57489873a13aa6a3ff7433b
             User : user1@<tenantname>.de
Private key state : Okay
     Device state : Unknown
```

На затронутых устройствах будет "Состояние устройства" со значением "Unknown". Если вывод "Устройство не присоединилось" или значение "Состояние устройства" является "Хорошо", игнорируйте следующие указания.

Только для устройств, которые показывают, что устройство соединено (в силу deviceId, thumbprint и так далее) и чье значение "Состояние устройства" является "Неизвестным", администраторы должны выполнить следующую команду в контексте регистрации пользователя домена на таком устройстве ниже уровня:

```console
"%programfiles%\Microsoft Workplace Join\autoworkplace /leave"
```

Предыдущей команде необходимо выполнить только один раз для каждого пользователя домена, входиющего на Windows на устройстве с Windows уровней. Эта команда должна запускаться в контексте регистрации пользователя домена. 

Необходимо заботиться о том, чтобы не запускать эту команду, когда пользователь впоследствии войду. При запуске предыдущей команды будет очищаться присоединилось состояние локального гибридного компьютера Azure AD,присоединившегося к пользователю, который вошел. И если компьютер по-прежнему настроен как гибридный Azure AD,присоединився к клиенту, он будет пытаться присоединиться, когда пользователь снова войдет.

### <a name="windows-current"></a>Windows Current

#### <a name="unjoin"></a>Unjoin

Чтобы определить, было ли Windows 10 ранее соединено с Azure AD, запустите следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство является гибридным Azure AD-joined, администратор увидит следующий вывод:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : YES
```

Если вывод "AzureAdJoined: Нет", игнорируйте следующие рекомендации.

Только для устройств, которые показывают, что устройство присоединяется к Azure AD, запустите следующую команду в качестве администратора, чтобы удалить соединяемого состояния устройства.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущей команде необходимо выполнить только один раз в административном контексте на Windows устройстве.

#### <a name="hybrid-ad-joinre-registration"></a>Гибридная регистрация AD\Re-Registration

Устройство автоматически присоединяется к Azure AD без вмешательства пользователя или администратора до тех пор, пока устройство подключено к глобальным конечным точкам Azure AD. 


## <a name="azure-ad-join"></a>Регистрация Azure AD

**ВАЖНО:** Принцип службы Intune будет включен после миграции коммерции, которая подразумевает активацию регистрации устройств Azure AD. Если вы заблокировали регистрацию устройств Azure AD перед миграцией, необходимо отключить главу службы Intune с PowerShell, чтобы снова отключить регистрацию устройств Azure AD на портале Azure AD. Вы можете отключить главу службы Intune с этой командой в Azure Active Directory PowerShell для Graph модуля.

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

### <a name="unjoin"></a>Unjoin

Чтобы определить, было ли Windows 10 ранее соединялось с Azure AD, пользователь или администратор может выполнить следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство присоединяется к Azure AD, пользователь или администратор увидят следующий вывод:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : NO
```

Если вывод "AzureAdJoined: НЕТ", игнорируйте следующие рекомендации.

Пользователь. Если к устройству присоединилась Azure AD, пользователь может отсоединять устройство от параметров. Убедитесь, что на устройстве есть учетная запись локального администратора, прежде чем отсоединять устройство от Azure AD. Для регистрации на устройстве требуется учетная запись локального администратора.

Администратор. Если администратор организации хочет отсоединять устройства пользователей, присоединив к Azure AD, они могут сделать это, выдав следующую команду на каждом из устройств с помощью механизма, например групповой политики. Администратор должен убедиться, что на устройстве есть учетная запись локального администратора, прежде чем отсоединять устройство от Azure AD. Чтобы войти в устройство, требуется учетная запись локального администратора.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущей команде необходимо выполнить только один раз в административном контексте на Windows устройстве. 

### <a name="azure-ad-joinre-registration"></a>Azure AD Join/Re-Registration

Пользователь может присоединиться к устройству Azure AD из параметров Windows: Параметры > учетных записей > доступа или школьных **> Подключение.**
 

## <a name="azure-ad-registered-company-owned"></a>Azure AD Registered (Company owned)

Чтобы определить, Windows 10 зарегистрировано ли устройство Azure AD, запустите следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство зарегистрировано Azure AD, вы увидите следующий вывод:

```console
+----------------------------------------------------------------------+
| User State                                                           |
+----------------------------------------------------------------------+
           WorkplaceJoined : YES
          WamDefaultSet : NO
          WamDefaultAuthority : organizations
```

Удаление существующей учетной записи Azure AD на устройстве:

- Чтобы удалить учетную запись Azure AD на устройстве, используйте Средство CleanupWPJ, которое можно скачать [ здесь:CleanupWPJ.zip](https://download.microsoft.com/download/8/e/f/8ef13ae0-6aa8-48a2-8697-5b1711134730/WPJCleanUp.zip).

- Извлеките файл ZIP и запустите **WPJCleanup.cmd**. Этот инструмент запустит правильный исполняемый на основе версии Windows на устройстве.

- С помощью такого механизма, как групповой политики, администратор может запустить команду на устройстве в контексте любого пользователя, который подписан на устройстве.

Чтобы отключить запросы диспетчера веб-учетных записей для регистрации устройства в Azure AD, добавьте это значение реестра: 

- Расположение: HKLM\SOFTWARE\Policies\Microsoft\Windows\WorkplaceJoin
- Тип: DWORD (32 бита)
- Имя: BlockAADWorkplaceJoin
- Данные значения: 1

Наличие этого значения реестра должно блокировать вступление на рабочем месте и запретить пользователям видеть подсказки для перейти на устройство.

## <a name="android"></a>Android

Для Android пользователям потребуется оторегистрируйте и перерегистрируйте свои устройства. Это можно сделать с помощью Microsoft Authenticator или Корпоративный портал приложения. 

- В приложении Microsoft Authenticator пользователи могут перейти к Параметры > **регистрации устройств.** Оттуда пользователи могут отрегистрируйте и перерегистрируйте свое устройство.
 
- С Корпоративный портал пользователи могут перейти на вкладку **Devices** и удалить устройство. После этого повторно зарегистрив устройство с помощью Корпоративный портал.
 
- Пользователи также могут отрегистрируйтесь и перерегистрируйтесь, удалив учетную запись со страницы параметров учетной записи, а затем повторно добавив учетную запись.

Для регистрации и перерегистрации устройства на Android с помощью Microsoft Authenticator приложения:

1.  Откройте приложение Microsoft Authenticator и перейдите **в Параметры**.
2.  Выберите **регистрацию устройства.**
3.  Незарегистрированное устройство, выбрав **Unregister**.
4.  Для **регистрации устройства** перерегистрируйте устройство, введя адрес электронной почты, а затем выберите **Register**.

Для регистрации и перерегистрации устройства Android на странице Android Параметры:

1.  Откройте **устройство Параметры** и перейдите к **учетным записям.**
2.  Выберите учетную запись работы, которую необходимо перерегистрируйте, и выберите **учетную запись Remove.**
3.  После удаления учетной записи со страницы **Учетные** записи выберите учетную запись **Add Account > work account.**
4.  Для **workplace Join** введите адрес электронной почты и выберите **Присоединиться,** чтобы завершить регистрацию устройства.

Для регистрации и перерегистрации устройства на Android с Корпоративный портал:

1.  Запустите Корпоративный портал и перейдите на **вкладку Devices.**
2.  Выберите устройство, чтобы увидеть сведения об устройстве.
3.  Из меню эллипсов (три точки) выберите **Remove Device** и выполните удаление, подтвердив в диалоговом окантовке.
4.  Теперь вы должны войти в систему из Корпоративный портал приложения. Выберите **вход для** повторной регистрации устройства.

Дополнительные сведения о любых действиях, необходимых на этапе миграции этой рабочей нагрузки или воздействии на администрирование или использование, см. в Azure Active Directory (Azure AD) в дополнительных сведениях Azure AD для миграции из [Microsoft Cloud Deutschland](ms-cloud-germany-transition-azure-ad.md).

## <a name="ios"></a>iOS

На устройствах iOS пользователю необходимо вручную удалить все кэшированные учетные записи из Microsoft Authenticator, отрегистрить устройство и выйти из любых родных приложений на устройстве.

### <a name="step-1-if-present-remove-the-account-from-the-microsoft-authenticator-app"></a>Шаг 1. Если она присутствует, удалите учетную запись из Microsoft Authenticator приложения

1. Нажмите на учетную запись в Microsoft Authenticator приложении.
2. Нажмите **значок Параметры** в правом верхнем углу. Если вы не видите значок **Параметры,** возможно, вы не используете последнюю версию Microsoft Authenticator.
3. Нажмите **кнопку Удалить учетную запись.**
4. Нажмите **кнопку Все приложения на этом устройстве**.
 
### <a name="step-2-unregister-the-device-from-the-microsoft-authenticator-app"></a>Шаг 2. Незарегистрированное устройство из Microsoft Authenticator приложения

1. Нажмите значок меню в правом верхнем углу.
2. Нажмите **кнопку Параметры,** а затем **регистрация устройств.**
4. Если учетная запись отображается, нажмите **кнопку Незарегистрированное устройство** **и продолжайте** в диалоговом окте. После этого вы не должны видеть учетную запись.
 
### <a name="step-3-sign-out-from-individual-apps-if-necessary"></a>Шаг 3. При необходимости выпишитесь из отдельных приложений

Пользователи могут перейти к отдельным приложениям, Outlook, Teams и OneDrive, а также удалить учетные записи из этих приложений.
