# <a name="additional-device-information-for-the-migration-from-microsoft-cloud-deutschland"></a>Дополнительные сведения об устройстве для миграции из Microsoft Cloud Deutschland

## <a name="frequently-asked-questions"></a>Вопросы и ответы

**Как узнать, затронута ли моя организация?**

Администраторам следует проверить, есть ли `https://portal.microsoftazure.de` у них зарегистрированные устройства. Если в вашей организации зарегистрированы устройства, это влияет на вас.

**Как это повлияет на пользователей?**

Пользователи с зарегистрированного устройства больше не смогут входить в учетную запись после того, как миграция будет завершена на этапе миграции [Azure AD.](ms-cloud-germany-transition.md#how-is-the-migration-organized)  

Убедитесь, что все ваши устройства зарегистрированы в конечной точке по всему миру, прежде чем организация будет отключена от Microsoft Cloud Deutschland.
  
**Когда пользователи повторно регистрируют свои устройства?**

Для успешной миграции крайне важно только отозарегистрировать и повторно зарегистрировать устройства на этапе миграции "Отдельно от [Microsoft Cloud Deutschland".](ms-cloud-germany-transition.md#how-is-the-migration-organized)

**Как восстановить состояние устройства после миграции?**

Для гибридных устройств с Azure AD и устройств с Windows, которые принадлежат компании, зарегистрированным в Azure AD, администраторы смогут управлять миграцией этих устройств с помощью удаленно активированных процессов, которые будут отрегистрируют старые состояния устройств.
  
Для всех остальных устройств, включая личные устройства с Windows, зарегистрированные в Azure AD, конечный пользователь должен выполнить эти действия вручную. Для устройств, которые присоединились к Azure AD, пользователям необходимо иметь учетную запись локального администратора, чтобы отозарегистрировать, а затем повторно зарегистрировать свои устройства.

Корпорация Майкрософт опубликует инструкции по успешному восстановлению состояния устройства. 
 
**Как узнать, что все мои устройства зарегистрированы в общедоступных облаках?**

Чтобы проверить, зарегистрированы ли устройства в общечном облаке, необходимо экспортировать и скачать список устройств с портала Azure AD в таблицу Excel. Затем отфильтруем устройства, зарегистрированные (с помощью столбца _registeredTime)_ после этапа миграции "Отдельно от [Microsoft Cloud Deutschland".](ms-cloud-germany-transition.md#how-is-the-migration-organized)

Регистрация устройств деактивируется после миграции клиента и не может быть включена или отключена. Если Intune не используется, во sign in to your subscription and run this command to re-activate the option:

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

## <a name="hybrid-azure-ad-join"></a>Гибридное присоединение к Azure AD

### <a name="windows-down-level"></a>Windows более высокого уровня

 Устройства более ранней версии Windows — это устройства с Windows, которые в настоящее время работают под более ранними версиями Windows (например, Windows 8.1 или Windows 7) или windows Server более ранних версий, чем 2019 и 2016. Если такие устройства были зарегистрированы ранее, вам потребуется отозарегистрировать и повторно зарегистрировать эти устройства. 

Чтобы определить, было ли устройство более высокого уровня Windows ранее присоединяться к Azure AD, используйте следующую команду на устройстве:

```console
%programfiles%\Microsoft Workplace Join\autoworkplace /status
```

Если устройство ранее было подключено к Azure AD и имеет сетевое подключение к глобальным конечным точкам Azure AD, вы увидите следующие выходные данные:

```console
+----------------------------------------------------------------------+
| Device Details                                                       |
+----------------------------------------------------------------------+
         DeviceId : AEE2B956-DA62-48D0-BB47-046DD992A110
       Thumbprint : 00fdfa2de5c32feae57489873a13aa6a3ff7433b
             User : user1@<tenantname>.de
Private key state : Okay
     Device state : Unknown
```

Затронутые устройства будут иметь состояние "Устройство" со значением "Неизвестно". Если вы выводит "Устройство не присоединилось" или "Состояние устройства" имеет значение "Окей", игнорируйте следующие рекомендации.

Только для устройств, которые показывают, что устройство присоединилось (с учетом deviceId, thumbprint и так далее) и состояние устройства "Неизвестно", администраторам следует выполнить следующую команду в контексте пользователя домена, который вошел на такое устройство более высокого уровня:

```console
"%programfiles%\Microsoft Workplace Join\autoworkplace /leave"
```

Предыдущее команду необходимо выполнить только один раз для каждого пользователя домена, который войт на устройстве более высокого уровня Windows. Эту команду следует выполнить в контексте пользователя домена, который войт. 

Чтобы не запускать эту команду при последующем под учетной записи пользователя, необходимо соблюдать достаточное внимание. При запуске предыдущей команды очищается состояние местного гибридного компьютера, присоединившегося к Azure AD, для пользователя, который вошел в службу. Кроме того, если компьютер по-прежнему настроен на гибридное использование Azure AD в клиенте, он попытается присоединиться, когда пользователь снова войдет.

### <a name="windows-current"></a>Windows Current

#### <a name="unjoin"></a>Unjoin

Чтобы определить, было ли устройство с Windows 10 ранее присоединяться к Azure AD, запустите следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство гибридное с azure AD, администратор увидит следующие выходные данные:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : YES
```

Если вы выводит "AzureAdJoined : Нет", игнорируйте следующие рекомендации.

Только для устройств, которые показывают, что устройство присоединилось к Azure AD, запустите следующую команду от имени администратора, чтобы удалить его.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущее команду необходимо выполнить только один раз в административном контексте на устройстве с Windows.

#### <a name="hybrid-ad-joinre-registration"></a>Hybrid AD Join\Re-Registration

Устройство автоматически присоединяется к Azure AD без вмешательства пользователя или администратора, если устройство подключено к глобальным конечным точкам Azure AD. 


## <a name="azure-ad-join"></a>Присоединиться к Azure AD

**ВАЖНО!** После коммерческой миграции, которая подразумевает активацию регистрации устройств Azure AD, будет включена основная служба Intune. Если вы заблокировали регистрацию устройств Azure AD перед миграцией, необходимо отключить участников-служб Intune с помощью PowerShell, чтобы снова отключить регистрацию устройств Azure AD на портале Azure AD. Вы можете отключить основное обслуживание Intune с помощью этой команды в модуле Azure Active Directory PowerShell для Graph.

```powershell
Get-AzureADServicePrincipal -All:$true |Where-object -Property AppId -eq "0000000a-0000-0000-c000-000000000000" | Set-AzureADServicePrincipal -AccountEnabled:$false
```

### <a name="unjoin"></a>Unjoin

Чтобы определить, было ли устройство с Windows 10 ранее присоединяться к Azure AD, пользователь или администратор может выполнить следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство присоединяется к Azure AD, пользователь или администратор увидят следующие выходные данные:

```console
+----------------------------------------------------------------------+
| Device State                                                         |
+----------------------------------------------------------------------+
 
             AzureAdJoined : YES
          EnterpriseJoined : NO
              DomainJoined : NO
```

Если вы выводит "AzureAdJoined : NO", игнорируйте следующие рекомендации.

Пользователь: если устройство присоединилось к Azure AD, пользователь может отсоединить устройство от параметров. Убедитесь, что на устройстве есть учетная запись локального администратора, прежде чем отсоединять устройство от Azure AD. Учетная запись локального администратора необходима для обратного вход в устройство.

Администратор: если администратору организации требуется отсоедидинять устройства пользователей, которые являются устройствами, которые являются устройствами, которые являются службами Azure AD, он может сделать это, выдав следующую команду на каждом устройстве с помощью такого механизма, как групповая политика. Администратор должен убедиться, что на устройстве есть учетная запись локального администратора, прежде чем отсоединять устройство от Azure AD. Для обратного вход в устройство необходима учетная запись локального администратора.

```console
%SystemRoot%\system32\dsregcmd.exe /leave
```

Предыдущее команду необходимо выполнить только один раз в административном контексте на устройстве с Windows. 

### <a name="azure-ad-joinre-registration"></a>Регистрация и регистрация в Azure AD

Пользователь может присоединить устройство к Azure AD из параметров Windows: параметры > учетные записи > **Access Work or School > Connect.**
 

## <a name="azure-ad-registered-company-owned"></a>Azure AD Registered (компания принадлежит)

Чтобы определить, зарегистрировано ли устройство с Windows 10 в Azure AD, запустите следующую команду на устройстве:

```console
%SystemRoot%\system32\dsregcmd.exe /status
```

Если устройство зарегистрировано в Azure AD, вы увидите следующие выходные данные:

```console
+----------------------------------------------------------------------+
| User State                                                           |
+----------------------------------------------------------------------+
           WorkplaceJoined : YES
          WamDefaultSet : NO
          WamDefaultAuthority : organizations
```

Чтобы удалить существующую учетную запись, зарегистрированную в Azure AD, на устройстве:

- Чтобы удалить зарегистрированную в Azure AD учетную запись на устройстве, используйте CleanupWPJ, средство, которое можно скачать здесь: [CleanupWPJ.zip. ](https://download.microsoft.com/download/8/e/f/8ef13ae0-6aa8-48a2-8697-5b1711134730/WPJCleanUp.zip)

- Извлеките ZIP-файл и запустите **файл WPJCleanup.cmd.** Это средство запустит нужный исполняемый исполняемый инструмент в зависимости от версии Windows на устройстве.

- Используя такой механизм, как групповая политика, администратор может выполнить команду на устройстве в контексте любого пользователя, который ввел учетные данные на устройстве.

Чтобы отключить запросы диспетчера учетных записей на регистрацию устройства в Azure AD, добавьте это значение реестра: 

- Расположение: HKLM\SOFTWARE\Policies\Microsoft\Windows\WorkplaceJoin
- Тип: DWORD (32 бита)
- Имя: BlockAADWorkplaceJoin
- Значение: 1

Наличие этого значения реестра должно блокировать участие на рабочем месте и запретить пользователям видеть запросы на присоединиться к устройству.

## <a name="android"></a>Android

Для Android пользователям потребуется отозарегистрировать и повторно зарегистрировать свои устройства. Это можно сделать с помощью приложения Microsoft Authenticator или приложения "Портал компании". 

- В приложении Microsoft Authenticator пользователи могут перейти в > **регистрации устройств.** После этого пользователи могут отозарегистрировать и повторно зарегистрировать свое устройство.
 
- На портале компании пользователи могут перейти на вкладку **"Устройства"** и удалить устройство. После этого повторно зарегистрите устройство с помощью портала компании.
 
- Пользователи также могут отозарегистрировать и повторно зарегистрироваться, удалив учетную запись со страницы параметров учетной записи, а затем повторно добавив эту учетную запись.

Чтобы отозарегистрировать и повторно зарегистрировать устройство на Android с помощью приложения Microsoft Authenticator:

1.  Откройте приложение Microsoft Authenticator и перейдите в **"Параметры".**
2.  Выберите **регистрацию устройства.**
3.  Отоберем регистрацию устройства, выбрав **"Отрегистрить".**
4.  Для **регистрации устройства** повторно зарегистрируйте устройство, введя свой адрес электронной почты, а затем выберите **"Регистрация".**

Чтобы отозарегистрировать и повторно зарегистрировать устройство Android на странице "Параметры Android":

1.  Откройте **"Параметры устройства"** и перейдите в **"Учетные записи".**
2.  Выберите учетную запись, которую нужно повторно зарегистрировать, и выберите **"Удалить учетную запись".**
3.  После удаления учетной записи  на странице "Учетные записи" выберите "Добавить учетную **запись > work".**
4.  Для **workplace Join** введите адрес электронной почты и выберите **"Присоединиться",** чтобы завершить регистрацию устройства.

Чтобы отозарегистрировать и повторно зарегистрировать устройство на Android с портала компании:

1.  Запустите портал компании и перейдите на **вкладку "Устройства".**
2.  Выберите устройство, чтобы увидеть сведения об устройстве.
3.  В меню многоточий (три точки) выберите "Удалить устройство" и завершите удаление, подтвердив это в диалоговом окле. 
4.  Теперь вы должны выйти из приложения "Портал компании". Выберите **"Войти",** чтобы повторно зарегистрировать устройство.

Дополнительные сведения о любых действиях, необходимых на этапе миграции этой рабочей нагрузки, а также о влиянии на администрирование или использование, см. в дополнительных сведениях о Azure AD для миграции из [Microsoft Cloud Deutschland.](ms-cloud-germany-transition-azure-ad.md)

## <a name="ios"></a>iOS

На устройствах с iOS пользователю потребуется вручную удалить кэшированные учетные записи из Средства проверки подлинности Майкрософт, отрегистратора устройства и выйти из любых приложений на устройстве.

### <a name="step-1-if-present-remove-the-account-from-the-microsoft-authenticator-app"></a>Шаг 1. При его наступлению удалите учетную запись из приложения Microsoft Authenticator

1. Коснитесь учетной записи в приложении Microsoft Authenticator.
2. **Коснитесь значка "Параметры"** в правом верхнем углу. Если вы не видите значок **"Параметры",** возможно, вы не используете последнюю версию Microsoft Authenticator.
3. Нажмите **кнопку "Удалить учетную запись".**
4. **Коснитесь всех приложений на этом устройстве.**
 
### <a name="step-2-unregister-the-device-from-the-microsoft-authenticator-app"></a>Шаг 2. Unregister the device from the Microsoft Authenticator app

1. Коснитесь значка меню в правом верхнем углу.
2. **Коснитесь параметров,** а **затем регистрация устройства.**
4. Если отображается ваша учетная запись, коснитесь устройства **"Unregister"** и **"Continue** in the dialog". После этого учетная запись не должна отсвеяться.
 
### <a name="step-3-sign-out-from-individual-apps-if-necessary"></a>Шаг 3. При необходимости выход из отдельных приложений

Пользователи могут перейти к отдельным приложениям, например Outlook, Teams и OneDrive, и удалить учетные записи из этих приложений.
