## <a name="tenant-id-filtering-and-tenant-id-term-prefixing"></a><span data-ttu-id="14dbd-121">Фильтрация ИД клиента и префикс идентификатора клиента</span><span class="sxs-lookup"><span data-stu-id="14dbd-121">Tenant ID Filtering and Tenant ID Term Prefixing</span></span>

<span data-ttu-id="14dbd-122">Поиск префиксов для каждого термина, индексируемого в полнотекстовом индексе, с ИДЕНТИФИКАТОРом клиента.</span><span class="sxs-lookup"><span data-stu-id="14dbd-122">Search prefixes every term that is indexed in the full-text index with the tenant ID.</span></span> <span data-ttu-id="14dbd-123">Например, если термин "*foo*" индексируется для клиента с идентификатором "*123*", запись в полнотекстовом индексе будет иметь значение "*123foo".*</span><span class="sxs-lookup"><span data-stu-id="14dbd-123">For example, when the term "*foo*" is indexed for a tenant with an ID of "*123*", the entry in the full-text index is "*123foo.*"</span></span>

<span data-ttu-id="14dbd-124">Каждый запрос преобразуется, чтобы включить идентификатор клиента, используя процесс, называемый фильтрацией кода клиента.</span><span class="sxs-lookup"><span data-stu-id="14dbd-124">Every query is converted to include the tenant ID using a process called tenant ID filtering.</span></span> <span data-ttu-id="14dbd-125">Например, запрос "*foo*" преобразуется в "<*GUID*>. *foo* И *tenantID*: <*GUID*> ", где <*GUID*> представляет клиент, выполняющий запрос.</span><span class="sxs-lookup"><span data-stu-id="14dbd-125">For example, the query "*foo*" is converted to "<*guid*>.*foo* AND *tenantID*:<*guid*>", where <*guid*> represents the tenant performing the query.</span></span> <span data-ttu-id="14dbd-126">Это преобразование запроса выполняется в каждом узле индекса, и на него не могут влиять ни запросы, ни обработка контента.</span><span class="sxs-lookup"><span data-stu-id="14dbd-126">This query conversion occurs within each index node and neither query nor content processing can influence it.</span></span> <span data-ttu-id="14dbd-127">Так как идентификатор клиента добавляется в каждый запрос, то частота терминов в других клиентах не может быть определена при поиске ранжирования по наилучшему совпадению в одном клиенте.</span><span class="sxs-lookup"><span data-stu-id="14dbd-127">Because the tenant ID is added to every query, the frequency of a term in other tenants can't be inferred by looking at best match ranking in one tenant.</span></span>
