# <a name="sharepoint-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки SharePoint Server 2007;

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

**10 октября 2017** г. поддержка SharePoint Server 2007 Microsoft Office 2007. Если вы не переходить с SharePoint Server 2007 на Microsoft 365 или более новую версию SharePoint Server в локальной версии, самое время приступить к планированию. В этой статье данная статья содержит ресурсы, которые помогут перенести данные в SharePoint Online или обновить sharePoint Server локально.
  
## <a name="what-does-end-of-support-mean"></a>Что означает *окончание поддержки?*

SharePoint Server, как и большинство продуктов Майкрософт, имеет жизненный цикл поддержки, в ходе которого корпорация Майкрософт предоставляет новые функции, исправления ошибок, исправления для системы безопасности и так далее. Этот жизненный цикл обычно длится 10 лет с начального выпуска продукта. Завершение этого жизненного цикла называется завершением поддержки продукта. После окончания поддержки корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка для проблем, которые могут возникнуть.
    
- Исправления ошибок, которые могут повлиять на стабильность и доступность сервера.
    
- Исправления безопасности для уязвимостей, которые могут сделать сервер уязвимым к нарушению безопасности.
    
- Обновления часовой пояс.
    
Ферма SharePoint Server 2007 будет работать после 10 октября 2017 г., но для продукта не будут выпущены дополнительные обновления, исправления и исправления, включая исправления и исправления для системы безопасности. Служба поддержки Майкрософт полностью перенацелила свои усилия на поддержку более последних версий продукта. Так как установка больше не поддерживается или не обновляется, необходимо обновить продукт или перенести важные данные.
  
> [!TIP]
> Если вы еще не запланировали обновление или миграцию, см. некоторые примеры начала миграции [SharePoint 2007.](sharepoint-2007-migration-options.md) Вы также можете найти партнеров [Майкрософт,](https://go.microsoft.com/fwlink/?linkid=841249) которые могут помочь в обновлении или миграции Microsoft 365 (или и того, и другого).
  
Дополнительные сведения о серверах Office 2007 и завершении поддержки см. в "Ресурсах", которые помогут вам обновить серверы и клиенты [Office 2007.](upgrade-from-office-2007-servers-and-products.md)
  
## <a name="what-are-my-options"></a>Какие у меня есть варианты?

Первой остановкой должен быть [сайт жизненного цикла продукта.](https://go.microsoft.com/fwlink/?linkid=843148) Если у вас есть продукт Майкрософт, который устарел, проверьте дату окончания поддержки, чтобы запланировать обновление или миграцию в течение года. При выборе следующего шага подумайте, какие функции продукта будут достаточно хорошо, лучше и лучше. Пример:

|SharePoint Server 2010  <br/> |SharePoint Server 2013  <br/> |SharePoint Online  <br/> |
||Гибридная среда SharePoint  <br/> |SharePoint Server 2016  <br/> |

Если выбрать вариант "достаточно хорошо", скоро потребуется начать планирование другого обновления после завершения миграции с SharePoint Server 2007. 

>[!NOTE] 
>Даты окончания поддержки могут быть измениться. Проверьте сайт [жизненного цикла продукта.](https://support.microsoft.com/lifecycle)
  
## <a name="where-can-i-go-next"></a>Что делать дальше?

SharePoint Server можно установить локально на ваших серверах. Вы также можете использовать SharePoint Online, веб-службу, которая входит в состав Microsoft 365. Доступны следующие параметры:
  
- Миграция в SharePoint Online.
    
- Обновив SharePoint Server локально.
    
- Сделайте оба из вышеперечисленного.
    
- Реализация [гибридного решения SharePoint.](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)
    
Следует помнить о скрытых затратах, связанных с обслуживанием фермы серверов, обслуживанием или переносом настроек и обновлением оборудования, необходимого SharePoint Server. Наличие локальной фермы SharePoint Server является полезным, если это необходимо. Но если вы запустите ферму на устаревших серверах SharePoint Server без интенсивной настройки, вы сможете воспользоваться преимуществами миграции в SharePoint Online.
  
> [!IMPORTANT]
> Есть и другой вариант, если контент в SharePoint 2007 используется редко. Некоторые администраторы SharePoint решили создать подписку на Microsoft 365, настроить новый сайт SharePoint Online, а затем очистить SharePoint 2007, убирая только основные документы на новые сайты SharePoint Online. После этого данные могут быть стекаются из сайта SharePoint 2007 в архивы. Подумайте, как пользователи работают с данными из установки SharePoint 2007. Для управления вашими потребностями могут быть творческие способы.
  
|**SharePoint Online (SPO)**|**Локальное sharePoint Server**|
|:-----|:-----|
|Высокая стоимость во времени (планирование, выполнение/проверка)  <br/> |Высокая стоимость во времени (планирование, выполнение/проверка)  <br/> |

При переходе на Microsoft 365 одновселевой переход будет иметь более значительные затраты на предварительные затраты, при этом вы упорядочайте данные и решаете, что необходимо перенести в облако и что оставить. Однако последующие обновления будут автоматически, и вам больше не придется управлять обновлениями оборудования и программного обеспечения. Кроме того, время работы фермы будет с помощью соглашения об уровне обслуживания[(SLA)](https://go.microsoft.com/fwlink/?linkid=843153)Майкрософт.

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локального сервера SharePoint Server

В обновлениях SharePoint нет способа пропустить версии. Обновления обновляют последовательно:
  
- SharePoint 2007 \> SharePoint Server 2010 \> SharePoint Server 2013 \> SharePoint Server 2016
   
Перейти с SharePoint 2007 на SharePoint Server 2016 означает значительные затраты времени и затраты на оборудование (также необходимо обновить SQL серверы), программное обеспечение и администрирование. Настройки необходимо обновить или отоимдить.
  
> [!NOTE]
> Можно поддерживать ферму SharePoint 2007, устанавливать ферму SharePoint Server 2016 на новом оборудовании (поэтому отдельные фермы работают рядом), а затем планировать и выполнять перенос контента вручную (например, для скачивания и повторной отправки контента). Однако остерегайтесь некоторых ошибок ручного перемещения, таких как перемещение документов, заменяющих последнюю измененную учетную запись псевдонимом учетной записи, которая перемещается вручную. Также рассмотрите работу, которую необходимо сделать заранее, например, воссоздание сайтов, подсайтов, разрешений и структур списков. Заранее продумайте, какие данные можно переместить в хранилище или удалить, чтобы снизить влияние миграции.
  
Перед обновлением важно очистить среду. Перед обновлением необходимо быть уверены, что существующая ферма работает и, конечно, перед списанием.

Подумайте, какие данные можно переместить в хранилище или удалить, чтобы снизить влияние миграции.
  
Очистите среду перед обновлением. Перед обновлением и перед списанием необходимо быть уверены, что существующая ферма работает.

### <a name="create-a-sharepoint-hybrid-solution-between-sharepoint-online-and-on-premises"></a>Создание гибридного решения SharePoint между SharePoint Online и локальной версией

Если ответ на ваши потребности в миграции находится где-то между самообустояемой локальной версией и более низкой стоимостью владения, предложенной SharePoint Online, вы можете подключить фермы SharePoint Server 2013 или 2016 к SharePoint Online с помощью гибридной системы. [Узнайте о гибридных решениях SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)
  
Если вы решите, что гибридная ферма SharePoint Server будет пользоваться преимуществами вашего бизнеса, ознакомьтесь с существующими типами гибридной конфигурации и настройкой подключения между локальной фермой SharePoint и подпиской на Microsoft 365.
  
Один из хороший способ узнать, как это работает, — создать среду тестирования и тестирования Microsoft 365, которую можно настроить с помощью руководств по лаборатории [тестирования.](m365-enterprise-test-lab-guides.md) Получив пробную или приобретенную подписку на Microsoft 365, вы можете создать в SharePoint Online коллекции веб-сайтов, веб-сайты и библиотеки документов, в которые можно перенести данные. Вы можете перейти вручную, с помощью API миграции или, если вы хотите перенести содержимое содержимого my Site в OneDrive для бизнеса, с помощью мастера гибридной миграции.
  
> [!NOTE]
> Помните, что для использования гибридного варианта ферму SharePoint 2007 необходимо обновить локально до SharePoint Server 2013 или SharePoint Server 2016.
