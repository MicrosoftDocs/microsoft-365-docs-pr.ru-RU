# <a name="sharepoint-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки SharePoint Server 2007;

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

**10 октября 2017** г. Microsoft Office SharePoint Server 2007 достиг конца поддержки. Если вы еще не выполнили миграцию из SharePoint Server 2007 в Microsoft 365 или более новую версию SharePoint Server, теперь настало время начать планирование. В этой статье представлены ресурсы, которые помогут вам перенести данные в SharePoint Online или обновить локальный сервер SharePoint Server.
  
## <a name="what-does-end-of-support-mean"></a>Что означает *Окончание поддержки* ?

В SharePoint Server, как и у большинства продуктов Майкрософт, предусмотрен жизненный цикл поддержки, в течение которого корпорация Майкрософт предоставляет новые функции, исправления ошибок, исправления для системы безопасности и т. д. Этот жизненный цикл обычно длится через 10 лет от первоначального выпуска продукта. Окончание этого жизненного цикла называется окончанием поддержки продукта. После завершения поддержки Корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка проблем, которые могут возникнуть.
    
- Исправление ошибок для проблем, которые могут повлиять на стабильность и удобство использования сервера.
    
- Исправления для системы безопасности уязвимостей, которые могут сделать сервер уязвимым для нарушений безопасности.
    
- Обновления часового пояса.
    
Ферма SharePoint Server 2007 будет по-прежнему работать после 10 октября 2017, но при этом не будут выпущены дополнительные обновления, исправления или исправления, в том числе исправления и исправления для системы безопасности. Служба поддержки Майкрософт полностью изменила возможности поддержки до последних версий продукта. Так как ваша установка больше не поддерживается или не исправлена, следует обновить продукт или перенести важные данные.
  
> [!TIP]
> Если вы еще не планировали обновление или миграцию, ознакомьтесь со статьей: [SharePoint 2007 Migration варианты миграции, которые необходимо учитывать](sharepoint-2007-migration-options.md) при запуске. Кроме того, можно выполнить поиск [партнеров Майкрософт](https://go.microsoft.com/fwlink/?linkid=841249) , которые могут помочь при обновлении или миграции Microsoft 365 (или в обоих вариантах).
  
Дополнительные сведения о серверах Office 2007 и о завершении поддержки можно найти в статье [ресурсы, которые помогут вам выполнить обновление серверов и клиентов office 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-are-my-options"></a>Что такое параметры?

Когда вы выбираете следующий шаг, определите, какие функции продукта хорошо подходят, лучше и лучше. Пример:

## <a name="where-can-i-go-next"></a>Что делать дальше?

SharePoint Server можно установить локально на собственных серверах. Или вы можете использовать SharePoint Online, который является веб-службой, которая входит в состав Microsoft 365. Доступны следующие параметры:
  
- Переход на SharePoint Online.
    
- Обновление локальной версии SharePoint Server.
    
- Выполните оба описанных выше действия.
    
- Реализация [гибридного решения SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx) .
    
Следует учитывать скрытые затраты, связанные с обслуживанием фермы серверов, обслуживанием и перенос настроек, а также обновление оборудования, необходимого для SharePoint Server. Использование локальной фермы SharePoint Server в случае необходимости. Но если вы выполняете ферму на старых серверах SharePoint без высокой настройки, вы можете воспользоваться преимуществами миграции в SharePoint Online.
  
> [!IMPORTANT]
> Существует еще один вариант, если контент в SharePoint 2007 нередко используется. Некоторые администраторы SharePoint решили создать подписку на Microsoft 365, настроить новый сайт SharePoint Online, а затем удалить их из SharePoint 2007, изменяя только важные документы на новые сайты SharePoint Online. После этого данные можно будет расрядиться из сайта SharePoint 2007 в архивы. Рассмотрите возможность работы пользователей с данными из установки SharePoint 2007. Для управления вашими потребностями могут быть изобретательными способами.
  
|**SharePoint Online (SPO)**|**Локальная среда SharePoint Server**|
|:-----|:-----|
|Высокая стоимость во время (планирование, выполнение и проверка)  <br/> |Высокая стоимость во время (планирование, выполнение и проверка)  <br/> |
|Снижение затрат на фонды (без приобретения оборудования)  <br/> |Более высокие затраты на фонды (оборудование + ИТ/администраторы)  <br/> |
|Одноразовые затраты в процессе миграции  <br/> |Одноразовые затраты, повторяющиеся в будущие миграции  <br/> |
|Снижение совокупной стоимости владения и обслуживания  <br/> |Большая совокупная стоимость владения/обслуживания  <br/> |
   
При переходе на Microsoft 365 одноразовое перемещение будет иметь более тяжелую стоимость, а вы организуете данные и решаете, что следует предпринять в облаке и что следует оставить позади. Однако будущие обновления будут выполняться автоматически, и вам больше не потребуется управлять оборудованием и программным обеспечением. Кроме того, время ожидания фермы будет заархивировано соглашением об уровне обслуживания ([SLA](https://go.microsoft.com/fwlink/?linkid=843153)) Майкрософт.

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локальной версии SharePoint Server

Невозможно пропускать версии при обновлении SharePoint. Последовательные обновления:
  
- SharePoint 2007 \> SharePoint server 2010 \> sharepoint Server 2013 \> SharePoint Server 2016
   
Переход с SharePoint 2007 на SharePoint Server 2016 означает значительную окупаемость времени и включает затраты на оборудование (серверы SQL Server также необходимо обновить), программное обеспечение и администрирование. Настройка должна быть обновлена или отменена.
  
> [!NOTE]
> Вы можете сохранить свою конечную ферму SharePoint 2007, установить ферму SharePoint Server 2016 на новом оборудовании (чтобы отдельные фермы выполнялись параллельно), а затем спланировать и выполнить миграцию контента вручную (например, для скачивания и повторной загрузки контента). Но помните о некоторых возможностях ручного перемещения, таких как перемещение документов, замена учетной записи последнего изменения псевдонимом учетной записи, выполняющей перемещение вручную. Кроме того, следует учитывать работу, которая должна быть выполнена заранее, например повторное создание сайтов, дочерних сайтов, разрешений и структур списков. Изучите данные, которые можно переместить в хранилище или удалить, чтобы уменьшить влияние миграции.
  
Перед обновлением необходимо очистить среду. Перед началом обновления необходимо убедиться в том, что существующая ферма работает, и определенно перед ликвидацией!

Переход с SharePoint 2007 на SharePoint Server 2016 приводит к значительным затратам времени и потребует затрат на оборудование (серверы SQL Server также необходимо обновить), программное обеспечение и администрирование. Настройка должна быть обновлена или отменена.
  
> [!NOTE]
> Вы можете сохранить свою конечную ферму SharePoint 2007, установить ферму SharePoint Server 2016 на новом оборудовании (чтобы отдельные фермы выполнялись параллельно), а затем спланировать и выполнить миграцию контента вручную (например, для скачивания и повторной загрузки контента). Но помните о возможных возможностях ручного перемещения, таких как перемещение документов, замена последней измененной учетной записи на псевдоним учетной записи, выполняющей перемещение вручную, и работу, которая должна выполняться заранее, например повторное создание сайтов, дочерних сайтов, разрешений и структур списков. Определите, какие данные можно переместить в хранилище или удалить, чтобы уменьшить влияние миграции.
  
Очистите среду перед обновлением. Перед началом обновления необходимо убедиться в том, что существующая ферма работает.

### <a name="create-a-sharepoint-hybrid-solution-between-sharepoint-online-and-on-premises"></a>Создание гибридного решения SharePoint между SharePoint Online и локальной системой

Если ответ на ваш запрос на миграцию находится где-то в локальной среде, а более низкие затраты на владение, предоставляемые SharePoint Online, вы можете подключить фермы SharePoint Server 2013 или 2016 к SharePoint Online с помощью гибридной среды. [Сведения о гибридных решениях SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)
  
Если вы решили, что Гибридная ферма SharePoint Server будет использовать преимущества вашей организации, ознакомьтесь с существующими типами гибридных развертываний и настройками подключения между локальной фермой SharePoint и подпиской на Microsoft 365.
  
Чтобы понять, как это работает, можно создать среду разработки и тестирования Microsoft 365, которую можно настроить с помощью [руководств по лаборатории тестирования](m365-enterprise-test-lab-guides.md). После получения пробной или приобретенной подписки на Microsoft 365 вы можете создать семейства веб-сайтов, веб-сайты и библиотеки документов в SharePoint Online, в которые можно перенести данные. Вы можете выполнить миграцию вручную, используя API миграции, или, если вы хотите перенести контент личного сайта в OneDrive для бизнеса с помощью мастера гибридной конфигурации.
  
> [!NOTE]
> Помните, что для использования гибридного развертывания ферма SharePoint 2007 должна быть обновлена локально до SharePoint Server 2013 или SharePoint Server 2016.
