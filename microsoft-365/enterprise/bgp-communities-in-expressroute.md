# <a name="using-bgp-communities-in-expressroute-for-office-365-scenarios"></a>Использование сообществ BGP в ExpressRoute для сценариев Office 365

Подключение к Office 365 с помощью Azure ExpressRoute основано на объявлениях BGP определенных IP-подсетей, которые представляют сети, в которых развернуты конечные точки Office 365. Из-за глобального характера Office 365 и количества служб, составляющих Office 365, клиентам часто приходится управлять рекламой, которую они принимают в своей сети. Уменьшение числа IP-подсетей; которые в оставшейся части этой статьи называются префиксами IP, чтобы в соответствии с терминологией управления сетями BGP, клиенты могут получить следующие конечные цели:
  
- Управляйте числом принятых **префиксов IP-** Клиенты с внутренней сетевой инфраструктурой или сетевым оператором, который поддерживает только ограниченное число IP-префиксов, и клиенты с сетевым оператором, который взимает плату за прием префиксов выше ограниченного числа, хотят оценить общее количество префиксов, уже объявленных в сети, и выбрать приложения Office 365, которые лучше всего подходят для ExpressRoute.

- Управление пропускной способностью, необходимой для канала **Azure ExpressRoute.** Клиентам может потребоваться управлять конвертом пропускной способности служб Office 365 по пути ExpressRoute и Интернету. Это позволяет клиентам резервировать пропускную способность ExpressRoute для определенных приложений, таких как Skype для бизнеса, и маршрутировать остальные приложения Office 365 по пути к Интернету.

Чтобы помочь клиентам с этими целями, префиксы IP Office 365, рекламируемые через ExpressRoute, помечаются значениями сообщества BGP, определенными для службы, как показано в примере ниже.
  
> [!NOTE]
> Следует ожидать, что в значение сообщества будет включен определенный сетевой трафик, связанный с другими приложениями. Это ожидаемое поведение для глобального предложения "Программное обеспечение как услуга" с общими службами и центрами обработки данных. По возможности это было свернуто с помощью двух вышеперечисленных целей, управляя количеством префиксов и/или пропускной способностью.

|**Служба**|**Значение сообщества BGP**|**Примечания**|
|:-----|:-----|:-----|
|Exchange Online\*  <br/> |12076:5010  <br/> |Включает службы Exchange и EOP\*  <br/> |
|SharePoint Online\*  <br/> |12076:5020  <br/> |SharePoint Online  <br/> |
|Skype для бизнеса\*  <br/> |12076:5030  <br/> |Skype для бизнеса Online & Microsoft Teams  <br/> |
|Другие службы Office 365\*  <br/> |12076:5100  <br/> |Включает Azure Active Directory (сценарии проверки подлинности и синхронизации каталогов), а также службы портала Office 365  <br/> |
|\*Область сценариев служб, включенных в ExpressRoute, задокументирована в статье о конечных [точках Office 365.](https://aka.ms/o365endpoints)  <br/> \*\*В будущем могут быть добавлены дополнительные службы и значения сообщества BGP. [См. текущий список сообществ BGP.](https://azure.microsoft.com/documentation/articles/expressroute-routing/)  <br/> |

## <a name="what-are-the-most-common-scenarios-for-using-bgp-communities"></a>Какие наиболее распространенные сценарии использования сообществ BGP?

Клиенты могут использовать сообщества BGP для управления группами IP-префиксов, которые принимаются их сетью через Azure ExpressRoute, что влияет на общее количество префиксов IP и ожидаемый конверт пропускной способности определенных служб Office 365. Важно понимать, что весь трафик Office 365 требует трафика, связанного с Интернетом, независимо от использования Azure ExpressRoute или сообщества BGP. Следующие три сценария являются наиболее распространенным использованием этой функции.
  
### <a name="scenario-1-minimizing-the-number-of-office-365-ip-prefixes"></a>Сценарий 1. Минимизация количества IP-префиксов Office 365

Корпорация Contoso — это компания с 50 000 сотрудников, которая в настоящее время использует Office 365 для Exchange Online и SharePoint Online. При проверке требований ExpressRoute contoso определяет, что сетевые устройства во многих регионах не могут обрабатывать размеры таблиц маршрутов, превышжающие 100 дополнительных записей маршрута. Компания Contoso проанализировала общее количество IP-префиксов, которые ExpressRoute будет объявлять для всего набора служб Office 365, и добавила, что оно превышает 100. Чтобы остаться в области 100 дополнительных записей маршрута, в contoso использование ExpressRoute для Office 365 можно сделать только значением сообщества BGP SharePoint Online, 12076:5020, полученным через пиринг Microsoft ExpressRoute.

|**Используемый тег сообщества BGP**|**Маршрут функциональность через Azure ExpressRoute**|**Необходимые интернет-маршруты**|
|:-----|:-----|:-----|
|SharePoint  <br/> (12076:5020)  <br/> |SharePoint Online &amp; OneDrive для бизнеса  <br/> | Запросы DNS, CRL, &amp; CDN  <br/>  Все остальные службы Office 365 не поддерживаются в Azure ExpressRoute  <br/>  Все остальные облачные службы Майкрософт  <br/>  Портал Office 365, проверка подлинности Office 365, &amp; Office в браузере  <br/>  Exchange Online, Exchange Online Protection и Skype для бизнеса Online  <br/> |

> [!NOTE]
> Чтобы добиться меньшего количества префиксов для каждой службы, будет сохраняться минимальное перекрытие между службами. Это поведение не является отклонением от нормы.
  
### <a name="scenario-2-scoping-expressroute-and-internal-bandwidth-use-to-some-office-365-services"></a>Сценарий 2. Ограничение области ExpressRoute и использование внутренней пропускной способности для некоторых служб Office 365

Компания Fabrikam Inc. — крупное многоэтапное предприятие с распределенной разнородной сетью — это подписчик многих служб Office 365, в том числе; Exchange Online, SharePoint Online и Skype для бизнеса Online. Внутренняя инфраструктура маршрутов Fabrikam может обрабатывать тысячи IP-префиксов в таблицах маршрутов; Однако компании Fabrikam нужно только подготовка ExpressRoute и внутренней пропускной способности для приложений Office 365, которые наиболее чувствительны к производительности сети, и использовать существующую пропускную способность Интернета для всех остальных приложений Office 365.
  
По этой причине Fabrikam передает своей пропускной способности Azure ExpressRoute только значение сообщества BGP Skype для бизнеса Online , 12076:5030, полученное через пиринг ExpressRoute Майкрософт. Оставшийся сетевой трафик, связанный с Office 365, продолжает использовать точки интернет-трафика.

|**Используемый тег сообщества BGP**|**Маршрут функциональность через Azure ExpressRoute**|**Необходимые интернет-маршруты**|
|:-----|:-----|:-----|
|Skype для бизнеса  <br/> (12076:5030)  <br/> |Сигналы SIP Skype, загрузки, голосовая, видеосвязь и общий доступ к рабочему столу  <br/> | Запросы DNS, CRL, &amp; CDN  <br/>  Все остальные службы Office 365 не поддерживаются в Azure ExpressRoute  <br/>  Все остальные облачные службы Майкрософт  <br/>  Портал Office 365, проверка подлинности Office 365, &amp; Office в браузере  <br/>  Телеметрия Skype для бизнеса, краткие советы по клиенту Skype, подключение к общедоступным мгновенным подключениям  <br/>  Exchange Online, Exchange Online Protection и SharePoint Online  <br/> |

### <a name="scenario-3-scoping-azure-expressroute-for-office-365-services-only"></a>Сценарий 3. Использование области azure ExpressRoute только для служб Office 365

Банк Woodgrove Bank является клиентом нескольких облачных служб Майкрософт, включая Office 365. После оценки пропускной способности сети и потребления банка Woodgrove Bank решает развернуть Azure ExpressRoute в качестве предпочтительного пути для поддерживаемых служб Office 365. Таблицы маршрутов могут поддерживать полный набор префиксов IP-адресов Office 365, а маршруты Azure ExpressRoute, которые они создали, поддерживают все прогнозируемые потребности в пропускной способности и задержке.
  
Чтобы обеспечить сетевой трафик, связанный с облачными службами Майкрософт, кроме Office 365, Банк Woodgrove Bank использует ExpressRoute для Office 365 для всех IP-префиксов, помеченных определенными значениями сообщества BGP в Office 365: 12076:5010, 12076:5020, 12076:5030, 12076:5100.

|**Используемый тег сообщества BGP**|**Маршрут функциональность через Azure ExpressRoute**|**Необходимые интернет-маршруты**|
|:-----|:-----|:-----|
|Exchange, Skype для бизнеса & Microsoft Teams, SharePoint и &amp; другие службы  <br/> (12076:5010, 12076:5020, 12076:5030, 12076:5100)  <br/> |Exchange Online &amp; Exchange Online Protection  <br/> SharePoint Online &amp; OneDrive для бизнеса  <br/> Сигналы SIP Skype, загрузки, голосовая, видеосвязь и общий доступ к рабочему столу  <br/> Портал Office 365, проверка подлинности Office 365, &amp; Office в браузере  <br/> | Запросы DNS, CRL, &amp; CDN  <br/>  Все остальные службы Office 365 не поддерживаются в Azure ExpressRoute  <br/>  Все остальные облачные службы Майкрософт  <br/> |

## <a name="key-planning-considerations-to-using-bgp-communities"></a>Основные вопросы планирования использования сообществ BGP

Клиенты, которые решили воспользоваться преимуществами сообществ BGP, чтобы повлиять на то, как ExpressRoute объявляется и распространяется через клиентскую сеть, должны принимать во внимание следующие соображения:
  
- При использовании сообществ BGP в проекте сети важно обеспечить сохранение симметрии маршрута. В некоторых случаях при добавлении или удалении сообществ BGP может возникнуть ситуация, в которой нарушена симметричная маршрутивка, и необходимо обновить конфигурацию маршрутов, чтобы восстановить симметричную маршрутику.

- Использование области Azure ExpressRoute с помощью значений сообщества BGP — это действие клиента. Корпорация Майкрософт будет объявлять все IP-префиксы, связанные с пиринговой связью, независимо от области, настроенной клиентом.

- Azure ExpressRoute не поддерживает какие-либо действия в сети Майкрософт на основе сообществ BGP, которые были назначены клиенту.

- Префиксы IP, используемые в Office 365, помечаются только значениями сообщества BGP, определенными службами. Сообщества BGP для определенных расположений не поддерживаются. Службы Office 365 имеют глобальный характер, фильтрация префиксов в зависимости от расположения клиента или данных в облаке Office 365 не поддерживается. Рекомендуется настроить сеть для координации самого короткого или предпочтительного сетевого пути из сетевого расположения пользователя в глобальную сеть Майкрософт независимо от физического расположения IP-адреса запрашиваемой службы Office 365.

- Префиксы IP, включенные в каждое значение сообщества BGP, представляют подсеть, которая содержит IP-адреса для приложения Office 365, связанного со значением. В некоторых случаях несколько приложений Office 365 имеет IP-адреса в подсети, в результате чего префикс IP существует в нескольких значениях сообщества. Это происходит редко из-за фрагментации выделения и не влияет на количество префиксов и цели управления пропускной способностью. Клиентам рекомендуется использовать подход "разрешить то, что необходимо", а не "запретить то, что не нужно" при использовании сообществ BGP для Office 365, чтобы свести к минимуму эффект.

- Использование сообществ BGP не меняет требования или конфигурацию сетевого подключения, необходимые для использования Office 365. Клиенты, желающие получить доступ к Office 365, по-прежнему должны иметь доступ к Интернету.

- Использование области Azure ExpressRoute с сообществами BGP влияет только на маршруты, которые ваша внутренняя сеть может видеть через пиринговую связь Майкрософт. Вам может потребоваться внести дополнительные настройки уровня приложений, такие как использование PAC или конфигурации WPAD в сочетании с маршрутирой с за областью действия.

- Помимо использования сообществ BGP, назначаемой Майкрософт, клиенты могут назначать собственные сообщества BGP префиксам IP Office 365, которые прошли через Azure ExpressRoute, чтобы повлиять на внутреннюю маршрутику. Распространенным вариантом использования является назначение сообщества BGP на основе расположения всем маршрутам, которые проходят через каждое заданное расположение пиринга ExpressRoute, а затем использование этой информации в сети клиента для координации самого короткого или предпочтительного сетевого пути в сеть Майкрософт. Использование сообществ BGP, которые были назначены клиенту, в сценариях ExpressRoute для Office 365 находится вне области контроля или видимости Корпорации Майкрософт.
