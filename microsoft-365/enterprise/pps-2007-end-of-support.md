## <a name="what-does-end-of-support-mean"></a>Что означает *завершение поддержки?*

Как и большинство продуктов Майкрософт, PerformancePoint Server 2007 с sp3, программным обеспечением ProClarity и SharePoint Server 2007 SP3 имеют жизненный цикл поддержки, в ходе которого корпорация Майкрософт предоставляет новые функции, исправления ошибок и обновления для системы безопасности. Жизненный цикл продукта обычно длится 10 лет с начального выпуска продукта. Завершение этого жизненного цикла называется завершением поддержки продукта. Так как proClarity, PerformancePoint Server и SharePoint Server 2007 достигли конца поддержки, Корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка для проблем, которые могут возникнуть.
    
- Исправления ошибок, обнаруженных и которые могут повлиять на стабильность и доступность серверов.
    
- Исправления безопасности для обнаруженных уязвимостей, которые могут сделать серверы или приложения уязвимыми для нарушений безопасности.
    
- Обновления часовой пояс.
    
Установка ProClarity, SharePoint Server 2007 с sp3 и PerformancePoint Server 2007 с sp3 продолжится, даже если поддержка завершена. Однако мы настоятельно рекомендуем как можно скорее перейти из этих приложений.

### <a name="what-if-i-dont-upgrade-now"></a>Что делать, если я не обновусь?

Вы можете не обновляться немедленно. Существующие серверы и приложения продолжат работать. Однако вы не будете получать дополнительные обновления, включая обновления для системы безопасности, так как поддержка завершена. Если что-то пойдет не так с серверными приложениями, вы не сможете получить помощь из службы технической поддержки Майкрософт.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Использование Power BI (в облаке или локальной)

Power BI — это набор средств бизнес-аналитики для анализа данных и обмена аналитическими данными. С помощью Power BI вы можете использовать локальное или сетевое источники данных для создания интерактивных отчетов и панелей мониторинга. Люди могут просматривать и использовать отчеты и панели мониторинга на своих компьютерах или мобильных устройствах.
  
Power BI не входит в состав Microsoft 365 или SharePoint Server. Это отдельное предложение, которое включает Power BI Desktop, шлюзы Power BI и службу Power BI. Power BI также интегрируется с SharePoint Online. Вы можете начать работу с Power BI бесплатно. В зависимости от использования данных и бизнес-потребностей вы можете позже перейти на Power BI Pro с помощью Microsoft 365 E5. Чтобы узнать больше, посмотрите, [что такое Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Использование служб Reporting Services (локально)

SQL Server Reporting Services предоставляет надежное решение для работы с отчетами. Службы Reporting Services можно настроить в режиме встроенного или интегрированного SharePoint. Для этого можно использовать несколько различных средств, в том числе конструктор отчетов, построитель отчетов и Power View. В последнем выпуске SQL Server можно также использовать SQL Server Mobile Report Publisher для доставки отчетов, масштабирования до любого размера экрана. Это позволяет пользователям использовать отчеты на своих мобильных устройствах. Дополнительные данные [см. в SQL Server Reporting Services (SSRS): создание,](https://go.microsoft.com/fwlink/?linkid=841342)развертывание и управление отчетами на мобильных устройствах и на разных устройствах.
  
### <a name="use-performancepoint-services-on-premises"></a>Использование PerformancePoint Services (локально)

PerformancePoint Server 2007 был продан отдельно от SharePoint Server 2007. Начиная с SharePoint Server 2010, PerformancePoint Services это приложение-служба в SharePoint Server. Поэтому вам не нужно приобретать отдельные серверные лицензии или оборудование для использования PerformancePoint Services.
  
Чтобы перейти с PerformancePoint Server 2007 на PerformancePoint Services, необходимо перейти на более новую версию SharePoint Server и настроить PerformancePoint Services. Перемещаемая версия SharePoint Server определяет, можно ли импортировать существующий контент панели мониторинга из PerformancePoint Server 2007 в PerformancePoint Services.
  
- При обновлении до SharePoint Server 2010 можно импортировать содержимое панели мониторинга PerformancePoint из PerformancePoint Server 2007 в PerformancePoint Services в SharePoint Server 2010. Чтобы узнать больше, см. мастер [импорта: PerformancePoint Server 2007 в SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- При переходе на SharePoint Server 2013 или SharePoint Server 2016, скорее всего, потребуется создать новый контент панели мониторинга (источники данных, отчеты, системы показателей и страницы панелей мониторинга).
    
Чтобы начать работу с планом PerformancePoint Services обновления, см. следующие ресурсы:
  
- [План действий после окончания поддержки SharePoint Server 2007](sharepoint-2007-end-of-support.md)
    
- Если вы знаете, в какую версию SharePoint вы переходите, см. соответствующую статью для PerformancePoint Services:
    
  - [Планирование PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [PerformancePoint Services sharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Обзор служб PerformancePoint Services в SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
При обновлении до PerformancePoint Services вы получаете несколько новых функций и улучшений. PerformancePoint Services предлагает улучшенные системы показателей; новые визуализации, такие как дерево декомпозиции и отчет о ХPI; дополнительные типы диаграмм; улучшение возможностей фильтрации аналитики времени; и улучшенное соответствие требованиям к специальным требованиям. Дополнительные возможности см. в PerformancePoint Services [SharePoint Server 2010.](https://go.microsoft.com/fwlink/?linkid=841343)
