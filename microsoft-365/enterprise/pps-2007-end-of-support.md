# <a name="performancepoint-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки PerformancePoint Server 2007.

Для серверов и приложений Office 2007 достигнут конец поддержки, в том числе серверы и приложения, которые могут использоваться в составе решений бизнес-аналитики (BI). В следующей таблице перечислены приложения бизнес-аналитики, которые затронуты:

|PerformancePoint Server 2007 с пакетом обновления 3 (SP3)  <br/> |9 января 2018 г.  <br/> |
   
Для получения дополнительных сведений ознакомьтесь [со статьей ресурсы, которые помогут вам выполнить обновление серверов и клиентов Office 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>Что означает окончание поддержки?

Для продуктов Майкрософт, таких как PerformancePoint Server 2007 с пакетом обновления 3 (SP3), программное обеспечение ProClarity и SharePoint Server 2007 с пакетом обновления 3 (SP3), предусмотрен жизненный цикл поддержки, в течение которого Майкрософт предоставляет новые функции, исправления ошибок и обновления безопасности Жизненный цикл продукта обычно длится в течение 10 лет с даты первоначального выпуска продукта, а окончание этого жизненного цикла называется окончанием поддержки продукта. По мере ProClarity, PerformancePoint Server и SharePoint Server 2007 достигнут конец поддержки, корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка проблем, которые могут возникнуть
    
- Исправление ошибок для обнаруженных проблем, которые могут повлиять на стабильность и удобство использования серверов
    
- Исправления для системы безопасности уязвимостей, которые могут привести к нарушениям безопасности серверов или приложений
    
- Обновления часового пояса
    
Установка ProClarity, SharePoint Server 2007 с пакетом обновления 3 (SP3) и PerformancePoint Server 2007 с пакетом обновления 3 (SP3) продолжит выполняться, несмотря на то, что поддержка закончилась. Однако настоятельно рекомендуется выполнить миграцию из этих приложений как можно скорее.

### <a name="what-happens-if-i-dont-upgrade-now"></a>Что произойдет, если не выполнить обновление сейчас?

В настоящее время вы можете сделать это без обновления. Существующие серверы и приложения продолжат работу. Однако вы не будете получать дополнительные обновления, в том числе обновления для системы безопасности, после завершения поддержки. Если что-то пошло не так с серверными приложениями, вы не сможете получить помощь в службе технической поддержки Майкрософт.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Использование Power BI (в облаке или локальной среде)

Power BI — это набор средств бизнес-аналитики для анализа данных и предоставления общих сведений. С помощью Power BI вы можете создавать интерактивные отчеты и панели мониторинга с помощью локальных и сетевых источников данных. Пользователи могут просматривать и использовать отчеты и панели мониторинга с помощью своих компьютеров или мобильных устройств.
  
Power BI не входит в состав Microsoft 365 или SharePoint Server, но является отдельным предложением, включающим Power BI Desktop, шлюзы Power BI и служба Power BI. Power BI также интегрируется с SharePoint Online. Вы можете приступить к работе с Power BI бесплатно и в зависимости от требований к использованию данных и бизнес-потребностям выполнить обновление до Power BI Pro с помощью Microsoft 365. Чтобы узнать больше, посмотрите, [что такое Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Использование служб отчетов (локальная среда)

Службы SQL Server Reporting Services предоставляют надежное решение для создания отчетов, а также возможность установки и настройки служб Reporting Services в режиме интеграции с SharePoint или в собственном режиме. Вы можете создавать отчеты с помощью нескольких инструментов, в том числе конструктора отчетов, построителя отчетов и Power View. В последнем выпуске SQL Server вы также можете использовать Microsoft SQL Server Mobile Reporting Report для доставки отчетов, которые масштабируются на любой размер экрана, что позволяет Организации использовать отчеты на мобильных устройствах. Дополнительные сведения см. в статье [SQL Server Reporting Services (SSRS): создание, развертывание и управление отчетами по мобильным и разбитым на страницы](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Использование PerformancePoint Services (локально)

Как вы знаете, PerformancePoint Server 2007 был приобретен отдельно от SharePoint Server 2007. Начиная с SharePoint Server 2010, PerformancePoint Services это приложение-служба в SharePoint Server. Это означает, что вам не нужно приобретать отдельные серверные лицензии или оборудование, чтобы использовать службы PerformancePoint Services.
  
Для перехода от PerformancePoint Server 2007 к службам PerformancePoint Services вы переходите к более новой версии SharePoint Server и настраиваете службы PerformancePoint Services. Версия SharePoint Server, на которую вы перемещаетесь, определит, можно ли импортировать имеющееся содержимое панели мониторинга из PerformancePoint Server 2007 в службы PerformancePoint Services.
  
- При обновлении до SharePoint Server 2010 вы можете импортировать содержимое панели мониторинга PerformancePoint из PerformancePoint Server 2007 в службы PerformancePoint Services в SharePoint Server 2010. Дополнительные сведения о том, как это работает, можно найти в статье [Мастер импорта: PerformancePoint Server 2007 Content to SharePoint server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Если вы перемещаетесь на SharePoint Server 2013 или SharePoint Server 2016, вам, скорее всего, потребуется создать новое содержимое панели мониторинга (источники данных, отчеты, системы показателей и страницы панели мониторинга).
    
Чтобы приступить к работе с планом обновления PerformancePoint Services, ознакомьтесь со следующими материалами:
  
1. [План поддержки SharePoint Server 2007](sharepoint-2007-end-of-support.md)
    
2. Если вы знаете, на какую версию SharePoint вы перемещаете, ознакомьтесь со статьей, в которой выполняется служба PerformancePoint Services:
    
  - [Планирование служб PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Общие сведения о службах PerformancePoint в SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Обзор служб PerformancePoint Services в SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
При обновлении до PerformancePoint Services вам понравится несколько новых функций и улучшений. Службы PerformancePoint Services предлагают улучшенные системы показателей, новые зрительные образы, такие как дерево декомпозиции и подробные сведения о ключевом показателе эффективности, а также другие типы диаграмм, улучшенные возможности фильтрации операций со временем и улучшения соответствия требованиям. Чтобы узнать больше, ознакомьтесь со статьей [новые возможности PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
  
## <a name="where-can-i-get-help-with-my-upgrade"></a>Где можно получить помощь по обновлению?

Как вы обновляете локально или переходите на Microsoft 365, мы рекомендуем вам работать с партнером Майкрософт. Квалифицированный партнер может помочь вам определить решение, которое соответствует потребностям вашей компании, и помочь вам в развертывании. Посетите [Центр партнеров Майкрософт](https://go.microsoft.com/fwlink/?linkid=841249)и используйте фильтры поиска, чтобы найти поставщика решений.
