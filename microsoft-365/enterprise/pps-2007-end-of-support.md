# <a name="performancepoint-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки PerformancePoint Server 2007.

Для серверов и приложений Office 2007 достигнут конец поддержки, в том числе серверы и приложения, которые могут использоваться в составе решений бизнес-аналитики (BI). В следующей таблице перечислены приложения бизнес-аналитики, которые затронуты:
  
|**Приложения бизнес-аналитики Майкрософт**|**Поддержка дат завершена**|
|:-----|:-----|

|PerformancePoint Server 2007 с пакетом обновления 3 (SP3)  <br/> |9 января 2018 г.  <br/> |
   
Для получения дополнительных сведений ознакомьтесь [со статьей ресурсы, которые помогут вам выполнить обновление серверов и клиентов Office 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>Что означает *Окончание поддержки* ?

Как и большинство продуктов Майкрософт, PerformancePoint Server 2007 с пакетом обновления 3 (SP3), ProClarity, программное обеспечение и SharePoint Server 2007 с пакетом обновления 3 (SP3) имеют жизненный цикл поддержки, в течение которого Майкрософт предоставляет новые функции, исправления ошибок и обновления безопасности. Жизненный цикл продукта обычно длится через 10 лет от первоначального выпуска продукта. Окончание этого жизненного цикла называется окончанием поддержки продукта. По мере ProClarity, PerformancePoint Server и SharePoint Server 2007 достигнут конец поддержки, корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка проблем, которые могут возникнуть.
    
- Исправление ошибок для обнаруженных проблем, которые могут повлиять на стабильность и удобство использования серверов.
    
- Исправления для системы безопасности уязвимостей, которые могут привести к нарушениям безопасности серверов или приложений.
    
- Обновления часового пояса.
    
Установка ProClarity, SharePoint Server 2007 с пакетом обновления 3 (SP3) и PerformancePoint Server 2007 с пакетом обновления 3 (SP3) продолжит выполняться, несмотря на то, что поддержка закончилась. Однако настоятельно рекомендуется выполнить миграцию из этих приложений как можно скорее.

### <a name="what-if-i-dont-upgrade-now"></a>Что делать, если обновление не выполняется?

Вы можете отказаться от немедленного обновления. Существующие серверы и приложения продолжат работу. Но вы не получите дальнейших обновлений, в том числе обновлений для системы безопасности, так как поддержка закончилась. Если что-то пошло не так с серверными приложениями, вы не сможете получить помощь в службе технической поддержки Майкрософт.

Службы PerformancePoint Services недоступны в Microsoft 365, поэтому вы будете заменять контент панели мониторинга PerformancePoint книгами Excel или другими отчетами. Хорошая новость состоит в том, что Excel 2016 имеет множество новых типов диаграмм, и вы легко создадите впечатляющие панели мониторинга в Excel. А новые функции регулярно добавляются. Чтобы узнать больше, ознакомьтесь со статьей [новые возможности Excel 2016 для Windows](https://support.office.com/article/5fdb9208-ff33-45b6-9e08-1f5cdb3a6c73.aspx).
  
Кроме того, если вы приобретаете 50 рабочих мест или более Microsoft 365, группа Microsoft FastTrack может помочь вам настроить. Чтобы узнать больше, посетите [FastTrack](https://www.microsoft.com/fasttrack/microsoft-365).

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Использование Power BI (в облаке или локальной среде)

Power BI — это набор средств бизнес-аналитики для анализа данных и предоставления общих сведений. С помощью Power BI вы можете использовать локальные и сетевые источники данных для создания интерактивных отчетов и панелей мониторинга. Пользователи могут просматривать и использовать отчеты и панели мониторинга на своих компьютерах или мобильных устройствах.
  
Power BI не входит в состав Microsoft 365 или SharePoint Server. Это отдельный выпуск, включающий Power BI Desktop, шлюзы Power BI и службу Power BI. Power BI также интегрируется с SharePoint Online. Вы можете приступить к работе с Power BI бесплатно. В зависимости от требований к использованию данных и бизнес-потребностям вы можете позднее выполнить обновление до Power BI Pro с помощью Microsoft 365. Чтобы узнать больше, посмотрите, [что такое Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Использование служб отчетов (локальная среда)

Службы SQL Server Reporting Services предоставляют надежное решение для создания отчетов. Службы Reporting Services можно настроить как в собственном режиме, так и в режиме интеграции с SharePoint. Вы можете использовать несколько различных средств для создания отчетов, в том числе конструктора отчетов, построителя отчетов и Power View. В последнем выпуске SQL Server вы также можете использовать Microsoft SQL Server Mobile Reporting Report для доставки отчетов, которые масштабируются на любой размер экрана. Это позволяет читателям использовать отчеты на своих мобильных устройствах. Дополнительные сведения см. в статье [SQL Server Reporting Services (SSRS): создание, развертывание и управление отчетами по мобильным и разбитым на страницы](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Использование PerformancePoint Services (локально)

PerformancePoint Server 2007 был продан отдельно от SharePoint Server 2007. Начиная с SharePoint Server 2010, PerformancePoint Services это приложение-служба в SharePoint Server. Поэтому вам не нужно приобретать отдельные серверные лицензии или оборудование для использования PerformancePoint Services.
  
Для перехода с PerformancePoint Server 2007 на службу PerformancePoint Services вы можете перейти на более новую версию SharePoint Server и настроить службы PerformancePoint Services. Версия SharePoint Server, на которую вы перемещаетесь, определяет, можно ли импортировать имеющееся содержимое панели мониторинга из PerformancePoint Server 2007 в службы PerformancePoint Services.
  
- При обновлении до SharePoint Server 2010 вы можете импортировать содержимое панели мониторинга PerformancePoint из PerformancePoint Server 2007 в службы PerformancePoint Services в SharePoint Server 2010. Чтобы узнать больше, ознакомьтесь со статьей [Мастер импорта: PerformancePoint Server 2007 Content to SharePoint server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Если вы перейдете на SharePoint Server 2013 или SharePoint Server 2016, вам, скорее всего, потребуется создать новое содержимое панели мониторинга (источники данных, отчеты, системы показателей и страницы панели мониторинга).
    
Чтобы приступить к работе с планом обновления PerformancePoint Services, ознакомьтесь со следующими материалами:
  
- [План поддержки SharePoint Server 2007](sharepoint-2007-end-of-support.md)
    
- Если вы знаете, на какую версию SharePoint вы перемещаете, ознакомьтесь со статьей, в которой выполняется служба PerformancePoint Services:
    
  - [Планирование служб PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Общие сведения о службах PerformancePoint в SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Обзор служб PerformancePoint Services в SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
При обновлении до PerformancePoint Services вы получаете несколько новых функций и улучшений. Службы PerformancePoint Services предлагают улучшенные системы показателей; новые зрительные образы, такие как дерево декомпозиции и подробные сведения о ключевом показателе эффективности; Дополнительные типы диаграмм; Улучшенные возможности фильтрации логики операций со временем; и улучшено соответствие требованиям специальных возможностей. Чтобы узнать больше, ознакомьтесь со статьей [новые возможности PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841343).
