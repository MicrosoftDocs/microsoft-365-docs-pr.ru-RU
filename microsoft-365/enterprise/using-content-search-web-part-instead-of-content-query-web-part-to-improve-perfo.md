# <a name="using-content-search-web-part-instead-of-content-query-web-part-to-improve-performance-in-sharepoint-online"></a>Использование веб-части поиска контента вместо веб-части запроса контента для повышения производительности SharePoint Online

В этой статье описывается, как повысить производительность путем замены веб-части запроса контента веб-частью поиска контента в SharePoint Server 2013 и SharePoint Online.
  
Одной из самых мощных новых функций SharePoint Server 2013 и SharePoint Online является веб-часть поиска контента (CSWP). Эта веб-часть использует индекс поиска для быстрого получения результатов, показанных пользователю. Используйте веб-часть поиска контента вместо веб-части запроса контента (CQWP) на своих страницах, чтобы повысить производительность для пользователей.
  
Использование веб-части поиска контента над веб-частью запроса контента почти всегда приведет к значительно лучшей производительности загрузки страниц SharePoint Online. Существует небольшая дополнительная конфигурация, чтобы получить правильный запрос, но награды улучшения производительности и более счастливых пользователей.
  
## <a name="comparing-the-performance-gain-you-get-from-using-content-search-web-part-instead-of-content-query-web-part"></a>Сравнение получения производительности с помощью веб-части поиска контента вместо веб-части запроса контента

В следующих примерах покажите относительную прибыль производительности, которая может быть при использовании веб-части поиска контента вместо веб-части запроса контента. Эффекты более очевидны со сложной структурой сайта и очень широкими запросами контента.
  
В этом примере сайта есть следующие характеристики:
  
- 8 уровней подвидов.
    
- Списки с использованием настраиваемого типа "фруктового" контента.
    
- В веб-части запрос контента широк, возвращая все элементы с типом контента "фрукты".
    
- В примере используется только 50 элементов на 8 сайтах. Эффекты будут еще более выраженными для сайтов с большим содержанием.
    
Вот снимок экрана результатов веб-части запроса контента.
  
![Рисунок: запрос контента для веб-части](../media/b3d41f20-dfe5-46ed-9c0a-31057e82de33.png)
  
В Internet Explorer используйте вкладку **Network** средств разработчика F12 для анализа сведений для загона ответа. В следующем снимке экрана значение **spRequestDuration** для этой страницы составляет 924 миллисекунд. 
  
![Снимок экрана со значением длительности запроса (924)](../media/343571f2-a249-4de2-bc11-2cee93498aea.png)
  
 **SPRequestDuration** указывает объем работы, которая делается на сервере для подготовки страницы. Переключение контента веб-части с контентом с помощью поиска веб-части значительно сокращает время, необходимое для отрисовки страницы. Напротив, страница с эквивалентной веб-частью поиска контента, возвращая такое же количество результатов, имеет значение **SPRequestDuration** в 106 миллисекунд, как показано на этом снимке экрана: 
  
![Снимок экрана со значением длительности запроса (106)](../media/b46387ac-660d-4e5e-a11c-cc430e912962.png)
