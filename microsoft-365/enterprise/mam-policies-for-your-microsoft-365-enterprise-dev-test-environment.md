# <a name="device-compliance-policies-for-your-microsoft-365-for-enterprise-test-environment"></a>Политики соответствия требованиям к устройствам для тестовой среды Microsoft 365 для предприятий

*Это руководство по лаборатории тестирования можно использовать только для Microsoft 365 для тестовых сред предприятия.*

В этой статье описывается, как добавить политику соответствия требованиям для устройств Intune для устройств с Windows 10 и приложений Microsoft 365 для предприятий в тестовую среду Microsoft 365 для предприятия.

Добавление политики соответствия требованиям к устройствам Intune включает в себя два этапа:
- [Этап 1: создание тестовой среды Microsoft 365 для предприятий](#phase-1-build-out-your-microsoft-365-for-enterprise-test-environment)
- [Этап 2: Создание политики соответствия требованиям устройств для устройств с Windows 10](#phase-2-create-a-device-compliance-policy-for-windows-10-devices)

![Руководства по лаборатории тестирования для облака Майкрософт](../media/m365-enterprise-test-lab-guides/cloud-tlg-icon.png)

> [!TIP]
> Для отображения всех статей, посвященных руководству по лаборатории тестирования Microsoft 365 для предприятий, перейдите к разделу [руководство по лаборатории тестирования microsoft 365 для предприятия](../downloads/Microsoft365EnterpriseTLGStack.pdf).

## <a name="phase-1-build-out-your-microsoft-365-for-enterprise-test-environment"></a>Этап 1: создание тестовой среды Microsoft 365 для предприятий

Если вы хотите настроить политики MAM только в упрощенном виде с минимальными требованиями, следуйте инструкциям в разделе [облегченная настройка конфигурации](lightweight-base-configuration-microsoft-365-enterprise.md).
  
Если вы хотите настроить политики MAM на имитации предприятия, следуйте инструкциям в [сквозной проверке подлинности](pass-through-auth-m365-ent-test-environment.md).
  
> [!NOTE]
> Для тестирования автоматического лицензирования и членства в группах не требуется имитация тестовой среды предприятия, которая включает имитируемую интрасеть, подключенную к Интернету и синхронизацию каталогов, для леса доменных служб Active Directory (AD DS). Он предоставляется в качестве варианта, чтобы можно было протестировать автоматизированное членство и членство в группах и поэкспериментировать с ним в среде, представляющей типичную организацию.
>  

## <a name="phase-2-create-a-device-compliance-policy-for-windows-10-devices"></a>Этап 2: Создание политики соответствия требованиям устройств для устройств с Windows 10

На этом этапе создается политика соответствия требованиям устройств для устройств с Windows 10. На этом этапе для добавления группы используется Microsoft Intune и [центр администрирования Microsoft Endpoint Manager](https://go.microsoft.com/fwlink/?linkid=2109431) , а также создается политика соответствия требованиям.

1. Перейдите в [центр администрирования microsoft 365](https://admin.microsoft.com)и войдите в свою подписку на тестовую лаборатория Microsoft 365 с помощью учетной записи глобального администратора. Выберите центр администрирования **диспетчера конечных точек** . Откроется [центр администрирования диспетчера конечных точек](https://go.microsoft.com/fwlink/?linkid=2109431) .

    Если сообщение, похожее на " **Управление устройствами" не включено** , отображается сообщение, выберите Intune в качестве администратора MDM. Конкретные действия приведены [в разделе Установка центра управления мобильными устройствами](/mem/intune/fundamentals/mdm-authority-set).

    Центр администрирования диспетчера конечных точек посвящен управлению устройствами и управлению приложениями. Обзор этого центра администрирования приведен [в разделе Tutorial: пошаговое руководство Intune в диспетчере конечных точек Майкрософт](/mem/intune/fundamentals/tutorial-walkthrough-endpoint-manager).

2. В **группы** добавьте новую группу **Microsoft 365** или группу **безопасности** с именем " **управляемые пользователи устройств Windows 10**" с **назначенным** типом членства. На следующих этапах вы назначите политику соответствия требованиям для этой группы. 

    Конкретные действия и сведения о **Microsoft 365** или группах **безопасности** содержатся в разделе [Добавление групп для организации пользователей и устройств](/mem/intune/fundamentals/groups-add).

3. В окне **устройства** создайте политику соответствия требованиям Windows 10. Назначьте эту политику для созданной **управляемой группы пользователей устройств с Windows 10** .

    В политике можно заблокировать простые пароли, для них требуется брандмауэр, необходимо запустить службу антивредоносной программы защитника Майкрософт и т. д. Политика соответствия требованиям обычно включает базовые параметры или значение по крайней мере, которое должно иметь каждое устройство.

    Конкретные действия и сведения о доступных параметрах соответствия, которые можно настроить, приведены в разделе [Использование политик соответствия для установки правил для управляемых устройств](/mem/intune/protect/device-compliance-get-started).

По завершении у вас есть политика соответствия требованиям устройств для тестирования членов в группе **управляемых пользователей устройств Windows 10** .
