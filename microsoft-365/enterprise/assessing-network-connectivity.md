# <a name="assessing-microsoft-365-network-connectivity"></a>Оценка сетевого подключения к Microsoft 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft 365 позволяет клиентам по всему миру подключаться к службе с помощью подключения к Интернету. По мере развития службы безопасность, производительность и надежность Microsoft 365 улучшаются в зависимости от клиентов, использующих Интернет для установления подключения к службе.
  
Клиенты, планируемые использовать Microsoft 365, должны оценить существующие и прогнозируемые потребности в подключении к Интернету в рамках проекта развертывания. Для развертывания корпоративного класса надежное и подходящее подключение к Интернету является важной частью использования функций и сценариев Microsoft 365.
  
Оценка сети может выполняться многими разными людьми и организациями в зависимости от размера и предпочтений. Область сети оценки также может отличаться в зависимости от того, где вы сейчас в процессе развертывания. Чтобы помочь вам лучше понять, что требуется для оценки сети, мы подготовили руководство по оценке сети, чтобы помочь вам понять доступные варианты. Эта оценка определяет, какие действия и ресурсы необходимо добавить в проект развертывания для успешного внедрения Microsoft 365.
  
Комплексная оценка сети предоставляет возможные решения проблем сетевого проектирования, а также детали реализации. Некоторые оценки сети показывают, что оптимальное сетевое подключение к Microsoft 365 можно обеспечить с помощью незначительных изменений конфигурации или изменений в структуре существующей инфраструктуры сети и интернета.

Некоторые оценки указывают, что для сетевого подключения к Microsoft 365 потребуются дополнительные инвестиции в сетевые компоненты. Например, для корпоративных сетей, охватывающих филиалы и различные географические регионы, могут потребоваться инвестиции в решения SD-WAN или оптимизированная инфраструктура маршрутов для поддержки подключения к Microsoft 365 через Интернет. Иногда оценка указывает, что на сетевое подключение к Microsoft 365 влияют нормативные требования или требования к производительности для таких сценариев, как качество мультимедиа в Skype для бизнеса [Online.](https://support.office.com/article/Media-Quality-and-Network-Connectivity-Performance-in-Skype-for-Business-Online-5fe3e01b-34cf-44e0-b897-b0b2a83f0917) Эти дополнительные требования могут привести к инвестициям в инфраструктуру подключения к Интернету, оптимизацию маршрутизации и специализированное прямое подключение.

Некоторые ресурсы, которые помогут вам оценить сеть:

- Общие сведения о сетевых подключениях [Microsoft 365](microsoft-365-networking-overview.md) см. в обзоре сетевых подключений Microsoft 365.
- Принципы сетевого подключения [Microsoft 365](https://aka.ms/o365networkingprinciples) см. в принципах подключения для безопасного управления трафиком Microsoft 365 и обеспечения максимальной производительности.
- Зарегистрируется в [Microsoft FastTrack для](https://www.microsoft.com/fasttrack) помощи в планировании, проектировании и развертывании Microsoft 365. 
- В разделе [тестирования подключения Microsoft 365](assessing-network-connectivity.md#the-microsoft-365-connectivity-test) ниже приведены основные тесты подключений, которые предоставляют конкретные рекомендации по улучшению сетевых подключений, которые могут быть сделаны между определенным расположением пользователя и Microsoft 365.

> [!NOTE]
> Для использования ExpressRoute для Office 365 требуется авторизация Майкрософт. Корпорация Майкрософт проверяет каждый запрос клиента и авторизирует ExpressRoute для использования Office 365 только в том случае, если нормативные требования клиента предписывают прямое подключение. Если у вас есть такие требования, укажете выдержку из текста и веб-ссылку на регламент, который вы интерпретируете, чтобы начать проверку Корпорацией Майкрософт, в форме запроса [ExpressRoute для Office 365](https://aka.ms/O365ERReview) требуется прямое подключение. Неавторизованные подписки, которые пытаются создать фильтры маршрутов для Office 365, получат сообщение [об ошибке.](https://support.microsoft.com/kb/3181709)
  
Ключевые моменты, которые следует учитывать при планировании оценки сети для Microsoft 365:
  
- Microsoft 365 — это безопасная, надежная высокоскоростная служба, которая работает через общедоступный Интернет. Мы продолжаем вкладывать средства для улучшения этих аспектов службы. Все службы Microsoft 365 доступны через подключение к Интернету.

- Мы постоянно оптимизируем основные аспекты Microsoft 365, такие как доступность, глобальный доступ и производительность для подключения к Интернету. Например, многие службы Microsoft 365 используют расширяющийся набор edge-узлов с доступом к Интернету. Эта по краям сеть обеспечивает наилучшее расположение и производительность подключений через Интернет.

- When considering using Microsoft 365 for any of the included services such as Teams or Skype for Business Online voice, video, or meeting capabilities, customers should complete an end to end network assessment and meet connectivity requirements using [Microsoft FastTrack](https://www.microsoft.com/fasttrack).

Если вы оцениваете Microsoft 365 и не знаете, с чего начать оценку сети или нашли проблемы с сетевым дизайном, которые вам нужна помощь, обратитесь к своей группе по работе с учетными данными Майкрософт.

## <a name="the-microsoft-365-connectivity-test"></a>Тест подключения Microsoft 365

Тест [подключения Microsoft 365](https://aka.ms/netonboard) — это средство оценки сети POC, которое выполняет основные тесты подключения к вашему клиенту Microsoft 365 и дает конкретные рекомендации по проектированию сети для оптимальной производительности Microsoft 365. Это средство выделяет распространенные варианты проектирования периметра крупных корпоративных сетей, которые полезны для просмотра веб-страниц в Интернете, но влияют на производительность больших приложений SaaS, таких как Microsoft 365.

Средство сетевой подсети делает следующее:

- Определяет ваше расположение или вы можете указать местоположение для тестирования
- Проверяет расположение сетевого эксгресса
- Проверяет сетевой путь к ближайшему входу в службу Microsoft 365
- Предоставляет расширенные тесты с помощью загружаемого приложения для Windows 10, которое предоставляет рекомендации по проектированию сети периметра, связанные с прокси-серверами, брандмауэрами и DNS. Средство также выполняет тесты производительности для Skype для бизнеса Online, Microsoft Teams, SharePoint Online и Exchange Online.

Средство имеет два компонента: браузерный пользовательский интерфейс, который собирает основные сведения о под подключениях, и загружаемое приложение для Windows 10, которое выполняет расширенные тесты и возвращает дополнительные данные оценки.

Средство на основе браузера отображает следующие сведения:

- Вкладка результатов и влияния
  - Расположение на карте входной ногой службы
  - Расположение на карте других входных входов службы, которые обеспечивают оптимальное подключение
  - Относительная производительность по сравнению с другими клиентами Microsoft 365 рядом с вами
- Вкладка "Сведения и решения"
  - Расположение пользователя по городу и стране
  - Расположение сетевого перенаступа по городам, штатам и стране
  - Расстояние между пользователем и сетевым эксгримом
  - Расположение переднего входа службы Microsoft 365 Exchange Online
  - Оптимальные входные точки службы Microsoft 365 Exchange Online для расположения пользователей
  - Клиенты в вашей области с более производительностью

Загружаемое приложение "Расширенные тесты" предоставляет следующие дополнительные сведения:

- Вкладка "Сведения и решения" (с приложением)
  - Шлюз пользователя по умолчанию
  - DNS-сервер клиента
  - Рекурсивное разрешение DNS клиента
  - DNS-сервер Exchange Online
  - DNS-сервер SharePoint Online
  - Идентификация прокси-сервера
  - Проверка возможности подключения к носительной сети
  - Потеря пакетов качества мультимедиа
  - Задержка качества мультимедиа
  - Дрожать качества мультимедиа
  - Переусортовка пакетов качества мультимедиа
- Тесты подключения к нескольким конечным точкам для определенных функций
- Диагностика сетевых путей с данными трассировки и задержки для служб Exchange Online, SharePoint Online и Teams

Вы можете ознакомиться с тестом подключения Microsoft 365 и предоставить отзывы на обновленном тестовом POC подключения [Microsoft 365](https://techcommunity.microsoft.com/t5/Office-365-Networking/Updated-Office-365-Network-Onboarding-Tool-POC-with-new-network/m-p/711130#M130) с новыми рекомендациями по проектированию сети в блоге. Сведения о будущих обновлениях этого средства и других сетевых обновлениях Microsoft 365 будут опубликованы в блоге о сети [Office 365.](https://techcommunity.microsoft.com/t5/Office-365-Networking/bd-p/Office365Networking)
