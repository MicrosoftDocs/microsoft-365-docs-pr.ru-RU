# <a name="network-requests-in-office-for-mac"></a>Сетевые запросы в Office для Mac

Приложения Office для Mac обеспечивают работу собственного приложения на платформе macOS. Каждое приложение предназначено для работы в различных сценариях, в том числе состояния, когда доступ к сети невозможен. Когда компьютер подключается к сети, приложения автоматически подключаются к ряду веб-служб, чтобы обеспечить расширенные функциональные возможности. Ниже приведены сведения о том, какие конечные точки и URL-адреса попытаются достичь приложения, и предоставлены ли службы. Эта информация полезна при устранении проблем с настройкой сети и настройке политик для серверов сетевых прокси-серверов. Сведения, приведенные в этой статье, предназначены для дополнения к [статье URL-адресов и диапазонов адресов Office 365](urls-and-ip-address-ranges.md), которая включает конечные точки для компьютеров под управлением Microsoft Windows. Если не указано иное, сведения, приведенные в этой статье, также применимы к Office 2019 для Mac и Office 2016 для Mac, которые доступны в качестве одноразовой покупки в розничном магазине или по соглашению о корпоративном лицензировании. 

  
В этой статье описываются таблицы, в которых описываются сетевые URL-адреса, тип и описание службы или функции, предоставляемые этой конечной точкой. Каждое приложение Office может различаться в разных службах и использовании конечных точек. В приведенных ниже таблицах определены указанные ниже приложения.
  
- W: Word
- P: PowerPoint
- X: Excel
- O: Outlook
- N: OneNote
   
Тип URL-адреса определяется следующим образом:
  
- ST: Static — URL-адрес жестко закодирован в клиентском приложении.
    
- СС: полупрозрачный URL-адрес кодируется как часть веб-страницы или перенаправителя.
    
- CS: config Service — URL-адрес возвращается в составе службы настройки Office.

|**URL-адрес**|**Тип**|**Описание**|

## <a name="options-for-reducing-network-requests-and-traffic"></a>Параметры для сокращения количества сетевых запросов и трафика

Конфигурация Office для Mac по умолчанию обеспечивает оптимальное взаимодействие с пользователем в терминах функциональности и своевременное обновление компьютера. В некоторых случаях может потребоваться запретить приложениям обращаться к конечным точкам сети. В этом разделе обсуждаются варианты этого.
  
 ### <a name="disabling-cloud-sign-in-and-office-add-ins"></a>Отключение входа в облако и надстроек Office
  
Клиенты с корпоративным лицензированием могут иметь ограниченные политики сохранения документов в облачном хранилище. Для следующих параметров приложения можно настроить отключение входа MSA/OrgID и доступ к надстройкам Office.
  
- ```defaults write com.microsoft.Word UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Excel UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Powerpoint UseOnlineContent -integer 0```

Если пользователи пытаются получить доступ к функции входа, появится сообщение о том, что отсутствует сетевое подключение. Так как этот параметр также блокирует активацию продукта в сети, его следует использовать только для установки с корпоративной лицензией. В частности, использование этого параметра предотвратит доступ приложений Office к следующим конечным точкам:
  
- ```https://odc.officeapps.live.com```
    
- ```https://*.firstpartyapps.oaspapps.com```
    
- Все конечные точки, перечисленные в разделе "вход" выше.
    
- Все конечные точки, перечисленные в разделе "интеллектуальный поиск" выше.
    
- Все конечные точки, перечисленные в разделе "Активация продукта" выше.
    
- Все конечные точки, перечисленные выше в разделе "приложения Office (или надстройки)".
    
Чтобы повторно установить все функции для пользователя, установите для него значение "2" или удалите его.
  
> [!NOTE]
> Для этого предпочтения требуется сборка Office для Mac 15,25 [160726] или более поздней версии. 
  
### <a name="telemetry"></a>Телеметрия 
  
Office для Mac отправляет данные телеметрии обратно в корпорацию Майкрософт через регулярные промежутки времени. Данные передаются в конечную точку "связующее". Данные телеметрии помогают инженерской команде оценить работоспособность и неожиданное поведение каждого приложения Office. Существует две категории телеметрии:
  
- **Пульс** содержит сведения о версии и лицензии. Эти данные отправляются сразу после запуска приложения. 
    
- **Использование** содержит сведения о том, как приложения используются и не являются неустранимыми ошибками. Эти данные отправляются каждые 60 минут. 
    
Корпорация Майкрософт очень серьезно рассматривает вашу конфиденциальность. Вы можете ознакомиться с политикой сбора данных Майкрософт на сайте [https://privacy.microsoft.com](https://privacy.microsoft.com) . Чтобы запретить приложениям отправку телеметрии "использование", можно настроить параметр **сендаллтелеметренаблед** . Приоритет применяется для каждого приложения и может настраиваться через профили конфигурации macOS или вручную из терминала: 
  
```defaults write com.microsoft.Word SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Excel SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Powerpoint SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Outlook SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.onenote.mac SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.autoupdate2 SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Office365ServiceV2 SendAllTelemetryEnabled -bool FALSE```

Телеметрии пульса всегда отправляются и не могут быть отключены.
  
### <a name="crash-reporting"></a>Отчеты о сбоях
  
При возникновении неустранимой ошибки приложения приложение неожиданно завершает работу и загружает отчет о сбое в службу "Watson". Отчет о недоставке состоит из стека вызовов, который представляет собой список действий, которые обходило обработка приложением в результате сбоя. С помощью этих действий группа разработчиков определяет точную функцию, которая не удалась и почему.
  
В некоторых случаях содержимое документа приводит к сбою приложения. Если приложение определяет документ как причину, он запросит пользователя, если он также может отправить документ вместе с стеком вызовов. Пользователи могут выбрать этот вопрос. ИТ администраторы могут иметь ограниченные требования относительно передачи документов и принимать решение от имени пользователя, чтобы никогда не отправлять документы. Чтобы запретить отправку документов и отключать запрос пользователя, можно задать следующие параметры.
  
```defaults write com.microsoft.errorreporting IsAttachFilesEnabled -bool FALSE```

> [!NOTE]
> Если для **сендаллтелеметренаблед** задано **значение false**, то все отчеты о сбоях для этого процесса отключаются. Чтобы включить отчеты о сбоях, не отправляя телеметрии использования, можно задать следующие параметры. ```defaults write com.microsoft.errorreporting IsMerpEnabled -bool TRUE``` 
  
### <a name="updates"></a>Обновления
  
Корпорация Майкрософт выпускает обновления Office для Mac через регулярные промежутки времени (как правило, раз в месяц). Мы настоятельно рекомендуем пользователям и ИТ администраторам хранить компьютеры в актуальном состоянии, чтобы убедиться в том, что установлены последние исправления для системы безопасности. В тех случаях, когда ИТ администраторы хотят управлять обновлениями компьютера и управлять ими, можно настроить следующие параметры, чтобы запретить автоматическому автоматическому обнаружению и отправку обновлений продуктов.
  
```defaults write com.microsoft.autoupdate2 HowToCheck -string 'Manual'```

### <a name="blocking-requests-with-a-firewallproxy"></a>Блокировка запросов с помощью брандмауэра или прокси-сервера
  
Если ваша организация блокирует запросы на URL-адреса через брандмауэр или прокси-сервер, необходимо настроить указанные ниже URL-адреса в этом документе или заблокировать их в списке с ответом 40X (например, 403 или 404). Ответ 40X позволит приложениям Office корректно принимать доступ к ресурсу, что обеспечивает более быстрое взаимодействие с пользователем, чем простое удаление подключения, которое, в свою очередь, приведет к повторной попытке клиента.
  
Если прокси-сервер требует проверки подлинности, клиенту будет возвращен ответ 407. Для достижения лучших результатов убедитесь, что вы используете Office для Mac версии 15,27 или более поздней, так как они включают конкретные исправления для работы с серверами NTLM и Kerberos.
