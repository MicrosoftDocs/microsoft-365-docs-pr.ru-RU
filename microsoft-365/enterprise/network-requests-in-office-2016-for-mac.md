# <a name="network-requests-in-office-for-mac"></a>Сетевые запросы в Office для Mac

Office для Mac приложения обеспечивают возможность работы с приложениями на платформе macOS. Каждое приложение предназначено для работы в различных сценариях, включая состояния, когда доступ к сети отсутствует. Когда компьютер подключен к сети, приложения автоматически подключаются к ряду веб-служб, обеспечивая расширенные функции. Ниже описаны конечные точки и URL-адреса, к которым приложения пытаются связаться, и предоставляемые службы. Эти сведения полезны при устранении неполадок конфигурации сети и настройке политик для прокси-серверов сети. Сведения в этой статье дополняют статью о URL-адресах и диапазонах адресов [Office 365,](urls-and-ip-address-ranges.md)которая включает конечные точки для компьютеров под управлением Microsoft Windows. Если не отмечено, сведения в этой статье также применимы к Office 2019 для Mac и Office 2016 для Mac, которые можно приобрести в розничном магазине или по соглашению о корпоративном лицензировании. 

  
В большинстве данной статьи представлены таблицы с подробным описанием сетевых URL-адресов, типов и описания служб или функций, предоставляемых этой конечной точкой. Каждое из приложений Office может отличаться по использованию службы и конечной точки. В таблицах ниже определены следующие приложения:
  
- W: Word
- P: PowerPoint
- X: Excel
- O: Outlook
- N: OneNote
   
Тип URL-адреса определяется следующим образом:
  
- ST: Static — URL-адрес жестко закодироваться в клиентском приложении.
    
- SS: Semi-Static - URL-адрес закодирован как часть веб-страницы или перенаправления.
    
- CS: config Service — URL-адрес возвращается как часть службы конфигурации Office.

## <a name="options-for-reducing-network-requests-and-traffic"></a>Варианты сокращения сетевых запросов и трафика

Конфигурация по умолчанию Office для Mac обеспечивает наилучшее пользовательское интерфейсное решение как с точки зрения функциональности, так и с точки зрения поддержания компьютера в курсе. В некоторых случаях может потребоваться запретить приложениям связываться с конечными точками сети. В этом разделе обсуждаются варианты для этого.
  
 ### <a name="disabling-cloud-sign-in-and-office-add-ins"></a>Отключение облачных Sign-In и Office Add-Ins
  
Клиенты с лицензиями на объемные лицензии могут иметь строгие политики сохранения документов в облачном хранилище. Ниже дается настройка для каждого приложения, чтобы отключить вход mSA/OrgID и доступ к надстройки Office.
  
- ```defaults write com.microsoft.Word UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Excel UseOnlineContent -integer 0```

- ```defaults write com.microsoft.Powerpoint UseOnlineContent -integer 0```

Если пользователи попытаются получить доступ к Sign-In функции, они увидят ошибку, что сетевого подключения нет. Так как этот параметр также блокирует активацию продукта через Интернет, его следует использовать только для установок с много лицензиями. В частности, использование этого параметра не позволит приложениям Office получить доступ к следующим конечным точкам:
  
- ```https://odc.officeapps.live.com```
    
- ```https://*.firstpartyapps.oaspapps.com```
    
- Все конечные точки, перечисленные в разделе "Вход" выше.
    
- Все конечные точки, перечисленные в разделе "Интеллектуальный подсмотр" выше.
    
- Все конечные точки, перечисленные в разделе "Активация продукта" выше.
    
- Все конечные точки, перечисленные в разделе "Приложения Office (надстройки)" выше.
    
Чтобы повторно установить полную функциональность для пользователя, либо задав для него параметр "2", либо удалите его.
  
> [!NOTE]
> Этот параметр требует Office для Mac сборки 15.25 [160726] или более поздней. 
  
### <a name="telemetry"></a>Телеметрия 
  
Office для Mac отправляет данные телеметрии обратно в корпорацию Майкрософт с регулярными интервалами. Данные загружаются в конечную точку Nexus. Данные телеметрии помогают инженерной группе оценить состояние и непредвиденное поведение каждого приложения Office. Существует две категории телеметрии:
  
- **Пульс содержит** сведения о версии и лицензии. Эти данные отправляются сразу после запуска приложения. 
    
- **Использование** содержит сведения об использовании приложений и некритальных ошибках. Эти данные отправляются каждые 60 минут. 
    
Корпорация Майкрософт очень серьезно относится к вашей конфиденциальности. Сведения о политике сбора данных корпорации Майкрософт можно прочитать по этой [https://privacy.microsoft.com](https://privacy.microsoft.com) теме. Чтобы приложения не отправляли телеметрию "Использование", можно скорректировать параметр **SendAllTelemetryEnabled.** Предпочтение отдается для каждого приложения и может быть задается с помощью профилей конфигурации macOS или вручную из терминала: 
  
```defaults write com.microsoft.Word SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Excel SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Powerpoint SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Outlook SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.onenote.mac SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.autoupdate2 SendAllTelemetryEnabled -bool FALSE```

```defaults write com.microsoft.Office365ServiceV2 SendAllTelemetryEnabled -bool FALSE```

Телеметрия пульса всегда отправляется и не может быть отключена.
  
### <a name="crash-reporting"></a>Отчеты о сбоях
  
В случае неувторяемой ошибки приложения приложение неожиданно завершает работу и передает отчет о сбое в службу "Watson". Отчет о сбоях состоит из стека вызовов, который представляет собой список действий, которые приложение обрабатывалось до сбоя. Эти действия помогают проектной группе определить точную функцию, которая не удалась и почему.
  
В некоторых случаях содержимое документа приведет к сбою приложения. Если приложение определяет документ в качестве причины, оно спросит пользователя, можно ли также отправить документ вместе с стеком вызовов. Пользователи могут сделать информированный выбор для этого вопроса. ИТ-администраторы могут предъявлять строгие требования к передаче документов и принимать решение от имени пользователя никогда не отправлять документы. Можно настроить следующие параметры, чтобы запретить отправление документов и запретить пользователю подсказку:
  
```defaults write com.microsoft.errorreporting IsAttachFilesEnabled -bool FALSE```

> [!NOTE]
> Если **для SendAllTelemetryEnabled** установлено **false,** все отчеты о сбоях для этого процесса отключены. Чтобы включить отчеты о сбоях без отправки телеметрии об использовании, можно настроить следующие параметры: ```defaults write com.microsoft.errorreporting IsMerpEnabled -bool TRUE``` 
  
### <a name="updates"></a>Обновления
  
Корпорация Майкрософт Office для Mac обновления с регулярными интервалами (обычно раз в месяц). Мы настоятельно рекомендуем пользователям и ИТ-администраторам поддерживать компьютеры в курсе последних исправлений безопасности. В случаях, когда ИТ-администраторам требуется тесно контролировать обновления компьютеров и управлять ими, можно настроить следующие параметры, чтобы предотвратить автоматическое обнаружение и предоставление обновлений продукта в процессе автоматического обновления.
  
```defaults write com.microsoft.autoupdate2 HowToCheck -string 'Manual'```

### <a name="blocking-requests-with-a-firewallproxy"></a>Блокировка запросов с помощью брандмауэра или прокси-сервера
  
Если ваша организация блокирует запросы на URL-адреса через брандмауэр или прокси-сервер, обязательно настройте URL-адреса, указанные в этом документе, как разрешенные или блокируемые с откликом 40X (например, 403 или 404). 40-х ответ позволит приложениям Office корректно принять невозможность доступа к ресурсу и обеспечит более быстрый пользовательский интерфейс, чем простое прерывание подключения, что, в свою очередь, приведет к повторной попытку клиента.
  
Если прокси-серверу требуется проверка подлинности, клиенту будет возвращен ответ 407. Для лучшего использования убедитесь, что вы используете сборки Office для Mac 15.27 или более поздней, так как они включают определенные исправления для работы с серверами NTLM и Kerberos.
