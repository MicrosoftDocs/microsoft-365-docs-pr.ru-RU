# <a name="project-server-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки Project Server 2007;

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Поддержка для серверов и приложений Office 2007 в 2017, и вам необходимо рассмотреть план миграции. Если вы используете Project Server 2007 и связанные продукты, обратите внимание на следующие даты завершения поддержки:
  
|**Продукт**|**Дата окончания поддержки**|
|:-----|:-----|
|Project Server 2007  <br/> |10 октября 2017 г.  <br/> |
|Project Portfolio Server 2007  <br/> |10 октября 2017 г.  <br/> |
|Project 2007 Стандартный  <br/> |10 октября 2017 г.  <br/> |
|Project 2007 профессиональный  <br/> |10 октября 2017 г.  <br/> |
   
Для получения дополнительных сведений о серверах Office 2007, которые достигают выбытие, ознакомьтесь со статьей [обновление с серверов и клиентских продуктов office 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>Что означает *Окончание поддержки* ?

Большинство продуктов корпорации Майкрософт имеют жизненный цикл поддержки, в течение которого они получают новые функции, исправления ошибок, исправления для системы безопасности и т. д. Этот жизненный цикл обычно длится через 10 лет от первоначального выпуска продукта. Окончание этого жизненного цикла называется окончанием поддержки продукта. Так как Project Server 2007 достиг конца поддержки 10 октября 2017, корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка проблем, которые могут возникнуть.
    
- Исправление ошибок для проблем, которые могут повлиять на стабильность и удобство использования сервера.
    
- Исправления для системы безопасности уязвимостей, которые могут сделать сервер уязвимым для нарушений безопасности.
    
- Обновления часового пояса.
    
Установка Project Server 2007 продолжит работу после этой даты. Но из-за перечисленных выше изменений настоятельно рекомендуется выполнить миграцию из Project Server 2007 как можно скорее.

- Миграция в Project Online
    
- Переход на более новую локальную версию Project Server (желательно Project Server 2016)

|:-----|:-----|

## <a name="important-considerations-when-you-migrate-from-project-server-2007"></a>Важные рекомендации по миграции из Project Server 2007

При планировании миграции из Project Server 2007 учитывайте следующее:
  
- **Получение помощи от партнеров Майкрософт** — обновление с Project Server 2007 может быть проблематичным и требует большого количества подготовки и планирования. Это может быть особенно проблематичным, если вы не настроили приложение Project Server 2007 изначально. К счастью, у вас есть партнеры Майкрософт, которые могут помочь вам, если планируется переход на Project Server 2016 или Project Online. Выполните поиск партнера Майкрософт, который поможет вам выполнить миграцию в [центре партнеров Майкрософт](https://go.microsoft.com/fwlink/p/?linkid=841249). Выполните поиск по термину "  *Золотой проект" и "Управление портфелем* ", чтобы просмотреть список всех партнеров Майкрософт, имеющих опыт в Project. 
    
- **Планирование настроек** — многие настройки, внесенные в среде project Server 2007, могут не работать при переходе на project Server 2016 или Project Online. Существует существенные различия архитектуры Project Server между версиями. Кроме того, поддерживаются и другие операционные системы, серверы баз данных и клиентские веб-браузеры. Спланируйте, как тестировать или перестраивать настройки для новой среды. Кроме того, планирование дает возможность определить, требуется ли все настройки. Для получения дополнительных сведений ознакомьтесь [со статьей Создание плана для текущих настроек во время обновления до SharePoint 2013](https://go.microsoft.com/fwlink/p/?linkid=842061). 
    
- При планировании, выполнении и тестировании обновления **время и терпение** могут занимать много времени и усилий, особенно при обновлении до Project Server 2016. Например, если вы переносите данные из Project Server 2007 в Project Server 2016, сначала необходимо выполнить миграцию в Project Server 2010, проверить данные, а затем выполнить те же действия при переходе к каждой последующей версии. Вы можете обратиться к партнеру Майкрософт, чтобы получить оценки того, сколько времени займет и что он будет стоить.
    
## <a name="migrate-to-project-online"></a>Миграция в Project Online

Если вы решили выполнить миграцию из Project Server 2007 в Project Online, можно выполнить следующие действия, чтобы вручную перенести данные плана проекта:
  
1. Сохраните планы проекта из Project Server 2003 в MPP формат.
    
2. В Project профессиональный 2013, Project профессиональный 2016 или клиент Project Online для настольных ПК откройте каждый MPP-файл, а затем сохраните его и опубликуйте в Project Online.
    
Вы можете вручную создать конфигурацию Microsoft Project Web App (PWA) в Project Online. Например, повторно создайте все необходимые настраиваемые поля или корпоративные календари. Партнеры корпорации Майкрософт также могут помочь этому процессу.

## <a name="migrate-to-a-newer-on-premises-version-of-project-server"></a>Переход на новую локальную версию Project Server

Мы настоятельно считаем, что вы получаете лучшие значения и удобство работы пользователей, переполнив их в Project Online. Но также мы понимаем, что в некоторых организациях необходимо хранить данные проекта в локальной среде. Если вы решили хранить данные проекта локально, вы можете перенести среду Project Server 2007 в Project Server 2010, Project Server 2013 или Project Server 2016.
  
Если вы не можете выполнить миграцию в Project Online, рекомендуется выполнить миграцию в Project Server 2016. Project Server 2016 включает все функции предыдущих выпусков Project Server. Он наиболее полно соответствует интерфейсу Project Online, хотя некоторые функции доступны только в Project Online.
  
После каждой миграции необходимо убедиться, что данные успешно перенесены.
  
> [!NOTE]
>
  
### <a name="how-do-i-migrate-to-project-server-2016"></a>Как перейти к Project Server 2016?

Различия в архитектуре Project Server 2007 и Project Server 2016 предотвращают прямой путь миграции. Поэтому необходимо перенести данные Project Server 2007 в каждую повернутую версию Project Server до тех пор, пока не будет достигнуто приложение Project Server 2016.
  
Выполните следующие действия для Project Server 2016:
  
1. Миграция из Project Server 2007 в Project Server 2010.
    
2. Миграция из Project обслуживает 2010 в Project Server 2013.
    
3. Миграция из Project Server 2013 в Project Server 2016.
    
После каждой миграции убедитесь, что данные успешно перенесены.

#### <a name="how-do-i-upgrade"></a>Как выполнить обновление?

Дополнительные сведения см. [в статье Upgrade to Project Server 2010](https://go.microsoft.com/fwlink/p/?linkid=841812). Важно понимать, что существует два различных метода, которые можно использовать для обновления:
  
- **Обновление с присоединением баз данных:** Этот метод обновляет только содержимое среды, а не параметры конфигурации. Это необходимо, если вы обновляете Office Project Server 2007, развернутое на оборудовании, поддерживающем только серверную операционную систему 32. Существует два типа методов обновления с присоединением баз данных:
    
  - ***Полное обновление* с присоединением базы данных** — перенос данных проекта, хранящихся в базах данных Office Project Server 2007, а также данных сайта Microsoft Project Web App, хранящихся в базе данных контента SharePoint.
    
  - ***Базовое обновление* с присоединением баз данных** — перенос только данных проекта, хранящихся в базах данных Project Server.
    
- **Обновление на месте**: данные конфигурации фермы и всего контента фермы обновляются на существующем оборудовании в фиксированном порядке. Когда вы начнете процесс обновления, программа установки переводит всю ферму в автономный режим. Сайты веб-сайтов и Microsoft Project Web App недоступны до завершения обновления, а затем программа установки перезапускает сервер. После обновления на месте невозможно приостановить обновление или восстановить предыдущую версию. Лучше всего сделать зеркальное изображение рабочей среды и выполнить обновление на месте для этой среды, а не в рабочей среде.

## <a name="migrate-from-portfolio-server-2007"></a>Миграция с сервера Portfolio Server 2007

Project Portfolio Server 2007 использовался с Project Server 2007 для стратегии, определения приоритетов и оптимизации портфеля. Но данные из Project Portfolio Server 2007 не могут быть перенесены в. Для таких данных, как бизнес-факторы, потребуется повторное создание.
