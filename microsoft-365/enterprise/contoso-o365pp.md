# <a name="microsoft-365-apps-for-enterprise-deployment-for-contoso"></a>Развертывание приложений Microsoft 365 для предприятий в компании Contoso

Компания Contoso обновила свои компьютеры до Windows 10 Корпоративная и Microsoft 365 приложений для предприятий, чтобы обеспечить более эффективную совместную работу, лучшую безопасность и более современные возможности для рабочего стола. После того, как они оценивают свою инфраструктуру и деловые потребности, компания Contoso определила следующие ключевые требования для развертывания:

- Все компьютеры должны запускать приложения Microsoft 365 для предприятий.
- По возможности развертывание должно использовать существующие средства управления и инфраструктуру.
- Развертывание должно поддерживать несколько языков и существующих архитектур на устройствах пользователей.
- ПК должны оставаться актуальными и обеспечивать безопасность с минимальными затратами на администрирование и минимальным влиянием на пользователей.

## <a name="deployment-tools"></a>Средства развертывания

В зависимости от требований компания Contoso решила развернуть приложения Windows 10 Корпоративная и Microsoft 365 для предприятий с помощью Configuration Manager (Текущая ветвь). Configuration Manager масштабируется для крупных сред и обеспечивает расширенный контроль над установкой, обновлениями и параметрами. Кроме того, у него есть встроенные функции, упрощающие и эффективное развертывание Office и управление им, в том числе:

- Одноранговый кэш, который может способствовать ограничению емкости сети при развертывании на устройствах в удаленных расположениях.
- Панель мониторинга управления клиентами Office, которая упрощает развертывание Office и отслеживание обновлений и предоставляет администраторам доступ к последним функциям развертывания и управления.
- Интеллектуальное развертывание языкового пакета, в том числе автоматический развертывание того же языка, что и операционная система.
- Полностью поддерживаемый и простой в использовании способ удаления существующих версий Office из клиента во время развертывания.

В дополнение к диспетчеру конфигураций компания Contoso использовала [набор средств готовности для надстройки Office и VBA](https://docs.microsoft.com/deployoffice/readiness-toolkit-application-compatibility-microsoft-365-apps), бесплатный инструмент Майкрософт, чтобы оценить проблемы совместимости с макросами и надстройками Office.

## <a name="managing-deployment-and-updates"></a>Управление развертыванием и обновлениями

В приложениях Microsoft 365 для предприятия используется новая модель выпуска: Office в качестве службы. Модель службы упрощает отслеживание новых функций. Однако для ИТ-отделов часто требуется изменить способ развертывания и тестирования новых выпусков. Чтобы свести к минимуму проблемы совместимости и обеспечить своевременное обновление компьютеров, развернутые Windows и Office разворачиваются в два этапа:

- Во первых, они развернули приложения Microsoft 365 для предприятий на небольшой набор репрезентативных устройств в Организации. Эта Пилотная группа использовалась для тестирования приложений, надстроек и оборудования с помощью приложений Microsoft 365 для предприятий.
- Четыре месяца спустя после устранения всех критических проблем, связанных с приложениями, надстройками и оборудованием, в пилотной группе специалисты компании Contoso выполнили развертывание приложений Microsoft 365 для предприятий на остальных устройствах в организации (в широкой группе).

Вместо того чтобы управлять обновлениями Office с помощью диспетчера конфигураций, компания Contoso включила автоматическое обновление из облака. Облачные обновления снижают затраты на администрирование и гарантируют, что устройства постоянно обновляются.

Компания Contoso предприняла один и тот же подход на основе двух стадий для обновления компонентов по мере того, как они использовались для развертывания Office: устройства в функции получения пилотной группы обновляются на четыре месяца раньше, чем в остальной части организации (широкая группа). Для реализации такого подхода для Office в компании Contoso использовали два указанных ниже рекомендованных [канала обновления](https://docs.microsoft.com/DeployOffice/overview-update-channels):

- Полугодовой канал (предварительная корпоративная версия) для обновлений в пилотной группе
- Semi-Annual корпоративный канал для обновлений широкой группы

Так как в Полугодовом канале (предварительной корпоративной версии) выпуск новых версий приложений Microsoft 365 для предприятий происходит на 4 месяца раньше, чем в Полугодовом канале (корпоративном), в распоряжении специалистов компании Contoso достаточно времени, чтобы проверить обновления без необходимости управлять ими.

## <a name="deployment-process"></a>Процесс развертывания

Чтобы выполнить развертывание Office, в компании Contoso реализовали указанный ниже процесс, в котором были использованы рекомендации корпорации Майкрософт.

1. Перед развертыванием компания Contoso использовала набор средств готовности для надстройки Office и VBA, чтобы протестировать свои приложения и надстройки Office, чтобы оценить их совместимость с приложениями Microsoft 365 для предприятий.
1. В диспетчере конфигурации они включили одноранговый кэш на своих клиентских устройствах, что помогает ограничить объем сети при развертывании на клиентских устройствах в удаленных расположениях. 
1. В качестве коллекций устройств в диспетчере конфигураций Contoso определила две группы развертывания: Пилотная группа и широкая группа. Пилотная группа, которая включала в себя небольшой набор репрезентативных устройств в Организации, использовалась для дополнительного тестирования приложений, надстроек и оборудования с приложениями Windows 10 Корпоративная и Microsoft 365 для предприятий.
1. Они создали пакеты развертывания для Office с помощью панели управления клиентов Office и мастера установщика Office 365, который является частью консоли Configuration Manager. Они создали два приложения Microsoft 365 для корпоративных пакетов, один для пилотной группы на Semi-Annual корпоративный канал (Предварительная версия) и один для широкой группы на корпоративном канале Semi-Annual.
2. Каждый пакет Office включал языковые пакеты для английского, французского и немецкого языков. Если устройство требует язык, не включенный в пакет Office, этот языковой пакет автоматически загружался из сети доставки содержимого (CDN) Office.
3. Перед установкой приложений Microsoft 365 для предприятий они автоматически удаляли все существующие MSI-версии Office с помощью встроенной в пакет Office функции.
4. В диспетчере конфигурации они развернули пакеты Windows и Office в точки распространения в сети. Затем они выполняли последовательности задач развертывания Configuration Manager, чтобы развернуть пилотный пакет приложений Microsoft 365 для предприятий в пилотной группе.
5. После устранения проблем совместимости с пилотной группой компания Contoso выполнила последовательности задач по развертыванию приложений Microsoft 365 для корпоративных пакетов в обширную группу.

В компании Contoso включено автоматическое обновление устройств из облака, поэтому нет необходимости управлять процессом в диспетчере конфигураций. Их устройства автоматически обновляются непосредственно в облаке на основе канала обновления, определенного при первоначальном развертывании.
