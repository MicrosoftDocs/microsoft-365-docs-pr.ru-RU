# <a name="microsoft-365-apps-for-enterprise-deployment-for-contoso"></a>Развертывание приложений Microsoft 365 для предприятий в компании Contoso

Компания Contoso обновила компьютеры до Windows 10 Корпоративная и приложений Microsoft 365 для предприятий, чтобы повысить эффективность совместной работы, повысить безопасность и повысить современный рабочий стол. После оценки инфраструктуры и бизнес-потребностей в компании Contoso определили эти ключевые требования для развертывания:

- На всех ПК должны работать приложения Microsoft 365 для предприятий.
- По возможности развертывание должно использовать существующие средства управления и инфраструктуру.
- Развертывание должно поддерживать несколько языков и существующие архитектуры на устройствах пользователей.
- Компьютеры должны оставаться в курсе и обеспечивать безопасность с минимальными затратами на ИТ-администрирование и минимальным влиянием на пользователей.

## <a name="deployment-tools"></a>Средства развертывания

В зависимости от своих требований компания Contoso решила развернуть Windows 10 Корпоративная и приложения Microsoft 365 для предприятий с помощью Configuration Manager (Current Branch). Configuration Manager масштабирования для крупных сред и предоставляет широкие возможности управления установкой, обновлениями и настройками. В нем также есть встроенные функции, которые упрощают и делают более эффективным развертывание Office и управление им, в том числе:

- Одноранговой кэш, который может помочь с ограниченной пропускной способностью сети при развертывании на устройствах в удаленных расположениях.
- Панель мониторинга управления клиентами Office, которая упрощает развертывание Office и мониторинг обновлений, а также предоставляет администраторам доступ к новейшим функциям развертывания и управления.
- Интеллектуальное развертывание языковых пакетов, включая автоматическое развертывание языка операционной системы.
- Полностью поддерживаемый и простой в использовании метод удаления существующих версий Office из клиента во время развертывания.

Помимо Configuration Manager, в компании Contoso использовали средство проверки готовности набор средств для надстроек Office и [VBA](https://docs.microsoft.com/deployoffice/readiness-toolkit-application-compatibility-microsoft-365-apps), бесплатное средство корпорации Майкрософт, для оценки проблем совместимости с макросами и надстройки Office.

## <a name="managing-deployment-and-updates"></a>Управление развертыванием и обновлениями

Приложения Microsoft 365 для предприятий используют новую модель выпуска: Office как услуга. Модель службы позволяет легко быть в курсе новых функций. Однако часто ИТ-отделам требуется изменить даты развертывания и тестирования новых выпусков. Чтобы свести к минимуму проблемы совместимости и обеспечить обновление компьютеров, в компании Contoso развернули Windows и Office в два этапа:

- Во-первых, они развернули приложения Microsoft 365 для предприятий на небольшом наборе представительных устройств в организации. Эта пилотная группа использовалась для тестирования приложений, надстройок и оборудования с приложениями Microsoft 365 для предприятий.
- Четыре месяца спустя после устранения всех критических проблем, связанных с приложениями, надстройками и оборудованием, в пилотной группе специалисты компании Contoso выполнили развертывание приложений Microsoft 365 для предприятий на остальных устройствах в организации (в широкой группе).

Вместо управления обновлениями Office с помощью Configuration Manager в компании Contoso включили автоматические обновления из облака. Облачные обновления сокращают затраты на администрирование, обеспечивая обновление устройств.

В компании Contoso использовали тот же двухугодный подход к обновлению функций, что и при развертывании Office: устройства в пилотной группе получали обновления функций на четыре месяца раньше, чем устройства в остальной части организации (широкая группа). Для реализации такого подхода для Office в компании Contoso использовали два указанных ниже рекомендованных [канала обновления](https://docs.microsoft.com/DeployOffice/overview-update-channels):

- Полугодовой канал (предварительная корпоративная версия) для обновлений в пилотной группе
- Semi-Annual Enterprise Channel для обновления для широкой группы

Так как в Полугодовом канале (предварительной корпоративной версии) выпуск новых версий приложений Microsoft 365 для предприятий происходит на 4 месяца раньше, чем в Полугодовом канале (корпоративном), в распоряжении специалистов компании Contoso достаточно времени, чтобы проверить обновления без необходимости управлять ими.

## <a name="deployment-process"></a>Процесс развертывания

Чтобы выполнить развертывание Office, в компании Contoso реализовали указанный ниже процесс, в котором были использованы рекомендации корпорации Майкрософт.

1. Перед развертыванием компания Contoso использовала надстройки Readiness набор средств для Office и VBA, чтобы протестировать свои приложения и надстройки Office, чтобы оценить их совместимость с приложениями Microsoft 365 для предприятий.
1. В Configuration Manager они включили одноранговой кэш на своих клиентских устройствах, что помогает с ограниченной пропускной способностью сети при развертывании на клиентских устройствах в удаленных расположениях. 
1. В Configuration Manager в компании Contoso определили две группы развертывания как коллекции устройств: пилотную и широкую. Пилотная группа, которая включала небольшой набор представительных устройств в организации, использовалась для дополнительного тестирования приложений, надстройок и оборудования с Windows 10 Корпоративная и приложениями Microsoft 365 для предприятий.
1. Они создали пакеты развертывания для Office с помощью панели мониторинга управления клиентами Office и мастера установщика Office 365, которые являются частью консоли Configuration Manager. Они создали два пакета приложений Microsoft 365 для предприятий: один для пилотной группы в корпоративном канале Semi-Annual (предварительная версия) и один для широкой группы в Semi-Annual Enterprise Channel.
2. Каждый пакет Office включал английский, французский и немецкий языковые пакеты. Если для устройства требовался язык, который не был включен в пакет Office, этот языковой пакет автоматически скачан из сети доставки содержимого (CDN) Office.
3. Перед установкой приложений Microsoft 365 для предприятий они автоматически удаляли все существующие MSI-версии Office с помощью встроенной в пакет Office функции.
4. В Configuration Manager они развернули пакеты Windows и Office в точках распространения в своей сети. Затем они запустили последовательности задач развертывания Configuration Manager, чтобы развернуть пилотный пакет приложений Microsoft 365 для предприятий в пилотной группе.
5. После решения проблем совместимости с пилотной группой компания Contoso запустила последовательности задач для развертывания пакета приложений Microsoft 365 для предприятий в широкой группе.

В компании Contoso включено автоматическое обновление устройств из облака, поэтому нет необходимости управлять процессом в диспетчере конфигураций. Их устройства автоматически обновляются непосредственно из облака на основе канала обновления, определенного в первоначальном развертывании.
