## <a name="deployment-tools"></a><span data-ttu-id="913f5-110">Средства развертывания</span><span class="sxs-lookup"><span data-stu-id="913f5-110">Deployment tools</span></span>

<span data-ttu-id="913f5-p102">В зависимости от требований компания Contoso решила развернуть приложения Windows 10 Корпоративная и Microsoft 365 для предприятий с помощью Configuration Manager (Текущая ветвь). Configuration Manager масштабируется для крупных сред и обеспечивает расширенный контроль над установкой, обновлениями и параметрами. Кроме того, у него есть встроенные функции, упрощающие и эффективное развертывание Office и управление им, в том числе:</span><span class="sxs-lookup"><span data-stu-id="913f5-p102">Based on their requirements, Contoso chose to deploy Windows 10 Enterprise and Microsoft 365 Apps for enterprise through Configuration Manager (Current Branch). Configuration Manager scales for large environments and provides extensive control over installation, updates, and settings. It also has built-in features to make it easier and more efficient to deploy and manage Office, including:</span></span>

- <span data-ttu-id="913f5-114">Одноранговый кэш, который может способствовать ограничению емкости сети при развертывании на устройствах в удаленных расположениях.</span><span class="sxs-lookup"><span data-stu-id="913f5-114">Peer cache, which can help with limited network capacity when deploying to devices in remote locations.</span></span>
- <span data-ttu-id="913f5-115">Панель мониторинга управления клиентами Office, которая упрощает развертывание Office и отслеживание обновлений и предоставляет администраторам доступ к последним функциям развертывания и управления.</span><span class="sxs-lookup"><span data-stu-id="913f5-115">The Office Client Management dashboard, which makes it easy to deploy Office and monitor updates and gives administrators access to the latest deployment and management features.</span></span>
- <span data-ttu-id="913f5-116">Интеллектуальное развертывание языкового пакета, в том числе автоматический развертывание того же языка, что и операционная система.</span><span class="sxs-lookup"><span data-stu-id="913f5-116">Intelligent language pack deployment, including automatically deploying the same language as the operating system.</span></span>
- <span data-ttu-id="913f5-117">Полностью поддерживаемый и простой в использовании способ удаления существующих версий Office из клиента во время развертывания.</span><span class="sxs-lookup"><span data-stu-id="913f5-117">A fully supported and easy-to-use method of removing existing versions of Office from a client during deployment.</span></span>

<span data-ttu-id="913f5-118">В дополнение к диспетчеру конфигураций компания Contoso использовала [набор средств готовности для надстройки Office и VBA](https://docs.microsoft.com/deployoffice/readiness-toolkit-application-compatibility-microsoft-365-apps), бесплатный инструмент Майкрософт, чтобы оценить проблемы совместимости с макросами и надстройками Office.</span><span class="sxs-lookup"><span data-stu-id="913f5-118">In addition to Configuration Manager, Contoso used the [Readiness Toolkit for Office add-in and VBA](https://docs.microsoft.com/deployoffice/readiness-toolkit-application-compatibility-microsoft-365-apps), a free tool from Microsoft, to assess compatibility issues with their Office macros and add-ins.</span></span>

## <a name="managing-deployment-and-updates"></a><span data-ttu-id="913f5-119">Управление развертыванием и обновлениями</span><span class="sxs-lookup"><span data-stu-id="913f5-119">Managing deployment and updates</span></span>

<span data-ttu-id="913f5-p103">В приложениях Microsoft 365 для предприятия используется новая модель выпуска: Office в качестве службы. Модель службы упрощает отслеживание новых функций. Однако для ИТ-отделов часто требуется изменить способ развертывания и тестирования новых выпусков. Чтобы свести к минимуму проблемы совместимости и обеспечить своевременное обновление компьютеров, развернутые Windows и Office разворачиваются в два этапа:</span><span class="sxs-lookup"><span data-stu-id="913f5-p103">Microsoft 365 Apps for enterprise has a new release model: Office as a service. The service model makes it easy to stay up to date with new features. But it often requires IT departments to change how they deploy and test new releases. To minimize compatibility issues and to ensure their computers stay up to date, Contoso deployed Windows and Office in two stages:</span></span>

- <span data-ttu-id="913f5-124">Во первых, они развернули приложения Microsoft 365 для предприятий на небольшой набор репрезентативных устройств в Организации.</span><span class="sxs-lookup"><span data-stu-id="913f5-124">First, they deployed Microsoft 365 Apps for enterprise to a small set of representative devices across the organization.</span></span> <span data-ttu-id="913f5-125">Эта Пилотная группа использовалась для тестирования приложений, надстроек и оборудования с помощью приложений Microsoft 365 для предприятий.</span><span class="sxs-lookup"><span data-stu-id="913f5-125">This pilot group was used to test apps, add-ins, and hardware with Microsoft 365 Apps for enterprise.</span></span>
- <span data-ttu-id="913f5-126">Четыре месяца спустя после устранения всех критических проблем, связанных с приложениями, надстройками и оборудованием, в пилотной группе специалисты компании Contoso выполнили развертывание приложений Microsoft 365 для предприятий на остальных устройствах в организации (в широкой группе).</span><span class="sxs-lookup"><span data-stu-id="913f5-126">Four months later, after addressing all critical issues with apps, add-ins, and hardware in the pilot group, Contoso deployed Microsoft 365 Apps for enterprise to the rest of the devices in the organization (the broad group).</span></span>

<span data-ttu-id="913f5-127">Вместо того чтобы управлять обновлениями Office с помощью диспетчера конфигураций, компания Contoso включила автоматическое обновление из облака.</span><span class="sxs-lookup"><span data-stu-id="913f5-127">Instead of managing updates to Office by using Configuration Manager, Contoso enabled automatic updates from the cloud.</span></span> <span data-ttu-id="913f5-128">Облачные обновления снижают затраты на администрирование и гарантируют, что устройства постоянно обновляются.</span><span class="sxs-lookup"><span data-stu-id="913f5-128">Cloud-based updates reduce administrative overhead while ensuring that devices stay up to date.</span></span>

<span data-ttu-id="913f5-129">Компания Contoso предприняла один и тот же подход на основе двух стадий для обновления компонентов по мере того, как они использовались для развертывания Office: устройства в функции получения пилотной группы обновляются на четыре месяца раньше, чем в остальной части организации (широкая группа).</span><span class="sxs-lookup"><span data-stu-id="913f5-129">Contoso followed the same two-stage approach for feature updates as they used for deploying Office: Devices in the pilot group received feature updates four months earlier than devices in the rest of the organization (the broad group).</span></span> <span data-ttu-id="913f5-130">Для реализации такого подхода для Office в компании Contoso использовали два указанных ниже рекомендованных [канала обновления](https://docs.microsoft.com/DeployOffice/overview-update-channels):</span><span class="sxs-lookup"><span data-stu-id="913f5-130">To enable this for Office, Contoso used two recommended [update channels](https://docs.microsoft.com/DeployOffice/overview-update-channels):</span></span>

- <span data-ttu-id="913f5-131">Полугодовой канал (предварительная корпоративная версия) для обновлений в пилотной группе</span><span class="sxs-lookup"><span data-stu-id="913f5-131">Semi-Annual Enterprise Channel (Preview) for updates to the pilot group</span></span>
- <span data-ttu-id="913f5-132">Semi-Annual корпоративный канал для обновлений широкой группы</span><span class="sxs-lookup"><span data-stu-id="913f5-132">Semi-Annual Enterprise Channel for updates to the broad group</span></span>

<span data-ttu-id="913f5-133">Так как в Полугодовом канале (предварительной корпоративной версии) выпуск новых версий приложений Microsoft 365 для предприятий происходит на 4 месяца раньше, чем в Полугодовом канале (корпоративном), в распоряжении специалистов компании Contoso достаточно времени, чтобы проверить обновления без необходимости управлять ими.</span><span class="sxs-lookup"><span data-stu-id="913f5-133">Because the Semi-Annual Enterprise Channel (Preview) releases a version of Microsoft 365 Apps for enterprise four months earlier than the Semi-Annual Enterprise Channel, Contoso has time to validate the updates without having to manage them.</span></span>

## <a name="deployment-process"></a><span data-ttu-id="913f5-134">Процесс развертывания</span><span class="sxs-lookup"><span data-stu-id="913f5-134">Deployment process</span></span>

<span data-ttu-id="913f5-135">Чтобы выполнить развертывание Office, в компании Contoso реализовали указанный ниже процесс, в котором были использованы рекомендации корпорации Майкрософт.</span><span class="sxs-lookup"><span data-stu-id="913f5-135">To complete the deployment of Office, Contoso implemented the following process, which includes best practice recommendations from Microsoft:</span></span>

1. <span data-ttu-id="913f5-137">В диспетчере конфигурации они включили одноранговый кэш на своих клиентских устройствах, что помогает ограничить объем сети при развертывании на клиентских устройствах в удаленных расположениях.</span><span class="sxs-lookup"><span data-stu-id="913f5-137">In Configuration Manager, they enabled peer cache on their client devices, which helps with limited network capacity when deploying to client devices in remote locations.</span></span>

2. <span data-ttu-id="913f5-142">Каждый пакет Office включал языковые пакеты для английского, французского и немецкого языков.</span><span class="sxs-lookup"><span data-stu-id="913f5-142">Each Office package included English, French, and German Language packs.</span></span> <span data-ttu-id="913f5-143">Если устройство требует язык, не включенный в пакет Office, этот языковой пакет автоматически загружался из сети доставки содержимого (CDN) Office.</span><span class="sxs-lookup"><span data-stu-id="913f5-143">If a device required a language that wasn't included in the Office package, that language pack was automatically downloaded from the Office Content Delivery Network (CDN).</span></span>

5. <span data-ttu-id="913f5-147">После устранения проблем совместимости с пилотной группой компания Contoso выполнила последовательности задач по развертыванию приложений Microsoft 365 для корпоративных пакетов в обширную группу.</span><span class="sxs-lookup"><span data-stu-id="913f5-147">After they addressed compatibility issues with the pilot group, Contoso ran the task sequences to deploy the Microsoft 365 Apps for enterprise package to the broad group.</span></span>

<span data-ttu-id="913f5-148">В компании Contoso включено автоматическое обновление устройств из облака, поэтому нет необходимости управлять процессом в диспетчере конфигураций.</span><span class="sxs-lookup"><span data-stu-id="913f5-148">Because Contoso chose to automatically update devices from the cloud, there was no need to manage the process in Configuration Manager.</span></span> <span data-ttu-id="913f5-149">Их устройства автоматически обновляются непосредственно в облаке на основе канала обновления, определенного при первоначальном развертывании.</span><span class="sxs-lookup"><span data-stu-id="913f5-149">Their devices are automatically updated directly from the cloud-based on the update channel that was defined in the initial deployment.</span></span>
