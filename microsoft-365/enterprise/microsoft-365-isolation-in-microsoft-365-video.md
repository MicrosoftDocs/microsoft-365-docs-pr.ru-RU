# <a name="tenant-isolation-in-office-365-video"></a>Изоляция клиента в Office 365 - Видео

> [!NOTE]
> Office 365 Video будет заменен Microsoft Stream. Дополнительные сведения о новой корпоративной видеослужба, которая добавляет аналитику для совместной работы с видео, и сведения о планах перехода для текущих клиентов Microsoft 365 Video см. в обзоре перехода [Office 365 Видео](https://docs.microsoft.com/stream/migrate-from-office-365)на Microsoft Stream.

## <a name="introduction"></a>Общие сведения

Служба хранилища Azure используется для хранения данных для нескольких служб Office 365, включая Office 365 Видео и Sway. Хранилище Azure включает хранилище больших объектов, которое является высоко масштабируемым облачным хранилищем объектов на основе REST, которое используется для хранения неструктурированных данных. Хранилище Azure использует простую модель управления доступом; Каждая подписка Azure может создать одну или несколько учетных записей хранения. Каждая учетная запись хранения имеет один секретный ключ, который используется для управления доступом ко всем данным в этой учетной записи хранения. Это поддерживает типичный сценарий, в котором хранилище связано с приложениями и эти приложения имеют полный контроль над связанными с ними данными; например, Sway, храня контент в хранилище Azure. Весь контент клиента для Sway хранится в общих учетных записях хранения Azure. Контент каждого пользователя находится в отдельном дереве каталогов BLOB-хранилищ Azure.

Системы, управляющие доступом к средам клиентов (например, порталу Azure, SMAPI и т. д.), изолированы в приложении Azure под управлением Майкрософт. Это логически отделяет инфраструктуру клиентского доступа от клиентских приложений и уровня хранилища.

## <a name="tenant-isolation-in-office-365-video"></a>Изоляция клиента в Office 365 - Видео

[Office 365 Видео](https://support.office.com/article/Meet-Office-365-Video-ca1cc1a9-a615-46e1-b6a3-40dbd99939a6) — это корпоративный портал, который предоставляет организациям высокобезопасный и безопасный для всей организации сайт для публикации, совместного использования и обнаружения видеоконтента. В Office 365 Видео видео каждого клиента хранятся в изоляции и шифруются во всех расположениях и доступны только пользователям, которые имеют доступ к видео организации и имеют разрешения на доступ к ним. Для этого Office 365 Видео использует сочетание технологий:

- SharePoint Online используется для хранения видео-файла и метаданных (название видео, описание и т. д.). Он также предоставляет уровень безопасности и соответствия требованиям (включая проверку подлинности) и функции поиска.
- Службы мультимедиа Azure предоставляют функции перекодинга, адаптивной потоковой передачи, безопасной доставки (с помощью шифрования AES) и эскизов.

[Azure Media Services](https://azure.microsoft.com/services/media-services/) — это предложение "платформа как услуга" для включения конечных процессов мультимедиа в облаке. Платформа предоставляет REST API для отправки, кодирования, шифрования (с помощью PlayReady) и доставки мультимедиа через центр обработки данных Azure по всему миру.

Все отправки для Office 365 Video происходят по протоколу HTTPS. При отправке видео файл сохраняется в SharePoint Online, а копия файла отправляется через зашифрованный канал в службы мультимедиа Azure. Службы мультимедиа Azure перекодируют видео в несколько форматов, оптимизированных для просмотра (например, мобильных устройств, низкой пропускной способности, высокой пропускной способности и т. д.). Эти файлы, а также исходный файл, полученный при отправке, хранятся в хранилище BLOB-файлов Azure. Файлы шифруются с помощью AES 128 по алгоритму упаковки общего шифрования MPEG (или более ранней версии PlayReady) для воспроизведения и шифруются с помощью шифрования хранилища AES 256 перед хранением в хранилище BLOB-файлов Azure. (Используя клиентский SDK служб мультимедиа Azure, клиенты могут контролировать, какое шифрование используется. Например, клиент может применить шифрование хранилища служб мультимедиа Azure (AES 256) к ресурсу мультимедиа с высоким значением перед отправкой в него хранилища BLOB-файлов Azure.)

Azure Media Services также создает эскиз для видео, которое оно передает вместе с метаданными эскиза в SharePoint Online через зашифрованный канал.
