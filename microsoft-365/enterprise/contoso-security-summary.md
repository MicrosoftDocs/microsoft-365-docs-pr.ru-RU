# <a name="summary-of-microsoft-365-for-enterprise-security-for-the-contoso-corporation"></a>Сводка Microsoft 365 для корпоративной безопасности корпорации Contoso

Чтобы получить разрешение на развертывание Microsoft 365 для предприятия, отдел ИТ-безопасности Contoso провел тщательный обзор безопасности. Они определили следующие требования к безопасности для облака:

- Используйте самые сильные методы проверки подлинности для доступа сотрудников к облачным ресурсам.
- Убедитесь, что компьютеры и мобильные устройства подключаются и подключаются к приложениям безопасным способом.
- Защита компьютеров и электронной почты от вредоносных программ.
- Разрешения на облачные цифровые активы определяют, кто может получить доступ к данным и что они могут сделать, и предназначены для доступа к данным с наименьшими привилегиями.
- Конфиденциальные и строго регулируемые цифровые активы маркируются, шифруются и хранятся в безопасных местах.
- Высокорегулируемые цифровые активы защищены дополнительным шифрованием и разрешениями.
- Сотрудники службы ИТ-безопасности могут отслеживать текущую позу безопасности с центральных панелей мониторинга и получать уведомления о событиях безопасности для быстрого реагирования и смягчения последствий.

## <a name="the-contoso-path-to-microsoft-365-security-readiness"></a>Путь Contoso к готовности к безопасности Microsoft 365

Contoso последовала этим шагам, чтобы подготовить свою безопасность для развертывания Microsoft 365 для предприятия:

1. Ограничение учетных записей администратора для облака

   Contoso подробно рассмотрела существующие учетные записи администраторов доменных служб Active Directory (AD DS) и настроил серию выделенных учетных записей и групп облачного администратора.

2. Классификация данных на три уровня безопасности

   Contoso тщательно проанализировала и определила три уровня, которые были использованы для определения microsoft 365 для корпоративных функций для защиты наиболее ценных данных.

3. Определение политик защиты информации, политик доступа к ней и ее хранения согласно уровням данных.

   На основе уровней данных Contoso определила подробные требования к квалификации будущих ИТ-рабочих нагрузок, которые перемещаются в облако.

Чтобы следовать требованиям безопасности и microsoft 365 для корпоративного развертывания, администраторы безопасности Contoso и его ИТ-отдел развернули множество функций и возможностей безопасности, как описано в следующих разделах.

## <a name="identity-and-access-management"></a>Управление идентификацией и доступом 

- Специальные учетные записи глобального администратора с использованием MFA и PIM

  Вместо того, чтобы назначать глобальную роль администратора учетным записям пользователей, Contoso создала три выделенные учетные записи глобального администратора с сильными паролями. Учетные записи защищены службами многофакторной проверки подлинности Azure AD (MFA) и Azure Active Directory (Azure AD) privileged Identity Management (PIM). *PIM доступно только с Microsoft 365 E5.*

  Вход с глобальной учетной записью администратора делается только для определенных административных задач. Пароли известны только назначенным сотрудникам и могут использоваться только в течение периода времени, настроенного в Azure AD PIM.

  Администраторы безопасности Contoso присвоили меньше ролей администратора учетным записям, соответствующим функции ИТ-сотрудника.

  Дополнительные сведения см. в статье [Роли администраторов в Microsoft 365](/office365/admin/add-users/about-admin-roles).

- Многофакторная проверка подлинности для всех учетных записей пользователей

  MFA добавляет дополнительный уровень защиты в процесс регистрации. Это требует, чтобы пользователи признали телефонный звонок, текстовое сообщение или уведомление приложения на своем смартфоне после правильного ввода пароля. В MFA учетные записи пользователей Azure AD защищены от несанкционированного входного доступа, даже если пароль учетной записи скомпрометирован.

   - Чтобы защититься от компрометации подписки Microsoft 365, Contoso требует MFA для всех учетных записей глобального администратора.
   - Для защиты от фишинговых атак, в ходе которых злоумышленник компрометирует учетные данные надежного пользователя в организации и отправляет вредоносные письма, в Contoso реализовали MFA для всех учетных записей пользователей, в том числе начальников отделов и высшего руководства.

- Повышение безопасности доступа для устройств и приложений с помощью политик условного доступа

  В Contoso применяются [политики условного доступа](../security/office-365-security/microsoft-365-policies-configurations.md) для удостоверений, устройств, Exchange Online и SharePoint. Политики условного доступа для удостоверений предусматривают обязательную смену паролей для пользователей с высокой степенью риска и блокирование использования клиентами приложений, которые не поддерживают современную проверку подлинности. Политики для устройств предусматривают определение утвержденных приложений и обязательное использование компьютеров и мобильных устройств, соответствующих требованиям. Политики условного доступа для Exchange Online предусматривают блокировку клиентов ActiveSync и настройку шифрования сообщений Office 365. Политики условного доступа для SharePoint предусматривают дополнительную защиту для конфиденциальных и жестко регулируемых сайтов.

- Windows Hello для бизнеса

  Contoso развернул [Windows Hello для](/windows/security/identity-protection/hello-for-business/hello-identity-verification) бизнеса, чтобы в конечном итоге устранить потребность в паролях с помощью сильной двух факторов проверки подлинности на ПК и мобильных устройствах под управлением Windows 10 Enterprise.

- Credential Guard в Защитнике Windows

  Чтобы блокировать целевые атаки и вредоносные программы, работающие в операционной системе с административными привилегиями, Contoso Защитник Windows credential [Guard](/windows/security/identity-protection/credential-guard/credential-guard) с помощью групповой политики AD DS.

## <a name="threat-protection"></a>Защита от угроз

- Защита от вредоносных программ с помощью антивирусной программы "Защитник Windows"

  С помощью [антивирусной программы "Защитник Windows"](/windows/security/threat-protection/windows-defender-antivirus/windows-defender-antivirus-in-windows-10) Contoso обеспечивает защиту от вредоносных программ и управляет такой защитой на компьютерах и устройствах под управлением Windows 10 Корпоративная.

- Защита потока электронной почты и ведения журнала аудита почтовых ящиков с помощью Microsoft Defender для Office 365 

  Contoso использует Exchange Online Protection и [Defender для Office 365](/office365/securitycompliance/office-365-atp) для защиты от неизвестных вредоносных программ, вирусов и вредоносных URL-адресов, передаваемых по электронной почте.

  Contoso также включила журнал аудита почтовых ящиков, чтобы определить, кто входит в почтовые ящики пользователей, отправляет сообщения и выполняет другие действия, выполняемые владельцем почтовых ящиков, делегированным пользователем или администратором.

- Мониторинг и предотвращение атак с помощью инструмента анализа угроз и реагирования на них в Office 365

  Contoso использует исследование и реагирование на угрозы [Office 365](/office365/securitycompliance/office-365-ti) для защиты пользователей, упростив выявление и устранение атак, а также предотвращение будущих атак.

- Защита от сложных угроз с помощью Advanced Threat Analytics

  С помощью [Advanced Threat Analytics (ATA)](/advanced-threat-analytics/what-is-ata) Contoso обеспечивает защиту от сложных целевых атак. ATA автоматически анализирует, изучает и определяет обычное и необычное поведение сущности (пользователя, устройства и ресурсов).

## <a name="information-protection"></a>Защита информации

- Защита конфиденциальных и жестко регулируемых цифровых ресурсов с помощью меток Azure Information Protection

  Contoso определила три уровня защиты данных и развернула метки конфиденциальности [Microsoft 365,](../compliance/sensitivity-labels.md) которые пользователи применяют к цифровым активам. Для своих торговых секретов и другой интеллектуальной собственности Contoso использует подлабели чувствительности для строго регулируемых данных. Этот процесс шифрует контент и ограничивает доступ к определенным учетным записям и группам пользователей.

- Предотвращение утечки данных интрасети с помощью защиты от потери данных

  Contoso настраивает политики предотвращения потери данных для Exchange Online, SharePoint и OneDrive для бизнеса, чтобы не допустить случайного или намеренного обмена конфиденциальными данными. [](../compliance/data-loss-prevention-policies.md)

- Предотвращение утечек данных на устройстве с помощью Windows Information Protection

  Contoso использует [Windows Information Protection (WIP)](/windows/security/information-protection/windows-information-protection/protect-enterprise-data-using-wip) для защиты от утечки данных с помощью интернет-приложений и служб, корпоративных приложений и данных на корпоративных устройствах и персональных устройствах, которые сотрудники приносят на работу.

- Мониторинг облака с помощью Microsoft Cloud App Security

  С помощью [Microsoft Cloud App Security](/cloud-app-security/what-is-cloud-app-security) корпорация Contoso создает схему своей облачной среды, отслеживает ее использование, определяет инциденты и события безопасности. *Microsoft Cloud App Security предлагается только в плане Microsoft 365 E5.*

- Управление устройствами с использованием Microsoft Intune

  Корпорация Contoso использует [Microsoft Intune](/intune/introduction-intune) для регистрации, контроля и настройки доступа для мобильных устройств и приложений, которые на них выполняются. Кроме того, в соответствии с политиками условного доступа для устройств необходимо использовать утвержденные приложения, а также компьютеры и мобильные устройства, соответствующие требованиям.

## <a name="security-management"></a>Управление безопасностью

- Центральная панель мониторинга безопасности для ИТ с Azure Defender

  Contoso использует [Защитник Azure,](https://azure.microsoft.com/services/security-center/) чтобы представить единое представление о безопасности и защите от угроз, управлять политиками безопасности во всех рабочих нагрузках и реагировать на кибератаки.

- Центральная панель мониторинга безопасности, доступная пользователям на консоли "Безопасность Windows"

  Contoso развернула приложение [Windows Security](/windows/security/threat-protection/windows-defender-security-center/windows-defender-security-center) на своих ПК и устройствах под управлением Windows 10 Enterprise, чтобы пользователи могли сразу увидеть свою позицию безопасности и принять меры.
