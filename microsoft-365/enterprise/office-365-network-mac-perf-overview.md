# <a name="network-connectivity-in-the-microsoft-365-admin-center"></a>Подключение к сети в центре Microsoft 365 администрирования

Центр администрирования Microsoft 365 теперь включает агрегированные метрики сетевой подключения, собранные из Microsoft 365 клиента и доступные для просмотра только административными пользователями в клиенте.

> [!div class="mx-imgBorder"]
> ![Средство тестирования подключения к сети](../media/m365-mac-perf/m365-mac-perf-admin-center.png)

**Сетевые оценки** и **сетевые** сведения отображаются в центре администрирования Microsoft 365 в центре **| Подключение к сети.**

> [!div class="mx-imgBorder"]
> ![Страница производительности сети](../media/m365-mac-perf/m365-mac-perf-page-nav.png)

>[!NOTE]
>Подключение к сети в Центре администрирования поддерживает клиентов в WW Commercial и Германии, но не GCC, GCC High, DoD или Китае.

При первом переходе на страницу производительности сети необходимо настроить расположения для того, чтобы увидеть карту производительности глобальной сети, оценку сети для всего клиента, процент пользователей, работающих удаленно или на месте, а также список текущих проблем для принятия мер и/или дальнейших исследований. На области обзоров можно сверлить, чтобы просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в обзоре производительности [сети в центре администрирования Microsoft 365 сети.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

Вам может быть предложено присоединиться к общедоступным предварительным просмотрам этой функции от имени организации. Принятие обычно происходит немедленно, после чего вы увидите страницу подключения к сети.

Чтобы получить доступ к странице подключения к сети, необходимо быть администратором организации в Microsoft 365. Административная роль "Читатель отчетов" будет иметь доступ к этой информации. Чтобы настроить расположения и другие элементы сетевого подключения, администратор должен быть частью роли администратора сервера, например роли администратора службы поддержки.

## <a name="pre-requisites-for-network-connectivity-assessments-to-appear"></a>Предварительные требования к оценке сетевого подключения

Чтобы начать работу, включите параметр выбора расположения, чтобы автоматически собирать данные с устройств с помощью служб Windows Location Services, перейдите в список расположения, чтобы добавить или загрузить данные о расположении, или запустите Microsoft 365 сетевого подключения из офисных расположений. В то время как возможность подключения к сети может быть оценена по всей организации, для определенных офисных местоположений необходимо будет сделать любые улучшения в области сетевого проектирования. Сведения о подключении к сети предоставляются для каждого расположения офиса после определения этих расположений. Существует три варианта получения сетевых оценок из офисных местоположений:

### <a name="1-enable-windows-location-services"></a>1. Включить Windows службы расположения

Для этого параметра необходимо иметь не менее двух компьютеров, работающих в каждом расположении офиса, которые поддерживают предварительные требования. OneDrive для Windows должна быть в курсе и устанавливаться на каждом компьютере. Дополнительные сведения о OneDrive версиях см. в OneDrive [заметки о выпуске.](https://support.office.com/article/onedrive-release-notes-845dcf18-f921-435e-bf28-4e24b95e5fc0) В ближайшем будущем планируется добавить сетевые измерения в другие Office 365 клиентские приложения.

Windows Служба расположения должна иметь согласие на компьютерах. Вы можете проверить это, запуская **приложение Карты** и найди себя. Его можно включить на одной машине с **Параметры | Конфиденциальность | Расположение,** в котором необходимо включить параметр _Разрешить приложениям_ доступ к вашему расположению. Windows Согласие служб расположения можно развернуть на ПК с помощью MDM или групповой политики с помощью параметра _LetAppsAccessLocation._

Вам не нужно добавлять расположения в Центре администрирования с помощью этого метода, так как они автоматически идентифицированы в разрешении города. Несколько расположений в одном городе не будут показаны при использовании служб Windows расположения. Сведения о расположении округлились до ближайших 300 метров на 300 метров, чтобы не было доступа к более точным сведениям о расположении.

Машины должны иметь Wi-Fi, а не кабель ethernet. Машины с кабелем ethernet не имеют точной информации о расположении.

Образцы измерений и расположения офисов должны появиться через 24 часа после того, как эти предварительные требования будут выполнены.

### <a name="2-add-locations-and-provide-lan-subnet-information"></a>2. Добавление местоположений и предоставление сведений о подсетях LAN

Для этого параметра Windows службы расположения или Wi-Fi не требуется. Ваша OneDrive для Windows должна быть в курсе и установлена по крайней мере на одном компьютере в этом расположении.

Кроме того, необходимо добавить расположения на странице **Locations или** импортировать их из CSV-файла. Добавленные расположения должны включать сведения о подсети локальной сети office.

Этот параметр позволяет иметь несколько офисов, определенных в городе.

Все тестовые измерения с клиентских компьютеров включают данные подсети LAN, которые соотносятся с данными о расположении офиса, которые вы ввели. Образцы измерений и расположения офисов должны появиться через 24 часа после того, как эти предварительные требования будут выполнены.

### <a name="3-manually-gather-test-reports-with-the-microsoft-365-network-connectivity-test-tool"></a>3. Вручную собирать тестовые отчеты с помощью средства Microsoft 365 подключения к сети

Для этого параметра необходимо идентифицировать человека в каждом расположении. Попросите их просмотреть [Microsoft 365](https://connectivity.office.com) тест подключения к сети на компьютере Windows, на котором у них есть административные разрешения. На веб-сайте они должны войти в свою учетную запись Office 365 для той же организации, что вы хотите увидеть результаты. Затем они должны щелкнуть **выполнить тест**. Во время теста есть скачаный тест подключения EXE. Они должны открыть и выполнить это. После завершения тестов результат теста загружается в Центр администрирования.

Тестовые отчеты связаны с расположением, если оно было добавлено с данными подсети LAN, в противном случае они показаны только в расположении города.

Образцы измерений и расположения офисов должны появиться через 2-3 минуты после завершения тестового отчета. Дополнительные сведения см. в Microsoft 365 тест сетевого подключения [(предварительный просмотр).](office-365-network-mac-perf-onboarding-tool.md)

## <a name="how-do-i-use-this-information"></a>Как использовать эти сведения?

**Сведения о сети,** связанные с ними рекомендации по производительности и сетевые оценки предназначены для разработки периметров сети для расположения офисов. Каждое представление содержит сведения о характеристиках производительности для определенного общего сетевого вопроса для каждого географического расположения, в котором пользователи имеют доступ к вашему клиенту. **Рекомендации по производительности** для каждой сетевой идеи предлагают определенные изменения в дизайне сетевой архитектуры, которые можно внести для улучшения взаимодействия пользователей, связанных Microsoft 365 подключения к сети. Оценка сети показывает, как подключение к сети влияет на взаимодействие пользователей, что позволяет сравнивать различные сетевые подключения расположения пользователей.

**Сетевые оценки** перегоняют совокупность многих показателей производительности сети в снимок работоспособности корпоративной сети, представляющую значение точки от 0 до 100. Сетевые оценки охватываются как для всего клиента, так и для каждого географического расположения, из которого пользователи подключаются к вашему клиенту, предоставляя администраторам Microsoft 365 простой способ мгновенно понять состояние сети предприятия и быстро сверлить подробный отчет для любого глобального расположения офиса.

Сложные предприятия с несколькими расположениями офисов и нетривиальными архитектурами периметра сети могут извлечь выгоду из этой информации либо во время первоначальной работы в Microsoft 365, либо для устранения проблем с производительностью сети, обнаруженных при росте использования. Обычно это необязательно для малых предприятий, использующих Microsoft 365, или для любых предприятий, у которых уже есть простое и прямое подключение к сети. Предприятия с более чем 500 пользователями и несколькими офисными расположениями, как ожидается, выиграют больше всего.

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительного просмотра и доступны только для Microsoft 365 клиентов, которые были зарегистрированы в программе предварительного просмотра функций.

## <a name="enterprise-network-connectivity-challenges"></a>Enterprise сетевых подключений

> [!div class="mx-imgBorder"]
> ![Клиентская сеть в облако](../media/m365-mac-perf/m365-mac-perf-first-last-mile.png)

Многие предприятия имеют конфигурации периметра сети, которые со временем выросли и в основном предназначены для размещения доступа сотрудников к веб-сайту, где большинство веб-сайтов не известны заранее и не соответствуют действительности. Основное внимание уделяется предотвращению вредоносных программ и фишинговых атак с этих неизвестных веб-сайтов. Эта стратегия конфигурации сети, хотя и полезна в целях безопасности, может привести к ухудшению Microsoft 365 производительности и пользовательского интерфейса.

## <a name="how-we-can-solve-these-challenges"></a>Как мы можем решить эти проблемы

Предприятия могут улучшить общее взаимодействие с [](./microsoft-365-network-connectivity-principles.md) пользователями и обеспечить безопасность среды, следуя Office 365 принципам подключения и используя функцию сетевого подключения Microsoft 365 Центра администрирования. В большинстве случаев выполнение этих общих принципов будет иметь значительное положительное влияние на задержку конечного пользователя, надежность службы и общую производительность Microsoft 365.

Иногда корпорации Майкрософт предложено исследовать проблемы производительности сети с Microsoft 365 для крупных корпоративных клиентов, и это часто имеет корневую причину, связанную с инфраструктурой периметра сети клиента. При обнаружении распространенной первопричины проблемы периметра клиентской сети мы стремимся определить простые тестовые измерения, которые ее идентифицируют. Тест с порогом измерения, который определяет определенную проблему, имеет важное значение, так как мы можем проверить одно и то же измерение в любом месте, определить, присутствует ли эта корневая причина, и поделиться им в качестве сетевой информации с администратором.

Некоторые сетевые сведения просто указывают на проблему, которая требует дополнительного исследования. Представление о сети, в котором у нас достаточно тестов, чтобы показать определенное действие по исправлению, чтобы исправить корневую причину, перечислены в качестве **рекомендуемых действий.** Эти рекомендации, основанные на живых метриках, которые раскрывают значения, которые находятся за пределами заранее определенного порога, гораздо более ценны, чем рекомендации по рекомендациям общей практики, так как они специфичная для вашей среды и покажут фактическое улучшение после внесения рекомендуемых изменений.

## <a name="network-connectivity-overview-in-the-microsoft-365-admin-center"></a>Обзор подключения к сети в центре администрирования Microsoft 365

Эти измерения в настоящее время используются для предоставления анализа архитектуры сетевой архитектуры и оценки сетевой сети, которые показаны на странице подключения к сети в центре администрирования Microsoft 365 сети. 

По умолчанию приблизительные сведения о расположении, связанные с сетевыми измерениями, определяют город, в котором находятся клиентские устройства. Оценка сети в каждом расположении отображается с цветом, а относительное число пользователей в каждом расположении представлено размером круга.

## <a name="remote-worker-assessment-and-user-connection-metrics"></a>Удаленная оценка рабочих и метрики подключения к пользователю

Мы классифицировать журналы сетевого трафика как удаленных или удаленных пользователей и показывать их проценты в разделе метрики подключения пользователей области обзора. Для городов, в которых есть удаленные пользователи, при открываемой странице этого расположения вы найдете конкретный балл оценки удаленной сети. В списке расположения будут как офисные расположения, так и удаленные рабочие города, которые можно отфильтровать и отсортировать. Мы предоставляем оценку удаленного работника с разбивкой точек для Exchange, SharePoint и Teams.

Сведения о сетевых сетях домашних пользователей агрегируются и сообщаются на уровне города и ограничиваются городами с не менее чем 5 удаленными сотрудниками. Мы не идентифицируем отдельных сотрудников, работающих из дома.

Расположения автоматически классифицируются как удаленные или удаленные, однако для обеспечения 100% классификации вы можете вводить все IP-адреса удаленного входа вручную. Если вы решите перейти по этому маршруту, после добавления всех IP-адресов **egress** enter вручную в флажке Locations Параметры после добавления всех IP-адресов отступа. Когда это будет сделано, все журналы сетевого трафика с IP-адресов, помеченных вами как на месте, всегда будут классифицироваться как офисы, а все другие IP-адреса отступа будут классифицироваться как удаленные.

## <a name="specific-office-location-network-performance-summary-and-insights"></a>Конкретные сводки и сведения о производительности сети расположения офисов

Выбор расположения офиса открывает страницу сводки, относящаяся к определенному расположению, с подробными сведениями о сетевом отступе, который был определен в результате измерений для этого расположения офиса.

> [!div class="mx-imgBorder"]
> ![Сведения о сетевых сведениях по расположению](../media/m365-mac-perf/m365-mac-perf-locations-plan-overview.png)

Карта сети периметра для пользователей организации в расположении показана с некоторыми или всеми этими элементами:

- **Office** - расположение офиса для страницы, на которая вы смотрите
- **Периметр сети** — расположение источника IP-адреса для подключений из расположения офиса. Это зависит от точности баз данных гео-IP-расположения
- **Exchange** передняя дверь службы — одна из рекомендуемых Exchange, к которую пользователи в этом расположении офиса должны подключаться
- **Exchange-** входная дверь Exchange службы, к которую подключены пользователи, но не рекомендуется
- **SharePoint входной** двери службы — одна из рекомендуемых SharePoint, к которую пользователи в этом расположении офиса должны подключаться
- **SharePoint-оптимальной** входной двери службы — входная дверь SharePoint, к которую подключены пользователи, но не рекомендуется
- **Сервер рекурсивного** решения DNS — расположение из базы данных геоИП обнаруженного рекурсивного рекурсизатора DNS, используемого для Exchange Online (при наличии)
- **Прокси-сервер** — расположение из базы данных ip-адресов обнаруженного прокси-сервера (при наличии) 

На странице сводки расположения офиса дополнительно показаны оценка сети расположения, история оценки сети, сравнение оценки этого расположения с другими клиентами в том же городе, а также список конкретных рекомендаций и рекомендаций, которые можно предпринять для повышения производительности и надежности сети.

Сравнение между клиентами в одном городе основано на ожидании того, что все клиенты имеют равный доступ к поставщикам сетевых услуг, инфраструктуре телекоммуникаций и расположенным поблизости точкам присутствия сети Майкрософт.

Имена местоположений можно настроить при добавлении нового расположения или редактировании существующего расположения в пролете расположения. Это обеспечивает гибкость для настройки имен расположения в любое время. Кроме того, при добавлении lan-подсетей непосредственно в вылет расположения мы показывем выпадающе список подсетей LAN с мягким соответствием, которые можно выбрать из. Можно добавить и изменить имена отдельных IP-адресов, отступив от офиса.

На вкладке подробные сведения на странице расположения офиса показаны конкретные результаты измерений, которые использовались для представления каких-либо сведений, рекомендаций и оценки сети. Это предусмотрено для того, чтобы сетевые инженеры могли проверять рекомендации и факторы в любых ограничениях или специфике в своей среде. Вы также найдете предполагаемое число пользователей для собранных образцов в этом офисе, а также удаленных рабочих в этом городе.

> [!div class="mx-imgBorder"]
> ![Сведения о местоположении](../media/m365-mac-perf/m365-mac-perf-locations-plan-details-all.png)


## <a name="sharing-network-assessment-data-with-microsoft"></a>Совместное использование данных оценки сети с Корпорацией Майкрософт

По умолчанию сетевые оценки для организации и сетевые сведения делятся с сотрудниками Майкрософт. Это не включает личные данные сотрудников, а только конкретные метрики сетевой оценки и сетевые сведения, показанные в центре администрирования для расположения вашего офиса. Кроме того, в него не включены имена расположения офиса или уличные адреса, поэтому необходимо сообщить им город и узнать, как будет поддерживаться имя офиса, который необходимо обсудить. Если это отключено, инженеры Корпорации Майкрософт, с помощью которые вы обсуждаете подключение к сети, не смогут просмотреть эту информацию. Включение этого параметра позволяет использовать только будущие данные, начиная со дня включения.

## <a name="csv-import-for-lan-subnet-office-locations"></a>CSV Import для расположения локального офиса подсети LAN

Для идентификации локального офиса подсети необходимо заранее добавить каждое расположение. Вместо добавления отдельных местоположений в вкладке **Locations** их можно импортировать из CSV-файла. Вы можете получить эти данные из других хранимых вами мест, таких как панель мониторинга качества вызовов или сайты и службы Active Directory.

В CSV-файле обнаруженное расположение города в столбце userEntered указывается как пустое, а вручную добавленное расположение офиса — 1.

1. В главном _окне Подключение к Microsoft 365_ щелкните вкладку **Locations.**

1. Нажмите **кнопку Импорт** чуть выше списка расположения. Появится **вылет местоположений** импортных офисов.

   > ![Сообщение импорта CSV](../media/m365-mac-perf/m365-mac-perf-import.png)

Это позволит вам правильно отформатировать CSV с заголовками столбцов, к которым можно добавить расположения. Вы можете оставить существующие экспортируемые расположения как они есть; они не будут дублироваться при импорте обновленного CSV. Если вы хотите изменить адрес существующего расположения, он будет обновлен при импорте CSV. Вы не можете изменить адрес обнаруженного города.

1. Откройте CSV и добавьте свои расположения, заполнив следующие поля в новой строке для каждого расположения, которые вы хотите добавить. Оставьте все остальные поля пустыми; значения, которые вы вводите в других полях, будут игнорироваться.

   1. **userEntered** (обязательно): должно быть 1 для добавленного нового расположения office subnet LAN
   1. **Имя** (обязательно): имя расположения офиса

   1. **Долгота** (необязательный вариант): заполненная Bing на картах при пустых адресах

Egress Диапазоны IP-адресов должны быть /24 размера сети и /24 не включены в ввод.

1. При добавлении расположения офиса и сберегнуть файл нажмите кнопку **Обзор** рядом с завершенным полем **Upload** и выберите сохраненный CSV-файл.

## <a name="faq"></a>Вопросы и ответы

### <a name="what-is-a-microsoft-365-service-front-door"></a>Что такое Microsoft 365 входная дверь службы?

Входная Microsoft 365 является точкой входа в глобальную сеть Майкрософт, где Office и службы прекращают свое сетевое подключение. Для оптимального подключения к Microsoft 365 сети рекомендуется прекратить подключение к сети в ближайшую Microsoft 365 входную дверь.

>[!NOTE]
>Microsoft 365 входная дверь службы не имеет прямого отношения к продукту службы входных дверей Azure, доступному на рынке Azure.

### <a name="what-is-an-optimal-microsoft-365-service-front-door"></a>Что является оптимальным Microsoft 365 входной двери службы?

Оптимальной Microsoft 365 входной двери службы является входная дверь, которая ближе всего к вашей сетевой отступной, как правило, в вашем городе или области метро. Используйте средство [Microsoft 365 (предварительный просмотр)](office-365-network-mac-perf-onboarding-tool.md) для определения расположения вашей входной двери Microsoft 365 службы и оптимальной входной двери службы. Если средство определяет оптимальное использование входной двери, вы оптимально подключайтесь к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение egress Интернета?

Расположение выхода из Интернета — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это также определено как место, где у вас есть устройство сетевого перевода адресов (NAT) и обычно, где вы подключаетсяе к поставщику интернет-служб (ISP). Если вы видите расстояние между расположением и расположением отступа к Интернету, это может указывать на значительный откат WAN.

### <a name="what-license-is-needed-for-this-capability"></a>Какая лицензия нужна для этой возможности?

Требуется лицензия, которая предоставляет доступ к Microsoft 365 центру администрирования.
