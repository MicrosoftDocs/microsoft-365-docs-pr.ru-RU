# <a name="network-connectivity-in-the-microsoft-365-admin-center-preview"></a>Сетевое подключение в Центре администрирования Microsoft 365 (предварительная версия)

Центр администрирования Microsoft 365 теперь включает сводные показатели сетевого подключения, собранные из клиента Microsoft 365 и доступные для просмотра только административными пользователями в клиенте.

> [!div class="mx-imgBorder"]
> ![Средство проверки сетевого подключения](../media/m365-mac-perf/m365-mac-perf-admin-center.png)

**Оценки сети и** **сетевые данные** отображаются в Центре администрирования Microsoft 365 в центре | **Подключение.**

> [!div class="mx-imgBorder"]
> ![Страница производительности сети](../media/m365-mac-perf/m365-mac-perf-page-nav.png)

>[!NOTE]
>Средство проверки сетевого подключения поддерживает клиентов в WW Commercial и Germany, но не на GCC Moderate, GCC High, DoD или Китае.

При первом переходе на страницу производительности сети вы увидите обзорную область, содержащую карту глобальной производительности сети, оценку сети для всего клиента и список текущих проблем. В этом обзоре вы можете просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в обзоре производительности сети в Центре администрирования [Microsoft 365.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

Вам может быть предложено присоединиться к предварительной версии этой функции от имени вашей организации. Как правило, принятие происходит немедленно, после чего вы увидите страницу подключения к сети. 

При переходе на страницу сетевого подключения вы увидите обзорную область, содержащую карту глобальной производительности сети, оценку сети для всего клиента и список текущих проблем. Для доступа к этой странице необходимо быть администратором организации в Microsoft 365. Административная роль "Читатель отчетов" будет иметь доступ на чтение этой информации. Для настройки местоположений и других элементов сетевого подключения администратор должен быть частью роли администратора сервера, такой как роль администратора службы поддержки. В этом обзоре вы можете просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в обзоре сетевого подключения в Центре администрирования [Microsoft 365.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

## <a name="pre-requisites-for-network-connectivity-assessments-to-appear"></a>Должны появиться предварительные требования для оценки сетевого подключения

Чтобы при начать работу, включите параметр определения местоположения для автоматического сбора данных с устройств с помощью служб расположения Windows, перейдите к списку расположений, чтобы добавить или отправить данные о расположении, или запустите проверку сетевого подключения Microsoft 365 из расположений вашего офиса. Несмотря на то что сетевое подключение можно оценить по всей организации, для определенных офисных мест потребуется внося любые улучшения сетевого дизайна. Сведения о сетевых подключениях предоставляются для каждого расположения офиса после определения этих расположений. Существует три варианта получения оценок сети из офисов:

### <a name="1-enable-windows-location-services"></a>1. Включить службы расположения Windows

Для этого варианта в каждом офисе должно быть по крайней мере два компьютера, которые поддерживают необходимые требования. На каждом компьютере необходимо установить OneDrive для Windows **версии 19.232** или выше. Дополнительные сведения о версиях OneDrive см. в заметках о [выпуске OneDrive.](https://support.office.com/article/onedrive-release-notes-845dcf18-f921-435e-bf28-4e24b95e5fc0) Показатели сети планируется добавить в другие клиентские приложения Office 365 в ближайшем будущем.

На компьютерах необходимо согласие службы расположения Windows. Это можно проверить, задав приложение **"Карты"** и выявив себя. Его можно включить на одном компьютере с **помощью | Конфиденциальность | Расположение,** где должен быть включен параметр _"Разрешить_ приложениям доступ к вашему расположению". Согласие служб расположения Windows можно развернуть на ПК с помощью MDM или групповой политики с параметром _LetAppsAccessLocation._

Добавлять расположения в Центре администрирования с помощью этого метода не требуется, так как они автоматически идентифицированы в разрешении города. С помощью службы расположения Windows нельзя показывать несколько офисов в городе. Перед отправкой сведения о расположении также округываются до ближайшего 300 метров на 300 метров, чтобы получить доступ к более точной информации о расположении было невозможно.

Компьютеры должны иметь Wi-Fi сети, а не кабель Ethernet. Компьютеры с кабель ethernet не имеют точной информации о расположении.

Примеры показателей и расположения офисов должны появиться через 24 часа после того, как эти требования будут выполнены.

### <a name="2-add-locations-and-provide-lan-subnet-information"></a>2. Добавление местоположений и предоставление сведений о подсети локальной сети

Для этого параметра ни службы расположения Windows, ни Wi-Fi не требуются. OneDrive для Windows **версии 20.161** или выше должен быть установлен на каждом компьютере в расположении.

Вам также необходимо добавить расположения на странице подключения к сети Центра администрирования или импортировать их из CSV-файла. Добавленные расположения должны включать сведения о подсети локальной сети Office.

Так как вы добавляете расположения, в городе может быть определено несколько офисов.

Все тестовые измерения с клиентских компьютеров включают сведения о подсети локальной сети, которые соотносятся с введенной информацией о расположении офиса. Примеры показателей и расположения офисов должны появиться через 24 часа после того, как эти требования будут выполнены.

### <a name="3-manually-gather-test-reports-with-the-microsoft-365-network-connectivity-test-tool"></a>3. Сбор отчетов о тестах вручную с помощью средства проверки сетевого подключения Microsoft 365

Для этого параметра необходимо идентифицировать человека в каждом расположении. Попросите пользователей просмотреть тест подключения к [сети Microsoft 365](https://connectivity.office.com) на компьютере с Windows, на котором у них есть административные разрешения. На веб-сайте им необходимо войти в свою учетную запись Office 365 для той же организации, в которую вы хотите увидеть результаты. Затем они должны нажать **кнопку "Выполнить тест".** Во время тестирования имеется скачавшая exE-файл теста подключения. Им также необходимо открыть и выполнить эту задачу. После завершения тестов результат теста будет загружен в Office 365.

Тестовые отчеты связаны с расположением, если оно было добавлено с информацией о подсети локальной сети, в противном случае они будут показаны только в расположении города.

Примеры показателей и расположения офисов должны появиться через 2-3 минуты после завершения тестового отчета. Дополнительные сведения см. в тесте сетевого подключения [Microsoft 365 (предварительная версия).](office-365-network-mac-perf-onboarding-tool.md)

## <a name="how-do-i-use-this-information"></a>Как использовать эти сведения?

**Анализ сети,** соответствующие рекомендации по производительности и оценки сети призваны помочь в разработке сетей периметров для офисов. Каждая информация предоставляет подробные сведения о характеристиках производительности для конкретной общей проблемы для каждого географического расположения, в котором пользователи имеют доступ к вашему арендатору. **Рекомендации по производительности** для каждого анализа сети предлагают определенные изменения в проекте сетевой архитектуры, которые можно внести для улучшения взаимодействия с пользователем, связанного с сетевым подключением Microsoft 365. Оценка сети показывает, как сетевое подключение влияет на взаимодействие с пользователем, что позволяет сравнить различные сетевые подключения к расположению пользователей.

**Оценки сети** объединяют множество метрик производительности сети в моментальный снимок работоспособности корпоративной сети, представленный значением 0–100. Оценка сети зависит как от всего клиента, так и для каждого географического расположения, из которого пользователи подключаются к вашему арендатору, предоставляя администраторам Microsoft 365 простой способ мгновенно понять состояние сети предприятия и быстро детализированную информацию о подробном отчете по любому расположению глобального офиса.

Сложные предприятия с несколькими офисными расположениями и нестандартными архитектурами периметра сети могут воспользоваться этой информацией при первоначальной подсети в Microsoft 365 или для устранения проблем с производительностью сети, обнаруженных при росте использования. Обычно это необязательно для малых предприятий, использующих Microsoft 365, или для тех предприятий, у которых уже есть простое и прямое сетевое подключение. Предприятия с более чем 500 пользователями и несколькими офисными расположениями получают наибольшее преимущество.

>[!IMPORTANT]
>Анализ сети, рекомендации по производительности и оценки в Центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, которые зарегистрированы в программе предварительного просмотра функций.

## <a name="enterprise-network-connectivity-challenges"></a>Проблемы с подключением к корпоративной сети

> [!div class="mx-imgBorder"]
> ![Клиентская сеть в облако](../media/m365-mac-perf/m365-mac-perf-first-last-mile.png)

Многие предприятия имеют конфигурации периметра сети, которые со временем стали все больше и в основном предназначены для размещения доступа сотрудников к веб-сайтам, где большинство веб-сайтов не известны заранее и не являются недоверными. Основное внимание уделяется предотвращению вредоносных программ и атак с неизвестных веб-сайтов. Эта стратегия конфигурации сети, хотя и полезна в целях безопасности, может привести к снижению производительности и пользовательского интерфейса Microsoft 365.

## <a name="how-we-can-solve-these-challenges"></a>Как мы можем решить эти проблемы

Предприятия могут улучшить общий пользовательский интерфейс и защитить свою среду, следуя принципам подключения [Office 365](https://aka.ms/pnc) и используя функцию сетевого подключения в Центре администрирования Microsoft 365. В большинстве случаев выполнение этих общих принципов оказывает значительное положительное влияние на задержку конечных пользователей, надежность служб и общую производительность Microsoft 365.

Иногда корпорации Майкрософт предложено изучить проблемы с производительностью сети с Microsoft 365 для крупных корпоративных клиентов, и они часто имеют первопричину, связанную с инфраструктурой периметра сети клиентов. При обнаружении распространенной основной причины проблемы периметра сети клиента мы стремимся определить простые тестовые показатели, которые ее идентифицируют. Тест с порогом измерения, который определяет конкретную проблему, имеет ценность, так как мы можем протестировать одно и то же измерение в любом месте, определить, присутствует ли эта корневая причина, и сообщить администратору о ней как о сетевом анализе.

Некоторые данные о сети просто указывают на проблему, которая требует дальнейшего анализа. Анализ сети, в котором у нас достаточно тестов, чтобы показать определенное действие по исправлению для устранения основной причины, указано в качестве **рекомендуемых действий.** Эти рекомендации, основанные на реальных метриках, которые показывают значения, которые не имеют предопределенного порогового значения, гораздо более ценны, чем рекомендации общего уровня, так как они специфически для вашей среды и будут показывать фактическое улучшение после внесения рекомендуемых изменений.

## <a name="specific-office-location-network-performance-summary-and-insights"></a>Сводка по производительности сети расположений в определенных офисах и анализ

При выборе расположения офиса откроется страница сводки, на которой отображаются подробные сведения об откате сети, определенной по измерениям для этого расположения офиса.

> [!div class="mx-imgBorder"]
> ![Сведения о сетевой статистике по расположению](../media/m365-mac-perf/m365-mac-perf-locations-plan-overview.png)

Отображается карта сети периметра для пользователей организации в расположении с некоторыми или всеми из этих элементов:

- **Расположение Office** — расположение офиса для страницы, на которая вы ищете
- **Периметр сети** — расположение ip-адреса источника для подключений из расположения офиса. Это зависит от точности баз данных географического расположения IP
- **Оптимальный вход в службу Exchange** — одна из рекомендуемых входных входных толок службы Exchange, к которую должны подключаться пользователи в этом офисе
- **Не оптимальный вход** в Exchange — входная входная возможность службы Exchange, к которую подключены пользователи, но не рекомендуется
- **Оптимальный вход в** службу SharePoint — одна из рекомендуемых входных точек службы SharePoint, к которую должны подключаться пользователи в этом офисе
- **Не оптимальный** вход в службу SharePoint — входная точка службы SharePoint, к которую подключены пользователи, но не рекомендуется
- **DNS-рекурсивный** разрешительный сервер — расположение из геообъективной базы данных IP обнаруженного рекурсивного разрешения DNS, используемого для Exchange Online (если доступно)
- **Прокси-сервер** — расположение из базы данных ГЕО IP обнаруженного прокси-сервера (если доступно) 

На странице сводки о расположении офиса также показана оценка сети расположения, история оценок сети, сравнение оценки этого расположения с другими клиентами в том же городе, а также список конкретных данных и рекомендаций, которые можно предпринять для повышения производительности и надежности сети.

Сравнение между клиентами в одном городе основано на том, что все клиенты имеют одинаковый доступ к поставщикам сетевых услуг, телекоммуникационной инфраструктуре и ближайшим сетевым точкам присутствия Майкрософт.

На вкладке "Сведения" на странице "Расположение офиса" показаны конкретные результаты измерения, которые использовались для анализа, рекомендаций и оценки сети. Это обеспечивается, чтобы сетевые инженеры могли проверить рекомендации и факторы в любых ограничениях или особенно областях своей среды.

## <a name="csv-import-for-lan-subnet-office-locations"></a>Импорт CSV для местоположений офисов подсети локальной сети

Для идентификации офиса подсети локальной сети необходимо добавить каждое расположение заранее. Вместо того чтобы добавлять отдельные офисы на вкладку **"Расположения",** их можно импортировать из CSV-файла. Вы можете получить эти данные из других мест, где они были сохранены, таких как панель мониторинга качества вызовов или сайты и службы Active Directory

В CSV-файле обнаруженное расположение города в столбце userEntered показывается пустым, а добавленное вручную расположение офиса показывается как 1.

1. В главном _окне подключения к Microsoft 365_ щелкните вкладку **"Расположения".**

1. Нажмите **кнопку "Импорт"** над списком местоположений. Появится **флайок** "Импорт местоположений офиса".

   > [!div class="mx-imgBorder"]
   > ![Сообщение импорта CSV](../media/m365-mac-perf/m365-mac-perf-import.png)

1. Щелкните ссылку "Загрузить текущие расположения **офиса" (CSV-файл),** чтобы экспортировать список текущих местоположений в CSV-файл и сохранить его на локальном жестком диске. Это обеспечит правильно отформатированный CSV-формат с заголовками столбцов, в которые можно добавлять расположения. Вы можете оставить существующие экспортные расположения как есть; они не будут дублироваться при импорте обновленной CSV-версии. Если вы хотите изменить адрес существующего расположения, он будет обновлен при импорте CSV.. Изменить адрес обнаруженного города невозможно.

1. Откройте CSV и добавьте расположения, заполнив следующие поля в новой строке для каждого расположения, который вы хотите добавить. Оставьте все остальные поля пустыми; значения, которые вы вводите в других полях, будут игнорироваться.

   1. **userEntered** (обязательно): должно быть 1 для нового расположения офиса подсети локальной сети
   1. **Адрес** (обязательно): физический адрес офиса
   1. **Широта** (необязательно). Заполнение из карт Bing подыском адреса, если пусто
   1. **Долгота** (необязательно). Заполнение из карт Bing подыском адреса, если пусто
   1. **Диапазоны** IP-адресов для выхода от 1 до 5 (необязательно). Для каждого диапазона введите имя цепи, за которым следует разделенный пробелом список допустимого IPv4- или IPv6-адресов CIDR. Эти значения используются для дифференциации нескольких офисов, где используются одинаковые IP-адреса подсети локальной сети. Все диапазоны IP-адресов на входе должны иметь размер /24, а /24 не включается в входные данные.
   1. **LanIps** (обязательно). Список диапазонов подсети локальной сети, которые используются в этом расположении офиса. Для ИД подсети локальной сети должен быть включен размер сети CIDR, в котором может быть от /8 до /29. Несколько диапазонов подсети локальной сети могут быть разделены запятой или запятой.
   
1. Когда вы добавили расположения в офисе и  сохранили файл, нажмите кнопку "Обзор" рядом с полем "Отправить завершенное" и выберите сохраненный CSV-файл. 

1. Файл будет автоматически проверен. Если имеются ошибки проверки, вы увидите сообщение об ошибке В файле импорта есть _некоторые ошибки. Просмотрите ошибки, исправте файл импорта и попробуйте еще раз._ Щелкните **ссылку "Открыть сведения об ошибке"** для списка определенных ошибок проверки полей.

   > [!div class="mx-imgBorder"]
   > ![Сообщение об ошибке импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-error.png)

1. Если в файле нет ошибок, вы увидите сообщение о _готовности отчета. Найдены расположения x, которые нужно добавить, и расположения x для обновления._ Нажмите **кнопку** "Импорт", чтобы отправить CSV-загрузку.

   > [!div class="mx-imgBorder"]
   > ![Сообщение о готовности к импорту CSV](../media/m365-mac-perf/m365-mac-perf-import-ready.png)

## <a name="faq"></a>Вопросы и ответы

### <a name="what-is-a-microsoft-365-service-front-door"></a>Что такое входной вход службы Microsoft 365?

Входная точка службы Microsoft 365 — это точка входа в глобальную сеть Майкрософт, в которой клиенты и службы Office прерывают сетевое подключение. Для оптимального сетевого подключения к Microsoft 365 рекомендуется прерывать сетевое подключение до ближайшего входа в Microsoft 365.

>[!NOTE]
>Клиентский доступ к службе Microsoft 365 не имеет прямой связи с продуктом Azure Front Door Service, доступным в Azure Marketplace.

### <a name="what-is-an-optimal-microsoft-365-service-front-door"></a>Что является оптимальным входным входом в службу Microsoft 365?

Оптимальным входным входом службы Microsoft 365 является тот, который находится ближе всего к выходу вашей сети, как правило, в вашем городе или области. Используйте средство [проверки подключения Microsoft 365 (предварительная версия),](office-365-network-mac-perf-onboarding-tool.md) чтобы определить расположение используемой службы Microsoft 365: переднего и оптимального. Если средство определяет оптимальный вариант использования переднего входа, вы оптимальным образом подключаетсяе к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение для интернет-эксгримации?

Выход из Интернета — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это расположение также определено как расположение, в котором имеется устройство трансляции сетевых адресов (NAT) и обычно используется подключение к поставщику услуг Интернета (ISP). Если между вашим расположением и расположением интернет-игреса вы увидите большой путь, это может указывать на значительную wan-сеть.
