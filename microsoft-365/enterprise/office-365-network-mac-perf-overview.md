# <a name="network-connectivity-in-the-microsoft-365-admin-center-preview"></a>Сетевое подключение в Центре администрирования Microsoft 365 (предварительная версия)

В Центре администрирования Microsoft 365 теперь включены собранные показатели подключения к сети, собранные из вашего клиента Microsoft 365 и доступные для просмотра только пользователями с правами администратора в вашем клиенте. **Сетевые оценки** и **сведения о сетевых аналитиках** отображаются в Центре администрирования Microsoft 365 в разделе "Здоровье" **| Подключение.**

![Страница "Производительность сети"](../media/m365-mac-perf/m365-mac-perf-page-nav.png)

При первом переходе на страницу сетевого подключения отображается панель обзора с картой глобальной производительности сети, оценка сети, областью действия которой ограничена весь клиент, и список текущих проблем. В этом обзоре можно детализировать данные, чтобы просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в [обзоре производительности сети в Центре администрирования Microsoft 365.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

## <a name="pre-requisites-for-network-connectivity-assessments-to-appear"></a>Предварительные требования для оценки сетевого подключения

Существует три варианта получения оценки сети из расположений офиса:

### <a name="1-enable-windows-location-services"></a>1. Включите службы расположения Windows

Для этого варианта необходимо, чтобы два компьютера работали в каждом офисе, которое поддерживает необходимые требования. OneDrive для Windows версии 19.232 или более поздней должен быть установлен на каждом компьютере. Дополнительные сведения о версиях OneDrive см. в [заметках о выпуске OneDrive.](https://support.office.com/article/onedrive-release-notes-845dcf18-f921-435e-bf28-4e24b95e5fc0) В ближайшем будущем планируется добавить меры сети в другие клиентские приложения Office 365.

Необходимо согласиться на компьютерах, используя службу местоположения Windows. Вы можете проверить это, **запустив** приложение "Карты" и выполнив поиск самостоятельно. Ее можно включить на одном компьютере с **расположением параметров**  ->  **Privacy**  ->  **конфиденциальности, где** параметр "Разрешить приложениям доступ к вашему местоположению" должен быть включен. Согласие на службы расположения Windows можно развернуть на пк с помощью MDM или групповой политики с _параметром LetAppsAccessLocation._

При этом добавлять в Центр администрирования расположения не нужно, так как они определяются автоматически по разрешению города. Нельзя показывать несколько местоположений офисов в городе с помощью служб определения расположения Windows.

На компьютерах следует использовать сеть Wi-Fi, а не кабель Ethernet. У компьютеров с кабелем Ethernet нет точной информации о расположении.

Образцы измерений и офисные расположения должны начинаться через 24 часа после выполнения этих необходимых условий.

### <a name="2-add-locations-and-provide-lan-subnet-information"></a>2. Добавьте расположения и укажите сведения о подсетях

Для этого варианта ни служба расположения Windows, ни Wi-Fi не требуется. Вам потребуется OneDrive для Windows версии 20.161.0811.0001 или более поздней версии на всех композициях в расположении.

Вам также потребуется добавить расположения на странице "Подключение к сети" Центра администрирования или импортировать их из CSV-файла. Добавляемые расположения должны содержать сведения о подсети, касающуюся организации Office.

Поскольку при добавлении расположений вы укажете в городе, в городе может быть определено несколько офисов.

Образцы измерений и офисные расположения должны начинаться через 24 часа после выполнения этих необходимых условий.

### <a name="3-manually-gather-test-reports-with-the-microsoft-365-network-connectivity-test-tool"></a>3. Сбор отчетов о тестировании вручную с помощью средства проверки возможности подключения к сети Microsoft 365

В этом случае необходимо определить человека в каждом месте. Попросите их [браузера для проверки сетевого подключения Microsoft 365](https://connectivity.office.com) на компьютере с Windows, на котором у них есть разрешения администратора. На веб-сайте необходимо войти в свою учетную запись Office 365 в том клиенте, для которого требуется просмотреть результаты. Затем они нажимат кнопку "Выполнить тест". В ходе теста отображается загруженный тестовый ЖУРНАЛ для подключения. THey необходимо открыть и выполнить. По завершении проверок результат теста отправляется в корпорацию Майкрософт.

Тестовые отчеты привязаны к расположению, если они были добавлены со сведениями о подсети, в противном случае они отображаются только в городе.

Образцы показателей и офисные расположения должны начинаться через 2–3 минуты после завершения результата теста.

## <a name="how-do-i-use-this-information"></a>Как использовать эти данные?

**Сетевая аналитика, связанные**с ними рекомендации по производительности и оценки сети предназначены для упрощения разработки периметров сети для расположений офисов. В каждом аналитике представлены сведения о характеристиках производительности для конкретной общей проблемы, касающиеся каждого географического расположения, в котором пользователи обращаются к вашему клиенту. **Рекомендации по производительности для** каждой сетей напредье определенные изменения в структуре сетевой архитектуры, которые можно внести для улучшения работы пользователей, связанных с сетевым подключением Microsoft 365. При оценке сети показано, как сетевое подключение влияет на влияние на взаимодействие с пользователем и позволяет сравнить различные сетевые подключения с расположениями.

**Оценки сети** разделяют на совокупность множества показателей производительности сети в снимок состояния корпоративной сети, который представлен значением точки от 0 до 100. Оценки сети относятся ко всему клиенту и каждому географическому расположению, из которого пользователи подключаются к клиенту, предоставляя администраторам Microsoft 365 простой способ быстро захвата идущего использования работоспособности сети предприятия и быстро изображения подробного отчета по всему глобальному расположению офиса.

Сложные предприятия с несколькими офисами или неупроизводимыми архитектурами периметра сети могут получить преимущества от этих сведений либо при первоначальном подключении к Microsoft 365, либо устранить проблемы с производительностью сети, обнаруженные ростом использования. Обычно это не требуется для малых предприятий с Microsoft 365 или для предприятий, которые уже вынуждены просто и напрямую подошло к сети. Ожидается, что преимущества в организации, содержащих более 500 пользователей, и расположения нескольких офисов отдается.

>[!IMPORTANT]
>Статус сети, рекомендации и оценки производительности в Центре администрирования Microsoft 365 в настоящее время найдены в состоянии предварительной версии и доступны только для клиентов Microsoft 365, которые были зарегистрированы в программе предварительной оценки функций.

## <a name="enterprise-network-connectivity-challenges"></a>Проблемы с подключением к корпоративной сети

![Сеть клиента в облако](../media/m365-mac-perf/m365-mac-perf-first-last-mile.png)

Во многих предприятиях конфигурации периметра сети отрочены со временем и в основном предназначены для поддержки доступа сотрудников к веб-сайту в Интернете, где большинство веб-сайтов неизвестны заранее и неступают в них. Избавление от вредоносных программ и фишинга для этих неизвестных веб-сайтов сосредочается на предыдущих обязанностях. Эта стратегия настройки сети, не сматриваясь в целях безопасности, может привлечь снижение производительности и удобства работы пользователей Microsoft 365.

## <a name="how-we-can-solve-these-challenges"></a>Как мы можем решить эти задачи

Предприятия могут повысить общий пользовательский интерфейс и защитить среду, используя [принципы подключения к Office 365](https://aka.ms/pnc) и используя функцию сетевого подключения к Центру администрирования Microsoft 365. В большинстве случаев приступая к этим общим принципам, существенное положительное воздействие на задержку конечного пользователя, надежность и общую производительность Microsoft 365 будет значительно возвращаться.

Иногда корпорация Майкрософт требует изучить проблемы с производительностью сети в Microsoft 365 для крупных предприятий, и часто возникает основная причина, связанная с инфраструктурой выхода из сети клиентов. Если обнаружена распространенная основная причина проблемы с периметром сети клиента, мы наберем определить тесты простого теста, которые определяют ее. Тест с пороговым значением измерения, определяющий конкретную проблему, полезно из-за того, что мы можем проверить то же самое измерение в любом местоположении, определить наличие основной причины и поделиться ею как сведения о сети для администратора.

В некоторых сетевых аналитиках просто указывается проблема, которая требует дальнейшего изучения. Аналитика сети, где у нас достаточно тестов, чтобы показать конкретные действия по исправлению, чтобы устранить основную причину, указано **в качестве рекомендуемого действия.** Эти рекомендации, основанные на динамическистях метриках, которые отражают значения, не выходят за заранее определенное пороговое значение, стоят существенно более ценные рекомендации, так как они являются специфическими для вашей среды и показывать фактическое улучшение после внесения рекомендуемых изменений.

## <a name="csv-import-for-lan-subnet-office-locations"></a>Импорт CSV-файлов для расположений офисов в локальной сети

Для идентификации офиса по локальной сети потребуется заранее добавить каждую полкупание. Вместо добавления отдельных расположений офисов на вкладке **"Расположения",** можно импортировать их из CSV-файла. Можно получить эти данные из других мест, сохраненных вами, например из панели мониторинга качества звонков или сайтов и служб Active Directory

В CSV-файле обнаруженному местоположению города присвоена метка **City,** а добавленное вручную расположение офиса помечается. **Location**

1. В основном _окне "Подключение к Microsoft 365"_ перейдите на **вкладку "Расположения".**
1. Нажмите **кнопку "Импорт"** над списком расположений. Появится **всплывающая поле** "Импорт местоположений офиса".

   ![Сообщение об ошибке импорта CSV](../media/m365-mac-perf/m365-mac-perf-import.png)

1. Щелкните **ссылку "Скачать текущее расположение офиса (CSV)", чтобы** экспортировать список текущих расположений в CSV-файл и сохранить его на локальный жесткий диск. В этом случае будет создан правильно отформатированный CSV-файл с заголовками столбцов, в которые можно добавить расположения. Можно оставить существующие экспортированные расположения как есть. они не будут дублироваться при импорте обновленного CSV. Если вы хотите изменить адрес существующего расположения, оно будет обновлено при импорте CSV- и мин. Адрес обнаруженного города изменить нельзя.
1. Откройте CSV-файл и добавьте расположения, заполнив следующие поля в новой строке для каждого добавляемого расположения. Все остальные поля оставьте пустыми; значения, вводимые в другие поля, будут игнорироваться.
   1. **Адрес** (обязательный): физический адрес офиса
   1. **Широта (необязательно):** заполняется из поиска карт Bing, если значение пустое
   1. **Долгота** (необязательно): заполняется из поиска карт Bing, если значение не задано
   1. **Диапазоны IP-адресов для выхода от 1 до 5** (необязательно): для каждого диапазона введите имя схемы, а затем — список действительных адресов IPv4 или IPv6. Эти значения используются для различения местоположений офисов, в которых используются одни и те же IP-адреса подсети локальной сети.
   1. **LanIps** (обязательно): создание списка диапазонов подсетей, используемых в этом офисе.
1. После добавления расположений офисов и сохранения файла нажмите кнопку **обзора** рядом с **полем "Отправить завершенное** поле" и выберите сохраненный CSV-файл.
1. Файл будет автоматически проверен. Если имеются ошибки проверки, в файле импорта будет _указаны ошибки. Проверьте ошибки, исправьте файл импорта и повторите попытку._ Щелкните ссылку **"Открыть сведения об ошибке** открытия" для списка определенных ошибок проверки полей.

   ![Сообщение об ошибке импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-error.png)

1. Если в файле нет ошибок, появится сообщение _"Отчет готов". Найдены расположения x, которые требуется добавить, и x расположений, которые требуется обновить._ Нажмите **кнопку** "Импорт", чтобы отправить CSV-файл.

   ![Готово к импорту CSV](../media/m365-mac-perf/m365-mac-perf-import-ready.png)

## <a name="faq"></a>Вопросы и ответы

### <a name="what-is-a-microsoft-365-service-front-door"></a>Что такое фондовая служба Microsoft 365?

The Microsoft 365 service front door is an entry point on global Microsoft network where Office clients and services terminate their network connection. Для оптимального сетевого подключения к Microsoft 365 рекомендуется прекращать сетевое подключение, ближайшее в директоре Microsoft 365.

>[!NOTE]
>Передний план службы Microsoft 365 не имеет прямого отношения с продуктом службы Microsoft Azure Front Door, доступным в Azure Marketplace.

### <a name="what-is-an-optimal-microsoft-365-service-front-door"></a>Что такое оптимальная передний план службы Microsoft 365?

Оптимальный интерфейс службы Microsoft 365 — это элемент, наиболее близкий к выходу сети, как правило, в вашем городе или метро в области. Используйте [тест возможности подключения Microsoft 365,](office-365-network-mac-perf-onboarding-tool.md) чтобы определить место переднего плана обслуживания Microsoft 365 и оптимальный интерфейсный дверь службы. Если средство определит, что передний дверь является оптимальным, вы оптимально подключаетесь к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое выходной интернет-адрес?

Интернет-адрес — это расположение, из которого сетевой трафик выходит из корпоративной сети и подключается к Интернету. Этот параметр также определяется как расположение, где используется устройство преобразования сетевых адресов (NAT) и, как правило, это местоположение установлено для подключения к поставщику услуг Интернета. Если у вас есть между вашим местоположением и расположением интернет-страницы, это может означать на значительное расходование глобальной сети.
