# <a name="network-connectivity-in-the-microsoft-365-admin-center-preview"></a>Подключение к сети в Центре администрирования Microsoft 365 (предварительный просмотр)

Центр администрирования Microsoft 365 теперь включает агрегированные метрики сетевого подключения, собранные из клиента Microsoft 365 и доступные для просмотра только административными пользователями в клиенте.

> [!div class="mx-imgBorder"]
> ![Средство тестирования подключения к сети](../media/m365-mac-perf/m365-mac-perf-admin-center.png)

**Сетевые оценки** и **сетевые** сведения отображаются в Центре администрирования Microsoft 365 в области **| Подключение.**

> [!div class="mx-imgBorder"]
> ![Страница производительности сети](../media/m365-mac-perf/m365-mac-perf-page-nav.png)

>[!NOTE]
>Средство тестирования подключения к сети поддерживает клиентов в WW Commercial и Германии, но не GCC Moderate, GCC High, DoD или Китае.

При первом переходе на страницу производительности сети вы увидите область обзора, содержащую карту производительности глобальной сети, оценку сети для всего клиента и список текущих проблем. В обзоре можно просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в [обзоре производительности Сети в Центре администрирования Microsoft 365.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

Вам может быть предложено присоединиться к общедоступным предварительным просмотрам этой функции от имени организации. Принятие обычно происходит немедленно, а затем вы увидите страницу подключения к сети. 

При переходе на страницу подключения к сети вы увидите область обзора, содержащую карту производительности глобальной сети, оценку сети для всего клиента и список текущих проблем. Чтобы получить доступ к этой странице, необходимо быть администратором организации в Microsoft 365. Административная роль "Читатель отчетов" будет иметь доступ к этой информации. Чтобы настроить расположения и другие элементы сетевого подключения, администратор должен быть частью роли администратора сервера, например роли администратора службы поддержки. В обзоре можно просмотреть конкретные показатели производительности сети и проблемы по расположению. Дополнительные сведения см. в обзоре подключения к сети в Центре администрирования [Microsoft 365.](#network-connectivity-overview-in-the-microsoft-365-admin-center)

## <a name="pre-requisites-for-network-connectivity-assessments-to-appear"></a>Предварительные требования к оценке сетевого подключения

Чтобы начать работу, включите параметр выбора расположения, чтобы автоматически собирать данные с устройств с помощью служб расположения Windows, перейдите в список расположения, чтобы добавить или загрузить данные о расположении, или запустите тест подключения к сети Microsoft 365 из офисных расположений. В то время как возможность подключения к сети может быть оценена по всей организации, для определенных офисных местоположений необходимо будет сделать любые улучшения в области сетевого проектирования. Сведения о подключении к сети предоставляются для каждого расположения офиса после определения этих расположений. Существует три варианта получения сетевых оценок из офисных местоположений:

### <a name="1-enable-windows-location-services"></a>1. Включить службы расположения Windows

Для этого параметра необходимо иметь не менее двух компьютеров, работающих в каждом расположении офиса, которые поддерживают предварительные требования. OneDrive для версии Windows должен быть в курсе и устанавливаться на каждом компьютере. Дополнительные сведения о версиях OneDrive см. в заметках [о выпуске OneDrive.](https://support.office.com/article/onedrive-release-notes-845dcf18-f921-435e-bf28-4e24b95e5fc0) В ближайшее время планируется добавить сетевые измерения в другие клиентские приложения Office 365.

На компьютерах необходимо получить согласие службы расположения Windows. Вы можете протестировать это, запуская приложение **Maps** и выявив себя. Его можно включить на одной машине с помощью **параметров | Конфиденциальность | Расположение,** в котором необходимо включить параметр _Разрешить приложениям_ доступ к вашему расположению. Согласие служб расположения Windows можно развернуть на ПК с помощью MDM или групповой политики с помощью параметра _LetAppsAccessLocation._

Вам не нужно добавлять расположения в Центре администрирования с помощью этого метода, так как они автоматически идентифицированы в разрешении города. Вы не можете показывать несколько офисных расположений в городе с помощью служб расположения Windows. Сведения о расположении также округлются до ближайших 300 метров на 300 метров перед отправкой, чтобы получить доступ к более точным сведениям о расположении.

Машины должны иметь Wi-Fi, а не кабель ethernet. Машины с кабелем ethernet не имеют точной информации о расположении.

Образцы измерений и расположения офисов должны появиться через 24 часа после того, как эти предварительные требования будут выполнены.

### <a name="2-add-locations-and-provide-lan-subnet-information"></a>2. Добавление местоположений и предоставление сведений о подсетях LAN

Для этого параметра не требуется Wi-Fi службы расположения Windows. Ваша версия OneDrive для Windows должна быть в курсе и устанавливаться на каждом компьютере в этом расположении.

Кроме того, необходимо добавить расположения на странице подключения к сети Центра администрирования или импортировать их из CSV-файла. Добавленные расположения должны включать сведения о подсети локальной сети office.

Так как вы добавляете расположения, вы можете иметь несколько офисов, определенных в городе.

Все тестовые измерения с клиентских компьютеров включают данные подсети LAN, которые соотносятся с данными о расположении офиса, которые вы ввели. Образцы измерений и расположения офисов должны появиться через 24 часа после того, как эти предварительные требования будут выполнены.

### <a name="3-manually-gather-test-reports-with-the-microsoft-365-network-connectivity-test-tool"></a>3. Вручную собирать тестовые отчеты с помощью средства тестирования сетевого подключения Microsoft 365

Для этого параметра необходимо идентифицировать человека в каждом расположении. Попросите их просмотреть тест [подключения к сети Microsoft 365](https://connectivity.office.com) на компьютере Windows, на котором у них есть административные разрешения. На веб-сайте им необходимо войти в свою учетную запись Office 365 для той же организации, что и вы хотите увидеть результаты. Затем они должны щелкнуть **выполнить тест**. Во время теста есть скачаный тест подключения EXE. Они должны открыть и выполнить это также. После завершения тестов результат теста будет загружен в Office 365.

Тестовые отчеты связаны с расположением, если оно было добавлено с данными подсети LAN, в противном случае они показаны только в расположении города.

Образцы измерений и расположения офисов должны появиться через 2-3 минуты после завершения тестового отчета. Дополнительные сведения см. [в веб-сайте Microsoft 365 network connectivity test (preview).](office-365-network-mac-perf-onboarding-tool.md)

## <a name="how-do-i-use-this-information"></a>Как использовать эти сведения?

**Сведения о сети,** связанные с ними рекомендации по производительности и сетевые оценки предназначены для разработки периметров сети для расположения офисов. Каждое представление содержит сведения о характеристиках производительности для определенной общей проблемы для каждого географического расположения, в котором пользователи имеют доступ к вашему клиенту. **Рекомендации по производительности** для каждой сетевой идеи предлагают определенные изменения в дизайне сетевой архитектуры, которые можно внести для улучшения взаимодействия пользователей, связанных с подключением к сети Microsoft 365. Оценка сети показывает, как подключение к сети влияет на взаимодействие пользователей, что позволяет сравнивать различные сетевые подключения расположения пользователей.

**Сетевые оценки** перегоняют совокупность многих показателей производительности сети в снимок работоспособности корпоративной сети, представляющую значение точки от 0 до 100. Оценки сети охватываются как для всего клиента, так и для каждого географического расположения, из которого пользователи подключаются к вашему клиенту, предоставляя администраторам Microsoft 365 простой способ мгновенно понять состояние сети предприятия и быстро сверлить подробный отчет для любого глобального расположения офиса.

Сложные предприятия с несколькими расположениями офисов и нетривиальными архитектурами периметра сети могут воспользоваться этой информацией либо во время их первоначальной работы в Microsoft 365, либо для устранения проблем с производительностью сети, обнаруженных при росте использования. Обычно это необязательно для малых предприятий, использующих Microsoft 365, или для любых предприятий, у которых уже есть простое и прямое подключение к сети. Предприятия с более чем 500 пользователями и несколькими офисными расположениями, как ожидается, выиграют больше всего.

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в Центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительного просмотра и доступны только для клиентов Microsoft 365, которые были зарегистрированы в программе предварительного просмотра функций.

## <a name="enterprise-network-connectivity-challenges"></a>Проблемы подключения к корпоративной сети

> [!div class="mx-imgBorder"]
> ![Клиентская сеть в облако](../media/m365-mac-perf/m365-mac-perf-first-last-mile.png)

Многие предприятия имеют конфигурации периметра сети, которые со временем выросли и в основном предназначены для размещения доступа сотрудников к веб-сайту, где большинство веб-сайтов не известны заранее и не соответствуют действительности. Основное внимание уделяется предотвращению вредоносных программ и атак на рыбалку с этих неизвестных веб-сайтов. Эта стратегия конфигурации сети, хотя и полезна в целях безопасности, может привести к ухудшению производительности и пользовательского интерфейса Microsoft 365.

## <a name="how-we-can-solve-these-challenges"></a>Как мы можем решить эти проблемы

Предприятия могут улучшить общее взаимодействие с пользователями и обеспечить безопасность среды, следуя принципам подключения [Office 365](https://aka.ms/pnc) и используя функцию сетевого подключения Центра администрирования Microsoft 365. В большинстве случаев выполнение этих общих принципов оказывает значительное положительное влияние на задержку конечного пользователя, надежность службы и общую производительность Microsoft 365.

Иногда корпорации Майкрософт приходится исследовать проблемы производительности сети с Microsoft 365 для крупных корпоративных клиентов, и это часто имеет корневую причину, связанную с инфраструктурой периметра сети клиентов. При обнаружении распространенной первопричины проблемы периметра клиентской сети мы стремимся определить простые тестовые измерения, которые ее идентифицируют. Тест с порогом измерения, который определяет определенную проблему, имеет важное значение, так как мы можем проверить одно и то же измерение в любом месте, определить, присутствует ли эта корневая причина, и поделиться им в качестве сетевой информации с администратором.

Некоторые сетевые сведения просто указывают на проблему, которая требует дополнительного исследования. Представление о сети, в котором у нас достаточно тестов, чтобы показать определенное действие по исправлению, чтобы исправить корневую причину, перечислены в качестве **рекомендуемых действий.** Эти рекомендации, основанные на живых метриках, которые раскрывают значения, которые находятся за пределами заранее определенного порога, гораздо более ценны, чем рекомендации по рекомендациям общей практики, так как они специфичная для вашей среды и покажут фактическое улучшение после внесения рекомендуемых изменений.

## <a name="specific-office-location-network-performance-summary-and-insights"></a>Конкретные сводки и сведения о производительности сети расположения офисов

Выбор расположения офиса открывает страницу сводки, относящаяся к определенному расположению, с подробными сведениями о сетевом отступе, который был определен в результате измерений для этого расположения офиса.

> [!div class="mx-imgBorder"]
> ![Сведения о сетевых сведениях по расположению](../media/m365-mac-perf/m365-mac-perf-locations-plan-overview.png)

Карта сети периметра для пользователей организации в расположении показана с некоторыми или всеми этими элементами:

- **Расположение Office** . Расположение офиса для страницы, на которая вы смотрите
- **Периметр сети** — расположение источника IP-адреса для подключений из расположения офиса. Это зависит от точности баз данных гео-IP-расположения
- **Входная дверь** оптимальной службы Exchange — одна из рекомендуемых входных дверей службы Exchange, к которую должны подключаться пользователи в этом расположении офиса.
- **Входная дверь** exchange, не оптимальная, — входная дверь службы Exchange, к которую подключены пользователи, но не рекомендуется
- **Входная дверь** оптимальной службы SharePoint — одна из рекомендуемых входных дверей службы SharePoint, к которую должны подключаться пользователи в этом расположении офиса.
- Входная дверь службы **SharePoint,** не оптимальная, — входная дверь службы SharePoint, к которую подключены пользователи, но не рекомендуется
- **DNS recursive resolver server** - The location from a geo IP database of the detected DNS recursive resolver used for Exchange Online (if available)
- **Прокси-сервер** — расположение из базы данных ip-адресов обнаруженного прокси-сервера (при наличии) 

На странице сводки расположения офиса дополнительно показаны оценка сети расположения, история оценки сети, сравнение оценки этого расположения с другими клиентами в том же городе, а также список конкретных рекомендаций и рекомендаций, которые можно предпринять для повышения производительности и надежности сети.

Сравнение между клиентами в одном городе основано на ожидании того, что все клиенты имеют равный доступ к поставщикам сетевых услуг, инфраструктуре телекоммуникаций и расположенным поблизости точкам присутствия сети Майкрософт.

На вкладке подробные сведения на странице расположения офиса показаны конкретные результаты измерений, которые использовались для представления каких-либо сведений, рекомендаций и оценки сети. Это предусмотрено для того, чтобы сетевые инженеры могли проверять рекомендации и факторы в любых ограничениях или специфике в своей среде.

## <a name="csv-import-for-lan-subnet-office-locations"></a>CSV Import для расположения локального офиса подсети LAN

Для идентификации локального офиса подсети необходимо заранее добавить каждое расположение. Вместо добавления отдельных местоположений в вкладке **Locations** их можно импортировать из CSV-файла. Вы можете получить эти данные из других хранимых вами мест, таких как панель мониторинга качества вызовов или сайты и службы Active Directory.

В CSV-файле обнаруженное расположение города показывает в столбце userEntered как пустое, а вручную добавленное расположение офиса — 1.

1. В главном _окне Подключение к Microsoft 365_ щелкните вкладку **Locations.**

1. Нажмите **кнопку Импорт** чуть выше списка расположения. Появится **вылет местоположений** импортных офисов.

   > [!div class="mx-imgBorder"]
   > ![Сообщение импорта CSV](../media/m365-mac-perf/m365-mac-perf-import.png)

1. Щелкните **ссылку Download current office (.csv)** для экспорта текущего списка местоположений в CSV-файл и сохраните его на локальном жестком диске. Это позволит вам правильно отформатировать CSV с заголовками столбцов, к которым можно добавить расположения. Вы можете оставить существующие экспортируемые расположения как они есть; они не будут дублироваться при импорте обновленного CSV. Если вы хотите изменить адрес существующего расположения, он будет обновлен при импорте CSV. Вы не можете изменить адрес обнаруженного города.

1. Откройте CSV и добавьте свои расположения, заполнив следующие поля в новой строке для каждого расположения, которые вы хотите добавить. Оставьте все остальные поля пустыми; значения, которые вы вводите в других полях, будут игнорироваться.

   1. **userEntered** (обязательно): должно быть 1 для нового расположения локальной подсети
   1. **Адрес** (обязательно): физический адрес офиса
   1. **Широта** (необязательный). Заполнение с карт Bing адреса, если пустой
   1. **Долгота** (необязательный вариант): Заполнение с карт Bing искомого адреса, если пусто
   1. Ip-адрес Egress в диапазоне **1-5** (необязательный): Для каждого диапазона введите имя схемы с раздельным списком действительных ip-адресов IPv4 или IPv6 CIDR. Эти значения используются для дифференцированного расположения нескольких офисов, в которых используются те же IP-адреса подсети LAN. Диапазоны IP-адресов Egress должны быть /24 размера сети, а /24 не включены в входные данные.
   1. **LanIps** (обязательно): Список диапазонов подсетей LAN, которые используются в этом расположении офиса. Для ID-id-сетей lan-сетей необходимо включить размер сети CIDR, в котором размер сети может быть между /8 и /29. Несколько диапазонов подсетей LAN можно разделить запятой или запятой.
   
1. При добавлении расположения офиса и сберегнуть файл нажмите кнопку **Обзор** рядом с завершенным полем **Upload** и выберите сохраненный CSV-файл.

1. Файл будет проверяться автоматически. Если есть ошибки проверки, вы увидите сообщение об ошибке _Есть некоторые ошибки в файле импорта. Просмотрите ошибки, исправь файл импорта и попробуйте еще раз._ Щелкните **ссылку Откройте сведения об** ошибках для списка определенных ошибок проверки поля.

   > [!div class="mx-imgBorder"]
   > ![Сообщение об ошибке импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-error.png)

1. Если в файле нет ошибок, вы увидите сообщение, _готовое к отчету. Найдено x расположения для добавления и x расположения для обновления._ Нажмите **кнопку Импорт,** чтобы загрузить CSV.

   > [!div class="mx-imgBorder"]
   > ![Готовое сообщение для импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-ready.png)

## <a name="faq"></a>Вопросы и ответы

### <a name="what-is-a-microsoft-365-service-front-door"></a>Что такое входная дверь службы Microsoft 365?

Входная дверь службы Microsoft 365 является точкой входа в глобальную сеть Майкрософт, где клиенты и службы Office прекращают подключение к сети. Для оптимального подключения к сети Microsoft 365 рекомендуется прекратить подключение к сети в ближайшую входную дверь Microsoft 365.

>[!NOTE]
>Входная дверь службы Microsoft 365 не имеет прямого отношения к продукту службы входных дверей Azure, доступному на рынке Azure.

### <a name="what-is-an-optimal-microsoft-365-service-front-door"></a>Что является оптимальной входной дверью службы Microsoft 365?

Оптимальная входная дверь службы Microsoft 365 является ближайшей к вашей сетевой отступной, как правило, в вашем городе или области метро. Используйте [средство тестирования подключения Microsoft 365 (предварительный просмотр),](office-365-network-mac-perf-onboarding-tool.md) чтобы определить расположение входной двери службы Microsoft 365 и оптимальной входной двери службы. Если средство определяет оптимальное использование входной двери, вы оптимально подключаетсяе к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение egress Интернета?

Расположение выхода из Интернета — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это также определено как место, где у вас есть устройство сетевого перевода адресов (NAT) и обычно, где вы подключаетсяе к поставщику интернет-служб (ISP). Если вы видите расстояние между расположением и расположением отступа к Интернету, это может указывать на значительный откат WAN.
