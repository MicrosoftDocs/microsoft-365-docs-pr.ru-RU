# <a name="network-connectivity-in-the-microsoft-365-admin-center-preview"></a>Сетевое подключение в центре администрирования Microsoft 365 (Предварительная версия)

Центр администрирования Microsoft 365 теперь включает Объединенные метрики сетевого подключения, полученные от клиента Microsoft 365 и доступные для просмотра только администраторами в вашем клиенте.

![Средство проверки сетевого подключения](../media/m365-mac-perf/m365-mac-perf-admin-center.png)

**Оценки сети** и **аналитика** сети отображаются в центре администрирования Microsoft 365 в разделе **работоспособность | Подключение**.

![Страница "производительность сети"](../media/m365-mac-perf/m365-mac-perf-page-nav.png)

При первом переходе на страницу сетевое подключение вы увидите область обзора, содержащую карту глобальной сети, оценку сети для всего клиента и список текущих проблем. С помощью обзора можно детализировать, чтобы просмотреть конкретные метрики производительности сети и проблемы по их расположению. Для получения дополнительных сведений ознакомьтесь [со статьей Обзор производительности сети в центре администрирования Microsoft 365](#network-connectivity-overview-in-the-microsoft-365-admin-center).

## <a name="pre-requisites-for-network-connectivity-assessments-to-appear"></a>Отображаются предварительные требования для оценок сетевого подключения

Существует три способа получить оценки сети из расположений в офисе:

### <a name="1-enable-windows-location-services"></a>1. Включение служб расположения Windows

Для этого необходимо по крайней мере два компьютера работают в каждом расположении комнаты, которые поддерживают необходимые требования. На каждом компьютере должен быть установлен OneDrive для Windows версии **19,232** или более поздней версии. Дополнительные сведения о версиях OneDrive приведены в [заметках о выпуске onedrive](https://support.office.com/article/onedrive-release-notes-845dcf18-f921-435e-bf28-4e24b95e5fc0). В ближайшем будущем планируется добавление сетевых измерений в другие клиентские приложения Office 365.

Служба расположения Windows должна быть отослана на компьютерах. Это можно проверить, запустив приложение **Maps** и наполняя Поиск. Его можно включить на отдельном компьютере с **параметрами | Конфиденциальность | Расположение** , в котором параметр _Разрешить приложениям доступ к расположению_ должен быть включен. Разрешение служб расположения Windows можно развернуть на компьютерах с помощью MDM или групповой политики с параметром _летаппсакцесслокатион_.

Этот метод не требуется добавлять в центре администрирования, так как он автоматически идентифицируется при разрешении городов. Вы не можете отобразить несколько расположений Office в городе с помощью служб расположения Windows.

Компьютеры должны иметь сеть Wi/Fi, а не кабель Ethernet. У компьютеров с кабелем Ethernet нет достоверных сведений о расположении.

Для образцов измерений и расположений Office должно начаться через 24 часа после выполнения этих предварительных условий.

### <a name="2-add-locations-and-provide-lan-subnet-information"></a>2. Добавление расположений и предоставление сведений о подсети для локальной сети

Для этого параметра не требуются ни службы расположения Windows, ни Wi/Fi. Вам потребуется OneDrive для Windows версии 20,161 или выше, установленной на каждом компьютере в расположении.

Кроме того, необходимо добавить расположения на странице "Сетевое подключение" центра администрирования или импортировать их из CSV-файла. Добавляемые расположения должны включать сведения о подсети Office LAN.

Так как вы добавляете расположения, в городе может быть определено несколько офисов.

Для образцов измерений и расположений Office должно начаться через 24 часа после выполнения этих предварительных условий.

### <a name="3-manually-gather-test-reports-with-the-microsoft-365-network-connectivity-test-tool"></a>3. вручную Соберите тестовые отчеты с помощью средства проверки сетевого подключения Microsoft 365.

В этом случае необходимо указать человека в каждом месте. Попросите их браузеру [проверить сетевое подключение к Microsoft 365](https://connectivity.office.com) на компьютере с Windows, для которого у них есть разрешения администратора. На веб-сайте им нужно выполнить вход в свою учетную запись Office 365 для того же клиента, который вы хотите увидеть результаты. Затем нажмите кнопку выполнить тест. Во время тестирования можно загрузить тестовый EXE-файл подключения. Их также нужно открыть и выполнить. По завершении тестирования результаты теста отправляются в корпорацию Майкрософт.

Тестовые отчеты связаны с расположением, если оно было добавлено со сведениями о подсети локальной сети, в противном случае они отображаются только в расположении City.

Примеры измерений и расположения Office должны начаться в течение 2-3 минут после завершения теста.

## <a name="how-do-i-use-this-information"></a>Как использовать эту информацию?

Сведения о **сети**, рекомендации по производительности и оценки сети предназначены для создания периметра сети для офисов. Каждый из них предоставляет подробные сведения о характеристиках производительности для определенной распространенной ситуации для каждого географического расположения, в котором пользователи обращаются к клиенту. **Рекомендации по производительности** для каждой сетевой аналитики предоставляют конкретные изменения структуры сети, которые можно сделать для улучшения взаимодействия с пользователем, связанными с подключением к сети Microsoft 365. Оценка сети показывает, как сетевое подключение влияет на взаимодействие с пользователем, что позволяет сравнивать различные сетевые подключения в расположении пользователей.

В ходе **оценки сети** проводится преобразование множества метрик производительности сети в снимок работоспособности корпоративной сети, представленный значением баллов из 0-100. Оценка сети распространяется как на весь клиент, так и на каждое географическое расположение, из которого пользователи подключаются к клиенту, предоставляя администраторам Microsoft 365 простой способ незамедлительно оценить работоспособность сети предприятия и быстро перейти к подробному отчету по любому глобальному расположению в офисе.

Сложные предприятия с несколькими расположениями Office и нетривиальными архитектурами периметра сети могут выиграть от этих сведений во время первоначальной миграции до Microsoft 365 или для устранения проблем с производительностью сети, обнаруженных с ростом использования. Как правило, это не требуется для малых предприятий, использующих Microsoft 365, или предприятий, у которых уже есть простое и прямое сетевое подключение. Предприятия с более чем 500 пользователями и несколькими расположениями в офисе могут использовать самые выгодные преимущества.

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, зарегистрированных в программе предварительного просмотра компонентов.

## <a name="enterprise-network-connectivity-challenges"></a>Проблемы с подключением к корпоративной сети

![Сеть клиента в облако](../media/m365-mac-perf/m365-mac-perf-first-last-mile.png)

Многие предприятия имеют конфигурации периметра сети, которые увеличились со временем и в основном предназначены для обеспечения доступа к веб-сайту сотрудника, где большинство веб-сайтов неизвестны заранее и не являются доверенными. Необходимый и необходимый фокус не добегают атак вредоносных программ и фишинг с неизвестных веб-сайтов. Эта стратегия конфигурации сети, которая полезна в целях обеспечения безопасности, может привести к ухудшению производительности пользователей Microsoft 365 и взаимодействия с пользователем.

## <a name="how-we-can-solve-these-challenges"></a>Как можно решить эти проблемы

Предприятия могут повышать общий пользовательский интерфейс и защищать среду от использования [принципов подключения Office 365](https://aka.ms/pnc) и с помощью функции сетевого подключения центра администрирования Майкрософт 365. В большинстве случаев, следуя этим общим принципам, вы получите значительное положительное воздействие на задержку конечных пользователей, надежность службы и общую производительность Microsoft 365.

В некоторых случаях корпорация Майкрософт запрашивает сведения о проблемах с производительностью, связанными с Microsoft 365 для крупных корпоративных клиентов, и у них есть основная причина, связанная с инфраструктурой исходящего сетевого выхода клиентов. Если обнаружена общая причина проблемы с периметром сети клиента, мы будем находить простые измерения, идентифицирующие их. Тест с пороговостью измерения, определяющий конкретную проблему, является ценным, так как мы можем проверить такое же измерение в любом расположении, выяснить, присутствует ли эта основная причина, и поделиться ею с администратором.

Некоторые сетевые аналитики просто указывают на проблему, требующую дальнейшего изучения. Сетевая аналитика, в которой у нас достаточно тестов для отображения определенных действий по исправлению для исправления основной причины, приведено в качестве **рекомендуемого действия**. Эти рекомендации основаны на реальных показателях, которые показывают значения, которые выходят за пределы предварительно определенного порогового значения, являются гораздо более ценными, чем общие рекомендации для вашей среды, и будут показывать фактическое улучшение после внесения рекомендуемых изменений.

## <a name="csv-import-for-lan-subnet-office-locations"></a>Импорт CSV-файлов для расположений в подсети локальной сети

Для идентификации Office в локальной сети необходимо заранее добавить каждое расположение. Вместо добавления отдельных расположений Office на вкладке **расположения** можно импортировать их из CSV-файла. Вы можете получить эти данные из других хранящихся в ней мест, таких как панель мониторинга качества звонков или Active Directory — сайты и службы.

В CSV-файле найденный город называется **City**, а местоположение Office, добавленное вручную, — это **Расположение**с пометкой.

1. В окне основное _Подключение к Microsoft 365_ перейдите на вкладку **расположения** .
1. Нажмите кнопку **Импорт** , расположенную непосредственно над списком расположений. Появится всплывающее меню **импорта мест для Office** .

   ![Сообщение импорта в формате CSV](../media/m365-mac-perf/m365-mac-perf-import.png)

1. Щелкните ссылку **скачать текущие расположения Office (. csv)** , чтобы экспортировать список текущих расположений в CSV-файл, и сохраните его на локальный жесткий диск. В этом случае вам потребуется правильно отформатированный CSV-файл с заголовками столбцов, в которые можно добавлять расположения. Вы можете оставить существующие экспортированные расположения как есть; они не будут дублироваться при импорте обновленного CSV-файла. Если вы хотите изменить адрес существующего расположения, он будет обновлен при импорте CSV-файла. Вы не можете изменить адрес обнаруженного города.
1. Откройте CSV-файл и добавьте расположения, заполнив следующие поля в новой строке для каждого расположения, которое вы хотите добавить. Оставьте все остальные поля пустыми; значения, введенные в других полях, будут игнорироваться.
   1. **Адрес** (обязательный): физический адрес Office
   1. **Широта** (необязательно): заполняется из поиска карт Bing, если это поле пусто
   1. **Долгота** (необязательно): заполняется при поиске в картах Bing, если это поле пусто
   1. **Диапазоны IP-адресов выхода 1-5** (необязательно): для каждого диапазона введите имя цепи, а затем разделенный пробелами список допустимых адресов IPv4 или IPv6 CIDR. Эти значения используются для различения нескольких расположений Office, в которых используются одни и те же IP-адреса подсети локальной сети.
   1. **Ланипс** (обязательный параметр): список диапазонов ПОДсетей локальной сети, используемых в этом расположении Office.
1. Когда вы добавите расположения Office и сохранили файл, нажмите кнопку **Обзор** рядом с полем **отправить завершенный** файл и выберите сохраненный CSV-файл.
1. Файл будет автоматически проверен. Если имеются ошибки проверки, появится сообщение об ошибке, _в файле импорта есть ошибки. Просмотрите ошибки, исправьте файл импорта и повторите попытку._ Щелкните ссылку **открыть сведения об ошибке открыть** , чтобы получить список определенных ошибок проверки полей.

   ![Сообщение об ошибке импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-error.png)

1. Если в файле нет ошибок, отобразится сообщение _о том, что отчет готов. Найдено расположений x для добавления и расположений для обновления._ Нажмите кнопку **Импорт** , чтобы отправить CSV-файл.

   ![Сообщение о готовности импорта CSV](../media/m365-mac-perf/m365-mac-perf-import-ready.png)

## <a name="faq"></a>Вопросы и ответы

### <a name="what-is-a-microsoft-365-service-front-door"></a>Что такое Передняя дверца службы Microsoft 365?

Передняя дверца службы Microsoft 365 — это точка входа в глобальной сети Майкрософт, в которой клиенты и службы Office завершают свое сетевое подключение. Для оптимального сетевого подключения к Microsoft 365 рекомендуется завершить сетевое подключение к ближайшему передней дверце Microsoft 365.

>[!NOTE]
>Передняя дверца службы Microsoft 365 не имеет прямого отношения к продукту службы Front дверь Azure, доступному в Azure Marketplace.

### <a name="what-is-an-optimal-microsoft-365-service-front-door"></a>Что такое оптимальная Передняя дверца службы Microsoft 365?

Оптимальная Передняя дверца службы Microsoft 365 — это наиболее близкий к выходу из сети, как правило, в области город или Metro. Используйте [Тест подключения microsoft 365](office-365-network-mac-perf-onboarding-tool.md) , чтобы определить расположение вашей работающей передней дверцы службы Microsoft 365 и оптимальной дверцы службы. Если вы определили, что используемая Передняя дверца является оптимальной, то вы оптимально подключаются к глобальной сети Майкрософт.

### <a name="what-is-an-internet-egress-location"></a>Что такое расположение для выхода в Интернет?

Расположение для выхода в Интернет — это расположение, в котором сетевой трафик выходит из корпоративной сети и подключается к Интернету. Это также определяется как расположение, в котором у вас есть устройство преобразования сетевых адресов (NAT) и который обычно используется для подключения к поставщику услуг Интернета. Если вы видите большое расстояние между расположением и расположением для выхода в Интернет, это может указывать на значительное обратное размещение глобальной сети.
