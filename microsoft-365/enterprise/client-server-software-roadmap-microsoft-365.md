# <a name="client-and-server-software-roadmap-for-microsoft-365"></a>Дорожная карта клиентского и серверного программного обеспечения для Microsoft 365

Большинство корпоративных организаций имеют разнородную среду, которая включает несколько выпусков операционных систем, клиентского программного обеспечения и серверного программного обеспечения. Microsoft 365 для Enterprise включает наиболее безопасные версии ключевых компонентов ИТ-инфраструктуры. Он также включает функции производительности, предназначенные для использования облачных технологий.

Чтобы максимально увеличить бизнес-значение интегрированного пакета Microsoft 365 для Enterprise, приступить к планированию и реализации стратегии переноса выпусков:

- Клиент Office на компьютерах для Приложения Microsoft 365 для предприятий.
- Серверы Office серверов, установленных на серверах, к их эквивалентным службам в Microsoft 365.
- Windows 7 и Windows 8.1 на устройствах для Windows 10 Корпоративная.

>[!Note]
>Поддержка Windows 7 завершилась *14 января 2020 г.* Дополнительные сведения см. в [дополнительных сведениях.](https://support.microsoft.com/help/4057281/windows-7-support-will-end-on-january-14-2020)
>

По мере выполнения этих миграций с течением времени ваша организация приближается к зрению современного [рабочего места.](https://www.microsoft.com/microsoft-365/blog/2018/04/27/making-it-simpler-with-a-modern-workplace/) Эта безопасная и интегрированная среда поможет вам разблокировать командную работу и творческий подход в организации. Microsoft 365 для Enterprise позволяет и расширяет возможности на всем пути.

## <a name="migration-for-office-client-products"></a>Миграция Office клиентских продуктов

Организации как больших, так и малых часто используют сочетание старых версий Office клиентских продуктов, таких как Word, Excel и PowerPoint. Эти более старые версии:

- Можно обновить [с помощью](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5) последних обновлений и исправлений службы поддержки. Но этот процесс иногда вручную и может не масштабироваться по всей организации.
- Не оптимизированы для использования облачных технологий Майкрософт, которые помогают цифровому преобразованию бизнеса.
- Не предоставлять последние функции.

Microsoft 365 для Enterprise включает Приложения Microsoft 365 для предприятий. Эта версия Office клиентских продуктов доступна с Microsoft 365 для Enterprise лицензии. Он установлен и обновляется из облака Майкрософт. Приложения Microsoft 365 для предприятий включает обновления системы безопасности и новейшие функции. Дополнительные сведения см. [в Приложения Microsoft 365 для предприятий.](/deployoffice/about-microsoft-365-apps)

### <a name="office-2007"></a>Office 2007

Для версий Office в Office 2007 г. конец поддержки уже пройден. Дополнительные сведения [см. в Office дорожной карте 2007](/deployoffice/office-2007-end-support-roadmap)г. на конец 2007 г.

Вместо обновления компьютеров Office 2007 г. до Office 2010 г. Office 2013 г. или Office 2016 г., рассмотрите следующие действия:

1. Получите и назначьте Microsoft 365 лицензию для пользователей.
2. Удалить Office 2007 на своих компьютерах.
3. Установите Приложения Microsoft 365 для предприятий, как индивидуально, так и во время ИТ-установки. Дополнительные сведения см. в [руководстве по развертыванию для Приложений Microsoft 365](/deployoffice/deployment-guide-microsoft-365-apps).

Приложения Microsoft 365 для предприятий устанавливает обновления автоматически. Он может использовать облачные службы для повышения безопасности и производительности.

### <a name="office-2010"></a>Office 2010

Для версий Office в Office 2010 г. поддержка завершилась *13 октября 2020 г.* Дополнительные сведения [см. Office дорожную карту 2010](/deployoffice/office-2010-end-support-roadmap)г. на конец 2010 г.

Возможно, вы рассмотрите возможность обновления компьютеров Office 2010 Office 2013 или Office 2016 года. Однако обе эти версии должны обновляться вручную. Поэтому вместо этого следует предпринять следующие действия:

1. Получите и назначьте Microsoft 365 лицензию для пользователей.
2. Удалить Office 2010 на своих компьютерах.
3. Установите Приложения Microsoft 365 для предприятий, как индивидуально, так и во время ИТ-установки. Дополнительные сведения см. в [руководстве по развертыванию для Приложений Microsoft 365](/deployoffice/deployment-guide-microsoft-365-apps).

Приложения Microsoft 365 для предприятий автоматически устанавливает как обновления безопасности, так и новые обновления функций. Он может использовать облачные службы в Microsoft 365 для повышения безопасности и производительности.

### <a name="office-2013-and-office-2016"></a>Office 2013 и Office 2016

См. дорожную карту конца поддержки на [Office 2013 г.](/lifecycle/products/microsoft-office-2013) Окончание поддержки Office 2016 г. еще не определено. В этих версиях, как Office 2010 г., необходимо установить [обновления безопасности.](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5) Эта задача может быть не очень масштабироваться в зависимости от размера организации.

Вместо того, чтобы поддерживать компьютеры с последними обновлениями безопасности за Office 2013 или Office 2016 г. или обновлять компьютеры с Office по Office 2016 г., подумайте о следующих действиях:

1. Получите и назначьте Microsoft 365 лицензию для пользователей.
2. Удалить Office 2013 или Office 2016 на своих компьютерах.
3. Установите Приложения Microsoft 365 для предприятий, как индивидуально, так и во время ИТ-установки. Дополнительные сведения см. в [руководстве по развертыванию для Приложений Microsoft 365](/deployoffice/deployment-guide-microsoft-365-apps).

Приложения Microsoft 365 для предприятий устанавливает как обновления безопасности, так и обновления новых функций автоматически. Он может использовать облачные службы в Microsoft 365 для повышения безопасности и производительности.

## <a name="migration-for-office-server-products"></a>Миграция Office серверных продуктов

Как крупные, так и небольшие организации часто используют сочетание старых версий продуктов Office серверов, таких как Exchange Server и SharePoint Server. Эти более старые версии:

- Эти старые версии нужно обновить, применив последние обновления для системы безопасности и исправления. В некоторых случаях эти обновления выходят ежемесячно.
- Не оптимизированы для использования облачных технологий Майкрософт, которые помогают цифровому преобразованию бизнеса.
- Не включайте новые приложения производительности, такие как Microsoft Teams.
- Не включайте последние функции безопасности, такие как Exchange и Defender для Office 365.

Microsoft 365 для Enterprise включает облачные версии служб Office серверов, которые используют одни и те же инструменты, что и локальное программное обеспечение Office сервера, например веб-браузеры и клиент Outlook. Эти службы автоматически обновляются для обеспечения безопасности. Таким образом, ИТ-персонал экономит время, необходимое для обслуживания и обновления локального сервера. Эти службы также предлагают новые улучшения функций, которые не присутствуют в программном обеспечении Office сервера.

Используйте следующие ресурсы для сведений о миграции пользователей и данных для определенных Microsoft 365 рабочих нагрузок:

- [Перемещение почтовых ящиков из локального Exchange Server в Exchange Online](/exchange/hybrid-deployment/move-mailboxes)
- [Перенос SharePoint данных с SharePoint Server на SharePoint Online](/sharepointmigration/migrate-to-sharepoint-online)
- [Перенос Skype для бизнеса в Microsoft Teams](/microsoftteams/migration-interop-guidance-for-teams-with-skype)
