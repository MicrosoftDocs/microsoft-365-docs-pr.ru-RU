# <a name="client-and-server-software-roadmap-for-microsoft-365"></a>Дорожная карта клиентского и серверного программного обеспечения для Microsoft 365

Большинство корпоративных организаций имеют разнородную среду, которая включает несколько выпусков операционных систем, клиентского программного обеспечения и серверного программного обеспечения. Microsoft 365 для предприятия включает самые безопасные версии ключевых компонентов ИТ-инфраструктуры. Он также включает функции производительности, предназначенные для использования облачных технологий.

Чтобы максимально увеличить бизнес-значение интегрированного пакета продуктов Microsoft 365 для Enterprise, приступить к планированию и реализации стратегии переноса выпусков:

- Клиент Office, установленный на компьютерах в Microsoft 365 Apps для предприятия.
- Серверы Office, установленные на серверах, на эквивалентные службы в Microsoft 365.
- Windows 7 и Windows 8.1 на устройствах в Windows 10 Enterprise.

>[!Note]
>Поддержка Windows 7 завершилась 14 января *2020 г.* Дополнительные сведения см. в [дополнительных сведениях.](https://support.microsoft.com/help/4057281/windows-7-support-will-end-on-january-14-2020)
>

По мере выполнения этих миграций с течением времени ваша организация приближается к зрению современного [рабочего места.](https://www.microsoft.com/microsoft-365/blog/2018/04/27/making-it-simpler-with-a-modern-workplace/) Эта безопасная и интегрированная среда поможет вам разблокировать командную работу и творческий подход в организации. Microsoft 365 для предприятия включает и предоставляет вам все возможности на этом пути.

## <a name="migration-for-office-client-products"></a>Миграция для клиентских продуктов Office

Организации как больших, так и малых часто используют сочетание старых версий клиентских продуктов Office, таких как Word, Excel и PowerPoint. Эти более старые версии:

- Можно обновить [с помощью](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5) последних обновлений и исправлений службы поддержки. Но этот процесс иногда вручную и может не масштабироваться по всей организации.
- Не оптимизированы для использования облачных технологий Майкрософт, которые помогают цифровому преобразованию бизнеса.
- Не предоставлять последние функции.

Microsoft 365 для предприятия включает приложения Microsoft 365 для предприятия. Эта версия клиентских продуктов Office доступна с лицензией Microsoft 365 для предприятия. Он установлен и обновляется из облака Майкрософт. Приложения Microsoft 365 для предприятий включает обновления системы безопасности и новейшие функции. Дополнительные сведения см. [в дополнительных сведениях о приложениях Microsoft 365 для предприятия.](/deployoffice/about-microsoft-365-apps)

### <a name="office-2007"></a>Office 2007

Для версий Office в выпуске Office 2007 конец поддержки уже пройден. Дополнительные сведения см. в дорожной карте [Office 2007 по завершении поддержки.](/deployoffice/office-2007-end-support-roadmap)

Вместо обновления компьютеров с Office 2007 до Office 2010, Office 2013 или Office 2016 рассмотрите следующие действия:

1. Получите и назначьте пользователям лицензию Microsoft 365.
2. Удалить Office 2007 на своих компьютерах.
3. Установите приложения Microsoft 365 для предприятия как индивидуально, так и во время ИТ-2018. Дополнительные сведения см. в [руководстве по развертыванию приложений Microsoft 365.](/deployoffice/deployment-guide-microsoft-365-apps)

Приложения Microsoft 365 для предприятия устанавливают обновления автоматически. Он может использовать облачные службы для повышения безопасности и производительности.

### <a name="office-2010"></a>Office 2010

Для версий Office в выпуске Office 2010 поддержка завершилась *13 октября 2020 г.* Дополнительные сведения см. в дорожной карте [Office 2010 по завершении](/deployoffice/office-2010-end-support-roadmap)поддержки.

Возможно, вы рассмотрите возможность обновления компьютеров с office 2010 до Office 2013 или Office 2016. Однако обе эти версии должны обновляться вручную. Поэтому вместо этого следует предпринять следующие действия:

1. Получите и назначьте пользователям лицензию Microsoft 365.
2. Удалить Office 2010 на своих компьютерах.
3. Установите приложения Microsoft 365 для предприятия как индивидуально, так и во время ИТ-2018. Дополнительные сведения см. в [руководстве по развертыванию приложений Microsoft 365.](/deployoffice/deployment-guide-microsoft-365-apps)

Приложения Microsoft 365 для предприятия автоматически устанавливают обновления безопасности и новые обновления функций. Он может использовать облачные службы в Microsoft 365 для повышения безопасности и производительности.

### <a name="office-2013-and-office-2016"></a>Office 2013 и Office 2016

См. [дорожную карту конца поддержки Для Office 2013](/lifecycle/products/microsoft-office-2013). Окончание поддержки Office 2016 еще не определено. В этих версиях, например в Office 2010, необходимо установить [обновления безопасности.](https://support.office.com/article/install-office-updates-2ab296f3-7f03-43a2-8e50-46de917611c5) Эта задача может быть не очень масштабироваться в зависимости от размера организации.

Вместо того, чтобы поддерживать компьютеры с последними обновлениями безопасности для Office 2013 или Office 2016 или обновлять компьютеры с Office 2013 до Office 2016, рассмотрите следующие действия:

1. Получите и назначьте пользователям лицензию Microsoft 365.
2. Удалить Office 2013 или Office 2016 на своих компьютерах.
3. Установите приложения Microsoft 365 для предприятия как индивидуально, так и во время ИТ-2018. Дополнительные сведения см. в [руководстве по развертыванию приложений Microsoft 365.](/deployoffice/deployment-guide-microsoft-365-apps)

Приложения Microsoft 365 для предприятия устанавливают обновления безопасности и обновления новых функций автоматически. Он может использовать облачные службы в Microsoft 365 для повышения безопасности и производительности.

## <a name="migration-for-office-server-products"></a>Миграция для продуктов сервера Office

Как крупные, так и малые организации часто используют сочетание старых версий продуктов сервера Office, таких как Exchange Server и SharePoint Server. Эти более старые версии:

- Следует обновить с помощью последних обновлений безопасности и исправлений поддержки. В некоторых случаях эти обновления выпускаются ежемесячно.
- Не оптимизированы для использования облачных технологий Майкрософт, которые помогают цифровому преобразованию бизнеса.
- Не включайте новые приложения производительности, такие как Microsoft Teams.
- Не включайте последние функции безопасности, такие как Exchange и Defender для Office 365.

Microsoft 365 для предприятия включает облачные версии служб office server, которые используют одни и те же инструменты, что и локальное программное обеспечение сервера Office, например веб-браузеры и клиент Outlook. Эти службы автоматически обновляются для обеспечения безопасности. Таким образом, ИТ-персонал экономит время, необходимое для обслуживания и обновления локального сервера. Эти службы также предлагают новые улучшения функций, которые не присутствуют в программном обеспечении сервера Office.

Используйте следующие ресурсы для сведений о миграции пользователей и данных для определенных рабочих нагрузок Microsoft 365:

- [Перемещение почтовых ящиков из локального Exchange Server в Exchange Online](/exchange/hybrid-deployment/move-mailboxes)
- [Перенос данных SharePoint с SharePoint Server на SharePoint Online](/sharepointmigration/migrate-to-sharepoint-online)
- [Перенос Skype для бизнеса в Microsoft Teams](/microsoftteams/migration-interop-guidance-for-teams-with-skype)
