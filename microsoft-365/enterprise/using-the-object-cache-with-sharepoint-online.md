# <a name="using-the-object-cache-with-sharepoint-online"></a>Использование кэша объектов в SharePoint Online

В этой статье объясняется разница между использованием кэша объектов в локальной sharePoint Server 2013 и SharePoint Online.
  
В развертывании SharePoint Online существует значительное отрицательное влияние на кэш объектов. Любая зависимость от кэша объектов в SharePoint Online снизит надежность страницы. 
  
## <a name="how-the-sharepoint-online-and-sharepoint-server-2013-object-cache-works"></a>Как работает кэш объектов SharePoint Online и SharePoint Server 2013

Когда SharePoint Server 2013 находится в локальной сети, у клиента есть частные серверы переднего сервера, на которые находится кэш объектов. Это означает, что кэш выделен для одного клиента и ограничен только объемом доступной памяти и выделенным для кэша объектов. Так как в локальном сценарии обслуживается только один клиент, на серверах переднего сервера обычно есть пользователи, которые снова и снова запрашивают один и тот же сайт. Это означает, что кэш быстро заполнен и остается полным результатов запроса списка и объектов SharePoint, которые пользователи запрашивают регулярно.
  
![Показывает трафик и нагрузку на локальные веб-серверы переднего плана](../media/a0d38b36-4909-4abb-8d4e-4930814bb3de.png)
  
В результате при втором посещении страницы время загрузки страницы улучшается. После не менее четырех нагрузок одной страницы страница кэшется на всех веб-серверах переднего уровня.
  
С другой стороны, в SharePoint Online существует намного больше серверов, но и намного больше сайтов. Каждый пользователь может подключаться к разным интерфейсным веб-серверам, кэш которых не заполнен. Или, возможно, кэш заполняется для сервера, но следующий пользователь на этом интерфейсном веб-сервере запрашивает страницу с другого сайта. Или, даже если следующий пользователь запросит ту же страницу, что и во время предыдущего посещения, он балансирует нагрузку на другой интерфейсный веб-сервер, на который эта страница не есть в кэше. В последнем случае кэшинг не помогает пользователям.
  
На следующем рисунке каждая точка представляет страницу, запрашиваемую пользователем, и ее кэш. Различные цвета представляют различные клиенты, которые совместно используют инфраструктуру SaaS.
  
![Показывает результаты кэширования объектов в SharePoint Online](../media/25d04011-ef83-4cb7-9e04-a6ed490f63c3.png)
  
Как видно из схемы, вероятность того, что какой-либо пользователь нападет на сервер с кэшной версией своей страницы, ниже. Кроме того, из-за большой пропускной способности и того, что серверы совместно размещены между многими сайтами, кэш не длится долго, так как доступно только столько места для кэшинга.
  
По всем этим причинам получение пользователями кэшных объектов не является эффективным способом обеспечения качественного пользовательского интерфейса и времени загрузки страниц в SharePoint Online.
  
## <a name="if-we-cant-rely-on-the-object-cache-to-improve-performance-in-sharepoint-online-what-do-we-use-instead"></a>Если мы не можем полагаться на кэш объектов для повышения производительности в SharePoint Online, что мы используем вместо этого?

Это означает использование подходов для проблем производительности, которые не зависят от кэшинга объектов, чтобы дать пользователям хорошие результаты.
