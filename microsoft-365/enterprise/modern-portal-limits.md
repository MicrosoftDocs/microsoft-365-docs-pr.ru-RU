# <a name="sharepoint-online-modern-portal-site-limits"></a>Ограничения для сайтов современного портала SharePoint Online

В этой статье данная статья содержит рекомендации по производительности для современных сайтов портала в SharePoint Online. Используйте рекомендации в этой статье, чтобы оптимизировать производительность современных сайтов портала и избежать распространенных проблем с производительностью.

## <a name="performance-considerations-for-modern-portal-sites"></a>Вопросы производительности для современных сайтов портала

С точки зрения оптимизации производительности существует несколько характеристик, которые делают современные сайты портала уникальными. Основное различие между сайтами совместной работы и портала в SharePoint Online — масштаб. Как правило, сайты портала обслуживают большее количество просмотров страниц, чем сайты совместной работы, и, скорее всего, будут содержать больше статического контента и меньше редактируемых ресурсов. Кроме того, архитектура современных сайтов отличается от классических сайтов тем, что большая часть обработки страниц отрисовки и выполнения кода выполняется на клиенте, а не на сервере.

Оптимизация производительности для современных сайтов портала в основном ориентирована на несколько общих целей:

- Уменьшение общего размера компонентов каждой страницы сайта
- Разгрузка размещения общих статических файлов, таких как изображения, таблицы стилей и сценарии, в CDN
- Ограничить вызовы SharePoint и внешних конечных точек только необходимыми данными
- Избегайте дублирования запросов для одного и того же содержимого

Многие рекомендации в этой статье посвящены минимизации и оптимизации вызовов SharePoint Online. Выполнение повторяющихся вызовов при каждой загрузке страницы влияет на производительность пользователей, так как информация будет извлекаться из службы каждый раз, даже если она не изменилась. Таким образом, запросы к SharePoint можно классифицируются как вызовы, которые являются общими для всех пользователей, или вызовы, необходимые для каждого отдельного пользователя. Результаты из этих двух категорий вызовов следует кэшируйте для оптимизации работы пользователей.

>[!NOTE]
>Используйте средство диагностики страниц [для SharePoint](https://aka.ms/perftool) в качестве отправной точки для анализа определенных метрик производительности на страницах сайта SharePoint Online.

## <a name="modern-portal-site-limits-and-recommendations"></a>Ограничения и рекомендации для современных сайтов портала

|**Limit**|**Максимальное рекомендуемое значение**|**Примечания**|
|:-----|:-----|:-----|:-----|
|Страницы и новостные элементы  <br/> |5000 для каждого сайта  <br/> |Рекомендуется ограничить число страниц и новостных элементов на современном сайте портала до 5000.  <br/> |
|Веб-части на странице  <br/> |20 на страницу  <br/> |Рекомендуется использовать не более 20 веб-частей на страницу, включая как веб-части Майкрософт, так и пользовательские веб-части. <br/> Дополнительные сведения см. в подсети "Оптимизация производительности веб-части на современных [страницах сайтов SharePoint Online".](modern-web-part-optimization.md)  <br/> |
|Динамические веб-части на странице  <br/> |4 на страницу  <br/> |Число динамических веб-частей, которые делают один или несколько запросов к SharePoint для получения последних данных, должно быть ограничено 4 на страницу. _Веб-часть_ "Новости" является примером динамической веб-части. <br/> Дополнительные сведения см. в подсети "Оптимизация производительности веб-части на современных [страницах сайтов SharePoint Online".](modern-web-part-optimization.md)    <br/> |
|Группы безопасности  <br/> |20 для каждого сайта  <br/> |Количество групп безопасности влияет на масштаб многих запросов на современных сайтах портала. Рекомендуется ограничить количество групп безопасности как можно меньше, не более 20 на сайт.  <br/> |
|Элементы в навигации по сайту  <br/> |100 для каждого сайта  <br/> |Мы рекомендуем добавить менее 100 элементов в навигацию сайта и использовать элементы управления навигацией, которые уже используются.  <br/> Дополнительные сведения см. в подсети "Оптимизация веса страниц" на современных [страницах сайтов SharePoint Online.](modern-page-weight-optimization.md) <br/> |
|Максимальный размер изображения  <br/> |300 КБ на изображение  <br/> |Рекомендуется ограничить размер изображений до 300 кб или меньше, а также использовать CDN для работы с изображениями, таблицами стилей и сценариями. <br/>Дополнительные сведения см. в раздел "Оптимизация изображений на современных страницах сайтов [SharePoint Online"](modern-image-optimization.md) и "Использование сети доставки [содержимого (CDN) Office 365 с SharePoint Online".](use-microsoft-365-cdn-with-spo.md)  <br/> |
|Пользователи с правами на редактирование  <br/> |200 пользователей на сайт  <br/> |Сайты портала SharePoint оптимизированы для просмотра и потребления контента. Разрешения на редактирование на портале должны быть ограничены ограниченной группой пользователей, так как разрешения на изменение загружают дополнительные элементы управления и, следовательно, будут работать медленнее для этих пользователей. Таким образом, чрезмерное количество пользователей с разрешениями на редактирование повлияет на общий процесс. <br/> |
|Сторонние iFrame  <br/> |2 на страницу  <br/> |IFrames are unpredictably slow because they load a separate external page including all associated content such as javascript, CSS and framework elements. Если необходимо использовать iFrame, ограничите их число до 2 или меньше для каждой страницы.<br/> Дополнительные сведения см. в раздел "Оптимизация iFrames на современных и классических страницах сайта [публикации SharePoint Online".](modern-iframe-optimization.md) <br/> |
|Вызовы службы UPA  <br/> |1 на пользователя в час  <br/> |Рекомендуется не делать никаких вызовов по _запросу_ в службу UPA (приложение профилей пользователей). API [Microsoft Graph](https://docs.microsoft.com/graph/call-api) и [PageContext](https://docs.microsoft.com/javascript/api/sp-page-context/pagecontext?view=sp-typescript-latest) можно использовать для запроса сведений о пользователе.  <br/> Если требуется вызов службы UPA, при необходимости сделайте один вызов, а затем кэшйте данные для повторного использования в том же сеансе. |
|Вызовы службы таксономии  <br/> |5 на пользователя в час  <br/> |Рекомендуется не делать  вызовов таксономии для каждого запроса. Если необходимы вызовы службы таксономии, кэшйте данные для повторного использования в том же сеансе. <br/> Дополнительные сведения см. в раздел "Оптимизация вызовов страниц на современных и классических страницах сайта [публикации SharePoint Online".](modern-page-call-optimization.md) <br/> |
