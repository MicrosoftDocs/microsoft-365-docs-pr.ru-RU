# <a name="nat-support-with-office-365"></a>Поддержка NAT в Office 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Ранее было предложено, чтобы максимальное число клиентов Exchange, которое необходимо использовать для подключения к Office 365, было около 2000 клиентов на сетевой порт.
  
## <a name="why-use-nat"></a>Зачем использовать NAT?

С помощью NAT тысячи пользователей в корпоративной сети могут "поделиться" несколькими общедоступными IP-адресами, которые можно маршрутить.
  
В большинстве корпоративных сетей используется частное ПРОСТРАНСТВО IP-адресов (RFC1918). Частное адресное пространство выделяется интернет-сайтом IANA и предназначено исключительно для сетей, которые не маршрутют напрямую в Глобальный интернет и из него.
  
Чтобы предоставить доступ к Интернету устройствам в закрытом IP-адресовом пространстве, организации используют такие технологии шлюза, как брандмауэры и прокси-порты, которые предоставляют службы трансляции сетевых адресов (NAT) или перевода адресов портов (PAT). Эти шлюзы делают трафик с внутренних устройств в Интернет (включая Office 365) исходя из одного или более общедоступных IP-адресов, которые можно маршрутить. Каждое исходящие подключения с внутреннего устройства преобразуется в другой исходный TCP-порт на общедоступный IP-адрес. 
  
## <a name="why-do-you-need-to-have-so-many-connections-open-to-office-365-at-the-same-time"></a>Почему необходимо одновременно открыть столько подключений к Office 365?

Outlook может открывать восемь или более подключений (в случаях, когда есть надстройки, общие календари, почтовые ящики и т. д.). Так как на устройстве NAT на базе Windows доступно не более 64 000 портов, перед исчерпанием портов число пользователей за IP-адресом может быть не более 8000. Обратите внимание, что если клиенты используют устройства, не использующие ОС Windows, для NAT, общее количество доступных портов зависит от используемого устройства NAT или программного обеспечения. В этом сценарии максимальное число портов может быть меньше 64 000. Доступность портов также зависит от других факторов, таких как Windows, ограничивающих 4000 портов для собственного использования, что сокращает общее число доступных портов до 60 000. Могут быть другие приложения, такие как Internet Explorer, которые могут подключаться одновременно, для которых требуются дополнительные порты.
  
## <a name="calculating-maximum-supported-devices-behind-a-single-public-ip-address-with-office-365"></a>Вычисление максимального поддерживаемых устройств за одним общедоступным IP-адресом в Office 365

Чтобы определить максимальное число устройств за одним общедоступным IP-адресом, необходимо отслеживать сетевой трафик, чтобы определить пиковое потребление портов на клиент. Кроме того, для использования портов следует использовать пиковый коэффициент (минимум 4). 
  
 **Используйте следующую формулу для расчета числа поддерживаемых устройств на IP-адрес:**
  
Максимальное количество поддерживаемых устройств за одним общедоступным IP-адресом = (64 000 — ограниченные порты)/(Пиковое потребление портов + пиковый коэффициент)
  
 **Например, если это так:**
  
- **Ограниченные порты:** 4000 для операционной системы

- **Пиковое потребление портов:** 6 на устройство

- **Пиковый коэффициент:** 4

Затем максимальное число поддерживаемых устройств за одним общедоступным IP-адресом = (64 000–4000)/(6 + 4) = 6000
  
С выпуском пакета размещения Office 365, включаемого в обновления за сентябрь 2011 г. для Microsoft Office Outlook 2007 или ноябрь 2011 г. для Microsoft Outlook 2010, русская версия или более поздней версии, число подключений из Outlook (Как Office Outlook 2007 с Пакет обновления 2, так и Outlook 2010) к Exchange может быть всего 2. Необходимо учитывать различные операционные системы, поведение пользователей и т. д., чтобы определить минимальное и максимальное число портов, которые потребуются для вашей сети в пиковом режиме.
  
Если вы хотите поддерживать больше устройств за одним общедоступным IP-адресом, выполните действия, описанные для оценки максимального числа поддерживаемых устройств:
  
Отслеживайте сетевой трафик, чтобы определить пиковое потребление портов для каждого клиента. Вам следует собрать эти данные:
  
- Из нескольких местоположений
    
- С нескольких устройств
    
- Несколько раз
    
С помощью предыдущей формулы можно вычислить максимальное количество пользователей на IP-адрес, которое может поддерживаться в их среде.
  
Существует несколько способов распределения клиентской нагрузки по дополнительным общедоступным IP-адресам. Доступные стратегии зависят от возможностей решения корпоративного шлюза. Проще всего сегментировать адресное пространство пользователя и статически "назначить" ряд IP-адресов каждому шлюзу. Другой альтернативой, которую предлагают многие шлюзы, является возможность использования пула IP-адресов. Преимущество пула адресов в том, что он гораздо более динамический и с меньшей вероятностью требует корректировки по мере роста пользовательской базы.
