# <a name="view-directory-synchronization-status-in-microsoft-365"></a>Просмотр состояния синхронизации каталогов в Microsoft 365

Если вы интегрировали локальное доменное обслуживание Active Directory (AD DS) с Azure Active Directory (Azure AD), синхронизируя локальное окружение с Microsoft 365, вы также можете проверить состояние синхронизации.
  
## <a name="view-directory-synchronization-status"></a>Просмотр состояния синхронизации службы каталогов

- Ворегистрируйся [в центр администрирования Microsoft 365](https://admin.microsoft.com) и выберите **состояние DirSync** на домашней странице.
- Поочередно вы можете перейти к **пользователям Active** Users, а на странице \>  **Активные** пользователи выберите **дополнительные** \> **синхронизации каталогов.** На области **синхронизации каталогов** выберите **Перейти к управлению DirSync.**

## <a name="information-on-the-manage-directory-synchronization-page"></a>Сведения на странице Синхронизация каталогов

В следующей таблице перечислены функции, которые можно получить на странице.
  
Если возникла проблема с синхронизацией каталогов, ошибки также перечислены на этой странице. Дополнительные сведения о различных ошибках, с которыми вы можете столкнуться, см. в статью Определение ошибок синхронизации каталогов [в Microsoft 365.](identify-directory-synchronization-errors.md)
  
|Item|Назначение|
|:-----|:-----|
|**Проверены домены** | Количество доменов в клиенте Microsoft 365, которые вы проверили. |
|**Не проверены домены** | Домены, добавленные, но не проверенные. |
|**Включена синхронизация каталогов** |True или False. Указывает, включена ли синхронизация каталогов. |
|**Последняя синхронизация каталогов** | Последняя синхронизация каталогов. Будет отображаться предупреждение и ссылка на средство устранения неполадок, если последняя синхронизация была более трех дней назад. |
|**Включена синхронизация паролей** | True или False. Указывает, есть ли у вас синхронизация с hash паролем между нашим локальной и клиентом Microsoft 365. |
|**Последняя синхронизация паролей** | В последний раз синхронизация с hash паролей была задана. Будет отображаться предупреждение и ссылка на средство устранения неполадок, если последняя синхронизация была более трех дней назад. |
|**Клиентская версия синхронизации каталогов** | Содержит ссылку на скачивание, если выпущена новая версия Azure AD Connect. |
|**Учетная запись службы синхронизации directory** | Отображает имя учетной записи службы синхронизации каталогов Microsoft 365. |

## <a name="monitor-synchronization-health"></a>Отслеживание работоспособности функции синхронизации

В этом разделе вы установите агент Azure AD Connect Health на все свои локальные контроллеры доменов AD DS, чтобы отслеживать инфраструктуру идентификации и службы синхронизации, предоставляемые Azure AD Connect. Сведения, полученные в результате отслеживания, доступны на портале Azure AD Connect Health. Там вы можете просматривать оповещения, результаты отслеживания производительности, аналитические данные об использовании и прочие сведения.

Ключевое решение о способе использования Azure AD Connect Health зависит от способа, которым вы используете Azure AD Connect.

- Если вы используете **управляемую проверку подлинности**, начните работу с прочтения статьи [Мониторинг синхронизации Azure AD Connect с помощью Azure AD Connect Health](/azure/active-directory/connect-health/active-directory-aadconnect-health-sync), чтобы понять и настроить Azure AD Connect Health.
- Если вы синхронизируете только имена учетных записей и групп, используя **федеративную проверку подлинности** с помощью служб федерации Active Directory (AD FS), для начала прочитайте статью [Мониторинг AD FS с помощью Azure AD Connect Health](/azure/active-directory/connect-health/active-directory-aadconnect-health-adfs), чтобы понять и настроить Azure AD Connect Health.

По завершению у вас будет:

- На каждом из ваших локальных серверов поставщиков удостоверений установлен агент Azure AD Connect Health.
- На портале Azure AD Connect Health отображается текущее состояние вашей локальной инфраструктуры и действий по синхронизации с клиентом Azure AD для вашей подписки на Microsoft 365.
