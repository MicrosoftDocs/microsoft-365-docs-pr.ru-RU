# <a name="data-move-general-faq"></a>Ответы на общие вопросы о перемещении данных

Вот ответы на общие вопросы о перемещении основных неавтратных данных клиента в новый геоцентр данных.
  
## <a name="what-customers-are-eligible-to-request-a-move"></a>Какие клиенты могут запросить перемещение?
  
Существующие коммерческие клиенты Microsoft 365, которые выбрали страну, подходящую для нового географического центра данных, смогут запросить перемещение. Эта программа существует только для клиентов с соответствующим кодом страны, присвоенным клиенту Microsoft 365, для переноса неанимных основных данных клиентов для соответствующих рабочих нагрузок в соответствующий геоцентр данных Microsoft 365. Сведения о том, как [запросить перемещение](request-your-data-move.md) данных, чтобы подтвердить право на получение данных в стране, можно найти на странице "Как запросить".   

## <a name="how-do-we-define-core-customer-data"></a>Как определить основные данные клиента?
 
Основные данные клиента — это термин, который ссылается на подмножество данных клиента, определенных в Microsoft Online Services [условиях:](https://aka.ms/ost) 
- Содержимое почтовых ящиков Exchange Online (тело электронной почты, записи календаря и содержимое вложений электронной почты)
- Контент сайта SharePoint Online и файлы, хранимые на этом сайте
- Файлы, загруженные в OneDrive для бизнеса 

## <a name="what-is-in-scope-for-teams-migration"></a>Что касается миграции Teams?

Помимо Exchange Online, SharePoint Online и OneDrive для бизнеса; Корпорация Майкрософт переведет данные Teams в локальный центр обработки данных. 
- Сообщения чата Teams, включая личные сообщения и сообщения каналов. 
- Изображения Teams, используемые в чатах. 

Файлы Teams хранятся в SharePoint Online, а файлы чата Teams — в OneDrive для бизнеса. Голосовая почта, календарь и контакты хранятся в Exchange Online. Во многих случаях Exchange Online, SharePoint Online и OneDrive для бизнеса уже используются клиентом в локальном географическом центрах обработки данных и также являются частью программы миграции Microsoft 365 для соответствующих стран пользователей.

## <a name="at-what-point-is-my-migration-complete-so-that-my-tenants-core-customer-data-is-being-stored-at-rest-in-my-new-geo"></a>На каком этапе миграция завершена, чтобы основные данные клиента хранились в неанимном месте в новом географическом месте?

Из-за общих зависимостей между Exchange Online и SharePoint Online/OneDrive для бизнеса любая миграция не может считаться завершенной, пока не будут перенесены обе службы. Exchange Online и SharePoint Online/OneDrive для бизнеса часто переносят в отдельное время и независимо друг от друга. Администраторы клиентов получают подтверждение в Центре сообщений после завершения каждой миграции службы и могут просмотреть карточку расположения данных в Центре администрирования в любое время, чтобы подтвердить основные данные клиентов в месте хранения для каждой службы.

## <a name="how-do-you-make-sure-my-customer-data-is-safe-during-the-move-and-that-i-wont-experience-downtime"></a>Как убедиться, что мои данные клиента безопасны во время перемещения и что не будет простоя?
  
Перемещение данных — это операция службы на тыловом компьютере с минимальным влиянием на конечных пользователей. Функции, которые могут повлиять, перечислены в списке ["Во время и после перемещения данных".](during-and-after-your-data-move.md) Мы соблюдаем Microsoft Online Services соглашению об уровне обслуживания [(SLA)](https://go.microsoft.com/fwlink/p/?LinkId=523897) для доступности, поэтому клиентам не нужно ничего подготавливать или отслеживать во время перемещения. 
  
Все службы Microsoft 365 работают в центрах обработки данных одинаковыми версиями, поэтому вы можете быть уверены в согласованности функций. Ваша служба полностью поддерживается на протяжении всего процесса.
  
## <a name="what-is-the-impact-of-having-different-services-located-in-different-geos"></a>Как это повлияет на то, что разные службы будут размещены в разных географических расположений?

Некоторые службы Microsoft 365 могут располагаться в разных географических расположений для некоторых существующих клиентов и для клиентов, которые находятся в середине процесса перемещения. Наши службы работают независимо друг от друга, и в этом случае они не влияют на пользовательский интерфейс. Однако в целях хранения данных миграцию клиента нельзя считать завершенной до тех пор, пока Exchange Online и SharePoint Online/OneDrive для бизнеса не будут перенесены в один и тот же центр обработки данных.

 ## <a name="where-is-my-core-customer-data-located"></a>Где находятся основные данные клиента?

Администраторы клиентов могут в любое время просмотреть карточку расположения данных в Центре администрирования, чтобы подтвердить основные данные клиентов для каждой службы, в частности для своего клиента.  Мы также публикуем расположение географических расположений центров обработки данных, центров обработки данных и расположение данных клиентов Office 365 на картах интерактивного центра обработки данных [Microsoft 365 ](https://office.com/datamaps) в качестве ссылки на текущие основные данные клиентов по умолчанию для новых клиентов. Вы можете проверить расположение неавтных данных клиента в разделе "Расположение данных" в профиле организации в Центре администрирования Microsoft 365.  
 
## <a name="when-will-i-be-able-to-request-a-move"></a>Когда я могу запросить перемещение?
  
Сведения о том, как [запросить](request-your-data-move.md) страницу перемещения данных для поддерживаемых временных периодов для вашего географического центра обработки данных, можно найти на странице "Как запросить".
  
## <a name="how-can-i-request-to-be-moved"></a>Как можно запросить перемещение?
  
Подходящие клиенты увидят страницу в Центре администрирования [Microsoft 365.](https://admin.microsoft.com/) См. [инструкции по](request-your-data-move.md) запросу перемещения данных. 
  
## <a name="can-i-change-my-selection-after-requesting-a-move"></a>Можно ли изменить выбор после запроса на перемещение?
  
Мы не будем удалять вас из процесса после отправки запроса.
  
## <a name="what-happens-if-i-do-not-request-a-move-before-the-deadline"></a>Что произойдет, если я не запросю перемещение до крайнего срока?
  
Мы не можем принимать запросы на миграцию после открытого периода регистрации.

## <a name="what-if-i-want-to-move-my-data-in-order-to-get-better-network-performance"></a>Что делать, если я хочу переместить данные, чтобы улучшить производительность сети?
  
Физическая близость к центру обработки данных Microsoft 365 не гарантирует лучшую производительность сети. Существует множество факторов и компонентов, влияющих на производительность сети между конечным пользователем и службой Microsoft 365. Дополнительные сведения об этой настройке и настройке производительности см. в подстройке сети [для Microsoft 365.](network-planning-and-performance.md)
  
 ## <a name="do-all-the-services-move-their-data-on-the-same-day"></a>Все ли службы перемещают свои данные в один день?
 
Каждая служба перемещается независимо и, скорее всего, будет перемещать свои данные в разное время.
  
 ## <a name="can-i-choose-when-i-want-my-data-to-be-moved"></a>Можно ли выбрать время, когда данные будут перемещены?
 
Клиенты не могут выбрать определенную дату, они не могут отложить перемещение, и мы не можем делиться определенной датой или временными рамками для перемещения.
  
 ## <a name="can-you-share-when-my-data-will-be-moved"></a>Можно ли делиться данными при перемещении?
  
Перемещение данных — это тыловая операция с минимальным влиянием на конечных пользователей. Сложность, точность и масштаб, при которых необходимо выполнять перемещение данных в глобальной и автоматизированной среде, не позволяет нам обмениваться данными, когда ожидается завершение перемещения данных для вашего клиента или любого другого отдельного клиента. После завершения перемещения данных клиенты получат одно подтверждение в Центре сообщений для службы- 
  
 ## <a name="what-happens-if-users-access-services-while-the-data-is-being-moved"></a>Что произойдет, если пользователи будут получать доступ к службам при перемещении данных?

Полный [список функций,](during-and-after-your-data-move.md) которые могут быть ограничены во время и после перемещения данных, см. в разных фрагментах перемещения данных для каждой службы. 
  
 ## <a name="how-do-i-know-the-move-is-complete"></a>Как узнать, что перемещение завершено?
  
Просмотрите Центр сообщений Microsoft 365, чтобы подтвердить, что перемещение данных каждой службы завершено. При перемещении данных каждой службы мы опредим уведомление о завершении, чтобы вы получите три уведомления о завершении: по одному для Exchange Online, SharePoint Online и Skype для бизнеса Online. Вы также можете проверить расположение неавтных данных клиента в разделе "Расположение данных" профиля организации в Центре администрирования Microsoft 365.  
  
## <a name="i-am-a-microsoft-365-customer-in-one-of-the-new-datacenter-geos-but-when-i-signed-up-i-selected-a-different-country-how-can-i-be-moved-to-the-new-datacenter-geo"></a>Я являюсь клиентом Microsoft 365 в одном из новых геолокационных центра обработки данных, но когда я подписывался, я выбрал другую страну. Как можно перемещении в новый центр обработки данных в географическом окте?

Изменить страну регистрации, связанную с клиентом, невозможно. Вместо этого необходимо создать новый клиент Microsoft 365 с новой подпиской и вручную переместить пользователей и данные в новый клиент.
  
## <a name="what-happens-if-we-are-in-process-of-email-data-migration-to-microsoft-365-during-the-exchange-online-move"></a>Что произойдет при переносе данных электронной почты в Microsoft 365 во время перемещения Exchange Online?

Это очень распространенный сценарий, который полностью поддерживается. Облачная миграция между геообъекторами центра обработки данных не препятствует миграции из локальной организации в облачные почтовые ящики.
  
 ## <a name="can-i-pilot-some-users"></a>Можно ли пилотное пилотное несколько пользователей?
  
Вы можете создать отдельный клиент для проверки возможности подключения, но его нельзя комбинировать с существующим клиентом.

## <a name="i-dont-want-to-wait-for-microsoft-to-move-my-data-can-i-just-create-a-new-tenant-and-move-myself"></a>Я не хочу ждать, пока Корпорация Майкрософт переместит мои данные. Можно ли просто создать новый клиент и переместить себя?
  
Да, однако процесс будет не таким плавным, как если бы корпорация Майкрософт выполняла перемещение данных.
  
Если новый клиент создается после того, как новый центр обработки данных будет доступен, он будет размещен в новом географическом. Этот новый клиент полностью отделен от предыдущего клиента, и вы несете ответственность за перемещение всех почтовых ящиков пользователей, контента сайта, доменных имен и любых других данных. Обратите внимание, что нельзя переместить имя клиента из одного клиента в другой. Мы рекомендуем дождаться программы перемещения, предоставляемой корпорацией Майкрософт, так как мы будем принимать решение о перемещении всех параметров, данных и подписок для ваших пользователей.
  
## <a name="my-customer-data-has-already-been-moved-to-a-new-datacenter-geo-can-i-move-back"></a>Данные моих клиентов уже перемещены в новый геоцентр обработки данных. Можно ли вернуться назад?
 
Нет, это невозможно. Клиентов, которые были перемещены в новые географические центра обработки данных, нельзя вернуть обратно. Как клиент в любом географическом сфере вы будете испытывать такое же качество обслуживания, производительности и элементов управления безопасностью, как и раньше. [Microsoft 365 с](https://aka.ms/multi-geo) несколькими географическими решениями доступен некоторым клиентам в качестве надстройки и позволяет одному клиенту создавать несколько дополнительных географических вариантов и перемещать пользовательские данные в эти географические точки с обязательствами по размещению данных.
  
## <a name="will-microsoft-365-tenants-hosted-in-the-new-datacenters-be-available-to-users-outside-of-the-country"></a>Будут ли клиенты Microsoft 365, которые размещены в новых центрах обработки данных, доступными пользователям за пределами страны?
  
Да. Корпорация Майкрософт поддерживает крупную глобальную сеть с общедоступными подключениями к Интернету в более чем 130 расположениях в 35 странах по всему миру с помощью пиринговых соглашений с более чем 2700 поставщиками услуг Интернета (ISP). Пользователи смогут получать доступ к центрам обработки данных из интернета.

## <a name="my-tenant-has-configured-the-multi-geo-add-on-can-i-still-enroll-in-my-tenant-in-the-microsoft-365-move-program-to-change-my-default-geo-and-move-any-user-not-in-a-satellite-region-to-the-new-default-geo"></a>Мой клиент настроил [надстройку с несколькими географическими настройками.](https://aka.ms/multi-geo) Могу ли я по-прежнему зарегистрироваться в моем клиенте в программе перемещения Microsoft 365, чтобы изменить свой географический регион по умолчанию и переместить любого пользователя, не в регионе-регионе по умолчанию?

Да, ваш клиент может зарегистрироваться, но существуют важные вопросы, так как перемещение на уровне клиента не полностью поддерживается для клиентов, которые настроили поддержку нескольких географических конфигураций.

SharePoint Online и OneDrive для бизнеса не могут перейти в новый центр обработки данных на уровне клиента с помощью этой программы. Администратор клиента может настроить перемещение ресурсов OneDrive для бизнеса в любой доступный регион с помощью поддержки нескольких регионов, но расположение по умолчанию для клиента нельзя изменить после настройки поддержки нескольких регионов для клиента.

Для клиентов, которые решили отказаться от миграции, мы переместим все почтовые ящики Exchange Online из текущего географического региона по умолчанию в новый локальный центр обработки данных и обновим регион Exchange Online по умолчанию. Мы не будем перемещать почтовые ящики EXO, настроенные в регионах с несколькими регионами-получателями, чтобы они по-прежнему соблюдали замество данных в регионах, которые вы планировали.
