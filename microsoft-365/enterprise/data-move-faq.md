# <a name="data-move-general-faq"></a>Ответы на общие вопросы о перемещении данных

Вот ответы на общие вопросы о перемещении основных данных клиентов в покое в новый геоцентр обработки данных.
  
## <a name="what-customers-are-eligible-to-request-a-move"></a>Какие клиенты могут запрашивать перемещение?
  
Существующие коммерческие клиенты Microsoft 365, выбраввшие страну, подходящую для нового геоцентра обработки данных, смогут запросить перемещение. Программа существует только для клиентов с соответствующим кодом страны, назначенного клиенту Microsoft 365 для переноса основных данных клиентов в покое для соответствующих рабочих нагрузок в соответствующий геоцентр данных Microsoft 365. Обратитесь к странице [How to request your data move](request-your-data-move.md) page to confirm country eligibility.   

## <a name="how-do-we-define-core-customer-data"></a>Как определить основные данные клиентов?
 
Основные данные клиента — это термин, который относится к подмножество данных клиентов, определенных в [Microsoft Online Services Термины:](https://aka.ms/ost) 
- Содержимое почтовых ящиков Exchange Online (тело электронной почты, записи календаря и содержимое вложений электронной почты)
- Содержимое сайта SharePoint Online и файлы, хранимые на этом сайте
- Файлы, загруженные в OneDrive для бизнеса 

## <a name="what-is-in-scope-for-teams-migration"></a>Что имеется в области миграции Teams?

Помимо Exchange Online, SharePoint Online и OneDrive для бизнеса; Корпорация Майкрософт будет мигрировать данные Teams в локальный центр обработки данных. 
- Группы сообщений чата, в том числе частных сообщений и сообщений канала. 
- Изображения teams, используемые в чатах. 

Файлы teams хранятся в SharePoint Online, а файлы чата Teams хранятся в OneDrive для бизнеса. Голосовая почта, календарь и контакты хранятся в Exchange Online. Во многих случаях Exchange Online, SharePoint Online и OneDrive для бизнеса уже используются клиентом в локальном геоцентре обработки данных, а также являются частью программы миграции Microsoft 365 для соответствующих стран-клиентов.

## <a name="at-what-point-is-my-migration-complete-so-that-my-tenants-core-customer-data-is-being-stored-at-rest-in-my-new-geo"></a>На каком этапе моя миграция завершена, чтобы основные данные клиента хранятся в покое в новом гео?

Из-за общих зависимостей между Exchange Online и SharePoint Online/OneDrive для бизнеса любая миграция не может считаться завершенной до переноса обеих служб. Exchange Online и SharePoint Online/OneDrive для бизнеса часто переносят в отдельное время и независимо друг от друга. Администраторы клиентского клиента получают подтверждение в Центре сообщений по завершению каждой миграции служб и могут просматривать карточку расположения данных в Центре администрирования в любое время, чтобы подтвердить основные данные клиентов в расположении отдыха для каждой службы.

## <a name="how-do-you-make-sure-my-customer-data-is-safe-during-the-move-and-that-i-wont-experience-downtime"></a>Как убедиться, что мои данные клиентов безопасны во время перемещения и что я не буду испытывать простои?
  
Перемещение данных — это операция службы с минимальным влиянием на конечных пользователей. Функции, на которые можно повлиять, перечислены во [время и после перемещения данных.](during-and-after-your-data-move.md) Мы придерживаемся соглашения Microsoft Online Services уровня обслуживания [(SLA)](https://go.microsoft.com/fwlink/p/?LinkId=523897) для доступности, поэтому нет ничего, что клиенты должны подготовиться или контролировать во время перемещения. 
  
Все службы Microsoft 365 запускают одинаковые версии в центрах обработки данных, поэтому вы можете быть уверены в согласованной функциональности. Ваша служба полностью поддерживается на протяжении всего процесса.
  
## <a name="what-is-the-impact-of-having-different-services-located-in-different-geos"></a>Какое влияние оказывает наличие различных служб, расположенных в разных географических географических службах?

Некоторые службы Microsoft 365 могут располагаться в разных географических странах для некоторых существующих клиентов и для клиентов, которые находятся в центре процесса перемещения. Наши службы работают независимо друг от друга и не влияют на пользовательский опыт, если это так. Однако для целей оседлости данных миграция клиента не может считаться полной до тех пор, пока в один и тот же геоцентр данных не будут перенесены как Exchange Online, так и SharePoint Online/OneDrive для бизнеса.

 ## <a name="where-is-my-core-customer-data-located"></a>Где находятся основные данные клиентов?

Администраторы клиентского клиента могут просматривать карточку расположения данных в Центре администрирования в любое время, чтобы подтвердить основные данные клиента в расположении отдыха для каждой службы, в частности для клиента.  Мы также публикуем расположение геоцентров, центров обработки данных и расположение данных клиентов Office 365 на интерактивных картах центра обработки данных [Microsoft 365 ](https://office.com/datamaps) в качестве ссылки для текущих базовых данных клиентов по умолчанию в местах отдыха для новых клиентов. Вы можете проверить расположение данных клиентов в покое с помощью раздела Расположение данных в профиле организации в центре администрирования Microsoft 365.  
 
## <a name="when-will-i-be-able-to-request-a-move"></a>Когда я смогу запросить перемещение?
  
Обратитесь к странице [How to request your data move](request-your-data-move.md) page for supported timeframes for your datacenter geo.
  
## <a name="how-can-i-request-to-be-moved"></a>Как можно попросить о перемещении?
  
Подходящие клиенты увидят страницу в центре администрирования [Microsoft 365.](https://admin.microsoft.com/) См. [инструкции по](request-your-data-move.md) запросу перемещения данных. 
  
## <a name="can-i-change-my-selection-after-requesting-a-move"></a>Могу ли я изменить выбор после запроса перемещения?
  
После отправки запроса вы не сможете удалить вас из процесса.
  
## <a name="what-happens-if-i-do-not-request-a-move-before-the-deadline"></a>Что произойдет, если я не запросят переместиться до крайнего срока?
  
Мы не можем принимать запросы на миграцию после периода открытой регистрации.

## <a name="what-if-i-want-to-move-my-data-in-order-to-get-better-network-performance"></a>Что делать, если я хочу переместить данные, чтобы улучшить производительность сети?
  
Физическая близость к центру обработки данных Microsoft 365 не гарантирует лучшую производительность сети. Существует множество факторов и компонентов, влияющих на производительность сети между конечным пользователем и службой Microsoft 365. Дополнительные сведения об этом и настройке производительности см. в [веб-сайте Network planning and performance tuning for Microsoft 365.](network-planning-and-performance.md)
  
 ## <a name="do-all-the-services-move-their-data-on-the-same-day"></a>Перемещают ли все службы данные в один и тот же день?
 
Каждая служба перемещается самостоятельно и, скорее всего, будет перемещать свои данные в разное время.
  
 ## <a name="can-i-choose-when-i-want-my-data-to-be-moved"></a>Могу ли я выбрать, когда нужно, чтобы мои данные перемещались?
 
Клиенты не могут выбрать конкретную дату, они не могут отложить их перемещение, и мы не можем поделиться определенной датой или временными рамками для ходов.
  
 ## <a name="can-you-share-when-my-data-will-be-moved"></a>Можете ли вы поделиться, когда мои данные будут перемещены?
  
Перемещение данных — это операция с минимальным влиянием на конечных пользователей. Сложность, точность и масштаб, с помощью которых мы должны выполнять перемещения данных в глобальной и автоматизированной среде, не позволяет нам обмениваться данными, когда ожидается завершение перемещения данных для клиента или любого другого отдельного клиента. По завершению перемещения данных клиенты получат одно подтверждение в Центре сообщений на одну службу-участницу. 
  
 ## <a name="what-happens-if-users-access-services-while-the-data-is-being-moved"></a>Что произойдет, если пользователи будут получать доступ к службам во время перемещений данных?

Сведения [о полном списке](during-and-after-your-data-move.md) функций, которые могут быть ограничены во время перемещения данных для каждой службы, см. во время и после перемещения данных. 
  
 ## <a name="how-do-i-know-the-move-is-complete"></a>Как узнать, что перемещение завершено?
  
Просмотрите Центр сообщений Microsoft 365, чтобы подтвердить, что перемещение данных каждой службы завершено. При перемещении данных каждой службы мы вывешим уведомление о завершении, чтобы вы могли получить три уведомления о завершении: по одному для Exchange Online, SharePoint Online и Skype для бизнеса Online. Вы также можете проверить расположение данных клиентов в покое с помощью раздела Расположение данных в профиле организации в центре администрирования Microsoft 365.  
  
## <a name="i-am-a-microsoft-365-customer-in-one-of-the-new-datacenter-geos-but-when-i-signed-up-i-selected-a-different-country-how-can-i-be-moved-to-the-new-datacenter-geo"></a>Я являюсь клиентом Microsoft 365 в одном из новых геоцентров центра обработки данных, но когда я подписался, я выбрал другую страну. Как меня можно переместить в новый геоцентр обработки данных?

Невозможно изменить страну регистрации, связанную с клиентом. Вместо этого необходимо создать новый клиент Microsoft 365 с новой подпиской и вручную переместить пользователей и данные в новый клиент.
  
## <a name="what-happens-if-we-are-in-process-of-email-data-migration-to-microsoft-365-during-the-exchange-online-move"></a>Что произойдет, если мы в процессе миграции данных электронной почты в Microsoft 365 во время перемещения Exchange Online?

Это очень распространенный сценарий, который полностью поддерживается.  Облачная миграция между геоцентрами обработки данных не мешает локальному переносу облачных почтовых ящиков.
  
 ## <a name="can-i-pilot-some-users"></a>Могу ли я пилотных некоторых пользователей?
  
Для проверки подключения можно создать отдельный клиент пробной пробной системы, но его нельзя комбинировать с существующим клиентом.

## <a name="i-dont-want-to-wait-for-microsoft-to-move-my-data-can-i-just-create-a-new-tenant-and-move-myself"></a>Я не хочу ждать, пока Корпорация Майкрософт будет перемещать мои данные. Могу ли я просто создать нового клиента и переместить себя?
  
Да, однако процесс будет не таким бесшовным, как если бы Microsoft выполняла перемещение данных.
  
При создании нового клиента после того, как будет доступен новый геоцентр обработки данных, новый клиент будет размещен в новом geo. Этот новый клиент полностью отделен от предыдущего клиента, и вы будете отвечать за перемещение всех почтовых ящиков пользователей, контента сайта, доменных имен и любых других данных. Обратите внимание, что нельзя переместить имя клиента из одного клиента в другой. Рекомендуется дождаться программы перемещения, предоставленной Корпорацией Майкрософт, так как мы позаботимся о перемещении всех параметров, данных и подписок для пользователей.
  
## <a name="my-customer-data-has-already-been-moved-to-a-new-datacenter-geo-can-i-move-back"></a>Мои данные о клиентах уже перемещены в новый геоцентр обработки данных. Могу ли я вернуться?
 
Нет, это невозможно. Клиенты, переведенные в новые геоцентры обработки данных, не могут быть перемещены обратно. Как клиент в любом гео, вы будете испытывать такое же качество управления службой, производительностью и безопасностью, как и раньше. [Microsoft 365 Multi Geo](https://aka.ms/multi-geo) доступна некоторым клиентам в качестве надстройки и позволяет одному клиенту создавать несколько спутниковых геос и перемещать данные пользователей в эти геос с обязательствами по размещению данных.
  
## <a name="will-microsoft-365-tenants-hosted-in-the-new-datacenters-be-available-to-users-outside-of-the-country"></a>Будут ли клиенты Microsoft 365, размещенные в новых центрах обработки данных, доступными пользователям за пределами страны?
  
Да. Корпорация Майкрософт поддерживает крупную глобальную сеть с общедоступными подключениями к Интернету в более чем 130 точках в 35 странах по всему миру с соглашениями о пиринге с более чем 2700 поставщиками интернет-услуг (ПОСТАВЩИКОВ). Пользователи смогут получать доступ к центрам обработки данных, где бы они ни были в Интернете.

## <a name="my-tenant-has-configured-the-multi-geo-add-on-can-i-still-enroll-in-my-tenant-in-the-microsoft-365-move-program-to-change-my-default-geo-and-move-any-user-not-in-a-satellite-region-to-the-new-default-geo"></a>Мой клиент настроил [надстройку Multi Geo.](https://aka.ms/multi-geo) Могу ли я по-прежнему зарегистрироваться в своем клиенте в программе Перемещения Microsoft 365, чтобы изменить гео по умолчанию и переместить любого пользователя, не в спутниковом регионе, на новый гео по умолчанию?

Да, клиент имеет право на регистрацию, но существуют существенные факторы, так как перемещение на уровне клиента не поддерживается полностью для клиентов, настроенных на Multi-Geo.

SharePoint Online и OneDrive для бизнеса не могут перейти на новый геоцентр обработки данных на уровне клиента с помощью этой программы. Администратор клиента может настроить акции OneDrive для бизнеса для перемещения в любой доступный регион с помощью Multi-Geo, но расположение клиента по умолчанию не может быть изменено после настройки Multi-Geo для клиента.

Для клиентов, которые выбирают миграцию, мы переместим все почтовые ящики Exchange Online из текущего гео по умолчанию в новый локальный центр обработки данных и обновим регион Exchange Online по умолчанию. Мы не будем перемещать любые почтовые ящики EXO, настроенные в регионах с несколькими спутниками Geo, чтобы продолжать уважать место жительства данных в регионе спутниковой связи, как вы планировали.
