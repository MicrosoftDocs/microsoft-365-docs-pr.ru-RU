# <a name="introduction-to-performance-tuning-for-sharepoint-online"></a>Общие сведения о настройке производительности для SharePoint Online

В этой статье объясняется, какие конкретные аспекты необходимо учитывать при разработке страниц для лучшей производительности в SharePoint Online.
     
## <a name="sharepoint-online-metrics"></a>Метрик SharePoint Online

Следующие широкие показатели для SharePoint Online предоставляют реальные данные о производительности:
  
- Скорость загрузки страниц
    
- Сколько разных поездок требуется на страницу
    
- Проблемы со службой
    
- Другие вещи, которые вызывают ухудшение производительности
    
### <a name="conclusions-reached-because-of-the-data"></a>Выводы, сделанные из-за данных

Данные говорят нам:
  
- Большинство страниц хорошо выполняются в SharePoint Online.
    
- Не настраиваемые страницы загружаются очень быстро.
    
- OneDrive для бизнеса, сайты команд и страницы системы, такие как _layouts и т.д., быстро загружаются.
    
- Для загрузки самых медленных 1% страниц SharePoint Online требуется более 5000 миллисекунд.
    
Один простой тест, который можно использовать для измерения производительности, сравнивая время загрузки собственного портала с временем загрузки домашней страницы OneDrive для бизнеса, так как она использует несколько настраиваемых функций. Это часто будет первым шагом, который поддержка попросит вас завершить при устранении неполадок с производительностью сети.
  
## <a name="use-a-standard-user-account-when-checking-performance"></a>Использование стандартной учетной записи пользователя при проверке производительности

Администратор коллекции сайтов, владелец сайта, редактор или участник относятся к дополнительным группам безопасности, имеют дополнительные разрешения и, следовательно, имеют дополнительные элементы, загружающие SharePoint на страницу.
  
Это применимо к локальной sharePoint и SharePoint Online, но в локальном сценарии различия будут не так легко заметить, как в SharePoint Online.
  
Чтобы правильно оценить, как будет работать страница для пользователей, следует использовать стандартную учетную запись пользователя, чтобы избежать загрузки элементов управления авторизации и дополнительного трафика, связанного с группами безопасности.
  
## <a name="connection-categories-for-performance-tuning"></a>Категории подключения для настройки производительности

Связи между сервером и пользователем можно классифицировать на три основных компонента. Рассмотрите их при разработке страниц SharePoint Online, чтобы получить представление о времени нагрузки.
  
- **Сервер** Серверы, на которые Майкрософт размещены в центрах обработки данных.
    
- **Сеть** Сеть Майкрософт, Интернет и локальное подключение между центром обработки данных и пользователями.
    
- **Браузер** Загрузка страницы.
    
В этих трех подключениях обычно существует пять причин, которые вызывают 95% медленных страниц. Каждая из этих причин обсуждается в этой статье:
  
- Проблемы с навигацией
    
- Свернутый контент
    
- Большие файлы
    
- Много запросов на сервер
    
- Обработка веб-части
    
### <a name="server-connection"></a>Подключение к серверу

Многие проблемы, влияющие на производительность в локальной сети SharePoint, также относятся к SharePoint Online.
  
Как вы и ожидали, у вас есть гораздо больше контроля над тем, как серверы выполняются с локальной SharePoint. С SharePoint Online все немного по-другому. Чем больше работы вы сделаете на сервере, тем дольше будет отрисовка страницы. В SharePoint самыми крупными виновниками в этом отношении являются сложные страницы с несколькими веб-частями.
  
Локальное помещение SharePoint Server

SharePoint Online
  
![Снимок экрана: сервер в сети](../media/46b27ded-d8a4-4287-b3e0-2603a764b8f8.png)
  
В SharePoint Online некоторые запросы на страницы могут вызывать несколько серверов. Вы можете получить матрицу запросов между серверами для отдельного запроса. Эти взаимодействия являются дорогостоящими с точки зрения нагрузки на страницу и замедляют процессы.
  
Примеры взаимодействия этих серверов с сервером:
  
- Веб-SQL серверов
    
- Веб-серверы приложений
    
Другая вещь, которая может замедлить взаимодействие серверов, это пропуска кэш. В отличие от локальной SharePoint, существует очень малая вероятность того, что вы нажмитесь на тот же сервер для страницы, которую вы посещали ранее; это делает кэшинг объектов устаревшим.
  
### <a name="network-connection"></a>Подключение к сети 

С локальной sharePoint, которая не использует WAN, вы можете использовать высокоскоростное подключение между центрами обработки данных и конечными пользователями. Как правило, с сетевой точки зрения все легко управлять.
  
В SharePoint Online необходимо учитывать еще несколько факторов. Например:
  
- Сеть Майкрософт
    
- Интернет
    
- The ISP
    
Независимо от того, какую версию SharePoint (и какую сеть) вы используете, к числу вещей, которые обычно приводят к загружению сети, относятся следующие:
  
- Большая полезной нагрузки
    
- Много файлов
    
- Большое физическое расстояние до сервера
    
Одной из функций, которую можно использовать в SharePoint Online, является Microsoft CDN (Сеть доставки контента). CDN — это в основном распределенная коллекция серверов, развернутых в нескольких центрах обработки данных. С помощью CDN содержимое на страницах может быть организовано на сервере, близком к клиенту, даже если клиент находится далеко от исходящего SharePoint Server. В будущем корпорация Майкрософт будет использовать эти дополнительные данные для хранения локальных экземпляров страниц, которые невозможно настроить, например домашней страницы администратора SharePoint Online. Дополнительные сведения о CDNs см. в [материалах сетей доставки контента.](content-delivery-networks.md)
  
Что-то, что вам нужно знать, но не может быть в состоянии сделать много о является скорость подключения вашего isP. Простой тестовый инструмент скорости скажет вам скорость подключения.
  
### <a name="browser-connection"></a>Подключение к браузеру

Существует несколько факторов, которые следует учитывать с помощью веб-браузеров с точки зрения производительности.
  
Посещение сложных страниц повлияет на производительность. Большинство браузеров имеют небольшой кэш (около 90 МБ), в то время как средняя веб-страница обычно составляет около 1,6 МБ. Это не займет много времени, чтобы привыкнуть.
  
Пропускная способность также может быть проблемой. Например, если пользователь смотрит видео на другом сеансе, это повлияет на производительность страницы SharePoint. Хотя вы не можете запретить пользователям потоковое мультимедиа, вы можете управлять загрузкой страницы для пользователей.
