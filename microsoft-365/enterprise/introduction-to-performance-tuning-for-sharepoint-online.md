# <a name="introduction-to-performance-tuning-for-sharepoint-online"></a>Общие сведения о настройке производительности для SharePoint Online

В этой статье объясняется, какие конкретные аспекты необходимо учитывать при разработке страниц для лучшей производительности в SharePoint Online.
     
## <a name="sharepoint-online-metrics"></a>Показатели SharePoint Online

Следующие показатели для SharePoint Online предоставляют реальные данные о производительности:
  
- Скорость загрузки страниц
    
- Количество необходимых кругов на страницу
    
- Проблемы со службой
    
- Другие причины ухудшения производительности
    
### <a name="conclusions-reached-because-of-the-data"></a>Выводы, полученные из-за данных

Данные подсказывает нам:
  
- Большинство страниц хорошо работает в SharePoint Online.
    
- Ненастройные страницы загружаются очень быстро.
    
- OneDrive для бизнеса, сайты группы и системные страницы, такие как _layouts и т. д., быстро загружаются.
    
- Загрузка самых медленных 1% страниц SharePoint Online составляет более 5000 миллисекунд.
    
Одним из простых тестов производительности может быть измерение производительности путем сравнения времени загрузки собственного портала с временем загрузки домашней страницы OneDrive для бизнеса, так как в ней используется мало пользовательских функций. Зачастую это первый шаг, который будет запросить у вас при устранении неполадок с производительностью сети.
  
## <a name="use-a-standard-user-account-when-checking-performance"></a>Использование стандартной учетной записи пользователя при проверке производительности

Администратор, владелец сайта, редактор или участник относятся к дополнительным группам безопасности, имеют дополнительные разрешения и, следовательно, имеют дополнительные элементы, которые SharePoint загружает на страницу.
  
Это применимо к локальной sharePoint и SharePoint Online, но в локальном сценарии различия будут не так легко заметить, как в SharePoint Online.
  
Чтобы правильно оценить, как страница будет работать для пользователей, следует использовать стандартную учетную запись пользователя, чтобы избежать загрузки элементов управления для авторизации и дополнительного трафика, связанного с группами безопасности.
  
## <a name="connection-categories-for-performance-tuning"></a>Категории подключений для настройки производительности

Связи между сервером и пользователем можно классифицировать на три основных компонента. Учитывайте эти моменты при разработке страниц SharePoint Online, чтобы получить представление о времени загрузки.
  
- **Сервер** Серверы, которые корпорация Майкрософт размещена в центрах обработки данных.
    
- **Сеть** Сеть Майкрософт, Интернет и ваша локальной сеть между центром обработки данных и вашими пользователями.
    
- **Браузер** Место загрузки страницы.
    
В этих трех подключениях обычно есть пять причин, которые вызывают 95 % медленных страниц. Каждая из этих причин обсуждается в этой статье:
  
- Проблемы навигации
    
- Свернутый контент
    
- Большие файлы
    
- Большое количество запросов к серверу
    
- Обработка веб-части
    
### <a name="server-connection"></a>Подключение к серверу

Многие проблемы, влияющие на производительность локальной точки SharePoint, также применимы к SharePoint Online.
  
Как и ожидаось, вы имеете гораздо больше контроля над тем, как серверы выполняются с локальной sharePoint. В SharePoint Online все немного по-другому. Чем больше работы вы делаете на сервере, тем дольше требуется отрисовка страницы. В SharePoint самыми главными злоумышленниками в этом отношении являются сложные страницы с несколькими веб-частями.
  
Локальное sharePoint Server
  
![Снимок экрана: локальный сервер](../media/a8e9b646-cdff-4131-976a-b5f891da44ac.png)
  
SharePoint Online
  
![Снимок экрана: сервер в сети](../media/46b27ded-d8a4-4287-b3e0-2603a764b8f8.png)
  
В SharePoint Online некоторые запросы страниц могут вызывать несколько серверов. Вы можете получить матрицу запросов между серверами для отдельного запроса. Эти взаимодействия являются дорогостоящими с точки зрения загрузки страницы и замедляют процесс.
  
Примеры взаимодействия между сервером и сервером:
  
- Веб-SQL серверов
    
- Веб-серверы приложений
    
Еще одно, что может замедлить взаимодействие с сервером, — промахи кэша. В отличие от локальной точки SharePoint существует очень большой вероятность того, что вы будете работать на том же сервере для страницы, которую вы уже посещали; это делает кэш объектов устаревшим.
  
### <a name="network-connection"></a>Подключение к сети 

В локальной sharePoint, которая не использует WAN, можно использовать высокоскоростное подключение между центрами обработки данных и конечными пользователями. Как правило, с точки зрения сети управление с точки зрения сети является простым.
  
В SharePoint Online следует учитывать еще несколько факторов. Например:
  
- Сеть Майкрософт
    
- Интернет
    
- The ISP
    
Независимо от используемой версии SharePoint (и сети) в сети обычно используются следующие точки:
  
- Большие полезной нагрузки
    
- Много файлов
    
- Большое физическое расстояние до сервера
    
Одной из функций, которую можно использовать в SharePoint Online, является сеть доставки содержимого (Майкрософт). CdN — это, по сути, распределенная коллекция серверов, развернутых в нескольких центрах обработки данных. С помощью сети CDN контент на страницах может быть находится на сервере рядом с клиентом, даже если клиент находится далеко от сервера SharePoint Server. В будущем корпорация Майкрософт будет использовать эту функцию для хранения локальных экземпляров страниц, которые невозможно настроить, например домашней страницы администратора SharePoint Online. Дополнительные сведения о сетях ДОСТАВКИ содержимого см. в сетях [доставки содержимого.](content-delivery-networks.md)
  
Что-то, о чем вам необходимо знать, но, возможно, не удастся многое сделать, — это скорость подключения вашего isP. Простое средство тестирования скорости покажет вам скорость подключения.
  
### <a name="browser-connection"></a>Подключение к браузеру

Существует несколько факторов, которые следует учитывать при работе с веб-браузерами с точки зрения производительности.
  
Просмотр сложных страниц влияет на производительность. В большинстве браузеров имеется только небольшой кэш (около 90 МБ), а средняя веб-страница обычно составляет около 1,6 МБ. Это не займет много времени.
  
Кроме того, может возникнуть проблема с пропускной способностью. Например, если пользователь смотрит видео в другом сеансе, это повлияет на производительность страницы SharePoint. Хотя вы не можете запретить пользователям потоковую передачу мультимедиа, вы можете управлять тем, как страница будет загружаться для пользователей.
