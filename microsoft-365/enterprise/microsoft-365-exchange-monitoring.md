# <a name="exchange-online-monitoring-for-microsoft-365"></a>Мониторинг Exchange Online для Microsoft 365

С помощью мониторинга Exchange Online в Центре администрирования Microsoft 365 вы можете отслеживать работоспособность службы Exchange в рамках подписки Microsoft 365 вашей организации. Мониторинг Exchange Online предоставляет информацию об инцидентах и предупреждениях, собираемых в следующих категориях:

- **Инфраструктура**. Проблема обнаружена в инфраструктуре Microsoft 365, принадлежащей корпорации Майкрософт и используемой для обеспечения регулярных обновлений и устранения проблем. Например, пользователи не могут получить доступ к Exchange Online из-за проблем в Exchange или другой облачной инфраструктуре Microsoft 365.
- **Сторонняя инфраструктура**. Проблема обнаружена в сторонней инфраструктуре, от которой зависит ваша организация. Для ее устранения требуется действие от вашей организации. Например, операции проверки подлинности пользователей регулируются сторонним поставщиком службы маркеров безопасности (STS), который не позволяет пользователям подключаться к Exchange Online.
- **Пользовательская инфраструктура**. Проблема обнаружена в инфраструктуре вашей организации, и для ее устранения требуется действие от вашей организации. Например, пользователи не могут получить доступ к Exchange Online, так как они не могут получить маркер проверки подлинности от поставщика STS, размещенного в вашей организации, из-за просроченного сертификата.

Ниже приведен пример страницы **Работоспособность служб** в Центре администрирования Microsoft 365, доступной в разделе **Работоспособность > Работоспособность службы**.

![Страница "Работоспособность служб" в Центре администрирования Microsoft 365](../media/microsoft-365-exchange-monitoring/service-health-dashboard-example.png)

Значение столбца **Состояние** указывает, является ли служба работоспособной или с ней связаны предупреждения и инциденты на основе облачных служб, поддерживаемых корпорацией Майкрософт. 

Значение в столбце **Проблемы вашей организации и сторонних поставщиков** указывает, что инфраструктура вашей организации или сторонние программы влияют на работоспособность служб пользователей в Exchange Online. Предупреждения и инциденты требуют *ваших* действий по устранению.

Ниже приведен пример страницы мониторинга **Exchange Online** в Центре администрирования Microsoft 365, доступной в разделе **Работоспособность > Работоспособность службы > Exchange Online**.

![Страница мониторинга Exchange Online в Центре администрирования Microsoft 365](../media/microsoft-365-exchange-monitoring/exhange-monitoring-example.png)

На странице мониторинга **Exchange Online** вы можете увидеть, работоспособна ли служба Exchange Online и связаны ли с ней какие-либо инциденты и предупреждения. С помощью мониторинга Exchange Online вы можете просмотреть работоспособность служб для определенных почтовых сценариев и увидеть сигналы почти в режиме реального времени для определения воздействия сценария. 

## <a name="requirements"></a>Требования

Эта предварительная версия включена для пользователей, отвечающих следующим требованиям: 

- У вашей организации должно быть не менее 5 000 лицензий на один или несколько следующих продуктов: Office 365 E3, Microsoft 365 E3, Office 365 E5, Microsoft 365 E5. 

  Например, у вашей организации может быть 3 000 лицензий Office 365 E3 и 2 500 лицензий Microsoft 365 E5, что в итоге составляет 5 500 лицензий на соответствующие продукты.

- У вашей организации должно быть как минимум 50 активных пользователей Exchange Online ежемесячно.

С помощью мониторинга Exchange Online вы можете просматривать сведения о работоспособности следующих почтовых клиентов на основе действий по чтению писем:

- Классическое приложение Outlook
- Outlook в Интернете
- Собственные почтовые клиенты iOS и Android 
- Приложение Outlook Mobile в iOS и Android 
- Клиент Outlook Mac

В этих клиентах вы можете просматривать количество активных пользователей за последние 30 минут на основе чтения писем пользователями, а также количество инцидентов и предупреждений в панели мониторинга. Эти данные сравниваются с тем же интервалом прошлой недели, чтобы проверить, имеется ли проблема. 

>[!Note]
> Количество активных пользователей измеряется по одному действию, например при чтении пользователем письма. В это показателе учитываются действия только за последние 30 минут.
>

Кроме того, вы можете отслеживать работоспособность Exchange Online в следующих сценариях:

- **Поток обработки почты**. Количество сообщений, успешно доставленных в почтовый ящик без задержек после того, как сообщение достигло сети Microsoft 365. 
- **Обычная проверка подлинности и современная проверка подлинности**. Количество пользователей, успешно прошедших проверку в службе Exchange Online.

![Пример мониторинга работоспособности Exchange для доставки почты](../media/microsoft-365-exchange-monitoring/exhange-monitoring-scenario-example.png)

Во всех этих сценариях ключевые показатели указываются в основной панели мониторинга за последние 30 минут. В подробных представлениях для каждого из этих сценариев демонстрируется тренд почти в режиме реального времени за последние семь дней с 30-минутным агрегированием в сравнении с прошлой неделей. 

## <a name="send-us-feedback"></a>Отправка отзывов и предложений

Существует два способа обратной связи:

- Использование параметра **Оставить отзыв**, доступного на каждой странице Центра администрирования Microsoft 365.
- Отправка отзыва с помощью ссылки **Эта запись полезна?** для определенного инцидента или предупреждения.

![Ссылка "Эта запись полезна?" для конкретного инцидента или предупреждения](../media/microsoft-365-exchange-monitoring/exhange-monitoring-example-incident-feedback.png)

## <a name="frequently-asked-questions"></a>Вопросы и ответы

#### <a name="1-why-dont-i-see-exchange-online-monitoring-under-health-in-the-microsoft-365-admin-center"></a>1. Почему я не вижу элемент "Мониторинг Exchange Online" в разделе "Работоспособность" Центра администрирования Microsoft 365? 

Сначала включите новый Центр администрирования на **главной** странице Центра администрирования Microsoft 365. 

Затем убедитесь, что вы отвечаете обоим следующим требованиям: 

- У вашей организации должно быть не менее 5 000 лицензий на один или несколько следующих продуктов: Office 365 E3, Microsoft 365 E3, Office 365 E5, Microsoft 365 E5. 
- У вашей организации должно быть как минимум 50 активных пользователей Exchange Online ежемесячно.

Если количество лицензий в вашей организации ниже 5 000, а число активных пользователей — менее 50 в месяц, мониторинг Exchange Online не будет включен, пока не будут выполнены эти требования.

#### <a name="2-the-active-user-count-in-the-dashboard-for-each-client-appears-to-be-low-we-have-a-lot-of-active-licenses-assigned-to-users-what-does-this-mean"></a>2. На панели мониторинга для каждого клиента отображается малое количество активных пользователей. У нас есть много активных лицензий, назначенных пользователям. Что это означает? 

Количество активных пользователей, отображаемое при мониторинге, основано на 30-минутном промежутке, в котором пользователи выполняли действия, указанные в функции. Это не следует путать с количеством использований. Чтобы просмотреть количество использований, воспользуйтесь отчетами о действиях в Центре администрирования Microsoft 365 (**Отчеты > Использование**).

#### <a name="3-will-there-be-other-monitoring-scenarios-for-other-services-such-as-teams-and-sharepoint"></a>3. Будут ли доступны другие сценарии мониторинга для других служб, таких как Teams и SharePoint? 

Майкрософт интегрирует эту функцию непосредственно в панель мониторинга работоспособности служб в Центре администрирования Microsoft 365. Это позволит корпорации Майкрософт расширить сценарии мониторинга для других служб, о чем будет объявлено при появлении соответствующих новостей. 

#### <a name="4-what-is-the-plan-for-general-availability-of-this-experience"></a>4. Каков план в отношении общей доступности этой функции? 

Корпорация Майкрософт интегрировала мониторинг Exchange Online непосредственно в панель мониторинга **Работоспособность служб** в Центре администрирования Microsoft 365. 

С помощью этой новой интегрированной функции корпорация Майкрософт планирует собрать ваши отзывы, а затем определить план в отношении общей доступности.

#### <a name="5-is-this-a-free-included-or-paid-extra-feature"></a>5. Это бесплатная (включенная) или платная (дополнительная) функция? 

Эта функция выпущена в общедоступной предварительной версии и доступна только для пользователей, удовлетворяющих требованиям из вопроса 1.

#### <a name="6-how-do-i-provide-feedback"></a>6. Как оставить отзыв? 

Для общего отзыва используйте значок **Оставить отзыв** в правом нижнем углу страницы мониторинга **Exchange Online**. 

Для отзыва об инцидентах или предупреждениях используйте ссылку **Эта запись полезна?**

#### <a name="7-where-is-the-data-instrumented-for-the-scenarios-that-show-activity-trends"></a>7. Где обрабатываются данные для сценариев, демонстрирующих тренды действий?

Данные обрабатываются в службе Exchange Online. Если возникнет сбой, прежде чем запрос достигнет службы Exchange Online, или возникнет сбой в Exchange Online, вы увидите падение сигнала активности.
