# <a name="exchange-2007-end-of-support-roadmap"></a>План действий после прекращения поддержки Exchange 2007;

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

**11 апреля 2017**г. сервер Exchange Server 2007 достиг конца поддержки. Если вы еще не начали миграцию с Exchange 2007 на Microsoft 365, Office 365 или Exchange 2016, настало время начать планирование. 
  
## <a name="what-does-end-of-support-mean"></a>Что означает окончание поддержки?

На сервере Exchange Server, как и почти все продукты корпорации Майкрософт, предусмотрены жизненные циклы поддержки, в которых мы предоставляем новые функции, исправления ошибок, исправления для системы безопасности и т. д. Этот жизненный цикл обычно продолжается в течение 10 лет с даты первоначального выпуска продукта, а окончание этого жизненного цикла называется окончанием поддержки продукта. Так как Exchange 2007 достиг конца поддержки 11 апреля 2017, корпорация Майкрософт больше не предоставляет:
  
- Техническая поддержка проблем, которые могут возникать;
    
- Исправление ошибок для обнаруженных проблем, которые могут повлиять на стабильность и удобство использования сервера;
    
- Исправления для системы безопасности уязвимостей, которые могут привести к уязвимости сервера, которые могут привести к нарушению безопасности;
    
- Обновления часового пояса.
    
Установка Exchange 2007 продолжится после этой даты. Однако из-за перечисленных выше изменений настоятельно рекомендуется выполнить миграцию из Exchange 2007 как можно скорее.
  
Дополнительные сведения о серверах Office 2007, близких в конце поддержки, приведены в статье [Планирование обновления с серверов и продуктов office 2007](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-are-my-options"></a>Что такое параметры?

Теперь, когда Exchange 2007 достиг конца поддержки, мы настоятельно рекомендуем ознакомиться со своими возможностями и подготовить план миграции. Вы можете выполнить указанные ниже действия.
  
- Миграция на Microsoft 365 с помощью прямой, поэтапной или гибридной миграции;
    
- Перенесите серверы Exchange 2007 в более новую версию Exchange на локальных серверах.
    
В следующих разделах каждый из этих вариантов рассматривается более подробно.
  
### <a name="migrate-to-microsoft-365"></a>Миграция в Microsoft 365

Вы можете перенести электронную почту в Microsoft 365 — это самый лучший и простой способ отказаться от использования развертывания Exchange 2007. При переходе на Microsoft 365 вы можете сделать один переход с более 10 лет старой технологии на функциональные возможности, такие как:
  
- Возможности обеспечения соответствия требованиям, такие как политики хранения, хранение на месте и хранение для судебного разбирательства, обнаружение электронных данных на месте и т. д.
    
- Группы Microsoft 365;
    
- Сортировка почты;
    
- MyAnalytics
    
- REST API для программного доступа к электронной почте, календарям, контактам и т. д.
    
Кроме того, Microsoft 365 также получает новые функции и возможности, и пользователи обычно могут приступить к их использованию сразу же. В дополнение к новым функциям вам не придется беспокоиться:
  
- Приобретение и обслуживание оборудования;
    
- Оплата от отопления и охлаждения серверов;
    
- Поддержка обновлений безопасности, продуктов и часовых поясов;
    
- Поддержка хранения и программного обеспечения для поддержки требований соответствия требованиям;
    
- Обновление до новой версии Exchange — вы всегда используете последнюю версию Exchange в Microsoft 365.

#### <a name="cutover-migration"></a>Прямая миграция

Прямой миграция это один, где в заранее выбранную дату и время вы перенесете все почтовые ящики, группы рассылки, контакты и т. д. в Microsoft 365; по завершении работы вы завершите работу локальных серверов Exchange и начнете использовать Microsoft 365 исключительно.
  
Метод миграции прямой отлично подходит для небольших организаций, у которых нет очень большого количества почтовых ящиков, и вы можете быстро перейти в Microsoft 365 и не хотите работать с некоторыми сложностями других методов. Кроме того, это несколько ограничено, так как оно должно быть завершено за неделю или меньше, так как для этого требуется, чтобы пользователи перестроили свои профили Outlook. Несмотря на то, что прямой миграция может обрабатывать до 2 000 почтовых ящиков, мы настоятельно рекомендуем перенести в этот метод максимум 150 почтовых ящиков. Если вы попытаетесь перенести более 150 почтовых ящиков, вам может быть недостаточно времени для переноса всех почтовых ящиков перед крайним сроком, а сотрудники ИТ, поддерживающих ИТ, могут перенастроить Outlook.
  
Если вы планируете выполнить миграцию прямой, вот несколько моментов, которые следует учесть:
  
- Microsoft 365 потребуется подключиться к серверам Exchange 2007 с помощью Outlook Anywhere через TCP-порт 443;
    
- Все локальные почтовые ящики будут перемещены в Microsoft 365;
    
- Вам потребуется локальная учетная запись администратора, которая имеет доступ для чтения содержимого почтовых ящиков пользователей;
    
- Обслуживаемые домены Exchange 2007, которые вы хотите использовать в Microsoft 365, необходимо добавить в качестве проверенных доменов в службе.
    
- Между моментом начала миграции и началом фазы завершения Microsoft 365 будет периодически синхронизировать почтовые ящики Microsoft 365 и локальные почтовые ящики. Это позволяет выполнить миграцию, не беспокоясь о том, что в локальных почтовых ящиках остались сообщения.
    
- Пользователи будут получать новые временные пароли для своей учетной записи Microsoft 365, которые необходимо будет изменить при первом входе в их почтовые ящики;
    
- Вам потребуется лицензия Microsoft 365, включающая Exchange Online для каждого переносимого почтового ящика пользователя.
    
- Пользователям потребуется настроить новый профиль Outlook на каждом устройстве и скачать электронную почту еще раз. Количество сообщений электронной почты, загружаемых Outlook, может различаться. Для получения дополнительных сведений об [изменении объема почты, который необходимо сохранить в автономном режиме](https://support.office.com/article/Change-how-much-mail-to-keep-offline-f3a1251c-6dd5-4208-aef9-7c8c9522d633?ui=en-US&amp;rs=en-US&amp;ad=US&amp;fromAR=1), изучите.
    
Чтобы узнать больше о прямой миграции, ознакомьтесь со следующего:
  
- [Что нужно знать о миграции электронной почты прямой](https://support.office.com/article/What-you-need-to-know-about-a-cutover-email-migration-to-Office-365-961978ef-f434-472d-a811-1801733869da)
    
- [Выполнение прямой миграции электронной почты](https://support.office.com/article/Perform-a-cutover-migration-of-email-to-Office-365-9496e93c-1e59-41a8-9bb3-6e8df0cd81b4)
    
#### <a name="staged-migration"></a>Поэтапная миграция

Поэтапная миграция состоит в том, что у вас есть несколько сотен или несколько тысяч почтовых ящиков, которые необходимо перенести в Microsoft 365, и вам не нужны какие-либо расширенные функции гибридной миграции, такие как общие сведения о доступности в календаре.
  
Поэтапная миграция отлично подходит для организаций, которым требуется больше времени на перенос почтовых ящиков в Microsoft 365, но по-прежнему планируете выполнить миграцию в течение нескольких недель. Вы можете перенести почтовые ящики в «пакетах», которые позволяют контролировать, сколько почтовых ящиков и какие из них переносятся в данный момент времени. Вы можете выполнять пакетную обработку почтовых ящиков пользователей в одном отделе, например, чтобы убедиться в том, что все они перемещены одновременно. Вы также можете оставлять почтовые ящики руководителей, пока не будет выполнен последний пакет. Как и при прямой миграции, пользователям потребуется воссоздать свои профили Outlook.
  
Если вы планируете поэтапную миграцию, примите во внимание следующие моменты:
  
- Microsoft 365 потребуется подключиться к серверам Exchange 2007 с помощью Outlook Anywhere через TCP-порт 443;
    
- Вам потребуется локальная учетная запись администратора, которая имеет доступ для чтения содержимого почтовых ящиков пользователей;
    
- Обслуживаемые домены Exchange 2007, которые вы хотите использовать в Microsoft 365, необходимо добавить в качестве проверенных доменов в службе.
    
- Вам потребуется создать CSV-файл с полным именем и адресом электронной почты каждого почтового ящика, который требуется перенести в пакет. Кроме того, необходимо включить новый пароль для каждого почтового ящика, который вы переносите, а затем отправить свой пароль каждому пользователю. Пользователю будет предложено сменить пароль при первом входе в новый почтовый ящик Microsoft 365;
    
- Между моментом запуска пакета миграции и началом фазы завершения Microsoft 365 будет периодически синхронизировать почтовые ящики Microsoft 365 и локальные почтовые ящики, включенные в пакет. Это позволяет выполнить миграцию, не беспокоясь о том, что в локальных почтовых ящиках остались сообщения.
    
- Пользователи будут получать новые временные пароли для своей учетной записи Microsoft 365, которые необходимо будет изменить при первом входе в почтовый ящик;
    
- Вам потребуется лицензия Microsoft 365, включающая Exchange Online для каждого переносимого почтового ящика пользователя.
    
- Пользователям потребуется настроить новый профиль Outlook на каждом устройстве и скачать электронную почту еще раз. Количество сообщений электронной почты, загружаемых Outlook, может различаться. Для получения дополнительных сведений об [изменении объема почты, который необходимо сохранить в автономном режиме](https://support.office.com/article/Change-how-much-mail-to-keep-offline-f3a1251c-6dd5-4208-aef9-7c8c9522d633?ui=en-US&amp;rs=en-US&amp;ad=US&amp;fromAR=1), изучите.
    
Чтобы узнать больше о поэтапной миграции, ознакомьтесь со следующего:

#### <a name="full-hybrid"></a>Полное гибридное развертывание

Полная Гибридная миграция — это одна организация, в которой у вашей организации много сотен и более десятков тысяч почтовых ящиков, и вы хотите переместить их в Microsoft 365. Так как эти миграции обычно являются более длительными, гибридная миграция позволяет выполнять следующие действия:
  
- Показывать локальным пользователям сведения о доступности в календаре для пользователей в Microsoft 365 и наоборот;
    
- Просмотр объединенного глобального списка адресов, который содержит получателей как в локальной среде, так и в Microsoft 365;
    
- Просмотр полных свойств получателей Outlook для всех пользователей независимо от того, являются ли они локальными или в Microsoft 365;
    
- Безопасная связь электронной почты между локальными серверами Exchange и Microsoft 365 с использованием TLS и сертификатов;
    
- Обрабатывать сообщения, отправляемые между локальными серверами Exchange и Microsoft 365 как внутренние, позволяя им:
    
  - Должны быть правильно оценены и обработаны транспортом и агентами соответствия требованиям, нацеленными на внутренние сообщения;
    
  - Обход фильтров защиты от нежелательной почты.
    
Полные гибридные миграции лучше всего подходят для организаций, которые должны оставаться в гибридной конфигурации в течение нескольких месяцев или больше. Вы получите функции, перечисленные ранее в этом разделе, а также синхронизацию службы каталогов, улучшения интегрированных функций соответствия и возможность перемещения почтовых ящиков в Microsoft 365 с помощью перемещения почтовых ящиков в сети. Microsoft 365 становится расширением локальной организации.
  
Если вы думаете, как выполнить полную гибридную миграцию, примите во внимание следующие моменты:
  
- Полные гибридные миграции подходят не ко всем типам организаций. Из-за сложности полных гибридных миграций Организации с менее чем несколькими сотнями почтовых ящиков обычно не видят преимущества, необходимые для установки. Если это звучит, как ваша организация, настоятельно рекомендуем рассмотреть прямой или поэтапную миграцию;
    
- Вам потребуется развернуть по крайней мере один сервер Exchange 2013 в организации Exchange 2007, чтобы он действовал в качестве гибридного сервера. Этот сервер будет обмениваться данными с Microsoft 365 от имени серверов Exchange 2007;
    
- Microsoft 365 вам потребуется подключиться к "гибридному серверу" с помощью Outlook Anywhere через TCP-порт 443;
    
- Вам потребуется настроить синхронизацию службы каталогов с помощью Azure Active Directory (Azure AD) для подключения между локальными серверами Active Directory и Microsoft 365;
    
- Пользователи смогут выполнять вход в свой почтовый ящик Microsoft 365, используя те же имя пользователя и пароль, которые они используют при входе в локальную сеть (требуется Azure AD Connect с синхронизацией паролей и/или службами федерации Active Directory);
    
- Вам потребуется лицензия Microsoft 365, включающая Exchange Online для каждого переносимого почтового ящика пользователя.
    
- Пользователям не нужно настраивать новый профиль Outlook на большинстве своих устройств (на некоторых старых телефонах Android может потребоваться новый профиль), и вам не придется повторно скачивать свои сообщения.
    
Если вы хотите выполнить полную гибридную миграцию, ознакомьтесь со следующими ресурсами, которые помогут вам выполнить миграцию:
  
- [Помощник по развертыванию Exchange](https://aka.ms/exdeploy)
    
- [Гибридные развертывания Exchange Server](https://technet.microsoft.com/library/jj200581%28v=exchg.150%29.aspx)
    
- [Мастер гибридной конфигурации](https://technet.microsoft.com/library/hh529921%28v=exchg.150%29.aspx)
    
- [Мастер гибридной конфигурации: вопросы и ответы](https://technet.microsoft.com/library/mt488940%28v=exchg.150%29.aspx)
    
- [Предварительные условия для гибридного развертывания](https://technet.microsoft.com/library/hh534377%28v=exchg.150%29.aspx)
    
### <a name="migrate-to-a-newer-version-of-exchange-server"></a>Переход на более новую версию Exchange Server

Несмотря на то, что вы уверены, что вы можете добиться наилучшего опыта работы пользователей, переполнив его на Microsoft 365, мы также понимаем, что в некоторых организациях необходимо хранить локальную электронную почту. Это может быть вызвано соблюдением нормативных требований, чтобы гарантировать, что данные не хранятся в центре обработки данных, расположенном в другой стране и т. д. Если вы решили сохранить локальную электронную почту, вы можете перенести среду Exchange 2007 в Exchange 2010, Exchange 2013 или Exchange 2016.
  
Рекомендуется выполнить миграцию в Exchange 2016, если вы не можете выполнить миграцию в Microsoft 365. Exchange 2016 включает все функции и улучшения, включенные в предыдущие выпуски Exchange, и наиболее полно соответствует возможностям, доступным в Microsoft 365 (хотя некоторые функции доступны только в Microsoft 365). Ознакомьтесь только с некоторыми из отсутствующих вещей.
  
|**Выпуск Exchange**|**Компоненты**|
|:-----|:-----|
|Exchange 2010  <br/> | Управление доступом на основе ролей (разрешения без списков ACL)  <br/>  Политики почтовых ящиков Outlook Web App  <br/>  Возможность совместного использования календарей сведений о занятости и делегировании между организациями  <br/> |

#### <a name="which-version-should-i-migrate-to"></a>В какой версии следует выполнить миграцию?

Рекомендуем сначала выполнить миграцию в Exchange 2016. Затем воспользуйтесь приведенными ниже сведениями, чтобы подтвердить ваше предположение или отправиль к Exchange 2016. Если вы не можете выполнить миграцию в Exchange 2016 по одной из причин или другой, выполните тот же процесс с Exchange 2013 и т. д.

Январь 2020 — гораздо больше, чем вы думаете!  <br/> |

Перемещение служб и другой инфраструктуры в Exchange 2010 или 2013 перемещение почтовых ящиков и общедоступных папок в Exchange 2010 или 2013, оставшиеся серверами Exchange Server с 2007 |

Переместите службы и другую инфраструктуру в Exchange 2013 перемещение почтовых ящиков и общедоступных папок в Exchange 2013 списанные серверы Exchange Server 2007 установите Exchange 2016 в существующую организацию Exchange Server 2013. Списать оставшиеся серверы Exchange 2013 > [!NOTE]> миграция с exchange 2013 на exchange 2016 проста. Обе версии имеют почти одинаковые аппаратные требования. ## <a name="what-if-i-need-help"></a>Что делать, если вам нужна помощь?

При переходе на Microsoft 365 вы можете использовать нашу службу Microsoft FastTrack. FastTrack предоставляет рекомендации, средства и ресурсы, позволяющие выполнить миграцию в Microsoft 365 по мере возможности. Все, что у вас есть, у вас есть реальный инженер по поддержке, который поможет вам выполнить миграцию, используя планирование и проектирование всех способов переноса последнего почтового ящика. Если вы хотите узнать больше о FastTrack, ознакомьтесь со сведениями о [Microsoft FastTrack](https://fasttrack.microsoft.com/).
