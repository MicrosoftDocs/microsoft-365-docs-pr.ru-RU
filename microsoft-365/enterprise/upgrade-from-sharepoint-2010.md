# <a name="upgrading-from-sharepoint-2010"></a>Переход с SharePoint 2010

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Поддержка Microsoft SharePoint 2010 и SharePoint Server 2010 завершится **13 апреля 2021 г.** В этой статье данная статья содержит ресурсы, которые помогут перенести существующие данные SharePoint Server 2010 в SharePoint Online в Microsoft 365 или обновить свою локальной среде SharePoint Server 2010.

## <a name="what-is-end-of-support"></a>Что такое *окончание поддержки?*

Большинство продуктов Майкрософт имеют жизненный цикл поддержки, в течение которого они получают новые функции, исправления ошибок, исправления безопасности и так далее. После даты окончания поддержки продукт не перестает работать, но корпорация Майкрософт больше не предоставляет:

- Техническая поддержка для проблем, которые могут возникнуть.

- Исправления ошибок, которые могут повлиять на стабильность и доступность сервера.

- Исправления безопасности для уязвимостей, которые могут сделать сервер уязвимым к нарушению безопасности.

- Обновления часовой пояс.

Это означает, что больше не будет обновлений, исправлений или исправлений для продукта (включая исправления и исправления для системы безопасности). Служба поддержки Майкрософт полностью перенацелит свои усилия на поддержку более последних версий.

По мере того как поддержка SharePoint Server 2010 приближается к концу, удалите данные, которые вам больше не нужны, перед обновлением продукта и переносом важных данных.

> [!NOTE]
> Жизненный цикл программного обеспечения обычно длится десять лет с начального выпуска. [Поставщики решений](https://go.microsoft.com/fwlink/?linkid=841249) Майкрософт могут помочь вам перейти на следующую версию программного обеспечения или перейти на миграцию Microsoft 365 (или и то, и другое). Убедитесь, что вы знаете даты окончания поддержки критически важных технологий, особенно для версии Microsoft SQL Server, используемой с SharePoint. Дополнительные сведения см. в [политике фиксированного жизненного цикла.](https://support.microsoft.com/help/14085)

## <a name="whats-next"></a>Что дальше?

SharePoint Server 2013 и SharePoint Foundation 2013 можно установить локально на ваших серверах. Вы также можете использовать SharePoint Online, веб-службу, которая входит в состав Microsoft 365. Вы можете выбрать:

- Миграция в SharePoint Online.

- Обновив SharePoint Server или SharePoint Foundation в локальной организации.

- Сделайте оба из вышеперечисленного.

- Реализация [гибридного решения SharePoint.](https://docs.microsoft.com/sharepoint/hybrid/hybrid)

Учитывайте скрытые затраты на обслуживание фермы серверов, включая обслуживание или перенос настроек и обновление оборудования. Если вы учитывали эти факторы, будет проще обновить локально. Если вы запустите ферму на устаревших серверах SharePoint Server без интенсивной настройки, вы сможете воспользоваться преимуществами запланированной миграции в SharePoint Online. В локальной среде SharePoint Server также можно рассмотреть возможность перемещения некоторых данных в SharePoint Online, чтобы сократить затраты на управление оборудованием.

> [!NOTE]
> Администраторы SharePoint могут создать подписку на Microsoft 365, настроить новые сайты SharePoint Online, а затем очистить SharePoint Server 2010, выверив только основные документы на новые сайты. Затем любые оставшиеся данные можно оцедить с сайта SharePoint Server 2010 в локальном архиве.

|SharePoint Online|Локальное sharePoint Server|
|---|---|
|Высокая стоимость во времени (планирование, выполнение/проверка)|Высокая стоимость во времени (планирование, выполнение/проверка)|
|Снижение стоимости в средствах (без покупок оборудования)|Более высокая стоимость в средствах (покупки оборудования)|
|Разовая стоимость при миграции|Однократные затраты, повторяемые при будущей миграции|
|Низкая общая стоимость владения или обслуживания|Высокая общая стоимость владения или обслуживания|

Разовая переход на Microsoft 365 будет стоить дороже, если вы упорядожите данные и решите, что нужно перенести в облако и что оставить. Но после переноса данных будущие обновления будут автоматически, так как вам больше не потребуется управлять обновлениями оборудования и программного обеспечения. Время работы фермы будет запротеститься соглашением об уровне обслуживания [(SLA)](https://go.microsoft.com/fwlink/?linkid=843153)Майкрософт.

### <a name="migrate-to-sharepoint-online"></a>Миграция на SharePoint Online

Убедитесь, что SharePoint Online предлагает все необходимые функции. См. [описание службы SharePoint.](https://docs.microsoft.com/office365/servicedescriptions/sharepoint-online-service-description/sharepoint-online-service-description)

Вы не можете перейти непосредственно с SharePoint Server 2010 (или SharePoint Foundation 2010) на SharePoint Online. Поэтому большая часть работы по миграции вручную. Но на этом этапе вы можете обрезать данные и сайты, которые больше не нужны перед перемещением. Другие данные можно архивировать в хранилище. 

Помните, что SharePoint Server 2010 и SharePoint Foundation 2010 не перестанут работать после окончания поддержки. Таким образом, у администраторов может быть период, когда SharePoint все еще работает, если их клиенты забывают переместить некоторые данные.

При обновлении до SharePoint Server 2013 или SharePoint Server 2016 и переносе данных в SharePoint Online можно использовать API миграции [SharePoint](https://support.office.com/article/Upload-on-premises-content-to-SharePoint-Online-using-PowerShell-cmdlets-555049c6-15ef-45a6-9a1f-a1ef673b867c?ui=en-US&amp;rs=en-US&amp;ad=US) для переноса информации в OneDrive для бизнеса.

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локального сервера SharePoint Server

В SharePoint Server 2019 обновления должны быть *последовательными.* Не существует способа обновления с SharePoint Server 2010 до SharePoint Server 2016 или непосредственно до SharePoint 2019. Путь последовательного обновления:

- SharePoint Server 2010 \> SharePoint Server 2013 \> SharePoint Server 2016

Для этого необходимо время и планирование всего пути от SharePoint 2010 к SharePoint Server 2016. Обновления связаны с затратами на оборудование (SQL серверы также должны быть обновлены), программное обеспечение и администрирование. Кроме того, может потребоваться обновление или даже отказ от настроек. Перед обновлением фермы SharePoint Server обязательно документировать критически важные настройки.

> [!NOTE]
> Вы можете поддерживать ферму SharePoint 2010, устанавливать ферму SharePoint Server 2016 на новом оборудовании (поэтому отдельные фермы работают рядом), а затем планировать и выполнять перенос контента вручную (например, для скачивания и повторной отправки контента). Однако существуют потенциальные ошибки в этих ручных перемещениях, например документы из 2010 года с текущей последней измененной учетной записью с псевдонимом учетной записи, которая перемещается вручную. Некоторые задачи необходимо делать заранее, такие как воссоздание сайтов, подсайтов, разрешений и структур списков. Перед обновлением необходимо очистить среду. Подумайте, какие данные можно переместить в хранилище или больше не нужны. Это может снизить влияние миграции. Перед обновлением необходимо быть уверены, что существующая ферма работает, и (конечно) перед списанием!

Не забудьте просмотреть поддерживаемые и *неподдержки путей обновления:*

### <a name="create-a-hybrid-solution-with-sharepoint-online-and-sharepoint-server-on-premises"></a>Создание гибридного решения с локальной версией SharePoint Online и SharePoint Server

Гибридная настройка обеспечивает наилучшие возможности локальной и сетевой конфигурации для некоторых потребностей миграции. Вы можете подключить фермы SharePoint Server 2013, 2016 или 2019 к SharePoint Online, чтобы создать гибридную сеть SharePoint. Узнайте о гибридных решениях [SharePoint.](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)

Если целью миграции является гибридная ферма SharePoint Server, выясните, какие сайты и пользователей нужно переместить в Интернет, а какие должны оставаться в локальной сети. Ранжирование контента фермы SharePoint Server с высоким, средним или низким влиянием на вашу компанию может помочь в этом решении. Вам может потребоваться только поделиться учетными записями пользователей для входа в систему и индексом поиска SharePoint Server с SharePoint Online. Но этот фактор может быть неясным, пока вы не посмотрите, как используются сайты. Если позже ваша компания решит перенести весь контент в SharePoint Online, вы можете переместить все оставшиеся учетные записи и данные в Интернет и списать свою локальной ферму. На этом этапе управление фермой SharePoint будет происходить с помощью консолей Microsoft 365.

Ознакомьтесь с существующими типами гибридной конфигурации и настройкой подключения между локальной фермой SharePoint и подпиской на Microsoft 365.

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локального сервера SharePoint Server

В обновлениях SharePoint нет способа пропустить версии. Это означает, что обновления будут обновляться последовательно:

- SharePoint 2007 \> SharePoint Server 2010 \> SharePoint Server 2013 \> SharePoint Server 2016

Чтобы перейти на весь путь от SharePoint 2007 к SharePoint Server 2016, необходимо значительно уделить время и затраты на оборудование (также необходимо обновить SQL серверы), программное обеспечение и административные расходы. Настройки необходимо обновить или отказаться от них в соответствии с критичностью функции.

> [!NOTE]
> Можно поддерживать ферму SharePoint 2007, устанавливать ферму SharePoint Server 2016 на новом оборудовании (поэтому отдельные фермы работают рядом), а затем планировать и выполнять перенос контента вручную (например, для скачивания и повторной отправки контента). Однако у этих ручных перемещения есть некоторые недостатки, например перемещение документов, заменяющих последнюю измененную учетную запись псевдонимом учетной записи, которая перемещается вручную. Также необходимо заранее сделать много работы, например повторное воссоздание сайтов, подсайтов, разрешений и структур списков. В любом случае подумайте, какие данные можно переместить в хранилище или больше не нужно снижать влияние миграции.

Перед обновлением необходимо очистить среду. Перед обновлением необходимо быть уверены, что существующая ферма работает и, конечно, перед списанием.

### <a name="create-a-sharepoint-hybrid-solution-between-sharepoint-online-and-on-premises"></a>Создание гибридного решения SharePoint между SharePoint Online и локальной версией

Если ответ на ваши потребности в миграции находится где-то между контролем, предлагаемым локально, и более низкой стоимостью владения, предложенной SharePoint Online, вы можете подключить фермы SharePoint Server 2013 или 2016 к SharePoint Online с помощью гибридной системы. [Узнайте о гибридных решениях SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)

Если вы решите, что гибридная ферма SharePoint Server будет пользоваться преимуществами вашей компании, ознакомьтесь с существующими типами гибридной конфигурации и настройкой подключения между локальной фермой SharePoint и подпиской на Microsoft 365.

Возможно, вы захотите создать среду тестирования и тестирования Microsoft 365, которую можно настроить с помощью руководств по [лаборатории тестирования.](m365-enterprise-test-lab-guides.md) Получив пробную или приобретенную подписку на Microsoft 365, вы можете создать в SharePoint Online коллекции веб-сайтов, веб-сайты и библиотеки документов, в которые можно перенести данные. Вы можете перейти вручную, с помощью API миграции или, если вы хотите перенести содержимое содержимого my Site в OneDrive для бизнеса, с помощью мастера гибридной миграции.

> [!NOTE]
> Для использования гибридного варианта ферму SharePoint Server 2010 необходимо сначала обновить локально до SharePoint Server 2013 или 2016. SharePoint Foundation 2010 и SharePoint Foundation 2013 не поддерживают гибридные подключения с SharePoint Online.
