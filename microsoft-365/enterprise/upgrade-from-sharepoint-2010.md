# <a name="upgrading-from-sharepoint-2010"></a>Переход с SharePoint 2010

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft SharePoint 2010 и SharePoint Server 2010 достигают окончания поддержки до **13 апреля 2021 г**. В этой статье представлены ресурсы, которые помогут вам перенести существующие данные SharePoint Server 2010 в SharePoint Online в Microsoft 365 или обновить локальную среду SharePoint Server 2010.

## <a name="what-is-end-of-support"></a>Что представляет собой *конец поддержки*?

Большинство продуктов корпорации Майкрософт имеют жизненный цикл поддержки, в течение которого они получают новые функции, исправления ошибок, исправления для системы безопасности и т. д. После даты окончания поддержки продукт не перестает работать, но корпорация Майкрософт больше не предоставляет:

- Техническая поддержка проблем, которые могут возникнуть.

- Исправление ошибок для проблем, которые могут повлиять на стабильность и удобство использования сервера.

- Исправления для системы безопасности уязвимостей, которые могут сделать сервер уязвимым для нарушений безопасности.

- Обновления часового пояса.

Это означает, что не будет больше обновлений, исправлений или исправлений для продукта (включая исправления системы безопасности и исправления). Служба поддержки Майкрософт полностью переместит свои действия по поддержке более поздних версий.

В конце поддержки подходов SharePoint Server 2010 удалите ненужные данные, прежде чем обновлять продукт и переносить важные данные.

> [!NOTE]
> Жизненный цикл программного обеспечения обычно длится не более десяти лет из первоначального выпуска. [Поставщики решений Майкрософт](https://go.microsoft.com/fwlink/?linkid=841249) могут помочь вам выполнить обновление до следующей версии программного обеспечения или выполнить миграцию на Microsoft 365 (или и то, и другое). Убедитесь, что у вас есть сведения о датах окончания поддержки для важнейших базовых технологий, особенно для версии Microsoft SQL Server, используемой с SharePoint. Дополнительные сведения см. в разделе [фиксированная политика жизненного цикла](https://support.microsoft.com/help/14085).

## <a name="whats-next"></a>Что дальше?

SharePoint Server 2013 и SharePoint Foundation 2013 можно установить локально на свои серверы. Или вы можете использовать SharePoint Online, который является веб-службой, которая входит в состав Microsoft 365. Вы можете выбрать один из следующих вариантов:

- Переход на SharePoint Online.

- Обновление SharePoint Server или SharePoint Foundation локально.

- Выполните оба описанных выше действия.

- Реализация [гибридного решения SharePoint](https://docs.microsoft.com/sharepoint/hybrid/hybrid) .

Рассмотрите скрытые затраты на обслуживание фермы серверов, в том числе обслуживание и перенос настроек и обновление оборудования. Если у вас есть учет этих факторов, вы сможете упростить обновление локальной системы. Если вы выполняете ферму на устаревших серверах SharePoint без высокой настройки, вы можете воспользоваться преимуществами запланированной миграции в SharePoint Online. Для локальной среды SharePoint Server можно также переместить некоторые данные в SharePoint Online, чтобы снизить затраты на управление оборудованием.

> [!NOTE]
> Администраторы SharePoint могут создать подписку на Microsoft 365, настроить новые сайты SharePoint Online, а затем удалить их из SharePoint Server 2010, изменяя только важные документы на новые сайты. Затем все оставшиеся данные можно очистить от сайта SharePoint Server 2010 к локальным архивам.

|SharePoint Online|Локальная среда SharePoint Server|
|---|---|
|Высокая стоимость во время (планирование, выполнение и проверка)|Высокая стоимость во время (планирование, выполнение и проверка)|
|Снижение затрат на фонды (без приобретения оборудования)|Более высокие затраты на фонды (приобретение оборудования)|
|Одноразовые затраты в процессе миграции|Одноразовые затраты, повторяющиеся в будущие миграции|
|Снижение совокупной стоимости владения и обслуживания|Большая совокупная стоимость владения/обслуживания|

Переход на Microsoft 365 в одноразовом режиме будет иметь более высокие затраты при Организации данных и принятия решения о том, что следует предпринять в облаке и что следует оставить позади. Но после переноса данных будущие обновления будут выполняться автоматически, так как больше не требуется управлять оборудованием и программным обеспечением. И время ожидания фермы будет подархивировано [соглашением об уровне обслуживания (SLA) Майкрософт](https://go.microsoft.com/fwlink/?linkid=843153).

### <a name="migrate-to-sharepoint-online"></a>Миграция на SharePoint Online

Убедитесь, что SharePoint Online предоставляет все необходимые функции. Ознакомьтесь с [описанием службы SharePoint](https://docs.microsoft.com/office365/servicedescriptions/sharepoint-online-service-description/sharepoint-online-service-description).

Вы не можете перенести непосредственно из SharePoint Server 2010 (или SharePoint Foundation 2010) в SharePoint Online. Выполнение большинства операций по миграции выполняется вручную. Но эта стадия позволяет удалять данные и сайты, которые больше не нужны перед перемещением. Вы можете архивировать другие данные в хранилище. 

Помните, что SharePoint Server 2010 и SharePoint Foundation 2010 не перестают работать в конце поддержки. Так что администраторы могут использовать период, когда SharePoint все еще работает, если их клиенты забывают переместить некоторые данные.

Если вы выполняете обновление до SharePoint Server 2013 или SharePoint Server 2016 и принимаете решение о переводе данных в SharePoint Online, вы можете использовать [API миграции SharePoint](https://support.office.com/article/Upload-on-premises-content-to-SharePoint-Online-using-PowerShell-cmdlets-555049c6-15ef-45a6-9a1f-a1ef673b867c?ui=en-US&amp;rs=en-US&amp;ad=US) для переноса данных в OneDrive для бизнеса.

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локальной версии SharePoint Server

В SharePoint Server 2019 обновление  *должно идти последовательно*. Переход с SharePoint Server 2010 на SharePoint Server 2016 или SharePoint 2019 напрямую не предусмотрен. Способ последовательного обновления:

- SharePoint Server 2010 \> SharePoint server 2013 \> sharepoint Server 2016

Это займет некоторое время и планирование для отслеживания всего пути от SharePoint 2010 до SharePoint Server 2016. Обновления подразумевают затраты на оборудование (серверы SQL Server также необходимо обновить), программное обеспечение и администрирование. Кроме того, настройки может потребоваться обновить или даже отброшены. Перед обновлением фермы SharePoint Server обязательно задокументируйте критические настройки.

> [!NOTE]
> Вы можете сохранить свою конечную ферму SharePoint 2010, установить ферму SharePoint Server 2016 на новом оборудовании (чтобы отдельные фермы выполнялись параллельно), а затем спланировать и выполнить миграцию контента вручную (например, для загрузки и повторной загрузки контента). Но существуют потенциальные ловушки для таких ручных перемещений, например документы, поступающие из 2010, у которых есть текущая учетная запись, которая была изменена с помощью псевдонима учетной записи, которая выполняет перемещение вручную. Некоторые действия должны быть выполнены заранее, например, для создания сайтов, дочерних сайтов, разрешений и структур списков. Перед обновлением обязательно очистите среду. Определите, какие данные можно переместить в хранилище или больше не нужны. Это может снизить влияние миграции. Перед началом обновления необходимо убедиться в том, что существующая ферма работает, и (безусловно) до ликвидации!

### <a name="create-a-hybrid-solution-with-sharepoint-online-and-sharepoint-server-on-premises"></a>Создание гибридного решения в локальной среде SharePoint Online и SharePoint Server

Гибридная Настройка предоставляет оптимальные как в локальной среде, так и в Интернете для некоторых задач миграции. Вы можете подключить фермы SharePoint Server 2013, 2016 или 2019 к SharePoint Online, чтобы создать гибридную среду SharePoint: [сведения о гибридных решениях SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx).

Если используется гибридная ферма SharePoint Server, укажите, какие сайты и пользователи будут перемещаться в сети и какие должны быть сохранены локально. Ранжирование контента фермы SharePoint Server в качестве высокого, среднего или незначительного воздействия на вашу компанию поможет ваше решение. Вам может потребоваться предоставить общий доступ к учетным записям пользователей только для входа и индекса поиска SharePoint Server в SharePoint Online. Но этот фактор может быть снят, пока не будет рассмотрено, как используются сайты. Если в вашей компании вы решите перенести весь контент в SharePoint Online, вы можете переместить все оставшиеся учетные записи и данные в оперативный режим и списать свою локальную ферму. Управление и администрирование фермы SharePoint выполняется с помощью консолей Microsoft 365 с этого точки.

Обязательно ознакомьтесь с существующими типами гибридных развертываний и настройками подключения между локальной фермой SharePoint и подпиской на Microsoft 365.

Если вы решили создать новый сайт Microsoft 365 и перенести в него данные вручную, проверьте [Параметры Microsoft 365](https://www.microsoft.com/microsoft-365/).

### <a name="upgrade-sharepoint-server-on-premises"></a>Обновление локальной версии SharePoint Server

Невозможно пропускать версии при обновлении SharePoint. Это означает, что обновление осуществляется последовательно:

- SharePoint 2007 \> SharePoint server 2010 \> sharepoint Server 2013 \> SharePoint Server 2016

Чтобы получить полный путь от SharePoint 2007 к SharePoint Server 2016, потребуется значительное количество времени и будет использоваться оборудование (серверы SQL Server также необходимо обновить), программное обеспечение и затраты на администрирование. Настройка должна быть обновлена или отменена в соответствии с критическим компонентом.

> [!NOTE]
> Вы можете сохранить свою конечную ферму SharePoint 2007, установить ферму SharePoint Server 2016 на новом оборудовании (чтобы отдельные фермы выполнялись параллельно), а затем спланировать и выполнить миграцию контента вручную (например, для скачивания и повторной загрузки контента). Но для этих ручных перемещений существует ряд недостатков, например перемещение документов с заменой последней измененной учетной записи на псевдоним учетной записи, которая выполняет перемещение вручную. И многие работы должны быть выполнены заранее, например, для повторного создания сайтов, дочерних сайтов, разрешений и структур списков. В любом случае определите, какие данные можно переместить в хранилище или больше не требуется для снижения влияния миграции.

Перед обновлением обязательно очистите среду. Перед началом обновления необходимо убедиться в том, что существующая ферма работает, и определенно перед ликвидацией!

### <a name="create-a-sharepoint-hybrid-solution-between-sharepoint-online-and-on-premises"></a>Создание гибридного решения SharePoint между SharePoint Online и локальной системой

Если вам нужен ответ на миграцию между элементом управления, предлагаемым локальной системой, и более низкие затраты на владение, предлагаемые SharePoint Online, вы можете подключить фермы SharePoint Server 2013 или 2016 к SharePoint Online с помощью гибридной среды. [Сведения о гибридных решениях SharePoint](https://support.office.com/article/4c89a95a-a58c-4fc1-974a-389d4f195383.aspx)

Если вы решили, что Гибридная ферма SharePoint Server будет использовать преимущества вашей организации, ознакомьтесь с существующими типами гибридных развертываний и настройками подключения между локальной фермой SharePoint и подпиской на Microsoft 365.

Вам может потребоваться создать среду разработки и тестирования Microsoft 365, которую можно настроить с помощью [руководств по лаборатории тестирования](m365-enterprise-test-lab-guides.md). После получения пробной или приобретенной подписки на Microsoft 365 вы можете создать семейства веб-сайтов, веб-сайты и библиотеки документов в SharePoint Online, в которые можно перенести данные. Вы можете выполнить миграцию вручную, используя API миграции, или, если вы хотите перенести контент личного сайта в OneDrive для бизнеса с помощью мастера гибридной конфигурации.

> [!NOTE]
> Чтобы использовать гибридную среду, необходимо сначала обновить локальную ферму SharePoint 2010 Server до SharePoint Server 2013 или 2016. SharePoint Foundation 2010 и SharePoint Foundation 2013 не поддерживают гибридные подключения к SharePoint Online.
