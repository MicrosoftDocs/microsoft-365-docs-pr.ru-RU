# <a name="multi-factor-authentication-for-your-microsoft-365-for-enterprise-test-environment"></a>Многофакторная проверка подлинности для Microsoft 365 для тестовой среды предприятия

*Это руководство по тестовой лаборатории можно использовать как для Microsoft 365, так и для Office 365 корпоративный среды тестирования.*

Чтобы обеспечить дополнительный уровень безопасности при входе в Microsoft 365 службу или приложение, использующее клиента Azure AD для подписки, можно включить многофакторную проверку подлинности Azure AD, для проверки учетной записи требуется не только имя пользователя и пароль.

При многофакторной проверке подлинности пользователи должны подтвердить телефонный звонок, ввести код проверки, отправленный в текстовом сообщении, или проверить проверку подлинности с помощью приложения на своих смартфонах после правильного ввода паролей. Они могут войти только после того, как этот второй фактор проверки подлинности будет удовлетворен.
  
В этой статье описывается, как включить и протестировать проверку подлинности на основе текстовых сообщений для определенной учетной записи пользователя.
  
Настройка многофакторной проверки подлинности для учетной записи в Microsoft 365 для корпоративной тестовой среды включает два этапа и третий необязательный этап:
- [Этап 1. Создание Microsoft 365 для корпоративной тестовой среды](#phase-1-build-out-your-microsoft-365-for-enterprise-test-environment)
- [Этап 2. Включение и проверка многофакторной проверки подлинности для учетной записи User 2](#phase-2-enable-and-test-multi-factor-authentication-for-the-user-2-account)
- [Этап 3. Включить и протестировать многофакторную проверку подлинности с помощью политики условного доступа](#phase-3-enable-and-test-multi-factor-authentication-with-a-conditional-access-policy)

## <a name="phase-1-build-out-your-microsoft-365-for-enterprise-test-environment"></a>Этап 1. Создание Microsoft 365 для корпоративной тестовой среды

Если вы просто хотите проверить многофакторную проверку подлинности в облегченом режиме с минимальными требованиями, следуйте инструкциям в [базовой конфигурации Lightweight.](lightweight-base-configuration-microsoft-365-enterprise.md)
  
Если вы хотите протестировать многофакторную проверку подлинности в смоделированном предприятии, следуйте инструкциям в [сквозной проверке подлинности.](pass-through-auth-m365-ent-test-environment.md)
  
> [!NOTE]
> Тестирование многофакторной проверки подлинности не требует имитации тестовой среды предприятия, которая включает смоделированную интрасеть, подключенную к Интернету, и синхронизацию каталогов для леса служб домена Active Directory (AD DS). Она показана здесь лишь для того, чтобы вы могли проверить многофакторную проверку подлинности и поэкспериментировать с этим компонентом в типичной для организаций среде.
  
## <a name="phase-2-enable-and-test-multi-factor-authentication-for-the-user-2-account"></a>Этап 2. Включение и проверка многофакторной проверки подлинности для учетной записи User 2

Чтобы включить многофакторную проверку подлинности для учетной записи User 2, сделайте следующее:
  
1. Откройте отдельный частный экземпляр браузера, перейдите в центр администрирования Microsoft 365 и войдите в глобальную учетную запись [https://portal.microsoft.com](https://portal.microsoft.com) администратора.
    
2. В левой навигации выберите **Пользователи**  >  **Активные пользователи**.
    
3. В области Активные пользователи выберите **многофакторную проверку подлинности.**
    
4. В списке выберите **учетную запись Пользователя 2.**
    
5. В разделе **User 2** в разделе **Быстрые действия** выберите **Включить**.
    
6. В **диалоговом окне О включаемом** многофакторном диалоговом окне Auth выберите **Включить многофакторный auth**.
    
7. В **диалоговом окне** Обновления выберите **Закрыть**.
    
8. На **вкладке Microsoft 365 центра администрирования** выберите значок учетной записи пользователя в правом верхнем справа, а затем выберите **Sign out**.
    
9. Закройте окно браузера.
   
Чтобы настроить отправку текстового сообщения для проверки учетной записи User 2 и проверить ее, сделайте следующее:
  
1. Откройте новый частный экземпляр браузера.
    
2. Перейдите в [центр администрирования Microsoft 365](https://admin.microsoft.com) и войдите с именем и паролем учетной записи Пользователя 2.
    
3. После регистрации вам будет предложено настроить учетную запись для получения дополнительных сведений. Нажмите кнопку **Далее**.
    
4. На странице **Дополнительная проверка безопасности**: 
    
   - Выберите свою страну или регион.
    
   - Введите номер телефона смартфона, который будет получать текстовые сообщения.
    
   - В **методе** выберите **Отправить мне код текстовым сообщением**.
    
5. Нажмите кнопку **Далее**.
    
6. Введите код проверки из текстового сообщения, полученного на смартфоне, а затем выберите **Verify**.
    
7. На **шаге 3. Сохранить существующие приложения** страницу, выберите **Готово**.
    
8. При первом входе в учетную запись User 2 вам будет предложено изменить пароль. Дважды введите исходный пароль и новый пароль, а затем выберите пароль **Update и войдите.** Запишите новый пароль в надежном месте.
    
    Вы должны увидеть Office для пользователя 2 на вкладке **Microsoft Office главная** в браузере.

## <a name="phase-3-enable-and-test-multi-factor-authentication-with-a-conditional-access-policy"></a>Этап 3. Включить и протестировать многофакторную проверку подлинности с помощью политики условного доступа

*Этот этап можно использовать только для Microsoft 365 для корпоративной тестовой среды.*

На этом этапе вы включаете многофакторную проверку подлинности для учетной записи Пользователя 3 с помощью группы и политики условного доступа.

Затем создайте новую группу MFAUsers и добавьте к ней учетную запись Пользователя 3.

1. На **вкладке Microsoft 365 центра администрирования** выберите **Группы** в левой навигации, а затем выберите **Группы**.
2. Выберите **Добавить группу**.
3. В области **Выберите групповую группу** выберите **безопасность** и выберите **Далее**.
4. В области **Настройка базовых элементов** выберите **Создать** группу, а затем выберите **Закрыть**.
5. В области **Просмотреть и завершить добавление групповой** области введите **MFAUsers** и выберите **Далее**.
6. В списке групп выберите **группу MFAUsers.**
7. В области **MFAUsers** выберите **Членов,** а затем выберите Просмотр всех и **управление участниками.**
8. В области **MFAUsers** выберите Добавить **участников,** выберите учетную запись **Пользователя 3** и выберите **сохранить**  >    >  **закрыться.**

Далее создайте политику условного доступа, которая потребует многофакторной проверки подлинности для членов группы MFAUsers.

1. В новой вкладке браузера перейдите к [https://portal.azure.com](https://portal.azure.com) .
2. Выберите **Azure Active Directory**  >  **условный**  >  **доступ к безопасности.**
3. В области **условного доступа — политики выберите** новую **политику.**
4. В новой **области** введите **MFA для учетных записей** пользователей в **поле Имя.**
5. В разделе **Назначения выберите** Пользователей **и группы.**
6. На **вкладке Включить** области **Пользователи** и группы выберите **Выберите** пользователей и группы Пользователи  >  **и группы**  >  **Выберите**.
7. В области **Выберите** выберите группу **MFAUsers** и выберите **Выберите**  >  **Готово.**
8. В разделе **Элементы управления доступом** в **новой области** выберите **Grant**.
9. В области **Грант** выберите **требуемую многофакторную** проверку подлинности, а затем выберите **Выберите**.
10. В новой **области** выберите **политику "Включить",** а затем выберите **Создать**.
11. Закрой **портал Azure** **и Microsoft 365 вкладки центра** администрирования.

Чтобы проверить эту политику, вопишитесь и вопишитесь в учетную запись Пользователя 3. Вы должны быть предложены для настройки MFA. Это показывает, что применяется политика MFAUsers.
