# <a name="microsoft-365-informed-network-routing-preview"></a>Microsoft 365 сообщил о маршрутизации в сети (Предварительная версия)

Уведомление о маршрутизации в сети — это функция, которая интегрирует различные приложения Microsoft 365 с решениями по сети с программным обеспечением по протоколу SD-WAN, чтобы оптимизировать и улучшить сетевое подключение к конечным точкам службы Майкрософт. Оптимизированное подключение по протоколу SD для глобальной сети может привести к повышению производительности и производительности пользователей.

>[!IMPORTANT]
>Служба Microsoft 365 сообщила о маршрутизации в сети в настоящее время в состоянии предварительной версии. Дополнительные сведения о предварительной версии, включая рекомендации по получению помощи, можно найти в [статье Microsoft 365 с уведомлением об общедоступной предварительной версии сетевой маршрутизации](https://go.microsoft.com/fwlink/?linkid=2151565)

## <a name="overview"></a>Обзор

Уведомление о маршрутизации в сети предоставляет канал обмена данными с двунаправленным письмом между корпорацией Майкрософт и решением SD-WAN. Для всех настроенных расположений Office и Интернет, корпорация Майкрософт периодически разделяет Отзывы и предложения с решением SD/WAN относительно качества выбранных приложений Microsoft 365 для сетевого трафика, связанного с конкретными Интернет каналами. С помощью этой обратной связи решение SD-WAN может выполнить интеллектуальные действия по восстановлению путем маршрутизации трафика приложений Microsoft 365 через альтернативные доступные ссылки. 

Снижение качества обслуживания в пути к определенному каналу Интернета, такому как Увеличенная задержка или высокая потеря пакетов, трудно обнаружить постоянно. Эти падения могут повредить взаимодействие с пользователем для таких приложений, как Exchange Online, SharePoint, OneDrive и Microsoft Teams. Распространенные признаки: медленный поиск контента Exchange, высокая скорость передачи при взаимодействии с библиотеками документов SharePoint или OneDrive, а также неудовлетворительный вызов или качество собрания в Microsoft Teams.

Механизм обратной связи и восстановления в рамках сетевой информированной маршрутизации позволяет динамически определять такие проблемы почти в режиме реального времени и уведомляет развернутое решение SD/WAN, чтобы выполнять автоматические действия по восстановлению.

Канал общего доступа к данным также используется для периодического получения данных оптики на уровне сети из решения SD-WAN, в том числе сведения о конфигурации и статистика использования, связанную с устройством и подключенными цепями. Личные сведения не собираются и не хранятся. Все собранные сведения объединены в расположения Office и подключенные Интернет-каналы. Эта информация поможет корпорации Майкрософт эффективнее и эффективно устранять сообщенные проблемы, связанные с использованием служб и приложений Microsoft 365.

>[!NOTE]
>Microsoft 365 сообщила о маршрутизации в сети для поддержки клиентов в приложении WW для коммерческого облака, но не в облаках GCC умеренный, GCC High, DoD, Германия или Китае.

## <a name="requirements"></a>Требования

### <a name="integrated-sd-wan-solutions"></a>Интегрированные решения SD и глобальной сети

Корпорация Майкрософт работает с различными участниками, чтобы обеспечить интеграцию с Microsoft 365 с уведомлением о маршрутизации в сети. В настоящее время поддерживаются следующие решения:

| Device Maker | Имя решения | Минимальная версия |
| --- | --- | --- |
| Cisco | [IOS XE SD — WAN](https://go.microsoft.com/fwlink/?linkid=2151460) | 20.4/17.4 |

### <a name="network-topology"></a>топологию сети; 

Информированный сетевой маршрут определяет трафик, связанный с определенным расположением Office и Интернет-цепью, на основе общедоступного IP-адреса, используемого для отправки сетевого трафика в корпорацию Майкрософт. 

Если по крайней мере одна сетевая цепь предоставляет прямой доступ к Интернету в расположении филиала, то маршрутизация по сети может не дать существенных значений.

### <a name="application-usage"></a>Использование приложений

Данные о взаимодействии приложений (отражается в метриках качества сети) собираются с помощью Microsoft Outlook на устройствах под управлением Windows, Teams, SharePoint и OneDrive. При оценке работоспособности сетевой цепи не учитывается и другой трафик приложения.

## <a name="enabling-informed-network-routing"></a>Включение информирования о маршрутизации в сети

Включение информирования о маршрутизации в сети требует нескольких действий, некоторые из которых необходимо выполнить в интерфейсе конфигурации решения SD — WAN. Обратитесь к поставщику решений SD для глобальной сети за инструкциями по запуску процесса информирования о сети в решении SD/WAN перед продолжением настройки в центре администрирования Microsoft 365.

После того как вы будете готовы к включению информирования о маршрутизации в сети в центре администрирования Microsoft 365, убедитесь, что у вас есть необходимые разрешения глобального администратора.

>[!IMPORTANT]
>Чтобы обеспечить необходимые разрешения на уровне клиента для выбранного решения SD/WAN для доступа к информированному каналу обмена данными маршрутизации в сети, необходимо выполнить следующие действия в качестве глобального администратора.


### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1: Открытие параметров конфигурации решения SD для глобальной сети

В [центре администрирования Microsoft 365](https://admin.microsoft.com/)выберите элемент **работоспособность > сетевое подключение** в левой области навигации.

В этом разделе центра администрирования представлены совокупные метрики сетевого подключения для Организации и рекомендации по повышению качества подключения. Для получения дополнительных сведений об этих функциях, доступных в центре администрирования, обратитесь к разделу [сетевое подключение в центре администрирования Microsoft 365 (Предварительная версия)](office-365-network-mac-perf-overview.md) .

Выберите **параметры > решение SD по глобальной** сети, чтобы открыть область информирования о конфигурации маршрутизации в сети. Другие параметры, которые отображаются в разделе **Параметры** , применимы к общим рекомендациям по подключению к сети в центре администрирования и не требуют включения информирования о маршрутизации в сети.

В области Конфигурация выберите **Добавить решение SD-WAN (Предварительная версия)**.

### <a name="step-2-select-your-sd-wan-solution-and-data-storage-location"></a>Шаг 2: выбор решения SD для глобальной сети и места хранения данных

В раскрывающихся списках выберите решение SD — WAN, которое вы развернули, и расположение, в котором будут сопоставлены данные, которые должны быть сопоставлены с уведомлением о сети. Дополнительные сведения можно найти в разделе [Data Storage](#data-storage) .

Нажмите кнопку **Далее**.

### <a name="step-3-accept-terms-for-sharing-of-data"></a>Шаг 3: принятие условий для совместного использования данных

Внимательно прочтите и подтвердите указанные термины, связанные с обменом данными между корпорацией Майкрософт и выбранным решением SD-WAN, а затем установите флажок указанный флажок.

Нажмите кнопку **Далее**.

### <a name="step-4-grant-permissions-to-the-sd-wan-solution"></a>Шаг 4: предоставление разрешений для решения SD/WAN

На этом этапе запустится запрос на предоставление разрешений с помощью Azure Active Directory (Azure AD). Вам будет предложено предоставить разрешения на уровне клиента, которые позволяют выбранному решению SD/WAN получать информированное хранилище данных о маршрутизации в сети и сведения о работоспособности службы, связанные с клиентом. Для этого действия требуются разрешения роли глобального администратора.

Выберите ссылку **предоставить разрешение для этого приложения** и следуйте запросам Azure AD.

Завершив предоставление разрешений, нажмите кнопку **Далее**.

### <a name="step-5-confirm-your-configuration-settings"></a>Шаг 5: Подтверждение параметров конфигурации

Последним шагом при включении информирования о сети для вашего клиента является страница подтверждения, в которой отображаются предоставленные параметры. 

Теперь для клиента включена маршрутизация сети.

Нажмите кнопку **Готово** , а затем закройте область Конфигурация решения SD/WAN.

## <a name="configuring-network-informed-routing"></a>Настройка маршрутизации в сети

В рамках решения по подключению к глобальной сети (SD) можно выполнить значительную часть конфигурации для информирования о маршрутизации сети, например настроить способ маршрутизации трафика в нормальных условиях и альтернативные пути, которые следует использовать при обнаружении проблем. Обратитесь к поставщику решений SD с глобальной сетью для получения подробных сведений об этих действиях по настройке.

Каждое место в офисе должно быть настроено в центре администрирования Microsoft 365, чтобы сообщить о маршрутизации сети правильно определить трафик, связанный с сетевыми цепями, обеспечивающими подключение к этим расположениям.

Расположения Office могут быть автоматически обнаружены в рамках непрерывной коллекции телеметрии сети Майкрософт. В результате некоторые расположения могут быть предварительно заполнены в центре администрирования клиента. 

Если эти расположения являются точными, необходимо включить функцию маршрутизации в сети для каждого места и настроить Интернет-каналы и их общедоступные IP-адреса. 

Если автоматически обнаруженные расположения неточны или в клиенте нет расположений, предварительно заполненных в клиенте, необходимо добавить или изменить расположения вручную в соответствии с точной топологией Организации.

### <a name="updating-locations"></a>Обновление расположений

Расположения клиента можно найти на вкладке " **расположения** ". Расположения могут редактироваться непосредственно в списке или обновляться с помощью CSV-файла.

Убедитесь, что в этом списке есть все расположения Office, в которых вы хотите включить маршрутизацию по сети.

>[!NOTE]
>Столбцы в списке **расположений** для собранных образцов и других данных, связанных с оценкой, не относятся к сообщению о маршрутизации в сети.

### <a name="enabling-a-location-for-informed-network-routing"></a>Включение расположения для информирования о маршрутизации в сети

1. В списке **расположения** выберите команду **изменить** в меню Быстрые действия, чтобы открыть область Конфигурация местоположения.

2. **В этом расположении выберите использовать Microsoft 365 с уведомлением о маршрутизации в сети**.

3. Добавьте все сетевые каналы, обеспечивающие подключение к этому расположению Office в разделе **диапазоны IP-адресов исходящих IP-адресов в этом разделе Расположение Office** . Убедитесь, что каждая цепь связана с уникальными подсетями общедоступного IP-адреса, представляющими сетевой трафик.

4. Нажмите кнопку **Сохранить**, чтобы сохранить изменения.

## <a name="disabling-network-informed-routing"></a>Отключение информирования маршрутизации в сети

Функция маршрутизации в виде информированной сети может быть отключена для всего клиента путем сброса параметров решения SD/WAN. Несмотря на то, что это остановит всю обработку данных в Microsoft 365, вам также следует отключить маршрутизацию по сети в центре администрирования.

### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1: Открытие параметров конфигурации решения SD для глобальной сети

В [центре администрирования Microsoft 365](https://admin.microsoft.com/) выберите пункт **работоспособность > сетевое подключение** в левой области навигации.

Выберите **параметры > решение SD по глобальной** сети, чтобы открыть область информирования о конфигурации маршрутизации в сети.

В области Конфигурация отображается сводка по настроенному решению SD для глобальной сети.

### <a name="step-2-reset-your-configuration"></a>Шаг 2: сброс конфигурации

В области Конфигурация выберите **Сброс параметров решения SD для глобальной сети**.

Теперь ваши параметры сброшены, а маршрутизация сети была отключена. Вы можете снова включить его в любое время, выполнив действия, описанные в статье [включение информирования о маршрутизации в сети](#enabling-informed-network-routing).

## <a name="data-storage"></a>Хранилище данных

Данные, которыми обмениваются Майкрософт и поставщик решений SD-WAN, хранятся в местоположении хранения данных, выбранном при первоначальном включении маршрутизации в сети. Параметры расположения хранилища данных представляют географические области, содержащие области Microsoft Azure, в которых хранятся данные.

>[!NOTE]
>На этапе предварительной версии единственным доступным местом хранения данных является " **Северная Америка**". Дополнительные места хранения данных будут доступны до общего доступа к сообщению о маршрутизации в сети.

Данные хранятся в этом расположении в течение 30 дней. Если этот ключ отключен, все оставшиеся данные удаляются в течение 30-дневного периода хранения.
