# <a name="microsoft-365-informed-network-routing-preview"></a>Microsoft 365 маршрутику сети (предварительный просмотр)

Информированная маршрутная сеть — это функция, которая интегрирует различные Microsoft 365 приложения с решениями, определенными для сторонних программных сетей (SD-WAN), чтобы оптимизировать и улучшить подключение сети к конечным точкам службы Майкрософт. Оптимизация подключения SD-WAN может привести к улучшению пользовательских возможностей и производительности.

>[!IMPORTANT]
>Microsoft 365 маршрутия сети в настоящее время находится в состоянии предварительного просмотра. Дополнительные сведения об этом предварительном просмотре, включая рекомендации по приему помощи, см. в Microsoft 365 сетевой [маршрутике Public Preview.](https://go.microsoft.com/fwlink/?linkid=2151565)

## <a name="overview"></a>Обзор

Разнонаправленная маршрутивка сети обеспечивает двухнаправленный канал обмена данными между Корпорацией Майкрософт и вашим решением SD-WAN. Для каждого настраиваемого расположения офиса и сети Интернет Корпорация Майкрософт периодически делится отзывами с решением SD-WAN о качестве выбранных Microsoft 365 приложений для сетевого трафика, связанных с каждым конкретным подключением к Интернету. С помощью этой обратной связи решение SD-WAN может принимать разумные действия по восстановлению, Microsoft 365 трафика приложений с помощью альтернативных доступных ссылок. 

Ухудшение качества службы на пути к определенной схеме Интернета, например увеличение задержки или высокая потеря пакета, трудно обнаружить на постоянной основе. Эти деградации могут отрицательно сказываться на пользовательских интерфейсах таких приложений, как Exchange Online, SharePoint, OneDrive и Microsoft Teams. Распространенные симптомы включают медленный поиск Exchange контента, высокое время передачи при взаимодействии с библиотеками SharePoint или OneDrive документов, а также плохое качество вызовов или собраний в Microsoft Teams.

Механизм обратной связи и восстановления в сетевой маршрутике стремится динамически обнаруживать такие проблемы в режиме реального времени и информирует развернутое решение SD-WAN для автоматического восстановления.

Канал обмена данными также используется для периодических приемов данных оптики на уровне сети из решения SD-WAN, включая сведения о конфигурации и статистику использования, связанные с устройством и присоединенными схемами. Персональные данные не собираются и не хранятся. Вся собранная информация агрегируется с расположениями офисов и подключенными интернет-схемами. Эти сведения помогут Корпорации Майкрософт более эффективно и эффективно устранять проблемы с использованием Microsoft 365 и приложений.

>[!NOTE]
>Microsoft 365 маршрутизации сети поддерживает клиентов в облаке WW Commercial, но не GCC, GCC high, DoD, Germany или China clouds.

## <a name="requirements"></a>Требования

### <a name="integrated-sd-wan-solutions"></a>Интегрированные решения SD-WAN

Корпорация Майкрософт работает с различными партнерами, чтобы включить интеграцию с Microsoft 365 сетевой маршрутией. В настоящее время включены следующие решения:

| Производитель устройств | Имя решения | Минимальная версия |
| --- | --- | --- |
| Cisco | [IOS XE SD-WAN](https://go.microsoft.com/fwlink/?linkid=2151460) | 20.4/17.4 |

### <a name="network-topology"></a>топологию сети; 

В настоящее время в маршрутике информированных сетей определяется трафик, связанный с определенным расположением офиса и интернет-схемой на основе общедоступных IP-адресов, используемых для отправки сетевого трафика в Корпорацию Майкрософт. 

В случае, если в филиале не существует хотя бы одной сетевой схемы, обеспечиваемой прямым доступом в Интернет, маршрутная маршрутивка с сетевыми данными может не иметь существенного значения.

### <a name="application-usage"></a>Использование приложения

Данные по опыту работы с приложениями (отраженные в метриках качества сети) собираются с помощью определенных клиентских приложений Майкрософт. Exchange метрики отражают использование клиента Outlook, а также некоторые Outlook Web App использования. SharePoint и OneDrive метрики отражают использование конечных точек SharePoint конечных точек, независимо от клиентского приложения. Teams метрики отражают использование Teams настольного клиента. Другой трафик приложений не рассматривается при оценке состояния сетевой схемы.

## <a name="enabling-informed-network-routing"></a>Включение информированной маршрутики сети

Включение информированной маршрутиации сети требует нескольких действий, некоторые из которых необходимо выполнить в интерфейсе конфигурации вашего решения SD-WAN. Проконсультируйтесь с поставщиком решений SD-WAN, чтобы узнать, как инициировать процесс включения сетевой маршрутизации в решении SD-WAN, прежде чем приступить к настройке в центре администрирования Microsoft 365.

После того как вы будете готовы включить информированную маршрутику сети в центре администрирования Microsoft 365, убедитесь, что у вас есть необходимые разрешения глобального администратора.

>[!IMPORTANT]
>Чтобы предоставить необходимые разрешения на использование приложений на уровне клиента для выбранного решения SD-WAN для доступа к каналу обмена данными по маршрутизам, необходимо выполнить следующие действия в качестве глобального администратора.


### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1. Параметры конфигурации решения SD-WAN

В центре [администрирования Microsoft 365](https://admin.microsoft.com/)выберите подключение **> сети** в области навигации слева.

В этом разделе центра администрирования для организации 2016 года 300 метрики сетевой подключений и рекомендации по улучшению подключения. Дополнительные сведения об этих [функциях,](office-365-network-mac-perf-overview.md) доступных в центре администрирования Microsoft 365 центра администрирования, см. в Microsoft 365 network connectivity.

Выберите **Параметры > SD-WAN для** открытия области конфигурации маршрутов сети. Другие параметры, которые отображаются в **Параметры,** применимы к общим рекомендациям по подключению к сети в центре администрирования и не требуются для подключения к информированным маршрутиям сети.

В области конфигурации выберите **Добавить решение SD-WAN (Предварительная версия).**

### <a name="step-2-select-your-sd-wan-solution-and-data-storage-location"></a>Шаг 2. Выберите решение SD-WAN и расположение хранилища данных

В выпадающих полях выберите развернутые решения SD-WAN и расположение, в котором будут сохранены данные, связанные с сетевой маршрутирой. Дополнительные [сведения см.](#data-storage) в разделе хранение данных.

Нажмите кнопку **Далее**.

### <a name="step-3-accept-terms-for-sharing-of-data"></a>Шаг 3. Принятие условий для общего доступа к данным

Внимательно ознакомьтесь с предоставленными терминами, связанными с обменом данными между Microsoft и выбранным решением SD-WAN, а затем выберите указанный почтовый ящик.

Нажмите кнопку **Далее**.

### <a name="step-4-grant-permissions-to-the-sd-wan-solution"></a>Шаг 4. Предоставление разрешений решению SD-WAN

На этом шаге будет инициироваться запрос на предоставление разрешений с Azure Active Directory (Azure AD). Вам будет попросят предоставить разрешения на уровне клиента, которые позволят выбранному решению SD-WAN получить доступ к хранилищу сведений о маршрутивке сети и сведениям о состоянии обслуживания, связанных с вашим клиентом. Это действие требует глобальных разрешений на роль администратора.

Выберите **ссылку Дать разрешение на эту ссылку приложения** и выполните запросы Azure AD.

После завершения предоставления разрешений выберите **Далее**.

### <a name="step-5-confirm-your-configuration-settings"></a>Шаг 5. Подтверждение параметров конфигурации

Заключительный шаг в создании сетевой маршрутики для клиента — страница подтверждения, отображающая предоставленные параметры. 

В настоящее время для клиента включена проинформа о сетевой маршрутике.

Выберите **Готово,** а затем закрой области конфигурации решения SD-WAN.

## <a name="configuring-network-informed-routing"></a>Настройка сетевой маршрутивки с сообщаемой информацию

В решении SD-WAN будет выполняться большая часть конфигурации для маршрутировки в сети, например настройка маршрутов трафика в обычных условиях и альтернативных путей, которые следует использовать при обнаружении проблем. Подробные сведения об этих шагах конфигурации обратитесь к поставщику решений SD-WAN.

Каждое расположение офиса должно быть настроено в центре администрирования Microsoft 365, чтобы информированная маршрутная сеть правильно определяла трафик, связанный с сетевыми схемами, обеспечивающими подключение к этим расположениям.

Office расположения могут быть автоматически обнаружены в рамках текущей коллекции сетевой телеметрии Корпорации Майкрософт. В результате некоторые расположения могут быть предварительно заполнены в центре администрирования для клиента. 

Если эти расположения являются точными, вам просто необходимо включить информированную функцию маршрутинга сети для каждого нужного расположения и настроить схемы Интернета и их общедоступные IP-адреса. 

Если автоматически обнаруженные расположения не являются точными или в клиенте нет предварительно заселяемого расположения, вам придется вручную добавлять или изменять расположения, чтобы отразить точную топологию организации.

### <a name="updating-locations"></a>Обновление расположения

Расположения для клиента можно найти в вкладке **Locations.** Расположения могут изменяться непосредственно в списке или обновляться с помощью CSV-файла.

Убедитесь, что в этом списке присутствует каждое расположение офиса, в котором необходимо включить информированную маршрутику сети.

>[!NOTE]
>Столбцы в списке **Расположения** для собранных образцов и других сведений, связанных с оценкой, не связаны с функцией маршрутиации сети.

### <a name="enabling-a-location-for-informed-network-routing"></a>Включение расположения для маршрутиации в сети

1. В **списке Расположения** выберите **Изменить** из меню быстрых действий, чтобы открыть области конфигурации расположения.

2. Выберите **использование Microsoft 365 маршрутику сети в этом расположении.**

3. Добавьте все сетевые схемы, обеспечивающие подключение к Интернету, в это расположение офиса в **диапазоне Egress IP-адресов** в этом разделе расположения офиса. Убедитесь, что каждая схема связана с уникальными общедоступными подсетями IP-адресов, представляющими сетевой трафик.

4. Нажмите кнопку **Сохранить**, чтобы сохранить изменения.

## <a name="disabling-network-informed-routing"></a>Отключение сетевой маршрутики

Функция маршрутирования сети может быть отключена для всего клиента путем сброса параметров решения SD-WAN. В то время как это остановит всю обработку данных в Microsoft 365, необходимо также отключить маршрутику сетевых данных в центре администрирования.

### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1. Параметры конфигурации решения SD-WAN

В центре [администрирования Microsoft 365](https://admin.microsoft.com/) **выберите подключение > сети** в области навигации слева.

Выберите **Параметры > SD-WAN для** открытия области конфигурации маршрутов сети.

На области конфигурации показан сводка настроенного в настоящее время решения SD-WAN.

### <a name="step-2-reset-your-configuration"></a>Шаг 2. Сброс конфигурации

В области конфигурации выберите **Сброс параметров решения SD-WAN.**

В настоящее время сброшены параметры и отключена маршрутная сеть. Вы можете повторно включить его в любое время, следуя шагам в [включив информированную маршрутику сети.](#enabling-informed-network-routing)

## <a name="data-storage"></a>Хранилище данных

Данные, обменивающиеся между Корпорацией Майкрософт и поставщиком решений SD-WAN, хранятся в месте хранения данных, выбранном во время начальной возможности сетевой маршрутивки. Параметры расположения хранилища данных представляют географические области, Microsoft Azure регионах, в которых хранятся данные.

>[!NOTE]
>На этапе Предварительного просмотра единственным доступным расположением для хранения данных является **Северная Америка.** Дополнительные места хранения данных станут доступны до общего доступа к информированной маршрутике сети.

Данные сохраняются в этом расположении до 30 дней. При отключении все оставшиеся данные удаляются в этом 30-дневном окне хранения.
