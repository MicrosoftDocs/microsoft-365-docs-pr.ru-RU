# <a name="microsoft-365-informed-network-routing-preview"></a>Маршрутная сеть с учетом информации Microsoft 365 (предварительная версия)

Проинформируемая маршрутизируемая сеть — это функция, которая интегрирует различные приложения Microsoft 365 со сторонними решениями для программно-определяемой сети (SD-WAN), чтобы оптимизировать и улучшить сетевое подключение к конечным точкам служб Майкрософт. Оптимизированная связь SD-WAN может улучшить взаимодействие с пользователем и повысить производительность.

>[!IMPORTANT]
>В настоящее время маршрутия сети с предварительной версией Microsoft 365 находится в состоянии предварительной версии. Дополнительные сведения об этой предварительной версии, включая рекомендации по получении помощи, см. в руководстве по маршрутиации сети с открытой предварительной версией [Microsoft 365.](https://go.microsoft.com/fwlink/?linkid=2151565)

## <a name="overview"></a>Обзор

Маршрутная маршрутка по сети обеспечивает двунаправленный канал общего доступа к данным между корпорацией Майкрософт и вашим решением SD-WAN. Корпорация Майкрософт периодически делится с решением SD-WAN отзывами о качестве выбранных приложений Microsoft 365 для сетевого трафика, связанного с каждой конкретной интернет-каналом, для каждого из настроенных расположений в офисе и интернет-каналов. Используя эту обратную связь, решение SD-WAN может затем принять интеллектуальные действия по восстановлению, перенанося трафик приложений Microsoft 365 по альтернативным доступным ссылкам. 

Снижение качества обслуживания на пути к определенной сети Интернета, например увеличение задержки или высокая потеря пакетов, трудно определить на постоянной основе. Это может отрицательно сказаться на пользовательском интерфейсе таких приложений, как Exchange Online, SharePoint, OneDrive и Microsoft Teams. Распространенными симптомами являются медленный поиск контента Exchange, высокое время передачи при взаимодействии с библиотеками документов SharePoint или OneDrive, а также неудовлетворительные вызовы или качество собраний в Microsoft Teams.

Механизм обратной связи и восстановления в сетевой маршрутике с информированием направлен на динамическое обнаружение таких проблем почти в режиме реального времени и информирует развернутое решение SD-WAN о том, что необходимо принять меры по автоматическому восстановлению.

Канал общего доступа к данным также используется для периодического получения данных оптики на уровне сети из решения SD-WAN, включая сведения о конфигурации и статистику использования, связанную с устройством и подключенными каналами. Персональные данные не собираются и не хранятся. Вся собранная информация агрегируется по расположениям офисов и подключенным интернет-схемам. Эти сведения могут помочь корпорации Майкрософт более эффективно и эффективно устранять проблемы, о чем сообщается, при использовании служб и приложений Microsoft 365.

>[!NOTE]
>Маршрутизации сети Microsoft 365 с поддержкой клиентов в коммерческом облаке WW, но не в облаке GCC Moderate, GCC High, DoD, Germany или Китае.

## <a name="requirements"></a>Requirements

### <a name="integrated-sd-wan-solutions"></a>Интегрированные решения SD-WAN

Корпорация Майкрософт совместно с различными партнерами обеспечивает интеграцию с маршрутией сети с учетом информации о Microsoft 365. В настоящее время включены следующие решения:

| Device Maker | Имя решения | Минимальная версия |
| --- | --- | --- |
| Cisco | [IOS XE SD-WAN](https://go.microsoft.com/fwlink/?linkid=2151460) | 20.4/17.4 |

### <a name="network-topology"></a>топологию сети; 

В настоящее время при маршрутиации в сети определяется трафик, связанный с определенным расположением офиса и интернет-каналом, на основе общего IP-адреса, используемого для отправки сетевого трафика в корпорацию Майкрософт. 

Если в филиале нет по крайней мере одной сетевой схемы, предоставляющая прямой доступ к Интернету, маршрутная маршрутная маршрутка с данными по сети может не иметь значительного значения.

### <a name="application-usage"></a>Использование приложений

Данные о работе с приложениями (отражаются с помощью метрик качества сети) собираются с помощью Microsoft Outlook на устройствах под управлением Windows, Teams, SharePoint и OneDrive. При оценке состояния сетевой сети другой трафик приложений не рассматривается.

## <a name="enabling-informed-network-routing"></a>Включение маршрутов в сети с использованием информации

Для включения маршрутов в сети с учетом информации требуется выполнить несколько действий, некоторые из которых необходимо выполнить в интерфейсе конфигурации решения SD-WAN. Обратитесь к поставщику решения SD-WAN, чтобы получить инструкции по инициации процесса включения сетевой маршрутизации в решении SD-WAN, прежде чем приступить к настройке в Центре администрирования Microsoft 365.

После того как вы будете готовы включить маршрутику по сети в Центре администрирования Microsoft 365, убедитесь, что у вас есть необходимые разрешения глобального администратора.

>[!IMPORTANT]
>Чтобы предоставить необходимые разрешения приложений на уровне клиента для выбранного решения SD-WAN на доступ к каналу общего доступа к данным маршрутной маршрутки в сети, необходимо выполнить следующие действия от глобального администратора.


### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1. Открытие параметров конфигурации решения SD-WAN

В Центре [администрирования Microsoft 365](https://admin.microsoft.com/)выберите "Health **> Network connectivity" (Подключение** к сети) в области навигации слева.

В этом разделе Центра администрирования приводится сводная метрика сетевого подключения для вашей организации и рекомендации по улучшению подключения. Дополнительные сведения об этих функциях, доступных в Центре администрирования, см. в центре администрирования [Microsoft 365 (предварительная](office-365-network-mac-perf-overview.md) версия).

Выберите **параметры > SD-WAN,** чтобы открыть области конфигурации маршрутов сети. Другие параметры,  которые отображаются в параметрах, применимы к общему руководству по сетевому подключению в Центре администрирования и не требуются для обеспечения маршрутации по сети.

В области конфигурации выберите "Добавить **решение SD-WAN (предварительная версия)".**

### <a name="step-2-select-your-sd-wan-solution-and-data-storage-location"></a>Шаг 2. Выбор решения SD-WAN и расположения для хранения данных

В выпадающих полях выберите развернутые решения SD-WAN и расположение, в котором будут храниться данные, связанные с маршрутирой с данными о сети. Дополнительные [сведения см.](#data-storage) в разделе хранилища данных.

Нажмите кнопку **Далее**.

### <a name="step-3-accept-terms-for-sharing-of-data"></a>Шаг 3. Принятие условий для общего доступа к данным

Внимательно прочитайте и подтвердите предоставленные условия, связанные с совместным использованием данных между Корпорацией Майкрософт и выбранным решением SD-WAN, а затем выберите указанный контрольный ящик.

Нажмите кнопку **Далее**.

### <a name="step-4-grant-permissions-to-the-sd-wan-solution"></a>Шаг 4. Предоставление разрешений для решения SD-WAN

На этом этапе будет инициирован запрос на предоставление разрешений с помощью Azure Active Directory (Azure AD). Вам будет запросят предоставить разрешения на уровне клиента, которые позволяют выбранному решению SD-WAN получать доступ к хранилищу данных маршрутной маршрутки в сети, а также сведения о состоянии службы, связанные с клиентом. Для этого действия требуются разрешения роли глобального администратора.

Выберите **ссылку "Предоставить разрешение для этого** приложения" и следуйте запросам Azure AD.

После предоставления разрешений выберите **"Далее".**

### <a name="step-5-confirm-your-configuration-settings"></a>Шаг 5. Подтверждение параметров конфигурации

Последний шаг по включите маршрутику с использованием сети для клиента — это страница подтверждения, на которую будут отображаться предоставленные параметры. 

Теперь для клиента включена маршрутная маршрутка по сети.

Выберите **"Готово",** а затем закроем области конфигурации решения SD-WAN.

## <a name="configuring-network-informed-routing"></a>Настройка сетевой маршрутной маршрутки

В решении SD-WAN будет выполняться большая часть конфигурации маршрутиации сети в рамках решения SD-WAN, например настройка маршрута трафика в обычных условиях и альтернативные пути, которые следует использовать при обнаружении проблем. Подробные сведения об этих действиях по настройке можно получить у поставщика решений SD-WAN.

Каждое расположение офиса должно быть настроено в Центре администрирования Microsoft 365, чтобы маршруты в сети могли правильно определять трафик, связанный с сетевыми каналами, обеспечивающими подключение к этим расположениям.

Расположения Office могут быть автоматически обнаружены как часть текущей коллекции телеметрии сети корпорации Майкрософт. В результате некоторые расположения могут быть предварительно заполнены в Центре администрирования вашего клиента. 

Если эти расположения точны, необходимо просто включить функцию маршрутации сети в зависимости от нужного расположения и настроить интернет-цепи и их общедоступные IP-адреса. 

Если автоматически обнаруженные расположения не точны или в клиенте нет предварительно заполненных местоположений, вам придется добавить или изменить расположения вручную, чтобы отразить точную топологию организации.

### <a name="updating-locations"></a>Обновление местоположений

Расположения клиента можно найти на вкладке **"Расположения".** Расположения могут изменяться непосредственно в списке или обновляться с помощью CSV-файла.

Убедитесь, что в этом списке присутствует каждое расположение офиса, в котором вы хотите включить маршрутику в сети.

>[!NOTE]
>Столбцы в списке **"Расположения"** для собранных примеров и других сведений, связанных с оценкой, не связаны с функцией маршрутации в сети.

### <a name="enabling-a-location-for-informed-network-routing"></a>Включение расположения для маршрутки в сети с использованием информации

1. В **списке "Расположения"** выберите **"Изменить"** в меню быстрых действий, чтобы открыть области конфигурации расположения.

2. Выберите "Использовать маршрутику сети с учетом **информации Microsoft 365" в этом расположении.**

3. Добавьте все сетевые сети, обеспечивающие подключение к Интернету, в это расположение офиса в диапазонах IP-адресов **egress** в этом разделе расположения офиса. Убедитесь, что каждая схема связана с уникальными подсетями общедоступных IP-адресов, представляющими сетевой трафик.

4. Нажмите кнопку **Сохранить**, чтобы сохранить изменения.

## <a name="disabling-network-informed-routing"></a>Отключение сетевой маршрутной маршрутки

Функция маршрутки в сети может быть отключена для всего клиента путем сброса параметров решения SD-WAN. Хотя это остановит всю обработку данных в Microsoft 365, вы также должны отключить маршрутику с учетом сетевых данных в Центре администрирования.

### <a name="step-1-open-sd-wan-solution-configuration-options"></a>Шаг 1. Открытие параметров конфигурации решения SD-WAN

В Центре [администрирования Microsoft 365](https://admin.microsoft.com/) выберите **"Health > Network connectivity" (Подключение** к сети) в области навигации слева.

Выберите **параметры > SD-WAN,** чтобы открыть области конфигурации маршрутов сети.

В области конфигурации показана сводка настроенного в настоящее время решения SD-WAN.

### <a name="step-2-reset-your-configuration"></a>Шаг 2. Сброс конфигурации

В области конфигурации выберите **"Сброс параметров решения SD-WAN".**

Теперь параметры были сброшены, а маршрутная маршрутка по сети отключена. Вы можете повторно включить его в любое время, вы следуя шагам, которые необходимо включить в маршруте [по](#enabling-informed-network-routing)сети.

## <a name="data-storage"></a>Хранилище данных

Данные, обмениваются данными между корпорацией Майкрософт и поставщиком решений SD-WAN, хранятся в расположении хранения данных, выбранном во время первоначальной реализации сетевой маршрутной маршрутации. Параметры расположения для хранения данных представляют географические области, содержащие регионы Microsoft Azure, в которых хранятся данные.

>[!NOTE]
>На этапе предварительного просмотра единственным доступным местом хранения данных является **Северная Америка.** Дополнительные места хранения данных станут доступны до общедоступной маршрутиации по сети.

Данные сохраняются в этом расположении в течение 30 дней. Если этот срок отключен, все оставшиеся данные удаляются в течение 30-дневного времени хранения.
