# <a name="plan-for-third-party-ssl-certificates-for-microsoft-365"></a>Планирование сторонних SSL-сертификатов для Microsoft 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Чтобы шифровать связь между клиентами и средой Microsoft 365, на серверах инфраструктуры должны быть установлены сторонние сертификаты SSL.

Эта статья входит в состав планирования [сети и настройки производительности для Microsoft 365.](https://aka.ms/tune)
   
Сертификаты необходимы для следующих компонентов Microsoft 365:
  
- Локальная среда Exchange
    
- Единый вход (SSO) (для серверов федерации служб федерации Active Directory (AD FS) и прокси-серверов федерации AD FS)
    
- Службы Exchange Online, такие как служба автообнаружия, outlook Anywhere и веб-службы Exchange
    
- Гибридный сервер Exchange
    
## <a name="certificates-for-exchange-on-premises"></a>Сертификаты для локального exchange

Обзор использования цифровых сертификатов для обеспечения безопасности обмена данными между локальной организацией Exchange и Exchange Online см. в статье TechNet "Общие сведения о требованиях к [сертификатам".](https://go.microsoft.com/fwlink/p/?LinkID=243657)
  
## <a name="certificates-for-single-sign-on"></a>Сертификаты для единого входа

Чтобы предоставить пользователям упрощенное единое входное решение с надежной безопасностью, сертификаты, показанные в следующей таблице, требуются на серверах федерации или прокси-серверах федерации. В приведенной ниже таблице основное внимание уделяется службам федерации Active Directory (AD FS), а также дополнительные сведения об использовании сторонних [поставщиков удостоверений.](https://docs.microsoft.com/azure/active-directory/hybrid/how-to-connect-fed-compatibility)
  
| Тип сертификата | Description | Что необходимо знать перед развертыванием |
|:-----|:-----|:-----|
|**SSL-сертификат (также называемый сертификатом проверки подлинности сервера)** <br/> |Это стандартный SSL-сертификат, используемый для обеспечения безопасности взаимодействия между серверами федерации, клиентами и прокси-серверами федерации.  <br/> |AD FS требует SSL-сертификата. По умолчанию AD FS использует SSL-сертификат, настроенный для веб-сайта по умолчанию в службах IIS.  <br/> Имя субъекта этого SSL-сертификата используется для определения имени службы федерации (FS) для каждого развернутого экземпляра AD FS. Рассмотрите возможность выбора имени субъекта для любых новых сертификатов, выданных ЦС, которые лучше всего представляют имя вашей компании или организации для Microsoft 365. Это имя должно быть маршрутизоваемым через Интернет.  <br/>**Внимание!** AD FS требует, чтобы у этого SSL-сертификата не было точечного (короткого имени) имени субъекта.          <br/> **Рекомендация:** Так как клиенты AD FS должны доверять этому сертификату, рекомендуется использовать SSL-сертификат, выданный общедоступным (сторонним) ЦС или ЦС, подчиненным публично доверенному корню; например, VeriSign или Thawte.  <br/> |
|**Сертификат для подписи маркера** <br/> |Это стандартный сертификат X.509, используемый для безопасной подписывания всех маркеров, выдававаемой сервером федерации, и который Microsoft 365 принимает и проверяет.  <br/> |Сертификат для подписи маркера должен содержать закрытый ключ, цепочек к доверенного корня в FS. По умолчанию AD FS создает самозаверя сертификат. Однако в зависимости от потребностей организации вы можете изменить этот сертификат на сертификат, выданный ЦС, с помощью оснастки управления AD FS.  <br/>**Внимание!** Сертификат для подписи маркера критически важен для стабильности FS. Если сертификат изменен, Microsoft 365 должен быть уведомлен об изменении. Если уведомление не предоставлено, пользователи не смогут войти в свои службы Microsoft 365.<br/>**Рекомендация:** Рекомендуется использовать самозаверяемый сертификат для подписи маркеров, созданный AD FS. Таким образом он управляет этим сертификатом за вас по умолчанию. Например, когда срок действия этого сертификата истекает, AD FS создает самозаверя сертификат.  <br/> |
   
Прокси-серверы федерации требуют сертификата, описанного в следующей таблице.
  
| Тип сертификата | Description | Что необходимо знать перед развертыванием |
|:-----|:-----|:-----|
|Сертификат SSL  <br/> |Это стандартный SSL-сертификат, используемый для защиты связи между сервером федерации, прокси-сервером федерации и клиентских компьютерами Интернета.  <br/> |Этот SSL-сертификат должен быть привязан к веб-сайту по умолчанию в IIS, прежде чем можно будет успешно запустить мастер настройки прокси-сервера федерации AD FS.  <br/> Этот сертификат должен иметь то же имя субъекта, что и SSL-сертификат, настроенный на сервере федерации в корпоративной сети.  <br/> **Рекомендация:** Рекомендуется использовать тот же сертификат проверки подлинности сервера, который настроен на сервере федерации, к который подключается прокси-сервер федерации.  <br/> |
   
## <a name="certificates-for-autodiscover-outlook-anywhere-and-active-directory-synchronization"></a>Сертификаты для автооружения, outlook Anywhere и синхронизации Active Directory

Для внешних серверов клиентского доступа Exchange 2013, Exchange 2010, Exchange 2007 и Exchange 2003 требуется сторонний SSL-сертификат для безопасных подключений для служб автообнаружения, outlook Anywhere и синхронизации Active Directory. Возможно, этот сертификат уже установлен в локальной среде.
  
## <a name="certificate-for-an-exchange-hybrid-server"></a>Сертификат для гибридного сервера Exchange

Для безопасного подключения к службе Exchange Online для внешнего гибридного сервера Или серверов Exchange требуется сторонний SSL-сертификат. Этот сертификат необходимо получить от стороного поставщика SSL.
