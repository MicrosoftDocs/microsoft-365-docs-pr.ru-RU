# <a name="plan-for-third-party-ssl-certificates-for-microsoft-365"></a>Планирование сторонних SSL-сертификатов для Microsoft 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Для шифрования сообщений между клиентами и средой Microsoft 365 на серверах инфраструктуры необходимо установить сторонние сертификаты Secure Socket Layer (SSL).

Эта статья является частью [планирования сети и настройки производительности для Microsoft 365](./network-planning-and-performance.md).
   
Сертификаты необходимы для следующих компонентов Microsoft 365:
  
- Локальная среда Exchange
    
- Единый вход (SSO) (как для серверов федерации Active Directory Federation Services (AD FS), так и для прокси-серверов федерации федерации AD FS)
    
- Службы Exchange Online, такие как Автообнаружить, Outlook Anywhere и Веб-службы Exchange
    
- Гибридный сервер Exchange
    
## <a name="certificates-for-exchange-on-premises"></a>Сертификаты для локальной биржи Exchange

Обзор использования цифровых сертификатов для обеспечения безопасности связи между локальной организацией Exchange и Exchange Online см. в статье TechNet [Understanding Certificate Requirements.](/previous-versions/exchange-server/exchange-141/gg476123(v=exchg.141))
  
## <a name="certificates-for-single-sign-on"></a>Сертификаты для единого входа

Чтобы предоставить пользователям упрощенный единый вход, который включает надежную безопасность, сертификаты, показанные в следующей таблице, необходимы либо на серверах федерации, либо на прокси-серверах федерации. В приведенной ниже таблице основное внимание уделяется службам Федерации Active Directory (AD FS), а также дополнительные сведения об использовании сторонних [поставщиков удостоверений.](/azure/active-directory/hybrid/how-to-connect-fed-compatibility)
  
| Тип сертификата | Описание | Что нужно знать перед развертыванием |
|:-----|:-----|:-----|
|**Сертификат SSL (также называемый сертификатом проверки подлинности сервера)** <br/> |Это стандартный сертификат SSL, который используется для обеспечения безопасности связи между серверами федерации, клиентами и прокси-компьютерами федерации.  <br/> |AD FS требует сертификатА SSL. По умолчанию AD FS использует SSL-сертификат, настроенный для веб-сайта по умолчанию в службах интернет-информации (IIS).  <br/> Имя субъекта этого сертификата SSL используется для определения имени службы федерации (FS) для каждого экземпляра AD FS, развернутого вами. Рассмотрите возможность выбора имени субъекта для любых новых сертификатов, выдающих сертификаты ca, которые лучше всего представляют имя вашей компании или организации для Microsoft 365. Это имя должно быть internet-routable.  <br/>**Внимание:** AD FS требует, чтобы у этого сертификата SSL не было безточного (короткое имя) имени субъекта.          <br/> **Рекомендация:** Поскольку этому сертификату должны доверять клиенты AD FS, рекомендуется использовать SSL-сертификат, выданный общедоступным (сторонним) ЦС или ЦС, подчиненным публично доверяемой корневой компании; например, VeriSign или Thawte.  <br/> |
|**Сертификат подписи маркеров** <br/> |Это стандартный сертификат X.509, который используется для безопасного подписания всех маркеров, которые выдают сервер федерации и которые принимает и проверяет Microsoft 365.  <br/> |Сертификат подписывания маркеров должен содержать закрытый ключ, который цепится с доверенным корнем в FS. По умолчанию AD FS создает самозаверяется сертификат. Однако в зависимости от потребностей организации этот сертификат можно изменить на сертификат, выданный ЦС, с помощью оснастки управления AD FS.  <br/>**Внимание:** Сертификат подписывания маркеров имеет решающее значение для стабильности FS. Если сертификат изменен, microsoft 365 должен быть уведомлен об изменении. Если уведомление не предоставлено, пользователи не могут войти в свои предложения службы Microsoft 365.<br/>**Рекомендация:** Рекомендуется использовать самозаверяемый сертификат подписи маркеров, созданный AD FS. Таким образом, он управляет этим сертификатом для вас по умолчанию. Например, когда срок действия этого сертификата истекает, AD FS будет создавать новый самозаверяется сертификат.  <br/> |
   
Прокси-серверы федерации требуют сертификат, описанный в следующей таблице.
  
| Тип сертификата | Описание | Что нужно знать перед развертыванием |
|:-----|:-----|:-----|
|Сертификат SSL  <br/> |Это стандартный сертификат SSL, который используется для обеспечения связи между сервером федерации, прокси-сервером федерации и компьютерами клиентов Интернета.  <br/> |Этот сертификат SSL должен быть привязан к веб-сайту по умолчанию в IIS, прежде чем вы сможете успешно запустить мастер конфигурации прокси-сервера федерации федерации AD FS.  <br/> Этот сертификат должен иметь то же имя субъекта, что и сертификат SSL, настроенный на сервер федерации в корпоративной сети.  <br/> **Рекомендация:** Рекомендуется использовать тот же сертификат проверки подлинности сервера, который настроен на сервере федерации, к который подключается этот прокси-сервер федерации.  <br/> |
   
## <a name="certificates-for-autodiscover-outlook-anywhere-and-active-directory-synchronization"></a>Сертификаты для автоматического поиска, синхронизации Outlook Anywhere и Active Directory

На внешних серверах Exchange 2013, Exchange 2010, Exchange 2007 и Exchange 2003 клиентского доступа (CASS) требуется сторонний сертификат SSL для безопасных подключений для служб автообнаружения, Outlook Anywhere и Active Directory. Возможно, этот сертификат уже установлен в локальной среде.
  
## <a name="certificate-for-an-exchange-hybrid-server"></a>Сертификат гибридного сервера Exchange

Для безопасного подключения к службе Exchange Online вашему гибридному серверу Или серверам Exchange требуется сторонний SSL-сертификат. Этот сертификат необходимо получить у сторонного поставщика SSL.
