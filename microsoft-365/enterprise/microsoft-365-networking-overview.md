# <a name="microsoft-365-network-connectivity-overview"></a>Обзор сетевого подключения Microsoft 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft 365 это распределенное программное обеспечение, которое обеспечивает производительность и совместную работу с помощью разнообразных служб и приложений. Клиентские компоненты Microsoft 365, такие как Outlook, Word и PowerPoint, выполняются на компьютерах пользователей и подключаются к другим компонентам Microsoft 365, которые работают в центрах обработки данных Майкрософт. Наиболее важный фактор, определяющий качество взаимодействия с пользователем Microsoft 365, — это надежность сети и небольшая задержка между клиентами Microsoft 365 и передними дверцами службы Microsoft 365.

В этой статье рассказывается о задачах сети Microsoft 365 и о том, почему для сети Microsoft 365 требуется другой подход к оптимизации по сравнению с общим Интернет-трафиком.

## <a name="microsoft-365-networking-goals"></a>Сетевые цели Microsoft 365

Конечная цель Microsoft 365 Network — оптимизация работы конечных пользователей, позволяющая наименее ограниченный доступ между клиентами и ближайшими конечными точками Microsoft 365. Качество работы конечных пользователей напрямую связано с производительностью и временем отклика приложения, которое использует пользователь. Например, Microsoft Teams полагается на низкую задержку, чтобы телефонные звонки, Конференции и совместное взаимодействие с экраном не выключались без сбоев, а Outlook использует беспроводное подключение для функций мгновенного поиска, которые используют индексирование на стороне сервера и возможности AI.

Основная цель структуры сети — уменьшить задержку, уменьшая время кругового пути с клиентских компьютеров в глобальную сеть Майкрософт, магистраль сети общедоступной сети корпорации Майкрософт, которая соединяет все центры обработки данных Майкрософт с небольшой задержкой, точки входа облачных приложений высокого уровня доступности, распределенные по всему миру. Дополнительные сведения о глобальной сети Майкрософт см. в статье [Как Майкрософт строит свою быструю и надежную глобальную сеть](https://azure.microsoft.com/blog/how-microsoft-builds-its-fast-and-reliable-global-network/).

Оптимизация производительности сети Microsoft 365 не должна быть сложной. Вы можете получить наилучшую производительность, выполнив несколько ключевых принципов:

- Определение сетевого трафика Microsoft 365
- Разрешить исходящий трафик локальной ветви сетевого трафика Microsoft 365 в Интернет из каждого места, где пользователи подключаются к Microsoft 365
- Разрешить трафику Microsoft 365 пропускать прокси-серверы и устройства проверки пакетов

Для получения дополнительных сведений о принципах сетевого подключения Microsoft 365, обратитесь к разделу [принципы сетевого подключения microsoft 365](microsoft-365-network-connectivity-principles.md).

## <a name="traditional-network-architectures-and-saas"></a>Традиционные архитектуры сети и SaaS

Традиционные принципы архитектуры сети для рабочих нагрузок клиентов и серверов разрабатываются в предположении, что трафик между клиентами и конечными точками не выходит за границы сети Организации. Кроме того, во многих корпоративных сетях все исходящие подключения к Интернету проходят через корпоративную сеть и отправляются из центрального расположения.

В традиционных сетевых архитектурах большая задержка для универсального Интернет-трафика является необходимым компромиссом для обеспечения безопасности периметра сети, а оптимизация производительности для трафика Интернета обычно включает обновление или масштабирование оборудования в точках выхода сети. Однако этот подход не выполняет требования к оптимальной производительности сети для служб SaaS, таких как Microsoft 365.

## <a name="identifying-microsoft-365-network-traffic"></a>Идентификация сетевого трафика Microsoft 365

Мы упростили идентификацию сетевого трафика Microsoft 365 и упрощает управление идентификацией сети.

- Новые категории конечных точек сети, чтобы отличать большой критический сетевой трафик от сетевого трафика, не влияющие на задержки в Интернете. Существует всего несколько URL-адресов и поддерживаемых IP-адресов в самой критической категории "оптимизировать".
- Веб-службы для использования сценариев или прямого конфигурирования устройств, а также управления изменениями идентификации сети Microsoft 365. Изменения доступны в веб-службе, в формате RSS или в электронной почте с помощью шаблона Microsoft Flow.
- [Партнерская программа для сетевых партнеров Office 365](https://aka.ms/Office365NPP) с партнерами корпорации Майкрософт, предоставляющих устройства и службы, которые соответствуют принципам сетевого подключения Microsoft 365 и имеют простую конфигурацию.

## <a name="securing-microsoft-365-connections"></a>Защита подключений Microsoft 365

Цель традиционной системы сетевой безопасности — укрепить защиту периметра корпоративной сети от вторжений и вредоносных эксплойтов. Большинство корпоративных сетей применяют безопасность сети для трафика Интернета с помощью таких технологий, как прокси-серверы, брандмауэры, SSL и анализ, глубокая проверка пакетов и защита от потери данных. Эти технологии выполняют важную задачу снижения риска для обычных интернет-запросов, но могут существенно снизить производительность, масштабируемость и удобство работы пользователей в отношении конечных точек Microsoft 365.

Microsoft 365 помогает обеспечить соответствие требованиям Организации требованиям к безопасности контента и использованию данных с помощью встроенных функций безопасности и управления, предназначенных специально для компонентов и рабочих нагрузок Microsoft 365. Для получения дополнительных сведений о безопасности и совместимости Microsoft 365 ознакомьтесь со статьей [план обеспечения безопасности Office 365](https://docs.microsoft.com/office365/securitycompliance/security-roadmap). Для получения дополнительных сведений о положениях и поддержке Майкрософт в расширенных сетевых решениях, выполняющих расширенную обработку трафика Microsoft 365, ознакомьтесь со статьей [Использование сторонних сетевых устройств или решений на трафике Office 365](https://support.microsoft.com/help/2690045).

## <a name="why-is-microsoft-365-networking-different"></a>Почему Microsoft 365 Network отличается?

Microsoft 365 разработана для оптимальной производительности с помощью безопасности конечной точки и зашифрованных сетевых подключений, что снижает потребность в обеспечении безопасности периметра. Центры обработки данных Microsoft 365 располагаются по всему миру, и служба разработана для использования различных способов подключения клиентов к наиболее доступным конечным точкам служб. Так как данные и обработка пользователей распределяются между многими центрами обработки данных Майкрософт, не существует одной конечной точки сети, к которой могут подключаться клиентские компьютеры. Фактически, данные и службы в клиенте Microsoft 365 динамически оптимизируются глобальной сетью Майкрософт для адаптации к географическим расположениям, от которых пользователи получают доступ.

Некоторые распространенные проблемы, связанные с производительностью, создаются при проверке пакетов и централизованной отгрузках трафика Microsoft 365:

- Высокая задержка может вызвать очень низкую производительность видеопотоков и звуковых потоков, а также замедлить отклики на получение данных, поиск, совместную работу в режиме реального времени, сведения о доступности в календаре, содержимом продукта и других службах
- Егрессинг подключения из центрального расположения не допускают возможности динамической маршрутизации для глобальной сети Microsoft 365, Добавление задержки и время кругового приема
- Расшифровка безопасного сетевого трафика Microsoft 365 и повторное шифрование может вызвать ошибки протокола и угрозу безопасности

Сокращение сетевого пути к точкам входа Microsoft 365, позволяя трафику клиента исключаться как можно ближе к их географическому расположению, могут повышать производительность подключения и работать с конечными пользователями в Microsoft 365. Кроме того, она может помочь снизить влияние будущих изменений архитектуры сети на производительность и надежность Microsoft 365. Модель оптимального подключения позволяет всегда предоставлять сетевой выход в расположении пользователя, независимо от того, находится ли он в корпоративной сети или в удаленных расположениях, таких как домашние, Гостиницы, кафе и аэропорты. Общий трафик в Интернете и корпоративный сетевой трафик, основанный на глобальной сети, будут раздельно маршрутизироваться и не используют локальную модель прямого выхода. Такая модель прямого выхода представлена на приведенной ниже схеме.

![Сетевая архитектура с локальными точками выхода](../media/6bc636b0-1234-4ceb-a45a-aadd1044b39c.png)

Архитектура локального выхода имеет следующие преимущества для сетевого трафика Microsoft 365 по сравнению с традиционной моделью:
  
- Обеспечивается оптимальная производительность Microsoft 365 благодаря оптимизации длины маршрутов. Подключения конечных пользователей динамически направляются в ближайшую точку входа Microsoft 365 по инфраструктуре _распределенной службы_ глобальной сети Microsoft, а трафик направляется внутренним образом в конечные точки данных и служб по сравнению с высокой доступностью корпорации Майкрософт.
- Снижает нагрузку на корпоративную инфраструктуру сети, разрешая локальный выход для трафика Microsoft 365, обход прокси-серверов и устройств проверки трафика.
- Обеспечивает безопасность подключений на обоих концах, используя функции безопасности конечной точки клиента и облачные функции безопасности, избегая применения избыточных сетевых технологий безопасности.

> [!NOTE]
> Инфраструктура " _Передняя дверца" распределенной службы_ — это высокодоступный и масштабируемый сетевой край Microsoft Global Network с географически распределенными расположениями. Он завершает подключения конечных пользователей и эффективно передает их в глобальную сеть Майкрософт. Дополнительные сведения о глобальной сети Майкрософт см. в статье [Как Майкрософт строит свою быструю и надежную глобальную сеть](https://azure.microsoft.com/blog/how-microsoft-builds-its-fast-and-reliable-global-network/).

Для получения дополнительных сведений об использовании принципов сетевого подключения Microsoft 365, обратитесь к разделу [принципы сетевого подключения microsoft 365](microsoft-365-network-connectivity-principles.md).

## <a name="conclusion"></a>Заключение

Оптимизация производительности сети Microsoft 365 связана с удалением ненужных препятствий. Разсматривая подключения Microsoft 365 как надежный трафик, вы можете предотвратить задержку проверки пакетов и конкуренцию за пропускную способность прокси-сервера. Разрешение локальных подключений между клиентскими компьютерами и конечными точками Office 365 позволяет динамически маршрутизировать трафик через глобальную сеть Майкрософт.
