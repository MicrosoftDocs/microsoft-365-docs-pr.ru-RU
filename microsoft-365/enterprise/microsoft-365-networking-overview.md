# <a name="microsoft-365-network-connectivity-overview"></a>Обзор сетевого подключения Microsoft 365

*Эта статья относится к Microsoft 365 корпоративный и Office 365 корпоративный.*

Microsoft 365 — это распределенное облако SaaS, которое обеспечивает производительность и совместную работу с помощью разнообразных микрослужб и приложений. Клиентские компоненты Microsoft 365, такие как Outlook, Word и PowerPoint, работают на компьютерах пользователей и подключаются к другим компонентам Microsoft 365, которые работают в центрах обработки данных Майкрософт. Наиболее важным фактором, определяющих качество работы конечных пользователей Microsoft 365, является надежность сети и низкая задержка между клиентами Microsoft 365 и интерфейсными клиентами службы Microsoft 365.

В этой статье вы узнаете о целях работы сети Microsoft 365 и о том, почему сети Microsoft 365 требуют другого подхода к оптимизации, чем общий интернет-трафик.

## <a name="microsoft-365-networking-goals"></a>Цели сети Microsoft 365

Конечная цель сети Microsoft 365 — оптимизировать работу конечных пользователей, включив наименее строгий доступ между клиентами и ближайшими конечными точками Microsoft 365. Качество работы конечного пользователя напрямую связано с производительностью и скоростью отклика приложения, которое использует пользователь. Например, Microsoft Teams полагается на низкую задержку, чтобы телефонные звонки, конференции и совместная работа с общим экраном были без сбоев, а Outlook использует отличные сетевые подключения для функций мгновенного поиска, которые используют возможности индексации на стороне сервера и ИИ.

Основной целью проекта сети должно быть минимизация задержки путем сокращения времени кругового пути (RTT) от клиентских компьютеров к глобальной сети Майкрософт, магистрали общедоступных сетей Майкрософт, которая пересекает все центр обработки данных Майкрософт с низкой задержкой, точками входа в облако с высокой доступностью, распространяемой по всему миру. Дополнительные сведения о глобальной сети Майкрософт см. в статье [Как Майкрософт строит свою быструю и надежную глобальную сеть](https://azure.microsoft.com/blog/how-microsoft-builds-its-fast-and-reliable-global-network/).

Оптимизация производительности сети Microsoft 365 не должна быть сложной. Вы можете получить максимально возможную производительность, следуя нескольким ключевым принципам:

- Определение сетевого трафика Microsoft 365
- Разрешить трафик локальной ветви Microsoft 365 в Интернет из каждого расположения, в котором пользователи подключаются к Microsoft 365
- Разрешить трафику Microsoft 365 обходить прокси-устройства и устройства проверки пакетов

Дополнительные сведения о принципах сетевого подключения Microsoft 365 см. в подсетях [Microsoft 365.](microsoft-365-network-connectivity-principles.md)

## <a name="traditional-network-architectures-and-saas"></a>Традиционные сетевые архитектуры и SaaS

Традиционные принципы сетевой архитектуры для рабочих нагрузок клиентов и серверов разработаны на основе предположения, что трафик между клиентами и конечными точками не распространяется за пределы периметра корпоративной сети. Кроме того, во многих корпоративных сетях все исходящие подключения к Интернету проходят через корпоративную сеть и исходящие из центрального расположения.

В традиционных сетевых архитектурах высокая задержка общего интернет-трафика является необходимым компромиссным решением для поддержания безопасности периметра сети, а оптимизация производительности интернет-трафика обычно включает обновление или масштабирование оборудования в точках сетевого выходного трафика. Однако этот подход не соответствует требованиям для оптимальной производительности сети служб SaaS, таких как Microsoft 365.

## <a name="identifying-microsoft-365-network-traffic"></a>Определение сетевого трафика Microsoft 365

Мы упростим идентификацию сетевого трафика Microsoft 365 и упростим управление сетевым трафиком.

- Новые категории конечных точек сети для различения высококритого сетевого трафика от сетевого трафика, на который не влияют задержки в Интернете. В самой важной категории "Оптимизация" имеется всего несколько URL-адресов и поддерживаемые IP-адреса.
- Веб-службы для использования сценариев или прямой настройки устройств и управления изменениями сетевой идентификации Microsoft 365. Изменения доступны в веб-службе, в формате RSS или по электронной почте с помощью шаблона Microsoft Flow.
- [Партнерской программой Office 365 network](https://aka.ms/Office365NPP) с партнерами Майкрософт, которые предоставляют устройства или службы, которые следуют принципам сетевого подключения Microsoft 365 и имеют простую конфигурацию.

## <a name="securing-microsoft-365-connections"></a>Защита подключений Microsoft 365

Цель традиционной системы сетевой безопасности — укрепить защиту периметра корпоративной сети от вторжений и вредоносных эксплойтов. Большинство корпоративных сетей обеспечивает безопасность сети для интернет-трафика с помощью таких технологий, как прокси-серверы, брандмауэры, разрыв и проверка SSL, глубокая проверка пакетов и системы защиты от потери данных. Эти технологии выполняют важную задачу снижения риска для обычных интернет-запросов, но могут существенно снизить производительность, масштабируемость и удобство работы пользователей в отношении конечных точек Microsoft 365.

Microsoft 365 помогает удовлетворить потребности вашей организации в обеспечении безопасности контента и соответствия использованию данных встроенными функциями безопасности и управления, разработанными специально для функций и рабочих нагрузок Microsoft 365. Дополнительные сведения о безопасности и соответствии требованиям Microsoft 365 см. в плане обеспечения [безопасности Office 365.](https://docs.microsoft.com/office365/securitycompliance/security-roadmap) Дополнительные сведения о рекомендациях корпорации Майкрософт и позиции поддержки в расширенных сетевых решениях, [](https://support.microsoft.com/help/2690045)которые выполняют обработку трафика Microsoft 365 на более высоком уровне, см. в этой теме.

## <a name="why-is-microsoft-365-networking-different"></a>Почему сеть Microsoft 365 отличается?

Microsoft 365 разработан для обеспечения оптимальной производительности с использованием безопасности конечных точек и зашифрованных сетевых подключений, что снижает необходимость в применении безопасности периметра. Центр обработки данных Microsoft 365 расположен по всему миру, и служба разработана для использования различных методов подключения клиентов к наиболее доступным конечным точкам службы. Так как пользовательские данные и обработка распределены между многими центрами обработки данных Майкрософт, нет единой конечной точки сети, к которой клиентские компьютеры могут подключаться. Фактически данные и службы в клиенте Microsoft 365 динамически оптимизированы глобальной сетью Майкрософт для адаптации к географическим расположениям, из которых конечные пользователи могут получать к ним доступ.

Некоторые распространенные проблемы производительности создаются, когда трафик Microsoft 365 подвергается проверке пакетов и централизованному egress:

- Высокая задержка может привести к крайне низкой производительности видео- и аудиопотоков, а также медленному отклику на запросы данных, поиск, совместную работу в режиме реального времени, сведения календаря о занятости, контент продукта и другие службы
- При наступлении подключений из центрального расположения возможности динамической маршрутизации в глобальной сети Microsoft 365 не повлияют на возможности, что добавляет задержку и время кругового пути
- Расшифровка защищенного SSL сетевого трафика Microsoft 365 и повторное шифрование может привести к ошибкам протокола и вызвать угрозу безопасности

Сокращение сетевого пути к точкам входа Microsoft 365, позволяя клиенту как можно ближе к их географическому расположению, повысить производительность подключения и улучшить взаимодействие с конечными пользователями в Microsoft 365. Это также поможет снизить влияние будущих изменений на сетевую архитектуру на производительность и надежность Microsoft 365. Оптимальная модель подключения — всегда обеспечивать сетевой экстренный доступ в расположении пользователя независимо от того, находится ли она в корпоративной сети или в удаленных расположениях, таких как дома, кафе и аэропорт. Общий интернет-трафик и корпоративный сетевой трафик на основе WAN будут маршрутизализируются отдельно и не будут использовать локализованную прямую исходящую модель. Такая модель прямого выхода представлена на приведенной ниже схеме.

![Сетевая архитектура с локальными точками выхода](../media/6bc636b0-1234-4ceb-a45a-aadd1044b39c.png)

Архитектура локального экскурса имеет следующие преимущества для сетевого трафика Microsoft 365 по традиционной модели:
  
- Обеспечивается оптимальная производительность Microsoft 365 благодаря оптимизации длины маршрутов. Подключения конечных пользователей динамически перенаправлются в ближайшую точку  входа Microsoft 365 инфраструктурой переднего входа распределенной службы Microsoft Global Network, а трафик затем внутренне перенаправскуется в конечные точки данных и служб через сверхнизкий оптоволокон высокой доступности корпорации Майкрософт с низкой задержкой.
- Снижает нагрузку на инфраструктуру корпоративной сети, позволяя локально использовать трафик Microsoft 365, минуя прокси и устройства проверки трафика.
- Обеспечивает безопасность подключений на обоих конечных точках путем использования функций безопасности клиентских конечных точек и облачной безопасности, избегая применения избыточных технологий сетевой безопасности.

> [!NOTE]
> Инфраструктура переднего входа _распределенной_ службы является высокодоступной и масштабируемой сетевой границей глобальной сети Майкрософт с географически распределенными расположениями. Он прерывает подключения конечных пользователей и эффективно маршрутирует их в глобальной сети Майкрософт. Дополнительные сведения о глобальной сети Майкрософт см. в статье [Как Майкрософт строит свою быструю и надежную глобальную сеть](https://azure.microsoft.com/blog/how-microsoft-builds-its-fast-and-reliable-global-network/).

Дополнительные сведения о принципах сетевого подключения Microsoft 365 и их применении см. в подсетях [Microsoft 365.](microsoft-365-network-connectivity-principles.md)

## <a name="conclusion"></a>Заключение

Оптимизация производительности сети Microsoft 365 на самом деле сводится к устранению ненужных помех. Обрабатывая подключения Microsoft 365 как надежный трафик, вы можете предотвратить задержку в результате проверки пакетов и состязания за пропускную способность прокси-сервера. Разрешение локальных подключений между клиентских компьютеров и конечными точками Office 365 позволяет динамически маршрутить трафик через глобальную сеть Майкрософт.
