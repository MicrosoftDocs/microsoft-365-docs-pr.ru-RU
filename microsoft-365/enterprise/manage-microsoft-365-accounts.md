# <a name="manage-microsoft-365-user-accounts"></a>Управление учетными записями пользователей Microsoft 365

Вы можете управлять учетными записями пользователей Microsoft 365 несколькими различными способами в зависимости от конфигурации. Вы можете управлять учетными записями пользователей в [центре администрирования Microsoft 365](https://docs.microsoft.com/microsoft-365/admin/add-users/), [PowerShell](manage-user-accounts-and-licenses-with-microsoft-365-powershell.md), в доменных службах Active Directory (AD DS) или на портале администрирования Azure Active Directory (Azure AD). 

Как только вы приобрели Microsoft 365, для управления учетными записями можно использовать центр администрирования Microsoft 365 и PowerShell. При управлении облачными удостоверениями у каждого пользователя в Организации есть отдельная учетная запись и пароль. Если вы хотите интегрировать свою локальную инфраструктуру с учетными записями пользователей, которые синхронизируются с Microsoft 365, вы можете использовать Azure AD Connect для обеспечения синхронизации удостоверений и паролей для функций единого входа (SSO).
  
## <a name="plan-for-where-and-how-you-will-manage-your-user-accounts"></a>Планирование того, где и как будут управляться учетными записями пользователей

Здесь и как можно управлять учетными записями пользователей зависит от модели удостоверений, которую вы хотите использовать для вашей Microsoft 365. Две общие модели являются только облачными и гибридными.
  
### <a name="cloud-only"></a>Только облако

Создавать пользователей и управлять ими можно в центре администрирования Microsoft 365. Вы также можете использовать PowerShell или центр администрирования Azure AD. 
    
### <a name="hybrid"></a>Гибридная среда

Учетные записи пользователей синхронизируются с Microsoft 365 из доменных служб Active Directory, поэтому для управления учетными записями пользователей необходимо использовать локальные средства AD DS. 
    
## <a name="managing-accounts"></a>Управление учетными записями

Принимая решение о том, как Организация будет создавать учетные записи и управлять ими, учитывайте следующие требования:
  
- Для подключения удостоверений между Microsoft 365 и AD DS необходимо установить программное обеспечение синхронизации каталогов на серверах в локальной среде.
    
- Любой вариант синхронизации каталогов, в том числе параметры единого входа, требует, чтобы атрибуты доменных служб Active Directory соответствовали стандартам. Особенности того, какие атрибуты используются в каталоге и какая очистка (если это требуется), описана в [статье Подготовка к синхронизации службы каталогов Microsoft 365](prepare-for-directory-synchronization.md). 
    
- Спланируйте, как вы собираетесь создавать учетные записи Microsoft 365.

- Независимо от того, как вы планируете добавлять учетные записи пользователей в Microsoft 365, вам необходимо управлять несколькими функциями учетной записи, такими как назначение лицензий, указание расположения и т. д. Эти функции можно управлять долгосрочным сроком действия из центра администрирования Microsoft 365, а также [создавать учетные записи пользователей с помощью PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=717083).
    
    Если вы решили добавить всех пользователей и управлять ими с помощью центра администрирования, вы сможете указать расположение и назначить лицензии одновременно с созданием учетной записи Microsoft 365. В результате не нужно ничего планировать.
    
    > [!IMPORTANT]
    > Создание учетных записей в Microsoft 365 без назначения лицензии (например, в SharePoint Online) означает, что владелец учетной записи может просматривать центр Майкрософт 365, но не может получить доступ к службам в подписке компании. После назначения расположения и лицензии учетная запись реплицируется на назначенную службу или службы. Пользователь может войти в свою учетную запись и использовать назначенные им службы.
