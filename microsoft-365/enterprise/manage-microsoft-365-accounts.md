# <a name="manage-microsoft-365-user-accounts"></a>Управление учетной записью пользователя Microsoft 365

Учетные записи пользователей Microsoft 365 можно управлять несколькими различными способами в зависимости от конфигурации. Вы можете управлять учетными записями пользователей в Центре администрирования [Microsoft 365,](https://docs.microsoft.com/microsoft-365/admin/add-users/) [PowerShell,](manage-user-accounts-and-licenses-with-microsoft-365-powershell.md)доменных службах Active Directory (AD DS) или на портале администрирования Azure Active Directory (Azure AD). 

Как только вы приобретаете Microsoft 365, Центр администрирования Microsoft 365 и PowerShell можно использовать для управления учетной записью. При управлении облачными удостоверениями у каждого пользователя в организации есть отдельные имя и пароль учетной записи пользователя. Если вы хотите интегрироваться с локальной инфраструктурой и синхронизировать учетные записи пользователей с Microsoft 365, вы можете использовать Azure AD Connect для синхронизации удостоверений и паролей для функций единого доступа.
  
## <a name="plan-for-where-and-how-you-will-manage-your-user-accounts"></a>Планирование места и управления учетной записью пользователя

Где и как можно управлять учетной записью пользователя, зависит от модели удостоверений, которая будет применяться для Microsoft 365. Две общие модели являются облачными и гибридными.
  
### <a name="cloud-only"></a>Только облако

Вы создаете пользователей и управляете ими в Центре администрирования Microsoft 365. Вы также можете использовать PowerShell или Центр администрирования Azure AD. 
    
### <a name="hybrid"></a>Гибридная среда

Учетные записи пользователей синхронизируются с Microsoft 365 из AD DS, поэтому для управления учетными записями пользователей необходимо использовать средства локальной службы AD DS. 
    
## <a name="managing-accounts"></a>Управление учетной записью

При выборе способа создания учетных записей в организации и управления ими учитывайте следующие требования:
  
- Программное обеспечение синхронизации службы каталогов должно быть установлено на серверах в локальной среде для подключения удостоверений между Microsoft 365 и вашей службой AD DS.
    
- Любой параметр синхронизации каталогов, включая параметры SSO, требует, чтобы атрибуты AD DS соответствуют стандартам. Особенности атрибутов, используемых в каталоге, и необходимые порядок очистки (при их необходимости) описаны в подготовьтесь к синхронизации службы каталогов с [Microsoft 365.](prepare-for-directory-synchronization.md) 
    
- Запланируйте создание учетных записей Microsoft 365.

- Независимо от того, как вы собираетесь добавлять учетные записи пользователей в Microsoft 365, вам необходимо управлять несколькими функциями учетных записей, такими как назначение лицензий, указание расположения и так далее. Этими функциями можно долгосрочно управлять в Центре администрирования Microsoft 365 или создавать учетные записи пользователей [с помощью PowerShell.](https://go.microsoft.com/fwlink/p/?LinkId=717083)
    
    Если вы решите добавить всех пользователей и управлять ими через Центр администрирования, укажите расположение и назначьте лицензии одновременно с созданием учетной записи Microsoft 365. В результате планирования не требуется много.
    
    > [!IMPORTANT]
    > Создание учетных записей в Microsoft 365 без назначения лицензии (например, SharePoint Online) означает, что владелец учетной записи может просматривать Центр Microsoft 365, но не может получить доступ к службам в подписке вашей компании. После назначения расположения и лицензии учетная запись реплицируется в назначенную службу или службу. Пользователь может войти в свою учетную запись и использовать службы, которые вы им назначены.
