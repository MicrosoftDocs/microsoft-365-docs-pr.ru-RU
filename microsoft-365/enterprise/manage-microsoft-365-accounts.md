# <a name="tools-to-manage-microsoft-365-user-accounts"></a>Средства для управления учетными записями пользователей Microsoft 365

Управлять пользователями Microsoft 365 можно несколькими различными способами в зависимости от конфигурации. Можно управлять пользователями в [центре администрирования Microsoft 365](https://admin.microsoft.com), Windows PowerShell, в доменных службах Active Directory (AD DS) или на портале Azure Active Directory (Azure AD). 

Как только вы приобрели Microsoft 365, для управления учетными записями можно использовать центр администрирования и Windows PowerShell. При управлении облачными удостоверениями у каждого сотрудника в Организации есть отдельный идентификатор пользователя и пароль для Microsoft 365. Если вы хотите интегрировать свою локальную инфраструктуру с учетными записями пользователей, которые синхронизируются с Microsoft 365, вы можете использовать Azure AD Connect для предоставления синхронизации удостоверений и паролей для функций единого входа.
  
## <a name="plan-for-where-and-how-you-will-manage-your-user-accounts"></a>Планирование того, где и как будут управляться учетными записями пользователей

Здесь и как можно управлять учетными записями пользователей зависит от модели удостоверений, которую вы хотите использовать для вашей Microsoft 365. Две общие модели — это облачная проверка подлинности и Федеративная проверка подлинности.
  
### <a name="cloud-authentication"></a>Проверка подлинности в облаке

- Облачная проверка подлинности — создание пользователей и управление ими в центре администрирования Microsoft 365. Вы также можете использовать Windows PowerShell или Azure AD. 
    
- Синхронизация хэша паролей (ФС) с помощью единого единого входа — самый простой способ включить проверку подлинности для объектов AD DS в Azure AD. С помощью ФС вы синхронизируете объекты учетной записи пользователя доменных служб Active Directory с Microsoft 365 и управляете локальными пользователями. 
    
- Сквозная проверка подлинности (ПТА) с единым входом — предоставляет простую проверку паролей для служб проверки подлинности Azure AD с помощью программного агента, работающего на одном или нескольких локальных серверах, для проверки пользователей непосредственно в доменных СЛУЖБах Active Directory. 
    
### <a name="federated-authentication"></a>Федеративная проверка подлинности

- Параметры федеративной проверки подлинности — в основном для крупных организаций с более сложными требованиями к проверке подлинности, объекты AD DS синхронизируются с Microsoft 365, а учетные записи пользователей управляются локально. 
    
- [Сторонние поставщики проверки подлинности и идентификации](about-microsoft-365-identity.md) — объекты доменных служб Active Directory могут синхронизироваться с Microsoft 365 и доступ к облачным ресурсам в основном управляются сторонним поставщиком удостоверений (IDP). 
    
## <a name="managing-accounts"></a>Управление учетными записями

Принимая решение о том, как Организация будет создавать учетные записи и управлять ими, учитывайте следующие требования:
  
- Для подключения удостоверений между Microsoft 365 и AD DS необходимо установить программное обеспечение синхронизации каталогов на серверах в локальной среде.
    
- Любой вариант синхронизации каталогов, в том числе параметры единого входа, требует, чтобы атрибуты доменных служб Active Directory соответствовали стандартам. Особенности того, какие атрибуты используются в каталоге и какая очистка (если это требуется), описана в [статье Подготовка к подготовке пользователей через синхронизацию службы каталогов к Microsoft 365](prepare-for-directory-synchronization.md). 
    
- Спланируйте, как вы собираетесь создавать учетные записи Microsoft 365.
    
    В следующей таблице перечислены различные средства управления учетными записями.

Для получения расширенных функциональных возможностей можно выполнить обновление до базового выпуска, Azure AD Premium P1 или Azure AD Premium P2. - Независимо от того, как вы планируете добавлять учетные записи пользователей в Microsoft 365, вам необходимо управлять несколькими функциями учетной записи, такими как назначение лицензий, указание расположения и т. д. Эти функции можно управлять в долгосрочной перспективе из центра администрирования, а также [создавать учетные записи пользователей с помощью PowerShell](https://go.microsoft.com/fwlink/p/?LinkId=717083).
    
    Если вы решили добавить всех пользователей и управлять ими с помощью центра администрирования, вы сможете указать расположение и назначить лицензии одновременно с созданием учетной записи Microsoft 365. В результате не нужно ничего планировать.
    
    > [!IMPORTANT]
    > Создание учетных записей в Microsoft 365 без назначения лицензии (например, в SharePoint Online) означает, что владелец учетной записи может просматривать центр Майкрософт 365, но не может получить доступ к службам в подписке компании. После назначения расположения и лицензии учетная запись реплицируется на назначенную службу или службы. Пользователь может войти в свою учетную запись и использовать назначенные им службы.
