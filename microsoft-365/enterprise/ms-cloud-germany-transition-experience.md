## <a name="azure-active-directory"></a>Azure Active Directory

Что не меняется:

- Начальный домен клиента (например) с ИД клиента (GUID) и пользовательскими доменами сохранится `contoso.onmicrosoft.de` после миграции. 

- Запросы на проверку подлинности для ресурсов, переносимых в службы Office 365, предоставляется службой проверки подлинности Office 365 Службой проверки подлинности Azure ( `login.microsoftonline.com` ). Во время миграции ресурсы, которые по-прежнему остаются в Office 365 в Германии, являются аутентификацией существующей службы Germany Azure `login.microsoftonline.de` ().

Соображения, которые следует отметить:

- Для управляемых учетных записей домена после копирования начального клиента Azure Active Directory (Azure AD) завершен (что является первым этапом миграции Azure AD в службу Azure AD Office 365), изменения паролей, изменения самообслуживающих паролей (SSPR) и сброс паролей администраторами должны быть сделаны с порталов служб Office 365. Запросы на обновление паролей из службы Германии на данном этапе не увенчаются успехом, так как клиент Azure AD был перенесен в службы Office 365. Сбросы федераированных паролей домена не влияют, так как они завершались в локальном каталоге.

- Входы Azure представлены на портале, на котором пользователь пытается получить доступ. Журналы аудита доступны только из конечной точки служб Office 365 после перехода. Перед переносом до завершения миграции необходимо сохранить журналы входа и аудита с портала Microsoft Cloud Deutschland.

- Сброс пароля, изменения паролей, сброс пароля администратором управляемых организаций (не использующих службы федерации Active Directory) должны выполняться через портал служб Office 365. Попытки пользователей, которые имеют доступ к порталам Microsoft Cloud Deutschland для сброса паролей, не увенчаются неудачей.

- Общие запросы субъектов защиты данных (GDPR) выполняются с портала администрирования служб Office 365 Azure для будущих запросов. Любые устаревшие или нестандартные диагностические данные, проживающие в Microsoft Cloud Deutschland, удаляются за 30 дней.

## <a name="subscriptions--licenses"></a>Подписки & лицензии

- Подписки на Office 365 и Dynamics из Microsoft Cloud Deutschland переходят в немецкий регион с помощью переноса Azure AD. Затем организация обновляется, чтобы отразить новые подписки на службы Office 365. Во время краткого процесса передачи подписки изменения подписки блокируют.

- Так как клиент переходит на службы Office 365, его подписки и лицензии, определенные в Германии, стандартизуются с новыми предложениями служб Office 365. Соответствующие подписки на службы Office 365 приобретаются для переданных подписок в Германии. Пользователям, у которых есть лицензии в Германии, будут назначены лицензии на службы Office 365. По завершении работы устаревшие подписки в Германии отменяются и удаляются из текущего клиента служб Office 365.

- После переноса отдельных рабочих нагрузок дополнительные функции доступны через службы Office 365 (например, Microsoft Planner и Microsoft Flow) из-за новых подписок на службы Office 365. Если это подходит для вашей организации, клиент или администратор лицензирования может отключить новые планы службы при планировании управления изменениями для внедрения новых служб. Инструкции по отключению планов служб, назначаемой лицензиям пользователей, см. в инструкции по отключению доступа к службам [Microsoft 365](/office365/enterprise/powershell/disable-access-to-services-while-assigning-user-licenses)при назначении лицензий пользователей.

## <a name="exchange-online"></a>Exchange Online

- URL-адреса ресурсов Exchange переходят с устаревшей конечной точки Германии на конечную точку `outlook.office.de` служб Office 365 `outlook.office365.com` после миграции. Пользователи могут получать доступ к перенесенным почтовым ящикам с помощью устаревшего URL-адреса до завершения миграции. Клиенты должны как можно скорее перейти на новый URL-адрес после начала миграции Exchange, чтобы избежать влияния на выход из среды Германии. URL-адреса служб Office 365 для служб Outlook становятся доступными только после начала миграции Exchange.

- Почтовые ящики переносят в качестве процесса отыгрывки. Пользователи в вашей организации могут быть в Microsoft Cloud Deutschland или в немецком регионе во время перехода и являются частью одной организации Exchange (в том же глобальном списке адресов).

- Пользователи веб-Outlook Web App, которые имеют доступ к службе с помощью URL-адреса, где их почтовый ящик не находится, увидят дополнительный запрос на проверку подлинности. Например, если почтовый ящик пользователя находится в службах Office 365, а подключение Outlook Web App пользователя использует устаревшую конечную точку, пользователь сначала пройдет проверку подлинности, а затем `outlook.office.de` `login.microsoftonline.de` — `login.microsoftonline.com` . После завершения миграции пользователь может получить доступ к новому URL-адресу (), и он увидит только один ожидаемый запрос `https://outlook.office365.com` на вход. 

## <a name="office-services"></a>Службы Office

Службы Office Online доступны до `office.de` и во время перехода. После перехода почтовых ящиков пользователей на службы Office 365 пользователи должны начать использовать URL-адреса служб Office 365. По мере переноса последующих рабочих нагрузок в службы Office 365 их интерфейс с office.com портала начнет работать.

Самая недавно используемая служба (MRU) в Office — это переход от Microsoft Cloud Deutschland к глобальным службам Office 365, а не миграции. Только ссылки на MRU со стороны глобальных служб Office 365 будут видны после миграции с Office.com портала. Ссылки MRU из Microsoft Cloud Deutschland не видны в качестве ссылок MRU в глобальных службах Office 365. В службах Office 365 Global ссылки MRU доступны только после того, как миграция клиента достигла этапа 9.

## <a name="exchange-online-protection"></a>Exchange Online Protection

- Функции back-end Exchange Online Protection (EOP) копируется в новом регионе Германии.
- Пользователям Центра безопасности и соответствия требованиям Office 365 необходимо перейти к использованию глобальных URL-адресов `https://protection.office.com` в рамках миграции.

## <a name="office-365-video"></a>Office 365 Видео

С 1 марта 2021 г. служба Office 365 Video не будет поддерживаться, и после завершения переноса SharePoint Online в новые регионы центра обработки данных в Германии видео office 365 не будет поддерживаться.
