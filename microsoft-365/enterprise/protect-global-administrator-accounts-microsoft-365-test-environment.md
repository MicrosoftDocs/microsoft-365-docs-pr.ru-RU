# <a name="protect-global-administrator-accounts-in-your-microsoft-365-for-enterprise-test-environment"></a>Защита учетных записей глобальных администраторов в тестовой среде Microsoft 365 для предприятий

*Это руководство по лаборатории тестирования можно использовать только для Microsoft 365 для корпоративных тестовых сред.*

Вы можете предотвращать цифровые атаки на вашу организацию, обеспечивая максимально надежную защиту учетных записей администраторов. 

В этой статье описывается использование политик условного доступа Azure Active Directory (Azure AD) для защиты учетных записей глобальных администраторов.

Защита учетных записей глобальных администраторов в тестовой среде Microsoft 365 для предприятий состоит из двух этапов:
- [Этап 1. Создание тестовой среды Microsoft 365 для предприятий](#phase-1-build-out-your-microsoft-365-for-enterprise-test-environment)
- [Этап 2. Настройка политик условного доступа](#phase-2-configure-conditional-access-policies)

> [!TIP]
> Чтобы получить визуальную карту всех статей в руководстве по лаборатории тестирования Microsoft 365 для предприятий, перейдите в стек руководств по лаборатории тестирования [Microsoft 365 для предприятий.](../downloads/Microsoft365EnterpriseTLGStack.pdf)

## <a name="phase-1-build-out-your-microsoft-365-for-enterprise-test-environment"></a>Этап 1. Создание тестовой среды Microsoft 365 для предприятий

Если вы хотите протестировать защиту учетных записей глобального администратора облегченным способом с минимальными требованиями, следуйте инструкциям в базовой конфигурации [lightweight.](lightweight-base-configuration-microsoft-365-enterprise.md)
  
Если вы хотите протестировать защиту учетных записей глобального администратора на имитированном предприятии, следуйте инструкциям в сквозной [проверке подлинности.](pass-through-auth-m365-ent-test-environment.md)
  
> [!NOTE]
> Для тестирования защиты учетных записей глобального администратора не требуется тестовая среда имитированной организации, которая включает имитированную интрасеть, подключенную к Интернету, и синхронизацию каталогов для доменных служб Active Directory (AD DS). Она предоставляется здесь в качестве варианта, чтобы вы могли протестировать защиту учетной записи глобального администратора и поэкспериментировать с ней в типичной для организации среде. 
  
## <a name="phase-2-configure-conditional-access-policies"></a>Этап 2. Настройка политик условного доступа

Сначала создайте новую учетную запись пользователя в качестве выделенного глобального администратора.

1. На отдельной вкладке откройте [Центр администрирования Microsoft 365.](https://admin.microsoft.com/)
2. Выберите   >  **"Пользователи активные пользователи",** а затем выберите **"Добавить пользователя".**
3. В области **"Добавить** пользователя" введите **DedicatedAdmin** в полях **"Имя",**"Отображаемая **имя"** и **"Имя пользователя".**
4. Выберите **"Пароль",** **"Позволить мне создать пароль",** а затем введите надежный пароль. Зафиксировать пароль для этой новой учетной записи в надежном месте.
5. Нажмите кнопку **Далее**.
6. В области **"Назначение лицензий на** продукты" выберите **Microsoft 365 E5** и затем выберите **"Далее".**
7. В области **"Необязательные параметры"** выберите **"Центр** администрирования  >  **ролей" "Глобальный**  >  **администратор**  >  **далее".**
8. On the **You're almost done** pane, select **Finish adding,** and then select **Close**.

Затем создайте группу GlobalAdmins и добавьте в нее учетную запись DedicatedAdmin.

1. На **вкладке Центра администрирования Microsoft 365** выберите **"Группы"** в области навигации слева, а затем выберите **"Группы".**
2. Выберите **"Добавить группу"**.
3. В области **"Выбор типа** группы" выберите "Безопасность" **и** выберите **"Далее".**
4. В области **"Настройка основ" выберите** "Создать **группу",** а затем выберите **"Закрыть".**
5. В области **просмотра и завершения добавления группы** введите **GlobalAdmins** и выберите **"Далее".**
7. В списке групп выберите группу **GlobalAdmins.**
8. В области **GlobalAdmins** выберите **"Члены",** а затем выберите **"Просмотреть все и управлять участниками".**
9. В области **GlobalAdmins** выберите "Добавить участников", выберите учетную запись **DedicatedAdmin** и учетную запись глобального администратора, а затем выберите "Сохранить    >    >  **закрыть".**

Затем создайте политики условного доступа, чтобы требовать многофакторную проверку подлинности для учетных записей глобальных администраторов и запретить проверку подлинности, если риск для входов средний или высокий.

Для этой первой политики требуется, чтобы все учетные записи глобальных администраторов использовали MFA.

1. На новой вкладке браузера перейдите к [https://portal.azure.com](https://portal.azure.com) .
2. Щелкните **условный доступ безопасности Azure Active Directory.**  >    >  
3. В области **"Условный доступ** — политики" выберите базовую политику: требовать многофакторную оценку для **администраторов (предварительная версия).**
4. В области **"Базовая** политика" выберите "Использовать **политику немедленно> сохранить.**

Эта вторая политика блокирует доступ к проверке подлинности учетной записи глобального администратора, если риск для входов средний или высокий.

1. В области **"Условный доступ — политики"** выберите **"Новая политика".**
2. В области **"Новая"** **введите "Глобальные администраторы" в** **"Имя".**
3. В разделе **"Назначения"** выберите **"Пользователи и группы".**
4. На **вкладке "Включить"** **области** "Пользователи и группы" выберите "Выбор пользователей и   >  **групп" "Выбор пользователей и**  >  **групп".**
5. В области **"Выбор"** выберите группу **GlobalAdmins** и выберите   >  **"Готово".**
6. В разделе **"Назначения"** выберите **"Условия".**
7.  В области **условий** выберите риск для  **входов,** выберите "Да" для **настройки,** выберите "Высокий" и "Средний", а затем выберите **"Выбрать"** и **"Готово".**
8. В разделе **элементов управления Access** на **новой** области выберите **"Предоставить".**
9. В области **предоставления** выберите **блокирование доступа,** а затем выберите **"Выбрать".**
10. В области **"Создать"** выберите **"Включить"** для политики **"Включить",** а затем выберите **"Создать".**
11. **Закроем портал Azure** и вкладки Центра администрирования Microsoft **365.**

Чтобы протестировать первую политику, выпишитесь и во sign in с помощью учетной записи DedicatedAdmin. Вам будет предложено настроить MFA. Это показывает, что применяется первая политика.
