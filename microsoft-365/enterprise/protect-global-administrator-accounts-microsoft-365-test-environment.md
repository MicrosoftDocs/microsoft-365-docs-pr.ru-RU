# <a name="protect-global-administrator-accounts-in-your-microsoft-365-for-enterprise-test-environment"></a>Защита учетных записей глобального администратора в Microsoft 365 для тестовой среды предприятия

*Это руководство по тестовой лаборатории можно использовать только для Microsoft 365 для корпоративных тестовых сред.*

Вы можете предотвратить цифровые атаки на организацию, обеспечив безопасность учетных записей администратора. 

В этой статье описывается использование политики условного доступа Azure Active Directory (Azure AD) для защиты учетных записей глобального администратора.

Защита учетных записей глобального администратора в Microsoft 365 для корпоративной тестовой среды включает два этапа:
- [Этап 1. Создание Microsoft 365 для корпоративной тестовой среды](#phase-1-build-out-your-microsoft-365-for-enterprise-test-environment)
- [Этап 2. Настройка политик условного доступа](#phase-2-configure-conditional-access-policies)

> Чтобы получить визуальную карту для всех статей в стеке руководства по Microsoft 365 для корпоративной лаборатории тестирования, перейдите Microsoft 365 для корпоративного руководства по [лаборатории тестирования.](../downloads/Microsoft365EnterpriseTLGStack.pdf)

## <a name="phase-1-build-out-your-microsoft-365-for-enterprise-test-environment"></a>Этап 1. Создание Microsoft 365 для корпоративной тестовой среды

Если вы хотите проверить глобальную защиту учетных записей администратора в легком режиме с минимальными требованиями, следуйте инструкциям в [базовой конфигурации Lightweight.](lightweight-base-configuration-microsoft-365-enterprise.md)
  
Если вы хотите протестировать глобальную защиту учетных записей администратора в смоделированном предприятии, следуйте инструкциям в сквозной [проверке подлинности.](pass-through-auth-m365-ent-test-environment.md)
  
> [!NOTE]
> Тестирование глобальной защиты учетных записей администратора не требует имитации тестовой среды предприятия, которая включает смоделированную интрасеть, подключенную к Интернету, и синхронизацию каталогов для служб домена Active Directory (AD DS). Он предоставляется здесь в качестве параметра, чтобы можно было протестировать глобальную защиту учетной записи администратора и поэкспериментировать с ней в среде, которая представляет типичную организацию. 
  
## <a name="phase-2-configure-conditional-access-policies"></a>Этап 2. Настройка политик условного доступа

Сначала создайте новую учетную запись пользователя в качестве выделенного глобального администратора.

1. На отдельной вкладке откройте [центр администрирования Microsoft 365.](https://admin.microsoft.com/)
2. Выберите **Пользователей**  >  **Активные пользователи,** а затем **выберите Добавить пользователя**.
3. В области **Добавить пользователя** введите **DedicatedAdmin** в поле **Имя,** Отображайте **имя** и **имя** пользователя.
4. Выберите **пароль,** **выберите Дайте мне создать пароль,** а затем введите надежный пароль. Запись пароля для этой новой учетной записи в безопасном расположении.
5. Нажмите кнопку **Далее**.
6. В области **Назначить лицензии продукта** выберите **Microsoft 365 E5** и выберите **Далее**.
7. В области **Необязательных параметров** выберите **центр доступа Центра администрирования ролей** к  >    >  **глобальному администратору**  >  **Далее.**
8. На области **Вы почти закончили,** выберите **добавление Готово,** а затем выберите **Закрыть**.

Далее создайте новую группу с именем GlobalAdmins и добавьте к ней учетную запись DedicatedAdmin.

1. На **вкладке Microsoft 365 центра администрирования** выберите **Группы** в левой навигации, а затем выберите **Группы**.
2. Выберите **Добавить группу**.
3. В области **Выберите групповую группу** выберите **безопасность** и выберите **Далее**.
4. В области **Настройка базовых элементов** выберите **Создать** группу, а затем выберите **Закрыть**.
5. В области **Просмотреть и завершить добавление групповой** области введите **GlobalAdmins** и выберите **Далее**.
7. В списке групп выберите **группу GlobalAdmins.**
8. В области **GlobalAdmins** выберите **Членов,** а затем выберите Просмотр всех и **управление участниками.**
9. В области **GlobalAdmins** выберите Добавить **участников,** выберите учетную запись **DedicatedAdmin** и глобальную учетную запись администратора, а затем выберите **Сохранить**  >  **закрыть**  >  **закрыть**.

Далее создайте политики условного доступа, чтобы требовать многофакторной проверки подлинности для учетных записей глобального администратора и отказывать в проверке подлинности, если риск входной записи средний или высокий.

Эта первая политика требует, чтобы все глобальные учетные записи администратора использовали MFA.

1. В новой вкладке браузера перейдите к [https://portal.azure.com](https://portal.azure.com) .
2. Щелкните **Azure Active Directory**  >  **безопасности**  >  **условного доступа**.
3. В области **условного доступа — политики** выберите базовую **политику: требуется MFA для администраторов (предварительный просмотр).**
4. В области **базовой политики** выберите **политику Use немедленно > Сохранить**.

Эта вторая политика блокирует доступ к глобальной проверке подлинности учетных записей администратора, если риск входной записи средний или высокий.

1. В области **условного доступа — политики выберите** новую **политику.**
2. В новой **области** введите **глобальные администраторы** в **Name**.
3. В разделе **Назначения выберите** Пользователей **и группы.**
4. На **вкладке Включить** области **Пользователи** и группы выберите **Выберите** пользователей и группы Пользователи  >  **и группы**  >  **Выберите**.
5. В области **Выбор** выберите группу **GlobalAdmins** и выберите **Выберите Сделано**  >  .
6. В разделе **Назначения выберите** **Условия**.
7. В области  **Условия** выберите риск для входов, выберите **Да** для **Настройки,** выберите высокой и средней, а затем **выберите Выберите** и **Сделать**.  
8. В разделе **Элементы управления доступом** в **новой области** выберите **Grant**.
9. В области **Грант** выберите **блок-доступ** и выберите **Выберите**.
10. В новой **области** выберите **политику "Включить",** а затем выберите **Создать**.
11. Закрой **портал Azure** **и Microsoft 365 вкладки центра** администрирования.

Чтобы протестировать первую политику, вопишитесь и вопишитесь в учетную запись DedicatedAdmin. Вы должны быть предложены для настройки MFA. Это показывает, что применяется первая политика.
