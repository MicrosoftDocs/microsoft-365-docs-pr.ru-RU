# <a name="microsoft-365-network-insights-preview"></a>Microsoft 365 Network Insights (Предварительная версия)

**Сетевая аналитика** — это показатели производительности, полученные от клиента Microsoft 365 и доступные для просмотра только администраторами в вашем клиенте. Аналитика отображается в центре администрирования Microsoft 365 по адресу <https://portal.microsoft.com/adminportal/home#/networkperformance> .

Insights призваны помочь при проектировании периметров сети для расположений в офисе. Каждый из них предоставляет сведения о характеристиках производительности для определенной распространенной ситуации для каждого географического расположения, в котором пользователи обращаются к клиенту.

Существует шесть сведений о сети, которые могут отображаться для каждого местоположения Office:

- [Выход сети с обратной допускной подсистемой](#backhauled-network-egress)
- [Более высокая производительность обнаружено соседних клиентов](#better-performance-detected-for-customers-near-you)
- [Использование неоптимальной передней дверцы службы Exchange Online](#use-of-a-non-optimal-exchange-online-service-front-door)
- [Использование неоптимальной передней дверцы службы SharePoint Online](#use-of-a-non-optimal-sharepoint-online-service-front-door)
- [Минимальная скорость скачивания из передней дверцы SharePoint](#low-download-speed-from-sharepoint-front-door)
- [Оптимальный сетевой выход для пользователя в Китае](#china-user-optimal-network-egress)

Существует два уровня обслуживания сети клиента, которые могут отображаться для клиента. Они также отображаются на страницах оценки продуквитиви:

- [Подключения, показанные в Exchange, на которые влияют проблемы подключения](#exchange-sampled-connections-impacted-by-connectivity-issues)
- [Подключения, показанные в SharePoint, на которые влияют проблемы подключения](#sharepoint-sampled-connections-impacted-by-connectivity-issues)

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, зарегистрированных в программе предварительного просмотра компонентов.

## <a name="backhauled-network-egress"></a>Выход сети с обратной допускной подсистемой

Это значение будет отображаться, если служба Network Insights обнаружит, что расстояние от конкретного пользователя до выхода в сеть превышает 500 миль (800 километров), что указывает на то, что трафик Microsoft 365 передается на распространенное пограничный порт или прокси-сервер Интернета.

Это сокращение является сокращенным в некоторых сводных представлениях.

![Выход сети с обратной допускной подсистемой](../media/m365-mac-perf/m365-mac-perf-insights-detail-backhauled.png)

### <a name="what-does-this-mean"></a>Сценарий

Это указывает, что расстояние между расположением офиса и выходом сети превышает 500 миль (800 километров). Расположение офиса идентифицируется с помощью непонятного места на клиентском компьютере, а расположение выходного трафика определяется с помощью обратного IP-адреса в базы данных расположения. Если служба расположения Windows отключена на компьютерах, расположение Office может быть неточным. Расположение выхода из сети может быть неточным, если информация о базе данных обратного IP-адреса неточна.

Подробные сведения об этом сопоставлении включают расположение Office, предполагаемый процент от общего числа пользователей клиента в расположении, расположение текущего сетевого выхода, релевантность расположения выходных данных, расстояние между расположением и текущей точкой выхода, Дата первого обнаружения условия и Дата разрешения условия.

### <a name="what-should-i-do"></a>Что делать?

Для этого мы рекомендуем использовать сетевой выход ближе к расположению Office, чтобы обеспечить оптимальную маршрутизацию подключений в глобальную сеть Майкрософт и на ближайшую переднюю дверцу службы Microsoft 365. Если вы развернете доступ к сетевым выходным пользователям для пользователей, вы также можете улучшить производительность в будущем, так как корпорация Майкрософт развертывает в будущем как сетевые точки присутствия, так и передние двери обслуживания Microsoft 365.

Для получения дополнительных сведений о том, как устранить эту проблему, просмотрите раздел исходящих [сетевых подключений локально](microsoft-365-network-connectivity-principles.md#egress-network-connections-locally) в [принципах подключения к сети Office 365](microsoft-365-network-connectivity-principles.md).

## <a name="better-performance-detected-for-customers-near-you"></a>Более высокая производительность обнаружено соседних клиентов

Это значение будет отображаться, если служба Network Insights обнаружит, что значительное количество клиентов в области Metro имеет лучшую производительность, чем у пользователей в вашей организации в этом расположении.

В некоторых сводных представлениях это аббревиатура "Peers".

![Относительная производительность сети](../media/m365-mac-perf/m365-mac-perf-insights-detail-cust-near-you.png)

### <a name="what-does-this-mean"></a>Сценарий

В этом изучении изучается сводная производительность клиентов Microsoft 365 в том же городе, что и это расположение Office. Это представление отображается в том случае, если средняя задержка пользователей составляет 10% выше средней задержки соседних клиентов.

### <a name="what-should-i-do"></a>Что делать?

Это условие может быть связано с несколькими причинами, включая задержку в корпоративной сети или поставщике услуг Интернета, узких мест или архитектурные проблемы. Проверьте задержку между каждым переходом между сетью Office и текущей передней дверцей Microsoft 365. Дополнительные сведения см. в статье [принципы сетевого подключения Office 365](microsoft-365-network-connectivity-principles.md).

## <a name="use-of-a-non-optimal-exchange-online-service-front-door"></a>Использование неоптимальной передней дверцы службы Exchange Online

Это значение будет отображаться, если служба Network Insights обнаруживает, что пользователи в определенном расположении не подключаются к оптимальной передней дверце службы Exchange Online.

Эта аналитика сокращена в виде "маршрутизации" в некоторых сводных представлениях.

![Неоптимальная передняя дверь EXO](../media/m365-mac-perf/m365-mac-perf-insights-detail-front-door-exo.png)

### <a name="what-does-this-mean"></a>Сценарий

Мы перечислим передние двери служб Exchange Online, которые подходят для использования из города местонахождения Office с хорошей производительностью. Если в текущем тесте показывается использование передней дверцы службы Exchange Online, которой нет в этом списке, мы вызываем эту рекомендацию.

### <a name="what-should-i-do"></a>Что делать?

Использование неоптимальной передней дверцы службы Exchange Online может быть вызвано обратной связи в сети перед выходом в корпоративную сеть, в этом случае мы рекомендуем использовать локальную и прямую сеть. Кроме того, это может быть вызвано использованием удаленного сервера рекурсивного разрешения DNS, в результате чего мы рекомендуем выравнивать рекурсивный сервер сопоставителя DNS с выходом в сеть.

## <a name="use-of-a-non-optimal-sharepoint-online-service-front-door"></a>Использование неоптимальной передней дверцы службы SharePoint Online

Это значение будет отображаться, если служба Network Insights обнаружит, что пользователи в определенном расположении не подключаются к ближайшей передней дверце службы SharePoint Online.

Это сокращение сокращается как "АФД" в некоторых сводных представлениях.

![Неоптимальная передняя дверь SPO](../media/m365-mac-perf/m365-mac-perf-insights-detail-front-door-spo.png)

### <a name="what-does-this-mean"></a>Сценарий

Мы идентифицируем переднюю дверцу службы SharePoint Online, к которой подключается тестовый клиент. Затем в городе местонахождения Office мы сравниваемся с ожидаемой внешней дверце службы SharePoint Online для этого города. Если это не так, мы будем использовать эту рекомендацию.

### <a name="what-should-i-do"></a>Что делать?

Использование неоптимальной передней дверцы службы SharePoint Online может быть вызвано обратной связи в сети перед выходом в корпоративную сеть, в этом случае мы рекомендуем использовать локальную и прямую сеть. Кроме того, это может быть вызвано использованием удаленного сервера рекурсивного разрешения DNS, в результате чего мы рекомендуем выравнивать рекурсивный сервер сопоставителя DNS с выходом в сеть.

## <a name="low-download-speed-from-sharepoint-front-door"></a>Минимальная скорость скачивания из передней дверцы SharePoint

Это значение будет отображаться, если служба Network Insights обнаружит, что пропускная способность между определенным расположением Office и SharePoint Online менее 1 Мбит/с.

Это сокращение сокращается до "пропускная способность" в некоторых сводных представлениях.

### <a name="what-does-this-mean"></a>Сценарий

Скорость скачивания, которую пользователь может получить из SharePoint Online и фронтальных дверей службы OneDrive для бизнеса, измеряется в мегабайтах в секунду (Мбит/с). Если это значение меньше 1 Мбит/с, мы предоставляем эту информацию.

### <a name="what-should-i-do"></a>Что делать?

Для повышения скорости загрузки может потребоваться увеличить пропускную способность. Кроме того, может возникнуть перегрузка сети между пользовательскими машинами в расположении Office и внешней дверце службы SharePoint Online. Иногда это называется перегруженной потерей и позволяет ограничить скорость загрузки, доступную пользователям даже при наличии достаточной пропускной способности.

## <a name="china-user-optimal-network-egress"></a>Оптимальный сетевой выход для пользователя в Китае

Это представление будет отображаться, если в Организации есть пользователи, подключающиеся к клиенту Microsoft 365 в других географических регионах. 

### <a name="what-does-this-mean"></a>Сценарий

Если в вашей организации есть подключение к частной глобальной сети, рекомендуется настроить сетевую цепь WAN из расположений Office в Китае, в которой есть сетевой выход в Интернет, в любом из следующих расположений:

- Гонконг
- Япония
- Тайвань
- Республика Корея
- Сингапур
- Малайзия

Исходящий трафик из Интернета больше не влияет на производительность, а исходящий трафик в Китае может привести к большим задержкам и проблемам с подключением из-за перегрузки с перекрестными границами.

### <a name="what-should-i-do"></a>Что делать?

Для получения дополнительных сведений об устранении проблем с производительностью, связанных с этим представлением, ознакомьтесь со статьей [Оптимизация производительности глобального клиента Office 365 для пользователей Китая](microsoft-365-networking-china.md).

## <a name="exchange-sampled-connections-impacted-by-connectivity-issues"></a>Подключения, показанные в Exchange, на которые влияют проблемы подключения

Это значение будет отображаться, когда затронуто 50% или больше подключений по образцу. Влияние определяется оценкой Exchange, приведенным ниже 60% для каждого примера.

### <a name="what-does-this-mean"></a>Сценарий

Это указывает на то, что большинство пользователей, скорее всего, испытывают проблемы с подключением Outlook к Exchange Online. Процентное отношение образцов, скорее всего, представляет процент пользователей, которые отображаются под 60 точек.  

### <a name="what-should-i-do"></a>Что делать?

Включите видимость сетевого подключения к сети Office, если это еще не сделано. Вы хотите определить, на какие офисы повлияли плохое сетевое подключение, которое влияет на Exchange, и Узнайте о способах, с помощью которых можно увеличить периметр сети для подключения пользователей к сети корпорации Майкрософт.

## <a name="sharepoint-sampled-connections-impacted-by-connectivity-issues"></a>Подключения, показанные в SharePoint, на которые влияют проблемы подключения

Это значение будет отображаться, когда затронуто 50% или больше подключений по образцу. Влияние оценки SharePoint ниже 40% для каждого примера.

### <a name="what-does-this-mean"></a>Сценарий

Это указывает на то, что большинство пользователей, скорее всего, столкнулись с проблемами взаимодействия с SharePoint и OneDrive. Процентное отношение образцов, скорее всего, представляет процент пользователей, которые отображаются под 40 точек.  

### <a name="what-should-i-do"></a>Что делать?

Включите видимость сетевого подключения к сети Office, если это еще не сделано. Вы хотите определить, на какие офисы повлияли плохое сетевое подключение, которое повлияет на SharePoint, и Узнайте о способах, позволяющих увеличить периметр сети для подключения пользователей к сети корпорации Майкрософт.
