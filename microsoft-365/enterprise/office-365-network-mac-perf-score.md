# <a name="microsoft-365-network-assessment-preview"></a>Оценка сети Microsoft 365 (Предварительная версия)

В сетевом подключении центра администрирования Microsoft 365 средства **оценки сети** заменяют статистические показатели производительности сети на снимок работоспособности сети периметра корпоративной сети, представленный значением баллов из 0-100. С помощью оценки сети вы узнаете, сколько ИТ-проекта сети влияет на работу пользователей Office 365. Оценка сети распространяется как на весь клиент, так и на каждое географическое расположение, из которого пользователи подключаются к клиенту, предоставляя администраторам Microsoft 365 простой способ незамедлительно оценить работоспособность сети предприятия и быстро перейти к подробному отчету по любому глобальному расположению в офисе.

Баллы оценки сети — это среднее значение задержки TCP, скорости загрузки и метрики качества подключения UDP, компилируемые один раз в день. Метрики производительности для сетей, принадлежащих корпорации Майкрософт, исключаются из этих измерений, чтобы убедиться, что результаты оценки ненеоднозначны и относятся к корпоративной сети.

![Значение оценки сети](../media/m365-mac-perf/m365-mac-perf-overview-score-top.png)

Слишком маленькое значение оценки сети предполагает, что клиенты Microsoft 365 могут испытывать серьезные проблемы с подключением к клиенту или обслуживанием непроизводительного пользовательского интерфейса, в то время как высокое значение указывает правильно настроенную сеть с минимальными проблемами производительности. Значение 80% представляет работоспособную базовую линию, на которой не следует ожидать, что вы не будете получать обычные жалобы пользователей по подключению к Microsoft 365 или скорости ответа из-за производительности сети. По мере появления итеративных улучшений сетевого подключения это значение увеличится вместе с пользовательским интерфейсом.

| Оценка сети | Ожидается взаимодействие с пользователем |
| :----------------- | :----------------------- |
| 100                | Лучший                     |
| 80                 | Соответствует рекомендациям    |
| 60                 | Хорошо               |
| 40                 | Пользователи могут столкнуться с проблемами |
| двадцать                 | Пользователи могут выдаст себя       |
| нуль                  | Проблемы с сетью распространенная тема обсуждения |

>[!IMPORTANT]
>Сведения о сети, рекомендации по производительности и оценки в центре администрирования Microsoft 365 в настоящее время находятся в состоянии предварительной версии и доступны только для клиентов Microsoft 365, зарегистрированных в программе предварительного просмотра компонентов.

## <a name="network-assessment-panel"></a>Панель оценки сети

Каждая оценка сети, в зависимости от того, находится ли область клиента или к определенному расположению Office, показывает панель с подробными сведениями об оценке. На этой панели отображается линейчатая диаграмма с оценкой как в процентах, так и в качестве общего количества баллов для каждой рабочей нагрузки компонентов, включая рабочие нагрузки, в которых были получены данные измерения. Для оценки сети местоположения Office мы также пропустим сравнение с клиентами Microsoft 365 в каждом из пяти куинтилес, которые сообщили данные в том же городе, что и Ваше местонахождение для офиса.

![Пример значения оценки сети](../media/m365-mac-perf/m365-mac-perf-overview-score.png)

При **разбивке оценки** на панели отображается оценка каждой рабочей нагрузки компонентов.

**Журнал оценки** показывает, что за прошлые 30 дней оценки и тестирования. Вы также можете отправить отчет о журнале метрик для любого местоположения Office в течение двух лет с помощью вкладки "журнал". На вкладке Журнал можно выбрать атрибуты для создания отчетов и, выбрав время отчета, вы можете выделить влияние проекта обновления сети и улучшения оценки сети.

## <a name="tenant-network-assessments-and-office-location-network-assessments"></a>Оценки сети клиента и оценки сети Office

Оценка сети измеряет Оформление периметра сети офиса в сети корпорации Майкрософт. Улучшения периметра сети лучше всего выполнять в каждом офисе.

Мы отображаем значение оценки сети для всего клиента Microsoft 365 на странице "Обзор сетевой производительности", который является взвешенным средним значением оценки сети для всех мест в офисе. Кроме того, для каждого обнаруженного расположения Office на странице сводки по этому расположению имеется определенное значение оценки сети.

## <a name="exchange-online"></a>Exchange Online

Для Exchange Online измеряется задержка TCP от клиентского компьютера до передней дверцы службы Exchange. Это может повлиять на расстояние, на которое сеть перемещается по локальной сети и глобальной сети клиентов. Кроме того, они могут быть затронуты сетевыми промежуточными устройствами или службами, которые отменяют подключение или вызывают повторную отправку пакетов. И он влияет на то, насколько близка Ближайшая Передняя дверца службы Exchange. Медиана (также известная как 50-й процентиль или P50 Measure) применяется для всех измерений за последние три дня.

Оценка Exchange Online выполняется с помощью следующей таблицы. Любой номер задержки TCP между пороговыми значениями назначается линейно в пределах диапазона.

| Задержка TCP   | Points |
| :------------ | :----- |
| 10ms или меньше  | 100    |
| 25ms          | 80     |
| 100 мс         | 60     |
| 200 мс         | 40     |
| 300ms         | двадцать     |
| 350ms или более | нуль      |

## <a name="sharepoint-online"></a>SharePoint Online

Для SharePoint Online скорость скачивания, доступная пользователям для доступа к документу из SharePoint или OneDrive, измеряется. Это может повлиять на пропускную способность, доступную для сетевых цепей между клиентским компьютером и сетью корпорации Майкрософт. Кроме того, это часто влияет на перегрузку сети, которая существует в узких местах в сложных сетевых устройствах или в областях с плохим покрытием. Скорость загрузки измеряется в мегабайтах в секунду, приблизительно в десятых долях от каналов в секунду. Количество мегабайт в секунду является полезным, так как вы можете напрямую узнать, какой размер файла можно скачать: 1 секунду. 25-й процентиль (который также называется мерой P25) используется для всех измерений за последние три дня. Этот 25 процентиль позволяет снизить влияние различных перегрузок с течением времени.

Оценка SharePoint Online выполняется с помощью следующей таблицы. Любой номер скорости скачивания между пороговыми значениями назначается линейно в пределах диапазона.

| Скорость скачивания | Points |
| :------------- | :----- |
| 20MBps или более | 100    |
| 14MBps         | 80     |
| 8MBps          | 60     |
| 4MBps          | 40     |
| 2MBps          | двадцать     |
| 0MBps          | нуль      |

## <a name="microsoft-teams"></a>Microsoft Teams

Для Microsoft Teams качество сети измеряется как задержка UDP, колебание UDP и потеря пакетов UDP. UDP используется для подключения аудио-и видеоконференций для Microsoft Teams. Это может затронуть те же факторы, что и для задержки и скорости скачивания в дополнение к разрывам подключений в сетевой поддержке UDP, так как протокол UDP настроен отдельно для более общего протокола TCP. Медиана (также известная как 50-й процентиль или P50 Measure) применяется для всех измерений за последние три дня. 

Мы вычисляем оценку очков из этих UDP-измерений в масштабе от 1 до 5. Затем мы сопоставлены с масштабом баллов 0-100 для оценки сети Microsoft Teams.  Общая хорошая — свыше 87,5 баллов, а общая плохая — менее 50 очков.
