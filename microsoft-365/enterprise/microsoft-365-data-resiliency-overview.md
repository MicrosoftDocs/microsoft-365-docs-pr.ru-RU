## <a name="microsoft-365-data-resiliency-principles"></a><span data-ttu-id="d63bf-121">Принципы устойчивости данных Microsoft 365</span><span class="sxs-lookup"><span data-stu-id="d63bf-121">Microsoft 365 Data Resiliency Principles</span></span>

<span data-ttu-id="d63bf-122">Устойчивость — это способность облачной службы выдерживаться определенных типов сбоев и все еще полностью работоспособна с точки зрения клиентов.</span><span class="sxs-lookup"><span data-stu-id="d63bf-122">Resiliency refers to the ability of a cloud-based service to withstand certain types of failures and yet remain fully-functional from the customers' perspective.</span></span> <span data-ttu-id="d63bf-123">Устойчивость данных означает, что неисправностей в Microsoft 365 не возникает, а критические данные клиента остаются без изменений и не затрагиваются.</span><span class="sxs-lookup"><span data-stu-id="d63bf-123">Data resiliency means that no matter what failures occur within Microsoft 365, critical customer data remains intact and unaffected.</span></span> <span data-ttu-id="d63bf-124">К этому конечному пользователю службы Microsoft 365 были разработаны на основе пяти определенных принципов устойчивости:</span><span class="sxs-lookup"><span data-stu-id="d63bf-124">To that end, Microsoft 365 services have been designed around five specific resiliency principles:</span></span>

- <span data-ttu-id="d63bf-125">Имеются критические и некритические данные.</span><span class="sxs-lookup"><span data-stu-id="d63bf-125">There is critical and non-critical data.</span></span> <span data-ttu-id="d63bf-126">Некритические данные (например, сведения о том, было ли сообщение прочитано) могут быть удалены в редких случаях сбоя.</span><span class="sxs-lookup"><span data-stu-id="d63bf-126">Non-critical data (for example, whether a message was read) can be dropped in rare failure scenarios.</span></span> <span data-ttu-id="d63bf-127">Важные данные (например, данные клиентов, такие как сообщения электронной почты) должны быть защищены от экстремальных затрат.</span><span class="sxs-lookup"><span data-stu-id="d63bf-127">Critical data (for example, customer data such as email messages) should be protected at extreme cost.</span></span> <span data-ttu-id="d63bf-128">В качестве цели проекта Доставка почтовых сообщений всегда является критической, а подобные сообщения не являются критическими.</span><span class="sxs-lookup"><span data-stu-id="d63bf-128">As a design goal, delivered mail messages are always critical, and things like whether a message has been read is non-critical.</span></span>

- <span data-ttu-id="d63bf-133">Большинство результатов потери данных от действий клиентов, поэтому пользователям разрешается самостоятельно восстанавливать себя с помощью графического интерфейса пользователя, который позволяет им восстанавливать случайно удаленные элементы.</span><span class="sxs-lookup"><span data-stu-id="d63bf-133">Most data loss results from customer actions, so allow customers to recover on their own using a GUI that enables them to restore accidentally deleted items.</span></span>
