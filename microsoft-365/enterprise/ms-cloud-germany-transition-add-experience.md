# <a name="migration-phases-actions-and-impacts-for-the-migration-from-microsoft-cloud-deutschland-advanced"></a>Миграция фазы действий и последствий для миграции из Microsoft Cloud Deutschland (расширенный) 

Миграции клиентов из Microsoft Cloud Deutschland в немецкий регион служб Microsoft Office 365 выполняются в наборе этапов и их настроенных действий для каждой рабочей нагрузки. На этом рисунке показаны девять этапов миграции в новые немецкие центр обработки данных.

![Девять этапов миграции в новые центр обработки данных в Германии](../media/ms-cloud-germany-migration-opt-in/migration-organization.png)

В следующих разделах приводится дополнительная информация о клиентских впечатлениях при переходе из Microsoft Cloud Germany (Microsoft Cloud Deutschland) в службы Office 365 в новом немецком регионе центра обработки данных.

## <a name="office-365-portal-services"></a>Службы портала Office 365

Между 2 этапом 9 и 3 этапом 9 портал партнеров может быть недоступным. В течение этого времени партнер может не иметь доступа к сведениям клиентов на портале партнеров. Так как каждая миграция отличается, продолжительность доступности может быть в часах.

Вызовы служб Azure AD перенаправляются из Microsoft Cloud Deutschland в службы Office 365 и прозрачны для служб. | Все клиенты Office | - Общие запросы субъектов защиты данных (GDPR) выполняются с портала Azure Admin для будущих запросов. Миграция должна быть завершена до завершения работы организации Azure AD в среде служб Office 365. | Клиенты Azure | Для миграций Azure см. в книге "Руководство по миграции Azure", обзор руководства по миграции [для Azure Germany.](https://docs.microsoft.com/azure/germany/germany-migration-main) |
