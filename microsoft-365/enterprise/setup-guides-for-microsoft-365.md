# <a name="setup-guides-for-microsoft-365-and-office-365-services"></a>Руководства по настройке для служб Microsoft 365 и Office 365

Руководства по установке Microsoft 365 и Office 365 предоставляют вам конкретные рекомендации и ресурсы по планированию и развертыванию клиентов, приложений и служб. Эти руководства создаются с помощью тех же рекомендаций, что и встроенные специалисты [Microsoft 365 FastTrack](https://www.microsoft.com/fasttrack/microsoft-365) для совместной работы в отдельных действиях, и доступны всем администраторам в центре администрирования Microsoft 365. Они предоставляют сведения о настройке продукта, включении функций безопасности, развертывании средств для совместной работы, а также предоставляют сценарии для ускорения расширенных развертываний.

## <a name="how-to-access-setup-guides-in-the-microsoft-365-admin-center"></a>Как получить доступ к руководствам по настройке в центре администрирования Microsoft 365

Руководства по настройке доступны на странице [руководства по установке](https://aka.ms/setupguidance) центра администрирования Microsoft 365. Вы можете следить за ходом выполнения, а также можете вернуться в любое время, чтобы завершить руководство. Чтобы открыть страницу **руководства по установке** , выполните следующие действия:

1. В [центре администрирования Microsoft 365](https://admin.microsoft.com/)перейдите на **домашнюю** страницу.

2. Найдите карточку **обучающие & Guide** . 

   ![Карточка учебного & Guide в центре администрирования Microsoft 365](../media/setup-guides-for-microsoft-365/adminportal-trainingandguides.png)

3. Выберите **настраиваемое руководство по установке**.

   ![Снимок экрана со страницей руководства по установке в центре администрирования Microsoft 365](../media/setup-guides-for-microsoft-365/adminportal-setupguidance.png)

>[!NOTE]
>Для доступа к центру администрирования Microsoft 365 требуется разрешение администратора клиента.

## <a name="how-do-setup-guides-work-in-the-microsoft-365-admin-center"></a>Как работают руководства по настройке в центре администрирования Microsoft 365?

В каждом руководстве представлены пошаговые инструкции, ресурсы, статьи и при необходимости, сценарии, которые можно использовать для внесения изменений в конфигурацию. В этих руководствах представлены варианты, отражающие конкретные потребности в малых и крупных организациях. Кроме того, в руководстве включена поддержка как для новых, так и для опытных администраторов.

![Пример руководства по настройке](../media/setup-guides-for-microsoft-365/m365-setupguide-example.png)

С помощью руководств вы можете узнать больше о конкретных возможностях Microsoft 365 и Office 365 на этапе планирования, во время развертывания и развертывания, а также после завершения развертывания для изменения параметра.

## <a name="guides-for-initial-setup"></a>Руководства по начальной настройке

### <a name="prepare-your-environment"></a>Подготовка среды

Руководство по [подготовке среды](https://aka.ms/prepareyourenvironment) поможет подготовить среду организации к работе со службами Microsoft 365 и Office 365. Независимо от ваших целей, для успешного развертывания необходимо выполнить некоторые задачи. Чтобы избежать ошибок при подготовке среды, вы можете получить пошаговые инструкции по подключению к домену, добавлению пользователей, назначению лицензий, настройке электронной почты с помощью Exchange Online, установке или развертыванию приложений Office. 

### <a name="email-setup-advisor"></a>Помощник по настройке электронной почты

[Помощник по настройке электронной почты](https://aka.ms/office365setup) содержит пошаговые инструкции по настройке Exchange Online для Организации. Сюда входит настройка новых учетных записей электронной почты, перенос электронной почты и настройка защиты электронной почты. Для успешной настройки электронной почты воспользуйтесь этим помощником, и вы получите рекомендуемый способ миграции на основе текущей почтовой системы Организации, количество переносимых почтовых ящиков и способ управления пользователями и их доступом.

### <a name="gmail-contacts-and-calendar-advisor"></a>Контакты Gmail и советник по календарю

При переносе почтового ящика пользователя Gmail в Microsoft 365 сообщения электронной почты переносятся, но контакты и элементы календаря не удаляются. С помощью [контактов и помощника по календарю Gmail](https://aka.ms/gmailcontactscalendar) вы можете импортировать контакты Google и элементы календаря Google в Microsoft 365 с помощью методов Import и export с Outlook.com, клиент Outlook или PowerShell.

### <a name="microsoft-365-deployment-advisor"></a>Помощник по развертыванию Microsoft 365

[Помощник по развертыванию Microsoft 365](https://aka.ms/microsoft365setupguide) предоставляет рекомендации по настройке средств повышения производительности, политик безопасности и возможностей управления устройствами. Если вы используете корпоративную подписку на Microsoft 365 Business Premium или Microsoft 365, вы можете использовать этот помощник для установки и настройки устройств Организации. 

Вы получите рекомендации и получите доступ к ресурсам, чтобы включить облачные службы, обновите устройства до последней поддерживаемой версии Windows 10 и присоедините устройства к Azure Active Directory (Azure AD), в одном централизованном расположении.


### <a name="remote-work-setup-guide"></a>Руководство по настройке удаленной работы

[Руководство по настройке удаленной работы](https://aka.ms/remoteworksetup) предоставляет организациям советы и ресурсы, необходимые для успешной работы пользователей, обеспечения безопасности данных и защиты учетных данных пользователей. 

Вы получите рекомендации по оптимизации трафика устройства удаленных рабочих процессов как в ресурсы Microsoft 365 в облаке, так и в сети организации, что снизит нагрузку на инфраструктуру VPN удаленного доступа.

[Руководство по настройке виртуальных рабочих столов Windows](https://aka.ms/wvdsetupguide) предоставляет администраторам планирование ресурсов и предварительных условий для развертывания, руководство по настройке и дополнительные ресурсы.

Вы можете скачать Microsoft EDGE на отдельные устройства или мы покажем, как развертывать их для нескольких пользователей в Организации с помощью Configuration Manager или Microsoft Intune.

### <a name="intune-configuration-manager-co-management-setup-guide"></a>Руководство по настройке совместного управления в диспетчере конфигураций Intune

Используйте [руководство по настройке управления Intune Configuration Manager](https://aka.ms/comanagementsetup) для существующих клиентских устройств Configuration Manager и новых устройств на основе Интернета, которыми ваша организация хочет совместно управлять с помощью Microsoft Intune и Configuration Manager. Это руководство по управлению совместным развертыванием позволяет управлять устройствами с Windows 10 и добавлять новые функции к устройствам организации, в то же время получая преимущества обоих решений.

## <a name="guides-for-authentication-and-access"></a>Руководство по проверке подлинности и доступу

### <a name="azure-ad-setup-guide"></a>Руководство по установке Azure AD

[Руководство по установке Azure AD](https://aka.ms/aadpguidance) содержит сведения о том, как убедиться в том, что в вашей организации используется надежный фундамент обеспечения безопасности. В этом руководстве вы настроите начальные функции, такие как управление доступом на основе ролей Azure (Azure RBAC) для администраторов, Azure AD Connect для локального каталога, и работоспособность Azure AD Connect, поэтому вы можете отслеживать работоспособность гибридного удостоверения во время автоматической синхронизации. 

Кроме того, он включает в себя важную информацию о включении автоматического сброса паролей, условного доступа и интегрированного стороннего входа, в том числе дополнительной расширенной защиты идентификации и автоматизации подготовки пользователей.

### <a name="sync-users-from-your-orgs-directory"></a>Синхронизация пользователей из каталога Организации

[Мастер синхронизации пользователей из каталога вашей организации](https://aka.ms/directorysyncsetup) поможет включить синхронизацию службы каталогов. Это позволяет использовать локальные и облачные удостоверения для упрощения доступа и упрощения управления. Разблокируйте новые возможности, такие как параметры единого входа, самостоятельного входа, автоматической подготовки учетных записей, управления условным доступом и политик соответствия требованиям. Это гарантирует, что ваши пользователи будут иметь доступ к необходимым им ресурсам из любого места.

### <a name="plan-your-passwordless-deployment"></a>Планирование развертывания с незащищенным паролем

Выполните обновление до альтернативного подхода, позволяющего пользователям безопасно получать доступ к своим устройствам с помощью одного из следующих методов проверки подлинности без пароля: 

- Windows Hello для бизнеса
- Приложение Microsoft Authenticator
- Ключи контроля доступа 

Используйте [Мастер планирования развертывания](https://aka.ms/passwordlesssetup) не только паролей, чтобы найти лучшие методы проверки подлинности, которые нужно использовать, и получить рекомендации по их развертыванию. 

### <a name="plan-your-self-service-password-reset-sspr-deployment"></a>Планирование развертывания SSPR для самостоятельного сброса пароля

Предоставление пользователям возможности изменять или сбрасывать свой пароль независимо от того, заблокирована ли учетная запись, или забыли пароль без необходимости обращаться к инженеру службы поддержки. 

Используйте [Мастер планирования самостоятельного сброса пароля](https://aka.ms/SSPRSetupGuide) для получения релевантных статей и инструкций по настройке соответствующих параметров портала Azure, которые помогут развернуть SSPR в вашей среде.

### <a name="active-directory-federation-services-ad-fs-deployment-advisor"></a>Помощник по развертыванию служб федерации Active Directory (AD FS)

[Помощник по развертыванию AD FS](https://aka.ms/adfsguidance) предоставляет пошаговые инструкции по развертыванию локальной инфраструктуры AD FS, которая выполняет проверку подлинности пользователей для Microsoft 365 и Office 365 Services. С помощью этого руководства ваша организация может просматривать компоненты и требования AD FS, получать и устанавливать сертификаты SSL, необходимые для развертывания, а также устанавливать требуемый прокси-сервер веб-приложений. 

## <a name="guides-for-security-and-compliance"></a>Руководства по обеспечению безопасности и соответствия требованиям

### <a name="microsoft-defender-advanced-threat-protection-atp-advisor"></a>Советник по защите от Advanced Threat protection (ATP) в защитнике Майкрософт

Сделайте оценку уязвимости Организации и решите, какой пакет развертывания и методы конфигурации лучше всего подходят. 

>[!NOTE]
>Для пакета ATP для защитника Майкрософт необходима корпоративная лицензия Майкрософт.

### <a name="exchange-online-protection-setup-guide"></a>Руководство по настройке Exchange Online Protection

Microsoft Exchange Online Protection (EOP) — это облачная служба фильтрации электронной почты для защиты от нежелательной почты и вредоносных программ, с помощью которой можно защитить организацию от нарушений политики обмена сообщениями. 

Используйте [руководство по настройке Exchange Online Protection](https://aka.ms/EOPguidance) , чтобы настроить EOP, выбрав, какие из трех сценариев развертывания используют локальные почтовые ящики &mdash; , гибридные (смесь локальных и облачных) почтовых ящиков, или все облачные почтовые ящики соответствуют &mdash; вашей организации. В этом руководстве представлены сведения и ресурсы для настройки и просмотра лицензирования пользователя, назначения разрешений в центре администрирования Microsoft 365 и настройки политик защиты от вредоносных программ и нежелательной почты в центре безопасности & соответствия требованиям. 

### <a name="office-365-advanced-threat-protection-advisor"></a>Опытный помощник по защите от угроз для Office 365

[Опытный помощник по защите от угроз Office 365](https://aka.ms/oatpsetup) защищает организацию от вредоносных угроз, которые могут возникать в вашей среде с помощью электронных сообщений, ссылок и сторонних средств совместной работы. В этом руководстве представлены ресурсы и сведения, помогающие подготовить и определить расширенный план защиты от угроз в соответствии с потребностями Организации. 

### <a name="microsoft-information-protection-setup-guide"></a>Руководство по настройке защиты информации (Майкрософт)

Обзор возможностей, которые можно применить к стратегии защиты информации, чтобы вы могли быть уверены, что конфиденциальные данные защищены. Используйте подход из четырех стадий жизненного цикла, в котором вы обнаруживаете, классифицирует, защищаете и следите за конфиденциальной информацией. [Руководство по настройке защиты информации (Майкрософт](https://aka.ms/mipsetupguide) ) предоставляет рекомендации по выполнению каждого из этих этапов.

### <a name="microsoft-information-governance-setup-guide"></a>Руководство по настройке Microsoft Information

В [руководстве по настройке системы управления сведениями Майкрософт](https://aka.ms/migsetupguide) представлены сведения, необходимые для настройки стратегии управления Организацией и управления ею, чтобы убедиться, что данные классифицируются и управляются в соответствии с определенными правилами жизненного цикла. С помощью этого руководства вы узнаете, как создавать, автоматически применять или публиковать метки, политики меток и политики хранения, применяемые к контенту Организации для повторного использования и записям соответствия требованиям. Кроме того, вы получите информацию об импорте CSV-файлов с помощью плана файлов для массовых сценариев или для их применения вручную к отдельным документам.

Вы получите инструкции по определению требований к сети Организации с помощью средства планирования сети и помощника Teams в центре администрирования Teams. По завершении развертывания руководство содержит полезные материалы, которые помогут вам начать работу с Teams.

### <a name="sharepoint-setup-guide"></a>Руководство по настройке SharePoint

[Руководство по настройке SharePoint](https://aka.ms/spoguidance) поможет вам настроить хранилище документов SharePoint и управление контентом, создавать сайты, настраивать внешний общий доступ, переносить данные и настраивать дополнительные параметры, а также управлять участием пользователей и общением в Организации. Выполните действия по настройке политик разрешений для совместного использования содержимого, выберите средства синхронизации миграции, а также включите параметры безопасности для среды SharePoint. 

### <a name="onedrive-setup-guide"></a>Руководство по установке OneDrive

Используйте [руководство по установке onedrive](https://aka.ms/ODfBquickstartguide) , чтобы приступить к работе с хранилищем файлов onedrive, общим доступом, совместной работой и синхронизацией. OneDrive предоставляет централизованное расположение, в котором пользователи могут синхронизировать файлы приложений Microsoft 365, настраивать внешний общий доступ, переносить данные пользователей и настраивать расширенные параметры безопасности и доступа к устройствам. Руководство по установке OneDrive можно развернуть с помощью подписки OneDrive или автономного плана OneDrive.

## <a name="advanced-wizards"></a>Расширенные мастера

### <a name="in-place-upgrade-with-configuration-manager"></a>Обновление на месте с помощью Configuration Manager

Используйте [обновление на месте с руководством Configuration Manager](https://aka.ms/win10upgradedemo) при обновлении устройств с ОС Windows 7 и Windows 8,1 до последней версии Windows 10. Вы будете использовать сценарий, предоставленный для проверки необходимых компонентов и автоматической настройки обновления на месте.

### <a name="deploy-office-to-your-users"></a>Развертывание Office для пользователей

Развертывание приложений Office из облака с возможностью настройки установки с помощью средства развертывания Office. [Руководство по развертыванию Office to your users](https://aka.ms/proplusodt) поможет вам создать настраиваемую конфигурацию Office с дополнительными параметрами или использовать предварительно созданную рекомендуемую конфигурацию. В этом дополнительном мастере представлены пошаговые инструкции по предоставлению пользователям установки Office, настроенной в Организации, с помощью пошаговых инструкций по самостоятельной установке или развертывания.

### <a name="deploy-office-to-remote-users"></a>Развертывание Office для удаленных пользователей

Теперь, когда работа выполняется удаленно, пользователям необходимо получить параметры Office вашей организации, когда они не подключены к внутренней сети или при использовании собственных устройств. 

Используйте [руководство развертывание Office для удаленных пользователей](https://aka.ms/officeremoteinstall) , чтобы создать настраиваемую установку Office и затем отправить пользователям созданный скрипт PowerShell, который будет беспрепятственно устанавливать Office вместе с вашей конфигурацией.

### <a name="deploy-and-update-microsoft-365-apps-with-configuration-manager"></a>Развертывание и обновление приложений Microsoft 365 с помощью диспетчера конфигураций

Для организаций, использующих Configuration Manager, вы можете использовать [развертывание и обновление приложений microsoft 365 с помощью консультанта Configuration Manager](https://aka.ms/oppinstall) , чтобы создать скрипт, который будет автоматически настраивать развертывание приложений Microsoft 365 с помощью рекомендаций, рекомендуемых специалистами FastTrack. Используйте это руководство для создания групп развертывания, настройки приложений и компонентов Office, настройки динамической или экономичной установки, а затем запустите сценарий, чтобы создать приложения, правила автоматического развертывания и коллекции устройств, которые необходимо выбрать для развертывания.
