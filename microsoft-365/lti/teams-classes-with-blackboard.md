# <a name="use-microsoft-teams-classes-with-blackboard-learn-ultra"></a>Использование Microsoft Teams классов с помощью Blackboard Learn Ultra

Командная работа является основой каждой современной организации. Поощряя совместную работу, это определяющая характеристика каждого успешного учреждения. Вы можете расширить все возможности и функции Blackboard Learn Ultra, соеряду их с Microsoft Teams классами.

Ваши классы могут включать беседы в режиме реального времени, видео-собрания или асинхронные взаимодействия. Вы можете добавить обмен файлами и совместное использование файлов для учащихся в одном месте. Microsoft Teams с помощью Learn Ultra переопределяет динамику обучения и то, что означает эффективное обучение.

> [!IMPORTANT]
> Убедитесь, что вы успешно создали поле электронной почты учебного заведения в учебной информационной системе (SIS) `help.blackboard.com/Learn/Administrator/SaaS/Integrations/Student\_Information\_System/SIS\_Planning`
>
>Интеграция Microsoft Teams классов зависит от поля электронной почты учреждения в вашем SIS, чтобы соедитировать правильное имя пользователя Microsoft Azure Active Directory (AAD) пользователя (UPN). Если электронная почта учреждения не была предусмотрена, это будет по умолчанию для существующей электронной почты. Рекомендуется установить это поле для каждого пользователя, чтобы обеспечить правильную синхронизацию данных и отсутствие конфликта данных электронной почты между Microsoft AAD и Blackboard Learn Ultra.
>
> Если в сопоставлении SIS это поле не установлено надлежащим образом, интеграция будет продолжать работать, но пользователи могут не отображаться в созданных классах Teams и могут возникать ошибки.

## <a name="supporting-institutional-data-mapping--institution-email-sis-field"></a>Поддержка институционального сопоставления данных — поле SIS электронной почты учреждения

В рамках эволюции с интеграцией поставщиков облачных решений Blackboard Learn Ultra создала новое поле электронной почты учебного заведения как в интеграции студенческой информационной системы, так и в общедоступных API REST, позволяя учреждениям эффективно управлять процессом синхронизации данных между Blackboard Learn Ultra и Microsoft AAD. 

### <a name="what-does-the-institution-email-mean-and-what-does-it-support"></a>Что означает электронная почта учреждения и что она поддерживает?

Поле **электронной почты** institution позволяет настраивать сопоставления полей между внешними источниками данных клиента и Blackboard Learn Ultra. Если источниками данных являются облачные поставщики, такие как Microsoft, имя принципа пользователя (UPN) является основным уникальным идентификатором для каждого пользователя, состоящим из префикса UPN (имя учетной записи пользователя) и суффикса upN (доменное имя DNS), соединяемого вместе с символом @. Это создает уникальный адрес электронной почты для каждого конкретного пользователя в Microsoft Azure Active Directory.

Чтобы обеспечить точность данных и правильность регистрации или членства между классами Learn Ultra и Microsoft Teams, адрес электронной почты пользователя должен совпадать между обеими системами. В Blackboard Learn Ultra пользователи могут изменять или переопределять существующий адрес электронной почты в пользовательском интерфейсе, что может привести к ошибкам синхронизации и неправильному добавлению пользователя в команду классов. **Сопоставление полей** электронной почты учреждения обеспечивает правильное управление этим уровнем проверки безопасности и проверки проверки, независимо от того, изменили ли пользователи электронную почту в Blackboard Learn Ultra или нет.

 Если два адреса электронной почты отличаются, либо:

- Необходимо принять решение о том, какой источник имеет приоритет и будет приниматься в качестве электронной почты человека и учреждения.
  или
- Учреждение может установить настраиваемую сопоставление полей в своей электронной почте institution, что может разрешить потенциальный конфликт.

**Сопоставление полей электронной** почты учреждения теперь доступно для всех существующих типов интеграции SIS в **Advanced Configuration Параметры** Пользователи узнают сопоставление полей типа  >    >  **объекта.**

> [!NOTE]
> Важно отметить, что по умолчанию электронная почта учреждения  устанавливается для электронной почты человека для всех форматов SIS и должна быть уникальной для каждого человека.  Все существующие интеграции, которые настроены и запущены, будут иметь это сопоставление данных, так как SIS не сможет импортировать пользователей, если их электронная почта будет дублироваться. Если учреждению требуется возможность изменить электронную почту учреждения на настраиваемую,  им потребуется управлять этим с помощью расширенных Параметры в SIS.

## <a name="requirements"></a>Requirements

Интеграция Microsoft Teams классов доступна только для **курсов Ultra Course View.** Чтобы использовать его, вашему учреждению необходимо выполнить эти требования:

- Have Blackboard Learn Ultra Learn SaaS with Ultra Base Navigation enabled

- Включить LTI для использования в курсах.

  А. Перейдите в **группу администраторов**  >  **поставщиков инструментов LTI**  >  **Manage Global Properties.**

  Б. Выберите **включенную LTI в курсах** и необязательно выберите **Включено в организациях.**

  В. Выберите **Отправить**.

- Должно быть настроено LTI

- Добавление blackboard Learn Ultra Teams LTI Integration

- Добавление Microsoft Teams классов LTI 1.3

- Добавление инструмента API REST и совместного использования ресурсов поперек происхождения

- Настройка и утверждение интеграции Microsoft Teams классов

## <a name="add-the-blackboard-learn-ultra-teams-classes-lti-13-tool"></a>Добавление средства Blackboard Learn Ultra Teams классов LTI 1.3

1. Из панели **администраторов** выберите **поставщиков инструментов LTI.**

2. Выберите **средство реестра LTI 1.3.**

3. В поле **Client ID** введите или скопируйте и введите этот ID:

   `f1561daa-1b21-4693-ba90-6c55f1a0eb41`

4. Просмотрите все параметры, которые были предварительно заполнены и в **состоянии инструмента,** а затем выберите **Включено**.

5. В **политиках учреждений** выберите **роль в курсе,** имени и адресе электронной почты, а затем выберите **Да** для обоих. 

6. Выберите **разрешить доступ к службе класса** и разрешить доступ к **службе членства.**

## <a name="add-the-microsoft-teams-classes-lti-13-tool"></a>Добавление средства Microsoft Teams классов LTI 1.3

1. Из панели **администраторов** выберите **поставщиков инструментов LTI.**

2. Выберите **средство реестра LTI 1.3.**

3. В поле **Client ID** введите или скопируйте и введите этот ID:

   `027328b7-c2e3-4c9e-aaa1-07802dae6c89`

4. Просмотрите все параметры, которые были предварительно заполнены и в состоянии *инструмента и* выберите *Включено.*

5. В **политиках учреждений** выберите **роль в курсе, имени и** **адресе электронной почты.** Выберите **Да** для обоих.

6. Выберите **разрешить доступ к службе класса** и разрешить доступ к **службе членства.**

## <a name="add-the-rest-api-tool"></a>Добавление инструмента API REST

1. Из панели **администраторов** перейдите к **интеграциям** и выберите **интеграции API rest.**

2. Выберите **Создание интеграции.**

3. В поле **Application ID** введите или скопируйте и введите этот ID:

   `f1561daa-1b21-4693-ba90-6c55f1a0eb41`

4. Введите пользователя для этой интеграции.

   Этот пользователь будет иметь доступ к домашнему API, с которым связано приложение.

5. Выберите **Отправить**.

## <a name="add-the-cross-origin-resource-sharing"></a>Добавление совместного доступа к ресурсам по пересеченным ресурсам

1. Из панели **администратора** перейдите к **интеграции** и выберите **Cross-origin Resource Sharing*.

2. Выберите **создание конфигурации.**

3. В поле **Origin** тип копирования и вклейки этого URL-адреса:

   `https://bb-ms-teams-ultra-ext.api.blackboard.com`

4. В поле **Разрешенные главы** введите **авторизацию**.

5. Set **Available** to **Yes**.

6. Выберите **Отправить**.

## <a name="configure-and-approve-microsoft-teams-classes-integration"></a>Настройка и утверждение интеграции Microsoft Teams классов

Чтобы успешно интегрировать экземпляр Blackboard Learn Ultra с Microsoft Teams классами, необходимо убедиться, что приложение Blackboard Learn Ultra утверждено для доступа в Microsoft Azure клиенте. Это процесс, который должен быть завершен глобальным администратором Microsoft 365 учреждения.

Этот процесс можно сделать до или после настройки приложений LTI в вашей blackboard Learn Ultra Instance.

### <a name="before-configuring-the-lti-applications"></a>Перед настройкой приложений LTI

Если вы решите утвердить приложение Blackboard Learn Ultra Teams Classes Azure перед настройкой интеграции LTI, вам потребуется перенаправить в конечную точку согласия администратора платформы администрирования Microsoft **Identity.** Показан URL-адрес:

`https://login.microsoftonline.com/{tenant}/adminconsent?client\_id=2d94989f-457a-47c1-a637-e75acdb11568`

> [!NOTE]
> Вы замените **{Tenant}** определенным Microsoft Azure клиентом.

### <a name="after-configuring-the-lti-applications"></a>После настройки приложений LTI

1. На панели **администратора** перейдите к средствам и **утилитам** и выберите **администратор Microsoft Teams интеграции.**

2. Выберите **Включить Microsoft Teams.**

3. Добавьте свой **microsoft Tenant ID** в доступное текстовое поле.

4. Выберите один из следующих вариантов:

   - Если приложение имеет предварительное согласие, оно будет показывать небольшой контрольный знак. Если закладки появляются, выберите **Отправить**.

   - Если согласие не утверждено, выполните описанные действия по генерации URL-адреса для согласия и отправьте его в глобальный администратор Microsoft 365 для утверждения.

5. После подтверждения утверждения выберите **retry,** чтобы подтвердить, а затем выберите **Отправить**.
