# <a name="get-your-environment-ready-for-microsoft-viva-topics"></a>Готовьте среду для Microsoft Viva Topics

Чтобы максимально использовать Viva Topics, необходимо включить как можно больше контента для обнаружения тем, чтобы у вас был богатый набор тем для пользователей. Но какой контент следует использовать для обнаружения темы? Как максимально увеличить индексируемую информацию, оставаясь при этом под контролем? Чем больше контента находится в области, тем лучше будут открываться сведения, которые может открыть искусственный интеллект. В этой статье вы можете сделать шаги по планированию, чтобы убедиться, что вы включите соответствующий контент и у вас есть подходящие люди и ресурсы, чтобы сделать хороший опыт для пользователей.

Чтобы спланировать Viva Topics, необходимо:

![Миграция, подключение, модернизация, безопасность и определение действий для подключения к управлению знаниями](../media/knowledge-management/km-adoption-onboarding-checklist.png)

1. [Перенос контента в SharePoint](#1-migrate-content-to-microsoft-365)
    - Индексация тем включает только контент на сайтах SharePoint.
      - По возможности перенос ценного контента в SharePoint Online из внешних источников.
      - Расставить приоритеты в источниках контента с высоким потенциалом молчаливого знания.
      - Выделим преимущества управления знаниями, чтобы побудить пользователей перемещать контент с сайтов OneDrive в SharePoint.

2. [Подключение сведений к Microsoft Graph](#2-connect-information-to-microsoft-graph)
    - В будущем внешний контент может быть занесен в график знаний и станет доступным.
    - Для содержимого, которое невозможно перемещено, рассмотрите возможность использования соединителю Graph для улучшения поиска и подготовки к будущему включению.

3. [Модернизация страниц SharePoint](#3-modernize-sharepoint-pages)
    - Карточки тем можно всплыть только на современных страницах.
    - Определение классических страниц с высоким профилем, которые являются кандидатами на модернизацию.

4. [Надлежащее обеспечение безопасности контента](#4-secure-content-appropriately)
    - Ресурсы темы отделаны безопасностью на основе разрешений пользователя.
    - Определите любое содержимое, которое может иметь неправильно широкие или ограничительные разрешения:
      - Поощряйте владельцев сайтов использовать отчеты о совместном использовании для проверки разрешений
      - У администраторов аудит широко общий контент с помощью поиска
      - Поощряйте владельцев контента делиться контентом, который не является конфиденциальным и может иметь более широкие преимущества для организации.
    - Просмотрите конфигурацию Microsoft Graph для пользователей и контента:
      - Конфигурация индексации тем не включает контент из Поиска или Delve (например, NOINDEX). Просмотрите, являются ли эти конфигурации по-прежнему актуальными.

5. [Определение менеджеров знаний и тем](#5-identify-knowledge-managers-and-topics)
    - Используйте существующие таксономии, чтобы вручную создавать темы или подтверждать темы, предложенные ИИ.
    - Определение экспертов по субъектам (МСП) для ожидаемых или засеяных тем.
    - Определите сайты, которые охватывают большой объем ценных данных, которые можно использовать для пилотного майнинга тем.
    - Вовлечение руководителей знаний и сообществ практики.

## <a name="1-migrate-content-to-microsoft-365"></a>1. Перенос контента в Microsoft 365

Существует несколько средств и служб, которые помогут вам в миграции — вы можете получить обзор и сведения о миграции при переносе контента в [Microsoft 365.](/sharepointmigration/migrate-to-sharepoint-online) Средства миграции включают:

- [Диспетчер переноса данных](/sharepointmigration/mm-get-started)
- [Средство миграции SharePoint (SPMT)](/sharepointmigration/introducing-the-sharepoint-migration-tool)
- [Microsoft 365 FastTrack](https://www.microsoft.com/fasttrack/microsoft-365)
- [Средства и службы миграции партнеров](https://www.microsoft.com/solution-providers)

Сделайте большую часть миграции:

- Миграция на современный сайт, который включает Microsoft Teams. Хотя индексация может происходить на любом сайте SharePoint (классическом или современном), отображение тем пользователям с помощью выделенок и карт происходит только на современных страницах.
- Сохранение имен пользователей — большинство средств миграции позволяют соизмеренить идентификаторы пользователей во время миграции, чтобы такие свойства, как Created By или Modified By, сохранялись после миграции. Это важно для тем, так как авторство файлов используется для идентификации экспертов, добавленных на страницу или карточку темы. 
- Описательное имя учетной записи службы . В некоторых случаях невозможно сохранить имена пользователей. Например, при переносе контента, созданного кем-то, кто больше не является сотрудником организации. В этом случае большинство средств миграции перемещают файл так, как если бы он был создан учетной записью администратора или учетной записью службы. Если это происходит часто, то эта учетная запись службы может быть указана в отношении тем в качестве эксперта. Здесь очень важно имя этой учетной записи. Если вы сделаете его описательным, наличие этих учетных записей, не относящающихся к человеку, будет понятно пользователям, потребляющих темы.

## <a name="2-connect-information-to-microsoft-graph"></a>2. Подключение сведений к Microsoft Graph

Если вы не можете перенести содержимое, подключите его к Microsoft Graph:

- Рассмотрите возможность [реализации соединители контента Graph](/microsoftsearch/connectors-overview). С помощью соединители внешний контент можно индексировать в Microsoft Graph, где пользователи смогут обнаружить его с помощью Microsoft Search.
- Будущие разработки привнесут внешние данные в Viva Topics.

## <a name="3-modernize-sharepoint-pages"></a>3. Модернизация страниц SharePoint

Так как карточки и основные темы могут отображаться только на современных страницах, обновите все страницы, которые вы хотите включить в Viva Topics от классических до современных. См. [в раздел Модернизация классических сайтов SharePoint.](/sharepoint/dev/transform/modernize-classic-sites) Сканер модернизации [SharePoint можно](/sharepoint/dev/transform/modernize-scanner) использовать для подготовки классических сайтов к модернизации.

Если у вас много классических сайтов, расставить приоритеты на страницах с высоким профилем для преобразования в современные.

## <a name="4-secure-content-appropriately"></a>4. Безопасность контента надлежащим образом

Когда пользователи взаимодействуют с карточкой темы или страницей темы, они могут видеть различные ресурсы. Это связано с тем, что они имеют доступ к различным файлам, связанным с этой темой. Если ваши основные разрешения слишком строги, то можно уменьшить количество аспектов обнаружения информации по темам. С другой стороны, если они слишком широкие, то тема может передать содержимое пользователю, которое вы не собираетесь видеть.
Здесь важное значение имеет управление хорошими разрешениями. И хорошее управление разрешениями основано на постоянном партнерстве между администраторами и владельцами контента. Хотя это может быть продолжающееся действие, при подготовке к темам можно предпринять ряд практических действий:

- Поощряйте владельцев сайтов к просмотру общего доступа и разрешений.

  Владельцы сайтов SharePoint могут просмотреть отчет о совместном использовании для своего сайта, в нем показаны полные сведения о всех разрешениях и ссылках общего доступа, настроенных на сайте, см. в [разделе Отчеты о совместном использовании.](/sharepoint/sharing-reports) В этом списке перечислены внутренние и внешние (гостевых) пользователей.

  Владельцы сайтов также могут видеть, у кого  есть разрешения на сайт, переехав на страницы "Разрешения сайта" и **"Параметры расширенных** разрешений".

  1. На своем сайте выберите **параметры**  >  **разрешений сайта**. Проверьте, кто указан в списке владельцев сайтов, членов сайта и посетителей сайтов. Проверьте для всех гостевых пользователей.
  2. На странице **Разрешения** выберите **параметры расширенных разрешений.** Вы можете проверить уникальные разрешения и узнать, кто имеет ограниченный доступ к любым пунктам на сайте.

- Аудит групп и групп Microsoft 365, чтобы убедиться, что они надлежащим образом задают себя в качестве общедоступных или частных групп или групп. New Teams и Microsoft 365 Groups по умолчанию должны быть закрытыми, но при первом их выпущении были общедоступными по умолчанию. Если вы были более ранними усыновителями этих технологий, вам может потребоваться просмотреть. Кроме того, функция группы часто развивается в течение жизненного цикла, и параметр может потребоваться обновить, чтобы отразить текущее использование группы.
- Просмотрите использование "всех", "всех, кроме внешних пользователей" или широких групп безопасности. Содержимое может быть неправильно общим с этими значениями. Чтобы просмотреть использование этих групп, можно:
  - Создание учетной записи без членства в группе
  - Используйте поиск с этой учетной записью для обнаружения контента, который широко используется.
  - Если ненадлежащее содержимое отображается в этой учетной записи с помощью поиска, вы можете работать с владельцами сайтов, чтобы исправить конфигурацию разрешений.

Помимо разрешений можно также управлять областью обнаружения с помощью разделов. Вы всегда контролируете индексацию.

Администраторы могут настроить индексацию в Центре администрирования Microsoft 365. При настройках [управления знаниями](set-up-topic-experiences.md)можно:

- Разрешить обнаружение на всех сайтах SharePoint или указать сайты, чтобы включить или исключить в качестве источников темы.
- Если у вас есть конфиденциальные термины, вы также можете исключить темы по имени. Например, если у вас есть имя конфиденциального проекта, в котором не нужно отображаться выделенка или карточка независимо от разрешений пользователя, вы можете исключить это имя проекта.

На уровне контента вы также можете управлять тем, что можно обнаружить. Любая конфигурация, сделанная для исключения контента из поиска, также будет использоваться при обнаружении контента. Например, если вы исключили из результатов поиска определенную библиотеку документов, эта библиотека документов не будет использоваться для обнаружения темы.

## <a name="5-identify-knowledge-managers-and-topics"></a>5. Определение менеджеров знаний и тем

Управление темами включает три ключевые роли, в том числе две новые роли Azure Active Directory (AAD): администратор знаний и менеджер знаний:

- Администратор знаний (KA) — это техническая роль, как правило, в ИТ-области. Эта роль позволяет настроить Разделы Viva в центре администрирования M365, а также конфигурацию обнаружения и видимости темы.
- Менеджер знаний (KM) работает с самими темами и контролирует их качество и полноту.
- Вкладчики темы (TCs) основаны не на роли AAD, а на разрешениях в центре администрирования. Они являются специалистами по темам, способными курировать контент по темам, добавляя ресурсы и людей.

В зависимости от организации в этих ролях может быть мало или много людей. Для некоторых организаций это могут быть одни и те же люди.

| Администратор знаний | Менеджер знаний | Участник темы |
|:-------|:-------|:-------|:-------|
| Роль AAD | Роль AAD | SME |
| Имеет доступ к центру администрирования | Имеет доступ к центру администрирования | Нет доступа к центру администрирования |
| Настраивает Viva Topics | Владеет управлением и качеством тем | Вносит вклад в темы, основанные на их опыте. |
| Обеспечивает соблюдение стандартов безопасности и соответствия требованиям и понимает лицензионное соглашение.| Выполняет задачи управления темами, такие как создание, редактирование, удаление и отклонение тем. Поддерживает участников темы своими задачами. | Curates the information and content on topic pages, including which people and resources are pinned to that topic. |

Основные моменты и карточки будут отображаться пользователям в контексте их работы, например при просмотре современных страниц в SharePoint. Вы контролируете конечный пользовательский интерфейс для тем.

- Кто может видеть Разделы? Обзор тем настраивается в Центре администрирования Microsoft 365. Выберите, какие группы позволяют видеть темы:
  - Все в моей организации. "Все" не включает гостей, это все внутренние пользователи в каталоге
  - Только выбранные люди или группы безопасности (этот параметр хорош, пока вы еще развертываете Viva Topics, поэтому вы можете протестировать с подмножество пользователей). Если вы хотите, чтобы гости просмотрели Разделы, необходимо использовать параметр "выбранные люди или группы безопасности" и предоставить им лицензию.
  - Ни к кому.

    Все пользователи, даже гостевых, должны иметь лицензию, применяемую для просмотра темы. И помните, что разрешения всегда контролируют то, что можно увидеть.

- Какие темы видны? Вы можете выбрать:
  - Показать все темы кандидатов.
  - Показывать только подтвержденные темы.

Теперь, когда у нас есть менеджеры, эксперты и пользователи, мы можем говорить о самих темах.

- Это хорошая практика для семени темы в вашем списке тем. Качество и количество тем основаны на контенте — он будет создан только в качестве темы, если он включен в контент, который находится в области. Если для этой темы достаточно информации и доказательств, она будет создана ИИ. Темы посева — это то, чем могут помочь менеджер знаний и эксперты по предметам. Сочетание знаний о человеке с ИИ — это оптимальный путь для тем качества. Поэтому если существуют темы, которые вы ожидаете, их можно создавать вручную в центре Topic. Это даст AI мощный сигнал о релевантности этой темы и определит ресурсы и людей, которые будут ассоциироваться с этой темой.
- Используйте существующие таксономии для планирования темы в SharePoint или в других местах. Существующие таксономии часто включают термины организации, продукты, области субъектов и так далее. Источники для тем также могут приходить из списков проектов, существующих закладки поиска и так далее.
