# <a name="plan-for-microsoft-viva-topics"></a>Планирование microsoft Viva Topics

Вы управляете темами работы в организации. Решения по планированию для разделов гарантируют, что для пользователей будут показаны темы высокого качества и у них есть необходимые разрешения на использование и вклад знаний.

В этой статье мы рассмотрим эти решения по планированию:

- Какие сайты SharePoint необходимо обходить для разделов
- Какие темы нужно исключить из тем
- Пользователей, для которых нужно сделать темы видимыми
- Какие пользователи вы хотите предоставить разрешения на управление темами в центре тем
- Какие пользователи вы хотите предоставить разрешения на создание или редактирование разделов в центре тем
- Имя центра тем

Безопасность и конфиденциальность ваших данных соблюдаются, а темы не предоставляет пользователям дополнительный доступ к файлам, на которые у них нет прав. Мы также рекомендуем ознакомиться с разделами о безопасности и конфиденциальности [Microsoft Viva Topics](topic-experiences-security-privacy.md) в процессе планирования.

## <a name="requirements"></a>Требования

Чтобы получить доступ к Центру администрирования Microsoft 365 и настроить разделы, необходимо быть глобальным администратором или администратором SharePoint.

Всем пользователям, которые будут использовать разделы, требуется **лицензия "Опыт работы с** разделами". Назначение лицензий освещается в теме ["Настройка Microsoft Viva Topics".](set-up-topic-experiences.md)

## <a name="topic-discovery"></a>Обнаружение тем

Параметры обнаружения тем указывают, какие сайты SharePoint используются в качестве источников для тем. Вы можете включить все сайты SharePoint, определенный список сайтов или нет сайтов. Рекомендуется выбрать все сайты, чтобы в темах можно было найти большое количество интересных тем для пользователей.

При настройки разделов можно выбрать один из следующих вариантов:

- **Все сайты**: все сайты SharePoint в организации. К ним относятся текущие и будущие сайты.
- **Все, кроме выбранных сайтов:** все сайты, кроме тех, которые вы указали. Сайты, созданные в будущем, будут включены в качестве источников для обнаружения тем. 
- **Только выбранные сайты:** только сайты, которые вы указали. Сайты, созданные в будущем, не будут включены в качестве источников для обнаружения тем.
- **Нет сайтов:** не включать сайты SharePoint.

Если выбран вариант **"Все",** за исключением выбранных сайтов или только выбранных **сайтов,** можно отправить CSV-файл со списком сайтов. Эти параметры полезны, если вы делаете пилотный проект и хотите включить ограниченное количество сайтов для запуска.

Шаблон CSV можно скопировать ниже:

``` csv
Site name,URL
```

Мы не рекомендуем выбирать "Нет **сайтов",** так как это предотвращает автоматическое создания или обновления тем. Однако этот параметр можно выбрать, если нужно настроить разделы, а затем добавить сайты позже.

Мы рекомендуем создать процесс для пользователей или менеджеров знаний, чтобы запросить удаление отдельных сайтов из обнаружения тем, если это необходимо в вашей организации.

## <a name="user-permissions"></a>Разрешения пользователей

Разрешения пользователей, которые вы указываете, определяют, какие пользователи в организации взаимодействуют с темами и что они могут делать.

*Управление темами*

Менеджеры по знаниям следит за качеством информации, ее структурой и другими лучшими методиками в организации. Они могут подтверждать и отклонить разделы.

Хотя вы можете указать отдельных руководителей тем, мы рекомендуем создать группу безопасности (или использовать существующую), которая содержит людей, которые вы хотите быть менеджерами знаний. Эту группу безопасности можно указать в процессе установки.

*Создание и редактирование разделов*

Участники темы — это специалисты по темам и эксперты по темам в вашей организации. Они могут создавать и редактировать разделы. 

Мы рекомендуем разрешить всем пользователям в организации создавать и редактировать темы, так как темы лучше всего работают, когда все пользователи могут обмениваться информацией.

Если вы хотите ограничить создание и редактирование тем определенными людьми или группами, создайте группу безопасности для них и укажите ее в процессе настройки.

Вы можете не разрешать кому-либо вносить свой вклад в темы, но это не рекомендуется. Менеджеры по знаниям по-прежнему смогут редактировать и создавать темы, если вы выберете этот вариант.

*Посетители тем*

Посетители тем могут видеть информацию на страницах тем, в результатах поиска и при выделении тем в содержимом, например на страницах SharePoint. Пользователи могут видеть обнаруженные темы, только если у них есть доступ к файлам и страницам, в которые была обнаружена тема.

При настройке посетителей тем можно выбрать один из вариантов:

- **Все в моей организации**
- **Только выбранные люди или группы безопасности**
- **Никто**

Мы рекомендуем **всем в моей** организации, но при пилотном проекте вы можете выбрать только выбранных людей или группы безопасности. Вы также можете выбрать **"Нет",** если вы хотите настроить разделы, но пока не разрешайте пользователям видеть разделы. (Менеджеры по знаниям по-прежнему смогут просматривать разделы и помогать в принятии решения о том, как сделать разделы доступными для широкой аудитории.)

## <a name="knowledge-rules"></a>Правила знаний

Администратор может исключить определенные темы из тем. Это полезно, если вы хотите, чтобы конфиденциальные данные не появлялись в темах. Менеджеры по знаниям могут исключать темы в центре тем, но темы, исключенные администратором, даже не видны руководителям знаний. (Менеджеры по знаниям также могут удалять разделы в центре тем после обнаружения.)

Чтобы исключить разделы на уровне администратора, необходимо добавить их в CSV-файл и отправить файл. Это можно сделать во время установки или позднее.

CSV-файл должен содержать следующие параметры:

- **Name**: Type the name of the topic you want to exclude. Это можно сделать двумя способами:
- **MatchType-Exact/Partial**: Type whether the name you entered was an *exact* or *partial* match type.
    - Точное совпадение: можно включить точное имя или аббревиатуру (например, *Contoso* или *ATL).*
    - Частичное совпадение: можно исключить все темы, в них есть определенное слово.  Например,  arc исключит все темы со словом *arc* в нем, такие как "Arc *circle",* *"Arc arc welding"* или *"Training arc".* Обратите внимание, что он не будет исключать темы, в которых текст включается как часть слова, например *"Архитектура".*
- **Означает (необязательно)**: (также известное как *расширение).* Если вы хотите исключить акроним, введите слова, за которые стоит аббревиатура.

    ![Исключение разделов в шаблоне CSV](../media/exclude-topics-csv.png) 

Шаблон CSV можно скопировать ниже:

``` csv
Name (required),Expansion,MatchType- Exact/Partial (required)
```

## <a name="administration"></a>Администрирование

При настройке разделов в процессе настройки автоматически создается центр тем. Подумайте о том, как вы хотите назвать центр темы и каким должен быть URL-адрес. В процессе настройки вы можете настроить имя и URL-адрес, а также изменить имя (но не URL-адрес) позже в Центре администрирования Microsoft 365. У вас может быть только один центр тем.

## <a name="setup-checklist"></a>Контрольный список программы установки

При настройке тем вам потребуется следующее:

> [!div class="checklist"]
> * Список сайтов, которые необходимо включить или исключить, если не включить все сайты для обнаружения тем
> * Группа безопасности для пользователей, просматривающих разделы, если не разрешается всем пользователям просматривать разделы
> * Группа безопасности для участников тем, если не разрешается всем пользователям создавать и редактировать разделы
> * Группа безопасности для руководителей знаний по теме, если не разрешает всем пользователям управлять темами
> * Список конфиденциальных разделов, которые необходимо исключить из обнаружения тем
> * Имя сайта центра тем
