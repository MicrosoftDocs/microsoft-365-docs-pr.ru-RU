# <a name="run-a-trial-of-microsoft-viva-topics"></a>Запуск пробной пробной части Microsoft Viva Topics

В этой статье описано, как настроить и запустить пробную пилотную программу для развертывания Viva Topics в организации. В этой статье также рекомендуется использовать рекомендации для судебного разбирательства.

## <a name="sign-up-for-a-trial"></a>Подпишитесь на пробную

Пробные работы доступны в одном из следующих источников. Эти исследования предоставляют 25 пользователям доступ к Viva Topics в течение 30 дней.

- Страница [продукта Viva Topics](https://www.microsoft.com/microsoft-viva/topics?activetab=pivot:overviewtab)

- The [Центр администрирования Microsoft 365](https://admin.microsoft.com)
    1.  Войдите в [центр администрирования Microsoft 365](https://admin.microsoft.com).
    2.  Перейдите **к службам**  >  **покупки биллинга.**
    3.  Прокрутите вниз до раздела **Надстройки**.
    4.  На **плитке "Опыты по** теме" выберите **Сведения**.
    5.  Нажмите **Получить бесплатную пробную версию**.
    6.  Выполните оставшиеся действия мастера, чтобы подтвердить пробную часть.

Чтобы активировать пробную Microsoft 365, необходимо быть глобальным администратором или администратором биллинга.

> [!NOTE]
> Общедоступные пробные процессы можно добавлять только один раз для Microsoft 365 клиента.

### <a name="who-should-be-involved-in-a-trial"></a>Кто должны быть вовлечены в судебное разбирательство

|Роль  |Действия  |
|---------|---------|
|Microsoft 365 администратора или администратора биллинга  |   Активация пробной пробной работы и назначение лицензий      |
|Microsoft 365 глобального администратора или SharePoint администратора    |       Настройка Viva Topics и создание тематических центров  |
|Бизнес-пользователь     |   Выполнение ролей менеджера знаний, вкладчика темы и ролей потребителей по темам      |

### <a name="before-you-activate-a-trial"></a>Перед активацией пробной

Планирование имеет важное значение для эффективной пробной работы над Viva Topics. Пробный период ограничен и должен включать обнаружение тем и изучение качества темы, управления и интерфейсов конечных пользователей.

#### <a name="discovery"></a>Discovery

Существует два варианта стратегии высокого уровня для настройки обнаружения темы во время пробного испытания:

- Индексировать все или большинство SharePoint контента в Интернете.
   - Для полной индексации крупных клиентов может занять до двух недель. В то время как разделы будут постепенно генерируться в течение этого периода, полная индексация может потреблять до половины пробного периода.
   - Для клиентов с большим объемом данных этот параметр может создавать очень большое количество тем, может быть, десятки тысяч.

- Определите подмножество SharePoint для индексации.

Выбор этих стратегий — это баланс следующих двух факторов:

- Наличие достаточного количества данных для создания значимых тем. AI в Viva Topics настроен на работу с большими наборами данных, в идеале с более чем 10 000 документов.
- В течение пробного периода не создается так много тем, что оценка их в течение доступного периода времени является подавляющей.

Для большинства организаций вторая стратегия дает наилучший результат.

> [!NOTE]
> В связи с количеством документов, необходимых ИИ, рекомендуется выполнить испытания Viva Topics на клиенте производства. Это не влияет на производительность клиента в течение этого периода. Доступ к пользовательским интерфейсам Viva Topics могут получать только пользователи, у которых есть пробная лицензия.

#### <a name="roles"></a>Роли

Во время судебного разбирательства необходимо активнее использовать три роли, описанные в следующей таблице.

|Роль  |Действия  |
|---------|---------|
|Управляющий базой знаний     |   Управление этапами жизненного цикла тем; подтверждение и удаление тем; выступать в качестве менеджера сообщества для участников темы      |
|Участник темы    |      Эксперты по теме контента, которые могут:<br> Обзор тем для оценки качества контента, определяемого ИИ<br>Curate обнаружил темы с дополнительным контентом<br>Создание дополнительных тем, которые не были обнаружены ИИ   |
|Тема потребителя    |     Потребление тем с помощью выделений страниц и поиска<br>Предоставление отзывов о значении представленных тем    |

#### <a name="expected-topics"></a>Ожидаемые темы

Может быть полезно документировать темы, которые, как ожидается, будут созданы ИИ, даже если они основаны только на предположениях. Эта задача наиболее легко выполнена при индексации определенного подмножества SharePoint сайтов, для которых можно легко определить мсп.

Наличие документального списка поможет вам:

- Просмотрите список тем, созданных на ИИ, которые могут быть больше, чем вы ожидаете.
- Знать темы, которые необходимо создавать вручную или которые являются приоритетами для курирования.

При успешном развертывании или пробном процессе Viva Topics всегда потребуется сочетание тем, определяемых ИИ и созданных людьми.

## <a name="activate-a-trial"></a>Активация пробной пробной

Когда вы инициируете судебное разбирательство, необходимо:

- Назначение лицензий соответствующим пользователям.
- Выполните [дополнительную настройку](set-up-topic-experiences.md) Viva Topics.

Когда пробная версия активируется, начинается процесс обнаружения темы.

## <a name="during-a-trial"></a>Во время судебного разбирательства

Пробный период следует использовать для оценки следующих компонентов Viva Topics:

- Предлагаемые ИИ темы и контент темы
- Интерфейсы конечных пользователей, наплавка тематических карт на современных SharePoint и в Поиск (Майкрософт)

Учтите следующие факторы:

- Чтобы в Viva Topics было максимальное значение, контент в темах должен быть сочетанием контента, определяемого ИИ, и контента, курированного людьми.
- Все пользовательские интерфейсы являются "обрезанными разрешениями" (включая представление руководителя знаний на странице **Управление темами).** Пользователи увидят тему только в том случае, если у них есть разрешения на просмотр некоторых ресурсов, которые были использованы для создания этой темы. Это означает, что различные пользователи могут видеть разные контенты на одной странице темы.
- Пользователи могут видеть несколько тем, которые имеют одно и то же имя на странице **Управление темами.** Эти темы не обязательно дублируются, но могут быть из-за одного термина, используемого в нескольких контекстах данных, таких как кодовое имя проекта, используемого двумя отдельными проектами.

## <a name="after-a-trial"></a>После пробной

По результатам судебного разбирательства можно решить, следует ли приступить к использованию Viva Topics в производстве.

### <a name="proceed-to-production-use"></a>Приступить к использованию производства

Чтобы обеспечить непрерывность службы, необходимо приобрести необходимое количество лицензий и назначить эти лицензии пользователям. Пробные пользователи, у которых по окончании пробного периода нет полной лицензии, не смогут получить доступ к любым опытом Viva Topics.

### <a name="dont-proceed-to-production-use"></a>Не переходите к производственному использованию

Если вы не покупаете лицензии после пробного процесса:

- Обнаружение темы остановится.
- Пользователи больше не будут видеть основные темы или карточки.
- Центр тем не будет удален, но предлагаемые темы и управление опытом тем не будут доступны.
- Любые темы, определяемые ИИ, будут потеряны.
- Темы, которые были отредактированы автором темы, останутся в библиотеке страниц центра тем. На этих страницах останется только предоставленное вручную содержимое, а не содержимое, предложенное ИИ.
