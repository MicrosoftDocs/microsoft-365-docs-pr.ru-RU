# <a name="create-an-extractor-preview"></a>Создание средства извлечения (Предварительная версия)

Содержимое этой статьи предназначено для проекта Кортекс Private Preview. [Узнайте больше о Кортекс Project](https://aka.ms/projectcortex).

</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4CL2G]

</br> 

До или после создания модели классификатора для автоматизации идентификации и классификации определенных типов документов можно дополнительно добавить в модель средства извлечения для извлечения определенных сведений из этих документов. Например, вам может потребоваться, чтобы модель не только определяла все документы по *обновлению контрактов* , добавленные в библиотеку документов, но и отображала *дату начала службы* для каждого документа в виде столбца в библиотеке документов.

Необходимо создать средство извлечения для каждой сущности в документе, который необходимо извлечь. В этом примере необходимо извлечь *дату запуска службы* для каждого документа *продления контракта* , идентифицируемого моделью. Это должно произойти, если вы хотите просмотреть представление в библиотеке документов для всех документов по *обновлению контрактов* со столбцом, в котором отображается значение даты начала службы для каждого документа.

> [!NOTE]
> Перед созданием средства извлечения необходимо [Добавить файлы примеров](https://docs.microsoft.com/microsoft-365/contentunderstanding/create-a-classifier#add-your-example-files) , чтобы помочь обучить модель для определения сведений, которые необходимо извлечь. Используйте те же примеры файлов, которые использовались для создания классификатора.

## <a name="name-your-extractor"></a>Имя средства извлечения

1. На плитке **Создание и обучение извлечений** на странице Домашняя модель выберите команду **обучение изучению**.
2. На экране **Создание средства извлечения сущностей** введите имя средства извлечения в поле **новое имя средства извлечения** . Например, введите **дату начала** для ИТ – службы, если вы хотите извлечь дату начала службы из каждого документа продления контрактов.
3. Нажмите кнопку **Создать**.

## <a name="add-a-label"></a>Добавление метки

Следующий шаг — маркировка информации, которую необходимо извлечь в примерах обучающих файлов.

Создание средства извлечения открывает страницу извлечения. Здесь отображается список файлов примеров с первым файлом в списке, отображаемом в средстве просмотра.

1. В средстве просмотра выберите данные, которые необходимо извлечь из файлов. Например, если вы хотите извлечь *дату начала обслуживания*, выделите значение даты в первом файле (*понедельник, 14 октября 2019*). а затем нажмите кнопку **сохранить**.  Значение отображается в списке примеры примеров под столбцом " **метки** ".
2. Выберите **следующий файл** для автоматического сохранения и откройте следующий файл в списке в средстве просмотра. Или нажмите кнопку **сохранить** , а затем выберите другой файл из списка **помеченные примеры** .
3. В средстве просмотра Повторяйте шаги 1 и 2, а затем повторяйте, пока не сохраните метку во всех пяти файлах.

    ![Дополнительные параметры](../media/content-understanding/select-service-start-date.png) 

### <a name="add-a-negative-example"></a>Добавление отрицательного примера

Подобно тому, как вы добавляете Неотрицательный образец файла при создании классификатора, необходимо добавить к средству извлечения отрицательный образец. Это должен быть файл, который не содержит значение даты запуска службы.

1. В списке **помеченные примеры** выберите отрицательный пример.
2. В средстве просмотра, расположенном в верхней части статьи, выберите **Метка отсутствует**.
3. Нажмите кнопку **Сохранить**.
 
Когда вы помечайте пять файлов, отображается баннер уведомления о том, что вы хотите перейти к учебному обучению. Вы можете выбрать дополнительные документы или перейти к учебному обучению. 

## <a name="add-an-explanation"></a>Добавление объяснения

В этом примере создается объяснение, в котором приводится подсказка о самом формате объекта и варианты, которые она может иметь в образце документов. Например, значение даты может быть в разных форматах, например:
- 10/14/2019
- 14 октября 2019 г.
- Понедельник, 14 октября 2019 г.
 

Чтобы помочь определить *дату начала службы* , вы создадите описание шаблона.

1. В разделе объяснение выберите **создать** и введите имя (например, *Date*).
2. В списке Тип выберите **шаблон**.
3. В поле Value (значение) укажите вариант даты, отображаемый в примерах файлов. Например, если у вас есть форматы дат 0/00/0000, вы вводите любые варианты, которые появятся в документах, например:
    - 0/0/0000
    - 0/00/0000
    - 00/0/0000
    - 00/00/0000
4. Нажмите **Сохранить**.

### <a name="use-the-explanation-library"></a>Использование библиотеки объяснений

Для создания объяснений элементов, таких как даты, проще использовать библиотеку объяснений, а не вводить все варианты вручную. Библиотека объяснений — это набор готовых пояснений и шаблонов фраз. Библиотека предоставляет все форматы для стандартных фраз или списков шаблонов, таких как даты, Номера телефонов, почтовый индекс и т. д. 

Для примера *даты запуска службы* более эффективно использовать предварительно построенное объяснение *даты* в библиотеке объяснений:

1. В **разделе объяснение**выберите **создать**, а затем выберите **из объяснения библиотека**.
2. В библиотеке объяснение выберите элемент **Дата**. Вы можете просмотреть все варианты распознанной даты.
3. Нажмите кнопку **Add** (Добавить).</br>

    ![Библиотека объяснений](../media/content-understanding/explanation-library.png) 

4. На странице **Создание объяснения** сведения о *дате* автоматически заполняют поля в библиотеке пояснения. Нажмите **Сохранить**.</br>

    ![Date](../media/content-understanding/date-explanation-library.png) 

## <a name="train-the-model"></a>Обучение модели 

Чтобы сохранить объяснение, Начните обучение. Если в вашей модели достаточно информации для извлечения данных из примеров файлов с метками, вы увидите каждый файл с меткой " **соответствует**".  

![ПОИСКПОЗ](../media/content-understanding/match2.png) 

Если в объяснении недостаточно сведений для поиска данных, которые необходимо извлечь, каждый файл будет снабжен **несовпадением**. Вы можете щелкнуть **несовпадающие** файлы, чтобы просмотреть дополнительные сведения о причине несовпадения.


## <a name="add-another-explanation"></a>Добавление дополнительного объяснения

Часто несовпадение указывает на то, что предоставленное объяснение не предоставило достаточно сведений для извлечения значения даты начала службы для соответствия нашим файлам с метками. Возможно, его потребуется изменить или добавить еще одно объяснение.

Обратите внимание, что в качестве значения параметра *Дата начала* для текстовой строки всегда предшествует фактическое значение. Чтобы помочь определить дату начала службы, необходимо создать описание фразы.

1. В разделе объяснение выберите **создать**, а затем введите имя (например, *Строка префикса*).
2. В поле Тип выберите пункт **список фраз**.
3. Используйте *дату начала службы* в качестве значения.
4. Нажмите **Сохранить**.

    ![Строка префикса](../media/content-understanding/prefix-string.png) 

## <a name="train-the-model-again"></a>Еще раз обучить модель

При сохранении объяснения вы снова начнете обучение, на этот раз используя оба объяснения в примере. Если в вашей модели достаточно информации для извлечения данных из помеченных примеров файлов, вы увидите каждый файл с меткой " **соответствует**". 

Если вы опять получите **несовпадение** с подписанными файлами, вам, скорее всего, потребуется создать другое пояснение, чтобы предоставить модели дополнительные сведения для определения типа документа, или рассмотреть возможность внесения изменений в образец модели.

## <a name="test-your-model"></a>Тестирование модели

Если вы получили такое же значение в примере файлов с метками, теперь вы можете проверить модель на оставшихся файлов примеров без метки.

1. На домашней странице модели перейдите на вкладку **тест** .  В этом примере выполняется модель для файлов примеров без метки.
2. В списке **тестовые файлы** показаны примеры файлов, которые отображаются, если модель может извлекать нужные сведения. Используйте эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

    ![Тестирование файлов](../media/content-understanding/test-filies-extractor.png)
