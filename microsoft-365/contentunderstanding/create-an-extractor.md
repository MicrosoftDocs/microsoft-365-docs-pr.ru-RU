# <a name="create-an-extractor-preview"></a>Создание средства извлечения (Предварительная версия)
> [!Note] 
> Содержимое этой статьи предназначено для Кортексного предварительного просмотра Project. [Узнайте больше о Кортекс Project](https://aka.ms/projectcortex).

</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4CL2G]

</br> 

До или после создания модели классификатора для автоматизации идентификации и классификации определенных типов документов можно дополнительно добавить в модель средства извлечения для извлечения определенных сведений из этих документов. Например, вам может потребоваться, чтобы модель не только определяла все документы по *обновлению контрактов* , добавленные в библиотеку документов, но и отображала *дату начала службы* для каждого документа в виде столбца в библиотеке документов.

Необходимо создать средство извлечения для каждой сущности в документе, который необходимо извлечь. В нашем примере мы хотим извлечь *дату начала службы* для каждого документа *продления контракта* , идентифицируемого моделью. Нам нужна возможность просматривать представление в библиотеке документов для всех документов по *обновлению контрактов* со столбцом, в котором отображается значение даты начала каждого документа.

> [!Note]
> Перед созданием средства извлечения необходимо [Добавить файлы примеров](https://docs.microsoft.com/microsoft-365/contentunderstanding/create-a-classifier?view=o365-worldwide#add-your-example-files) , чтобы обучить модель, чтобы определить сведения, которые необходимо извлечь. Используйте те же примеры файлов, которые использовались для создания классификатора.


## <a name="name-your-extractor"></a>Имя средства извлечения

1. На плитке " **Создание и обучение** " в разделе "Домашняя страница модели" щелкните " **Извлечение**и обучение".
2. На экране **Создание средства извлечения сущностей** введите имя средства извлечения в поле **новое имя средства извлечения** . Например, если нам нужно извлечь дату начала службы из каждого документа продления контракта, мы можем назвать ее как **дату начала** .
3. Нажмите **Создать**.

## <a name="add-a-label"></a>Добавление метки

Следующий шаг — маркировка сведений, которые необходимо извлечь в примерах обучающих файлов.

При создании средства извлечения будет открыта страница извлечения, в которой вы увидите список файлов с примерами, с первым файлом в списке, отображаемом в средстве просмотра.

1. В средстве просмотра выберите данные, которые необходимо извлечь из файлов. Например, если вы хотите извлечь *дату начала обслуживания*, в первом файле будет выделено значение даты, которое будет выделено в первом файле (*понедельник, 14 октября 2019*). Затем нажмите кнопку **Сохранить**.  Отобразится значение для файла в списке помеченные примеры под столбцом **метки** .
2. Выберите пункт **следующий файл** для автосохранения и откройте следующий файл в списке в средстве просмотра или выберите **сохранить** и выберите другой файл из списка **помеченных примеров** .
3. В средстве просмотра повторите шаги 1 и 2 и сделайте это, пока метка не будет сохранена в пяти файлах.

    ![Дополнительные параметры](../media/content-understanding/select-service-start-date.png) 


### <a name="add-a-negative-example"></a>Добавление отрицательного примера

Аналогично добавлению файла с отрицательным примером при создании классификатора, необходимо добавить к средству извлечения отрицательный пример. В нашем примере это должен быть файл, который не содержит значение даты начала службы.

1. В списке **помеченные примеры** выберите отрицательный пример.
2. В средстве просмотра, расположенном в верхней части статьи, выберите **Метка отсутствует**.
3. Щелкните **Сохранить**.
 
Получив пять файлов, отображается баннер уведомления о том, что вы перейдете к учебному обучению. Вы можете выбрать дополнительные документы или перейти к учебному обучению. 

## <a name="add-an-explanation"></a>Добавление объяснения

В нашем примере мы создадим объяснение, в котором приводится подсказка о самом формате сущностей и варианты, которые она может содержать в документах примера. Например, значение даты может быть в разных форматах, например:
- 10/14/2019
- 14 октября 2019 г.
- Понедельник, 14 октября 2019 г.
 

Чтобы помочь определить *дату начала службы* , можно создать описание шаблона.

1. В разделе объяснение выберите **создать**, а затем введите имя (например, *Date*).
2. В поле Тип выберите **шаблон список**.
3. Чтобы указать значение, необходимо указать вариант даты, отображаемый в примерах файлов. Например, если у вас есть форматы дат 0/00/0000, введите любые варианты, которые могут отображаться в документах, например:
    - 0/0/0000
    - 0/00/0000
    - 00/0/0000
    - 00/00/0000
4. Нажмите **Сохранить**.


### <a name="use-the-explanation-library"></a>Использование библиотеки объяснений

Для создания объяснений для таких вещей, как даты, значительно проще использовать библиотеку объяснений, чем вводить все варианты вручную. Библиотека объяснений — это набор готовых пояснений и шаблонов фраз. Библиотека пытается предоставить все форматы для стандартных фраз или списков шаблонов, таких как даты, Номера телефонов, почтовый индекс и многие другие. 

Для нашего примера *даты начала службы* более эффективно использовать предварительно построенное объяснение *даты* в библиотеке объяснений:

1. В **разделе объяснение*** * выберите **создать**, а затем выберите **из объяснения библиотеки**.
2. В библиотеке объяснение выберите элемент **Дата**. Вы можете просмотреть все варианты даты, которые будут распознаны.
3. Нажмите кнопку **Добавить**.</br>

    ![Библиотека объяснений](../media/content-understanding/explanation-library.png) 

4. На странице **Создание объяснения** информация о *дате* в библиотеке объяснений будет заполнена полями. Нажмите **Сохранить**.</br>

    ![Библиотека объяснений](../media/content-understanding/date-explanation-library.png) 

 
## <a name="train-the-model"></a>Обучение модели 

Чтобы начать обучение, сохраните объяснение. Если в вашей модели достаточно информации для извлечения данных из примеров файлов с метками, вы увидите каждый файл с меткой " **соответствует**".  

![Библиотека объяснений](../media/content-understanding/match2.png) 

Если в объяснении недостаточно сведений для поиска данных, которые необходимо извлечь, каждый файл будет снабжен **несовпадением**. Вы можете щелкнуть несовпадающие файлы, чтобы просмотреть дополнительные сведения о причине несовпадения.


## <a name="add-another-explanation"></a>Добавление дополнительного объяснения

Часто несовпадение указывает на то, что предоставленное объяснение не предоставило достаточно сведений для извлечения значения даты начала службы для соответствия нашим файлам с метками. Возможно, его потребуется изменить или добавить еще одно объяснение.

В нашем примере обратите внимание на то, что текстовая строка " *Начало обслуживания* " всегда предшествует фактическому значению. Чтобы помочь определить дату начала службы, мы можем создать фразу объяснение.

1. В разделе объяснение выберите **создать**, а затем введите имя (например, *Строка префикса*).
2. В поле Тип выберите пункт **список фраз**.
3. Используйте *дату начала службы* в качестве значения.
4. Нажмите **Сохранить**.

    ![Библиотека объяснений](../media/content-understanding/prefix-string.png) 


## <a name="train-the-model"></a>Обучение модели

При сохранении объяснения вы снова начнете обучение, на этот раз используя оба объяснения в нашем примере. Если в вашей модели достаточно информации для извлечения данных из примеров файлов с метками, вы увидите каждый файл с меткой " **соответствует**". 

Если вы опять получите **несовпадение** с подписанными файлами, вам может потребоваться создать другое пояснение, чтобы предоставить модели дополнительные сведения для определения типа документа, или просмотреть изменения, внесенные в существующие.

## <a name="test-your-model"></a>Тестирование модели

Если вы получили соответствующее приложение с помеченными примерами файлов, теперь вы можете проверить модель на оставшихся файлов примеров без метки.

1. На домашней странице модели перейдите на вкладку **тест** .  Это приведет к запуску модели для файлов примеров без метки.
2. В списке **тестовые файлы** будут отображаться файлы, которые будут отображаться, и отобразится, если модель сможет извлекать нужные сведения. Вы можете использовать эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

    ![Тестирование файлов](../media/content-understanding/test-filies-extractor.png)
