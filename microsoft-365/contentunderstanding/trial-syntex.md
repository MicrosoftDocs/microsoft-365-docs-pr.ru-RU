# <a name="run-a-trial-of-microsoft-sharepoint-syntex"></a>Запустите пробную SharePoint Syntex

В этой статье описано, как настроить и запустить пробную пилотную программу для развертывания SharePoint Syntex организации. Кроме того, рекомендуется использовать рекомендации для судебного разбирательства.

## <a name="sign-up-for-a-trial"></a>Подпишитесь на пробную

Пробная версия SharePoint Syntex предоставляет доступ 300 пользователям в течение 30 дней.

> [!NOTE]
> Для автоматического добавления 1 миллиона кредитов AI Builder в пробную часть включены до 300 пользователей. Для успешного пробного испытания не нужно включать 300 пользователей.

Пробную версию можно получить из одного из следующих источников:

- Страница [SharePoint Syntex продукта](https://www.microsoft.com/microsoft-365/enterprise/sharepoint-syntex?activetab=pivot:overviewtab)

- The [Центр администрирования Microsoft 365](https://admin.microsoft.com)
    1.  Войдите в [центр администрирования Microsoft 365](https://admin.microsoft.com).
    2.  Перейдите **к службам**  >  **покупки биллинга.**
    3.  Прокрутите вниз до раздела **Надстройки**.
    4.  На плитке SharePoint Syntex выберите **Детали**.
    5.  Нажмите **Получить бесплатную пробную версию**.
    6.  Чтобы подтвердить пробную часть, выполните оставшиеся действия мастера.

Чтобы активировать пробную Microsoft 365, необходимо быть глобальным администратором или администратором биллинга.

### <a name="who-should-be-involved-in-a-trial"></a>Кто должны быть вовлечены в судебное разбирательство

|Роль  |Действия  |
|---------|---------|
|Microsoft 365 администратора или администратора биллинга    |     Активация пробной пробной работы и назначение лицензий    |
|Microsoft 365 глобального администратора или SharePoint администратора     |   Настройка SharePoint Syntex и создание центров контента      |
|Бизнес-пользователи     |    Построение и тестирование модели     |

### <a name="before-you-activate-a-trial"></a>Перед активацией пробной

Чтобы успешно спланировать SharePoint Syntex, рассмотрим следующие факторы:

- Наиболее значимое тестирование выполнено в сценариях и данных "реального мира".
- Вы можете активировать пробную SharePoint Syntex только один раз для каждого клиента.

Клиент тестовых или демонстрационных версий может использоваться в качестве "сухого запуска" для проверки действий активации и административных элементов управления. Но, вероятно, лучше оценить построение модели на производственном клиенте.

Чтобы максимально увеличить значение пробной пробной части для клиента производства, необходимо планирование и вовлечение бизнеса. Для определения трех-шести случаев использования, которые потенциально могут быть устранены SharePoint Syntex, следует вовлекть одну или несколько областей бизнеса. Эти случаи использования должны:

- Включай сценарии, которые можно решить с помощью модели обработки форм или понимания документов.
- Четкое понимание цели для любых извлеченных метаданных; например, просмотр форматирования или автоматизации с помощью Power Automate. Хотя SharePoint Syntex основное внимание уделялось классификации документов и извлечению метаданных, значение для количественной оценки — это то, что эти метаданные позволяют.
- Базируются на определенном наборе данных; например, конкретные SharePoint сайты или библиотеки. Распространенным заблуждением SharePoint Syntex является то, что общие модели целей можно применять во всем контенте организации. Более точное представление: модели построены для решения определенных бизнес-проблем в целевых расположениях.

Все эти случаи использования могут оказаться не очень подходящими для SharePoint Syntex. Цель пробной пробной работы не в том, чтобы доказать, что SharePoint Syntex будет соответствовать всем сценариям. Вместо этого пробная версия должна помочь вам лучше понять значение продукта.

В каждом из запланированных случаев использования определите пользователей, которые являются экспертами по теме в связанном содержимом или процессе. Создание моделей SharePoint Syntex фокусируется на экспертах по доменам в контенте, а не на ИТ-специалистах или ресурсах разработчиков.

## <a name="activate-a-trial"></a>Активация пробной пробной

Когда вы инициируете судебное разбирательство, необходимо:

- Назначение лицензий соответствующим пользователям.
- Выполните [дополнительную настройку SharePoint Syntex.](set-up-content-understanding.md)
    - Возможно, вам нужно [создать дополнительные центры контента.](create-a-content-center.md)

После активации пробного процесса можно создавать модели и обрабатывать файлы. См. [инструкции по созданию модели.](create-a-content-center.md)

## <a name="during-a-trial"></a>Во время судебного разбирательства

Пробные периоды ограничены, поэтому лучше сначала сосредоточиться на том, могут ли SharePoint Syntex классифицировать документы и извлекать метаданные для определенных случаев использования. После окончания пробного периода можно оценить, как можно использовать метаданные.

## <a name="after-a-trial"></a>После пробной

На основании результатов судебного разбирательства можно решить, следует ли приступить к использованию SharePoint Syntex.

### <a name="proceed-to-production-use"></a>Приступить к использованию производства

Чтобы обеспечить непрерывность службы, необходимо приобрести необходимое количество лицензий и назначить эти лицензии пользователям. Пробные пользователи, у которых в конце пробного периода нет полной лицензии, не смогут полностью использовать SharePoint Syntex.

Возможно, вам придется оценить предполагаемое использование обработки форм и спланировать ожидаемое количество кредитов AI Builder. Справки см. [в справке Estimate the AI Builder capacity that's right for you.](https://powerapps.microsoft.com/ai-builder-calculator/)

### <a name="dont-proceed-to-production-use"></a>Не переходите к производственному использованию

Если вы не покупаете лицензии после пробного процесса:

- Вы не сможете создавать новые модели.
- Библиотеки, в которые запускались модели, больше не будут автоматически классифицировать файлы или извлекать модели.
- Все ранее засекреченные файлы или извлеченные метаданные не будут затронуты. 
- Центры контента и любые модели понимания документов не будут автоматически удалены. Они останутся доступными для использования, если вы решите приобрести лицензии в будущем.
- Модели обработки форм будут храниться в экземпляре Общих служб данных (CDS) среды Power Platform по умолчанию. Их можно использовать при будущем лицензировании для SharePoint Syntex или с возможностями AI Builder в платформе Power.
