# <a name="step-1-use-sharepoint-syntex-to-identify-contract-files-and-extract-data"></a>Этап 1. Использование SharePoint Syntex для определения файлов контрактов и извлечения данных

Вашей организации необходим способ определения и классификации всех документов по контрактам из множества файлов, которые вы получаете. Кроме *того,* необходимо быстро просмотреть несколько ключевых элементов в каждом из идентифицированных файлов контрактов (например, клиент, *подрядчик* и сумма *платы).* Это можно сделать с помощью [SharePoint Syntex](index.md) для создания модели понимания документов и ее применения в библиотеке документов.

## <a name="overview-of-the-process"></a>Обзор процесса

[Для автоматизации](document-understanding-overview.md) классификации файлов и извлечения информации для понимания документов используются модели искусственного интеллекта (AI). Модели понимания документов также являются оптимальными для извлечения информации из неструктурированных и полуструктурированных документов, в которых необходимая информация не содержится в таблицах или формах, например в контрактах.

1. Сначала необходимо найти по крайней мере пять примеров файлов, которые можно использовать для "обучения" модели для поиска характеристик, определенных типу контента, который вы пытаетесь определить (контракт). 

2. С SharePoint Syntex создайте новую модель понимания документов. С помощью примеров файлов необходимо [создать классификатор](create-a-classifier.md). Обучив классификатор с примерными файлами, вы научите его искать характеристики, характерные для того, что вы увидите в контрактах вашей компании. Например, [создайте "объяснение",](create-a-classifier.md#create-an-explanation) которое ищет определенные строки, которые находятся в ваших контрактах, таких как соглашение об обслуживании, условия соглашения *и* *компенсации*. Вы даже можете обучить объяснению искать эти строки в определенных разделах документа или расположенных рядом с другими строками. Если вы считаете, что обучили классификатор сведениям, которые ему необходимы, вы можете проверить модель на примере набора примеров файлов, чтобы узнать, насколько она эффективна. После тестирования можно при необходимости внести изменения в свои объяснения, чтобы сделать их более эффективными. 

3. В модели можно [создать](create-an-extractor.md) экстрактор для получения определенных фрагментов данных из каждого контракта. Например, для каждого контракта больше всего вас беспокоит информация о том, кто клиент, имя подрядчика и общая стоимость.

4. После успешного создания модели применяйте ее в библиотеке SharePoint [документов.](apply-a-model.md) При отправке документов в библиотеку документов будет работать модель понимания документов, которая будет определять и классифицировать все файлы, соответствующие типу контента контрактов, определенному в модели. Все файлы, классифицируются как контракты, будут отображаться в пользовательском представлении библиотеки. В файлах также будут отображаться значения из каждого контракта, определенного в экстракторе.

   ![Контракты в библиотеке документов](../media/content-understanding/doc-lib-solution.png)

5. Если у вас есть требования к удержанию для [](apply-a-retention-label-to-a-model.md) контрактов, вы также можете использовать модель для применения метки хранения, которая не позволит удалить контракты в течение определенного периода времени.

## <a name="steps-to-create-and-train-your-model"></a>Действия по созданию и подготовке модели

> [!NOTE]
> Для этих действий можно использовать примеры файлов в репозитории ресурсов решений по управлению [контрактами.](https://github.com/pnp/syntex-samples/tree/main/scenario%20assets/Contracts%20Management) Примеры в этом репозитории содержат как файлы модели для понимания документов, так и файлы, используемые для подготовки модели.

### <a name="create-a-contract-model"></a>Создание модели Contract

Первым шагом является создание модели Контракта.

1. В центре контента выберите **Создать**, а затем —**Создать модель**.

2. В поле **"Новое понимание документа"** в поле **Имя** введите имя модели. Для этого решения по управлению контрактом можно назвать типовой *контракт.*

4. Нажмите **Создать**. В результате будет создана домашняя страница для модели.</br>

    ![Снимок экрана домашней страницы Contract.](../media/content-understanding/models-contract-home-page.png)


### <a name="train-your-model-to-classify-a-type-of-file"></a>Обучение модели классификации типа файла

#### <a name="add-example-files-for-your-model"></a>Добавление примеров файлов для модели

Необходимо добавить по крайней мере пять примеров файлов, которые являются документами контрактов, и один пример файла, который не является документом контракта (например, заявление о работе). 

1. На странице **Models >,** в статье **Основные** действия  >  **Добавить примеры файлов**, выберите Добавить **файлы**.

   ![Снимок экрана, показывающий страницу Контракты с выделенной опцией Добавить примеры файлов.](../media/content-understanding/key-actions-add-example-files.png)

2. В **примере Выберите файлы для страницы** модели откройте папку Контракт, выберите файлы, которые вы хотите использовать, а затем выберите **Добавить**. Если у вас нет примеров файлов, выберите Upload **добавить** их.

#### <a name="label-the-files-as-positive-or-negative-examples"></a>Пометить файлы как положительные или отрицательные примеры

1. На странице **Models >,** в статье **Ключевые** действия классифицировать файлы и выполнить обучение, выберите  >   **классификатор Train**.

   ![Снимок экрана, показывающий страницу Контракты с классифицировать файлы и выделенную возможность обучения.](../media/content-understanding/key-actions-classify-files.png)

2. На странице классификатора > contract **> Contract** в верхней части файла первого примера будет см. текст с вопросом, является ли файл примером созданной модели Contract. Если это положительный пример выберите **Да**. Если это отрицательный пример выберите **Нет**.

3. В **списке примеры с** меткой слева выберите другие файлы, которые необходимо использовать в качестве примеров, и пометить их. 

    ![Домашняя страница классификатора](../media/content-understanding/models-contract-classifier.png) 

#### <a name="add-at-least-one-explanation-to-train-the-classifier&quot;></a>Добавьте хотя бы одно объяснение для обучения классификатора 

1. На странице **классификатора > contract > contract** выберите вкладку **Train.**

2. В разделе **&quot;Обученные файлы&quot;** вы увидите список файлов примеров, которые были помечены ранее. Выберите один из положительных файлов из списка, чтобы отобразить его в зритель.

3. В разделе **Пояснения** выберите **New,** а затем **Blank**.

4. На странице **Создание объяснения**:

    а. В поле **Name** введите имя объяснения (например, &quot;Согласование").

    Б. В поле **"Разъяснение"** выберите **список Фразы,** так как вы добавляете текстовую строку.

    В. В поле **Список фраз** введите строку (например, "AGREEMENT"). Вы можете выбрать **чувствительный к делу,** если строка должна быть чувствительной к делу.

    г. Выберите **Сохранить и обучить**.

    ![Снимок экрана панели "Создание панели объяснений".](../media/content-understanding/contract-classifier-create-explanation.png) 

#### <a name="test-your-model"></a>Тестирование модели

Вы можете протестировать модель контракта на примере файлов, которые она еще не видела. Это необязательно, но это может быть полезной наилучшей практикой.

1. На странице **классификатора > contract > contract** выберите вкладку **Test.** Эта модель выполняется в неослабных файлах примеров.

2. В **списке Тестовые файлы** отображаются файлы примера и показано, предсказывает ли модель положительный или отрицательный результат. Используйте эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

    ![Снимок экрана неослабных файлов в списке текстовых файлов](../media/content-understanding/test-on-files.png) 

3. После этого выберите **обучение выходу.**

### <a name="create-and-train-an-extractor"></a>Создание и обучение экстрактора

1. На странице **Models > Contract** в статье Key **Actions** Create and  >  **train extractors** выберите Create **extractor**.

   ![Снимок экрана, показывающий страницу Контракты с выделенным параметром Create and Train extractors.](../media/content-understanding/key-actions-create-extractors.png)

2. На панели **извлечений нового** объекта в поле **Новое** имя введите имя извлечения. Например, назови его *Клиент,* если вы хотите извлечь имя клиента из каждого контракта.

3. Когда вы закончили, выберите **Создать**.

#### <a name="label-the-entity-you-want-to-extract"></a>Метка объекта, который необходимо извлечь

При создании экстрактора откроется страница экстрактора. Здесь вы видите список ваших примеров файлов, причем первый файл в списке отображается в средстве просмотра.

![Снимок экрана страницы примеры примеры с меткой client extractor.](../media/content-understanding/client-extractor-labeled-examples.png) 

Чтобы пометить объект:

1. В средстве просмотра выберите данные, которые нужно извлечь из файлов. Например, если вы хотите извлечь *клиента,* вы выделяете значение клиента в первом файле (в этом примере Best *For You Organics),* а затем выберите **Сохранить**. В столбце **Метка** отображается отображение  значения из файла в списке примеров с меткой.

2. Выберите **следующий файл** для автозаданий и откройте следующий файл в списке в зрителье. Или выберите **Сохранить,** а затем выберите другой файл из списка **примеров с меткой.**

3. В представлении повторите шаги 1 и 2, а затем повторите, пока не сэкономит метку во всех файлах.

После маркировки файлов отображается баннер уведомлений, информирующий о переходе на обучение. Вы можете выбрать метку дополнительных документов или продвижения к обучению.

#### <a name="add-an-explanation"></a>Добавление объяснений

Вы можете создать объяснение, которое содержит подсказку о самом формате сущности и вариантах, которые он может иметь в примерах файлов. Например, значение даты может быть в разных форматах, таких как:

- 10/14/2019
- 14 октября 2019 г.
- Понедельник, 14 октября 2019 г.

Чтобы определить дату *начала контракта,* можно создать объяснение шаблона.

1. В разделе **Пояснения** выберите **New,** а затем **Blank**.

2. На странице **Создание объяснения**:

    а. В поле **Имя** введите имя объяснения (например, *Дата).*

    Б. В поле **"Разъяснение"** выберите **список шаблонов**.

    В. В поле **Значение** укай вариант даты, как они отображаются в примере файлов. Например, если у вас есть форматы даты, которые отображаются как 0/00/0000, вы вводите любые варианты, которые появляются в ваших документах, например:

    - 0/0/0000
    - 0/00/0000
    - 00/0/0000
    - 00/00/0000

4. Выберите **Сохранить и обучить**.

#### <a name="test-your-model-again"></a>Снова протестировать модель

Вы можете протестировать модель контракта на примере файлов, которые она еще не видела. Это необязательно, но это может быть полезной наилучшей практикой.

1. На странице **классификатора > contract > contract** выберите вкладку **Test.** Эта модель выполняется в неослабных файлах примеров.

2. В **списке тестовых файлов** отображаются файлы примера и показано, может ли модель извлекать необходимые сведения. Используйте эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

3. После этого выберите **обучение выходу.**

### <a name="apply-your-model-to-a-document-library"></a>Применение модели в библиотеке документов

Чтобы применить модель к библиотеке SharePoint документов:

1. На странице **Models >, в** статье **Ключевые** действия Применить модель к  >  **библиотекам,** выберите **Применить модель**.

   ![Снимок экрана, на котором показана страница Контракты с моделью Apply to libraries option highlighted.](../media/content-understanding/key-actions-apply-model.png)

2. На панели **Добавить контракт** выберите SharePoint, на который содержится библиотека документов, на которую необходимо применить модель. Если сайт не отображается в списке, используйте поле поиска, чтобы найти его. Нажмите **Добавить**.

    > [!NOTE]
    > Вам потребуются разрешения *Управление списком* или права на *редактирование* для библиотеки документов, к которой применяется модель.

3. После выбора сайта выберите библиотеку документов, к которой необходимо применить модель.

4. Так как модель связана с типом контента, при ее применении к библиотеке будет добавлен тип контента и его представление с метки, извлеченные в виде столбцов. Это представление является представлением библиотеки по умолчанию, но по умолчанию можно выбрать, чтобы оно не было представлением  по умолчанию, выбрав **расширенные** параметры и очистив это новое представление в качестве контрольного окна по умолчанию.

5. Нажмите кнопку **Добавить**, чтобы применить модель к библиотеке.

6. На странице **Models > Contract** в  разделе Библиотеки с этой моделью вы увидите URL-адрес SharePoint сайта.

    ![Снимок экрана домашней страницы Contract, показывающая библиотеки с этим разделом модели.](../media/content-understanding/contract-libraries-with-this-model.png)

После применения модели к библиотеке документов можно приступить к отправке документов на сайт и увидеть результаты.
