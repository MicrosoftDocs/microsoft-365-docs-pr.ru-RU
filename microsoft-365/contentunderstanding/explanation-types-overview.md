# <a name="introduction-to-explanation-types"></a>Общие сведения о типах объяснения

Используйте объяснения, чтобы определить сведения, которые необходимо обозначить, и извлеките в документ общие сведения о моделях для Microsoft SharePoint Синтекс. При создании объяснения обязательно Выбирайте тип объяснения. 

В этой статье рассказывается о различных типах объяснений и способах их использования.

   ![Типы объяснения](../media/content-understanding/explanation-types.png) 
   
Эти типы объяснения доступны:

- **Список фраз**: список слов, фраз, цифр и других знаков, которые можно использовать в документе или извлекаемой информации. Например, Текстовая строка, **ссылающаяся на Doctor** , относится ко всем идентифицирующим документам медицинских ссылок, которые вы идентифицируете.</br>

- **Список "шаблон**": шаблоны списка чисел, букв или других символов, которые можно использовать для определения извлекаемой информации. Например, вы можете извлечь **номер телефона** ссылающегося врача из всех идентифицирующих документов, которые вы идентифицируете.</br>

- **Близость**: сведения о том, как близкие объяснения относятся друг к другу. Например, список "шаблон *номера дома* " отправляется сразу перед списком " *название улицы* ", но без маркеров между ними (вы узнаете о маркерах далее в этой статье). 
 
## <a name="phrase-list"></a>Список фраз

Тип объяснения списка фразы обычно используется для идентификации и классификации документа с помощью модели. Как описано в примере кода с именем *Doctor* , это строка слов, фраз, чисел или символов, которые постоянно используются в документах, которые вы идентифицируете.

В то же время вы можете добиться лучшей успеха с объяснением, если заданная вами фраза находится в согласованном расположении в документе. Например, *ссылающаяся на врач* метку может быть постоянно расположена в первом абзаце документа.

Если для определения метки требуется чувствительность к регистру, то с помощью типа список фраз вы можете указать его в объяснении, выбрав только флажок **точный регистр** .

   ![Учет регистра](../media/content-understanding/case-sensitivity.png) 

## <a name="pattern-lists"></a>Списки шаблонов

Тип списка "шаблон" особенно полезен при создании объяснения, идентифицирующего и извлекающее сведения из документа. Как правило, они представлены в разных форматах, таких как даты, Номера телефонов или номера кредитных карт. Например, дата может отображаться в различных форматах (1/1/2020, 1-1-2020, 01/01/20, 01/01/2020, 1 января, 2020 и т. д.). Определение шаблона позволяет повысить эффективность объяснений, записывая все возможные варианты в данных, которые вы пытаетесь идентифицировать и извлечь. 

В качестве примера **номера телефона** извлеките номер телефона для каждой ссылки врача из всех документов медицинских ссылок, которые идентифицирует модель. При создании объяснения выберите тип списка шаблон, чтобы разрешить возвращение различных форматов.

   ![Список шаблонов телефонных номеров](../media/content-understanding/pattern-list.png)

В этом примере установите флажок **любая цифра из 0-9** . При выборе этого значения каждое значение "0" в списке шаблонов распознается как любая цифра в диапазоне от 0 до 9.

   ![Любая цифра из 0-9](../media/content-understanding/digit-identity.png)

Аналогичным образом, если вы создаете список шаблонов, включающий текстовые символы, установите флажок **любая буква от a до z** . При выборе этого параметра каждый символ "a", используемый в списке шаблонов, распознается как любой символ из "a" в "z".

Например, если вы создадите список шаблонов **дат** и хотите убедиться в том, что формат даты, такой как *1 января, 2020* , то необходимо выполнить указанные ниже действия.
- Добавьте *AAA 0, 0000* и *AAA 00, 0000* в свой список шаблонов.
- Убедитесь, что также выбрана **любая буква от a до z** .

   ![Любая буква a – z](../media/content-understanding/any-letter.png)

Кроме того, если в вашем списке есть требования к капитализации, можно выбрать **только флажок точное заглавные прописные буквы** . Например, если для первой буквы месяца требуется использовать прописные буквы, необходимо выполнить следующие действия:

- Добавьте *AAA 0, 0000* и *AAA 00, 0000* в свой список шаблонов.
- Убедитесь, что выбраны **только точные прописные и строчные буквы** .

   ![Только точный регистр](../media/content-understanding/exact-caps.png)

> [!NOTE]
> Вместо того чтобы поступать со списками шаблонов вручную, используйте [библиотеку объяснений]() для использования готовых шаблонов списка шаблонов для общего списка, например *даты*, *номера телефона*, *номера кредитной карты*и т. д. 

## <a name="proximity"></a>Компонент ранжирования с учетом расположения 

Тип объяснения близлежащих элементов помогает модели определять данные с помощью определения способа закрытия другой части данных. Например, в вашей модели вы определили два объяснения, которые помечают как *номер почтового адреса* клиента, так и *номер телефона*. 

Кроме того, обратите внимание на то, что номера телефонов клиентов всегда отображаются перед номером почтового адреса. 

Алекс Вилбурн<br>
555-555-5555<br>
One Microsoft Way<br>
Redmond, WA 98034<br>

Используйте описание близости, чтобы определить, насколько далеко из объяснения номера телефона лучше определить номер почтового адреса в документах.

   ![Объяснение близости](../media/content-understanding/proximity.png)</br>

#### <a name="what-are-tokens"></a>Что такое маркеры?

Чтобы использовать тип объяснения близости, Узнайте, что такое маркер, так как количество маркеров определяет расстояние от одного объяснения до другого.  

Маркер — это непрерывный диапазон (без пробелов и знаков препинания) букв и цифр. Пробел не является маркером. Каждый знак пунктуации является маркером. В следующей таблице приведено несколько примеров того, как определить количество маркеров в фразе.

|Программа|Количество маркеров|Объяснение|
|--|--|--|
|`Dog`|1 |Одно слово без знаков препинания и пробелов.|
|`RMT33W`|1 |Номер локатора записи. Он может иметь цифры и буквы, но не имеет знаков препинания.|
|`425-555-5555`|5 |Номер телефона. Каждый знак пунктуации является одним маркером, поэтому он будет  `425-555-5555` иметь 5 маркеров:<br>`425`<br>`-`<br>`555`<br>`-`<br>`5555` |
|`https://luis.ai`|7 |`https`<br>`:`<br>`/`<br>`/`<br>`luis`<br>`.`<br>`ai`<br>|

#### <a name="configure-the-proximity-explanation-type"></a>Настройка типа объяснения близости

В этом примере настраивается параметр близости, чтобы можно было определить диапазон количества маркеров, на которые объясняется номер *телефона* , из объяснения *номера улицы* .

Необходимо убедиться, что в качестве минимального диапазона задано значение 0, поскольку между номером телефона и номером улицы нет маркеров.

## <a name="use-the-explanation-library"></a>Использование библиотеки объяснений

Несмотря на то, что вы можете вручную добавлять различные значения списка шаблонов для вашего объяснения, значительно упрощается использование готовых шаблонов, предоставленных в библиотеке объяснений.

Например, вместо того чтобы вручную добавлять все варианты для *Date*, используйте шаблон списка шаблон для параметра *Дата*, который уже включает ряд значений шаблона:</br>

   ![Библиотека объяснений](../media/content-understanding/explanation-template.png)</br>
 
Библиотека объяснений содержит ряд часто используемых объяснений в списке шаблонов, в том числе:</br>

- Date</br>
- Дата (число)</br>
- Time</br>
- Числовой</br>
- Номер телефона</br>
- Почтовый индекс</br>
- Первое слово предложения</br>
- Кредитная карта</br>
- Номер социального страхования</br>

Обратите внимание, что библиотека объяснений также включает шаблоны для объяснений со списками фраз, в том числе:
- Конец предложения
- Денежный

#### <a name="to-use-a-template-from-the-explanation-library"></a>Использование шаблона из библиотеки объяснений

1. В разделе **объяснения** на странице **обучение** модели выберите **создать**, а затем выберите **из шаблона**.</br>

   ![Создание из шаблона](../media/content-understanding/from-template.png)</br>

2.  На странице " **объяснения шаблонов** " выберите объяснение, которое необходимо использовать, а затем нажмите кнопку **Добавить**.</br>

       ![Выбор шаблона](../media/content-understanding/phone-template.png)</br>

3. Сведения о выбранном шаблоне будут отображаться на странице **Создание объяснения** . При необходимости измените имя объяснения и добавьте или удалите элементы из списка шаблон. </br> 

   ![Изменение шаблона](../media/content-understanding/phone-template-live.png)</br>

4. По завершении нажмите кнопку **сохранить**.
