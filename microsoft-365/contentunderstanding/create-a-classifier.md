# <a name="create-a-classifier-preview"></a>Создание классификатора (Предварительная версия)

> [!Note] 
> Содержимое этой статьи предназначено для Кортексного предварительного просмотра Project. [Узнайте больше о Кортекс Project](https://aka.ms/projectcortex).

</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4CL0R]  

</br>

Классификатор — это тип модели, автоматизирующий идентификацию и классификацию типа документа. Например, вам может потребоваться определить все документы по *обновлению контрактов* , добавленные в библиотеку документов, например:

![Документ по обновлению контракта](../media/content-understanding/contract-renewal.png)

При создании классификатора будет создан новый [тип контента SharePoint](https://docs.microsoft.com/sharepoint/governance/content-type-and-workflow-planning#content-type-overview) , который будет связан с моделью.

При создании классификатора необходимо создать *объяснения* , которые помогут определить модель, указав общие данные, которые будут ожидать согласования для этого типа документа. 

Вы используете примеры типа документа ("example Files"), чтобы помочь "обучить" модель для определения файлов с одинаковым типом контента.

Чтобы создать классификатор, необходимо выполнить следующие действия:
1. Назовите модель
2. Добавление файлов примеров
3. Добавление меток к примерам файлов
4. Создание объяснения
5. Тестирование модели 

> [!Note]
> В то время как классификатор использует модель для определения и классификации типов документов, можно также запросить определенные части информации из каждого файла, идентифицируемого моделью. Для этого необходимо создать средство **извлечения** для модели, которое описывается в статье [Создание средства извлечения](create-an-extractor.md).

## <a name="name-your-model"></a>Назовите модель

Первым шагом является создание модели в центре контента, указав ее имя:

1. В центре управления контентом нажмите кнопку **создать**, а затем выберите команду **создать модель**.
2. В области **модель нового документа "Общие сведения** " в поле **имя** введите имя модели. Для нашего примера, если мы хотим определить документы для продления контрактов, мы можем назвать эту модель *продление контракта*.
3. Нажмите **Создать**. При этом будет создана домашняя страница модели.</br>

    ![Домашняя страница модели классификатора](../media/content-understanding/model-home.png)

При создании модели создается новый тип контента SharePoint. Тип контента SharePoint представляет категорию документов, имеющих общие характеристики и совместное использование коллекции столбцов или свойств метаданных для конкретного содержимого. Управление типами контента SharePoint осуществляется с помощью [коллекции типов контента](). Для нашего примера при создании модели мы создадим новый тип контента " *Обновление контракта* ".

Выберите **Дополнительные параметры** , если вы хотите сопоставить эту модель с существующим типом контента в коллекции типов контента SharePoint, чтобы использовать ее схему. Обратите внимание, что хотя вы можете использовать существующий тип контента для использования его схемы, чтобы упростить идентификацию и классификацию, вам все равно потребуется научить модель для извлечения информации из файлов, которые она идентифицирует.</br>

![Дополнительные параметры](../media/content-understanding/advanced-settings.png)

## <a name="add-your-example-files"></a>Добавление файлов примеров

На домашней странице модели можно добавить файлы примеров, которые помогут вам обучить модель для определения типа документа. </br>
</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4D0iX] 

</br>

> [!Note]
> Эти же файлы должны использоваться для обучения классификатора и [извлечения](create-an-extractor.md). Вы всегда можете добавить более позднее, но обычно необходимо добавить полный набор файлов примеров. Вы увидите метку для обучения модели и протестируйте оставшиеся непомеченные, чтобы оценить фитнес модели. 

Для вашего учебного набора вам потребуется использовать как положительные, так и отрицательные примеры:
- Положительный пример: документы, представляющие тип документа. Они содержат строки и сведения, которые всегда будут находиться в документе этого типа.
- Негативный пример: документы, не представляющие тип документа.  В них отсутствуют строки и сведения, которые должны присутствовать в документе этого типа.

Для обучения модели необходимо использовать по крайней мере пять положительных примеров и один отрицательный пример.  Вам потребуются дополнительные шаблоны для тестирования модели после обучения.

Добавление файлов примеров:

1. На домашней странице модели в плитке **Библиотека образца построения** щелкните **Добавить файлы**.
2. На странице **Выбор примеров файлов для модели** выберите файлы примера из библиотеки образцов файлов в центре управления контентом. Если вы еще не отправляли их там, вы можете отправить их сейчас, нажав кнопку **Отправить** , чтобы переместить их в библиотеку образцов файлов.
3. Выбрав файлы примеров, которые необходимо использовать для обучения модели, нажмите кнопку **Добавить**.

    ![Выбор файлов примеров](../media/content-understanding/select-sample.png) 

## <a name="label-your-example-files"></a>Добавление меток к примерам файлов

После добавления файлов примеров необходимо добавить их в качестве положительных или отрицательных примеров.

1. На домашней странице модели в разделе **Классификация файлов и запуск учебной** плитки щелкните элемент **классификатор**.
   При этом отобразится страница метки со списком файлов с примерами, где первый файл будет виден в средстве просмотра.
2. В средстве просмотра, расположенном в верхней части первого файла примера, вы увидите сообщение с вопросом, является ли этот файл примером модели, которую вы только что создали. Если это положительный пример, нажмите **кнопку Да**. Если это отрицательный пример, выберите **нет**.
3. В списке **примеры с заголовками** слева выберите дополнительные файлы, которые необходимо использовать в качестве примеров, а также пометьте их. 

    ![Домашняя страница модели классификатора](../media/content-understanding/classifier-home-page.png) 


> [!Note]
> Подпись по крайней мере из пяти положительных примеров и один отрицательный пример. 

## <a name="create-an-explanation"></a>Создание объяснения

Следующий шаг — создание объяснения на странице "обучение".  Объяснение — это подсказка или подсказка, которая поможет модели понять, как распознать этот документ. Например, документы для продления контракта всегда содержат *запрос на дополнительную строку с раскрытием* текста.

> [!Note]
> При использовании с извлечением объяснение используется для идентификации строки, которую необходимо извлечь из документа. 

Чтобы создать объяснение:

1. На домашней странице модели перейдите на вкладку **обучение** , чтобы перейти на страницу обучение.
2. На странице обучение в разделе " **обученные файлы** " вы увидите список примеров файлов, помеченных ранее. Выберите из списка один из ваших положительных файлов, который будет отображаться в средстве просмотра.
3. В разделе объяснение нажмите кнопку **создать**, а затем — **пустое**.
4. На странице **Создание объяснения** :</br>
    а. Введите **имя** (например, "блок раскрытия").</br>
    б. Выберите **тип**. В нашем примере мы будем выбирать **список фраз**, так как добавляется текстовая строка.</br>
    в. В поле **Введите здесь** введите строку.  В нашем примере мы добавим "запрос на дополнительную раскрытие". Вы можете выбрать параметр **с учетом регистра** , если строка должна учитывать регистр.</br>
    г. Щелкните **Сохранить**.

    ![Создание объяснения](../media/content-understanding/explanation.png) 
    
 
5.  Теперь модель проверит, было ли созданное вами объяснение достаточно хорошо, чтобы правильно определить промаркированные примеры файлов, как положительные и отрицательные. В разделе "подготовленные файлы" Проверьте столбец **оценки** после завершения обучения, чтобы увидеть результаты.  Если созданное объяснение было достаточно для того, чтобы оно было было отмечено как (положительное или отрицательное), файлы будут отображаться **соответствующим** образом.

    ![Создание объяснения](../media/content-understanding/match.png) 

Если вы получили **несовпадение** с подписанными файлами, вам может потребоваться создать дополнительное пояснение, чтобы предоставить модели дополнительные сведения для определения типа документа. Вы можете щелкнуть файл, чтобы получить дополнительные сведения о причине несоответствия.

## <a name="test-your-model"></a>Тестирование модели

Если вы получили соответствующее приложение с помеченными примерами файлов, теперь вы можете проверить модель на оставшихся файлов примеров без метки.

1. На домашней странице модели перейдите на вкладку **тест** .  Это приведет к запуску модели для файлов примеров без метки.
2. В списке **тестовые файлы** будут отображаться файлы, которые будут отображаться, и отобразится, если модель прогнозируется как положительные или отрицательные примеры. Вы можете использовать эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

    ![Проверка непомеченных файлов](../media/content-understanding/test-on-files.png)
