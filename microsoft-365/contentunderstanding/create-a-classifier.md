# <a name="create-a-classifier-in-microsoft-sharepoint-syntex"></a>Создание классификатора в Microsoft SharePoint Синтекс

Содержимое этой статьи предназначено для проекта Кортекс Private Preview. [Узнайте больше о Кортекс Project](https://aka.ms/projectcortex).

</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4CL0R]  

</br>

Классификатор — это тип модели, который можно использовать для автоматизации идентификации и классификации типов документов. Например, вы можете определить все документы для *продления контрактов* , добавленные в библиотеку документов, например, показанные на следующем рисунке.

![Документ по обновлению контракта](../media/content-understanding/contract-renewal.png)

Создание классификатора позволяет создать новый [тип контента SharePoint](https://docs.microsoft.com/sharepoint/governance/content-type-and-workflow-planning#content-type-overview) , который будет связан с моделью.

При создании классификатора необходимо создать *объяснения* для определения модели. Это позволяет заметок о распространенных данных, которые будут ожидать согласованного поиска этого типа документа. 

Используйте примеры типа документа ("example Files"), чтобы "обучить" модель для идентификации файлов с одинаковым типом контента.

Чтобы создать классификатор, необходимо выполнить следующие действия:
1. Назовите модель.
2. Добавьте файлы примеров.
3. Пометьте примеры файлов.
4. Создание объяснения.
5. Протестируйте модель.

> [!NOTE]
> Пока модель использует классификатор для определения и классификации типов документов, вы также можете получить определенные фрагменты данных из каждого файла, идентифицируемого моделью. Для этого создайте средство **извлечения** , которое будет добавлено в модель. Ознакомьтесь [со](create-an-extractor.md)статьей Создание средства извлечения.

## <a name="name-your-model"></a>Назовите модель

Первым этапом создания модели является присвоение ему имени:

1. В центре управления контентом выберите **создать**, а затем **Создайте модель**.
2. В области **модель нового документа "Общие сведения** " в поле **имя** введите имя модели. Например, если вы хотите определить документы для продления контрактов, можно назвать *Обновление контракта*модели.
3. Нажмите кнопку **Создать**. При этом будет создана домашняя страница модели.</br>

    ![Домашняя страница модели классификатора](../media/content-understanding/model-home.png)

При создании модели также создается новый тип контента SharePoint. Тип контента SharePoint представляет категорию документов, имеющих общие характеристики и совместное использование коллекции столбцов или свойств метаданных для конкретного содержимого. Управление типами контента SharePoint осуществляется с помощью [коллекции типов контента](https://support.microsoft.com/office/create-or-customize-a-site-content-type-27eb6551-9867-4201-a819-620c5658a60f). В этом примере при создании модели создается новый тип контента *обновления контракта* .

Выберите **Дополнительные параметры** , если вы хотите сопоставить эту модель с существующим типом контента в коллекции типов контента SharePoint, чтобы использовать ее схему. Обратите внимание, что хотя вы можете использовать существующий тип контента для использования его схемы, чтобы упростить идентификацию и классификацию, вам по-прежнему нужно провести обучение модели, чтобы извлечь информацию из файлов, которые она идентифицирует.</br>

![Дополнительные параметры](../media/content-understanding/advanced-settings.png)

## <a name="add-your-example-files"></a>Добавление файлов примеров

На домашней странице модели добавьте файлы примеров, которые понадобятся вам при обучении модели, чтобы определить тип документа. </br>
</br>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4D0iX] 

</br>

> [!NOTE]
> Вы должны использовать одни и те же файлы для обучения классификатора и [извлечения](create-an-extractor.md). Вы всегда можете добавить более позднее, но обычно вы добавляете полный набор образцов файлов. Пометьте, что нужно обучить модель, и протестируйте оставшиеся непомеченные, чтобы оценить фитнес модели. 

Для вашего учебного набора вы хотите использовать как положительные, так и отрицательные примеры:
- Положительный пример: документы, представляющие тип документа. Они содержат строки и сведения, которые всегда будут находиться в документе этого типа.
- Негативный пример: документы, не представляющие тип документа. В них отсутствуют строки и сведения, которые должны присутствовать в документе этого типа.

Обязательно используйте по крайней мере пять положительных примеров и по крайней мере один отрицательный пример для обучения модели.  Вы хотите создать дополнительные единицы, чтобы протестировать модель после учебного процесса.

Добавление файлов примеров:

1. На домашней странице модели в плитке **Библиотека образцов** нажмите кнопку **Добавить файлы**.
2. На странице **Выбор примеров файлов для модели** выберите файлы примера из библиотеки образцов файлов в центре управления контентом. Если вы еще не отправляли их там, нажмите кнопку **Отправить** , чтобы переместить их в библиотеку образцов файлов.
3. Выбрав файлы примеров, которые необходимо использовать для обучения модели, нажмите кнопку **Добавить**.

    ![Выбор файлов примеров](../media/content-understanding/select-sample.png) 

## <a name="label-your-example-files"></a>Добавление меток к примерам файлов

После добавления файлов примеров их необходимо добавить в качестве положительного или отрицательного примера.

1. На домашней странице модели в разделе **Классификация файлов и запуск обучающих материалов** нажмите кнопку **обучение классификатора**.
   Откроется страница метки со списком файлов с примерами, где первый файл виден в средстве просмотра.
2. В средстве просмотра, расположенном в верхней части первого файла примера, вы увидите вопрос, является ли он примером модели, которую вы только что создали. Если это положительный пример, нажмите **кнопку Да**. Если это отрицательный пример, выберите **нет**.
3. В списке **помеченных примеров** слева выберите дополнительные файлы, которые нужно использовать в качестве примеров, и пометьте их. 

    ![Главная страница классификатора](../media/content-understanding/classifier-home-page.png) 


> [!NOTE]
> Подпись по крайней мере из пяти положительных примеров и один отрицательный пример. 

## <a name="create-an-explanation"></a>Создание объяснения

Следующий шаг предназначен для создания объяснения на странице "обучение". Объяснение помогает модели понять, как распознать документ. Например, документы для продления контракта всегда содержат *запрос на дополнительную строку с раскрытием* текста.

> [!Note]
> При использовании с извлечением объяснение определяет строку, которую необходимо извлечь из документа. 

Чтобы создать объяснение:

1. На домашней странице модели выберите вкладку **обучение** , чтобы перейти на страницу обучение.
2. На странице "обучение" в разделе " **подготовленные файлы** " вы увидите список образцов файлов, помеченных ранее. Выберите один из положительных файлов в списке и отобразится в средстве просмотра.
3. В разделе объяснение выберите **создать** , а затем — **пустое**.
4. На странице **Создание объяснения** :</br>
    а. Введите **имя** (например, "блок раскрытия").</br>
    б. Выберите **тип**. Для примера выберите пункт **список фраз**, так как вы добавите текстовую строку.</br>
    в. В поле **Введите здесь** введите строку. Для примера добавьте "запрос на дополнительную раскрытие". Вы можете выбрать параметр **с учетом регистра** , если строка должна учитывать регистр.</br>
    г. Нажмите кнопку **Сохранить**.

    ![Создание объяснения](../media/content-understanding/explanation.png) 
    
 
5. Теперь модель проверяет, было ли созданное вами объяснение достаточно хорошо, чтобы правильно определить оставшиеся помеченные файлы примеров, как положительные и отрицательные примеры. В разделе ученые файлы проверьте столбец **Оценка** после завершения обучения, чтобы увидеть результаты. В файлах отображается значение " **совпадает**", если созданные вами объяснения были достаточно, чтобы соблюдались как положительные, так и отрицательные.

    ![Значение параметра ПОИСКПОЗ](../media/content-understanding/match.png) 

Если вы получаете **несовпадение** с помеченными файлами, вам может потребоваться создать дополнительное пояснение, чтобы предоставить модели дополнительные сведения для определения типа документа. Если это произойдет, щелкните файл, чтобы получить дополнительные сведения о причине несоответствия.

## <a name="test-your-model"></a>Тестирование модели

Если вы получили соответствующее помеченные файлы примеров, теперь вы можете проверить модель на оставшихся файлов примеров без метки.

1. На домашней странице модели перейдите на вкладку **тест** .  В этом примере выполняется модель для файлов примеров без метки.
2. В списке **тестовые файлы** показаны примеры файлов и показывается, если модель прогнозируется как положительное или отрицательное. Используйте эти сведения, чтобы определить эффективность классификатора в определении ваших документов.

    ![Проверка непомеченных файлов](../media/content-understanding/test-on-files.png)
