# <a name="microsoft-sharepoint-syntex-adoption-get-started"></a>Внедрение Microsoft SharePoint Синтекс: Начало работы

Вы считаете, что интеллектуальные службы контента, доступные в Microsoft SharePoint Синтекс, имеют три части:

- **Общие сведения о содержимом:** создание AI-моделей без кода для классификации и извлечения данных из контента для автоматического применения метаданных для обнаружения и повторного использования знаний. Узнайте больше о [содержимом](document-understanding-overview.md).
- **Обработка контента:** Автоматизация сбора, приема и классификации контента и упрощения процессов, ориентированных на контент, с помощью автоматизации энергопотребления. Узнайте больше об [обработке контента](form-processing-overview.md).
- **Соответствие содержимого:** Управление контентом и управление им для усовершенствования безопасности и управления с интеграцией в Microsoft Information Protection.

С помощью новых служб и возможностей AI можно создавать общие сведения и классификации приложений непосредственно в потоки управления контентом с помощью Microsoft SharePoint Синтекс:

|Ввод вручную| Обработка форм | Общие сведения о документе |
|:-------|:--------|:--------|
| Ввод данных и трудовые ресурсы для любого содержимого | Обработка цифрового контента — фотографий, сканирований, чеков, визитных карточек, видеороликов с OCR & текст |  Сбор типов контента и метаданных из контрактов, резюме и других структурированных документов |
| Интерактивны   | Предварительно созданные, автоматизированные   | Настраиваемая, с поддержкой   | Настраиваемый, соответствует требованиям |
| Люди, выполняющие работу | Научились экспертами по темам (смес). Сбор типов контента и метаданных из контрактов, возобновления и других неструктурированных документов. | Смес менее задействованы. из заказов на покупку, приложений, а также других структурированных и структурированных документов |

Microsoft SharePoint Синтекс интегрируется со средствами обеспечения соответствия Microsoft 365, такими как:

- Метки хранения, определяющие политику записей на основе возраста или внешних событий.
- Метки конфиденциальности, которые задают политики защиты от потери данных, шифрования, совместного использования и условного доступа.

Пользователи могут применять метки или автоматически применять модели AI для SharePoint Синтекс. Аналитики и планы файлов обеспечивают масштабированное управление использованием и политиками меток.

## <a name="identify-pilot-business-scenarios-to-optimize"></a>Оценка пилотных бизнес-сценариев для оптимизации

Чтобы подготовиться к использованию Microsoft SharePoint Синтекс в Организации, сначала необходимо ознакомиться со сценариями, в которых она будет полезна. Почему помогает определить требуемую модель и как структурировать организационную модель в зависимости от того, где будет применена модель. Ниже приведено несколько сценариев, в которых понимание документов может помочь вашей организации:

- Обработка контента: контракты процессов, операторы работы и другие документы, подобные формам. Интаке формы, обучить модель для определения и сопоставления полей, а затем запустите формы для автоматического сбора данных. Дополнительные сведения [: Обзор обработки форм](form-processing-overview.md).
- Анализ счета: Извлекайте соответствующие сведения из счетов и убедитесь, что они соответствуют политике или обрабатываются правильно.

Подумайте о способах, с помощью которых Microsoft SharePoint Синтекс может помочь вашей организации:

- Автоматизация бизнес-процессов
- Улучшенная точность поиска
- Управление риском для обеспечения соответствия требованиям

### <a name="form-processing-scenario-example"></a>Пример сценария обработки формы

Например, вы можете настроить процесс, используя Microsoft SharePoint Синтекс и функции автоматизации Power, для отслеживания и мониторинга накладных.

1. Настройка библиотеки для хранения документов счетов.
1. Обучение модели для распознавания полей в документах.
1. Извлеките поля, которые необходимо отслеживать в списке.
1. Настройте последовательность для уведомления определенных событий, например:
    - Добавляется новый счет.
    - Срок выполнения накладной истек.
    - Счет — это сумма, превышающая сумму автоматического утверждения.

![Отслеживание и мониторинг накладных с помощью Microsoft SharePoint Синтекс и автоматизации управления питанием](../media/content-understanding/process-invoices-flow.png)

При автоматизации этого сценария можно выполнить указанные ниже действия.

- Экономия времени и денег путем автоматического извлечения данных из накладных вместо ручного извлечения данных.
- Уменьшение потенциальных ошибок и обеспечение лучшей совместимости с помощью рабочих процессов для выполнения накладных и уведомления о возникших проблемах.

### <a name="document-understanding-scenario-example"></a>Пример сценария "Общие сведения о документе"

В качестве другого примера можно настроить процесс определения контрактов, которые компания использует для других компаний или лиц. Вы можете настроить модель для извлечения ключевых данных из этих контрактов, таких как имя клиента, сборы, даты или другие важные сведения, и добавить их в библиотеку в качестве полей, которые можно быстро просмотреть. Вы также можете применить метку хранения в библиотеке документов, чтобы убедиться, что контракты невозможно удалить до определенного периода времени для соответствия нормативным требованиям.

1. Начните с центра контента и создайте новый документ общие сведения о модели для контрактов.
1. Отправьте образцы документов для положительных и отрицательных примеров, запустите обучение, чтобы определить документы контрактов и просмотреть результаты.
1. Изучите средство извлечения, чтобы определить поля в контрактах, такие как имя клиента, сборы и дата, а затем протестируйте средство извлечения.
1. По завершении модели примените модель к библиотеке, в которой можно отправлять контракты.
1. Примените метку хранения к полю Date (дата), чтобы контракты сохранялись в библиотеке в течение периода времени, требуемого для Организации контрактов.

![Отслеживание и отслеживание контрактов с помощью Microsoft SharePoint Синтекс и меток хранения](../media/content-understanding/process-contracts-flow.png)

При автоматизации этого сценария можно выполнить указанные ниже действия.

- Экономия времени и денег путем автоматического извлечения данных из контрактов, а не ручного извлечения данных.
- Обеспечьте лучшую совместимость, используя метки хранения, чтобы убедиться в том, что контракты хранятся соответствующим образом.

### <a name="tips-for-identifying-scenarios"></a>Советы по идентификации сценариев

При планировании бизнес-сценариев, которые следует учитывать, ответьте на следующие вопросы.

- Устранена ли проблема?
- Будет ли он широко использоваться или иметь широкое воздействие?
- Доступно?
- Можно измерять успех?

Определение приоритетов сценариев на основе влияния и простоты реализации. Сделайте исходную область фокуса более возможной, а также легко реализуемой. Отменяет приоритеты сценариев с низким влиянием, которые трудно реализовать.

## <a name="identify-roles--responsibilities"></a>Определение ролей & обязанностей

| Администратор SharePoint/базы знаний | Администратор платформы управления питанием | Диспетчер знаний | Владелец модели |

Руководитель базы знаний, владелец бизнес-процесса и владелец модели контента создают образцы моделей и внедрение лидера по в Организации.

Где они будут создавать и применять модели? Существуют ли уже существующие процессы или репозитории, которые можно было бы улучшить?

- Обработка форм: решите, какие сайты будут получать действие обработки формы.
- Общие сведения о документе: вы можете создать несколько центров контента для разных областей бизнеса.

## <a name="readiness-checklist"></a>Контрольный список готовности

Чтобы подготовиться к реализации Microsoft SharePoint Синтекс (Предварительная версия), необходимо выполнить следующие действия:

![Сведения о готовности контента](../media/content-understanding/cu-adoption-readinesschecklist.png)

1. Планирование конечного состояния
    - Общие сведения о моделях — это, а не конец.
    - Планирование использования значения извлеченных метаданных с:
      - Поиск
      - Фильтрация и форматирование представления
      - Соответствие требованиям
      - Автоматизация
2. Однозначно
    - Узнайте, как использовать существующую информационную архитектуру и функцию управления контентом.
    - Существуют какие типы контента, которые подходят для моделей?
    - Какие существующие процессы были усовершенствованы метаданными?
3. Разработка
    - Проектирование подхода к информационной архитектуре, управляемым метаданным и типам контента
    - Разработайте процесс для определения, создания, управления.
