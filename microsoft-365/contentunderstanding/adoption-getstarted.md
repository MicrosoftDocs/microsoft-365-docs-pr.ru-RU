# <a name="microsoft-sharepoint-syntex-adoption-get-started"></a>Microsoft SharePoint Syntex adoption: Get started

Думайте о том, что интеллектуальные службы контента, доступные в SharePoint Syntex, имеют три части:

- **Понимание контента:** создание моделей AI без кода для классификации и извлечения информации из контента для автоматического применения метаданных для обнаружения и повторного использования знаний. Узнайте больше о [понимании контента.](document-understanding-overview.md)
- **Обработка контента:** Автоматизация захвата, ingestion и классификации контента и оптимизация процессов, ориентированных на контент, с помощью Power Automate. Дополнительные материалы об [обработке контента.](form-processing-overview.md)
- **Соответствие требованиям контента:** Управление и управление контентом для улучшения безопасности и управления с помощью интеграции с Microsoft Information Protection.

С помощью новых служб и возможностей ИИ можно создавать приложения для понимания контента и классификации непосредственно в поток управления контентом с помощью SharePoint Syntex. Существует два различных способа понимания контента. Тип модели, который вы используете, основан на формате файла и случае использования:

| Обработка форм | Понимание документов |
|:-------|:-------|
| Создан из библиотеки документов. | Созданная в центре контента часть SharePoint Syntex. |
| Модель, созданная в конструкторе ИИ. | Модель, созданная в родном интерфейсе. |
| Используется для полуструктурных форматов файлов. | Используется для неструктурированных форматов файлов. |
| Классификатор settable. | Классификатор с необязательными экстракторами. |
| Ограничен одной библиотекой. | Может применяться к нескольким библиотекам. |
| Обучение в формате PDF, JPG, PNG общей стоимостью 50 МБ/500 pp. | Тренируйся по 5-10 PDF-файлам, Office или электронной почте, включая отрицательные примеры. |

Для более полного сравнения возможностей см. разницу между пониманием документов и [моделями обработки форм.](difference-between-document-understanding-and-form-processing-model.md)

## <a name="identify-pilot-business-scenarios-to-optimize"></a>Определение пилотных бизнес-сценариев для оптимизации

Чтобы подготовиться к использованию SharePoint Syntex в организации, сначала необходимо понять сценарии, в которых это будет полезно. "Почему" помогает определить, какая модель будет нужна, и как структурировать свою организацию на основе того, где модель будет применяться. Вот несколько сценариев, в которых понимание документов может помочь организации:

- **Обработка контента:** Обработка контрактов, отчетов о работе и других документов, похожих на форму. Включите формы, обучите модель понимать поля и сопобирать их, а затем запустите формы, чтобы автоматически собирать данные. Дополнительные сведения см. в [обзоре обработки форм.](form-processing-overview.md)
- **Анализ счетов:** Извлеките соответствующие сведения из счетов и убедитесь, что они соответствуют политике или обрабатываются надлежащим образом.

Подумайте о том, как SharePoint Syntex может помочь вашей организации:

- Автоматизация бизнес-процессов
- Повышение точности поиска
- Управление рисками соответствия требованиям

При рассмотрении бизнес-сценариев задайте себе следующие вопросы:

- Решает ли это реальную проблему?
- Будет ли он широко использоваться или будет иметь широкое влияние?
- Можно ли получить его?
- Можете ли вы измерить успех?

Расставить приоритеты сценариев на основе влияния и простоты реализации. Сделайте начальную область фокуса более высокими сценариями воздействия, которые также можно легко реализовать. Расставить приоритеты при снижении приоритетов сценариев воздействия, которые трудно реализовать.

Используйте следующие сценарии, чтобы подсказыть, как можно использовать SharePoint Syntex в организации.

### <a name="scenario-track-data-from-invoices-with-form-processing"></a>Сценарий: отслеживание данных из счетов-фактур с обработкой форм

Например, вы можете настроить процесс с помощью функций SharePoint Syntex и Power Automate для отслеживания и мониторинга счетов-фактур.

1. Настройка библиотеки для хранения документов-фактур.
1. Обучение модели распознаванию полей в документах.
1. Извлечение полей, которые необходимо отслеживать в списке.
1. Настройка потока для уведомления о определенных событиях, таких как:
    - Добавлен новый счет.
    - Счет-фактура засвечен.
    - Счет за сумму, которая превышает автоматическую сумму утверждения.

![Отслеживание и отслеживание счетов с помощью SharePoint Syntex и Power Automate](../media/content-understanding/process-invoices-flow.png)

При автоматизации этого сценария можно:

- Экономите время и деньги, автоматически извлекая данные из счетов, а не вручную.
- Уменьшение потенциальных ошибок и обеспечение соответствия требованиям с помощью рабочего процесса для проверки счетов и уведомления о любых проблемах.

### <a name="scenario-track-information-from-contracts-with-document-understanding"></a>Сценарий: отслеживание сведений из контрактов с пониманием документов

В качестве другого примера можно настроить процесс определения контрактов, заключаемого вашей компанией с другими компаниями или отдельными лицами. Настройка модели для извлечения ключевых сведений из этих контрактов, таких как имя клиента, сборы, даты или другие важные сведения, а также добавление сведений в библиотеку в качестве полей, которые можно быстро просмотреть. Нанесите метку хранения в библиотеку документов, чтобы убедиться, что контракты не могут быть удалены до определенного времени для соответствующего соблюдения бизнес-правил.

1. Начните с центра контента и создайте новую модель понимания документов для контрактов.
1. Загрузите примеры документов для положительных и отрицательных примеров, а затем запустите обучение, чтобы определить контрактные документы и просмотреть результаты.
1. Обучаем извлечения для определения полей в контрактах, таких как имя клиента, плата и дата, а затем протестировать экстрактор.
1. По завершению модели примените модель к библиотеке, в которой можно загрузить контракты.
1. Нанесите метку хранения на поле дат, чтобы контракты сохранялись в библиотеке в течение необходимого времени.

![Отслеживайте и отслеживайте контракты с Метки SharePoint Syntex и хранения](../media/content-understanding/process-contracts-flow.png)

При автоматизации этого сценария можно:

- Экономите время и деньги, автоматически извлекая данные из контрактов, а не вручную.
- Обеспечение соответствия требованиям с помощью меток хранения для обеспечения надлежащего хранения контрактов.

### <a name="scenario-avoid-risk-with-records-management-document-governance-and-compliance-processes-based-on-sharepoint-syntex"></a>Сценарий: избегайте рисков при управлении записями, управлении документами и процессах соответствия требованиям на основе SharePoint Syntex

Снижение рисков является общей целью для большинства компаний. Возможно, потребуется:

- Лучший способ обеспечения и обеспечения информационного управления в клиенте.
- Усовершенствовать систему классификации документов, электронных писем и других форм связи, которые считаются "записями" для проектов.
- Аудит квитанций, контрактов и так далее для обеспечения соответствия политикам компании.
- Чтобы убедиться, что у проектов есть вся документация, необходимая для соответствия требованиям.

Настройка некоторых процессов для соответствия требованиям SharePoint Syntex для захвата и надлежащей классификации, аудита и флага документов и форм, которые требуют более эффективного управления. Вы можете полагаться на SharePoint Syntex, чтобы автоматически классифицировать контент, а не полагаться на то, что конечные пользователи будут вручную тегировать или команда соответствия требованиям вручную применять правила управления и архивировать. Вы также можете включить упрощенный режим поиска, управлять объемами данных, применять политики управления записями и хранения, обеспечивать соответствие требованиям, а также применять методы архива и чистки.

Когда вы автоматизируете этот сценарий, вы можете чувствовать себя в безопасности, что:

- Соответствие требованиям поддерживаются, а риск снижается.
- Управление таксономией и записями применяется последовательно и точно.
- Контролируются объемы контента.
- Сотрудники могут легко найти нужные сведения в нужном контексте.

### <a name="scenario-capture-information-from-previously-inaccessible-documents"></a>Сценарий: захват сведений из ранее недоступных документов

Большинство организаций имеют большие репозитории юридических документов, политик, контрактов, кадровой документации и руководства по управлению. Мои эти хранилища данных для извлечения ценных сведений, таких как: проекты, сектора, темы, люди, географические области и так далее.

Например, директору отдела кадров необходимо быстро получить доступ ко всем hr-документам, включая резюме, политики кадров и другие формы. Кроме того, они хотят быстро определить необходимые сведения из резюме и других документов, связанных с персоналом, без ручного сеяния документов. Они ищут решение, которое позволяет им быстро находить необходимую информацию без необходимости вручную искать тысячи резюме, политики кадров и другую документацию, которая может быть распространена на нескольких сайтах.

При автоматизации этого сценария можно:

- Разблокировка знаний из цифрового контента.
- Классифицировать политики, резюме, документы о продажах, технические чертежи, планы учетных записей и извлечение сведений.
- Быстро найдите правильные сведения или документ, которые вы ищете.
- Получите мгновенный доступ к последней информации.
- Сократите время поиска.

### <a name="scenario-improve-data-processing-to-provide-insights--analytics"></a>Сценарий. Улучшение обработки данных для предоставления & аналитики

Например, фармацевтическая компания может использовать SharePoint Syntex для извлечения информации из документов FDA для ответа на вопросы, которые есть у их руководителей. Более легкодоступные ответы могут сократить время, необходимое для получения этих ответов, и повысить доступность данных для получения более точных ответов на вопросы руководства.

Например, руководитель проекта должен быстро предоставить ответы на вопросы, связанные с продуктом, из моей команды руководства. Им необходимо найти сведения и метрики, связанные с запросами в одной консолидированной панели мониторинга. Они ищут решение, которое извлекает необходимую информацию из меток продуктов, брошюр продуктов и других материалов и создает консолидированный отчет, который они могут использовать при отчитывающейся группе руководства.

При автоматизации этого сценария можно:

- Сократите время для получения ответов.
- Повышение доступности данных.
- Предоставление более точных ответов.

### <a name="scenario-automate-order-processing"></a>Сценарий: автоматизация обработки заказов

Благодаря SharePoint Syntex можно сократить время ручной обработки заказов клиентов. Например, вы можете загружать заказы из факса, электронной почты или бумаги в SharePoint с помощью обработки OCR, а затем извлекать метаданные из этих заказов, чтобы вы могли выполнять их с помощью автоматизированных процессов.

Например, руководитель цепочки поставок хочет уменьшить количество ошибок, вызванных вводом данных вручную. Они хотят избегать ручного просмотра и ввода данных входящие клиенты (бумажные, факсимиловые или электронные сообщения), чтобы уменьшить количество ошибок при входе в бизнес-системы. Они хотят, чтобы решение, применяее методы ИИ и машинного обучения, проверяли входящие сведения о порядке, извлекали основные данные и автоматически нажимали их в свою систему ERP для выполнения и согласования заказов.

При автоматизации этого сценария можно убедиться, что:

- Повышается точность заказа и отгрузки.
- Уменьшаются сборы или штрафы, связанные с ошибками заказа или отправки.
- Задержки в взимание или оплата уменьшаются.
- Сокращаются затраты на персонал.

### <a name="scenario-simplify-visa-renewal-process"></a>Сценарий: упрощение процесса продления визы

SharePoint Syntex поможет вам автоматизировать напоминания и обновления для ключевых сведений о контракте. Например, директору отдела кадров необходимо убедиться, что визы сотрудников обновляются и/или обновляются вовремя. Они хотят предоставить людям простой и интуитивно понятный процесс обновления виз. Им нужно решение, которое извлекает даты продления из контрактов и автоматически отправляет напоминания сотрудникам, когда приближаются даты их продления.

При автоматизации этого сценария можно убедиться, что:

- Снижается уровень несоблюдения.
- Количество ручных напоминаний уменьшается.
- Уменьшается количество штрафов за несоблюдение.

## <a name="readiness-checklist"></a>Контрольный список готовности

Чтобы быть готовым к реализации SharePoint Syntex, необходимо:

![Готовность к пониманию контента](../media/content-understanding/cu-adoption-readinesschecklist.png)

1. Планирование конечных состояния
    - Модели понимания документов — это средства, а не конец.
    - Планирование использования значения извлеченных метаданных с помощью:
      - Поиск
      - Фильтрация и форматирование представлений
      - Соответствие требованиям
      - Автоматизация
2. Определение
    - Сведения о существующей информационной архитектуре и использовании функций управления контентом.
    - Являются ли существующие типы контента хорошими кандидатами для моделей?
    - Какие существующие процессы будут улучшены с помощью метаданных?
