# <a name="microsoft-sharepoint-syntex-adoption-get-started"></a>Внедрение microsoft SharePoint Syntex: начало работы

Интеллектуальные службы контента, доступные в синтаксисе SharePoint, считаются тремя частями:

- **Понимание содержимого:** создайте модели ИИ без кода для классификации и извлечения информации из контента, чтобы автоматически применять метаданные для обнаружения и повторного использования знаний. Узнайте больше о [содержимом.](document-understanding-overview.md)
- **Обработка контента:** Автоматизация захвата, присоединения и классификации контента, а также упрощение процессов, ориентированных на контент, с помощью Power Automate. Узнайте больше об [обработке контента.](form-processing-overview.md)
- **Соответствие контенту:** Контроль и управление контентом для повышения безопасности и управления с помощью интеграции с Microsoft Information Protection.

Благодаря новым службам и возможностям ИИ вы можете создавать приложения для понимания контента и классификации непосредственно в потоке управления содержимым с помощью синтаксита SharePoint. Существует два разных способа понимания содержимого. Тип используемой модели зависит от формата файла и случая использования:

| Обработка форм | Понимание документа |
|:-------|:-------|
| Создается из библиотеки документов. | Создается в центре содержимого, в составе Синтаксис SharePoint. |
| Модель, созданная в построителье ИИ. | Модель, созданная в пользовательском интерфейсе. |
| Используется для полуструктурных форматов файлов. | Используется для неструктурированных форматов файлов. |
| Классификатор settable. | Обучаемый классификатор с необязательными извлечениями. |
| Ограничен одной библиотекой. | Может применяться к нескольким библиотекам. |
| Обучение в формате PDF, JPG, PNG, всего 50 МБ/500 pp. | Обучись 5–10 PDF-файлам, Office или файлам электронной почты, включая отрицательные примеры. |

В следующей таблице рассказывается о доступности и лицензировании для Синтаксис SharePoint:

| Обработка форм | Понимание документа |
|:-------|:-------|
| Обработка форм зависит от Платформы Power Platform. <br>Сведения о глобальной доступности для Power Platform и построитель ИИ см. в сведениях [о доступности Power Platform.](https://dynamics.microsoft.com/geographic-availability/) | Доступно во всех регионах. |
| Использует кредиты построщика ИИ.<br>Кредиты можно приобрести пакетами в 1 млн.<br>При приобретении 300 и более лицензий на синтаксис SharePoint включаются 1M-кредиты.<br>1M-кредиты позволяют обрабатывать 2000 страниц файлов. | Модели работают на всех языках латинских алфавитов. Помимо английского: немецкий, шведский, французский, испанский, итальянский и португальский. |
| Подготовка к общей среде службы данных по умолчанию. | Не имеет ограничений емкости. |

Дополнительные сведения о кредитах и единицах построитель ИИ см. в лицензировании [построитель ИИ.](https://docs.microsoft.com/ai-builder/administer-licensing)

Синтаксис SharePoint интегрируется с функциями соответствия требованиям Microsoft 365, например:

- Метки хранения, которые определяют политику записей на основе возраста документа или внешних событий.
- Метки конфиденциальности, которые устанавливают политики DLP, шифрования, общего доступа и условного доступа.

Пользователи могут применять метки или автоматически применять их с помощью моделей ИИ синтаксис SharePoint. Аналитика и планы файлов обеспечивают масштабное управление использованием меток и политиками.

## <a name="identify-pilot-business-scenarios-to-optimize"></a>Определение пилотных бизнес-сценариев для оптимизации

Чтобы подготовиться к использованию синтаксита SharePoint в организации, сначала необходимо понять сценарии, в которых он будет полезен. Причины, по которым можно определить, какая модель потребуется, и как структурировать организацию в зависимости от того, где будет применяться модель. Вот несколько сценариев, в которых понимание документов может помочь вашей организации.

- Обработка контента: контракты обработки, заявления о работе и другие документы, похожие на формы. Вберите формы, научите модель понимать и сопобирать поля, а затем запустите формы, чтобы автоматически собрать данные. Дополнительные сведения [см. в обзоре обработки форм.](form-processing-overview.md)
- Анализ счета: извлеките соответствующие сведения из своих счетов и убедитесь, что они соответствуют политике или обрабатываются соответствующим образом.

Подумайте о том, как синтаксис SharePoint может помочь вашей организации:

- Автоматизация бизнес-процессов
- Повышение точности поиска
- Управление рисками соответствия требованиям

### <a name="form-processing-scenario-example"></a>Пример сценария обработки форм

Например, можно настроить процесс с помощью функций Синтаксис SharePoint и Power Automate для отслеживания и отслеживания накладных.

1. Настройка библиотеки для хранения документов накладных.
1. Обучение модели распознаванию полей в документах.
1. Извлекать поля, которые нужно отслеживать, в список.
1. Настройка потока для уведомления о конкретных событиях, таких как:
    - Добавляется новый счет.
    - Срок действия счета-фактуры устарел.
    - Счет-фактура на сумму, которая превышает сумму автоматического утверждения.

![Отслеживание и отслеживание накладных с помощью Синтаксис SharePoint и Power Automate](../media/content-understanding/process-invoices-flow.png)

Автоматизировать этот сценарий можно:

- Экономите время и деньги, автоматически извлекая данные из накладных, а не вручную.
- Уменьшите количество потенциальных ошибок и обесплатите соответствие требованиям с помощью рабочего процесса, чтобы действовать на счетах и уведомлять вас о любых проблемах.

### <a name="document-understanding-scenario-example"></a>Пример сценария с пониманием документа

В качестве другого примера можно настроить процесс определения договоров, которые имеются у вашей компании с другими компаниями или отдельными лицами. Можно настроить модель для извлечения ключевых сведений из этих контрактов, таких как имя клиента, стоимость, даты и другие важные сведения, и добавить их в библиотеку в качестве полей, которые можно быстро просмотреть. Вы также можете применить метку хранения к библиотеке документов, чтобы гарантировать, что контракты не могут быть удалены до определенного времени для соответствующего соответствия бизнес-нормам.

1. Начните с центра содержимого и создайте новую модель понимания документов для контрактов.
1. Загрузите образцы документов для положительных и отрицательных примеров, а затем запустите обучение, чтобы определить контрактные документы и просмотреть результаты.
1. Обучаем извлечения определять поля в контрактах, такие как имя клиента, комиссия и дата, а затем протестировать извлечения.
1. После завершения работы модели примените модель к библиотеке, в которой можно отправить контракты.
1. Примените метку хранения к полю даты, чтобы контракты хранилися в библиотеке в течение времени, необходимого организации для контрактов.

![Отслеживание и мониторинг контрактов с помощью синтаксита SharePoint и меток хранения](../media/content-understanding/process-contracts-flow.png)

Автоматизировать этот сценарий можно:

- Экономите время и деньги, автоматически извлекая данные из контрактов, а не вручную.
- Чтобы обеспечить соответствие требованиям, с помощью меток хранения, убедитесь, что контракты сохраняются соответствующим образом.

### <a name="tips-for-identifying-scenarios"></a>Советы по идентификации сценариев

При рассмотрении бизнес-сценариев задайте себе следующие вопросы:

- Решает ли она реальную проблему?
- Будет ли он широко использоваться или иметь широкое влияние?
- Можно ли получить его?
- Можно ли оценить успешность?

Расставить приоритеты сценариев на основе влияния и простоты реализации. Сделайте начальную область фокуса более сильной, которую также можно легко реализовать. Сценарии понижения приоритетов, которые сложно реализовать.

## <a name="identify-roles--responsibilities"></a>Определение ролей & обязанностей

Определите, кто в вашей организации будет создавать модели и управлять ими? Могут быть задействованы следующие роли:

| Администратор SharePoint/Знаний | Администратор Power Platform | Диспетчер знаний | Владелец модели |
|:-------|:-------|:-------|:-------|
| Роль AAD| Роль ADD | Роль AAD | Лидеры |
| Настройка обработки форм | Настройка общей среды службы данных для обработки форм | Сбор случаев использования | Сбор случаев бизнес-использования |
| Управление центрами содержимого и разрешениями| Приобретение и выделение кредитов AIB | Создание практических методик и анализ моделей проверки | Создание и применение моделей |

Менеджер знаний, владелец владельца бизнес-процесса и владелец модели контента создают образцы моделей и внедрение в организации по-разному.
Другие, которые могут быть вовлечены: администратор соответствия требованиям, менеджеры таксономии.

Где будут создаваться и применяться модели? Существуют ли существующие процессы или репозитории, которые можно улучшить?

- Обработка форм: определите, какие сайты будут получать действие обработки форм.
- Понимание документа. Вы можете создать несколько центров контента для различных бизнес-областей.

## <a name="readiness-checklist"></a>Контрольный список готовности

Чтобы быть готовым к внедрению синтаксита SharePoint, необходимо выполнить:

![Готовность к пониманию контента](../media/content-understanding/cu-adoption-readinesschecklist.png)

1. Планирование состояния окончания
    - Модели понимания документов — это средство, а не конец.
    - Запланируйте использование значения извлеченных метаданных с помощью:
      - Поиск
      - Фильтрация и форматирование представлений
      - Соответствие требованиям
      - Автоматизация
2. Определение
    - Сведения о существующей информационной архитектуре и использовании функций управления контентом.
    - Являются ли существующие типы контента хорошими кандидатами для моделей?
    - Какие существующие процессы будут улучшены с помощью метаданных?
3. Разработка
    - Проектирование подхода к информационной архитектуре, управляемым метаданным и типам контента
    - Разработать процесс определения, создания и управления.
