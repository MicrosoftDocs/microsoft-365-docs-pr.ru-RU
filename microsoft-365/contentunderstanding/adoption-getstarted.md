# <a name="microsoft-sharepoint-syntex-adoption-get-started"></a>Microsoft SharePoint Syntex: начало работы

Подумайте, что интеллектуальные службы контента, доступные в SharePoint Syntex, имеют три части:

- **Понимание контента:** Создайте модели AI без кода, чтобы классифицировать и извлекать информацию из контента, чтобы автоматически применять метаданные для обнаружения и повторного использования знаний. Узнайте больше о [понимании контента.](document-understanding-overview.md)
- **Обработка контента:** Автоматизация захвата, ingestion и классификации контента и оптимизация процессов, ориентированных на контент, с помощью Power Automate. Дополнительные материалы об [обработке контента.](form-processing-overview.md)
- **Соответствие требованиям контента:** Управление и управление контентом для улучшения безопасности и управления с помощью интеграции с Microsoft Information Protection.

С помощью новых служб и возможностей ИИ можно создавать приложения для понимания контента и классификации непосредственно в поток управления контентом SharePoint Syntex. Существует два различных способа понимания контента. Тип модели, который вы используете, основан на формате файла и случае использования:

| Обработка форм | Осмысление документации |
|:-------|:-------|
| Создан из библиотеки документов. | Созданная в центре контента часть SharePoint Syntex. |
| Модель, созданная в конструкторе ИИ. | Модель, созданная в родном интерфейсе. |
| Используется для полуструктурных форматов файлов. | Используется для неструктурированных форматов файлов. |
| Классификатор settable. | Классификатор с необязательными экстракторами. |
| Ограничен одной библиотекой. | Может применяться к нескольким библиотекам. |
| Обучение в формате PDF, JPG, PNG общей стоимостью 50 МБ/500 pp. | Обучение на 5–10 PDF-файлах, файлах Office или электронной почты, в том числе отрицательные примеры. |

Для более полного сравнения возможностей см. разницу между пониманием документов и [моделями обработки форм.](difference-between-document-understanding-and-form-processing-model.md)

## <a name="identify-pilot-business-scenarios-to-optimize"></a>Определение пилотных бизнес-сценариев для оптимизации

Чтобы подготовиться к использованию SharePoint Syntex в организации, сначала необходимо понять сценарии, в которых он будет полезен. "Почему" помогает определить, какая модель будет нужна, и как структурировать свою организацию на основе того, где модель будет применяться. Вот несколько сценариев, в которых понимание документов может помочь организации:

- **Обработка контента:** Обработка контрактов, отчетов о работе и других документов, похожих на форму. Включите формы, обучите модель понимать поля и сопобирать их, а затем запустите формы, чтобы автоматически собирать данные. Дополнительные сведения см. в [обзоре обработки форм.](form-processing-overview.md)
- **Анализ счетов:** Извлеките соответствующие сведения из счетов и убедитесь, что они соответствуют политике или обрабатываются надлежащим образом.

Подумайте, как SharePoint Syntex может помочь вашей организации:

- Автоматизация бизнес-процессов
- Повышение точности поиска
- Управление рисками соответствия требованиям

При рассмотрении бизнес-сценариев задайте себе следующие вопросы:

- Решает ли это реальную проблему?
- Будет ли он широко использоваться или будет иметь широкое влияние?
- Можно ли получить его?
- Можете ли вы измерить успех?

Расставить приоритеты сценариев на основе влияния и простоты реализации. Сделайте начальную область фокуса более высокими сценариями воздействия, которые также можно легко реализовать. Расставить приоритеты при снижении приоритетов сценариев воздействия, которые трудно реализовать.

Используйте примеры [сценариев и](adoption-scenarios.md) используйте примеры, чтобы подсказыть идеи о том, как можно использовать SharePoint Syntex в организации.

| Роль AAD| Роль AAD | Роль AAD | Лидеры |

Менеджер знаний, владелец бизнес-процессов и владелец модели контента создают примерные модели и утверждение чемпиона в организации.

- Обработка форм. Определите, какие сайты получат действие обработки форм.

## <a name="readiness-checklist"></a>Контрольный список готовности

Чтобы быть готовым к реализации SharePoint Syntex, необходимо:

![Готовность к пониманию контента](../media/content-understanding/cu-adoption-readinesschecklist.png)

1. Планирование конечных состояния
    - Модели понимания документов — это средства, а не конец.
    - Планирование использования значения извлеченных метаданных с помощью:
      - Поиск
      - Фильтрация и форматирование представлений
      - Соответствие требованиям
      - Автоматизация
2. Определение
    - Сведения о существующей информационной архитектуре и использовании функций управления контентом.
    - Являются ли существующие типы контента хорошими кандидатами для моделей?
    - Какие существующие процессы будут улучшены с помощью метаданных?
3. Разработка
    - Разработка подхода к информационной архитектуре, управляемым метаданным и типам контента
    - Разработка процесса определения, создания, управления.
