# <a name="create-a-search"></a>Создание поискового запроса

На **вкладке Поиск** в вашем случае можно создать новый поиск, щелкнув **новый** поиск и следуя за мастером.

![Мастер поиска в Advanced eDiscovery случае](../media/AeDSearch1.png)

## <a name="name-the-search-and-give-it-a-description"></a>Назови поиск и дайте ему описание

Каждый поиск с случаем должен иметь уникальное имя. Вы можете дополнительно предоставить описание для поиска. 

## <a name="choose-the-custodians-and-custodial-locations-to-search"></a>Выбор хранителей и местоположений для поиска

Выберите расположения контента хранителя для поиска, указав, что хранители, добавленные в дело. Выбрав хранителя, вы запустите поиск по всем источникам данных, отображымся к хранителям. У вас также есть возможность сузить поиск до выбранных источников данных для каждого хранителя. Дополнительные сведения о том, как добавлять хранителей и управлять их источниками данных, см. в этой ссылке [Работа с хранителями.](managing-custodians.md)

## <a name="choose-non-custodial-locations"></a>Выбор местоположений, не отбираемых для содержания под стражей

В некоторых случаях может потребоваться поиск источников данных, не связанных с хранителями. В этом случае можно указать расположения, которые необходимо искать, или выбрать для поиска все расположения контента для определенной службы Майкрософт (например, поиска всех Exchange почтовых ящиков или всех SharePoint сайтов и OneDrive учетных записей).

## <a name="define-the-search-query-and-conditions"></a>Определение запроса и условий поиска

Вы можете определить запрос по ключевым словам и любые условия для поиска с помощью предварительно построенных карт условий или с помощью языка запросов ключевых слов (KQL). Дополнительные сведения см. в [см. в руб. Сборка поисковых запросов.](building-search-queries.md)
