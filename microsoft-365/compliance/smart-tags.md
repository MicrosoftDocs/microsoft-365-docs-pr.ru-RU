# <a name="set-up-smart-tags-in-advanced-ediscovery"></a>Настройка смарт-тегов в Advanced eDiscovery

Возможности машинного обучения (ML) в Advanced eDiscovery помогут вам сделать процесс принятия решений более эффективным при просмотре документов дела в наборе для проверки. Смарт-теги — это способ принятия решений с учетом возможностей ML: при пометке документов во время проверки. При создании группы смарт-тегов решения, которые являются результатом модели ML, связанной с группой смарт-тегов, отображаются вместе с тегами в группе тегов. Это помогает просмотреть сведения о результатах ML в строке при просмотре определенных документов.

## <a name="how-to-set-up-a-smart-tag-group"></a>Настройка группы смарт-тегов

1. В наборе для проверки **щелкните "Управление набором** для проверки" и выберите **"Управление тегами".**

2. Нажмите **кнопку "Добавить группу тегов"** и выберите **"Добавить группу смарт-тегов".**

3. Выберите модель ML, которую необходимо связать с группой тегов.
    
   При этом создается группа тегов и N-потомки, где *N* — это число возможных выходных данных модели.  Например, модель [обнаружения адвокатских привилегий](attorney-privilege-detection.md) имеет два возможных выходных результата: 

   - **Положительное** — используется для пометки документов, содержащих привилегированное содержимое адвокатского клиента.
   
   - **Отрицательный** — используется для пометки документов, которые не содержат привилегированного содержимого адвокатского клиента.
    
    При выборе этой модели создается группа тегов с двумя потомков (один потомок с именем **Positive,** а другой отрицательный) для набора для проверки. В этом примере каждый из этих тегов соответствует одному из возможных результатов из модели обнаружения адвокатских привилегий.

4. При желании можно переименовать группу тегов и их потомки. Например, можно переименовать тег Positive в  **Privileged,** а отрицательный — на  **"Не привилегированный".**

## <a name="how-to-use-smart-tags"></a>Использование смарт-тегов

При просмотре документа результаты модели отображаются рядом с соответствующим тегом потомка. Например, если у вас есть смарт-группа тегов для обнаружения адвокатских привилегий и вы просматриваете документ, который потенциально привилегированный, причина такого вывода отображается рядом с соответствующим тегом. Важно отметить, что тег не применяется к документу автоматически. Рецензент принимает решение о том, как пометить документ.
