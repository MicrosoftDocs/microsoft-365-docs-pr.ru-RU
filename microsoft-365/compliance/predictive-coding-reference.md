# <a name="predictive-coding-reference-preview"></a>Ссылка на прогностическое кодирование (предварительный просмотр)

В этой статье описываются основные понятия и метрики средства прогностического кодирования в Advanced eDiscovery. Разделы в статье перечислены в алфавитном порядке.

## <a name="confidence-level"></a>Уровень вероятности

Уровень доверия — это расширенный параметр при создании модели прогностического кодирования. В нем определяется, что показатели производительности модели (например, богатство, точность и отзыв) попадают в указанный диапазон (это определяет допустимый для модели диапазон ошибок), который является представителем истинных значений оценки прогноза, которые модель назначает для элементов в наборе обзоров. Значения уровня доверия и погрешности также помогают определить количество элементов, включенных в набор элементов управления. Значение по умолчанию для уровня доверия — 0,95 или 95%.

## <a name="control-set"></a>Набор управления

Набор управления используется в процессе подготовки модели прогностического кодирования. Набор элементов управления — оценка баллов предсказаний, которые модель назначает для элементов с меткой, выполняемой во время учебных раундов. Размер набора элементов управления зависит от количества элементов в наборе обзоров и уровня доверия и значения ошибок, заданная при создании модели. Элементы в наборе управления никогда не изменяются и не идентифицируются для пользователей. Общее число элементов в наборе элементов управления отображается на странице вылетов для учебного раунда.

## <a name="control-set-confusion-matrix"></a>Матрица путаницы набора управления

После завершения учебного раунда модель назначает оценку прогноза 10 элементов в наборе элементов управления, помеченных во время учебного раунда. Модель сравнивает оценку прогнозирования этих 10 элементов с фактической меткой, назначенной элементу во время учебного раунда. На основе этого сравнения модель определяет следующие классификации для оценки производительности прогнозирования модели:

<br>

****

|Label|Модель предсказывает, что элемент актуален|Модель предсказывает, что элемент не имеет значения|
|---|---|---|
|**Элемент рецензента помечен как релевантный**|True positive|Ложное срабатывание|
|**Элемент рецензента помечен как не релевантный**|Ложный отрицательный|True negative|
|

На основе этих сопоставлений модель получает значения для метрик F-score, точности и отзыва и погрешности для каждого из них. Число каждого типа путаницы из матрицы отображается на странице вылетов для учебного раунда.

## <a name="f-score"></a>F-score

F-score — это средневзвешеное значение баллов для метрик точности и отзыва.  Диапазон баллов для этого показателя составляет от **0** до **1**. Оценка ближе **к 1** указывает, что модель будет более точно обнаруживать соответствующие элементы. Метрика F-score отображается на панели мониторинга модели и на странице вылетов для каждого учебного раунда.

## <a name="margin-of-error"></a>Погрешность

Предел ошибки — это расширенный параметр при создании режима прогностического кодирования. В нем указывается степень ошибки в метриках производительности (например, богатство, точность и отзыв), полученных из случайной выборки элементов в наборе элементов управления. Для более низкой погрешности требуется больший набор управления для обеспечения того, чтобы показатели производительности модели были меньше. Значения для уровня погрешности и уверенности также помогают определить количество элементов, включенных в набор элементов управления. Значение по умолчанию для погрешности составляет 0,05 или 5%.

## <a name="model-stability"></a>Стабильность модели

Стабильность модели указывает на способность модели точно прогнозировать релевантность документа в наборе отзывов. Если модель нестабильная, может потребоваться выполнить дополнительные учебные раунды, чтобы включить стабильность модели. Если модель стабильна, не требуется выполнять дополнительные учебные раунды. Панель мониторинга модели указывает текущее состояние стабильности модели. Если модель стабильна, показатели производительности достигли уровня, который соответствует параметрам уровня уверенности и погрешности.

## <a name="overturn-rate"></a>Коэффициент отмены

Коэффициент отмены — это процент элементов в наборе обзоров, где оценка прогноза изменилась между учебными раундами. Модель считается стабильной, если скорость опрокидывания не превышает 5%. Метрика скорости опрокидывания отображается на панели мониторинга модели и на странице вылетов для каждого учебного раунда. Коэффициент отмены для первого учебного раунда ноль, так как предыдущий показатель прогноза не отменяется.

## <a name="precision"></a>Точность

Метрика точности измеряет долю элементов, которые действительно актуальны для элементов, прогнозируемых моделью. Это означает, что элементы в контрольном наборе, где метка как релевантная рецензентом и предсказана как релевантная моделью. Диапазон баллов для этого показателя составляет от **0** до **1**. Оценка ближе **к 1** указывает, что модель будет определять меньше не соответствующих элементов. Метрика точности отображается на панели мониторинга модели и на странице вылетов для каждого учебного раунда.

## <a name="prediction-score"></a>Оценка прогноза

Это оценка, которую модель назначает каждому документу в наборе отзывов. Оценка основана на релевантности документа по сравнению с обучением модели в учебных раундах. Как правило, элементы с оценками прогноза от **0** до **0,5** считаются не соответствующими, а элементы с оценками прогноза от **0,5** до **1** считаются релевантными. Оценка прогноза содержится в поле метаданных документов. Фильтр прогнозирования можно использовать для отображения элементов в наборе обзоров, которые попадают в определенный диапазон прогнозов.

## <a name="recall"></a>Recall

Метрика отзывов измеряет долю элементов, прогнозируемых моделью, среди элементов, которые действительно актуальны. Это означает, что элементы в наборе элементов управления, которые были релевантны модели, также были помечены рецензентом как релевантные. Диапазон баллов для этого показателя составляет от **0** до **1**. Оценка ближе **к 1** указывает, что модель определяет большую часть соответствующих элементов. Метрика отзыва отображается на панели мониторинга модели и на странице вылетов для каждого учебного раунда.

## <a name="review-set"></a>Набор для проверки

Набор обзоров предоставляет область модели прогностического кодирования. При создании новой модели для набора обзоров из набора обзоров выбираются элементы для набора элементов управления и наборов подготовки. Когда модель назначает оценки прогнозирования, она назначает эти оценки элементов в обзоре. Перед созданием модели прогностического кодирования необходимо добавить все элементы в набор обзоров. При добавлении элементов после создания модели этим пунктам не будет назначена оценка прогноза.

## <a name="richness"></a>Богатство

Метрика богатства измеряет процент пунктов набора обзоров, которые модель предсказывает как релевантные. Диапазон баллов для этого показателя составляет от **0** до **1**. Метрика насыщенности отображается на панели мониторинга модели.

## <a name="sampled-items"></a>Примеры элементов

Выбранные  элементы терминов являются ссылкой на случайный пример элементов в наборе обзоров (содержащих текст), выбранных и связанных с набором элементов управления при создании модели прогностического кодирования. Для каждого учебного раунда также выбран случайный пример элементов. Элементы, выбранные для набора элементов управления модели, никогда не включаются в учебный набор для той же модели. Обратное также верно: элементы набора подготовки никогда не включаются в набор элементов управления.

## <a name="training-set"></a>Набор учебных занятий

Модель случайным образом выбирает элементы из набора обзоров и добавляет их в учебный набор. Во время учебного раунда вам будут представлены элементы из учебного набора (помимо элементов из набора управления), чтобы можно было пометить каждый из них как "релевантный" или "не соответствующий". Этот процесс маркировки или "обучения" помогает модели узнать, как предсказать, какие элементы в обзоре являются релевантными или не релевантными. Каждый раз, когда вы выполняете учебный раунд, модель выбирает больше элементов из обзора и добавляет их в набор подготовки для этого учебного раунда. Элементы из набора элементов управления никогда не выбираются для учебного набора.
