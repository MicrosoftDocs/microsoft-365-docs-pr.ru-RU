# <a name="set-up-a-connector-to-archive-facebook-data-preview"></a>Настройка соединитетеля для архива данных Facebook (предварительный просмотр)

Используйте соединители в центре соответствия требованиям Microsoft 365 для импорта и архива данных со страниц Facebook Business в Microsoft 365. После настройки соединители он подключается к странице Facebook Business (на плановой основе), преобразует содержимое элементов Facebook в формат сообщений электронной почты, а затем импортирует эти элементы в почтовый ящик в Microsoft 365.

После импорта данных Facebook можно применить к данным Facebook такие функции соответствия требованиям Microsoft 365, как хранение судебного разбирательства, поиск контента, In-Place архива, аудит, соответствие требованиям связи и политики хранения Microsoft 365. Например, при размещении почтового ящика в удержании судебного разбирательства или при наложении на политику хранения данные Facebook сохраняются. Вы можете искать сторонние данные с помощью поиска контента или связывать почтовый ящик, в котором хранятся данные Facebook, с хранителями в случае предварительного поиска электронных данных. Использование соединиттеля для импорта и архива данных Facebook в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="prerequisites-for-setting-up-a-connector-for-facebook-business-pages"></a>Необходимые условия для настройки соединители для страниц Facebook Business

Выполните следующие необходимые условия, прежде чем вы сможете настроить соединители в центре соответствия требованиям Microsoft 365 для импорта и архива данных со страниц facebook business вашей организации. 

- Вам нужна учетная запись Facebook для бизнес-страниц организации (необходимо войти в эту учетную запись при настройке соединитетеля). В настоящее время архивировать данные можно только со страниц Facebook Business; вы не можете архивировать данные из отдельных профилей Facebook.

- В организации должна быть действительная подписка Azure. Если у вас нет существующей подписки Azure, вы можете зарегистрироваться для одного из этих параметров:

    - [Зарегистрив бесплатную подписку на Azure на один год](https://azure.microsoft.com/free)

    - [Подпишитесь на подписку Pay-As-You-Go Azure](https://azure.microsoft.com/pricing/purchase-options/pay-as-you-go/)

    > [!NOTE]
    > Бесплатная [подписка Azure Active Directory,](use-your-free-azure-ad-subscription-in-office-365.md) включенная в подписку Microsoft 365, не поддерживает соединители в Центре & соответствия требованиям.

- Соединители для страниц Facebook Business могут импортировать в общей сложности 200 000 элементов за один день. Если в день имеется более 200 000 элементов Facebook Business, ни один из этих элементов не будет импортироваться в Microsoft 365.

- Пользователю, который настраивает настраиваемый соединитель в центре соответствия требованиям Microsoft 365 (в шаге 5), необходимо уступить роль экспорта импорта почтовых ящиков в Exchange Online. По умолчанию эта роль не назначена ни одной группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах Создание групп  [ролей](/Exchange/permissions-exo/role-groups#create-role-groups) или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-create-an-app-in-azure-active-directory"></a>Шаг 1. Создание приложения в Azure Active Directory

Первым шагом является регистрация нового приложения в Azure Active Directory (AAD). Это приложение соответствует ресурсу веб-приложения, который реализуется в шаге 4 и шаге 5 для соединиттеля Facebook. 

Пошаговая инструкция см. в приложении [Create in Azure Active Directory.](deploy-facebook-connector.md#step-1-create-an-app-in-azure-active-directory)

Во время выполнения этого шага (с помощью предыдущих пошагових инструкций) вы сохраните следующую информацию в текстовом файле. Эти значения используются в последующих шагах в процессе развертывания.

- ID приложения AAD

- Секрет приложения AAD

- ID клиента

## <a name="step-2-deploy-the-connector-web-service-from-github-to-your-azure-account"></a>Шаг 2. Развертывание веб-службы соединителя из GitHub в учетную запись Azure

Следующий шаг — развертывание исходных кодов для приложения-соединиттеля страниц Facebook, которое будет использовать API Facebook для подключения к вашей учетной записи Facebook и извлечения данных, чтобы вы могли импортировать его в Microsoft 365. Соединитатель Facebook, развернутый для организации, будет загружать элементы со страниц бизнеса Facebook в расположение Azure Storage, созданное на этом шаге. После создания соединиттеля бизнес-страниц Facebook в центре соответствия требованиям Microsoft 365 (на шаге 5) служба импорта будет копировать данные бизнес-страниц Facebook из расположения Хранилища Azure в почтовый ящик в организации Microsoft 365. Как было ранее объяснено в разделе [Необходимые](#prerequisites-for-setting-up-a-connector-for-facebook-business-pages) условия, для создания учетной записи Azure storage необходимо иметь действительная подписка Azure.

Пошаговые инструкции см. в странице Развертывание веб-службы соединителя [из GitHub в учетную](deploy-facebook-connector.md#step-2-deploy-the-connector-web-service-from-github-to-your-azure-account)запись Azure.

В пошаговом инструкции по завершению этого шага будут указаны следующие сведения:

- APISecretKey. Этот секрет создается при завершении этого шага. Он используется в шаге 5.

- TenantId. ID клиента вашей организации Microsoft 365, который вы скопировали после создания приложения-соединиттеля Facebook в Azure Active Directory на шаге 1.

После выполнения этого шага обязательно скопируйте URL-адрес службы приложений Azure https://fbconnector.azurewebsites.net) (например. Этот URL-адрес необходимо использовать для выполнения 3-го, 4-го и 5-го этапов.

## <a name="step-3-register-the-web-app-on-facebook"></a>Шаг 3. Регистрация веб-приложения на Facebook

Следующий шаг — создание и настройка нового приложения на Facebook. Соединители бизнес-страниц Facebook, которые вы создаете в шаге 5, используют веб-приложение Facebook для взаимодействия с API Facebook для получения данных со страниц facebook business вашей организации.

Пошаговая инструкция см. в странице [Регистрация приложения Facebook.](deploy-facebook-connector.md#step-3-register-the-facebook-app)

Во время выполнения этого шага (следуя пошаговим инструкциям), вы сохраните следующую информацию в текстовом файле. Эти значения используются для настройки приложения соединители Facebook в шаге 4.

- ID приложения Facebook

- Секрет приложения Facebook

- Веб-сайты Facebook проверяют маркер

## <a name="step-4-configure-the-facebook-connector-app"></a>Шаг 4. Настройка приложения соединители Facebook

Следующий шаг — добавление параметров конфигурации в приложение соединителю Facebook, которое было загружено при создании ресурса веб-приложения Azure на шаге 1. Это необходимо сделать, переехав на домашную страницу приложения-соединиттеля и настроив его.

Пошаговая инструкция см. в приложении [Configure the Facebook connector.](archive-facebook-data-with-sample-connector.md#step-4-configure-the-facebook-connector-app)

Во время выполнения этого шага (следуя пошаговом инструкции), вы предоставляете следующую информацию (которую вы скопировали в текстовый файл после завершения предыдущих действий):

- ID приложения Facebook (получен в шаге 3)

- Секрет приложения Facebook (получен в шаге 3)

- Веб-сайты Facebook проверяют маркер (полученный в шаге 3)

- Azure Active Directory application ID (AAD application ID obtained in Step 1)

- Секрет приложения Azure Active Directory (секрет приложения AAD, полученный в шаге 1)

## <a name="step-5-set-up-a-facebook-business-pages-connector-in-the-microsoft-365-compliance-center"></a>Шаг 5. Настройка соединитетеля страниц Facebook в центре соответствия требованиям Microsoft 365

Заключительный шаг — настройка соединитетеля в центре соответствия требованиям Microsoft 365, который будет импортировать данные со страниц Facebook Business в указанный почтовый ящик в Microsoft 365. После завершения этого шага служба импорта Microsoft 365 начнет импорт данных со страниц Facebook Business в Microsoft 365.

Пошаговые инструкции см. в шаге 5. Настройка соединитетеля Facebook в центре соответствия [требованиям Microsoft 365.](deploy-facebook-connector.md#step-5-set-up-a-facebook-connector-in-the-microsoft-365-compliance-center) 

Во время выполнения этого шага (следуя пошаговой инструкции), вы предоставляете следующую информацию (которую вы скопировали в текстовый файл после выполнения шагов).

- AAD application ID (obtained in Step 1)

- URL-адрес службы приложений Azure (полученный в шаге 1; например, https://fbconnector.azurewebsites.net)

- APISecretKey (созданный в шаге 2)
