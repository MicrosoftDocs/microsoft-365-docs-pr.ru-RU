# <a name="review-conversations-in-advanced-ediscovery"></a>Просмотр бесед в Advanced eDiscovery 

Обмен мгновенными сообщениями — это удобный способ задать вопросы, поделиться идеями или быстро обмениваться идеями с широкой аудиторией. По мере того как платформы обмена мгновенными сообщениями, такие как Microsoft Teams, становятся основными для совместной работы предприятия, организации должны оценить, как их рабочий процесс eDiscovery адресует эти новые формы взаимодействия и совместной работы. 

Функция реконструкции бесед в Advanced eDiscovery предназначена для определения контекстного содержимого и создания отдельных представлений бесед. Эта возможность позволяет эффективно и быстро просмотреть полные мгновенные сообщения (также называемые цепочками *бесед),* которые создаются на таких платформах, как Microsoft Teams.

С помощью реконструкции бесед можно использовать встроенные возможности для воссоздание, просмотра и экспорта бесед. Использование реконструкции беседы Advanced eDiscovery для:

- Сохранение уникальных метаданных на уровне сообщений во всех сообщениях в беседе.

- Сбор контекстных сообщений вокруг результатов поиска.

- Просмотр, аннотации и отредактировать беседы с потоками.

- Экспорт отдельных сообщений или бесед с цепочками

## <a name="terminology"></a>Терминология

Вот несколько определений, которые помогут вам приступить к использованию реконструкции беседы.

- **Сообщения:** Представляет наименьшую единицу беседы. Размер, структура и метаданные сообщений могут отличаться. 

- **Беседа:** Представляет группировку одного или более сообщений. В разных приложениях беседы могут быть представлены различными способами. В некоторых приложениях существует явное действие, которое приводит к ответу на существующее сообщение. Беседы формируются явно в результате этого действия пользователя. Например, вот снимок экрана с беседой канала в Microsoft Teams.

   ![Беседа на канале Microsoft Teams](../media/threadedchat.png)

   В других приложениях (например, сообщениях чата 1xN в Teams) нет формальной цепочки ответов, а вместо этого сообщения отображаются как "плоская речка сообщений" в одном потоке. В приложениях этих типов беседы высмеяются из группы сообщений, которые происходят в течение определенного времени. Такая "мягкая группировка" сообщений (в отличие от цепочки ответов) представляет беседу "назад и вперед" по интересующих вас темах. 

## <a name="step-1-run-a-search"></a>Шаг 1. Запуск поиска

После того как вы определили релевантные хранители и расположения контента, вы можете создать поиск потенциально релевантной информации. На **вкладке "Поиск"** в деле Advanced eDiscovery можно создать поиск, нажав кнопку **"Создать поиск"** и следуя за мастером. Сведения о том, как создать поиск, создать поисковый запрос и просмотреть результаты поиска, см. в подстройки ["Сбор данных для дела".](create-search-to-collect-data.md)

## <a name="step-2-create-a-conversation-review-set"></a>Шаг 2. Создание набора для проверки беседы

В наборе для проверки можно искать, помечать, примечать и редактировать документы, сообщения электронной почты и беседы чата. В Advanced eDiscovery можно настроить просмотр бесед на основе отдельных сообщений или бесед. Это определяется типом набора для проверки, в который добавляются результаты поиска, созданного на шаге 1. Существует два разных типа наборов для проверки: 
  
  - **Стандартные наборы для проверки:** Сообщения в беседах обрабатываются и отображаются как отдельные элементы. 
  
  -  **Наборы для просмотра беседы:** Сообщения в беседах обрабатываются по отдельности, но отображаются в представлении беседы. В наборе для проверки беседы можно отмечать, помечать и редактировать сообщения в потоковом представлении беседы. 

Дополнительные сведения о том, как просмотреть контент в наборе для проверки и управлять им, см. в документе ["Управление наборами отзывов".](managing-review-sets.md) 

## <a name="step-3-enable-conversation-retrieval-options"></a>Шаг 3. Включить параметры и получить беседу

После проверки и завершения поискового запроса вы можете добавить результаты поиска в набор для проверки. При добавлении результатов поиска в набор для проверки исходные данные копируется в хранилище Azure для упрощения процесса проверки и анализа. Дополнительные сведения о добавлении результатов поиска в набор для проверки см. в подстройки "Добавление результатов поиска [в набор для проверки".](add-data-to-review-set.md) 

При добавлении данных из бесед в набор для проверки вы можете использовать параметры ищите беседы для расширения поиска и добавления контекстных сообщений. После настройки параметров ирисовки беседы могут произойти следующие вещи:

  ![Ирисовка беседы](../media/messagesandconversations.png)
  
1. С помощью запроса по ключевому слову и диапазону дат поиск вернул сообщение *3.* Это сообщение было частью более крупной беседы, иллюстрированной *CRC1.* 
  
2. Когда вы добавляете данные в набор для проверки и включаете параметры иска беседы, Advanced eDiscovery возвращается и собирает другие элементы в *CRC1.* 
  
3. После того как элементы будут добавлены в набор для проверки, вы можете просмотреть все отдельные сообщения из *CRC1.* 

Чтобы включить и получить беседу:
  
1. На **вкладке "Поиск"** в деле Advanced eDiscovery выберите поиск, а затем нажмите кнопку "Добавить для **проверки", задав** флажок на странице".
  
2. Выберите существующий набор для проверки или создайте набор для проверки. Параметры ищите можно настроить при добавлении результатов поиска в стандартный набор или набор для просмотра бесед.
  
3. В **параметрах коллекции** настройте параметры и получения бесед для источников контента, которые нужно развернуть в поиске, а затем нажмите кнопку **"Добавить",** чтобы начать процесс.  
  
4. После завершения **задания "Добавить для проверки"** на вкладке **"Задания"** можно начать просмотр бесед.

## <a name="step-4-review-and-export-conversations-in-a-review-set"></a>Шаг 4. Просмотр и экспорт бесед в наборе для проверки

После обработки контента и его добавления в набор для проверки можно приступить к просмотру данных в наборе для проверки. Возможности проверки отличаются в зависимости от того, было ли содержимое добавлено в стандартный набор для проверки или набор анализа бесед. 

### <a name="reviewing-conversations-in-a-standard-review-set"></a>Просмотр бесед в стандартном наборе для проверки

В стандартном наборе для проверки сообщения обрабатываются и отображаются как отдельные элементы, аналогично хранимые в папке почтового ящика. В этом рабочий процесс каждое сообщение обрабатывается как отдельный элемент. В результате потоковые параметры сводки и экспорта недоступны в стандартном наборе для проверки. 

  ![Стандартный набор для проверки](../media/standardrs.PNG)

### <a name="reviewing-conversations-in-a-conversation-review-set"></a>Просмотр бесед в наборе для проверки бесед

В наборе для просмотра бесед отдельные сообщения объединяется и представляются как беседы. Это позволяет просмотреть и экспортировать контекстные беседы. 

  ![Набор для проверки беседы](../media/ConversationRSOptions.PNG)

В следующих разделах описывается просмотр и экспорт бесед в наборе для проверки бесед.

#### <a name="reviewing-conversations"></a>Просмотр бесед

В наборе для проверки беседы можно использовать следующие параметры для упрощения процесса проверки.

- **Группировка по беседе:** Группировать сообщения в одной беседе, чтобы упростить и ускорить процесс проверки. 

- **Сводное представление:** Отображает цепочку беседы. В этом представлении можно просмотреть всю беседу, а также получить доступ к метаданным для каждого отдельного сообщения.  
  
   - Просмотр метаданных для отдельных сообщений
   
   - Скачивание отдельных сообщений

- **Текстовое представление:** Предоставляет извлеченный текст для всей беседы. 

- **Представление "Аннотировать":** Позволяет разметить потоковые представления беседы. Все сообщения в беседе имеют один и тот же документ с заносями.

- **Добавление тегов:** При просмотре бесед в наборе для проверки можно просматривать и применять теги, щелкнув панель тегов **на** панели кодирования.

- **Повторное преобразование беседы:** При добавлении сообщений в набор для проверки бесед автоматически запускается задание преобразования для создания потоковой сводки и создания заносок в представления. Если задание реконструкции беседы не удается, вы можете повторно его перезапись, нажав кнопку **Action > Create conversation PDFs** in the review set.

#### <a name="exporting-conversations"></a>Экспорт бесед

В наборе для проверки бесед можно настроить следующие параметры экспорта бесед:

![Параметры экспорта для бесед](../media/export.png)

а. Параметры метаданных

   - **Загрузим файл:** Метаданные включаются для каждого отдельного сообщения, электронной почты и документа. Для каждого сообщения в беседе есть одна строка. 

   - **Теги:** Теги из процесса проверки включаются в файл метаданных. Сообщения в беседе имеют одинаковые теги. 

б. Параметры беседы
  
   - **Файлы беседы:** При экспорте файлов бесед представление с заносом преобразуется в PDF-файл и загружается в папку экспорта. Сообщения в одном файле беседы указывают на PDF-версию того же файла беседы.  
  
   - **Отдельные сообщения чата:** При экспорте отдельных сообщений каждое уникальное сообщение в беседе экспортируется как отдельный элемент. Файл экспортируется в том же формате, что и в почтовом ящике. Для конкретной беседы вы получаете несколько MSG-файлов. 

     >[!NOTE]
     > Если вы применили примечания к файлу беседы, эти примечания не будут перенесены в отдельные сообщения. 

в. Другие параметры

   - **Создание текстовых файлов для всего экспортируемой информации:** Создает текстовый файл для каждой беседы, экспортируемой из набора для проверки. 

   - **Замените экспортируемую информацию на отредактировали PDFs:** Если в процессе проверки создаются отредактные файлы бесед, то эти файлы доступны во время экспорта. Вы можете определить, следует ли экспортировать только встроенные файлы (не выбрав этот параметр) или заменить их на отредактировали версии встроенных файлов (выбрав этот параметр), которые экспортируются как PDF-файлы.
