# <a name="conversation-threading-in-advanced-ediscovery"></a>Потоковая потоковая беседа в advanced eDiscovery

Мгновенные сообщения — это удобный способ задавать вопросы, обмениваться идеями или быстро общаться в больших аудиториях. Поскольку платформы обмена мгновенными сообщениями, такие как группы Microsoft Teams и Yammer, становятся ключевыми для совместной работы предприятий, организации должны оценить, как их рабочий процесс по обнаружению электронных данных решает эти новые формы взаимодействия и совместной работы.

Функция Реконструкции беседы в advanced eDiscovery предназначена для определения контекстного контента и создания различных представлений беседы. Эта возможность позволяет эффективно и быстро просмотреть полные мгновенные беседы сообщений (также называемые беседами с потоками), которые создаются на таких платформах, как Microsoft Teams.

С помощью conversation Reconstruction можно использовать встроенные возможности для восстановления, просмотра и экспорта потоковой беседы. Используйте расширенные операции по восстановлению беседы с электронными данными для:

- Сохранение уникальных метаданных на уровне сообщений во всех сообщениях в ходе беседы.

- Сбор контекстных сообщений вокруг результатов поиска.

- Обзор, аннотации и отредактировать потоковые беседы.

- Экспорт отдельных сообщений или потоковой беседы

## <a name="terminology"></a>Терминология

Вот несколько определений, которые помогут приступить к использованию реконструкции беседы.

- **Сообщения:** Представляет наименьший блок беседы. Сообщения могут отличаться по размеру, структуре и метаданным. 

- **Беседа:** Представляет группу одного или более сообщений. В различных приложениях беседы могут быть представлены разными способами. В некоторых приложениях существует явное действие, которое приводит к ответу на существующее сообщение. Беседы формируются явно в результате этого действия пользователя. Например, вот снимок экрана беседы канала в Microsoft Teams.

   ![Разговор каналов Microsoft Teams](../media/threadedchat.png)

   В других приложениях (например, сообщения чата 1xN в Teams) нет официальной цепочки ответов, а вместо них сообщения отображаются как "плоская река сообщений" в одном потоке. В приложениях этих типов беседы высвечены из группы сообщений, которые происходят в течение определенного времени. Эта "мягкая группировка" сообщений (в отличие от цепочки ответов) представляет беседу "назад и вперед" на определенную интересуемую тему.

## <a name="step-1-create-a-draft-collection"></a>Шаг 1. Создание коллекции черновиков

После того как вы определили соответствующие хранители и расположения контента, можно создать поиск, чтобы найти потенциально релевантный контент. На **вкладке Коллекции** в деле Advanced eDiscovery можно создать  коллекцию, щелкнув новую коллекцию и следуя за мастером. Сведения о том, как создать коллекцию, создать запрос поиска и просмотреть результаты поиска, см. в статью [Создание коллекции черновиков.](create-draft-collection.md)

## <a name="step-2-commit-a-draft-collection-to-a-review-set"></a>Шаг 2. Совершение набора черновиков в набор отзывов

После проверки и завершения поиска в коллекции можно добавить результаты поиска в набор отзывов. При добавлении результатов поиска в набор отзывов исходные данные копируется в область хранения Azure для облегчения процесса проверки и анализа. Дополнительные сведения о добавлении результатов поиска в набор отзывов см. в документе [Commit a draft collection to a review set.](commit-draft-collection.md)

При добавлении элементов из бесед в набор отзывов можно использовать параметр threaded conversations для сбора контекстных сообщений из бесед, содержащих элементы, которые соответствуют критериям поиска коллекции. После выбора параметра беседы потоков могут произойти следующие вещи:

  ![Переоценка беседы](../media/messagesandconversations.png)
  
1. С помощью ключевого слова и запроса диапазона дат поиск возвращал хит в *Message 3.* Это сообщение было частью более масштабной беседы, иллюстрированной *CRC1*.
  
2. Если добавить данные в набор отзывов и включить параметры для иска беседы, advanced eDiscovery вернется и соберет другие элементы *в CRC1.*
  
3. После того как элементы были добавлены в набор отзывов, вы можете просмотреть все отдельные сообщения *из CRC1.*

Чтобы включить параметр threaded conversations, см. в рубке [Commit a draft collection to a review set.](commit-draft-collection.md#commit-a-draft-collection-to-a-review-set)
  
## <a name="step-3-review-and-export-threaded-conversations"></a>Шаг 3. Просмотр и экспорт бесед с потоками

После обработки и добавления контента в набор отзывов можно приступить к проверке данных в наборе обзоров. Возможности проверки отличаются в зависимости от того, был ли контент добавлен в стандартный набор обзоров или набор обзоров беседы.

### <a name="reviewing-conversations-in-a-standard-review-set"></a>Просмотр бесед в стандартном наборе обзоров

В стандартном наборе обзоров сообщения обрабатываются и отображаются как отдельные элементы, аналогичные тому, как они хранятся в папке почтовых ящиков. В этом рабочего процесса каждое сообщение обрабатывается как отдельный элемент. В результате параметры потоковой сводки и экспорта недоступны в стандартном наборе обзоров.

  ![Стандартный набор обзоров](../media/standardrs.PNG)

### <a name="reviewing-conversations-in-a-conversation-review-set"></a>Просмотр бесед в наборе обзоров бесед

В наборе обзоров бесед отдельные сообщения совмещение и преподносясь как беседы. Это позволяет просмотреть и экспортировать контекстные беседы. 

  ![Набор обзоров беседы](../media/ConversationRSOptions.PNG)

В следующих разделах описаны обзор и экспорт бесед в наборе обзоров бесед.

#### <a name="reviewing-conversations"></a>Просмотр бесед

В наборе обзоров бесед можно использовать следующие параметры для облегчения процесса проверки.

- **Группа по разговору:** Группировать сообщения в одной беседе вместе, чтобы помочь пользователям упростить и ускорить процесс проверки.

- **Сводное представление:** Отображает резьбовую беседу. В этом представлении можно просмотреть весь разговор, а также получить доступ к метаданным для каждого отдельного сообщения.  
  
   - Просмотр метаданных для отдельных сообщений
   
   - Скачивание отдельных сообщений

- **Представление текста:** Предоставляет извлеченный текст для всего разговора.

- **Аннотировать представление:** Позволяет разметку потокового представления беседы. Все сообщения в беседе имеют один и тот же аннотированный документ.

- **Теги:** При просмотре бесед в наборе обзоров можно просматривать и применять теги, щелкнув панель **Теги** в панели кодирования.

- **Повторное преобразование беседы:** При добавлении сообщений в набор обзоров бесед автоматически запускается задание преобразования для создания потоковой сводки и аннотации представлений. Если задание реконструкции беседы не удается, вы можете повторно использовать эту работу, нажав кнопку Action > Создать диалоговую **PDF в** наборе обзоров.

#### <a name="exporting-conversations"></a>Экспорт разговоров

В наборе обзоров бесед можно установить следующие параметры экспорта бесед:

![Экспорт вариантов для бесед](../media/export.png)

а. Параметры метаданных

   - **Файл нагрузки:** Метаданные включены для каждого отдельного сообщения, электронной почты и документа. Для каждого сообщения в беседе имеется одна строка. 

   - **Теги:** Теги из процесса проверки включены в файл метаданных. Сообщения в беседе имеют одинаковые теги. 

б. Параметры беседы
  
   - **Файлы беседы:** При экспорте файлов беседы аннотированное представление преобразуется в файл PDF и загружается в папку экспорта. Сообщения в одном файле беседы указывают на PDF-версию того же файла беседы.  
  
   - **Отдельные сообщения чата:** При экспорте отдельных сообщений каждое уникальное сообщение в беседе экспортируется в качестве отдельного элемента. Файл экспортируется в том же формате, что и в почтовом ящике. Для определенного разговора вы получаете несколько файлов msg.

     >[!NOTE]
     > Если вы применили аннотации к файлу беседы, эти аннотации не будут перенесены в отдельные сообщения.

в. Другие параметры

   - **Создание текстовых файлов для всего экспортируемой содержимого:** Создает текстовый файл для каждого разговора, экспортируемой из набора отзывов.

   - **Замените экспортируемую контентную информацию на отредактировали PDF:** Если в процессе проверки создаются отредактированные файлы беседы, эти файлы доступны во время экспорта. Можно определить, следует ли экспортировать только личные файлы (не выбрав этот параметр) или заменить личные файлы отредактными версиями родных файлов (выбрав этот параметр), экспорт которых экспортируется в формате PDF-файлов.
