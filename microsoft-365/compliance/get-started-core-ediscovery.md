# <a name="get-started-with-core-ediscovery-in-microsoft-365"></a>Начало работы с core eDiscovery в Microsoft 365

Основное открытие электронных Microsoft 365 предоставляет базовый инструмент для поиска и экспорта контента в Microsoft 365 и Office 365. Кроме того, с помощью core eDiscovery можно разместить удержание для электронных открытий в расположениях контента, таких как почтовые ящики Exchange, SharePoint сайты, OneDrive учетные записи и Microsoft Teams. Для развертывания core eDiscovery ничего не требуется, но перед организацией, чтобы начать поиск, экспорт и сохранение контента, необходимо выполнить некоторые обязательные задачи, которые должен выполнить ИТ-администратор и диспетчер по обнаружению электронных обнаружений.

В этой статье обсуждаются действия, необходимые для настройка core eDiscovery. Это включает обеспечение надлежащего лицензирования, необходимого для доступа к core eDiscovery и размещения удержания электронных разыскных материалов в расположениях контента, а также назначения разрешений вашей ИТ-, юридической и следственной группе, чтобы они могли получать доступ к делам и управлять ими. В этой статье также представлен краткий обзор использования кейсов для поиска и экспорта контента.

## <a name="step-1-verify-and-assign-appropriate-licenses"></a>Шаг 1. Проверка и назначение соответствующих лицензий

Лицензирование для core eDiscovery требует соответствующей подписки организации и лицензирования для каждого пользователя.

- **Подписка на организацию:** Чтобы получить доступ к core eDiscovery в центре Центр соответствия требованиям Microsoft 365 или Office 365 безопасности & и использовать функции удержания и экспорта, организация должна иметь подписку Microsoft 365 E3 или Office 365 E3 или выше.

- **Лицензирование для каждого пользователя:** Чтобы разместить удержание электронных данных на почтовых ящиках и сайтах, пользователю должна быть назначена одна из следующих лицензий, в зависимости от вашей подписки на организацию:

  - Лицензия Microsoft 365 E3 или Office 365 E3 или выше

   ИЛИ

  - Office 365 E1 лицензии с Exchange Online плана 2 или Exchange Online Archiving надстройки

  И

  - Office 365 E1 лицензии с SharePoint Online Plan 2 или OneDrive для бизнеса plan 2 надстройки
  
  Сведения о назначении лицензий см. в руб. [Назначение лицензий пользователям.](../admin/manage/assign-licenses-to-users.md)

Сведения о лицензировании:

- Скачайте и посмотрите решение "Откройте & ответ" [в Microsoft 365 сопоставление лицензирования соответствия](/office365/servicedescriptions/downloads/microsoft-365-compliance-licensing-comparison.xlsx)требованиям.

- См. [описание службы центра &](/office365/servicedescriptions/office-365-platform-service-description/office-365-securitycompliance-center)безопасности.

## <a name="step-2-assign-ediscovery-permissions"></a>Шаг 2. Назначение разрешений на открытие электронной почты

Чтобы получить доступ к core eDiscovery или добавиться в качестве участника дела об обнаружении основных данных, пользователю необходимо получить соответствующие разрешения. В частности, пользователь должен быть добавлен в качестве члена группы ролей диспетчера электронных данных в центре Office 365 безопасности & соответствия требованиям. Члены этой группы ролей могут создавать и управлять делами по обнаружению core eDiscovery. Они могут добавлять и удалять участников, проводить удержание электронных данных для пользователей, создавать и редактировать поиски и экспортировать контент из дела core eDiscovery.

Выполните следующие действия, чтобы добавить пользователей в группу ролей диспетчера электронных данных:

1. Перейдите и войдите в использование учетных данных для учетной записи администратора в <https://compliance.microsoft.com/permissions> Microsoft 365 или Office 365 организации.

2. На странице **Разрешения выберите** группу **ролей диспетчера** электронных открытий.

3. На странице флажок диспетчера электронных обнаружений нажмите **кнопку Изменить** рядом с разделом Диспетчер по обнаружению **электронных обнаружений.**

4. На странице **Choose eDiscovery Manager** в мастере группы редактирования ролей нажмите **кнопку Выберите Диспетчер обнаружения**.

5. Нажмите **кнопку Добавить,** а затем выберите почтовый ящик для всех пользователей, которые необходимо добавить в группу ролей.

6. Нажмите **кнопку Добавить,** чтобы добавить выбранных пользователей, а затем нажмите кнопку **Готово**.

7. Нажмите **Кнопку Сохранить,** чтобы добавить пользователей в группу ролей, а затем нажмите **кнопку Закрыть,** чтобы завершить шаг.

### <a name="more-information-about-the-ediscovery-manager-role-group"></a>Дополнительные сведения о группе ролей диспетчера электронных данных

В группе ролей диспетчера электронных обнаружений есть две подгруппы. Разница между этими подгруппами заключается в области их действия.

- **Диспетчер по обнаружению** электронных обнаружений: может просматривать и управлять случаями, в которые они создаются или являются членами core eDiscovery. Если другой диспетчер по обнаружению электронных данных создает дело, но не добавляет второго диспетчера по обнаружению электронных данных в качестве участника этого дела, второй диспетчер по обнаружению электронных данных не сможет просмотреть или открыть дело на странице Core eDiscovery в центре соответствия требованиям. Как правило, большинство людей в вашей организации могут быть добавлены в подгруппу диспетчера электронных обнаружений.

- **Администратор по обнаружению** электронной почты: может выполнять все задачи по управлению случаями, которые может выполнять диспетчер по обнаружению электронных обнаружений. Кроме того, администратор, ответственный за обнаружение электронных данных, может выполнять указанные ниже действия.

  - Просмотр всех случаев, перечисленных на странице Core eDiscovery.
  
  - Управлять любым делом в организации после того, как добавит себя в качестве его участника.

  - Получать доступ к данным любого дела в организации и экспортировать эти данные.

  Из-за широкой области доступа в организации должно быть только несколько администраторов, которые являются членами подгруппы администраторов обнаружения электронных данных.

Дополнительные сведения о разрешениях на открытие электронных данных и описании каждой роли, назначенной группе ролей диспетчера электронных данных, см. в руб. Назначение разрешений на открытие [электронных данных.](assign-ediscovery-permissions.md)

## <a name="step-3-create-a-core-ediscovery-case"></a>Шаг 3. Создание дела об обнаружении основных электронных обнаружений

Следующим шагом является создание случая и начало использования core eDiscovery. Выполните следующие действия, чтобы создать случай и добавить участников. Пользователь, создавший корпус, автоматически добавляется в качестве участника.

1. Перейдите и войдите в использование учетных данных учетной записи пользователя, которая получила соответствующие разрешения на [https://compliance.microsoft.com](https://compliance.microsoft.com) открытие электронных данных. Члены группы ролей управления организацией также могут создавать случаи core eDiscovery.

2. В левой области навигации Центр соответствия требованиям Microsoft 365 показать **все,** а затем нажмите **кнопку eDiscovery > Core**.

3. На странице **Core eDiscovery** нажмите **кнопку Создать случай**.

4. На странице **Вылет нового** случая укатите случае имя (обязательно), а затем введите необязательное описание. Имя случая должно быть уникальным в организации.

5. Нажмите **кнопку Сохранить,** чтобы создать случай.

   Новый случай создается и отображается на странице Core eDiscovery. Возможно, вам придется нажать **кнопку Обновить,** чтобы отобразить новый корпус.

## <a name="step-4-optional-add-members-to-a-core-ediscovery-case"></a>Шаг 4 (необязательный): Добавление участников в дело core eDiscovery

Если вы создаете случай в шаге 3 и вы единственный человек, который будет использовать этот случай, вам не нужно выполнять этот шаг. Вы можете начать использовать этот случай для создания содержимого, поиска контента и экспорта результатов поиска. Выполните этот шаг, если вы хотите предоставить другим пользователям (или группе ролей) доступ к делу.

1. На странице **Core eDiscovery** в Центр соответствия требованиям Microsoft 365 нажмите имя дела, в которое нужно добавить участников.

2. На домашней странице дела выберите **вкладку Параметры,** а затем выберите **разрешения Access &.**

3. На странице **& разрешений** на странице **Участники** нажмите кнопку **Добавить,** чтобы добавить участников в дело.

    Вы также можете добавить группы ролей в качестве участников дела. В **группах Ролей** щелкните **Добавить**. Вы можете назначить только группы ролей, в которые вы входите, в случае. Это потому, что группы ролей контролируют, кто может назначать участников делу об обнаружении электронных обнаружений.

4. В списке людей или групп ролей, которые можно добавить в качестве участников дела, щелкните слева от имени людей (или групп ролей), которые вы хотите добавить. Если у вас есть большой список людей или групп ролей, которые могут быть добавлены в качестве участников, используйте поле Поиска для поиска определенного лица или группы ролей в списке. 
  
5. После выбора групп людей или ролей, которые необходимо добавить в качестве участников дела, щелкните **Сохранить,** чтобы сохранить новых членов или группы ролей.

## <a name="explore-the-core-ediscovery-workflow"></a>Изучение рабочего процесса core eDiscovery

Чтобы начать работу с основными данными об обнаружении электронных данных, вы можете найти простой рабочий процесс создания электронных данных для интересующих людей, поиска контента, соответствующего вашему расследованию, а затем экспорта этих данных для дальнейшего анализа. На каждом из этих этапов мы также рассмотрим некоторые расширенные функции по обнаружению основных электронных данных, которые можно изучить.

![Рабочий процесс core eDiscovery](../media/CoreEdiscoveryWorkflow.png)

1. **[Создайте удержание для электронных обнаружений.](create-ediscovery-holds.md)** Первым шагом после создания дела является размещение удержания (также называемого удержанием для проверки электронных обнаружений) для расположения контента интересуемого в вашем расследовании человека. Расположения контента включают Exchange почтовые ящики, SharePoint сайты, OneDrive учетные записи, а также почтовые ящики и сайты, связанные с Microsoft Teams и Office 365 Groups. Хотя этот шаг необязателен, создание удержания для проверки электронных обнаружений сохраняет содержимое, которое может иметь отношение к делу во время расследования. При создании удержания электронных данных можно сохранить все содержимое в определенных расположениях контента или создать удержание на основе запроса, чтобы сохранить только содержимое, которое соответствует запросу удержания. Помимо сохранения контента, еще одна веские причины для создания удержания электронных данных заключается в быстром поиске расположений контента на удержании (вместо необходимости выбора каждого расположения для поиска) при создании и запуске поиска на следующем шаге. После завершения расследования можно освободить любое созданное удержание.

2. **[Поиск контента](search-for-content-in-core-ediscovery.md)**. После создания удержаний для поиска содержимого используйте встроенный инструмент поиска. Вы также можете искать другие расположения контента для данных, которые могут иметь отношение к делу. Можно создавать и запускать различные поиски, связанные с делом. Вы используете ключевые слова, свойства [](keyword-queries-and-search-conditions.md) и условия для создания поисковых запросов, возвращая результаты поиска с данными, которые, скорее всего, имеют отношение к делу. Вы также можете:

   - Просмотр статистики поиска, которая может помочь вам уточнить запрос поиска, чтобы сузить результаты.

   - Просмотрите результаты поиска, чтобы быстро проверить, находятся ли соответствующие данные.

   - Пересмотр запроса и повторное повторы поиска.

3. **[Экспорт и скачивание результатов поиска.](export-content-in-core-ediscovery.md)** После поиска и поиска данных, которые имеют отношение к вашему расследованию, вы можете экспортировать их из Office 365 для проверки людьми, не в составе группы расследования. Экспорт данных — это двухшаговая процедура. Первый шаг — экспорт результатов поиска в случае из Office 365. Это достигается путем копирования результатов поиска в расположение служба хранилища Azure Майкрософт. Следующим шагом является использование средства экспорта электронных обнаружений для загрузки контента на локальный компьютер. Помимо экспортных файлов данных, в пакете экспорта также содержатся отчет об экспорте, сводный отчет и отчет об ошибках.
