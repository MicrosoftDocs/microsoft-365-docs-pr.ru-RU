# <a name="search-for-content-in-a-core-ediscovery-case"></a>Поиск контента в случае core eDiscovery

После создания дела core eDiscovery и удержания людей, интересу которых является этот случай, можно создать и запустить один или несколько поисков контента, соответствующего делу. Поиски, связанные с делом об обнаружении  основных электронных данных, не перечислены на странице поиска контента в центре Microsoft 365 соответствия требованиям. Эти поиски перечислены на странице **Поиски** в случае core eDiscover, с чем связаны поиски. Это также означает, что к поискам, связанным с делом, можно получить доступ только к участникам дела.

Чтобы создать поиск по обнаружению основных электронных обнаружений:
  
1. Перейдите к учетным данным для учетной записи пользователя, которая получила соответствующие разрешения на открытие электронных данных и является участником <https://compliance.microsoft.com> дела.

2. В левой области навигации центра Microsoft 365 нажмите кнопку **Показать** все, а затем нажмите **кнопку eDiscovery > Core**.

3. На странице **Core eDiscovery** выберите случай, в который необходимо создать связанный поиск, а затем нажмите **кнопку Открыть дело**.

4. На **домашней странице** для дела нажмите вкладку **Поиск** и нажмите **кнопку Новый поиск**.

   ![Щелкните Новый поиск для создания поиска core eDiscovery](../media/CoreeDiscoverySearch1.png)

   > [!NOTE]
   > Параметр **Search by ID list** позволяет искать определенные сообщения электронной почты и другие элементы почтовых ящиков с помощью списка Exchange ID. Чтобы создать поиск по списку идентификаторов, нужно отправить CSV-файл, в котором указаны конкретные элементы почтового ящика для поиска. Инструкции см. в статье [Подготовка CSV-файла для поиска контента по списку идентификаторов](csv-file-for-an-id-list-content-search.md).

5. В **мастере поиска New** введите имя для поиска и необязательное описание, которое поможет определить поиск. Имя поиска должно быть уникальным в организации.

6. На странице **Расположения** выберите расположения контента, которые необходимо искать. Вы можете искать почтовые ящики, сайты и общедоступные папки.

    ![Выбор расположения контента для постановки на удержание](../media/ContentSearchLocations.png)
  
   1. **Exchange** почтовых ящиков: Установите кнопку  "Вкл", а затем нажмите кнопку Выберите **пользователей,** групп или групп, чтобы указать почтовые ящики для удержания. Используйте поле поиска, чтобы найти почтовые ящики пользователей и группы рассылки (чтобы разместить удержание на почтовых ящиках членов группы) для удержания. Вы также можете искать почтовый ящик, связанный с командой Майкрософт (для сообщений каналов), Office 365 group и Yammer Group. Дополнительные сведения о данных приложений, хранимых в почтовых ящиках, см. в материалах Content, хранимых в почтовых ящиках [для получения сведений об обнаружении электронных данных.](what-is-stored-in-exo-mailbox.md)

   2. **SharePoint** сайтов: Установите кнопку "Нажатие", а затем нажмите кнопку Выберите сайты, чтобы указать SharePoint сайты и OneDrive учетные записи для удержания.   Укажите URL-адрес каждого сайта, который вы хотите поставить на удержание. Вы также можете добавить URL-адрес SharePoint для microsoft Team, Office 365 группы или Yammer Group.
  
   3. **Exchange общедоступных папок:** Установите перегной, чтобы все общедоступные папки в организации Exchange Online на удержание.  Вы не можете выбрать определенные общедоступные папки для удержания. Если вы не хотите удерживать общедоступные папки, отключите переключатель.
  
   4. Выберите этот почтовый ящик для поиска Teams контента для локального пользователя. Например, при поиске Exchange почтовых ящиков в организации и выбора этого почтового ящика облачное хранилище, используемого для хранения данных чата Teams для локального пользователя, будет включено в область поиска. Дополнительные сведения см. в статье [Поиск данных в разделе "чат" для локальных пользователей](search-cloud-based-mailboxes-for-on-premises-users.md).

7. На странице **Определение условий поиска** введите ключевой запрос и при необходимости добавьте условия в поисковый запрос.

   ![Настройка запроса поиска](../media/ContentSearchQuery.png)

   1. Укажите ключевые слова, свойства сообщений, такие как отправленные и полученные даты, или свойства документов, такие как имена файлов или дата последнего изменения документа. Можно создавать более сложные запросы, включающие логические операторы, например **AND**, **OR**, **NOT** и **NEAR**. Если оставить незаполненным поле для ввода ключевых слов, в результаты поиска будет включено все содержимое из указанных расположений контента. Дополнительные сведения см. в [статье Keyword queries and search conditions for eDiscovery.](keyword-queries-and-search-conditions.md)

   2. Кроме того, вы можете установить флажок **Отобразить список ключевых слов** и вводить ключевые слова в каждой строке. В этом случае в создаваемом поисковом запросе ключевые слова из каждой строки будут соединены логическим оператором (**c:s**), аналогичным по функциональности оператору **OR**.

      Зачем использовать список ключевых слов? Вы можете получить статистические сведения о том, сколько элементов соответствует каждому ключевому слову. Это поможет быстро определить, какие ключевые слова наиболее (и наименее) эффективны. В строке можно также использовать ключевую фразу (в скобках). Дополнительные сведения о списке ключевых слов и статистике поиска см. в статье [Get keyword statistics for searches.](view-keyword-statistics-for-content-search.md#get-keyword-statistics-for-searches)

      > [!NOTE]
      > Чтобы уменьшить количество проблем, вызванных большими списками ключевых слов, в списке ключевых слов не более 20 строк.

   3. Можно добавить условия поиска, чтобы сузить поиск и вернуть более уточненный набор результатов. Каждое условие добавляет предложение к поисковому запросу, которое создается и запускается в начале поиска. Условие логически связано с запросом по ключевым словам, указанным в поле для ввода ключевых слов, с помощью логического оператора (**c:c**), аналогичного по функциональности оператору **AND**. Это означает, что элементы попадают в результаты поиска, если соответствуют как запросу по ключевому слову, так и одному или нескольким условиям. Таким образом условия помогают сузить область результатов поиска. Перечень и описание условий, которые можно использовать в поисковом запросе, см. в статье [условия поиска](keyword-queries-and-search-conditions.md#search-conditions).

8. Просмотрите параметры поиска (и при необходимости отредактировать), а затем отправьте поиск для его запуска.

После завершения поиска можно просмотреть его результаты. При необходимости **щелкните Обновление** на странице **Поиски,** чтобы отобразить созданный поиск.

## <a name="more-information-about-searching-content-locations"></a>Дополнительные сведения о поиске местоположений контента

- При нажатии кнопки **Выберите пользователей, групп** или групп, чтобы указать почтовые ящики для поиска, отображаемая выборка почтовых ящиков пуста. Это вызвано мерами по повышению скорости работы. Чтобы добавить получателей в этот список, щелкните Выберите **пользователей,** групп или групп, введите имя (не менее трех символов) в поле поиска, выберите поле рядом с именем, а затем нажмите **кнопку Выберите**.

- В список почтовых ящиков для поиска можно добавить неактивные почтовые ящики, Microsoft Teams, Yammer группы, группы Office 365 и группы рассылки. Динамические группы рассылки не поддерживаются. Если вы добавляете Microsoft Teams, Yammer группы или Office 365 группы, группа или почтовый ящик группы будут искать; почтовые ящики участников группы не ищутся.

- Чтобы добавить сайты в поиск, включайте очки и нажмите **кнопку Выберите сайты.** Введите URL-адрес для каждого сайта, который необходимо искать. Вы также можете добавить URL SharePoint для microsoft Team, Yammer группы или Office 365 группы.
