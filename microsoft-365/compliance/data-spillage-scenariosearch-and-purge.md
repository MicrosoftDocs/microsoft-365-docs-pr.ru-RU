# <a name="ediscovery-solution-series-data-spillage-scenario---search-and-purge"></a>Серия решений для eDiscovery: сценарий утечки данных — поиск и очистка

 **Что такое утечка данных и почему вас это необходимо?** Утечка данных происходит, когда конфиденциальный документ выходит в недопустимую среду. При обнаружении инцидента утечки данных важно быстро оценить размер и расположение утечки, изучить действия пользователей вокруг него, а затем окончательно очистить утечку данных из системы. 
  
## <a name="data-spillage-scenario"></a>Сценарий утечки данных

Вы ведущий сотрудник по информационной безопасности в компании Contoso. Вы будете знать о ситуации утечки данных, когда сотрудник непреднаученно поделился строго конфиденциальным документом с несколькими людьми по электронной почте. Вы хотите быстро оценить, кто получил этот документ внутри и извне. После этого вы хотите поделиться результатами дела с другими следователями для проверки, а затем окончательно удалить данные из Office 365. По завершению исследования необходимо создать отчет с свидетельством о окончательном удалении и другими сведениями дела для любой дальнейшей ссылки.
  
### <a name="scope-of-this-article"></a>Область действия этой статьи

В этом документе содержится список инструкций по окончательному удалению сообщения из Microsoft 365, чтобы оно было недоступным и не может быть восстановлено. Чтобы удалить сообщение и сделать его восстановленным до истечения срока хранения удаленных элементов, см. поиск и удаление сообщений электронной почты [в организации.](search-for-and-delete-messages-in-your-organization.md)

## <a name="things-to-know-before-you-start"></a>Что нужно знать перед началом

- Если почтовый ящик находится на удержании, удаленное сообщение остается в папке "Элементы с восстановления" до истечения срока хранения или с момента сохранения. [В шаге 6](#step-6-prepare-the-mailboxes) описано, как удалить удержание для почтовых ящиков. Перед снятием удержания обратитесь в отдел управления записями или юридические отделы. В вашей организации может быть политика, которая определяет, имеет ли приоритет почтовый ящик на удержании или инцидент утечки данных. 
    
- Чтобы контролировать, какие почтовые ящики пользователей может искать и управлять тем, кто может получить доступ к делу, можно настроить границы соответствия требованиям и создать настраиваемую группу ролей, как описано в шаге [1.](#optional-step-1-manage-who-can-access-the-case-and-set-compliance-boundaries) Для этого необходимо быть членом группы ролей "Управление организацией" или иметь роль управления ролем. Если вы или администратор вашей организации уже установили границы соответствия требованиям, вы можете пропустить шаг 1.
    
- Чтобы создать дело, необходимо быть членом группы ролей "Руководитель eDiscovery" или членом настраиваемой группы ролей, которая назначена ролью управления делами. Если вы не участник, попросите администратора Microsoft 365 добавить вас в группу ролей диспетчера [eDiscovery.](assign-ediscovery-permissions.md)
    
- Создавать и выполнять поиск контента могут только члены группы ролей "Диспетчер eDiscovery" и пользователи с ролью "Управление поиском соответствия". Удалять сообщения могут только члены группы ролей "Управление организацией" и пользователи с ролью "Управление поиском и очисткой". Сведения о добавлении пользователей в группу ролей см. в статье [Назначение разрешений обнаружения электронных данных в Центре безопасности и соответствия требованиям](https://docs.microsoft.com/microsoft-365/compliance/assign-ediscovery-permissions).
    
- Для поиска действий по обнаружению электронных данным в журнале аудита на шаге 8 аудит должен быть включен для вашей организации. Вы можете искать действия, которые были выполнены за последние 90 дней. Дополнительные сведения о в том, как [](#auditing-the-data-spillage-investigation-process) включить и использовать аудит, см. в разделе аудита процесса анализа утечки данных в действии 8. 
    
## <a name="optional-step-1-manage-who-can-access-the-case-and-set-compliance-boundaries"></a>(Необязательно) Шаг 1. Управление доступом к делу и настройка границ соответствия требованиям

В зависимости от вашей организационной практики необходимо контролировать, кто может получить доступ к делу eDiscovery, используемого для расследования инцидента утечки данных, и настроить границы соответствия. Самый простой способ сделать это — добавить следователей в качестве членов существующей группы ролей в Центре безопасности & соответствия требованиям, а затем добавить группу ролей в качестве участника дела eDiscovery. Сведения о встроенных группах ролей eDiscovery и добавлении участников в дело eDiscovery см. в подстройки "Назначение разрешений на [eDiscovery".](assign-ediscovery-permissions.md)
  
Вы также можете создать новую группу ролей, которая соответствует потребностям организации. Например, может потребоваться, чтобы группа экспертов по утечке данных в организации могла получать доступ ко всем делам утечки данных и совместно работать над ними. Это можно сделать, создав группу ролей "Раскрытый агент утечки данных", назначив соответствующие роли (экспорт, расшифровка RMS, проверка, просмотр, поиск соответствия и управление делами), добавив в группу ролей расследования утечки данных, а затем добавив группу ролей в качестве участника дела eDiscovery утечки данных. Подробные инструкции по этому действию см. в описании границ соответствия требованиям для расследований [eDiscovery в Office 365.](tagging-and-assessment-in-advanced-ediscovery.md) 
  
## <a name="step-2-create-an-ediscovery-case"></a>Шаг 2. Создание дела eDiscovery

Дело eDiscovery обеспечивает эффективный способ управления анализом утечки данных. Вы можете добавить участников в группу ролей, созданную на шаге 1, добавить группу ролей в качестве участника нового дела eDiscovery, выполнить итеративные поиски для поиска утечки данных, экспортировать отчет для обмена, отслеживать состояние дела, а затем при необходимости вернуться к сведениям дела. Рассмотрите возможность создания соглашения об именованиях для дел обнаружения электронных данных, используемых для инцидентов утечки данных, и укажайте как можно больше информации в имени и описании дела, чтобы при необходимости вы могли находить и ссылаться на них в будущем.
  
Для создания нового дела можно использовать eDiscovery в Центре безопасности и соответствия требованиям. См. "Создание нового дела" в делах [eDiscovery.](ediscovery-cases.md#step-2-create-a-new-case)
  
## <a name="step-3-search-for-the-spilled-data"></a>Шаг 3. Поиск утечки данных

Теперь, когда вы создали дело и управляемый доступ, вы можете использовать его для итеративного поиска, чтобы найти утечки данных и определить почтовые ящики, содержащие утечки данных. Вы будете использовать тот же поисковый запрос, который использовался для поиска сообщений электронной почты для удаления этих сообщений на шаге [7.](#step-7-permanently-delete-the-spilled-data)
  
Чтобы создать поиск контента, связанный с делом eDiscovery, см. статью "Создание и запуск поиска контента, связанного с делом" в делах [eDiscovery.](ediscovery-cases.md#step-5-create-and-run-a-content-search-associated-with-a-case)
  
 **Важно!** Ключевые слова, которые вы используете в поисковом запросе, могут содержать фактически утечки данных, которые вы ищете. Например, если вы ищете документы, содержащие номер социального обеспечения, и используете его в качестве ключевого слова поиска, необходимо удалить запрос позже, чтобы избежать дальнейшего утечки. См. ["Удаление поискового запроса"](#deleting-the-search-query) на шаге 8. 
  
## <a name="step-4-review-and-validate-case-findings"></a>Шаг 4. Проверка и проверка результатов дела

После создания поиска контента необходимо проверить результаты поиска и убедиться, что они состоят только из сообщений электронной почты, которые необходимо удалить. При поиске контента можно просмотреть случайную выборку из 1000 сообщений электронной почты без экспорта результатов поиска, чтобы избежать дальнейшего утечки данных. Дополнительные материалы об ограничениях предварительного просмотра можно найти на [этой теме.](limits-for-content-search.md)
  
Если у вас более 1000 почтовых ящиков или более 100 сообщений электронной почты на каждый почтовый ящик для проверки, вы можете разделить первоначальный поиск на несколько поисковых запросов, используя дополнительные ключевые слова или условия, такие как диапазон дат или отправитель/получатель, и просмотреть результаты каждого поиска по отдельности. Обязательно запишите все поисковые запросы, которые будут применяться при удалении сообщений на [шаге 7.](#step-7-permanently-delete-the-spilled-data)

Если хранителям или конечным пользователям назначена лицензия на Office 365 E5, вы можете проверить до 10 000 результатов поиска одновременно с помощью Advanced eDiscovery. Если просматривается более 10 000 сообщений электронной почты, вы можете разделить поисковый запрос по диапазону дат и просмотреть каждый результат по отдельности, так как результаты поиска сортироваться по дате. В Advanced eDiscovery можно пометить результаты поиска с помощью метки как функции на панели предварительного просмотра и отфильтровать результаты поиска по метке.  Это полезно при совместной работе со вторичным рецензентом. С помощью дополнительных средств аналитики в Advanced eDiscovery, таких как распознавание текста, потоки электронной почты и прогнозирование кодирования, вы можете быстро обработать и просмотреть тысячи сообщений и пометить их для дальнейшего просмотра. См. ["Быстрая настройка Advanced eDiscovery".](quick-setup-for-advanced-ediscovery.md)

Когда вы найдете сообщение электронной почты, содержащую утечку данных, проверьте получателей сообщения, чтобы определить, был ли он отправлен извне. Для дальнейшей трассировки сообщения можно собирать сведения об отправителе и диапазоне дат, чтобы использовать журналы трассировки сообщений, описанные в [шаге 5.](#step-5-use-message-trace-log-to-check-how-spilled-data-was-shared)

После проверки результатов поиска может потребоваться поделиться своими результатами с другими для дополнительного просмотра. Люди, которые были назначены делу на шаге 1, могут просмотреть содержимое дела как в eDiscovery, так и в Advanced eDiscovery, и утвердить результаты дела. Вы также можете создать отчет без экспорта фактического контента. Вы также можете использовать этот отчет в качестве подтверждения удаления, как описано в [шаге 8.](#step-8-verify-provide-a-proof-of-deletion-and-audit)
  
 **Чтобы создать статистический отчет:**
  
1. Перейдите на **страницу** поиска в деле eDiscovery и щелкните поиск, для чего нужно создать отчет. 
    
2. На странице "Flyout" щелкните **"Отчет > экспорта".**
 
      Отображается страница отчета "Экспорт".

    ![Выберите поиск, а затем нажмите кнопку "> Экспорт" на странице".](../media/O365-eDiscoverySolutions-DataSpillage-ExportReport1.png)
    
3. Выберите **все элементы,** в том числе те, которые имеют неоформатизированный формат, зашифрованы или не были проиндексированы по другим причинам, а затем нажмите кнопку **"Создать отчет".**

4. В случае eDiscovery нажмите кнопку **"Экспорт",** чтобы отобразить список заданий экспорта. Может потребоваться  нажать кнопку "Обновить", чтобы обновить список, чтобы отобразить только что созданное задание экспорта.

5. Щелкните задание экспорта и нажмите кнопку **"Загрузить** отчет" на странице".
 
    ![На странице "Экспорт" щелкните экспорт и выберите "Загрузить отчет"](../media/O365-eDiscoverySolutions-DataSpillage-ExportReport2.png)

Сводный **отчет об экспорте** содержит количество найденных местоположений с результатами и размер результатов поиска. Его можно использовать для сравнения с отчетом, созданным после удаления, и в качестве подтверждения удаления. Отчет **"Результаты"** содержит более подробную сводку результатов поиска, включая тему, отправитель, получателей, если сообщение было прочитано, даты и размер каждого сообщения. Если какие-либо сведения в этом отчете содержат фактические данные, не забудьте окончательно удалить Results.csv после завершения исследования.

Дополнительные сведения об экспорте отчетов см. в [отчете "Экспорт отчета о поиске контента".](export-a-content-search-report.md)
    
## <a name="step-5-use-message-trace-log-to-check-how-spilled-data-was-shared"></a>Шаг 5. Использование журнала трассировки сообщений для проверки общего использования утечки данных

Чтобы дополнительно выяснить, был ли общий доступ к электронной почте с утечками данных, можно запросить журналы трассировки сообщений с помощью сведений об отправителе и диапазона дат, собранных на шаге 4. Обратите внимание, что период хранения для трассировки сообщений составляет 30 дней для данных в режиме реального времени и 90 дней для исторических данных.
  
Трассировка сообщений можно использовать в Центре безопасности и соответствия требованиям или с помощью соответствующих cmdlets в Exchange Online PowerShell. Важно отметить, что трассировка сообщений не дает полных гарантий по полноте возвращаемой информации. Дополнительные сведения об использовании трассировки сообщений см. в: 
  
- [Трассировка сообщений в Центре безопасности и соответствия требованиям](https://docs.microsoft.com/microsoft-365/security/office-365-security/message-trace-scc)
    
- [Новая трассировка сообщений в Центре & соответствия требованиям](https://blogs.technet.microsoft.com/exchange/2018/05/02/new-message-trace-in-office-365-security-compliance-center/)
    
## <a name="step-6-prepare-the-mailboxes"></a>Шаг 6. Подготовка почтовых ящиков

После проверки и проверки того, что результаты поиска содержат только те сообщения, которые необходимо удалить, необходимо собрать список адресов электронной почты почтовых ящиков, которые будут применяться в шаге 7 при удалении утечки данных. Вам также может потребоваться подготовить почтовые ящики, прежде чем вы сможете окончательно удалить сообщения электронной почты в зависимости от того, включено ли восстановление одного элемента для почтовых ящиков, содержащих утечку данных, или если какой-либо из этих почтовых ящиков находится на удержании.
  
### <a name="get-a-list-of-addresses-of-mailboxes-with-spilled-data"></a>Получить список адресов почтовых ящиков с утечками данных

Существует два способа сбора списка адресов электронной почты почтовых ящиков с утечками данных.

**Вариант 1. Получите список адресов почтовых ящиков с утечками данных**

1. Откройте дело eDiscovery, перейдите **на** страницу поиска и выберите соответствующий поиск контента. 
    
2. На странице "Flyout" щелкните **"Просмотреть результаты".**
    
3. В раскрывающемся списке **Отдельные результаты** выберите пункт **Статистика поиска**.
    
4. В **выпадаемом** списке "Тип" щелкните **"Лучшие расположения".**
    
    ![Получить список почтовых ящиков, содержащих результаты поиска на странице "Лучшие расположения" в статистике поиска](../media/O365-eDiscoverySolutions-DataSpillage-TopLocations.png)

    Отображается список почтовых ящиков, содержащих результаты поиска. Также отображается количество элементов в каждом почтовом ящике, которое соответствует поисковому запросу.
    
5. Скопируйте сведения из списка и сохраните  их в файл или нажмите кнопку "Скачать", чтобы скачать их в CSV-файл. 
    
**Вариант 2. Получите расположения почтовых ящиков из отчета об экспорте**

Откройте сводный отчет об экспорте, загруженный на [шаге 4.](#step-4-review-and-validate-case-findings) В первом столбце отчета адрес электронной почты каждого почтового ящика указан в списке **"Расположения".**
  
### <a name="prepare-the-mailboxes-so-you-can-delete-the-spilled-data"></a>Подготовка почтовых ящиков для удаления утечки данных

Если восстановление отдельного элемента включено или почтовый ящик помещен на удержание, окончательно удаленное (удаленное) сообщение будет храниться в папке "Элементы с возможностью восстановления". Поэтому перед очисткой утечки данных необходимо проверить существующие конфигурации почтовых ящиков и отключить восстановление одного элемента, а также отключить удержание или политику хранения. Помните, что вы можете подготовить один почтовый ящик за раз, а затем выполнить одну команду для разных почтовых ящиков или создать сценарий PowerShell для подготовки нескольких почтовых ящиков одновременно.

- Инструкции о том, как проверить, включено ли восстановление отдельного элемента или включен ли почтовый ящик на удержании или назначены ли ему политики хранения, см. в статье "Шаг 1. Сбор сведений о почтовом ящике" в статье "Удаление элементов в папке "Элементы с возможностью восстановления" облачных почтовых ящиков на удержании. [](delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md#step-1-collect-information-about-the-mailbox) 

- Инструкции по отключению восстановления одного [](delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md#step-2-prepare-the-mailbox) элемента см. в подготовимом шаге "Шаг 2. Подготовка почтового ящика" в статье "Удаление элементов в папке "Элементы с возможностью восстановления" облачных почтовых ящиков на удержании. 

- Инструкции по удалению удержания или политики [](delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md#step-3-remove-all-holds-from-the-mailbox) хранения из почтового ящика см. в под вопросе "Шаг 3. Удаление всех удержаний из почтового ящика" статьи "Удаление элементов в папке "Элементы с возможностью восстановления" облачных почтовых ящиков на удержании. 

- См. "Шаг 4. Удаление удержания задержки для почтового ящика" статьи "Удаление элементов в папке "Элементы с возможностью восстановления" облачных почтовых ящиков на удержании для указания об удалении удержания задержки, которое помещается на почтовый ящик после снятия удержания любого типа. [](delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md#step-4-remove-the-delay-hold-from-the-mailbox)

> [!IMPORTANT]
> Перед удалением удержания или политики хранения обратитесь в отдел управления записями или юридические отделы. В вашей организации может быть политика, которая определяет, имеет ли почтовый ящик на удержании или инцидент утечки данных приоритет. 
  
Не забудьте вернуться к предыдущим конфигурациям почтового ящика после того, как убедитесь, что утечка данных была окончательно удалена. См. сведения в [шаге 7.](#step-7-permanently-delete-the-spilled-data)

## <a name="step-7-permanently-delete-the-spilled-data"></a>Шаг 7. Окончательное удаление утечки данных

Используя расположения почтовых ящиков, собранные и подготовленные на шаге 6, и поисковый запрос, созданный и уточненный в шаге 3 для поиска сообщений электронной почты, содержащих утечку данных, теперь вы можете окончательно удалить утечку данных.  Как объяснялось ранее, для удаления сообщений необходимо быть членом группы ролей "Управление организацией" или иметь роль управления "Поиск и очистка". Сведения о добавлении пользователей в группу ролей см. в статье [Назначение разрешений обнаружения электронных данных в Центре безопасности и соответствия требованиям](https://docs.microsoft.com/microsoft-365/compliance/assign-ediscovery-permissions).

Чтобы удалить пролитые сообщения, см. действия 2 & 3 в поиске и [удалении сообщений электронной почты](https://docs.microsoft.com/microsoft-365/compliance/search-for-and-delete-messages-in-your-organization)

> [!IMPORTANT]
> Элементы электронной почты в наборе для проверки в деле Advanced eDiscovery невозможно удалить, используя процедуры, описанные в этой статье. Это потому, что элементы в наборе для проверки — это копии элементов в службе, которые копируется и хранится в хранилище Azure. Это означает, что они не будут возвращены поиском контента, который вы создали на шаге 3. Чтобы удалить элементы в наборе для проверки, необходимо удалить дело Advanced eDiscovery, содержащее этот набор для проверки. Дополнительные сведения см. в статье [Закрытие или удаление дела Advanced eDiscovery](close-or-delete-case.md).
  
## <a name="step-8-verify-provide-a-proof-of-deletion-and-audit"></a>Шаг 8. Проверка, предоставление подтверждения удаления и аудит

Последний шаг рабочего процесса по управлению инцидентом утечки данных — убедиться, что утечка данных была окончательно удалена из почтового ящика, переходить к делу eDiscovery и повторно запускать тот же поисковый запрос, который использовался для удаления этих данных, чтобы убедиться, что результаты не возвращаются. После подтверждения того, что утечка данных была окончательно удалена, вы можете экспортировать отчет и включить его (вместе с исходным отчетом) в качестве подтверждения удаления. Затем можно [закрыть дело,](ediscovery-cases.md#optional-step-9-close-a-case)что позволит вам повторно открыть его, если вы будете ссылаться на него в будущем. Кроме того, можно также привести почтовые ящики в прежнее состояние, удалить поисковый запрос, используемый для поиска утечки данных, и найти записи аудита задач, выполняемых при управлении инцидентом утечки данных. 
  
### <a name="reverting-the-mailboxes-to-their-previous-state"></a>Обращение почтовых ящиков в предыдущее состояние

Если в шаге 6 была изменена конфигурация почтовых ящиков для подготовки почтовых ящиков до удаления утечки данных, необходимо будет вернуться к их предыдущему состоянии. См. "Шаг 6. Восстановление предыдущего состояния почтового ящика" статьи "Удаление элементов в папке "Элементы с возможностью восстановления" облачных почтовых ящиков на [удержании.](delete-items-in-the-recoverable-items-folder-of-mailboxes-on-hold.md#step-6-revert-the-mailbox-to-its-previous-state)
  
### <a name="deleting-the-search-query"></a>Удаление поискового запроса

Если ключевые слова в поисковом запросе, созданном и использованного на шаге 3, содержат некоторые из фактических утечки данных, удалите поисковый запрос, чтобы предотвратить дальнейшее утечку данных.
  
1. В Центре безопасности и соответствия требованиям откройте дело eDiscovery, перейдите на страницу поиска и выберите соответствующий поиск контента. 
    
2. На странице "Flyout" нажмите кнопку **"Удалить".**

    ![Выберите поиск и нажмите кнопку "Удалить" на странице".](../media/O365-eDiscoverySolutions-DataSpillage-DeleteSearch.png)
