# <a name="compare-versions-of-ome"></a>Сравнение версий OME

В этой статье сравнивается устаревшее шифрование сообщений Office 365 (OME) с новыми возможностями OME и расширенным шифрованием сообщений Office 365. Новые возможности — это слияние и более новая версия как OME, так и управления правами на доступ к данным (IRM). Также обрабатываются уникальные характеристики развертывания в GCC High. Эти два могут сосуществовать в Организации. Сведения о том, как работают новые возможности, можно найти в статье [Шифрование сообщений в Office 365 (OME)](ome.md).

## <a name="advantages-of-the-new-ome-capabilities-over-legacy-ome"></a>Преимущества новых возможностей OME по сравнению с прежними версиями OME

Новые возможности предоставляют следующие преимущества:

- Возможность использовать Encrypt-Only (что обеспечивает безопасную совместную работу), не пересылать и настраиваемые ограничения.
- Отправители могут отправлять сообщения, зашифрованные с новыми возможностями, вручную из Outlook Desktop, Outlook для Mac и Outlook в веб-клиентах.
- Пользователи Microsoft 365 получают возможность использовать встроенные возможности в поддерживаемых клиентах Outlook. Кроме того, администраторы могут показывать фирменный опыт для получателей Microsoft 365.
- Учетные записи, которые не входят в состав Microsoft 365, например, Gmail, Yahoo и учетные записи Майкрософт, являются федеративными с порталом OME, что обеспечивает более удобную работу пользователей для этих получателей. Все остальные удостоверения используют однократный код прохода для доступа к зашифрованным сообщениям.
- Администраторы могут настраивать фирменную символику и создавать различные шаблоны фирменного стиля.
- Администраторы могут отозвать электронные сообщения, зашифрованные с новыми возможностями.
- Новые возможности предоставляют подробные отчеты об использовании в центре безопасности и &amp; соответствия требованиям.

## <a name="office-365-advanced-message-encryption-capabilities"></a>Расширенные возможности шифрования сообщений в Office 365

Расширенное шифрование сообщений Office 365 предоставляет дополнительные возможности на основе новых возможностей OME. Для использования расширенных возможностей шифрования сообщений необходимо настроить новые возможности шифрования сообщений Office 365 в вашей организации. Кроме того, чтобы использовать эти возможности, получатели должны просматривать защищенную почту через портал OME и отвечать на них. К дополнительным возможностям относятся:

- Отзыв сообщений

- Истечение срока действия сообщения

- Несколько шаблонов фирменной символики

Расширенное шифрование сообщений Office 365 не поддерживается в GCC High.

Более подробную информацию об использовании расширенного шифрования сообщений можно узнать в статье [Office 365 Advanced Message Encryption](ome-advanced-message-encryption.md).

## <a name="unique-characteristics-of-office-365-message-encryption-in-a-gcc-high-deployment"></a>Уникальные характеристики шифрования сообщений Office 365 при высоком развертывании GCC

Если вы планируете использовать шифрование сообщений Office 365 в среде с большим числом GCC, существуют уникальные характеристики, касающиеся взаимодействия с получателем.

### <a name="encrypted-email-between-gcc-high-and-gcc-high-recipients"></a>Зашифрованная электронная почта между GCC High и большим получателям GCC

Отправители могут вручную зашифровывать сообщения в Outlook для ПК и Mac, а также Outlook в Интернете, а организации могут настраивать политику для шифрования электронных сообщений с помощью правил для почтовых ящиков Exchange.

Получатели в режиме GCC High получают одинаковый режим чтения в Outlook для компьютеров с Windows и Mac и Outlook в Интернете в качестве всех остальных пользователей.

### <a name="encrypted-email-between-gcc-high-and-non-gcc-high-recipients"></a>Зашифрованные сообщения электронной почты между большинством получателей GCC High и non GCC High

Отправители внутри GCC High могут отправлять зашифрованную электронную почту за пределами самой большой границы GCC и наоборот.

Все получатели за преличную высоту GCC, в том числе коммерческие пользователи Microsoft 365, пользователи Outlook.com и другие пользователи других поставщиков электронной почты, такие как Gmail и Yahoo, получают сообщения программы-оболочки. Эта почтовая оболочка перенаправляет получателя на портал OME, где получатель может прочитать сообщение и ответить на него. Это также относится к отправителям за пресылаемыми сообщениями GCC High Send OME encrypted mail to GCC High.

## <a name="coexistence-of-legacy-ome-and-the-new-capabilities-in-the-same-tenant"></a>Сосуществование устаревших OME и новых возможностей в одном клиенте

Вы можете использовать как устаревшие OME, так и новые возможности в одном клиенте. Как администратор, вы можете сделать это, выбрав версию OME, которую нужно использовать при создании правил для своего почтового процесса.

- Чтобы указать устаревшую версию OME, используйте действие правила обработки почтовых ящиков Exchange **Примените предыдущую версию OME**.

- Чтобы указать новые возможности, используйте действие правила обработки почтового ящика Exchange для **применения шифрования сообщений Office 365 и защиты прав**.

Пользователи могут вручную отправлять сообщения, зашифрованные с помощью новых возможностей с рабочего стола Outlook, Outlook для Mac и Outlook в Интернете.

## <a name="migrate-from-legacy-ome-to-the-new-capabilities"></a>Переход с устаревших OME на новые возможности

Несмотря на то, что обе версии OME могут сосуществовать, настоятельно рекомендуется изменить старые правила обработки почтового ящика, которые используют действие правила, **Примените предыдущую версию OME** , чтобы использовать новые возможности. Обновите эти правила, чтобы использовать действие правила обработки почтового ящика **применить шифрование и защиту сообщений Office 365**. Инструкции приведены в разделе [Определение правил для почтового процесса для шифрования сообщений электронной почты в Office 365](define-mail-flow-rules-to-encrypt-email.md).

## <a name="get-started-with-ome"></a>Начало работы с OME

Как правило, для организации автоматически включаются новые возможности OME. Дополнительные сведения о новых возможностях OME в Организации можно узнать в статье [Настройка новых возможностей шифрования сообщений Office 365](set-up-new-message-encryption-capabilities.md).

Устаревшая версия OME автоматически включается для вашей организации, если вы включили Azure Information Protection. Ранее устаревшие OME работали, даже если не включена служба Azure Information Protection. Теперь это не так.

Чтобы начать работу с устаревшей OME, если вы включили Azure Information Protection, настройте правила для обработки почтового ящика, которые используют действие правила, **и примените предыдущую версию OME**. Инструкции приведены в разделе [Определение правил для почтового процесса для шифрования сообщений электронной](define-mail-flow-rules-to-encrypt-email.md)почты.
