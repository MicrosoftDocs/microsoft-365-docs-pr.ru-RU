## <a name="overview-of-ad-rms-deprecation-in-exchange-online"></a>Обзор амортизации AD RMS в Exchange Online

Exchange Online включает функции управления правами на информацию (IRM), которые обеспечивают защиту сообщений и вложений электронной почты в интернете и автономном режиме. По умолчанию Exchange Online Azure Information Protection. Однако ваша организация, возможно, Exchange Online IRM для использования локальной службы управления правами active Directory (AD RMS). Поддержка AD RMS в Exchange Online уходит в отставку. Вместо этого Azure Information Protection полностью заменит AD RMS.

Чтобы оценить, влияет ли это амортизации на организацию, см. в рублях Как перенести [AD RMS в Azure RMS в Exchange Online.](https://support.microsoft.com/help/5001237) В этой статье данная статья содержит рекомендации по вариантам миграции.

## <a name="advantages-of-the-new-ome-capabilities-over-legacy-ome"></a>Преимущества новых возможностей OME по сравнению с устаревшим OME

Новые возможности предоставляют следующие преимущества:

- Возможность использования параметра только для шифрования (который позволяет обеспечить безопасное совместное использование), параметра Не перенадвигать и настраиваемые ограничения.
- Отправители могут отправлять почту в зашифрованном виде с новыми возможностями вручную из Outlook, Outlook для Mac и Outlook веб-клиентов.
- Microsoft 365 получатели получают возможность использовать в поддерживаемых клиентах Outlook опыт работы с Outlook. Кроме того, администраторы могут Microsoft 365 получателям фирменный интерфейс.
- Учетные записи за пределами Microsoft 365, такие как учетные записи Gmail, Yahoo и Microsoft, федератированы с помощью портала OME, что обеспечивает более удобное пользовательский интерфейс для этих получателей. Для доступа к зашифрованным сообщениям все другие идентификаторы используют код одновекового пропуска.
- Администраторы могут настраивать брендинг и создавать несколько шаблонов брендинга.
- Администраторы могут отоскить сообщения электронной почты, зашифрованные с помощью новых возможностей.
- Новые возможности предоставляют подробные отчеты об использовании через Центр соответствия &amp; требованиям безопасности.

## <a name="office-365-advanced-message-encryption-capabilities"></a>Расширенное шифрование сообщений Office 365 возможности

Расширенное шифрование сообщений Office 365 дополнительные возможности в верхней части новых возможностей OME. Для использования возможностей предварительного шифрования сообщений необходимо создать новые шифрование сообщений Office 365 в организации. Кроме того, для использования этих возможностей получатели должны просматривать и отвечать на них, чтобы обеспечить безопасность почты через портал OME. Расширенные возможности включают:

- Отзыв сообщений

- Срок действия сообщения

- Несколько шаблонов брендинга

Расширенное шифрование сообщений Office 365 не поддерживается в GCC High.

Сведения об использовании предварительного шифрования сообщений [см. в](ome-advanced-message-encryption.md)Расширенное шифрование сообщений Office 365.

## <a name="unique-characteristics-of-office-365-message-encryption-in-a-gcc-high-deployment"></a>Уникальные характеристики шифрование сообщений Office 365 в GCC высоком развертывании

Если вы планируете использовать шифрование сообщений Office 365 в GCC среде High, существуют некоторые уникальные характеристики, касающиеся опыта получателя.

### <a name="encrypted-email-between-gcc-high-and-gcc-high-recipients"></a>Зашифрованная электронная почта между GCC и GCC высокими получателями

Отправители могут вручную шифровать сообщения электронной почты в Outlook для ПК и Mac и Outlook в Интернете, или организации могут настроить политику шифрования электронной почты с помощью Exchange правил потока почты.

Получатели GCC high получают такой же опыт чтения в Outlook для ПК и Mac и Outlook в Интернете, как и все другие пользователи.

### <a name="encrypted-email-between-gcc-high-and-non-gcc-high-recipients"></a>Зашифрованная электронная почта GCC высокой и не GCC высокой получателей

Отправители GCC high могут отправлять зашифрованную электронную почту за пределами GCC высокой границы и наоборот.

Все получатели за пределами GCC High, включая коммерческих Microsoft 365 пользователей, пользователей Outlook.com и других пользователей других поставщиков электронной почты, таких как Gmail и Yahoo, получают почту-оболочку. Эта почта обертки перенаправляет получателя на портал OME, где получатель может читать и отвечать на сообщение. Это также относится к отправителям за пределами GCC высокой отправки зашифрованной почты OME в GCC High.

## <a name="coexistence-of-legacy-ome-and-the-new-capabilities-in-the-same-tenant"></a>Сосуществование устаревших OME и новых возможностей в том же клиенте

В одном клиенте можно использовать как устаревшие OME, так и новые возможности. В качестве администратора вы выбираете, какую версию OME вы хотите использовать при создании правил потока почты.

- Чтобы указать устаревшую версию OME, Exchange действие правила потока почты **Применить предыдущую версию OME.**

- Чтобы указать новые возможности, используйте Exchange правила потока почты **Apply шифрование сообщений Office 365 и защиты прав.**

Пользователи могут вручную отправлять почту, зашифрованную с помощью новых возможностей Outlook, Outlook для Mac и Outlook в Интернете.

## <a name="migrate-from-legacy-ome-to-the-new-capabilities"></a>Перенос из устаревшей OME в новые возможности

Несмотря на то, что обе версии OME могут сосуществовать, мы настоятельно рекомендуем изменить старые правила потока почты, которые используют действие правила Применить предыдущую версию **OME** для использования новых возможностей. Обнови эти правила, чтобы использовать действие правила потока почты **Apply шифрование сообщений Office 365 и защиты прав.** Инструкции см. в тексте Определение правил потока почты для шифрования сообщений электронной почты [в Office 365.](define-mail-flow-rules-to-encrypt-email.md)

## <a name="get-started-with-ome"></a>Начало работы с OME

Как правило, новые возможности OME автоматически включены для организации. Дополнительные сведения о новых возможностях OME в организации см. в [шифрование сообщений Office 365.](set-up-new-message-encryption-capabilities.md)

Устаревшая версия OME автоматически включена для организации, если вы включили Azure Information Protection. В прошлом устаревший OME работал, даже если защита информации Azure не включена. Теперь это не так.

Чтобы начать использовать устаревший OME, если вы включили Azure Information Protection, настройте правила потока почты, использующие действие правила Применить предыдущую **версию OME.** Инструкции см. в [тексте Определение правил потока почты для шифрования сообщений электронной почты.](define-mail-flow-rules-to-encrypt-email.md)
