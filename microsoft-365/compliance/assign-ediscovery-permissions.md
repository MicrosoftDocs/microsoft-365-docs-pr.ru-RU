# <a name="assign-ediscovery-permissions-in-the-microsoft-365-compliance-center"></a>Назначение разрешений на открытие электронных данных в центре Microsoft 365 соответствия требованиям

Если вы хотите, чтобы люди использовали [какие-либо](ediscovery.md) средства, связанные с электронным открытием, в центре Microsoft 365 соответствия требованиям, необходимо назначить им соответствующие разрешения. Самый простой способ сделать это — добавить человеку соответствующую группу ролей на странице **Permissions** в центре соответствия требованиям. В этом разделе описываются разрешения, необходимые для выполнения задач по обнаружению электронных обнаружений.
  
Основная группа ролей, связанных с электронным открытием, в Microsoft 365 центра соответствия требованиям называется **диспетчером eDiscovery.** В ней есть две подгруппы.
  
- **Менеджеры** по обнаружению электронных обнаружений — менеджер по обнаружению электронных обнаружений может использовать средства поиска для поиска содержимого в организации и выполнять различные действия, связанные с поиском, такие как предварительный просмотр и экспорт результатов поиска. Участники также могут создавать и управлять случаями в core eDiscovery и Advanced eDiscovery, добавлять и удалять членов в случае, создавать личные дела, запускать поиски, связанные с делом, и получать доступ к данным дела. Менеджеры по обнаружению электронных данных могут получать доступ только к создаваемым ими делам. Они не могут получать доступ к делам, созданным другими менеджерами по обнаружению электронных данных.
  
- **Администраторы обнаружения электронных данных**. Администратор обнаружения электронных данных является участником группы ролей "Менеджер по обнаружению электронных данных" и может выполнять те же задачи, связанные с поиском контента и управлением делами, что и менеджер по обнаружению электронных данных. Кроме того, администратор обнаружения электронных данных может выполнять следующие действия.
  
  - Доступ ко всем случаям, перечисленным на основных страницах **Advanced eDiscovery** в центре Microsoft 365 соответствия требованиям. 

  - Получать доступ к данным дела в Advanced eDiscovery для любого дела в организации.
  
  - Управлять всеми делами обнаружения электронных данных после добавления себя в качестве участника дела.
  
  Причины, по которым в организации могут потребоваться администраторы по обнаружению электронных данных, см. [в дополнительных сведениях.](#more-information)

> [!NOTE]
> Чтобы проанализировать данные пользователя с помощью Advanced eDiscovery, пользователю (хранителям данных) должна быть назначена Office 365 E5 или Microsoft 365 E5 лицензии. Кроме того, пользователям с Office 365 E1 или лицензией Office 365 или Microsoft 365 E3 может быть назначена лицензия Соответствие требованиям Microsoft 365 E5 или Microsoft 365 и надстройка аудита. Администраторы, сотрудники по соблюдению требований или юристы, которым назначены дела в качестве членов и Advanced eDiscovery для сбора, просмотра и анализа данных, не требуется лицензия E5. Дополнительные сведения о лицензировании Advanced eDiscovery см. в [Advanced eDiscovery.](overview-ediscovery-20.md#subscriptions-and-licensing)
  
## <a name="before-you-assign-permissions"></a>Перед назначением разрешений

- Вам необходимо быть членом группы ролей управления организацией или назначать роль управления ролью для назначения разрешений на открытие электронных данных в центре Microsoft 365 соответствия требованиям.

- С помощью команды [Add-RoleGroupMember](/powershell/module/exchange/Add-RoleGroupMember) в Центре обеспечения безопасности & PowerShell можно добавить группу безопасности с включенной почтой в качестве члена подгруппы менеджеров по обнаружению электронных данных в группе ролей диспетчера электронных данных. Однако вы не можете добавить группу безопасности с включенной почтой в подгруппу администраторов по обнаружению электронных данных. Дополнительные сведения [см. в дополнительных сведениях.](#more-information) 
  
## <a name="assign-ediscovery-permissions"></a>Назначение разрешений на обнаружение электронных данных

1. Перейдите к учетной записи и войдите в нее с помощью учетной <https://compliance.microsoft.com> записи, которая может назначать разрешения.
  
2. В левой области центра соответствия Microsoft 365 выберите **Разрешения**.

3. На странице **Permissions & Roles** в **центре** соответствия требованиям щелкните **Роли**.

4. На странице **Роли центра соответствия требованиям** выберите диспетчер по **обнаружению электронных данных.**
  
5. На странице **flyout manager eDiscovery сделайте** одно из следующих разрешений на основе разрешений на открытие электронных данных, которые необходимо назначить.
  
    **Чтобы сделать пользователя менеджером по обнаружению электронных данных:** Рядом с **менеджером по обнаружению электронных обнаружений** выберите **Изменить**. На странице **Мастера диспетчера электронных** обнаружений нажмите ![ кнопку Добавить значок ](../media/ITPro-EAC-AddIcon.gif) **Добавить**. Выберите пользователя (или пользователей), который необходимо добавить в качестве менеджера по обнаружению электронных данных, а затем **добавьте**. Когда вы закончите добавлять пользователей, выберите **Готово**. Затем на странице **Мастер диспетчера** редактирования выберите диспетчер по обнаружению электронных обнаружений выберите **Сохранить,** чтобы сохранить изменения в членстве диспетчера электронных обнаружений.
  
    **Чтобы сделать пользователя администратором по обнаружению электронных данных:** Рядом с **администратором eDiscovery** выберите **Изменить**. На странице **Выберите администратора по обнаружению электронных обнаружений** нажмите ![ кнопку Добавить ](../media/ITPro-EAC-AddIcon.gif) **значок Добавить**. Выберите пользователя (или пользователей), который необходимо добавить в качестве администратора по обнаружению электронных **данных,** а затем  **добавить**. Когда вы закончите добавлять пользователей, выберите **Готово**. Затем на странице **Мастер администратора** редактирования Выберите администратора электронных открытий выберите **Сохранить,** чтобы сохранить изменения в членстве администратора по обнаружению электронных обнаружений.
  
> [!NOTE]
> Вы также можете использовать **кодлет Add-eDiscoveryCaseAdmin,** чтобы сделать пользователя администратором по обнаружению электронных данных. Однако перед использованием этого комлета пользователю должна быть назначена роль управления случаями, чтобы сделать его администратором по обнаружению электронных данных. Дополнительные сведения см. [в добавлении-eDiscoveryCaseAdmin](/powershell/module/exchange/add-ediscoverycaseadmin). 
  
На странице **Permissions** в центре Microsoft 365 соответствия требованиям можно также назначить пользователям разрешения, связанные с электронным открытием, добавив их в группы ролей администратора соответствия требованиям, управления организацией и рецензента. Описание ролей RBAC, связанных с электронным открытием, которые назначены каждой из этих групп ролей, см. в описании ролей [RBAC, связанных с электронным открытием.](#rbac-roles-related-to-ediscovery)

### <a name="case-management"></a>Управление случаем

Эта роль позволяет пользователям создавать, редактировать, удалять и управлять доступом к основным Advanced eDiscovery и Advanced eDiscovery в центре Microsoft 365 соответствия требованиям. Как объяснялось ранее, перед использованием команды **Add-eDiscoveryCaseAdmin** пользователю должна быть назначена роль управления случаями, чтобы сделать его администратором по обнаружению электронных данных.

Дополнительные сведения см. в указанных ниже статьях.

- [Начало работы с Core eDiscovery](get-started-core-ediscovery.md)

- [Начало работы с Advanced eDiscovery](get-started-with-advanced-ediscovery.md)

### <a name="communication"></a>Обмен информацией

Эта роль позволяет пользователям управлять всеми коммуникациями с хранителями, идентифицированными в Advanced eDiscovery случае. Это включает создание уведомлений о удержании, напоминаний о удержании и эскалаций для управления. Кроме того, пользователь может отслеживать подтверждение хранителя уведомлений о удержании и управлять доступом к порталу хранителя, который используется каждым хранителями для отслеживания сообщений о случаях, когда они были идентифицированы как хранители.

Дополнительные сведения см. в [сообщении Work with communications in Advanced eDiscovery.](managing-custodian-communications.md)

### <a name="compliance-search"></a>Поиск соответствия

Эта роль позволяет пользователям запускать средство поиска контента в центре соответствия Microsoft 365 для поиска почтовых ящиков и общедоступных папок, сайтов SharePoint Online, OneDrive для бизнеса сайтов, Skype для бизнеса бесед, Microsoft 365 групп и Microsoft Teams и Yammer групп. Эта роль позволяет пользователю получать оценку результатов поиска и создавать отчеты об экспорте, но для инициировать действия по поиску контента, такие как предварительный просмотр, экспорт или удаление результатов поиска, необходимы другие роли.

Пользователи, которым назначена роль поиска соответствия требованиям, но не роль Preview, могут просмотреть результаты поиска, в котором действие предварительного просмотра было инициировано пользователем, которому назначена роль Preview. Пользователь без роли Preview может просматривать результаты в течение двух недель после создания начального действия предварительного просмотра.

Точно так же пользователи, которым назначена роль Поиска соответствия требованиям, но не роль экспорта, могут скачать результаты поиска, в котором действие экспорта было инициировано пользователем, которому назначена роль Export. Пользователь без роли Экспорт может скачивать результаты поиска в течение двух недель после создания исходного действия экспорта. После этого они не смогут скачать результаты, если кто-то с ролью Экспорта не перезапустит экспорт.

Дополнительные сведения см. в [Office 365.](content-search.md)

### <a name="custodian"></a>Хранитель

Эта роль позволяет пользователям определять и управлять хранителями для Advanced eDiscovery случаев и использовать информацию из Azure Active Directory и других источников для поиска источников данных, связанных с хранителями. Пользователь может связать другие источники данных, такие как почтовые ящики, SharePoint сайты и Teams с хранителями в случае. Кроме того, пользователь может использовать юридические источники данных, связанные с хранителями, для сохранения контента в контексте дела.

Дополнительные сведения см. в [ссылке Работа с хранителями в Advanced eDiscovery.](managing-custodians.md)

### <a name="export"></a>Экспорт

Роль позволяет пользователям экспортировать результаты поиска контента на локальный компьютер. Он также позволяет подготовить результаты поиска для анализа в Advanced eDiscovery.

Дополнительные сведения об экспорте результатов поиска см. в Microsoft 365 центра [соответствия](export-search-results.md)требованиям.

### <a name="hold"></a>Hold

Эта роль позволяет пользователям удерживать контент в почтовых ящиках, общедоступных папках, сайтах, Skype для бизнеса беседах и Microsoft 365 группах. Если содержимое находится на удержании, его владельцы по-прежнему могут изменять или удалять его, но исходный контент будет сохранен до отмены удержания или истечения его срока.

Дополнительные сведения о удерживает см. в.

- [Создание удержания в core eDiscovery](create-ediscovery-holds.md) 

- [Создание удержания в Advanced eDiscovery](add-custodians-to-case.md)

### <a name="preview"></a>Предварительный просмотр

Эта роль позволяет пользователям просматривать список элементов, возвращенных из поиска контента. Пользователи также могут открывать и просматривать каждый элемент из списка, чтобы просмотреть его содержимое.

### <a name="review"></a>Проверка

Эта роль позволяет пользователям получать доступ к наборам обзоров [в Advanced eDiscovery](overview-ediscovery-20.md). Пользователи, которым назначена эта роль, могут видеть и открывать список случаев на странице > **Advanced** в центре Microsoft 365 соответствия требованиям, в которых они являются членами. После того как пользователь получает доступ к Advanced eDiscovery, он может выбрать **наборы обзоров** для доступа к данным кейсов. Эта роль не позволяет пользователю просматривать результаты поиска коллекции, связанного с делом, или выполнять другие задачи по управлению поиском или делом. Пользователи с этой ролью могут получать доступ только к данным в наборе обзоров.

### <a name="rms-decrypt"></a>Расшифровка RMS

Эта роль позволяет пользователям просматривать защищенные правами сообщения электронной почты при просмотре результатов поиска и экспорте расшифрованных сообщений электронной почты, защищенных правами. Эта роль также позволяет пользователям просматривать (и экспортировать) файл, зашифрованный с помощью технологии шифрования [Майкрософт,](encryption.md) когда зашифрованный файл присоединен к сообщению электронной почты, включенному в результаты поиска по обнаружению электронных данных. Кроме того, эта роль позволяет пользователям рассматривать и запрашивать зашифрованные вложения электронной почты, которые добавляются в набор отзывов в Advanced eDiscovery. Дополнительные сведения о расшифровке в eDiscovery см. в Microsoft 365 [средствах eDiscovery.](ediscovery-decryption.md)

### <a name="search-and-purge"></a>Поиск и очистка

Эта роль позволяет пользователям выполнять массовое удаление данных, совпадающих с критериями поиска контента. Дополнительные сведения см. в публикации Поиск и удаление сообщений электронной [почты в организации.](search-for-and-delete-messages-in-your-organization.md)

## <a name="more-information"></a>Дополнительные сведения

- **Зачем создавать администратора, ответственного за обнаружение электронных данных?** Как объяснялось ранее, администратор обнаружения электронных данных входит в группу ролей "Диспетчер eDiscovery" и может просматривать все дела eDiscovery в организации. Эта возможность доступа ко всем делам eDiscovery имеет две важные цели:

  - Если единственный член дела eDiscovery уволится из вашей организации, никто (в том числе члены группы ролей "Управление организацией" или другой участник группы ролей "Диспетчер eDiscovery") не сможет получить доступ к этому делу, так как никто не является его членом. В этой ситуации доступ к данным в деле будет невозможен. Но так как администратор по обнаружению электронных обнаружений может получить доступ ко всем делам об обнаружении электронных обнаружений в организации, он может просмотреть дело и добавить себя или другого руководителя по обнаружению электронных обнаружений в качестве участника дела.

  - Так как администратор по обнаружению электронных обнаружений может просматривать и получать доступ ко всем случаям и Advanced eDiscovery core, он может проверять и контролировать все случаи и связанные поиски соответствия требованиям. Это может помочь предотвратить любое неправомерное использование запросов на проверку соответствия требованиям или случаев поиска электронной информации. Так как администраторы электронных данных могут получать доступ к потенциально конфиденциальным сведениям в результатах поиска соответствия требованиям, необходимо ограничить число пользователей, которые являются администраторами электронных данных.

- **Могу ли я добавить группу в качестве члена группы ролей диспетчера электронных открытий?** Как было объяснено ранее, можно добавить группу безопасности с включенной почтой в качестве члена подгруппы менеджеров по электронным данным в группу ролей диспетчера электронных данных с помощью команды **Add-RoleGroupMember** в центре обеспечения безопасности & Compliance Center PowerShell. Например, можно запустить следующую команду, чтобы добавить группу безопасности с включенной почтой в группу ролей диспетчера электронных данных. 

  ```powershell
  Add-RoleGroupMember "eDiscovery Manager" -Member <name of security group>
  ```

    Exchange группы рассылки и Microsoft 365 группы не поддерживаются. Необходимо использовать группу безопасности с поддержкой почты, которую можно создать в Exchange Online PowerShell с помощью запуска `New-DistributionGroup -Type Security` . Вы также можете создать группу безопасности с включенной почтой (и добавить членов) в центре администрирования Exchange центре администрирования или Microsoft 365 центре администрирования. Это может занять до 60 минут после создания новой безопасности с поддержкой почты, которая будет доступна для добавления в группу ролей диспетчеров электронных данных. 

    Кроме того, как было указано ранее, нельзя сделать группу безопасности с включенной почтой администратором по обнаружению электронных данных с помощью cmdlet **Add-eDiscoveryCaseAdmin** в центре безопасности & Compliance Center PowerShell. Можно добавлять только отдельных пользователей в качестве администраторов по обнаружению электронных данных.

    Вы также не можете добавить группу безопасности с поддержкой почты в качестве участника дела.
