# <a name="insider-risk-management-cases"></a>Варианты по управлению рисками для участников

Это сердце управления рисками для предварительной оценки и позволяет глубоко исследовать проблемы, возникающие в результате использования индикаторов риска, определенных в политиках. Сценарии создаются вручную из оповещений в ситуациях, когда требуется дальнейшие действия для решения проблемы, связанной с соответствием требованиям для пользователя. Для каждого случая применяется один пользователь, и несколько оповещений для пользователя можно добавить в существующее или новое обращение. 

После изучения сведений о случае можно выполнить следующие действия:

- Отправка уведомления пользователю
- устранение неблагоприятных требований
- предоставление общего доступа к обращению с экземпляром ServiceNow или с получателем электронной почты
- эскалация дела для расширенного исследования обнаружения электронных данных

## <a name="cases-dashboard"></a>Панель мониторинга "обращения"

**Панель мониторинга** рисков по управлению рисками позволяет просматривать и реагировать на обращения. Каждый мини-приложение отчета на панели мониторинга отображает сведения за последние 30 дней.

- Количество **активных случаев** : общее количество активных случаев в ходе расследования.
- Количество **обращений за прошлые 30 дней** : общее количество созданных обращений, отсортированных по *активному* и *закрытому* состоянию.
- **Статистика** : среднее время активных обращений, указанных в часах, днях или месяцах.

В очереди обращений отображается список всех активных и закрытых обращений в Организации в дополнение к текущему состоянию следующих атрибутов case:

- **Имя случая** : имя случая, которое определяется при подтверждении оповещения и создании обращения.  
- **Состояние** : состояние обращения — " *активно* " или " *закрыто* ".
- **Пользователь** : пользователь для обращения. Если функция анонимности для имен пользователей включена, отображается анонимная информация.
- **Время открытия** : время, прошедшее с момента открытия обращения.
- **Общее количество оповещений политики** : количество совпадений политик, включенных в это обращение. Это число может увеличиваться, если в обращение добавляются новые оповещения.
- **Последнее обновление** : время, прошедшее с момента добавления примечания к случаю или изменения состояния обращения.
- **Последнее обновление** : имя аналитики управления рисками для участников программы или инвестигатор, которые последним обновили дело.

![Панель мониторинга "варианты управления рисками"](../media/insider-risk-cases-dashboard.png)

Используйте элемент управления **поиском** , чтобы выполнять поиск по именам дел для определенного текста и использовать фильтр вариантов для сортировки обращений по следующим атрибутам:

- Status
- Время открытия, Дата начала и Дата окончания
- Дата последнего обновления, Дата начала и Дата окончания

## <a name="filter-cases"></a>Варианты фильтрации

В зависимости от числа и типа активных политик управления рисками для оценки в Организации, просмотр больших очередей может быть непростой задачей. Использование фильтров Case помогает аналитикам и исследованиям сортировать варианты по нескольким атрибутам. Чтобы отфильтровать оповещения на **панели мониторинга "обращения"** , выберите элемент управления **фильтра** . Вы можете фильтровать обращения по одному или нескольким атрибутам:

- **Состояние** : выберите одно или несколько значений состояния, чтобы отфильтровать список дел. Параметры *активны* и *закрыты* .
- **Временный регистр открыт** : выберите начальную и конечную даты, когда обращение было открыто.
- **Последнее обновление** : выберите начальную и конечную даты, когда обращение было обновлено.

## <a name="investigate-a-case"></a>Исследование дела

Более глубокое исследование оповещений об управлении рисками очень важно для выполнения правильных действий. Варианты, связанные с управлением рисками для оценки риска — это центральное средство управления для подробного изучения истории действий и оповещений о рисках пользователей, а также для изучения контента и сообщений, предоставляемых рискам. Аналитики и изучает риски также используют варианты для централизации анализа отзывов и примечаний, а также для обработки сценариев.

При выборе обращения открываются средства управления делами, а аналитики и изучении позволяют проанализировать сведения об этих случаях.

### <a name="case-overview"></a>Обзор обращения

На вкладке **Обзор обращения** обобщены сведения о действиях оповещений и уровне риска для случая. 

- Мини-приложение **Alerts** отображает соответствие политике для случая, включая состояние оповещения, серьезность риска для оповещений и время обнаружения оповещения. 
- На диаграмме " **Журнал уровня риска** " отображается уровень риска пользователя за последние 30 дней. График позволяет аналитикам и исследованиям быстро видеть тенденцию в общем риске для пользователей с течением времени. 
- Мини-приложение для работы с **содержимым "действие риска** " суммирует типы данных и содержимого, которые содержатся в оповещениях, добавленных в обращение. Этот мини-приложение предоставляет полное представление о всех данных и наборе контента под угрозой в случае.

Область " **сведения о делах** " доступна на всех вкладках управления делами и суммирует сведения о вариантах для аналитиков и обучений по рискам. Он включает в себя следующие области:

- **Имя случая** : имя варианта с автоматически сгенерированным порядковым номером и именем риска, связанным с шаблоном, который первым подтвердил оповещение. 
- **Состояние обращения** : текущее состояние обращения — " *активно* " или " *закрыто* ".
- **Показатель риска пользователя** : текущий рассчитанный уровень риска для пользователя в случае. Этот показатель вычисляется каждые 24 часа и использует показатели риска оповещений из всех активных оповещений, связанных с пользователем.
- **Подтверждено оповещений** : список оповещений для пользователя, подтвержденный для обращения.
- **Связанное содержимое** : список контента, отсортированного по источникам контента и типам. Например, для контента оповещений в SharePoint Online могут отображаться имена папок или файлов, связанные с действием риска для оповещений в случае.

### <a name="alerts"></a>Оповещения

На вкладке **оповещения** вы общаетесь с текущими оповещениями, включенными в обращение. В существующее обращение можно добавить новые оповещения, которые будут добавлены в очередь **оповещений** по мере их назначения. В очередь перечисляются следующие атрибуты оповещения:

- Status
- Severity
- Время обнаружения

Выберите оповещение из очереди, чтобы отобразить страницу **сведений о предупреждении** .

Используйте элемент управления поиском для поиска по именам оповещений для определенного текста и используйте фильтр оповещений для сортировки обращений по следующим атрибутам:

- Status
- Severity
- Время обнаружения, Дата начала и Дата окончания

Используйте элемент управления фильтром для фильтрации предупреждений по нескольким атрибутам, в том числе:

- **Состояние** : выберите одно или несколько значений состояния, чтобы отфильтровать список оповещений. Параметры *подтверждены* *, отменяются,* *требуют проверки* и *разрешаются* .
- **Severity** : выберите один или несколько уровней серьезности угроз оповещений, чтобы отфильтровать список оповещений. Возможные значения: *Высокая* , *Средняя* и *Минимальная* .
- **Время обнаружения** : выберите начальную и конечную даты создания оповещения.
- **Политика** : выберите одну или несколько политик, чтобы отфильтровать оповещения, созданные выбранными политиками.

### <a name="user-activity"></a>Действия пользователей

Вкладка **активности пользователей** — это один из самых мощных средств для анализа внутреннего риска и исследования в случае решения по управлению рисками для оценки. Эта вкладка структурирована, чтобы обеспечить быструю проверку случая, включая историю временной шкалы для всех оповещений, сведения обо всех оповещениях, о текущем показателе риска для пользователя, а также элементы управления для эффективного выполнения действий, которые могут содержать риски в случае.

![Действия пользователя по управлению рисками для участников](../media/insider-risk-user-activities.png)

1. **Фильтры даты и времени окна** : по умолчанию последние шесть месяцев оповещений, подтвержденных в случае, отображаются на диаграмме активности пользователей. Вы можете легко отфильтровать представление диаграммы с помощью ползунков на обоих концах окна диаграммы или путем определения определенных начальных и конечных дат в элементе управления фильтра диаграммы.
2. **Действия и сведения об оповещениях о рисках** : визуальные действия визуально отображаются в виде цветных пузырьков на диаграмме активности пользователей. Для разных категорий риска создаются пузырьковые подзадачи, размер которых пропорциональен числу действий риска для категории. Выберите подпузырька, чтобы отобразить сведения о каждом мероприятии риска. Ниже приведены дополнительные сведения.
    - **Дата** действия риска.
    - **Категория "действие риска** ". Например, *сообщения электронной почты с вложениями, отправленными за* прев организации или *файлов, загруженных из SharePoint Online* .
    - **Показатель риска** для оповещения. Этот показатель является числовым показателем уровня серьезности риска для предупреждения.
    - Количество событий, связанных с предупреждением. Также доступны ссылки на каждый файл или электронную почту, связанную с действием риска.
3. **Условные обозначения действий риска** : в нижней части диаграммы активности пользователей, условные обозначения с цветовым кодированием помогают быстро определить категорию риска для каждого оповещения.
4. **Действие риска чронологи** : перечислены все оповещения о рисках, связанные с обращением, в том числе все сведения, доступные в соответствующем всплывающем подокне оповещений.
5. **Действия при обращении** : параметры для разрешения регистра находятся на панели инструментов действие в действии. Вы можете разрешить обращение, отправить пользователю уведомление по электронной почте или эскалировать его для данных или расследования пользователей.

### <a name="activity-explorer-preview"></a>Обозреватель действий (Предварительная версия)

>[!IMPORTANT]
>Вкладка Обозреватель активности доступна в области управления делами для пользователей, которые запускают события после того, как эта функция доступна в Организации.

Вкладка **Обозреватель действий** позволяет аналитикам и обучениям просматривать сведения о действиях, связанных с оповещениями о рисках. Например, в рамках действий управления делами для исследования и аналитик может потребоваться просмотреть все действия, связанные с обращением, чтобы получить дополнительные сведения. С помощью **обозревателя действий** рецензенты могут быстро просмотреть временную шкалу обнаруженных рискованных действий, а также определить и отфильтровать все действия, связанные с оповещениями.

Дополнительные сведения об обозревателе действий можно найти в статье [Alert Alert Management Alert Alerts](insider-risk-management-alerts.md#activity-explorer-preview) .

### <a name="content-explorer"></a>Обозреватель контента

Вкладка " **Обозреватель содержимого** " позволяет аналитикам и исследованиям риска просматривать копии всех отдельных файлов и сообщений электронной почты, связанных с оповещениями о рисках. Например, если вы создаете оповещение, когда пользователь загружает сотни файлов из SharePoint Online и действие запускает оповещение политики, все загруженные файлы для оповещения фиксируются и копируются в обращение по управлению рисками для участников из исходных источников хранения.

Обозреватель контента — это мощное средство для базовых и расширенных функций поиска и фильтрации. Для получения дополнительных сведений об использовании обозревателя содержимого, ознакомьтесь со статьей [Центр справки по управлению рисками](insider-risk-management-content-explorer.md).

### <a name="case-notes"></a>Примечания к случаю

На вкладке " **заметки к случаю** " в случае, когда аналитики и исследования рисков совместно используют комментарии, Отзывы и ценные сведения о работе в случае. Заметки — это постоянные дополнения к обращению, которые невозможно изменить или удалить после сохранения заметки. Когда обращение создается из оповещения, комментарии, введенные в диалоговом окне " **Подтверждение оповещения" и "Создание примера риска** ", автоматически добавляются в качестве примечания к случаю.

Панель мониторинга заметок о вариантах отображает заметки у пользователя, который создал заметку, и время, прошедшее с момента сохранения заметки. Чтобы выполнить поиск определенного ключевого слова в текстовом поле Note Note (Примечание), используйте кнопку **поиска** на панели мониторинга "обращение" и введите конкретное ключевое слово.

Чтобы добавить заметку к случаю, выполните следующие действия:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **обращения** .
2. Выберите вариант, а затем перейдите на вкладку **заметки к случаю** .
3. Выберите **Добавить заметку к случаю** .
4. В диалоговом окне **Добавление заметки к случаю** введите заметку для обращения. Нажмите кнопку **сохранить** , чтобы добавить заметку к регистру, или кнопку **Отмена** закрытия без сохранения заметки в случае.

### <a name="contributors"></a>Авторы

На вкладке **Участники** в случае, когда аналитики и исследования риска могут добавлять других проверяющих в дело. По умолчанию все пользователи, которым назначены **аналитики управления рисками** и **оценки управления** рисками для оценки безопасности, перечислены как участники для каждого активного и закрытого случая.

Все варианты управления рисками для предварительной оценки должны управляться с помощью соответствующих средств контроля доступа на месте, чтобы обеспечить конфиденциальность и целостность расследования. Для поддержки управления доступом для обращений пользователям назначается один из двух типов доступа к делам:

- **Постоянный доступ** : постоянный доступ автоматически предоставляется пользователям с **аналитиками управления рисками и экспертами** по **управлению рисками для оценки** , когда обращение создается из оповещения. Постоянный доступ обеспечивает полный контроль за время жизни этого случая и предоставляет возможность добавлять других участников дела.
- **Временный доступ** : временный доступ предоставляется только пользователям, имеющим постоянный доступ к этому случаю. Как правило, этот уровень доступа предоставляется пользователю, который должен добавлять примечания к обращению. Участники с временным доступом имеют все элементы управления управления делами, кроме:
    - Разрешение на подтверждение или отклонение оповещений
    - Разрешение на изменение участников для обращений
    - Разрешение на просмотр файлов и сообщений в обозревателе контента

Чтобы добавить участника в дело, выполните следующие действия:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **обращения** .
2. Выберите вариант, а затем перейдите на вкладку **Участники** .
3. Нажмите кнопку **Добавить участника** .
4. В диалоговом окне **Добавление участника** введите имя пользователя, которого нужно добавить, а затем выберите пользователя из списка предлагаемых пользователей. Этот список создается из Azure Active Directory подписки клиента.
5. В диалоговом окне **Добавление участника** выберите уровень доступа для участника. Вы можете выбрать **постоянное** или **временное** .
6. Нажмите кнопку **Добавить** , чтобы добавить пользователя в качестве участника, или кнопку **Отмена** , чтобы закрыть диалоговое окно, не добавляя пользователя в качестве участника.

## <a name="case-actions"></a>Действия при обращении

Аналитики и исследования рисков могут выполнять действия в одном из нескольких способов, в зависимости от степени серьезности обращения, истории риска пользователя и рекомендаций по риску в Организации. В некоторых ситуациях вам может потребоваться эскалировать обращение к пользователю или анализу данных для совместной работы с другими областями Организации, а также для углубленного изучения действий по риску. Управление рисками для предварительной оценки тесно интегрировано с другими решениями Microsoft 365 на соответствие требованиям, чтобы обеспечить Управление разрешением.

### <a name="send-email-notice"></a>Уведомление о отправке сообщения электронной почты

В большинстве случаев действия пользователей, создающие оповещения о рисках для участников программы оценки, являются случайными или случайными. Отправка уведомления с напоминанием пользователю с помощью электронной почты является эффективным методом для документирования обзора и действия, а также способов напоминания пользователей о корпоративных политиках или для их указания на актуальное обучение. Уведомления создаются на основе [шаблонов примечаний, которые вы создаете](insider-risk-management-notices.md) для вашей инфраструктуры управления рисками для оценки.

Важно помнить о том, что отправка уведомления по электронной почте пользователю * *_не_* разрешается как _Closed *. В некоторых случаях может потребоваться оставить обращение открытым, отправив пользователю уведомление о необходимости поиска дополнительных действий, не открывая новое обращение. Если вы хотите устранить ситуацию после отправки уведомления, необходимо выбрать **вариант разрешения** в качестве дальнейших действий после отправки уведомления.

Чтобы отправить уведомление пользователю, назначенному для обращения, выполните следующие действия:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **обращения** .
2. Выберите вариант, а затем нажмите кнопку **Отправить уведомление по электронной почте** на панели инструментов действие на обращение.
3. В диалоговом окне **Отправка уведомления по электронной почте** выберите элемент управления раскрывающееся меню **Выбор шаблона уведомления** , чтобы выбрать шаблон уведомления для уведомления. Этот вариант предварительно заполняет другие поля в уведомлении.
4. Изучите поля примечаний и обновите их соответствующим образом. Введенные здесь значения будут переопределять значения шаблона.
5. Нажмите кнопку **Отправить** , чтобы отправить уведомление пользователю, или кнопку **Отмена** , чтобы закрыть диалоговое окно без отправки уведомления пользователю. Все отправленные уведомления добавляются в очередь заметок о делах на панели мониторинга **заметок о вариантах** .

### <a name="escalate-for-investigation"></a>Эскалация для исследования

Следует эскалировать в случаях, когда требуется дополнительная юридическая информация о риске для пользователя. Это эскалация открывает новое расширенное дело обнаружения электронных данных в организации Microsoft 365. Расширенное обнаружение электронных данных предоставляет сквозной рабочий процесс для сохранения, сбора, проверки, анализа и экспорта содержимого, которое отвечает за внутреннее и внешнее юридическое расследование Организации. Кроме того, она позволяет юридическим командам управлять всем рабочим процессом уведомления по юридическим удержаниям, чтобы общаться с custodians, вовлеченным в дело. Назначение проверяющего в качестве хранитель в расширенном варианте обнаружения электронных данных, созданном на основе случая управления рисками, поможет юридическим командам предпринять соответствующие действия и управлять сохранением контента. Дополнительные сведения о дополнительных возможностях обнаружения электронных данных приведены [в статье обзор расширенного обнаружения электронных данных в Microsoft 365](overview-ediscovery-20.md).

Чтобы эскалировать обращение к расследованию пользователя:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **обращения** .
2. Выберите вариант, а затем нажмите кнопку **эскалация for расследования** на панели инструментов действие на обращение.
3. В диалоговом окне **укрупнение для исследования** введите имя нового расследования пользователя. При необходимости введите примечания к этому случаю и выберите **эскалировать** .
4. Изучите поля примечаний и обновите их соответствующим образом. Введенные здесь значения будут переопределять значения шаблона.
5. Нажмите кнопку **подтвердить** , чтобы создать сценарий расследования пользователей, или кнопку **Отмена** , чтобы закрыть диалоговое окно без создания нового случая расследования пользователя.

После того как обращение по управлению рисками эксперта эскалируется к новому случаю расследования пользователей, вы можете просмотреть новое обращение в области **eDiscovery**  >  **Advanced** eDiscovery в центре соответствия требованиям Microsoft 365.

### <a name="run-automated-tasks-with-power-automate-flows-for-the-case"></a>Запуск автоматизированных задач с автоматизированными потоками управления для обращения

С помощью рекомендуемых автоматизированных потоков, средства анализа и аналитики риска могут быстро выполнить следующие действия:

- Запрос информации из отдела кадров или бизнеса о пользователе в случае с риском для оценки
- Уведомить менеджера при наличии у пользователя оповещения о риске для оценки
- Добавление напоминания календаря к исполнению с учетом случая оценки риска

Для запуска, управления или создания автоматизированных потоков для случая управления рисками при предварительной выдаче:

1. На панели инструментов действия с действием выберите элемент **Автоматизация** . 
2. Выберите поток автоматизации Power для запуска, а затем выберите **поток выполнения** . 
3. После завершения процесса нажмите кнопку Done ( **Готово** ).

Дополнительные сведения о потоках автоматизации энергопотребления для системы управления рисками можно найти в статье [Начало работы с параметрами управления рисками](insider-risk-management-settings.md#power-automate-flows-preview)для участников программы.

### <a name="view-or-create-a-microsoft-teams-team-for-the-case"></a>Просмотр или создание группы Microsoft Teams для обращения

Если в настройках включена интеграция Microsoft Teams для управления рисками, группа Microsoft Teams создается автоматически при каждом подтверждении оповещения и создании обращения. С помощью расследований и аналитик риска можно быстро открыть Microsoft Teams и перейти непосредственно к группе, выбрав пункт **Просмотр группы Microsoft** Teams на панели инструментов действие на обращение.

Для обращений, открытых перед активацией Microsoft Team Integration, специалисты по вопросам и аналитики могут создать новую группу Microsoft Teams для случая, выбрав **создать группу Microsoft** Teams на панели инструментов действие на обращение.

При разрешении дела связанная команда Майкрософт будет автоматически архивироваться (скрыта и включена только для чтения).

Чтобы узнать больше о Microsoft Teams для управления рисками, ознакомьтесь с разделом [Начало работы с параметрами управления рисками для оценки](insider-risk-management-settings.md#microsoft-teams-preview).

### <a name="share-the-case"></a>Общий доступ к обращению

Совместное использование случая управления рисками для исследования позволяет экспертам и аналитикам легко совместно работать с другими заинтересованными лицами в Организации. Вы можете быстро поделиться ссылкой на обращение по управлению рисками для участников с внешними заинтересованными сторонами из области управления делами. Чтобы получить доступ к обращению по программе "Предварительная оценка" из связи, заинтересованные лица должны быть включены в любую группу ролей "Управление рисками".

Доступны следующие варианты общего доступа:

- **ServiceNow** : после настройки соединителя Microsoft 365 ServiceNow для вашей организации Microsoft 365 вы можете легко поделиться ссылкой на обращение, открыть инцидент или запросить изменение в Организации ServiceNow. Чтобы поделиться делами с ServiceNow, выберите **общий доступ к**  >  **servicenow** из действия дела. Интеграция с помощью функции "Управление рисками" в ServiceNow включает следующие сведения о сценариях и действиях:

    - **Тип задачи** : тип задачи для новой задачи ServiceNow — *инцидент* или *запрос на изменение* .
    - **Приоритет** : приоритет новой задачи ServiceNow: *планирование* , *небольшое* , *умеренное* , *высокое* или *критическое* .

### <a name="resolve-the-case"></a>Разрешение обращения

После того, как аналитики и расследования по рискам прошли проверку и расследования, можно выполнить разрешение на выполнение всех оповещений, включенных в данный момент. Разрешение обращения добавляет классификацию разрешения, изменяет состояние обращения на *Closed* , а причины действий по разрешению автоматически добавляются в очередь заметок о делах на панели мониторинга **заметок** . Варианты разрешаются как один из следующих вариантов:

- **Мягкое** : классификация случаев, когда оповещения о соответствии политик оцениваются как низкие риски, несерьезные или ложные срабатывания.
- **Подтвержденное нарушение политики** : классификация случаев, когда оповещения о соответствии политик оцениваются как рискованные, серьезные или могут быть вызваны злонамеренным намерением.

Чтобы устранить проблему, выполните указанные ниже действия.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **обращения** .
2. Выберите вариант, а затем нажмите кнопку **Разрешить обращение** на панели инструментов действие на обращение.
3. В диалоговом окне **Разрешить обращение** выберите элемент элемент **управления раскрывающееся меню разрешение,** чтобы выбрать классификацию разрешения для этого случая. Эти параметры являются **неблагоприятными** или **подтвержденными нарушением политики** .
4. В диалоговом окне **Разрешение обращения** введите причины классификации разрешения в текстовом поле **предпринимаемое действие** .
5. Нажмите кнопку **Разрешить** , чтобы закрыть это обращение, или кнопку **Отмена** , чтобы закрыть диалоговое окно без разрешения обращения.
