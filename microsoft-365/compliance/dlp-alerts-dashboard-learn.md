# <a name="learn-about-the-data-loss-prevention-alerts-dashboard"></a>Сведения о панели оповещений о предотвращении потери данных

Если критерии в политике предотвращения потери данных совпадают с действиями, которые пользователь принимает на конфиденциальный элемент, политика может создать оповещение. Это может привести к высокому объему оповещений. Оповещения DLP собираются на панели оповещений. Панель оповещений предоставляет одно место для глубокого исследования всех сведений о совпадении политик.  

<!-- [Microsoft 365 compliance center](https://compliance.microsoft.com/)-->

## <a name="workloads"></a>Рабочие нагрузки

Панель [мониторинга управления оповещениями DLP](https://compliance.microsoft.com/datalossprevention?viewid=dlpalerts)в центре соответствия требованиям [Microsoft 365](https://compliance.microsoft.com/)отображает оповещения для политик DLP по этим рабочим нагрузкам:

- Exchange
- SharePoint;
- OneDrive;
- Teams
- устройства с Windows 10 

> [!TIP]
> Клиенты, [](endpoint-dlp-learn-about.md) которые используют DLP конечных точек, которые имеют право на получение DLP команд, увидят оповещения о политике [DLP](dlp-microsoft-teams.md) конечной точки и оповещения о политике команд DLP в панели управления оповещениями DLP.

## <a name="single-alert-and-aggregate-alert"></a>Единое оповещение и сводное оповещение

В политиках DLP можно настроить два типа оповещений.

Оповещения **об** одном событии обычно используются в политиках, отслеживают высокочувствительные события, происходящие в низком объеме, например одно сообщение электронной почты с 10 или более номерами кредитных карт клиентов, отправленных за пределами организации.

**Оповещения об агрегированных** событиях обычно используются в политиках, отслеживая события, происходящие в более высоком объеме в течение определенного периода времени. Например, агрегированное оповещение может быть вызвано, когда 10 отдельных сообщений электронной почты с одним номером кредитной карты клиента отправляются за пределы вашей организации в течение 48 часов.
