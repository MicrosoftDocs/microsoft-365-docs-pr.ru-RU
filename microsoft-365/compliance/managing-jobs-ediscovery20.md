# <a name="manage-jobs-in-advanced-ediscovery"></a>Управление заданиями в Advanced eDiscovery

Вот список заданий (которые обычно являются длительными процессами), которые отслеживаются на вкладке **"Задания"** дела в Advanced eDiscovery. Эти задания запускаются действиями пользователей при использовании дел и управлении этими случаями.

| Тип задания            | Описание     |
| :----------------- | :----------     |
|Добавление данных в набор для проверки | Пользователь добавляет результаты поиска в набор для проверки. Это задание состоит из двух подустанавлий: </br>• **GatheringItems** — создается список элементов, которые соответствуют поисковому запросу (и источнику данных Microsoft 365, в который они находятся). </br>• **Ingestion & Indexing —** элементы, которые соответствуют поисковому запросу, копируется в хранилище Azure (в процессе, называемом использованием), а затем эти элементы в хранилище Azure переиндексируется. Этот новый индекс используется при запросе и анализе элементов в наборе данных. </br></br>Дополнительные сведения см. в [подстройки "Добавление результатов поиска в набор для проверки".](add-data-to-review-set.md) |
|Добавление данных в другой набор для проверки | Пользователь добавляет документы из одного набора для проверки в другой набор для проверки в том же случае. Дополнительные сведения см. в [подстройки](add-data-to-review-set-from-another-review-set.md)"Добавление данных в набор для проверки" из другого набора.|
|Добавление данных, не вносяых в Microsoft 365, в набор для проверки | Пользователь загружает данные, не относя такие как Microsoft 365, в набор для проверки. Данные также индексироваться во время этого процесса. Например, файлы с локального файлового сервера или клиентского компьютера загружаются в набор для проверки. Дополнительные сведения см. в подстройки "Загрузка данных, не относяхся к [Microsoft 365".](load-non-office-365-data-into-a-review-set.md)| 
|Добавление исправленных данных в набор для проверки | Данные с ошибками обработки устраняются и загружаются обратно в набор для проверки. Дополнительные сведения см. в указанных ниже статьях.</br>• [Исправление ошибок при обработке данных](error-remediation-when-processing-data-in-advanced-ediscovery.md)</br>• [Исправление ошибок одного элемента](single-item-error-remediation.md)| 
|Сравнение наборов нагрузки | Пользователь просматривает различия между различными наборами нагрузки в наборе для проверки. Набор нагрузки — это экземпляр добавления данных в набор для проверки. Например, если добавить результаты двух разных поисковых запросов в один и тот же набор для проверки, каждый из них будет представлять набор нагрузки. |
|Реконструкция беседы|Когда пользователь добавляет результаты поиска в набор для проверки бесед, мгновенные сообщения (также называемые цепочками *бесед)* в таких службах, как Microsoft Teams, воссоздаются в PDF-файле. Это задание также запускается, когда пользователь нажимает кнопку **Action > Create conversation PDFs** in a review set. Дополнительные сведения [см. в обзоре бесед в Advanced eDiscovery.](conversation-review-sets.md)
|Преобразование отредактируемых документов в ФОРМАТ PDF|После того как пользователь примечает документ в наборе для проверки и редактирует его часть, он может преобразовать отредактируемый документ в PDF-файл. Это гарантирует, что отредактируемая часть не будет видна, если документ экспортируется для презентации. Дополнительные сведения [см. в документе "Просмотр документов в наборе для проверки".](annotating-and-redacting-documents.md) |
|Оценка результатов поиска | После создания и выполнения пользователем нового поиска (или повторного выполнения существующего поиска) средство поиска выполняет поиск в индексе элементов, которые соответствуют поисковому запросу, и подготавливает оценку, включающую число и общий размер всех элементов поиска, а также количество источников данных, в которые был включен поиск.  Дополнительные сведения [см. в теме "Сбор данных для дела".](collecting-data-for-ediscovery.md) | 
|Подготовка данных для экспорта | Пользователь экспортирует документы из набора для проверки. После завершения процесса экспорта они могут скачать экспортные данные на локальный компьютер. Дополнительные сведения см. в [экспорте данных дела.](exporting-data-ediscover20.md) | 
|Подготовка к разрешению ошибок |Когда пользователь выбирает файл и создает исправление ошибки в представлении ошибки на вкладке обработки дела, первым шагом процесса является отправка файла с ошибкой обработки в хранилище Azure в Microsoft Cloud.  Это задание отслеживает ход процесса отправки. Дополнительные сведения о рабочий процесс устранения ошибок см. в исправлении [ошибок при обработке данных.](error-remediation-when-processing-data-in-advanced-ediscovery.md) | 
|Подготовка предварительного просмотра поиска | После создания и выполнения пользователем нового поиска (или повторного выполнения существующего поиска) средство поиска подготавливает пример подмножество элементов (которые соответствуют поисковому запросу), которые можно просмотреть. Предварительный просмотр результатов поиска помогает определить эффективность поиска.  Дополнительные сведения [см. в теме "Сбор данных для дела".](collecting-data-for-ediscovery.md#view-search-results-and-statistics) | 
|Переиндексация данных хранителя | При добавлении хранителя в дело все частично индексные элементы в выбранных источниках данных хранителя переиндексются процессом, называемым *"Расширенным индексасом".* Это задание также запускается  при нажатии кнопки "Обновить индекс" на вкладке "Обработка" дела и при обновлении индекса для определенного хранителя на странице "Свойства хранителя".  Дополнительные сведения [см. в дополнительных сведениях об индексации данных хранителя.](indexing-custodian-data.md)
|Запуск аналитики | Пользователь анализирует данные в наборе для проверки, запуская средства аналитики Advanced eDiscovery, такие как обнаружение практически дубликатов, анализ потоков электронной почты и анализ тем. Дополнительные сведения см. [в анализе данных в наборе для проверки.](analyzing-data-in-review-set.md) | 
|Добавление тегов к документам | Это задание запускается, когда пользователь нажимает кнопку **"Начать** задание маркировки" на панели тегов при просмотре документов в наборе для проверки.  Пользователь может запустить это задание после маркировки документов в наборе для проверки и их пакетного выбора на панели документов представления. Дополнительные сведения [см. в документе Tag в наборе для проверки.](tagging-documents.md) | 
|||

## <a name="job-status"></a>Состояние задания

В следующей таблице описываются различные состояния состояния заданий.

| Состояние           | Описание     |
| :----------------- | :----------     |
| Submitted | Было создано новое задание.  Дата и время отправки задания отображаются в  столбце "Создано" на **вкладке "Задания".** |
| Сбой отправки | Сбой отправки задания.  Следует попытаться повторно перезапуска действия, которое инициировало задание. |
| В процессе выполнения | Задание находится в процессе выполнения, вы можете отслеживать ход выполнения задания на вкладке **"Задания".** |
| Успешно | Задание успешно выполнено. Дата и время завершения задания отображаются в столбце **"Завершено"** на **вкладке "Задания".** |
| Частично успешно | Задание было частично успешно. Это состояние обычно возвращается, когда задание не находит частично индексировать данные (также называемые неиндексными данными) в некоторых источниках данных хранителя.  |
| Не выполнено | Сбой задания.  Следует попытаться повторно перезапуска действия, которое инициировало задание. Если задание не будет работать во второй раз, рекомендуется обратиться в службу поддержки Майкрософт и предоставить информацию о поддержке. |
