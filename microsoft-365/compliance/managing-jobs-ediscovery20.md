# <a name="manage-jobs-in-advanced-ediscovery"></a>Управление заданиями в области расширенных электронных обнаружений

Вот список заданий (которые обычно являются длительными процессами), отслеживаются на вкладке **Jobs** в случае с расширенным электронным открытием. Эти задания запускаются действиями пользователя при использовании и управлении делами.

| Тип задания            | Описание     |
| :----------------- | :----------     |
|Добавление данных в набор отзывов | Пользователь добавляет результаты поиска в набор отзывов. Это задание состоит из двух рабочих мест подгруппы: </br>• **GatheringItems** — создается список элементов, которые соответствуют запросу поиска (и источнику данных Microsoft 365, в который они находятся). </br>• Индексация & *ingestion*— элементы, совпадающие с запросом поиска, копируется в расположение Azure Storage (в процессе, называемом вложением), а затем эти элементы в расположении Azure Storage переиндексируется.  Этот новый индекс используется при запросе и анализе элементов в наборе данных. </br></br>Дополнительные сведения см. [в добавлении результатов поиска в набор отзывов.](add-data-to-review-set.md) |
|Добавление данных в другой набор обзоров | Пользователь добавляет документы из одного набора отзывов в другой набор отзывов в том же случае. Дополнительные сведения см. в примере Добавление данных [в набор отзывов из другого набора обзоров.](add-data-to-review-set-from-another-review-set.md)|
|Добавление данных, не в microsoft 365, в набор отзывов | Пользователь загружает данные, не вложенные в Microsoft 365, в набор отзывов. Данные также индексироваться в ходе этого процесса. Например, файлы с локального файлового сервера или клиентского компьютера загружаются в набор отзывов. Дополнительные сведения см. в [обзоре Load non-Microsoft 365 data.](load-non-office-365-data-into-a-review-set.md)| 
|Добавление исправленных данных в набор отзывов | Данные с ошибками обработки устраняются и загружаются обратно в набор отзывов. Дополнительные сведения см. в указанных ниже статьях.</br>• [Устранение ошибок при обработке данных](error-remediation-when-processing-data-in-advanced-ediscovery.md)</br>• [Исправление ошибки одного элемента](single-item-error-remediation.md)| 
|Сравнение наборов нагрузки | Пользователь рассматривает различия между различными наборами нагрузки в наборе обзоров. Набор нагрузки — это пример добавления данных в набор отзывов. Например, если добавить результаты двух разных поисков в один и тот же набор отзывов, каждый будет представлять набор нагрузки. |
|Реконструкция беседы|Когда пользователь добавляет результаты поиска в набор обзоров бесед, мгновенные беседы сообщений (также называемые беседами с потоками) в службах, таких как Microsoft Teams, реконструируют в файле PDF. Эта работа также запускается, когда пользователь щелкает действие > **создать диалог PDF в** наборе обзоров. Дополнительные сведения см. в [обзоре бесед в расширенных электронных данных.](conversation-review-sets.md)
|Преобразование отредактируемых документов в PDF|После того как пользователь аннотирует документ в наборе отзывов и отредактирует его часть, он может преобразовать отредактированную документацию в файл PDF. Это гарантирует, что отредактируемая часть не будет видна, если документ экспортируется для презентации. Дополнительные сведения см. в обзоре документов [в наборе обзоров.](annotating-and-redacting-documents.md) |
|Оценка результатов поиска | После создания и выполнения нового поиска (или повторного выполнения существующего поиска) средство поиска ищет индекс элементов, которые соответствуют запросу поиска, и подготавливает оценку, которая включает в себя число и общий размер всех элементов поиска, а также количество источников данных.  Дополнительные сведения см. [в примере Сбор данных для дела.](collecting-data-for-ediscovery.md) | 
|Подготовка данных для экспорта | Пользователь экспортирует документы из набора отзывов. По завершению процесса экспорта они могут скачать экспортные данные на локальный компьютер. Дополнительные сведения см. в [примере Export case data](exporting-data-ediscover20.md). | 
|Подготовка к разрешению ошибок |Когда пользователь выбирает файл и создает новое исправление ошибок в  представлении Об ошибке на вкладке Обработка дела, первым шагом в этом процессе является отправка файла с ошибкой обработки в расположение хранилища Azure в облаке Microsoft. Это задание отслеживает ход процесса загрузки. Дополнительные сведения о работе по исправлению [](error-remediation-when-processing-data-in-advanced-ediscovery.md)ошибок см. в этой ссылке. | 
|Подготовка предварительного просмотра поиска | После создания и выполнения пользователем нового поиска (или повторного выполнения существующего поиска) средство поиска готовит пример подмножество элементов (которые соответствуют запросу поиска), которые можно просмотреть. Просмотр результатов поиска поможет определить эффективность поиска.  Дополнительные сведения см. [в примере Сбор данных для дела.](collecting-data-for-ediscovery.md#view-search-results-and-statistics) | 
|Переиндексация данных хранителя | При добавлении хранителя в дело все частично индексные элементы в выбранных источниках данных хранителя переиндексироваться путем процесса *advanced indexing.* Эта работа также запускается при нажатии индекса **Обновления** на вкладке Обработка дела и при обновлении индекса для конкретного хранителя на странице флажок свойств хранителя.  Дополнительные сведения см. [в расширенных индексациях данных хранителя.](indexing-custodian-data.md)
|Запуск аналитики | Пользователь анализирует данные в наборе обзоров, запуская средства аналитики расширенных электронных данных, такие как обнаружение дублирующихся данных, анализ потоков электронной почты и анализ тем. Дополнительные сведения см. в [обзоре Анализ данных.](analyzing-data-in-review-set.md) | 
|Пометка документов | Эта задача запускается, когда пользователь щелкает  кнопку **Пуск** пометки задания в панели тегов при просмотре документов в наборе отзывов. Пользователь может приступить к этой работе после пометки документов в наборе обзоров, а затем для их массового выбора в панели документов представления. Дополнительные сведения см. [в теге документов в наборе обзоров.](tagging-documents.md) |

## <a name="job-status"></a>Состояние задания

В следующей таблице описываются различные состояния для заданий.

| Состояние           | Описание     |
| :----------------- | :----------     |
| Submitted | Было создано новое задание.  Дата и время отправки задания отображаются в столбце **Created** на **вкладке Jobs.** |
| Сбой отправки | Отправка задания не удалась.  Необходимо попытаться перезапуск действия, которое вызвало задание. |
| В процессе выполнения | Задание находится в процессе выполнения, вы можете отслеживать ход выполнения задания на вкладке **Jobs.** |
| Успешное | Задание успешно выполнено. Дата и время завершения задания отображаются в столбце **Завершено** на **вкладке Задания.** |
| Частично успешный | Задание было успешным. Этот статус обычно возвращается, когда задание не находит частично индексных данных (также называемых неиндексуалными данными) в некоторых источниках данных хранителя.  |
| Не выполнено | Задание не удалось.  Необходимо попытаться перезапуск действия, которое вызвало задание. Если задание не удается во второй раз, рекомендуется обратиться в службу поддержки Майкрософт и предоставить сведения о поддержке с задания. |
