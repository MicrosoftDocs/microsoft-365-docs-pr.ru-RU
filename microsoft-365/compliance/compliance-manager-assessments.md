# <a name="build-and-manage-assessments-in-compliance-manager"></a>Создание и управление оценками в диспетчере соответствия требованиям

**В этой статье:** Узнайте, как настроить диспетчер соответствия требованиям для организации, создав и управляя **оценками.** В этой статье вы можете узнать, как создавать оценки, организовывать их в **группы,** работать с средствами **управления,** принимать обновления и экспортировать **отчеты об оценке.**

## <a name="introduction-to-assessments"></a>Введение в оценки

Диспетчер соответствия требованиям помогает вам создавать оценки, которые оценивают соответствие требованиям отраслевых и региональных правил, применимых к вашей организации. Оценки строятся на основе шаблонов оценки, которые содержат необходимые элементы управления, действия по улучшению и действия Майкрософт для завершения оценки. Настройка наиболее подходящих оценок для организации поможет вам реализовать политики и операционные процедуры, чтобы ограничить риск соответствия требованиям.

Все ваши оценки перечислены на вкладке оценки Диспетчер соответствия требованиям. Узнайте больше о [том, как фильтровать представление о своих оценках и интерпретировать состояния состояния.](compliance-manager-setup.md#assessments-page)

> [!IMPORTANT]
> Шаблоны, доступные вашей организации для оценки построения, зависят от лицензионного соглашения. [Просмотрите сведения о лицензировании.](/office365/servicedescriptions/microsoft-365-service-descriptions/microsoft-365-tenantlevel-services-licensing-guidance/microsoft-365-security-compliance-licensing-guidance)

## <a name="data-protection-baseline-default-assessment"></a>Оценка базовой оценки по умолчанию для защиты данных

Чтобы начать работу, Корпорация  Майкрософт предоставляет оценку по умолчанию в диспетчере соответствия требованиям для базовой Microsoft 365 **защиты данных.** Эта базовая оценка имеет набор элементов управления для ключевых правил и стандартов защиты данных и общего управления данными. В этом базовом сценарии используются элементы, главным образом из NIST CSF (National Institute of Standards and Technology Cybersecurity Framework) и ISO (Международная организация по стандартизации), а также из FedRAMP (Федеральная программа управления рисками и авторизациями) и GDPR (Общее регулирование защиты данных Европейского союза).

Эта оценка используется для вычисления начальной оценки соответствия требованиям при первом при первом подстройке диспетчера соответствия требованиям перед настройкой других оценок. Диспетчер соответствия требованиям собирает начальные сигналы из Microsoft 365 решений. Вы сразу увидите, как ваша организация выполняет свои действия относительно ключевых стандартов и правил защиты данных, а также предлагаемые действия по улучшению.

Диспетчер соответствия требованиям становится более полезным при создании и управлении собственными оценками в соответствии с конкретными потребностями организации.

## <a name="understand-groups-before-creating-assessments"></a>Понимание групп перед созданием оценок

При создании оценки необходимо назначить ее группе. Группы — это контейнеры, которые позволяют организовывать оценки таким логическим образом, как по годам или нормативным данным, или на основе подразделений или географических данных организации. Поэтому перед созданием оценок рекомендуется планировать стратегию группировки.

Ниже приведены примеры двух групп и их оценок:

- **Оценка ИГ FFIEC 2020**
  - FFIEC IS
- **Оценки безопасности данных и конфиденциальности**
  - ISO 27001:2013
  - ISO 27018:2014

Если две различные оценки в одной группе разделяют действия по улучшению, которые вы управляете, любые обновления, которые вы делаете для деталей или состояния реализации действия, будут автоматически синхронизироваться по всей группе. Эта синхронизация позволяет реализовать одно действие по улучшению и одновременно выполнить несколько требований.

### <a name="create-a-group"></a>Создание группы

Вы можете создать группу при создании новой оценки. Группы не могут быть созданы в качестве автономных сущностями.

### <a name="what-to-know-when-working-with-groups"></a>Что нужно знать при работе с группами

- Группа должна содержать по крайней мере одну оценку.
- Имена групп должны быть уникальными в организации.
- Группы не имеют свойств безопасности. Все разрешения связаны с оценками.
- После добавления оценки в группу группу изменить нельзя.
- Если вы добавляете новую оценку в существующую группу, общие сведения из оценок в этой группе копируется в новую оценку.
- Связанные элементы управления оценкой при различных оценках в одной группе автоматически обновляются по завершению.
- При изменении улучшения, которое отображается в нескольких группах, это изменение находит отражение во всех случаях этого действия по улучшению.
- Группы могут содержать оценки для одной и той же сертификации или регулирования, но каждая группа может содержать только одну оценку для определенной пары сертификации продуктов. Например, группа не может содержать две оценки для Office 365 и NIST CSF. Группа может содержать несколько оценок для одного и того же продукта, только если соответствующая сертификация или регулирование для каждого из них отличаются.
- Удаление оценки нарушает связь между этой оценкой и группой.
- Группы нельзя удалять вручную.

## <a name="create-assessments"></a>Создание оценок

> [!NOTE]
> Создавать и изменять оценки могут только пользователи, которые занимают роль глобального администратора, администратора соответствия требованиям или руководителя по оценке соответствия требованиям. Дополнительные новости [о ролях и разрешениях.](compliance-manager-setup.md#set-user-permissions-and-assign-roles)

Чтобы приступить к оценке построения, выполните следующие действия.

1. Знайте, в какую группу вы назначите свою оценку, или будьте готовы создать новую для этой оценки.

2. Откройте мастер оценки. Вы можете получить доступ к этой области вылетов из одного из двух мест:
    - Перейдите на **страницу оценки** в диспетчере соответствия требованиям и выберите **добавить оценку;** или
    - Найдите шаблон, который необходимо использовать на вкладке **шаблоны** оценки, просматривайте его сведения и выберите **Создание оценки.** Это заполнит поле выбора шаблона мастера для вас.

3. **Выберите шаблон.** Если вы еще не выбрали шаблон на шаге 2, выберите шаблон, который будет служить основой для оценки. Вы увидите список шаблонов, разделенных на включенные и премиум-категории (дополнительные сведения см. в [разделе Типы](compliance-manager-templates.md#template-availability-and-licensing) шаблонов). Выберите кнопку радио рядом с выбранным шаблоном, а затем выберите **Далее**.

4. **Имя и группа:** Установите эти свойства, чтобы определить оценку и назначить ее группе.
    - **Имя.** Введите имя для оценки в поле **Имя оценки.** Имена оценки должны быть уникальными в группах. Если имя вашей оценки совпадает с именем другой оценки в любой группе, вы получите ошибку с просьбой создать другое имя.
    - **Группа.** Назначьте свою оценку группе. Вы можете либо:
        - Выберите **Использовать существующую группу,** чтобы назначить ее уже созданной группе; или
        - Выберите **Создать новую группу** для создания новой группы и назначить эту оценку:
            - Определите имя группы и введите его в поле под кнопкой радио.
            - Вы можете **скопировать данные из существующей группы,** такие как сведения о реализации и тестировании и документы, выбрав соответствующие поля.

    По завершению выберите **Далее**.

5. **Обзор и отделка:** На последнем экране мастера показан шаблон, имя и группа, выбранные для оценки. Вы можете изменить любой из этих параметров из ссылок на экране, которые будут возвращаться к соответствующим шагам мастера. Когда вы будете готовы, выберите **Создать оценку**.

6. Следующий экран подтверждает, что вы успешно создали новую оценку. Выберите **Сделано,** чтобы закрыть мастера, и на экране будут отображаться сведения о новой оценке.

Если после выбора **"Создать** оценку" экран сбойной оценки, выберите **Попробуйте** еще раз создать свою оценку. 

Вы можете изменить имя оценки после ее создания, выбрав кнопку **Изменить** имя в верхнем правом углу страницы сведений [оценки.](#monitor-assessment-progress-and-controls)

## <a name="monitor-assessment-progress-and-controls"></a>Мониторинг хода оценки и элементов управления

Каждая оценка имеет страницу с подробными сведениями, на которых по-быстрому просматривается ваш прогресс в завершении оценки. На странице показаны ваши успехи в выполнении элементов управления, а также состояние тестирования действий по улучшению ключевых элементов управления.

### <a name="overview-tab"></a>Вкладка Обзор

В вкладке обзор содержится график, показывающий процент, относящаяся к завершению оценки. На этом графике содержится разбивка точек из действий, которые вы имеете, и указывается на действия, которые принадлежат Корпорации Майкрософт, поэтому вы можете увидеть, сколько еще баллов необходимо для завершения оценки.

Основные действия по улучшению элементов управления в оценке перечислены в порядке наибольшего потенциального воздействия для получения баллов. На связанном графике подробно совокупный тестовый статус действий по улучшению, чтобы можно было быстро оценить, что было протестированно и что еще нужно сделать.

Чтобы получить доступ к отдельным действиям по улучшению, посетите вкладку **Controls** или **вкладку Действия по улучшению.**

### <a name="controls-tab"></a>Вкладка с элементами управления

На вкладке элементов управления отображаются подробные сведения о каждом элементе управления, отображемом для оценки. Диаграмма **разбивки** состояния управления показывает состояние элементов управления по семейство, поэтому сразу видно, какие группы элементов управления нуждаются в внимании.

Под диаграммой в таблице перечислены подробные сведения о каждом контроле в рамках оценки. Элементы управления группуются семейством элементов управления. Расширь каждую фамилию, чтобы выявить отдельные элементы управления, которые она содержит. Сведения, перечисленные для каждого управления, включают в себя:

- **Название управления**
- **Состояние**: отражает состояние теста действий по улучшению в области управления 
    - **Пройдено** — все действия по улучшению имеют тестовый статус "пройденный", или по крайней мере один передается, а остальные " вне области"
    -  **Не удалось** — по крайней мере одно действие по улучшению имеет тестовый статус "failed"
    - **Нет** — все действия по улучшению не были протестированы
    - **Вне области —** все действия по улучшению для этой оценки не подвластные.
    - **В процессе** выполнения — действия по улучшению имеют состояние, помимо перечисленных выше, которое может включать в себя "прогресс", "частичный кредит" или "незащищенный"
- **Удостоверение управления:** идентификационный номер управления, задаваемый соответствующим регулированием, стандартом или политикой.
- **Достигнутые** очки: количество баллов, заработанных при выполнении действий, из общего количества достижимых баллов 
- **Ваши действия:** количество ваших действий, завершенных из общего числа действий, которые необходимо сделать
- **Действия Майкрософт:** количество действий, завершенных Корпорацией Майкрософт 

Чтобы просмотреть сведения о контроле, выберите его из строки в таблице. На странице сведения о контроле показан график с указанием тестового состояния действий в этом контроле. В таблице ниже графика показаны основные действия по улучшению этого управления.

Выберите действие по улучшению из списка для сверки на страницу сведений о действии по улучшению. На страницах подробных сведений показано состояние теста, заметки о реализации и запуск в рекомендуемое решение.

### <a name="your-improvement-actions-tab"></a>Вкладка действия по улучшению

На вкладке для действий по улучшению перечислены все элементы управления в оценке, управляемые вашей организацией. В панели состояния подробно сообщается об агрегированном тестовом состоянии действий по улучшению в оценке, чтобы можно было быстро оценить, что было протестированно и что еще нужно сделать. Под панели находится полный список действий по улучшению и ключевых сведений, включая: состояние тестирования, количество потенциальных и заработанных баллов, соответствующие правила и стандарты, применимое решение, тип действия и семейство элементов управления. Узнайте больше о [том, как действия вносят вклад в оценку соответствия](compliance-score-calculation.md#action-types-and-points)требованиям.

Выберите действие по улучшению, чтобы просмотреть страницу сведений, и выберите ссылку **Запуск,** чтобы открыть решение для принятия действий.

### <a name="microsoft-actions-tab"></a>Вкладка действия Майкрософт

На вкладке Действия Майкрософт перечислены все действия в оценке, управляемой Корпорацией Майкрософт. В списке показаны основные сведения о действии, в том числе: состояние теста, точки, которые способствуют достижению общей оценки соответствия требованиям, связанным требованиям и стандартам, применимое решение, тип действия и семейство элементов управления. Выберите действие по улучшению, чтобы просмотреть его страницу сведений.

Узнайте больше о [том, как отслеживаются](compliance-score-calculation.md) и забиваются действия по контролю и улучшению.

## <a name="accept-updates-to-assessments"></a>Принятие обновлений к оценкам

Когда обновление доступно для оценки, вы увидите уведомление и сможете принять обновление или отложить его на более позднее время.

### <a name="what-causes-an-update"></a>Причины обновления

Обновление оценки происходит при внесении изменений в шаблоны, которые влияют на скоринг. Изменения могут включать корректировку сопоставления управления или других указаний на основе нормативных изменений или изменений продукта. Обновления для оценки могут поступать из организации (например, при внесении изменений в настраиваемый [шаблон),](compliance-manager-templates.md#modify-a-template)а также из Корпорации Майкрософт.

Если Корпорация Майкрософт обновит расширенный шаблон диспетчера соответствия требованиям, ваша оценка наследует эти обновления, как только вы их примете. При ее продлении при оценке будут сохранены дополнительные атрибуты, примененные к оценке.

Настраиваемые оценки, которые вы создаете, не получают обновлений шаблонов от Microsoft. Пользовательские оценки могут получать обновления действий по улучшению, но любые обновления Майкрософт для управления сопоставлением между оценками и действиями по улучшению не применяются к настраиваемой шаблонам.

> [!NOTE]
> Обновления к оценкам применяются только на групповом уровне. Если у вас есть две оценки, построенные из одного шаблона, который существует в двух разных группах, каждая оценка будет иметь ожидающих уведомления об обновлении, и вам нужно будет принять обновление для каждой оценки в своей соответствующей группе по отдельности.

#### <a name="where-youll-see-assessment-update-notifications"></a>Где вы увидите уведомления об обновлении оценки

На странице сведения об оценке также показана метка **"Ожидание обновления"** рядом с оценкой с обновлением. Выберите эту оценку, чтобы добраться до ее страницы сведений.

Сообщение, размещенное в верхней части страницы сведений об оценке, показывает, что для этой оценки доступно обновление. Выберите **кнопку Обновления Обзор** на баннере, чтобы просмотреть определенные изменения и принять или отложить обновление.

На странице сведения об оценке могут также перечисляться действия по улучшению, которые имеют рядом метку **Ожидающих** обновлений. Эти обновления необходимы для определенных изменений самих действий по улучшению и должны приниматься отдельно. Узнайте больше о приеме обновлений [для действий по](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions) улучшению.

#### <a name="review-update-to-accept-or-defer"></a>Просмотр обновления для принятие или отсрочка

После выбора обновления **Обзор со** страницы сведений об оценке на правой стороне экрана отображается поле для вылетов. На области вылетов приведены основные сведения о ожидаемом обновлении:

- Название шаблона
- Источник обновления (Microsoft, ваша организация или определенный пользователь)
- Дата создания обновления
- Обзор, объясняющий обновление
- Конкретные сведения об изменениях, в том числе о влиянии на оценку соответствия требованиям, объеме прогресса в завершении оценки и определенном количестве изменений в действиях по улучшению и средствах управления.

Выбор **ссылки обновленный** шаблон загрузит Excel, содержащий данные управления для версии шаблона с ожидающих обновлений. Выбор **ссылки текущий шаблон** загружает файл существующего шаблона без изменений.

Чтобы принять обновление и внести изменения в свою оценку, выберите **accept update**. Принятые изменения являются постоянными.

Если вы выберите **Отмена,** обновление не будет применено к оценке. Тем не менее, вы будете продолжать видеть уведомление об ожидающих **обновлениях,** пока не примете обновление.

**Почему рекомендуется принимать обновления**

Принятие обновлений поможет вам получить самые обновленные рекомендации по использованию решений и принятию соответствующих действий по улучшению, чтобы соответствовать требованиям сертификации.

**Почему может потребоваться отложить обновление**

Если вы в середине завершения оценки, вы можете убедиться, что вы закончили работу над ней, прежде чем принять обновление к оценке, которая может нарушить сопоставление управления. Вы можете отложить обновление на более позднее время, выбрав **отмену** в области флааутов обновления обзоров.

## <a name="export-an-assessment-report"></a>Экспорт отчета об оценке

Вы можете экспортировать оценку в Excel для заинтересованных сторон соответствия требованиям в вашей организации или для внешних аудиторов и регулирующих органов. На странице Сведения о оценке выберите кнопку **Создание** отчета в верхней части страницы, которая создает Excel файл, который можно сохранить и поделиться.

Отчет — это снимок оценки даты и времени экспорта. Он содержит сведения о управлениях, управляемых как вами, так и Корпорацией Майкрософт, включая состояние реализации, дату тестирования и результаты тестирования.

## <a name="delete-an-assessment"></a>Удаление оценки

Удаление оценки удаляет ее из списка на странице оценки. Обратите внимание на эти важные моменты об удалении оценок:

- **Удаление оценки является постоянным; вы не можете получить его обратно.** Если вы хотите снова использовать ту же оценку, ее необходимо заново создать.
- Если действия по улучшению в оценке не отображаются ни в одной другой оценке, они будут удалены при удалении оценки.
- Мы рекомендуем [экспортировать отчет об](#export-an-assessment-report) оценке, прежде чем окончательно удалить его.

Чтобы удалить оценку, выполните следующие действия:

1. На странице **оценки** выберите оценку, которую вы хотите удалить, чтобы открыть страницу сведений о оценке.

2. Выберите **оценку Удаления** в правом верхнем углу экрана.

3. Появится окно с просьбой подтвердить, что вы хотите удалить оценку навсегда. Выберите **оценку Удаления,** чтобы закрыть окно. Вы получите окно подтверждения того, что ваша оценка была удалена из диспетчера соответствия требованиям.

Если удалить единственную оценку в группе, эта группа также удаляется из диспетчера соответствия требованиям.

> [!NOTE]
> Вы не можете удалить все свои оценки. Для правильной работы диспетчера соответствия требованиям организациям требуется по крайней мере одна оценка. Если оценка, которая необходимо удалить, является единственной, добавьте другую оценку перед удалением другой оценки.
