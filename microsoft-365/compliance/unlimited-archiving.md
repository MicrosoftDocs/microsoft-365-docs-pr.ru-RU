# <a name="overview-of-unlimited-archiving"></a>Общие сведения о неограниченной архивации

В Office 365 архивные почтовые ящики предоставляют пользователям дополнительное пространство для хранения почтовых ящиков. После включения архивного почтового ящика пользователя доступно до 100 ГБ дополнительного хранилища. В прошлом, когда была достигнута квота хранения на 100 ГБ, организациям приходилось обращаться в Корпорацию Майкрософт, чтобы запросить дополнительное пространство для хранения архивного почтового ящика. Это уже не так.

Функция неограниченного архивирования в Microsoft 365 (называемая автоматической расширенной архива) предоставляет дополнительное хранилище в архивных почтовых ящиках.  Когда квота хранения в архивном почтовом ящике будет достигнута, Microsoft 365 автоматически увеличивает размер архива, что означает, что у пользователей не будет места для хранения почтовых ящиков, а администраторам не придется запрашивать дополнительное хранилище для архивных почтовых ящиков.

Инструкции по пошаговому включению автоматического расширения архива см. в пункте [Включение неограниченного архива.](enable-unlimited-archiving.md)

> [!NOTE]
> Автоматически расширяющиеся архивы также доступны для общих почтовых ящиков. Чтобы включить архив общего почтового ящика, требуется лицензия Exchange Online Plan 2 или лицензия Exchange Online Plan 1 с архивация на базе Exchange Online лицензией.

## <a name="how-auto-expanding-archiving-works"></a>Как работает автоматическое расширение архива

Как объяснялось ранее, при включенном архивном почтовом ящике пользователя создается дополнительное пространство для хранения почтовых ящиков. При включенном автоматическом расширении архива Microsoft 365 периодически проверяет размер архивного почтового ящика. Когда почтовый ящик архива становится ближе к пределу хранения, Microsoft 365 автоматически создает дополнительное пространство для хранения архива. Если у пользователя заканчивается это дополнительное пространство для хранения, Microsoft 365 добавляет больше места для хранения в архив пользователя. Этот процесс происходит автоматически, что означает, что администраторам не нужно запрашивать дополнительное хранилище архива или управлять автоматическим расширением архивирования.

Ниже представлен краткий обзор процесса.

![Обзор процесса автоматического расширения архивации](../media/74355385-d990-44fe-8a87-6c3639d1f63f.png)

1. Архивативная возможность включена для почтового ящика пользователя или общего почтового ящика. Создается архивный почтовый ящик со 100 ГБ пространства для хранения, а квота предупреждения для архивного почтового ящика установлена до 90 ГБ.

2. Администратор включает автоматическое расширение архива для почтового ящика. Когда архивный почтовый ящик (в том числе папка "Извлекаемые элементы") достигает 90 ГБ, он преобразуется в архив автоматического расширения, а Microsoft 365 добавляет пространство для хранения в архив. Подготовка дополнительного пространства для хранения может занять до 30 дней.

   > [!NOTE]
   > Если почтовый ящик находится на удержании или назначен политике хранения, квота хранения архивного почтового ящика увеличивается до 110 ГБ при включенном автоматическом расширении архивирования. Кроме того, квота предупреждения в архиве увеличивается до 100 ГБ.

3. Microsoft 365 автоматически добавляет больше места для хранения при необходимости.

> [!IMPORTANT]
> Архив автоматического расширения поддерживается только для почтовых ящиков, используемых для отдельных пользователей (или общих почтовых ящиков), с темпом роста не более 1 ГБ в день. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Использование правил журналов, транспорта или автотранспорта для копирования сообщений в архивный почтовый ящик запрещено. Корпорация Майкрософт оставляет за собой право отказать в неограниченной архивной обработке в тех случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных для других пользователей или в других случаях ненадлежащего использования.

## <a name="what-gets-moved-to-the-additional-archive-storage-space"></a>Что перемещается в дополнительное хранилище архива?

Чтобы эффективно использовать автоматическое расширение архивного хранилища, папки могут быть перемещены. Microsoft 365 определяет, какие папки перемещаются при добавлении дополнительного хранилища в архив. Иногда при перемещении папки автоматически создается один или несколько подвещений, а элементы из исходной папки распределяются по этим папкам для облегчения процесса перемещения. При просмотре архивной части списка папок в Outlook эти подмостки отображаются в исходной папке.  Конвенция именования, которую Microsoft 365 использует для именования этих подвещений, _yyyy (Создан на **\<folder name\> mmm dd, yyyy h_mm)**, где:

- **yyyy —** это год, когда сообщения в папке были получены.

- **mmm dd, yyyy h_m** — это дата и время создания подмостков Office 365 в формате UTC на основе часовой пояс пользователя и региональных параметров в Outlook.

На следующих скриншотах покажите список папок до и после того, как сообщения будут перемещены в автоматически расширенный архив.

 **Перед добавлением дополнительного хранилища**

![Список папки архивного почтового ящика перед предоставлением автоматического расширения архива](../media/5d6d6420-e562-4912-aaab-1c111762b3f6.png)

 **После добавленного дополнительного хранилища**

![Список папок архивного почтового ящика после автоматического расширения архива](../media/c03c5f51-23fa-4fc2-b887-7e7e5cce30da.png)

> [!NOTE]
> Как описано ранее, Microsoft 365 перемещает элементы в подмостки (и называет их с помощью описанной выше конвенции имен), чтобы помочь распространять контент в вспомогательный архив. Но перемещение элементов в подмостки может быть не всегда так. Иногда вся папка может быть перемещена в вспомогательный архив. В этом случае папка сохранит свое исходное имя.  В списке папок в Outlook не будет видно, что папка была перемещена в вспомогательный архив.

## <a name="outlook-requirements-for-accessing-items-in-an-auto-expanded-archive"></a>Требования Outlook для доступа к пунктам в автоматическом расширенном архиве

Чтобы получить доступ к сообщениям, хранимым в автоматическом архиве, пользователям необходимо использовать одного из следующих клиентов Outlook:

- Outlook 2016 или Outlook 2019 для Windows

- Outlook в Интернете

- Outlook 2016 или Outlook 2019 для Mac

Вот некоторые моменты, которые следует учитывать при использовании Outlook или Outlook в Интернете для доступа к сообщениям, хранимым в автоматическом расширенном архиве.

- Вы можете получить доступ к любой папке в почтовом ящике архива, включая те, которые были перемещены в область автоматического расширения хранилища.

- Поиск автоматического расширенного архива доступен в Outlook для веб-сайта (OWA). Как и в Online Archive, можно искать элементы, которые были перемещены в дополнительную область хранения. При выборе архива в качестве области поиска в OWA будут искаться все архивы (включая автоматически расширенные архивы) и соответствующие подмостки.

- Автоматически расширенный поиск в архиве доступен в Outlook Desktop в current Channel (Preview). В этом предварительном просмотре доступна область Текущий почтовый ящик, что позволяет искать автоматически расширенный архив. Дополнительные сведения об этом и других функций поддержки microsoft Search см. в обзоре Как Outlook для Windows, подключенная к [Exchange Online, использует Microsoft Search.](https://techcommunity.microsoft.com/t5/outlook-global-customer-service/how-outlook-for-windows-connected-to-exchange-online-utilizes/ba-p/1715045) 

- Количество элементов в графах Outlook и Read/Unread (в Outlook и Outlook в Интернете) в автоматическом расширенном архиве может быть не точным.

- Вы можете удалить элементы в подмостках, которые указывает на область хранения с автоматическим расширением, но сама папка не может быть удалена.

- Вы не можете использовать функцию Восстановление удаленных элементов для восстановления элемента, который был удален из области автоматического расширения хранилища.

## <a name="auto-expanding-archiving-and-other-compliance-features"></a>Автоматическое расширение архива и другие функции соответствия требованиям

В этом разделе объясняется функциональность между автоматическим расширением архива и другими функциями управления соответствием требованиям и данными.

- **eDiscovery:** При использовании средства поиска электронных данных, например поиска контента или In-Place поиска электронных данных, также ищутся дополнительные области хранения в автоматическом расширенном архиве.

- **Хранение:** При удержании почтового ящика с помощью таких средств, как хранение судебного разбирательства в Exchange Online или политика хранения электронных данных в центре безопасности и соответствия требованиям, содержимое, расположенное в автоматическом расширенном архиве, также помещается на удержание.

- **Управление записями обмена сообщениями (MRM):** Если вы используете политики удаления MRM в Exchange Online для удаления элементов почтовых ящиков с истекшим сроком годности, элементы с истекшим сроком действия, расположенные в автоматическом расширенном архиве, также будут удалены.

- **Служба импорта:** Службу Импорта Office 365 можно использовать для импорта PST-файлов в автоматически расширенный архив пользователя. Вы можете импортировать до 100 ГБ данных из PST-файлов в архивный почтовый ящик пользователя.
