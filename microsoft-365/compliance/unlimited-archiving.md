# <a name="overview-of-unlimited-archiving"></a>Общие сведения о неограниченной архивации

В Office 365 архивные почтовые ящики предоставляют пользователям дополнительное место в хранилище почтовых ящиков. После включения архивного почтового ящика пользователя доступно до 100 ГБ дополнительного хранилища. В прошлом, когда была достигнута квота хранилища в 100 ГБ, организациям приходилось обращаться в корпорацию Майкрософт, чтобы запросить дополнительное место для архивного почтового ящика. Это больше не так.

Функция неограниченной архивирования в Microsoft 365 (называемая функцией автоматического расширения *архивирования)* предоставляет дополнительное хранилище в архивных почтовых ящиках. Когда квота хранилища в архивном почтовом ящике достигается, Microsoft 365 автоматически увеличивает размер архива, что означает, что у пользователей не будет места на хранилище почтовых ящиков, и администраторам не придется запрашивать дополнительное хранилище для архивных почтовых ящиков.

Пошаговая инструкция по включению автоматического расширения архива см. в подзаговоре ["Включение неограниченной архивной архива".](enable-unlimited-archiving.md)

> [!NOTE]
> Автоматически расширяющиеся архивы также доступны для общих почтовых ящиков. Чтобы включить архив для общего почтового ящика, необходима лицензия exchange Online (план 2) или Exchange Online (план 1) с архивация на базе Exchange Online лицензии.

## <a name="how-auto-expanding-archiving-works"></a>Как работает автоматическое расширение архива

Как объяснялось ранее, при включаемом архивном почтовом ящике пользователя создается дополнительное место в хранилище почтовых ящиков. Если включено автоматическое расширение архива, Microsoft 365 периодически проверяет размер архивного почтового ящика. Когда архивный почтовый ящик близко к предельному объему хранилища, Microsoft 365 автоматически создает дополнительное место для архива. Если пользователю не место в этом дополнительном хранилище, Microsoft 365 добавляет больше места в архив пользователя. Этот процесс происходит автоматически, что означает, что администраторам не нужно запрашивать дополнительное архивное хранилище или управлять архивативам с автоматическим расширением.

Ниже представлен краткий обзор процесса.

![Обзор процесса автоматического расширения архивации](../media/74355385-d990-44fe-8a87-6c3639d1f63f.png)

1. Архивная архива включена для почтового ящика пользователя или общего почтового ящика. Создается архивный почтовый ящик с объемом хранилища 100 ГБ, а квота предупреждения для архивного почтового ящика составляет 90 ГБ.

2. Администратор включает автоматическое расширение архива для почтового ящика. Когда архивный почтовый ящик (включая папку "Элементы с возможностью восстановления") достигает 90 ГБ, он преобразуется в архив с автоматическим расширением, а Microsoft 365 добавляет место в архив. Подготовка дополнительного пространства для хранения может занять до 30 дней.

   > [!NOTE]
   > Если почтовый ящик помещен на удержание или назначен политике хранения, квота хранилища для архивного почтового ящика увеличивается до 110 ГБ при включенной автоматическом расширении архивирования. Аналогичным образом, квота предупреждения для архива увеличена до 100 ГБ.

3. При необходимости Microsoft 365 автоматически добавляет больше места в хранилище.

> [!IMPORTANT]
> Автоматически расширяющийся архив поддерживается только для почтовых ящиков, используемых для отдельных пользователей (или общих почтовых ящиков) с скоростью роста не более 1 ГБ в день. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Использование журналов, правил транспорта или правил автоматической пересылки для копирования сообщений в архивный почтовый ящик запрещено. Корпорация Майкрософт оставляет за собой право отказать в неограниченном архиве в тех случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных других пользователей или в других случаях ненадлежащего использования.

## <a name="what-gets-moved-to-the-additional-archive-storage-space"></a>Что перемещается в дополнительное архивное хранилище?

Для эффективного использования автоматически расширяемого архивного хранилища папки могут быть перемещены. Microsoft 365 определяет, какие папки перемещаются при добавлении дополнительного хранилища в архив. Иногда при перемещении папки автоматически создается одна или несколько вложенных папок, а элементы из исходной папки распространяются в эти папки для упрощения процесса перемещения. При просмотре архивной части списка папок в Outlook эти вложенные папки отображаются под исходной папкой.  Соглашение об именах, которое Microsoft 365 использует для именования этих ветвей, _yyyy (создается на **\<folder name\> MMM dd, yyyy h_mm)**, где:

- **yyyy** — это год, в который были получены сообщения в папке.

- **mmm dd, yyyy h_m** — это дата и время создания подкапки в Office 365 в формате UTC в зависимости от часовой пояс пользователя и региональных параметров в Outlook.

На следующих снимках экрана список папок показывается до и после того, как сообщения перемещаются в архив с автоматическим расширением.

 **Перед добавлением дополнительного хранилища**

![Список папок архивного почтового ящика перед подготовкаю архива с автоматическим расширением](../media/5d6d6420-e562-4912-aaab-1c111762b3f6.png)

 **После того как будет добавлено дополнительное хранилище**

![Список папок архивного почтового ящика после предоставления архива с автоматическим расширением](../media/c03c5f51-23fa-4fc2-b887-7e7e5cce30da.png)

> [!NOTE]
> Как было описано выше, Microsoft 365 перемещает элементы во ведерки (и именует их с помощью описанного выше соглашения об именовом имени) для распространения контента в вспомогательный архив. Но перемещение элементов во вудерки может не всегда иметь дело. Иногда в вспомогательный архив может быть перемещена вся папка. В этом случае папка сохранит свое исходное имя.  В списке папок в Outlook не будет видно, что папка была перемещена в вспомогательный архив.

## <a name="outlook-requirements-for-accessing-items-in-an-auto-expanded-archive"></a>Требования Outlook для доступа к элементов в архиве с автоматическим расширением

Для доступа к сообщениям, хранимым в архиве с автоматическим расширением, пользователям необходимо использовать один из следующих клиентов Outlook:

- Outlook 2016 или Outlook 2019 для Windows

- Outlook в Интернете

- Outlook 2016 или Outlook 2019 для Mac

Вот некоторые моменты, которые следует учитывать при использовании Outlook или Outlook в Интернете для доступа к сообщениям, хранимым в архиве с автоматическим расширением.

- Вы можете получить доступ к любой папке в архивном почтовом ящике, включая те, которые были перемещены в автоматически расширяемую область хранения.

- В Outlook в Интернете доступен поиск архива с автоматическим расширением. Как и в оперативном архиве, вы можете искать элементы, которые были перемещены в дополнительную область хранения, только выяя текущую папку. Это означает, что необходимо выбрать архивную папку в списке папок, а затем выбрать одну папку в качестве области поиска. Аналогичным образом, если папка в автоматически расширенной области хранения содержит вложенные папки, необходимо искать каждую вложенную папку отдельно.
- Автоматически расширенный поиск в архиве доступен в outlook Desktop в current Channel (предварительная версия). В этой предварительной версии доступна область текущего почтового ящика, что позволяет искать в архиве с автоматическим расширением. Дополнительные сведения об этом и других функций поддержки Поиска (Майкрософт) см. в том, как Outlook для Windows, подключенный к [Exchange Online, использует Поиск (Майкрософт).](https://techcommunity.microsoft.com/t5/outlook-global-customer-service/how-outlook-for-windows-connected-to-exchange-online-utilizes/ba-p/1715045) 

- Количество элементов в Outlook и количество просмотров и непрочитанные (в Outlook и Outlook в Интернете) в автоматически расширенном архиве могут быть не точными.

- Вы можете удалить элементы во вложенной папке, которая указывает на автоматически расширяемую область хранения, но сама папка не может быть удалена.

- Функцию восстановления удаленных элементов нельзя использовать для восстановления элемента, удаленного из автоматически расширяемого хранилища.

## <a name="auto-expanding-archiving-and-other-compliance-features"></a>Автоматическое расширение архива и других функций соответствия требованиям

В этом разделе поясняется функциональность между автоматически расширяемой архиватива и другими функциями обеспечения соответствия требованиям и управления данными.

- **EDiscovery:** При использовании средства eDiscovery, такого как поиск контента или In-Place eDiscovery, поиск также ведется в дополнительных областях хранения в архиве с автоматическим расширением.

- **Хранение:** Когда вы помещаете почтовый ящик на удержание с помощью таких средств, как хранение для судебного разбирательства в Exchange Online или удержание дел eDiscovery и политики хранения в Центре безопасности и соответствия требованиям, контент, расположенный в архиве с автоматическим расширением, также помещается на удержание.

- **Управление записями сообщений (MRM):** Если вы используете политики удаления сообщений в Exchange Online для окончательного удаления элементов почтового ящика с истекшим сроком действия, элементы с истекшим сроком действия, расположенные в архиве с автоматическим расширением, также будут удалены.

- **Служба импорта:** Службу импорта Office 365 можно использовать для импорта PST-файлов в автоматически расширяемой архив пользователя. В архивный почтовый ящик пользователя можно импортировать до 100 ГБ данных из PST-файлов.
