# <a name="insider-risk-management-users-dashboard"></a>Панель мониторинга "Управление рисками для участников в организации"

Информационная **панель "Пользователи"** является важным средством в рабочий процесс управления рисками внутри организации и помогает следователям и аналитикам лучше понимать действия с рисками. На этой панели мониторинга представлены представления и функции управления для удовлетворения административных потребностей между созданием политик управления рисками внутри организации и управлением делами управления рисками внутри организации.

После того как пользователи добавлены в политики управления рисками внутри организации, фоновые процессы автоматически оценивают действия пользователей для [запуска индикаторов.](insider-risk-management-settings.md#indicators) После того как активирует индикаторы, действиям пользователей будут назначены показатели риска. Некоторые из этих действий могут привести к оповещению о insider risk, но некоторые действия могут не соответствовать минимальному уровню оценки риска, а оповещение о insider risk не будет создано. Информационная **панель "Пользователи"** позволяет просматривать пользователей с такими индикаторами и показателями риска, а также пользователей с активными оповещениями о риске внутри системы.

Кроме того, могут возникнуть ситуации, в которых может потребоваться временно добавить пользователей в внутренние политики риска после того, как о необычном событии сообщается за пределами рабочего процесса управления рисками внутри организации. Информационная **панель** "Пользователи" позволяет вручную добавить пользователя в политику риска для участников в течение определенного времени и обойти требование, чтобы у пользователя был индикатор запуска. Эти пользователи всегда отображаются на панели мониторинга "Пользователи" при активном присвоении политики.

Узнайте больше о том, как панель мониторинга "Пользователи" отображает пользователей в следующих сценариях:

- Пользователи панели мониторинга с активными оповещениями политики внутренней оценки риска
- Пользователи панели мониторинга с активными индикаторами
- Пользователи панели мониторинга, временно добавленные к политикам

## <a name="dashboard-users-with-active-insider-risk-policy-alerts"></a>Пользователи панели мониторинга с активными оповещениями политики внутренней оценки риска

На **информационной панели "Пользователи"** автоматически отображаются все пользователи с активными оповещениями о политике внутренней оценки риска. Эти пользователи с оповещениями имеют индикатор запуска и показатель риска активности, который соответствует требованиям для создания оповещения о риске внутри системы. Действия для этих пользователей просматриваются путем выбора  пользователя на панели мониторинга "Пользователи" и переходом на вкладку **"Действия пользователя".**

## <a name="dashboard-users-with-triggering-indicators"></a>Пользователи панели мониторинга с активными индикаторами

Панель **мониторинга "Пользователи"** автоматически отображает всех пользователей с активными индикаторами, но не имеет оценки риска активности, которая создает оповещение о риске внутри системы. Например, отображается пользователь с датой сообщаемой даты, так как это событие является индикатором запуска, но не является действием, которое имеет показатель риска. Действия для этих пользователей просматриваются путем выбора  пользователя на панели мониторинга "Пользователи" и переходов на вкладку **"Действия пользователя".**

## <a name="dashboard-users-added-temporarily-to-policies"></a>Пользователи панели мониторинга, временно добавленные к политикам

Информационная **панель "Пользователи"** позволяет временно добавлять пользователей в существующую политику управления рисками внутри организации после необычного события за пределами рабочего процесса управления рисками внутри организации. Временное добавление пользователей также позволяет добавить пользователей в политику управления рисками для тестирования политики, даже если необходимый соединителю не настроен.

Когда пользователь вручную добавляется в политику, действия пользователя за предыдущие 90 дней оформяются и добавляются на временную шкалу **действий** пользователя. Например, у вас есть пользователь, который в настоящее время не находится в области действия в политике внутренней оценки риска, а сведения о действиях, связанных с утечкой данных, сообщаются в юридический отдел организации. Юридический отдел рекомендует настроить новые краткосрочные требования к мониторингу для пользователя. Вы можете временно назначить  пользователя политике утечки данных на указанный период времени (окно активации). Все пользователи, добавленные временно,  отображаются на панели мониторинга "Пользователи", так как требования к индикаторам не запускаются.

>[!NOTE]
>Чтобы новые пользователи, добавленные вручную, появились на панели мониторинга **"Пользователи",** может потребоваться несколько часов. Отображение действий за предыдущие 90 дней для этих пользователей может занять до 24 часов. Чтобы просмотреть действия для пользователей, добавленных  вручную,  выберите пользователя на панели мониторинга "Пользователи" и откройте вкладку "Действия пользователя" в области сведений.

Пользователь автоматически удаляется из политики внутренней  безопасности и панели мониторинга "Пользователи", когда истекает время, определенное в окне активации, если: 

- у пользователя нет запускающих индикаторов или оповещений внутренней политики риска, а также
- если продолжительность периода  активации, определяемая вручную, превышает длительность окна активации **глобальной** политики. 

Параметр **окна активации** с самой продолжительной  длительностью всегда переопределяет параметр окна активации с более короткой длительностью. Например, вы настроили  окно активации  на вкладке "Временные рамки глобальной политики" в глобальных параметрах управления рисками внутри организации в течение 15 дней, которое автоматически применяется во всех ваших политиках. 

Вы временно добавляете пользователя  в политику внутренней защиты от утечки данных и определяете 30 дней в качестве окна активации **для** этого пользователя. Параметр **глобального окна активации,** который составляет 15 дней, переопределяется путем определения 30 дней для временно добавленного пользователя.  Временно добавленный пользователь останется на панели мониторинга **"Пользователи"** и будет в области действия политики в течение 30 дней.

В противоположном сценарии, где параметр глобального окна активации превышает параметр окна активации, определенный для  временно добавленного пользователя, параметр глобального окна активации переопределит параметр окна активации для временно добавленного пользователя.    Временно добавленный пользователь останется  на панели мониторинга "Пользователи" и будет в области действия политики в течение количества дней, определенного в глобальных параметрах **окна** активации.

## <a name="view-user-information-on-the-users-dashboard"></a>Просмотр сведений о пользователе на панели мониторинга "Пользователи"

Каждый пользователь,  отображаемой на панели мониторинга "Пользователи", имеет следующие сведения:

- **Пользователи**: имя пользователя. Это поле анонимизировано, если включен параметр глобальной анонимизации для управления рисками внутри организации.
- **Уровень риска**: текущий вычисляемого уровня риска пользователя. Этот показатель рассчитывается каждые 24 часа и использует оценки риска оповещений из всех активных оповещений, связанных с пользователем. Для пользователей с активными индикаторами уровень риска — ноль.
- **Активные оповещения:** количество активных оповещений для всех политик.
- **Подтвержденные нарушения**: количество дел, разрешенных как подтвержденные нарушения политики *для* пользователя.
- **Case**: текущий активный случай для пользователя.

![Панель мониторинга пользователей по управлению рисками внутри организации](../media/insider-risk-users-dashboard.png)

>[!NOTE]
>В некоторых случаях число  пользователей, отображающихся на панели мониторинга "Пользователи", может быть ограничено в зависимости от количества активных оповещений и политик совпадения. Пользователи с активными оповещениями  отображаются на панели мониторинга "Пользователи" по мере их сгенерирований, и в редких случаях достигается максимальное количество отображаемых пользователей. В этом случае пользователи с активными оповещениями, которые не  отображаются, будут добавлены на панель мониторинга "Пользователи", так как существующие оповещения пользователей будут переустанавлищены.

## <a name="view-user-details"></a>Просмотр сведений о пользователе

Чтобы просмотреть дополнительные сведения о действиях с рисками для пользователя, откройте панель сведений о пользователе, дважды щелкнув пользователя на панели мониторинга **"Пользователи".** В области сведений можно просмотреть следующие сведения:

- **Вкладка профиля** пользователя
    - **Имя и название:** имя и должность пользователя из Azure Active Directory. Эти поля пользователей будут анонимными или пустыми, если включен параметр глобальной анонимизации для управления рисками внутри организации.
    - **Электронная почта** пользователя: адрес электронной почты пользователя.
    - **Псевдоним:** сетевой псевдоним пользователя.
    - **Организация или отдел**: организация или отдел пользователя.

- **Вкладка действий** пользователя
    - **История последних действий пользователей:** перечисляет как индикаторы запуска, так и индикаторы внутренней опасности для действий пользователей за последние 180 дней. Все действия, которые имеются в индикаторах внутренней опасности, также оценки, хотя эти действия могут или не могли привести к оповещению о insider risk. Примерами запуска индикаторов могут быть дата или дата последнего запланированного действия для пользователя. Индикаторы риска внутри организации — это действия, определенные как элемент риска и определенные в политиках, в которые входит пользователь. События и действия с рисками перечислены с последним элементом, указанным первым.

## <a name="temporarily-add-a-user-to-a-policy"></a>Временное добавление пользователя в политику

Чтобы временно добавить пользователя в политику управления рисками для  участников программы,  используйте вкладку "Пользователи" в решении по управлению рисками для участников программы в Центре соответствия требованиям Microsoft 365. Пользователи, добавленные вручную, обходят триггерные требования к индикаторам для политики, в которые они добавляются и отображаются на панели мониторинга **"Пользователи".** Чтобы окончательно добавить пользователя в политику управления рисками внутри организации, используйте мастер политики.

Выполните следующие действия, чтобы добавить пользователя в существующую политику insider risk.

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для участников программы оценки и выберите вкладку **"Пользователи".** 
2. Выберите **"Добавить пользователя в политику"** на панели инструментов.
3. В **диалоговом окте "Добавление нового** пользователя" начните вводить имя пользователя в **поле "Пользователь".** Выберите пользователя, который нужно добавить в политику.
4. Выберите стрелку в поле **"Политика",** чтобы отобразить настроенные политики управления рисками внутри организации. Выберите политику для добавления пользователя.
5. Используйте **ползунок окна** активации, чтобы определить, как долго пользователь включается в политику и отображается на панели мониторинга "Пользователи". Указав время, вы определяете период активности политики для этого пользователя и запускается при генерировании первого оповещения или обнаружении индикатора запуска (например, совпадения с политикой DLP). Диапазон для окна **активации составляет** от 5 до 30 дней.
6. Выберите  "Добавить", **а** затем "Подтвердить", чтобы добавить пользователя в политику.

>[!NOTE]
>Чтобы новые пользователи, добавленные вручную, появились на панели мониторинга **"Пользователи",** может потребоваться несколько часов. Отображение действий за предыдущие 90 дней для этих пользователей может занять до 24 часов. Чтобы просмотреть действия для пользователей, добавленных  вручную,  выберите пользователя на панели мониторинга "Пользователи" и откройте вкладку "Действия пользователя" в области сведений.

## <a name="run-automated-tasks-with-power-automate-flows-for-a-user"></a>Выполнение автоматизированных задач с помощью потоков Power Automate для пользователя

Используя рекомендуемые потоки Power Automate, специалисты по рискам и аналитики могут быстро принять меры для:

- Уведомление пользователей при добавлении их в политику insider risk

Запуск, управление или создание потоков Power Automate для пользователя управления рисками внутри организации:

1. Выберите **"Автоматизировать"** на панели инструментов действий пользователя.
2. Выберите поток Power Automate для запуска, а затем выберите **"Поток запуска".**
3. После завершения потока выберите **"Готово"**.

Дополнительные информацию о потоках Power Automate для управления insider risk см. в руководстве "Начало работы с настройками управления рисками [внутри организации".](insider-risk-management-settings.md#power-automate-flows-preview)
