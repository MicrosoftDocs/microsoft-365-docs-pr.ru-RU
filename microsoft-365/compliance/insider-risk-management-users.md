# <a name="insider-risk-management-users-dashboard"></a>Панель мониторинга управления рисками пользователей изнутри

Панель **мониторинга пользователей** — это важный инструмент в работе по управлению рисками, который помогает следователям и аналитикам более полно понимать действия, связанные с рисками. Эта панель мониторинга предоставляет представления и функции управления для удовлетворения административных потребностей между созданием политики управления рисками изнутри и управление делами по управлению рисками.

После того как пользователи будут добавлены в инсайдерскую политику управления рисками, фоновые процессы автоматически оценивают действия пользователей для [запуска индикаторов.](insider-risk-management-settings.md#indicators) После запуска индикаторов пользователю назначены оценки риска. Некоторые из этих действий могут привести к оповещению о рисках, но некоторые действия могут не соответствовать минимальному уровню оценки риска, а оповещение о рисках изнутри не создается. Панель **мониторинга Пользователей** позволяет просматривать пользователей с данными типами показателей и показателей риска, а также пользователей с активными оповещений о рисках.

Дополнительные данные о том, как панель мониторинга пользователей отображает пользователей в следующих сценариях:

- Пользователи с активными оповещений о политике риска инсайдерской политики
- Пользователи с запусками событий
- Пользователи, временно добавленные в политики

## <a name="users-with-active-insider-risk-policy-alerts"></a>Пользователи с активными оповещений о политике риска инсайдерской политики

Панель **мониторинга пользователей** автоматически отображает всех пользователей с активными оповещений о политике риска. Эти пользователи с оповещениями имеют индикатор запуска и оценку риска активности, которая соответствует требованиям для создания оповещений о рисках, связанных с инсайдерской деятельностью. Действия для этих пользователей рассматриваются путем выбора пользователя на панели мониторинга **пользователей** и навигации на вкладке **Действия пользователя.**

## <a name="users-with-triggering-events"></a>Пользователи с запусками событий

Панель **мониторинга Пользователей** автоматически отображает всех пользователей с запусками событий, но у которых нет оценки риска действий, которая создаст оповещение о рисках, связанных с инсайдерской деятельностью. Например, отображается пользователь с датой отставку, так как это действие является событием-триггером, но не является действием, которое имеет оценку риска. Действия для этих пользователей рассматриваются путем выбора пользователя на панели мониторинга **пользователей** и навигации на вкладке **Действия пользователя.**

## <a name="users-added-temporarily-to-policies"></a>Пользователи, временно добавленные в политики

Панель **мониторинга пользователей** включает пользователей, добавленных в политики управления рисками инсайдеров после необычного события за пределами рабочего процесса управления рисками. Временное добавление пользователей (с панели мониторинга политики) также позволяет начать оценку активности пользователя для политики управления рисками для тестирования политики, даже если не настроен необходимый соединителю.

Когда пользователь вручную добавляется в политику, действия пользователя в течение предыдущих 90 дней засмеяются и добавляются в хронологию **действий пользователя.** Например, в настоящее время пользователю не назначены оценки риска для инсайдерской политики риска, а сведения об утечке данных сообщаются в юридический отдел организации. Юридический отдел рекомендует настроить новые требования к краткосрочному мониторингу для пользователя. Вы можете временно назначить  пользователя политике утечки данных в течение определенного времени (окно активации). Все пользователи, добавленные временно, отображаются на панели мониторинга **пользователей,** так как требования к событию запуска отменяются.

> [!NOTE]
> Может потребоваться несколько часов, чтобы новые пользователи, добавленные вручную, появились на панели **мониторинга пользователей.** Отображение действий этих пользователей за предыдущие 90 дней может занять до 24 часов. Чтобы просмотреть действия для пользователей, добавленных  вручную, выберите пользователя на панели мониторинга пользователей и откройте вкладку **Действия** пользователя на области сведений.

Пользователь автоматически удаляется из панели мониторинга **пользователей,** и при этом  время, задаваемое в окне Активация, прекращается, если:

- у пользователя нет дополнительных событий запуска или оповещений о политике риска, а также
- если продолжительность окна  активации вручную превышает продолжительность окна активации **глобальной** политики.

Параметр **Окна активации** с самой длинной продолжительностью всегда переопределяет параметр **окна** активации с более короткой продолжительностью. Например, вы настроили  окно активации  на вкладке "Глобальные таймфреймы политики" в глобальных параметрах управления рисками в течение 15 дней, которое автоматически применяется во всех ваших политиках риска, связанных с инсайдерской политикой.

Вы временно добавляете пользователя  в политику инсайдерской политики риска утечки данных и определяете 30 дней в качестве окна активации **для** этого пользователя. Параметр **глобального окна активации** в 15 дней  переопределяется путем определения параметра окна активации в 30 дней для временно добавленного пользователя. Временно добавленный пользователь останется в панели мониторинга **пользователей** и будет в поле для политики в течение 30 дней.

В противоположном сценарии, в котором параметр глобального окна активации длиннее параметра  окна активации, определенного  для временно добавленного пользователя, параметр глобального окна активации переопределит параметр окна активации для временно добавленного пользователя.   Временно добавленный пользователь останется в панели мониторинга **пользователей** и будет в поле для политики в течение нескольких дней, определенных в глобальных параметрах **окна активации.**

## <a name="view-user-information-on-the-users-dashboard"></a>Просмотр сведений о пользователях на панели мониторинга пользователей

Каждый пользователь, отображаемый на панели **мониторинга пользователей,** имеет следующие сведения:

- **Пользователи.** Имя пользователя для пользователя. Это поле анонимизировано, если включен глобальный параметр анонимизации для управления рисками изнутри.
- **Уровень риска:** текущий рассчитанный уровень риска пользователя. Эта оценка рассчитывается каждые 24 часа и использует оценки риска оповещений из всех активных оповещений, связанных с пользователем. Для пользователей с только индикаторами запуска уровень риска нулевой.
- **Активные оповещения.** Количество активных оповещений для всех политик.
- **Подтвержденные нарушения.** Количество случаев, разрешенных в качестве подтвержденных нарушений политики *для* пользователя.
- **Пример.** Текущий активный случай для пользователя.

![Панель мониторинга пользователей управления рисками изнутри](../media/insider-risk-users-dashboard.png)

> [!NOTE]
> В некоторых случаях число пользователей, отображающихся на панели мониторинга **пользователей,** может быть ограничено в зависимости от объема активных оповещений и политик, совпадающих с данными. Пользователи с активными оповещениями отображаются на панели мониторинга **пользователей** по мере их сгенерирований, и при максимальном количестве отображаемых пользователей могут быть редкие случаи. Если это ограничение произойдет, пользователи с активными оповещений, которые не отображаются, будут добавлены в панель мониторинга пользователей по мере триагности существующих оповещений пользователей. 

## <a name="view-user-details"></a>Просмотр сведений о пользователях

Чтобы просмотреть дополнительные сведения об активности риска для пользователя, откройте панель сведений пользователя, дважды щелкнув пользователя на панели **мониторинга пользователей.** На области сведений можно просмотреть следующие сведения:

- **Вкладка профилей** пользователей
  - **Имя и название.** Имя и название позиции для пользователя из Azure Active Directory. Эти поля пользователей будут анонимными или пустыми, если включена глобальная настройка анонимизации для управления рисками изнутри.
  - **Электронная почта** пользователя. Адрес электронной почты пользователя.
  - **Псевдоним:** сетевой псевдоним пользователя.
  - **Организация или отдел:** организация или отдел для пользователя.

- **Вкладка действий** пользователя
  - **История последних действий пользователей:** списки как индикаторов запуска, так и индикаторов риска для действий пользователей в течение последних 180 дней. Также засняты все действия, относящаемые к инсайдерской оценке рисков, хотя эти действия могут быть или не вызвали оповещение о рисках, связанных с инсайдерской деятельностью. Примерами запуска индикатора может быть дата отставки или последняя запланированная дата работы для пользователя. Инсайдерские индикаторы риска — это действия, определяющие элемент риска, которые определяются в политиках, в которые включен пользователь. События и действия риска перечислены в списке самых последних элементов, перечисленных в первую очередь.

## <a name="remove-users-from-in-scope-assignment-to-policies"></a>Удаление пользователей из назначения в области политик

Возможны сценарии, в которых необходимо перестать назначать оценки риска действию пользователя в политиках управления рисками. Используйте **Удаление** пользователей  на странице панели мониторинга пользователей, чтобы прекратить назначение баллов риска для одного или более пользователей из всех политик управления рисками, для которых они в настоящее время находятся в области. Это действие не удаляет пользователей из общего назначения политики (при добавлении пользователей или групп в конфигурацию политики), а просто удаляет пользователей от активной обработки политиками после текущих событий запуска. Если в будущем у пользователей будет еще одно событие-триггер, оценки риска из политик будут автоматически назначены пользователям снова. Все существующие оповещений или случаев для этого пользователя не будут удалены.

> [!NOTE]
> Удаление пользователя из политики может занять несколько минут. По завершению пользователь больше не будет перечислены на странице Пользователи. Если у удаляемого пользователя есть активные оповещения или случаи, пользователь останется на странице "Пользователи", а сведения для пользователя покажут, что он больше не является областью действия политики.

Чтобы вручную удалить пользователей из состояния в области во всех инсайдерской политике управления рисками, выполните следующие действия:

1. В [Центр соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к **управлению** рисками insider и выберите вкладку **Пользователи.**
2. На панели **мониторинга пользователей** выберите пользователя или пользователей, которых необходимо удалить из области применения политик управления рисками.
3. Выберите **Удаление пользователей**.
4. На области **Удалить пользователя** выберите  **удаление** или отмену, чтобы отменить изменения и закрыть диалоговое окно.
5. Выберите **Удалить** на области подтверждения, чтобы удалить пользователя.

## <a name="run-automated-tasks-with-power-automate-flows-for-a-user"></a>Запуск автоматизированных задач с Power Automate потоками для пользователя

Используя рекомендуемые Power Automate потоки, исследователи рисков и аналитики могут быстро принять меры к:

- Оповещать пользователей о добавлении в внутреннюю политику риска

Для запуска, управления или создания потоков Power Automate для пользователя управления рисками изнутри:

1. Выберите **Automate** на панели инструментов действий пользователя.
2. Выберите поток Power Automate для запуска, а затем выберите **поток Run**.
3. После завершения потока выберите **Готово**.

Дополнительные дополнительные Power Automate для управления рисками изнутри см. в рубрике Начало работы с настройками управления [рисками.](insider-risk-management-settings.md#power-automate-flows-preview)
