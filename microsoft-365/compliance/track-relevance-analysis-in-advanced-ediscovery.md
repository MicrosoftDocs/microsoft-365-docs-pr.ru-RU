# <a name="track-relevance-analysis-in-advanced-ediscovery"></a>Отслеживание анализа релевантности в Advanced eDiscovery
  
В Advanced eDiscovery на вкладке "Отслеживание релевантности" отображается вычисляемая действительность учебного процесса Релевантность, выполняемого на вкладке Tag, и указывается следующий шаг, который необходимо выполнить в процессе итерации обучения в Релевантность. 
  
## <a name="tracking-relevance-training-status"></a>Отслеживание состояния подготовки релевантности

1. Просмотр следующих сведений в отслеживании релевантности для проблем, как показано в следующем примере диалоговое окно **имя** проблемы ниже.

   - **Оценка.** Этот индикатор прогресса показывает, в какой степени выполненная к этому моменту подготовка релевантности достигла цели оценки с точки зрения погрешности. Также отображается богатство результатов обучения релевантности.

   - **Обучение.** Этот индикатор прогресса с кодом цвета и подсказка инструментов указывает на стабильность результатов обучения релевантности и цифровую шкалу, показывающая количество примеров подготовки релевантности, помеченных для каждой проблемы. Эксперт отслеживает ход итеративного учебного процесса релевантности. 
  
   - **Пакетный** расчет. Этот индикатор прогресса предоставляет сведения о завершении вычисления пакета.
  
   - **Следующий шаг.** Отображается рекомендация для следующего шага, который будет выполнен. 
  
    В примере показана успешно выполненная оценка проблемы, указанная индикатором выполнения цвета и задаткой. Пометка ведется, но случай по-прежнему считается неустойчивым (состояние стабильности также отображается в подсказке инструмента). Следующая рекомендация шага — "Обучение". 
  
    ![Данные об обучении на вкладке "Релевантность" > "Отслеживание" (этап 1)](../media/a00fe607-680a-48eb-9d61-4565319f7ab6.png)
  
    В расширенном представлении отображаются дополнительные сведения и параметры. Отображаемая текущая погрешность — это погрешность отзыва в текущем состоянии оценки, учитывая существующие (уже помеченные) файлы оценки.
  
    > [!NOTE]
    >  Этап оценки можно обойти путем очистки контрольного окна **оценки** за одну проблему, а затем для "всех проблем". Однако в результате статистики по этому вопросу не будет. > очистку контрольного окна **оценки** можно выполнить только до выполнения оценки. Если в деле существует несколько проблем, оценка обходится без проверки только в том случае, если для каждой проблемы будет очищено поле 
  
    Если оценка не завершена с первым набором примеров файлов, оценка может быть следующим шагом для тегов дополнительных файлов.
  
    В **"Отслеживание** релевантности" индикатор хода обучения и подсказка по инструментам указывают предполагаемое количество дополнительных образцов, необходимых для \> достижения стабильности. В этой оценке содержится руководство по дополнительному обучению, необходимому.
  
    ![Данные об обучении на вкладке "Релевантность" > "Отслеживание"](../media/98dbc3f5-5238-4d73-9f88-1aa4d77ea729.png)
  
2. Когда вы закончили теги и если вам необходимо продолжить обучение, нажмите кнопку **Обучение**. Другой пример набора файлов создается из загружаемого набора файлов для дополнительного обучения. Затем вы возвращается на вкладку Tag для тегов и подготовки дополнительных файлов.

### <a name="reaching-stable-training-levels"></a>Достижение стабильного уровня подготовки

После того, как файлы оценки достигнут стабильного уровня подготовки, Advanced eDiscovery готовы к вычислению Batch.
  
> [!NOTE]
> Обычно после трех стабильных образцов обучения следующим шагом является "Пакетный расчет". Могут быть исключения, например, при внесении изменений в теги файлов из более ранних образцов или при добавлении файлов семян. 
  
### <a name="performing-batch-calculation"></a>Выполнение вычисления пакета

Пакетный расчет выполняется по мере успешного завершения обучения на следующем этапе (когда в панели прогресса отображается стабильный учебный статус, контрольный знак и стабильный статус в подсказке инструмента).) Пакетный расчет применяет знания, полученные во время обучения релевантности для всего населения файлов, для оценки релевантности файлов и назначения баллов релевантности.
  
Если существует несколько проблем, для каждой проблемы делается пакетный расчет. Во время вычислений batch отслеживается прогресс при обработке всех файлов. 
  
Здесь рекомендуется следующий шаг "Нет", что указывает на то, что дополнительные итеративные учебные курсы релевантности не требуются на данном этапе. Следующий этап — **вкладка \> Релевантность Решите.** 
  
Если после вычисления пакета необходимо импортировать новые файлы, администратор может добавить импортируемые файлы в новую нагрузку.
  
> [!NOTE]
> Если вы **нажмете Отмена** во время вычисления пакета, процесс сохранит то, что уже выполнено. При повторном выполнении вычисления пакетов процесс будет продолжаться с последней выполненной точки. 
  
### <a name="assessing-tagging-consistency"></a>Оценка согласованности тегов

Если в тегах файлов есть несоответствия, это может повлиять на анализ. Процесс Advanced eDiscovery пометок можно использовать, если результаты не являются оптимальными или под вопросом согласованность. Возвращается список возможных непоследовательных помеченных файлов, которые при необходимости можно просмотреть и повторно просмотреть.
  
> [!NOTE]
> После семи или более учебных раундов после оценки  можно просмотреть последовательность тегов в выпуске подробных результатов обучения \>  \>  \>  \> релевантности. Этот обзор делается по одной проблеме одновременно.
  
1. В **\> отслеживании релевантности** разо расширении строки проблемы.
  
2. Справа от следующего **шага** нажмите кнопку **Изменить**.
  
3. Выберите **несоответствия тегов** в качестве **параметра Следующий** шаг после семи примеров обучения и нажмите **кнопку ОК**.
  
4. Выберите **несоответствия тегов.** Вкладка **Tag** открывает список несоответствий, которые необходимо переоценить.
  
5. Щелкните **Вычислять,** чтобы отправить изменения. Следующим шагом после пометки несоответствий является "Обучение".

### <a name="keywords"></a>Ключевые слова

Ключевое слово — это уникальная строка, слово, фраза или последовательность слов в файле, Advanced eDiscovery в качестве важного индикатора актуальности файла. В списке столбцов "Включить" ключевые слова и весы в файлах с тегами "Релевантные", а в столбцах "Исключить" перечислены ключевые слова и весы в файлах, помеченных как Не релевантные.
  
Advanced eDiscovery назначает отрицательные или положительные значения веса ключевого слова. Чем больше вес, тем выше вероятность того, что файлу, в котором появится ключевое слово, назначена более высокая оценка релевантности при вычислении пакета.
  
Список Advanced eDiscovery ключевых слов можно использовать для дополнения списка, построенного экспертом, или в качестве косвенной проверки вменяемости в любой момент процесса проверки файлов.
  
### <a name="training-progress"></a>Ход обучения

В **области "Прогресс** обучения" содержится график прогресса подготовки и отображение индикатора качества, как показано в примере ниже.
  
![Индикатор выполнения обучения на вкладке "Релевантность" > "Отслеживание"](../media/8a5089f5-a162-4246-ae09-bc1921859860.png)
  
**Индикатор качества подготовки:** отображает рейтинг согласованности тегов следующим образом:
  
- **Хорошо:** файлы помечены последовательно. (Отображается зеленый свет)
  
- **Medium.** Некоторые файлы могут быть помечены несогласованно. (Отображается желтый свет)

- **Предупреждение.** Многие файлы могут быть помечены несогласованно. (Отображается красный свет)

**График хода обучения:** показывает степень устойчивости подготовки релевантности после многих циклов подготовки релевантности по сравнению со значением F-measure. При переходе слева направо по графику интервал доверия сужается и, наряду с F-мерой, используется Advanced eDiscovery Релевантность для определения стабильности при оптимизации результатов обучения релевантности.
  
> [!NOTE]
> Релевантность использует F2, метрику F-measure, в которой recall получает в два раза больше веса, чем Точность. Для случаев с высоким уровнем насыщенности (более 25%) релевантность использует коэффициент F1 (1:1). Коэффициент F-measure можно настроить в настройках **релевантности** \> **Advanced.**
  
### <a name="batch-calculation-results"></a>Результаты вычислений пакетов

В **области результатов** пакетных вычислений содержится количество файлов, которые были забиты для релевантности: 
  
- **Success**
  
- **Empty:** Не содержит текста, например, только пробелы/вкладки
  
- **Failed:** Due to excessive size or could not be read
  
- **Игнорируется:** Из-за чрезмерного размера
  
- **Туманность:** содержит бессмысленным текстом или не содержит функций, соответствующих проблеме
  
> [!NOTE]
> Пустой, неудачный, проигнорированный или туманный получит значение релевантности -1.

![Статистика обучения на вкладке "Релевантность" > "Отслеживание"](../media/9a07740e-20d3-49fb-b9b9-84265e0a1836.png)
