# <a name="microsoft-compliance-configuration-analyzer-for-compliance-manager-preview"></a>Анализатор конфигурации соответствия требованиям Майкрософт для диспетчера соответствия требованиям (предварительный просмотр)

**В этой статье:** Узнайте, как установить и запустить средство Microsoft Compliance Configure Analyzer, чтобы быстро начать работу с microsoft Compliance Manger.

## <a name="microsoft-compliance-configuration-analyzer-mcca-preview-overview"></a>Обзор анализатора конфигурации соответствия требованиям Майкрософт (MCCA) (предварительный просмотр)

Анализатор конфигурации соответствия требованиям Майкрософт (MCCA) — это средство предварительного просмотра, которое поможет вам начать работу с [Microsoft Compliance Manager.](compliance-manager.md) MCCA — это утилита на основе PowerShell, которая будет получать текущие конфигурации организации и проверять их в отношении рекомендуемых методов Microsoft 365. Эти методы основаны на наборе элементов управления, которые включают ключевые правила и стандарты для защиты данных и управления данными.

MCCA поможет вам быстро узнать, какие действия по улучшению в Яслях соответствия требованиям применяются к текущей среде Microsoft 365. Каждое действие, которое определено MCCA, даст вам рекомендации по реализации с прямыми ссылками на диспетчер соответствия требованиям и применимым решением для начала принятия коррективных действий.

Дополнительным ресурсом для понимания MCCA является посещение инструкций [README на сайте GitHub.](https://github.com/OfficeDev/MCCA#overview) На этой странице данная страница содержит подробные сведения о необходимых предпосылках и содержит полные инструкции по установке. Для доступа к этой странице не требуется учетная запись GitHub.

**Доступность.** MCCA доступна всем организациям с лицензиями Office 365 и Microsoft 365, а также клиентам с умеренными и высокими уровнями GCC и GCC, в планах по расширению службы для клиентов DOD.

## <a name="install-mcca-and-run-a-report"></a>Установка MCCA и запуск отчета

Инструмент MCCA можно установить с помощью Windows PowerShell. После загрузки и установки средства не нужно повторять эти действия для запуска отчетов. Каждый раз, когда вы открываете MCCA, он будет спрашивать у вас учетные данные входа, и он будет генерировать новый обновленный отчет.

#### <a name="step-1-install-windows-powershell"></a>Шаг 1. Установка Windows PowerShell
Для начала вам потребуется модуль PowerShell Exchange Online (v2.0.3 или более), доступный в галерее PowerShell. [Получить инструкции по установке](https://www.powershellgallery.com/packages/ExchangeOnlineManagement/2.0.3).

#### <a name="step-2-install-mcca"></a>Шаг 2. Установка MCCA

Чтобы установить MCCA, начните с использования PowerShell в режиме администратора. Выполните следующие действия:

1. Выберите кнопку Windows **Start.**
2. Введите **PowerShell,** щелкните правой кнопкой мыши **по Windows PowerShell,** а затем выберите **Выполнить в качестве администратора**.
1. В командной строке введите следующие команды:

    ```powershell
    Install-Module -Name MCCAPreview
    ```

#### <a name="step-3-run-a-report"></a>Шаг 3. Запуск отчета

После установки MCCA можно запустить MCCA и создать отчет. Для запуска отчета:

1. Open PowerShell
2. Запустите cmdlet:

    ```powershell
    Get-MCCAReport
    ```

   Если вы клиент GCC High, вам потребуется предоставить дополнительный параметр ввода для запуска отчета:

    ```powershell
    Get-MCCAReport -ExchangeEnvironmentName O365USGovGCCHigh
    ```

3. После того как MCCA запускается, он выполняет проверку начальной версии и запросит учетные данные. На вводе подсказки имени пользователя вопишитесь в адрес электронной почты учетной записи Microsoft 365 (просмотреть роли, подходящие[для создания отчетов).](#role-based-reporting) Затем введите пароль по запросу пароля.

Затем на создание отчета уйма времени займет около 2-5 минут. Когда это будет сделано, откроется окно браузера и будет отображаться отчет ПО HTML. Каждый раз, когда вы запустите средство, он будет спрашивать учетные данные и создавать новый отчет. Этот отчет хранится локально в следующем каталоге:

C:\Users \<username> \AppData\Local\Microsoft\MCCA. 

Вы можете получить доступ к ранее созданным отчетам из этого каталога.

## <a name="understanding-your-report"></a>Понимание отчета

Отчет отражает данные, основанные на дате и времени его сгенерировании. В верхнем разделе приводится подробная информация о том, когда он был создан, имя организации и ID клиента.

#### <a name="geolocation-based-reporting"></a>Отчеты на основе геолокации

В **разделе Примечание** показано, что отчет настраивается в зависимости от географического расположения клиента. Рекомендации, перечисленные в этом средстве, будут специфи перечислены в вашей стране или регионе.

Выбор геолокации используется для оценки типов конфиденциальной информации (SITs), которые имеют отношение к этой геолокации, и создания отчета, соответствующего вашей стране или региону. Выберите геолокации на основе данных, которые имеются в клиенте.

Чтобы изменить сведения о расположении отчета, необходимо предоставить параметр ввода geolocation (-Geo). Вы можете выбрать одно или несколько геолокаций, применимых к вашему клиенту.

Следуйте инструкциям по запуску отчета на основе определенного расположения:

1. Open PowerShell
2. Чтобы указать определенный регион, вы запустите cmdlet с помощью номеров из таблицы ниже, соответствующих стране или региону. Введите несколько номеров, разделив их с запятой. Например, в смдлете ниже будет работать настраиваемый отчет для Asia-Pacific и Японии:

    ```powershell
    Get-MCCAReport -Geo @(1,7)
    ```
  | Input |  Страна | 
  | :------------- | :------------: |
  | 1  | Азиатско-Тихоокеанский регион |
  | 2  | Австралия |
  | 3  | Канада |
  | 4  | Европа (за исключением Франции) / Ближний Восток / Африка |
  | 5  | Франция |
  | 6  | Индия |
  | 7  | Япония |
  | 8  | Республика Корея |
  | 9  | Северная Америка (за исключением Канады) |
  | 10  | Южная Америка |
  | 11 | Южная Африка |
  | 12  | Швейцария |
  | 13  | Объединенные Арабские Эмираты |
  | 14  | Великобритания |


 > [!NOTE]
> В отчете всегда будут включены поддерживаемые MCCA международные типы конфиденциальной информации, такие как код SWIFT, номер кредитной карты и т. д.

#### <a name="role-based-reporting"></a>Отчеты на основе ролей

Отчет также будет настраиваться в зависимости от роли.

В таблице ниже показано, какие роли имеют доступ к разделам отчета. Другие роли в организации (не указанные в таблице ниже) могут не работать с инструментом, а также могут запускать средство и иметь ограниченный доступ к сведениям в итоговом отчете.

![MCCA — роли](../media/compliance-manager-mcca-roles.png "Роли MCCA")

Ниже перечислены исключения для этой ситуации.
1. Пользователь не сможет создавать отчет для IP, кроме раздела "Использование IRM для Exchange Online".
2. Пользователь сможет создавать отчет для IP отдельно от раздела "Использование IRM для Exchange Online".
3. Пользователь сможет создавать отчет для IP отдельно от раздела "Включить соответствие требованиям связи в O365".
4. Пользователь не сможет создавать отчеты для IP, кроме раздела "Включить аудит в Office 365".
5. Пользователь сможет создавать отчет для IP отдельно от раздела "Включить аудит в Office 365".

#### <a name="solutions-summary-section"></a>Раздел Сводка решений

В **разделе Сводка** решений в отчете представлен обзор действий по улучшению, которые ваша организация может принять в диспетчере соответствия требованиям, чтобы улучшить вашу осанку соответствия требованиям.

![MCCA — сводка решений](../media/compliance-manager-mcca-solutions.png "Сводный экран решений MCCA")

MCCA оценивает текущие конфигурации в соответствии с рекомендуемой мерой по улучшению в Диспетчере соответствия требованиям. Любое действие по улучшению, которое будет определено средством MCCA как необходимое внимание, будет перечислены в этом разделе.

Рядом с каждым решением Майкрософт находятся цветные поля с указанием количества элементов, соответствующих действиям по улучшению в диспетчере соответствия требованиям. Действия разбиты на три состояния:

- **OK:** действия, которые соответствуют рекомендуемые условия и не требуют внимания в настоящее время
- **Улучшение:** действия, которые требуют внимания
- **Рекомендация:** действия, которые не требуют внимания, но для которых мы рекомендуем рекомендации
 
Выберите поле для просмотра улучшений и рекомендаций.

**Элементы с состоянием Улучшения**

Выберите отсев рядом со меткой **Улучшение** справа от действия улучшения. Вы увидите краткие сводки и сведения о текущих параметрах и рекомендуемых действиях по улучшению. Сводка включает прямые ссылки на диспетчер соответствия требованиям, применимое решение в центре соответствия требованиям Microsoft 365 и соответствующую документацию.

Нажав на ссылку Диспетчер соответствия требованиям, вы сможете отфильтровать все действия по улучшению в этом решении, которое еще не реализовано. Оттуда можно увидеть количество баллов, которые можно достичь для увеличения показателей соответствия [требованиям,](compliance-score-calculation.md)а также применяемых ими оценок, а также применимых правил и сертификатов.

Для DLP существует кнопка **Сценарий** восстановления, которая предоставляет предварительно созданный скрипт PowerShell на основе рекомендуемых. Вы можете скопировать и вклеить его непосредственно в консоли PowerShell. Он создаст политику DLP в тестовом режиме

**Элементы со статусом Рекомендации**

Выберите отсев рядом со **меткой Рекомендации** справа от действия по улучшению. Вы увидите сводку текущей среды Microsoft 365 организации, связанную с действием по улучшению, а также рекомендуемые рекомендации.
