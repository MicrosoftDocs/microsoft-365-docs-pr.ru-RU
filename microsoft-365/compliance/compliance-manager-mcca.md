# <a name="microsoft-compliance-configuration-analyzer-for-compliance-manager"></a>Анализатор соответствия настройкам Майкрософт для диспетчера соответствия требованиям

**В этой статье:** Узнайте, как установить и запустить средство анализатора соответствия требованиям Майкрософт, чтобы быстро начать работу с диспетчером соответствия Майкрософт.

## <a name="microsoft-compliance-configuration-analyzer-mcca-overview"></a>Обзор анализатора соответствия требованиям Майкрософт (МККА)

Анализатор соответствия требованиям Майкрософт (МККА) — это средство, с помощью которого можно начать работу с [диспетчером соответствия требованиям Майкрософт](compliance-manager.md). МККА — это средство на основе PowerShell, которое выдает текущие конфигурации вашей организации и проверяет их на соответствие рекомендациям Microsoft 365, рекомендуемым. Эти рекомендации основаны на наборе элементов управления, включающих ключевые нормативные требования и стандарты для защиты данных и управления данными.

МККА поможет быстро узнать, какие действия по улучшению в диспетчере соответствия относятся к текущей среде Microsoft 365. Каждое действие, определенное с помощью МККА, предоставит рекомендации по реализации, с прямыми ссылками на диспетчер соответствия требованиям и соответствующим решением, чтобы начать делать корректирующие действия.

Дополнительный ресурс для понимания МККА заключается в [статье readme на сайте GitHub](https://github.com/OfficeDev/MCCA#overview). На этой странице представлены подробные сведения о необходимых компонентах и инструкции по полной установке. Для доступа к этой странице не требуется учетная запись GitHub.

## <a name="install-mcca-and-run-a-report"></a>Установка МККА и запуск отчета

Средство МККА можно установить с помощью Windows PowerShell. После загрузки и установки средства нет необходимости повторять эти действия для запуска отчетов. При каждом открытии МККА запрашиваются учетные данные для входа и создается новый, обновленный отчет.

#### <a name="step-1-install-windows-powershell"></a>Шаг 1: Установка Windows PowerShell
Для начала вам потребуется модуль PowerShell Exchange Online (v 2.0.3 или более поздней версии), доступный в галерее PowerShell. [Получение инструкций по установке](https://www.powershellgallery.com/packages/ExchangeOnlineManagement/2.0.3).

#### <a name="step-2-install-mcca"></a>Шаг 2: Установка МККА

Чтобы установить МККА, запустите PowerShell с помощью PowerShell в режиме администратора. Выполните указанные ниже действия.

1. Нажмите кнопку **Пуск** Windows.
2. Введите **PowerShell** , щелкните правой кнопкой мыши **Windows PowerShell** , а затем выберите пункт **Запуск от имени администратора**.
1. В командной строке введите следующие команды:

    ```powershell
    Install-Module -Name MCCAPreview
    ```

#### <a name="step-3-run-a-report"></a>Шаг 3: Запуск отчета

После установки МККА можно запустить МККА и создать отчет. Чтобы запустить отчет, выполните указанные ниже действия.

1. Открытие PowerShell
2. Выполните командлет:

    ```powershell
    Get-MCCAReport
    ```
3. После запуска МККА выполняет проверку начальной версии и запрашивает учетные данные. В поле введите запрос имени пользователя Войдите с помощью электронного адреса учетной записи Microsoft 365 ([Просмотрите роли, доступные для создания отчетов](#role-based-reporting)). Затем введите пароль в запросе пароля.

Создание отчета займет около 2-5 минут. После этого откроется окно браузера, в котором будет отображаться HTML-отчет. При каждом запуске средства будет запрашивать учетные данные и создавать новый отчет. Этот отчет хранится локально в следующем каталоге:

C:\Users \<username> \аппдата\локал\микрософт\мкка. 

Вы можете получать доступ к ранее созданным отчетам из этого каталога.

## <a name="understanding-your-report"></a>Общие сведения об отчете

Отчет отражает данные в соответствии с датой и временем его создания. В верхней части представлены сведения о процессе создания, имени Организации и идентификатора клиента.

#### <a name="geolocation-based-reporting"></a>Отчетность на основе географического расположения

В разделе **Note (Примечание** ) показано, что отчет настроен в соответствии с географическим расположением клиента. Рекомендации, приведенные в этом средстве, относятся к вашей стране или региону.

Ваш выбор географического расположения используется для оценки типов конфиденциальной информации (налево), которые относятся к этому географическому положению, и создания отчета, который соответствует вашей стране или региону. Выберите пункт географические расположения на основе данных, содержащихся в клиенте.

Чтобы изменить сведения о расположении отчета, необходимо указать входной параметр географического расположения (-GEO). Вы можете выбрать одно или несколько географов, доступных для клиента.

Выполните следующие инструкции, чтобы запустить отчет, основанный на определенном расположении:

1. Открытие PowerShell
2. Чтобы указать определенный регион, выполните командлет, используя номера из приведенной ниже таблицы, которые соответствуют стране или региону. Введите несколько чисел, разделяя их запятыми. Например, приведенный ниже командлет запустит настраиваемый отчет для Asia-Pacific и Японии:

    ```powershell
    Get-MCCAReport -Geo @(1,7)
    ```
  | Input |  Страна | 
  | :------------- | :------------: |
  | 1,1 | Азиатско-Тихоокеанский регион |
  | 2 | Австралия |
  | 4 | Канада |
  | 4  | Европа (без Франции)/Ближнего Востока и Африки |
  | 5  | Франция |
  | 6  | Индия |
  | 7  | Япония |
  | 8  | Республика Корея |
  | 9  | Северная Америка (за исключением Канады) |
  | 10  | Южная Америка |
  | 11  | Южная Африка |
  | 12  | Швейцария |
  | 13  | Объединенные Арабские Эмираты |
  | 14  | Соединенное Королевство |


 > [!NOTE]
> Отчет всегда будет включать поддерживаемые международные типы конфиденциальной информации, например код SWIFT, номер кредитной карты и т. д.

#### <a name="role-based-reporting"></a>Создание отчетов на основе ролей

Отчет также будет настраиваемым в зависимости от вашей роли.

В приведенной ниже таблице показано, какие роли имеют доступ к разделам отчета. Другие роли в Организации (не перечисленные в приведенной ниже таблице) могут не иметь функции запустить средство или запустить средство и иметь ограниченный доступ к данным в итоговом отчете.

![МККА — роли](../media/compliance-manager-mcca-roles.png "Роли МККА")

Ниже перечислены исключения для этой ситуации.
1. Пользователь не сможет создавать отчет об IP-адресах отдельно от раздела "использование IRM для Exchange Online".
2. Пользователь сможет создавать отчет об IP-адресе отдельно от раздела "использование IRM для Exchange Online".
3. Пользователь сможет создавать отчет об IP-адресе отдельно от раздела "Включение соответствия требованиям в O365".
4. Пользователь не сможет создавать отчет об IP-адресах отдельно от раздела "Включение аудита в Office 365".
5. Пользователь может формировать отчет об IP-адресах, отличных от раздела "Включение аудита в Office 365".

#### <a name="solutions-summary-section"></a>Раздел "Сводка по решениям"

В разделе " **Сводка по решениям** " этого отчета приводится обзор действий по улучшению, которые Организация может принять в диспетчере соответствия требованиям, чтобы повысить уровень соответствия требованиям.

![МККА — сводка по решениям](../media/compliance-manager-mcca-solutions.png "Экран сводки решений МККА")

МККА оценивает текущие конфигурации в соответствии с рекомендуемыми действиями по улучшению в диспетчере соответствия требованиям. В этом разделе приводится любое действие по улучшению, определенное средством МККА.

Рядом с каждым решением Майкрософт отображаются штриховые флажки, указывающие количество элементов, соответствующих действиям по улучшению в диспетчере соответствия требованиям. Действия разбиваются на три состояния:

- **ОК** : действия, которые удовлетворяют рекомендуемым условиям и не требуют внимания в настоящее время
- **Улучшение** : действия, требующие внимания
- **Рекомендация** : действия, которые не требуют внимания, но мы рекомендуем использовать рекомендации
 
Выберите поле, чтобы просмотреть улучшения и рекомендации.

**Элементы с состоянием улучшения**

Выберите раскрывающийся список рядом с меткой **улучшения** справа от действия улучшения. Вы увидите краткие сводки и подробные сведения о текущих параметрах и рекомендуемых действиях по улучшению. Сводка включает прямые ссылки в Диспетчер соответствия требованиям, применимое решение в центре соответствия требованиям Microsoft 365 и релевантную документацию.

Если щелкнуть ссылку диспетчер соответствия требованиям, вы получите отфильтрованное представление всех действий по улучшению, которые еще не реализованы в этом решении. В этой статье вы можете увидеть количество баллов, которые можно достигнуть, чтобы повысить уровень [соответствия требованиям](compliance-score-calculation.md), оценить их, а также соответствующие нормативные требования и сертификаты.

Для защиты от потери данных существует кнопка **сценария исправления** , предоставляющая предварительно созданный сценарий PowerShell в зависимости от рекомендуемых действий. Вы можете скопировать и вставить его непосредственно в консоли PowerShell. В тестовом режиме будет создана политика защиты от потери данных.

**Элементы с состоянием рекомендации**

Выберите раскрывающийся список рядом с меткой **Рекомендация** справа от действия улучшения. Вы увидите сводную информацию о текущей среде Microsoft 365, которая связана с действием по улучшению, а также рекомендуем рекомендациям.
