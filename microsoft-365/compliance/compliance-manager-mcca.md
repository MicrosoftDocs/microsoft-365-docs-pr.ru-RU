# <a name="microsoft-compliance-configuration-analyzer-for-compliance-manager-preview"></a>Анализатор конфигурации соответствия требованиям (Майкрософт) для диспетчера соответствия требованиям (предварительная версия)

**В этой статье:** Узнайте, как установить и запустить анализатор соответствия требованиям (Майкрософт), чтобы быстро начать работу с Microsoft Compliance Manger.

## <a name="microsoft-compliance-configuration-analyzer-mcca-preview-overview"></a>Обзор анализатора конфигурации соответствия (MCCA) (предварительная версия)

Анализатор конфигурации соответствия требованиям (Майкрософт) (MCCA) — это средство предварительной версии, которое поможет вам начать работу с [диспетчером соответствия требованиям Майкрософт.](compliance-manager.md) MCCA — это программа на основе PowerShell, которая извлекет текущие конфигурации вашей организации и проверяет их на основе рекомендуемых microsoft 365. Эти практические методики основаны на наборе средств контроля, которые включают ключевые нормативы и стандарты для защиты данных и управления данными.

MCCA поможет вам быстро узнать, какие действия по улучшению в Службе соответствия требованиям применяются к текущей среде Microsoft 365. Каждое действие, которое определено MCCA, дает рекомендации по реализации с прямыми ссылками на диспетчер соответствия требованиям и применимым решением для принятия коррективных мер.

Дополнительный ресурс для понимания MCCA — ознакомиться с инструкциями [README на GitHub.](https://github.com/OfficeDev/MCCA#overview) На этой странице приводится подробная информация о предварительных условиях и инструкции по полной установке. Для доступа к этой странице не требуется учетная запись GitHub.

**Доступность:** MCCA доступна всем организациям с лицензиями на Office 365 и Microsoft 365, а также клиентам среднего уровня сообщества государственных организаций США (GCC). В настоящее время планируется расширить возможности обслуживания до клиентов GCC High.

## <a name="install-mcca-and-run-a-report"></a>Установка MCCA и запуск отчета

Средство MCCA можно установить с помощью Windows PowerShell. Скачав и установив средство, вам не нужно повторять эти действия, чтобы запускать отчеты. Каждый раз, когда вы открываете MCCA, вам будут запросить учетные данные для входа, и будет создаваться новый обновленный отчет.

#### <a name="step-1-install-windows-powershell"></a>Шаг 1. Установка Windows PowerShell
Для начала вам потребуется модуль Exchange Online PowerShell (2.0.3 или выше), доступный в коллекции PowerShell. [Получите инструкции по установке.](https://www.powershellgallery.com/packages/ExchangeOnlineManagement/2.0.3)

#### <a name="step-2-install-mcca"></a>Шаг 2. Установка MCCA

Чтобы установить MCCA, начните с использования PowerShell в режиме администратора. Выполните следующие действия:

1. Выберите кнопку **"Начните"** в Windows.
2. Введите **PowerShell,** щелкните правой кнопкой **мыши** Windows PowerShell выберите "Запуск **от администратора".**
1. В командной строке введите следующие команды:

    ```powershell
    Install-Module -Name MCCAPreview
    ```

#### <a name="step-3-run-a-report"></a>Шаг 3. Запуск отчета

После установки MCCA можно запустить MCCA и создать отчет. Чтобы запустить отчет:

1. Откройте PowerShell
2. Запустите этот cmdlet:

    ```powershell
    Get-MCCAReport
    ```
3. После того как MCCA выполняется, он выполняет первоначальную проверку версии и запросит учетные данные. Во входе запроса имени пользователя во sign in with your Microsoft 365 account email address[(view the roles eligible to create reports).](#role-based-reporting) Затем введите пароль в запросе пароля.

После этого создание отчета займет около 2-5 минут. После этого откроется окно браузера и отобразит htmL-отчет. При каждом запуске средства оно будет запросить ваши учетные данные и создать новый отчет. Этот отчет хранится локально в следующем каталоге:

C:\Users \<username> \AppData\Local\Microsoft\MCCA. 

Вы можете получить доступ к ранее созданным отчетам из этого каталога.

## <a name="understanding-your-report"></a>Понимание отчета

Отчет отражает данные на основе даты и времени, в которое он был создан. В верхнем разделе приводится информация о том, когда он был создан, название организации и ИД клиента.

#### <a name="geolocation-based-reporting"></a>Отчеты на основе географического местонахождения

В **разделе** "Примечание" показано, что отчет настраивается в зависимости от географического расположения клиента. Рекомендации, перечисленные в средстве, будут перечислены в конкретной стране или регионе.

Выбор географического местонахождения используется для оценки типов конфиденциальной информации, соответствующих этому географическому местонахождению, и создания отчета, соответствующего вашей стране или региону. Выберите географическое местонахождение на основе данных, которые у вас есть в клиенте.

Чтобы изменить сведения о расположении отчета, необходимо предоставить входной параметр географического расположения (-Geo). Вы можете выбрать одно или несколько географических местонахождений, применимых к вашему арендатору.

Выполните эти инструкции, чтобы запустить отчет на основе определенного расположения:

1. Откройте PowerShell
2. Чтобы указать определенный регион, необходимо выполнить его с помощью номеров из таблицы ниже, соответствующих стране или региону. Введите несколько цифр, разделив их запятой. Например, в приведенной ниже таблице будет запускаться настраиваемый отчет для Asia-Pacific и Японии:

    ```powershell
    Get-MCCAReport -Geo @(1,7)
    ```
  | Input |  Страна | 
  | :------------- | :------------: |
  | 1  | Азиатско-Тихоокеанский регион |
  | 2  | Австралия |
  | 3  | Канада |
  | 4  | Европа (за исключением Франции), Ближний Восток и Африка |
  | 5  | Франция |
  | 6  | Индия |
  | 7  | Япония |
  | 8  | Республика Корея |
  | 9  | Северная Америка (за исключением Канады) |
  | 10  | Южная Америка |
  | 11  | Южная Африка |
  | 12  | Швейцария |
  | 13  | Объединенные Арабские Эмираты |
  | 14  | Великобритания |


 > [!NOTE]
> Отчет всегда будет включать поддерживаемые MCCA международные типы конфиденциальной информации, такие как код SWIFT, номер кредитной карты и т. д.

#### <a name="solutions-summary-section"></a>Раздел "Сводка по решениям"

В **разделе "Сводка** решений" отчета приводится обзор действий по улучшению, которые ваша организация может принять в диспетчере соответствия требованиям, чтобы повысить уровень соответствия требованиям.

![MCCA — сводка по решениям](../media/compliance-manager-mcca-solutions.png "Экран сводки по решениям MCCA")

MCCA оценивает текущие конфигурации с соответствием рекомендуемых действий по улучшению в диспетчере соответствия требованиям. Все действия по улучшению, которые средство MCCA определило как необходимые для внимания, будут перечислены в этом разделе.

Рядом с каждым решением Майкрософт находятся цветные поля, указывающие количество элементов, соответствующих действиям по улучшению в диспетчере соответствия требованиям. Действия разбиты на три состояния:

- **OK**: действия, которые соответствуют рекомендуемые условия и не требуют внимания в настоящее время
- **Улучшение**: действия, которые требуют внимания
- **Рекомендация**: действия, которые не требуют внимания, но для которых мы рекомендуем рекомендации
 
Выберите поле для просмотра улучшений и рекомендаций.

**Элементы с состоянием улучшения**

Выберите выпадание рядом с меткой **"Улучшение"** справа от действия по улучшению. Вы увидите краткую сводку и подробные сведения о текущих параметрах и рекомендуемых действиях по улучшению. Сводка содержит прямые ссылки на диспетчер соответствия требованиям, соответствующее решение в Центре соответствия требованиям Microsoft 365 и соответствующую документацию.

Щелкнув ссылку "Диспетчер соответствия требованиям", вы отфильтрованное представление всех действий по улучшению в этом решении, которые еще не реализованы. Здесь можно увидеть количество баллов, которые можно достичь для повышения оценки соответствия [требованиям,](compliance-score-calculation.md)а также применяемые ими оценки, а также применимые нормативы и сертификации.

Для DLP есть кнопка "Скрипт устранения", которая предоставляет готовый сценарий PowerShell, основанный на рекомендуемых рекомендациях.  Вы можете скопировать и вкопировать его непосредственно в консоль PowerShell. Она создаст политику DLP в тестовом режиме

**Элементы с состоянием "Рекомендации"**

Выберите выпадание рядом с меткой **"Рекомендация"** справа от действия по улучшению. Вы увидите сводку текущей среды Microsoft 365 вашей организации, связанную с действием по улучшению, а также рекомендуемые рекомендации.
