# <a name="learn-about-visibility-and-insights"></a>Сведения о видимости и аналитике

>*[Руководство по лицензированию Microsoft 365 для обеспечения безопасности и соответствия требованиям](https://aka.ms/ComplianceSD).*

Управление приложениями Майкрософт позволяет быстро получать сведения о видимости и полезную аналитику об экосистеме приложений Microsoft 365. Начните с панели мониторинга управления приложениями, которая предоставляет высокоуровневую сводку оповещений и приложений в клиенте, требующих внимания администратора.

Видимость и аналитика управления приложениями позволяет увидеть следующее:

- Список приложений с поддержкой OAuth, которые используют API Microsoft Graph для доступа к данным Microsoft 365.
- Обширное представление действий приложений, чтобы вы могли реагировать или отвечать на них.

>[!Note]
>Приложения только для Azure, которым не предоставлены разрешения на доступ к ресурсам Microsoft 365, не отображаются в управлении приложениями.
>

Просматривайте [роли администратора](app-governance-get-started.md#administrator-roles) для обзора необходимых ролей администратора для видимости и аналитики.

С помощью управления приложениями можно увидеть следующее:

- Панель мониторинга всей аналитики.
- Данные, доступ к данным осуществляется с помощью одного приложения и всех приложений с рабочей нагрузкой, и аналитика на уровне пользователя.
- Сведения о приложении и метаданные, такие как разрешения, дата регистрации и сертификация.
- Сведения об издателе и метаданные, такие как имя и состояние проверки.
- Использование основных ресурсов (сообщения электронной почты и файлы) в клиенте.
- Аналитика по следующим пунктам:

  - Приложения с высоким уровнем привилегий.
  - Приложения с избыточными правами.
  - Часто используемые приложения.
  - Давшие согласие основные пользователи с данными, к которым доступ может получить определенное приложение.
  - Приоритетные учетные записи с данными, к которым доступ может получить определенное приложение.

- Общее представление пользователей с доступом приложений.
- Аналитика оповещений.
- Аналитика списка политик.
<!-->
- Политики, созданные в MCAS на портале управления приложениями.

- Оповещения для приложений OAuth, созданных в MCAS, на портале управления приложениями.

Вы также можете:

- Выполнить детализацию до отдельного приложения (страница приложения) со всей его связанной аналитикой.
- Выполнить детализацию основных пользователей по данным и приоритетным учетным записям в пределах одного приложения.
