# <a name="limits-in-advanced-ediscovery"></a>Ограничения в Advanced eDiscovery

В этой статье описываются ограничения в решении Advanced eDiscovery в Microsoft 365.

## <a name="case-and-review-set-limits"></a>Ограничения набора для дела и проверки

В следующей таблице перечислены ограничения для дел и наборов для проверки в Advanced eDiscovery.

| Описание ограничения | Ограничение |
|:-----|:-----|
|Общее количество документов, которые можно добавить в дело (для всех наборов для проверки в случае).  <br/> |3 млн <br/> |
|Общий размер файла на набор нагрузки. Это включает загрузку office 365 в набор для проверки.  <br/> |300 ГБ <br/> |
|Общий объем данных, загруженных во все наборы для проверки в организации за день.<br/> |2 ТБ <br/> |
|Максимальное количество наборов нагрузок для каждого случая.  <br/> |200 <br/> |
|Максимальное количество наборов проверки для каждого случая.  <br/> |20 <br/> |
|Максимальное количество групп тегов для каждого случая.  <br/> |1000 <br/> |
|Максимальное количество тегов для каждого случая.  <br/> |1000 <br/> |

<br/>
<br/>

> [!NOTE]
> <sup>1.</sup> Любой элемент, превышающий ограничение на один файл, будет отозвалися как ошибка обработки.
>
> <sup>2</sup> При поиске в расположениях SharePoint и OneDrive для бизнеса символы в URL-адресах сайтов, на которых ведется поиск, учитываются в этом ограничении.
>
> <sup>3</sup> Для запросов без фраз (ключевое слово, которое не использует двойные кавычка) мы используем специальный индекс префикса. Это сообщает нам, что слово встречается в документе, но не там, где оно происходит в документе. Для выполнения запроса фразы (ключевое слово со двойными кавычками) необходимо сравнить позицию в документе для слов в фразе. Это означает, что мы не можем использовать индекс префикса для запросов фраз. В этом случае мы внутренне расширяем запрос со всеми возможными словами, до которые расширяется префикс; например, **время \* *_может развернуться* до _ "time OR timer OR times OR timex OR timex OR timeboxed OR ..."**. Ограничение в 10 000 — это максимальное число вариантов, до которые может расширяться слово, а не количество документов, совпадающих с запросом. Не существует верхнего предела для нефвационных терминов.
>
> <sup>4</sup> Это ограничение общее для всех средств eDiscovery. Это означает, что к этому ограничению применяются одновременно экспорты при поиске контента, core eDiscovery и Advanced eDiscovery.
>
> <sup>5</sup> Это ограничение применяется к загрузке выбранных документов из набора для проверки. Это не относится к экспорту документов из набора для проверки. Дополнительные сведения о загрузке и экспорте документов см. в экспорте данных дела [в Advanced eDiscovery.](exporting-data-ediscover20.md)
>
> <sup>6</sup> ограничений индексации на организацию в день. В качестве обходного решения можно выбрать несколько  хранителей на вкладке  "Источники данных" в случае, а затем нажать кнопку "Обновить индекс", чтобы избежать создания отдельного задания индекса для каждого хранителя.
