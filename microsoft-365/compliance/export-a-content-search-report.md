# <a name="export-a-content-search-report"></a>Экспорт отчета о поиске контента

Вместо экспорта полного набора результатов поиска из центра обеспечения соответствия требованиям & контента (а также из поиска контента, связанного с делом об обнаружении электронных данных), можно экспортировать те же отчеты, которые создаются при экспорте результатов поиска.
  
При экспорте отчета он загружается в папку с тем же именем, что и поиск контента, но она _ReportsOnly *.* Например, если поиск контента называется  *ContosoCase0815,* отчет загружается в папку с именем *ContosoCase0815_ReportsOnly*. Список документов, включенных в отчет, см. в документе [What's included in the report.](#whats-included-in-the-report)

## <a name="assign-roles-and-check-system-requirements"></a>Назначение ролей и требования системы проверки

- Чтобы экспортировать отчет о поиске контента, необходимо навести роль управления поиском соответствия требованиям в Центре & соответствия требованиям. Эта роль по умолчанию назначена встроенным группам управления электронными данными и группами ролей управления организацией. Дополнительные сведения см. в статье [Назначение разрешений на обнаружение электронных данных](assign-ediscovery-permissions.md).

- При экспорте отчета данные временно хранятся в уникальной служба хранилища Azure области в облаке Майкрософт перед загрузкой на локальный компьютер. Убедитесь, что ваша организация может подключиться к конечной точке Azure, которая **\* является .blob.core.windows.net** (подмывка представляет уникальный идентификатор для экспорта). Данные результатов поиска удаляются из области служба хранилища Azure через две недели после создания.

- Компьютер, используемый для экспорта результатов поиска, должен соответствовать указанным ниже требованиям к системе.

  - Последняя версия Windows (32-битная или 64-битная)

  - Microsoft .NET Framework 4.7

- Для запуска экспортного средства eDiscovery 1 необходимо использовать один из следующих поддерживаемых<sup>браузеров.</sup>

  - Microsoft Edge <sup>2</sup>

    OR

  - Microsoft Internet Explorer 10 и более поздних версиях

  > [!NOTE]
  > <sup>1</sup> Корпорация Майкрософт не производит сторонние расширения или надстройки для ClickOnce приложений. Экспорт результатов поиска с помощью неподдермеченного браузера с сторонними расширениями или надстройки не поддерживается.<br/>
  > <sup>2</sup> В результате недавних изменений Microsoft Edge ClickOnce по умолчанию поддержка больше не включена. Инструкции по включению ClickOnce поддержки в Edge см. в инструкции [Use the eDiscovery Export Tool in Microsoft Edge.](configure-edge-to-export-search-results.md)

- Если предполагаемый общий размер результатов, возвращенных поиском контента, превышает 2 ТБ, экспорт отчета не удается. Чтобы успешно экспортировать отчет, попробуйте сузить область поиска и перезавести его, чтобы предполагаемый размер результатов не превышает 2 ТБ.

- Экспорт отчетов о поиске контента учитывает максимальное количество экспортов, запущенных одновременно, и максимальное число экспортов, которые может выполнить один пользователь. Дополнительные сведения об ограничениях экспорта см. в результатах [поиска по экспорту контента.](export-search-results.md#export-limits)

## <a name="generate-and-download-a-content-search-report"></a>Создание и скачивание отчета о поиске контента

Действия по генерации и загрузке отчета поиска контента аналогичны фактически экспорту результатов поиска.
  
## <a name="step-1-generate-the-report-for-export"></a>Шаг 1. Создание отчета для экспорта

Первый шаг — подготовка отчета для загрузки на экспорт компьютера. При отправке отчета документы отчета загружаются в область служба хранилища Azure в облаке Майкрософт.
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
    
2. Выполните вход с помощью рабочей или учебной учетной записи.
    
3. В левой области Центра & безопасности нажмите кнопку **Поиск** \> **контента**.
    
4. На странице **поиска контента** выберите поиск. 
    
5. В области сведений, в **статье Экспорт отчет на компьютер,** нажмите Создать **отчет**.
    
    > [!NOTE]
    > Если результаты поиска старше 7 дней, вам будет предложено обновить результаты поиска. Если это произойдет, отмените экспорт, нажмите кнопку **Обновление** результатов поиска в области сведений для выбранного поиска, а затем снова запустите экспорт отчета после обновления результатов. 
  
6. На странице **Экспорт отчетов** в статье **Включить эти элементы** из поиска выберите один из следующих вариантов:
    
    - Экспортировать только индексированные элементы.
    
    - Экспортировать индексированные и неиндексированные элементы.
    
    - Экспортировать только неиндексированные элементы.
    
    Дополнительные сведения о неиндексуальных элементов см. в статьи Частично индексация элементов [в поиске контента.](partially-indexed-items-in-content-search.md)
    
7. Выберите, чтобы включить статистику поиска для всех версий SharePoint документов. Этот параметр отображается только в том случае, если источники контента поиска включают SharePoint или OneDrive для бизнеса сайты.
    
8. Щелкните **Создать отчет**.
    
    Отчет о результатах поиска подготовлен для скачивания, что означает, что документы отчета будут загружены в область служба хранилища Azure в облаке Майкрософт. Когда отчет будет готов к скачивке, **в** области сведений в отчете **"Экспорт"** на компьютер отображается ссылка "Скачать отчет". 
    
> [!NOTE]
> Вы также можете экспортировать отчет для поиска контента, связанного с делом об обнаружении электронных сообщений. Для этого перейдите в **eDiscovery eDiscovery,** выберите случай и нажмите \>  **кнопку Изменить значок** ![ Редактирование ](../media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif) . На странице **Поиск** выберите поиск, а затем нажмите значок **экспортировать** результаты поиска ![ ](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) \> **экспортировать отчет**. 
  
## <a name="step-2-download-the-report"></a>Шаг 2. Скачайте отчет

Следующий шаг — скачивание отчета из области служба хранилища Azure на локальный компьютер.
  
1. В области сведений для поиска, для который вы создали отчет, в статье Экспорт отчет на **компьютер,** нажмите **скачать отчет**.
    
    Отображается **страница отчета** Загрузка и содержит следующую информацию о отчете, скачаемом на компьютер. 
    
    - Количество скачиваемых элементов.
    
    - Предполагаемый общий размер скачиваемых элементов.
    
    - Сведения о том, какие элементы будут экспортированы, индексированные или неиндексированные. Элементы unindexed — это элементы, которые имеют признанный формат, шифруются или не индексироваться по другим причинам.
    
    - Будут ли загружены SharePoint документы.
    
    - Состояние процесса экспорта отчетов. Вы можете начать скачивать отчет, даже если подготовка отчета не завершена.
    
2. В разделе **Ключ экспорта** нажмите **Скопировать в буфер обмена**. Этот ключ используется в шаге 5 для загрузки отчета.
    
    > [!IMPORTANT]
    > Поскольку любой человек может установить и запустить средство экспорта электронных данных, а затем использовать этот ключ для загрузки отчета о поиске, не забудьте принять меры предосторожности для защиты этого ключа так же, как вы бы защитить пароли или другую информацию, связанную с безопасностью. 
  
3. Нажмите **кнопку Скачать отчет**.
    
4. Если вам предложено установить средство экспорта **электронных** обнаружений, нажмите кнопку **Установите**.
    
5. В **средстве экспорта службы обнаружения электронных данных** в соответствующее поле вставьте ключ экспорта, который вы скопировали в действии 2. 
    
6. Щелкните **Обзор,** чтобы указать расположение, в котором необходимо скачать отчет. 
    
7. Нажмите кнопку **Пуск**, чтобы скачать результаты поиска на свой компьютер. 
    
    **Средство экспорта службы обнаружения электронных данных** отображает сведения о состоянии процесса экспорта, в том числе о предполагаемом количестве (и размере) элементов, которые осталось скачать. По завершению процесса экспорта можно получить доступ к файлам в том месте, где они были загружены. 
    
> [!NOTE]
> Вы можете скачать отчет для поиска контента, связанного с делом об обнаружении электронных сообщений. Для этого перейдите в **eDiscovery eDiscovery,** выберите случай и нажмите \>  **кнопку Изменить значок** ![ Редактирование ](../media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif) . На странице **Экспорт выберите** экспорт отчета и нажмите кнопку **Скачать отчет** в области сведений. 
  
## <a name="whats-included-in-the-report"></a>Что включено в отчет

При генерации и экспорте отчета о результатах поиска контента загружаются следующие документы:
  
- **Сводка экспорта:** Документ Excel, содержащий сводку экспорта. К ним относятся такие сведения, как количество поисковых источников контента, количество результатов поиска из каждого расположения контента, предполагаемое количество элементов, фактическое число экспортируемого товара, а также предполагаемый и фактический размер экспортируемого товара. 
    
    > [!NOTE]
    > Если при экспорте отчета включены неидексные элементы, количество неиндексуальных элементов включается в общее число предполагаемых результатов поиска и общее число скачанных результатов поиска (если необходимо экспортировать результаты поиска), указанных в отчете Export Summary. Другими словами, общее число скачанных элементов равно общему числу оценных результатов и общему числу неиндексуальных элементов. 
  
- **Манифест:** Файл манифеста (в формате XML), содержащий сведения о каждом элементе, включенном в результаты поиска. 
    
- **Результаты:** Документ Excel, содержащий строку с информацией о каждом индексируемом элементе, который будет экспортироваться с результатами поиска. При выполнении поиска в почте журнал результатов содержит сведения о каждом сообщении, включая указанные ниже. 
    
  - Расположение сообщения в исходном почтовом ящике (включая сведения о том, в каком почтовом ящике находится сообщение, в основном или в архивном).
    
  - Дата отправки или получения сообщения.
    
  - Тема сообщения.
    
  - Отправитель и получатели сообщения.
    
    Для документов с SharePoint OneDrive для бизнеса сайтов журнал Results содержит сведения о каждом документе, включая:
    
  - URL-адрес документа.
    
  - URL-адрес семейства веб-сайтов, в котором расположен документ.
    
  - Дата последнего изменения документа.
    
  - Имя документа (которое указано в столбце "Тема" журнала результатов).
    
    > [!NOTE]
    > Количество строк в отчете **"Результаты"** должно быть равно общему числу результатов поиска за вычетом общего числа элементов, перечисленных в отчете Об элементах **Unindexed.** 
  
- **Элементы unindexed:** Документ Excel, содержащий сведения о любых неиндексуальных элементов, включенных в результаты поиска. Если при генерации отчета о результатах поиска не будут включены неидексные элементы, этот отчет будет по-прежнему загружен, но будет пустым.
