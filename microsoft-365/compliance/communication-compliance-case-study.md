# <a name="case-study---contoso-quickly-configures-an-offensive-language-policy-for-microsoft-teams-exchange-and-yammer-communications"></a>Пример. Contoso быстро настраивает оскорбительную языковую политику для Microsoft Teams, Exchange и Yammer коммуникаций

Соответствие требованиям к Microsoft 365 позволяет минимизировать риски для связи, помогая обнаруживать, захватывать и действовать в организации при ненадлежащих сообщениях. Предопределенные и настраиваемые политики дают возможность сканировать внутренние и внешние сообщения на соответствие политикам, чтобы их могли изучать специальные проверяющие. Проверяющие могут исследовать отсканированные сообщения электронной почты, Microsoft Teams, Yammer или сторонние сообщения в вашей организации и принимать соответствующие меры по исправлению, чтобы убедиться, что они соответствуют стандартам сообщений организации.

Корпорация Contoso — это вымышленная организация, которой необходимо быстро настроить политику для отслеживания оскорбительных выражений. Они используют Microsoft 365 в основном для электронной почты, Microsoft Teams и Yammer для своих пользователей, но имеют новые требования для применения политики компании в отношении домогательств на рабочем месте. ИТ-администраторы и специалисты contoso имеют базовое представление об основах работы с Microsoft 365 и ищут рекомендации по быстрому началу работы с соблюдением требований к связи.

В данном примере рассматриваются рекомендации по быстрой настройке политики соответствия требованиям к обмену данными для проверки сообщений на предмет оскорбительных выражений. Это руководство включает:

- Этап 1. Планирование соответствия требованиям к обмену данными
- Этап 2. Получение доступа к соответствию требованиям к обмену данными в Microsoft 365
- Этап 3. Настройка обязательных компонентов и создание политики соответствия требованиям к обмену данными
- Этап 4. Исследование и исправление оповещений

## <a name="step-1-planning-for-communication-compliance"></a>Шаг 1. Планирование соответствия требованиям связи

Администраторы ИТ-специалистов Contoso и специалисты по обеспечению соответствия требованиям посещали веб-вебинары в Интернете о решениях по обеспечению соответствия требованиям в Microsoft 365 и решили, что политики соответствия требованиям к коммуникациям помогут им соответствовать обновленным требованиям корпоративной политики для сокращения притеснений на рабочем месте. Совместно они разработали план по созданию и созданию политики соответствия требованиям к коммуникациям, которая будет отслеживать оскорбительные выражения для чатов, отправленных в Microsoft Teams, частных сообщений и бесед сообщества в Yammer, а также в сообщениях электронной почты, отправленных в Exchange Online. Они планируют выполнить следующие действия:

- ИТ-администраторы, которые нуждаются в доступе к функциям соответствия требованиям связи.
- Специалисты по обеспечению соответствия требованиям, которые должны создавать и управлять политиками связи.
- Специалисты по обеспечению соответствия требованиям и другие коллеги из других отделов (отделы кадров, юридические и т.д.), которые должны исследовать и устранять оповещения о соответствии требованиям.
- Пользователи, которые будут в области для коммуникации соответствия оскорбительным языковой политике.

### <a name="licensing"></a>Лицензирование

Первым шагом является подтверждение того, что лицензирование Microsoft 365 Contoso включает поддержку решения по обеспечению соответствия требованиям связи. Чтобы получить доступ и использовать соответствие требованиям связи, ИТ-администраторы Contoso должны убедиться, что у Contoso есть один из следующих ниже:

- Microsoft 365 E5 подписки (платная или пробная версия)
- Microsoft 365 E3 подписка + Соответствие требованиям Microsoft 365 E5 надстройка
- Microsoft 365 E3 + надстройка Microsoft 365 E5 insider Risk Management
- Microsoft 365 A5 подписки (платная или пробная версия)
- Microsoft 365 A3 + надстройка Microsoft 365 A5 соответствия требованиям
- Microsoft 365 A3 + надстройка Microsoft 365 A5 управления рисками
- Microsoft 365 Подписка на G5 (платная или пробная версия)
- Microsoft 365 Подписка на G5 + надстройка Microsoft 365 соответствия требованиям G5
- Microsoft 365 Подписка на G5 + Microsoft 365 надстройка управления рисками для инсайдеров G5
- Office 365 корпоративный Подписка на E5 (платная или пробная версия)
- Office 365 корпоративный Подписка на E3 + Office 365 Advanced Compliance надстройка (больше недоступна для новых подписок, см. примечание)

Они также должны подтвердить, что пользователям, включенным в политики соответствия требованиям к связи, должна быть назначена одна из вышенаказанных лицензий.

> [!IMPORTANT]
> Office 365 Advanced Compliance больше не продается в качестве независимой подписки. По истечении срока действия текущих подписок клиенты должны перейти к одной из вышеуказанных подписок, которые содержат те же или дополнительные функции соответствия требованиям.

ИТ-администраторы Contoso предпринимают следующие действия, чтобы проверить поддержку лицензирования для Contoso:

1. ИТ-администраторы войдите в **Центр администрирования Microsoft 365** <https://admin.microsoft.com> и перейдите **Центр администрирования Microsoft 365**  >  **биллинг**  >  **лицензий**.

2. Здесь они подтверждают, что у них есть один из вариантов [лицензии,](communication-compliance-configure.md#subscriptions-and-licensing) который включает поддержку соответствия требованиям связи.

![Лицензирование соответствия требованиям связи](../media/communication-compliance-case-licenses.png)

### <a name="permissions-for-communication-compliance"></a>Разрешения на соответствие требованиям связи

Существует пять групп ролей, используемых для настройки разрешений для управления функциями соответствия требованиям связи. Чтобы сделать **связь** доступной в качестве параметра меню в Центр соответствия требованиям Microsoft 365 и продолжить выполнение этих действий по настройке, администраторам Contoso назначена роль администратора соответствия требованиям *связи.*

Contoso решает использовать  группу ролей соответствия требованиям связи, назначив в группу всех администраторов соответствия требованиям связи, аналитиков, следователей и зрителей. Это упрощает для Contoso быстрое начало работы и наилучшим образом соответствует их требованиям по управлению соответствием требованиям.

|**Role**|**Разрешения роли**|
|:-----|:-----|
| **Соответствие требованиям связи** | Используйте эту группу ролей для управления соответствием требованиям к общению для организации в одной группе. Добавляя все учетные записи пользователей для назначенных администраторов, аналитиков, следователей и зрителей, можно настроить разрешения на соответствие требованиям связи в одной группе. Эта группа ролей содержит все роли разрешений на связь. Эта конфигурация является самым простым способом быстрого начала работы с соответствием требованиям к связи и подходит для организаций, которые не нуждаются в отдельных разрешениях, определенных для отдельных групп пользователей. |
| **Администратор соответствия требованиям к обмену данными** | Используйте эту группу ролей для первоначальной настройки соответствия требованиям к коммуникациям, а затем для разделения администраторов соответствия требованиям связи в определенную группу. Пользователи, назначенные этой группе ролей, могут создавать, читать, обновлять и удалять политики соответствия требованиям к связи, глобальные параметры и назначения групп ролей. Пользователи, назначенные этой группе ролей, не могут просматривать оповещения о сообщениях. |
| **Аналитик соответствия требованиям связи** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве аналитиков соответствия требованиям связи. Пользователи, назначенные этой группе ролей, могут просматривать политики, в которых они назначены в качестве рецензентов, просматривать метаданные сообщений (а не содержимое сообщений), переадрастать в дополнительные рецензенты или отправлять уведомления пользователям. Аналитики не могут разрешить ожидающих оповещений. |
| **Следователь соответствия требованиям связи** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве исследователей соответствия требованиям связи. Пользователи, назначенные этой группе ролей, могут просматривать метаданные и содержимое сообщений, перенаправляться в дополнительные рецензенты, перерастать в Advanced eDiscovery, отправлять уведомления пользователям и устранять предупреждение. |
| **Зритель соответствия требованиям к обмену данными** | Используйте эту группу для назначения разрешений пользователям, которые будут управлять отчетами о связи. Пользователи, заметив эту группу ролей, могут получать доступ ко всем виджетам отчетности на домашней странице соответствия требованиям связи и просматривать все отчеты о соответствии требованиям связи. |

1. ИТ-администраторы Contoso впишутся на страницу **разрешений** центра Office 365 безопасности & соответствия [требованиям https://protection.office.com/permissions) (с](https://protection.office.com/permissions) помощью учетных данных для глобальной учетной записи администратора и выберите ссылку для просмотра и управления ролями в Microsoft 365.
2. В Центре **&** безопасности они перейдите к **разрешениям** и выберите ссылку для просмотра ролей в Office 365.
3. Администраторы выбирают группу *ролей соответствия* требованиям связи, а затем **группу изменить роль.**
4. Администраторы выбирают **выбрать членов из** левой области навигации, а затем изменить . 
5. Они выбирают **Добавить** и затем выбрать почтовый ящик для всех пользователей Contoso, которые будут управлять соответствием требованиям связи, исследовать и проверять оповещения.
6. Администраторы выбирают **Добавить,** а затем выбрать **Готово**.
7. Они выбирают **Сохранить,** чтобы добавить пользователей Contoso в группу ролей. Они выбирают **Close** для выполнения действий.

## <a name="step-2-accessing-communication-compliance-in-microsoft-365"></a>Шаг 2. Доступ к требованиям для связи в Microsoft 365

После настройки разрешений на соответствие требованиям связи ИТ-администраторы Contoso и специалисты по обеспечению соответствия требованиям, которые назначены группе ролей соответствия требованиям связи, могут получить доступ к решению по обеспечению соответствия Microsoft 365. Администраторы ИТ-специалистов Contoso и специалисты по соблюдению требований имеют несколько способов получения доступа к требованиям к связи и начать создание новой политики:

- Начиная непосредственно с решения по обеспечению соответствия требованиям связи
- Начиная с Центр соответствия требованиям Microsoft 365
- Начиная с каталога Microsoft 365 решений
- Начиная с Центр администрирования Microsoft 365

С помощью этой ссылки ИТ-администраторы и специалисты по обеспечению соответствия требованиям Contoso будут направлены на панель мониторинга соответствия требованиям связи, где можно быстро просмотреть состояние оповещений и создать новые политики из заранее определенных шаблонов.

## <a name="step-3-configuring-prerequisites-and-creating-a-communication-compliance-policy"></a>Шаг 3. Настройка необходимых условий и создание политики соответствия требованиям к коммуникациям

Чтобы начать работу с политикой соответствия требованиям к обмену данными и приступить к настройке новой политики для отслеживания оскорбительных выражений, ИТ-администраторам Contoso необходимо настроить несколько обязательных компонентов. После настройки этих обязательных компонентов ИТ-администраторы и специалисты по соответствию требованиям Contoso могут настроить новую политику, а специалисты по соответствию требованиям могут начать исследование и исправление создаваемых оповещений.

### <a name="enabling-auditing-in-microsoft-365"></a>Включение аудита в Microsoft 365

Соответствие требованиям к обмену данными требует, чтобы журналы аудита отображали оповещения и отслеживали действия по исправлению, предпринимаемые проверяющими. Журналы аудита представляют собой сводку всех действий, связанных с определенной политикой организации, или любых изменений, вносимых в политику соответствия требованиям к обмену данными.

Чтобы включить аудит, ИТ-администраторам Contoso необходимо изучить и выполнить [пошаговые инструкции](turn-audit-log-search-on-or-off.md). После включения аудита появится сообщение о том, что журнал аудита готовится и что через несколько часов после завершения подготовки можно запустить поиск. ИТ-администраторам Contoso необходимо выполнить это действие только один раз.

### <a name="configuring-yammer-tenant-for-native-mode"></a>Настройка Yammer для родного режима

Соответствие требованиям связи требует, чтобы клиент Yammer организации был в режиме native, чтобы отслеживать оскорбительные выражения в частных сообщениях и публичных беседах сообщества.

ИТ-администраторы Contoso проверяют сведения в обзоре Yammer Native Mode в [статье Microsoft 365](/yammer/configure-your-yammer-network/overview-native-mode) и следят за действиями по запуску средства миграции в статье Настройка сети Yammer для родного режима для [Microsoft 365](/yammer/configure-your-yammer-network/native-mode) статьи.

### <a name="setting-up-a-group-for-in-scope-users"></a>Настройка группы для пользователей в области

Специалисты по комплаенсу Contoso хотят добавить всех пользователей в политику связи, которая будет отслеживать оскорбительные выражения. Они могли бы добавить каждую учетную запись пользователя в политику отдельно, но решили, что  это намного проще и экономит время на использование группы рассылки "Все пользователи" для пользователей для этой политики.

Им необходимо создать новую группу, включаемую всех пользователей Contoso, чтобы они могли предпринять следующие действия:

1. ИТ-администраторы Contoso войдите в Центр администрирования Microsoft 365  [(и https://admin.microsoft.com)](https://admin.microsoft.com) перейдите **Центр администрирования Microsoft 365**  >    >  **группы** групп.
2. Они выбирают **Добавить группу** и завершить мастер, чтобы создать новую группу Microsoft 365 *группу* или *группу рассылки.*

    ![Группы](../media/communication-compliance-case-all-employees.png)

3. После создания новой группы необходимо добавить всех пользователей Contoso в эту группу. Они открывают центр **администрирования Exchange** [ https://outlook.office365.com/ecp) (и](https://outlook.office365.com/ecp) переходят Exchange **группы** получателей центра  >    >  **администрирования.** ИТ-администраторы Contoso выбирают область членства и созданную ими группу *All Employees* и выберите управление **Редактирование,** чтобы добавить всех пользователей Contoso в новую группу мастера.

    ![Центр администрирования Exchange](../media/communication-compliance-case-eac.png)

### <a name="creating-the-policy-to-monitor-for-offensive-language"></a>Создание политики для отслеживания оскорбительных выражений

После настройки всех обязательных компонентов ИТ-администраторы и специалисты по соответствию требованиям Contoso могут настроить политику соответствия требованиям к обмену данными для отслеживания оскорбительных выражений. Новый шаблон политики оскорбительных выражений позволяет настроить эту политику легко и быстро.

1. ИТ-администраторам и специалистам по соответствию требованиям Contoso необходимо войти в **Центр соответствия требованиям Microsoft 365** и выбрать **Соответствие требованиям к обмену данными** в области навигации слева. Это действие откроет **обзорную** панель мониторинга, содержащую быстрые ссылки на шаблоны политики соответствия требованиям к обмену данными. Чтобы выбрать шаблон **Отслеживание оскорбительных выражений**, необходимо нажать кнопку **Начать работу**.

    ![Шаблон наступательного языка соответствия требованиям связи](../media/communication-compliance-case-template.png)

2. В мастере шаблонов политик ИТ-администраторы и специалисты по соответствию требованиям Contoso должны работать вместе, чтобы заполнить три обязательных поля: **Имя политики**, **Контролируемые пользователи или группы** и **Проверяющие**.
3. ИТ-администраторы и специалисты по соответствию требованиям могут сохранить предложенное мастером политик имя и сосредоточиться на оставшихся полях. Они выбирают группу *Все* пользователи для пользователей или групп для контроля поля и выбирают специалистов по соблюдению требований, которые должны исследовать и устранять оповещения политик для поля **Рецензенты.**  Последний шаг для настройки политики и начала сбора сведений об оповещении — это выбор **политики Create.**

    ![Мастер языка соответствия требованиям связи](../media/communication-compliance-case-wizard.png)

## <a name="step-4-investigate-and-remediate-alerts"></a>Шаг 4. Изучение и исправление оповещений

После настройки политики соответствия требованиям к обмену данными для отслеживания оскорбительных выражений специалисты по соответствию требованиям Contoso могут приступить к исследованию и исправлению оповещений, создаваемых политикой. Чтобы политика полностью обработала сообщения во всех каналах источника коммуникации и чтобы оповещения отобразились на **Панели мониторинга оповещений**, потребуется до 24 часов.

После сгенерации оповещений специалисты [](communication-compliance-investigate-remediate.md) по комплаенсу Contoso будут следовать инструкциям рабочего процесса по расследованию и исправлению проблем с оскорбительным языком.
