# <a name="collect-ediscovery-diagnostic-information"></a>Сбор диагностических сведений об обнаружении электронных данных

Иногда инженерам службы поддержки Майкрософт требуется определенная информация о вашей проблеме при открытии дела поддержки, связанного с core eDiscovery или Advanced eDiscovery. В этой статье содержится руководство по сбору диагностических сведений, которые помогут инженерам службы поддержки изучить и устранить проблемы. Как правило, вам не нужно собирать эти сведения, пока не будет предложено сделать это от инженера службы поддержки Майкрософт.

> [!IMPORTANT]
> Выходные данные, которые содержатся в этих данных, могут включать конфиденциальную информацию о судебных разбирательствах или внутренних расследованиях в организации. Перед отправкой необработанных диагностических сведений в службу поддержки Майкрософт необходимо просмотреть эти сведения и отредакциовать любую конфиденциальную информацию (например, имена или другую информацию о лицах для судебного разбирательства или расследования), заменив их на `XXXXXXX` . Использование этого метода также указывает инженеру службы поддержки Майкрософт, что сведения были отредактировали.

## <a name="collect-diagnostic-information-for-core-ediscovery"></a>Сбор диагностических сведений для core eDiscovery

Сбор диагностической информации для Core eDiscovery основан на использовании & PowerShell в Центре безопасности и соответствия требованиям. В следующих примерах PowerShell будут запускаться и сохраняться выходные данные в указанном текстовом файле. В большинстве случаев необходимо выполнить только одну из этих команд.

Чтобы выполнить следующие cmdlets, подключите [к PowerShell </span> ](https://docs.microsoft.com/powershell/exchange/connect-to-scc-powershell)Центра & безопасности. После подключения запустите одну или несколько из следующих команд и замените их фактическими именами объектов.

После просмотра сгенерированного текстового файла и отредактки конфиденциальных сведений отправьте его инженеру службы поддержки Майкрософт, который работает над вашим делом.

> [!NOTE]
> Вы также можете выполнить команды, указанные в этом разделе, для  сбора диагностических сведений об поисковых запросах и экспорте, перечисленных на странице "Поиск контента" в Центре соответствия требованиям Microsoft 365.

### <a name="collect-information-about-searches"></a>Сбор сведений о поиске

Следующая команда собирает сведения, полезные при расследовании проблем с поиском контента или поиском, связанным с основным делом eDiscovery.

```powershell
Get-ComplianceSearch "<Search name>" | FL > "ComplianceSearch.txt"
```

### <a name="collect-information-about-search-actions"></a>Сбор сведений о действиях поиска

Следующая команда собирает сведения, чтобы исследовать проблемы с предварительным просмотром, экспортом или сгружение результатов поиска контента или поиска, связанного с основным делом eDiscovery. Вы можете определить имя действия поиска, щелкнув экспорт, указанный на вкладке **"Экспорт".** Чтобы определить имена действий предварительного просмотра и очистки, вы можете выполнить для отображения списка всех действий с его целью выполнить выполнение слета **Get-ComplianceSearchAction.** Формат имени действия поиска составляется с помощью приложения или имени `_Preview` `_Export` `_Purge` соответствующего поиска.

```powershell
Get-ComplianceSearchAction "<Search action name>" | FL > "ComplianceSearchAction.txt"
```

### <a name="collect-information-about-ediscovery-holds"></a>Сбор сведений об обнаружении электронных данных

Если удержание eDiscovery, связанное с основным делом eDiscovery, не работает должным образом, запустите следующую команду, чтобы собрать сведения о политике удержания дела и связанном с ним правиле удержания дела для удержания eDiscovery. Имя *политики удержания дела* в следующей команде такое же, как имя удержания eDiscovery. Это имя можно идентифицировать на вкладке **"Содержит"** в случае core eDiscovery.

```powershell
Get-CaseHoldPolicy "<Case hold policy name>" | %{"--CaseHoldPolicy--";$_|FL;"--CaseHoldRule--";Get-CaseHoldRule -Policy $_.Name | FL} > "eDiscoveryCaseHold.txt"
```

### <a name="collect-all-case-information"></a>Сбор всех сведений о делах

Иногда не очевидно, какие сведения требуются службе поддержки Майкрософт для изучения вашей проблемы. В этой ситуации можно собрать все диагностические сведения для основного дела eDiscovery. Имя основного дела *eDiscovery* в следующей команде такое же, как имя дела, которое отображается на странице Core **eDiscovery** в Центре соответствия требованиям Microsoft 365.

```powershell
Get-ComplianceCase "<Core eDiscovery case name>"| %{"$($_.Name)";"`t==Searches==";Get-ComplianceSearch -Case $_.Name | FL;"`t==Search Actions==";Get-ComplianceSearchAction -Case $_.Name |FL;"`t==Holds==";Get-CaseHoldPolicy -Case $_.Name | %{$_|FL;"`t`t ==$($_.Name) Rules==";Get-CaseHoldRule -Policy $_.Name | FL}} > "eDiscoveryCase.txt"
```

## <a name="collect-diagnostic-information-for-advanced-ediscovery"></a>Сбор диагностических сведений для Advanced eDiscovery

Вкладка **"Параметры"** в случае Advanced eDiscovery позволяет быстро скопировать диагностические сведения для дела. Диагностические сведения сохраняются в буфер обмена, чтобы вы могли в paste it to a text file and send to Microsoft Support.

1. Go to [https://compliance.microsoft.com](https://compliance.microsoft.com/) and then click Show all > **eDiscovery > Advanced**.

2. Выберите дело и щелкните вкладку **"Параметры".**

3. В **области "Сведения о деле"** нажмите **кнопку "Выбрать".**

4. На странице "Flyout" щелкните **"Копировать диагностические сведения",** чтобы скопировать эти сведения в буфер обмена.

5. Откройте текстовый файл (в Блокноте) и в paste the information in the text file.

6. Сохраните текстовый файл и назовите его как-то `AeD Diagnostic Info YYYY.MM.DD` вроде (например, `AeD Diagnostic Info 2020.11.03` ).

После просмотра файла и отредактки конфиденциальных сведений отправьте его инженеру службы поддержки Майкрософт, который работает над вашим делом.
