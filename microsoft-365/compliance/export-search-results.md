# <a name="export-content-search-results"></a>Экспорт результатов поиска контента

После успешного запуска поиска контента можно экспортировать результаты поиска на локальный компьютер. При экспорте результатов для почты они будут скачаны на ваш компьютер в виде PST-файлов. При экспорте контента с сайтов SharePoint и OneDrive для бизнеса экспортируются копии документов Office. Существуют другие документы и отчеты, включенные в экспортные результаты поиска.
  
Экспорт результатов поиска контента включает подготовку результатов и их загрузку на локальный компьютер.
  
## <a name="before-you-export-content-search-results"></a>Перед экспортом результатов поиска контента

- Чтобы экспортировать результаты поиска, вам должна быть назначена роль управления экспортом в Центре безопасности & соответствия требованиям. Эта роль назначается встроенной группе ролей "Руководитель службы обнаружения электронных данных". По умолчанию эта роль не назначена группе ролей "Управление организацией". Дополнительные сведения см. в статье [Назначение разрешений на обнаружение электронных данных](assign-ediscovery-permissions.md).

- Компьютер, используемый для экспорта результатов поиска, должен соответствовать указанным ниже требованиям к системе.
  
  - 32- или 64-битные версии Windows 7 и более поздних версий
  
  - Microsoft .NET Framework 4.7
  
- Для запуска средства экспорта eDiscovery 1 необходимо использовать один из следующих поддерживаемых<sup>браузеров:</sup>

  - Microsoft Edge <sup>2</sup>
  
    OR

  - Microsoft Internet Explorer 10 и более поздних версий
  
  > [!NOTE]
  > <sup>1</sup> Корпорация Майкрософт не выпускает сторонние расширения или надстройки для ClickOnce приложений. Экспорт результатов поиска с помощью неподдергодного браузера со сторонними расширениями или надстройки не поддерживается.<br/>
  > <sup>2.</sup> В результате последних изменений в Microsoft Edge поддержка ClickOnce больше не включена по умолчанию. Инструкции по включению поддержки ClickOnce в Edge см. в средстве экспорта [eDiscovery в Microsoft Edge.](configure-edge-to-export-search-results.md)
  
- Рекомендуется скачивать результаты поиска на локальный компьютер. Однако чтобы исключить проблемы с брандмауэром или прокси-инфраструктурой вашей компании при загрузке результатов поиска, можно скачать результаты поиска на виртуальный рабочий стол за пределами сети. Это может уменьшить время, которое происходит в подключениях к данным Azure при экспорте большого количества файлов. Дополнительные сведения о виртуальных рабочих столах см. в [этой теме.](https://azure.microsoft.com/services/virtual-desktop) 

- Чтобы повысить производительность при загрузке результатов поиска, рассмотрите возможность деления поиска, возвращающих большой набор результатов, на более мелкие. Например, вы можете использовать диапазоны дат в поисковых запросах, чтобы получить меньший набор результатов, который можно скачать быстрее.
  
- При экспорте результатов поиска данные временно хранятся в предоставленном Майкрософт хранилище Azure в Облаке Майкрософт перед их загрузкой на локальный компьютер. Убедитесь, что ваша организация может подключиться к конечной точке в **\* Azure,blob.core.windows.net** (поддиапная знак представляет уникальный идентификатор для экспорта). Данные результатов поиска удаляются из хранилища Azure через две недели после их создания. 
  
- Если в организации для связи с Интернетом используется прокси-сервер, необходимо определить параметры прокси-сервера на компьютере, который используется для экспорта результатов поиска (чтобы средство экспорта можно было проверить на прокси-сервере). Для этого откройте  *файл* machine.configв расположении, которое соответствует вашей версии Windows. 
  
  - **32-битная:**`%windir%\Microsoft.NET\Framework\[version]\Config\machine.config`
  
  - **64-битная:**`%windir%\Microsoft.NET\Framework64\[version]\Config\machine.config`
  
    Добавьте следующие строки в  *machine.config*  между  `<configuration>`  `</configuration>` тегами и тегами. Не забудьте заменить  `ProxyServer`  `Port` и на правильные значения для вашей организации, например  `proxy01.contoso.com:80` . 
  
    ```xml
    <system.net>
       <defaultProxy enabled="true" useDefaultCredentials="true">
         <proxy proxyaddress="https://ProxyServer :Port " 
                usesystemdefault="False" 
                bypassonlocal="True" 
                autoDetect="False" />
       </defaultProxy>
    </system.net>
    ```

## <a name="step-1-prepare-search-results-for-export"></a>Шаг 1. Подготовка результатов поиска к экспорту

Прежде всего необходимо подготовить результаты поиска для экспорта. При подготовке результатов они загружаются в предоставленное Корпорацией Майкрософт хранилище Azure в Облаке Майкрософт. Содержимое из почтовых ящиков и сайтов загружается с максимальной скоростью 2 ГБ в час.
  
1. Перейдите по ссылке [https://protection.office.com](https://protection.office.com).
  
2. Выполните вход с помощью рабочей или учебной учетной записи.
  
3. В левой области Центра безопасности и & соответствия требованиям щелкните **"Поиск** \> **контента".**
  
4. На странице **"Поиск контента"** выберите поиск. 
  
5. В области сведений в разделе **Экспорт результатов на компьютер** щелкните **Начать экспорт**.
  
    > [!NOTE]
    > Если результаты поиска сформированы более 7 дней назад, вам будет предложено обновить их. В этом случае отмените экспорт, в области сведений щелкните **Обновить результаты поиска** для выбранного поиска. После обновления результатов еще раз запустите экспорт.  
  
6. На странице **"Экспорт результатов поиска"** в области **"Параметры вывода"** выберите один из следующих параметров:
  
    - Все элементы, за исключением тех, которые имеют неоформатизированный формат, шифруются или не индексируются по другим причинам.
  
    - Все элементы, в том числе неударованные в формате, шифруются или не индексируются по другим причинам.
  
    - Только элементы, которые имеют неизвестный формат, зашифрованы или не индексированы по другим причинам
  
    Описание [того,](#more-information) как экспортируются частично индексные элементы, см. в разделе "Дополнительные сведения". Дополнительные сведения о частично индексных элементов см. в части индексных элементов [в поиске контента.](partially-indexed-items-in-content-search.md)
  
7. В **области "Экспорт содержимого Exchange как"** выберите один из следующих параметров:
  
    - **Один PST-файл для каждого почтового ящика:** Экспортирует один PST-файл для каждого почтового ящика пользователя, содержащем результаты поиска. Все результаты из архивного почтового ящика пользователя включаются в один PST-файл. Этот параметр воспроизводит структуру папок почтового ящика из исходных почтовых ящиков.
  
    - **Один PST-файл, содержащий все сообщения:** Экспортирует один PST-файл (с именем *Exchange.pst),* содержащий результаты поиска из всех исходных почтовых ящиков, включенных в поиск. Этот параметр воспроизводит структуру папок почтовых ящиков для каждого сообщения.
  
    - **Один PST-файл, содержащий все сообщения в одной папке:** Экспортирует результаты поиска в один PST-файл, в котором все сообщения находятся в одной папке верхнего уровня. Этот параметр позволяет рецензентам просмотреть элементы в хронологическом порядке (элементы сортироваться по дате передачи), не переходя по исходной структуре папок почтовых ящиков для каждого элемента.
  
    - **Отдельные сообщения:** Экспортирует результаты поиска в виде отдельных сообщений электронной почты в формате MSG. При выборе этого параметра результаты поиска по электронной почте экспортируются в папку в файловой системе. Путь к папке для отдельных сообщений такой же, как при экспорте результатов в PST-файлы.
  
      > [!IMPORTANT]
      > Чтобы расшифровать защищенные RMS сообщения при их экспорте, необходимо экспортировать результаты поиска по электронной почте как отдельные сообщения. Зашифрованные сообщения останутся зашифрованными при экспорте результатов поиска в виде PST-файла. Дополнительные сведения см. в этой статье в статье о расшифровке сообщений электронной почты, защищенных с помощью [RMS,](#decrypting-rms-protected-email-messages-and-encrypted-file-attachments) и зашифрованных вложений.
  
8. Чтобы **исключить повторяющиеся** сообщения, нажмите кнопку "Включить дублирование". Этот параметр отображается, только если источниками контента поиска являются почтовые ящики Exchange или общедоступные папки. 
  
    При выборе этого параметра будет экспортирована только одна копия сообщения, даже если в почтовых ящиках, в которые был отправлен поиск, найдено несколько копий одного сообщения. Отчет о результатах экспорта (Results.csv) будет содержать строку для каждой копии дубликата сообщения, чтобы можно было определить почтовые ящики (или общедоступные папки), содержащие копию дубликата сообщения. Дополнительные сведения о дублировании и обнаружении дублирующихся элементов см. в документе [De-duplication in eDiscovery search results.](de-duplication-in-ediscovery-search-results.md)
  
9. Чтобы экспортировать все версии документов **SharePoint,** щелкните "Включить версии для документов SharePoint". Этот параметр отображается, только если источниками контента поиска являются сайты SharePoint или OneDrive для бизнеса. 
  
10. Щелкните **"Экспорт файлов" в сжатом (сжатом) ящике** папки, чтобы экспортировать результаты поиска в сжатые папки. Этот параметр доступен, только если вы решили экспортировать элементы Exchange в качестве отдельных сообщений, а результаты поиска включают документы SharePoint или OneDrive. Этот параметр в основном используется для работы с ограничением в 260 символов в именах путей к файлам Windows при экспорте элементов. См. раздел "Имена файлов экспортных элементов" в [разделе "Дополнительные](#more-information) сведения". 
  
11. Нажмите кнопку **Начать экспорт**. Результаты поиска готовы к загрузке, то есть они загружаются в хранилище Azure в Облаке Майкрософт. Это может занять несколько минут.

Инструкции по загрузке экспортных результатов поиска см. в следующем разделе.
  
## <a name="step-2-download-the-search-results"></a>Шаг 2. Скачивание результатов поиска

Следующим шагом является загрузка результатов поиска из хранилища Azure на локальный компьютер.
  
1. На странице **"Поиск контента"** перейдите на вкладку **"Экспорт".** 
  
   Может потребоваться  нажать кнопку "Обновить", чтобы обновить список заданий экспорта, чтобы в нем отображалось созданное задание экспорта. Задания экспорта имеют то же имя, что и соответствующий поиск, **_Export** к имени поиска.
  
2. Выберите задание экспорта, созданное на шаге 1.

3. На странице с элементом "Экспорт" нажмите кнопку **"Копировать в буфер обмена".** Этот ключ используется в шаге 6 для скачивания результатов поиска.
  
4. Нажмите кнопку **Скачать результаты**.

5. Если вам будет предложено установить средство экспорта **для eDiscovery,** нажмите кнопку **"Установить".**

6. В **средстве экспорта для eDiscovery** сделайте следующее:

   ![Средство экспорта eDiscovery](../media/eDiscoveryExportTool.png)

   1. В соответствующее поле в paste the export key that you copied in step 3.
  
   2. Нажмите кнопку **Обзор** и укажите расположение, в которое вы хотите скачать файлы результатов поиска.
  
      > [!IMPORTANT]
      >  Из-за высокой активности сети во время скачивания результаты поиска следует загружать только в расположение на внутреннем диске на локальном компьютере. Для наилучшего скачивания следуйте следующим рекомендациям: <br/>
      >- Не скачайте результаты поиска на UNC-путь, сетевой диск, внешний USB-накопитель или синхронизированную учетную запись OneDrive для бизнеса.<br/>
      >- Отключите антивирусную проверку для папки, в которую вы скачиваете результат поиска.<br/>
      >- Скачайте результаты поиска в разные папки для заданий загрузки одновременно.

6. Нажмите кнопку **Пуск**, чтобы скачать результаты поиска на свой компьютер.
  
    **Средство экспорта службы обнаружения электронных данных** отображает сведения о состоянии процесса экспорта, в том числе о предполагаемом количестве (и размере) элементов, которые осталось скачать. После завершения экспорта вы сможете получить доступ к файлам в расположении, где они были загружены.

### <a name="export-reports"></a>Экспорт отчетов
  
- При экспорте результатов поиска помимо результатов поиска включаются следующие отчеты.
  
  - **Сводка по экспорту** Документ Excel, содержащий сводку по экспорту. К ним относятся такие сведения, как количество источников контента, в которые был включен поиск, предполагаемые и загруженные размеры результатов поиска, а также предполагаемое и загруженное количество экспортируемого элементов.
  
  - **Манифест** Файл манифеста (в формате XML), содержащий сведения о каждом элементе, включенном в результаты поиска.
  
  - **Результаты** Документ Excel, содержащий сведения о каждом элементе, загружаемом в качестве результата поиска. При выполнении поиска в почте журнал результатов содержит сведения о каждом сообщении, включая указанные ниже.
  
    - Расположение сообщения в исходном почтовом ящике (включая сведения о том, в каком почтовом ящике находится сообщение, в основном или в архивном).
  
    - Дата отправки или получения сообщения.

    - Тема сообщения.

    - Отправитель и получатели сообщения.

    - Является ли сообщение дублирующимся сообщением, если при экспорте результатов поиска был включен параметр де дубликата. Дублирующиеся сообщения имеют  значение в столбце "Дубликат элемента", которое идентифицирует сообщение как дублирующееся. Значение в столбце **"Дубликат** элемента" содержит идентификатор элемента экспортируемого сообщения. Дополнительные сведения см. в [поддиаптеризации](de-duplication-in-ediscovery-search-results.md)результатов поиска при обнаружении электронных данных.

      Для документов с сайтов SharePoint и OneDrive для бизнеса журнал результатов содержит сведения о каждом документе, в том числе:

      - URL-адрес документа.

      - URL-адрес семейства веб-сайтов, в котором расположен документ.

      - Дата последнего изменения документа.

      - Имя документа (которое указано в столбце "Тема" журнала результатов).

  - **Неndexed Items** Документ Excel, содержащий сведения о частично индексных элементов, которые будут включены в результаты поиска. Если при генерации отчета о результатах поиска не будут включены частично индексные элементы, этот отчет будет по-прежнему загружен, но будет пустым.

  - **Ошибки и предупреждения** Содержит ошибки и предупреждения для файлов, которые произошли во время экспорта. Сведения о каждой отдельной ошибке или предупреждении см. в столбце "Сведения об ошибке".

  - **Пропущенные элементы** При экспорте результатов поиска с сайтов SharePoint и OneDrive для бизнеса экспорт обычно включает отчет о пропущенных SkippedItems.csv. Элементы, приведенные в этом отчете, обычно не загружаются, например папка или набор документов. Не экспортировать эти типы элементов по проекту. Для других пропущенных элементов в поле "Тип ошибки" и "Сведения об ошибке" в отчете о пропущенных элементах покажите причину, по которой элемент был пропущен и не загружен вместе с другими результатами поиска.

  - **Журнал трассировки** Содержит подробные сведения о процессе экспорта и помогает выявить проблемы во время экспорта.
  
    > [!NOTE]
    > Вы можете просто экспортировать эти документы, не экспортировать фактические результаты поиска. См. [отчет "Экспорт отчета о поиске контента".](export-a-content-search-report.md) 
  
### <a name="exporting-partially-indexed-items"></a>Экспорт частично индексных элементов
  
- При экспорте элементов почтового ящика из поиска контента, который возвращает все элементы почтового ящика в результатах поиска (так как ключевые слова не включены в поисковый запрос), частично индексированные элементы не будут скопироваться в PST-файл, содержащий неиндексированные элементы. Это происходит потому, что все элементы, включая частично индексные элементы, автоматически включаются в обычные результаты поиска. Это означает, что частично индексные элементы будут включены в PST-файл (или как отдельные сообщения), который содержит другие индексные элементы.

    При экспорте индексных и частично индексных элементов или экспорте только индексных элементов из поиска контента, который возвращает все элементы, будет загружено одинаковое количество элементов. Это происходит, даже если приблизительный результат поиска контента (отображаемой в статистике поиска в Центре безопасности & соответствия требованиям) по-прежнему будет включать отдельную оценку количества частично индексных элементов. Например, предположим, что оценка поиска, включаемая все элементы (без ключевых слов в поисковом запросе), показывает, что было найдено 1000 элементов, а также 200 частично индексных элементов. В этом случае 1000 элементов включают частично индексные элементы, так как поиск возвращает все элементы. Другими словами, поиск возвращает 1000 элементов, а не 1200 (как можно ожидать). Если экспортировать результаты этого поиска и выбрать экспорт индексных и частично индексных элементов (или экспортировать только частично индексные элементы), будет загружено 1000 элементов. Опять же, это происходит потому, что частично индексные элементы включаются в обычные (индексные) результаты при использовании пустого поискового запроса для возврата всех элементов. В этом же примере при экспорте только частично индексных элементов будет загружено только 200 неиндексных элементов.

    Кроме того, обратите внимание, что в предыдущем примере (при экспорте индексных  и частично индексных элементов или экспорте только индексных элементов) в отчете "Сводка по экспорту", включенного в экспортные результаты поиска, будет 1000 элементов с оценкой и 1000 загруженных элементов по тем же причинам, что и описано выше. 

- Если поиск, из которых экспортируются результаты, был поиском определенных местоположений контента или всех местоположений контента в организации, будут экспортироваться только частичные элементы из местоположений контента, содержащих элементы, которые соответствуют условиям поиска. Другими словами, если в почтовом ящике или на сайте не найдены результаты поиска, то частично индексные элементы в этом почтовом ящике или на сайте не будут экспортироваться. Причина этого заключается в том, что экспорт частично индексных элементов из большого числа мест в организации может увеличить вероятность ошибок экспорта и увеличить время, необходимое для экспорта и загрузки результатов поиска.

    Чтобы экспортировать частично индексируемые элементы из всех местоположений контента для поиска, настройте поиск для возврата всех элементов (удалив все ключевые слова из поискового запроса), а затем экспортировать только частично индексируемые элементы при экспорте результатов поиска.

    ![Использование третьего варианта экспорта для экспорта только неиndexed элементов](../media/5d7be338-a0e5-425f-8ba5-92769c24bf75.png)
  
- При экспорте результатов поиска с сайтов SharePoint или OneDrive для бизнеса возможность экспорта неиндексных элементов также зависит от выбранного варианта экспорта и от того, содержит ли сайт, на который был велись поиск, индексный элемент, который соответствует условиям поиска. Например, если поиск на определенных сайтах SharePoint или OneDrive для бизнеса не найден, то неиндексные элементы с этих сайтов не будут экспортироваться, если выбрать второй вариант экспорта для экспорта индексных и неиндексных элементов. Если индексный элемент сайта соответствует условиям поиска, при экспорте как индексных, так и неиндексных элементов будут экспортироваться все неиндексные элементы с этого сайта. На следующем рисунке описаны параметры экспорта в зависимости от того, содержит ли сайт индексный элемент, который соответствует условиям поиска.

    ![Выберите вариант экспорта в зависимости от того, содержит ли сайт индексный элемент, который соответствует условиям поиска](../media/94f78786-c6bb-42fb-96b3-7ea3998bcd39.png)

    1. Экспортируются только индексные элементы, которые соответствуют условиям поиска. Частично индексные элементы не экспортируются.

    2. Если никакие индексные элементы с сайта не соответствуют условиям поиска, то частично индексные элементы с того же сайта не экспортируются. Если индексные элементы с сайта возвращаются в результатах поиска, то частично индексные элементы с этого сайта экспортируются. Другими словами, экспортируются только частично индексные элементы с сайтов, которые содержат элементы, которые соответствуют условиям поиска.

    3. Все частично индексные элементы со всех сайтов в поиске экспортируются независимо от того, содержит ли сайт элементы, которые соответствуют условиям поиска.

    Если вы решите экспортировать частично индексные элементы, частично индексные элементы почтовых ящиков будут экспортироваться в отдельный PST-файл независимо от параметра, который вы выбрали в списке "Экспорт содержимого **Exchange как"**.

- Если в результатах поиска возвращаются частично индексные элементы (так как другие свойства частично индексных элементов соответствуют условиям поиска), то частично индексные элементы экспортируются с обычными результатами поиска. Таким образом, если вы решите экспортировать как индексные, так и частично индексизированные элементы (выбрав элементы "Все", в том числе элементы с невыявными **форматами,** зашифрованные или не проиндексированы по другим причинам), частично индексизированные элементы, экспортированы с помощью обычных результатов, будут перечислены в отчете Results.csv. Они не будут перечислены в отчете о items.csv не items.csv.
  
### <a name="exporting-individual-messages-or-pst-files"></a>Экспорт отдельных сообщений или PST-файлов
  
- Если имя пути к файлу сообщения превышает максимальное ограничение символов для Windows, имя пути к файлу усечено. Но исходное имя пути к файлу будет указано в манифесте и в списке результатов.
  
- Как объяснялось ранее, результаты поиска по электронной почте экспортируются в папку в файловой системе. Путь к папке для отдельных сообщений будет реплицировать путь к папке в почтовом ящике пользователя. Например, для поиска с именем "ContosoCase101" сообщения в папке "Входящие" пользователя будут расположены по пути к  `~ContosoCase101\\<date of export\Exchange\user@contoso.com (Primary)\Top of Information Store\Inbox` папке.

- Если вы решите экспортировать сообщения электронной почты в один PST-файл, содержащий все  сообщения в одной папке, папка "Удаленные" и папка "Папки поиска" будут включены в верхний уровень папки PST.  Эти папки пусты.

- Как было сказано ранее, для расшифровки сообщений с защитой RMS при их экспорте необходимо экспортировать результаты поиска по электронной почте в качестве отдельных сообщений. Зашифрованные сообщения останутся зашифрованными при экспорте результатов поиска по электронной почте в виде PST-файла.
  
### <a name="exporting-results-from-more-than-100000-mailboxes"></a>Экспорт результатов из более чем 100 000 почтовых ящиков

- Как объяснялось ранее &, для скачивания результатов поиска из более чем 100 000 почтовых ящиков необходимо использовать PowerShell Центра безопасности и соответствия требованиям. Чтобы скачать эти результаты поиска, можно выполнить следующий сценарий в этом разделе. При использовании этого сценария предполагается, что вы уже экспортировали результаты поиска (задание экспорта отображается на вкладке **"Экспорт"** в средстве поиска контента) и хотите скачать их.

   ```powershell
   $export=Get-ComplianceSearchAction SEARCHNAME_Export -IncludeCredential;
   $exportUrl=   [System.Uri]::EscapeDataString(($export.Results.Split(";") | ?{$_ -like '*Container url*'} | %{$_.Split(":",2)} | select -last 1).Trim());
   $exportToken=($export.Results.Split(";") | ?{$_ -like '*SAS Token*'} | %{$_.Split(":",2)} | select -last 1).Trim();
   ."$env:ProgramFiles\Internet Explorer\IEXPLORE.EXE" "https://complianceclientsdf.blob.core.windows.net/v16/Microsoft.Office.Client.Discovery.UnifiedExportTool.application?name=$($export.Name)&source=$exportUrl&zip=allow&trace=1";
   $exportToken | clip;
   ```

  В сценарии необходимо указать имя поиска, для которого нужно экспортировать результаты. Например, для поиска с именем замените SEARCHNAME_Export `SearchAllMailboxes` на `SearchAllMailboxes_Export` .

  После добавления имени поиска в сценарий можно скопировать текст сценария и вложить его в окно Windows PowerShell, подключенное к [PowerShell](https://docs.microsoft.com/powershell/exchange/connect-to-scc-powershell)Центра безопасности & соответствия требованиям. После вписания скрипта отобразилось средство экспорта для eDiscovery (как при загрузке результатов поиска с помощью пользовательского интерфейса):

  ![Средство экспорта eDiscovery](../media/eDiscoveryExportTool.png)

  Щелкните поле "Ключ экспорта" и нажмите, чтобы вставку ключа экспорта (сценарий копирует ключ экспорта `CTRL + V` в буфер обмена). Нажмите **кнопку** "Обзор", чтобы указать расположение для скачивания файлов, а затем запустите скачивание.

  Как было сказано ранее, мы рекомендуем скачивать результаты поиска на локальный диск из-за высокой активности диска (чтение и записи). Не скачивать результаты поиска на сетевой диск или другое сетевое расположение.

### <a name="decrypting-rms-protected-email-messages-and-encrypted-file-attachments"></a>Расшифровка сообщений электронной почты, защищенных с помощью RMS, и вложений зашифрованных файлов

Все сообщения электронной почты, защищенные правами (с защитой RMS), включенные в результаты поиска контента, будут расшифровываться при экспорте. Кроме того, любой файл, зашифрованный с помощью технологии шифрования [Майкрософт](encryption.md) и вложенный в сообщение электронной почты, включенное в результаты поиска, также будет расшифровываться при его экспорте. Эта возможность расшифровки включена по умолчанию для членов группы ролей "Менеджер по обнаружению электронных данных". Это необходимо, поскольку роль управления расшифровка RMS по умолчанию назначена этой группе ролей. При экспорте зашифрованных сообщений электронной почты и вложений следует иметь в виду следующее:
  
- Как объяснялось ранее, для расшифровки сообщений с защитой RMS при их экспорте необходимо экспортировать результаты поиска в качестве отдельных сообщений. При экспорте результатов поиска в PST-файл сообщения, защищенные службой RMS, остаются зашифрованными.

- Расшифровка сообщений идентифицирована в **отчете ResultsLog.** Этот отчет содержит столбец с именем  **"Decode Status",** а значение расшифрованного в этом столбце определяет расшифровку сообщений.

- Помимо расшифровки вложений при экспорте результатов поиска, можно также просмотреть расшифрованный файл при предварительном просмотре результатов поиска. Вы можете просматривать защищенное правами сообщение электронной почты только после его экспорта.

- В настоящее время возможность расшифровки при экспорте результатов поиска не включает зашифрованное содержимое с сайтов SharePoint и OneDrive для бизнеса. Однако скоро появится поддержка для документов, зашифрованных с помощью технологий шифрования Майкрософт и хранимых в SharePoint Online и OneDrive для бизнеса.

- Если необходимо предотвратить расшифровку сообщений, защищенных с помощью RMS, и зашифрованных вложений файлов, необходимо создать настраиваемую группу ролей (скопируя встроенную группу ролей диспетчера eDiscovery), а затем удалить роль управления расшифровкой RMS из настраиваемой группы ролей. Затем добавьте человека, которого вы не хотите расшифровывать, в качестве члена настраиваемой группы ролей.
  
### <a name="filenames-of-exported-items"></a>Имена файлов экспортных элементов
  
- Существует ограничение в 260 символов (налагается операционной системой) для полного пути для сообщений электронной почты и документов сайта, экспортируемого на локальный компьютер. Полное имя пути для экспортных элементов включает исходное расположение элемента и папку на локальном компьютере, на который загружаются результаты поиска. Например, если вы укаружите загрузку результатов поиска в средство экспорта для  `C:\Users\Admin\Desktop\SearchResults` eDiscovery, то полный путь для скачанный элемент электронной почты будет  `C:\Users\Admin\Desktop\SearchResults\ContentSearch1\03.15.2017-1242PM\Exchange\sarad@contoso.com (Primary)\Top of Information Store\Inbox\Insider trading investigation.msg` .

    Если ограничение в 260 символов превышено, полное имя пути для элемента будет усечено.

  - Если полное имя пути больше 260 символов, имя файла будет сокращено, чтобы попасть под ограничение; обратите внимание, что усеченное имя файла (за исключением расширения файла) не должно быть меньше восьми символов.

  - Если после сокращения имени файла имя полного пути остается слишком длинным, элемент перемещается из текущего расположения в родительную папку. Если путь слишком длинный, процесс повторяется: сокращается имя файла и при необходимости снова перемещается в родительную папку. Этот процесс повторяется до тех пор, пока полное имя пути не будет под ограничением в 260 символов.

  - Если полное имя усеченного пути уже существует, номер версии добавляется в конец имени файла; например,  `statusmessage(2).msg` .

    Чтобы устранить эту проблему, рассмотрите возможность загрузки результатов поиска в расположение с коротким именем; Например, скачивание результатов поиска в именоваемую папку добавит меньше символов к именам путей экспортных элементов, чем их загрузка в папку с именем  `C:\Results`  `C:\Users\Admin\Desktop\Results` .

- При экспорте документов сайта также может быть изменено исходное имя файла документа. Это происходит специально для документов, удаленных с сайта SharePoint или OneDrive для бизнеса, помещенного на удержание. После удаления документа на сайте, который находится на удержании, удаленный документ автоматически перемещается в архивную библиотеку для сайта (который был создан при удержании сайта). При перемещении удаленного документа в архивную библиотеку к исходному имени файла документа будет придан случайный и уникальный ИД. Например, если имя файла документа является и этот документ позднее удаляется и перемещается в архивную библиотеку, имя файла документа, перемещенного в архивную библиотеку, будет изменено на что-то вроде  `FY2017Budget.xlsx`  `FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx` . Если документ в архивной библиотеке соответствует запросу поиска контента и вы экспортируете результаты этого поиска, в экспортируемом файле будет изменено имя файла; в этом примере имя файла экспортируемого документа будет  `FY2017Budget_DEAF727D-0478-4A7F-87DE-5487F033C81A2000-07-05T10-37-55.xlsx` таким.

    Когда документ на сайте, на удержании, изменен (и включено хранение версий для библиотеки документов на сайте), копия файла автоматически создается в архивной библиотеке. В этом случае к имени файла документа, скопированного в архивную библиотеку, также будет сгенерирован случайным образом уникальный ИД.

    Причина, по которой имена файлов документов, перемещаются или копируется в архивную библиотеку, — это предотвращение конфликтующих имен файлов. Дополнительные сведения о удержании сайтов и архивной библиотеки см. в обзоре хранения на месте [в SharePoint Server 2016.](https://support.office.com/article/5e400d68-cd51-444a-8fe6-e4df1d20aa95)

### <a name="miscellaneous"></a>Разное
  
- При скачивании результатов поиска с помощью средства экспорта eDiscovery может возникнуть следующая ошибка: это временные ошибки, которые обычно возникают в хранилище `System.Net.WebException: The remote server returned an error: (412) The condition specified using HTTP conditional header(s) is not met.` Azure. Чтобы устранить эту проблему, повторите скачивание результатов [поиска,](#step-2-download-the-search-results)после чего перезапустите средство экспорта eDiscovery.

- Все результаты поиска и отчеты об экспорте содержатся в папке, имя которой совпадает с именем поиска контента. Экспортированные письма расположены в папке с именем **Exchange**. Документы расположены в папке с именем **SharePoint**.

- Метаданные файловой системы для документов на сайтах SharePoint и OneDrive для бизнеса сохраняются при экспорте документов на локальный компьютер. Это означает, что при экспорте документов их свойства, например даты создания и последнего изменения, не изменяются.

- Если результаты поиска включают элемент списка из SharePoint, который соответствует поисковому запросу, все строки в списке будут экспортироваться в дополнение к элементу, который соответствует поисковому запросу, и всем вложениям в списке. Причиной такого поведения является предоставление контекста для элементов списка, возвращаемого в результатах поиска. Также обратите внимание, что дополнительные элементы списка и вложения могут привести к тому, что количество экспортных элементов будет не таким, как при первоначальной оценке результатов поиска.
