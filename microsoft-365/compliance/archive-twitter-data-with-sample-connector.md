# <a name="set-up-a-connector-to-archive-twitter-data-preview"></a>Настройка соединитетеля для архива данных Twitter (предварительный просмотр)

Используйте соединители в центре соответствия требованиям Microsoft 365 для импорта и архива данных из Twitter в Microsoft 365. После настройки и настройки соединители подключаются к учетной записи Twitter организации (на плановой основе), преобразует содержимое элемента в формат сообщений электронной почты, а затем импортирует эти элементы в почтовый ящик в Microsoft 365.

После импорта данных Twitter можно применить к данным Twitter такие функции соответствия требованиям Microsoft 365, как хранение судебного разбирательства, поиск контента, In-Place архива, аудит и политики хранения Microsoft 365. Например, при размещении почтового ящика в удержании судебного разбирательства или при наложении на политику хранения данные Twitter сохраняются. Вы можете искать сторонние данные с помощью поиска контента или связывать почтовый ящик, в котором хранятся данные Twitter, с хранителями в случае предварительного поиска электронных данных. Использование соединиттеля для импорта и архива данных Twitter в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

После импорта данных Twitter можно применить к данным, хранимым в почтовом ящике, такие функции соответствия требованиям Microsoft 365, как хранение судебного разбирательства, поиск контента, In-Place архива, аудит, соответствие требованиям связи и политики хранения Microsoft 365. Например, можно искать данные в Twitter с помощью поиска контента или связывать почтовый ящик, в котором хранятся данные, с хранителями в случае предварительного поиска электронных данных. Использование соединиттеля для импорта и архива данных Twitter в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="before-you-set-up-a-connector"></a>Перед настройками соединитетеля

Выполните следующие необходимые условия, прежде чем можно настроить соединители в центре соответствия требованиям Microsoft 365 для импорта и архива данных из учетной записи Twitter организации.

- Для организации нужна учетная запись Twitter; при настройке соединитетеля необходимо войти в эту учетную запись.

- В организации должна быть действительная подписка Azure. Если у вас нет существующей подписки Azure, вы можете зарегистрироваться для одного из этих параметров:

    - [Зарегистрив бесплатную подписку на Azure на один год](https://azure.microsoft.com/free) 

    - [Подпишитесь на подписку Pay-As-You-Go Azure](https://azure.microsoft.com/pricing/purchase-options/pay-as-you-go/)

    > [!NOTE]
    > Бесплатная [подписка Azure Active Directory,](use-your-free-azure-ad-subscription-in-office-365.md) включенная в подписку Microsoft 365, не поддерживает соединители в Центре & соответствия требованиям.

- Соединители Twitter могут импортировать в общей сложности 200 000 элементов за один день. Если в день имеется более 200 000 элементов Twitter, ни один из этих элементов не будет импортироваться в Microsoft 365.

- Пользователю, который настраивает соединитель Twitter в центре соответствия требованиям Microsoft 365 (на шаге 5), необходимо уступить роль экспорта импорта почтовых ящиков в Exchange Online. По умолчанию эта роль не назначена ни одной группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах Создание групп  [ролей](/Exchange/permissions-exo/role-groups#create-role-groups) или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-create-an-app-in-azure-active-directory"></a>Шаг 1. Создание приложения в Azure Active Directory

Первым шагом является регистрация нового приложения в Azure Active Directory (AAD). Это приложение соответствует ресурсу веб-приложения, который реализуется в шаге 2 для соединиттеля Twitter.

Пошаговая инструкция см. в приложении [Create in Azure Active Directory.](deploy-twitter-connector.md#step-1-create-an-app-in-azure-active-directory)

Во время выполнения этого шага (следуя пошаговой инструкции), вы сохраните следующую информацию в текстовом файле. Эти значения будут использоваться в последующих шагах в процессе развертывания.

- ID приложения AAD

- Секрет приложения AAD

- ID клиента

## <a name="step-2-deploy-connector-web-service-from-github-repository-to-your-azure-account"></a>Шаг 2. Развертывание веб-службы соединителя из репозитория GitHub в учетную запись Azure

Следующим шагом является развертывание исходных кодов для приложения соединиттеля Twitter, которое будет использовать API Twitter для подключения к вашей учетной записи Twitter и извлечения данных, чтобы вы могли импортировать его в Microsoft 365. Соединитатель Twitter, развернутый для организации, будет загружать элементы из учетной записи Twitter организации в расположение Хранилища Azure, созданное на этом шаге. После создания соединитетеля Twitter в центре соответствия требованиям Microsoft 365 (на шаге 5) служба импорта Microsoft 365 скопирует данные Twitter из расположения Хранилища Azure в почтовый ящик в Microsoft 365. Как было ранее [](#before-you-set-up-a-connector) объяснено в разделе Перед созданием соединитетеля, для создания учетной записи Azure необходимо иметь допустимую подписку Azure.

Чтобы развернуть исходный код для приложения соединители Twitter:

1. Перейдите [на этот сайт GitHub](https://github.com/microsoft/m365-sample-twitter-connector-csharp-aspnet).

2. Щелкните **Развертывание в Azure.**

Пошаговые инструкции см. в странице Развертывание веб-службы соединителя [из GitHub в учетную](deploy-twitter-connector.md#step-2-deploy-the-connector-web-service-from-github-to-your-azure-account)запись Azure.

Следуя пошаговую инструкцию по завершению этого шага, вы предоставляете следующие сведения

- APISecretKey. Этот секрет создается при завершении этого шага. Он используется в шаге 5.

- tenantId. ID клиента вашей организации Microsoft 365, который вы скопировали после создания приложения Twitter в Azure Active Directory на шаге 1.

После завершения этого шага обязательно скопируйте URL-адрес службы приложений (например, `https://twitterconnector.azurewebsites.net` ). Этот URL-адрес необходимо использовать для выполнения 3-го, 4-го и 5-го этапов.

## <a name="step-3-create-developer-app-on-twitter"></a>Шаг 3. Создание приложения-разработчика в Twitter

Следующий шаг — создание и настройка приложения-разработчика в Twitter. Настраиваемый соединитатель, созданный в шаге 7, использует приложение Twitter для взаимодействия с API Twitter для получения данных из учетной записи Twitter организации.

Пошаговая инструкция см. в [странице Create the Twitter app.](deploy-twitter-connector.md#step-3-create-the-twitter-app)

Во время выполнения этого шага (следуя пошаговим инструкциям), вы сохраните следующую информацию в текстовом файле. Эти значения будут использоваться для настройки приложения соединители Twitter в шаге 4.

- Ключ API Twitter

- Секретный ключ API Twitter

- Маркер доступа к Twitter

- Секрет маркера доступа к Twitter

## <a name="step-4-configure-the-twitter-connector-app"></a>Шаг 4. Настройка приложения соединители Twitter

Следующий шаг — добавление параметров конфигураций в приложение соединителю Twitter, развернутое в шаге 2. Это необходимо сделать, переехав на домашную страницу приложения-соединиттеля и настроив его.

Пошаговая инструкция см. в приложении [Configure the connector web app.](deploy-twitter-connector.md#step-4-configure-the-connector-web-app)

Во время выполнения этого шага (следуя пошаговим инструкциям), вы предоставите следующую информацию (которую вы скопировали в текстовый файл после завершения предыдущих действий):

- Ключ API Twitter (получен в шаге 3)

- Секретный ключ API Twitter (получен в шаге 3)

- Маркер доступа к Twitter (получен в шаге 3)

- Секрет маркера доступа к Twitter (получен в шаге 3)

- Azure Active Directory application ID (AAD application ID obtained in Step 1)

- Секрет приложения Azure Active Directory (секрет приложения AAD, полученный в шаге 1)

## <a name="step-5-set-up-a-twitter-connector-in-the-microsoft-365-compliance-center"></a>Шаг 5. Настройка соединитетеля Twitter в центре соответствия требованиям Microsoft 365

Заключительный шаг — настройка соединитетеля Twitter в центре соответствия требованиям Microsoft 365, который будет импортировать данные из учетной записи Twitter организации в указанный почтовый ящик в Microsoft 365. После завершения этого шага служба импорта Microsoft 365 начнет импорт данных из учетной записи Twitter организации в Microsoft 365.

Пошаговая инструкция см. в странице Настройка соединитетеля Twitter в центре соответствия [требованиям Microsoft 365.](deploy-twitter-connector.md#step-5-set-up-a-twitter-connector-in-the-microsoft-365-compliance-center) 

Во время выполнения этого шага (следуя пошаговой инструкции), вы предоставите следующую информацию (которую вы скопировали в текстовый файл после выполнения действий).

- URL-адрес службы приложений Azure (полученный в шаге 2; например, `https://twitterconnector.azurewebsites.net` )

- APISecretKey (созданный в шаге 2)
