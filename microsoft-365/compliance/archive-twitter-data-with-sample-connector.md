# <a name="set-up-a-connector-to-archive-twitter-data-preview"></a>Настройка соединитела для архива данных Twitter (предварительная версия)

Используйте соединители в Центре соответствия требованиям Microsoft 365, чтобы импортировать и архивировать данные из Twitter в Microsoft 365. После настройки соединители подключаются к учетной записи Twitter вашей организации (по расписанию), преобразуют содержимое элемента в формат сообщений электронной почты, а затем импортируют их в почтовый ящик в Microsoft 365.

После импорта данных Twitter к данным Twitter можно применить такие функции соответствия требованиям Microsoft 365, как хранение для судебного разбирательства, поиск контента, архивировать In-Place, аудит и политики хранения Microsoft 365. Например, если почтовый ящик помещен на хранение для судебного разбирательства или назначен политике хранения, данные Twitter сохраняются. Вы можете искать данные сторонних пользователей с помощью поиска контента или связывать почтовый ящик, в котором хранятся данные Twitter, с хранителями в деле Advanced eDiscovery. Использование соединители для импорта и архива данных Twitter в Microsoft 365 может помочь вашей организации соблюдать правительственные и нормативные политики.

После импорта данных Twitter к данным, хранимым в почтовом ящике, можно применить такие функции соответствия требованиям Microsoft 365, как хранение для судебного разбирательства, поиск контента, архивировать In-Place, аудит, соответствие коммуникациям и политики хранения Microsoft 365. Например, вы можете искать данные в Twitter с помощью поиска контента или связывать почтовый ящик, в котором хранятся данные, с хранителями в деле Advanced eDiscovery. Использование соединители для импорта и архива данных Twitter в Microsoft 365 может помочь вашей организации соблюдать правительственные и нормативные политики.

## <a name="prerequisites-for-setting-up-a-connector-for-twitter"></a>Необходимые условия для настройки соединители для Twitter

Выполните следующие необходимые условия, прежде чем настраивать соединители в Центре соответствия требованиям Microsoft 365 для импорта и архива данных из учетной записи Twitter вашей организации.

- Вам нужна учетная запись Twitter для вашей организации; При настройке соединители необходимо войти в эту учетную запись.

- У вашей организации должна быть действительная подписка На Azure. Если у вас нет существующей подписки Azure, вы можете зарегистрироваться для использования одного из указанных здесь вариантов.

    - [Зарегистрироваться для бесплатной подписки Azure на один год](https://azure.microsoft.com/free) 

    - [Sign up for a Pay-As-You-Go Azure subscription](https://azure.microsoft.com/pricing/purchase-options/pay-as-you-go/)

    > [!NOTE]
    > Бесплатная [подписка на Azure Active Directory,](use-your-free-azure-ad-subscription-in-office-365.md) включенная в вашу подписку на Microsoft 365, не поддерживает соединители в Центре безопасности & соответствия требованиям.

- Пользователю, который настраивает соединитель Twitter в Центре соответствия требованиям Microsoft 365 (на шаге 5), должна быть назначена роль импорта и экспорта почтовых ящиков в Exchange Online. По умолчанию эта роль не назначена ни одной группе ролей в Exchange Online. Вы можете добавить роль "Импорт и экспорт почтового ящика" в группу ролей "Управление организацией" в Exchange Online. Можно также создать группу ролей, назначить роль "Импорт и экспорт почтового ящика" и добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в [](https://docs.microsoft.com/Exchange/permissions-exo/role-groups#modify-role-groups) разделах ["Создание](https://docs.microsoft.com/Exchange/permissions-exo/role-groups#create-role-groups) групп ролей" или "Изменение групп ролей" статьи "Управление группами ролей в Exchange Online".

## <a name="step-1-create-an-app-in-azure-active-directory"></a>Шаг 1. Создание приложения в Azure Active Directory

Первым шагом является регистрация нового приложения в Azure Active Directory (AAD). Это приложение соответствует ресурсу веб-приложения, который вы реализуете в шаге 2 для соединители Twitter.

Пошаговая инструкция см. в инструкциях по [созданию приложения в Azure Active Directory.](deploy-twitter-connector.md#step-1-create-an-app-in-azure-active-directory)

Во время выполнения этого шага (следуя пошаговой инструкции), вы сохраните следующие сведения в текстовом файле. Эти значения будут использоваться на последующих шагах процесса развертывания.

- ИД приложения AAD

- Секрет приложения AAD

- ИД клиента

## <a name="step-2-deploy-connector-web-service-from-github-repository-to-your-azure-account"></a>Шаг 2. Развертывание веб-службы соединителя из репозитория GitHub в учетной записи Azure

Далее необходимо развернуть исходный код для приложения соединители Twitter, которое будет использовать API Twitter для подключения к вашей учетной записи Twitter и извлечения данных, чтобы вы могли импортировать его в Microsoft 365. Соединитатель Twitter, развернутый для вашей организации, будет загружать элементы из учетной записи Twitter вашей организации в хранилище Azure, созданное на этом шаге. После создания соединитерия Twitter в Центре соответствия требованиям Microsoft 365 (на шаге 5) служба импорта Microsoft 365 скопирует данные Twitter из хранилища Azure в почтовый ящик в Microsoft 365. Как было поясняется ранее в разделе ["Необходимые](#prerequisites-for-setting-up-a-connector-for-twitter) условия", для создания учетной записи службы хранилища Azure необходимо иметь действительная подписка azure.

Чтобы развернуть исходный код для приложения соединители Twitter:

1. Перейдите [на этот сайт GitHub.](https://github.com/microsoft/m365-sample-twitter-connector-csharp-aspnet)

2. Нажмите **кнопку "Развернуть в Azure".**

Пошаговые инструкции см. в инструкциях по развертыванию веб-службы соединителя из [GitHub в учетной записи Azure.](deploy-twitter-connector.md#step-2-deploy-the-connector-web-service-from-github-to-your-azure-account)

Следуя пошаговой инструкции для выполнения этого шага, вы предоставляете следующие сведения:

- APISecretKey: этот секрет создается во время выполнения этого шага. Он используется в шаге 5.

- tenantId: ИД клиента вашей организации Microsoft 365, скопированные после создания приложения Twitter в Azure Active Directory на шаге 1.

После выполнения этого шага обязательно скопируйте URL-адрес службы приложения (например, `https://twitterconnector.azurewebsites.net` ). Этот URL-адрес необходимо использовать для выполнения шагов 3, 4 и 5.

## <a name="step-3-create-developer-app-on-twitter"></a>Шаг 3. Создание приложения для разработчиков в Twitter

Далее необходимо создать и настроить приложение разработчика в Twitter. Настраиваемый соединитатель, созданный на шаге 7, использует приложение Twitter для взаимодействия с API Twitter для получения данных из учетной записи Twitter вашей организации.

Пошаговая инструкция по созданию [приложения Twitter.](deploy-twitter-connector.md#step-3-create-the-twitter-app)

Во время выполнения этого шага (следуя пошаговой инструкции), вы сохраняете следующие сведения в текстовый файл. Эти значения будут использоваться для настройки приложения соединители Twitter на шаге 4.

- Ключ API Twitter

- Секретный ключ API Twitter

- Маркер доступа Twitter

- Секрет маркера доступа Twitter

## <a name="step-4-configure-the-twitter-connector-app"></a>Шаг 4. Настройка приложения соединители Twitter

Следующий шаг — добавление параметров конфигурации в приложение соединителю Twitter, развернутое на шаге 2. Для этого переходить на home-страницу приложения соединитела и настраивать его.

Пошаговая инструкция см. в подстройке ["Настройка веб-приложения соединители".](deploy-twitter-connector.md#step-4-configure-the-connector-web-app)

Во время выполнения этого шага (следуя пошаговой инструкции), вы предоставите следующие сведения (которые вы скопировали в текстовый файл после выполнения предыдущих действий):

- Ключ API Twitter (полученный на шаге 3)

- Секретный ключ API Twitter (полученный на шаге 3)

- Маркер доступа Twitter (полученный на шаге 3)

- Секрет маркера доступа Twitter (полученный на шаге 3)

- ИД приложения Azure Active Directory (ИД приложения AAD, полученный на шаге 1)

- Секрет приложения Azure Active Directory (секрет приложения AAD, полученный на шаге 1)

## <a name="step-5-set-up-a-twitter-connector-in-the-microsoft-365-compliance-center"></a>Шаг 5. Настройка соединители Twitter в Центре соответствия требованиям Microsoft 365

Последний шаг состоит в том, чтобы настроить соединиталь Twitter в Центре соответствия требованиям Microsoft 365, который будет импортировать данные из учетной записи Twitter вашей организации в указанный почтовый ящик в Microsoft 365. После выполнения этого шага служба импорта Microsoft 365 начнет импортировать данные из учетной записи Twitter вашей организации в Microsoft 365.

Пошаговая инструкция см. в записи "Настройка соединители Twitter" в Центре соответствия требованиям [Microsoft 365.](deploy-twitter-connector.md#step-5-set-up-a-twitter-connector-in-the-microsoft-365-compliance-center) 

Во время выполнения этого шага (следуя пошаговой инструкции), вы предоставите следующие сведения (которые вы скопировали в текстовый файл после выполнения этих действий).

- URL-адрес службы приложений Azure (полученный на шаге 2, `https://twitterconnector.azurewebsites.net` например)

- APISecretKey (созданный на шаге 2)
