# <a name="archive-third-party-data"></a>Архивация сторонних данных

Microsoft 365 позволяет администраторам использовать соединители данных для импорта и архивации сторонних данных на платформах социальных сетей, платформах обмена мгновенными сообщениями и платформах совместной работы с документами в почтовые ящики в организации Microsoft 365. Одним из основных преимуществ использования соединителей данных для импорта и архивации сторонних данных в Microsoft 365 является то, что после импорта вы можете применить различные решения Microsoft 365 о соответствии. Это позволяет убедиться, что данные, не относящиеся к корпорации Майкрософт, соответствуют нормативам и стандартам, влияющим на вашу организацию.

## <a name="overview-of-compliance-solutions-that-support-third-party-data"></a>Общие сведения о решениях для обеспечения соответствия требованиям, которые поддерживают сторонние данные

В следующих разделах описываются некоторые аспекты, которые решения по обеспечению соответствия требованиям Microsoft 365 могут помочь в управлении данными третьих лиц, перечисленными в предыдущей таблице.

### <a name="litigation-hold"></a>Удержание для судебного разбирательства

[Хранение для судебного разбирательства](create-a-litigation-hold.md) размещается в почтовом ящике пользователя, чтобы сохранить сторонние данные. При создании удержания можно указать продолжительность удержания (также называемую *удержанием на основе времени*), чтобы удаленные и измененные сторонние данные сохранялись в течение указанного периода, а затем окончательно удалены из почтового ящика. Или вы можете просто хранить контент в неопределенном состоянии (называемом *бесконечное удержание*) или пока не будет удалено удержание для судебного разбирательства.

### <a name="ediscovery"></a>Обнаружение электронных данных

Три основных средства обнаружения электронных данных в Microsoft 365: поиск контента, основные функции обнаружения электронных данных и расширенные функции обнаружения электронных данных.

- **[Поиск контента](content-search.md).** С помощью средства поиска контента можно искать в почтовых ящиках импортируемые данные сторонних поставщиков. Вы можете использовать поисковые запросы и условия для сужения результатов поиска и экспорта результатов поиска.

- **[Основное обнаружение электронных](get-started-core-ediscovery.md)данных.** Это средство строится на основных функциях поиска и экспорта, позволяя создавать обращения, позволяющие контролировать доступ к данным Case, размещать почтовые ящики пользователей или содержимое почтовых ящиков, которые соответствуют условиям поиска. Это означает, что вы можете поместить удержание обнаружения электронных данных на сторонние данные, импортированные в почтовые ящики пользователей.

- **[Расширенное обнаружение электронных](overview-ediscovery-20.md)данных.** Это мощное средство расширяет функциональные возможности основного обнаружения электронных данных, позволяя добавлять custodians к случаю, размещая хранитель данные на удержании, а затем загружая сторонние данные хранитель в проверку для дальнейшего анализа, например, темы и обнаружение повторяющихся данных. После загрузки сторонних данных в набор проверки можно выполнить запрос и отфильтровать их по суженному набору результатов.

   Как базовые обнаружения электронных данных, так и расширенные функции обнаружения электронных данных позволяют управлять сторонними данными, которые могут быть важны для юридических или внутренних исследований Организации.

### <a name="retention-policies"></a>Политики хранения

Вы можете применить [политику хранения](retention-policies.md) к почтовым ящикам пользователей, чтобы сохранять и удалять сторонние данные (и другое содержимое почтового ящика) после истечения периода хранения. Политики хранения также можно использовать для удаления сторонних данных определенного возраста или запуска проверки ликвидации по истечении срока хранения.

### <a name="records-management"></a>Управление записями

Функция [управления записями](records-management.md) в Microsoft 365 позволяет объявлять сторонние данные как записи. Это можно сделать вручную пользователями, которые применяют метку хранения, которая помечает сторонние данные в своем почтовом ящике как записи. Кроме того, вы можете автоматически применять метки хранения, определив конфиденциальную информацию, ключевые слова или типы контента в сторонних данных.

### <a name="communication-compliance"></a>Соответствие требованиям к обмену данными

Вы можете использовать [обеспечение соответствия требованиям](communication-compliance.md) для проверки данных сторонних производителей, чтобы убедиться, что они соответствуют стандартам данных вашей организации. Это можно сделать с помощью обнаружения, записи и принятия действий по исправлению для неуместных сообщений в Организации. Например, вы можете отслеживать сторонние данные, импортируемые для оскорбительного языка, конфиденциальной информации и соответствия нормативным требованиям.

### <a name="insider-risk-management"></a>Управление рисками, связанными с инсайдерами

Сигналы от сторонних данных, таких как выборочные данные о ПЕРСОНАЛе, могут быть использованы в решении по [управлению рисками](insider-risk-management.md) , чтобы свести к минимуму внутренние риски, позволяя обнаруживать, изучать и исправлять рискованные действия в Организации. Например, данные, импортированные с помощью Data Connector Connector, используются в качестве индикаторов риска, чтобы помочь обнаружить, что может привести к утечке данных сотрудников.

## <a name="working-with-a-microsoft-partner-to-archive-third-party-data"></a>Работа с партнером Майкрософт для архивации сторонних данных

Кроме того, вы можете импортировать и архивировать сторонние данные, чтобы ваша организация работала с партнером корпорации Майкрософт. Если сторонние типы данных не поддерживаются соединителями данных, доступными в центре соответствия требованиям Майкрософт, вы можете работать с партнером, который может предоставить настраиваемый соединитель, который будет настроен на регулярное извлечение элементов из стороннего источника данных, а затем подключается к Microsoft Cloud с помощью стороннего API и импортирует эти элементы в Microsoft 365. Соединитель партнера также Преобразовывает содержимое элемента из стороннего источника данных в сообщение электронной почты, а затем импортирует его в почтовый ящик в Microsoft 365.

Список партнеров, с которыми вы можете работать, а также пошагового процесса для этого метода, приведен в статье [Работа с партнером для архивации сторонних данных в Microsoft 365](work-with-partner-to-archive-third-party-data.md).
