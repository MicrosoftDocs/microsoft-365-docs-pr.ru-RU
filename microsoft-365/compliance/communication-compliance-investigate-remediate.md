# <a name="investigate-and-remediate-communication-compliance-alerts"></a>Анализ и устранений оповещений соответствия требованиям к обмену данными

После настройки политик соответствия требованиям к коммуникациям вы начнете получать в Центре соответствия требованиям Microsoft 365 оповещения о проблемах с сообщениями, которые соответствуют условиям политики. Следуйте инструкциям рабочего процесса, чтобы изучить и устранять проблемы оповещений.

## <a name="investigate-alerts"></a>Изучение оповещений

Первым шагом для изучения проблем, обнаруженных вашими политиками, является проверка оповещений о соответствии требованиям в Центре соответствия требованиям Microsoft 365. Существует несколько областей, которые помогут вам быстро изучить оповещения в зависимости от того, как вы предпочитаете просматривать группировку оповещений:

- **Страница политики соответствия** коммуникациям: при входе с использованием учетных данных для учетной записи администратора в организации [https://compliance.microsoft.com](https://compliance.microsoft.com) Microsoft 365  выберите соответствие коммуникациям, чтобы отобразить страницу политики соответствия требованиям.  На этой странице отображаются политики соответствия требованиям связи, настроенные для вашей организации Microsoft 365, и ссылки на рекомендуемые шаблоны политик. Каждая из перечисленных политик включает в себя количество оповещений, которые необходимо проверить, количество элементов, которые были рассмотрены и разрешены, состояние политики, а также дату и время последней проверки политики. При выборе политики отображаются все ожидающие оповещения о совпадениях с политикой, выберите конкретное оповещение, чтобы запустить страницу сведений о политике и начать действия по исправлению.
- **Alerts**: Navigate to **Communication compliance**  >  **Alerts** to display the last 30 days of alerts grouped by policy matches. Это представление позволяет быстро увидеть, какие политики соответствия требованиям к коммуникациям генерируют больше всего оповещений, упорядоченных по степени серьезности. Чтобы начать действия по исправлению, выберите политику, связанную с оповещением, чтобы запустить страницу **сведений о политике.** На  странице "Сведения о политике" можно просмотреть  сводку действий на странице "Обзор", просмотреть сообщения оповещений на странице "Ожидание" и просмотреть историю закрытых оповещений на странице  "Решение". 
- **Отчеты:** перейдите в **отчеты о** соответствии  >  **коммуникациям,** чтобы отобразить мини-приложения отчетов о соответствии коммуникациям. Каждое мини-приложения предоставляет обзор действий и состояния соответствия требованиям в отношении связи, включая доступ к более глубокой информации о совпадениях политик и действиях по исправлению.

### <a name="using-filters"></a>Использование фильтров

Далее необходимо отсортировать сообщения, чтобы вам было проще изучить оповещения. На странице **сведений о** политике соответствие требованиям к коммуникациям поддерживает многоуровневую фильтрацию для нескольких полей сообщений, что помогает быстро исследовать и просмотреть сообщения с совпадениями политик. Фильтрация доступна для ожидающих и разрешенных элементов для каждой настроенной политики. Можно настроить запросы фильтров для политики или настроить и сохранить пользовательские и стандартные запросы фильтров для использования в каждой конкретной политике. После настройки полей для фильтра вы увидите поля фильтра, отображаемые в верхней части очереди оповещений, которую можно настроить для определенных значений фильтра.

Полный список фильтров и сведений о полях см. в справочной статье [о](communication-compliance-feature-reference.md#filters) фильтрах.

#### <a name="to-configure-a-filter"></a>Настройка фильтра

1. Во sign [https://compliance.microsoft.com](https://compliance.microsoft.com) into using credentials for an admin account in your Microsoft 365 organization.

2. В Центре соответствия требованиям Microsoft 365 перейдите к **информационному обеспечению** соответствия.

3. Выберите **вкладку "Политики",** а затем выберите политику для исследования, дважды щелкните, чтобы открыть **страницу** "Политика".

4. На странице **"Политика"** выберите  вкладку **"Ожидание"** или "Разрешено", чтобы отобразить элементы для фильтрации.

5. Выберите **"Фильтры",** чтобы открыть страницу **сведений фильтров.**

6. Выберите один или несколько контрольных точек, чтобы включить фильтры для этих оповещений. Вы можете выбрать один из множества фильтров, в том числе *Date,* *Sender,* *Subject/Title,* *Classifiers* и т. д.

7. Если вы хотите сохранить фильтр, выбранный в качестве фильтра по умолчанию, выберите "Сохранить **по умолчанию".** Если вы хотите использовать этот фильтр в качестве сохраненного фильтра, выберите **"Готово".**

8. Если вы хотите сохранить выбранные фильтры в качестве  запроса фильтра, выберите "Сохранить" после настройки по крайней мере одного значения фильтра. Введите имя запроса фильтра и выберите **"Сохранить".** Этот фильтр доступен только для этой политики и  указан в разделе "Сохраненные запросы фильтра" на странице сведений **о фильтрах.**

    ![Средства управления подробными деталями фильтра соответствия требованиям к коммуникациям](../media/communication-compliance-filter-detail-controls.png)

### <a name="using-near-and-exact-duplicate-analysis"></a>Использование близкого и точного повторяемого анализа

Политики соответствия требованиям к коммуникациям автоматически сканируют и предварительно группируют копии сообщений без дополнительных действий по настройке. Это представление позволяет быстро действовать над похожими сообщениями по одному или как группа, снижая нагрузку на анализ сообщений для проверяющих. При обнаружении дубликатов на панели  инструментов действия по исправлению отображаются элементы управления "Близкие дубликаты" и /или "Точные дубликаты".  Это представление недоступно, если не найдены близкие или точные дубликаты.

#### <a name="to-remediate-duplicates"></a>Исправление дубликатов

1. Во sign [https://compliance.microsoft.com](https://compliance.microsoft.com) into using credentials for an admin account in your Microsoft 365 organization.

2. В Центре соответствия требованиям Microsoft 365 перейдите к **информационному обеспечению** соответствия.

3. Выберите **вкладку "Политики",** а затем выберите политику для исследования, дважды щелкните, чтобы открыть **страницу** "Политика".

4. На странице **"Политика"** выберите  вкладку **"Ожидание"** или "Разрешено", чтобы отобразить повторяющиеся сообщения.

5. Выберите элементы **управления "Близкие** дубликаты" или **"Точные** дубликаты", чтобы открыть страницу сведений о дубликатах.

6. Выберите одно или несколько сообщений для элементов управления действиями по исправлению для этих сообщений.

7. Выберите **"Разрешить",** **"Уведомить",** **"Эскалация"** или "Скачать", чтобы применить действие к выбранным дублирующимся сообщениям в качестве фильтра по умолчанию. 

8. Выберите **"Закрыть"** после выполнения действий по исправлению сообщений.

    ![Точное дублирование средств контроля соответствия коммуникации](../media/communication-compliance-duplicates-controls.png)

## <a name="remediate-alerts"></a>Исправление оповещений

Независимо от того, где вы начинаете просмотр оповещений или настраиваемой фильтрации, следующим шагом является принятие мер по исправлению оповещения. Запустите исправление оповещений, используя следующий рабочий процесс на страницах **"Политика"** **или "Оповещения".**

### <a name="step-1-examine-the-message-basics"></a>Шаг 1. Изучение основ сообщений

 Иногда источнику или теме очевидно, что сообщение может быть немедленно исправлено. Возможно, сообщение сомнительным или неправильно соответствует политике и должно быть разрешено как ложное срабатыващение. Выберите этот **контроль,** чтобы немедленно устранить оповещение и удалить его из очереди ожидающих оповещений. Из сведений об источнике или отправителю вы, возможно, уже знаете, как сообщение должно быть перенаправляться или обрабатываться в этих обстоятельствах. Рассмотрите возможность использования  **тега** "Как" или "Эскалация" для назначения тега применимым сообщениям или отправки сообщений назначенной проверяемой.

![Средства контроля соответствия коммуникациям требованиям](../media/communication-compliance-remediation-controls.png)

### <a name="step-2-examine-the-message-details"></a>Шаг 2. Изучение сведений о сообщении

После изучения основ сообщения необходимо открыть сообщение, чтобы изучить сведения и определить дальнейшие действия по исправлению. Выберите сообщение, чтобы просмотреть полные сведения о заголовке и тексте сообщения. Доступно несколько различных представлений, которые помогут вам определить правильный курс действий:

- **Представление источника:** это представление является стандартным представлением сообщений, которое часто встречается на большинстве веб-платформ обмена сообщениями. Сведения о заголовке форматированы в обычном стиле, а текст сообщения поддерживает imbedded графические файлы и текст в виде оболочки слова.
- Представление **текста:** текстовое представление отображает текстовое представление сообщения только в виде строки и включает выделение ключевых слов в сообщениях и вложениях для терминов, которые соответствуют связанной политике соответствия коммуникации. Выделение ключевых слов поможет быстро сканировать длинные сообщения и вложения в интересующие их области. В некоторых случаях выделенное текст может быть только во вложениях для сообщений, которые соответствуют условиям политики. Внедренные файлы не отображаются, и нумеруемая строка этого представления полезна для ссылки на подробные сведения для нескольких рецензентов.
- **Представление "Аннотация":** это представление позволяет рецензентам добавлять аннотации непосредственно в сообщение, сохраненное в представлении сообщения.
- **История пользователей:** в представлении истории пользователей отображаются все остальные оповещения, созданные любой политикой соответствия коммуникации пользователю, отправляя сообщение.
- **Представление сведений о сообщении:** расширенный просмотр метаданных сообщения и сведений о конфигурации.
- **Уведомление об обнаружении** шаблона: многие действия, которые со временем подавляют и запугивая, и включают повторную повторную передачу экземпляров того же поведения пользователя. Уведомление *об обнаружении* шаблона отображается в сведениях оповещений и вызывает внимание к оповещению. Обнаружение шаблонов зависит от политики и оценивает поведение в течение последних 30 дней, когда отправитель отправляет как минимум два сообщения одному получателю. С помощью этого уведомления следователи и проверяющие могут определить повторяющееся поведение, чтобы оценить оповещение соответствующим образом.
- **Отображение представления**"Перевод": это представление автоматически преобразует текст оповещения  в язык, настроенный в параметре языка "Отображаемый язык" в подписке Microsoft 365 для каждого рецензента. Представление "Перевод" помогает расширить возможности анализа для организаций с многоязычными пользователями и устраняет необходимость в дополнительных службах перевода вне процесса проверки соответствия коммуникации требованиям. С помощью служб Microsoft Translate представление "Перевод" можно по мере необходимости отключить и отключить, а также поддерживать широкий спектр языков. Полный список поддерживаемых языков см. на [веб-сайте Microsoft Translator Languages.](https://www.microsoft.com/translator/business/languages/) Языки, перечисленные в *списке языков переводчика,* поддерживаются в представлении "Перевод".

    ![Элементы управления представлением сообщений соответствия требованиям к коммуникациям](../media/communication-compliance-message-views.png)

### <a name="step-3-decide-on-a-remediation-action"></a>Шаг 3. Принятие решения о действии по исправлению

После просмотра сведений о сообщении для оповещения можно выбрать несколько действий по исправлению:

- **Решение:** при выборе управления **"Разрешить"** сообщение  немедленно удаляется из очереди ожидающих оповещений, и к сообщению не могут быть предприняты дополнительные действия. Выбрав **"Разрешить",** вы, по сути, закрыли оповещение без дополнительной классификации, и его нельзя повторно открыть для дальнейших действий. Все разрешенные сообщения отображаются на вкладке **"Решение".**
- **Ложное** срабатывание: вы всегда можете разрешить сообщение как ложное срабатывание в любой момент во время рабочего процесса проверки сообщения. Ложное срабатывание означает, что оповещение не является действием или что оповещение было сгенерировано неправильно процессом оповещения. Сообщение не может быть повторно повторно открыто, и все сообщения о ложном срабатывке отображаются на вкладке **"Решение".**
- **Power Automate (предварительная версия):** использование потока Power Automate для автоматизации задач обработки оповещения. По умолчанию соответствие коммуникации  включает диспетчер уведомлений, если у пользователя есть шаблон потока оповещений о соответствии коммуникации требованиям, который проверяющих может использовать для автоматизации процесса уведомлений для пользователей с оповещениями о сообщениях. Дополнительные сведения о создании потоков Power Automate и [](communication-compliance-feature-reference.md#power-automate-flows) управлении потоками в соответствии с коммуникациями см. в справочной статье по обеспечению соответствия коммуникациям.
- **Пометить** как : пометить сообщение как  совместимое, не совместимое или как сомнительный, так как оно относится к политикам и стандартам организации.  Добавление тегов и комментариев тегов помогает фильтровать оповещения политики микрофильтров для эскалаций или в рамках других внутренних процессов проверки. После завершения маркировки вы также можете разрешить сообщение, чтобы переместить его из очереди ожидающих проверки.
- **Уведомление:** вы можете использовать управление **Notify,** чтобы назначить оповещение пользовательскому шаблону уведомления и отправить пользователю предупреждение. Выберите подходящий шаблон уведомлений, настроенный в  области параметров соответствия коммуникациям, и выберите "Отправить по электронной почте напоминание" пользователю, который отправил сообщение, и устранить проблему. 
- **Escalate**: Using the **Escalate** control, you can choose who else in your organization should review the message. Выберите из списка проверяющих, настроенных в политике соответствия требованиям для связи, чтобы отправить уведомление по электронной почте с запросом на дополнительную проверку оповещения о сообщении. Выбранный рецензент может использовать ссылку в уведомлении по электронной почте, чтобы перейти непосредственно к тем пунктам, которые им были отправлены на проверку.
- **Эскалация для исследования:** с помощью эскалации для контроля исследования можно создать новое дело [Advanced eDiscovery](overview-ediscovery-20.md) для одного или нескольких сообщений.  Вы предоставите имя и примечания для нового дела, и пользователь, отправивший сообщение, которое соответствует политике, автоматически будет назначен в качестве хранителя дела. Для управления делом дополнительные разрешения не требуются. Создание дела не устраняет или не создает новый тег для сообщения. Вы можете выбрать в общей сложности 100 сообщений при создании дела Advanced eDiscovery во время процесса устранения. Сообщения во всех каналах связи, отслеживаемом соответствием коммуникации, поддерживаются. Например, вы можете выбрать 50 чатов Microsoft Teams, 25 сообщений электронной почты Exchange Online и 25 сообщений Yammer при открытии нового дела Advanced eDiscovery для пользователя.
- **Улучшение классификации (предварительная версия):** оповещения, созданные на основе совпадений типов классификаторов, могут потребовать обратной связи, чтобы свести к минимуму ложные срабатывые в организации. Используйте **усовершенствование средства** контроля классификации, чтобы предоставить отзыв о том, является ли классификация соответствия коммуникации допустимой, или предложить другие обучаемые классификаторы для этого типа соответствия. Вы можете подтвердить, что классификаторы соответствуют или не совпадают, или предложить другим обучаемым классификаторам связать с этим типом активности оповещений в будущем.  

    1. Выберите сообщение из списка оповещений.
    2. Выберите многоязык и выберите **"Улучшить классификацию".**
    3. В области **отзывов "Подробные** классификаторы", если элемент действительно положительный, выберите **"Совпадение".**  Если элемент был неправильно включен в категорию как ложное срабатывное, выберите **"Не совпадать".**
    4. Если есть другой классификатор, который больше подходит для элемента, выберите его в списке "Предложить другие обучаемые **классификаторы".** Этот отзыв инициирует оценку элемента другим классификатором.

    > [!TIP]
    > Вы можете одновременно предоставить отзывы о нескольких пунктах, выбрав их все, а затем выбрав команду "Предоставить **подробные** отзывы" на панели команд.

    5. Choose **Send feedback** to send your evaluation of the **Match** and Not **a match** classifications and suggest other trainable classifiers. Когда вы предоставили классификатору 30 экземпляров обратной связи, он автоматически переучен. Переподготовка может занять от 1 до 4 часов. Классификаторы можно переподготовить только дважды в день.

    > [!IMPORTANT]
    > Эти сведения перейти к классификатору в клиенте, **они не будут возвращаться в Корпорацию Майкрософт.**

    Дополнительные информацию о переподготовке классификатора для обеспечения соответствия коммуникациям см. в статье о переподготовке классификатора в [статье о соответствии коммуникациям.](classifier-how-to-retrain-comms-compliance.md)

- **Удаление сообщения в Teams:** с помощью команды "Удалить сообщение" вы можете блокировать недопустимые сообщения и содержимое, выявленные в оповещениях из каналов Microsoft Teams, а также чаты 1:1 и групповые чаты.  Удалены сообщения и содержимое заменены подсказкой политики, объясняя, что она заблокирована, и политикой, применяемой к удалению из представления. Получателям предоставляется ссылка в подсказке политики, чтобы узнать больше о применимой политике и процессе проверки. Отправитель получает подсказку политики для заблокированного сообщения и содержимого, но может просмотреть сведения о заблокированных сообщениях и содержимом для контекста удаления.
