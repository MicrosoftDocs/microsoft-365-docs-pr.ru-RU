# <a name="view-process-module-results-in-advanced-ediscovery-classic"></a>Просмотр результатов модуля процесса в Advanced eDiscovery (классическая)

После  \> **инициации** процесса подготовки можно просмотреть ход выполнения и результаты. 
  
> [!NOTE]
> Чтобы можно было использовать Advanced eDiscovery, требуется подписка на Office 365 E3 с надстройкой Advanced Compliance или E5 для организации. Если у вас этого плана нет и вы хотите попробовать Advanced eDiscovery, можете [зарегистрироваться для получения пробной версии Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
## <a name="process-task-status"></a>Состояние задачи обработки

В **области** \> **"Подготовка результатов** процесса" на странице отображается текущее состояние (если процесс запущен) или последнее состояние задачи состояния процесса, как показано в \> следующем примере.
  
![Состояние задач для модуля обработки](../media/9430f9e7-a4dd-47c7-ac2e-2c6a60fc948b.png)
  
Отображаемая задача может отличаться в зависимости от выбранных параметров процесса. 
  
- **Inventory**: Advanced eDiscovery выполняет итерации по всем файлам, выбранным для процесса, и выполняет базовое сбор данных.
    
- **Расчет подписей:** вычисляет цифровые подписи MD5.
    
- **Извлечение составных** данных: рекурсивно извлекает внутренние или содержащиеся файлы из составных файлов (например, PST, ZIP, MSG). Извлеченные файлы хранятся в папке дела.
    
- **Синхронизация базы данных**: внутренний процесс базы данных.
    
- **Копирование файлов:** копирует файлы процесса. Эта задача всегда отображается, даже если выбран параметр "Расширенные файлы копирования".
    
- **Извлечение** текста: при нативных файлах Advanced eDiscovery извлекает текст из этих файлов с помощью DTSearch. Извлеченный текст этих файлов хранится в виде текстовых файлов в папке дела.
    
- **Обновление метаданных:** обрабатывает загруженные метаданные. 
    
- **Завершение:** внутренняя обработка, которая завершает данные загруженных файлов дел (например, определение ошибок и файлов успешного завершения). 
    
Состояние задачи: отображается после завершения задачи. Во время выполнения задач отображается продолжительность выполнения.
  
> [!NOTE]
> Завершенные задачи также могут включать итоги для файлов, которые завершили обработку, или файлы с ошибками. 
  
> [!TIP]
> «Отмена» предоставляет возможность отката, чтобы остановить выполнение процесса, а затем откатить к предыдущему набору данных или сохраненным обработанным данным. Откат очищает все обработанные данные. Если вы не хотите, чтобы обработанные данные были потеряны (например, вы планируете перезагрузить эти файлы), выберите в этом окне параметр "Отмена", чтобы не откатываться. 
  
## <a name="process-summary"></a>Сводка по процессам

В сводке "Подготовка процесса результатов процесса" отображается разбивка результатов загруженных файлов в соответствии с результатами успешной обработки файлов и \> \> \> ошибками.
  
Области отображают графику импортируемой статистики файлов следующим образом:
  
- **Сводка процесса накапливает** d: все файлы в данном случае.
    
- **Сводка последнего процесса:** файлы, загруженные из последнего сеанса или действия. 
    
- **Фамилия**: сведения о фамилии в случае (если таковые есть).
    
- Если **были** добавлены файлы seed, количество файлов с заданным количеством файлов будет указано для каждого вопроса. 
    
    Если не удалось **пометить файлы seed,** это также отмечено. 
    
- Если **добавлены предварительно** помеченные файлы, количество предварительно помеченных файлов будет указано для каждого вопроса, определенного для файлов. 
    
    Если не удалось  пометить предварительно помеченные файлы, это также отмечено. 
    
![Сводка по модулю обработки](../media/2086a691-9e3d-4117-beb2-a5c3a9a4cc94.png)
  
## <a name="process-summary-accumulated-and-last-charts"></a>Сводные сведения о процессах, собранные и последние диаграммы

На левой панели находятся исходные и извлеченные файлы— все найденные файлы. 
  
Правая г., обработанная, включает:
  
- Файлы с ошибками загрузки
    
- Успешно загруженные файлы, которые могут включать: 
    
  - **Существующие**: файлы, которые были загружены до и теперь снова загружены (включая дубликаты).
    
  - **Текст:** уникальные файлы с текстом.
    
  - **Нетектовые:** пустые текстовые файлы, пустые текстовые файлы, нетексовые файлы. 
    
  - **Дублирующиеся** файлы: дублирующиеся файлы с текстом.
    
## <a name="last-process-errors"></a>Ошибки последнего процесса

В области "Подготовка ошибок последнего процесса" отображаются сведения об ошибках последнего сеанса или \> \> \> выполненного действия.
