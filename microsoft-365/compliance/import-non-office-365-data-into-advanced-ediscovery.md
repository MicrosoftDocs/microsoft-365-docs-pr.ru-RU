# <a name="import-non-microsoft-365-content-for-advanced-ediscovery-classic-analysis"></a>Импорт содержимого, не относяного к Microsoft 365, для анализа Advanced eDiscovery (классическая)

Не все документы, которые вам может потребоваться проанализировать с помощью Advanced eDiscovery, будут размещены в Microsoft 365. С помощью функции импорта содержимого, не от microsoft 365 в Advanced eDiscovery, вы можете отправить документы, не хранимые в Microsoft 365 (кроме PST-файлов), в дело, связанное с хранилищем Azure, и проанализировать их с помощью Advanced eDiscovery. В этой процедуре показано, как свести документы, не относясь к Microsoft 365, в Advanced eDiscovery для анализа.
  
> [!NOTE]
> Чтобы можно было использовать Advanced eDiscovery, требуется подписка на Office 365 E3 с надстройкой Advanced Compliance или E5 для организации. Если у вас этого плана нет и вы хотите попробовать Advanced eDiscovery, можете [зарегистрироваться для получения пробной версии Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
> [!NOTE]
> Вы можете приобрести подписку на надстройку для хранения данных Advanced eDiscovery для контента за исключением Microsoft 365. Эта возможность доступна только для контента, который необходимо проанализировать с помощью Advanced eDiscovery. Выполните действия, которые необходимо предпринять при покупке или редактировании надстройки [для Microsoft 365](https://docs.microsoft.com/microsoft-365/commerce/buy-or-edit-an-add-on) для бизнеса, и приобрети надстройку для хранения Advanced eDiscovery. 
  
## <a name="requirements-to-upload-non-office-365-content"></a>Требования к отправке содержимого, не относяного к Office 365

Для использования функции отправки не из Office 365, как описано в этой процедуре, требуется:
  
- Надстройка Office 365 E3 с дополнительным соответствием требованиям или подписка E5.
    
- Все хранители, содержимое которых не из Office 365 будет загружено, должны иметь лицензии на надстройки E3 с расширенным соответствием требованиям или E5.
    
- Существующее дело eDiscovery.
    
- Все файлы для отправки собираются в папки, в которых на одного хранителя есть одна папка, а имя папки в этом формате *alias@domainname.* В  *alias@domainname*  должны быть пользователи с псевдонимом и доменом Office 365. Вы можете собрать  *все*  alias@domainname в корневую папку. Корневая папка может  содержать только alias@domainname, в ней не должно быть свободных файлов.
    
- Учетная запись, которая является либо диспетчером eDiscovery, либо администратором eDiscovery.
    
- [Средства хранения Microsoft Azure,](https://aka.ms/downloadazcopy) установленные на компьютере, который имеет доступ к структуре папок контента, не относячеству к Office 365. 
    
## <a name="upload-non-office-365-content-into-advanced-ediscovery"></a>Отправка не из Office 365 содержимого в Advanced eDiscovery


1. В качестве менеджера по обнаружению электронных данных или администратора eDiscovery откройте **eDiscovery** и откройте дело, в которое будут загружены данные, не относя такие как Office 365. Если вам нужно создать дело, см. "Управление делами [eDiscovery" в Центре соответствия &amp; требованиям безопасности.](ediscovery-cases.md)
    
2. Нажмите **кнопку "Переключиться на Advanced eDiscovery".**

3. Выберите **"Просмотр наборов"** в меню.

4. Выберите существующий набор для проверки или выберите **"Добавить набор для проверки".**

5. Выберите **"Управление набором для проверки".**

6. В карточке с данными, не относякой к Office 365, выберите **"Просмотреть отправки".**

7. Выберите **"Отправить файлы",** чтобы запустить мастер отправки файлов.

8. Первая вкладка — **1. Подготовка шага**. Выберите **"Далее": отправка файлов.**

9. На **2. На вкладке** "Отправка файлов" вам будет предложено скачать AzCopy.exe, если вы еще этого не сделали, а затем укать путь к расположению файла. Например, команда будет выполняться `C:\Upload`  AzCopy.exe. Используя, `C:\Upload` вы увидите:

   `"%ProgramFiles(x86)%\Microsoft SDKs\Azure\AzCopy\AzCopy.exe" /Source:"c:\upload" /Dest:"https://spnam03salinkexternal003.blob.core.windows.net/16d13440-a6a4-4bc5-a82b-10ac9cfe9d7c-1601401811-externalstore?sv=2017-07-29&sr=c&si=ExternalStore63%7C0&sig=9Dq5v20TwkxByYDHhIEx%2FHSLlmlqUjY0njkJyTO0zGA%3D" /s`
  
10. Откройте окно командной AzCopy.exe, чтобы импортировать данные в Azure. После загрузки всех данных выберите **"Далее: файлы обработки".**

11. Следующая вкладка **— 3. Обработать файлы,** в которых вы увидите хранителей с данными, связанными с ними, а также покажет ход выполнения импортируемых данных.
        
    Дополнительные сведения о синтаксис Azcopy см. в сведениях о передаче данных с [помощью AzCopy в Windows.](https://docs.microsoft.com/azure/storage/common/storage-use-azcopy) 
    
    Дополнительные сведения о advanced eDiscovery Processing см. в модуле "Процесс" и загрузке данных в [Advanced eDiscovery (классическая).](run-the-process-module-and-load-data-in-advanced-ediscovery.md) 
    
    > [!IMPORTANT]
    > На пользователя должна быть одна корневая папка, <b></b> имя папки должно быть в alias@domainname формате. 
   
    > [!IMPORTANT]
    > После успешной обработки контейнера в Advanced eDiscovery вы больше не сможете добавлять новое содержимое в хранилище SAS в Azure. Если вы собираете дополнительный контент и хотите добавить его в дело для анализа Advanced eDiscovery, необходимо создать новый контейнер данных, не относячий к **Office 365,** и повторить эту процедуру. 
  
    > [!NOTE]
    > Если контейнер  не будет успешно обработаться из-за проблем с именами папок, а затем устранить проблемы, вам все равно придется создать новый контейнер, повторно подключиться и снова отправить его, используя процедуры, которые данной статьи.
