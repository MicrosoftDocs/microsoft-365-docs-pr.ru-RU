# <a name="set-up-attorney-client-privilege-detection-in-advanced-ediscovery"></a>Настройка обнаружения привилегий между адвокатом и клиентом в Advanced eDiscovery

Основным и дорогостоящим аспектом на этапе проверки любого процесса проверки электронных документов является проверка документов для привилегированного контента. Advanced eDiscovery обеспечивает обнаружение привилегированного контента на основе машинного обучения, чтобы сделать этот процесс более эффективным. Эта функция называется *обнаружением привилегий между адвокатом и клиентом.*

## <a name="how-does-it-work"></a>Как это работает?

Когда будет включено обнаружение привилегий между адвокатом и клиентом, все документы в [](analyzing-data-in-review-set.md) наборе отзывов будут обрабатываться моделью обнаружения привилегий между адвокатом и клиентом при анализе данных в наборе отзывов. Модель ищет две вещи:

- Привилегированное содержимое — модель использует машинное обучение для определения вероятности того, что документ содержит легальный по своему характеру контент.

- Участники . В рамках настройки обнаружения привилегий между адвокатом и клиентом необходимо представить список адвокатов для вашей организации. Затем модель сравнивает соавторов документа со списком юристов, чтобы определить, есть ли среди соавторов хотя бы один адвокат.

Модель создает следующие три свойства для каждого документа:

- **AttorneyClientPrivilegeScore:** Вероятность того, что документ является юридическим по своему характеру; значения для оценки от **0** до **1**.

- **HasAttorney:** Это свойство имеет свойство **true,** если один из участников документа указан в списке адвоката; в противном случае значение является **ложным**. Значение также равно **false**, если ваша организация не определила список юристов.

- **IsPrivilege:** Это свойство имеет значение **true,** если значение **для AttorneyClientPrivilegeScore** превышает пороговое значение или если в документе есть участник адвокатского процесса;  в противном случае значение заведомо **ложное.**

Эти свойства (и их соответствующие значения) добавляются в метаданные файлов документов в наборе отзывов, как показано на следующем скриншоте:

![Свойства привилегий адвоката и клиента, показанные в метаданных файлов](../media/AeDAttorneyClientPrivilegeMetadata.png)

Эти три свойства также можно искать в наборе обзоров. Дополнительные сведения см. в обзоре Запрос данных в [наборе обзоров.](review-set-search.md)

## <a name="set-up-the-attorney-client-privilege-detection-model"></a>Настройка модели обнаружения привилегий между адвокатом и клиентом

Чтобы включить модель обнаружения привилегий между адвокатом и клиентом, организации необходимо включить ее и загрузить список адвокатов.

### <a name="step-1-turn-on-attorney-client-privilege-detection"></a>Шаг 1. Включив обнаружение привилегий между адвокатом и клиентом

Человек, который является администратором по обнаружению электронных данных в вашей организации (член подгруппы администратора по обнаружению электронных данных в группе ролей диспетчера электронных данных), должен сделать модель доступной в Advanced eDiscovery случаях.

1. В Центре & безопасности перейдите в **центр eDiscovery > Advanced eDiscovery.**

2. На **домашней Advanced eDiscovery** на Параметры  нажмите кнопку Настройка **параметров глобальной аналитики.**

   ![Выберите "Настройка экспериментальных функций"](../media/AeDExperimentalFeatures.png)

3. На **вкладке Параметры аналитики** выберите **параметр Управление привилегиями адвоката и клиента.**

4. На странице **Адвокатская тайна** включите эту функцию, а затем выберите **Сохранить**.

### <a name="step-2-upload-a-list-of-attorneys-optional"></a>Шаг 2. Upload список адвокатов (необязательно)

Чтобы в полной мере использовать модель обнаружения привилегий между адвокатом и клиентом и использовать результаты ранее описанного обнаружения has **attorney** или **Potentially Privileged,** рекомендуем загрузить список адресов электронной почты для юристов и юристов, которые работают в вашей организации. 

Чтобы загрузить список адвокатов для использования моделью обнаружения привилегий между адвокатом и клиентом:

1. Создайте CSV-файл (без строки заголовка) и добавьте каждый адрес электронной почты для каждого нужного человека в отдельной строке. Сохраните этот файл на своем компьютере.

2. На **домашней Advanced eDiscovery** на Параметры  выберите параметр **Configure experimental features**, а затем выберите параметр **Manage attorney-client privilege setting**.

   Отображается **страница привилегий "Адвокат-клиент"** и включена возможность обнаружения привилегий между адвокатом и клиентом. 

   ![Страница вылетов привилегий "Адвокат-клиент"](../media/AeDUploadAttorneyList.png)

3. Выберите **Обзор,** а затем найдите .csv файл, созданный на шаге 1.

4. Выберите **Сохранить,** чтобы загрузить список адвокатов.

## <a name="use-the-attorney-client-privilege-detection-model"></a>Использование модели обнаружения привилегий между адвокатом и клиентом

Следуйте шагам в этом разделе, чтобы использовать обнаружение привилегий адвоката и клиента для документов в наборе отзывов.

### <a name="step-1-create-a-smart-tag-group-with-attorney-client-privilege-detection-model"></a>Шаг 1. Создание группы смарт-тегов с моделью обнаружения привилегий между адвокатом и клиентом

Один из основных способов увидеть результаты обнаружения адвокатской тайны в ходе проверки — использование группы смарт-тегов. Группа смарт-тегов указывает результаты обнаружения адвокатской тайны и выводит их построчно рядом с тегами в группе смарт-тегов. Это позволяет быстро выявлять потенциально привилегированные документы во время проверки документов. Кроме того, теги в группе смарт-тегов можно использовать для пометки документов как конфиденциальных или не конфиденциальных. Дополнительные сведения о смарт-тегах см. в дополнительных сведениях о настройках [смарт-тегов в Advanced eDiscovery.](smart-tags.md)

1. В наборе обзоров, который содержит документы, проанализированые  в шаге 1, выберите управляйте набором обзоров и выберите **теги Manage**.
 
2. В **тегах** выберите тягу рядом с **группой Добавить,** а затем **выберите добавить группу смарт-тегов.**

   ![Выберите "Добавить группу смарт-тегов"](../media/AeDCreateSmartTag.png)

3. На странице **Выбор модели для умной** страницы тегов выберите **Выберите** рядом с привилегией **"Адвокат-клиент".**

   Отображается группа тегов **с именем "Адвокат-клиент".** Он содержит два детских тега **с именем Positive** и **Negative**, которые соответствуют возможным результатам, полученным моделью.

   ![Группа смарт-тегов привилегий "Адвокат-клиент"](../media/AeDAttorneyClientSmartTagGroup.png)

3. Переименуйте группу тегов и теги, соответствующие вашему обзору. Например, можно переименовать Positive в **Privileged** и **Negative** to **Not privileged**. 

### <a name="step-2-analyze-a-review-set"></a>Шаг 2. Анализ набора отзывов

При анализе документов в наборе отзывов будет также работать модель обнаружения привилегий между адвокатом и клиентом, а соответствующие свойства (описанные в описании How [does it work?](#how-does-it-work) будут добавлены к каждому документу в наборе обзоров. Дополнительные сведения об анализе данных в наборе обзоров см. в обзоре Анализ данных в наборе [Advanced eDiscovery.](analyzing-data-in-review-set.md)

### <a name="step-3-use-the-smart-tag-group-for-review-of-privileged-content"></a>Шаг 3. Используйте группу смарт-тегов для просмотра привилегированного контента

После анализа набора отзывов и настройки смарт-тегов следующим шагом является проверка документов. Если модель определила, что документ потенциально привилегированный, соответствующий смарт-тег на панели тегов будет указывать следующие результаты, полученные при обнаружении привилегий между адвокатом и клиентом: 

- Если в документе есть содержимое, которое может быть законным по своему характеру, метка **Legal content** отображается рядом с соответствующим смарт-тегом (который в данном случае является тегом **Positive по** умолчанию).

- Если в документе есть участник, найденный в списке  адвоката организации, метка Attorney отображается рядом с соответствующим  смарт-тегом (который в данном случае также является положительным тегом по умолчанию).

- Если в документе содержится содержимое, которое может быть законным по  своему характеру и имеет участника, найденного в списке адвоката, отображаются как юридический контент, так и метки Attorney.   

Если модель определяет, что документ не содержит содержимого, которое является законным по своему характеру или не содержит участника из списка адвокатов, то ни одна метка не отображается на панели тегов.

Например, на следующих скриншотах покажут два документа. Первый содержит содержимое, которое является законным по своему характеру и содержит участника в списке адвокатов. Второй не содержит ни того, ни другго, и поэтому не отображает метки.

![Документ с метами адвокатского и юридического контента](../media/AeDTaggingPanelLegalContentAttorney.png)

![Документ без меток](../media/AeDTaggingPanelNegative.png)

После просмотра документа, чтобы узнать, содержится ли в нем привилегированное содержимое, можно пометить документ соответствующим тегом.
