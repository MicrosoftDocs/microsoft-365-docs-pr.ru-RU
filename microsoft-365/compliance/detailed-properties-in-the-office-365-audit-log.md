# <a name="detailed-properties-in-the-audit-log"></a>Подробные свойства в журнале аудита

При экспорте результатов поиска журнала аудита из центра безопасности & соответствия требованиям можно скачать все результаты, соответствующие условиям поиска. Для этого выберите пункт **Экспорт результатов** \> **скачать все результаты** на странице **Поиск журнала аудита** . Дополнительные сведения см в разделе [Поиск в журнале аудита](search-the-audit-log-in-security-and-compliance.md).
  
 При экспорте результатов поиска журнала аудита необработанные данные из единого журнала аудита копируются в файл значений с разделителями-запятыми (CSV), который загружается на локальный компьютер. Этот файл содержит дополнительные сведения из каждой записи аудита в столбце с именем **аудитдата**. Этот столбец содержит свойство с несколькими значениями для нескольких свойств из записи журнала аудита. Каждая пара **свойство: значение** в этом многозначном свойстве отделяется запятыми. 
  
В следующей таблице описываются включенные свойства (в зависимости от службы, в которой происходит событие) в столбце **аудитдата** с несколькими свойствами. **Служба Office 365 с этим** столбцом свойства указывает службу и тип действия (пользователя или администратора), который включает свойство. Для получения более подробных сведений об этих свойствах или о свойствах, которые могут не переводиться в этой статье, ознакомьтесь со статьей [схема API действий управления](https://go.microsoft.com/fwlink/p/?LinkId=717993).
  
> [!TIP]
> Можно использовать функцию преобразования JSON в Power Query в Excel, чтобы разделить столбец **аудитдата** на несколько столбцов, чтобы каждое свойство было иметь собственный столбец. Это позволит выполнять сортировку и фильтрацию по одному или нескольким из этих свойств. Сведения о том, как это сделать, можно найти в статье [Экспорт, Настройка и просмотр записей журнала аудита](export-view-audit-log-records.md).

|Workload|Служба Microsoft 365, в которой возникло действие.|Все|
||||

> [!NOTE]
><sup>1</sup> для событий, связанных с Azure Active Directory, значение для администратора не используется в записи аудита. Записи аудита для действий, выполняемых администраторами, показывают, что обычный пользователь (например, **usertype: 0**) выполнил действие. Свойство **UserID** будет определять пользователя (обычного пользователя или администратора), выполнившего действие.<br/>

Описанные выше свойства также отображаются при просмотре сведений о конкретном событии по нажатию кнопки **Дополнительные сведения** .
