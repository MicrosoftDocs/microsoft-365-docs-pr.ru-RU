# <a name="insider-risk-management-content-explorer"></a>Обозреватель содержимого для управления рисками для оценки рисков

Обозреватель контента для управления insider risk management позволяет пользователям, которые назначены роли *"Insider Risk Management Researchers",* проверять контекст и сведения о содержимом, связанном с действиями в оповещениях. Для всех оповещений, которые подтверждены для дела, копии данных и файлов сообщений архивируется как моментальный снимок во время элементов, сохраняя исходные файлы и сообщения в источниках хранения. Копирование данных и сообщений прозрачно для пользователя, связанного с оповещением, и для владельца содержимого. Если контент содержит разрешения на управление правами на доступ к данным, эти разрешения сохраняются для скопированного контента, и пользователям, для пользователей с назначенной ролью *"Insider Risk Management Investigators",* необходимы эти разрешения и права, если им нужно открыть и просмотреть файлы. Каждому файлу и сообщению автоматически назначен уникальный ИД файла в случае управления рисками внутри организации в целях управления. Документы, связанные с действиями индикаторов устройств, не включаются в обозреватель содержимого.

## <a name="column-options"></a>Параметры столбцов

Чтобы упростить анализаторам рисков и следователям просмотр захваченных данных и сообщений и контекст дела, в обозреватель содержимого включено несколько средств фильтрации и сортировки. Для базовой сортировки столбцы класса **Date** и **File** поддерживают сортировку с помощью заголовков столбцов в области очереди содержимого. Другие столбцы очереди можно добавить в представление для предоставления различных севовидов для файлов и сообщений.

## <a name="advanced-search-conditions"></a>Расширенные условия поиска

Можно добавить условия поиска, чтобы сузить область поиска и получить более уточненный набор результатов. Каждое условие добавляет предложение к поисковому запросу, которое создается и запускается в начале поиска. Условие логически подключается к запросу ключевого слова (указанному в поле ключевого слова) логическим оператором (представленным как c:c), который по функциональности аналогичен оператору AND. Это означает, что элементы должны удовлетворять как запросу по ключевому слову, так и одному или одному или одному из условий, которые должны быть включены в результаты поиска. Эта функция помогает сузить результаты с помощью условий.

Для расширенных средств фильтрации  и поиска раздвигает области фильтров в левой части очереди контента. Выберите **кнопку "Добавить условие",** чтобы открыть список условий:

### <a name="operators-used-with-conditions"></a>Операторы, используемые с условиями

|**Описание**|**Эквивалент запроса**|**Описание**|
|:-----------|:-------------------|:--------------|
| **After** |`property>date`| Используется с условиями даты. Возвращает элементы, отправленные, полученные или измененные после указанной даты.|
| **Before** |`property<date`| Используется с условиями даты. Возвращает элементы, отправленные, полученные или измененные до указанной даты.|
| **Между** |`date..date`| Используется с условиями даты и размера. При использовании с условием даты возвращает элементы, отправленные, полученные или измененные в указанный временной период. При использовании с условием размера возвращает элементы, размер которых находится в заданном диапазоне.|
| **Содержит все** |`(property:value) OR (property:value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые содержат все одно или несколько указанных строкных значений. |
| **Contains any of** |`(property:value) OR (property:value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые содержат любую часть одного или нескольких указанных строковых значений.|
| **Не содержит ни одного из** |`-property:value`  <br/> `NOT property:value`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не содержат ни одной части указанного строкового значения.|
| **Не равно любому из** |`-property=value`  <br/> `NOT property=value`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не содержат определенную строку.|
| **Равно** |`size=value`| Возвращает элементы, которые равны указанному размеру. <sup>1</sup>|
| **Equals any of** |`(property=value) OR (property=value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые полностью совпадают с одним или несколькими указанными строковыми значениями.|
| **Равно ни одному из** |`(property=value) OR (property=value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не соответствуют одному или более указанным строкам. |
| **Больше** |`size>value`| Возвращает элементы, у которых указанное свойство больше указанного значения. <sup>1</sup>|
| **Greater or equal** |`size>=value`| Возвращает элементы, у которых указанное свойство больше или равно указанному значению. <sup>1</sup>|
| **Меньше** |`size<value`| Возвращает элементы, которые больше или равны определенному значению. <sup>1</sup>|
| **Less or equal** |`size<=value`| Возвращает элементы, которые больше или равны определенному значению. <sup>1</sup>|
| **Not equal** |`size<>value`| Возвращает элементы, не равные указанному размеру. <sup>1</sup>|

> [!NOTE]
> <sup>1</sup> Этот оператор доступен только для условий, в которых используется свойство Size.
