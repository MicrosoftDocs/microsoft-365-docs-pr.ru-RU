# <a name="insider-risk-management-content-explorer"></a>Обозреватель контента для управления рисками

Обозреватель контента  управления рисками инсайдерской работы позволяет пользователям, на которые назначена роль *исследователей* по управлению рисками, для изучения контекста и сведений о контенте, связанном с действиями в оповещении. Данные о случаях в обозревателе контента обновляются ежедневно, чтобы включить новые действия. Для всех оповещений, подтвержденных в случае, копии данных и файлов сообщений архивируется в качестве снимка во время элементов при сохранении исходных файлов и сообщений в источниках хранения. При необходимости файлы данных дел могут экспортироваться как переносной файл документов (PDF) или в исходном формате файла.

Копирование данных и сообщений является прозрачным для пользователя, связанного с оповещением, и для владельца контента. В новых случаях заполнение контента в обозревателе контента обычно занимает около часа. Для случаев с большим количеством контента может потребоваться больше времени для создания снимка. Если содержимое по-прежнему загружается в обозревателе контента, вы увидите индикатор прогресса, отображающий процент завершения.

В некоторых случаях данные, связанные с случаем, могут быть недоступны в качестве снимка для просмотра в обозревателе контента. Такая ситуация может возникнуть при удалении или перемещении данных дела, а также при временной ошибке при обработке данных. Если такая ситуация возникает, выберите **Просмотр** файлов в панели предупреждений, чтобы просмотреть имена файлов, путь к файлам и причину сбоя для каждого файла. При необходимости эти сведения можно экспортировать в .csv (разделенные запятой значения).

Если контент содержит разрешения на управление правами на информацию, эти разрешения сохраняются  для скопированного контента, и пользователям, на которые назначена роль исследователей по управлению рисками, необходимы эти разрешения и права, если им необходимо открыть и просмотреть файлы. Каждому файлу и сообщению автоматически назначен уникальный файл ID в случае управления рисками для целей управления. Документы, связанные с действиями индикатора устройства, не включаются в обозреватель контента.

>[!Note]
>Обозреватель контента включает действия, связанные с Microsoft Office файлами. Действия на уровне сайта, такие как удаление SharePoint сайта или изменения разрешений сайта, не включаются в обозреватель контента.

## <a name="column-options"></a>Параметры столбцов

Чтобы аналитикам и следователям по рискам было проще просмотреть захваченные данные и сообщения и просмотреть контекст дела, в обозреватель контента включены несколько средств фильтрации и сортировки. Для базовой сортировки столбцы классов **Date** и **File** поддерживают сортировку с помощью заголовков столбцов в области очереди контента. Другие столбцы очереди доступны для добавления в представление для предоставления разных поворотов в файлах и сообщениях.

Чтобы добавить или удалить заголовки столбцов для очереди контента, используйте управление **столбцами Редактирование** и выберите из следующих параметров столбца. Эти столбцы соотносят с общими условиями свойств электронной почты и документов, поддерживаемых в обозревателе контента и перечисленных ниже в этой статье.

|:------------------|:----------------|
| **Author** | Поле автора в документах Microsoft Office, которое сохраняется при копировании документа. Например, если пользователь создает документ и передает его другому пользователю, который затем загружает его в SharePoint, документ по-прежнему сохраняет исходный автор. |
| **Bcc** | Доступные для сообщений электронной почты, пользователи в поле сообщений Bcc. |
| **Cc** | Доступные для сообщений электронной почты, пользователи в поле сообщений Cc. |
| **Сложный путь** | Понятный для человека путь, описываемый источником элемента. |
| **ID беседы** | Id беседы из сообщения. |
| **Индекс разговоров** | Индекс беседы из сообщения. |
| **Время создания.** | Время создания файла или сообщения электронной почты. |
| **Дата (UTC)** | Для электронной почты: дата получения сообщения адресатом или его отправки отправителем. Для документов дата последнего изменения документа. Дата находится в согласованном универсальном времени (UTC).|
| **Доминирующая тема** | Доминирующая тема, рассчитанная для аналитики. |
| **ID набора электронной почты** | Групповой ID для всех сообщений в одном наборе электронной почты. |

| **Класс File** | Для контента из SharePoint и OneDrive: **Document;** для контента из Exchange: **электронная почта** или **вложение**. |
| **Файл ID** | Идентификатор документа, уникальный в данном случае. |
| **Значок типа файла** | Расширение файла; например, docx, one, pptx или xlsx. Это поле является тем же свойством, что и свойство сайта FileExtension. |
| **ID** | Идентификатор GUID для файла. |

| **Включительно тип** | Инклюзивный тип, рассчитанный для аналитики: **0** — не включено; **1** — включительно; **2** — включительно минус; **3** — включительно. |

| **Тип сообщения** | Тип сообщения электронной почты для поиска. Возможные значения: контакты, документы, электронная почта, внешние данные, факсы, im, журналы, собрания, группы Майкрософт (возвращает элементы из чатов, собраний и вызовов в Microsoft Teams), заметки, сообщения, RSS-каналы, задачи, голосовая почта |
| **Участники** | Список всех участников сообщения; например, Sender, To, Cc, Bcc. |

| **Размер** | Для электронной почты и документов: размер элемента (в байтах). |
| **Тема** | Текст в строке темы сообщения электронной почты. |
| **Subject/Title** | Для электронной почты: текст в строке темы сообщения. Для документов: заголовок документа. Как объяснялось ранее, свойство Title — это метаданные, указанные в Microsoft Office документах. Вы можете ввести имя более чем одного субъекта или названия, разделенных запятой. Два или более значений, логически соединенных с помощью оператора OR. |
| **Список тем** | Список тем, рассчитанный для аналитики. |
| **Title** | Заголовок документа. Свойство Title — это метаданные, указанные в документах Office. Оно отличается от имени файла документа. |
| **To** | Получатель сообщения электронной почты в поле To. |

## <a name="filtering"></a>Фильтрация

Вы можете использовать один или несколько фильтров, чтобы сузить область поиска и вернуть более уточненный набор результатов. Чтобы установить фильтр, **выберите фильтры** в верхней части очереди контента. Многие фильтры включают дополнительные параметры фильтрации, чтобы помочь сузить результаты, возвращенные фильтром. Например, фильтр *Date* включает элементы управления  для настройки даты начала и *даты* окончания для **фильтра Date.** Выберите один или несколько элементов фильтра из следующих категорий:

### <a name="common-filters"></a>Общие фильтры

| **Фильтр** | **Описание** |
|:---------------------|:----------------|
| **Дата (UTC)** | Для электронной почты: дата получения сообщения адресатом или его отправки отправителем. Для документов дата последнего изменения документа. |
| **Отправитель/автор** | Для электронной почты: отправитель сообщения. Для документов человек, приведенный в поле *Автор* из Office документов. Можно ввести несколько имен, разделенных запятой. |
| **Source** | Расположение документа в организации. Например, определенное расположение SharePoint сайта. |
| **Subject/Title** | Для электронной почты: текст в строке темы сообщения. Для документов: заголовок документа. Свойство Title в документах — метаданные, указанные в Microsoft Office документах. Вы можете ввести имя более чем одного субъекта или названия, разделенных запятой. Два или более значений, логически соединенных с помощью оператора OR. |

### <a name="email-filters"></a>Фильтры электронной почты

| **Фильтр** | **Описание** |
|:---------------------|:----------------|
| **Bcc** | Поле Bcc сообщения электронной почты. |
| **Cc** | Поле Cc сообщения электронной почты. |
| **Имеет вложение** | Указывает, есть ли у сообщения вложение. Значения перечислены как **true или** **false**. |
| **Является ли вложение электронной почты** | Если документ является вложением, значение перечислены как **Да**. |
| **Встроенный документ** | Если документ встроен в сообщение электронной почты, значение перечислены как **Да**. |
| **Является ли вложение вложением в линию** | Если документ является вложением в почтовое сообщение, значение перечислены как **Да**. |
| **Участники** | Все поля людей в сообщении электронной почты. Это поля From, To, Cc и Bcc. |
| **Received** | Дата получения сообщения адресатом. |
| **Домены получателей** | Список всех доменов получателей сообщения. |
| **Recipients** | Получатели сообщений электронной почты. |
| **Sender** | Поле отправитель (From) для типов сообщений.  Формат **DisplayName \<SmtpAddress>**. |
| **Домен отправителя** | Домен отправитель. |
| **To** | Поле "Кому" электронного письма. |
| **Уникальный в наборе электронной почты** | False, если в наборе электронной почты есть дубликат вложения. |

## <a name="document-filters"></a>Фильтры документов

| **Filters** | **Описание** |
|:---------------------|:----------------|
| **Метки соответствия требованиям** | Метки соответствия требованиям, применяемые в Office 365. |
| **Созданное время (UTC)** | Дата и время создания файла или сообщения электронной почты. Дата и время находятся в согласованном универсальном времени (UTC). |
| **Последняя измененная дата (UTC)** | Дата последнего изменения документа. Дата и время находятся в согласованном универсальном времени (UTC). |
| **Расширение файла** | Тип расширения файла. |
| **События действий пользователя** | Действие для элементов, связанных с определенной активностью пользователя в конкретном случае.  Например, при выборе ссылки на "Изучение контента"  для действия на странице Действия пользователя в случае этот фильтр используется для отображения элементов, связанных с этим действием. |
| **Рабочий продукт** | Тип продукта для документа. Например, аннотации или теги в документе. |
