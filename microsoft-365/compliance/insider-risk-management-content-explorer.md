# <a name="insider-risk-management-content-explorer"></a>Обозреватель контента для управления рисками

Обозреватель контента  управления рисками инсайдерской работы позволяет пользователям, на которые назначена роль *исследователей* по управлению рисками, для изучения контекста и сведений о контенте, связанном с действиями в оповещении. Данные о случаях в обозревателе контента обновляются ежедневно, чтобы включить новые действия. Для всех оповещений, подтвержденных в случае, копии данных и файлов сообщений архивируется в качестве снимка во время элементов при сохранении исходных файлов и сообщений в источниках хранения. Копирование данных и сообщений является прозрачным для пользователя, связанного с оповещением, и для владельца контента. В новых случаях заполнение контента в обозревателе контента обычно занимает около часа. Для случаев с большим количеством контента может потребоваться больше времени для создания снимка. Если содержимое по-прежнему загружается в обозревателе контента, вы увидите индикатор прогресса, отображающий процент завершения.

В некоторых случаях данные, связанные с случаем, могут быть недоступны в качестве снимка для просмотра в обозревателе контента. Такая ситуация может возникнуть при удалении или перемещении данных дела, а также при временной ошибке при обработке данных. Если такая ситуация возникает, выберите **Просмотр** файлов в панели предупреждений, чтобы просмотреть имена файлов, путь к файлам и причину сбоя для каждого файла. При необходимости эти сведения можно экспортировать в файл csv (разделенные запятой).

Если контент содержит разрешения на управление правами на информацию, эти разрешения сохраняются  для скопированного контента, и пользователям, на которые назначена роль исследователей по управлению рисками, необходимы эти разрешения и права, если им необходимо открыть и просмотреть файлы. Каждому файлу и сообщению автоматически назначен уникальный файл ID в случае управления рисками для целей управления. Документы, связанные с действиями индикатора устройства, не включаются в обозреватель контента.

![Обозреватель контента для управления рисками](../media/insider-risk-content-explorer.png)

>[!Note]
>Обозреватель контента включает действия, связанные с Microsoft Office файлами. Действия на уровне сайта, такие как удаление сайта SharePoint или изменения разрешений на сайт, не включаются в обозреватель контента.

## <a name="column-options"></a>Параметры столбцов

Для базовой сортировки столбцы классов **Date** и **File** поддерживают сортировку с помощью заголовков столбцов в области очереди контента. Другие столбцы очереди доступны для добавления в представление для предоставления разных поворотов в файлах и сообщениях.

Чтобы добавить или удалить заголовки столбцов для очереди контента, используйте управление **столбцами Редактирование** и выберите из следующих параметров столбца. ## <a name="advanced-search-conditions"></a>Расширенные условия поиска

Можно добавить условия поиска, чтобы сузить область поиска и вернуть более уточненный набор результатов. Каждое условие добавляет предложение к поисковому запросу, которое создается и запускается в начале поиска. Условие логически подключено к запросу ключевого слова (указанному в поле ключевого слова) логическим оператором (который представлен как c:c), схожим по функциональности с оператором AND. Это означает, что элементы должны удовлетворять как ключевому запросу, так и одному или более условиям, которые будут включены в результаты поиска. Эти функции помогают сузить результаты.

Для расширенных средств фильтрации и поиска раздвигаем области **фильтра** на левой стороне очереди контента. Выберите **кнопку Добавить условие,** чтобы открыть список условий:

### <a name="operators-used-with-conditions"></a>Операторы, используемые с условиями

|**Описание**|**Эквивалент запроса**|**Описание**|
|:-----------|:-------------------|:--------------|
| **After** |`property>date`| Используется с условиями даты. Возвращает элементы, отправленные, полученные или измененные после указанной даты.|
| **Before** |`property<date`| Используется с условиями даты. Возвращает элементы, отправленные, полученные или измененные до указанной даты.|
| **Между** |`date..date`| Используется с условиями даты и размера. При использовании с условием даты возвращает элементы, отправленные, полученные или измененные в указанный временной период. При использовании с условием размера возвращает элементы, размер которых находится в заданном диапазоне.|
| **Содержит все** |`(property:value) OR (property:value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, содержащие все одно или несколько заданных значений строки. |
| **Contains any of** |`(property:value) OR (property:value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые содержат любую часть одного или нескольких указанных строковых значений.|
| **Не содержит ни одного** |`-property:value`  <br/> `NOT property:value`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не содержат ни одной части указанного строкового значения.|
| **Не равно любому из** |`-property=value`  <br/> `NOT property=value`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не содержат определенную строку.|
| **Equals** |`size=value`| Возвращает элементы, равные указанному размеру. <sup>1</sup>|
| **Equals any of** |`(property=value) OR (property=value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые полностью совпадают с одним или несколькими указанными строковыми значениями.|
| **Равно ни одному из** |`(property=value) OR (property=value)`| Используется с условиями для свойств, определяющих строковые значения. Возвращает элементы, которые не соответствуют одному или более заданным значениям строки. |
| **Больше, чем** |`size>value`| Возвращает элементы, в которых указанное свойство больше указанного значения. <sup>1</sup>|
| **Greater or equal** |`size>=value`| Возвращает элементы, в которых указанное свойство больше или равно указанному значению. <sup>1</sup>|
| **Меньше, чем** |`size<value`| Возвращает элементы, которые больше или равны определенному значению. <sup>1</sup>|
| **Less or equal** |`size<=value`| Возвращает элементы, которые больше или равны определенному значению. <sup>1</sup>|
| **Not equal** |`size<>value`| Возвращает элементы, не равные указанному размеру. <sup>1</sup>|

> [!NOTE]
> <sup>1</sup> Этот оператор доступен только для условий, в которых используется свойство Size.

| **Класс File** | Для контента из SharePoint и OneDrive: **Документ;** для контента из Exchange: **Email или **Attachment**. |
| **Типы файлов** | Расширение файла; например, docx, one, pptx или xlsx. |
| **Имеет участника адвокат** | True, если хотя бы один из участников находится в списке адвокатов; в противном случае значение false. |
| **Неизменяемый идентификатор** | Неопроменяемый Id, хранимый в Office 365. |
| **Включительно тип** | Инклюзивный тип, рассчитанный для аналитики: **0** — не включено; **1** — включительно; **2** — включительно минус; **3** — включительно. |
| **Класс Item** | Класс item, поставляемого сервером exchange; например, **IPM. Примечание** |
| **Дата последнего изменения** | Дата последнего изменения документа. |
| **Load ID** | Загрузи id, в котором элемент был загружен в набор отзывов. |
| **Имя расположения** | Строка, определяемая источником элемента.  Для exchange это поле будет SMTP-адресом почтового ящика. Для SharePoint и OneDrive URL-адрес коллекции сайтов. |
| **Помечено как представительное** | Один документ из каждого набора точных дубликатов помечен как представители. |
| **Расширение файлов native** | Родной расширение элемента. |
| **Имя файла native** | Имя родного файла элемента. |
| **NdEtSortExclAttach** | Concatenation набора электронной почты и набора ND для эффективной сортировки во время проверки; D добавляется в качестве префикса в наборы ND, а E добавляется в наборы электронной почты. |
| **Pivot ID** | ID поворота. |
| **Потенциально привилегированные** | True, если модель обнаружения привилегий между адвокатом и клиентом считает документ потенциально привилегированным. |
| **Состояние обработки** | Состояние обработки после того, как элемент был добавлен в набор отзывов. |
| **Процентил чтения** | Чтение процентали для документа на основе релевантности. |
| **Оценка релевантности** | Оценка релевантности документа на основе релевантности. |
| **Тег релевантности** | Оценка релевантности документа на основе релевантности. |
| **ID представителя** | Числимый идентификатор каждого набора точных дубликатов. |
| **Tags** | Теги, применяемые в наборе отзывов. |
| **Список тем** | Список тем, рассчитанный для аналитики. |
| **Название** | Заголовок документа. Свойство Title — это метаданные, указанные в документах Office. Оно отличается от имени файла документа. |
| **Исправлено** | True, если элемент был исправлен, в противном случае False. |
| **Статистика** | Количество слов в файле. |
