# <a name="manage-mailbox-auditing"></a>Управление аудитом почтовых ящиков

С января 2019 г. корпорация Майкрософт по умолчанию включит ведение журнала аудита почтовых ящиков для всех организаций. Это означает, что некоторые действия, выполняемые владельцами, делегатами и администраторами почтовых ящиков, автоматически записываются в журнал аудита почтовых ящиков, а соответствующие записи аудита почтовых ящиков будут доступны при их поиске в журнале аудита почтовых ящиков. Прежде чем аудит почтовых ящиков был включен по умолчанию, необходимо было вручную включить его для каждого почтового ящика пользователя в организации.

Вот некоторые преимущества аудита почтовых ящиков по умолчанию:

- Аудит автоматически включен при создании нового почтового ящика. Вам не нужно вручную включить его для новых пользователей.

- Вам не нужно управлять действиями почтовых ящиков, которые проверяются. Стандартный набор действий почтового ящика по умолчанию проверяется для каждого типа (администратора, делегата и владельца).

- Когда корпорация Майкрософт выпускает новое действие почтового ящика, это действие может быть автоматически добавлено в список действий почтового ящика, которые проверяются по умолчанию (при условии, что у пользователя есть соответствующая лицензия). Это означает, что вам не нужно отслеживать добавление новых действий в почтовых ящиках.

- В организации имеется согласованная политика аудита почтовых ящиков (так как вы проводите аудит одинаковых действий для всех почтовых ящиков).

> [!NOTE]
>* Важно помнить о выпуске аудита почтовых ящиков по умолчанию: вам не нужно ничего делать для управления аудитом почтовых ящиков. Тем не менее, чтобы узнать больше, настройте аудит почтовых ящиков с помощью параметров по умолчанию или вообще отключите его, этот раздел поможет вам.
>- По умолчанию только события аудита почтовых ящиков для пользователей E5 доступны при поиске в журнале аудита в Центре безопасности & соответствия требованиям или через API действий управления Office 365. Дополнительные сведения см. в [разделе](#more-information) "Дополнительные сведения" этой статьи.

## <a name="verify-mailbox-auditing-on-by-default-is-turned-on"></a>Проверка того, что аудит почтовых ящиков включен по умолчанию

Чтобы убедиться, что аудит почтовых ящиков по умолчанию включен для вашей организации, в [Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/connect-to-exchange-online-powershell)запустите следующую команду:

```PowerShell
Get-OrganizationConfig | Format-List AuditDisabled
```

Значение **False** указывает, что аудит почтовых ящиков включен по умолчанию для организации. Это значение по умолчанию переопределяет параметр аудита почтовых ящиков для определенных почтовых ящиков. Например, если для почтового ящика отключен аудит почтовых ящиков (для почтового ящика свойство *AuditEnabled* имеет значение **False),** для почтового ящика будут по-прежнему проверяться действия почтового ящика по умолчанию, так как аудит почтовых ящиков по умолчанию включен для организации.

Чтобы отключить аудит почтовых ящиков для определенных почтовых ящиков, необходимо настроить обход аудита для владельца почтового ящика и других пользователей, которым делегирован доступ к почтовому ящику. Дополнительные сведения см. [в](#bypass-mailbox-audit-logging) разделе "Обход журнала аудита почтовых ящиков" этой статьи.

> [!NOTE]
> Если аудит почтовых ящиков включен по умолчанию для организации, свойство *AuditEnabled* для затронутых почтовых ящиков не будет изменено с **False** на **True.** Другими словами, при аудите почтовых ящиков по умолчанию игнорируется свойство *AuditEnabled* для почтовых ящиков.

## <a name="logon-types-and-mailbox-actions"></a>Типы и действия почтовых ящиков для логотипа

Типы для входящих данных классифицируют пользователя, выдав проверенные действия в почтовом ящике. В следующем списке описываются типы входа, используемые в журнале аудита почтовых ящиков:

- **Владелец**: владелец почтового ящика (учетная запись, связанная с почтовым ящиком).

- **Делегат:**

  - Пользователь, которому назначено разрешение SendAs, SendOnBehalf или FullAccess для другого почтового ящика.

  - Администратор, которому назначено разрешение на полный доступ к почтовому ящику пользователя.

- **Администратор**:

  - Поиск в почтовом ящике ведется с помощью одного из следующих средств microsoft eDiscovery:

    - Поиск контента в Центре соответствия требованиям.

    - eDiscovery или Advanced eDiscovery в Центре соответствия требованиям.

    - In-Place eDiscovery в Exchange Online.

  - Доступ к почтовому ящику можно получить с помощью редактора Microsoft Exchange Server MAPI.

### <a name="verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type"></a>Проверка записи действий почтового ящика по умолчанию для каждого типа входа

Аудит почтовых ящиков по умолчанию добавляет новое свойство *DefaultAuditSet* во все почтовые ящики. Значение этого свойства указывает, проверяются ли действия почтового ящика по умолчанию (управляемые корпорацией Майкрософт) для почтового ящика.

Чтобы отобразить значение для почтовых ящиков пользователей или общих почтовых ящиков, замените имя, псевдоним, адрес электронной почты или имя пользователя-пользователя (имя пользователя) почтового ящика и запустите следующую команду в \<MailboxIdentity\> Exchange Online PowerShell:

```PowerShell
Get-Mailbox -Identity <MailboxIdentity> | Format-List DefaultAuditSet
```

Чтобы отобразить значение в почтовых ящиках группы Microsoft 365, замените имя, псевдоним или адрес электронной почты общего почтового ящика и запустите следующую команду в \<MailboxIdentity\> Exchange Online PowerShell:

```PowerShell
Get-Mailbox -Identity <MailboxIdentity> -GroupMailbox | Format-List DefaultAuditSet
```

Значение `Admin, Delegate, Owner` указывает:

- Проверяются действия почтовых ящиков по умолчанию для всех трех типов. Это единственное значение, которые вы увидите в почтовых ящиках группы Microsoft 365.

- Администратор не *изменил проверенные* действия почтового ящика для любого типа для пользователя или общего почтового ящика. Обратите внимание, что это состояние по умолчанию после первоначального отключения аудита почтовых ящиков в организации.

Если администратор когда-либо изменял действия почтового ящика, которые проверяются для типа для учетных сообщений (с помощью параметров *AuditAdmin,* *AuditDelegate* или *AuditOwner* в cmdlet **Set-Mailbox),** значение свойства будет другим.

Например, значение свойства `Owner` *DefaultAuditSet* для почтового ящика пользователя или общего почтового ящика указывает:

- Действия почтового ящика по умолчанию для владельца почтового ящика проверяются.

- Действия для проверенных почтовых ящиков для типов и типов для их входящих были изменены по `Delegate` `Admin` умолчанию.

Пустое значение свойства *DefaultAuditSet* указывает, что действия с почтовым ящиком для всех трех типов были изменены в почтовом ящике пользователя или общем почтовом ящике.

Дополнительные сведения см. в разделе "Изменение [или](#change-or-restore-mailbox-actions-logged-by-default) восстановление действий почтового ящика", занося в журнал по умолчанию в этой статье

### <a name="display-the-mailbox-actions-that-are-being-logged-on-mailboxes"></a>Отображение действий почтовых ящиков, которые регистрируются в почтовых ящиках

Чтобы увидеть действия почтовых ящиков, которые в настоящее время регистрируются в почтовых ящиках пользователей или общих почтовых ящиках, замените имя, псевдоним, адрес электронной почты или имя пользователя (имя пользователя) почтового ящика и запустите одну или несколько из следующих команд в \<MailboxIdentity\> Exchange Online PowerShell.

> [!NOTE]
> Хотя вы можете добавить переключатель в следующие команды Get-Mailbox для почтовых ящиков группы `-GroupMailbox` Microsoft 365, не верьте возвращаемой информации.  Действия по умолчанию и статические действия почтовых ящиков, которые проверяются для почтовых ящиков группы Microsoft 365, описаны в разделе "Действия с почтовыми ящиками группы [Microsoft 365"](#mailbox-actions-for-microsoft-365-group-mailboxes) выше в этой статье.

#### <a name="owner-actions"></a>Действия владельца

```PowerShell
Get-Mailbox -Identity <MailboxIdentity> | Select-Object -ExpandProperty AuditOwner
```

#### <a name="delegate-actions"></a>Делегирование действий

```PowerShell
Get-Mailbox -Identity <MailboxIdentity> | Select-Object -ExpandProperty AuditDelegate
```

#### <a name="admin-actions"></a>Действия администратора

```PowerShell
Get-Mailbox -Identity <MailboxIdentity> | Select-Object -ExpandProperty AuditAdmin
```

## <a name="change-or-restore-mailbox-actions-logged-by-default"></a>Изменение или восстановление действий почтовых ящиков, зарегистрированных по умолчанию

Как объяснялось ранее, одним из ключевых преимуществ аудита почтовых ящиков по умолчанию является то, что вам не нужно управлять действиями почтовых ящиков, которые проверяются. Корпорация Майкрософт делает это за вас, и мы автоматически добавим новые действия почтовых ящиков для аудита по умолчанию по мере их выпуска.

Однако вашей организации может потребоваться аудит другого набора действий почтовых ящиков для почтовых ящиков пользователей и общих почтовых ящиков. В процедурах этого раздела покажем, как изменить действия почтовых ящиков, которые проверяются для каждого типа для регистрации, и как вернуться к действиям по умолчанию под управлением Майкрософт.

> [!IMPORTANT]
> Если для настройки действий почтовых ящиков, зарегистрированных в почтовых ящиках пользователей или общих почтовых ящиках, используются следующие процедуры, новые действия по умолчанию, выпущенные корпорацией Майкрософт, не будут автоматически проверяться на эти почтовые ящики. Вам потребуется вручную добавить новые действия почтового ящика в настроенный список действий.

### <a name="change-the-mailbox-actions-to-audit"></a>Изменение действий почтового ящика для аудита

Вы можете использовать *параметры AuditAdmin,* *AuditDelegate* или *AuditOwner* в cmdlet **Set-Mailbox,** чтобы изменить действия почтовых ящиков, которые проверяются для почтовых ящиков пользователей и общих почтовых ящиков (аудит действий для почтовых ящиков групп Microsoft 365 не может быть настроен).

Для указания действий почтового ящика можно использовать два разных метода:

- *Замените* (переписать) существующие действия почтового ящика с помощью этого синтаксиса: `action1,action2,...actionN` .

- *Добавление или удаление действий* почтового ящика без влияния на другие существующие значения с помощью этого синтаксиса: `@{Add="action1","action2",..."actionN"}` или `@{Remove="action1","action2",..."actionN"}` .

В этом примере изменения действий почтового ящика администратора для почтового ящика с именем "Отрочено" путем перезаписи действий по умолчанию с помощью SoftDelete и HardDelete.

```PowerShell
Set-Mailbox -Identity "Gabriela Laureano" -AuditAdmin HardDelete,SoftDelete
```

В этом примере действие владельца MailboxLogin добавляется в почтовый ящик laura@contoso.onmicrosoft.com.

```PowerShell
Set-Mailbox -Identity laura@contoso.onmicrosoft.com -AuditOwner @{Add="MailboxLogin"}
```

В этом примере удаляется действие делегата MoveToDeletedItems для почтового ящика обсуждения группы.

```PowerShell
Set-Mailbox -Identity "Team Discussion" -AuditDelegate @{Remove="MoveToDeletedItems"}
```

Независимо от используемого метода настройка действий аудита почтовых ящиков для почтовых ящиков пользователей или общих почтовых ящиков имеет следующие результаты:

- Для настроенного типа учетных записей действия аудита почтовых ящиков больше не управляются корпорацией Майкрософт.

- Настроенный тип для логотипа больше не отображается в значении свойства *DefaultAuditSet* для почтового ящика, как [описано выше.](#verify-that-default-mailbox-actions-are-being-logged-for-each-logon-type)

### <a name="restore-the-default-mailbox-actions"></a>Восстановление действий почтового ящика по умолчанию

Если вы настроили действия почтовых ящиков, которые проверяются в почтовом ящике пользователя или общем почтовом ящике, вы можете восстановить действия по умолчанию для одного или всех типов для каждого типа для пользователя, используя такой синтаксис:

```PowerShell
Set-Mailbox -Identity <MailboxIdentity> -DefaultAuditSet <Admin | Delegate | Owner>
```

Можно указать несколько *значений DefaultAuditSet,* разделив их запятой

**Примечание.** Следующие процедуры не применяются к почтовым ящикам группы Microsoft 365 (они ограничены действиями по умолчанию, как [описано здесь).](#mailbox-actions-for-microsoft-365-group-mailboxes)

В этом примере восстанавливаются действия по умолчанию для всех типов почтовых ящиков, для mark@contoso.onmicrosoft.com.

```PowerShell
Set-Mailbox -Identity mark@contoso.onmicrosoft.com -DefaultAuditSet Admin,Delegate,Owner
```

В этом примере восстанавливаются действия почтового ящика, по умолчанию проверенные для типа "Администратор", в почтовом ящике chris@contoso.onmicrosoft.com, но настраиваемые проверенные действия почтовых ящиков для типов для учетных данных "Делегат" и "Владелец".

```PowerShell
Set-Mailbox -Identity chris@contoso.onmicrosoft.com -DefaultAuditSet Admin
```

Восстановление действий почтового ящика, по умолчанию проверенных по умолчанию, для типа для логотипа приводит к следующим результатам:

- Текущий список действий почтового ящика заменен действиями почтового ящика по умолчанию для типа для логотипа.

- Все новые действия почтовых ящиков, выпущенные корпорацией Майкрософт, автоматически добавляются в список проверенных действий для типа для выхода в учетную запись.

- Значение *свойства DefaultAuditSet* для почтового ящика обновляется, включив восстановленный тип для логотипа.

## <a name="turn-off-mailbox-auditing-on-by-default-for-your-organization"></a>Отключение аудита почтовых ящиков по умолчанию для организации

Вы можете отключить аудит почтовых ящиков по умолчанию для всей организации, выменив следующую команду в Exchange Online PowerShell:

```PowerShell
Set-OrganizationConfig -AuditDisabled $true
```

Отключение аудита почтовых ящиков по умолчанию приводит к следующим результатам:

- Аудит почтовых ящиков отключен для вашей организации.

- С отключения аудита почтовых ящиков по умолчанию никакие действия не проверяются, даже если аудит включен для почтового ящика (свойство *AuditEnabled* в почтовом ящике имеет значение **True).**

- Аудит почтовых ящиков не включен для новых почтовых ящиков, и установка свойства  *AuditEnabled* для нового или существующего почтового ящика в true будет проигнорирована.

- Все параметры связи обхода аудита почтовых ящиков (настраиваемые с помощью **cmdlet Set-MailboxAuditBypassAssociation)** игнорируются.

- Существующие записи аудита почтовых ящиков сохраняются до истечения срока действия журнала аудита для записи.

### <a name="turn-on-mailbox-auditing-on-by-default"></a>Включить аудит почтовых ящиков по умолчанию

Чтобы включить аудит почтовых ящиков для организации, в Exchange Online PowerShell запустите следующую команду:

```PowerShell
Set-OrganizationConfig -AuditDisabled $false
```

## <a name="bypass-mailbox-audit-logging"></a>Обход ведения журнала аудита почтовых ящиков

В настоящее время нельзя отключить аудит почтовых ящиков для определенных почтовых ящиков, если в организации по умолчанию включен аудит почтовых ящиков. Например, установка для свойства почтового ящика *AuditEnabled* свойства **False** игнорируется.

Однако вы **по-прежнему** можете использовать в Exchange Online PowerShell для предотвращения записи  в журнал любых и всех действий почтовых ящиков указанных пользователей независимо от того, где они происходят. Пример.

- Действия владельца почтового ящика, выполняемые обойденными пользователями, не регистрируются в журнале.

- Делегирование действий, выполняемых обойденными пользователями в почтовых ящиках других пользователей (включая общие почтовые ящики), не регистрируется в журнале.

- Действия администратора, выполняемые обойденными пользователями, не регистрируются в журнале.

Чтобы обойти ведение журнала аудита почтовых ящиков для определенного пользователя, замените имя, адрес электронной почты, псевдоним или имя пользователя-пользователя (имя пользователя) и запустите \<MailboxIdentity\> следующую команду:

```PowerShell
Set-MailboxAuditBypassAssociation -Identity <MailboxIdentity> -AuditByPassEnabled $true
```

Чтобы убедиться, что аудит для указанного пользователя не обходится, запустите следующую команду:

```PowerShell
Get-MailboxAuditBypassAssociation -Identity <MailboxIdentity> | Format-List AuditByPassEnabled
```

Значение **True указывает,** что ведение журнала аудита почтовых ящиков для пользователя обходится.

## <a name="more-information"></a>Дополнительная информация

- Хотя ведение журнала аудита почтовых ящиков по умолчанию включено для всех организаций, только пользователи с лицензиями E5 возвращают события журнала аудита почтовых ящиков при поиске в журнале аудита в Центре безопасности [& и](search-the-audit-log-in-security-and-compliance.md) соответствия требованиям или через API действий управления [Office 365](https://docs.microsoft.com/office/office-365-management-api/office-365-management-activity-api-reference) по **умолчанию.**

  Чтобы получить записи журнала аудита почтовых ящиков для пользователей без лицензий E5, вы можете:

  - Вручную включить аудит почтовых ящиков для отдельных почтовых ящиков (выполнить `Set-Mailbox -Identity <MailboxIdentity> -AuditEnabled $true` команду). После этого можно использовать поиск в журнале аудита в Центре безопасности & соответствия требованиям или с помощью API действий управления Office 365.
  
    > [!NOTE]
    > Если аудит почтовых ящиков уже включен для почтового ящика, но поиск не возвращает результатов, измените значение параметра _AuditEnabled_ на и `$false` `$true` обратно.
  
  - Используйте следующие cmdlets в Exchange Online PowerShell:

    - [Search-MailboxAuditLog](https://docs.microsoft.com/powershell/module/exchange/search-mailboxauditlog) для поиска определенных пользователей в журнале аудита почтовых ящиков.

    - [New-MailboxAuditLogSearch](https://docs.microsoft.com/powershell/module/exchange/new-mailboxauditlogsearch) для поиска определенных пользователей в журнале аудита почтовых ящиков и отправки результатов по электронной почте указанным получателям.

  - Используйте Центр администрирования Exchange (EAC) в Exchange Online для следующих действий:

    - [Экспорт журналов аудита почтовых ящиков](https://docs.microsoft.com/Exchange/security-and-compliance/exchange-auditing-reports/export-mailbox-audit-logs)

    - [Запуск отчета об обращениях к почтовым ящикам от пользователей, не являющихся их владельцами](https://docs.microsoft.com/Exchange/security-and-compliance/exchange-auditing-reports/non-owner-mailbox-access-report)

- По умолчанию записи журнала аудита почтовых ящиков сохраняются в течение 90 дней перед их удалением. Вы можете изменить срок жизни записей журнала аудита с помощью параметра *AuditLogAgeLimit* в cmdlet **Set-Mailbox** в Exchange Online PowerShell. Однако увеличение этого значения не позволяет искать события старше 90 дней в журнале аудита.

  При увеличении срока действия необходимо использовать в Exchange Online PowerShell для поиска записей старше 90 дней в журнале аудита почтовых ящиков пользователя с помощью [cmdlet Search-MailboxAuditLog.](https://docs.microsoft.com/powershell/module/exchange/search-mailboxauditlog)

- Если вы изменили свойство *AuditLogAgeLimit* для почтового ящика до того, как аудит почтовых ящиков был включен по умолчанию для организации, то срок жизни существующего журнала аудита почтового ящика не меняется. Другими словами, аудит почтовых ящиков по умолчанию не влияет на текущий срок жизни записей аудита почтовых ящиков.

- Чтобы изменить *значение AuditLogAgeLimit* для почтового ящика группы Microsoft 365, необходимо включить этот переключатель в команду `-GroupMailbox` **Set-Mailbox.**

- Записи журнала аудита почтовых ящиков хранятся во вложенной папке (с именем *Audits)* в папке "Элементы с восстановления" в почтовом ящике каждого пользователя. О записях аудита почтовых ящиков и папке "Элементы с восстановлениями" помните следующее:

  - В записях аудита почтовых ящиков учитывается квота хранилища папки "Элементы с восстановления", которая по умолчанию составляет 30 ГБ (квота предупреждения составляет 20 ГБ). Квота хранилища автоматически увеличивается до 100 ГБ (с квотой предупреждения 90 ГБ), если:

    - Для почтового ящика наложено удержание.

    - Почтовый ящик назначен политике хранения в Центре соответствия требованиям.

  - Записи аудита почтовых ящиков также учитываются в ограничении для папки "Элементы с [восстановления".](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#mailbox-folder-limits) В подкамере аудита можно хранить не более 3 миллионов элементов (записей аудита).

    > [!NOTE]
    > Маловероятно, что аудит почтовых ящиков по умолчанию повлияет на квоту хранилища или ограничение на папку для папки "Элементы с восстановления".

    - Чтобы отобразить размер и количество элементов во вложенной папке "Аудит" в папке "Элементы с возможностью восстановления", можно выполнить следующую команду в Exchange Online PowerShell:

      ```PowerShell
      Get-MailboxFolderStatistics -Identity <MailboxIdentity> -FolderScope RecoverableItems | Where-Object {$_.Name -eq 'Audits'} | Format-List FolderPath,FolderSize,ItemsInFolder
      ```

    - Невозможно напрямую получить доступ к записи журнала аудита в папке "Элементы с возможностью восстановления"; вместо этого для  поиска и просмотра записей аудита почтовых ящиков можно использовать для поиска и просмотра записей аудита почтовых ящиков или поиск в журнале аудита.

- Если почтовый ящик помещен на удержание или назначен политике хранения в Центре соответствия требованиям, записи журнала аудита сохраняются в течение срока, определенного свойством *AuditLogAgeLimit* почтового ящика (90 дней по умолчанию). Чтобы хранить записи журнала аудита дольше для почтовых ящиков на удержании, необходимо увеличить значение *AuditLogAgeLimit почтового* ящика.

- В среде с несколькими регионами аудит почтовых ящиков для разных географических расположений не поддерживается. Например, если пользователю назначены разрешения на доступ к общему почтовому ящику в другом географическом расположении, действия с почтовым ящиком, выполняемые этим пользователем, не регистрируются в журнале аудита почтовых ящиков общего почтового ящика.
