# <a name="learn-about-insider-risk-management-in-microsoft-365"></a>Узнайте об управлении insider risk в Microsoft 365

Управление внутренними рисками — это решение для обеспечения соответствия требованиям в Microsoft 365, которое помогает свести к минимуму внутренние риски, позволяя обнаруживать, исследовать и действовать над вредоносными и непреднамеренными действиями в организации. Политики предварительной оценки рисков позволяют определить типы рисков для идентификации и обнаружения в организации, в том числе действовать по делам и при необходимости переадружать дела в Службу Microsoft Advanced eDiscovery. Аналитики рисков в организации могут быстро принять соответствующие меры, чтобы убедиться, что пользователи соответствуют стандартам соответствия вашей организации.

Посмотрите видео ниже, чтобы узнать, как управление insider risk помогает организации предотвращать, обнаруживать и содержать риски при приоритете значений, культуры и пользовательского интерфейса организации:

## <a name="modern-risk-pain-points"></a>Современные точки риска

Управление и минимизация рисков в организации начинается с понимания типов рисков, найденных на современном рабочем месте. Некоторые риски обусловлены внешними событиями и факторами, которые не контролируются напрямую. Другие риски обусловлены внутренними событиями и действиями пользователей, которые можно свести к минимуму и избежать. В качестве примеров можно привести риски, связанных с несанкционированным, несанкционированным или неэтичным поведением и действиями пользователей в организации. К таким поведениям относится широкий спектр внутренних рисков, связанных с пользователями:

- Утечки конфиденциальных данных и утечка данных
- Нарушения конфиденциальности
- Кража интеллектуальной собственности (IP)
- Мошенничество
- Insider trading
- Нарушения нормативных требований

Пользователи на современном рабочем месте имеют доступ к созданию, управлению и совместному доступу к данным на широком спектре платформ и служб. В большинстве случаев организации имеют ограниченные ресурсы и средства для выявления и снижения рисков на уровне организации, а также для обеспечения безопасности пользователей.

Для управления рисками внутри организации используется весь спектр служб и сторонних индикаторов, которые помогают быстро выявлять, реагировать на действия с рисками и реагировать на них. Используя журналы из Microsoft 365 и Microsoft Graph, управление рисками внутри организации позволяет определять определенные политики для определения индикаторов риска. Эти политики позволяют выявлять рискованные действия и действовать для их снижения.

Управление рисками внутри организации происходит на основе следующих принципов:

- **Прозрачность:** баланс между конфиденциальностью пользователей и рисками организации с архитектурой конфиденциальности по дизайну.
- **Настраиваемая:** настраиваемые политики, основанные на отраслевых, географических и бизнес-группах.
- **Интегрированный**: интегрированный рабочий процесс в решениях для обеспечения соответствия требованиям Microsoft 365.
- **Actionable**: предоставляет сведения для уведомлений пользователей, анализа данных и расследований пользователей.

С помощью шаблонов политик, комплексных сигналов о действиях в службе Microsoft 365, а также средств управления оповещениями и случаями вы можете использовать анализ действий, чтобы быстро определять рискованные действия и действовать на них.

### <a name="alerts"></a>Оповещения

Оповещения автоматически создаются индикаторами риска, которые соответствуют условиям политики и отображаются на панели [мониторинга оповещений.](insider-risk-management-alerts.md) Эта панель мониторинга позволяет быстро просматривать все оповещения, которые требуют проверки, открывать оповещения со временем и статистику оповещений для организации. Все оповещения политики отображаются со следующими сведениями, которые помогут быстро определить состояние существующих оповещений и новых оповещений, которые требуют действий:

- Status
- Severity
- Обнаружено время
- Ситуация
- Состояние дела

![Панель мониторинга оповещений по управлению рисками в организации](../media/insider-risk-alerts-dashboard.png)

### <a name="triage"></a>Triage

Новые действия пользователей, которые нуждаются в анализе, автоматически создают оповещения с состоянием *проверки* потребностей. Проверяющих можно быстро идентифицировать и просмотреть, оценить и рассмотреть эти оповещения.

Для разрешения оповещений необходимо открыть новое дело, назначить оповещение существующему делу или отклонять оповещение. С помощью фильтров оповещений можно легко быстро определить оповещения по обнаружению состояния, серьезности или времени. В процессе рассмотрения проверяющих могут просматривать сведения оповещений о действиях, выявленных политикой, просматривать действия пользователей, связанные с соответствием политики, просматривать степень серьезности оповещения и просматривать сведения профиля пользователя.

![Триадж по управлению рисками внутри организации](../media/insider-risk-triage.png)

### <a name="investigate"></a>Исследование

[Создаются](insider-risk-management-cases.md) дела для оповещений, которые требуют более глубокой проверки и анализа сведений о действиях и обстоятельств, которые соответствуют политике. Панель **мониторинга case** предоставляет все представление всех активных дел, открытых дел с течением времени и статистики дела для вашей организации. Проверяющих можно быстро фильтровать дела по статусу, дате открытия дела и дате последнего обновления дела.

Выбор дела на панели мониторинга дела открывает дело для расследования и проверки. Этот шаг является частью рабочего процесса управления рисками внутри организации. В этой области действия с рисками, условия политики, сведения об оповещениях и сведения о пользователях синтезируются в интегрированном представлении для проверяющих. Основными средствами исследования в этой области являются:

- **Активность пользователей:** активность пользователя автоматически отображается на интерактивной диаграмме, которая отображает действия с течением времени и по уровню риска для текущих или прошлых действий с риском. Рецензенты могут быстро отфильтровать и просмотреть всю историю рисков для пользователя и детализовать конкретные действия для получения дополнительных сведений.
- **Обозреватель содержимого:** все файлы данных и сообщения электронной почты, связанные с действиями оповещений, автоматически захватываются и отображаются в обозревателе содержимого. Рецензенты могут фильтровать и просматривать файлы и сообщения по источнику данных, типу файла, тегам, беседам и многим дополнительным атрибутам.
- **Примечания** к делу: рецензенты могут предоставлять заметки по делу в разделе "Примечания к делу". Этот список объединяет все заметки в центральном представлении и включает сведения об рецензенте и дате отправки.

![Исследование управления рисками внутри организации](../media/insider-risk-investigate.png)

### <a name="action"></a>Action

После расследования дел проверяющих могут быстро решить проблему или сотрудничать с другими заинтересованными лицами в вашей организации. Если пользователи случайно или случайно нарушают условия политики, пользователю можно отправить простое уведомление с напоминанием из шаблонов уведомлений, которые можно настроить для вашей организации. Эти уведомления могут служить простыми напоминаниями или помогают пользователю обновлять учебные курсы или инструкции, чтобы предотвратить дальнейшее рискованные действия. Дополнительные сведения см. в [шаблонах уведомлений об управлении рисками в организации.](insider-risk-management-notices.md)

В более серьезных ситуациях может потребоваться поделиться сведениями о деле управления рисками внутри организации с другими проверятелями или службами в вашей организации. Управление рисками внутри организации тесно интегрировано с другими решениями для обеспечения соответствия требованиям Microsoft 365, чтобы помочь вам в комплексном разрешении рисков.

- **Advanced eDiscovery**: передача дела для расследования позволяет передавать данные и управление делом в Advanced eDiscovery в Microsoft 365. Advanced eDiscovery предоставляет конечный рабочий процесс для сохранения, сбора, проверки, анализа и экспорта контента, который отвечает на внутренние и внешние исследования в организации. Это позволяет юридическим группам управлять всем процессом уведомлений об удержании по юридическим основаниям. Дополнительные сведения о случаях Advanced eDiscovery см. в обзоре [Advanced eDiscovery в Microsoft 365.](overview-ediscovery-20.md)
- **ServiceNow (предварительная версия).** ServiceNow — это популярные облачные вычислительные платформы, которые помогают организациям управлять цифровыми процессами для корпоративных операций. Управление рисками внутри организации поддерживает совместное использование оповещений о случаях со службой ServiceNow и позволяет создавать инциденты и запросы на изменение, связанные с отдельными случаями риска внутри организации. Дополнительные сведения о совместном использовании сведений оповещений с serviceNow см. в разделе ["Общий доступ к делу с serviceNow".](insider-risk-management-cases.md#share-the-case)
- Интеграция API управления **Office 365 (предварительная версия).** Управление рисками для предварительной оценки поддерживает экспорт сведений оповещений в службы SIEM через API управления Office 365. Доступ к информации оповещений на платформе лучше всего подходит для процессов риска вашей организации, что позволяет более гибко действовать над действиями с рисками. Дополнительные сведения об экспорте сведений оповещений с помощью API управления Office 365 см. в [экспорте оповещений.](insider-risk-management-settings.md#export-alerts-preview)

## <a name="scenarios"></a>Сценарии

Управление внутренними рисками помогает обнаруживать, исследовать и принимать меры для снижения внутренних рисков в организации в нескольких распространенных сценариях:

### <a name="data-theft-by-departing-users"></a>Кража данных путем убывающих пользователей

Когда пользователи покидают организацию либо по собственному желанию, либо в результате увольнения, часто существует законная проблема, в результате чего данные компании, клиента и пользователя находятся под угрозой. Пользователи могут с нею предположить, что данные проекта не проприетары, или они могут быть захищаются получить данные компании для личной выгоды и в нарушение политики компании и правовых стандартов. Политики управления рисками внутри [](insider-risk-management-policies.md#policy-templates) организации, которые используют кражу данных при отступе от шаблона политики пользователей, автоматически обнаруживают действия, обычно связанные с этим типом кражи. С помощью этой политики вы автоматически будете получать оповещения о подозрительных действиях, связанных с кражей данных, отправляя пользователей, чтобы вы могли принять соответствующие меры расследования. Для этого шаблона политики необходимо настроить соединители [Microsoft 365](import-hr-data.md) HR для вашей организации.

### <a name="intentional-or-unintentional-leak-of-sensitive-or-confidential-information"></a>Преднамеренные или непреднамеренные утечки конфиденциальной или конфиденциальной информации

В большинстве случаев пользователи стараются правильно обрабатывать конфиденциальную или конфиденциальную информацию. Но иногда пользователи могут ошибаться, и информация случайно передается за пределы вашей организации или нарушает ваши политики защиты информации. В других случаях пользователи могут намеренно утечек или обмена конфиденциальной информацией с злонамеренным намерением и для потенциальной личной выгоды. Политики управления рисками внутри организации, созданные с помощью следующих шаблонов политики утечки данных, автоматически обнаруживают действия, обычно связанные с обменом конфиденциальной или конфиденциальной информацией:

- [Общие утечки данных](insider-risk-management-policies.md#general-data-leaks)
- [Утечки данных по приоритетным пользователям (предварительная версия)](insider-risk-management-policies.md#data-leaks-by-priority-users-preview)
- [Утечки данных несовершимыми пользователями (предварительная версия)](insider-risk-management-policies.md#data-leaks-by-disgruntled-users-preview)

## <a name="intentional-or-unintentional-security-policy-violations-preview"></a>Преднамеренные или непреднамеренные нарушения политики безопасности (предварительная версия)

Как правило, пользователи имеют большой уровень контроля при управлении устройствами на современном рабочем месте. Это может включать разрешения на установку или удалить приложения, необходимые для выполнения их обязанностей, или возможность временного отключения функций безопасности устройств. Независимо от того, является ли это действие случайным, случайным или вредоносным, это поведение может представлять риск для вашей организации и важно идентифицировать и действовать, чтобы свести к минимуму. Чтобы помочь в идентификации этих рискованных действий по обеспечению безопасности, следующие шаблоны нарушения политики безопасности для управления рисками внутри организации дают оценку индикаторов риска безопасности и используют Microsoft Defender для оповещений конечных точек для предоставления статистики о действиях, связанных с безопасностью:

- [Общие нарушения политики безопасности (предварительная версия)](insider-risk-management-policies.md#general-security-policy-violations-preview)
- [Нарушения политики безопасности при отступе от пользователей (предварительная версия)](insider-risk-management-policies.md#security-policy-violations-by-departing-users-preview)

- [Нарушения политики безопасности со стороны несоединых пользователей (предварительная версия)](insider-risk-management-policies.md#security-policy-violations-by-disgruntled-users-preview)

## <a name="policies-for-users-based-on-position-access-level-or-risk-history-preview"></a>Политики для пользователей на основе положения, уровня доступа или истории рисков (предварительная версия)

Пользователи в организации могут иметь разные уровни риска в зависимости от их положения, уровня доступа к конфиденциальной информации или истории рисков. Это могут быть члены группы руководителей организации, ИТ-администраторы с обширными привилегиями доступа к данным и сети, а также пользователи с прошлым опытом рискованных действий. В этих обстоятельствах более тщательное исследование и более агрессивное оценку рисков важно, чтобы помочь в предупреждении для исследования и быстрого реагирования. Чтобы определить рискованные действия для этих типов пользователей, можно создать группы пользователей с приоритетами и создать политики на следующую политику шаблонов:

- [Нарушения политики безопасности пользователями с приоритетами (предварительная версия)](insider-risk-management-policies.md#security-policy-violations-by-priority-users-preview)
- [Утечки данных по приоритетным пользователям (предварительная версия)](insider-risk-management-policies.md#data-leaks-by-priority-users-preview)

## <a name="actions-and-behaviors-by-disgruntled-users-preview"></a>Действия и поведение несовершимые пользователи (предварительная версия)

События, связанные с нагрузкой на работу, могут влиять на поведение пользователей несколькими способами, которые связаны с рисками внутри пользователя. Эти нагрузки могут быть анализом низкой производительности, понижением позиции или размещением пользователя в плане проверки производительности. Хотя большинство пользователей не реагируют на эти события злонамеренными действиями, нагрузка этих действий может привести к тем, что некоторые пользователи будут делать действия, которые обычно не рассматриваются в обычных обстоятельствах. Для идентификации таких типов рискованных действий следующие шаблоны политики управления рисками внутри организации используют соединители Microsoft 365 HR и начинают оценку индикаторов риска, связанных с поведением, которое может произойти вблизи событий, связанных с нагрузкой на работу:

- [Утечки данных несовершимыми пользователями (предварительная версия)](insider-risk-management-policies.md#data-leaks-by-disgruntled-users-preview)
- [Нарушения политики безопасности со стороны несоединых пользователей (предварительная версия)](insider-risk-management-policies.md#security-policy-violations-by-disgruntled-users-preview)
