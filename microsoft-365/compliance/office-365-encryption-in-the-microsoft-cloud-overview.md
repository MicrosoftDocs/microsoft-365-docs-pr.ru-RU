# <a name="encryption-in-the-microsoft-cloud"></a>Шифрование в Microsoft Cloud

Данные клиентов в корпоративных облачных службах Майкрософт защищены различными технологиями и процессами, включая различные виды шифрования. (Данные клиента в этом документе включают содержимое почтовых ящиков Exchange Online, текст электронной почты, записи календаря и содержимое вложений электронной почты, а также контент Skype для бизнеса, контент сайта SharePoint Online и файлы, хранимые на сайтах, а также файлы, отправленные в OneDrive для бизнеса или Skype для бизнеса.) Корпорация Майкрософт использует несколько методов шифрования, протоколов и шифров в своих продуктах и службах, чтобы обеспечить безопасный путь для передачи данных клиентов через наши облачные службы и защитить конфиденциальность данных клиентов, хранимых в наших облачных службах. Корпорация Майкрософт использует одни из самых надежных и безопасных протоколов шифрования, чтобы обеспечить препятствия для несанкционированного доступа к данным клиентов. Надлежащее управление ключами также является важным элементом практических методик шифрования, и Корпорация Майкрософт обеспечивает надлежащую безопасность всех ключей шифрования, управляемых Корпорацией Майкрософт.

Независимо от конфигурации клиента данные клиентов, хранимые в корпоративных облачных службах Майкрософт, защищены с помощью одной или нескольких форм шифрования. (Проверка нашей политики криптографии и ее принудительная проверка независимо проверяются несколькими сторонними аудиторами, и отчеты об этих аудитах доступны на портале [Service Trust Portal.)](https://aka.ms/stp)

Корпорация Майкрософт предоставляет технологии на стороне службы, которые шифруют неавтовые и транзитные данные клиентов. Например, для неавторических данных клиентов Microsoft Azure использует [BitLocker](https://docs.microsoft.com/windows/device-security/bitlocker/bitlocker-overview) и [DM-Crypt,](https://en.wikipedia.org/wiki/Dm-crypt)а Microsoft 365 использует BitLocker, шифрование службы хранилища [Azure,](https://docs.microsoft.com/azure/) [диспетчер](https://docs.microsoft.com/microsoft-365/compliance/exchange-online-secures-email-secrets) распределенных ключей (DKM) и шифрование службы Microsoft 365. Для транзитных данных клиентов Azure, Office 365, microsoft Commercial Support, Microsoft Dynamics 365, Microsoft Power BI и Visual Studio Team Services используют стандартные отраслевые протоколы безопасного транспорта, такие как IPsec и TLS, между центрами обработки данных Майкрософт, а также между пользовательскими устройствами и центрами обработки данных Майкрософт.

Помимо базового уровня криптографической безопасности, предоставляемого корпорацией Майкрософт, наши облачные службы также включают дополнительные параметры шифрования, которые можно управлять. Например, можно включить шифрование для трафика между виртуальными машинами Azure и их пользователями. С [помощью виртуальных](https://azure.microsoft.com/services/virtual-network/)сетей Azure вы можете использовать стандартный протокол IPsec для шифрования трафика между корпоративным VPN-шлюзом и Azure, а также между виртуальными виртуальной сетью. Кроме того, новые возможности шифрования сообщений [Office 365](set-up-new-message-encryption-capabilities.md) позволяют отправлять зашифрованную почту всем пользователям.

В соответствии со стандартом оперативной безопасности инфраструктуры открытых ключей, который является компонентом политики безопасности [Майкрософт,](https://servicetrust.microsoft.com/ViewPage/TrustDocuments?command=Download&downloadType=Document&downloadId=5868ecc8-50b7-4f91-b43f-640e2b99e86e&docTab=6d000410-c9e9-11e7-9a91-892aae8839ad_FAQ%20and%20White%20Papers)корпорация Майкрософт использует возможности шифрования, включенные в операционную систему Windows, для сертификатов [](https://csrc.nist.gov/publications/PubsFIPS.html) и механизмов проверки подлинности, в том числе использование криптографических модулей, которые соответствуют стандарту FIPS 140-2 федерального правительства США. (Соответствующие номера сертификатов NIST для Майкрософт можно найти по https://csrc.nist.gov/groups/STM/cmvp/documents/140-1/1401vend.htm.)

> [ПРИМЕЧАНИЕ] Чтобы получить доступ к политике безопасности Майкрософт в качестве ресурса, необходимо войти с помощью своей учетной записи на работе или в учебном зачете. Если у вас еще нет подписки, вы можете зарегистрироваться для [бесплатной пробной подписки.](https://servicetrust.microsoft.com/Home/TrialSubscriptions)

FIPS 140-2 — это стандарт, разработанный специально для проверки модулей продуктов, реализующих шифрование, а не продуктов, которые их используют. Криптографические модули, реализованные в службе, могут быть сертифицированы в качестве требований к безопасности, управлению ключами и выполнению других требований. Каждый раз, когда криптографические возможности используются для защиты конфиденциальности, целостности или доступности данных в облачных службах Майкрософт, используемые модули и шифры соответствуют стандарту FIPS 140-2.

Корпорация Майкрософт сертифицировать модули шифрования, используемые в наших облачных службах, с каждым новым выпуском операционной системы Windows:

- Azure и Azure для государственных органов США
- Dynamics 365 и Dynamics 365 для государственных организаций США
- Office 365, Office 365 для государственных организаций США и Office 365 U.S. Government Defense

Шифрование хранимых данных клиентов обеспечивается несколькими технологиями на стороне службы, включая BitLocker, DKM, шифрование службы хранилища Azure и шифрование служб в Exchange Online, Skype для бизнеса, OneDrive для бизнеса и SharePoint Online. Шифрование службы Office 365 включает возможность использовать управляемые клиентом ключи шифрования, хранимые в хранилище Azure Key Vault. Этот управляемый клиентом ключ, называемый [ключом](https://docs.microsoft.com/microsoft-365/compliance/customer-key-overview)клиента, доступен для Exchange Online, SharePoint Online, Skype для бизнеса и OneDrive для бизнеса.

Для передачи данных клиентов все серверы Office 365 согласовывать безопасные сеансы с использованием TLS по умолчанию с клиентские компьютеры для защиты данных клиентов.  Это относится к протоколам на любом устройстве, используемом клиентами, например Skype для бизнеса, Outlook и Outlook в Интернете, мобильных клиентах и веб-браузерах.

(По умолчанию все серверы с клиентами согласовыются с TLS 1.2, но при необходимости мы также поддерживаем согласование до более низкого стандарта.)
