# <a name="disable-reports-when-you-export-content-search-results"></a>Отключение отчетов при экспорте результатов поиска контента

При экспорте результатов поиска контента в Центре соответствия требованиям & eDiscovery Export средство автоматически создает и экспортирует два отчета, содержащих дополнительные сведения об экспортируемом контенте. Эти отчеты являются Results.csv и файл Manifest.xml (см. [](#frequently-asked-questions-about-disabling-export-reports) часто задамые вопросы об отключении экспорта отчетов в этом разделе для подробных описаний этих отчетов). Поскольку эти файлы могут быть очень большими, можно ускорить время скачивания и сохранить пространство диска, не мешая экспортировать эти файлы. Это можно сделать, изменив Windows реестра на компьютере, который используется для экспорта результатов поиска. Если вы хотите включить отчеты в более позднее время, вы можете изменить параметр реестра. 
  
## <a name="create-registry-settings-to-disable-the-export-reports"></a>Создание параметров реестра для отключения отчетов об экспорте

Выполните следующую процедуру на компьютере, который будет использовать для экспорта результатов поиска контента.
  
1. Закрой средство экспорта электронных открытий, если оно открыто.
    
2. Выполните один или оба следующих шага в зависимости от того, какой отчет экспорта необходимо отключить.
    
    - **Results.csv**
    
      Сохраните следующий текст в Windows реестра с помощью суффикса .reg; например, DisableResultsCsv.reg.
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultCsvEnabled /t REG_SZ /d False 
      ```

    - **Manifest.xml**
    
      Сохраните следующий текст в Windows реестра с помощью суффикса .reg; например, DisableManifestXml.reg.
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultEdrmEnabled /t REG_SZ /d False 
      ```

3. В Windows Explorer щелкните или дважды щелкните файл .reg, созданный в предыдущих действиях.
    
4. В окне Управления доступом пользователей щелкните **Да,** чтобы редактор реестра внести изменения. 
    
5. Если вам будет предложено продолжить, нажмите **кнопку Да**.
    
    Редактор реестра отображает сообщение о том, что параметр успешно добавлен в реестр.
  
## <a name="edit-registry-settings-to-re-enable-the-export-reports"></a>Изменение параметров реестра, чтобы повторно включить отчеты об экспорте

Если вы отключили отчеты Results.csv и Manifest.xml, создав файлы .reg в предыдущей процедуре, вы можете изменить эти файлы, чтобы повторно включить отчет, чтобы он экспортировался с результатами поиска. Снова выполните следующую процедуру на компьютере, который будет использовать для экспорта результатов поиска контента.
  
1. Закрой средство экспорта электронных открытий, если оно открыто.
    
2. Изменить один или оба файла редактирования .reg, созданных в предыдущей процедуре.
    
    - **Results.csv**
    
        Откройте файл DisableResultsCsv.reg в Блокнот, измените значение и `False` `True` сохраните файл. Например, после редактирования файла выглядит так:
    
        ```text
        Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultCsvEnabled /t REG_SZ /d True
        ```

    - **Manifest.xml**
    
        Откройте файл DisableManifestXml.reg в Блокнот, измените значение и `False` `True` сохраните файл. Например, после редактирования файла выглядит так:
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultEdrmEnabled /t REG_SZ /d True
      ```

3. В Windows Explorer щелкните или дважды щелкните файл .reg, который был отредактирован на предыдущем шаге.
    
4. В окне Управления доступом пользователей щелкните **Да,** чтобы редактор реестра внести изменения. 
    
5. Если вам будет предложено продолжить, нажмите **кнопку Да**.
    
    Редактор реестра отображает сообщение о том, что параметр успешно добавлен в реестр.
  
## <a name="frequently-asked-questions-about-disabling-export-reports"></a>Часто задамые вопросы об отключении отчетов об экспорте

 **Какие отчеты Results.csv Manifest.xml отчеты?**
  
Файлы Results.csv Manifest.xml содержат дополнительные сведения о экспортируемом контенте.
  
- **Results.csv** Документ Excel, содержащий сведения о каждом элементе, скачаемом в результате поиска. При выполнении поиска в почте журнал результатов содержит сведения о каждом сообщении, включая указанные ниже. 
    
  - Расположение сообщения в исходном почтовом ящике (включая сведения о том, в каком почтовом ящике находится сообщение, в основном или в архивном).
    
  - Дата отправки или получения сообщения.
    
  - Тема сообщения.
    
  - Отправитель и получатели сообщения.
    
  - Является ли сообщение дублирующим сообщением, если при экспорте результатов поиска включено де-дублирование. Дубликат сообщений будет иметь значение в столбце **Parent ItemId,** которое определяет сообщение как дубликат. Значение в столбце **Parent ItemId** такое же, как и значение в столбце **Item DocumentId** экспортируемого сообщения. 
    
    Для документов с SharePoint OneDrive для бизнеса сайтов журнал результатов содержит сведения о каждом документе, в том числе:
    
  - URL-адрес документа.
    
  - URL-адрес семейства веб-сайтов, в котором расположен документ.
    
  - Дата последнего изменения документа.
    
  - Имя документа (которое указано в столбце "Тема" журнала результатов).
    
- **Manifest.xml** Файл манифеста (в формате XML), содержащий сведения о каждом элементе, включенном в результаты поиска. Сведения в этом отчете те же, что и Results.csv отчете, но они в формате, указанном справочной моделью электронного обнаружения (EDRM). Дополнительные сведения о EDRM перейдите к [https://www.edrm.net](https://www.edrm.net) .
    
 **Когда следует отключить экспорт этих отчетов?**
  
Это зависит от конкретных потребностей. Многие организации не требуют дополнительных сведений о результатах поиска и не нуждаются в этих отчетах.
  
 **На каком компьютере я должен это сделать?**
  
 Необходимо изменить параметр реестра на любом локальном компьютере, на который вы запустите средство экспорта электронных открытий. 
  
 **После изменения этого параметра необходимо перезапустить компьютер?**
  
Нет, вам не нужно перезапустить компьютер. Но если средство экспорта электронных обнаружений запущено, его необходимо закрыть, а затем перезапустить после изменения параметра реестра.
  
 **Редактируется ли существующий ключ реестра или создается новый ключ?**
  
При первом запуске файла .reg, созданного в процедуре в этом разделе, создается новый ключ реестра. Затем параметр редактируется при каждом изменении и повторном запуске файла редактирования .reg.
