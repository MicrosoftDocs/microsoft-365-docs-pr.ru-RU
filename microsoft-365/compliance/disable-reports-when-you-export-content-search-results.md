# <a name="disable-reports-when-you-export-content-search-results"></a>Отключение отчетов при экспорте результатов поиска контента

При использовании средства экспорта eDiscovery для экспорта результатов поиска контента в Центре безопасности и соответствия требованиям & средство автоматически создает и экспортирует два отчета, содержащих дополнительные сведения об экспортируемом контенте. Эти отчеты являются Results.csv и Manifest.xml (подробные описания [](#frequently-asked-questions-about-disabling-export-reports) этих отчетов см. в разделе "Вопросы и ответы об отключаемом экспорте отчетов" этой статьи). Так как эти файлы могут быть очень большими, вы можете ускорить скачивание и сэкономить место на диске, не мешая экспорту этих файлов. Это можно сделать, изменив реестр Windows на компьютере, который используется для экспорта результатов поиска. Если вы хотите включить отчеты позже, можно изменить параметр реестра. 
  
## <a name="create-registry-settings-to-disable-the-export-reports"></a>Создание параметров реестра для отключения отчетов об экспорте

Выполните следующую процедуру на компьютере, который будет использовать для экспорта результатов поиска контента.
  
1. Закроем средство экспорта для eDiscovery, если оно открыто.
    
2. Выполните одно или оба следующих действия в зависимости от отчета об экспорте, который нужно отключить.
    
    - **Results.csv**
    
      Сохраните следующий текст в файле реестра Windows с помощью суффикса REG. например, DisableResultsCsv.reg.
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultCsvEnabled /t REG_SZ /d False 
      ```

    - **Manifest.xml**
    
      Сохраните следующий текст в файле реестра Windows с помощью суффикса REG. например, DisableManifestXml.reg.
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultEdrmEnabled /t REG_SZ /d False 
      ```

3. В проводнике Windows щелкните или дважды щелкните REG-файл, созданный на предыдущих шагах.
    
4. В окне "Управление доступом пользователей" нажмите кнопку **"Да",** чтобы позволить редактору реестра внести изменения. 
    
5. Когда вам будет предложено продолжить, нажмите кнопку **"Да".**
    
    В редакторе реестра отображается сообщение об успешном добавлении параметра в реестр.
  
## <a name="edit-registry-settings-to-re-enable-the-export-reports"></a>Изменение параметров реестра, чтобы повторно включить отчеты об экспорте

Если вы отключили отчеты Results.csv и Manifest.xml, создав REG-файлы в предыдущей процедуре, вы можете изменить эти файлы, чтобы повторно включить отчет, чтобы он экспортировался вместе с результатами поиска. Снова выполните следующую процедуру на компьютере, который будет использовать для экспорта результатов поиска контента.
  
1. Закроем средство экспорта для eDiscovery, если оно открыто.
    
2. Изменение одного или обоих файлов редактирования REG, созданных в предыдущей процедуре.
    
    - **Results.csv**
    
        Откройте файл DisableResultsCsv.reg в Блокноте, измените значение на и  `False`  `True` сохраните файл. Например, после редактирования файла он выглядит так:
    
        ```text
        Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultCsvEnabled /t REG_SZ /d True
        ```

    - **Manifest.xml**
    
        Откройте файл DisableManifestXml.reg в Блокноте, измените значение на и  `False`  `True` сохраните файл. Например, после редактирования файла он выглядит так:
    
      ```text
      Windows Registry Editor Version 5.00
      reg add HKLM\SOFTWARE\Microsoft\Exchange\Client\eDiscovery\ExportTool /v ResultEdrmEnabled /t REG_SZ /d True
      ```

3. В проводнике Windows щелкните или дважды щелкните REG-файл, измененный на предыдущем шаге.
    
4. В окне "Управление доступом пользователей" нажмите кнопку **"Да",** чтобы позволить редактору реестра внести изменения. 
    
5. Когда вам будет предложено продолжить, нажмите кнопку **"Да".**
    
    В редакторе реестра отображается сообщение об успешном добавлении параметра в реестр.
  
## <a name="frequently-asked-questions-about-disabling-export-reports"></a>Вопросы и ответы об отключении отчетов об экспорте

 **Каковы отчеты Results.csv и Manifest.xml отчетов?**
  
Файлы Results.csv и Manifest.xml содержат дополнительные сведения об экспортируемом содержимом.
  
- **Results.csv** Документ Excel, содержащий сведения о каждом элементе, загружаемом в качестве результата поиска. При выполнении поиска в почте журнал результатов содержит сведения о каждом сообщении, включая указанные ниже. 
    
  - Расположение сообщения в исходном почтовом ящике (включая сведения о том, в каком почтовом ящике находится сообщение, в основном или в архивном).
    
  - Дата отправки или получения сообщения.
    
  - Тема сообщения.
    
  - Отправитель и получатели сообщения.
    
  - Является ли сообщение дублирующимся, если при экспорте результатов поиска включено дублирование. Дублирующиеся сообщения будут иметь значение в столбце **Parent ItemId,** которое идентифицирует сообщение как дубликат. Значение в столбце **Parent ItemId** такое же, как и значение в столбце **Item DocumentId** экспортируемого сообщения. 
    
    Для документов с сайтов SharePoint и OneDrive для бизнеса журнал результатов содержит сведения о каждом документе, в том числе:
    
  - URL-адрес документа.
    
  - URL-адрес семейства веб-сайтов, в котором расположен документ.
    
  - Дата последнего изменения документа.
    
  - Имя документа (которое указано в столбце "Тема" журнала результатов).
    
- **Manifest.xml** Файл манифеста (в формате XML), содержащий сведения о каждом элементе, включенном в результаты поиска. Сведения в этом отчете такие же, как Results.csv отчете, но в формате, заданном в справочной модели электронного обнаружения (EDRM). Дополнительные сведения о EDRM: [https://www.edrm.net](https://www.edrm.net)
    
 **Когда следует отключить экспорт этих отчетов?**
  
Это зависит от конкретных потребностей. Многие организации не требуют дополнительных сведений о результатах поиска и не нуждаются в этих отчетах.
  
 **На каком компьютере это необходимо сделать?**
  
 Необходимо изменить параметр реестра на любом локальном компьютере, на который вы запустите средство экспорта eDiscovery. 
  
 **Нужно ли перезагрузить компьютер после изменения этого параметра?**
  
Нет, перезагружать компьютер не нужно. Но если средство экспорта для eDiscovery запущено, его необходимо закрыть и перезапустить после изменения параметра реестра.
  
 **Редактирует ли существующий ключ реестра или создается новый?**
  
Новый раздел реестра создается при первом запуске REG-файла, созданного в процедуре, которая была создана в этом разделе. Затем параметр редактируется при каждом изменении и повторном запуске REG-файла редактирования.
