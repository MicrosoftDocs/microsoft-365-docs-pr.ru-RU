# <a name="plan-for-insider-risk-management"></a>Планирование управления внутренними рисками

Перед началом [](insider-risk-management.md) работы с управлением рисками внутри организации необходимо рассмотреть важные действия и вопросы планирования, которые должны быть рассмотрены вашими группами по управлению информационными технологиями и соответствием требованиям. Тщательное понимание и планирование развертывания в следующих областях поможет обеспечить плавную реализацию и использование функций управления рисками внутри организации и соответствие с лучшими методиками для решения.

## <a name="work-with-stakeholders-in-your-organization"></a>Работа с заинтересованными лицами в организации

Определите соответствующих заинтересованных лиц в организации для совместной работы над действиями по оповещениям и делам управления рисками внутри организации. Некоторые рекомендуемые заинтересованные стороны, которые должны учитывать при [](insider-risk-management.md#workflow) первоначальном планировании и в процесс управления внутри организации, — это люди из следующих областей организации:

- Информационные технологии
- Соответствие требованиям
- Конфиденциальность
- Безопасность
- Управление персоналом
- Юридические аспекты

## <a name="determine-any-regional-compliance-requirements"></a>Определение региональных требований соответствия требованиям

В различных географических и организационных областях могут быть требования к обеспечению соответствия требованиям и конфиденциальности, которые отличаются от других областей организации. Работайте с заинтересованными лицами в этих областях, чтобы убедиться, что они понимают соответствие требованиям и средства контроля конфиденциальности в управлении рисками внутри организации и как их следует использовать в различных областях организации. В некоторых сценариях для соблюдения требований соответствия требованиям и конфиденциальности могут потребоваться политики, которые назначают или ограничивают некоторых заинтересованных лиц в расследованиях и делах на основе дела пользователя, нормативных или законодательных требований для области.

Если у вас есть требования к определенным заинтересованным лицам, которые будут участвовать в расследованиях дела, в котором участвуют пользователи в определенных регионах, ролях или подразделениях, может потребоваться реализовать отдельные (даже [идентичные)](insider-risk-management-policies.md) внутренние политики управления рисками, нацеленные на различные регионы и регионы. Эта конфигурация упростит для соответствующих заинтересованных лиц оценку дел, соответствующих их ролям и регионам, и управление ими. Кроме того, может потребоваться создать процессы и политики для регионов, где следователи и рецензенты говорят на том же языке, что и пользователи, чтобы упростить процесс эскалации для оповещений и дел управления рисками внутри организации.

## <a name="plan-for-the-review-and-investigation-workflow"></a>Планирование рабочего процесса проверки и исследования

Выберите выделенных заинтересованных лиц для регулярного отслеживания и просмотра оповещений и дел в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/) Убедитесь, что вы знаете, как назначать различных заинтересованных лиц различным группам ролей, доступным в управлении рисками внутри организации.

В зависимости от структуры группы управления соответствием требованиям вы можете назначать пользователей определенным группам ролей для управления различными наборами функций управления рисками внутри организации. Чтобы просмотреть вкладку "Разрешения" в Центре безопасности и соответствия & Безопасности Office 365  и управлять группами ролей, необходимо быть назначенной группе ролей "Управление организацией" или ролью "Управление  ролями".  Выберите один из этих параметров группы ролей при настройке управления рисками внутри организации:

| **Группа ролей** | **Разрешения роли** |
| :------------- | :------------------- |
| **Управление рисками внутри организации** | Используйте эту группу ролей для управления управлением рисками внутри организации в одной группе. Добавив все учетные записи пользователей для назначенных администраторов, аналитиков и следователей, вы можете настроить разрешения управления рисками внутри организации в одной группе. Эта группа ролей содержит все роли разрешений для управления рисками внутри организации. Эта конфигурация является самым простым способом быстрого начала работы с управлением рисками внутри организации и подходит для организаций, которые не нуждаются в отдельных разрешениях, определенных для отдельных групп пользователей.|
| **Администратор управления рисками для внутренней оценки** | Используйте эту группу ролей для первоначальной настройки управления рисками внутри организации, а затем для разделения администраторов внутри организации на определенную группу.  Пользователи в этой группе ролей могут создавать, читать, обновлять и удалять внутренние политики управления рисками и глобальные параметры. |
| **Аналитики по управлению внутренними рисками** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве аналитиков по рискам внутри компании. Пользователи в этой группе ролей могут получать доступ ко всем оповещениям, делам и шаблонам уведомлений об управлении рисками внутри организации. Они не могут получить доступ к обозревателю контента с insider risk. |
| **Исследователи управления внутренними рисками** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве участников расследования по данным о рисках. Пользователи в этой группе ролей могут получать доступ ко всем оповещениям, делам, шаблонам уведомлений и обозревателю содержимого для всех случаев. |

## <a name="understand-requirements-and-dependencies"></a>Требования и зависимости

В зависимости от того, как вы планируете реализовать политики управления рисками внутри организации, необходимо иметь подходящие лицензионные подписки на Microsoft 365, а также понимать и планировать некоторые необходимые условия для решения.

**Лицензирование:** Управление рисками внутри организации доступно в рамках широкого выбора подписок на лицензирование Microsoft 365. Подробные сведения см. в статье ["Начало работы с управлением рисками внутри](insider-risk-management-configure.md#subscriptions-and-licensing) организации".

Если у вас нет плана Microsoft 365 корпоративный E5 и вы хотите попробовать управление рисками внутри организации, вы можете добавить [Microsoft 365](/office365/admin/try-or-buy-microsoft-365) в существующую подписку или зарегистрироваться для пробной подписки на Microsoft 365 корпоративный E5. [](https://www.microsoft.com/microsoft-365/enterprise)

**Требования шаблона политики:** В зависимости от шаблона политики необходимо понимать и планировать некоторые требования перед настройкой управления insider risk management в организации.

- При использовании  кражи данных путем выхода из шаблона пользователя необходимо настроить соединители Microsoft 365 HR для периодического импорта сведений о дате прекращения действия для пользователей в организации. [Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 HR для организации см. в статье "Импорт данных с помощью соединители отдела кадров".
- При  использовании шаблонов утечек данных необходимо настроить по крайней мере одну политику защиты от потери данных (DLP), чтобы определить конфиденциальную информацию в организации и получать оповещения о рисках внутри организации для оповещений политики защиты от потери данных с высокой степенью серьезности. Пошаговые инструкции по настройке политик DLP для организации см. в статье "Создание, тестирование и настройка политики [DLP".](create-test-tune-dlp-policy.md)
- При использовании **шаблонов** нарушения политики безопасности необходимо включить Microsoft Defender для конечной точки для интеграции управления insider risk management в Центре безопасности Защитника для импорта оповещений о нарушении безопасности. [Пошаговая](/windows/security/threat-protection/microsoft-defender-atp/advanced-features) инструкция по настройке расширенных функций в Microsoft Defender содержит пошаговую инструкцию по интеграции Защитника для конечных точек с управлением рисками внутри организации.
- При  использовании несоединяемых шаблонов пользователей необходимо настроить соединители Microsoft 365 HR для периодического импорта сведений о производительности или понижении статуса для пользователей в организации. [Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 HR для организации см. в статье "Импорт данных с помощью соединители отдела кадров".

## <a name="test-with-a-small-group-of-users-in-a-production-environment"></a>Тестирование с небольшой группой пользователей в рабочей среде

Прежде чем широкое включение решения в производственной среде, можно рассмотреть возможность тестирования политик с небольшим набором пользователей в производственной среде при проведении необходимых проверок соответствия требованиям, конфиденциальности и юридических требований в организации. Для оценки управления insider risk в тестовой среде потребуется создать имитацию действий пользователя и другие сигналы для создания оповещений для рассмотрения и дел для обработки. Этот подход не является практическим для большинства организаций, поэтому предпочтительнее тестирование управления рисками внутри организации с небольшой группой пользователей в рабочей среде.

Чтобы обеспечить конфиденциальность в средстве, оохраняемую функцию анонимизации в параметрах политики, оохраняемую для анонимизации отображаемого имени пользователей в консоли управления рисками внутри организации, во время этого тестирования. Этот параметр помогает защитить конфиденциальность пользователей, которые имеют совпадения с политикой, и повышает объектность при анализе и анализе оповещений о insider risk.

Если вы не видите оповещений сразу после настройки политики управления рисками внутри организации, это может означать, что минимальное пороговое значение риска еще не было установлено. Хороший способ проверить, активна ли политика и работает ли она, — проверить, находится  ли пользователь в области действия политики на странице "Пользователи".
