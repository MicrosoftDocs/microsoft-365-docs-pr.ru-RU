# <a name="define-issues-and-assign-users-in-advanced-ediscovery-classic"></a>Определение проблем и назначение пользователей в Advanced eDiscovery (классическая)

> [!NOTE]
> Чтобы можно было использовать Advanced eDiscovery, требуется подписка на Office 365 E3 с надстройкой Advanced Compliance или E5 для организации. Если у вас этого плана нет и вы хотите попробовать Advanced eDiscovery, можете [зарегистрироваться для получения пробной версии Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
В Advanced eDiscovery можно определить одну или несколько проблем в рамках дела. Определение проблем позволяет далее классифицифицация разделов. При подключении к новому делу предоставляется одна проблема по умолчанию. Вы можете изменить имя проблемы по умолчанию и назначить ее пользователям. 
  
## <a name="adding-or-editing-an-issue-and-assigning-users"></a>Добавление или изменение проблемы и назначение пользователей

1. На **вкладке \> "Релевантность" выберите** \> **"Проблемы".**
    
    ![Элементы для оценивания на вкладке "Настройка релевантности"](../media/dfd8f9ef-b167-4ed9-980e-00ae98a97169.png)
  
2. Чтобы добавить проблему, щелкните значок ** + **. **Отобразилось** диалоговое окно "Добавление проблемы". 
    
    ![Добавление элементов для оценивания на вкладке "Настройка релевантности"](../media/c8e94982-139a-472a-b85d-282f2d742046.png)
  
    Чтобы изменить проблему, щелкните значок **редактирования.** 
    
3. В **имени проблемы** введите имя, которое является описательным и значимым для дела. 
    
4. В **описании** введите сведения о проблеме.
    
5. Чтобы включить **этот параметр,** выберите параметр "Включить параллельное обучение". Этот параметр позволяет нескольким рецензентам работать над одной проблемой одновременно (в отдельных примерах). 
    
6. В **списке**"Все  пользователи" выберите пользователя, который будет назначен проблеме, и щелкните стрелку вправо, чтобы добавить пользователя в список выбранных **пользователей.** Повторите это при необходимости. В окне выше в качестве выбранного пользователя отображается "Администратор". 
    
    > [!NOTE]
    > Назначение пользователям проблем можно изменить до или после цикла обучения релевантности. 
  
7. В **списке "Выбранные пользователи"** рядом с именем выбранного пользователя выберите один из следующих режимов выборки: 
    
  - **On**: The files can be viewed and tagged. Это значение по умолчанию.
    
  - **Бездействие:** файлы можно просмотреть; пометка необязательна.
    
  - **Выключите:** файлы невозможно просмотреть или пометить.
    
8. После добавления проблем нажмите кнопку **"ОК".**
    
## <a name="deleting-issues"></a>Удаление проблем

Проблемы могут быть удалены (то есть удалены из базы данных) только сразу после их определения, а фактическая работа по этой проблеме не была сделана. 
  
1. На **вкладке \> "Релевантность" выберите** **"Проблемы".**
    
2. Выберите проблему, удаляемую из базы данных, и нажмите кнопку **"Удалить".**
    
3. Отобразилось сообщение с подтверждением. Нажмите кнопку **Да** для подтверждения. 
    
4. Нажмите кнопку **ОК**.
