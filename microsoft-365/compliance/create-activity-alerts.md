# <a name="create-activity-alerts"></a>Создание оповещений об активности

Вы можете создать оповещение об активности, которое будет отправлять вам уведомление по электронной почте при выполнении пользователями определенных действий в Office 365. Оповещения об активности подобны поиску событий в журнале аудита, за исключением того, что вам будет отправлено сообщение электронной почты, когда происходит событие для действия, для которое вы создали оповещение. 
  
 **Почему вместо поиска в журнале аудита используются оповещения об активности?** Могут быть определенные виды действий или действий, выполняемые определенными пользователями, о которые вы действительно хотите знать. Вместо того чтобы искать эти действия в журнале аудита, вы можете использовать оповещения о действиях, чтобы Microsoft 365 отправил вам сообщение электронной почты при выполнении пользователями этих действий. Например, вы можете создать оповещение об активности, чтобы уведомить вас об удалении пользователем файлов в SharePoint, или создать оповещение о том, что пользователь окончательно удаляет сообщения из своего почтового ящика. Отправленные по электронной почте уведомления содержат сведения о том, какое действие было выполнено, и о пользователе, который его выполнил.

> [!NOTE]
> Оповещения об активности являются неподготовленными. Рекомендуется начать использовать политики оповещений в Центре безопасности и соответствия требованиям вместо создания новых оповещений об активности. Политики оповещений предоставляют дополнительные функции, такие как возможность создания политики оповещений, которая вызывает  оповещение при выполнении пользователем указанного действия, и отображение оповещений на странице "Просмотр оповещений" в Центре безопасности и соответствия требованиям. Дополнительные сведения [см. в политиках оповещений.](alert-policies.md)
  
## <a name="confirm-roles-and-configure-audit-logging"></a>Подтверждение ролей и настройка ведения журнала аудита

- Для управления оповещениями об активности вам должна быть назначена роль конфигурации организации в Центре & соответствия требованиям. По умолчанию эта роль назначена администратору соответствия требованиям и группам ролей управления организацией. Дополнительные сведения о добавлении членов в группы ролей см. в подсети "Предоставить пользователям доступ к Центру безопасности [& соответствия требованиям".](../security/office-365-security/grant-access-to-the-security-and-compliance-center.md)
    
- Прежде чем приступить к использованию оповещений об активности, вам (или другому администратору) необходимо включить ведение журнала аудита для вашей организации. Для этого просто нажмите кнопку "Начать запись действий пользователя и **администратора"** на странице **оповещений об активности.** (Если вы не видите эту ссылку, аудит для вашей организации уже включен.) Вы также можете включить аудит  на странице поиска в журнале аудита в  Центре безопасности & соответствия требованиям (перейдите к поиску \> **в журнале аудита поиска).** Это необходимо сделать только один раз для организации.
  
- Вы можете создавать оповещения для тех же действий, которые можно искать в журнале аудита. Список [распространенных](#more-information) сценариев (и отслеживаемого действия), для создания оповещений см. в разделе "Дополнительные сведения". 
    
- Вы можете  использовать страницу оповещений об активности в Центре безопасности & соответствия требованиям для создания оповещений только для действий пользователей, указанных в адресной книге организации. Эту страницу нельзя использовать для создания оповещений о действиях внешних пользователей, не указанных в адресной книге. 
    
## <a name="create-an-activity-alert"></a>Создание оповещения об активности

1. Перейдите по ссылке [https://protection.office.com/managealerts](https://protection.office.com/managealerts).
    
2. Выполните вход с помощью рабочей или учебной учетной записи.
    
3. На странице **оповещений об активности** нажмите ![ значок "Добавить". ](../media/8ee52980-254b-440b-99a2-18d068de62d3.gif) 

   Отобразить отображаемую страницу для создания оповещения об активности.

    
    ![Создание оповещения об активности](../media/53888bd5-9fa2-4398-8ccc-1a9dc72517ac.png)
  
4. Заполняйте следующие поля, чтобы создать оповещение об активности:
    
    а. **Name** — введите имя оповещения. Имена оповещений должны быть уникальными в организации.
    
    б. **Описание** (необязательно) — опишите оповещение, например отслеживаемую активность и пользователей, а также пользователей, на которые отправляются уведомления по электронной почте. Описания предоставляют быстрый и простой способ описать назначение оповещения другим администраторам.
    
    в. **Тип оповещения:** убедитесь, что выбран **параметр Custom.** 

    г. **Отправьте это оповещение при** нажатии **кнопки "Отправить это оповещение"** при настройке этих двух полей:
    
    - **Действия** — щелкните в выпадаемом списке, чтобы отобразить действия, для которые можно создать оповещение. Это тот же список действий, который отображается при поиске в журнале аудита. Можно выбрать одно или несколько определенных действий или щелкнуть имя группы действий, чтобы выбрать все действия в группе. Описание этих действий см. в разделе "Аудит действий" в разделе ["Поиск в журнале аудита".](search-the-audit-log-in-security-and-compliance.md#audited-activities) Когда пользователь выполняет какие-либо действия, добавленные в оповещение, отправляется уведомление по электронной почте. 
    
     - **Пользователи** — щелкните это поле, а затем выберите одного или несколько пользователей. Если пользователи в этом поле выполняют действия,  добавленные в поле "Действия", будет отправлено оповещение. Оставьте **поле "Пользователи"** пустым, чтобы отправить оповещение, когда любой пользователь в вашей организации выполняет действия, указанные оповещением. 

    д. Отправьте это оповещение по ссылке **"Отправить** это оповещение", а затем в поле "Получатели" введите имя, чтобы добавить пользователей, которые  получат уведомление по электронной почте, когда пользователь (указанный в поле "Пользователи") выполнит действие (указанное в поле "Действия").   Обратите внимание, что вы добавлены в список получателей по умолчанию. Вы можете удалить свое имя из этого списка.
    
5. Нажмите **кнопку** "Сохранить", чтобы создать оповещение. 
    
    Новое оповещение отображается в списке на странице **оповещений об активности.** 
    
    ![Список оповещений отображается на странице оповещений об активности](../media/02b774f2-1719-41de-bbc9-5e5b7576f335.png)
  
    Состояние оповещения — **"В** состоянии". Обратите внимание, что также указаны получатели, которые получат уведомление по электронной почте при отправке оповещения. 
  
## <a name="turn-off-an-activity-alert"></a>Отключение оповещения о действии

Вы можете отключить оповещение о действии, чтобы уведомление по электронной почте не было отправлено. После отключения оповещения о действии оно по-прежнему отображается в списке оповещений об активности для организации, и вы по-прежнему можете просматривать его свойства.
  
1. Go to Go to [https://protection.office.com/managealerts](https://protection.office.com/managealerts) .
    
2. Выполните вход с помощью учетной записи вашей организации или учебного заведения.
    
3. В списке оповещений об активности для организации щелкните оповещение, которое нужно отключить.
    
4. На странице **"Изменение оповещения"** щелкните переключатель "Вкл.", чтобы изменить состояние на **"Выкл",** и нажмите кнопку **"Сохранить".** 
    
    Состояние оповещения на страницах  оповещений об активности **отключено.** 
    
Чтобы снова включить оповещение об активности,  просто повторите эти действия и нажмите переключатель "Выкл.", чтобы изменить состояние на **"Включено".**

- Для создания и изменения оповещений об активности также можно использовать в PowerShell Центра безопасности и соответствия требованиям с помощью & **New-ActivityAlert** и **Set-ActivityAlert.** При использовании этих cmdlets для создания или изменения оповещений об активности следует иметь в виду следующее: 
    
  - Если вы добавляете действие в оповещение, которое не указано  в выпадаемом списке "Действия", на странице свойств отображается сообщение "Это оповещение имеет пользовательские операции, не указанные в списке".

Этот внешний пользователь будет указан в списке получателей оповещения. Но если удалить этого внешнего пользователя из оповещения, этот пользователь не сможет быть добавлен в оповещение повторно с помощью страницы "Изменение **оповещения".** You'll have to re-add the external user using the **Set-ActivityAlert** cmdlet, or use the **New-ActivityAlert** cmdlet to add the same (or different) external user to a new alert.
