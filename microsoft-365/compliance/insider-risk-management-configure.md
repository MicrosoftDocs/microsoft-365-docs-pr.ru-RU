# <a name="get-started-with-insider-risk-management"></a>Начало работы с управлением внутренними рисками

Используйте политики управления рисками для оценки рискованных действий и средств управления в Организации. Выполните указанные ниже действия, чтобы настроить необходимые компоненты и настроить политику управления рисками для предварительной оценки.

>[!IMPORTANT]
>Решение по управлению рисками для Microsoft 365 для оценки безопасности предоставляет возможность клиента для упрощения внутреннего управления на уровне пользователя. Администраторы уровня клиента могут настроить разрешения на предоставление доступа к этому решению для участников Организации и Настройка соединителей данных в центре соответствия требованиям Microsoft 365 для поддержки идентификации потенциально опасной информации на уровне пользователя. Клиенты могут получить подробные сведения о поведении, символе или материале о производительности, связанном с трудовым занятием, могут рассчитываться администратором и предоставляться для других пользователей в Организации.

Для получения дополнительных сведений о том, как политики риска для оценки могут помочь в управлении рисками в Организации, ознакомьтесь с разкрывающимся [руководством "Управление рисками в Microsoft 365](insider-risk-management.md)".

## <a name="before-you-begin"></a>Прежде чем начать

Прежде чем приступить к управлению рисками для оценки, необходимо подтвердить вашу [подписку на Microsoft 365](https://www.microsoft.com/microsoft-365/compare-all-microsoft-365-plans) и все надстройки. Чтобы получить доступ к управлению рисками и использовать ее, ваша организация должна иметь одну из следующих подписок или надстроек:

- Подписка на Microsoft 365 (платная или пробная версия)
- Подписка на Microsoft 365 E3 + надстройка для обеспечения соответствия требованиям корпорации Майкрософт 365
- Подписка на Microsoft 365 E3 + надстройка "Управление рисками для Майкрософт 365 для участников программы предварительной оценки"
- Подписка на Microsoft 365 A5 (платная или пробная версия)
- Подписка на Microsoft 365 a3 и надстройка Microsoft 365 A5 для обеспечения соответствия требованиям
- Подписка на Microsoft 365 a3 и надстройка "Управление рисками Майкрософт 365 A5"

Пользователям, включенным в политики управления рисками для предварительной оценки, должна быть назначена одна из указанных выше лицензий.

Если у вас нет существующего плана Microsoft 365 корпоративный, и вы хотите испытать ознакомительную версию управления рисками, вы можете [Добавить Microsoft 365](https://docs.microsoft.com/office365/admin/try-or-buy-microsoft-365) в существующую подписку или [зарегистрироваться для получения ознакомительной](https://www.microsoft.com/microsoft-365/enterprise) версии Microsoft 365 Enterprise.

## <a name="step-1-enable-permissions-for-insider-risk-management"></a>Шаг 1: Включение разрешений для управления рисками для оценки безопасности

Существует четыре группы ролей, которые используются для настройки разрешений на управление функциями управления рисками для предварительной оценки. Чтобы продолжить выполнение этих действий, администратору клиента необходимо назначить группу ролей **администратора "** **Управление рисками** " или "Управление рисками". Чтобы получить доступ к функциям управления рисками и управлять ими после начальной настройки, пользователи должны быть членами по крайней мере одной группы ролей "Управление рисками".

В зависимости от структуры группы управления соответствием вы можете назначать пользователям конкретные группы ролей для управления различными наборами функций управления рисками для участников. При настройке управления рисками для оценки выберите следующие параметры группы ролей:

| **Группа ролей** | **Разрешения роли** |
| :---- | :---------------- |
| **Предварительное управление рисками** | Используйте эту группу ролей, чтобы управлять управлением рисками для оценки Организации в одной группе. Добавляя все учетные записи пользователей для назначенных администраторов, аналитик и изучений, вы можете настроить разрешения на управление рисками для участников в одной группе. Эта группа ролей содержит все роли разрешений по управлению рисками для оценки. Эта конфигурация является самым простым способом начать работу с управлением рисками для оценки и подходит для организаций, которым не требуются отдельные разрешения, определенные для отдельных групп пользователей.|
| **Администратор управления рисками для участников** | Используйте эту группу ролей, чтобы первоначально настроить управление рисками для предварительной оценки и позднее, чтобы отделить администраторов о рисках для предварительной оценки в определенной группе.  Пользователи в этой группе ролей могут создавать, читать, обновлять и удалять политики управления рисками для оценки, глобальные параметры и назначения групп ролей. |
| **Аналитики управления рисками для участников** | Используйте эту группу для назначения разрешений пользователям, которые будут использоваться в качестве аналитиков вариантов риска для оценки. Пользователи в этой группе ролей могут получать доступ ко всем шаблонам оповещений, обращений и уведомлений об управлении рисками для оценки. Они не могут получать доступ к обозревателю контента для оценки риска. |
| **Исследования по управлению рисками для участников** | Используйте эту группу для назначения разрешений пользователям, которые будут использоваться в качестве расследований данных о рисках для оценки. Пользователи в этой группе ролей могут получать доступ ко всем оповещениям об управлении рисками, обращениям, заданным шаблонам и обозревателю контента. |

### <a name="add-users-to-an-insider-risk-management-role-group"></a>Добавление пользователей в группу ролей "Управление рисками"

Выполните следующие действия, чтобы добавить пользователей в группу ролей "Управление рисками" для участников:

1. Войдите в систему [https://protection.office.com/permissions](https://protection.office.com/permissions) , используя учетные данные для учетной записи администратора в организации Microsoft 365.

2. В центре безопасности &amp; и соответствия требованиям выберите **разрешения**. Выберите ссылку для просмотра ролей в Office 365 и управления ими.

3. Выберите группу ролей "Управление рисками для участников", к которой нужно добавить пользователей, а затем выберите команду **изменить группу ролей**.

4. Выберите **выбрать элементы** в левой области навигации, а затем нажмите кнопку **изменить**.

5. Нажмите кнопку **Добавить** , а затем установите флажок для всех пользователей, которых вы хотите добавить в группу ролей.

6. Нажмите кнопку **Добавить**, а затем нажмите кнопку **Готово**.

7. Нажмите кнопку **сохранить** , чтобы добавить пользователей в группу ролей. Нажмите кнопку **Закрыть** , чтобы завершить процедуру.

## <a name="step-2-enable-the-audit-log"></a>Шаг 2: Включение журнала аудита

Центр управления рисками для оценки безопасности использует журналы аудита для сведений о пользователях и действий, настроенных в политиках. Журналы аудита представляют собой сводку по всем действиям, связанным с политикой управления рисками для оценки или в момент изменения политики.

Пошаговые инструкции по включению аудита содержатся [в разделе Включение или отключение поиска в журнале аудита](turn-audit-log-search-on-or-off.md). После включения аудита отображается сообщение с сообщением о том, что журнал аудита подготавливается, и вы можете выполнить поиск в течение нескольких часов после завершения подготовки. Это действие необходимо выполнить только один раз. Для получения дополнительных сведений об использовании журнала аудита обратитесь к разделу [Поиск в журнале аудита](search-the-audit-log-in-security-and-compliance.md).

## <a name="step-3-configure-prerequisites-for-templates"></a>Шаг 3: Настройка необходимых компонентов для шаблонов

Большинство шаблонов управления рисками для предварительной оценки имеют необходимые компоненты, которые необходимо настроить для индикаторов политики, чтобы создать подходящие оповещения о действиях. Настройте соответствующие компоненты в зависимости от политик, которые планируется настроить для вашей организации.

Если вы настраиваете политику на основе *оскорбительного языка в* шаблоне политики электронной почты, вы можете пропустить этот шаг и перейти непосредственно к **шагу 4**.

### <a name="configure-microsoft-365-hr-connector"></a>Настройка Microsoft 365 HR Connector

Центр управления рисками для предварительной оценки поддерживает импорт данных пользователей и журналов, импортированных с платформ управления рисками сторонних производителей и платформ управления персоналом. Соединитель данных отдела кадров Microsoft 365 (HR) позволяет получить данные о персонале из CSV-файлов, в том числе даты увольнения пользователей, последние даты занятости, уведомления плана по улучшению производительности, действия по анализу производительности и состояние изменения уровня задания. Эти данные помогают отменять индикаторы оповещений в политиках "Управление рисками" и являются важной частью настройки полного охвата управления рисками в Организации. Если для вашей организации настроено несколько соединителей для отдела кадров, управление рисками для предварительной оценки будет автоматически получать индикаторы со всех соединителей отдела кадров.
Соединитель Microsoft 365 HR необходим при использовании следующих шаблонов политик:

- Частичная кража данных пользователей
- Нарушения политики безопасности за счет посторонних пользователей
- Нарушения политики безопасности пользователями дисгрунтлед
- Утечки данных пользователями дисгрунтлед

Пошаговое руководство по настройке соединителя Microsoft 365 HR для организации можно посмотреть в статье [Настройка соединителя для импорта](import-hr-data.md) статьи с данными о HR. После настройки соединителя кадров вернитесь к этим этапам настройки.

### <a name="configure-data-loss-prevention-dlp-policies"></a>Настройка политик защиты от потери данных (DLP)

С помощью политик защиты от нежелательной информации можно определить преднамеренное или случайное воздействие конфиденциальной информации нежелательной стороне для оповещений DLP с высоким уровнем серьезности. При настройке политики управления рисками для предварительной оценки с помощью любого из шаблонов **утечки данных** необходимо назначить политику защиты от потери данных.

Политики защиты от потери данных помогают определить, какие пользователи могут активировать риск риска при управлении рисками в процессе подготовки для получения оповещений о высокой степени серьезности DLP для конфиденциальной информации и являются важной частью настройки всего охвата управления рисками в Организации. Для получения дополнительных сведений об управлении рисками и интеграции и планировании политик DLP ознакомьтесь с разделами [политики управления рисками для участников](insider-risk-management-policies.md#general-data-leaks).

>[!IMPORTANT]
>Убедитесь, что вы выполнили следующие действия:
>
>- Вы осознаете и правильно настроите пользователей в области в политиках управления рисками DLP и Insider для получения ожидаемого покрытия политики.
>- Убедитесь, что параметры **отчетов об инцидентах** в политике защиты от потери данных для управления рисками, используемыми для этих шаблонов, настроены на получение оповещений о *высоком* уровне серьезности. Оповещения **об** управлении рисками для участников программы предварительной оценки не будут создаваться в политиках защиты от потери данных с установленным на *низком* или *среднем*уровне.

При использовании следующих шаблонов политики необходима политика защиты от потери данных:

- Общие утечки данных
- Утечки данных по приоритетным пользователям

В этой статье описывается [Создание, тестирование и Настройка политики DLP](create-test-tune-dlp-policy.md) для пошагового указания по настройке политик защиты от потери данных в Организации. После настройки политики защиты от потери данных вернитесь к этим этапам настройки.

### <a name="configure-priority-user-groups"></a>Настройка приоритетных групп пользователей

Управление рисками для системы оценки безопасности включает в себя поддержку по назначению приоритетных групп пользователей политикам, которые помогают обеспечить уникальную степень риска для пользователей с критическими положениями, высокими уровнями данных и доступа к сети, а также прошлую историю поведения риска. Создание группы приоритетов пользователей и назначение пользователям групповых политик области справки в уникальные обстоятельства, представленные этими пользователями.

При использовании следующих шаблонов политики необходимо указать группу приоритетов пользователей:

- Нарушения политики безопасности по приоритету пользователей 
- Утечки данных по приоритетным пользователям

Пошаговое руководство по созданию группы приоритетов пользователей можно найти в статье [Приступая к работе с параметрами управления рисками для предварительной оценки](insider-risk-management-settings.md#priority-user-groups-preview) . Когда вы настроили группу приоритетов пользователей, вернитесь к этим этапам настройки.

## <a name="step-4-configure-insider-risk-settings"></a>Шаг 4: Настройка параметров риска для оценки

[Параметры риска для участников программы предварительной](insider-risk-management-settings.md) оценки применяются ко всем политикам управления рисками для оценки, независимо от шаблона, выбранного при создании политики. Параметры настраиваются с помощью элемента управления " **параметры риска" для предварительной оценки** , расположенного в верхней части всех вкладок управления рисками для оценки. Эти параметры управляют конфиденциальностью, индикаторами, мониторингом окон и интеллектуальным обнаружением.

Перед настройкой политики определите следующие параметры риска для предварительной оценки:

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и выберите **параметры риска для участников программы предварительной оценки** в правом верхнем углу любой страницы.
2. На странице " **Конфиденциальность** " выберите параметр конфиденциальности для отображения имен пользователей для оповещений политики.
3. На странице " **индикаторы** " выберите индикаторы оповещений, которые необходимо применить ко всем политикам риска для оценки.

    >[!IMPORTANT]
    >Чтобы получать оповещения о рискованных действиях, определенных в политиках, необходимо выбрать один или несколько индикаторов.

4. На странице " **временные рамки политики** " выберите [временные рамки политики](insider-risk-management-settings.md#policy-timeframes) , которые будут применяться для вступления в действия пользователя, когда они вызывают соответствующую политике риска для оценки.
5. На странице " **интеллектуальные обнаружения** " настройте политики риска для предварительной оценки следующими параметрами:
    - [Обнаружение аномалий](insider-risk-management-settings.md#anomaly-detections)
    - [Нежелательные обнаружения языков](insider-risk-management-settings.md#offensive-language-detections)
    - [Уровень громкости оповещений](insider-risk-management-settings.md#alert-volume)
    - [Состояние оповещений Advanced Threat Protection в защитнике Microsoft 365](insider-risk-management-settings.md#microsoft-defender-advanced-threat-protection-preview)
    - [Параметры домена](insider-risk-management-settings.md#domains-preview)
6. На странице **оповещение об экспорте** включите возможность экспорта оповещений о рисках для участников программы предварительной оценки с помощью API управления Office 365 (при необходимости).
7. На странице **приоритет групп пользователей** создайте группу приоритетов пользователей и добавьте пользователей, если они не созданы на **шаге 3**.
8. Нажмите кнопку **сохранить** , чтобы включить эти параметры для политик оценки риска.

## <a name="step-5-create-an-insider-risk-management-policy"></a>Шаг 5: Создание политики управления рисками для предварительной оценки

К политикам управления рисками для предварительной оценки относятся назначенные пользователи и определены типы индикаторов риска, настроенных для оповещений. Прежде чем действия смогут инициировать оповещения, необходимо настроить политику.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. Выберите **создать политику** , чтобы открыть мастер политик
3. На странице **Создание политики для оценки риска** заполните следующие поля:
    - **Name (обязательно)**: введите понятное имя политики.
    - **Description (необязательно)**: введите описание политики.
    - **Выбор шаблона политики (обязательно)**: выберите один из [шаблонов политики](insider-risk-management-policies.md#policy-templates) , чтобы определить типы индикаторов риска, контролируемые политикой.

    >[!IMPORTANT]
    >Большинство шаблонов политики имеют необходимые компоненты, которые необходимо настроить для создания соответствующих оповещений в политике. Если вы не настроили соответствующие необходимые компоненты политики, ознакомьтесь с **шагом 3** .

4. Нажмите кнопку **Далее** , чтобы продолжить.
5. На странице **Пользователи** выберите **Добавить пользователя или группу** или **выберите приоритетные группы пользователей** , чтобы определить, какие пользователи или приоритетные группы пользователей включены в политику, в зависимости от выбранного шаблона политики. Установите флажок **все пользователи и группы с включенной поддержкой почты,** если это возможно (если вы не выбрали приоритетный шаблон на основе пользователей). Нажмите кнопку **Далее** , чтобы продолжить.
6. На странице **Указание содержимого для определения приоритета (необязательно)** можно назначить источникам приоритеты для улучшения показателей риска. Однако некоторые действия не приводят к созданию оповещения, если связанный контент не содержит встроенных или настраиваемых типов конфиденциальной информации или не был указан как приоритет на этой странице:
    - **Сайты SharePoint**: выберите пункт **Добавить сайт SharePoint** и выберите Организации SharePoint, для которых необходимо задать приоритет. Например, *"Group1@contoso.sharepoint.com/sites/group1"*.
    - **Тип конфиденциальной информации**: выберите **Добавить тип конфиденциальной информации** и выберите типы чувствительности, которые нужно назначить приоритетам. Например, *"номер банковского счета США"* и *"номер кредитной карты"*.
    - **Метки конфиденциальности**: выберите **Добавить метку конфиденциальности** и выберите метки, для которых требуется задать приоритет. Например, *"конфиденциальный"* и *"секрет"*.
7. Нажмите кнопку **Далее** , чтобы продолжить.
8. На странице " **Выбор индикаторов политики** " вы увидите [индикаторы](insider-risk-management-settings.md#indicators) , которые вы определили как доступные на странице "Индикаторы **параметров оценки риска**"  >  **Indicators** . Если вы выбрали шаблон " *утечки данных* " в начале работы мастера, необходимо выбрать политику защиты от потери данных в раскрывающемся списке **Политика защиты от потери** данных, чтобы включить индикаторы триггера для политики. Выберите индикаторы, которые необходимо применить к политике. Если вы предпочитаете не использовать параметры пороговых значений политики по умолчанию для этих индикаторов, отключите **Использование пороговых значений по умолчанию, рекомендуемых корпорацией Майкрософт** , и введите пороговые значения для каждого выбранного индикатора. Если вы выбрали по крайней мере один индикатор *Office* или *устройства* , выберите соответствующие **ускорители оценки риска** . Ускорители оценки риска применяются только для выбранных индикаторов.

    >[!IMPORTANT]
    >Если не удается выбрать индикаторы на этой странице, необходимо выбрать индикаторы, которые необходимо включить для всех политик на странице " **Insider risk management**  >  **Settings**  >  **индикаторы политики** управления рисками".

9. Нажмите кнопку **Далее** , чтобы продолжить.
10. На странице " **временные рамки политик** " вы увидите [условия окна активации](insider-risk-management-settings.md#policy-timeframes) для политики, которая находится на странице " **Insider risk settings**  >  **временные рамки политики** параметров для оценки риска".
11. Нажмите кнопку **Далее** , чтобы продолжить.
12. На странице " **Проверка** " Проверьте параметры, выбранные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы создать и активировать политику.

Настройте дополнительные политики по мере необходимости, используя инструкции, приведенные в шаге 4 данной статьи, или действия, описанные в статье [Создание новой политики риска для оценки](insider-risk-management-policies.md#create-a-new-policy).
