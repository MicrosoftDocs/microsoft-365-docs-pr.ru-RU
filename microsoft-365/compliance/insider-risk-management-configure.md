# <a name="get-started-with-insider-risk-management"></a>Начало работы с управлением внутренними рисками

Используйте внутренние политики управления рисками для определения рискованных действий и средств управления для работы с оповещениями о рисках в организации. Выполните следующие действия для настройки необходимых условий и настройки политики управления рисками внутри организации.

>[!IMPORTANT]
>Решение microsoft 365 для управления внутренними рисками предоставляет возможность на уровне клиента, помогая клиентам упростить внутреннее управление на уровне пользователей. Администраторы на уровне клиента могут настроить разрешения для предоставления доступа к этому решению для сотрудников организации и настроить соединители данных в Центре соответствия требованиям Microsoft 365 для импорта соответствующих данных для поддержки идентификации потенциально рискованных действий на уровне пользователя. Клиенты подтверждают, что администратор может вычислить информацию, связанную с поведением, характером или производительностью отдельного пользователя, и сделать ее доступной другим пользователям в организации. Кроме того, клиенты могут подтвердить, что им необходимо провести собственное полное исследование, связанное с поведением, характером или производительностью отдельного пользователя, связанными с работой, а не только полагаться на данные из службы управления рисками внутри организации. Клиенты несут полную ответственность за использование службы управления рисками в microsoft 365, а также за все связанные функции или службы в соответствии со всеми применимыми законами, включая законы, относящиеся к идентификации отдельных пользователей и любые действия по исправлению.

Дополнительные сведения о том, как политики insider risk могут помочь вам управлять рисками в организации, см. в руководстве по управлению рисками для программы оценки в [Microsoft 365.](insider-risk-management.md)

## <a name="subscriptions-and-licensing"></a>Подписки и лицензирование

Перед началом работы с управлением рисками внутри организации необходимо подтвердить подписку [на Microsoft 365](https://www.microsoft.com/microsoft-365/compare-all-microsoft-365-plans) и все надстройки. Чтобы получить доступ к управлению рисками внутри организации и использовать ее, у вашей организации должна быть одна из следующих подписок или надстройок:

- Подписка microsoft 365 E5 (платная или пробная версия)
- Подписка microsoft 365 E3 + надстройка соответствия требованиям Microsoft 365 E5
- Подписка microsoft 365 E3 + надстройка Microsoft 365 E5 Insider Risk Management
- Подписка на Microsoft 365 A5 (платная или пробная версия)
- Подписка на Microsoft 365 A3 + надстройка соответствия требованиям Microsoft 365 A5
- Подписка на Microsoft 365 A3 + надстройка microsoft 365 A5 Insider Risk Management
- Подписка на Microsoft 365 G5 (платная или пробная версия)
- Подписка на Microsoft 365 G3 + надстройка соответствия требованиям Microsoft 365 G5
- Подписка на Microsoft 365 G3 + надстройка Microsoft 365 G5 Insider Risk Management

Пользователям, включенным в внутренние политики управления рисками, должна быть назначена одна из вышеуказанных лицензий.

Если у вас нет плана Microsoft 365 корпоративный E5 и вы хотите попробовать управление рисками внутри организации, вы можете добавить [Microsoft 365](/office365/admin/try-or-buy-microsoft-365) в существующую подписку или зарегистрироваться для пробной подписки на Microsoft 365 корпоративный E5. [](https://www.microsoft.com/microsoft-365/enterprise)

## <a name="step-1-enable-permissions-for-insider-risk-management"></a>Шаг 1. Разрешение для управления рисками внутри организации

>[!Important]
>После настройки групп ролей может потребоваться до 30 минут, чтобы разрешения группы ролей применялись к назначенным пользователям в организации.

Существует четыре группы ролей, которые используются для настройки разрешений на управление функциями управления рисками внутри организации. Чтобы продолжить эти действия по настройке, администраторы  клиента должны сначала  назначить вас группе ролей "Управление рисками для программы оценки риска" или "Администратор управления рисками". Чтобы получить доступ к функциям управления insider risk management после начальной настройки и управлять ими, пользователи должны быть членами по крайней мере одной группы ролей управления рисками внутри организации.

В зависимости от структуры группы управления соответствием требованиям вы можете назначать пользователей определенным группам ролей для управления различными наборами функций управления рисками внутри организации. Выберите один из этих параметров группы ролей при настройке управления рисками внутри организации:

| **Группа ролей** | **Разрешения роли** |
| :---- | :---------------- |
| **Управление рисками внутри организации** | Используйте эту группу ролей для управления управлением рисками внутри организации в одной группе. Добавив все учетные записи пользователей для назначенных администраторов, аналитиков и следователей, вы можете настроить разрешения управления рисками внутри организации в одной группе. Эта группа ролей содержит все роли разрешений для управления рисками внутри организации. Эта конфигурация является самым простым способом быстрого начала управления рисками внутри организации и подходит для организаций, не нуждающихся в отдельных разрешениях, определенных для отдельных групп пользователей.|
| **Администратор управления рисками для внутренней оценки** | Используйте эту группу ролей для первоначальной настройки управления рисками внутри организации, а затем для разделения администраторов внутри организации на определенную группу.  Пользователи в этой группе ролей могут создавать, читать, обновлять и удалять политики управления рисками внутри организации, глобальные параметры и назначения групп ролей. |
| **Аналитики по управлению внутренними рисками** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве аналитиков по рискам внутри компании. Пользователи в этой группе ролей могут получать доступ ко всем оповещениям, делам и шаблонам уведомлений об управлении рисками внутри организации. Они не могут получить доступ к обозревателю контента с insider risk. |
| **Исследователи управления внутренними рисками** | Используйте эту группу для назначения разрешений пользователям, которые будут выступать в качестве участников расследования данных о рисках. Пользователи в этой группе ролей могут получать доступ ко всем оповещениям, делам, шаблонам уведомлений и обозревателю содержимого для управления рисками внутри организации. |

> [!NOTE]
> В настоящее время эти группы ролей не поддерживаются в privileged Identity Management (PIM). Дополнительные данные о PIM см. в руководстве "Назначение ролей [Azure AD в privileged Identity Management".](/azure/active-directory/privileged-identity-management/pim-how-to-add-role-to-user)

### <a name="add-users-to-an-insider-risk-management-role-group"></a>Добавление пользователей в группу ролей управления рисками внутри организации

Выполните следующие действия, чтобы добавить пользователей в группу ролей управления рисками внутри организации:

1. Во sign [https://protection.office.com/permissions](https://protection.office.com/permissions) into using credentials for an admin account in your Microsoft 365 organization.

2. В Центре соответствия &amp; требованиям безопасности перейдите к **"Разрешениям".** Выберите ссылку для просмотра ролей в Office 365 и управления ими.

3. Выберите группу ролей управления рисками внутри организации, в которую нужно добавить пользователей, а затем выберите **"Изменить группу ролей".**

4. Выберите **"Выбрать члены"** в левой области навигации, а затем выберите **"Изменить".**

5. Выберите  "Добавить", а затем — для всех пользователей, которые нужно добавить в группу ролей.

6. Выберите **"Добавить",** затем **"Готово"**

7. Выберите **"Сохранить",** чтобы добавить пользователей в группу ролей. Выберите **"Закрыть",** чтобы завершить действия.

## <a name="step-2-enable-the-audit-log"></a>Шаг 2. В включить журнал аудита

Для управления рисками внутри организации используются журналы аудита для анализа пользователей и действий, настроенных в политиках. Журналы аудита — это сводка всех действий, связанных с политикой управления рисками внутри организации или при любом ее правлении.

Пошаговая инструкция по включению аудита см. в инструкциях по включению и отключению поиска в [журнале аудита.](turn-audit-log-search-on-or-off.md) После того как вы включите аудит, отобразится сообщение о подготовке журнала аудита и о том, что вы можете выполнить поиск через несколько часов после завершения подготовки. Это действие необходимо сделать только один раз. Дополнительные сведения об использовании журнала аудита см. в [поиске в журнале аудита.](search-the-audit-log-in-security-and-compliance.md)

## <a name="step-3-configure-prerequisites-for-templates"></a>Шаг 3. Настройка необходимых условий для шаблонов

Большинство шаблонов управления рисками внутри организации имеют необходимые условия, которые необходимо настроить для индикаторов политики для создания соответствующих оповещений об активности. Настройте необходимые условия в зависимости от политик, которые планируется настроить для организации.

### <a name="configure-microsoft-365-hr-connector"></a>Настройка соединители Microsoft 365 HR

Управление рисками внутри организации поддерживает импорт данных пользователей и журналов, импортируемых из сторонних платформ управления рисками и управления кадрами. Соединиталь данных microsoft 365 human Resources (HR) позволяет получать данные о персонале из CSV-файлов, включая даты увольнения пользователей, даты последней занятости, уведомления о плане улучшения производительности, действия по проверке производительности и состояние изменения уровня должности. Эти данные помогают индикаторам оповещений в политиках управления рисками внутри организации и являются важной частью настройки полного охвата управления рисками в организации. Если в организации настроено несколько соединителю управления персоналом, управление рисками внутри организации автоматически извлекет индикаторы из всех соединители управления персоналом.

Соединители Microsoft 365 HR необходимы при использовании следующих шаблонов политики:

- Убывая кража данных пользователя
- Нарушения политики безопасности путем отступа пользователей
- Нарушения политики безопасности со стороны несоединых пользователей
- Утечки данных несовершимыми пользователями

[Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 hr для организации см. в статье "Настройка соединители для импорта данных отдела кадров". После настройки соединители отдела кадров вернись к этим шагам настройки.

### <a name="configure-data-loss-prevention-dlp-policies"></a>Настройка политик защиты от потери данных (DLP)

Управление рисками внутри организации поддерживает использование политик DLP, чтобы определить преднамеренное или случайное развещение конфиденциальной информации нежелательным сторонам для оповещений О DLP высокого уровня серьезности. При настройке политики управления рисками внутри  организации с любым из шаблонов утечек данных необходимо назначить политике определенную политику DLP.

Политики DLP помогают определять пользователей для активации оценки рисков в управлении рисками внутри организации для оповещений О DLP с высокой степенью серьезности для конфиденциальной информации и являются важной частью настройки полного охвата управления рисками в организации. Дополнительные сведения об управлении рисками внутри организации и интеграции политики DLP и планировании см. в политиках управления рисками [для insider.](insider-risk-management-policies.md#general-data-leaks)

>[!IMPORTANT]
>Убедитесь, что вы выполнили следующие работы:
>
>- Вы понимаете и правильно настраиваете пользователей в области как в политиках DLP, так и в политиках управления рисками внутри организации, чтобы получить ожидаемую область применения политики.
>- Убедитесь, что параметр **отчетов** об инцидентах в политике DLP для управления  insider risk management, используемый с этими шаблонами, настроен для оповещений высокого уровня серьезности. Оповещения для управления рисками внутри организации не создаются  из политик DLP с полем "Отчеты об инцидентах" на *низком* или среднем *уровне.*

Политика DLP требуется при использовании следующих шаблонов политики:

- Общие утечки данных
- Утечки данных по приоритетным пользователям

Пошаговые инструкции по настройке политик DLP для организации см. в статье "Создание, тестирование и настройка политики [DLP".](create-test-tune-dlp-policy.md) После настройки политики DLP вернись к этим шагам настройки.

### <a name="configure-priority-user-groups"></a>Настройка групп пользователей с приоритетами

Управление insider risk включает поддержку назначения приоритетных групп пользователей политикам, чтобы помочь идентификации уникальных действий риска для пользователей с критическими должностями, высокого уровня данных и сетевого доступа, а также прошлые истории поведения риска. Создание группы пользователей с приоритетом и назначение пользователей политикам области справки группы для уникальных условий, представляемых этими пользователями.

При использовании следующих шаблонов политики требуется группа пользователей с приоритетом:

- Нарушения политики безопасности пользователями с приоритетами
- Утечки данных по приоритетным пользователям

[Пошаговая](insider-risk-management-settings.md#priority-user-groups-preview) инструкция по созданию группы пользователей с приоритетами см. в статье "Начало работы с настройками управления рисками внутри организации". После настройки группы пользователей с приоритетом вернись к этим шагам настройки.

### <a name="configure-physical-badging-connector-optional"></a>Настройка физического соединители badging (необязательно)

Управление рисками внутри организации поддерживает импорт данных пользователей и журналов, импортируемых с физических платформ управления и доступа. Физический соединитель ненадежных подключений позволяет получать данные о доступе из JSON-файлов, включая ИД пользователей, ИД точек доступа, время и даты доступа, а также состояние доступа. Эти данные помогают индикаторам оповещений в политиках управления рисками внутри организации и являются важной частью настройки полного охвата управления рисками в организации. Если в организации настроено несколько физических соединителю, управление рисками внутри организации автоматически извлекет индикаторы из всех физических соединителю. Сведения из физического соединитела плохой нагрузки дополняет другие сигналы о внутренней опасности при использовании всех шаблонов политики insider risk.

>[!IMPORTANT]
>Чтобы политики управления рисками внутри организации могли использовать и сопоставляют данные сигналов, относящиеся к убывающих и завершенных пользователей, с данными событий от физических платформ управления и доступа, необходимо также настроить соединители Microsoft 365 HR. Если вы включите физический соединитеж, не включив соединители Microsoft 365 HR, политики управления рисками внутри организации будут обрабатывать события только для несанкционированного физического доступа пользователей в организации.

[Пошаговая](import-physical-badging-data.md) инструкция по настройке физического соединитела для организации см. в статье "Настройка соединители для импорта физических ненадежных данных". После настройки соединители вернись к этим шагам настройки.

## <a name="step-4-configure-insider-risk-settings"></a>Шаг 4. Настройка параметров insider risk

[Параметры insider risk применяются](insider-risk-management-settings.md) во всех политиках управления рисками внутри организации независимо от шаблона, выбранного при создании политики. Параметры настраиваются с  помощью управления "Параметры риска для оценки", расположенного в верхней части всех вкладок управления рисками внутри организации. Эти параметры контролируют конфиденциальность, индикаторы, окна мониторинга и интеллектуальные обнаружения.

Перед настройкой политики определите следующие параметры риска для внутренней оценки:

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для программы **insider** **и** выберите параметры риска для программы оценки в правом верхнем углу любой страницы.
2. На странице **"Конфиденциальность"** выберите параметр конфиденциальности для отображения имен пользователей для оповещений политики.
3. На странице **"Индикаторы"** выберите индикаторы оповещений, которые необходимо применить во всех политиках внутри системы риска.

    >[!IMPORTANT]
    >Чтобы получать оповещения о рискованных действиях, определенных в политиках, необходимо выбрать один или несколько индикаторов.

4. На странице **"Временные** рамки [](insider-risk-management-settings.md#policy-timeframes) политики" выберите временные рамки политики, которые должны входить в силу для пользователя, когда он инициирует совпадение с политикой insider risk.
5. На странице **интеллектуальных обнаружений** настройте следующие параметры для политик внутри риска:
    - [Обнаружение аномалий](insider-risk-management-settings.md#anomaly-detections)
    - [Уровень громкости оповещений](insider-risk-management-settings.md#alert-volume)
    - [Состояние оповещений в Microsoft Defender для конечной точки](insider-risk-management-settings.md#microsoft-defender-for-endpoint-preview)
    - [Параметры домена](insider-risk-management-settings.md#domains-preview)
6. На странице **"Экспорт оповещений"** в случае необходимости экспортировать сведения оповещений о insider risk с помощью API управления Office 365.
7. На странице **"Группы пользователей с приоритетом"** создайте группу пользователей с приоритетом и добавьте пользователей, если они не созданы на **шаге 3.**
8. На странице **потоков Power Automate** настройте поток из шаблонов потока insider risk flow или создайте новый поток. [Пошаговую инструкцию](insider-risk-management-settings.md#power-automate-flows-preview) см. в статье "Начало работы с настройками управления рисками внутри организации".
9. На странице **"Ресурсы** с приоритетом" настройте ресурсы приоритета так, чтобы они использовали данные из физического управления и платформы доступа, импортируемые физическим соединитетелем плохой конфигурации. [Пошаговую инструкцию](insider-risk-management-settings.md#priority-physical-assets-preview) см. в статье "Начало работы с настройками управления рисками внутри организации".
10. На странице **Microsoft Teams включите** интеграцию Microsoft Teams с управлением insider risk, чтобы автоматически создать команду для совместной работы пользователей или дела. [Пошаговую инструкцию](insider-risk-management-settings.md#microsoft-teams-preview) см. в статье "Начало работы с настройками управления рисками внутри организации".
11. Выберите  "Сохранить", чтобы включить эти параметры для политик внутри риска.

## <a name="step-5-create-an-insider-risk-management-policy"></a>Шаг 5. Создание политики управления рисками внутри организации

Политики управления рисками внутри организации включают в себя назначенного пользователя и определяют, какие типы индикаторов риска настроены для оповещений. Прежде чем действия смогут запускать оповещения, необходимо настроить политику.

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для программы оценки безопасности и выберите вкладку **"Политики".** 
2. Выберите **"Создать политику",** чтобы открыть мастер политики.
3. На странице **"Новая политика insider risk policy"** заполните следующие поля:
    - **Имя (обязательно)**: введите имя политики.
    - **Описание (необязательно).** Введите описание политики.
    - **Выберите шаблон политики (обязательно).** Выберите [](insider-risk-management-policies.md#policy-templates) один из шаблонов политики, чтобы определить типы индикаторов риска, отслеживаемого политикой.

    >[!IMPORTANT]
    >Большинство шаблонов политик имеют необходимые условия, которые необходимо настроить для создания соответствующих оповещений. Если вы не настроили необходимые условия для соответствующей политики, **см. шаг 3** выше.

4. Выберите **"Далее",** чтобы продолжить.
5. На  странице "Пользователи" выберите "Добавить пользователя или группу" или выберите группы пользователей "Приоритет", чтобы определить, какие пользователи или группы пользователей с приоритетом включены в политику в зависимости от выбранного шаблона политики.   Выберите **"Все пользователи и группы** с включенной поддержкой почты", если это возможно (если вы не выбрали шаблон, основанный на приоритете пользователя). Выберите **"Далее",** чтобы продолжить.
6. На странице **"Укажите, какое содержимое** необходимо назначить приоритету (необязательно)", можно назначить источники для приоритетов для повышения оценки риска. Однако некоторые действия не будут создавать оповещение, если только связанное содержимое не содержит встроенные или настраиваемые типы конфиденциальной информации или не было указано в качестве приоритета на этой странице:
    - **Сайты SharePoint:** выберите **"Добавить сайт SharePoint"** и выберите организации SharePoint, приоритеты для них. Например, *"group1@contoso.sharepoint.com/sites/group1".*
    - **Тип конфиденциальной информации:** выберите **"Добавить тип** конфиденциальной информации" и выберите типы конфиденциальности, которые необходимо расставить по приоритету. Например, *"Номер* банковского счета в США" и *"Номер кредитной карты".*
    - **Метки конфиденциальности:** выберите "Добавить метку **конфиденциальности"** и выберите метки, которые необходимо расставить по приоритету. Например, *"Конфиденциально"* и *"Секрет".*
7. Выберите **"Далее",** чтобы продолжить.
8. На странице **"Выбор индикаторов** политики" [](insider-risk-management-settings.md#indicators) вы увидите индикаторы, которые вы определили как доступные на странице "Индикаторы параметров риска для оценки  >   риска". Если вы выбрали  шаблон утечки данных в начале мастера, необходимо выбрать политику DLP из выпадающих списков политик **DLP,** чтобы включить индикаторы запуска для политики. Выберите индикаторы, которые необходимо применить к политике. Если вы предпочитаете не использовать пороговые значения политики по  умолчанию для этих индикаторов, отключаем рекомендуемые корпорацией Майкрософт пороговые значения по умолчанию и введите пороговые значения для каждого выбранного индикатора. Если вы выбрали хотя бы один  индикатор *Office*  или устройства, выберите нужные повышения для оценки риска. Повышение оценки риска применимо только к выбранным индикаторам.

    >[!IMPORTANT]
    >Если индикаторы на этой странице не могут быть выбраны, необходимо выбрать индикаторы, которые необходимо включить для всех политик на странице "Параметры политики управления рисками для оценки  >    >   риска".

9. Выберите **"Далее",** чтобы продолжить.
10. На странице **"Временные** рамки политики" [](insider-risk-management-settings.md#policy-timeframes) вы увидите условия окна активации для политики, которая находится на странице "Параметры политики для оценки   >   рисков".
11. Выберите **"Далее",** чтобы продолжить.
12. На странице **"Проверка"** просмотрите параметры, выбранные для политики. Выберите  "Изменить", чтобы изменить любое из значений политики, или выберите **"Отправить",** чтобы создать и активировать политику.

Настройте дополнительные политики по мере необходимости с помощью инструкций, которые были указаны в шаге 4 этой статьи, или действий, которые необходимо предпринять в статье "Создание новой внутренней политики [риска".](insider-risk-management-policies.md#create-a-new-policy)
