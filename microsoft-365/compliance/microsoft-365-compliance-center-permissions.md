# <a name="permissions-in-the-microsoft-365-compliance-center"></a>Разрешения в центре Microsoft 365 соответствия требованиям

Центр Microsoft 365 недавно был обновлен и теперь поддерживает непосредственное управление разрешениями для пользователей, которые выполняют задачи соответствия требованиям в Microsoft 365. Это обновление означает, что вам больше не придется использовать центр Office 365 безопасности & для управления разрешениями для решений по обеспечению соответствия требованиям. Используя новую  страницу Разрешений в центре Microsoft 365 соответствия требованиям, вы можете управлять разрешениями для пользователей для выполнения задач в таких функциях, как управление устройствами, предотвращение потери данных, eDiscovery, управление рисками, хранение и многие другие. Пользователи могут выполнять только задачи соответствия требованиям, к которые вы явно предоставляете им доступ.

Чтобы просмотреть вкладку **Permissions** в центре Microsoft 365 соответствия требованиям, пользователям необходимо быть  глобальным администратором или им должна быть  назначена роль управления ролью (роль назначена только группе ролей управления организацией). Роль *Управления ролью* позволяет пользователям просматривать, создавать и изменять группы ролей.

![Страница Разрешений в центре Microsoft 365 соответствия требованиям](../media/m365-compliance-center-permissions.png)

Разрешения в центре Microsoft 365 соответствия основаны на модели разрешений управления доступом на основе ролей (RBAC). RBAC — это та же модель разрешений, которая используется большинством служб Microsoft 365, поэтому если вы знакомы со структурой разрешений в этих службах, предоставление разрешений в центре Microsoft 365 соответствия требованиям будет знакомым. Важно помнить, что разрешения, управляемые в центре Microsoft 365 соответствия требованиям, не охватывают управление всеми разрешениями, необходимыми для каждой отдельной службы. Вам по-прежнему необходимо управлять определенными разрешениями, определенными для службы, в центре администрирования для конкретной службы. Например, если необходимо назначить разрешения для политик архива, аудита и хранения, необходимо управлять этими разрешениями в центре администрирования Exchange.

## <a name="relationship-of-members-roles-and-role-groups"></a>Связи между членами, ролями и группами ролей

Роль предоставляет разрешения на выполнение набора задач; например, роль "Управление делами" позволяет пользователям работать с делами по обнаружению электронных данных.

Группа ролей — это набор ролей, которые позволяют пользователям делать свою работу с помощью решений соответствия требованиям, Microsoft 365 центра соответствия требованиям. Например, добавляя пользователей в группу ролей *insider Risk Management,* назначенные администраторы, аналитики, следователи и аудиторы настраиваются для получения необходимых разрешений на управление рисками в одной группе. Центр Microsoft 365 соответствия требованиям включает группы ролей по умолчанию для задач и функций для каждого решения соответствия требованиям, на которые необходимо назначить людей. Как правило, мы рекомендуем просто добавлять отдельных пользователей в группы ролей соответствия требованиям по умолчанию по мере необходимости.

## <a name="permissions-needed-to-use-features-in-the-microsoft-365-compliance-center"></a>Разрешения, необходимые для использования функций в центре Microsoft 365 соответствия требованиям

Чтобы просмотреть все группы ролей по умолчанию, доступные в центре Microsoft 365 соответствия требованиям, а также роли, которые назначены группам ролей по умолчанию, см. в документе [Permissions in the Security & Compliance Center](/microsoft-365/security/office-365-security/permissions-in-the-security-and-compliance-center).

Управление разрешениями в центре Microsoft 365 соответствия требованиям предоставляет пользователям доступ только к функциям соответствия требованиям, доступными в центре Microsoft 365 соответствия требованиям. Если вы хотите предоставить разрешения другим функциям, которые не используются в центре Microsoft 365 соответствия требованиям, например Exchange правил потока почты (также известных как правила транспорта), необходимо использовать центр администрирования Exchange.

## <a name="azure-roles-in-the-microsoft-365-compliance-center"></a>Роли Azure в центре Microsoft 365 соответствия требованиям

Роли, которые отображаются в разделе **Роли Azure AD** на странице Microsoft 365 центра разрешений,  >   Azure Active Directory ролей. Чтобы управлять участниками роли Azure AD, выберите Управление участниками в Azure AD. Этот выбор перенаправляет вас на портал управления Azure.

## <a name="add-users-to-a-compliance-role-group"></a>Добавление пользователей в группу ролей соответствия требованиям

Выполните следующие действия, чтобы добавить пользователей в группу ролей соответствия требованиям:

1. Вопишитесь в область разрешений центра [Microsoft 365,](https://compliance.microsoft.com/permissions) используя учетные данные для учетной записи администратора в Microsoft 365 организации.
2. В центре Microsoft 365 соответствия требованиям перейдите к **разрешениям.** Выберите ссылку для просмотра ролей соответствия требованиям и управления ими в Microsoft 365.
3. Расширь **раздел Центр соответствия** требованиям и выберите **Роли.**
4. На странице **Роли центра соответствия** требованиям выберите группу ролей соответствия требованиям, в которую необходимо добавить пользователей, а затем выберите группу **изменить** роль на области сведений.
5. Выберите **Выберите членов из** левой области навигации, а затем выберите **Изменить**.
6. Выберите **Добавить,** а затем выберите почтовый ящик для всех пользователей, которые необходимо добавить в группу ролей.
7. Нажмите кнопку **Добавить**, затем нажмите кнопку **Готово**.
8. Выберите **Сохранить,** чтобы добавить пользователей в группу ролей. Выберите **Close** для выполнения действий.

## <a name="remove-users-from-a-compliance-role-group"></a>Удаление пользователей из группы ролей соответствия требованиям

Выполните следующие действия по удалению пользователей из группы ролей соответствия требованиям:

1. Вопишитесь в область разрешений центра [Microsoft 365,](https://compliance.microsoft.com/permissions) используя учетные данные для учетной записи администратора в Microsoft 365 организации.
2. В центре Microsoft 365 соответствия требованиям перейдите к **разрешениям.** Выберите ссылку для просмотра ролей соответствия требованиям и управления ими в Microsoft 365.
3. Расширь раздел Центр соответствия требованиям и выберите **Роли.**
4. На странице **Роли центра соответствия** требованиям выберите группу ролей соответствия требованиям, из которую необходимо удалить пользователей, а затем выберите группу **изменить** роль на области сведений.
5. Выберите **Выберите членов из** левой области навигации, а затем выберите **Изменить**.
6. Выберите **Удаление** и выберите почтовый ящик для всех пользователей, которые необходимо удалить из группы ролей.
7. Выберите **Удалить,** а затем выберите **Готово**.
8. Выберите **Сохранить,** чтобы удалить пользователей из группы ролей. Выберите **Close** для выполнения действий.
