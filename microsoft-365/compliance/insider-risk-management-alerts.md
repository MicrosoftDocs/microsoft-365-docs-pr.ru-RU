# <a name="insider-risk-management-alerts"></a>Оповещения по управлению рисками внутри организации

Оповещения для управления рисками в организации автоматически создаются индикаторами риска, определенными в политиках управления рисками внутри организации. Эти оповещения дают аналитикам соответствия требованиям и следователям всесторонние представления о текущем состоянии риска и позволяют организации рассматривать обнаруженные риски и принимать меры. По умолчанию политики создают определенное количество оповещений низкого, среднего [](insider-risk-management-settings.md#alert-volume) и высокого уровня серьезности, но вы можете увеличить или уменьшить объем оповещений в соответствии с потребностями. Кроме того, можно настроить [](insider-risk-management-settings.md#indicator-level-settings-preview) порог оповещения для индикаторов политики при создании новой политики с помощью мастера политики.

## <a name="alert-dashboard"></a>Панель мониторинга оповещений

Информационная панель  оповещений о рисках внутри системы позволяет просматривать оповещения, созданные политиками внутри риска, и действовать над них. В каждом мини-мини-отчете отображаются сведения за последние 30 дней.

- **Оповещения для проверки:** общее число оповещений, для которых требуется проверка и рассмотрение, включая разбивку по степени серьезности оповещения.
- Открытие оповещений за **последние 30** дней: общее количество оповещений, созданных политиками за последние 30 дней, отсортировали по высокой, средней и низкой степени серьезности оповещений.
- **Среднее время для разрешения оповещений:** сводка полезной статистики оповещений:
    - Среднее время для устранения оповещений высокого уровня серьезности, указанных в часах, днях или месяцах.
    - Среднее время для устранения оповещений средней серьезности, указанных в часах, днях или месяцах.
    - Среднее время для устранения оповещений с низкой степенью серьезности, указанных в часах, днях или месяцах.

![Панель мониторинга оповещений по управлению рисками в организации](../media/insider-risk-alerts-dashboard.png)

>[!NOTE]
>Для управления рисками внутри организации используется встроенное регулирование оповещений для защиты и оптимизации анализа и анализа рисков. Это регулирование предотвещает проблемы, которые могут привести к перегрузке оповещений политики, таких как неправильно заданные соединители данных или политики DLP. В результате может возникнуть задержка отображения новых оповещений для пользователя.

## <a name="alert-status-and-severity"></a>Состояние и серьезность оповещений

Оповещения можно учитывать в одном из следующих статусов:

- **Confirmed**: оповещение подтверждено и назначено новому или существующему делу.
- **Dismissed**: оповещение, отклонено как безвластие в процессе обработки.
- **Требуется обзор:** новое оповещение, в котором еще не были предприняты действия по рассмотрению.
- **Решение:** оповещение, которое является частью закрытого и разрешенного дела.

Оценки риска оповещений автоматически вычисляются на ряду индикаторов активности риска. К этим индикаторам относятся тип действия с риском, количество и частота его возникновения, история активности пользователей и добавление рисков активности, которые могут повысить серьезность активности. Оценка риска оповещений позволяет программным программам определить степень серьезности для каждого оповещения и не может быть настроена. Если оповещения остаются невыпущенными, а действия по рискам продолжают накапливаться в оповещении, степень серьезности риска может увеличиться. Аналитики рисков и следователи могут использовать степень серьезности риска оповещений для проверки оповещений в соответствии с политиками и стандартами риска в организации.

Уровень серьезности риска оповещений:

- **Высокая серьезность:** действия и индикаторы оповещения представляют значительный риск. Связанные действия с рисками являются серьезными, повторяющимися и сильно связаны с другими значительными факторами риска.
- **Средняя степень серьезности:** действия и индикаторы оповещения представляют средний риск. Связанные действия с рисками являются умеренной, частой и в некоторой степени связаны с другими факторами риска.
- **Низкий уровень серьезности:** действия и индикаторы оповещения представляют незначительный риск. Связанные действия с рисками являются незначительными, более редкими и не являются основными для других важных факторов риска.

## <a name="filter-alerts-on-the-alert-dashboard"></a>Фильтрация оповещений на панели мониторинга оповещений

В зависимости от количества и типа активных политик управления рисками внутри организации просмотр большой очереди оповещений может быть сложной задачей. Использование фильтров оповещений может помочь аналитикам и следователям сортировать оповещения по нескольким атрибутам. Чтобы отфильтровать оповещения на панели **оповещений,** выберите этот **фильтр.** Оповещения можно фильтровать по одному или несколько атрибутов:

- **Состояние:** выберите одно или несколько значений состояния для фильтрации списка оповещений. Возможные варианты: *"Подтверждено",* *"Отклонено",* *"Требуется проверка"* и *"Разрешено".*
- **Серьезность:** выберите один или несколько уровней серьезности риска оповещения, чтобы отфильтровать список оповещений. Возможные варианты: *High,* *Medium* и *Low.*
- **Обнаружено время:** выберите даты начала и окончания создания оповещения.
- **Политика:** выберите одну или несколько политик для фильтрации оповещений, созданных выбранными политиками.

## <a name="search-alerts-on-the-alert-dashboard"></a>Оповещения о поиске на информационной панели оповещений

Чтобы найти определенное слово в имени  оповещения, выберите его и введите слово для поиска. В результатах поиска отображается любое оповещение политики, содержащее слово, определенное в поиске.

## <a name="triage-alerts"></a>Предупреждения для триады

Чтобы выполнить оценку оповещения о риске для внутренней оценки, выполните следующие действия.

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для программы **insider и** выберите вкладку **"Оповещения".**
2. На **информационной панели оповещений** выберите оповещение, о чем вы хотите узнать.
3. В области **оповещений** можно просмотреть следующие вкладки и рассмотреть оповещение:
    - **Сводка.** Эта вкладка содержит общие сведения об оповещении и позволяет подтвердить оповещение и создать новое дело или отклонять оповещение. Он включает текущее состояние оповещения и уровень серьезности риска, указанный как *высокий,* *средний* или *низкий*. Степень серьезности со временем может увеличиваться или уменьшаться, если оповещение не является обойденным.
        - **What happened**: Displays the top three risk activities and policy matches during the activity evaluation period, including the type of violation associated with the activity.
        - **Сведения о** пользователе: отображаются общие сведения о пользователе, назначенного оповещением. Если анонимизация включена, имя пользователя, адрес электронной почты, псевдоним и поля организации анонимизируются.
        - **Сведения** оповещений: включает продолжительность времени с момента с момента сгенерии оповещения, политики, которые создали оповещение, и в списке перечислены дела, созданные из оповещения. Для новых оповещений в **поле Case** отображается "Нет".
        - **Обнаруженное содержимое:** включает содержимое, связанное с действиями риска для оповещения, и суммирует события активности по ключевым областям. При выборе ссылки действия открывается обозреватель действий и отображаются дополнительные сведения о действии.
    - **Активность пользователя:** на этой вкладке отображается история действий пользователя, связанного с оповещением. Эта история включает другие оповещения и действия, связанные с индикаторами риска, определенными в шаблоне, назначенного политике для этого оповещения. Эта история позволяет аналитикам рисков и следователям учитывать любые прошлые рискованные действия для сотрудника в процессе оценки.
    - **Действия:** для каждого оповещения доступны следующие действия:
        - **Открыть расширенное представление:** открывает панель **мониторинга обозревателя действий.**
        - **Подтверждение и создание дела:** это действие используется для подтверждения и создания нового случая для всех оповещений, связанных с пользователем. Это действие автоматически изменяет состояние оповещения на *Confirmed*.
        - **Оповещение об** отклонять: используйте это действие, чтобы отклонять оповещение. Это действие изменяет состояние оповещения на *"Разрешено".*

## <a name="activity-explorer-preview"></a>Обозреватель действий (предварительная версия)

>[!NOTE]
>Обозреватель действий доступен в области управления оповещениями для пользователей с запусками событий после того, как эта функция доступна в вашей организации.

Обозреватель действий предоставляет анализатору рисков и аналитикам комплексное аналитическое средство, которое предоставляет подробные сведения об оповещениях. С помощью обозревателя действий рецензенты могут быстро просмотреть временную шкалу обнаруженных рискованных действий, а также определить и фильтровать все действия с рисками, связанные с оповещениями. Для фильтрации оповещений в обозревателе действий выберите фильтр. Оповещения можно фильтровать по одному или более атрибутам, указанным в области сведений для оповещения. Обозреватель действий также поддерживает настраиваемые столбцы, чтобы помочь следователям и аналитикам сосредоточиться на самой важной для них информации.

![Обзор обозревателя действий по управлению рисками внутри организации](../media/insider-risk-management-activity-explorer.png)

Чтобы использовать **обозреватель действий,** выполните следующие действия:

1. В Центре соответствия требованиям Microsoft 365 перейдите в центр управления рисками для программы оценки безопасности и выберите вкладку **"Оповещения".** 
2. На **информационной панели оповещений** выберите оповещение, о чем вы хотите узнать.
3. В области **оповещений выберите**"Открыть **расширенное представление".**
4. На странице выбранного оповещения выберите вкладку обозревателя **действий.**

При просмотре действий в обозревателе действий, следователи и аналитики могут выбрать определенное действие и открыть области сведений об активности. В области отображаются подробные сведения о действиях, которые могут использовать следователи и аналитики в процессе проверки оповещений. Подробные сведения могут предоставить контекст для оповещения и помочь определить всю область действия по риску, которое вызвало оповещение.

![Сведения об обозревателе действий по управлению рисками в организации](../media/insider-risk-management-activity-explorer-details.png)

## <a name="create-a-case-for-an-alert"></a>Создание дела для оповещения

По мере проверки и рассмотрения оповещений можно создать новое дело для дальнейшего изучения действий с риском. Чтобы создать дело для оповещения, выполните следующие действия:

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для программы **insider и** выберите вкладку **"Оповещения".**
2. На **информационной панели** оповещений выберите оповещение, для чего нужно подтвердить, и создайте новое дело.
3. В области **сведений оповещений** выберите **"Действия** подтвердит&  >  **создать дело.**
4. В **диалоговом** окте "Подтверждение оповещения" и создайте дело для участников, введите имя дела, выберите пользователей, добавляемых в качестве участников, и добавьте комментарии по мере возможности. Комментарии автоматически добавляются в дело в качестве заметки.
5. Выберите **"Создать дело",** чтобы  создать дело, или "Отмена", чтобы закрыть диалоговое окно без создания дела.

После создания дела следователи и аналитики могут управлять этим делом и действовать над этим делом. Дополнительные [сведения см.](insider-risk-management-cases.md) в статье по управлению рисками для insider.
