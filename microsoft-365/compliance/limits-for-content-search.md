# <a name="limits-for-content-search-in-the-security--compliance-center"></a>Ограничения для поиска контента в Центре безопасности & соответствия требованиям

> [!NOTE]
> Ограничения в этом разделе отличаются от текущих ограничений для In-Place eDiscovery в Exchange Online и для центра eDiscovery в SharePoint Online.
  
К функции поиска контента в Центре безопасности & соответствия требованиям применяются различные ограничения. Это включает поиск, который выполняется **на** странице поиска контента, и поиски, связанные с делом eDiscovery. Эти ограничения помогают поддерживать состояние и качество услуг, предоставляемых организациям. Кроме того, существуют ограничения, связанные с индексащением сообщений электронной почты в Exchange Online для поиска. Вы не можете изменить ограничения поиска контента или индексации электронной почты, но следует знать о них, чтобы эти ограничения учитывались при планировании, запуске и устранении неполадок при поиске контента.
  
## <a name="content-search-limits"></a>Ограничения для поиска контента

В следующей таблице перечислены ограничения поиска в Центре безопасности & соответствия требованиям.
  
| Описание ограничения | Ограничение |
|:-----|:-----|
|Максимальное количество почтовых ящиков или сайтов, для поиска в одном поиске контента  <br/> |Без ограничения <sup>1</sup> <br/> |
|Максимальное количество одновременно запуска поисков контента в организации.  <br/> |30  <br/> |
|Максимальное количество одновременно запускаемого пользователем поиска контента. Это ограничение наиболее вероятно, если пользователь пытается запустить несколько поисков с помощью команды **Get-ComplianceSearch \| Start-ComplianceSearch в PowerShell** Центра & безопасности.  <br/> |10   <br/> |
|Максимальное количество элементов на почтовый ящик пользователя, которые отображаются на странице предварительного просмотра результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках пользователей, которые отображаются на странице предварительного просмотра результатов поиска контента. Отображаются самые новые элементы.  <br/> |1,000  <br/> |
|Максимальное количество почтовых ящиков пользователей, для которых возможен предварительный просмотр результатов поиска. Если поисковому запросу соответствует содержимое более 1000 почтовых ящиков, предварительный просмотр будет возможен только для первой 1000 почтовых ящиков с наибольшим количеством результатов поиска.  <br/> |1,000  <br/> |
|Максимальное количество элементов, найденных на сайтах SharePoint и OneDrive для бизнеса, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента. Отображаются самые новые элементы.  <br/> |200  <br/> |
|Максимальное количество сайтов (в SharePoint и OneDrive для бизнеса), которые можно просмотреть для результатов поиска. При наличии более 200 сайтов, содержащих контент, который соответствует поисковому запросу, предварительный просмотр будет доступен только для 200 сайтов с наиболее большим количеством результатов поиска.  <br/> |200  <br/> |
|Максимальное количество элементов на почтовый ящик общедоступных папок, отображающихся на странице предварительного просмотра результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках общедоступных папок, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента.  <br/> |200  <br/> |
|Максимальное количество общедоступных почтовых ящиков, которые можно просмотреть для результатов поиска. Если имеется более 500 почтовых ящиков общедоступных папок, содержащих содержимое, которое соответствует поисковому запросу, для предварительного просмотра будут доступны только 500 почтовых ящиков общедоступных папок с самыми результатами поиска.  <br/> |500  <br/> |
|Максимальное количество символов для поискового запроса (включая операторы и условия) для поиска контента.  <br/><br/> **Примечание.** Это ограничение вступает в силу после расширения запроса, что означает, что запрос будет расширен для каждого ключевого слова. Например, если поисковый запрос имеет 15 ключевых слов и дополнительные параметры и условия, запрос расширяется 15 раз, каждый из которых имеет другие параметры и условия в запросе. Поэтому даже если количество символов в поисковом запросе может быть меньше предельного, это расширенный запрос, который может привести к превышению этого ограничения.  <br/> |**Почтовые ящики:** 10 000  <br/> **Сайты:** 4000 при поиске по всем сайтам или 2000 при поиске до 20 <sup>сайтов 2</sup> <br/> |
|Максимальное количество вариантов, возвращаемого при использовании поддиапазона префикса для поиска точной фразы в поисковом запросе или при использовании поддиапазона префикса и оператора **boolean NEAR.**  <br/> |10 000 <sup>3</sup> <br/> |
|Минимальное количество альфа-символов для поддиапазонов префиксов; например,  `time*` ,  `one*` или  `set*` .  <br/> |3   <br/> |
|Максимальное количество почтовых ящиков в поиске контента, в которые можно удалить элементы с помощью действия "Поиск и очистка" (с помощью команды **New-ComplianceSearchAction -Purge).** Если при поиске контента, для который вы делаете очистку, количество исходных почтовых ящиков превышает это ограничение, действие очистки не будет работать. Дополнительные сведения о поиске и очистке см. в поиске и удалении сообщений электронной почты [в организации.](search-for-and-delete-messages-in-your-organization.md)  <br/> |50 000  <br/> |
|Максимальное количество местоположений в поиске контента, из которые можно экспортировать элементы. Если в экспортируемом поиске контента больше местоположений, чем это ограничение, экспорт не удастся. Дополнительные сведения см. в [экспорте результатов поиска контента.](export-search-results.md)  <br/> |100 000  <br/> |

> [!NOTE]
> <sup>1</sup> Хотя вы можете искать неограниченное количество почтовых ящиков в одном поиске, вы можете скачать только экспортные результаты поиска из не более 100 000 почтовых ящиков с помощью средства экспорта для eDiscovery в Центре безопасности и соответствия требованиям Office 365 & или Центре соответствия требованиям Microsoft 365. Чтобы скачать результаты поиска из более чем 100 000 почтовых ящиков, необходимо использовать PowerShell Центра & соответствия требованиям. Дополнительные сведения и пример сценария см. в экспорте результатов из более [чем 100 000 почтовых ящиков.](export-search-results.md#exporting-results-from-more-than-100000-mailboxes) <br/><br/> <sup>2</sup> При поиске в расположениях SharePoint и OneDrive для бизнеса символы URL-адресов искомого сайта учитываются в этом ограничении. <br/><br/> <sup>3</sup> Для запросов без фраз (ключевое слово, которое не использует двойные кавычка) мы используем специальный индекс префикса. Это сообщает нам, что слово встречается в документе, но не там, где оно происходит в документе. Чтобы сделать запрос фразы (ключевое слово со двойными кавычками), необходимо сравнить позицию в документе для слов в фразе. Это означает, что мы не можем использовать индекс префикса для запросов фраз. В этом случае мы внутренне расширяем запрос со всеми возможными словами, до которые расширяется префикс; например,  `"time*"` можно развернуть до  `"time OR timer OR times OR timex OR timeboxed OR …"` . Максимальное число вариантов, которые можно добавить  10 000, а не число документов, соответствующих запросов. Не существует верхнего предела для нефвационных терминов. 
  
## <a name="indexing-limits-for-email-messages"></a>Ограничения индексации для сообщений электронной почты

В следующей таблице описываются ограничения индексации, которые могут привести к возвращению сообщения электронной почты в качестве неиндексации элемента или частично индексного элемента в результатах поиска контента.
  
| Ограничение индексации | Максимальное значение | Описание |
|:-----|:-----|:-----|
|Максимальный размер вложения|150 МБ  <br/> |Максимальный размер вложения электронной почты для индексации. Любое вложение, которое превышает это ограничение, не будет разобрано для индексации, а сообщение с вложением будет помечено как частично индексироваться.  <br/> <br/>**Примечание.** Разбиение — это процесс, в котором служба индексирования извлекает текст из вложения, удаляет ненужные символы, такие как знаки препинания и пробелы, а затем делит текст на слова (в процессе, называемом маркеризацией), которые затем сохраняются в индексе.           |
|Максимальное количество вложений  <br/> |250  <br/> |Максимальное количество файлов, вложенных в сообщение электронной почты, которое будет разобрано для индексации. Если сообщение имеет более 250 вложений, первые 250 вложений будут проиндексированы и помечены как частично проиндексированы, так как у него были дополнительные вложения, которые не были проиндексированы.  <br/> |
|Максимальная глубина вложения  <br/> |30  <br/> |Максимальное количество вложенных вложений, которые можно разогнать. Например, если в сообщение электронной почты вложено еще одно сообщение, а вложенное сообщение имеет вложенный документ Word, документ Word и вложенное сообщение будут индексироваться. Это поведение будет продолжено для до 30 вложенных вложений.  <br/> |
|Максимальное количество прикрепленных изображений  <br/> |0  <br/> |Изображение, вложенное в сообщение электронной почты, пропускается с помощью разберегатель и не индексироваться.  <br/> |
|Максимальное время, затраченное на разбиение элемента  <br/> |30 секунд  <br/> |На разбиение элемента на индексацию тратится не более 30 секунд. Если время разметки превышает 30 секунд, элемент помечается как частично индексный.  <br/> |
|Максимальное количество выходных данных разберегатель  <br/> |2 миллиона символов  <br/> |Максимальный объем выходных данных текста из индексного обсчета. Например, если разберите из документа 8 миллионов символов, индексироваться будут только первые 2 миллиона символов.  <br/> |
|Максимальное число маркеров аннотаций  <br/> |2 миллиона  <br/> |При индексации сообщения электронной почты каждое слово обозначается различными инструкциями по обработке, которые определяют способ индексации этого слова. Каждый набор инструкций обработки называется маркером аннотации. Для поддержания качества обслуживания в Office 365 существует ограничение в 2 миллиона маркеров заносок для сообщения электронной почты.  <br/> |
|Максимальный размер тела в индексе  <br/> |67 миллионов символов  <br/> |Общее количество символов в тексте сообщения электронной почты и всех его вложений. При индексации сообщения электронной почты весь текст в тексте сообщения и во всех вложениях совмещен в одну строку. Максимальный размер индекса этой строки составляет 67 миллионов символов.  <br/> |
|Максимальное количество уникальных маркеров в теле  <br/> |1 миллион  <br/> |Как объяснялось ранее, маркеры являются результатом извлечения текста из содержимого, удаления знаков препинания и пробелов, а затем его деления на слова (называемые маркерами), хранимые в индексе. Например, фраза  `"cat, mouse, bird, dog, dog"` содержит 5 маркеров. Но только 4 из них являются уникальными маркерами. Для каждого сообщения электронной почты существует ограничение в 1 миллион уникальных маркеров, что позволяет предотвратить слишком большой размер индекса с помощью случайных маркеров.  <br/> |
