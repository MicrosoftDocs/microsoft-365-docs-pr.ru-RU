# <a name="limits-for-content-search"></a>Ограничения для поиска контента 
К средству поиска контента в Центре соответствия требованиям Microsoft 365 применяются различные ограничения. Это включает поиск,  который выполняется на странице поиска контента, и поиски, связанные с делом eDiscovery на основной странице **eDiscovery.** Эти ограничения помогают поддерживать состояние и качество услуг, предоставляемых организациям. Кроме того, существуют ограничения, связанные с индексащением сообщений электронной почты в Exchange Online для поиска. Вы не можете изменить ограничения для поиска контента или индексации электронной почты, но следует знать о них, чтобы эти ограничения учитывались при планировании, запуске и устранении неполадок поиска контента.
  
## <a name="search-limits"></a>Ограничения поиска

В следующей таблице перечислены ограничения поиска при использовании средства поиска контента в Центре соответствия требованиям Microsoft 365 и при поиске, связанном с основным делом eDiscovery.

|:-----|:-----|
|Максимальное количество почтовых ящиков или сайтов, в которые можно искать в одном поиске  <br/> |Без ограничения <sup>1</sup> <br/> |
|Максимальное количество поисковых запросов, которые могут одновременно запускаться в организации.  <br/> |30  <br/> |
|Максимальное число поисков, которые может одновременно запустить один пользователь. Это ограничение наиболее вероятно, если пользователь пытается запустить несколько поисков с помощью команды **Get-ComplianceSearch \| Start-ComplianceSearch в PowerShell** Центра & безопасности.  <br/> |10   <br/> |
|Максимальное количество элементов на почтовый ящик пользователя, которые отображаются на странице предварительного просмотра результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках пользователей, которые отображаются на странице предварительного просмотра при просмотре результатов поиска. Отображаются самые новые элементы.  <br/> |1,000  <br/> |

Если поисковому запросу соответствует содержимое более 1000 почтовых ящиков, предварительный просмотр будет возможен только для первой 1000 почтовых ящиков с наибольшим количеством результатов поиска.  <br/> |1,000  <br/> |
|Максимальное количество элементов, найденных на сайтах SharePoint и OneDrive для бизнеса, которые отображаются на странице предварительного просмотра при просмотре результатов поиска. Отображаются самые новые элементы.  <br/> |200  <br/> |
|Максимальное количество сайтов (в SharePoint и OneDrive для бизнеса), которые можно просмотреть для результатов поиска. При наличии более 200 сайтов, содержащих контент, который соответствует поисковому запросу, предварительный просмотр будет доступен только для 200 самых популярных сайтов с наиболее большим количеством результатов поиска.  <br/> |200  <br/> |
|Максимальное количество элементов на почтовый ящик общедоступных папок, отображающихся на странице предварительного просмотра результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках общедоступных папок, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента.  <br/> |200  <br/> |
|Максимальное количество общедоступных почтовых ящиков, которые можно просмотреть для результатов поиска. Если имеется более 500 почтовых ящиков общедоступных папок, содержащих содержимое, которое соответствует поисковому запросу, для предварительного просмотра будут доступны только 500 почтовых ящиков общедоступных папок с самыми результатами поиска.  <br/> |500  <br/> |
|Максимальное количество символов для поискового запроса (включая операторы и условия) для поиска.  <br/><br/> **Примечание.** Это ограничение вступает в силу после расширения запроса, что означает, что запрос будет расширен для каждого ключевого слова. Например, если поисковый запрос имеет 15 ключевых слов и дополнительные параметры и условия, запрос расширяется 15 раз, каждый из которых имеет другие параметры и условия в запросе. Поэтому даже если количество символов в поисковом запросе может быть меньше предельного, это расширенный запрос, который может привести к превышению этого ограничения.  <br/> |**Почтовые ящики:** 10 000  <br/> **Сайты:** 4000 при поиске по всем сайтам или 2000 при поиске до 20 <sup>сайтов 2</sup> <br/> |

|Минимальное количество альфа-символов для поддиапазонов префиксов; например,  `time*` ,  `one*` или  `set*` .  <br/> |3   <br/> |
|Максимальное количество почтовых ящиков в поиске, в которые можно удалить элементы с помощью действия "Поиск и очистка" (с помощью команды **New-ComplianceSearchAction -Purge).** Если при поиске, для который вы делаете очистку, больше исходных почтовых ящиков, чем это ограничение, действие очистки не будет работать. Дополнительные сведения о поиске и очистке см. в поиске и удалении сообщений электронной почты [в организации.](search-for-and-delete-messages-in-your-organization.md)  <br/> |50 000  <br/> |
|Максимальное количество местоположений в поиске, из которые можно экспортировать элементы. Если экспортируемая поисковая система имеет большее количество местоположений, чем это ограничение, экспорт не удастся. Дополнительные сведения см. в [экспорте результатов поиска контента.](export-search-results.md)  <br/> |100 000  <br/> |

> [!NOTE]
> <sup>1</sup> Хотя вы можете искать неограниченное количество почтовых ящиков в одном поиске, вы можете скачать только экспортные результаты поиска из не более 100 000 почтовых ящиков с помощью средства экспорта для eDiscovery в Центре соответствия требованиям Microsoft 365. Чтобы скачать результаты поиска из более чем 100 000 почтовых ящиков, необходимо использовать PowerShell Центра & соответствия требованиям. Дополнительные сведения и пример сценария см. в экспорте результатов из более [чем 100 000 почтовых ящиков.](export-search-results.md#exporting-results-from-more-than-100000-mailboxes) <br/><br/> <sup>2</sup> При поиске в расположениях SharePoint и OneDrive для бизнеса символы URL-адресов искомого сайта учитываются в этом ограничении. <br/><br/> <sup>3</sup> Для запросов без фраз (ключевое слово, которое не использует двойные кавычка) мы используем специальный индекс префикса. Это сообщает нам, что слово встречается в документе, но не там, где оно встречается в документе. Чтобы сделать запрос фразы (ключевое слово со двойными кавычками), необходимо сравнить позицию в документе для слов в фразе. Это означает, что мы не можем использовать индекс префикса для запросов фраз. В этом случае мы внутренне расширяем запрос со всеми возможными словами, до которые расширяется префикс; например,  `"time*"` можно развернуть до  `"time OR timer OR times OR timex OR timeboxed OR …"` . Максимальное число вариантов, которые можно добавить  10 000, а не число документов, соответствующих запросов. Не существует верхнего предела для нефвационных терминов.

|Максимум, который организация может экспортировать за один день <br/><br/> **Примечание.** Это ограничение сбрасывается ежедневно в 12:00 (UTC) <br/> |2 ТБ <br/> |
|Максимальное число одновлаверных экспортов, которые можно одновременно использовать в организации <br/><br/> **Примечание.** Запуск количества **экспортов только** отчетов по отношению к общему параллельному экспорту для организации. Если каждый из трех пользователей выполняет 3 экспорта, можно выполнить только один экспорт. Независимо от того, экспортируется ли отчет или результаты поиска, другие экспорты не могут выполняться до тех пор, пока не будет выполнен один из них.   <br/> |10  <br/> |

Инструкции см. в [экспорте результатов из более чем 100 000 почтовых ящиков.](export-search-results.md#exporting-results-from-more-than-100000-mailboxes) <br/> | 100 000 <br/>|

Если экспортировать все результаты поиска в одном PST-файле, PST-файл будет разбит на дополнительные PST-файлы, если общий размер результатов поиска превышает 10 ГБ. Если вы хотите изменить этот размер по умолчанию, вы можете изменить реестр Windows на компьютере, который используется для экспорта результатов поиска. См. [изменение размера PST-файлов при экспорте результатов поиска при обнаружении электронных данных.](change-the-size-of-pst-files-when-exporting-results.md) Результаты поиска из определенного почтового ящика не будут разделены между несколькими PST-файлами, если содержимое одного почтового ящика не превышает 10 ГБ. Если вы решили экспортировать результаты поиска в один PST-файл, содержащий все сообщения в одной папке, а размер результатов поиска превышает 10 ГБ, элементы будут по-прежнему организованы в хронологическом порядке, поэтому они будут разбиты на дополнительные PST-файлы в зависимости от даты, отправленной.<br/> | 10 ГБ <br/> |

## <a name="indexing-limits-for-email-messages"></a>Ограничения индексации для сообщений электронной почты

В следующей таблице описываются ограничения индексации, которые могут привести к возвращению сообщения электронной почты в качестве неиндексации элемента или частично индексного элемента в результатах поиска контента.
  
| Ограничение индексации | Максимальное значение | Описание |
|:-----|:-----|:-----|
|Максимальный размер вложения|150 МБ  <br/> |Максимальный размер вложения электронной почты для индексации. Любое вложение, которое превышает это ограничение, не будет разобрано для индексации, а сообщение с вложением будет помечено как частично индексироваться.  <br/> <br/>**Примечание.** Разбиение — это процесс, в котором служба индексирования извлекает текст из вложения, удаляет ненужные символы, такие как знаки препинания и пробелы, а затем делит текст на слова (в процессе, называемом маркеризацией), которые затем сохраняются в индексе.           |
|Максимальное количество вложений  <br/> |250  <br/> |Максимальное количество файлов, вложенных в сообщение электронной почты, которое будет разобрано для индексации. Если сообщение имеет более 250 вложений, первые 250 вложений будут проиндексированы и помечены как частично проиндексированы, так как у него были дополнительные вложения, которые не были проиндексированы.  <br/> |
|Максимальная глубина вложения  <br/> |30  <br/> |Максимальное количество вложенных вложений, которые будут размеяно. Например, если в сообщение электронной почты вложено еще одно сообщение, а вложенное сообщение имеет вложенный документ Word, документ Word и вложенное сообщение будут проиндексироваться. Это поведение будет продолжено для до 30 вложенных вложений.  <br/> |
|Максимальное количество прикрепленных изображений  <br/> |0  <br/> |Изображение, вложенное в сообщение электронной почты, пропускается с помощью разберегатель и не индексироваться.  <br/> |
|Максимальное время, затраченное на разбиение элемента  <br/> |30 секунд  <br/> |На разбиение элемента на индексацию тратится не более 30 секунд. Если время разметки превышает 30 секунд, элемент помечается как частично индексный.  <br/> |
|Максимальное количество выходных данных разберегатель  <br/> |2 миллиона символов  <br/> |Максимальный объем выходных данных текста из индексного обсчета. Например, если разберите 8 миллионов символов из документа, индексироваться будут только первые 2 миллиона символов.  <br/> |
|Максимальное число маркеров аннотаций  <br/> |2 миллиона  <br/> |При индексации сообщения электронной почты каждое слово обозначается различными инструкциями по обработке, которые определяют способ индексации этого слова. Каждый набор инструкций обработки называется маркером аннотации. Для поддержания качества обслуживания в Office 365 существует ограничение в 2 миллиона маркеров заносок для сообщения электронной почты.  <br/> |
|Максимальный размер тела в индексе  <br/> |67 миллионов символов  <br/> |Общее количество символов в тексте сообщения электронной почты и всех его вложений. При индексации сообщения электронной почты весь текст в тексте сообщения и во всех вложениях совмещен в одну строку. Максимальный размер индекса этой строки составляет 67 миллионов символов.  <br/> |
|Максимальное количество уникальных маркеров в теле  <br/> |1 миллион  <br/> |Как объяснялось ранее, маркеры являются результатом извлечения текста из содержимого, удаления знаков препинания и пробелов, а затем его деления на слова (называемые маркерами), хранимые в индексе. Например, фраза  `"cat, mouse, bird, dog, dog"` содержит 5 маркеров. Но только 4 из них являются уникальными маркерами. Для каждого сообщения электронной почты существует ограничение в 1 миллион уникальных маркеров, что позволяет предотвратить слишком большой размер индекса с помощью случайных маркеров.  <br/> |
|||
