# <a name="limits-for-content-search-in-the-security--compliance-center"></a>Ограничения для поиска контента в центре безопасности & соответствия требованиям

> [!NOTE]
> В этой статье предельные значения для In-Place обнаружения электронных данных в Exchange Online и для центра обнаружения электронных данных в SharePoint Online отличаются от текущих пределов.
  
Различные ограничения применяются к функции поиска контента в центре безопасности & соответствия требованиям. Это включает поиск на странице " **Поиск контента** " и выполняет поиск, связанный с вариантом обнаружения электронных данных. Эти пределы помогают поддерживать работоспособность и качество служб, предоставляемых организациям. Кроме того, существуют ограничения, связанные с индексацией сообщений электронной почты в Exchange Online для поиска. Вы не можете изменять ограничения на поиск контента и индексацию электронной почты, но вам следует знать о них, чтобы при планировании, запуске и устранении неполадок при поиске контента задействовать эти ограничения.
  
## <a name="content-search-limits"></a>Ограничения на поиск контента

В следующей таблице перечислены пределы поиска в центре безопасности & соответствия требованиям.
  
| Описание ограничения | Ограничение |
|:-----|:-----|
|Максимальное количество почтовых ящиков или сайтов, для которых можно выполнить поиск в едином поисковом контенте  <br/> |Без ограничения <sup>1</sup> <br/> |
|Максимальное количество поисков контента, которое может выполняться одновременно в Организации.  <br/> |более  <br/> |
|Максимальное количество поисков контента, которое может одновременно запустить один пользователь. Это количество, скорее всего, загружается, если пользователь пытается запустить несколько поисков с помощью команды **Get-ComplianceSearch \| Start-ComplianceSearch** в средстве безопасности & центре соответствия требованиям PowerShell.  <br/> |10   <br/> |
|Максимальное количество элементов в почтовом ящике пользователя, отображаемых на странице предварительного просмотра при просмотре результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках пользователей, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента. Отображаются самые новые элементы.  <br/> |1,000  <br/> |
|Максимальное количество почтовых ящиков пользователей, для которых возможен предварительный просмотр результатов поиска. Если поисковому запросу соответствует содержимое более 1000 почтовых ящиков, предварительный просмотр будет возможен только для первой 1000 почтовых ящиков с наибольшим количеством результатов поиска.  <br/> |1,000  <br/> |
|Максимальное количество элементов, найденных на сайтах SharePoint и OneDrive для бизнеса, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента. Отображаются самые новые элементы.  <br/> |200  <br/> |
|Максимальное число сайтов (в SharePoint и OneDrive для бизнеса), которые можно просмотреть для результатов поиска. Если количество сайтов, содержащих контент, которое соответствует поисковому запросу, превышает 200, для предварительного просмотра будет доступен только сайт с самыми верхними 200ми с большинством результатов поиска.  <br/> |200  <br/> |
|Максимальное количество элементов в почтовом ящике общедоступных папок, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента.  <br/> |100  <br/> |
|Максимальное количество элементов, найденных во всех почтовых ящиках общедоступных папок, которые отображаются на странице предварительного просмотра при просмотре результатов поиска контента.  <br/> |200  <br/> |
|Максимальное количество общедоступных почтовых ящиков, которые можно просмотреть для результатов поиска. Если количество почтовых ящиков общедоступных папок, которые соответствуют поисковому запросу, превышает 500, для предварительного просмотра будут доступны только первые 500 почтовых ящиков общедоступных папок с большинством результатов поиска.  <br/> |500  <br/> |
|Максимальное число символов для поискового запроса (включая операторы и условия) для поиска контента.  <br/><br/> **Примечание:** Это предельное значение вступает в силу после развертывания запроса, что означает, что запрос будет развернут по отношению к каждому из ключевых слов. Например, если поисковый запрос содержит 15 ключевых слов и дополнительные параметры и условия, то запрос будет расширен 15 раз, каждый из которых содержит другие параметры и условия в запросе. Таким образом, несмотря на то, что количество символов в поисковом запросе может быть ниже ограничения, это расширенный запрос, который может привести к превышению этого ограничения.  <br/> |**Почтовые ящики:** 10 000  <br/> **Sites:** 4 000 при поиске на всех сайтах или 2 000 при поиске до 20 сайтов <sup>2</sup> <br/> |
|Максимальное число вариантов, возвращаемых при использовании подстановочного знака префикса для поиска точной фразы в поисковом запросе или при использовании подстановочного знака префикса и оператора **NEAR** Boolean.  <br/> |10 000 <sup>3</sup> <br/> |
|Минимальное количество альфа-знаков для подстановочных знаков префикса; Например,,  `time*`  `one*` или  `set*` .  <br/> |4  <br/> |
|Максимальное количество почтовых ящиков в поиске контента, в которых можно удалить элементы, выполнив действие "Поиск и очистка" (с помощью команды **New-ComplianceSearchAction-unуничтожать** ). Если при поиске контента, для которого выполняется действие по очистке, больше исходных почтовых ящиков, то операция очистки завершится с ошибками. Дополнительные сведения о поиске и очистке можно найти [в статье Поиск и удаление сообщений электронной почты в Организации](search-for-and-delete-messages-in-your-organization.md).  <br/> |50 000  <br/> |
|Максимальное количество почтовых ящиков в поиске контента, из которых можно экспортировать элементы. Если экспортируемый контент содержит больше исходных почтовых ящиков, чем это ограничение, экспорт завершается с ошибками. Дополнительные сведения можно найти в статье [Экспорт результатов поиска контента](export-search-results.md).  <br/> |100 000  <br/> |

> [!NOTE]
> <sup>1</sup> несмотря на то, что вы можете выполнять поиск неограниченного числа почтовых ящиков в одной системе поиска, вы можете скачать результаты из максимального числа почтовых ящиков 100 000 с помощью средства экспорта электронных данных в центре безопасности & соответствия требованиям Office 365 или центра соответствия требованиям Microsoft 365. Чтобы скачать результаты поиска из более чем 100 000 почтовых ящиков, необходимо использовать оболочку безопасности & центре соответствия требованиям. Более подробную информацию и пример сценария можно узнать [в статье экспорт результатов из более чем 100 000 почтовых ящиков](export-search-results.md#exporting-results-from-more-than-100000-mailboxes). <br/><br/> <sup>2</sup> при поиске расположений SharePoint и OneDrive для бизнеса символы в URL-адресах просматриваемых сайтов подсчитываются с учетом этого ограничения. <br/><br/> <sup>3</sup> для запросов без фразы (значение ключевого слова, не использующее двойные кавычки), используется особый Индекс префикса. Это говорит о том, что слово встречается в документе, но не в том месте, где он встречается в документе. Чтобы выполнить запрос фразы (значение ключевого слова с двойными кавычками), необходимо сравнить положение в документе для слов в фразе. Это означает, что мы не можем использовать индекс префикса для запросов фраз. В этом случае мы внутренним образом Раскройте запрос со всеми возможными словами, к которым префикс расширяется; Например,  `"time*"` можно развернуть до  `"time OR timer OR times OR timex OR timeboxed OR …"` . Максимальное число вариантов, которые можно добавить  10 000, а не число документов, соответствующих запросов. Отсутствует верхний предел для нефразных терминов. 
  
## <a name="indexing-limits-for-email-messages"></a>Ограничения индексации сообщений электронной почты

В следующей таблице описываются ограничения индексации, которые могут привести к возврату сообщения электронной почты в виде неиндексированного элемента или частично индексированного элемента в результатах поиска контента.
  
| Предельное число индексации | Максимальное значение | Описание |
|:-----|:-----|:-----|
|Максимальный размер вложения|150 МБ  <br/> |Максимальный размер вложения электронной почты, которое будет анализироваться для индексирования. Любое вложение, размер которого превышает это значение, не будет анализироваться для индексирования, а сообщение с вложением будет помечено как частично индексированное.  <br/> <br/>**Примечание:** Анализ — это процесс, при котором служба индексирования извлекает текст из вложения, удаляет лишние символы, такие как пунктуация и пробелы, а затем разделяет текст на слова (в процессе, называемом разметки), которые затем хранятся в индексе.           |
|Максимальное количество вложений  <br/> |250  <br/> |Максимальное количество файлов, вложенных в сообщение электронной почты, которое будет анализироваться для индексирования. Если сообщение содержит более 250 вложений, анализируются и индексируются первые 250 вложений, а сообщение помечается как частично индексированное, так как оно имело дополнительные вложения, которые не были проанализированы.  <br/> |
|Максимальная глубина вложения  <br/> |более  <br/> |Максимальное количество анализируемых вложенных вложений. Например, если с сообщением электронной почты связано другое сообщение, а у вложенного сообщения есть вложенный документ Word, то документ Word и вложенное сообщение будут индексироваться. Это будет продолжаться до 30 вложенных вложений.  <br/> |
|Максимальное число подключенных изображений  <br/> |нуль  <br/> |Изображение, вложенное в сообщение электронной почты, пропускается средством синтаксического анализа и не индексируется.  <br/> |
|Максимальное время, затраченное на синтаксический анализ элемента  <br/> |30 секунд  <br/> |При синтаксическом разборе элемента для индексирования задается не более 30 секунд. Если время синтаксического анализа превышает 30 секунд, элемент помечается как частично индексированный.  <br/> |
|Максимальные выходные данные синтаксического анализатора  <br/> |2 миллиона символов  <br/> |Максимальный объем текстовых выходных данных синтаксического анализатора, индексируемого по индексу. Например, если парсер извлекли 8 000 000 символов из документа, индексируются только первые 2 000 000 символов.  <br/> |
|Максимальное количество маркеров аннотации  <br/> |2 000 000  <br/> |При индексировании сообщения электронной почты каждое слово помечается различными инструкциями по обработке, определяющими способ индексации этого приложения. Каждый набор инструкций обработки называется маркером заметки. Для поддержки качества обслуживания в Office 365 существует ограниченное количество маркеров аннотации 2 000 000 для сообщения электронной почты.  <br/> |
|Максимальный размер основного текста в индексе  <br/> |67 000 000 символов  <br/> |Общее количество символов в тексте сообщения электронной почты и всех его вложениях. При индексировании сообщения электронной почты весь текст в тексте сообщения и во всех вложениях объединяется в одну строку. Максимальный размер индексируемой строки составляет 67 000 000 символов.  <br/> |
|Максимальное число уникальных маркеров в теле  <br/> |1 миллион  <br/> |Как описывалось ранее, маркеры — это результат извлечения текста из содержимого, удаления знаков препинания и пробелов, а затем разделения этого текста на слова (называемые токенами), которые хранятся в индексе. Например, фраза  `"cat, mouse, bird, dog, dog"` содержит 5 маркеров. Но только 4 из них являются уникальными маркерами. Количество уникальных маркеров 1 000 000 для каждого сообщения электронной почты, которое помогает предотвратить слишком большой размер индекса с помощью случайных маркеров.  <br/> |
