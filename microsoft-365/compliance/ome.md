# <a name="message-encryption"></a>Шифрование сообщений

Электронная почта часто используется для обмена конфиденциальными сведениями, такими как финансовые данные, контракты, конфиденциальные сведения о продуктах, прогнозы объема реализации и отчеты о продажах, сведения о пациентах, а также данные о клиентах и сотрудниках. В результате почтовые ящики превратились в хранилища, содержащие большое количество потенциально конфиденциальных сведений, и утечка информации становится серьезной угрозой для организаций.

С помощью шифрования сообщений Office 365 ваша организация может отправлять и получать зашифрованные сообщения электронной почты между людьми внутри и за пределами организации. Шифрование сообщений Office 365 работает с Outlook.com, Yahoo!, Gmail и другими службами электронной почты. Шифрование сообщений электронной почты позволяет гарантировать, что только получатели смогут просматривать содержимое сообщения.

## <a name="how-office-365-message-encryption-works"></a>Как работает шифрование сообщений Office 365

Остальная часть этой статьи относится к новым возможностям OME.

Шифрование сообщений Office 365 — это веб-служба, созданная на основе Службы управления правами Microsoft Azure (Azure RMS), которая входит в состав Azure Information Protection. К ним относятся политики шифрования, удостоверений и авторизации для защиты электронной почты. Вы можете шифровать сообщения с помощью шаблонов управления правами, параметра ["Не](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#do-not-forward-option-for-emails)переадправление" и параметра ["Только шифрование".](https://docs.microsoft.com/information-protection/deploy-use/configure-usage-rights#encrypt-only-option-for-emails)

Затем с помощью этих параметров пользователи могут шифровать сообщения электронной почты и различные вложения. Полный список поддерживаемых типов вложений см. в подразделе "Типы файлов, на которые распространяется политика IRM при их вложении в сообщения" в введение в IRM для сообщений [электронной почты.](https://support.office.com/article/bb643d33-4a3f-4ac7-9770-fd50d95f58dc#FileTypesforIRM)

Администратор также может определить правила потока почты для применения этой защиты. Например, можно создать правило, которое требует шифрования всех сообщений, адресованных определенному получателю, или которое содержит определенные слова в строке темы, а также указать, что получатели не могут копировать или печатать содержимое сообщения.

В отличие от предыдущей версии OME, новые возможности обеспечивают единое решение для отправитель независимо от того, отправляете ли вы почту внутри организации или получателям за пределами Microsoft 365. Кроме того, получателям, которые получают защищенное сообщение электронной почты, отправленное в учетную запись Microsoft 365 в Outlook 2016 или Outlook в Интернете, не нужно делать никаких дополнительных действий для просмотра сообщения. Это работает без проблем. Получатели, использующие другие почтовые клиенты и поставщики услуг электронной почты, также имеют улучшенный опыт. Сведения см. в [сведениях о защищенных сообщениях в Office 365](https://support.office.com/article/Learn-about-protected-messages-in-Office-365-2baf3ac7-12db-40a4-8af7-1852204b4b67) и о том, как открыть [защищенное сообщение.](https://support.office.com/article/How-do-I-open-a-protected-message-1157a286-8ecc-4b1e-ac43-2a608fbf3098)

Подробный список различий между предыдущей версией OME и новыми возможностями OME см. в описании сравнения версий [OME.](ome-version-comparison.md)

Когда кто-то отправляет сообщение электронной почты, которое соответствует правилу потока почты шифрования, оно шифруется перед отправкой. Все конечные пользователи Microsoft 365, которые используют клиенты Outlook для чтения почты, получают первоклассные средства чтения зашифрованной и защищенной правами почты, даже если они не в той же организации, что и отправитель. Поддерживаемые клиенты Outlook включают классические приложения Outlook, Outlook Mac, Outlook Mobile для iOS и Android и Outlook в Интернете (прежнее название — Outlook Web App).

Получатели зашифрованных сообщений, которые получают зашифрованную или защищенную правами почту, отправленную в свои учетные записи Outlook.com, Gmail и Yahoo, получают сообщение-оболочку, которое направляет их на портал OME, где они могут легко проверить подлинность с помощью учетной записи Майкрософт, Gmail или Yahoo.

Конечные пользователи, которые читают зашифрованную или защищенную правами почту на клиентах, кроме Outlook, также используют портал OME для просмотра зашифрованных и защищенных правами сообщений, которые они получают.

Если отправитель защищенной почты находится в GCC High, а получатель находится за пределами GCC High, включая коммерческих пользователей, Outlook.com пользователей и пользователей других поставщиков электронной почты, таких как Gmail, получатель получает сообщение-оболочку. Сообщение-оболочка направляет получателя на портал OME, где получатель может читать и отвечать на сообщение. В противном случае, если отправитель и получатель находятся в среде GCC High, даже если они находятся в разных организациях, получатели, которые используют клиенты Outlook для чтения почты, получают в первую очередь средства чтения зашифрованной и защищенной правами почты. Дополнительные сведения о различных вариантах работы в GCC High см. в сравнениях версий [OME.](ome-version-comparison.md)

Дополнительные сведения об ограничениях на размер сообщений и вложений, которые можно зашифровать с помощью OME, см. в [exchange Online Limits.](https://technet.microsoft.com/library/exchange-online-limits.aspx)

## <a name="how-office-365-advanced-message-encryption-works-on-top-of-ome"></a>Как office 365 Advanced Message Encryption работает поверх OME

Office 365 Advanced Message Encryption позволяет создавать несколько шаблонов фирменной настройки, чтобы можно было точно настраивать контроль над почтой получателей и создавать пользовательские возможности фирменной настройки для поддержки различных организационных структур.

Расширенные возможности шифрования сообщений в Microsoft 365 помогают выполнять обязательства по обеспечению соответствия требованиям, которые требуют более гибкого контроля над доступом внешних получателей к зашифрованным электронным письмам. С помощью расширенного шифрования сообщений в Office 365 администратор может управлять конфиденциальными сообщениями электронной почты за пределами организации с помощью автоматических политик, обнаруживающих типы конфиденциальной информации (например, личные сведения, финансовые или медицинские коды) или ключевые слова для повышения защиты путем истечения срока доступа через защищенный веб-портал к зашифрованным письмам. Как администратор вы можете дополнительно управлять зашифрованными электронными письмами, которые доступны на веб-портале Microsoft 365, отменив доступ к электронной почте в любое время.

Отзыв и истечение срока действия сообщений работают только для сообщений электронной почты, отправляемой пользователями за пределами организации. Кроме того, получатели должны получить доступ к электронной почте через веб-портал. Чтобы получатель использовал портал для получения электронной почты, необходимо настроить настраиваемый шаблон фирменой настройки, который применяет оболочку. Затем шаблон фирменой марки применяется в правиле потока почты. Дополнительные сведения о advanced Message Encryption см. в office [365 Advanced Message Encryption.](ome-advanced-message-encryption.md)

## <a name="defining-rules-for-office-365-message-encryption"></a>Определение правил для шифрования сообщений Office 365

Один из способов включить новые возможности шифрования сообщений Office 365 — определить правила потока почты для администраторов Exchange Online и Exchange Online Protection. Эти правила определяют, при каких условиях сообщения электронной почты должны быть зашифрованы. Если для правила настроено действие шифрования, все сообщения, которые соответствуют условиям правила, шифруются перед их отправлением.

Правила потока почты являются гибкими, позволяя сочетать условия, чтобы соответствовать определенным требованиям безопасности в одном правиле. Например, можно создать правило для шифрования всех сообщений, которые содержат указанные ключевые слова и адресуются внешним получателям. Новые возможности шифрования сообщений Office 365 также шифруют ответы от получателей зашифрованной электронной почты.

Дополнительные сведения о том, как создать правила потока почты для использования новых возможностей OME, см. в теме "Определение правил для шифрования сообщений [Office 365".](define-mail-flow-rules-to-encrypt-email.md)

## <a name="sending-viewing-and-replying-to-encrypted-email-messages"></a>Отправка, просмотр и ответ на зашифрованные сообщения электронной почты

С помощью шифрования сообщений Office 365 пользователи могут отправлять зашифрованные сообщения из Outlook и Outlook в Интернете. Кроме того, администраторы могут настроить правила потока почты в Microsoft 365 для автоматического шифрования сообщений электронной почты в зависимости от совпадения ключевых слов или других условий.

Получатели зашифрованных сообщений, которые находятся в организациях, смогут легко читать эти сообщения в любой версии Outlook, включая Outlook для ПК, Outlook для Mac, Outlook в Интернете, Outlook для iOS и Outlook для Android. Пользователи, которые получают зашифрованные сообщения в других почтовых клиентах, могут просматривать их на портале OME.
