# <a name="set-up-a-connector-to-archive-cisco-jabber-data"></a>Настройка соединителя для архива данных Cisco Jabber

Используйте соединителю Globanet в центре соответствия требованиям Microsoft 365 для импорта и архива данных с платформы Cisco Jabber в почтовые ящики пользователей в организации Microsoft 365. Globanet предоставляет вам соединителю [Cisco Jabber,](https://globanet.com/jabber/) который настроен для захвата элементов из базы данных ms SQL Jabber, таких как сообщения чата 1:1 и групповые чаты, а затем импортировать эти элементы в Microsoft 365. Соединителю извлекает данные из базы данных MS SQL Cisco Jabber, обрабатывает его и преобразует содержимое из учетной записи пользователя Cisco Jabber в формат сообщения электронной почты, а затем импортирует эти элементы в почтовый ящик пользователя в Microsoft 365.

После хранения данных Cisco Jabber в почтовых ящиках пользователей можно применить функции соответствия требованиям Microsoft 365, такие как хранение судебного разбирательства, открытие электронных данных, политики хранения и метки хранения, а также соответствие требованиям к связи. Использование соединителя Cisco Jabber для импорта и архива данных в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="overview-of-archiving-cisco-jabber-data"></a>Обзор архива данных Cisco Jabber

В следующем обзоре рассказывается о процессе использования соединителя для архивации данных Cisco Jabber в Microsoft 365.

![Архивативная работа для данных Cisco Jabber](../media/CiscoJabberonMSSQLConnectorWorkflow.png)

1. Ваша организация работает с Cisco для настройки и настройки Cisco Jabber в базе данных MS SQL.

2. Каждые 24 часа элементы Cisco Jabber копируется из базы данных MS SQL на сайт Globanet Merge1. Соединителет также преобразует содержимое сообщений чата в формат сообщений электронной почты.

3. Соединителю Cisco Jabber, который вы создаете в центре соответствия требованиям Microsoft 365, ежедневно подключается к сайту Globanet Merge1 и передает элементы в безопасное хранилище Azure в облаке Microsoft.

4. Автоматическое сопоставление пользователей в качестве соединиттеля импортирует элементы в почтовые ящики определенных пользователей, используя значение свойства *Email,* описанного в [шаге 3.](#step-3-map-users-and-complete-the-connector-setup) Подмостки в папке "Входящие" с именем **Cisco Jabber** в MS SQL создаются в почтовых ящиках пользователей, и элементы сообщения импортируется в эту папку. Соединитатель определяет, в какой почтовый ящик импортировать элементы, используя значение свойства *Email.* Каждый элемент Cisco Jabber содержит это свойство, которое заполняется адресом электронной почты каждого участника.

## <a name="before-you-begin"></a>Подготовка

- Создайте учетную запись Globanet Merge1 для соединители Microsoft. Чтобы создать эту учетную запись, обратитесь в службу поддержки клиентов [Globanet.](https://globanet.com/ms-connectors-contact/) При создании соединитетеля в шаге 1 вы вопишитесь в эту учетную запись.

- Настройка базы данных ms SQL для получения элементов Jabber перед созданием соединителя в шаге 1. Параметры подключения для базы данных MS SQL при настройке соединителя Cisco Jabber в шаге 2. Дополнительные сведения см. в руководстве пользователя по сторонним [соединитетелям Merge1.](https://docs.ms.merge1.globanetportal.com/Merge1%20Third-Party%20Connectors%20Cisco%20Jabber%20on%20MS%20SQL%20User%20Guide%20.pdf)

- Пользователь, создавший соединитель Cisco Jabber в шаге 1 (и завершивший его в шаге 3), должен быть назначен роли экспорта импорта почтовых ящиков в Exchange Online. Эта роль необходима для добавления соединители на странице **соединители** данных в центре соответствия требованиям Microsoft 365. По умолчанию эта роль не назначена группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах Создание групп [ролей](/Exchange/permissions-exo/role-groups#create-role-groups) или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-set-up-the-cisco-jabber-connector"></a>Шаг 1. Настройка соединителя Cisco Jabber

Первым шагом является доступ  к соединителям данных в центре соответствия требованиям Microsoft 365 и создание соединителя для Cisco Jabber на данных MS SQL.

1. Перейдите [https://compliance.microsoft.com](https://compliance.microsoft.com/) и нажмите **соединители данных** Cisco  >  **Jabber** на MS SQL .

2. На странице **описания продукта Cisco Jabber SQL** MS нажмите **кнопку Добавить соединителю**.

3. На странице **Условия службы нажмите** кнопку **Принять**.

4. Введите уникальное имя, которое идентифицирует соединителен, а затем нажмите **кнопку Далее**.

5. Впишитесь в свою учетную запись Merge1, чтобы настроить соединители.

## <a name="step-2-configure-the-cisco-jabber-connector-on-the-globanet-merge1-site"></a>Шаг 2. Настройка соединителя Cisco Jabber на сайте Globanet Merge1

Второй шаг — настройка соединиттеля Cisco Jabber на ms SQL на сайте Globanet Merge1. Сведения о настройке соединиттеля Cisco Jabber на ms SQL см. в руководстве по пользователю [connectors merge1.](https://docs.ms.merge1.globanetportal.com/Merge1%20Third-Party%20Connectors%20Cisco%20Jabber%20on%20MS%20SQL%20User%20Guide%20.pdf)

После **нажатия кнопки Сохранить &**  finish отображается страница сопоставления пользователей в мастере соединители в центре соответствия требованиям Microsoft 365.

## <a name="step-3-map-users-and-complete-the-connector-setup"></a>Шаг 3. Карта пользователей и завершение установки соединитетеля

Чтобы составить карту пользователей и завершить соединители, установленные в центре соответствия требованиям Microsoft 365, выполните следующие действия:

1. На странице Map Cisco Jabber на **ms SQL пользователи на странице Microsoft 365** включат автоматическое сопоставление пользователей. Элементы Cisco Jabber на ms SQL содержат свойство *Email,* содержаное адреса электронной почты для пользователей в организации. Если соединитатель может связать этот адрес с пользователем Microsoft 365, элементы импортируется в почтовый ящик этого пользователя.

2. Нажмите **кнопку Далее,** просмотрите  параметры и перейдите на страницу соединители данных, чтобы просмотреть ход процесса импорта нового соединитетеля.

## <a name="step-4-monitor-the-cisco-jabber-connector"></a>Шаг 4. Мониторинг соединителя Cisco Jabber

После создания соединителя Cisco Jabber SQL ms можно просмотреть состояние соединителя в центре соответствия требованиям Microsoft 365.

1. Перейдите [https://compliance.microsoft.com](https://compliance.microsoft.com) и щелкните **соединители данных** в левом nav.

2. Щелкните **вкладку** Соединители, а затем выберите **соединителю Cisco Jabber** на ms SQL, чтобы отобразить страницу вылетов. На этой странице содержатся свойства и сведения о соединители.

3. В **состоянии Соединитель с исходным кодом** щелкните ссылку **Журнал** загрузки, чтобы открыть (или сохранить) журнал состояния соединитетеля. В этом журнале содержатся данные, импортируемые в облако Майкрософт.

## <a name="known-issues"></a>Известные проблемы

- В настоящее время мы не поддерживаем импорт вложений или элементов размером более 10 МБ. Поддержка более крупных элементов будет доступна позднее.
