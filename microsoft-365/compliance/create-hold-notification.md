# <a name="create-a-legal-hold-notice"></a>Создание юридического уведомления о удержании

Используя коммуникации хранителя Advanced eDiscovery, организации могут управлять своим процессом взаимодействия с хранителями. С помощью средства "Связь" юридические группы могут систематически отправлять, собирать и отслеживать уведомления об удержании по юридическим основаниям. Гибкий процесс создания также позволяет группам настраивать рабочий процесс уведомлений об удержании и содержимое уведомлений, отправленных хранителям.

![Страница "Связь"](../media/CommunicationPage.PNG)

В статье описаны действия в рабочий процесс уведомления об удержании.

## <a name="step-1-specify-communication-details"></a>Шаг 1. Указание сведений о связи

Первый шаг — указать соответствующие сведения для уведомлений об удержании по юридическим и другим данным хранителя.

![Страница "Связь с именем"](../media/NameCommunication.PNG)

1. В Центре безопасности & соответствия требованиям перейдите в > **Advanced eDiscovery,** чтобы отобразить список дел в организации.

2. Выберите дело, щелкните **вкладку "Связь"** и выберите "Новое **взаимодействие".**

3. На странице **"Связь с именем"** укажите следующие (необходимые) сведения о связи.

    - **Name**: this is the name for the communication.

    - **Выдача** сотрудника: в выпадаемом списке отображается список участников дела. Дополнительные сведения о добавлении новых участников в дело см. в примере создания дела [Advanced eDiscovery.](create-and-manage-advanced-ediscoveryv2-case.md#create-a-case) Каждое уведомление, отправленные хранителям, будет отправлено от имени указанного выдавляемого сотрудника.

> [!NOTE]
> У выдавляемой должности должен быть **активный** почтовый ящик, чтобы он мог демонстрироваться в выпадаемом списке "Сотрудник по выдаче".


4. Нажмите **Далее**.

## <a name="step-2-define-the-portal-content"></a>Шаг 2. Определение содержимого портала

Затем можно создать и добавить содержимое уведомления об удержании. На странице **"Определение содержимого портала"** в **мастере** создания взаимодействия укажите содержимое уведомления об удержании. Это содержимое будет автоматически примещение к уведомлениям о выдаче, повторной выдаче, напоминании и эскалации. Кроме того, этот контент будет отображаться на портале соответствия требованиям хранителя. 

![Страница содержимого портала](../media/PortalContent.PNG)

Чтобы создать контент портала:

1. Введите (или вырезание и ветвь из другого документа) уведомление о удержании в текстовом ящике для содержимого портала. 

2. Вставьте переменные объединения в уведомление, чтобы настроить уведомление и поделиться порталом соответствия требованиям custodian.

3. Нажмите **Далее**.

  >[!Tip]
  >Чтобы узнать больше о том, как настроить контент и формат содержимого портала, см. статью ["Использование редактора коммуникаций".](using-communications-editor.md)

## <a name="step-3-set-the-required-notifications"></a>Шаг 3. Настройка необходимых уведомлений

После определения содержимого уведомления об удержании можно настроить рабочий процесс отправки и управления процессом уведомления. Уведомления — это сообщения электронной почты, которые отправляются для уведомления и последующего уведомления хранителям. Все хранители, добавленные в связь, получат одинаковое уведомление. 

Чтобы настроить и отправить уведомление об удержании, необходимо включить уведомления о выдаче, повторной выдаче и выпуске.

### <a name="issuance-notification"></a>Уведомление о выдаче 

После создания сообщения уведомление  о выдаче инициалось указанным сотрудником по выдаче. Уведомление о выдаче — это первое сообщение, от отправленное хранителям, чтобы уведомить его об их обязательствах по сохранению. 

Чтобы создать уведомление о выдаче:

1. На **плитке** "Выдача" нажмите кнопку **"Изменить".**

2. При необходимости добавьте в поля "Ск" и "СК" дополнительных участников дела или **сотрудников.**  Чтобы добавить в эти поля нескольких пользователей, разделяйте адреса электронной почты за двоеточием.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранителям (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления о выдаче. 

5. Нажмите кнопку **Сохранить**.

### <a name="re-issuance-notification"></a>Re-Issuance уведомления

По мере выполнения дела хранителям может потребоваться сохранить дополнительные или меньшие данные, чем было подано ранее. После обновления содержимого портала отправляется уведомление о повторной выдаче, которое оповещает хранителей о любых изменениях в их обязательствах по сохранению.

Чтобы создать уведомление о повторной выдаче:

1. На **плитке "Повторное** редактирование" нажмите кнопку **"Изменить".**

2. При необходимости добавьте в поля "Ск" и "Ск" дополнительных участников дела или **сотрудников.**  Чтобы добавить в эти поля нескольких пользователей, разделяйте адреса электронной почты за двоеточием.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранителям (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления о повторной выдаче.

5. Нажмите кнопку **Сохранить**.

> [!NOTE]
> Если содержимое портала изменено  (на странице  "Определение содержимого портала" в мастере редактирования связи), уведомление о повторной выдаче будет автоматически отправлено всем хранителям, которые назначены уведомлению. После того как уведомление будет отправлено, хранителям будет предложено повторно подтвердить свое уведомление о удержании. Если вы настроили какие-либо процессы напоминания или эскалации, они также будут повторно запускаться. Дополнительные сведения о том, какие другие события управления случаями инициирует связь, см. в сведениях о событиях, [которые запускают уведомления.](#events-that-trigger-notifications)

### <a name="release-notification"></a>Уведомление о выпуске

После разрешения вопроса или если хранители больше не подлежат сохранению содержимого, вы можете освободить хранителя из дела. Если хранителям ранее было выдано уведомление о удержании, уведомление о выпуске можно использовать для оповещения хранителей о том, что они освобождены от своих обязательств.

Чтобы создать уведомление о выпуске: 

1. На **плитке "Выпуск"** нажмите кнопку **"Изменить".**

2. При необходимости добавьте в поля "Ск" и "СК" дополнительных участников дела или **сотрудников.**  Чтобы добавить в эти поля нескольких пользователей, разделяйте адреса электронной почты за двоеточием.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранителям (обязательно).

5. Нажмите **кнопку** "Сохранить" и перейдите к следующему шагу.

## <a name="optional-step-4-set-the-optional-notifications"></a>(Необязательно) Шаг 4. Настройка необязательных уведомлений

При желании можно упростить рабочий процесс для дальнейшего с неотвещения хранителями, создав и запланируя автоматические напоминания и уведомления об эскалации.

![Страница напоминания/эскалации](../media/ReminderEscalations.PNG)

### <a name="reminders"></a>Reminders

После того как вы отправили уведомление об удержании, вы можете следить за неотвеживающими хранителями, определив рабочий процесс напоминания.

Чтобы запланировать напоминания:

1. На **плитке "Напоминание"** нажмите кнопку **"Изменить".**

2. **Включит** рабочий процесс напоминания, включив **переключель состояния** (обязательно).

3. Укажите интервал **напоминания (в днях)** (обязательно). Это количество дней до отправки первых и последующих уведомлений с напоминанием. Например, если для интервала напоминания установлено семь дней, то первое напоминание будет отправлено через семь дней после первоначальной выдачи уведомления об удержании. Все последующие напоминания также отправляются каждые семь дней.

4. Укажите **количество напоминаний** (обязательно). В этом поле указывается количество напоминаний, отправляемых неотвечивым хранителям. Например, если установить для количества напоминаний значение 3, хранители получат не более трех напоминаний. После того как хранител подтвердит уведомление об удержании, напоминания больше не будут отправлены этому пользователю.

5. Укажите **тему** уведомления (обязательно). 

6. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранителям (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления с напоминанием.

7. Нажмите **кнопку "Сохранить"** и перейдите к следующему шагу.

### <a name="escalations"></a>Эскалации

В некоторых ситуациях могут потребоваться дополнительные способы данной меры с неотвечивой хранителями. Если хранител не подтверждает уведомление об удержании после получения указанного количества напоминаний, группа юристов может указать рабочий процесс для автоматической отправки уведомления об эскалации хранителю и его руководителю.

Чтобы запланировать эскалацию, спланировать:

1. На **плитке "Эскалация"** нажмите кнопку **"Изменить".**

2. **Включите** рабочий процесс эскалации, включив **переключель состояния.**

3. Укажите **интервал эскалации (в днях)** (обязательно).

4. Укажите **количество эскалаций** (обязательно). В этом поле указывается количество эскалаций для отправки неотвечивым хранителям. Например, если вы установите для количества эскалаций значение 3, уведомление об эскалации будет отправлено хранителям и их руководителю не более трех раз. После того как хранител подтвердит уведомление об удержании, эскалации больше не будут отправлены.

5. Укажите **тему** уведомления (обязательно). 

6. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранителям (обязательно). Содержимое портала, определенное на шаге 2, добавляется в конец уведомления об эскалации.

7. Нажмите **кнопку "Сохранить"** и перейдите к следующему шагу.

## <a name="step-5-assign-custodians-to-receive-notifications"></a>Шаг 5. Назначение хранителей для получения уведомлений

Завершив отправку содержимого для уведомлений, выберите хранителей, в которые вы хотите отправлять уведомления. 

![Страница "Выбор хранителей"](../media/SelectCustodians.PNG)

Чтобы добавить хранителей:

1. Назначьте хранителей сообщению, щелкнув рядом с их именем контрольный ящик.

    После создания сообщения рабочий процесс уведомления автоматически применяется к выбранным хранителям.

2. Нажмите **кнопку** "Далее", чтобы просмотреть параметры связи и сведения.

>[!NOTE]
>Вы можете добавлять только хранителей, которые были добавлены в дело и не были отправлены в рамках дела еще одно уведомление.

## <a name="step-6-review-settings"></a>Шаг 6. Просмотр параметров

После просмотра параметров и нажатия кнопки **"Отправить"** для завершения связи система автоматически запустит рабочий процесс связи, отправив уведомление о выдаче.

## <a name="events-that-trigger-notifications"></a>События, которые запускают уведомления

В следующей таблице описываются события в процессе управления делами, которые запускаются при отправлении хранителям различных типов уведомлений.

|Тип связи|Триггер |
|:---------|:---------|
|Уведомления о выдаче|Начальное создание уведомления. Вы также можете вручную повторно уведомить об удержании. |
|Уведомления о повторной выдаче|Обновление содержимого портала на странице **"Определение содержимого портала"** в **мастере редактирования связи.**|
|Уведомления о выпуске|Хранители освобождены из дела.|
|Reminders|Интервал и количество напоминаний, настроенных для напоминания.|
|Эскалации|Интервал и количество напоминаний, настроенных для эскалации.|
