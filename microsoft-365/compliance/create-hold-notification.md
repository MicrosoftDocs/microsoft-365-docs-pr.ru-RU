# <a name="create-a-legal-hold-notice"></a>Создание уведомления об судебном удержании

С помощью расширенных коммуникаций хранитель для обнаружения электронных данных Организации могут управлять своими рабочими процессами, взаимодействующими с custodians. С помощью средства связи юридические команды могут систематически отправлять, собирать и отслеживать уведомления о юридических удержаниях. Гибкий процесс создания также позволяет Teams настраивать рабочий процесс уведомления об удержаниях и контент в уведомлениях, отправляемых в custodians. 

![Страница "связь"](../media/CommunicationPage.PNG)

В этой статье описываются действия, описанные в рабочем процессе уведомления о удержании.

## <a name="step-1-specify-communication-details"></a>Шаг 1: Указание сведений о общении

Первый шаг — указать соответствующие сведения для уведомлений о судебном удержании или других коммуникаций хранитель.

![Страница "связь с именем"](../media/NameCommunication.PNG)

1. В центре безопасности & соответствия требованиям перейдите на **Обнаружение электронных данных > Advanced eDiscovery** для отображения списка дел в Организации.

2. Выберите вариант, перейдите на вкладку **связь** и нажмите кнопку **создать связь**.

3. На странице " **связь с именем** " укажите следующие (обязательные) сведения о связи.

    - **Name**: имя связи.

    - **Выдача директора**: в раскрывающемся списке отображается список элементов дела. Дополнительные сведения о том, как добавлять в обращение новые элементы, приведены в статье [Создание расширенного случая обнаружения электронных](get-started-with-advanced-ediscovery.md#step-4-create-an-advanced-ediscovery-case)данных. Каждое уведомление, отправленное в custodians, будет отправлено от имени указанного директора выпуска.

> [!NOTE]
> У ответственного руководителя должен быть **активный почтовый ящик** для отображения в раскрывающемся меню выдачи директора


4. Нажмите кнопку **Далее**.

## <a name="step-2-define-the-portal-content"></a>Шаг 2: Определение контента портала

Затем вы можете создать и добавить содержимое уведомления об удержании. На странице **Определение контента портала** в мастере **создания связи** укажите содержимое уведомления об удержании. Это содержимое будет автоматически добавлено к уведомлениям о выдаче, повторной выдаче, напоминании и эскалации. Кроме того, этот контент будет отображаться на портале соответствия требованиям хранитель. 

![Страница контента портала](../media/PortalContent.PNG)

Чтобы создать контент портала, выполните следующие действия.

1. Введите (или скопируйте и вставьте из другого документа) ваше уведомление об удержании в текстовом поле для контента портала. 

2. Вставьте переменные слияния в уведомление о настройке уведомления и предоставлении доступа к порталу обеспечения соответствия требованиям хранитель.

3. Нажмите кнопку **Далее**.

  >[!Tip]
  >Дополнительные сведения о способах настройки контента и формата контента портала см [в разделе Использование редактора взаимодействий](using-communications-editor.md).

## <a name="step-3-set-the-required-notifications"></a>Шаг 3: Установка необходимых уведомлений

Определив содержимое уведомления об удержании, вы можете настроить рабочие процессы для отправки и управления процессом уведомления. Уведомления — это сообщения электронной почты, которые отправляются для уведомления и дальнейших действий с custodians. Каждый хранитель, добавляемый в связь, получает одно и то же уведомление. 

Чтобы настроить и отправить уведомление о удержании, необходимо включить уведомления о выдаче, повторной выдаче и выпуске.

### <a name="issuance-notification"></a>Уведомление о выдаче 

После создания связи **уведомление выдачи** инициируется указанным должностным лицом. Уведомление о выдаче — первое сообщение, отправляемое в хранитель, чтобы сообщить им о обязательствах по их сохранности. 

Чтобы создать уведомление о выдаче:

1. В плитке **выдачи** нажмите кнопку **изменить**.

2. При необходимости добавьте дополнительные элементы или сотрудники в поля **"копия"** и **"Скрытая копия"** . Чтобы добавить в эти поля несколько пользователей, разделяйте адреса электронной почты точками с запятой.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранитель (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления о выдаче. 

5. Щелкните **Сохранить**.

### <a name="re-issuance-notification"></a>Уведомление о Re-Issuance

По мере выполнения сценария может потребоваться custodians для сохранения дополнительных или меньших данных, чем ранее было указано. После обновления контента портала отправляются уведомления о повторной выдаче и отправляются оповещения custodians о любых изменениях в обязательствах сохранения.

Чтобы создать уведомление о повторной выдаче:

1. В плитке **повторной выпуске** щелкните **изменить**.

2. При необходимости добавьте дополнительные элементы или сотрудники в поля **"копия"** и **"Скрытая копия"** . Чтобы добавить в эти поля несколько пользователей, разделяйте адреса электронной почты точками с запятой.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранитель (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления о повторной выдаче.

5. Щелкните **Сохранить**.

> [!NOTE]
> При изменении контента портала (на странице **Определение содержимого портала** в мастере **изменения связи** ) уведомление о повторной выдаче будет автоматически отправлено всем custodians, назначенным этому уведомлению. После отправки уведомления пользователю custodians будет предложено повторно подтвердить свое уведомление об удержании. Если вы настроили какие-либо напоминания или рабочие процессы эскалации, они также будут перезапущены. Дополнительные сведения о том, что другие события управления обращениями запускают сообщения, приведены в разделе [события, инициирующие уведомления](#events-that-trigger-notifications).

### <a name="release-notification"></a>Уведомление о выпуске

После того как пользователь разрешается или хранитель больше не сохраняет содержимое, вы можете освободить хранитель из случая. Если в Хранитель ранее было отправлено уведомление об удержании, уведомление о выпуске можно использовать для оповещения custodians о том, что они были выпущены из обязательств.

Чтобы создать уведомление о выпуске, выполните указанные ниже действия. 

1. В плитке **выпуск** нажмите кнопку **изменить**.

2. При необходимости добавьте дополнительные элементы или сотрудники в поля **"копия"** и **"Скрытая копия"** . Чтобы добавить в эти поля несколько пользователей, разделяйте адреса электронной почты точками с запятой.

3. Укажите **тему** уведомления (обязательно).

4. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранитель (обязательно).

5. Нажмите кнопку **сохранить** и перейдите к следующему шагу.

## <a name="optional-step-4-set-the-optional-notifications"></a>Необязательно Шаг 4: Настройка дополнительных уведомлений

Кроме того, вы можете упростить рабочий процесс, чтобы не отвечать на custodians, создавая и планируя автоматические напоминания и уведомления о эскалации.

![Страница напоминания/эскалации](../media/ReminderEscalations.PNG)

### <a name="reminders"></a>Reminders

После отправки уведомления об удержании вы можете подписаться на неотвечающие custodians, определив рабочий процесс напоминания.

Чтобы запланировать напоминания:

1. В плитке **напоминание** нажмите кнопку **изменить**.

2. Включите рабочий процесс **напоминания** , включив переключатель **состояния** (обязательно).

3. Укажите **интервал напоминаний (в днях)** (обязательный). Это количество дней, в течение которых необходимо подождать, прежде чем отправлять уведомления о начале и исполнении напоминания. Например, если задать для интервала напоминания значение семь дней, первое напоминание будет отправлено на семь дней после первоначального выдачи уведомления об удержании. Все последующие напоминания также будут отправляться каждые семь дней.

4. Укажите **количество памяток** (обязательный параметр). В этом поле указывается, сколько напоминаний следует отправлять в неотвечающие custodians. Например, если задать для параметра Количество напоминаний значение 3, то в Хранитель будет получено не более трех напоминаний. После того как хранитель признает уведомление об удержании, напоминания больше не будут отправляться этому пользователю.

5. Укажите **тему** уведомления (обязательно). 

6. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранитель (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления о напоминании.

7. Нажмите кнопку **сохранить** и перейдите к следующему шагу.

### <a name="escalations"></a>Эскалации

В некоторых ситуациях могут потребоваться дополнительные способы отслеживания неотвечающего custodians. Если хранитель не распознает уведомление о удержании после получения указанного количества напоминаний, юридический специалист может указать рабочий процесс для автоматической отправки уведомления о эскалации в хранитель и руководителю.

Планирование укрупнений:

1. В плитке **эскалация** нажмите кнопку **изменить**.

2. Включите рабочий процесс **эскалации** , включив переключатель **Status** .

3. Укажите **интервал эскалации (в днях)** (обязательно).

4. Укажите **количество укрупнений** (обязательный параметр). В этом поле указывается количество укрупнений, которые могут отправляться в неотвечающие custodians. Например, если вы установили число укрупнений равным 3, то уведомление о эскалации будет отправлено в хранитель, а их менеджеру не более трех раз. После того как хранитель признает уведомление об удержании, эскалации больше не будут отправляться.

5. Укажите **тему** уведомления (обязательно). 

6. Укажите содержимое или дополнительные инструкции, которые вы хотите предоставить хранитель (обязательно). Содержимое портала, определенное в шаге 2, добавляется в конец уведомления об эскалации.

7. Нажмите кнопку **сохранить** и перейдите к следующему шагу.

## <a name="step-5-assign-custodians-to-receive-notifications"></a>Шаг 5: назначение custodians для получения уведомлений

После завершения работы с содержимым для уведомлений выберите custodians, на которые вы хотите отправлять уведомления. 

![Выбор страницы custodians](../media/SelectCustodians.PNG)

Чтобы добавить custodians, выполните указанные ниже действия.

1. Назначьте custodians связи, установив флажок рядом с именем.

    После создания связи рабочий процесс уведомления будет автоматически применен к выбранному custodians.

2. Нажмите кнопку **Далее** , чтобы просмотреть сведения о параметрах и сведениях о общении.

>[!NOTE]
>Вы можете добавлять только custodians, добавленные к случаю, и не отправляли еще одно уведомление в случае.

## <a name="step-6-review-settings"></a>Шаг 6: Просмотр параметров

После просмотра параметров и нажатия кнопки **Отправить** для завершения связи система автоматически запустит рабочий процесс связи, отправив уведомление о выдаче.

## <a name="events-that-trigger-notifications"></a>События, инициирующие уведомления

В следующей таблице описываются события в процессе управления делами, которые инициируются при отправке различных типов уведомлений в custodians.

|Тип общения|Триггер |
|:---------|:---------|
|Уведомления о выдачах|Начальное создание уведомления. Вы также можете вручную повторно отправить уведомление об удержании. |
|Повторное уведомление о выдаче|Обновление контента портала на странице **Определение содержимого портала** в мастере **изменения взаимодействия** .|
|Уведомления о выпуске|Хранитель выпускается из этого случая.|
|Reminders|Интервал и количество напоминаний, настроенных для напоминания.|
|Эскалации|Интервал и количество напоминаний, настроенных для эскалации.|
