# <a name="manage-holds-in-advanced-ediscovery"></a>Управление удерживает в Advanced eDiscovery

Вы можете использовать Advanced eDiscovery для создания содержимого, которое может иметь отношение к вашему делу. Используя возможности Advanced eDiscovery хранения, можно разместить удержания на хранителях и их источниках данных. Кроме того, можно разместить удержание без содержания в почтовых ящиках и OneDrive для бизнеса сайтах. Вы также можете разместить удержание на групповом почтовом ящике, SharePoint сайте и OneDrive для бизнеса для Microsoft 365 Group. Аналогичным образом можно разместить удержание на почтовом ящике и сайте, связанных с Microsoft Teams. При удержании расположения контента содержимое удерживается до тех пор, пока вы не отпустите хранителя, удалите определенное расположение данных или полностью удалите политику хранения.

## <a name="manage-custodian-based-holds"></a>Управление хранением на основе хранителя

В некоторых случаях у вас может быть набор хранителей, которые вы идентифицировали и решили сохранить свои данные во время дела. В Advanced eDiscovery, когда эти хранители помещаются на удержание, пользователь и выбранные источники данных автоматически добавляются в политику хранения хранителей.

Чтобы просмотреть политику удержания хранителя:

1. В центре Microsoft 365 для отображения списка случаев в организации **щелкните > eDiscovery > Advanced.**

2. Перейдите на **вкладку Источники,** чтобы добавить хранителей в вашем случае. Чтобы узнать, как можно добавлять и удерживать хранителей в Advanced eDiscovery случае, см. в примере Добавление хранителей [в дело](add-custodians-to-case.md). Если вы уже добавили хранителей и поместили их на удержание, перейдите к шагу 3.

3. Перейдите на **вкладку Удержание** и нажмите **Кнопку CustodianHold \<HoldId>**.

4. На странице вылет можно увидеть статистику удержания для политики. Вы также можете выполнять действия, например применить запрос к удержанию на основе хранителя. Дополнительные сведения о создании запроса удержания и использовании условий см. в статье [Keyword querys and search conditions for Content Search.](keyword-queries-and-search-conditions.md)

## <a name="manage-non-custodial-holds"></a>Управление неконтлиционным хранением

При создании удержания у вас есть следующие параметры для области контента, который находится в указанных расположениях контента:

- Создается бесконечное удержание, в котором все содержимое помещается на удержание. Кроме того, можно создать удержание на основе запросов, в котором отложено только содержимое, которое соответствует запросу поиска.
  
- Вы можете указать диапазон дат для удержания только контента, отправленного, полученного или созданного в этом диапазоне дат. Кроме того, вы можете держать все содержимое независимо от того, когда оно было отправлено, получено или создано.

Чтобы создать удержание без содержания под стражей для Advanced eDiscovery дела:

1. В центре Microsoft 365 для отображения списка случаев в организации **щелкните > eDiscovery > Advanced.**
  
2. Нажмите **Кнопку Открыть** рядом с случаем, в который необходимо создать удерживает.
  
3. На домашней странице для дела щелкните вкладку **Держит.**
  
4. На **вкладке Держит** нажмите **кнопку Создать**.
  
5. На странице **Имя удерживаемой** страницы укайте имя удержания. Имя удержания должно быть уникальным в пределах организации.
 
6. (Необязательный) В поле **Описание** добавьте описание удержания.
  
7. Нажмите **Далее**.
  
8. Выберите расположения контента, которые необходимо разместить на удержании. Вы можете поместить на удержание почтовые ящики, сайты и общедоступные папки.

   1. **Exchange** электронной почты нажмите кнопку **Выберите пользователей,** групп или групп, а затем нажмите кнопку Выберите **пользователей,** групп или групп снова, чтобы указать почтовые ящики для удержания. Для поиска почтовых ящиков пользователей и групп рассылки, которые необходимо поставить на удержание (чтобы поставить почтовые ящики членов групп на удержание), воспользуйтесь полем поиска. Вы также можете разместить удержание в связанном почтовом ящике для Microsoft 365 группы или группы Майкрософт. Выберите пользователя, группу, поле проверки группы, нажмите **кнопку Выберите**, а затем нажмите кнопку **Готово**.
 
      > [!NOTE]
      > При нажатии кнопки **Выберите пользователей, групп** или групп, чтобы указать почтовые ящики для удержания, отображаемая выборка почтовых ящиков пуста. Это вызвано мерами по повышению скорости работы. Чтобы добавить людей в этот список, введите имя (минимум 3 символа) в поле поиска.

   1. **SharePoint** сайты — **выберите** сайты и  нажмите кнопку Выберите сайты снова, чтобы указать SharePoint и OneDrive для бизнеса сайты для удержания. Укажите URL-адрес каждого сайта, который вы хотите поставить на удержание. Вы также можете добавить URL-адрес SharePoint для Microsoft 365 группы или microsoft Team. Нажмите **кнопку Выберите,** а затем нажмите **кнопку Готово**.

      > [!NOTE]
      > URL-адрес учетной записи OneDrive пользователя включает основное имя пользователя (UPN) `https://alpinehouse-my.sharepoint.com/personal/sarad_alpinehouse_onmicrosoft_com` (например). В редких случаях, когда УИБ человека меняется, OneDrive URL-адрес будет также изменен для включения нового upN. Если учетная запись пользователя OneDrive является частью удержания, не на хранение, и его upN изменен, необходимо обновить удержание и указать на новый URL-OneDrive. Дополнительные сведения см. в разделе [Как изменения UPN влияют на URL-адрес OneDrive](/onedrive/upn-changes).

   1. **Exchange** общедоступных папок — переместите переключатель переключателя на положение Все, чтобы все общедоступные папки Exchange Online организации. Обратите внимание, что вы не можете выбрать определенные общедоступные папки для удержания. Оставьте переключатель toggle **в None,** если вы не хотите удерживать общедоступные папки.

9. После добавления расположения контента в удержание нажмите **кнопку Далее**.
  
10. Чтобы создать удержание на основе запроса с условиями, выполните следующее. В противном случае нажмите **кнопку Далее**.
    
    - В поле под **ключевыми словами** введите поисковый запрос в поле, чтобы приодержится только содержимое, отвечающий критериям поиска. Можно указать ключевые слова, свойства сообщений или свойства документов, например имена файлов. Вы также можете использовать более сложные запросы, которые используют оператора Boolean, например AND, OR или NOT. Если поле ключевого слова останется пустым, все содержимое, расположенное в указанных расположениях контента, будет помещено в удержание.

    - Нажмите  **кнопку** Добавить условия, чтобы добавить одно или несколько условий, чтобы сузить запрос поиска для удержания. Каждое условие добавляет пункт в поисковый запрос KQL, созданный и запускающийся при создании удержания. Например, можно указать диапазон дат, чтобы приодержится электронная почта или документы сайта, созданные в диапазоне дат. Условие логически соединяется с запросом по ключевому слову (указанному в соответствующем поле) оператором AND. Это означает, что элементы должны удовлетворять как ключевому запросу, так и условию, которое должно быть отложено.

     Дополнительные сведения о создании запроса поиска и использовании условий см. в статье [Keyword querys and search conditions for Content Search.](/office365/SecurityCompliance/keyword-queries-and-search-conditions)

11. После настройки удержания на основе запросов нажмите кнопку **Далее**.

12. Просмотрите параметры и нажмите кнопку **Создать этот удержание.**

## <a name="view-hold-statistics"></a>Просмотр статистики удержания

Через некоторое время сведения о новом удержании отображаются в области сведений на вкладке **Hold** для выбранного удержания. Эта информация включает количество почтовых ящиков и сайтов в удержании, а также статистику содержимого, помещенного на удержание, например общее число и размер элементов, размещенных на удержании, и последний раз, когда вычислялась статистика удержания. Эти статистические данные о удержании помогают определить, сколько контента, связанного с делом об обнаружении электронных данных, проводится.

Имейте в виду следующие вещи о статистике удержания:

- Общее число элементов в удержании указывает количество элементов из всех источников контента, которые размещены на удержании. Если вы создали удержание на основе запроса, эта статистика указывает количество элементов, которые соответствуют запросу.
  
- В число удерживаемого количества элементов также входят неиндексы, найденные в расположениях контента. Обратите внимание, что при создании удержания на основе запросов все неиндексы в расположениях контента помещаются на удержание. Это включает неиндексы, которые не соответствуют критериям поиска для удержания на основе запроса, и неиндексуальных элементов, которые могут не соответствовать условиям диапазона дат. Это отличается от того, что происходит при запуске поиска контента, в котором неиндексуалные элементы, не совпадающий с запросом поиска или исключенные из состояния диапазона дат, не включаются в результаты поиска. Дополнительные сведения о неиндексуальных элементов см. в статьи Частично индексация элементов в [поиске контента в Office 365.](partially-indexed-items-in-content-search.md) 

- Вы можете получить последнюю статистику удержания, щелкнув статистику Обновления, чтобы повторно запустить поисковую оценку, которая вычисляет текущее количество элементов в удержании.

- При необходимости щелкните Обновить на панели инструментов, чтобы обновить статистику удержания в области сведений.

- Со временем число удерживаемых элементов обычно увеличивается, так как пользователи, чей почтовый ящик или сайт находятся на удержании, обычно отправляют или получают новое сообщение электронной почты и создают новые SharePoint и OneDrive для бизнеса документы.

- Если SharePoint или OneDrive учетную запись перемещается в другой регион в среде с несколькими геодезами, статистика для этого сайта не будет включена в статистику удержания. Однако содержимое сайта по-прежнему будет удержано. Кроме того, если сайт перемещается в другой регион, URL-адрес, отображаемый в удержании, не будет обновляться. Вам придется изменить удержание и обновить URL-адрес.

## <a name="place-a-hold-on-microsoft-teams-and-office-365-groups"></a>Поместите удержание Microsoft Teams и Office 365 групп

Microsoft Teams построены на Office 365 Groups. Поэтому их удержание в Advanced eDiscovery очень похоже.

- **Как составить дополнительную Microsoft 365 групп или Microsoft Teams для хранителя? А как насчет размещения не-хранение на Microsoft 365 групп и Microsoft Teams?** Microsoft Teams построены на Microsoft 365 Groups. Таким образом, размещение их на удержание в случае с электронным открытием очень похоже. При размещении Microsoft 365 групп и Microsoft Teams следует помнить о следующих вещах.
  - Чтобы разместить контент, расположенный в Microsoft 365 группах и Microsoft Teams, необходимо указать почтовый ящик и SharePoint сайт, связанный с группой или группой.
  
  - Запустите **командлет Get-UnifiedGroup** в Exchange Online, чтобы просмотреть свойства для Microsoft 365 или Microsoft Team. Это хороший способ получить URL-адрес сайта, связанного с группой Microsoft 365 или командой Майкрософт. Например, следующая команда отображает выбранные свойства для группы Microsoft 365 с именем "Senior Leadership Team":


    ```console
    Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
    DisplayName            : Senior Leadership Team
    Alias                  : seniorleadershipteam
    PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
    SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
    ```

    > [!NOTE]
    > Чтобы запустить командлет Get-UnifiedGroup, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

 - При поиске почтового ящика пользователя поиск Microsoft 365 group или Microsoft Team, в который входит пользователь, не будет искаться. Кроме того, при размещении Microsoft 365 или Microsoft Team на удержание помещаются только почтовый ящик группы и сайт группы; почтовые ящики и OneDrive для бизнеса сайтов участников группы не помещаются на удержание, если только вы не добавим их в качестве хранителей или не разместим их источники данных. Поэтому, если вам необходимо придержать Microsoft 365 group или Microsoft Team для конкретного хранителя, рассмотрите возможность сопоставления сайта группы и группового почтового ящика с хранителями (см. в Advanced eDiscovery). Если Microsoft 365 group или Microsoft Team не может быть отнесена к одному хранителу, рассмотрите возможность добавления источника в удержание без содержания. 
 
 - Чтобы получить список членов Microsoft 365 или Microsoft Team, вы можете просмотреть свойства на странице Home > Groups в центре администрирования Microsoft 365. Или можно выполнить следующую команду в Exchange Online PowerShell:

   ```powershell
   Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress
   ```

    > [!NOTE]
    > Чтобы запустить командлет **Get-UnifiedGroupLinks**, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

- Телефонные беседы, которые являются частью Microsoft Teams канала, хранятся в почтовом ящике, связанном с командой. Файлы, которыми обмениваются участники команды, также сохраняются на сайте команды SharePoint. Поэтому для сохранения разговоров и файлов в канале необходимо SharePoint и SharePoint веб-сайт.
  
- Кроме того, беседы, которые являются частью списка чата в Microsoft Teams, хранятся в почтовом ящике пользователя, который участвует в чате.  Файлы, которыми пользователь делится в беседах чата, хранятся OneDrive для бизнеса веб-сайте пользователя, который разделяет файл. Поэтому для сохранения разговоров и файлов в списке Чата необходимо OneDrive для бизнеса отдельных почтовых ящиков и OneDrive для бизнеса сайтов. 
  
- Каждый канал microsoft Team или team содержит Вики для записи и совместной работы. Содержимое вики-сайта автоматически сохраняется в файле формата MHT. Этот файл хранится в библиотеке документов Teams Wiki Data на сайте команды в SharePoint. Контент вики можно поставить на удержание, поместив на удержание сайт SharePoint команды.

  > [!NOTE]
  > Возможность сохранения контента вики для microsoft Team или канала группы (при удержании SharePoint сайта команды) была выпущена 22 июня 2017 г. Если сайт группы находится на удержании, содержимое Вики сохраняется начиная с этой даты. Однако если сайт группы находится в удержании и содержимое Вики было удалено до 22 июня 2017 г., содержимое Вики не было сохранено.
