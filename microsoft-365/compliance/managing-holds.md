# <a name="manage-holds-in-advanced-ediscovery"></a>Управление удерживает в advanced eDiscovery

Вы можете использовать дело advanced eDiscovery для создания содержимого, которое может иметь отношение к вашему делу. С помощью возможностей хранения расширенных электронных данных можно разместить удержания на хранителях и их источниках данных. Кроме того, можно разместить удержание без содержания в почтовых ящиках и на сайтах OneDrive для бизнеса. Вы также можете разместить удержание на групповом почтовом ящике, сайте SharePoint и сайте OneDrive для бизнеса для группы Microsoft 365. Кроме того, можно разместить удержание на почтовом ящике и сайте, связанных с Microsoft Teams. При удержании расположения контента содержимое удерживается до тех пор, пока вы не отпустите хранителя, удалите определенное расположение данных или полностью удалите политику хранения.

## <a name="manage-custodian-based-holds"></a>Управление хранением на основе хранителя

В некоторых случаях у вас может быть набор хранителей, которые вы идентифицировали и решили сохранить свои данные во время дела. В advanced eDiscovery, когда эти хранители находятся на удержании, пользователь и выбранные источники данных автоматически добавляются в политику хранения хранителей.

Чтобы просмотреть политику удержания хранителя:

1. В центре соответствия требованиям Microsoft 365 **нажмите кнопку eDiscovery > Advanced,** чтобы отобразить список случаев в организации.

2. Перейдите на **вкладку Источники,** чтобы добавить хранителей в вашем случае. Подробнее о том, как можно добавить и разместить хранителей для удержания в деле advanced eDiscovery, см. в примере [Add Custodians to a case.](add-custodians-to-case.md) Если вы уже добавили хранителей и поместили их на удержание, перейдите к шагу 3.

3. Перейдите на **вкладку Удержание** и нажмите **Кнопку CustodianHold \<HoldId>**.

4. На странице вылет можно увидеть статистику удержания для политики. Вы также можете выполнять действия, например применить запрос к удержанию на основе хранителя. Дополнительные сведения о создании запроса удержания и использовании условий см. в статье [Keyword querys and search conditions for Content Search.](keyword-queries-and-search-conditions.md)

## <a name="manage-non-custodial-holds"></a>Управление неконтлиционным хранением

При создании удержания у вас есть следующие параметры для области контента, который находится в указанных расположениях контента:

- Создается бесконечное удержание, в котором все содержимое помещается на удержание. Кроме того, можно создать удержание на основе запросов, в котором отложено только содержимое, которое соответствует запросу поиска.
  
- Вы можете указать диапазон дат для удержания только контента, отправленного, полученного или созданного в этом диапазоне дат. Кроме того, вы можете держать все содержимое независимо от того, когда оно было отправлено, получено или создано.

Чтобы создать удержание без предварительного заключения для дела advanced eDiscovery:

1. В центре соответствия требованиям Microsoft 365 **нажмите кнопку eDiscovery > Advanced,** чтобы отобразить список случаев в организации.
  
2. Нажмите **Кнопку Открыть** рядом с случаем, в который необходимо создать удерживает.
  
3. На домашней странице для дела щелкните вкладку **Держит.**
  
4. На **вкладке Держит** нажмите **кнопку Создать**.
  
5. На странице **Имя удерживаемой** страницы укайте имя удержания. Имя удержания должно быть уникальным в пределах организации.
 
6. (Необязательный) В поле **Описание** добавьте описание удержания.
  
7. Нажмите **Далее**.
  
8. Выберите расположения контента, которые необходимо разместить на удержании. Можно разместить почтовые ящики, сайты и общедоступные папки на удержание.

   1. **Электронная** почта Exchange . Щелкните **Выберите пользователей, групп** или групп, а затем нажмите кнопку Выберите **пользователей,** групп или групп снова, чтобы указать почтовые ящики для удержания. Используйте поле поиска, чтобы найти почтовые ящики пользователей и группы рассылки (чтобы разместить удержание на почтовых ящиках членов группы) для удержания. Вы также можете разместить удержание в связанном почтовом ящике для группы Microsoft 365 или microsoft Team. Выберите пользователя, группу, поле проверки группы, нажмите **кнопку Выберите**, а затем нажмите кнопку **Готово**.
 
      > [!NOTE]
      > При нажатии кнопки **Выберите пользователей, групп** или групп, чтобы указать почтовые ящики для удержания, отображаемая выборка почтовых ящиков пуста. Это вызвано мерами по повышению скорости работы. Чтобы добавить людей в этот список, введите имя (минимум 3 символа) в поле поиска.

   1. **Сайты SharePoint** - Выберите **сайты** и нажмите кнопку **Выберите** сайты снова, чтобы указать сайты SharePoint и OneDrive для бизнеса, чтобы разместить на удержание. Укажите URL-адрес каждого сайта, который вы хотите поставить на удержание. Вы также можете добавить URL-адрес сайта SharePoint для группы Microsoft 365 или microsoft Team. Нажмите **кнопку Выберите,** а затем нажмите **кнопку Готово**.
    
      Советы по приодержите Microsoft 365 Groups и Microsoft Teams, см. в разделе **FaQ.**

      > [!NOTE]
      > URL-адрес учетной записи OneDrive пользователя включает основное имя пользователя (UPN) `https://alpinehouse-my.sharepoint.com/personal/sarad_alpinehouse_onmicrosoft_com` (например). В редких случаях изменения upN человека url-адрес OneDrive также изменится, чтобы включить новый upN. Если учетная запись OneDrive пользователя является частью удержания, не удерживаемого, и его upN изменен, необходимо обновить удержание и указать на новый URL-адрес OneDrive. Дополнительные сведения см. в разделе [Как изменения UPN влияют на URL-адрес OneDrive](https://docs.microsoft.com/onedrive/upn-changes).

   1. **Общедоступные папки** Exchange — переместите переключатель переключателя на позицию Все, чтобы придержать все общедоступные папки в организации Exchange Online. Обратите внимание, что вы не можете выбрать определенные общедоступные папки для удержания. Оставьте переключатель toggle **в None,** если вы не хотите удерживать общедоступные папки.

9. После добавления расположения контента в удержание нажмите **кнопку Далее**.
  
10. Чтобы создать удержание на основе запроса с условиями, выполните следующее. В противном случае нажмите **кнопку Далее**.
    
    - В поле под **ключевыми словами** введите поисковый запрос в поле, чтобы приодержится только содержимое, отвечающий критериям поиска. Можно указать ключевые слова, свойства сообщений или свойства документов, например имена файлов. Вы также можете использовать более сложные запросы, которые используют оператора Boolean, например AND, OR или NOT. Если поле ключевого слова останется пустым, все содержимое, расположенное в указанных расположениях контента, будет помещено в удержание.

    - Нажмите  **кнопку** Добавить условия, чтобы добавить одно или несколько условий, чтобы сузить запрос поиска для удержания. Каждое условие добавляет пункт в поисковый запрос KQL, созданный и запускающийся при создании удержания. Например, можно указать диапазон дат, чтобы приодержится электронная почта или документы сайта, созданные в диапазоне дат. Условие логически соединяется с запросом по ключевому слову (указанному в соответствующем поле) оператором AND. Это означает, что элементы должны удовлетворять как ключевому запросу, так и условию, которое должно быть отложено.

     Дополнительные сведения о создании запроса поиска и использовании условий см. в статье [Keyword querys and search conditions for Content Search.](https://docs.microsoft.com/office365/SecurityCompliance/keyword-queries-and-search-conditions)

11. После настройки удержания на основе запросов нажмите кнопку **Далее**.

12. Просмотрите параметры и нажмите кнопку **Создать этот удержание.**

## <a name="view-hold-statistics"></a>Просмотр статистики удержания

Через некоторое время сведения о новом удержании отображаются в области сведений на вкладке **Hold** для выбранного удержания. Эта информация включает количество почтовых ящиков и сайтов в удержании, а также статистику содержимого, помещенного на удержание, например общее число и размер элементов, размещенных на удержании, и последний раз, когда вычислялась статистика удержания. Эти статистические данные о удержании помогают определить, сколько контента, связанного с делом об обнаружении электронных данных, проводится.

Имейте в виду следующие вещи о статистике удержания:

- Общее число элементов в удержании указывает количество элементов из всех источников контента, которые размещены на удержании. Если вы создали удержание на основе запроса, эта статистика указывает количество элементов, которые соответствуют запросу.
  
- В число удерживаемого количества элементов также входят неиндексы, найденные в расположениях контента. Обратите внимание, что при создании удержания на основе запросов все неиндексы в расположениях контента помещаются на удержание. Это включает неиндексы, которые не соответствуют критериям поиска для удержания на основе запроса, и неиндексуальных элементов, которые могут не соответствовать условиям диапазона дат. Это отличается от того, что происходит при запуске поиска контента, в котором неиндексуалные элементы, не совпадающий с запросом поиска или исключенные из состояния диапазона дат, не включаются в результаты поиска. Дополнительные сведения о неиндексуальных элементов см. в статьи Частично индексация элементов в поиске контента [в Office 365.](partially-indexed-items-in-content-search.md) 

- Вы можете получить последнюю статистику удержания, щелкнув статистику Обновления, чтобы повторно запустить поисковую оценку, которая вычисляет текущее количество элементов в удержании.

- При необходимости щелкните Обновить на панели инструментов, чтобы обновить статистику удержания в области сведений.

- Со временем число удерживаемых элементов обычно увеличивается, так как пользователи, чей почтовый ящик или сайт находятся на удержании, обычно отправляют или получают новое сообщение электронной почты и создают новые документы SharePoint и OneDrive для бизнеса.

- Если сайт SharePoint или учетная запись OneDrive перемещается в другой регион в многоэтапной среде, статистика для этого сайта не будет включена в статистику удержания. Однако содержимое сайта по-прежнему будет удержано. Кроме того, если сайт перемещается в другой регион, URL-адрес, отображаемый в удержании, не будет обновляться. Вам придется изменить удержание и обновить URL-адрес.

## <a name="place-a-hold-on-microsoft-teams-and-office-365-groups"></a>Поместите удержание на группы Microsoft Teams и Office 365

Microsoft Teams построена на Группах Office 365. Поэтому размещение их на удержание в advanced eDiscovery очень похоже. 

- **Как сооставить дополнительный сайт Microsoft 365 Groups или Microsoft Teams с хранителями? А как насчет размещения неконтлидентального удержания в Microsoft 365 Groups и Microsoft Teams?** Microsoft Teams построена на Microsoft 365 Groups. Таким образом, размещение их на удержание в случае с электронным открытием очень похоже. При размещении групп Microsoft 365 и Microsoft Teams следует помнить о следующих вещах.
  - Чтобы разместить контент, расположенный в Microsoft 365 Groups и Microsoft Teams, необходимо указать почтовый ящик и сайт SharePoint, связанные с группой или группой.
  
  - Запустите **командлет Get-UnifiedGroup** в Exchange Online, чтобы просмотреть свойства для microsoft 365 Group или Microsoft Team. Это хороший способ получить URL-адрес сайта, связанного с группой Microsoft 365 или командой Майкрософт. Например, следующая команда отображает выбранные свойства для группы Microsoft 365 с именем "Senior Leadership Team":


    ```console
    Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
    DisplayName            : Senior Leadership Team
    Alias                  : seniorleadershipteam
    PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
    SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
    ```

    > [!NOTE]
    > Чтобы запустить командлет Get-UnifiedGroup, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

 - При поиске почтового ящика пользователя не будут искаться любые группы Microsoft 365 или Microsoft Team, в которые входит пользователь. Кроме того, при размещении удержания Группы Microsoft 365 или Microsoft Team на удержание помещаются только почтовый ящик группы и сайт группы; почтовые ящики и сайты OneDrive для бизнеса участников группы не помещаются на удержание, если вы не добавим их в качестве хранителей или не разместим их источники данных. Поэтому, если вам необходимо придержать группу Microsoft 365 или Microsoft Team для конкретного хранителя, рассмотрите возможность сопоставления сайта группы и группового почтового ящика с хранителями (см. статью Управление хранителями в advanced eDiscovery). Если группа Microsoft 365 или Microsoft Team не связаны с одним хранителями, рассмотрите возможность добавления источника в удержание, не относя к нему хранение. 
 
 - Чтобы получить список членов группы Microsoft 365 или Microsoft Team, вы можете просмотреть свойства на странице Home > Groups в центре администрирования Microsoft 365. Или можно выполнить следующую команду в Exchange Online PowerShell:

   ```powershell
   Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress
   ```

    > [!NOTE]
    > Чтобы запустить командлет **Get-UnifiedGroupLinks**, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

- Диалоги каналов, которые являются частью канала Microsoft Teams, хранятся в почтовом ящике, связанном с командой. Файлы, которыми обмениваются участники команды, также сохраняются на сайте команды SharePoint. Поэтому для сохранения разговоров и файлов в канале необходимо придержать почтовый ящик Microsoft Team и сайт SharePoint.
  
- Кроме того, беседы, входящие в список чатов в Microsoft Teams, хранятся в почтовом ящике пользователя, который участвует в чате.  Файлы, которыми пользователь делится в беседах чата, хранятся на сайте OneDrive для бизнеса пользователя, который разделяет файл. Поэтому для сохранения разговоров и файлов в списке Чата необходимо придержать отдельные почтовые ящики пользователей и сайты OneDrive для бизнеса. 
  
- Каждый канал microsoft Team или team содержит Вики для записи и совместной работы. Содержимое вики-сайта автоматически сохраняется в файле формата MHT. Этот файл хранится в библиотеке документов Teams Wiki Data на сайте команды в SharePoint. Содержимое в Вики можно поместить на удержание, поставив сайт SharePoint группы на удержание.

  > [!NOTE]
  > Возможность сохранять контент вики для microsoft Team или канала группы (при удержании сайта SharePoint) была выпущена 22 июня 2017 г. Если сайт группы находится на удержании, содержимое Вики сохраняется начиная с этой даты. Однако если сайт группы находится в удержании и содержимое Вики было удалено до 22 июня 2017 г., содержимое Вики не было сохранено.
