# <a name="manage-holds-in-advanced-ediscovery"></a>Управление удерживаемой возможностью в Advanced eDiscovery

Вы можете использовать дело Advanced eDiscovery для создания сохранения содержимого, которое может быть релевантно для вашего дела. С помощью функций удержания Advanced eDiscovery можно разместить удержания для хранителей и их источников данных. Кроме того, вы можете удерживать почтовые ящики и сайты OneDrive для бизнеса, не входящие в хранение. Вы также можете на удержание разместить почтовый ящик группы, сайт SharePoint и сайт OneDrive для бизнеса для группы Microsoft 365. Аналогичным образом можно на удержание разместить почтовый ящик и сайт, связанные с Microsoft Teams. Если вы поместите расположения контента на удержание, содержимое будет удерживаться до тех пор, пока вы не освободите хранителя, не удалите определенное расположение данных или не удалите политику удержания полностью.

## <a name="manage-custodian-based-holds"></a>Управление хранением на основе хранителя

В некоторых случаях у вас может быть определенный набор хранителей, которые вы определили и решили сохранить их данные во время дела. В Advanced eDiscovery, когда эти хранители помещаются на удержание, пользователь и выбранные им источники данных автоматически добавляются в политику удержания хранителей.

Чтобы просмотреть политику удержания хранителя:

1. В Центре соответствия требованиям Microsoft 365 щелкните **службу eDiscovery > Advanced,** чтобы отобразить список дел в организации.

2. Перейдите на **вкладку "Источники",** чтобы добавить хранителей в вашем случае. Чтобы узнать, как добавить хранителей и разместить их на удержании в рамках дела Advanced eDiscovery, см. статью ["Добавление хранителей в дело"](add-custodians-to-case.md). Если вы уже добавили хранителей и поместили их на удержание, перейдите к шагу 3.

3. Перейдите на **вкладку "Удержания"** и **\<HoldId> щелкните CustodianHold.**

4. На странице вы можете увидеть статистику удержания для политики. Вы также можете выполнять такие действия, как применение запроса к удержанию на основе хранителя. Дополнительные сведения о создании запроса удержания и использовании условий см. в статье "Запросы по ключевым словам и условия поиска [контента".](keyword-queries-and-search-conditions.md)

## <a name="manage-non-custodial-holds"></a>Управление хранением, не относящаться к хранением

При создании удержания у вас есть следующие параметры для области контента, который находится в указанных расположениях контента:

- Создается бесконечное удержание, при котором все содержимое помещается на удержание. Кроме того, можно создать удержание на основе запроса, в котором на удержание помещается только контент, который соответствует поисковому запросу.
  
- Можно указать диапазон дат для удержания только содержимого, отправленного, полученного или созданного в пределах этого диапазона дат. Кроме того, можно удерживать все содержимое независимо от того, когда оно было отправлено, получено или создано.

Чтобы создать удержание, не относяцие к удержанию, для дела Advanced eDiscovery:

1. В Центре соответствия требованиям Microsoft 365 щелкните **службу eDiscovery > Advanced,** чтобы отобразить список дел в организации.
  
2. Нажмите **кнопку "Открыть"** рядом с тем делом, в который необходимо влиться.
  
3. На домашней странице дела щелкните вкладку **"Содержит".**
  
4. На **вкладке "Содержит"** нажмите кнопку **"Создать".**
  
5. На странице **"Имя удержания"** придайте ему имя. Имя удержания должно быть уникальным в пределах организации.
 
6. (Необязательно) В поле **"Описание"** добавьте описание удержания.
  
7. Нажмите **Далее**.
  
8. Выберите расположения содержимого, которые нужно разместить на удержании. Почтовые ящики, сайты и общедоступные папки можно разместить на удержании.

   1. **Электронная почта Exchange.** Нажмите кнопку "Выбрать **пользователей,** группы или команды", а затем снова выберите пользователей, группы или **команды,** чтобы указать почтовые ящики для удержания. Используйте поле поиска для поиска почтовых ящиков пользователей и групп рассылки (для применения удержания к почтовым ящикам членов группы) для применения удержания. Вы также можете надержать связанный почтовый ящик для группы Microsoft 365 или группы Microsoft Team. Выберите пользователя, группу, командный контроль, нажмите кнопку **"Выбрать"** и нажмите кнопку **"Готово".**
 
      > [!NOTE]
      > При нажатии кнопки "Выбрать **пользователей, группы** или команды" для указания почтовых ящиков, которые необходимо разместить на удержании, отображается пустое сообщение выбора почтовых ящиков. Это вызвано мерами по повышению скорости работы. Чтобы добавить людей в этот список, введите имя (не менее 3 символов) в поле поиска.

   1. **Сайты SharePoint** — щелкните  **"Выбрать** сайты", а затем снова выберите "Выбрать сайты", чтобы указать сайты SharePoint и OneDrive для бизнеса, которые необходимо разместить на удержании. Укажите URL-адрес каждого сайта, который вы хотите поставить на удержание. Вы также можете добавить URL-адрес сайта SharePoint для группы Microsoft 365 или команды Microsoft Team. Нажмите **кнопку "Выбрать",** а затем нажмите кнопку **"Готово"**.
    
      Советы **по удержанию** групп Microsoft 365 и Microsoft Teams см. в разделе "FaQ".

      > [!NOTE]
      > URL-адрес учетной записи OneDrive пользователя включает его имя-пользователя (UPN) `https://alpinehouse-my.sharepoint.com/personal/sarad_alpinehouse_onmicrosoft_com` (например). В редких случаях, когда имя upN человека меняется, его URL-адрес OneDrive также меняется для включения нового upN. Если учетная запись OneDrive пользователя является частью удержания, не относячеству к хранением, и его имя-пользователя изменено, необходимо обновить удержание и указать новый URL-адрес OneDrive. Дополнительные сведения см. в разделе [Как изменения UPN влияют на URL-адрес OneDrive](https://docs.microsoft.com/onedrive/upn-changes).

   1. **Общедоступные папки Exchange—** переместите переключатель в положение "Все", чтобы поместить все общедоступные папки в организации Exchange Online на удержание. Обратите внимание, что нельзя выбрать конкретные общедоступные папки для удержания. Если вы не хотите помещать удержание на общедоступные папки, оставьте для переключателя **"Нет".**

9. После добавления местоположений содержимого в удержание нажмите кнопку **"Далее".**
  
10. Чтобы создать удержание на основе запроса с условиями, выполните следующие условия. В противном случае просто нажмите **кнопку "Далее".**
    
    - В поле **"Ключевые** слова" введите поисковый запрос в поле, чтобы на удержание помещалась только контент, удовлетворяющий условиям поиска. Можно указать ключевые слова, свойства сообщений или свойства документа, например имена файлов. Можно также использовать более сложные запросы, которые используют boolean оператор, например AND, OR или NOT. Если оставить поле ключевого слова пустым, все содержимое, расположенное в указанных расположениях контента, будет помещено на удержание.

    - Нажмите  **кнопку** "Добавить условия", чтобы добавить одно или несколько условий, чтобы сузить поисковый запрос для удержания. Каждое условие добавляет предложение в поисковый KQL-запрос, который создается и работает при создании удержания. Например, можно указать диапазон дат, чтобы документы электронной почты или сайта, созданные в диапазоне дат, помещались на удержание. Условие логически соединяется с запросом по ключевому слову (указанному в соответствующем поле) оператором AND. Это означает, что элементы должны удовлетворять как запросу ключевого слова, так и условию, которое необходимо поместить на удержание.

     Дополнительные сведения о создании поискового запроса и использовании условий см. в статье "Запросы по ключевым словам и условия поиска [контента".](https://docs.microsoft.com/office365/SecurityCompliance/keyword-queries-and-search-conditions)

11. После настройки удержания на основе запроса нажмите кнопку **"Далее".**

12. Просмотрите параметры и нажмите кнопку **"Создать это удержание".**

## <a name="view-hold-statistics"></a>Просмотр статистики удержания

Через некоторое время сведения о новом удержании отображаются в области сведений на вкладке **"Удержания"** для выбранного удержания. Эти сведения включают количество почтовых ящиков и сайтов на удержании, а также статистику по контенту, помещенного на удержание, например общее число и размер элементов, помещенных на удержание, и время последнего вычисления статистики удержания. Эти статистические данные по удержанию помогают определить, какой контент связан с делом eDiscovery.

О статистике удержания следует иметь в виду следующее:

- Общее число элементов на удержании указывает количество элементов из всех источников контента, помещенных на удержание. Если вы создали удержание на основе запроса, эта статистика указывает количество элементов, которые соответствуют запросу.
  
- Число элементов на удержании также включает неndexed элементы, найденные в расположениях контента. Обратите внимание, что при создании удержания на основе запроса все неиndexed элементы в расположениях контента помещаются на удержание. К ним относятся неndexed элементы, которые не соответствуют условиям поиска удержания на основе запроса, и неndexed элементы, которые могут не соответствовать условию диапазона дат. Это отличается от того, что происходит при запуске поиска контента, в котором неndexed элементы, которые не соответствуют поисковому запросу или исключены условием диапазона дат, не включаются в результаты поиска. Дополнительные сведения о неиндексации элементов см. в подразделе "Частично индексация элементов в поиске контента [в Office 365".](partially-indexed-items-in-content-search.md) 

- Вы можете получить последнюю статистику удержания, щелкнув "Обновить статистику", чтобы повторно запустить оценку поиска, которая вычисляет текущее число элементов на удержании.

- При необходимости нажмите кнопку "Обновить" на панели инструментов, чтобы обновить статистику удержания в области сведений.

- Со временем число элементов на удержании увеличивается, так как пользователи, чьи почтовые ящики или сайт находятся на удержании, обычно отправляют или получают новые сообщения электронной почты, а также создают новые документы SharePoint и OneDrive для бизнеса.

- Если сайт SharePoint или учетная запись OneDrive перемещена в другой регион в среде с несколькими регионами, статистика для этого сайта не будет включена в статистику удержания. Однако контент сайта будет по-прежнему находится на удержании. Кроме того, если сайт перемещается в другую область, URL-адрес, отображаемый в удержании, не будет обновлен. Вам придется изменить удержание и обновить URL-адрес.

## <a name="place-a-hold-on-microsoft-teams-and-office-365-groups"></a>Удержание групп Microsoft Teams и Office 365

Microsoft Teams создан на основе групп Office 365. Поэтому помещение их на удержание в Advanced eDiscovery очень похоже. 

- **Как сооставить дополнительный сайт групп Microsoft 365 или Microsoft Teams с хранителями? А что делать с удержанием, не относянымся к хранителям, для групп Microsoft 365 и Microsoft Teams?** Microsoft Teams создан на основе групп Microsoft 365. Таким образом, помещение их на удержание в случае eDiscovery очень похоже. При постановке групп Microsoft 365 и Microsoft Teams на удержание следует иметь в виду следующее.
  - Чтобы разместить контент, расположенный в группах Microsoft 365 и Microsoft Teams, на удержание, необходимо указать почтовый ящик и сайт SharePoint, связанные с группой или командой.
  
  - Запустите **командлет Get-UnifiedGroup** в Exchange Online, чтобы просмотреть свойства группы Microsoft 365 или Microsoft Team. Это хороший способ получить URL-адрес сайта, связанного с группой Microsoft 365 или командой Microsoft. Например, следующая команда отображает выбранные свойства для группы Microsoft 365 с именем "Senior Leadership Team":


    ```console
    Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl
    DisplayName            : Senior Leadership Team
    Alias                  : seniorleadershipteam
    PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
    SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
    ```

    > [!NOTE]
    > Чтобы запустить командлет Get-UnifiedGroup, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

 - При поиске в почтовом ящике пользователя поиск в группах Microsoft 365 или Microsoft Team, в которые входит пользователь, не будет искаться. Аналогичным образом, при удержании группы Microsoft 365 или Microsoft Team на удержание помещаются только почтовый ящик группы и сайт группы; Почтовые ящики и сайты OneDrive для бизнеса участников группы не помещаются на удержание, если вы не добавили их в качестве хранителей или не поместили их в источники данных. Таким образом, если вам нужно разместить группу Microsoft 365 или Microsoft Team на удержание для определенного хранителя, рассмотрите возможность сопоставления сайта группы и почтового ящика группы с хранителями (см. "Управление хранителями в Advanced eDiscovery"). Если группа Microsoft 365 или Microsoft Team не связаны с одним хранителями, рассмотрите возможность добавления источника к удержанию, не относя добавим его в хранение. 
 
 - Чтобы получить список членов группы Microsoft 365 или microsoft Team, вы можете просмотреть свойства на странице "Группы домашней >" в Центре администрирования Microsoft 365. Или можно выполнить следующую команду в Exchange Online PowerShell:

   ```powershell
   Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress
   ```

    > [!NOTE]
    > Чтобы запустить командлет **Get-UnifiedGroupLinks**, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

- Беседы канала, которые являются частью канала Microsoft Teams, хранятся в почтовом ящике, связанном с командой. Файлы, которыми обмениваются участники команды, также сохраняются на сайте команды SharePoint. Поэтому для сохранения бесед и файлов в канале необходимо на удержание разместить почтовый ящик Microsoft Team и сайт SharePoint.
  
- Кроме того, беседы, входящие в список чатов в Microsoft Teams, хранятся в почтовом ящике пользователя, который участвует в чате.  Файлы, которыми пользователь делится в беседах чата, хранятся на сайте OneDrive для бизнеса пользователя, который делится файлом. Таким образом, необходимо на удержание хранить беседы и файлы в списке чатов для отдельных почтовых ящиков пользователей и сайтов OneDrive для бизнеса. 
  
- Каждый канал microsoft Team или команды содержит вики-сайт для заметок и совместной работы. Содержимое вики-сайта автоматически сохраняется в файле формата MHT. Этот файл хранится в библиотеке документов Teams Wiki Data на сайте команды в SharePoint. Вы можете поместить контент в вики-сайт на удержание, поместив сайт группы SharePoint на удержание.

  > [!NOTE]
  > Возможность сохранения вики-контента для microsoft Team или канала команды (при удержании сайта SharePoint команды) была выпущена 22 июня 2017 г. Если сайт группы находится на удержании, содержимое вики-сайта будет храниться начиная с этой даты. Однако если сайт группы находится на удержании и вики-контент был удален до 22 июня 2017 г., вики-контент не сохраняется.
