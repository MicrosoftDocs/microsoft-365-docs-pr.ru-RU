# <a name="assign-and-complete-improvement-actions-in-compliance-manager"></a>Назначение и выполнение действий по улучшению в диспетчере соответствия требованиям

**В этой статье:** В этой статье объясняется, как **управлять рабочим процессом соответствия требованиям** с помощью действий по улучшению. Узнайте, как **назначать действия по улучшению** для реализации и тестирования, **управлять обновлениями** и экспортировать **отчеты**.

## <a name="manage-compliance-workflows-with-improvement-actions"></a>Управление рабочими процессами соответствия с действиями по улучшению

Действия по улучшению централизует действия по обеспечению соответствия требованиям. Каждое действие по улучшению приводит к подробному использованию рекомендаций по обеспечению защиты данных и стандартам. Для выполнения реализации и тестирования можно назначить действия пользователям в вашей организации. Вы также можете хранить в действии документацию, заметки и обновления состояния записи.

Все действия по улучшению перечислены на странице действия по улучшению. Узнайте больше о [просмотре действий по улучшению](compliance-manager-setup.md#improvement-actions-page).

## <a name="improvement-actions-details-page"></a>Страница сведений о действиях по улучшению

В каждом действии улучшения отображается страница сведений о текущем состоянии, связанных стандартах и нормативных требованиях, а также рекомендуемых рекомендациях по реализации. [Технические действия](compliance-score-calculation.md#technical-and-non-technical-actions) включают ссылку **Запустить сейчас** , которая переводит вас в решение, необходимое для реализации. Документация по реализации и тестированию можно присоединить непосредственно к странице сведений действия по улучшению.

Чтобы просмотреть страницу сведений действия по улучшению:

1. Перейдите на страницу действий по улучшению.
2. Выберите строку предполагаемого действия по улучшению, чтобы открыть страницу со сведениями.

Вы можете легко просмотреть следующее или предыдущее действие по улучшению в списке, щелкнув стрелку вверх или вниз в правом верхнем углу экрана. Если вы фильтруете список на странице действия по улучшению, перемещение вверх или вниз позволяет перейти к следующему элементу в этом отфильтрованном списке.

## <a name="assign-improvement-actions"></a>Назначение действий по улучшению

Чтобы начать работу над действием по улучшению, вы можете выполнить работу самостоятельно или назначить его другому пользователю. Назначенный пользователь может иметь следующие значения:

- владелец бизнес-политики;
- специалист по внедрению ИТ;
- Другой сотрудник с ответственностью за выполнение задачи

Определив соответствующий уполномоченный объект, убедитесь, что они содержат достаточную [роль диспетчера соответствия требованиям](compliance-manager-setup.md#set-user-permissions-and-assign-roles) для выполнения работы. Затем выполните указанные ниже действия, чтобы назначить действие улучшения:

1. На странице "сведения о действиях по улучшению" выберите **изменить состояние** в левом верхнем углу экрана.

2. В всплывающей панели изменить состояние выберите поле **назначено** , чтобы показать список **предлагаемых** пользователей. Вы можете выбрать пользователя из списка или ввести адрес электронной почты человека, которому вы хотите назначить его.

3. Нажмите кнопку **сохранить и закрыть**. Назначенный пользователь получит электронное письмо с объяснением того, что ему назначено действие по улучшению, и прямую ссылку на действие улучшения. (Примечание: сообщество Майкрософт для государственных учреждений (GCC) высокая клиенты не получали сообщение электронной почты при назначении им действий.)

Затем назначенный пользователь может выполнить Рекомендуемые действия.

## <a name="perform-work-and-store-documentation"></a>Выполнение работы и сохранение документации

Вы можете отправлять файлы и заметки, связанные с реализацией и тестированием, непосредственно в раздел **Notes and Documentation** . Эта среда является безопасным централизованным репозиторием, который поможет продемонстрировать степень удовлетворенности элементов управления для соответствия стандартам и нормативным требованиям. Любой пользователь с доступом только для чтения может читать содержимое этого раздела. Только пользователи с правами на редактирование могут отправлять и загружать файлы, а также вводить и редактировать заметки.

Раздел **Notes and Documentation** содержит поля для отправленных документов, примечаний по внедрению, тестовых примечаний и дополнительных заметок.

#### <a name="uploaded-documents"></a>Отправленные документы

- Выберите **Управление документами** , чтобы отправить нужные файлы.
- Когда откроется всплывающая область Управление документами, выберите **Добавить документ**, а затем выберите файл из вашей системы. Допустимые типы файлов:
    - Документы (doc, XLS, PPT, txt, PDF)
    - Изображения (JPG, PNG)
    - Видео (. МКВ)
    - Сжатые файлы (ZIP, RAR)
- Когда файл разрешается в области, нажмите кнопку **Закрыть**, который автоматически сохраняет вложенный файл. После этого вы увидите файл, указанный в **передаваемых документах**.
- Чтобы скачать или удалить документ, выберите **Управление документами** в списке документов. В раскрывающейся области выберите строку документа, чтобы выделить ее, а затем выберите **загрузить** или **Удалить**.

#### <a name="implementation-notes-test-notes-and-additional-notes"></a>Примечания по реализации, тестовые заметки и дополнительные примечания

- Чтобы добавить примечания в любое из этих трех полей, выберите **изменить заметки реализации** под любым из этих полей.
- Когда откроется раскрывающаяся панель, введите примечания в текстовое поле, а затем нажмите кнопку **сохранить и закрыть**.
- Чтобы изменить заметки, выберите **изменить заметки реализации**, внесите изменения, а затем нажмите кнопку **сохранить и закрыть**.

В полях Notes нет ограниченных символов. Рекомендуется хранить заметки, чтобы их можно было легко просматривать и редактировать на странице "сведения о действиях по улучшению".

## <a name="change-improvement-action-status"></a>Изменение состояния действия по улучшению

Вы можете записать состояние и дату реализации, а также состояние и дату тестирования для каждого действия по улучшению. Поля " **Реализация** " и " **Состояние тестирования** " могут редактироваться любым пользователем с разрешениями на редактирование, а не только назначенным пользователем.

Чтобы изменить состояние действия улучшения, выберите **изменить состояние** в левом верхнем углу страницы сведений. Ниже перечислены доступные поля и параметры состояния.

- **Состояние реализации**
    - **Не реализовано** — действие еще не реализовано
    - **Реализованное** действие Реализовано
    - **Альтернативная реализация** — выберите этот вариант, если вы использовали другие сторонние средства или выполняли другие действия, не включенные в рекомендации Майкрософт
    - **Запланированное** действие запланировано для реализации
    - **Вне области** действия — действие не относится к Организации и не влияет на вашу оценку.
- **Дата реализации**: доступна для выбора, когда состояние реализации — "реализовано" или "альтернативная реализация"
- **Состояние проверки**: доступно для выбора, когда состояние реализации — "реализовано" или "альтернативная реализация":
    - **Не оценено** — действие еще не тестировалось
    - **Переданная** реализация проверена средством оценки
    - **Сбой при низком риске** — сбой тестирования, недостаток риска
    - **Сбой среднего риска** при неудачном тестировании, умеренный риск
    - **Высокий риск** : сбой тестирования, высокий риск
    - **Вне области** действия — действие не является областью для оценки и не влияет на оценку.
- **Тестовая дата**: переключитесь в меню "Календарь", чтобы выбрать дату.

Общие действия синхронизируются между группами. При двух различных оценках, выполняемых вами в одной группе, любые обновления, вносимые в сведения о реализации действия или состояние, будут автоматически синхронизированы с одним действием в любой другой оценке в группе. Эта синхронизация позволяет реализовать одно действие по улучшению и отвечать нескольким требованиям в отношении нескольких нормативных требований.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>Назначение действия по улучшению для оценки завершения

После выполнения работ, проведения тестирования и отправки свидетельства необходимо назначить действие по улучшению для проверки. Проверяющий проверяет работу и проверяет документацию, а также выбирает соответствующее состояние проверки.

**Если для состояния тестирования задано значение "передано"**: действие завершено, а достигнуто максимальное количество баллов. Затем баллы подсчитываются в общий рейтинг соответствия требованиям.

**Если для состояния тестирования задано значение "Failed" (Failed**): действие не соответствует требованиям, и может быть назначено соответствующему пользователю для дополнительной работы.

## <a name="accepting-updates-to-improvement-actions"></a>Принятие обновлений для действий по улучшению

Когда обновление становится доступным для действия по улучшению, вы увидите уведомление рядом с его именем. Вы можете либо принять обновление, либо отложить его на более позднее время.

#### <a name="what-causes-an-update"></a>Что приводит к обновлению

Обновление происходит при наличии изменений, связанных с оценками, автоматизацией или областью. Изменения могут включать в себя новые рекомендации по улучшению изменений на основе нормативных требований или из-за изменения продукта. Только действия по улучшению, управляемые вашей организацией, получают уведомления об обновлениях.

#### <a name="where-youll-see-assessment-update-notifications"></a>Где вы увидите уведомления об обновлениях оценки

После обновления действия по улучшению вы увидите метку **ожидающих обновлений** рядом с именем на странице действия по улучшению и на странице сведений о связанных с ней оценках.

Перейдите на страницу сведения действия улучшения и нажмите кнопку **проверить обновление** в верхнем заголовке, чтобы просмотреть сведения об изменениях и принять или отложить обновление.

#### <a name="review-update-to-accept-or-defer"></a>Просмотрите обновление, чтобы принять или отложить

После выбора пункта **Обзор обновления** на странице сведения о действии по улучшению в правой части экрана отображается раскрывающаяся панель. В раскрывающейся области представлены основные сведения об обновлении, такие как затронутые оценки и изменения в показателях и области.

Нажмите кнопку **принять обновление** , чтобы принять все изменения, внесенные в действие улучшения. **Допустимые изменения носят постоянный характер**.

> [!NOTE]
> Если вы принимаете обновление действия, вы также принимаете обновления для других версий или экземпляров этого действия. Обновления распространяются на все технические действия на уровне клиента и распространяются на всю группу по нетехническим действиям.

Если вы нажимаете **кнопку Отмена**, обновление не будет применено к действию улучшения. Тем не менее, вы продолжаете видеть уведомление об **ожидающем обновлении** , пока вы не примете обновление.

**Почему мы рекомендуем принять обновления**

Принятие обновлений поможет убедиться в том, что у вас есть самые обновленные рекомендации по использованию решений и какие действия по улучшению должны быть выполнены в соответствии с требованиями сертификации.

**Почему вам может потребоваться отложить обновление**

Если вы находитесь в центре выполнения оценки, включающей действие по улучшению, перед принятием обновления может потребоваться убедиться в том, что работа над ним закончена. Вы можете отложить обновление на более позднее время, нажав **кнопку Отмена** на панели всплывающих обновлений проверки.

## <a name="export-a-report"></a>Экспорт отчета

Выберите **Экспорт** в левом верхнем углу экрана, чтобы скачать лист Excel, содержащий все действия по улучшению, а также категории фильтров, показанные на странице действия по улучшению.
