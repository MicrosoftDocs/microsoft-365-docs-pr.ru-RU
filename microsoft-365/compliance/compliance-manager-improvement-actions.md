# <a name="assign-and-complete-improvement-actions-in-compliance-manager"></a>Назначение и выполнение действий по улучшению в диспетчере соответствия требованиям

**В этой статье:** В этой статье рассказывается, как управлять **процессом** соответствия требованиям с помощью действий по улучшению. Узнайте, как **назначать действия по улучшению** для реализации и тестирования, управлять **обновлениями** и **экспортировать отчеты.**

## <a name="manage-compliance-workflows-with-improvement-actions"></a>Управление процессами соответствия требованиям с помощью действий по улучшению

Действия по улучшению централизуют действия по обеспечению соответствия требованиям. Каждое действие по улучшению дает подробные инструкции по реализации, которые помогут вам согласовать правила и стандарты защиты данных. Действия могут быть назначены пользователям в вашей организации для выполнения работы по реализации и тестированию. Вы также можете хранить документацию, заметки и записывать обновления состояния в действии.

Все действия по улучшению перечислены на странице действия по улучшению. Узнайте больше о [просмотре действий по улучшению.](compliance-manager-setup.md#improvement-actions-page)

## <a name="improvement-actions-details-page"></a>Страница Сведения о действиях по улучшению

Каждое действие по улучшению имеет страницу сведений, показывающая его текущее состояние, соответствующие стандарты и нормативные требования, а также рекомендуемые рекомендации по реализации. [Технические действия](compliance-score-calculation.md#technical-and-non-technical-actions) включают **ссылку Launch now,** которая принимает вас к соответствующему решению для реализации. Вы можете прикрепить документацию по реализации и тестированию непосредственно на страницу сведений о действии по улучшению.

Чтобы просмотреть страницу сведений о действии по улучшению:

1. Перейдите на страницу действия по улучшению.
2. Выберите строку действия по улучшению, которое откроет страницу сведений.

Вы можете легко просмотреть следующее или предыдущее действие по улучшению в списке, выбрав стрелку вверх или вниз в правом верхнем углу экрана. Если список фильтруется на странице действия по улучшению, перемещение вверх или вниз переводится к следующему элементу в этом фильтрованом списке.

## <a name="assign-improvement-actions"></a>Назначение действий по улучшению

Чтобы приступить к выполнению действия по улучшению, вы можете сделать работу самостоятельно или назначить ее другому пользователю. Назначенное лицо может быть:

- владелец бизнес-политики;
- специалист по внедрению ИТ;
- Другой сотрудник, на который возложена ответственность за выполнение задачи

После определения соответствующего назначаемой должности [](compliance-manager-setup.md#set-user-permissions-and-assign-roles) убедитесь, что он выполняет достаточную роль диспетчера соответствия требованиям для выполнения работы. Затем выполните ниже шаги, чтобы назначить действие по улучшению:

1. На странице сведения о действиях по улучшению выберите **состояние Редактирование** в левом верхнем разделе экрана.

2. В области флажок состояния редактирования выберите **назначенное** поле, чтобы показать список пользователей **"Предложенные** люди". Вы можете выбрать пользователя из списка или ввести адрес электронной почты человека, которого вы хотите назначить.

3. Выберите **Сохранить и закрыть**. Назначенное пользователю будет отправлено письмо с объяснением того, что ему назначено действие по улучшению, с прямой ссылкой на действие по улучшению. 
> [!NOTE]
> Клиенты государственного сообщества США (GCC) и Министерства обороны (DoD) не получат электронной почты при назначении им действий по улучшению.

После этого назначенный пользователь может выполнять рекомендуемые действия.

#### <a name="assign-multiple-improvement-actions-to-a-single-user"></a>Назначение нескольких действий по улучшению одному пользователю

Вы можете назначить несколько действий по улучшению одному пользователю, следуя следующим шагам:

1. Перейдите на страницу Действия по улучшению.
2. Выберите область слева от имени действия улучшения. Появится значок круглой проверки, указывающий, что вы выбрали это действие. Проверьте все действия, которые необходимо назначить.
3. Выберите **ссылку Назначение пользователю** в верхней части таблицы действий по улучшению.
4. Открывается всплывающее окно. В **поле Назначение** для начала введите имя человека, которого нужно назначить действия. Вы также можете выбрать из списка предложенных людей.
5. После заполнения поля **Назначение в** поле с именем назначит назначение. 
6. Затем вы увидите страницу действий по улучшению с помощью нового назначенного вами назначенного.

## <a name="perform-work-and-store-documentation"></a>Выполнение документации по работе и храню

Файлы и заметки, связанные с реализацией и тестированием, можно загружать непосредственно в раздел **Заметки и документацию.** Эта среда является безопасным, централизованным репозиторием, который поможет вам продемонстрировать соответствие элементов управления требованиям и требованиям. Любой пользователь с доступом только для чтения может читать контент в этом разделе. Только пользователи с правами на редактирование могут загружать и скачивать файлы и вводить или изменять заметки.

Раздел **Заметки и документация** содержит поля для загруженных документов, примечаний к реализации, тестовых заметок и дополнительных заметок.

#### <a name="uploaded-documents"></a>Загруженные документы

- Выберите **Управление документами для** отправки соответствующих файлов.
- Когда откроется флайер управления документами, выберите **добавить** документ, а затем выберите файл из системы. Типы принятых файлов:
    - Документы (.doc, .xls, .ppt, .txt, .pdf)
    - Изображения (.jpg, .png)
    - Видео (.mkv)
    - Сжатые файлы (.zip, .rar)
- После разрешения файла в области выберите **Закрыть,** что автоматически сохраняет вложение файла. Затем вы увидите файл, указанный под **загруженными документами.**
- Чтобы загрузить или удалить документ, выберите **Управление** документами из списка документов. На области вылетов выберите строку документа, чтобы выделить ее, а затем выберите **Скачать** или **Удалить**.

#### <a name="implementation-notes-test-notes-and-additional-notes"></a>Заметки о реализации, заметки для тестирования и дополнительные заметки

- Чтобы добавить заметки в любом из этих трех полей, выберите **заметки об редактировании** реализации под любым из этих полей.
- Когда откроется область вылетов, введите заметки в текстовом поле, а затем выберите **Сохранить и закрыть**.
- Чтобы изменить заметки, **выберите заметки по** реализации, внести изменения, а затем **сохранить и закрыть**.

В полях примечаний нет ограничения символов. Рекомендуется сохранять заметки краткими, чтобы их можно было легко просматривать и редактировать на странице сведения о действиях по улучшению.

## <a name="change-improvement-action-status"></a>Изменение состояния действия по улучшению

Для каждого действия по улучшению можно записывать состояние и дату реализации, а также состояние и дату тестирования. Поля **реализации** **и состояния** тестирования могут быть изменены любым пользователем с разрешениями на редактирование, а не только назначенным пользователем.

Чтобы изменить состояние действия по улучшению, выберите **состояние Изменить** в левом верхнем разделе страницы сведений. Ниже приведены доступные поля и параметры состояния:

- **Состояние реализации**
    - **Не реализовано** — действие еще не реализовано
    - **Реализовано** — реализовано действие
    - **Альтернативная реализация** — выберите этот параметр, если вы использовали другие сторонние средства или приняли другие действия, не включенные в рекомендации Майкрософт.
    - **Запланированные** действия для реализации
    - **Вне области** действия не имеют отношения к организации и не способствуют оценке
- **Дата реализации:** можно выбрать, когда состояние реализации "реализовано" или "альтернативная реализация"
- **Состояние тестирования:** доступно для выбора при "реализации" или "альтернативной реализации":
    - **Не оценено** — действие не было протестировано
    - **Пройдено** — реализация была проверена оценчиком
    - **Неудачный низкий риск** — тестирование не удалось, низкий риск
    - **Неудачный средний риск** — сбой тестирования, средний риск
    - **Сбой с высоким риском** — сбой тестирования, высокий риск
    - **Вне сферы действия** для оценки и не внося свой вклад в оценку
- **Дата тестирования:** переопробовка в всплывающее всплывающее в календаре для выбора даты

Общие действия синхронизируются между группами. Если две различные оценки в одной группе разделяют действия по улучшению, которыми вы управляете, любые обновления, которые вы делаете для деталей или состояния реализации действия, будут автоматически синхронизироваться с тем же действием в любой другой оценке в группе. Эта синхронизация позволяет реализовать одно действие по улучшению и выполнить несколько требований по нескольким правилам.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>Назначение действий по улучшению оценчику для завершения

После завершения работы выполните тестирование и загрузите доказательства, а затем назначьте действие по улучшению оценчику для проверки. Оценчик проверяет работу и проверяет документацию и выбирает соответствующий тестовый статус.

**Если состояние теста установлено как "Пройдено":** действие завершено, а достигнутые очки показывают максимальное количество достигнутых баллов. Затем очки засчитываются в общую оценку соответствия требованиям.

Если состояние теста установлено **как "Failed":** действие не соответствует требованиям, и оценчик может назначить его соответствующему пользователю для дополнительной работы.

## <a name="accepting-updates-to-improvement-actions"></a>Принятие обновлений к действиям по улучшению

Когда обновление доступно для действия по улучшению, рядом с его именем будет видно уведомление. Вы можете принять обновление или отложить его на более позднее время.

#### <a name="what-causes-an-update"></a>Причины обновления

Обновление происходит при изменениях, связанных с скорингом, автоматизацией или областью. Изменения могут включать новые рекомендации по действиям по улучшению, основанные на нормативных изменениях, или могут быть из-за изменений продукта. Уведомления об обновлении получают только действия по улучшению, управляемые организациями.

#### <a name="where-youll-see-assessment-update-notifications"></a>Где вы увидите уведомления об обновлении оценки

При обновлении действия по улучшению рядом  со своим именем на странице действия по улучшению и на странице сведений о связанных с ним оценках вы увидите метку "Ожидание обновления".

Перейдите на страницу сведений о  действии по улучшению и выберите кнопку Обновление Обзор в верхнем баннере, чтобы просмотреть сведения об изменениях и принять или отложить обновление.

#### <a name="review-update-to-accept-or-defer"></a>Просмотр обновления для принятие или отсрочка

После выбора обновления **"Обзор"** на странице сведения об улучшении на правой стороне экрана отображается поле для вылетов. В области вылетов ключевых сведений об обновлении, таких как оценки влияние и изменения в оценках и области.

Выберите **Принять обновление,** чтобы принять все изменения в действии по улучшению. **Принятые изменения являются постоянными.**

> [!NOTE]
> Когда вы принимаете обновление к действию, вы также принимаете обновления для любых других версий или экземпляров этого действия. Обновления будут распространяться на всех клиентах для технических действий и распространяться по всей группе для не-технических действий.

Если вы выберите **Отмена,** обновление не будет применено к действию по улучшению. Тем не менее, вы будете продолжать видеть уведомление об ожидающих **обновлениях,** пока не примете обновление.

**Почему рекомендуется принимать обновления**

Принятие обновлений поможет вам получить самые обновленные рекомендации по использованию решений и принятию соответствующих действий по улучшению, чтобы соответствовать требованиям сертификации.

**Почему может потребоваться отложить обновление**

Если вы в середине завершения оценки, которая включает действие по улучшению, вы можете убедиться, что вы закончили работу над ним, прежде чем принять обновление. Вы можете отложить обновление на более позднее время, выбрав **отмену** в области флааутов обновления обзоров.

#### <a name="accept-all-updates-at-once"></a>Принимать все обновления одновременно

Если у вас есть несколько обновлений и вы хотите  принять их все одновременно, выберите ссылку Принять все обновления в верхней части таблицы действий по улучшению. Появится поле flyout, в котором перечислены количество действий, которые необходимо обновить. Выберите **кнопку Принять обновления,** чтобы применить все обновления.

Обратите внимание, что при возвращении на страницу действия по улучшению в верхней части страницы может появиться сообщение с просьбой обновить страницу для завершения обновлений.

## <a name="export-a-report"></a>Экспорт отчета

Выберите **Экспорт** в верхнем левом углу экрана, чтобы скачать лист Excel, содержащий все действия по улучшению, и категории фильтра, показанные на странице действия улучшения.
