# <a name="assign-and-complete-improvement-actions-in-compliance-manager"></a>Назначение и выполнение действий по улучшению в диспетчере соответствия требованиям

**В этой статье:** В этой статье объясняется, как управлять **процессом** обеспечения соответствия требованиям с помощью действий по улучшению. Узнайте, как **назначать действия по улучшению** для реализации и тестирования, **управления обновлениями** и экспорта **отчетов.**

## <a name="manage-compliance-workflows-with-improvement-actions"></a>Управление процессами соответствия требованиям с помощью действий по улучшению

Действия по улучшению централизуют действия по обеспечению соответствия требованиям. Каждое действие по улучшению предоставляет подробные рекомендации по реализации, которые помогут вам согласовать нормы и стандарты защиты данных. Пользователям в организации могут быть назначены действия по реализации и тестированию. Вы также можете хранить документацию, заметки и записывать обновления состояния в рамках действия.

Все действия по улучшению перечислены на странице действий по улучшению. Узнайте больше о [просмотре действий по улучшению.](compliance-manager-setup.md#improvement-actions-page)

## <a name="improvement-actions-details-page"></a>Страница сведений о действиях по улучшению

Каждое действие по улучшению имеет страницу со сведениями о текущем состоянии, связанных стандартах и нормативных требованиях, а также рекомендации по реализации. [Технические](compliance-score-calculation.md#technical-and-non-technical-actions) действия включают **ссылку "Запуск"** для реализации соответствующего решения. Документацию по реализации и тестированию можно присоединить непосредственно на страницу сведений о действии по улучшению.

Чтобы просмотреть страницу сведений о действии по улучшению:

1. Перейдите на страницу действий по улучшению.
2. Выберите строку вашего действия по улучшению, которая открывает страницу сведений.

Вы можете легко просмотреть следующее или предыдущее действие улучшения в списке, выбрав стрелку вверх или вниз в правом верхнем углу экрана. Если вы отфильтровали список на странице действий по улучшению, перемещение вверх или вниз перемещает вас к следующему элементу этого отфильтрованного списка.

## <a name="assign-improvement-actions"></a>Назначение действий по улучшению

Чтобы начать работу по реализации действия по улучшению, вы можете самостоятельно сделать это или назначить его другому пользователю. Назначенное лицо может быть:

- владелец бизнес-политики;
- специалист по внедрению ИТ;
- Другой сотрудник, который отвечает за выполнение задачи

После того как вы определите соответствующего руководителя, убедитесь, что он имеет достаточную роль диспетчера соответствия требованиям [для](compliance-manager-setup.md#set-user-permissions-and-assign-roles) выполнения работы. Затем выполните следующие действия, чтобы назначить действие по улучшению:

1. На странице сведений о действиях по улучшению выберите "Изменить **состояние"** в левом верхнем разделе экрана.

2. В области изменения состояния выберите поле  "Назначено",  чтобы отобрать список пользователей "Предложенные люди". Вы можете выбрать пользователя в списке или ввести адрес электронной почты пользователя, которого вы хотите назначить.

3. Выберите **"Сохранить и закрыть".** Назначенное пользователю будет отправлено сообщение электронной почты с сообщением о том, что ему назначено действие по улучшению, с прямой ссылкой на действие по улучшению. (Примечание. Пользователи сообщества государственных организаций США (GCC) с высоким уровнем качества не получат сообщение электронной почты при назначении им действий.)

После этого соответствующий пользователь может выполнить рекомендуемые действия.

#### <a name="assign-multiple-improvement-actions-to-a-single-user"></a>Назначение нескольких действий по улучшению одному пользователю

Чтобы назначить несколько действий по улучшению одному пользователю, с помощью следующих действий:

1. Перейдите на страницу действий улучшения.
2. Выберите область слева от имени действия улучшения. Появится значок круговой проверки, указывающий, что вы выбрали это действие. Проверьте все действия, которые необходимо назначить.
3. Выберите **ссылку "Назначить пользователю"** в верхней части таблицы действий по улучшению.
4. Открывается всплывающее окно. В поле **"Назначить"** начните вводить имя человека, для которого нужно назначить действия. Вы также можете выбрать из списка предложенных людей.
5. После заполнения поля **"Назначить в** поле" с именем assignee выберите **"Назначить".**
6. После этого вы увидите страницу действий по улучшению с новыми назначенными действиями.

## <a name="perform-work-and-store-documentation"></a>Выполнение документации по работе и храну

Вы можете отправить файлы и заметки, связанные с реализацией и тестированием, непосредственно в раздел **"Заметки" и "Документация".** Эта среда является безопасным, централизованным репозиторием, который поможет вам продемонстрировать удовлетворенность средств контроля в соответствии со стандартами и нормативными актами. Любой пользователь с доступом только для чтения может читать содержимое в этом разделе. Только пользователи с правами на редактирование могут загружать и загружать файлы, а также вводить или редактировать заметки.

Раздел **"Примечания** и документация" содержит поля для загруженных документов, заметок о реализации, тестовых примечаний и дополнительных заметок.

#### <a name="uploaded-documents"></a>Загруженные документы

- Выберите **"Управление документами",** чтобы отправить все необходимые файлы.
- Когда откроется окно управления документами, выберите "Добавить **документ"** и выберите файл в системе. Типы файлов:
    - Документы (DOC, XLS, PPT, TXT, PDF)
    - Изображения (JPG, PNG)
    - Видео (MKV)
    - Сжатые файлы (ZIP, RAR)
- После разрешения файла в области выберите **"Закрыть",** чтобы автоматически сохранить вложение. После этого вы увидите файл, указанный под **загруженными документами.**
- Чтобы скачать или удалить документ, выберите **"Управление** документами" в списке документов. На flyout pane, select the document row to highlight it, then select **Download** or **Delete**.

#### <a name="implementation-notes-test-notes-and-additional-notes"></a>Примечания по реализации, тестовые заметки и дополнительные примечания

- Чтобы добавить примечания в любое из этих трех полей, выберите "Изменить примечания **по** реализации" под любым из этих полей.
- Когда откроется область, введите заметки в текстовом поле, а затем выберите "Сохранить **и закрыть".**
- Чтобы редактировать заметки, выберите **"Изменить примечания по** реализации", "Внести изменения" и "Сохранить **и закрыть".**

В полях заметок нет ограничений по символам. Мы рекомендуем сохранять краткие заметки, чтобы их можно было легко просматривать и редактировать на странице сведений о действиях по улучшению.

## <a name="change-improvement-action-status"></a>Изменение состояния действия по улучшению

Для каждого действия по улучшению можно записать состояние и дату реализации, а также состояние и дату тестирования. Поля  состояния **реализации** и тестирования может изменять любой пользователь с разрешениями на редактирование, а не только назначенное лицо.

Чтобы изменить состояние действия улучшения, выберите **"Изменить** состояние" в левом верхнем разделе страницы сведений. Ниже приведены доступные поля и параметры состояния:

- **Состояние реализации**
    - **Не реализовано** — действие еще не реализовано
    - **Implemented** - action implemented
    - **Альтернативная реализация** — выберите этот параметр, если вы использовали другие сторонние средства или приняли другие действия, не включенные в рекомендации Майкрософт
    - **Запланировано —** действие запланировано для реализации
    - **Вне области** действия — действие не относится к вашей организации и не вносит свой вклад в оценку
- **Дата реализации:** доступно для выбора, когда состояние реализации "реализовано" или "альтернативная реализация"
- **Состояние тестирования:** доступно для выбора, когда состояние реализации "реализовано" или "альтернативная реализация":
    - **Не оценено** — действие не было протестировано
    - **Пройдено** — реализация проверена оценчиком
    - **Сбой с низким риском** — сбой тестирования, низкий риск
    - **Сбой среднего риска** — сбой тестирования, средний риск
    - **Сбой с высоким уровнем риска** — сбой тестирования, высокий риск
    - **Вне области действия** — действие выходит за рамки оценки и не вносит свой вклад в оценку
- **Тестовая дата:** перелистывка всплывающее окна календаря для выбора даты

Распространенные действия синхронизируются между группами. Если две разные оценки в одной и той же группе совместно делают действия по улучшению, управляемые вами, любые обновления сведений или состояния реализации действия автоматически синхронизируются с тем же действием в любой другой оценке в группе. Такая синхронизация позволяет реализовать одно действие по улучшению и соответствовать нескольким требованиям в нескольких нормативных актах.

## <a name="assign-improvement-action-to-assessor-for-completion"></a>Назначение оценителем действия по улучшению для завершения

После завершения работы, проведения тестирования и отправки свидетельств необходимо назначить оценчику действие по улучшению для проверки. Оценчик проверяет работу, проверяет документацию и выбирает соответствующее состояние тестирования.

**Если для тестового состояния установлено значение "Passed":** действие завершено, а набраные точки показывают максимальные точки. Затем эти баллы учитываются в общем рейтинге соответствия требованиям.

**Если для тестового** состояния установлено состояние "Сбой": действие не соответствует требованиям, и оценчик может назначить его соответствующему пользователю для дополнительной работы.

## <a name="accepting-updates-to-improvement-actions"></a>Принятие обновлений для действий по улучшению

Если обновление доступно для действия по улучшению, рядом с его именем вы увидите уведомление. Вы можете принять обновление или отложить его на более позднее время.

#### <a name="what-causes-an-update"></a>Причины обновления

Обновление происходит при внесении изменений, связанных с озвучкой, автоматизацией или областью. Изменения могут включать новое руководство по действиям по улучшению на основе изменений нормативных органов или могут быть связаны с изменениями продукта. Уведомления об обновлении получают только действия по улучшению, управляемые организациями.

#### <a name="where-youll-see-assessment-update-notifications"></a>Где вы увидите уведомления об обновлении оценки

При обновлении действия по улучшению рядом  с его именем на странице действий по улучшению и на странице сведений о связанных с ним оценках будет отложена метка обновления.

Перейдите на страницу сведений о  действии по улучшению и выберите кнопку "Обзор обновления" в верхнем баннере, чтобы просмотреть сведения об изменениях и принять или отложить обновление.

#### <a name="review-update-to-accept-or-defer"></a>Проверка обновления на принятие или откладывание

После выбора обновления **"Проверка"** на странице сведений о действии по улучшению в правой части экрана появится экранная страница. На этой области ключевых сведений об обновлении, таких как оценки и изменения показателей и области применения.

Выберите **"Принять обновление",** чтобы принять все изменения в действии по улучшению. **Принятые изменения являются постоянными.**

> [!NOTE]
> Когда вы принимаете обновление действия, вы также принимаете обновления для любых других версий или экземпляров этого действия. Обновления распространяются на все клиенты для технических действий и распространяются на всей группе для не-технических действий.

Если выбрана **отмена,** обновление не будет применено к действию по улучшению. Тем не менее вы будете продолжать видеть уведомление об ожидающих **обновлениях,** пока не примете обновление.

**Почему мы рекомендуем принимать обновления**

Принятие обновлений помогает вам получить самые обновленные рекомендации по использованию решений и принятию соответствующих действий по улучшению для удовлетворения требований сертификации.

**Почему может потребоваться отложить обновление**

Если вы уже завершили оценку, включаемую действие по улучшению, перед принятием обновления может потребоваться убедиться, что вы завершили работу над ним. Вы можете отложить обновление на более позднее время, выбрав "Отмена" во flyout области обновления отзыва. 

#### <a name="accept-all-updates-at-once"></a>Принятие всех обновлений одновременно

Если у вас есть несколько обновлений и вы хотите  принять их все одновременно, выберите ссылку "Принять все обновления" в верхней части таблицы действий по улучшению. Появится выдавная области, в которой перечислены действия, которые необходимо обновить. Выберите **кнопку "Принять обновления",** чтобы применить все обновления.

Обратите внимание, что при возвращении на страницу действий по улучшению в верхней части страницы может появиться сообщение с просьбой обновить страницу для завершения обновлений.

## <a name="export-a-report"></a>Экспорт отчета

Выберите **"Экспорт"** в левом верхнем углу экрана, чтобы скачать лист Excel, содержащий все действия по улучшению, и категории фильтров, показанные на странице действий по улучшению.
