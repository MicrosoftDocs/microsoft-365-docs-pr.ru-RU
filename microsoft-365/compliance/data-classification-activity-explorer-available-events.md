# <a name="labeling-activities-that-are-available-in-activity-explorer"></a>Действия по маркировке, доступные в проводнике действий

## <a name="sensitivity-label-applied"></a>Метка "Чувствительность", примененная

Это событие создается каждый раз, когда документ без маркировки или отправка электронной почты с меткой. 

- Оно фиксируется во время сохранения в приложениях и веб-приложениях Office. 
- Она фиксируется во время появления в надстройки для защиты информации Azure. 
- Действия по обновлению и понижению меток также можно отслеживать с помощью поля типа событий *Label* и фильтра.   


|Source  |Сообщается в проводнике действий | Примечание  |
|---------|---------|---------|
| Word, Excel, PowerPoint|да |
|Outlook| да |из Win 32 |
|SharePoint online, OneDrive|да | |
|Exchange        |да         | |
|Унифицированный клиент Azure Information Protection (AIP) и единый сканер AIP |да |действие новой *метки* AIP сопоположено с *меткой, примененной* в проводнике действий   |
|Microsoft information protection (MIP) SDK         |да|действие новой *метки* AIP сопоположено с *меткой, примененной* в проводнике действий|
|Служба управления правами (RMS)         |неприменимо         | |
|Power BI desktop and web        | нет| доступны в журналах аудита Microsoft 365         |
|Microsoft Cloud App Security (MCAS)         |нет|         |

## <a name="sensitivity-label-changed"></a>Метка "Чувствительность" изменена

Это событие создается каждый раз, когда метка обновляется в документе или электронной почте.

- Для единого клиента AIP, источников единого сканера и SDK  *MIP* метка обновления AIP и карты действий метки понижения до метки обозревателя *действий изменены.*

- Он фиксируется в точке сохранения в приложениях Office и веб-приложениях. 
- Он фиксируется во время появления в Azure Information Protection унифицированных клиентских надстройок и правоохранительных органов сканера.
- Действия по обновлению и понижению меток также можно отслеживать с помощью поля типа событий *Label* и фильтра. Текст *обоснования* также захвачен, за исключением SharePoint Online и OneDrive.
- Метка конфиденциальности, сделанная в родных приложениях Office в Outlook, собирает последнее действие, которое было сгенерировано до сохранения файлов или действий отправки электронной почты. Например, если перед отправкой пользователь несколько раз меняет метку на электронной почте, последняя метка, найденная в электронной почте при отправке, запечатлена в журнале аудита и затем сообщается в проводнике действий. 


|Source  |Сообщается в проводнике действий|Примечание  |
|---------|---------|---------| 
|Word, Excel, PowerPoint         |да         |
|Outlook         |да         |Win 32|
|SharePoint Online, OneDrive         |да         |
|Exchange         |да         |
|Единый клиент AIP         |да         |
|Единый сканер AIP         |да         |
|MIP SDK         |да         |

## <a name="known-issues"></a>Известные проблемы
  
- Когда рекомендуемый совет средства метки отображается конечному пользователю, он не захватывается. Но если пользователь решил применить рекомендуемую метку, метка будет показана в поле *How applied* as *Recommended*  

- Текст обоснования в настоящее время не доступен для понижения метки конфиденциальности с Sharepoint и OneDrive.  

- В настоящее время типы конфиденциальной информации недоступны для действий автолабелинга из Word, Excel, PowerPoint и Outlook, а также SharePoint Online и OneDrive.
