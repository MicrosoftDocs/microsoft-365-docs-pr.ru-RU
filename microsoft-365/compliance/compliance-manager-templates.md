# <a name="working-with-assessment-templates-in-compliance-manager"></a>Работа с шаблонами оценки в диспетчере соответствия требованиям

**В этой статье:** Узнайте, **как работают шаблоны** и **как управлять** ими на странице "шаблоны оценки". Инструкции по **созданию** новых шаблонов, **изменению** существующих шаблонов, **форматированию данных шаблона с помощью Excel** и экспорту **отчетов о** шаблонах.

> [!IMPORTANT]
> Шаблоны оценки, доступные для Организации, зависят от лицензионного соглашения. [Просмотрите подробные сведения](https://go.microsoft.com/fwlink/?linkid=2132371).

## <a name="templates-overview"></a>Общие сведения о шаблонах

Шаблон — это платформа для создания оценки в диспетчере соответствия требованиям. Они содержат элементы управления для проведения требований сертификации с использованием определенного продукта. Диспетчер соответствия требованиям предоставляет исчерпывающий набор шаблонов, позволяющий организациям соответствовать национальным, региональным и отраслевым требованиям, управляющим сбором и использованием данных.

## <a name="list-of-pre-built-templates-for-assessments"></a>Список предварительно созданных шаблонов для оценок

Диспетчер соответствия требованиям предоставляет шаблоны для создания оценок, которые помогут обеспечить соблюдение различных нормативных требований и стандартов. Просмотр [списка шаблонов](compliance-manager-templates-list.md) , предоставляемых диспетчером соответствия требованиям. Новые шаблоны регулярно добавляются, поэтому проверьте список часто.

## <a name="viewing-and-managing-templates-from-the-assessment-templates-page"></a>Просмотр шаблонов на странице "шаблоны оценки" и управление ими

На странице "шаблоны оценки" в диспетчере соответствия требованиям отображается список шаблонов и ключевых сведений. Список включает шаблоны, предоставляемые диспетчером соответствия требованиям, а также шаблоны, которые были изменены или созданы в Организации. Фильтры можно применять для поиска шаблонов на основе сертификации, области продукта, страны, отрасли, создавшей ее, а также от того, включен ли шаблон для создания оценки.

Выберите шаблон из строки, чтобы отобразить его страницу сведений. На этой странице представлено описание шаблона и дополнительные сведения о сертификации, области и элементах управления. На этой странице можно выбрать соответствующие кнопки для создания оценки, экспорта данных шаблона в Excel или изменения шаблона.

## <a name="creating-and-modifying-templates-overview"></a>Общие сведения о создании и изменении шаблонов

Чтобы изменить существующий шаблон или создать новый шаблон, используйте специально отформатированную электронную таблицу Excel ([например, скачать пример](https://go.microsoft.com/fwlink/?linkid=2124865)), чтобы собрать необходимые данные элемента управления. После заполнения электронной таблицы импортируйте ее в Диспетчер соответствия требованиям в процессе создания или изменения шаблона.

> [!NOTE]
> Электронная таблица имеет определенный формат и схему, которые необходимо использовать, или они не будут правильно импортироваться в Диспетчер соответствия требованиям. Ниже приведены [инструкции по форматированию](#formatting-your-template-data-with-excel) .

**Обязательные роли**

Создавать и изменять шаблоны могут только пользователи с административной ролью глобального администратора или диспетчера соответствия требованиям. Дополнительные сведения о [ролях и разрешениях](compliance-manager-setup.md#set-user-permissions-and-assign-roles).

## <a name="create-a-new-template"></a>Создание нового шаблона

Чтобы создать новый шаблон (используемый для создания настраиваемых оценок), выполните указанные ниже действия.

1. Перейдите на страницу " **шаблоны оценки** " в диспетчере соответствия требованиям.
2. Выберите **создать новый шаблон** . Откроется мастер создания шаблонов.
3. Выберите тип шаблона, который вы хотите создать. В этом случае выберите **создать настраиваемый шаблон** , а затем нажмите кнопку **Далее** .
4. На экране **отправить файл** нажмите кнопку **Обзор** , чтобы найти и отправить отформатированный файл Excel со всеми необходимыми данными шаблона (см. [инструкции для правильного форматирования файла](#formatting-your-template-data-with-excel)).
5. Если в файле нет проблем, будет отображено имя отправленного файла. Нажмите кнопку **Далее** , чтобы продолжить. (Если необходимо изменить файл, выберите **Отправить другой файл** ).
    - Если при работе с файлом возникла ошибка, сообщение об ошибке вверху содержит сведения о том, что не так. Вам потребуется исправить файл и отправить его повторно. При неправильном форматировании электронной таблицы или при наличии недопустимых данных в определенных полях могут возникать ошибки (снова обратиться к [инструкциям по форматированию](#formatting-your-template-data-with-excel)).  
    
6. На экране **"Просмотр и завершение"** отображается количество действий и элементов управления, а также максимальный показатель для шаблона. Когда все будет готово к утверждению, выберите **создать шаблон.** (Если необходимо внести изменения, нажмите кнопку **назад** ).
7. Последний экран подтверждает, что был создан новый шаблон. Нажмите кнопку **Готово** , чтобы завершить работу мастера.
8. Вы будете приступать к странице сведений о новом шаблоне, где вы можете [создать свою оценку](compliance-manager-assessments.md#create-your-own-custom-assessment).

## <a name="formatting-your-template-data-with-excel"></a>Форматирование данных шаблона с помощью Excel

Таблица Excel, используемая для создания шаблонов, содержит четыре вкладки, три из которых являются обязательными:

1. [Шаблон](#template-tab) (обязательный)
2. [Контролфамили](#controlfamily-tab) (обязательный)
3. [Действия](#actions-tab) (обязательные)
4. [Измерения](#dimensions-tab) (необязательно)

При заполнении электронной таблицы данными шаблона она  **должна включать вкладки в** указанном порядке, в противном случае данные не будут успешно импортированы в шаблон.

##### <a name="template-tab"></a>Вкладка "шаблон"

Вкладка " **шаблон** " является обязательной. Информация на этой вкладке содержит метаданные о шаблоне. Имеется четыре обязательных столбца. Столбцы должны храниться в таблице Excel, как показано ниже. Вы можете добавить свой собственный столбец **после** четырех столбцов для предоставления собственных измерений. В этом случае не забудьте добавить их на вкладку **измерения** с помощью приведенных [ниже инструкций](#dimensions-tab).

- **название** : это название шаблона, которое должно быть уникальным. Имя не может быть предоставлено другому шаблону, который находится в диспетчере соответствия требованиям, включая собственные шаблоны или шаблон диспетчера соответствия требованиям.

- **продукт** : это обязательное измерение. Перечислите продукты, связанные с шаблоном.

- **сертификация** : это нормативное значение, которое вы используете для шаблона.

- **инскопесервицес** : эти службы в рамках продукта, на которые распространяется этот оценочный адрес (например, если в качестве продукта указан Office 365, Microsoft Teams может быть службой в пределах области). Можно перечислить несколько служб, разделенных двумя точками с запятой.

> [!NOTE]
> Данные, вставленные в ячейки **продукта** и **сертификации** , невозможно изменить после импорта электронной таблицы для создания или настройки шаблона. Кроме того, Группа не может содержать две оценки, имеющие одинаковую комбинацию **продукта и сертификации** . Можно использовать несколько шаблонов с одинаковой комбинацией продукта или сертификации.

##### <a name="controlfamily-tab"></a>Вкладка "Контролфамили"

Вкладка **контролфамили** является обязательной.  Обязательные столбцы на этой вкладке, которые должны соответствовать порядку, указанному в примере электронной таблицы,:

- **контролнаме** : имя элемента управления из сертификата, стандартного или нормативного имени, который обычно имеет НЕКОТОРЫЙ тип идентификатора. Имена элементов управления должны быть уникальными в шаблоне. В электронной таблице не может быть нескольких элементов управления с одинаковыми именами.

- **контролфамили** : предоставление слова или фразы для контролфамили, которая определяет широкое группирование элементов управления. Контролфамили не обязательно должен быть уникальным; она может быть указана несколько раз в электронной таблице. Один и тот же Контролфамили также может быть указан в нескольких шаблонах, хотя они не имеют отношения друг к другу. Каждый Контролфамили должен быть сопоставлен по крайней мере с одним элементом управления.

- **контролтитле** : предоставьте заголовок для элемента управления. В то время как Контролнаме — это справочный код, заголовок — это формат форматированного текста, который обычно отображается в нормативных положениях.

- **контролдескриптион** : предоставьте описание элемента управления.

- **контролактионтитле** : это название действия, которое нужно связать с этим элементом управления. Можно добавить несколько действий, разделив их двумя точками с запятой, не пробелья между ними. Каждый список элементов управления должен включать по крайней мере одно действие (это означает, что вы можете перечислить список действий на вкладке **действия** той же электронной таблицы, действия, которые существуют в другом шаблоне или действием, созданным корпорацией Майкрософт). Разные элементы управления могут ссылаться на одно и то же действие.

##### <a name="actions-tab"></a>Вкладка "действия"

Вкладка **действия** является обязательной.  Она определяет действия по улучшению, которыми управляет организация, а не только те, которые являются корпорацией Майкрософт, которые уже существуют в диспетчере соответствия требованиям. Обязательные столбцы для этой вкладки, которые должны соответствовать порядку, указанному в примере электронной таблицы,:

- **актионтитле** : это название действия и обязательное поле. Заданное название должно быть уникальным. **Важно!** если вы ссылаетесь на действие, которое уже существует (например, в другом шаблоне) и изменяли любой из его элементов в последующих столбцах, эти изменения будут распространяться на те же действия в других шаблонах.

- **имплементатионтипе** : в этом обязательном поле перечислите один из трех следующих типов реализации:
    - **Рабочие** действия, реализуемые людьми и процессами для защиты конфиденциальности, целостности и доступности организационных систем, активов, данных и сотрудников (например: осведомленность и обучение безопасности)
    - **Технические** действия, выполненные с помощью технологий и механизмов, которые входят в состав аппаратного, программного обеспечения или встроенного программного обеспечения информационной системы для защиты конфиденциальности, целостности и доступности организационных систем и данных (пример: многофакторная проверка подлинности)
    - **Документация** — действия, реализуемые с помощью документированных политик и процедур установки и определения элементов управления, необходимых для защиты конфиденциальности, целостности и доступности организационных систем, активов, данных и сотрудников (пример: политика информационной безопасности)

- **актионскоре** : в этом обязательном поле Укажите числовое значение оценки для действия. Он должен быть целым числом в диапазоне от 1 до 99; Он не может иметь значение 0, null или пустое значение. Чем больше число, тем больше его значение в сторону улучшения соответствия требованиям. На изображении ниже показано, как диспетчер соответствия отвечает за контроль оценок.

![Значения точек управления диспетчера соответствия требованиям](../media/compliance-score-action-scoring.png "Значения точек управления диспетчера соответствия требованиям")

- **актиондескриптионтитле** : это название описания и является обязательным. Это название описания позволяет выполнять одно и то же действие в нескольких шаблонах и иметь разные описания в каждом шаблоне.  Это поле позволяет определить шаблон, на который ссылается описание. В большинстве случаев в это поле можно ввести имя шаблона, который вы создаете.

- **актиондескриптион** : предоставьте описание действия. Вы можете применить форматирование, например полужирный шрифт и гиперссылки. Это обязательное поле.

- **Dimension — назначение действия** : это необязательное поле. Если включить его, заголовок должен содержать префикс "Dimension –". Все измерения, которые вы включаете здесь, будут использоваться в качестве фильтров в диспетчере соответствия требованиям и будут отображаться на странице "сведения о действиях по улучшению" в диспетчере соответствия требованиям.

##### <a name="dimensions-tab"></a>Вкладка "измерения"

Вкладка " **измерения** " является необязательной. Тем не менее, если вы ссылаетесь на измерение в другом месте, вам нужно указать его здесь, если он не существует в шаблоне, созданном ранее, или в шаблоне Microsoft. Столбцы для этой вкладки перечислены ниже.

- **дименсионкэй** : список "продукт", "сертификация", "цель действия"
- **дименсионвалуе** : примеры: Office 365, Хиппа, превентивные, обнаруживающее

Вы можете просмотреть существующие измерения, перейдя в **Управление клиентами** и выбрав вкладку **измерения** . Кроме того, при экспорте существующего шаблона у экспортированной таблицы будет вкладка **измерения** , в которой перечислены все измерения, используемые в шаблоне.

## <a name="modify-a-template"></a>Изменение шаблона

Возможно, вы захотите изменить уже созданный шаблон, например добавить элементы управления или добавить или удалить действия по улучшению. Этот процесс аналогичен процессу создания шаблона в том, что вы отправите отформатированный файл Excel с данными шаблона.

Тем не менее, существуют определенные сведения о том, как вы форматируете файл, изменяя существующие данные шаблона. **Рекомендуется внимательно проанализировать эти инструкции, чтобы не перезаписать существующие данные, которые вы хотите сохранить.**

### <a name="template-modification-process-steps"></a>Этапы процесса изменения шаблона

Чтобы изменить шаблон, выполните указанные ниже действия.

1. На странице " **шаблоны оценки** " выберите шаблон, который вы хотите изменить, и отобразит его страницу сведений.
2. Выберите **Экспорт в Excel** . Будет загружен файл Excel со всеми данными шаблона. Сохраните файл на локальном компьютере.
3. Внесите изменения в шаблон, [изменив файл Excel, выполнив приведенные ниже инструкции](#formatting-your-excel-file-to-modify-a-template).
4. Завершив внесение изменений в файл Excel, сохраните файл.
5. На странице сведения о вашем шаблоне выберите **изменить шаблон** , чтобы запустить мастер изменения. 
6. На экране **отправить файл** нажмите кнопку **Обзор** , чтобы найти и отправить файл Excel.
7. При отсутствии проблем с файлом на следующем экране отображается имя отправленного файла. Нажмите кнопку **Далее** , чтобы продолжить (если вам нужно изменить файл, выберите **Отправить другой файл** ).
    - Если при работе с файлом возникла проблема, в верхней части сообщения об ошибке поясняется, что не так. Вам потребуется исправить файл и отправить его повторно. При неправильном форматировании электронной таблицы или при наличии недопустимых данных в определенных полях могут возникать ошибки.

8. На экране **"Просмотр и завершение"** отображается количество действий и элементов управления, а также максимальный показатель для шаблона. Когда все будет готово к утверждению, нажмите кнопку **Далее** .
9. Последний экран подтверждает, что шаблон был изменен. Нажмите кнопку **Готово** , чтобы завершить работу мастера.

Теперь шаблон будет включать внесенные изменения. Все оценки, использующие этот измененный шаблон, теперь будут показывать ожидающие обновления, и вам потребуется принять обновления для оценок, чтобы отразить изменения, внесенные в шаблон. Узнайте больше об [обновлениях для оценки](compliance-manager-assessments.md#accepting-updates-to-assessments).

> [!NOTE]
> Если вы используете диспетчер соответствия требованиям на языке, отличном от английского, вы заметите, что при экспорте шаблона в Excel текст будет отображаться на английском языке. Названия действий (как действия по улучшению, так и действия Майкрософт) должны распознаются на английском языке и распознаются элементами управления. Если вы вносите изменения в название действия, обязательно запишите его на английском языке, чтобы правильно импортировать файл.

#### <a name="edit-the-main-template-attributes"></a>Изменение атрибутов основного шаблона

На вкладке **шаблоны** можно изменить содержимое столбца **название** , столбец **инскопесервицес** и любой другой столбец, который вы можете добавить. Однако вы не можете изменять данные в столбцах **Product** или **Certification** .

#### <a name="add-an-improvement-action"></a>Добавление действия по улучшению

1. Перейдите на вкладку **действия** . Добавьте сведения в обязательные поля первой пустой строки под существующими действиями.
2. Перейдите на вкладку **контролфамили** . Найдите строку с элементом управления, с которым сопоставляется действие улучшения. Добавьте новое действие в столбец **контролактионтитле** в этой строке (не забывайте, что в этом поле разделяются несколько действий с двумя точками с запятыми, между которыми нет пробела).
3. Сохраните электронную таблицу.

#### <a name="edit-an-improvement-actions-information"></a>Изменение сведений о действии улучшения

Вы можете изменить любые сведения о действиях *по улучшению, кроме заголовков* . Вы можете изменять любую ячейку из столбцов B, а когда вы импортируете файл обратно в шаблон, действия улучшения в этом шаблоне теперь будут содержать обновленные данные.

Невозможно изменить **актионтитле** (столбец A), так как в этом случае диспетчер соответствия требованиям считает это новым действием по улучшению. Если вы хотите изменить имя действия улучшения, ознакомьтесь с инструкциями, приведенными ниже.

#### <a name="change-an-improvement-actions-name"></a>Изменение имени действия улучшения

Если вы хотите изменить имя действия улучшения, необходимо явно указать в электронной таблице, что вы заменяете существующее имя новым. Выполните следующие действия:

1. На вкладке **действия** электронной таблицы добавьте новый столбец после столбца a.
2. В этом новом столбце, который теперь является столбцом B, поместите в качестве его заголовка в строку 1: **олдактионтитле** .
3. Скопируйте содержимое столбца A и вставьте его в столбец B. В этом случае существующие названия действий по улучшению, которые нужно изменить, будут помещены в столбец B.
4. В столбце A **актионтитле** удалите старое имя и замените его новым именем для действия улучшения.

Обратите внимание, что названия действий, как для действий по улучшению, так и для действий Майкрософт, должны быть написаны на английском языке, чтобы их можно было распознать при указании ссылок в элементах управления.

#### <a name="remove-an-improvement-action"></a>Удаление действия улучшения

Удаление действия по улучшению из строки электронной таблицы **не приводит** к удалению действия из шаблона, который вы изменяете. Вместо этого выполните описанные ниже действия.

1. На вкладке **действия** Вставьте новый столбец в виде **операции** a и PUT в строку заголовка, которая является номером строки 1.
2. В строке для действия по улучшению, которое нужно удалить, добавьте в столбец A команду **Delete** для этой строки.
3. Убедитесь, что элемент управления не ссылается на действие улучшения. Перейдите на вкладку **контролфамили** и найдите название действия улучшения в столбце F, который является **контролактионтитле** .
4. Когда вы найдете действие по улучшению, указанное в столбце **контролактионтитле** , удалите его.
5. Сохраните электронную таблицу.

При импорте электронной таблицы обратно в шаблон ваше действие будет удалено из шаблона. Удаление действия из шаблона не приводит к полному удалению действия. На это действие по-прежнему можно ссылаться из другого шаблона.

Если вы удаляете Последнее действие по улучшению, на которое ссылается элемент управления, необходимо удалить элемент управления.

#### <a name="remove-a-control"></a>Удаление элемента управления

Чтобы удалить элемент управления, выполните те же действия, что и для удаления действия по улучшению, как описано выше. На вкладке **контролфамили** добавьте столбец **Операция** и вставьте **Delete** рядом с элементом управления, который нужно удалить.

## <a name="export-a-template"></a>Экспорт шаблона

Вы можете экспортировать файл Excel, содержащий все данные шаблона. Чтобы изменить шаблон, необходимо экспортировать шаблон, так как это будет файл Excel, который будет редактироваться и отправляться в [процессе изменения](#modify-a-template).

Чтобы экспортировать шаблон, перейдите на страницу сведений о шаблоне и нажмите кнопку **Экспорт в Excel** .

Обратите внимание, что при экспорте шаблона, расширенного из шаблона "Диспетчер соответствия требованиям", экспортированный файл будет содержать только атрибуты, добавленные в шаблон. Экспортированный файл не включает исходные данные шаблона, предоставленные корпорацией Майкрософт. Чтобы получить такой отчет, ознакомьтесь с инструкциями по [экспорту отчета о оценке](compliance-manager-assessments.md#export-an-assessment-report).
