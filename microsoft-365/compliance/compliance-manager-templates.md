# <a name="working-with-assessment-templates-in-compliance-manager"></a>Работа с шаблонами оценки в диспетчере соответствия требованиям

**В этой статье:** Понять, **как работают шаблоны** **и как управлять ими** на странице шаблонов оценки. Получите инструкции по **созданию** **новых шаблонов,** изменению существующих шаблонов, форматирования данных шаблонов с помощью **Excel** и экспорту отчетов о **шаблонах.**

> [!IMPORTANT]
> Шаблоны оценки, доступные вашей организации, зависят от лицензионного соглашения. [Просмотрите сведения.](https://go.microsoft.com/fwlink/?linkid=2132371)

## <a name="templates-overview"></a>Обзор шаблонов

Шаблон — это основа для создания оценки в диспетчере соответствия требованиям. Они содержат элементы управления, необходимые для применения требований сертификации с использованием определенного продукта. Диспетчер соответствия требованиям предоставляет полный набор шаблонов, которые помогают вашей организации соблюдать национальные, региональные и отраслевые требования, управляющие сбором и использованием данных.

## <a name="list-of-pre-built-templates-for-assessments"></a>Список встроенных шаблонов для оценок

Диспетчер соответствия требованиям предоставляет шаблоны для создания оценок, которые помогут вам соответствовать различным нормативным требованиям и стандартам. Просмотр списка [шаблонов, предоставленных](compliance-manager-templates-list.md) диспетчером соответствия требованиям. Новые шаблоны добавляются регулярно, поэтому часто проверяйте список.

## <a name="viewing-and-managing-templates-from-the-assessment-templates-page"></a>Просмотр и управление шаблонами со страницы шаблонов оценки

На странице "Шаблоны оценки" в диспетчере соответствия требованиям отображается список шаблонов и основные сведения. Этот список включает шаблоны, предоставляемые диспетчером соответствия требованиям, а также шаблоны, измененные или созданные организацией. Вы можете применить фильтры, чтобы найти шаблон на основе сертификации, области продукта, страны, отрасли, его создавшего, а также на том, включен ли шаблон для создания оценки.

Выберите шаблон в строке, чтобы отвести его страницу сведений. На этой странице содержится описание шаблона и дополнительные сведения о сертификации, области и сведениях об элементе управления. На этой странице можно выбрать соответствующие кнопки для создания оценки, экспорта данных шаблона в Excel или изменения шаблона.

## <a name="creating-and-modifying-templates-overview"></a>Общие сведения о создании и изменении шаблонов

Чтобы изменить существующий шаблон или создать собственный шаблон, используйте специально отформатированную таблицу Excel[(пример](https://go.microsoft.com/fwlink/?linkid=2124865)загрузки) для сборки необходимых данных управления. После заполнения таблицы ее необходимо импортировать в диспетчер соответствия требованиям во время создания или изменения шаблона.

> [!NOTE]
> Таблица имеет определенный формат и схему, которые необходимо использовать, или она будет неправильно импортироваться в диспетчер соответствия требованиям. Ниже [приведены инструкции по](#formatting-your-template-data-with-excel) форматированием.

**Необходимые роли**

Создавать и изменять шаблоны могут только пользователи с ролью глобального администратора или администратора в диспетчере соответствия требованиям. Узнайте больше о [ролях и разрешениях.](compliance-manager-setup.md#set-user-permissions-and-assign-roles)

## <a name="create-a-new-template"></a>Создание нового шаблона

Чтобы создать собственный шаблон (используемый для создания пользовательских оценок), выполните следующие действия.

1. Перейдите на **страницу шаблонов оценки** в диспетчере соответствия требованиям.
2. Выберите **"Создать новый шаблон"**. Откроется мастер создания шаблонов.
3. Выберите тип шаблона, который нужно создать. В этом случае выберите **"Создать настраиваемый шаблон",** а затем выберите **"Далее".**
4. На экране **отправки**  файла выберите "Обзор", чтобы найти и отправить отформатированный файл Excel, содержащий все необходимые данные шаблона (см. инструкции по правильному форматированию [файла).](#formatting-your-template-data-with-excel)
5. Если проблем с файлом нет, будет отображаться имя загруженного файла. Выберите **"Далее",** чтобы продолжить. (Если необходимо изменить файл, выберите **"Отправить другой файл").**
    - Если в файле произошла ошибка, в верхней части сообщения об ошибке объясняется, что происходит. Вам потребуется исправить файл и отправить его еще раз. Ошибки могут привести к неправильному форматированию таблицы или недопустимым сведениям в определенных полях (см. инструкции по [форматированию).](#formatting-your-template-data-with-excel)  
    
6. На **экране проверки и завершения** показано количество действий по улучшению и элементов управления, а также максимальная оценка для шаблона. Когда все готово к утверждению, выберите **"Создать шаблон".** (Если необходимо внести изменения, выберите **"Назад".**
7. На последнем экране подтверждается, что новый шаблон создан. Выберите **"Готово",** чтобы выйти из мастера.
8. Вы получите страницу сведений о новом шаблоне, где вы сможете [создать оценку.](compliance-manager-assessments.md#create-your-own-custom-assessment)

## <a name="formatting-your-template-data-with-excel"></a>Форматирование данных шаблона с помощью Excel

Таблица Excel, используемая для создания шаблонов, содержит четыре вкладки, три из которых являются обязательной:

1. [Шаблон](#template-tab) (обязательно)
2. [ControlFamily](#controlfamily-tab) (обязательно)
3. [Действия](#actions-tab) (обязательно)
4. [Измерения](#dimensions-tab) (необязательно)

При заполнении таблицы данными шаблона она должна включать вкладки в порядке, перечисленном  **выше,** в противном случае данные не будут успешно импортироваться в шаблон.

##### <a name="template-tab"></a>Вкладка "Шаблон"

Вкладка **"Шаблон"** является обязательной. Сведения на этой вкладке предоставляют метаданные о шаблоне. Существует четыре необходимых столбца. В столбцах должен храниться порядок на листе Excel, как по указано ниже. Вы можете добавить собственный столбец **после** четырех столбцов, чтобы предоставить собственные размеры. В этом случае обязательно добавьте их на вкладку **"Измерения"** с помощью [инструкций ниже.](#dimensions-tab)

- **title**: это название шаблона, которое должно быть уникальным. Оно не может делиться именем с другим шаблоном, который у вас есть в диспетчере соответствия требованиям, включая собственные шаблоны или шаблон диспетчера соответствия требованиям.

- **product**: это необходимое измерение. Список продукта, связанного с шаблоном.

- **сертификация**: это норматив, который вы используете для шаблона.

- **inScopeServices**: это службы в продукте, к который обращается эта оценка (например, если вы указали Office 365 как продукт, Microsoft Teams может быть службой в области действия). Можно перечислить несколько служб, разделенных двумя двоеточиями.

> [!NOTE]
> Данные, вставленные  в  ячейки продукта и сертификации, не могут быть изменены после импорта таблицы для создания или настройки шаблона. Кроме того, группа не может содержать две оценки с одинаковым сочетанием продуктов **и сертификатов.** Можно использовать несколько шаблонов с одинаковым сочетанием продуктов и сертификатов.

##### <a name="controlfamily-tab"></a>Вкладка ControlFamily

Вкладка **ControlFamily** является обязательной.  Необходимые столбцы на этой вкладке, которые должны следовать порядку, представленного в примере таблицы, являются следующими:

- **controlName**: это имя управления из стандарта сертификации, стандарта или норматива, обычно это определенный тип ИД. Имена управления должны быть уникальными в шаблоне. В электронных таблицах нельзя использовать несколько элементов управления с одинаковыми именами.

- **controlFamily**: предоставление слова или фразы для controlFamily, который определяет широкую группировку элементов управления. ControlFamily не должен быть уникальным; он может быть указан в электронных таблицах несколько раз. Один и тот же controlFamily также может быть указан в нескольких шаблонах, хотя они не имеют отношения друг к другу. Каждый controlFamily должен быть соотнося по крайней мере с одним из них.

- **controlTitle**: в качестве заголовка для этого управления. В то время как controlName является эталонным кодом, заголовок является форматом формата ДД, который обычно рассматривается в нормативных актах.

- **controlDescription**: предоставление описания этого управления.

- **controlActionTitle**: это заголовок действия, которое вы хотите связать с этим управлением. Можно добавить несколько действий, разделив их двумя двоеточиями без пробела между ними. Каждый список должен включать по крайней мере одно действие, которое должно существовать (это  означает, что вы можете перечислить действие, которое вы перечисляете на вкладке "Действия" той же таблицы, действие, которое существует в другом шаблоне, или действие, созданное корпорацией Майкрософт). Разные элементы управления могут ссылаться на одно и то же действие.

##### <a name="actions-tab"></a>Вкладка "Действия"

Вкладка **"Действия"** является обязательной.  Он обозначает действия по улучшению, управляемые вашей организацией, а не действиями Майкрософт, которые уже существуют в диспетчере соответствия требованиям. Необходимые столбцы для этой вкладки, которые должны следовать порядку, представленного в примере таблицы, являются следующими:

- **actionTitle**: это заголовок для вашего действия и обязательное поле. Заголовок должен быть уникальным. **Важно!** Если вы ссылались на самостоятельное действие (например, в другом шаблоне) и изменяете любой из его элементов в последующих столбцах, эти изменения будут распространяться на то же действие в других шаблонах.

- **implementationType**: в этом обязательном поле перечислить один из трех типов реализации ниже:
    - **Операционные** действия, реализованные людьми и процессами для защиты конфиденциальности, целостности и доступности организационных систем, активов, данных и персонала (например: осведомленность о безопасности и обучение)
    -  Технические действия, завершенные с помощью технологий и механизмов, содержащихся в компонентах оборудования, программного обеспечения или микропрограмм информационной системы, для защиты конфиденциальности, целостности и доступности организационных систем и данных (например, многофакторная проверка подлинности)
    - **Документация** - действия, реализованные посредством документированных политик и процедур, устанавливающих и определяющих средства контроля, необходимые для защиты конфиденциальности, целостности и доступности организационных систем, активов, данных и персонала (например, политика информационной безопасности)

- **actionScore**: в этом обязательном поле укайте числовые значения для вашего действия. Это должно быть целое число от 1 до 99; не может быть 0, null или пустым. Чем больше число, тем больше его значение для повышения уровня соответствия требованиям. На рисунке ниже показано, как диспетчер соответствия требованиям управляет результатами:

![Значения точки элементов управления в диспетчере соответствия требованиям](../media/compliance-score-action-scoring.png "Значения точки элементов управления в диспетчере соответствия требованиям")

- **actionDescriptionTitle**: это название описания, необходимое. Это название описания позволяет использовать одно и то же действие в нескольких шаблонах и разное описание в каждом шаблоне.  Это поле помогает уточнить, на какой шаблон ссылаются описания. В большинстве случаев в это поле можно поместить имя создаемого шаблона.

- **actionDescription**: предоставление описания действия. Можно применять форматирование, например полужирный текст и гиперссылки. Это обязательное поле.

- **dimension-Action Purpose**: это необязательное поле. Если вы включит его, в заголке должен быть префикс "dimension-". Все измерения, которые вы здесь включаете, будут использоваться в качестве фильтров в диспетчере соответствия требованиям и отображаться на странице сведений о действиях по улучшению в диспетчере соответствия требованиям.

##### <a name="dimensions-tab"></a>Вкладка "Измерения"

Вкладка **"Измерения"** не является обязательной. Однако если вы ссылались на измерение в другом месте, укажите его здесь, если оно не существует в шаблоне, который вы уже создали или в шаблоне Майкрософт. Столбцы для этой вкладки перечислены ниже:

- **dimensionKey**: список "product", "certifications", "action purpose"
- **dimensionValue**: примеры: Office 365, HIPPA, Preventative, Detective

Вы можете просмотреть существующие  измерения, выбрав вкладку **"Измерения".** Кроме того, при экспорте существующего шаблона экспортируемая  таблица будет иметь вкладку "Измерения", на которой перечислены все измерения, используемые в шаблоне.

## <a name="modify-a-template"></a>Изменение шаблона

Возможно, вы захотите изменить уже созданный шаблон, например добавить элементы управления или добавить или удалить действия по улучшению. Этот процесс похож на процесс создания шаблона, так как вы будете загружать форматированный файл Excel с данными шаблона.

Однако при форматирование файла с изменениями существующих данных шаблона необходимо иметь в виду определенные сведения. **Мы рекомендуем внимательно и внимательно и внимательно просмотреть эти инструкции, чтобы не переописывать существующие данные, которые вы хотите сохранить.**

### <a name="template-modification-process-steps"></a>Этапы процесса изменения шаблона

Чтобы изменить шаблон, выполните следующие действия:

1. На странице **"Шаблоны оценки"** выберите шаблон, который нужно изменить, и на этой странице будут выводиться сведения.
2. Выберите **"Экспорт в Excel"** Будет загружен файл Excel со всеми данными шаблона. Сохраните файл на локальном компьютере.
3. Внести изменения в [шаблон, изменяя файл Excel, используя инструкции ниже.](#formatting-your-excel-file-to-modify-a-template)
4. После внесения изменений в файл Excel сохраните его.
5. На странице сведений шаблона выберите **"Изменить** шаблон", чтобы инициировать мастер изменения. 
6. На экране **отправки файла** выберите "Обзор", чтобы найти и отправить файл Excel. 
7. Если проблем с файлом нет, на следующем экране будет показано имя загруженного файла. Select **Next** to continue (if you need to change the file, select **Upload a different file**).
    - Если с файлом возникла проблема, в верхней части сообщения об ошибке объясняется, что происходит. Вам потребуется исправить файл и отправить его еще раз. Ошибки могут привести к неправильному форматированию таблицы или недопустимым сведениям в определенных полях.

8. На **экране проверки и завершения** показано количество действий по улучшению и элементов управления, а также максимальная оценка для шаблона. Когда все готово к утверждению, выберите **"Далее".**
9. Последний экран подтверждает, что шаблон был изменен. Выберите **"Готово",** чтобы выйти из мастера.

Теперь ваш шаблон будет включать внесенные изменения. Все оценки, которые используют этот измененный шаблон, теперь будут показывать ожидающие обновления, и вам потребуется принять обновления для оценок, чтобы отразить изменения, внесенные в шаблон. Узнайте больше об [обновлениях оценок.](compliance-manager-assessments.md#accepting-updates-to-assessments)

> [!NOTE]
> Если вы используете диспетчер соответствия требованиям на языке, отличном от английского, вы заметите, что при экспорте шаблона в Excel какой-либо текст отображается на английском языке. Названия действий (как ваших действий по улучшению, так и действий Майкрософт) должны быть на английском языке, чтобы элементы управления распознали их. При внесении изменений в название действия обязательно напишите его на английском языке, чтобы файл импортируется правильно.

#### <a name="edit-the-main-template-attributes"></a>Изменение атрибутов основного шаблона

На **вкладке** "Шаблоны" можно  редактировать любые изменения в столбце заголовка, в **столбце inScopeServices** и в любом другом столбце, который вы могли добавить. Однако в столбцах продукта  или сертификации нельзя редактировать **что-либо.**

#### <a name="add-an-improvement-action"></a>Добавление действия по улучшению

1. Перейдите на **вкладку "Действия".** Добавьте сведения в необходимые поля в первой пустой строке под существующими действиями.
2. Перейдите на **вкладку ControlFamily.** Найдите строку, содержащую тот или иной контроль, с которой соподержает действие по улучшению. Добавьте новое действие в столбец **controlActionTitle** в этой строке (не забудьте разделить несколько действий в этом поле двумя двоеточиями без пробела между ними).
3. Сохраните таблицу.

#### <a name="edit-an-improvement-actions-information"></a>Изменение сведений о действии по улучшению

Вы можете изменить сведения о любых действиях по улучшению, *за исключением его названия.* Можно редактировать любую ячейку из столбцов B и более новые, а при импорте файла обратно в шаблон действия по улучшению в этом шаблоне теперь будут содержать обновленные данные.

Вы не можете изменить **actionTitle** (столбец A), так как в этом случае диспетчер соответствия требованиям считает это новым действием по улучшению. Если вы хотите изменить имя действия по улучшению, см. инструкции ниже.

#### <a name="change-an-improvement-actions-name"></a>Изменение имени действия по улучшению

Если вы хотите изменить имя действия по улучшению, необходимо явно назначить в электронных таблицах, которые заменяют существующее имя новым именем. Выполните приведенные ниже действия.

1. На **вкладке "Действия"** в электронных таблицах добавьте новый столбец после столбца A.
2. В этом новом столбце, который теперь является столбцом B, поместите его в строку 1: **oldActionTitle**.
3. Скопируйте содержимое столбца A и в paste их в столбец B. Это помещает существующие заголовки действий по улучшению, которые нужно изменить, в столбец B.
4. В столбце A **actionTitle** удалите старое имя и замените его новым именем для действия улучшения.

Обратите внимание, что названия действий, как для действий по улучшению, так и для действий Майкрософт, должны быть написаны на английском языке, чтобы они распознались при ссылке в средствах управления.

#### <a name="remove-an-improvement-action"></a>Удаление действия по улучшению

Чтобы удалить действие улучшения из шаблона, необходимо удалить его из всех ссылок на него. Чтобы изменить таблицу, выполните следующие действия.

1. На **вкладке ControlFamily** найщите заголовок удаляемого действия по улучшению.
2. Удалите заголовок действия улучшения в ячейках, в которых оно отображается. Если действие улучшения является единственным действием в этой строке, удалите всю строку (при этом удаляется управление).
3. На **вкладке "Действия"** удалите строку с удаляемой мерой улучшения.
4. Сохраните таблицу.

При импорте таблицы обратно в шаблон действие улучшения будет удалено из шаблона.

Удаление действия по улучшению из шаблона не полностью удаляет действие улучшения из диспетчера соответствия требованиям. На это действие по-прежнему может ссылаться другой шаблон.

#### <a name="remove-a-control"></a>Удаление управления

Чтобы удалить контроль, измените таблицу, выполив следующие действия, а затем повторно импортировать таблицу:

1. На **вкладке ControlFamily** найдите удаляемый в столбце **controlName.**
2. Удалите строку для этого управления.
    - Если этот удаленный контроль содержит действия по улучшению, на которые не ссылается какой-либо другой контроль, вам потребуется удалить эти действия с вкладки **"Действия".** В противном случае вы получите ошибку проверки.

3. Сохраните таблицу.

При импорте таблицы обратно в шаблон ваш контроль будет удален из шаблона.

## <a name="export-a-template"></a>Экспорт шаблона

Можно экспортировать файл Excel, содержащий все данные шаблона. Чтобы изменить шаблон, необходимо экспортировать шаблон, так как это будет файл Excel, который вы редактируете и загружаете [в процессе изменения.](#modify-a-template)

Чтобы экспортировать шаблон, перейдите на страницу сведений о шаблоне и выберите кнопку **"Экспорт в Excel".**

Обратите внимание, что при экспорте шаблона, расширенного из шаблона диспетчера соответствия требованиям, экспортируется файл будет содержать только атрибуты, добавленные в шаблон. В экспортируемом файле не будут включены исходные данные шаблона, предоставленные корпорацией Майкрософт. Чтобы получить такой отчет, см. инструкции по [экспорту отчета об оценке.](compliance-manager-assessments.md#export-an-assessment-report)
