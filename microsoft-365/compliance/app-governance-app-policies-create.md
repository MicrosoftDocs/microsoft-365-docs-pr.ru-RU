# <a name="create-app-policies"></a>Создание политик приложений

>*[Руководство по лицензированию Microsoft 365 для обеспечения безопасности и соответствия требованиям](https://aka.ms/ComplianceSD).*

Наряду со встроенным набором возможностей для обнаружения аномального поведения приложений и создания оповещений, политики приложений в системе управления приложениями Майкрософт позволяют:

- Указывать условия, на основании которых система управления приложениями может оповещать вас о поведении приложений для автоматического или ручного исправления.
- Реализовать политики приложений по соответствию требованиям для вашей организации.

Политики приложений можно создавать на основе предоставленных настраиваемых шаблонов, а также можно создать собственную политику приложений.

Чтобы создать новую политику приложений, перейдите в раздел **Центр соответствия требованиям Microsoft 365 > Управление приложениями > Обзор > Политики**:

- Чтобы создать политику приложений на основе шаблонов, предназначенных для использования приложений, выберите **Создать политику** в разделе **Создание политики использования приложений**.
- Чтобы создать политику приложений на основе шаблонов, предназначенных для разрешений приложений, выберите **Создать политику** в разделе **Создание политики разрешений**.
- Чтобы создать политику приложений для сертификации приложения или для настраиваемой политики, выберите **Создать**.

## <a name="app-policy-templates"></a>Шаблоны политики приложений

Чтобы создать политику приложений на основе шаблона политики приложений, на странице **Выбор шаблона политики приложений** выберите категорию шаблона приложения, задайте имя шаблона и затем нажмите **Далее**.

В системе управления приложениями есть три категории шаблонов политик приложений.

### <a name="app-users-and-data-access"></a>Доступ пользователей и данных к приложению

Управление приложениями включает следующие шаблоны для создания оповещений об использовании приложений.

| Имя шаблона | Описание |
|:-------|:-----|
| Новое приложение с доступом к большому объему данных | Выделяет недавно зарегистрированные приложения с доступом к большому объему данных, чтобы обеспечить предсказуемость шаблонов этих данных. <br><br> По умолчанию эта политика помечает все приложения, которые были зарегистрированы за последние 7 дней и которые имели доступ к более чем 1 ГБ данных за этот период. Эту политику можно настроить с помощью дополнительных условий и действий. |
|||

### <a name="app-permissions"></a>Разрешения приложений

Управление приложениями включает следующие шаблоны для создания оповещений для разрешений приложений.

| Имя шаблона | Описание |
|:-------|:-----|
| Приложения с избыточными правами | Выделяет все приложения, которым предоставлено больше разрешений, чем они используют, для выявления возможностей сокращения разрешений. <br><br> По умолчанию эта политика помечает все приложения, которые помечены как приложения с избыточными правами, если они не используются в течение 90 дней. Фильтр периода времени можно настроить с помощью дополнительных условий и действий. |
| Новое приложение с разрешениями с высоким уровнем привилегий | Выделяет все новые приложения с разрешениями с высоким уровнем привилегий для выявления потенциально важных приложений, которые могут потребовать дальнейших исследований. <br><br> По умолчанию эта политика помечает все приложения, зарегистрированные за последние 7 дней и имеющие широкий спектр разрешений. |
|||

### <a name="app-certification"></a>Сертификация приложения

Управление приложениями включает следующие шаблоны для создания оповещений для сертификации приложений.

| Имя шаблона | Описание |
|:-------|:-----|
| Новое несертифицированное приложение | Выделяет новые приложения, которые еще не прошли сертификацию, чтобы убедиться, что они ожидаются в клиенте. <br><br> По умолчанию эта политика помечает все приложения, зарегистрированные за последние 7 дней и не прошедшие сертификацию. |
|||

## <a name="custom-app-policies"></a>Настраиваемые политики приложений

Используйте настраиваемую политику приложений, когда необходимое вам действие не покрывается ни одним из встроенных шаблонов.

Чтобы создать настраиваемую политику приложений, сначала выберите **Создать** на странице **Политики**. На странице **Выбор шаблона политики приложений** выберите категорию **Настраиваемая**, затем шаблон **Настраиваемая политика**, а после нажмите **Далее**.

На странице **Название и описание** настройте следующее:

- Имя политики

- Описание политики

- Выберите уровень серьезности политики, который определяет уровень серьезности оповещений, создаваемых этой политикой.

  - Высокий
  - Средний
  - Низкий

На странице **Выбор параметров и условий политики** в разделе **Выбор приложений, к которым применима эта политика** выберите:

- Все приложения
- Выбор определенных приложений

  На панели можно выбрать одно или несколько приложений.
  Нажмите **Добавить**.

Нажмите **Далее**.

На странице **Выбор параметров и условий политики** выберите **Установить новые условия для политики**, а затем нажмите **Далее**.

На панели **Создание правила** можно выбрать условия для нового правила. Нажмите **Добавить условие** и выберите условие из списка, а затем укажите значение условия. Можно добавлять несколько условий.

Ниже перечислены доступные условия для настраиваемой политики приложений.

|Условие | Допустимые значения условия | Дополнительные сведения |
|:-------|:-----|:-------|
| Возраст регистрации приложения | В течение последних X дней |  |
| Сертификация приложения | Базовое соответствие требованиям, соответствие требованиям MCAS или не применимо | [Сертификация Microsoft 365](https://docs.microsoft.com/microsoft-365-app-certification/docs/enterprise-app-certification-guide) |
| Проверка издателя | Да или нет | [Проверка издателя](https://docs.microsoft.com/azure/active-directory/develop/publisher-verification-overview) |
| Разрешение приложения | Выберите одно или несколько разрешений API из списка | [Справочник по разрешениям Microsoft Graph](https://docs.microsoft.com/graph/permissions-reference) |
| Делегированное разрешение | Выберите одно или несколько разрешений API из списка | [Справочник по разрешениям Microsoft Graph](https://docs.microsoft.com/graph/permissions-reference) |
| Высокий уровень привилегий | Да или нет | Это внутреннее обозначение, основанное на той же логике, которая используется MCAS. |
| Приложение с избыточными правами | Да или нет | Приложения, которым предоставлено больше разрешений, чем ими используется. |
| Доступ приложения к данным | Доступ к данным размером более X ГБ в час |  |
| Тенденция доступа приложения к данным | Увеличение использования данных на X% за последние 7 дней |  |
| Доступ приложения к API | Число вызовов API, превышающее X в час |  |
| Тенденция доступа приложения к API | Увеличение вызовов API на X% за последние 7 дней     |  |
| Пользователи, давшие согласие | Пользователей, давших согласие: (более или менее) X |  |
| Приоритетный пользователь дал согласие | Да или нет | Пользователь с [приоритетной учетной записью](https://docs.microsoft.com/microsoft-365/admin/setup/priority-accounts). |
| Согласие на приложение получено от | Выберите пользователей из списка |  |
| Роль пользователя, давшего согласие | Выберите одну или несколько ролей: администратор Teams, читатели каталогов, читатель сведений о безопасности, администратор соответствия требованиям, администратор безопасности, администратор службы поддержки, администратор SharePoint, администратор Exchange, глобальный читатель, глобальный администратор, администратор данных соответствия требованиям, администратор пользователя, администратор поддержки служб | Разрешен выбор нескольких вариантов. <br><br> В этом списке должна быть доступна любая роль Azure AD с назначенным участником. |
| Рабочая нагрузка, к которой получен доступ | OneDrive и/или SharePoint и/или Exchange | Разрешен выбор нескольких вариантов. |
| Частота ошибок | Частота ошибок превышает X% за последние 7 дней, где значение X определяется администратором |  |

Чтобы эта политика приложений применялась, должны быть выполнены все указанные условия.

После указания условий нажмите **Сохранить**, а затем нажмите **Далее**.

Если вы хотите, чтобы система управления приложениями отключила приложение при создании оповещения на основе этой политики, на странице **Определение действий политики** выберите **Отключить приложение**, а затем нажмите **Далее**.

На странице **Определение состояния политики** выберите один из перечисленных вариантов.

- **Режим аудита**: политики оцениваются, но настроенные действия не выполняются. Политики в режиме аудита отображаются с состоянием **Аудит** в списке политик.
- **Активная**: политики оцениваются, и выполняются настроенные действия.
- **Неактивная**: политики не оцениваются, и настроенные действия не выполняются.

## <a name="test-and-monitor-your-new-app-policy"></a>Тестирование и мониторинг новой политики приложений

Теперь, когда политика приложений создана, необходимо отслеживать ее на странице **Политики**, чтобы убедиться, что она регистрирует ожидаемое количество активных оповещений и общее количество оповещений во время тестирования. 

![Страница сводки политик MAPG в Центре соответствия требованиям Microsoft 365 с выделенной политикой](..\media\manage-app-protection-governance\mapg-cc-policies-policy.png)

Если количество оповещений неожиданно низкое, перед установкой состояния политики приложений отредактируйте ее параметры, чтобы убедиться, что настройка сделана правильно.

Ниже показан пример процесса создания новой политики, ее тестирования и активации.

1. Создайте новую политику с начальными значениями для уровня серьезности, приложений, условий и действий и со значением состояния **Режим аудита**.
2. Проверьте ожидаемое поведение, например созданные оповещения.
3. Если поведение не соответствует ожидаемому, отредактируйте приложения политики, условия и параметры действий по мере необходимости и вернитесь к шагу 2.
4. Если поведение соответствует ожидаемому, измените политику и переведите ее состояние в **Активная**.

![Рабочий процесс создания политики приложений](../media/manage-app-protection-governance/mapg-create-new-policy-process.png)
