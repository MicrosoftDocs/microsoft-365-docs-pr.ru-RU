## <a name="august-2020"></a>Август 2020 г.

### <a name="spotlight-insider-risk-and-communication-compliance-updates"></a>Прожектор: риски с рисками и обновлениями для обеспечения соответствия требованиям

Несколько новых и улучшенных функций, которые доступны в общедоступной предварительной версии в этом месяце:

**Внутреннее управление рисками**

- Ознакомьтесь с шестью новыми [шаблонами политик](insider-risk-management-policies.md#policy-templates):
    - Утечки данных по приоритетным пользователям
    - Утечки данных пользователями дисгрунтлед
    - Общие нарушения политики безопасности
    - Нарушения политики безопасности за счет посторонних пользователей
    - Нарушения политики безопасности по приоритету пользователей
    - Нарушения политики безопасности пользователями дисгрунтлед

- Интеграция с [защитником Майкрософт для конечной точки](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection) позволяет импортировать и фильтровать защитник Майкрософт для оповещений конечных точек для действий, обнаруженных политиками, созданными с помощью новых шаблонов политики нарушения безопасности. Кроме того, существует связанный [параметр риска предварительной оценки](insider-risk-management-settings.md#microsoft-defender-for-endpoint-preview) , где вы можете импортировать оповещения системы безопасности в систему управления рисками на основе состояния "защитник Майкрософт" для оценки оповещения о конечных точках.

    > [!NOTE]
    > Чтобы воспользоваться преимуществами защитника Майкрософт для интеграции конечных точек (в том числе новых шаблонов нарушения политики безопасности), необходимо настроить в Организации защитник Майкрософт для конечной точки. Кроме того, необходимо включить защитник Майкрософт для конечной точки для интеграции управления рисками для участников, [настроив расширенные функции в защитнике Майкрософт для конечной точки](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center).
 
- Настройка пороговых значений индикатора при [создании политики](insider-risk-management-policies.md#create-a-new-policy).
- Настройте [приоритетные группы пользователей](insider-risk-management-settings.md#priority-user-groups-preview) , чтобы определить пользователей в Организации, чьи действия требуют более детальной проверки на основе таких факторов, как их положение, уровень доступа к конфиденциальным сведениям или история риска.
- Используйте API действий управления Office 365, чтобы [экспортировать сведения об оповещениях о рисках для оценки риска](insider-risk-management-settings.md#export-alerts-preview) в другие приложения, которые Организация может использовать для управления или объединения данных о рисках для оценки.
- Новые [Параметры домена](insider-risk-management-settings.md#domains-preview) помогают определять уровни риска для действий в определенных доменах и контролировать их.

**Соответствие требованиям к обмену данными**

- При [просмотре сообщений в оповещении](communication-compliance-investigate-remediate.md#step-3-decide-on-a-remediation-action)теперь можно удалять недопустимые сообщения в каналах Microsoft Teams, 1:1 и в беседах групп. Удаленные сообщения и содержимое заменяются подсказкой политики, поясняющей, что она была удалена из-за конфиденциального содержимого.
- Новые [роли взаимодействия](communication-compliance-configure.md#step-1-required-enable-permissions-for-communication-compliance) (они также будут включены в новые группы ролей для обеспечения соответствия требованиям к связи в сентябре).
- Новые параметры обеспечения соответствия требованиям на связь, включающие параметры [шаблонов](communication-compliance-feature-reference.md#notice-templates)для [обеспечения конфиденциальности](communication-compliance-feature-reference.md#privacy-preview) и уведомления.
- Новые [классификаторы](communication-compliance-feature-reference.md#classifiers) , помогающие обнаруживать изображения для взрослых, раци и Гори.
- Уведомление о новом обнаруженном шаблоне, которое появляется при [просмотре сообщений в оповещении](communication-compliance-investigate-remediate.md#step-2-examine-the-message-details) , позволяет узнать о возможном поведении экземпляров пользователя.

По умолчанию Microsoft 365 создает определенный объем оповещений о низком уровне, средней и высокой серьезности, но при этом [параметр громкости оповещений](insider-risk-management-settings.md#alert-volume)можно увеличить или уменьшить в соответствии с вашими потребностями.

Это полезно, если необходимо защитить содержимое контейнера с помощью метки конфиденциальности, но разрешить пользователям настраивать параметры конфиденциальности.

### <a name="new-encryption-options-for-sensitivity-labels"></a>Новые параметры шифрования для меток конфиденциальности 

При настройке шифрования для метки конфиденциальности теперь у вас есть два параметра, которые позволяют пользователям назначать разрешения, когда они вручную применяют эту метку к электронной почте и документам:<br>
- При применении метки к **электронной почте Outlook** пользователи могут применять ограничения, эквивалентные параметру "не пересылать". Получатели смогут прочитать сообщение, но не пересылать, печатать или копировать содержимое.
- При применении метки к **файлам Word, PowerPoint и Excel** пользователям будет предложено назначить разрешения на доступ определенным пользователям и группам.

Чтобы получить дополнительные сведения, перейдите к разделу [ограничение доступа к содержимому с помощью меток конфиденциальности, чтобы применить шифрование](encryption-sensitivity-labels.md#let-users-assign-permissions) .
