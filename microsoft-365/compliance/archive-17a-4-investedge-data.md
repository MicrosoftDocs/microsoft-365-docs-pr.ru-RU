# <a name="set-up-a-connector-to-archive-investedge-data"></a>Настройка соединитетеля для архива данных InvestEdge

Используйте [InvestEdge DataParser](https://www.17a-4.com/investedge-dataparser/) от 17a-4 LLC для импорта и архива данных из InvestEdge в почтовые ящики пользователей в Microsoft 365 организации. В dataParser включен соединиттель InvestEdge, настроенный для захвата элементов из стороннего источника данных и импорта этих элементов в Microsoft 365. Соединиттель InvestEdge DataParser преобразует данные InvestEdge в формат сообщений электронной почты, а затем импортирует эти элементы в почтовые ящики пользователей в Microsoft 365.

После хранения данных InvestEdge в почтовых ящиках пользователей можно применить Microsoft 365, такие как хранение судебного разбирательства, электронные сведения, политики хранения и метки хранения, а также соответствие требованиям связи. Использование соединиттеля InvestEdge для импорта и архива данных в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="overview-of-archiving-investedge-data"></a>Обзор архива данных InvestEdge

В следующем обзоре объясняется процесс использования соединиттеля данных для архивации данных InvestEdge в Microsoft 365.

![Архивировать рабочий процесс для данных InvestEdge с 17a-4](../media/InvestEdgeDataParserConnectorWorkflow.png)

1. Ваша организация работает с 17a-4 для настройки и настройки investEdge DataParser.

2. Элементы InvestEdge регулярно собираются в DataParser. DataParser также преобразует содержимое сообщения в формат сообщения электронной почты.

3. Соединиттель InvestEdge DataParser, который вы создаете в Центр соответствия требованиям Microsoft 365, подключается к DataParser и передает сообщения в безопасное расположение служба хранилища Azure в облаке Майкрософт.

4. В почтовых ящиках пользователей создается подмостки в папке "Входящие" с именем **InvestEdge DataParser,** и элементы InvestEdge импортируется в эту папку. Соединитатель определяет, в какой почтовый ящик импортировать элементы, используя значение свойства *Email.* Каждый элемент InvestEdge содержит это свойство, которое заполняется адресом электронной почты каждого участника.

## <a name="before-you-set-up-a-connector"></a>Перед настройками соединитетеля

- Создайте учетную запись DataParser для соединители Microsoft. Для этого обратитесь в [ООО "17a-4".](https://www.17a-4.com/contact/) При создании соединитетеля в шаге 1 необходимо войти в эту учетную запись.

- Пользователь, создававший соединиттель InvestEdge DataParser в шаге 1 (и завершавший его в шаге 3), должен быть назначен роли экспорта импорта почтовых ящиков в Exchange Online. Эта роль требуется для добавления соединители на странице **соединители** данных в Центр соответствия требованиям Microsoft 365. По умолчанию эта роль не назначена группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах [Создание](/Exchange/permissions-exo/role-groups#create-role-groups) групп ролей или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-set-up-a-investedge-dataparser-connector"></a>Шаг 1. Настройка соединиттеля InvestEdge DataParser

Первым шагом является доступ к странице соединители данных в Центр соответствия требованиям Microsoft 365 и создание соединителя 17a-4 для данных InvestEdge.

1. Перейдите <https://compliance.microsoft.com> и нажмите **кнопку Соединители данных**  >  **InvestEdge DataParser**.

2. На странице **описания продукта InvestEdge DataParser** нажмите **кнопку Добавить соединителю**.

3. На странице **Условия службы нажмите** кнопку **Принять**.

4. Введите уникальное имя, которое идентифицирует соединителен, а затем нажмите **кнопку Далее**.

5. Вопишите в свою учетную запись 17a-4 и выполните действия мастера подключения InvestEdge DataParser.

## <a name="step-2-configure-the-investedge-dataparser-connector"></a>Шаг 2. Настройка соединиттеля InvestEdge DataParser

Работайте с поддержкой 17a-4 для настройки соединителя DataParser InvestEdge.

## <a name="step-3-map-users"></a>Шаг 3. Пользователи карт

Соединиттель InvestEdge DataParser автоматически соединит пользователей с Microsoft 365 адресами электронной почты перед импортом данных в Microsoft 365.

## <a name="step-4-monitor-the-investedge-dataparser-connector"></a>Шаг 4. Мониторинг соединиттеля InvestEdge DataParser

После создания соединиттеля InvestEdge DataParser можно просмотреть состояние соединитетеля в Центр соответствия требованиям Microsoft 365.

1. Перейдите <https://compliance.microsoft.com> и щелкните **соединители данных** в левом nav.

2. Щелкните  вкладку Соединители, а затем выберите соединиттель InvestEdge DataParser, созданный для отображения страницы вылетов, которая содержит свойства и сведения о соединителе.

3. В **состоянии Соединитель с исходным кодом** щелкните ссылку **Журнал** загрузки, чтобы открыть (или сохранить) журнал состояния соединитетеля. В этом журнале содержатся данные, импортируемые в облако Майкрософт.

## <a name="known-issues"></a>Известные проблемы

В настоящее время мы не поддерживаем импорт вложений или элементов размером более 10 МБ. Поддержка более крупных элементов будет доступна позднее.
