# <a name="microsoft-privacy-management-preview"></a>Управление конфиденциальностью Майкрософт (предварительный просмотр)

## <a name="what-is-privacy-management"></a>Что такое управление конфиденциальностью

По мере роста размеров и сложности облачных данных вашей компании необходимо понимать и защищать персональные данные, удерживаемые в вашей среде. Управление конфиденциальностью в Microsoft 365 позволяет сотрудникам принимать разумные решения по обработке данных и решать критически важные риски конфиденциальности, предоставляя эффективные способы поиска и управления личными данными, автоматизации операций конфиденциальности и выполнения запросов на права субъекта. Эти решения позволят создать надежное рабочее место для конфиденциальности и решить проблемы в масштабе.

> [!NOTE]
> Эти службы в настоящее время находятся в стадии предварительного просмотра и подчиняются условиям, указанным в [термине Online Services Terms.](https://www.microsoft.com/en-us/licensing/product-licensing/products)

## <a name="privacy-management-solutions"></a>Решения по управлению конфиденциальностью

В современном ландшафте данных организации должны вкладывать средства в управление и защиту хранимых ими персональных данных. Правила конфиденциальности во всем мире устанавливают стандарты, которые следует соблюдать, и права отдельных лиц на управление собственной личной информацией, а также постоянно совершенствуются методы для бизнеса, чтобы сохранить информацию о своих сотрудниках и клиентах в безопасности.

Microsoft 365 конфиденциальности предоставляет решения, которые позволяют:

- **Поиск и** визуализация персональных данных. Защита данных начинается с тщательного понимания того, что хранятся в организации, где они живут в службах, которые вы используете, и условий, в которых они управляются в долгосрочной перспективе.
- Управление рисками конфиденциальности. Используйте управление конфиденциальностью для оценки данных для ключевых сценариев риска и используйте встроенные средства для настройки оповещений и устранения проблем, таких как непреднамеренное чрезмерное хранение, экспозиция или ненужное хранение личной информации.
- **Эффективно выполнять запросы** на персональные данные. Когда люди запрашивают управление личными данными, хранимые о них, используйте решение по управлению конфиденциальностью для сбора данных, проверки результатов и создания отчетов.

Управление конфиденциальностью предоставляет вам инструменты, которые помогут вам ориентироваться в этих ситуациях, автоматизировать ключевые задачи и легко управлять процессами обработки данных.

### <a name="understand-your-data"></a>Понимание данных

С помощью управления конфиденциальностью можно получить сведения о всей области конфиденциальности Microsoft 365 организации. Решение позволит оценить ваши данные для личной информации, дать вам четкое представление о том, что вы храните, и предложить возможности для изучения областей, ключевых интересов.

Эти сведения представлены на панели мониторинга **Обзор,** которая предоставляет динамические обновления о ваших данных и важных тенденциях, а также профиль **данных,** который позволяет изучить текущую аналитику. Результаты и оповещения здесь могут информировать о следующих действиях по управлению данными.

Дополнительные сведения см. в [дополнительных сведениях о поиске и визуализации персональных данных.](privacy-management-data-profile.md)

### <a name="manage-risk-scenarios"></a>Управление сценариями риска

Сложные среды данных могут представлять потенциально рискованные сценарии для персональных данных. Управление конфиденциальностью предоставляет простые способы обнаружения рисков в следующих областях и создания текущих процессов для обработки этих сценариев.

- **Overexposed personal data:** Companies may collect various types of information that can be used to identify individual customers or employees. Необходимо правильно управлять правами доступа к этим данным для защиты конфиденциальности и предотвращения ненадлежащего использования.
- **Передача данных:** передача персональных данных между отделами в вашей организации или через страны или региональные границы может повысить риск воздействия данных или выйти из законодательства и правил конфиденциальности.
- **Минимизация данных.** Компании могут собирать избыточные сведения или хранить их дольше, чем необходимо, что приводит к хранению неиспользованых персональных данных. Эти данные следует свести к минимуму, чтобы предотвратить риски конфиденциальности.

Управление конфиденциальностью предоставляет встроенные шаблоны для настройки политик обработки данных в этих областях. Эти политики могут оценивать Microsoft 365 данных на постоянной основе, предупреждать о возможных проблемах и помогают устранять эти проблемы. В результате ваши сотрудники могут лучше следовать рекомендациям конфиденциальности и быть в курсе рекомендуемых действий. Шаблоны можно использовать как предоставленные, так и настраиваемые для удовлетворения конкретных потребностей вашей компании.

Дополнительные дополнительные возможности см. в [см. в руб. Создание и управление политиками.](privacy-management-policies.md)

### <a name="manage-subject-rights-requests"></a>Управление запросами на права субъекта

В соответствии с определенными правилами конфиденциальности, например Общим регулированием защиты данных (GDPR) в ЕС, отдельные лица могут запрашивать проверку или управление персональными данными о себе, собранными компаниями. Для компаний, которые хранят большие объемы информации, поиск соответствующих данных может показаться сложной задачей.

Управление конфиденциальностью в Microsoft 365 может помочь вам обрабатывать эти запросы с помощью решения запроса на права субъекта. Она предоставляет возможности автоматизации и рабочего процесса для поиска данных субъектов, хранимых в Microsoft 365, проверки результатов, выбора соответствующих файлов и создания отчетов. По пути вы можете безопасно сотрудничать с другими специалистами в вашей организации, чтобы довести запрос до завершения.

Обратите внимание, что это решение запроса на права субъекта предоставляет возможности, помимо исходного решения по запросам субъекта данных (DSR) в центре соответствия требованиям, и между ними нет подключения или общего доступа к рабочего процессам.

Дополнительные дополнительные новости см. в [руб. Управление запросами на права субъекта.](privacy-management-subject-rights-requests.md)

## <a name="how-we-evaluate-your-data"></a>Оценка данных

Чтобы показать личную информацию в Microsoft 365 среде и предоставить возможности для управления данными и устранения проблем, управление конфиденциальностью оценивает данные в следующей области.

### <a name="what-privacy-management-evaluates"></a>Оценка управления конфиденциальностью

- Конфиденциальные сведения, которые мы называем личными данными, и это данные, поддерживаемые классификацией данных Майкрософт, например, имя, адрес или номер социального обеспечения
- Персональные данные людей, связанных с вашей организацией, таких как клиенты и сотрудники
- Обработка данных сотрудниками, работающими с персональными данными, такими как владельцы файлов и сотрудники бизнес-операций

Дополнительные сведения о том, Microsoft 365 определяет конфиденциальные сведения, см. в дополнительных сведениях [о типах конфиденциальной информации.](sensitive-information-type-learn-about.md)

### <a name="where-privacy-management-identifies-personal-data"></a>Где управление конфиденциальностью идентифицирует персональные данные

Решение по управлению конфиденциальностью для Microsoft 365 оценивает данные и файлы, хранимые вашей организацией в облачной службе Microsoft 365, чтобы помочь вам определять и управлять рисками конфиденциальности в этом пространстве. Это включает:

- Microsoft Exchange
- Microsoft SharePoint
- Microsoft OneDrive
- Microsoft Teams

Так как управление конфиденциальностью фокусируется на данных, определенных вашей организации, любые личные учетные записи ваших сотрудников или клиентов, которые могут иметься в этих службах, не будут в области.
