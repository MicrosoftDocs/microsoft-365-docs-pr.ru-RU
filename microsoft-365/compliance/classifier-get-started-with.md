# <a name="get-started-with-trainable-classifiers"></a>Начало работы с обучаемыми классификаторами

Обучаемый классификатор Microsoft 365 — это средство, которое можно обучить распознавать различные типы контента, предоставляя ему примеры для анализа. После обучения его можно использовать для идентификации элемента для применения меток конфиденциальности Office, политик соответствия коммуникациям и политик меток хранения.

Для создания настраиваемого обучаемого классификатора сначала нужно дать ему образцы, которые человек выбирает и положительно соответствует категории. Затем после их обработки вы тестируете способность классификаторов прогнозировать, предоставляя ей сочетание положительных и отрицательных примеров. В этой статье показано, как создать и обучить настраиваемый классификатор, а также повысить производительность настраиваемых обучаемых классификаторов и предварительно обученных классификаторов за время их существования путем переподготовки.

Дополнительные информацию о различных типах классификаторов см. в дополнительных данных об обучаемых [классификаторах.](classifier-learn-about.md)

## <a name="prerequisites"></a>Предварительные условия

### <a name="licensing-requirements"></a>Требования к лицензированию

Классификаторы — это функция соответствия требованиям Microsoft 365 E5 или E5. Чтобы использовать их, необходимо иметь одну из этих подписок.

### <a name="permissions"></a>Разрешения

Для доступа к классификаторам в пользовательском интерфейсе: 

- Глобальный администратор должен выбрать для клиента создание настраиваемые классификаторы.
- Для обучения классификатора необходима роль администратора соответствия требованиям.

Для использования классификаторов в таких сценариях необходимы учетные записи с этими разрешениями:

- Сценарий политики меток хранения: роли управления записями и управления хранением 
- Сценарий политики меток конфиденциальности: администратор безопасности, администратор соответствия требованиям, администратор данных соответствия требованиям
- Сценарий политики соответствия коммуникациям: администратор управления рисками, администратор надзорной проверки 

> [!IMPORTANT]
> По умолчанию только пользователь, создавший настраиваемый классификатор, может обучить и просмотреть прогнозы, сделанные этим классификатором.

### <a name="seed-content"></a>Содержимое "Seed"

Если вы хотите, чтобы обучаемый классификатор независимо и точно определял элемент как определенный классификатор, сначала необходимо представить ему множество примеров типа контента, которые находятся в этой категории. Это подача образцов в обучаемый классификатор называется *засеяния.* Содержимое seed выбирается человеком и представляет категорию содержимого.

> [!TIP]
> Необходимо иметь не менее 50 положительных примеров и до 500. Обучаемый классификатор обрабатывает до 500 последних созданных примеров (по отметке даты и времени, созданной файлом). Чем больше примеров вы предоставляете, тем точнее прогнозы, которые будет делать классификатор.

### <a name="testing-content"></a>Тестирование контента

После того как обучаемый классификатор обработает достаточно положительных примеров для построения модели прогнозирования, необходимо проверить прогнозы, которые он делает, чтобы проверить, может ли классификатор правильно различать элементы, которые соответствуют категории и элементы, которые не. Для этого нужно выбрать другой, надеемся, больший набор содержимого, состоящего из примеров, которые должны попасть в категорию, и примеров, которые не будут. Следует протестировать данные, отличались от данных начального начального уровня, которые вы предоставили. После их обработки вы вручную пройдите результаты и убедитесь, что каждое прогнозирование верно, неверно или вы не уверены. Обучаемый классификатор использует эту обратную связь для улучшения модели прогнозирования.

> [!TIP]
> Для наилучших результатов в тестовом примере установлено не менее 200 элементов с ряду положительных и отрицательных совпадений.

## <a name="how-to-create-a-trainable-classifier"></a>Создание обучаемого классификатора

1. Соберет от 50 до 500 элементов контента. Это должны быть только образцы, которые строго представляют тип контента, который обучаемый классификатор должен положительно определить как в категории классификации. Для поддерживаемых типов файлов см. расширения имен файлов для обхода по умолчанию и типы файлов для разлифментации в [SharePoint Server.](https://docs.microsoft.com/sharepoint/technical-reference/default-crawled-file-name-extensions-and-parsed-file-types)

   > [!IMPORTANT]
   > Элементы образца для тестирования и их seed не должны быть зашифрованы и должны быть на английском языке.

   > [!IMPORTANT]
   > Убедитесь, что элементы в наборе элементов **являются** примерами категории. Обучаемый классификатор сначала создает свою модель на основе того, чем вы ее засеяли. Классификатор предполагает, что все образцы данных являются положительными и не могут узнать, является ли образец слабым или отрицательным совпадением с категорией.

2. Поместите содержимое для закладки в папку SharePoint Online, выделенную для удержания *только содержимого.* Заметьте сайт, библиотеку и URL-адрес папки.

   > [!TIP]
   > Если вы создаете новый сайт и папку для данных о заметиве, то перед созданием обучаемого классификатора, который будет использовать эти данные, необходимо проиндексировать это расположение по крайней мере в течение часа.

3. Во sign in to Microsoft 365 compliance center with compliance admin or security admin role access and open **Microsoft 365 compliance center** or Microsoft **365 security center** Data  >  **classification**.

4. Выберите **вкладку "Обучаемые классификаторы".**

5. Choose **Create trainable classifier**.

6. Заполните соответствующие значения для категорий элементов и полей, которые должен определить обучаемый `Name` `Description` классификатор.

7. Выберите сайт, библиотеку и URL-адрес папки SharePoint Online для сайта контента для закладки из шага 2. Выберите `Add` .

8. Просмотрите параметры и выберите `Create trainable classifier` .

9. В течение 24 часов обучаемый классификатор обрабатывает данные о качестве и создает модель прогнозирования. Состояние классификатора находится `In progress` во время обработки данных о заметиве. Когда классификатор завершит обработку данных о замечении, состояние изменится на `Need test items` .

10. Теперь можно просмотреть страницу сведений, выбрав классификатор.

    > [!div class="mx-imgBorder"]
    > ![обучаемый классификатор, готовый к тестированию](../media/classifier-trainable-ready-to-test-detail.png)

11. Соберет не менее 200 тестовых элементов контента (10 000 максимально) для наилучших результатов. Это должен быть набор элементов, которые являются положительными, отрицательными и некоторыми, которые немного менее очевидны по своей природе. Для поддерживаемых типов файлов см. расширения имен файлов для обхода по умолчанию и типы файлов для разлифментации в [SharePoint Server.](https://docs.microsoft.com/sharepoint/technical-reference/default-crawled-file-name-extensions-and-parsed-file-types)

    > [!IMPORTANT]
    > Образцы элементов не должны быть зашифрованы и должны быть на английском языке.

12. Поместите тестовое содержимое в папку SharePoint Online, выделенную только для удержания *тестового содержимого.* Заметьте URL-адрес сайта, библиотеки и папки SharePoint Online.

    > [!TIP]
    > Если вы создаете новый сайт и папку для тестовых данных, перед созданием обучаемого классификатора, который будет использовать эти данные, проиндексируете это расположение по крайней мере в течение часа.

13. Выберите `Add items to test` .

14. Выберите сайт, библиотеку и URL-адрес папки SharePoint Online для тестового сайта контента на шаге 12. Выберите `Add` .

15. Завершите работу мастера, выбрав `Done` . Обработка тестовых файлов обучаемым классификатором займет до часа.

16. Когда обучаемый классификатор будет обрабатывать тестовые файлы, состояние на странице сведений изменится на `Ready to review` . Если вам нужно увеличить размер тестового примера, выберите и разрешим обучаемой `Add items to test` классификатору обрабатывать дополнительные элементы.

    > [!div class="mx-imgBorder"]
    > ![готовы к просмотру снимка экрана](../media/classifier-trainable-ready-to-review-detail.png)

17. Выберите `Tested items to review` вкладку для просмотра элементов.

18. Microsoft 365 будет одновременно представлять 30 элементов. Просмотрите их и в поле `We predict this item is "Relevant". Do you agree?` выберите либо `Yes` либо `No` `Not sure, skip to next item` . Точность модели автоматически обновляется после каждого 30 элементов.

    > [!div class="mx-imgBorder"]
    > ![поле "Просмотр элементов"](../media/classifier-trainable-review-detail.png)

19. Просмотрите *не менее* 200 элементов. После стабилизации оценки точности параметр **публикации** станет доступным и будет говорить состояние классификатора. `Ready to use`

    > [!div class="mx-imgBorder"]
    > ![точность и готовность к публикации](../media/classifier-trainable-review-ready-to-publish.png)

20. Опубликуем классификатор.

21. После публикации классификатор будет доступен в качестве условия в автоматическом [](apply-retention-labels-automatically.md#configuring-conditions-for-auto-apply-retention-labels) применении меток [конфиденциальности Office,](apply-sensitivity-label-automatically.md)политика автоматического применения меток хранения на основе условия и соответствия [коммуникации.](communication-compliance.md)
