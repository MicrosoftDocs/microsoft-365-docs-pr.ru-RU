# <a name="get-started-with-trainable-classifiers"></a>Начало работы с обучаемыми классификаторами

Классификатор Microsoft 365, который можно обучить распознавать различные типы контента, предоставляя ему примеры для анализа. После обучения вы можете использовать его для определения элемента для применения меток конфиденциальности Office, политик соответствия требованиям к коммуникациям и политик меток хранения.

Создание настраиваемого классификатора, который можно обучить, сначала включает в себя предоставление ему образцов, которые были выбрали люди и положительно совпадали с этой категорией. Затем, после обработки этих данных, вы тестируете способность классификаторов предсказывать, предоставляя ему сочетание положительных и отрицательных образцов. В этой статье показано, как создать и обучить настраиваемый классификатор и повысить производительность настраиваемых классификаторов и предварительно подготовленных классификаторов за время их жизни путем переподготовки.

Дополнительные новости о различных типах классификаторов см. в дополнительных подробной информации о [классификаторах,](classifier-learn-about.md)которые можно обучить.

Просмотрите это видео для краткой сводки создания классификатора, который можно обучить. Вам по-прежнему необходимо прочитать эту полную статью, чтобы получить подробные сведения.

## <a name="prerequisites"></a>Необходимые условия

### <a name="licensing-requirements"></a>Требования к лицензированию

Классификаторы — это функция Microsoft 365 E5 или соответствия требованиям E5. Вы должны иметь одну из этих подписок, чтобы использовать их.

### <a name="permissions"></a>Разрешения

Чтобы получить доступ к классификаторам в пользовательском интерфейсе: 

- Глобальный администратор должен выбрать для клиента создание настраиваемые классификаторы.
- Для подготовки классификатора требуется роль администратора соответствия требованиям.

Для использования классификаторов в этих сценариях потребуется учетная запись с этими разрешениями:

- Сценарий политики меток хранения: роли управления записями и управления хранением 
- Сценарий политики меток конфиденциальности: администратор безопасности, администратор соответствия требованиям, администратор данных соответствия требованиям
- Сценарий политики соответствия требованиям к коммуникации: администратор управления рисками изнутри, администратор надзорных обзоров 

> [!IMPORTANT]
> По умолчанию только пользователь, создавший настраиваемый классификатор, может обучить и просмотреть прогнозы, сделанные этим классификатором.

### <a name="seed-content"></a>Содержимое seed

Если вы хотите, чтобы классификатор, с помощью подготовки, мог самостоятельно и точно определить элемент как определенный классификатор, сначала необходимо представить ему множество примеров типа контента, который находится в этой категории. Это кормление образцов в обучаемый классификатор называется *посевом.* Содержимое семени выбирается человеком и, по мнению экспертов, представляет категорию контента.

> [!TIP]
> Необходимо иметь не менее 50 положительных примеров и до 500. Классификатор, который можно обучить, будет обрабатывать до 500 последних созданных образцов (по созданному файлу дата/время). Чем больше примеров вы предоставите, тем точнее будут прогнозы, которые будет делать классификатор.

### <a name="testing-content"></a>Тестирование контента

После обработки классификатором для обучения достаточно положительных образцов для создания модели прогнозирования необходимо проверить прогнозы, которые он делает, чтобы проверить, может ли классификатор правильно различать элементы, которые соответствуют категории, и элементы, которые этого не делают. Вы делаете это, выбрав другой, мы надеемся, больший набор выбранных людьми контента, который состоит из образцов, которые должны попасть в категорию и образцы, которые не будут. Вы должны протестировать с помощью различных данных, чем исходные исходные данные семян, которые вы впервые предоставили. После обработки этих результатов вы вручную проверяете, является ли каждый прогноз правильным, неправильным или вы не уверены. Классификатор, который можно обучить, использует эту обратную связь для улучшения модели прогнозирования.

> [!TIP]
> Для наилучших результатов в тестовом наборе имеется не менее 200 элементов с ряду положительных и отрицательных совпадений.

## <a name="how-to-create-a-trainable-classifier"></a>Создание классификатора, который можно обучить

1. Сбор между 50-500 элементами контента для семян. Это должны быть только образцы, которые сильно представляют тип контента, который необходимо классификатору, который можно использовать для положительного определения в категории классификации. См., что по умолчанию для поддерживаемых типов файлов SharePoint [сервере](/sharepoint/technical-reference/default-crawled-file-name-extensions-and-parsed-file-types) по умолчанию обходили расширения имен файлов и размежевые типы файлов.

   > [!IMPORTANT]
   > Элементы семенного и тестового образцов не должны быть зашифрованы и должны быть на английском языке.

   > [!IMPORTANT]
   > Убедитесь, что элементы в наборе семян **являются** сильными примерами категории. Классификатор, который можно обучить, сначала создает свою модель на основе того, с чем вы ее сеяли. Классификатор предполагает, что все образцы семян являются сильными позитивами и не могут узнать, является ли образец слабым или отрицательным.

2. Поместите содержимое семени в папку SharePoint Online, которая посвящена только содержанию *семян.* Обратите внимание на URL-адрес сайта, библиотеки и папки.

   > [!TIP]
   > Если вы создаете новый сайт и папку для семенных данных, то перед созданием классификатора, который будет использовать эти данные, необходимо проиндексировать это расположение по крайней мере за час.

3. Во входе Центр соответствия требованиям Microsoft 365 с доступом администратора или администратора безопасности к роли администратора и **откройте** Центр соответствия требованиям Microsoft 365 или **Microsoft 365** центра безопасности  >  **классификации данных.**

4. Выберите **вкладку Классификаторы,** которые можно обучить.

5. Выберите **Создание классификатора, который можно обучить.**

6. Заполните соответствующие значения для и полей категории элементов, которые необходимо определить этому `Name` `Description` классификатору для обучения.

7. Выберите URL SharePoint веб-сайт, библиотеку и папку для сайта семенного контента с шага 2. Выберите `Add` .

8. Просмотрите параметры и выберите `Create trainable classifier` .

9. В течение 24 часов обучаемый классификатор обработать данные семян и создать модель прогнозирования. Состояние классификатора — `In progress` это процесс обработки данных семян. Когда классификатор завершает обработку данных семян, состояние меняется на `Need test items` .

10. Теперь вы можете просмотреть страницу сведений, выбрав классификатор.

    > [!div class="mx-imgBorder"]
    > ![классификатор, готовый к тестированию](../media/classifier-trainable-ready-to-test-detail.png)

11. Сбор по крайней мере 200 элементов тестового контента (10 000 макс) для наилучших результатов. Это должно быть сочетание элементов, которые являются сильными позитивами, сильными негативами и некоторыми, которые немного менее очевидны по своей природе. См., что по умолчанию для поддерживаемых типов файлов SharePoint [сервере](/sharepoint/technical-reference/default-crawled-file-name-extensions-and-parsed-file-types) по умолчанию обходили расширения имен файлов и размежевые типы файлов.

    > [!IMPORTANT]
    > Образцы элементов не должны быть зашифрованы и должны быть на английском языке.

12. Поместите тестовый контент в папку SharePoint Online, которая предназначена только для проведения *тестового контента.* Обратите внимание на URL SharePoint веб-сайт, библиотеку и папку.

    > [!TIP]
    > Если вы создаете новый сайт и папку для тестовых данных, то перед созданием классификатора, который будет использовать эти семенные данные, необходимо проиндексировать это расположение по крайней мере за час.

13. Выберите `Add items to test` .

14. Выберите URL SharePoint веб-сайт, библиотеку и папку для сайта тестового контента с шага 12. Выберите `Add` .

15. Завершите мастер, выбрав `Done` . Процесс обработки тестовых файлов для классификатора, который можно обучить, займет до часа.

16. Когда обучаемый классификатор будет обрабатывать тестовые файлы, состояние на странице сведений изменится на `Ready to review` . Если необходимо увеличить размер тестового образца, выберите и разрешим классификатору обучить `Add items to test` дополнительные элементы.

    > [!div class="mx-imgBorder"]
    > ![готовый к просмотру скриншота](../media/classifier-trainable-ready-to-review-detail.png)

17. Выберите `Tested items to review` вкладку для просмотра элементов.

18. Microsoft 365 одновременно будет представлено 30 элементов. Просмотрите их и в `We predict this item is "Relevant". Do you agree?` поле выберите либо или `Yes` `No` `Not sure, skip to next item` . Точность модели автоматически обновляется после каждого 30 элементов.

    > [!div class="mx-imgBorder"]
    > ![поле элементов просмотра](../media/classifier-trainable-review-detail.png)

19. Просмотрите *не менее* 200 элементов. Как только оценка точности стабилизируется, будет доступен вариант **публикации** и будет говориться состояние классификатора `Ready to use` .

    > [!div class="mx-imgBorder"]
    > ![оценка точности и готов к публикации](../media/classifier-trainable-review-ready-to-publish.png)

20. Опубликуй классификатор.

21. После публикации классификатор будет доступен в качестве условия в Office автоматической маркировки с метки чувствительности, автоматическое применение политики метки хранения на основе условия и в соответствии с [связью](communication-compliance.md).
