# <a name="create-an-ediscovery-hold"></a>Создание запроса на удержание в случае обнаружения электронных данных

Вы можете использовать основное дело eDiscovery для создания сохранения содержимого, которое может быть релевантно для дела. Вы можете на удержание разместить почтовые ящики Exchange и учетные записи OneDrive для бизнеса людей, которых вы изучаете по этому делу. Вы также можете удерживать почтовые ящики и сайты, связанные с Microsoft Teams, группами Office 365 и группами Yammer. При удержании расположений контента содержимое сохраняется до тех пор, пока вы не удалите удержание из расположения содержимого или пока не удалите удержание.

После создания удержания для eDiscovery может потребоваться до 24 часов, чтобы удержание вступает в силу. 

При создании удержания у вас есть следующие параметры для области контента, который сохраняется в указанных расположениях контента:
  
- Создается бесконечное удержание, при котором все содержимое в указанных расположениях помещается на удержание. Кроме того, можно создать удержание на основе запроса, в котором на удержание помещается только контент в указанных расположениях, которые совпадают с поисковым запросом.

- Можно указать диапазон дат, чтобы сохранить только контент, отправленный, полученный или созданный в пределах этого диапазона дат. Кроме того, можно удерживать весь контент в указанных расположениях независимо от того, когда он был отправлен, получен или создан.
  
## <a name="how-to-create-an-ediscovery-hold"></a>Создание удержания для eDiscovery

Чтобы создать удержание eDiscovery, связанное с основным делом eDiscovery:
  
1. Войдите и войдите, используя учетные данные для учетной записи пользователя, которая имеет соответствующие разрешения на [https://compliance.microsoft.com](https://compliance.microsoft.com) eDiscovery.

2. В левой области навигации Центра соответствия требованиям Microsoft 365 щелкните "Показать **все",** а затем выберите "> основные электронные **> основные".**

3. На странице **core eDiscovery** выберите дело, в которое нужно создать удержание, и нажмите кнопку **"Открыть дело".**

4. На **домашней странице** для дела щелкните вкладку **"Содержит".**
  
5. На странице **"Удерживает"** нажмите кнопку **"Создать".**

6. На странице **"Имя мастера удержания"** придайте имени удержания и добавьте необязательное описание, а затем нажмите кнопку **"Далее".** Имя удержания должно быть уникальным в пределах организации.

7. На странице **"Расположения содержимого"** выберите расположения контента, которые нужно разместить на удержании. Почтовые ящики, сайты и общедоступные папки можно разместить на удержании.

    ![Выбор расположения контента для постановки на удержание](../media/a59e4265-9151-4dbf-913f-6a4ab8db06b4.png)
  
   1. **Расположения почтовых** ящиков— щелкните "Выбрать **пользователей,** группы или команды", а затем снова выберите пользователей, группы или **команды,** чтобы указать почтовые ящики для удержания. Используйте поле поиска для поиска почтовых ящиков пользователей и групп рассылки (для применения удержания к почтовым ящикам членов группы) для применения удержания. Вы также можете надержать связанный почтовый ящик для группы Microsoft Team, группы Office 365 или группы Yammer. Выберите пользователя, группу, командный контроль, нажмите кнопку **"Выбрать"** и нажмите кнопку **"Готово".**

   1. **Расположения сайтов—**  **щелкните "Выбрать сайты",** а затем снова выберите "Выбрать сайты", чтобы указать учетные записи SharePoint и OneDrive для удержания. Укажите URL-адрес каждого сайта, который вы хотите поставить на удержание. Вы также можете добавить URL-адрес сайта SharePoint для группы Microsoft Team, группы Office 365 или группы Yammer. Нажмите **кнопку "Выбрать",** а затем нажмите кнопку **"Готово"**.
  
   1. **Общедоступные папки Exchange.** Переместите переключатель с переключателем в положение "Все", чтобы поместить все общедоступные папки в организации ![ ](../media/scc-toggle-on.png) Exchange Online на удержание.  Нельзя выбрать конкретные общедоступные папки для удержания. Если вы не хотите ставить удержание на общедоступные папки, оставьте для переключателя **"Нет".**

   > [!NOTE]
   > К удержанию необходимо добавить по крайней мере одно расположение содержимого. В противном случае статические статические элементы удержания для eDiscovery покажут, что никакие элементы не находятся на удержании.

8. После добавления местоположений содержимого в удержание нажмите кнопку **"Далее".**

9. Чтобы создать удержание на основе запроса с условиями, выполните следующие условия. В противном случае, чтобы сохранить весь контент в указанных расположениях контента, нажмите кнопку **"Далее".**

    ![Создание удержания на основе запроса с условиями](../media/d587b58e-d05c-4ac0-b0fe-09019e4f1063.png)
  
    1. В поле **"Ключевые слова"** введите поисковый запрос, чтобы сохранялся только контент, который соответствует условиям поиска. Можно указать ключевые слова, свойства сообщений электронной почты или свойства документов, например имена файлов. Вы также можете использовать более сложные запросы, которые используют boolean оператор, например **AND,** **OR** или **NOT**.

    1. Нажмите **кнопку "Добавить условия",** чтобы добавить одно или несколько условий, чтобы сузить поисковый запрос для удержания. Каждое условие добавляет предложение в поисковый KQL-запрос, созданный и запускающийся при создании удержания. Например, можно указать диапазон дат, чтобы документы электронной почты или сайта, созданные в диапазоне дат, помещались на удержание. Условие логически подключается к запросу ключевого слова (указанному в поле **"Ключевые** слова") **оператором AND.** Это означает, что элементы должны удовлетворять как запросу ключевого слова, так и условию для сохранения.

    Дополнительные сведения о создании поискового запроса и использовании условий см. в статье "Запросы по ключевым словам и условия поиска [контента".](keyword-queries-and-search-conditions.md)

10. После настройки удержания на основе запроса нажмите кнопку **"Далее".**

11. Просмотрите параметры (и при необходимости отредактируете их) и нажмите кнопку **"Создать это удержание".**

## <a name="query-based-holds-placed-on-site-documents"></a>Запросы, размещенные на документах сайта

При размещении удержания электронных данных на основе запросов на документах, расположенных на сайтах SharePoint, учитывайте следующие моменты:

- Удержание на основе запроса сначала сохраняет все документы на сайте в течение короткого периода времени после их удаления. Это означает, что при удалении документа он будет перемещен в архивную библиотеку, даже если он не соответствует критериям удержания на основе запроса. Однако удаленные документы, не соответствующие удержанию на основе запроса, будут удалены заданием времени, которое обрабатывает архивную библиотеку. Задание timer выполняется периодически и сравнивает все документы в архивной библиотеке с удержанием eDiscovery на основе запросов (и другими типами удержаний и политик хранения). Заданием времени удаляются документы, не соответствующие удержанию на основе запроса, и сохраняются соответствующие документы.

- Удержания на основе запросов не следует использовать для целевого сохранения, например сохранения документов в определенной папке или на сайте или с помощью других критериев хранения на основе расположения. Это может привести к непредвиденным результатам. Для сохранения документов сайта рекомендуется использовать условия удержания, не основанные на расположении, такие как ключевые слова, диапазоны дат или другие свойства документов.

## <a name="ediscovery-hold-statistics"></a>Статистика удержания при обнаружении электронных данных

После создания удержания для eDiscovery сведения о новом удержании отображаются на странице flyout для выбранного удержания. Эти сведения включают количество почтовых ящиков и сайтов на удержании, а также статистику по контенту, помещенного на удержание, например общее число и размер элементов, помещенных на удержание, и время последнего вычисления статистики удержания. Эти статистические данные удержания помогают определить объем контента, связанного с делом, который сохраняется.
  
![Статистика удержания](../media/575cfe0a-9210-4ae4-8df8-65665d66712e.png)
  
О статистике удержания при обнаружении электронных данных следует помнить следующее:
  
- Общее число элементов на удержании указывает количество элементов из всех источников контента, помещенных на удержание. Если вы создали удержание на основе запроса, эта статистика указывает количество элементов, которые соответствуют запросу.

- Число элементов на удержании также включает неndexed элементы, найденные в расположениях контента. При создании удержания на основе запроса все неиndexed элементы в расположениях контента помещаются на удержание. К ним относятся неndexed элементы, которые не соответствуют условиям поиска удержания на основе запроса, и неndexed элементы, которые могут не соответствовать условию диапазона дат. Это отличается от того, что происходит при запуске поиска, в котором неndexed элементы, которые не соответствуют поисковому запросу или исключены условием диапазона дат, не включаются в результаты поиска. Дополнительные сведения о неиндексациях элементов см. в подразделе ["Частично индексные элементы".](partially-indexed-items-in-content-search.md)

- Вы можете получить последнюю статистику  удержания, щелкнув "Обновить статистику", чтобы повторно провести оценку поиска, которая вычисляет текущее число элементов на удержании.

- Со временем количество элементов на удержании увеличивается, так как пользователи, чьи почтовые ящики или сайт находятся на удержании, обычно отправляют или получают новые сообщения электронной почты, а также создают новые документы в SharePoint и OneDrive.

- Если почтовый ящик Exchange, сайт SharePoint или учетная запись OneDrive перемещены в другой регион в среде с несколькими регионами, статистика для этого сайта не будет включена в статистику удержания. Но контент в этих расположениях по-прежнему будет сохранен. Кроме того, если почтовый ящик или сайт перемещается в другой регион, SMTP-адрес или URL-адрес, отображаемая в удержании, не будут автоматически обновлены. Вам придется изменить удержание и обновить URL-адрес или SMTP-адрес, чтобы расположения контента снова включались в статистику удержания

## <a name="search-locations-on-ediscovery-hold"></a>Поиск в расположениях с удержанием eDiscovery

При [поиске](search-for-content-in-core-ediscovery.md) контента в случае core eDiscovery можно быстро настроить поиск только для тех расположений контента, которые были помещены на удержание, связанное с делом.

![Расположения на удержании](../media/d56398aa-0b20-4500-8e26-494eab92a99f.png)

Выберите параметр **"Расположения на удержании"** для поиска во всех расположениях контента, помещенных на удержание. Если дело содержит несколько функций eDiscovery, при выборе этого параметра будут искаться расположения контента из всех содержимого. Кроме того, если расположение контента было помещено на удержание на основе запроса, при запуске поиска будут искаться только элементы, которые соответствуют запросу на удержание. Другими словами, в результаты поиска возвращается только контент, который соответствует условиям удержания и условиям поиска. Например, если пользователь был помещен на удержание дела на основе запроса, которое сохраняет элементы, отправленные или созданные до определенной даты, поиск будет только в этих элементов. Это достигается путем подключения запроса удержания дела и поискового запроса оператором **AND.**

Вот некоторые другие моменты, которые необходимо помнить при поиске в расположениях с удержанием eDiscovery:

- Если расположение контента является частью нескольких удержаний в одном случае, запросы удержания объединяются операторами **OR** при поиске в этом расположении контента с использованием параметра содержимого "Все дела". Аналогичным образом, если расположение контента является частью двух разных удержаний, где один из них основан на запросах, а другой является бесконечным удержанием (где все содержимое помещается на удержание), то весь контент будет искаться из-за бесконечного удержания.

- Если поиск настроен для поиска расположений на удержании, а затем вы изменяете удержание eDiscovery в случае (путем добавления или удаления расположения или изменения запроса на удержание), конфигурация поиска обновляется с помощью этих изменений. Однако после изменения удержания необходимо повторно провести поиск, чтобы обновить результаты поиска.

- Если несколько удержаний при обнаружении электронных данных размещаются в одном расположении в деле eDiscovery и вы выбираете поиск в расположениях на удержании, максимальное число ключевых слов для этого поискового запроса равно 500. Это потому, что при поиске совмещаются все запросы с помощью оператора **OR.** Если в объединенных запросах удержания и поисковом запросе содержится более 500 ключевых слов, поиск будет проводиться не только в содержимом, которое соответствует запросу.

- Если удержание для eDiscovery имеет состояние "Включено", вы по-прежнему можете искать расположения на удержании, пока удержание включено.

## <a name="preserve-content-in-microsoft-teams"></a>Сохранение контента в Microsoft Teams

Беседы, которые являются частью канала Microsoft Teams, хранятся в почтовом ящике, связанном с командой Microsoft Team. Файлы, которыми обмениваются участники команды, также сохраняются на сайте команды SharePoint. Поэтому для сохранения бесед и файлов в канале необходимо разместить почтовый ящик группы и сайт SharePoint на удержание для eDiscovery.

Кроме того, беседы, которые являются частью списка чата в Teams (называемые *чатыми 1:1* или групповыми чатами *1:N),* хранятся в почтовых ящиках пользователей, которые участвуют в чате. Файлы, к которым пользователи делятся в беседах чата, хранятся в учетной записи OneDrive пользователя, который делится файлом. Поэтому необходимо добавить отдельные почтовые ящики пользователей и учетные записи OneDrive в удержание для eDiscovery, чтобы сохранить беседы и файлы в списке чатов. Кроме помещения почтового ящика и сайта группы на удержание, на почтовые ящики участников команды Майкрософт лучше поставить на удержание почтовые ящики и сайт группы.

> [!IMPORTANT]
> В облачной организации пользователи, которые участвуют в беседах, входящих в список чатов в Teams, должны иметь почтовый ящик Exchange Online, чтобы сохранять беседы чата при удержании почтового ящика при обнаружении электронных данных. Это потому, что беседы, входящие в список чатов, хранятся в облачных почтовых ящиках участников чата. Если у участника чата нет почтового ящика Exchange Online, вы не сможете сохранить эти беседы. Например, в гибридном развертывании Exchange пользователи с почтовым ящиком могут участвовать в беседах, входящих в список чатов в Teams. Но в этом случае содержимое из этих бесед не может быть сохранено, так как у этих пользователей нет облачных почтовых ящиков, которые можно поместить на удержание.

Дополнительные сведения о сохранении контента Teams см. в поддержимом пользователя [или команды Microsoft Teams.](https://docs.microsoft.com/MicrosoftTeams/legal-hold)

### <a name="preserve-card-content"></a>Сохранение содержимого карточки

Аналогичным образом содержимое карточки, созданные приложениями в каналах Teams, чатах 1:1 и групповых чатах 1:N, сохраняется в почтовых ящиках и сохраняется при удержании почтового ящика при обнаружении электронных данных. *Карточка* — это контейнер пользовательского интерфейса для коротких фрагментов содержимого. Карточки могут иметь несколько свойств и вложений, а также могут включать кнопки, запуская действия карт. Дополнительные сведения см. в [карточках.](https://docs.microsoft.com/microsoftteams/platform/task-modules-and-cards/what-are-cards) Как и в других материалах Teams, содержимое карточки хранится в зависимости от того, где была использована карточка. Содержимое карточек, используемых в канале Teams, хранится в почтовом ящике группы Teams. Содержимое карточки для чатов 1:1 и 1xN хранится в почтовых ящиках участников чата.

### <a name="preserve-meeting-and-call-information"></a>Сохранение сведений о собраниях и звонках

Сводная информация о собраниях и звонках в канале Teams также хранится в почтовых ящиках пользователей, набравших номер для собрания или звонка. Это содержимое также сохраняется при удержании eDiscovery для почтовых ящиков пользователей.

### <a name="preserve-content-in-private-channels"></a>Сохранение содержимого в частных каналах

Начиная с февраля 2020 г. мы также включили возможность сохранения содержимого в частных каналах. Так как чаты частного канала хранятся в почтовых ящиках участников чата, помещение почтового ящика пользователя на удержание при обнаружении электронных данных сохранит чаты частного канала. Кроме того, если почтовый ящик пользователя был помещен на удержание для eDiscovery до февраля 2020 г., удержание теперь будет автоматически применяться к частным каналам сообщений, хранимых в этом почтовом ящике. Также поддерживается сохранение файлов, общий доступ к файлам в частных каналах.

### <a name="preserve-wiki-content"></a>Сохранение вики-содержимого

Каждый канал команды или команды также содержит вики-сайт для принятия заметок и совместной работы. Содержимое вики-сайта автоматически сохраняется в файле формата MHT. Этот файл хранится в библиотеке документов Teams Wiki Data на сайте команды в SharePoint. Вы можете сохранить вики-контент, добавив сайт SharePoint команды в удержание eDiscovery.

> [!NOTE]
> Возможность сохранения вики-контента для канала команды или команды (при удержании сайта sharePoint команды) была выпущена 22 июня 2017 г. Если сайт группы находится на удержании, содержимое вики-сайта будет храниться начиная с этой даты. Однако если сайт группы находится на удержании и вики-контент был удален до 22 июня 2017 г., вики-контент не был сохранен.

### <a name="office-365-groups"></a>Группы Office 365

Teams создан на основе групп Office 365. Таким образом, помещение групп Office 365 на удержание для eDiscovery аналогично удержанию контента Teams.

При размещении групп Teams и Office 365 на удержание для eDiscovery следует иметь в виду следующее:

- Как объяснялось ранее, для удержания контента, расположенного в Teams и группах Office 365, необходимо указать почтовый ящик и сайт SharePoint, связанные с группой или командой.

- Запустите **командлет Get-UnifiedGroup** в [Exchange Online PowerShell,](https://docs.microsoft.com/powershell/exchange/connect-to-exchange-online-powershell) чтобы просмотреть свойства teams и групп Office 365. Это хороший способ получить URL-адрес сайта, связанного с командой или группой Office 365. Например, следующая команда отображает выбранные свойства для группы Office365 с именем "Senior Leadership Team":

    ```text
    Get-UnifiedGroup "Senior Leadership Team" | FL DisplayName,Alias,PrimarySmtpAddress,SharePointSiteUrl

    DisplayName            : Senior Leadership Team
    Alias                  : seniorleadershipteam
    PrimarySmtpAddress     : seniorleadershipteam@contoso.onmicrosoft.com
    SharePointSiteUrl      : https://contoso.sharepoint.com/sites/seniorleadershipteam
    ```

    > [!NOTE]
    > Чтобы запустить командлет **Get-UnifiedGroup**, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена. 
  
- При поиске в почтовом ящике пользователя поиск в любой группе Team или Office 365, в которую входит пользователь, не будет искаться. Аналогичным образом, когда вы помещаете группу или группу Office 365 на удержание для eDiscovery, на удержание помещаются только почтовый ящик группы и сайт группы. Почтовые ящики и сайты OneDrive для бизнеса участников группы не помещаются на удержание, если вы явно не добавите их в удержание для eDiscovery. Поэтому, если вам нужно по юридическим причинам разместить группу Team или Office 365 на удержание, рассмотрите возможность добавления почтовых ящиков и учетных записей OneDrive участников группы или группы на удержание.

- Чтобы получить список участников группы или группы Office 365, вы можете  просмотреть свойства на странице "Группы" в Центре администрирования Microsoft 365. Или можно выполнить следующую команду в Exchange Online PowerShell:

    ```powershell
    Get-UnifiedGroupLinks <group or team name> -LinkType Members | FL DisplayName,PrimarySmtpAddress
    ```

    > [!NOTE]
    > Чтобы запустить командлет **Get-UnifiedGroupLinks**, в Exchange Online вам должна быть назначена роль "получатели только для чтения" или вы должны входить в группу пользователей, которым она назначена.

## <a name="preserve-content-in-onedrive-accounts"></a>Сохранение содержимого в учетных записях OneDrive

Чтобы собрать список URL-адресов сайтов OneDrive для бизнеса в организации, чтобы добавить их в удержание или поиск, связанный с делом eDiscovery, см. статью "Создание списка всех местоположений [OneDrive](https://docs.microsoft.com/onedrive/list-onedrive-urls)в организации". Сценарий в этой статье создает текстовый файл, содержащий список всех сайтов OneDrive в организации. Чтобы запустить этот скрипт, требуется установить и использовать командную консоль SharePoint Online. Не забудьте добавить URL-адрес домена личного сайта вашей организации к каждому сайту OneDrive, на котором нужно выполнить поиск. Это домен, содержащий все хранилище OneDrive. Пример: `https://contoso-my.sharepoint.com`. Вот пример URL-адреса для сайта OneDrive пользователя: `https://contoso-my.sharepoint.com/personal/sarad_contoso_onmicrosoft.com`.

> [!IMPORTANT]
> URL-адрес учетной записи OneDrive пользователя включает его имя-пользователя (UPN) `https://alpinehouse-my.sharepoint.com/personal/sarad_alpinehouse_onmicrosoft_com` (например). В редких случаях, когда имя upN человека меняется, его URL-адрес OneDrive также меняется для включения нового upN. Если учетная запись OneDrive пользователя является частью удержания при обнаружении электронных данных, старая учетная запись пользователя изменена, необходимо обновить удержание, а также добавить новый URL-адрес OneDrive пользователя и удалить старый. Дополнительные сведения см. в разделе [Как изменения UPN влияют на URL-адрес OneDrive](https://docs.microsoft.com/onedrive/upn-changes).

## <a name="removing-content-locations-from-an-ediscovery-hold"></a>Удаление местоположений контента из удержания eDiscovery

После удаления почтового ящика, сайта SharePoint или учетной записи OneDrive из удержания для eDiscovery применяется удержание *задержки.* Это означает, что фактическое удаление удержания откладывается на 30 дней, чтобы предотвратить окончательное удаление (удаление) данных из расположения контента. Это дает администраторам возможность искать или восстанавливать содержимое, которое будет удалено после снятия удержания eDiscovery. Сведения о работе удержания задержки для почтовых ящиков и сайтов отличаются.

- **Почтовые ящики:** Удержание задержки помещается на почтовый ящик при следующей обработке почтового ящика помощником для управляемых папок и обнаружении того, что удержание при обнаружении электронных сообщений было удалено. В частности, удержание задержки применяется к почтовому ящику, когда помощник для управляемых папок задает для одного из следующих свойств почтового ящика **true:**

   - **DelayHoldApplied:** Это свойство применяется к содержимому, связанному с электронной почтой (созданным пользователями Outlook и Outlook в Интернете), который хранится в почтовом ящике пользователя.

   - **DelayReleaseHoldApplied:** Это свойство применяется к облачному содержимому (которое создается приложениями, не относя такими приложениями Outlook, как Microsoft Teams, Microsoft Forms и Microsoft Yammer), хранимым в почтовом ящике пользователя. Облачные данные, созданные приложением Майкрософт, обычно хранятся в скрытой папке в почтовом ящике пользователя.

   Если для почтового ящика заложено удержание задержки (если для одного из предыдущих свойств установлено **true),** почтовый ящик по-прежнему считается на удержании в течение неограниченного срока, как если бы почтовый ящик удерживался для судебного разбирательства. По истечении 30 дней срок удержания истекает, и Microsoft 365 автоматически попытается удалить удержание задержки (задав для свойства DelayHoldApplied или DelayReleaseHoldApplied свойство **False),** чтобы удержание было удалено. После того как для любого из этих свойств задается **false,** соответствующие элементы, помеченные для удаления, будут удалены при следующей обработке почтового ящика помощником для управляемых папок.

   Дополнительные сведения см. в разделе [Управление почтовыми ящиками при удержании с задержкой](identify-a-hold-on-an-exchange-online-mailbox.md#managing-mailboxes-on-delay-hold).

- **Сайты SharePoint и OneDrive:** Все содержимое SharePoint или OneDrive, которое сохраняется в архивной библиотеке, не удаляется в течение 30-дневного периода удержания с задержкой после удаления сайта из удержания при обнаружении электронных данных. Это аналогично тому, что происходит, когда сайт выходит из политики хранения. Кроме того, вы не можете вручную удалить это содержимое в архивной библиотеке в течение 30-дневного периода удержания. 

   Дополнительные сведения см. [в публикации политики хранения.](retention.md#releasing-a-policy-for-retention)

Удержание с задержкой также применяется к расположениям контента на удержании при закрытии основного дела eDiscovery, так как удержания отключаются при закрытии дела. Дополнительные сведения о закрытии дела см. в сведениях о закрытии, повторном открытии и удалении основного дела [eDiscovery.](close-reopen-delete-core-ediscovery-cases.md)
