# <a name="run-the-process-module-and-load-data-in-advanced-ediscovery-classic"></a>Запуск модуля process и загрузка данных в Advanced eDiscovery (классическая)

> [!NOTE]
> Чтобы можно было использовать Advanced eDiscovery, требуется подписка на Office 365 E3 с надстройкой Advanced Compliance или E5 для организации. Если у вас этого плана нет и вы хотите попробовать Advanced eDiscovery, можете [зарегистрироваться для получения пробной версии Office 365 корпоративный E5](https://go.microsoft.com/fwlink/p/?LinkID=698279). 
  
В этом разделе описаны функции модуля advanced eDiscovery. 
  
Помимо данных файлов, метаданные, такие как тип файла, расширение, расположение или путь, дата и время создания, автор, хранитеель и тема, можно загрузить в Advanced eDiscovery и сохранить для каждого случая. Некоторые метаданные вычисляются с помощью Advanced eDiscovery, например при загрузке файлов native. 
  
Advanced eDiscovery предоставляет значения системных метаданных, например почти дубликаты групп или оценки релевантности. Администратор может добавить другие метаданные, например примечания к файлам. 
  
## <a name="running-process"></a>Запущенный процесс

> [!NOTE]
> Пакетные номера назначены файлу во время процесса, чтобы разрешить отслеживание файлов. Номер пакета также позволяет идентификации пакетов процессов для параметров повторной обработки. Для фильтрации по номерам пакетов и сеансам доступны дополнительные фильтры. 
  
Чтобы запустить процесс, выполните следующие действия.
  
1. [Открытие службы безопасности &amp; Центр соответствия требованиям.](go-to-the-securitycompliance-center.md) 
    
2. Go to **Search &amp; investigation** \> **eDiscovery** and then click **Go to Advanced eDiscovery**.
    
3. В Advanced eDiscovery выберите соответствующий случай  на странице "Дела" и нажмите кнопку **"Перейти к делу".**
    
4. В **процессе** \>  \> **подготовки программы** установки выберите контейнер из списка доступных контейнеров.
    
    ![Щелкните "Процесс", чтобы добавить результаты поиска в дело](../media/50bdc55c-d378-4881-b302-31ef785fa359.png)
  
5. Нажмите **кнопку "Дополнительные параметры"...** Если вы хотите добавить контейнер в качестве файлов с замещями или в качестве предварительно помеченных файлов. 
    
    Используйте файлы с заданными данными, чтобы ускорить обучение по вопросам с низкой насыщенностью (как правило, 2 % или меньше). Для файлов с заданными значениями рекомендуется выбирать различные файлы и обрабатывать около 20–50 семечелей на проблему (слишком большое количество файлов с заместью релевантности может привести к перекосу результатов релевантности). Файлы с заданными файлами должны быть просмотрены тем же человеком, который обучает проблему.
    
    Используйте предварительно помеченные файлы для автоматизации обучения релевантности. Необходимо пометить по крайней мере 1500 файлов и сохранить пропорции релевантных файлов так же, как и в коллекции, добавленной к релевантности. Эти файлы должны быть помечены вручную, и вы должны быть уверены в качестве тегов.
    
    ![Снимок экрана: страница "Дополнительные параметры" для обработки пакетных файлов](../media/3c25cb78-4484-41e5-bd34-3753c7ab6cf2.jpg)
  
  - В разделе **"Seed":** 
    
    Выберите **"Пометить как seed files",** чтобы пометить контейнер как файлы для хранения. Вам также необходимо назначить их для каждого вопроса в выпадаемом ок **недоуме.** Выберите **"Релевантно"** или "Не **релевантно"** в **выпадаемом теге.** 
    
    > [!NOTE]
    > После того как вы настроили файлы как **seed,** вы не сможете пометить их как **предварительно помеченные.** 
  
  - В разделе **предварительно помеченных файлов:** 
    
    Выберите **Mark как предварительно помеченные файлы,** чтобы пометить контейнер как предварительно помеченные файлы. Вам также необходимо назначить их для каждого вопроса из **выпадаемого переназначаемого** выпуска "For issue". Выберите **"Релевантно"** или "Не **релевантно"** в **выпадаемом теге.** 
    
    > [!NOTE]
    > После того как вы настроили файлы как **предварительно помеченные,** вы не сможете пометить их как **seed**. 
  
  - В разделе **тегов электронной почты.** установите, какая часть обработанного сообщения электронной почты будет помечена как "Seed" или "Pre-tagged". 
    
6. Для начала щелкните **"Процесс".** По завершению отображаются результаты процесса.
    
7. (Необязательно) Если вам нужно назначить источники данных определенному хранителю, вы можете добавить и изменить имена хранителей в средстве **Custodians** Manage и назначить хранителей в \>  **custodians** \> **Assign.** 
    
При добавлении в дело можно повторно обработать.
