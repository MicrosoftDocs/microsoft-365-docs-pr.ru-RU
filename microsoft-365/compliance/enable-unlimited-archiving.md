# <a name="enable-unlimited-archiving---admin-help"></a>Включить неограниченный архив — справка для администраторов

Вы можете использовать функцию архивирования с автоматическим расширением Exchange Online, чтобы включить неограниченное пространство для архивных почтовых ящиков. Если включено автоматическое расширение архивирования, при приближении к предельному объему хранилища в архивный почтовый ящик пользователя автоматически добавляется дополнительное место. В результате объем хранилища почтовых ящиков будет неограниченным. Вы можете включить автоматическое расширение архива для всех пользователей в организации или только для определенных пользователей. Дополнительные сведения об автоматическом расширении архивных данных см. в обзоре неограниченной архиватуры [в Office 365.](unlimited-archiving.md)

## <a name="before-you-enable-auto-expanding-archiving"></a>Перед тем как включить автоматическое расширение архива

- Чтобы включить автоматическое расширение архива для всей организации или отдельных пользователей, необходимо быть глобальным администратором в организации или членом группы ролей "Управление организацией" в организации Exchange Online. Кроме того, вы должны быть членом группы ролей, которая назначена роль "Получатели почты", чтобы включить автоматическое расширение архива для определенных пользователей.

- Для включения автоматического расширения архива необходимо включить архивный почтовый ящик пользователя. Чтобы включить архивный почтовый ящик, пользователю должна быть назначена лицензия на Exchange Online (план 2). Если пользователю назначена лицензия на Exchange Online (план 1), необходимо назначить ему отдельную лицензию архивация на базе Exchange Online, чтобы включить архивный почтовый ящик. См. ["Включить архивные почтовые ящики" в](enable-archive-mailboxes.md)Центре & соответствия требованиям.

- Вы также можете включить архивные почтовые ящики с помощью PowerShell. Пример [команды](#more-information) PowerShell, которая позволяет включить архивные почтовые ящики для всех пользователей в организации, см. в разделе "Дополнительные сведения".

- Автоматически расширяющиеся архивы также доступны для общих почтовых ящиков. Чтобы включить архив для общего почтового ящика, необходима лицензия exchange Online (план 2) или Exchange Online (план 1) с архивация на базе Exchange Online лицензии.

- Автоматическое расширение архива предотвращает восстановление или восстановление [неактивного почтового ящика.](inactive-mailboxes-in-office-365.md#what-are-inactive-mailboxes) Это означает, что если включить автоматическое расширение архивации для почтового ящика и он станет неактивным [](recover-an-inactive-mailbox.md) позднее, вы не сможете восстановить неактивный почтовый ящик (преобразовав его в активный) или восстановить [его](restore-an-inactive-mailbox.md) (путем объединения содержимого в существующий почтовый ящик). Если для неактивного почтового ящика включено автоматическое расширение архива, единственным способом восстановления данных является использование средства поиска контента в Центре соответствия требованиям Microsoft 365 для экспорта данных из почтового ящика и импорта в другой почтовый ящик. Дополнительные сведения см. в разделе "Неактивные почтовые ящики и автоматически расширяющиеся архивы" статьи "Обзор неактивных [почтовых ящиков".](inactive-mailboxes-in-office-365.md#inactive-mailboxes-and-auto-expanding-archives)

- Для автоматического расширения архива нельзя использовать Центр администрирования Exchange или Центр & соответствия требованиям. Необходимо использовать Exchange Online PowerShell. Чтобы подключиться к организации Exchange Online с помощью удаленной powerShell, см. "Подключение [к Exchange Online PowerShell".](https://go.microsoft.com/fwlink/p/?linkid=396554)

## <a name="enable-auto-expanding-archiving-for-your-entire-organization"></a>Включить автоматическое расширение архива для всей организации

Вы можете включить автоматическое расширение архива для всей организации. После включения функции автоматическое расширение архива будет включено для существующих почтовых ящиков пользователей и для новых созданных почтовых ящиков пользователей. При создании почтовых ящиков пользователей не забудьте включить основной архивный почтовый ящик пользователя, чтобы функция автоматического расширения архива была работать для нового почтового ящика пользователя.
  
1. [Подключение к PowerShell Exchange Online](https://go.microsoft.com/fwlink/p/?linkid=396554)

2. Чтобы включить автоматическое расширение архива для всей организации, запустите следующую команду в Exchange Online PowerShell.

    ```powershell
    Set-OrganizationConfig -AutoExpandingArchive
    ```

## <a name="enable-auto-expanding-archiving-for-specific-users"></a>Включить автоматическое расширение архива для определенных пользователей

Вместо включения функции автоматического расширения архивов для каждого пользователя в организации ее можно включить только для определенных пользователей. Это можно сделать, так как только некоторым пользователям может потребоваться большой объем архивного хранилища.
  
Когда вы включаете автоматическое расширение архивации для определенного пользователя и почтового ящика пользователя на удержании или назначены политике хранения, в конфигурацию внося следующие два изменения:
  
- Квота хранилища для основного архивного почтового ящика пользователя увеличена на 10 ГБ (с 100 ГБ до 110 ГБ). Квота предупреждения для архива также увеличена на 10 ГБ (с 90 ГБ до 100 ГБ).

- Квота хранилища для папки "Элементы с возможностью восстановления" в основном почтовом ящике пользователя увеличена на 10 ГБ (также с 100 ГБ до 110 ГБ). Квота предупреждения элементов для восстановления также увеличена на 10 ГБ (с 90 ГБ до 100 ГБ). Эти изменения применяются только в том случае, если почтовый ящик на удержании или назначен политике хранения.

Это дополнительное пространство добавляется для предотвращения проблем с хранилищем, которые могут возникнуть перед подготовкаю архива с автоматическим расширением. При встраии  *автоматического*  расширения архивирования для всей организации дополнительное место в хранилище не добавляется, как описано в предыдущем разделе.
  
1. [Подключение к PowerShell Exchange Online](https://go.microsoft.com/fwlink/p/?linkid=396554)

2. Чтобы включить автоматическое расширение архива для определенного пользователя, в Exchange Online PowerShell запустите следующую команду. Как объяснялось ранее, архивный почтовый ящик пользователя (основной архив) необходимо включить, прежде чем можно будет включить автоматическое расширение архива для этого пользователя.

    ```powershell
    Enable-Mailbox <user mailbox> -AutoExpandingArchive
    ```

> [!IMPORTANT]
> В гибридном развертывании Exchange нельзя использовать команду **Enable-Mailbox -AutoExpandingArchive,** чтобы включить автоматическое расширение архивации для определенного пользователя, основной почтовый ящик которого находится локально, а архивный почтовый ящик — облачный. Чтобы включить автоматическое расширение архивации для облачных архивных почтовых ящиков в гибридном развертывании Exchange, необходимо выполнить команду **Set-OrganizationConfig -AutoExpandingArchive** в Exchange Online PowerShell, чтобы включить автоматическое расширение архивации для всей организации. Если основные и архивные почтовые ящики пользователя являются облачными, можно использовать команду **Enable-Mailbox -AutoExpandingArchive,** чтобы включить автоматическое расширение архива для этого пользователя.
  
## <a name="verify-that-auto-expanding-archiving-is-enabled"></a>Проверка включения автоматического расширения архива

Чтобы убедиться, что для вашей организации включена архивативная архива с автоматическим расширением, запустите следующую команду в Exchange Online PowerShell.

```powershell
Get-OrganizationConfig | FL AutoExpandingArchiveEnabled
```

Значение указывает, что для организации включено автоматическое  `True` расширение архива. 
  
Чтобы убедиться, что для определенного пользователя включена архивная архива с автоматическим расширением, запустите следующую команду в Exchange Online PowerShell.
  
```powershell
Get-Mailbox <user mailbox> | FL AutoExpandingArchiveEnabled
```

Значение указывает, что для пользователя включена архивная  `True` архива с автоматическим расширением.
  
Чтобы определить, включена ли автоматическое расширение архива для неактивных почтовых ящиков, запустите следующую команду в Exchange Online PowerShell.
  
```powershell
Get-Mailbox -InactiveMailboxOnly | FL UserPrincipalName,AutoExpandingArchiveEnabled
```

Значение указывает, что для неактивного почтового ящика включена автоматическое расширение  `True` архива. Значение указывает, что автоматическое расширение архива `False` не включено.

После автоматического расширения архива следует иметь в виду следующее:
  
- Если вы запустите команду **Set-OrganizationConfig -AutoExpandingArchive,** чтобы включить автоматическое расширение архива для организации, вам не нужно запускать **enable-Mailbox -AutoExpandingArchive** для отдельных почтовых ящиков. При запуске для организации автоматического расширения архивных данных с помощью набора архивов **set-OrganizationConfig** свойство  *AutoExpandingArchiveEnabled*  в почтовых ящиках пользователей не меняется на `True` .

- Аналогичным образом, значения свойств почтового ящика  *ArchiveQuota*  и  *ArchiveWarningQuota*  не меняются при встраии в архивирование с автоматическим расширением. На самом деле, если включить автоматическое расширение архивирования для почтового ящика пользователя и для свойства  *AutoExpandingArchiveEnabled*  установлено такое же свойство, свойства  `True`  *ArchiveQuota*  и  *ArchiveWarningQuota*  будут игнорироваться. Вот пример этих свойств почтового ящика после включения автоматического расширения архива для почтового ящика пользователя. 

    ![Свойства ArchiveQuota и ArchiveWarningQuota игнорируются после автоматического расширения архивирования](../media/6a1c1b69-5c4c-4267-aac8-53577667f03e.png)

## <a name="more-information"></a>Дополнительная информация

- Вы также можете включить архивные почтовые ящики с помощью PowerShell. Например, можно выполнить следующую команду в Exchange Online PowerShell, чтобы включить архивные почтовые ящики для всех пользователей, архивный почтовый ящик которых еще не включен.

    ```powershell
    Get-Mailbox -Filter {ArchiveStatus -Eq "None" -AND RecipientTypeDetails -eq "UserMailbox"} | Enable-Mailbox -Archive
    ```

- После того как вы включите автоматическое расширение архивации для вашей организации или для определенного пользователя, архивный почтовый ящик преобразуется в архив автоматического расширения, когда архивный почтовый ящик (включая папку "Элементы с возможностью восстановления") достигает 90 ГБ. Подготовка дополнительного пространства для хранения может занять до 30 дней.

- После того как вы включите автоматическое расширение архива, его нельзя отключить. Кроме того, администраторы не могут настраивать квоту хранилища для автоматического расширения архивирования.

- Автоматическое расширение архивации поддерживается для облачных архивных почтовых ящиков в гибридном развертывании Exchange для пользователей с основным почтовым ящиком в локальной сети. Однако после включения автоматического расширения архива для облачного архивного почтового ящика вы не сможете вернуть этот архивный почтовый ящик в локальной организации Exchange. Автоматическое расширение архива не поддерживается для локального почтового ящика в любой версии Exchange Server.

- Список клиентов Outlook, которые пользователи могут использовать для доступа к элементов в дополнительном хранилище в архивном почтовом ящике, см. в разделе "Требования Outlook для доступа к элементов в архиве с автоматическим расширением" в разделе ["Обзор](unlimited-archiving.md#outlook-requirements-for-accessing-items-in-an-auto-expanded-archive)неограниченной архивирования".

- Как объяснялось ранее, при запуске команды **Enable-Mailbox -AutoExpandingArchive** к квоте хранилища основного архивного почтового ящика пользователя (и к папке "Элементы с возможностью восстановления", если почтовый ящик находится на удержании), добавляется 10 ГБ. Это обеспечивает дополнительное хранилище до тех пор, пока не будет подойтогано автоматически расширенное место хранения (что может занять до 30 дней). Это дополнительное место не добавляется при запуске **set-OrganizationConfig -AutoExpandingArchive** для автоматического расширения архивирования для всех почтовых ящиков в организации. Если включено автоматическое расширение архивирования для всей организации, но необходимо добавить дополнительные 10 ГБ пространства для определенного пользователя, можно выполнить команду **Enable-Mailbox -AutoExpandingArchive** для этого почтового ящика. Вы получите сообщение об ошибке, что автоматическое расширение архивирования уже включено, но в почтовый ящик будет добавлено дополнительное место для хранения.

> [!IMPORTANT]
> Автоматическое расширение архивирования поддерживается только для почтовых ящиков, используемых для отдельных пользователей или общих почтовых ящиков с скоростью роста не более 1 ГБ в день. Использование журналов, правил транспорта или правил автоматической пересылки для копирования сообщений в архивный почтовый ящик для архива запрещено. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Корпорация Майкрософт оставляет за собой право отказать в неограниченном архивном архиве в тех случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных других пользователей или в других случаях ненадлежащего использования.
