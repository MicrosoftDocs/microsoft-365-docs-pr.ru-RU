# <a name="enable-unlimited-archiving---admin-help"></a>Включение неограниченной архивации — Справка для администраторов

Вы можете использовать функцию автоматического развертывания Exchange Online для включения неограниченного дискового пространства для архивных почтовых ящиков. Если включено автоматическое расширение архивации, дополнительное дисковое пространство автоматически добавляется в архивный почтовый ящик пользователя, если он приближается к максимальному объему хранилища. В результате будет неограниченный объем хранилища почтовых ящиков. Вы можете включить автоматическое расширение архивации для всех пользователей в организации или только для определенных пользователей. Дополнительные сведения об автоматическом развертывании архивации приведены [в статье Обзор неограниченного архивирования в Office 365](unlimited-archiving.md).

## <a name="before-you-enable-auto-expanding-archiving"></a>Перед включением автоматического развертывания архивации

- Необходимо быть глобальным администратором в организации или членом группы ролей Управление организацией в организации Exchange Online, чтобы включить автоматическое расширение архивации для всей организации или для определенных пользователей. Кроме того, необходимо быть участником группы ролей, которой назначена роль получателей почты, чтобы включить автоматическое расширение архивации для определенных пользователей.

- Архивный почтовый ящик пользователя должен быть включен, прежде чем вы сможете включить автоматическое расширение архивации. Для включения архивного почтового ящика пользователю должна быть назначена лицензия на Exchange Online (план 2) Exchange Online. Если пользователю назначена лицензия на Exchange Online (план 1), необходимо назначить ему отдельную лицензию архивации на базе Exchange Online, чтобы включить архивный почтовый ящик. [В разделе Включение архивных почтовых ящиков в центре безопасности & соответствия требованиям](enable-archive-mailboxes.md).

- Вы также можете использовать PowerShell для включения архивных почтовых ящиков. В разделе [Дополнительные сведения](#more-information) вы найдете пример команды PowerShell, которую можно использовать для включения архивных почтовых ящиков для всех пользователей в Организации.

- Автоматически расширяющиеся архивы также доступны для общих почтовых ящиков. Для включения архива для общего почтового ящика требуется лицензия на Exchange Online (план 2) или лицензию на Exchange Online (план 1) с лицензией на архивацию на базе Exchange Online.

- Вы не можете использовать центр администрирования Exchange или центр соответствия требованиям & безопасности, чтобы включить автоматическое расширение архивации. Необходимо использовать Exchange Online PowerShell. Чтобы подключиться к организации Exchange Online с помощью удаленной оболочки PowerShell, ознакомьтесь со статьей [Подключение к Exchange Online PowerShell](https://go.microsoft.com/fwlink/p/?linkid=396554).

## <a name="enable-auto-expanding-archiving-for-your-entire-organization"></a>Включение автоматического развертывания архивации для всей Организации

Вы можете включить автоматическое расширение архивации для всей Организации. После его включения автоматически развернутая Архивация будет включена для существующих почтовых ящиков пользователей и новых почтовых ящиков пользователей, которые были созданы. Когда вы создаете почтовые ящики пользователей, обязательно включите основной архивный почтовый ящик пользователя, чтобы автоматически развернутая функция архивации работала для нового почтового ящика пользователя.
  
1. [Подключение к PowerShell Exchange Online](https://go.microsoft.com/fwlink/p/?linkid=396554)

2. Выполните следующую команду в Exchange Online PowerShell, чтобы включить автоматическое расширение архивации для всей Организации.

    ```powershell
    Set-OrganizationConfig -AutoExpandingArchive
    ```

## <a name="enable-auto-expanding-archiving-for-specific-users"></a>Включение автоматического развертывания архивирования для определенных пользователей

Вместо включения автоматического расширения архивации для каждого пользователя в организации его можно включить только для определенных пользователей. Это может быть вызвано тем, что для большого объема архивного хранилища может понадобиться только несколько пользователей.
  
При включении автоматического расширения архивации для определенного пользователя и почтового ящика пользователя при удержании или назначении политики хранения выполняются следующие два изменения конфигурации:
  
- Квота хранилища основного архивного почтового ящика пользователя увеличивается на 10 ГБ (от 100 ГБ до 110 ГБ). Квота предупреждения для архива также увеличивается на 10 ГБ (от 90 ГБ до 100 ГБ).

- Квота хранилища для папки "элементы с возможностью восстановления" в основном почтовом ящике пользователя увеличивается на 10 ГБ (также от 100 ГБ до 110 ГБ). Квота предупреждения для восстанавливаемых элементов также увеличивается на 10 ГБ (от 90 ГБ до 100 ГБ). Эти изменения применяются только в том случае, если почтовый ящик находится на удержании или назначен политике хранения.

Это дополнительное пространство добавляется для предотвращения проблем с хранилищем, которые могут возникнуть до подготовки автоматически развертываемого архива. Дополнительное дисковое пространство *не* добавляется при включении автоматического развертывания архивации для всей Организации, как описано в предыдущем разделе.
  
1. [Подключение к PowerShell Exchange Online](https://go.microsoft.com/fwlink/p/?linkid=396554)

2. Выполните следующую команду в Exchange Online PowerShell, чтобы включить автоматическое расширение архивации для определенного пользователя. Как было сказано ранее, архивный почтовый ящик пользователя (основной архив) должен быть включен, прежде чем вы сможете включить автоматическое расширение архивации для этого пользователя.

    ```powershell
    Enable-Mailbox <user mailbox> -AutoExpandingArchive
    ```

> [!IMPORTANT]
> В гибридном развертывании Exchange невозможно использовать команду **Enable-Mailbox-аутоекспандингарчиве** , чтобы включить автоматическое расширение архивации для определенного пользователя, основной почтовый ящик которого является локальным, а архивный почтовый ящик — на основе облака. Чтобы включить автоматическое расширение архивации для облачных архивных почтовых ящиков в гибридном развертывании Exchange, необходимо выполнить команду **Set-OrganizationConfig-аутоекспандингарчиве** в Exchange Online PowerShell, чтобы включить автоматическое расширение архивации для всей Организации. Если основной и архивный почтовые ящики пользователя размещаются в облаке, вы можете использовать команду **Enable/Mailbox – аутоекспандингарчиве** , чтобы включить автоматическое расширение архивации для определенного пользователя.
  
## <a name="verify-that-auto-expanding-archiving-is-enabled"></a>Проверка включения автоматического развертывания архивации

Чтобы убедиться, что автоматическое расширение архивации включено для вашей организации, выполните следующую команду в Exchange Online PowerShell.

```powershell
Get-OrganizationConfig | FL AutoExpandingArchiveEnabled
```

Значение указывает на то `True` , что автоматическое расширение архивации включено для Организации. 
  
Чтобы убедиться, что автоматическое расширение архивации включено для определенного пользователя, выполните следующую команду в Exchange Online PowerShell.
  
```powershell
Get-Mailbox <user mailbox> | FL AutoExpandingArchiveEnabled
```

Значение указывает на то `True` , что автоматическое расширение архивации включено для пользователя. 
  
После включения автоматического развертывания архивации следует учитывать следующие моменты.
  
- Если выполнить команду **Set – OrganizationConfig – аутоекспандингарчиве** , чтобы включить автоматическое расширение архивации для Организации, не нужно запускать командлет **Enable/Mailbox/аутоекспандингарчиве** в отдельных почтовых ящиках. Выполнение командлета **Set – OrganizationConfig** для включения автоматического развертывания архивации в Организации не приводит к изменению свойства *аутоекспандингарчивинаблед* для почтовых ящиков пользователей на `True` .

- Аналогично, значения свойств *ArchiveQuota* и *ArchiveWarningQuota* почтового ящика не изменяются при включении автоматического развертывания архивации. На самом деле, при включении автоматического развертывания архивации для почтового ящика пользователя, если свойство *аутоекспандингарчивинаблед* имеет значение `True` , свойства *ArchiveQuota* и *ArchiveWarningQuota* игнорируются. Ниже приведен пример свойств этого почтового ящика после включения автоматического развертывания архивации для почтового ящика пользователя.

## <a name="more-information"></a>Дополнительные сведения

- Вы также можете использовать PowerShell для включения архивных почтовых ящиков. Например, можно выполнить следующую команду в Exchange Online PowerShell, чтобы включить архивные почтовые ящики для всех пользователей, чей архивный почтовый ящик еще не включен.

    ```powershell
    Get-Mailbox -Filter {ArchiveStatus -Eq "None" -AND RecipientTypeDetails -eq "UserMailbox"} | Enable-Mailbox -Archive
    ```

- После включения автоматического расширения архивации для организации или определенного пользователя архивный почтовый ящик преобразуется в автоматически расширяемый Архив при достижении размера архивного почтового ящика (включая папку "элементы с возможностью восстановления") 90 ГБ. Для подготовки дополнительного дискового пространства может потребоваться до 30 дней.

- После включения автоматического развертывания архивации его невозможно отключить.

- Автоматическое расширение архивации поддерживается для облачных архивных почтовых ящиков в гибридном развертывании Exchange для пользователей с локальным основным почтовым ящиком. Однако после включения автоматического расширения архивации для облачного архивного почтового ящика его нельзя отархивировать обратно в локальную организацию Exchange. Автоматическое расширение архивации не поддерживается для локальных почтовых ящиков в Exchange Server 2010.

- Список клиентов Outlook, которые пользователи могут использовать для доступа к элементам в дополнительной области хранения в архивном почтовом ящике, приведен в разделе "требования к Outlook для доступа к элементам в автоматическом развернутом архиве" в разделе [Обзор неограниченной архивации](unlimited-archiving.md#outlook-requirements-for-accessing-items-in-an-auto-expanded-archive).

- Как описывалось ранее, 10 ГБ добавляются к квоте хранилища основного архивного почтового ящика пользователя (и в папку "элементы с возможностью восстановления", если почтовый ящик находится на удержании) при выполнении команды **Enable-Mailbox-аутоекспандингарчиве** . Это обеспечивает дополнительное хранилище до тех пор, пока не будет подготовлено автоматически расширенное дисковое пространство (которое может занять до 30 дней). Это дополнительное дисковое пространство не добавляется при запуске командлета **Set-OrganizationConfig-аутоекспандингарчиве** , чтобы включить автоматическое расширение архивации для всех почтовых ящиков в Организации. Если включено автоматическое расширение архивации для всей Организации, но необходимо добавить дополнительные 10 ГБ дискового пространства для определенного пользователя, можно выполнить команду **Enable-Mailbox-аутоекспандингарчиве** в этом почтовом ящике. Вы получите сообщение об ошибке, сообщающее о том, что автоматическое расширение архивации уже включено, но в почтовый ящик будет добавлено дополнительное место для хранения.

- Администраторы не могут настраивать квоту хранилища.

> [!IMPORTANT]
> Автоматическое расширение архивации поддерживается только для тех почтовых ящиков, которые используются для отдельных пользователей или общих почтовых ящиков, а скорость роста не превышает 1 ГБ в день. Использование операций ведения журнала, правил транспорта или правил автоматической пересылки для копирования сообщений в архивный почтовый ящик в целях архивации не разрешено. Архивный почтовый ящик пользователя предназначен только для этого пользователя. Корпорация Майкрософт оставляет за собой право отказаться от неограниченного архивирования в случаях, когда архивный почтовый ящик пользователя используется для хранения архивных данных для других пользователей или в других случаях недопустимого использования.
