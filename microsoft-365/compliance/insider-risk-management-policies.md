# <a name="insider-risk-management-policies"></a>Политики управления рисками для предварительной оценки

Политики управления рисками для предварительной оценки определяют, какие пользователи находятся в области действия и какие типы индикаторов риска настроены для оповещений. Можно быстро создать политику, которая применяется ко всем пользователям в организации или определять отдельных пользователей или группы для управления в политике. Политики поддерживают приоритеты содержимого для фокусировки условий политики на нескольких или определенных Microsoft Teams, на сайтах SharePoint, типах чувствительности данных и метках данных. С помощью шаблонов можно выбирать определенные индикаторы риска и настраивать пороги событий для индикаторов политики, эффективно настраивая показатели риска, а также уровень и частоту оповещений. Кроме того, с помощью ускорителей и аномалий риска определяются действия пользователей, имеющие более высокую или нетипичную важность. Окна политик позволяют определить период времени, в течение которого политика будет применяться к действиям оповещения, и используется для определения продолжительности политики после активации.

## <a name="policy-dashboard"></a>Панель мониторинга политик

**Панель мониторинга политики** позволяет быстро просматривать политики в Организации и текущие состояния оповещений, связанных с каждой политикой.

- **Имя политики**: имя, назначенное политике в мастере политик.
- **Активные оповещения**: количество активных оповещений для каждой политики.
- **Подтвержденные оповещения**: общее число оповещений, которые привели к случаям из политики за последние 365 дней.
- **Действия, выполняемые с оповещениями**: общее количество оповещений, которые были подтверждены или отклонения за последние 365 дней.
- **Эффективность политики**: процентное отношение, определяемое общим количеством подтвержденных оповещений, разделенных на общее количество действий, выполняемых над оповещениями (это сумма подтвержденных или отклоненных оповещений за последний год).
- **Active**: состояние обращения ( *Да* или *нет*).

## <a name="policy-templates"></a>Шаблоны политики

Шаблоны управления рисками для предварительной оценки — это предопределенные условия политики, которые определяют типы индикаторов риска и модель оценки риска, используемую политикой. У каждой политики должен быть шаблон, назначенный в мастере создания политики до создания политики. Центр управления рисками для предварительной оценки поддерживает до пяти политик для каждого шаблона политики. При создании новой политики риска для оценки с помощью мастера политики вы можете выбрать один из следующих шаблонов политики:

### <a name="data-theft-by-departing-users"></a>Хищение данных за счет посторонних пользователей

Когда пользователи покидают организацию, существуют определенные индикаторы риска, которые обычно связаны с хищением данных путем их посторонних пользователей. Этот шаблон политики использует индикаторы для оценки риска и фокусирования для определения и оповещений в этой области риска. Кража данных для пользователей может включать в себя загрузку файлов из SharePoint Online, печать файлов и копирование данных в личные облачные службы обмена сообщениями и службы хранилища вблизи ресигнатион и конечных дат. Этот шаблон начинает подсчет показателей риска, связанных с этими действиями, и их взаимоотношения с состоянием занятости пользователя.

>[!IMPORTANT]
>При использовании этого шаблона необходимо настроить соединитель Microsoft 365 HR для периодического импорта сведений о ресигнатион и даты увольнения для пользователей в Организации. Пошаговое руководство по настройке соединителя Microsoft 365 HR для вашей организации представлено в статье [Импорт данных с помощью СОЕДИНИТЕЛЯ HR Connector](import-hr-data.md) .

### <a name="general-data-leaks"></a>Общие утечки данных

Защита данных и предотвращение потерь данных является постоянной задачей для большинства организаций, особенно с быстрым увеличением новых данных, созданных пользователями, устройствами и службами. Пользователи могут создавать, хранить и обмениваться информацией между службами и устройствами, которые делают управление утечками данных более сложными и сложными. Утечки данных могут включать случайный общий доступ к информации вне организации или кражи данных с помощью злонамеренного способа. В сочетании с политикой защиты от потери данных (DLP) Этот шаблон начинает вычислить обнаружение подозрительных данных загрузки данных SharePoint Online, общий доступ к файлам и папкам, печать файлов и копирование данных в личные облачные службы обмена сообщениями и хранилищами.

При использовании шаблона **утечек данных** необходимо назначить политику защиты от потери данных, чтобы активировать индикаторы в политике риска предварительной оценки для оповещений высокой серьезности в Организации. Если правило политики DLP добавляет предупреждение о высоком уровне серьезности к журналу аудита Office 365, то политики риска, созданные с помощью этого шаблона, автоматически проверяют оповещение о высокой серьезности DLP. Если оповещение содержит пользователя в области, определенного в политике риска, оповещение обрабатывается политикой риска оценки как новое оповещение и назначается уровень опасности и оценки риска для оценки риска. Эта политика позволяет оценивать это предупреждение в контексте с другими действиями, включенными в случае.

#### <a name="data-leaks-policy-guidelines"></a>Рекомендации по использованию утечек данных

При создании или изменении политик защиты от потери данных для использования с политиками управления рисками для предварительной оценки учитывайте следующие рекомендации.

- Определите приоритеты событий ексфилтратион и избирательно при назначении параметров **отчетов об инциденте** *высокой* важности при настройке правил в политиках защиты от потери данных. Например, сообщение электронной почты с конфиденциальными документами должно быть событием ексфилтратион *высокого* уровня оповещений. Переопределение *высокого* уровня в параметрах **отчетов об инцидентах** в других правилах политики защиты от потери данных может увеличить шум в рабочем процессе оповещения об угрозе для контроля угроз и усложнить проверку этих оповещений. Например, назначение *высокого* уровня оповещений для доступа к действиям типа "отказ" в политиках защиты от потери данных облегчает оценку действительно опасного поведения и действий пользователей.
- Убедитесь, что вы знаете и правильно настроили пользователей в области в политиках управления рисками DLP и Insider. Оповещения политики защиты от потери данных будут обработаны только для пользователей, определенных в пределах области для политик управления рисками для оценки, использующих шаблон " **утечки данных** ". Кроме того, политика управления рисками для высокой серьезности будет проверяться политикой управления рисками для защиты от потери данных. Очень важно неясно настраивать пользователей в области политики рисков защиты от потери данных и оценки.

     Например, если правила политики защиты от потери данных ограничены только пользователями группы продаж, а политика риска оценки, созданная с помощью шаблона " **утечки данных** ", имеет все пользователи в пределах области, политика риска предварительной оценки будет действительно ОБРАБАТЫВАТЬ оповещения DLP высокой серьезности для пользователей в группе продаж. Политика риска предварительной оценки не получает оповещений о высоком приоритете DLP для пользователей, которые не определены в правилах защиты от потери данных в этом примере. И наоборот, если политика управления рисками, созданная с помощью шаблонов **утечки данных** , ограничена только пользователями в группе продаж, а назначенная политика защиты от потери данных распространяется на всех пользователей, то политика риска оценки будет обрабатывать только оповещения о защите от потери данных для участников группы продаж. Политика управления рисками для предварительной оценки будет игнорировать оповещения о высокой серьезности DLP для всех пользователей, не включенных в группу продаж.

- Убедитесь, что параметр правила " **отчеты об инцидентах** " в политике DLP, используемом для этого шаблона управления рисками для оценки, настроен на *высокий* уровень серьезности оповещений. *Высокий* уровень серьезности событий и оповещений об управлении рисками при предварительной работе не создаются на основе правил в политиках защиты от потери данных с *нехваткой* или *средним*полем " **отчеты об инцидентах** ".

    ![Настройка оповещений о политике защиты от потери данных](../media/insider-risk-DLP-policy-high-severity.png)

     >[!NOTE]
     >При создании новой политики защиты от потери данных с помощью встроенных шаблонов необходимо выбрать параметр **создать или настроить Расширенные правила DLP** , чтобы настроить параметры **отчетов об инцидентах** для уровня *высокой* важности.

Каждая политика управления рисками, созданная с помощью шаблона " **утечки данных** ", может иметь только одну назначенную политику DLP. Рассмотрите возможность создания выделенной политики защиты от потери данных, которая сочетает различные действия, которые вы хотите обнаружить и выиграть в качестве триггера событий для политик оценки риска, использующих шаблон " **утечки данных** ".

В разделе [Создание, тестирование и Настройка политики DLP](create-test-tune-dlp-policy.md) пошаговое руководство по настройке политик защиты от потери данных в Организации.

### <a name="data-leaks-by-priority-users-preview"></a>Утечки данных по приоритетным пользователям (Предварительная версия)

Защита данных и предотвращение потерь данных для пользователей в организации могут зависеть от их положения, уровня доступа к конфиденциальным сведениям или истории риска. Утечки данных могут включать случайный общий доступ к конфиденциальным сведениям вне организации или кражи данных с помощью злонамеренного способа. В сочетании с политикой защиты от потери данных (DLP) Этот шаблон начинает вычислить обнаружение подозрительных действий в режиме реального времени и приводит к увеличению вероятности оповещений о рисках и оповещений с более высоким уровнем серьезности. Приоритет пользователей определяется в разделе [приоритет групп пользователей](insider-risk-management-settings.md#priority-user-groups-preview) , настроенных в области Параметры управления рисками для участников.

Как и в случае с **шаблоном общих потерь данных**, необходимо назначить политику защиты от потери данных, чтобы активировать индикаторы в политике риска предварительной оценки для оповещений высокой серьезности в Организации. При создании политики с помощью этого шаблона следуйте рекомендациям, приведенным выше в разделе "утечки данных". Кроме того, необходимо присвоить приоритет группам пользователей, созданным с помощью параметров **управления рисками для предварительной оценки**  >  **Settings**  >  ,**приоритетных групп пользователей** в политику.

### <a name="data-leaks-by-disgruntled-users-preview"></a>Утечки данных пользователями дисгрунтлед (Предварительная версия)

Когда пользователи получают доступ к нагрузочным испытаниям, они могут стать дисгрунтлед, что может привести к увеличению вероятности риска для оценки. Этот шаблон начинает изменять действия пользователей, когда идентифицируется индикатор, связанный с дисгрунтлемент. Примеры включают уведомления о повышении производительности, неудовлетворительные обзоры производительности или изменения состояния уровня задания. Утечки данных для пользователей дисгрунтлед могут включать загрузку файлов из SharePoint Online и копирование данных в персональные облачные службы обмена сообщениями и хранения данных о событиях стрессор о занятости.

При использовании этого шаблона также необходимо настроить соединитель Microsoft 365 HR для периодического импорта уведомлений о повышении производительности, состояния некачественной проверки производительности или сведений об изменении уровня задания для пользователей в Организации. Пошаговое руководство по настройке соединителя Microsoft 365 HR для вашей организации представлено в статье [Импорт данных с помощью СОЕДИНИТЕЛЯ HR Connector](import-hr-data.md) .

### <a name="general-security-policy-violations-preview"></a>Общие нарушения политики безопасности (Предварительная версия)

Во многих организациях пользователи имеют разрешения на установку программного обеспечения на своих устройствах или на изменение параметров устройств для оказания помощи в задачах. Случайно или с вредоносными намерениями пользователи могут устанавливать вредоносные программы или отключать важные функции безопасности, помогающие защитить информацию на устройстве или в сетевых ресурсах. Этот шаблон политики использует оповещения системы безопасности от Advanced Threat Protection в защитнике Майкрософт (ATP), чтобы начать подсчет этих действий и обнаружения фокуса и оповещений в эту область риска. Используйте этот шаблон для получения ценных сведений о нарушениях политики безопасности в сценариях, когда у пользователей может быть журнал нарушений политики безопасности, который может быть индикатором риска для оценки.

Вам потребуется настроить в Организации пакет ATP для защитника Microsoft и включить пакет ATP для защитника Майкрософт для интеграции управления рисками в центре безопасности защитника, чтобы импортировать оповещения о нарушениях безопасности. Дополнительные сведения о настройке пакета ATP для защитника Майкрософт для интеграции управления рисками для ИТ можно найти [в статье configure Advanced Features in Microsoft Defender ATP](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center).

### <a name="security-policy-violations-by-departing-users-preview"></a>Нарушения политики безопасности за счет посторонних пользователей (Предварительная версия)

Отменяя положительные или отрицательные термины, могут возникать более риски, связанные с нарушениями политики безопасности. Чтобы защититься от случайных или вредоносных нарушений безопасности для пользователей, этот шаблон политики использует оповещения Microsoft Defender ATP для получения сведений о действиях, связанных с безопасностью. К этим действиям относятся пользователь, устанавливающий вредоносные программы или другие потенциально опасные приложения и исключающий функции безопасности на своих устройствах. Индикаторы политики активируются после того, как пользователи имеют ресигнатион или дату завершения, импортированные из соединителя Microsoft 365 HR в качестве события триггера.

При использовании этого шаблона необходимо настроить соединитель Microsoft 365 HR для периодического импорта сведений о ресигнатион и даты увольнения для пользователей в Организации. Пошаговое руководство по настройке соединителя Microsoft 365 HR для вашей организации представлено в статье [Импорт данных с помощью СОЕДИНИТЕЛЯ HR Connector](import-hr-data.md) .

Вам потребуется настроить в Организации пакет ATP для защитника Microsoft и включить пакет ATP для защитника Майкрософт для интеграции управления рисками в центре безопасности защитника, чтобы импортировать оповещения о нарушениях безопасности. Дополнительные сведения о настройке пакета ATP для защитника Майкрософт для интеграции управления рисками для ИТ можно найти [в статье configure Advanced Features in Microsoft Defender ATP](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center).

### <a name="security-policy-violations-by-priority-users-preview"></a>Нарушения политики безопасности по приоритету пользователей (Предварительная версия)

Защита от нарушений безопасности для пользователей в Организации может зависеть от их положения, уровня доступа к конфиденциальным сведениям или истории риска. Так как нарушения безопасности, связанные с приоритетом пользователей, могут оказать влияние на критические области вашей организации, этот шаблон политики начинает оценку этих показателей и использует оповещения Microsoft Defender ATP для получения сведений о действиях, связанных с безопасностью, для этих пользователей. К ним могут относиться приоритетные пользователи, которые устанавливают вредоносные программы или другие потенциально опасные приложения и отключают функции безопасности на своих устройствах. Приоритет пользователей определяется в разделе приоритет групп пользователей, настроенных в области Параметры управления рисками для участников.

Вам потребуется настроить в Организации пакет ATP для защитника Microsoft и включить пакет ATP для защитника Майкрософт для интеграции управления рисками в центре безопасности защитника, чтобы импортировать оповещения о нарушениях безопасности. Дополнительные сведения о настройке пакета ATP для защитника Майкрософт для интеграции управления рисками для ИТ можно найти [в статье configure Advanced Features in Microsoft Defender ATP](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center). Кроме того, необходимо присвоить приоритет группам пользователей, созданным с помощью параметров **управления рисками для предварительной оценки**  >  **Settings**  >  ,**приоритетных групп пользователей** в политику.

### <a name="security-policy-violations-by-disgruntled-users-preview"></a>Нарушения политики безопасности пользователями дисгрунтлед (Предварительная версия)

Пользователи, которые испытывают риск для сотрудников, могут быть подвержены повышенному риску в случае случайных или вредоносных нарушений политики безопасности. Эти нагрузочные испытания могут включать пользователя в план улучшения производительности, состояние некачественной проверки производительности или понижение его текущей позиции. Этот шаблон политики начинает подсчет рисков на основе этих индикаторов и действий, связанных с этими событиями для этих пользователей.

При использовании этого шаблона также необходимо настроить соединитель Microsoft 365 HR для периодического импорта уведомлений о повышении производительности, состояния некачественной проверки производительности или сведений об изменении уровня задания для пользователей в Организации. Пошаговое руководство по настройке соединителя Microsoft 365 HR для вашей организации представлено в статье [Импорт данных с помощью СОЕДИНИТЕЛЯ HR Connector](import-hr-data.md) .

Кроме того, необходимо настроить в Организации пакет ATP для защитника Microsoft и включить пакет ATP ATP для интеграции управления рисками в центре безопасности защитника, чтобы импортировать оповещения о нарушениях безопасности. Дополнительные сведения о настройке пакета ATP для защитника Майкрософт для интеграции управления рисками для ИТ можно найти [в статье configure Advanced Features in Microsoft Defender ATP](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center).

### <a name="offensive-language-in-email"></a>Оскорбительный язык в сообщении электронной почты

Обнаружение и принятие меры по предотвращению оскорбительных и нежелательных функций является важнейшим компонентом предотвращения риска. Встроенные классификаторы в Microsoft 365 сканируют отправленные сообщения электронной почты от почтовых ящиков Exchange Online в Организации для различных типов проблем с соответствием. Эти классификаторы используют сочетание искусственного интеллекта и ключевых слов для определения языка в сообщениях электронной почты, которые могут нарушить политику защиты от преследований. Используйте этот шаблон, чтобы быстро создать политику, которая использует эти классификаторы для автоматического обнаружения сообщений электронной почты, которые могут считаться оскорбительными или оскорбительными. Центр управления рисками для предварительной оценки использует классификаторы, которые проверяют посланные сообщения электронной почты на англоязычные языковые термины и мнений для оскорбительного языка.

## <a name="prioritize-content-in-policies"></a>Определение приоритетов контента в политиках

Политики управления рисками для предварительной оценки поддерживают указание более высокого приоритета для контента в зависимости от того, где он хранится или как он классифицируется. Указание контента в качестве приоритета увеличивает показатель риска для любой связанной деятельности, который, в свою очередь, повышает вероятность создания оповещения о высоком уровне серьезности. Однако некоторые действия не приводят к созданию оповещения, если связанный контент не содержит встроенных или настраиваемых типов конфиденциальной информации или не был указан в политике как приоритет.

Например, у вашей организации есть выделенный сайт SharePoint для строго конфиденциального проекта. Утечки данных на этом сайте SharePoint могут нарушить безопасность проекта и повлиять на его успешность. Наблюдая приоритеты для этого сайта SharePoint в политике утечки данных, показатели риска для уточненных действий автоматически увеличиваются. Это повышает вероятность того, что эти действия создают оповещение о риске для оценки и уровень серьезности для оповещения.

При создании политики управления рисками для предварительной оценки в мастере политики можно выбрать один из следующих приоритетов:

- **Сайты SharePoint**: всем действиям, связанным со всеми типами файлов в определенных сайтах SharePoint, назначается более высокий показатель риска. 
- **Типы конфиденциальной информации**: всем действиям, связанным с контентом, содержащим [типы конфиденциальной информации](sensitive-information-type-entity-definitions.md) , назначается более высокий показатель риска.
- **Метки конфиденциальности**: для всех действий, связанных с контентом, к которому применены конкретные [метки чувствительности](sensitivity-labels.md) , назначен более высокий показатель риска.

## <a name="create-a-new-policy"></a>Создание новой политики

Для создания новой политики управления рисками для предварительной оценки вы будете использовать мастер политик в решении " **Управление рисками** " в центре соответствия требованиям Microsoft 365.

Выполните указанные ниже действия, чтобы создать новую политику.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. Выберите **создать политику** , чтобы открыть мастер политик
3. На странице **Создание политики для оценки риска** заполните следующие поля:
    - **Name (обязательно)**: введите понятное имя политики.
    - **Description (необязательно)**: введите описание политики.
    - **Выбор шаблона политики (обязательно)**: выберите один из [шаблонов политики](insider-risk-management-policies.md#policy-templates) , чтобы определить типы индикаторов риска, контролируемые политикой.

    >[!IMPORTANT]
    >Большинство шаблонов политики имеют необходимые компоненты, которые необходимо настроить для создания соответствующих оповещений в политике. Если вы не настроили соответствующие необходимые компоненты политики, ознакомьтесь [со статьей начало работы по управлению рисками для оценки](insider-risk-management-configure.md#step-3-configure-prerequisites-for-templates).

4. Нажмите кнопку **Далее** , чтобы продолжить.
5. На странице **Пользователи** выберите **Добавить пользователя или группу** или **выберите приоритетные группы пользователей** , чтобы определить, какие пользователи или приоритетные группы пользователей включены в политику, в зависимости от выбранного шаблона политики. Установите флажок **все пользователи и группы с включенной поддержкой почты,** если это возможно (если вы не выбрали приоритетный шаблон на основе пользователей). Нажмите кнопку **Далее** , чтобы продолжить.
6. На странице **Указание содержимого для определения приоритета (необязательно)** можно назначить источникам приоритеты для улучшения показателей риска. Однако некоторые действия не приводят к созданию оповещения, если связанный контент не содержит встроенных или настраиваемых типов конфиденциальной информации или не был указан как приоритет на этой странице:
    - **Сайты SharePoint**: выберите пункт **Добавить сайт SharePoint** и выберите Организации SharePoint, для которых необходимо задать приоритет. Например, *"Group1@contoso.sharepoint.com/sites/group1"*.
    - **Тип конфиденциальной информации**: выберите **Добавить тип конфиденциальной информации** и выберите типы чувствительности, которые нужно назначить приоритетам. Например, *"номер банковского счета США"* и *"номер кредитной карты"*.
    - **Метки конфиденциальности**: выберите **Добавить метку конфиденциальности** и выберите метки, для которых требуется задать приоритет. Например, *"конфиденциальный"* и *"секрет"*.
7. Нажмите кнопку **Далее** , чтобы продолжить.
8. На странице " **Выбор индикаторов политики** " вы увидите [индикаторы](insider-risk-management-settings.md#indicators) , которые вы определили как доступные на странице "Индикаторы **параметров оценки риска**"  >  **Indicators** . Если вы выбрали шаблон " *утечки данных* " в начале работы мастера, необходимо выбрать политику защиты от потери данных в раскрывающемся списке **Политика защиты от потери** данных, чтобы включить индикаторы триггера для политики. Выберите индикаторы, которые необходимо применить к политике. Если вы предпочитаете не использовать параметры пороговых значений политики по умолчанию для этих индикаторов, отключите **Использование пороговых значений по умолчанию, рекомендуемых корпорацией Майкрософт** , и введите пороговые значения для каждого выбранного индикатора. Если вы выбрали по крайней мере один индикатор *Office* или *устройства* , выберите соответствующие **ускорители оценки риска** . Ускорители оценки риска применяются только для выбранных индикаторов.

    >[!IMPORTANT]
    >Если не удается выбрать индикаторы на этой странице, необходимо выбрать индикаторы, которые необходимо включить для всех политик на странице " **Insider risk management**  >  **Settings**  >  **индикаторы политики** управления рисками".

9. Нажмите кнопку **Далее** , чтобы продолжить.
10. На странице " **временные рамки политик** " вы увидите [условия окна активации](insider-risk-management-settings.md#policy-timeframes) для политики, которая находится на странице " **Insider risk settings**  >  **временные рамки политики** параметров для оценки риска".
11. Нажмите кнопку **Далее** , чтобы продолжить.
12. На странице " **Проверка** " Проверьте параметры, выбранные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы создать и активировать политику.

Ускорители оценки риска применяются только для выбранных индикаторов.

    >[!IMPORTANT]
    >Если не удается выбрать индикаторы на этой странице, необходимо выбрать индикаторы, которые необходимо включить для всех политик на странице " **Insider risk management**  >  **Settings**  >  **индикаторы политики** управления рисками".

11. Нажмите кнопку **Далее** , чтобы продолжить.
12. На странице " **временные рамки политик** " вы увидите [условия окна активации](insider-risk-management-settings.md#policy-timeframes) для политики, которая находится на странице " **Insider risk settings**  >  **временные рамки политики** параметров для оценки риска".
13. Нажмите кнопку **Далее** , чтобы продолжить.
14. На странице **Проверка** просмотрите параметры, обновленные для политики. Нажмите кнопку **изменить** , чтобы изменить любое значение политики, или кнопку **послать** , чтобы обновить и активировать политику.

## <a name="delete-a-policy"></a>Удаление политики

>[!NOTE]
>Удаление политики не приводит к удалению активных или архивированных оповещений, созданных из политики.

Чтобы удалить существующую политику безопасности для предварительной оценки, выполните указанные ниже действия.

1. В [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите к разделу **Управление рисками для оценки** и перейдите на вкладку **политики** .
2. В панели мониторинга политика выберите политику, которую нужно удалить.
3. Нажмите кнопку **Удалить** на панели инструментов панели мониторинга.
4. В диалоговом окне **Удалить** нажмите кнопку **Да** , чтобы удалить политику, или кнопку **Отмена** , чтобы закрыть диалоговое окно.
