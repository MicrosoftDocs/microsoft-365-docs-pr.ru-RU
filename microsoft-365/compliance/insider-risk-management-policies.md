# <a name="insider-risk-management-policies"></a>Политики управления рисками внутри организации

Политики управления рисками внутри организации определяют, какие пользователи находятся в области действия и какие типы индикаторов риска настроены для оповещений. Вы можете быстро создать политику, применяемую ко всем пользователям в организации, или определить отдельных пользователей или группы для управления политикой. Политики поддерживают приоритеты контента, чтобы сосредоточиться на условиях политики для нескольких или определенных Microsoft Teams, сайтов SharePoint, типов конфиденциальности данных и меток данных. С помощью шаблонов можно выбирать конкретные индикаторы риска и настраивать пороговые значения событий для индикаторов политики, эффективно настраивая оценки рисков, а также уровень и частоту оповещений. Кроме того, средства повышения и обнаружения аномалий оценки риска помогают определить активность пользователей, которая имеет более высокий или необычный вид. Окна политик позволяют определить сроки применения политики к действиям оповещения и используются для определения длительности активированной политики.

## <a name="policy-dashboard"></a>Панель мониторинга политики

Панель **мониторинга политики** позволяет быстро видеть политики в организации и текущее состояние оповещений, связанных с каждой политикой.

- **Имя политики:** имя, назначенное политике в мастере политики.
- **Активные оповещения:** количество активных оповещений для каждой политики.
- **Подтвержденные оповещения:** общее количество оповещений, которые были выявимы в случаях из политики за последние 365 дней.
- **Действия, принятые в отношении** оповещений: общее количество оповещений, которые были подтверждены или отклонены за последние 365 дней.
- **Эффективность политики**: процентное соотношение, определяемого итогом подтвержденных оповещений, разделенным на общее количество действий, принятых в отношении оповещений (суммы оповещений, которые были подтверждены или отклонены за последний год).
- **Активный:** состояние дела: *Да* или *Нет.*

## <a name="policy-templates"></a>Шаблоны политики

Шаблоны управления рисками внутри организации — это предварительно определенные условия политики, определяющие типы индикаторов рисков и модель оценки рисков, используемую политикой. Перед созданием политики в мастере создания каждой политики должен быть назначен шаблон. Управление рисками внутри организации поддерживает до пяти политик для каждого шаблона политики. При создании новой политики insider risk с помощью мастера политики вы выберете один из следующих шаблонов политики:

### <a name="data-theft-by-departing-users"></a>Кража данных путем убывающих пользователей

Когда пользователи покидают организацию, существуют определенные индикаторы риска, обычно связанные с кражей данных при отступе от пользователей. Этот шаблон политики использует индикаторы для оценки рисков и фокусирует обнаружение и оповещения для этой области риска. Кража данных у отправляющихся пользователей может включать загрузку файлов из SharePoint Online, печать файлов и копирование данных в личные облачные службы обмена сообщениями и службы хранения вблизи их трудовых и конечных дат. Этот шаблон начинает оценку индикаторов риска, связанных с этими действиями, и их корреляции со статусом занятости пользователей.

>[!IMPORTANT]
>При использовании этого шаблона необходимо настроить соединители Microsoft 365 HR для периодического импорта сведений о дате увольнения и увольнения для пользователей в организации. [Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 HR для организации см. в статье "Импорт данных с помощью соединители отдела кадров".

### <a name="general-data-leaks"></a>Общие утечки данных

Защита данных и предотвращение утечек данных — постоянная задача для большинства организаций, особенно в связи с быстрым ростом новых данных, созданных пользователями, устройствами и службами. Пользователи могут создавать, хранить и обмениваться информацией между службами и устройствами, что делает управление утечками данных все более сложным и сложным. Утечки данных могут включать случайную передачу информации за пределы организации или кражу данных с злонамеренной целью. В сочетании с назначенной политикой защиты от потери данных этот шаблон начинает выявлять подозрительные загрузки данных SharePoint Online в режиме реального времени, общий доступ к файлам и папок, печать файлов и копирование данных в личные облачные службы обмена сообщениями и службы хранения.

При использовании **шаблона "Утечки** данных" необходимо назначить политику защиты от утечки данных для запуска индикаторов в политике insider risk для оповещений высокого уровня серьезности в организации. Каждый раз, когда в журнал аудита Office 365 добавляется оповещение о высокой серьезности с помощью правила политики DLP, политики внутренней опасности, созданные с помощью этого шаблона, автоматически проверяют оповещение О DLP с высокой степенью серьезности. Если оповещение содержит пользователя в области действия, определенного в политике риска для внутренней оценки, то оповещение обрабатывается политикой риска внутри системы как новое оповещение и получает степень серьезности и оценку риска. Эта политика позволяет оценить это оповещение в контексте других действий, включенных в дело.

#### <a name="data-leaks-policy-guidelines"></a>Рекомендации политики утечки данных

При создании или изменении политик DLP для использования с политиками управления рисками внутри организации рассмотрите следующие рекомендации:

- Расставить приоритеты событий фильтрации данных  и быть выборочными при назначении параметров отчетов об инцидентах высокому при настройке правил в политиках DLP.  Например, отправка конфиденциальных документов известному конкуренту по электронной почте должна быть событием с высоким уровнем фильтрации оповещений.  Переназначение  высокого уровня  в параметрах отчетов об инцидентах в других правилах политики DLP может повысить уровень шума в рабочего процесса оповещения управления рисками внутри организации и затруднить оценку оповещений вашими следователями и аналитиками данных. Например, назначение  высоких уровней оповещений для доступа к действиям отказа в политиках DLP усложняет оценку действительно рискованных действий и действий пользователей.
- Убедитесь, что вы понимаете и правильно настраиваете пользователей в области как в политиках DLP, так и в политиках управления рисками внутри организации. Только пользователи, определенные как в области применения политик  управления рисками внутри организации, использующие шаблон "Утечки данных", будут обрабатывать оповещения политики DLP с высокой степенью серьезности. Кроме того, только пользователи, определенные как в области действия правила для оповещения О DLP с высокой степенью серьезности, будут рассмотрены политикой управления рисками внутри организации для рассмотрения. Важно не настраивать пользователей в области как в политиках DLP, так и в политиках внутренней безопасности конфликтующих способов.

     Например, если правила политики защиты от защите данных имеют область действия только для  пользователей из группы продаж, а политика insider risk policy, созданная на основе шаблона "Утечки данных", определяет всех пользователей как в области действия, политика риска для участников группы продаж будет обрабатывать только оповещения о защите от lp высокого уровня серьезности для пользователей из группы продаж. Политика insider risk не будет получать оповещения о DLP с высоким приоритетом для обработки пользователями, которые не определены в правилах DLP в этом примере. И наоборот, если политика управления  рисками внутри организации, созданная на основе шаблонов утечек данных, имеет область действия только для пользователей из отдела продаж, а назначенная политика защиты от защите от реализации защите от реализации пользования (DLP) — только для всех пользователей, политика риска для участников группы продаж будет обрабатывать только оповещения о высокой серьезности DLP. Политика управления рисками внутри организации будет игнорировать оповещения О DLP с высокой степенью серьезности для всех пользователей, не в группе продаж.

- Убедитесь, **что** параметр правила отчетов об инцидентах в политике DLP,  используемой для этого шаблона управления insider risk management, настроен для оповещений высокого уровня серьезности. Высокий *уровень* серьезности — это события, запускающие события, и оповещения об управлении рисками  внутри организации не  создаются из правил в политиках DLP с полем отчетов об инцидентах, установленным на низком или среднем *уровне.*

    ![Параметр оповещения политики DLP](../media/insider-risk-DLP-policy-high-severity.png)

     >[!NOTE]
     >При создании новой политики DLP с использованием встроенных шаблонов необходимо выбрать параметр "Создать или настроить расширенные правила **DLP",** чтобы настроить параметр отчетов об инцидентах для уровня серьезности.  

Каждой политике управления рисками внутри организации, созданной из шаблона **"Утечки** данных", может быть назначена только одна политика DLP. Рассмотрите возможность создания выделенной политики DLP, объединяющей различные действия, которые вы хотите обнаружить и выступать в качестве запуска событий для политик внутренней защиты от рисков, которые используют шаблон **утечки** данных.

Пошаговые инструкции по настройке политик DLP для организации см. в статье "Создание, тестирование и настройка политики [DLP".](create-test-tune-dlp-policy.md)

### <a name="data-leaks-by-priority-users-preview"></a>Утечки данных по приоритетным пользователям (предварительная версия)

Защита данных и предотвращение утечек данных для пользователей в организации может зависеть от их положения, уровня доступа к конфиденциальной информации или истории рисков. Утечки данных могут включать случайную передачу конфиденциальных сведений за пределы организации или кражу данных с злонамеренной целью. В сочетании с назначенной политикой защиты от потери данных этот шаблон начинает выявлять подозрительные действия в режиме реального времени и повышает вероятность появления оповещений и оповещений с более высоким уровнем серьезности. Пользователи с приоритетами определяются в группах пользователей [с](insider-risk-management-settings.md#priority-user-groups-preview) приоритетами, настроенных в области параметров управления рисками внутри организации.

Как и **в** шаблоне общих утечек данных, необходимо назначить политику DLP, чтобы активаторы в политике insider risk для оповещений высокого уровня серьезности в организации. Следуйте рекомендациям политики утечки данных, которые были указаны выше при создании политики с использованием этого шаблона. Кроме того, необходимо назначить политике группы пользователей с приоритетами, созданные в параметрах управления рисками для оценки  >    >   рисков.

### <a name="data-leaks-by-disgruntled-users-preview"></a>Утечки данных несовершимыми пользователями (предварительная версия)

Когда пользователи испытывают нагрузку на работу, они могут стать несоразмерными, что может повысить вероятность действий с внутриорганиалистией. Этот шаблон начинает озвучить активность пользователей при обнаружении индикатора, связанного с распускаемой нагрузкой. В качестве примеров можно привести уведомления об улучшении производительности, проверки низкой производительности или изменения состояния уровня задания. Утечки данных для неигрываемых пользователей могут включать загрузку файлов из SharePoint Online и копирование данных в личные облачные службы обмена сообщениями и службы хранения вблизи событий, которые могут привести к нагрузке на работу.

При использовании этого шаблона также необходимо настроить соединители Microsoft 365 HR для периодического импорта уведомлений об улучшении производительности, проверки низкой производительности или сведений об изменении уровня должности для пользователей в организации. [Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 HR для организации см. в статье "Импорт данных с помощью соединители отдела кадров".

### <a name="general-security-policy-violations-preview"></a>Общие нарушения политики безопасности (предварительная версия)

Во многих организациях пользователи имеют разрешения на установку программного обеспечения на своих устройствах или изменение параметров устройств для выполнения задач. Случайно или с злонамеренным намерением пользователи могут установить вредоносное ПО или отключить важные функции безопасности, которые помогают защитить информацию на своем устройстве или в сетевых ресурсах. Этот шаблон политики использует оповещения системы безопасности из Microsoft Defender для конечной точки, чтобы начать оценку этих действий и сосредоточиться на обнаружении и оповещениях для этой области риска. Используйте этот шаблон для предоставления статистики о нарушениях политики безопасности в сценариях, когда пользователи могут иметь историю нарушений политики безопасности, которая может быть индикатором внутренней угрозы.

Необходимо настроить Microsoft Defender для конечной точки в вашей организации и включить Защитник для конечной точки для интеграции управления insider risk management в Центре безопасности Защитника для импорта оповещений о нарушении безопасности. Дополнительные сведения о настройке Защитника для конечной точки для интеграции управления insider risk management см. в подстройке "Настройка расширенных функций в [Защитнике для конечной точки".](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center)

### <a name="security-policy-violations-by-departing-users-preview"></a>Нарушения политики безопасности при отступе от пользователей (предварительная версия)

Выход пользователей из системы в случае положительного или отрицательного срабатывания может привести к более высокому риску нарушения политики безопасности. Чтобы защититься от непреднамеренных или вредоносных нарушений безопасности для увольненных пользователей, этот шаблон политики использует защитник для оповещений конечных точек, чтобы получить представление о действиях, связанных с безопасностью. К таким действиям относится установка пользователем вредоносных программ или других потенциально опасных приложений и отключение функций безопасности на своих устройствах. Индикаторы политики активируются после того, как пользователи имеют дату увольнения или завершения, импортированные из соединители Microsoft 365 HR в качестве события запуска.

При использовании этого шаблона необходимо настроить соединители Microsoft 365 HR для периодического импорта сведений о дате увольнения и увольнения для пользователей в организации. [Пошаговая](import-hr-data.md) инструкция по настройке соединители Microsoft 365 HR для организации см. в статье "Импорт данных с помощью соединители отдела кадров".

Необходимо настроить Microsoft Defender для конечной точки в вашей организации и включить Защитник для конечной точки для интеграции управления insider risk management в Центре безопасности Защитника для импорта оповещений о нарушении безопасности. Дополнительные сведения о настройке Защитника для конечной точки для интеграции управления insider risk management см. в подстройке "Настройка расширенных функций в [Защитнике для конечной точки".](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-features#share-endpoint-alerts-with-microsoft-compliance-center)

### <a name="security-policy-violations-by-priority-users-preview"></a>Нарушения политики безопасности пользователями с приоритетами (предварительная версия)

Защита от нарушений безопасности для пользователей в организации может зависеть от их положения, уровня доступа к конфиденциальной информации или истории рисков. Поскольку нарушения безопасности пользователями с приоритетом могут оказывать влияние на критически важные области вашей организации, этот шаблон политики начинает учитывать эти индикаторы и использует Microsoft Defender для оповещений конечных точек, чтобы получить представление о действиях, связанных с безопасностью, для этих пользователей. К ним могут относится установка вредоносных программ или других потенциально опасных приложений и отключение функций безопасности на своих устройствах. Пользователи с приоритетами определяются в группах пользователей с приоритетами, настроенных в области параметров управления рисками внутри организации.

Необходимо настроить Microsoft Defender для конечной точки в вашей организации и включить Защитник для конечной точки для интеграции управления insider risk management в Центре безопасности Защитника для импорта оповещений о нарушении безопасности. ## <a name="prioritize-content-in-policies"></a>Приоритет контента в политиках

Политики управления рисками внутри организации поддерживают указание более высокого приоритета для контента в зависимости от того, где он хранится или как классифицируется. Указание содержимого в качестве приоритета увеличивает оценку риска для всех связанных действий, что, в свою очередь, повышает вероятность создания оповещения с высокой степенью серьезности. Однако некоторые действия не будут создавать оповещение, если только связанное содержимое не содержит встроенные или настраиваемые типы конфиденциальной информации или не указано в качестве приоритета в политике.

Например, в организации имеется выделенный сайт SharePoint для строго конфиденциального проекта. Утечка данных для информации на этом сайте SharePoint может нарушить проект и значительно повлиять на его успех. При назначив этому сайту SharePoint приоритет в политике утечки данных, показатели риска для действий, квалифицирующегося, автоматически увеличиваются. Такая расоритизация приоритетов повышает вероятность того, что эти действия создают оповещение о риске внутри организации, и повышает степень серьезности оповещения.

При создании политики управления рисками внутри организации в мастере политики можно выбрать один из следующих приоритетов:

- **Сайты SharePoint:** всем действиям, связанным со всеми типами файлов на определенных сайтах SharePoint, назначена более высокая оценка риска. 
- **Типы конфиденциальной информации:** всем действиям, связанным с содержимым с типами конфиденциальной информации, присвоена более высокая оценка риска. [](sensitive-information-type-entity-definitions.md)
- **Метки конфиденциальности:** любому действию, связанному с содержимым с примененными метами конфиденциальности, присвоена более высокая оценка риска. [](sensitivity-labels.md)

## <a name="create-a-new-policy"></a>Создание новой политики

Чтобы создать новую политику управления рисками внутри организации, используйте мастер политики в решении **для** управления рисками для программы оценки в Центре соответствия требованиям Microsoft 365.

Выполните следующие действия, чтобы создать новую политику:

1. В Центре [соответствия требованиям Microsoft 365](https://compliance.microsoft.com)перейдите в центр управления рисками для программы **insider и** выберите вкладку **"Политики".**
2. Выберите **"Создать политику",** чтобы открыть мастер политики
3. На странице **"Новая политика insider risk policy"** заполните следующие поля:
    - **Имя (обязательно)**: введите имя политики.
    - **Описание (необязательно).** Введите описание политики.
    - **Выберите шаблон политики (обязательно).** Выберите [](insider-risk-management-policies.md#policy-templates) один из шаблонов политики, чтобы определить типы индикаторов риска, которые отслеживаются политикой.

    >[!IMPORTANT]
    >Большинство шаблонов политик имеют необходимые условия, которые необходимо настроить для создания соответствующих оповещений. If you haven't configured the applicable policy prerequisites, see [Get started with insider risk management](insider-risk-management-configure.md#step-3-configure-prerequisites-for-templates).

4. Выберите **"Далее",** чтобы продолжить.
5. На  странице "Пользователи" выберите "Добавить пользователя или группу" или выберите группы пользователей "Приоритет", чтобы определить, какие пользователи или группы пользователей с приоритетом включены в политику в зависимости от выбранного шаблона политики.   Выберите **"Все пользователи и группы** с включенной поддержкой почты", если это возможно (если вы не выбрали шаблон, основанный на приоритете пользователя). Выберите **"Далее",** чтобы продолжить.
6. На странице **"Укажите, какой** контент необходимо назначить приоритету (необязательно)", можно назначить источники для приоритетов для повышения оценки риска. Однако некоторые действия не будут создавать оповещение, если только связанное содержимое не содержит встроенные или настраиваемые типы конфиденциальной информации или не было указано в качестве приоритета на этой странице:
    - **Сайты SharePoint:** выберите **"Добавить сайт SharePoint"** и выберите организации SharePoint, приоритеты для них. Например, *"group1@contoso.sharepoint.com/sites/group1".*
    - **Тип конфиденциальной информации:** выберите **"Добавить тип** конфиденциальной информации" и выберите типы конфиденциальности, которые необходимо расставить по приоритету. Например, *"Номер* банковского счета в США" и *"Номер кредитной карты".*
    - **Метки конфиденциальности:** выберите "Добавить метку **конфиденциальности"** и выберите метки, приоритеты для них. Например, *"Конфиденциально"* и *"Секрет".*
7. Выберите **"Далее",** чтобы продолжить.
8. На странице **"Выбор индикаторов** политики" [](insider-risk-management-settings.md#indicators) вы увидите индикаторы, которые вы определили как доступные на странице "Индикаторы параметров риска для оценки  >   риска". Если вы выбрали  шаблон утечки данных в начале мастера, необходимо выбрать политику DLP из выпадающих списков политик **DLP,** чтобы включить индикаторы запуска для политики. Выберите индикаторы, которые необходимо применить к политике. Если вы предпочитаете не использовать пороговые значения политики по  умолчанию для этих индикаторов, отключаем рекомендуемые корпорацией Майкрософт пороговые значения по умолчанию и введите пороговые значения для каждого выбранного индикатора. Если вы выбрали хотя бы один  индикатор *Office*  или устройства, выберите нужные повышения для оценки риска. Повышение оценки риска применимо только к выбранным индикаторам.

    >[!IMPORTANT]
    >Если индикаторы на этой странице не могут быть выбраны, необходимо выбрать индикаторы, которые необходимо включить для всех политик на странице "Параметры политики управления рисками для оценки  >    >   риска".

9. Выберите **"Далее",** чтобы продолжить.
10. На странице **"Временные** рамки политики" [](insider-risk-management-settings.md#policy-timeframes) вы увидите условия окна активации для политики, которая находится на странице "Параметры политики для оценки   >   рисков".
11. Выберите **"Далее",** чтобы продолжить.
12. На странице **"Проверка"** просмотрите параметры, выбранные для политики. Выберите  "Изменить", чтобы изменить любое из значений политики, или выберите **"Отправить",** чтобы создать и активировать политику.

## <a name="delete-a-policy"></a>Удаление политики

>[!NOTE]

Чтобы удалить существующую политику управления рисками внутри организации, выполните следующие действия.
