# <a name="set-up-encryption-in-office-365-enterprise"></a>Настройка шифрования в Office 365 корпоративный

Шифрование позволяет защитить содержимое от чтения неавторизованным пользователем. Поскольку [шифрование в Office 365](encryption.md) можно сделать с помощью различных технологий и методов, не существует одного места, где можно включить или настроить шифрование. В этой статье данная статья содержит сведения о различных способах настройки или настройки шифрования в рамках стратегии защиты информации.
  
> [!TIP]
> Дополнительные технические сведения о шифровании см. в техническом справочнике по шифрованию [в Office 365.](technical-reference-details-about-encryption.md)
  
В Office 365 по умолчанию доступно несколько возможностей шифрования. Дополнительные возможности шифрования можно настроить в соответствии с определенными требованиями или юридическими требованиями. В следующей таблице описаны несколько методов шифрования для различных сценариев.
  
|**Сценарий**|**Методы шифрования**|
|:-----|:-----|
|Файлы сохраняются на компьютерах с Windows  <br/> |Шифрование на уровне компьютера можно сделать с помощью BitLocker на устройствах с Windows. Как администратор предприятия или ИТ-специалисты вы можете настроить эту возможность с помощью Microsoft Deployment набор средств (MDT). См. ["Настройка MDT для BitLocker".](https://go.microsoft.com/fwlink/?linkid=849282)  <br/> |
|Файлы сохраняются на мобильных устройствах  <br/> |Некоторые виды мобильных устройств шифруют файлы, сохраненные на этих устройствах по умолчанию. С помощью встроенных функций управления мобильными устройствами для [Office 365](https://support.microsoft.com/en-us/office/capabilities-of-built-in-mobile-device-management-for-microsoft-365-a1da44e5-7475-4992-be91-9ccec25905b0)вы можете настроить политики, которые определяют, следует ли разрешить мобильным устройствам доступ к данным в Office 365. Например, можно настроить политику, которая разрешает доступ к данным Office 365 только устройствам, шифруя содержимое. См. ["Создание и развертывание политик безопасности устройств".](https://support.microsoft.com/office/create-and-deploy-device-security-policies-d310f556-8bfb-497b-9bd7-fe3c36ea2fd6)  <br/> Чтобы дополнительно контролировать взаимодействие мобильных устройств с Office 365, можно добавить [Microsoft Intune.](https://docs.microsoft.com/mem/intune/fundamentals/setup-steps)  <br/> |
|Вам необходимо управлять ключами шифрования, используемыми для шифрования данных в центрах обработки данных Майкрософт  <br/> | Как администратор Office 365 вы можете управлять ключами шифрования вашей организации, а затем настраивать Office 365 так, чтобы они использовали их для шифрования неавтовых данных в центрах обработки данных Майкрософт.  <br/> [Шифрование службы с помощью ключа клиента в Office 365](customer-key-overview.md) <br/> |
|Люди общаются по электронной почте (Exchange Online)  <br/> | Администратор Exchange Online может настроить шифрование электронной почты несколькими вариантами. Такие законодательные акты перечислены ниже.  <br/>  Использование [шифрования сообщений Office 365 (OME)](set-up-new-message-encryption-capabilities.md) с Azure Rights Management (Azure RMS) для отправки зашифрованных сообщений внутри или за пределами организации  <br/>  Использование [S/MIME для подписи и шифрования](https://aka.ms/c6dozg) сообщений для шифрования и цифровой подписи сообщений электронной почты  <br/>  Использование TLS для [настроить соединители для безопасного потока почты с другой организацией](https://aka.ms/hs809p) <br/>  См. [шифрование электронной почты в Office 365.](https://aka.ms/hic3f7)  <br/> |
|Доступ к файлам с сайтов группы или библиотек документов (OneDrive для бизнеса или SharePoint Online)  <br/> |При работе с файлами, сохраненными в OneDrive для бизнеса или SharePoint Online, используются подключения TLS. Эта служба встроена в Office 365 автоматически. См. [раздел "Шифрование данных в OneDrive для бизнеса и SharePoint Online".](https://go.microsoft.com/fwlink/?linkid=526379)  <br/> |
|Общий доступ к файлам в собраниях по сети и мгновенных беседах (Skype для бизнеса Online)  <br/> |When people are working with files using Skype for Business Online, TLS is used for the connection. Эта служба встроена в Office 365 автоматически. См. ["Безопасность и архив "Безопасность" (Skype для бизнеса Online).](https://aka.ms/nuq4ws)  <br/> |
|Общий доступ к файлам в собраниях по сети и мгновенных беседах (Microsoft Teams)  <br/> |Когда люди работают с файлами с помощью Microsoft Teams, для подключения используется TLS. Эта служба встроена в Office 365 автоматически. В настоящее время Microsoft Teams не поддерживает отрисовку зашифрованных сообщений электронной почты. Чтобы предотвратить отправку зашифрованной электронной почты в Microsoft Teams как зашифрованную, см. в этой [теме.](https://docs.microsoft.com/microsoft-365/compliance/ome-faq?view=o365-worldwide#can-i-automatically-remove-encryption-on-incoming-and-outgoing-mail)  <br/> 

## <a name="additional-information"></a>Дополнительные сведения

Дополнительные информацию о решениях для защиты файлов, которые включают параметры шифрования, см. в подключе "Решения для защиты [файлов" в Office 365.](https://www.microsoft.com/download/details.aspx?id=55523)
