# <a name="set-up-a-connector-to-archive-data-from-ms-sql-database"></a>Настройка соединитетеля для архива данных из ms База данных SQL

Используйте соединитель Veritas в центре Microsoft 365 для импорта и архива данных ms База данных SQL для почтовых ящиков пользователей в Microsoft 365 организации. Veritas предоставляет вам соединителю ms База данных SQL importer, который настроен для захвата элементов из базы данных с помощью XML-файла конфигурации и импорта этих элементов в Microsoft 365. Соединитатель преобразует содержимое из MS База данных SQL в формат сообщений электронной почты, а затем импортирует эти элементы в почтовые ящики пользователей в Microsoft 365.

После хранения контента из ms База данных SQL в почтовых ящиках пользователей вы можете применить Microsoft 365, такие как хранение судебного разбирательства, открытие электронных данных, политики хранения и метки хранения. Использование соединиттеля ms База данных SQL для импорта и архива данных в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="overview-of-archiving-the-ms-sql-data"></a>Обзор архива данных ms SQL

В следующем обзоре объясняется процесс использования соединитетеля для архивации данных ms SQL в Microsoft 365.

![Процесс архива для данных MS SQL](../media/MSSQLDatabaseConnectorWorkflow.png)

1. Организация работает с поставщиком ms База данных SQL для настройки и настройки сайта ms База данных SQL.

2. Раз в 24 часа элементы MS База данных SQL копируется на сайте Veritas Merge1. Соединителет также преобразует этот контент в формат сообщений электронной почты.

3. Соединиттель ms База данных SQL importer, который вы создаете в центре соответствия требованиям Microsoft 365, ежедневно подключается к сайту Veritas Merge1 и передает сообщения в безопасное служба хранилища Azure в облаке Майкрософт.

4. Соединитатель импортирует преобразованные элементы MS База данных SQL в почтовые ящики определенных пользователей, используя значение свойства *Email* автоматического сопоставления пользователей, как описано в шаге [3](#step-3-map-users-and-complete-the-connector-setup). В почтовых ящиках пользователя создается подмостки в папке **"Входящие"** с именем MS База данных SQL Importer, и элементы импортируется в эту папку. Соединитатель определяет, в какой почтовый ящик импортировать элементы, используя значение свойства *Email.* Каждый элемент из ms База данных SQL содержит это свойство, заполненное адресом электронной почты каждого участника элемента.

## <a name="before-you-begin"></a>Прежде чем начать

- Создайте учетную запись Veritas Merge1 для соединители Microsoft. Чтобы создать учетную запись, обратитесь [в службу поддержки клиентов Veritas.](https://www.veritas.com/content/support/) При создании соединитетеля в шаге 1 необходимо войти в эту учетную запись.

- Пользователю, который создает соединителер ms База данных SQL importer в шаге 1 (и завершает его в шаге 3), необходимо приступить к роли экспортировать импорт почтовых ящиков в Exchange Online. Эта роль требуется для добавления соединители на странице соединители данных в центре Microsoft 365 соответствия требованиям. По умолчанию эта роль не назначена какой-либо группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах [Создание](/Exchange/permissions-exo/role-groups#create-role-groups) групп ролей или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-set-up-the-ms-sql-database-importer-connector"></a>Шаг 1. Настройка соединиттеля База данных SQL ms

Первым шагом является доступ к странице **Соединители** данных в центре соответствия требованиям Microsoft365 и создание соединитетеля для ms База данных SQL.

1. Перейдите [https://compliance.microsoft.com](https://compliance.microsoft.com) к и нажмите **соединители**  >  **данных MS База данных SQL Importer**.

2. На странице **описания продукта База данных SQL ms База данных SQL** нажмите **кнопку Добавить новый соединититель.**

3. На странице **Условия службы нажмите** кнопку **Принять**.

4. Введите уникальное имя, идентифицируемое соединитетелем, а затем нажмите **кнопку Далее**.

5. Впишитесь в свою учетную запись Merge1, чтобы настроить соединители.

## <a name="step-2-configure-the-ms-sql-database-importer-connector-on-the-veritas-merge1-site"></a>Шаг 2. Настройка соединителя База данных SQL ms на сайте Veritas Merge1

На втором этапе необходимо настроить соединители ms База данных SQL importer на сайте Merge1. Сведения о настройке импортера ms База данных SQL см. в руководстве по пользователю [Merge1 Сторонние соединители.](https://docs.ms.merge1.globanetportal.com/Merge1%20Third-Party%20Connectors%20MS%20SQL%20Database%20Importer%20User%20Guide%20.pdf)

После нажатия **кнопки Сохранить &**  finish отображается страница сопоставления пользователей в мастере соединители в центре Microsoft 365 соответствия требованиям.

## <a name="step-3-map-users-and-complete-the-connector-setup"></a>Шаг 3. Карта пользователей и завершение установки соединитетеля

Чтобы составить карту пользователей и выполнить установку соединитетеля, выполните следующие действия:

1. На странице **Map MS База данных SQL импортера** для Microsoft 365 пользователей включаем автоматическое сопоставление пользователей. Элементы ms База данных SQL включают свойство *Email,* которое содержит адреса электронной почты для пользователей в организации. Если соединитатель может связать этот адрес с Microsoft 365 пользователем, элементы импортируется в почтовый ящик этого пользователя.

2. Нажмите **кнопку Далее,** просмотрите  параметры и перейдите на страницу соединители данных, чтобы просмотреть ход процесса импорта нового соединитетеля.

## <a name="step-4-monitor-the-ms-sql-database-importer-connector"></a>Шаг 4. Мониторинг соединиттеля База данных SQL ms

После создания соединиттеля База данных SQL ms можно просмотреть состояние соединитетеля в центре Microsoft 365 соответствия требованиям.

1. Перейдите <https://compliance.microsoft.com/> и щелкните **соединители данных** в левом nav.

2. Щелкните  вкладку Соединители, а затем **выберите соединитетель ms База данных SQL** **importer,** чтобы отобразить страницу вылетов, которая содержит свойства и сведения о соединителе.

3. В **состоянии Соединитель с исходным кодом** щелкните ссылку **Журнал** загрузки, чтобы открыть (или сохранить) журнал состояния соединитетеля. В этом журнале содержатся данные, импортируемые в облако Майкрософт.

## <a name="known-issues"></a>Известные проблемы

- В настоящее время мы не поддерживаем импорт вложений или элементов размером более 10 МБ. Поддержка более крупных элементов будет доступна позднее.
