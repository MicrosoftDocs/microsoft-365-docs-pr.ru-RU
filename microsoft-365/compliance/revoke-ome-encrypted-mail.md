# <a name="revoke-email-encrypted-by-advanced-message-encryption"></a>Отодвигать сообщения электронной почты, зашифрованные с помощью advanced Message Encryption

Отзыв электронной почты предлагается в рамках службы office 365 Advanced Message Encryption. Office 365 Advanced Message Encryption входит в [состав Microsoft 365](https://www.microsoft.com/microsoft-365/enterprise/home)корпоративный E5, Office 365 E5, Microsoft 365 E5 (цены для некоммерческих сотрудников), Office 365 корпоративный E5 (цены для некоммерческих сотрудников) и Office 365 для образования A5. Если у вашей организации есть подписка, не включаемая в расширенный шифрование сообщений Office 365, вы можете приобрести ее с помощью надстройки SKU соответствия требованиям Microsoft 365 E5 для Microsoft 365 E3, Microsoft 365 E3 (цены для некоммерческих сотрудников) или надстройки Office 365 Advanced Compliance SKU для Microsoft 365 E3, Microsoft 365 E3 (цены для некоммерческих сотрудников) или SKU Office 365.

Эта статья входит в большую серию статей о шифровании сообщений [Office 365.](ome.md)

Если сообщение было зашифровано с помощью office 365 Advanced Message Encryption и вы администратор Microsoft 365 или вы отправитель сообщения, вы можете отослать сообщение при определенных условиях. Администраторы отозовют сообщения с помощью PowerShell. Отправитель отзовет сообщение, отправленное непосредственно из Outlook в Интернете. В этой статье описываются условия, при которых отзыв возможен, и описывается, как это сделать.
  
## <a name="encrypted-emails-that-you-can-revoke"></a>Зашифрованные сообщения электронной почты, которые можно отооскить

Администраторы и отправители сообщений могут отослать зашифрованные сообщения, если получатель получил сообщение электронной почты с фирменным шифрованием на основе ссылок. Если получатель получил опыт работы с текстом в поддерживаемом клиенте Outlook, вы не сможете отоискить сообщение.

Тип удостоверения получателя зависит от типа удостоверения получателя: получатели учетных записей Office 365 и Майкрософт (например, outlook.com пользователей) получают возможность работы в поддерживаемых клиентах Outlook. Все другие типы получателей, такие как Gmail и Yahoo, получают ссылки на основе ссылок.

Администраторы и отправители сообщений могут отослать сообщения, зашифрованные с помощью шифрования, примененного непосредственно из Outlook в Интернете. Например, сообщения, зашифрованные с помощью параметра "Только шифрование".

:::image type="content" source="../media/adhocencryptionrevoke.png" alt-text="Снимок экрана с параметром &quot;Только шифрование&quot; в Outlook в Интернете.":::

## <a name="recipient-experience-for-revoked-encrypted-emails"></a>Опыт получателей для отозванных зашифрованных сообщений электронной почты

После отзыва сообщения электронной почты получатель получает сообщение об ошибке при доступе к зашифрованному сообщению на портале шифрования сообщений Office 365: "Сообщение было отозвано отправиком".

![Снимок экрана: отозванное зашифрованное сообщение электронной почты.](../media/revoked-encrypted-email.png)

## <a name="how-to-revoke-an-encrypted-message-that-you-sent"></a>Как отоискить отправленное зашифрованное сообщение

Вы можете отоискить почту, отправленную одному получателю, использующему учетную запись социальных сетей, например gmail.com или yahoo.com. Другими словами, вы можете отоскить сообщение электронной почты, отправленное одному получателю, который получил ссылку.

Вы не можете отоискить почту, отправленную получателю, использующему учетную запись для работы или учебного заведения из Office 365 или Microsoft 365, или пользователя, который использует учетную запись Майкрософт, например учетную запись outlook.com учетной записи. 

Чтобы отоискать отправленное зашифрованное сообщение, выполните следующие действия.

1. В Outlook в Интернете  в папке "Отправлено" перейдите к сообщению, которое нужно отоискить.

   Если сообщение отозвано, в верхней части сообщения будет отобрана ссылка "Удалить внешний доступ".

    :::image type="content" source="../media/infoprotect-email-encryption/adhocencryptionrevokesentmsg.png" alt-text="Screenshot showing encrypted mail that you want to revoke in Outlook on the web.":::

2. Нажмите **кнопку "Удалить внешний доступ",** чтобы отоискать сообщение.

   Сообщение показывает, что его состояние отозвано.

   :::image type="content" source="../media/adhocencryptionrevokedmsg.png" alt-text="Снимок экрана: отозванное зашифрованное сообщение в Outlook в Интернете.":::

## <a name="how-to-revoke-an-encrypted-message-as-an-administrator"></a>Как отоискить зашифрованное сообщение от администратора

Администраторы Microsoft 365 выполните следующие общие действия, чтобы отоискить доступную зашифрованную электронную почту:

- Получите ИД сообщения электронной почты.
- Убедитесь, что вы можете отоскить сообщение.
- Отозовет сообщение.

### <a name="step-1-obtain-the-message-id-of-the-email"></a>Этап 1. Получение ИД сообщения электронной почты

Прежде чем отоскить зашифрованное сообщение, соберем его код. MessageId обычно имеет формат:

`<xxxxxxxxxxxxxxxxxxxxxxx@xxxxxx.xxxx.prod.outlook.com>`  

Существует несколько способов найти ИД сообщения электронной почты, которое требуется отоискить. В этом разделе описывается несколько параметров, но можно использовать любой метод, который предоставляет ИД.

#### <a name="to-identify-the-message-id-of-the-email-you-want-to-revoke-by-using-message-trace-in-the-security-amp-compliance-center"></a>Идентификация ИД сообщения, отозоваемого с помощью трассировки сообщений, в Центре соответствия &amp; требованиям безопасности

1. Поиск электронной почты по отправителю или получателю с помощью новой трассировки сообщений в Центре [& соответствия требованиям.](https://blogs.technet.microsoft.com/exchange/2018/05/02/new-message-trace-in-office-365-security-compliance-center/)

2. После того как вы нашли сообщение электронной почты, выберите его, чтобы отследить его в области **сведений о трассировке** сообщений. **Разойдите дополнительные** сведения, чтобы найти ИД сообщения.

#### <a name="to-identify-the-message-id-of-the-email-you-want-to-revoke-by-using-office-message-encryption-reports-in-the-security-amp-compliance-center"></a>Определение ИД сообщения, которое требуется отооскить, с помощью отчетов шифрования сообщений Office в Центре безопасности &amp; и соответствия требованиям

1. В Центре соответствия &amp; требованиям безопасности перейдите к отчету о шифровании **сообщений.** Сведения об этом отчете см. в отчете о безопасности электронной [почты в Центре соответствия &amp; требованиям безопасности.](../security/office-365-security/view-email-security-reports.md)

2. Выберите **таблицу сведений о** представлении и определите сообщение, которое нужно отоискить.

3. Дважды щелкните сообщение, чтобы просмотреть сведения с ИД сообщения.

### <a name="step-2-verify-that-the-mail-is-revocable"></a>Шаг 2. Убедитесь, что сообщение отозвано

Чтобы проверить возможность отзыва сообщения, проверьте, отображается ли поле "Состояние отзыва" в отчете  "Шифрование" в таблице "Сведения" в Центре соответствия &amp; требованиям безопасности.

Чтобы убедиться, что вы можете отоискить определенное сообщение электронной почты с помощью Windows PowerShell, выполните следующие действия.

1. Используя учетную запись для работы или учебного заведения с разрешениями глобального администратора в организации, Windows PowerShell сеанс и подключите его к Exchange Online. Инструкции см. в статье [Подключение к Exchange Online PowerShell](https://aka.ms/exopowershell).

2. Выполните Get-OMEMessageStatus следующим образом:

     ```powershell
     Get-OMEMessageStatus -MessageId "<message id>" | ft -a  Subject, IsRevocable
     ```

   Эта команда возвращает тему сообщения и сообщает, можно ли отозвано сообщение. Пример.

     ```console
     Subject        IsRevocable
     -------        -----------
     "Test message" True
     ```

### <a name="step-3-revoke-the-mail"></a>Шаг 3. Отоискить почту

После того как вы знаете ИД сообщения, которое вы хотите отооскить, и убедились, что сообщение отзывается, вы можете отоискить сообщение с помощью Центра соответствия требованиям безопасности или &amp; Windows PowerShell.

Как отоискить сообщение с помощью Центра соответствия &amp; требованиям безопасности

1. Используя учетную запись для работы или учебного заведения с разрешениями глобального администратора в организации, подключите ее к Центру безопасности & соответствия требованиям.

2. В **отчете Encryption**  (Шифрование) в таблице сведений для сообщения выберите **"Отоскить сообщение".**

Чтобы отоискить сообщение электронной почты с помощью Windows PowerShell, используйте Set-OMEMessageRevocation.

1. Используя учетную запись для работы или учебного заведения с разрешениями глобального администратора в организации, подключите к [Exchange Online PowerShell.](https://aka.ms/exopowershell)

2. Выполните Set-OMEMessageRevocation следующим образом:

    ```powershell
    Set-OMEMessageRevocation -Revoke $true -MessageId "<messageId>"
    ```

3. Чтобы проверить, отозвано ли сообщение электронной почты, выполните Get-OMEMessageStatus следующим образом:

    ```powershell
    Get-OMEMessageStatus -MessageId "<messageId>" | ft -a  Subject, Revoked
    ```

    В случае успешного отзыва он возвращает следующий результат:  

     ```console
     Revoked: True
     ```
