# <a name="compliance-score-calculation"></a>Расчет оценки соответствия требованиям

> [!IMPORTANT]
> Рекомендации от диспетчера соответствия требованиям не интерпретируются как гарантию соответствия требованиям. Вы можете оценивать и проверять эффективность клиентских элементов управления в соответствии с нормативной средой. Эти службы подчиняются условиям и условиям, указанным в [терминах веб-служб](https://go.microsoft.com/fwlink/?linkid=2108910). Кроме того [, вы также найдете рекомендации по лицензированию Microsoft 365 для обеспечения безопасности и соответствия требованиям](https://docs.microsoft.com/office365/servicedescriptions/microsoft-365-service-descriptions/microsoft-365-tenantlevel-services-licensing-guidance/microsoft-365-security-compliance-licensing-guidance).

## <a name="how-to-read-your-compliance-score"></a>Просмотр оценки соответствия требованиям

В панели мониторинга "Управление соответствием требованиям" отображается общий рейтинг соответствия требованиям. Эта оценка измеряет выполнение рекомендуемых действий по улучшению в элементах управления. Ваш рейтинг поможет вам понять текущую оценку соответствия требованиям. Кроме того, она может помочь вам определить приоритеты действий в зависимости от их потенциального снижения риска.

Значение оценки назначается на трех уровнях:

1. **Оценка действий по улучшению**: каждое действие оказывает влияние на вашу оценку в зависимости от потенциального риска

2. **Контрольный показатель**: этот показатель представляет собой сумму баллов, полученных при выполнении действий по улучшению в элементе управления. Эта сумма применяется полностью по отношению к общему показателю соответствия требованиям, когда элемент управления отвечает обоим следующим условиям:
    - **Состояние реализации** равно **реализованной** или **альтернативной реализации**, а также
    - **Результат проверки** равен **пройденному**.

3. Оценка **оценки**: этот показатель представляет собой сумму контрольных показателей. Он вычисляется с помощью оценок действий. Каждое действие Майкрософт и каждое действие улучшения, управляемое организацией, считаются один раз, независимо от того, как часто они ссылаются на элемент управления.

Общий показатель соответствия вычисляется с помощью оценок действий, где каждое действие Майкрософт подсчитывается один раз, каждое техническое действие, которое вы управляете, учитывается один раз, и каждое нетехническое действие, которое вы управляете, учитывается один раз в каждой группе. Эта логика предназначена для предоставления наиболее точных учетных данных по реализации и тестированию действий в Организации. Вы можете заметить, что это может привести к тому, что общий показатель соответствия требованиям будет отличаться от среднего оценки. Узнайте больше о [том, как оцениваются действия](#action-types-and-points).

## <a name="initial-score-based-on-microsoft-365-data-protection-baseline"></a>Первоначальный балл, основанный на базовом показателе защиты данных Microsoft 365
  
Диспетчер соответствия требованиям предоставляет начальный рейтинг на основе базового уровня защиты данных Microsoft 365. Этот базовый план представляет собой набор элементов управления, включающих ключевые нормативные требования и стандарты для защиты данных и общего управления данными. Этот базовый план рисует элементы в основном с помощью NIST CSF (Национальный институт стандартов и технологий Циберсекурити Framework) и ISO (Международная организация по стандартизации), а также из FedRAMP (Федеральный показатель рисков и программного обеспечения управления авторизацией) и GDPR (общее количество требований к защите данных Европейского союза).

Первоначальный показатель рассчитывается в соответствии с оценкой по умолчанию для системы защиты данных, предоставленной всем организациям. При первом посещении диспетчер соответствия требованиям уже собирает сигналы из решений Microsoft 365. Вы увидите, как выполняется Организация относительно основных стандартов защиты данных и нормативных требований, а также ознакомьтесь с предлагаемыми действиями по улучшению.

Так как каждая организация имеет определенные потребности, диспетчер соответствия требованиям зависит от того, как настроить оценки и управлять ими для упрощения и смягчения риска, как можно более детально.

## <a name="how-compliance-manager-continuously-assesses-controls"></a>Как диспетчер соответствия требованиям постоянно оценивает элементы управления

Диспетчер соответствия требованиям автоматически сканирует среду Microsoft 365 и определяет системные параметры, непрерывно и автоматически обновляет состояние технического действия. Microsoft Secure Score — это основной механизм, который выполняет мониторинг.

Состояние действия обновляется на панели мониторинга каждые 24 часа. Когда вы следуйте рекомендациям по внедрению элемента управления, вы обычно видите состояние элемента управления, обновленного на следующий день.

Например, если включить многофакторную проверку подлинности (MFA) на портале Azure AD, диспетчер соответствия требованиям обнаруживает этот параметр и отображает его в сведениях о решении на доступ к управлению. И наоборот, если вы не включили MFA, то есть флаги диспетчера соответствия требованиям, которые необходимо предпринять.

Узнайте больше о [безопасном показателе и принципах его работы](../security/mtp/microsoft-secure-score-new.md).
  
## <a name="action-types-and-points"></a>Типы и точки действий

Диспетчер соответствия требованиям отслеживает два типа действий:

1. **Действия по улучшению**: действия, которыми управляет ваша организация.
2. **Действия Майкрософт**: действия, которыми управляет Майкрософт.

Оба типа действий имеют точки, которые подсчитаны к общему показателю после завершения.

### <a name="technical-and-non-technical-actions"></a>Технические и нетехнические действия

Действия группируются по определенному техническим или нетехническим видам. Влияние каждого действия на результат зависит от типа.

- **Технические действия** реализуются путем взаимодействия с технологией решения (например, для изменения конфигурации). Баллы для технических действий предоставляются один раз для каждого действия, независимо от того, сколько групп им принадлежит.

- **Нетехнические действия** управляются вашей организацией и реализуются способами, отличными от работы с технологией решения. Существует два типа не технических действий: **Документация** **и работа**. Баллы для этих действий применяются к рейтингу соответствия на уровне группы. Это означает, что если действие существует в нескольких группах, вы получите значение точки действия каждый раз при его реализации в группе.

**Пример оценки технических и нетехнических действий:**

Предположим, что у вас есть техническое действие с тремя точками, которые существуют в пяти группах, и у вас не было технического действия, посвященном трем точкам, которые существуют в одной группе 5.

При успешном внедрении технического действия общее число баллов, которые вы получаете, равно 3. Это связано с тем, что для клиента требуется только один раз реализовать действие. Реализация и состояние тестирования для технического действия будут отображаться одинаково во всех экземплярах этого действия в каждой группе, к которой он принадлежит.

Если вы успешно реализовали нетехнические действия в каждой из пяти групп, общее количество получаемых баллов составляет 15. Это связано с тем, что необходимо реализовать действие в каждой группе. Состояние реализации и тестирования для нетехнических действий будет отличаться в разных группах, так как действие реализуется отдельно в каждой из групп.

Эта логика оценки предназначена для предоставления наиболее точных учетных данных по реализации и тестированию действий в Организации.

### <a name="how-score-values-are-determined"></a>Как определяются значения оценок
 
Действиям назначаются значения оценки, зависящие от того, являются ли они обязательными или непревентивными, а также являются ли они превентивными, обнаруживающее или корректирующими.

### <a name="mandatory-and-discretionary-actions"></a>Обязательные и избирательные действия

 - **Обязательные действия** не могут быть обходиться намеренно или случайно. Примером обязательного действия является централизованно управляемая политика паролей, устанавливающая требования к длине, сложности и сроку действия пароля. Для доступа к системе пользователи должны выполнить эти требования.
  
 - Для выполнения **определенных действий** пользователи должны знать и придерживаться политики. Например, политика, требующая, чтобы пользователи блокировали свой компьютер при их выходе, это неизбирательное действие, так как оно зависит от пользователя.
  
### <a name="preventative-detective-and-corrective-actions"></a>Меры по превентивной, обнаруживающее и корректирующим действиям
  
 - Действия, связанные с **превентивной превентивной** адресацией. Например, защита информации на REST с помощью шифрования является предупредительным действием при атаках и нарушениях безопасности. Разделение обязанностей является превентивным действием по управлению конфликтами интересов и защите от мошенничества.
  
 - **Действия обнаруживающее** активно контролируют системы на обнаружение нерегулярных условий или поведений, представляющих риск, или которые можно использовать для обнаружения вторжений или нарушений. Примеры включают аудит системного доступа и привилегированные административные действия. Аудит соответствия нормативным требованиям — это тип действия обнаруживающее, используемого для обнаружения проблем с процессами.
  
- **Действия по исправлению** попытаются сохранить неблагоприятные последствия инцидента безопасности как минимум, предпринять корректирующие меры, чтобы уменьшить немедленный эффект, и при необходимости отменить повреждение. Ответ на инцидент конфиденциальности — это корректирующее действие по ограничению повреждений и восстановления систем в работоспособном состоянии после нарушения безопасности.
  
Каждое действие имеет назначенное значение в диспетчере соответствия требованиям в зависимости от того, какой риск он представляет:

|**Type**|**Назначенная Оценка**|
|:-----|:-----|
| Обязательная превентивная | 27 |
| Неизбирательные дискреционные меры | 9  |
| Обнаруживающее является обязательным | 4 |
| Дискреционный обнаруживающее | 1,1 |
| Корректирующая обязательная | 4 |
| Корректирующая избирательность | 1,1 |
