 - <span data-ttu-id="268f2-168">**Обязательные действия** не могут быть обходиться намеренно или случайно.</span><span class="sxs-lookup"><span data-stu-id="268f2-168">**Mandatory actions** can't be bypassed, either intentionally or accidentally.</span></span> <span data-ttu-id="268f2-169">Примером обязательного действия является централизованно управляемая политика паролей, устанавливающая требования к длине, сложности и сроку действия пароля.</span><span class="sxs-lookup"><span data-stu-id="268f2-169">An example of a mandatory action is a centrally managed password policy that sets requirements for password length, complexity, and expiration.</span></span> <span data-ttu-id="268f2-170">Для доступа к системе пользователи должны выполнить эти требования.</span><span class="sxs-lookup"><span data-stu-id="268f2-170">Users must follow these requirements to access the system.</span></span>
  
 - <span data-ttu-id="268f2-171">Для выполнения **определенных действий** пользователи должны знать и придерживаться политики.</span><span class="sxs-lookup"><span data-stu-id="268f2-171">**Discretionary actions** rely upon users to understand and adhere to a policy.</span></span> <span data-ttu-id="268f2-172">Например, политика, требующая, чтобы пользователи блокировали свой компьютер при их выходе, это неизбирательное действие, так как оно зависит от пользователя.</span><span class="sxs-lookup"><span data-stu-id="268f2-172">For example, a policy requiring users to lock their computer when they leave it is a discretionary action because it relies on the user.</span></span>
  
### <a name="preventative-detective-and-corrective-actions"></a><span data-ttu-id="268f2-173">Меры по превентивной, обнаруживающее и корректирующим действиям</span><span class="sxs-lookup"><span data-stu-id="268f2-173">Preventative, detective, and corrective actions</span></span>
  
 - <span data-ttu-id="268f2-174">Действия, связанные с **превентивной превентивной** адресацией.</span><span class="sxs-lookup"><span data-stu-id="268f2-174">**Preventative actions** address specific risks.</span></span> <span data-ttu-id="268f2-175">Например, защита информации на REST с помощью шифрования является предупредительным действием при атаках и нарушениях безопасности.</span><span class="sxs-lookup"><span data-stu-id="268f2-175">For example, protecting information at rest using encryption is a preventative action against attacks and breaches.</span></span> <span data-ttu-id="268f2-176">Разделение обязанностей является превентивным действием по управлению конфликтами интересов и защите от мошенничества.</span><span class="sxs-lookup"><span data-stu-id="268f2-176">Separation of duties is a preventative action to manage conflict of interest and guard against fraud.</span></span>
  
 - <span data-ttu-id="268f2-177">**Действия обнаруживающее** активно контролируют системы на обнаружение нерегулярных условий или поведений, представляющих риск, или которые можно использовать для обнаружения вторжений или нарушений.</span><span class="sxs-lookup"><span data-stu-id="268f2-177">**Detective actions** actively monitor systems to identify irregular conditions or behaviors that represent risk, or that can be used to detect intrusions or breaches.</span></span> <span data-ttu-id="268f2-178">Примеры включают аудит системного доступа и привилегированные административные действия.</span><span class="sxs-lookup"><span data-stu-id="268f2-178">Examples include system access auditing and privileged administrative actions.</span></span> <span data-ttu-id="268f2-179">Аудит соответствия нормативным требованиям — это тип действия обнаруживающее, используемого для обнаружения проблем с процессами.</span><span class="sxs-lookup"><span data-stu-id="268f2-179">Regulatory compliance audits are a type of detective action used to find process issues.</span></span>
