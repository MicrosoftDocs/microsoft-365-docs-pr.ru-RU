# <a name="get-started-with-compliance-manager"></a>Начало работы с диспетчером соответствия требованиям

**В этой статье:** Эта статья поможет вам настроить диспетчер соответствия требованиям. Узнайте, как **получить доступ к** диспетчеру соответствия требованиям, **задать роли и разрешения**, а также настроить **Автоматическое тестирование действий по улучшению**. Просмотрите **панель мониторинга управления соответствием требованиям** и ознакомьтесь с основными страницами: страница "действия по улучшению", страница "решения", страница "оценки" и страница "шаблоны оценки".

## <a name="who-can-access-compliance-manager"></a>Кто может получить доступ к диспетчеру соответствия требованиям

Диспетчер соответствия требованиям доступен для организаций с лицензиями Office 365 и Microsoft 365. Возможности оценки доступности и управления зависят от лицензионного соглашения.  [Просмотр сведений об описании службы](https://go.microsoft.com/fwlink/?linkid=2132371).

## <a name="before-you-begin"></a>Прежде чем начать

Глобальный администратор Microsoft 365 для Организации, скорее всего, будет первым пользователем для доступа к диспетчеру соответствия требованиям. Мы рекомендуем глобальному администратору войти и настроить разрешения пользователей, как описано ниже, при первом запуске диспетчера соответствия требованиям.

## <a name="sign-in"></a>Вход

1. Перейдите в [центр соответствия требованиям microsoft 365](https://compliance.microsoft.com/) и **Войдите в** систему с помощью учетной записи глобального администратора Microsoft 365.
2. В области навигации слева выберите элемент **Диспетчер соответствия требованиям** . Вы будете приступать к [панели мониторинга "Управление соответствием требованиям"](#understand-the-compliance-manger-dashboard).

Прямой канал для доступа к диспетчеру соответствия требованиям: [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) .

## <a name="set-user-permissions-and-assign-roles"></a>Настройка разрешений пользователей и назначение ролей

Диспетчер соответствия требованиям использует модель разрешений управления доступом на основе ролей (RBAC). Только пользователи, которым назначена роль, могут получать доступ к диспетчеру соответствия требованиям, а действия, разрешенные каждым пользователем, ограничиваются [типом роли](#role-types).

### <a name="where-to-set-permissions"></a>Место для задания разрешений

Пользователь, владеющий ролью глобального администратора организации, может устанавливать разрешения пользователей в центре соответствия требованиям Microsoft 365, а также в Azure Active Directory (Azure AD).

Чтобы задать разрешения и назначить роли в центре соответствия требованиям Microsoft 365, выполните следующие действия:

1. Выберите **разрешения** для навигации в левой части [центра соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. В верхней части экрана выберите ссылку **"для просмотра и управления ролями в Office 365...".** Откроется новая вкладка в центре безопасности & безопасности Office 365 ([сведения о том, почему вы перенаправлены](microsoft-365-compliance-center.md#frequently-asked-questions)).

3. Найдите группу ролей, в которую необходимо добавить одного или нескольких пользователей, и установите флажок слева от имени группы. (Обратитесь к [списку ролей и связанных функций ниже](#role-types). Имена групп ролей имитируют имя роли.

4. В раскрывающейся области для этой группы выберите **изменить** под заголовком **элементы** .

5. Нажмите **кнопку Выбрать элементы**. Появится еще одно всплывающее окно.

6. Нажмите кнопку **+ Добавить** , чтобы выбрать одного или нескольких пользователей, которых нужно добавить в группу.

7. Установите флажок рядом с именами, которые вы хотите добавить, а затем нажмите кнопку **Добавить** в нижней части страницы.

8. Завершив назначение пользователей, нажмите кнопку **Готово**, а затем нажмите кнопку **сохранить**, а затем **Закрыть**.

Чтобы обратиться за помощью к диспетчеру соответствия требованиям, обратитесь в [cmresearch@microsoft.com](mailto:cmresearch@microsoft.com).

## <a name="settings-for-automated-testing-and-user-history"></a>Параметры автоматического тестирования и истории пользователей

Параметры диспетчера соответствия требованиям в центре соответствия требованиям Microsoft 365 позволяют включать и отключать автоматическое тестирование действий по улучшению. Кроме того, эти параметры позволяют управлять данными пользователей, связанными с действиями по улучшению, включая возможность повторного назначения действий по улучшению другому пользователю.  Только пользователи с ролью глобального администратора или диспетчера соответствия требованиям могут получить доступ к параметрам диспетчера соответствия требованиям.

### <a name="set-up-automated-testing"></a>Настройка автоматического тестирования

Некоторые действия по улучшению в диспетчере соответствия требованиям отслеживаются также с помощью [оценки безопасности Майкрософт](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score). Вы можете настроить автоматическое тестирование действий, которые будут совместно отслеживаться, что означает, что если действие тестируется и обновляется по безопасному показателю, результаты синхронизируются с теми же действиями в диспетчере соответствия требованиям и подсчета на оценку соответствия требованиям.

Автоматическое тестирование включается по умолчанию для новых организаций в диспетчере соответствия требованиям. При первом развертывании Microsoft 365 или Office 365 необходимо примерно семь дней, чтобы обеспечить полную сбор данных и присвоить им оценку соответствия требованиям.  При включенном автоматическом тестировании Дата тестовой даты действия не будет обновлена, но состояние проверки обновится. При создании новых оценок результаты автоматически включают показатели управления Microsoft и интеграцию рейтинга.

Глобальный администратор организации может изменить параметры автоматического тестирования в любое время. Вы можете отключить автоматическое тестирование для стандартных действий по улучшению или включить его для отдельных действий. Чтобы изменить параметры автоматического тестирования, следуйте приведенным ниже инструкциям.

#### <a name="to-manage-your-automated-testing-settings"></a>Для управления параметрами автоматического тестирования сделайте следующее:

1. Выберите **Параметры** в левой области навигации в [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. На странице Параметры выберите **Диспетчер соответствия требованиям**.

3. Выберите **Автоматическое тестирование** в левой области навигации.

4. Нажмите кнопку "Применить", чтобы включить автоматическое тестирование для всех действий по улучшению, отключите его для всех действий или включите отдельное действие.

5. Если выбрано **Включение для каждого действия по улучшению**, в списке отобразятся все доступные действия по улучшению.  Установите флажок рядом с каждым действием, которое необходимо автоматически протестировать.

6. Нажмите кнопку **сохранить** , чтобы сохранить параметры. В верхней части экрана будет выведено сообщение о том, что выбор сохранен. Если появится сообщение об ошибке, повторите попытку.

**Примечание:** Включать и отключать автоматическое обновление для всех действий может только глобальный администратор. Администратор управления соответствием может включить автоматическое обновление для отдельных действий, но не для глобальных действий.

### <a name="manage-user-history"></a>Управление журналом пользователей

Параметры " **Управление журналом пользователей** " помогают быстро определить, какие пользователи работали с действиями по улучшению в диспетчере соответствия требованиям. Данные идентифицируемого пользователя, связанные с действиями по улучшению, включают все выполненные операции по реализации и тестированию, документы, отправленные и все введенные вами заметки. Понимание и получение данных этого типа может потребоваться для собственных потребностей Организации.

Кроме того, параметры журнала пользователей позволяют переназначить все действия по улучшению от одного пользователя к другому.

**Чтобы найти параметры журнала пользователей, выполните указанные ниже действия.**

1. Выберите параметры в левой области навигации в [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. На странице Параметры выберите **Диспетчер соответствия требованиям**.

3. Выберите **Управление журналом пользователей** с помощью левой панели навигации.

На странице " **Управление журналом пользователей** " отображается список всех пользователей по адресу электронной почты, назначенному действию улучшения. С помощью кнопки **поиска** можно быстро найти определенного пользователя, введя его адрес электронной почты.

Справа от каждого адреса электронной почты каждого пользователя в раскрывающемся меню **Выбор** предоставляются параметры для экспорта отчета, повторное назначение действий по улучшению или удаление журнала. В каждом разделе ниже приведены подробные сведения о каждом параметре.

#### <a name="export-a-report-of-user-history-data"></a>Экспорт отчета о данных истории пользователей

Вы можете экспортировать файл Excel, содержащий список действий по улучшению, назначенных пользователю в данный момент.  В отчете также перечислены все файлы свидетельств, отправленные этим пользователем. Эта информация поможет вам повторно назначить открытые действия по улучшению.

Отчет отражает состояние действия по улучшению в соответствии с его датой создания. Он не является историческим отчетом о всех предыдущих изменениях его состояния или назначения (Узнайте, как [экспортировать отчет со страницы "действия по улучшению](compliance-manager-improvement-actions.md#export-a-report)").

**Выполните указанные ниже действия, чтобы экспортировать отчет по пользователю.**

1. Выберите **Параметры** в левой области навигации в [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. На странице Параметры выберите **Диспетчер соответствия требованиям**.

3. Выберите **Управление журналом пользователей** в левой части окна навигации.

4. Найдите предполагаемого пользователя, выполнив поиск в списке адресов электронной почты или выбрав **Поиск** и введя адрес электронной почты пользователя.

5. В раскрывающемся меню **Выбор** выберите пункт **Экспорт отчета**.

6. После создания файла Excel вы можете открыть его и сохранить на локальном компьютере.

#### <a name="reassign-improvement-actions-to-another-user"></a>Повторное назначение действий по улучшению другому пользователю

Вы можете переназначить действия по улучшению от одного пользователя к другому. При повторном назначении действия журнал отправки документов не изменяется, но имя пользователя, который изначально передал эту документацию, больше не отображается в действии улучшения.

**Выполните указанные ниже действия, чтобы переназначить действия по улучшению другому пользователю:**

1. Выберите **Параметры** в левой области навигации в [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. На странице Параметры выберите **Диспетчер соответствия требованиям**.

3. Выберите **Управление журналом пользователей** в левой части окна навигации.

4. Найдите пользователя, выполнив поиск в списке адресов электронной почты или выбрав **Поиск** и введя адрес электронной почты этого пользователя.

5. В раскрывающемся меню **Выбор** выберите команду **переназначить действия по улучшению**. Откроется раскрывающаяся панель " **переназначить действия улучшения** ".

6. В поле **Поиск пользователей** введите имя или адрес электронной почты пользователя *, которому необходимо назначить действия по*улучшению.

7. Когда появится имя предполагаемого пользователя, которому **будут назначены действия по улучшению**, выберите пользователя, а затем нажмите **назначить действия**.

8. После завершения перераспределения вы увидите сообщение с подтверждением в раскрывающейся области, подтверждающее, что все действия по улучшению предыдущего пользователя были переназначены новому пользователю. При получении уведомления об отказе в переназначении закройте окно и повторите попытку. Чтобы закрыть раскрывающуюся область, нажмите кнопку **done (Готово**).

Новый Уполномоченный пользователь получает электронное письмо, которое было назначено действию улучшения. Сообщение электронной почты содержит прямую ссылку на страницу сведений действия по улучшению.
 
 > [!NOTE]
> Если вы переназначите действие с ожидающим обновлением, прямая ссылка на действие в сообщении с переназначением будет прервана, если обновление будет принято после перераспределения. Это можно исправить, повторно назначив пользователю действие после принятия обновления. Узнайте больше об [обновлениях действий по улучшению](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions).

#### <a name="delete-user-history"></a>Удаление журнала пользователя

Удаление журнала пользователя приведет к его удалению в качестве владельца действий по улучшению и удалению имени из всех остальных полей в диспетчере соответствия требованиям. Когда вы удаляете историю пользователя, действия по улучшению, которыми они владеют, не будут отображать **назначенные** значения до тех пор, пока не будет назначен новый пользователь. Все документы, отправленные на действие улучшения, будут показываться **пользователю, удаленному** вместо имени удаленного пользователя. Удаление журнала пользователей является необратимым.

Чтобы удалить журнал пользователя, выполните указанные ниже действия.

1. Выберите **Параметры** в левой области навигации в [центре соответствия требованиям Microsoft 365](https://compliance.microsoft.com/).

2. На странице Параметры выберите **Диспетчер соответствия требованиям**.

3. Выберите **Управление журналом пользователей** в левой части окна навигации.

4. Найдите пользователя, выполнив поиск в списке адресов электронной почты или выбрав **Поиск** и введя адрес электронной почты этого пользователя.

5. В раскрывающемся меню **Выбор** выберите команду **Удалить журнал**.

6. Откроется окно с запросом на подтверждение окончательного удаления журнала пользователя. Чтобы продолжить удаление, выберите **Удалить журнал**. Чтобы оставить журнал без удаления, нажмите **кнопку Отмена**.

7. Вы вернетесь на страницу " **Управление журналом пользователей** " с сообщением подтверждения в начале удаления журнала для пользователя.

## <a name="understand-the-compliance-manger-dashboard"></a>Общие сведения о панели мониторинга диспетчера соответствия требованиям

Панель мониторинга "Управление соответствием требованиям" предназначена для того, чтобы предоставить вам наглядный вид текущей проверки соответствия требованиям.

![Диспетчер соответствия требованиям — панель мониторинга](../media/compliance-manager-dashboard.png "Панель мониторинга диспетчера соответствия требованиям")

### <a name="overall-compliance-score"></a>Общий показатель соответствия требованиям

Ваш рейтинг соответствия требованиям очень наглядно представлен вверху. В нем показано процентное отношение на основе баллов, достижимых для выполнения действий по улучшению, которые имеют ключевые стандарты защиты данных и нормативные требования. Точки из [действий Майкрософт](compliance-manager-assessments.md#microsoft-actions-tab), управляемых моей Майкрософт, также подносят оценку соответствия требованиям.

Когда вы впервые переходите к диспетчеру соответствия требованиям, первоначальный показатель основан на [базовом уровне защиты данных Microsoft 365](compliance-manager-assessments.md#data-protection-baseline-default-assessment). Эта базовая оценка, доступная всем организациям, представляет собой набор элементов управления, включающих общие отраслевые нормы и стандарты. Диспетчер соответствия требованиям сканирует существующие решения Microsoft 365 и предоставляет начальную оценку в соответствии с текущими параметрами конфиденциальности и безопасности. При добавлении оценок, относящихся к вашей организации, ваш рейтинг становится более понятным для вас.

**Подробнее:** [Узнайте, как рассчитывается рейтинг соответствия требованиям](compliance-score-calculation.md).

### <a name="key-improvement-actions"></a>Основные действия по улучшению

В этом разделе перечислены основные действия по улучшению, которые можно выполнить прямо сейчас, чтобы получить наибольшее положительное воздействие на общий рейтинг соответствия требованиям. Выберите **Просмотреть все действия по улучшению** , чтобы перейти на страницу действий по улучшению.

### <a name="solutions-that-affect-your-score"></a>Решения, влияющие на оценку

В этом разделе описываются решения, содержащие действия по улучшению, которые могут оказать положительное воздействие на вашу оценку, а также количество незаконченных действий по улучшению этих решений. Выберите **Просмотреть все решения** , чтобы посетить страницу решений.

### <a name="compliance-score-breakdown"></a>Детализация оценки соответствия требованиям

В этом разделе представлены более подробные сведения о рейтинге двумя различными способами:

- **Categories**: показывает процент общей оценки в категориях защиты данных, например "Защита данных" или "Управление устройствами".
- **Оценки**: показывает процентное отношение процесса управления оценкой для определенных требований и стандартов защиты данных, нормативов или законов, таких как GDPR или NIST 800-53.

### <a name="filtering-your-dashboard-view"></a>Фильтрация представления панели мониторинга

Вы можете отфильтровать представление панели мониторинга, чтобы увидеть только те элементы, которые относятся к определенным нормативам, стандартам, решениям, типам действий, группам оценки или категориям защиты данных. Таким образом, Фильтрация представления позволяет отфильтровать оценку на панели мониторинга, отображая количество баллов, которые вы получили из всех возможных точек на основе условий фильтрации.

Чтобы применить фильтры:

1. Выберите **Фильтр** в верхней правой части панели мониторинга.
2. Выберите критерии фильтрации в раскрывающейся панели **фильтры** , а затем нажмите кнопку **Применить**.

После применения фильтра вы увидите, что ваш рейтинг скорректирован в режиме реального времени. Процент соответствия требованиям и сведения о разбиении на них, а также действия и решения для улучшения теперь относятся только к данным, включенным в условия фильтрации. Если вы выйдете из диспетчера соответствия требованиям, отфильтрованное представление останется при входе в систему.

Чтобы удалить фильтры:

- В заголовке **Примененные фильтры** нажмите кнопку **X** рядом с нужным фильтром, чтобы удалить его. также
- Выберите **Фильтр** в верхней правой части панели мониторинга, а затем в раскрывающемся меню **фильтры** выберите **Очистить фильтры**.

## <a name="improvement-actions-page"></a>Страница "действия по улучшению"

[Действия по улучшению](compliance-manager-improvement-actions.md) — это действия, которыми управляет Организация. Работа с действиями по улучшению помогает централизовать действия по обеспечению соответствия требованиям и обеспечить соответствие нормативным требованиям и стандартам защиты данных. Каждое действие по улучшению содержит подробные инструкции по реализации и ссылку для запуска соответствующего решения. Действия по улучшению могут быть назначены пользователям в Организации для выполнения реализации и тестирования. Вы также можете хранить документацию, заметки и обновления состояния записи в действии улучшения.

### <a name="view-your-improvement-actions"></a>Просмотр действий по улучшению

В панели мониторинга "Управление соответствием" отображаются **основные действия по улучшению.** Чтобы просмотреть все действия по улучшению, перейдите на вкладку действия по улучшению на панели мониторинга, на которой вы переводите страницу в действие улучшения. Вы также можете выбрать Просмотр всех действий по улучшению в списке ключевых действий улучшения на панели мониторинга, чтобы перейти на страницу действий по улучшению.

На странице действия по улучшению отображаются все действия по улучшению, которые управляются вашей организацией. Действия, которые управляются корпорацией Майкрософт, можно просматривать в каждой оценке (Дополнительные сведения о [действиях Майкрософт](compliance-manager-assessments.md#microsoft-actions-tab)).

Если на странице действия по улучшению имеется длинный список действий, может быть полезно отфильтровать представление. Выберите пункт **Фильтр** в правом верхнем углу списка действия. Когда откроется раскрывающаяся панель **фильтры** , выберите критерии на основе нормативных требований, стандартов, решений и групп. Кроме того, вы можете настроить представление, выбрав **группу** в правом верхнем углу. В раскрывающемся меню выберите для просмотра по группе, решению, категории, типу действия или состоянию.

Представление по умолчанию для этой страницы не показывает действия по улучшению с **пройденным**состоянием тестирования. Чтобы просмотреть прошедшее тестирование действия, установите флажок **переданное** поле в раскрывающейся панели фильтры. Только действия со статусом проверки **переданных** на ваш счет. Некоторые действия могут показывать **отложенные метки обновления.** Узнайте больше об [обновлениях действий по улучшению](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions).

На странице действия по улучшению отображаются следующие точки данных для каждого действия по улучшению:

- **Достигнуто баллов**: количество баллов, полученных из общего количества баллов за счет выполнения действия.
- **Нормативные требования**: требования или стандарты, относящиеся к действию
- **Группа**: группа, которой назначено действие
- **Решения**: решение, с помощью которого можно выполнить действие
- **Оценки**: оценки, содержащие действие
- **Категории**: связанная Категория защиты данных (например, защита информации, Управление устройствами и т. д.)
- **Состояние тестирования**:
    - **Нет** – обновление состояния не записано
    - **Не оценено** — тестирование не было начато
    - **Успешное** тестирование реализации
    - **Сбой при низком риске** — сбой тестирования, недостаток риска
    - **Сбой среднего риска** при неудачном тестировании, умеренный риск
    - **Сбой при высоком риске** — сбой тестирования, высокий риск
    - **Вне области** действия — действие не входит в область оценки и не влияет на оценку.
    - **Для обнаружения** при ручном тестировании указывает, что действие было реализовано, но не проверено; для автоматических тестов указывает, что действие ожидает результатов автоматизации.
    - **Не удалось обнаружить** : невозможно определить автоматическое состояние
    - **Частично протестировано** — Автоматическая оценка с неполными баллами

**Подробнее:** Узнайте [, как назначать и выполнять работу над действиями по улучшению](compliance-manager-improvement-actions.md).

## <a name="solutions-page"></a>Страница "решения"

На странице Solutions показан общий доступ к общим и потенциальным точкам, упорядоченный по решениям. Просмотр оставшихся баллов и действий по улучшению в этом представлении помогает определить, какие решения требуют более быстрого внимания.

Найдите страницу решения, выбрав вкладку **решения** на панели мониторинга диспетчера соответствия требованиям. Вы также можете выбрать **Просмотр всех решений** под **решениями, которые влияют на рейтинг** в правом верхнем углу панели мониторинга.

### <a name="filtering-your-solutions-view"></a>Фильтрация представления решений

Чтобы отфильтровать представление решений, выполните указанные ниже действия.

1. Выберите пункт **Фильтр** в левом верхнем углу списка оценки.
2. В раскрывающейся области **фильтры** установите флажок рядом с нужным критерием (стандарты и нормативы, решение, тип действия, Группа диспетчера соответствия требованиям, категория).
3. Нажмите кнопку **Apply (применить** ). Область фильтра будет закрыта, и вы увидите отфильтрованное представление.

Вы также можете изменить представление, чтобы просмотреть оценки по группам, продуктам или нормативным данным, выбрав тип группировки в раскрывающемся меню **Группа** над списком оценки.

### <a name="taking-action-from-the-solution-page"></a>Выполнение действий со страницы решения

На странице "решения" отображаются решения вашей организации, которые подключены к действиям по улучшению. В таблице перечислены вклады каждого решения в общий рейтинг, количество баллов, которые можно достигнуть в этом решении, а также оставшееся количество действий по улучшению, сгруппированных в этом решении, которые могут увеличить рейтинг.

На этом экране можно выполнить два способа действий:

1. В столбце " **оставшиеся** решения" строки предполагаемого решения выберите номер гиперссылки. На экране действия по улучшению отображается отфильтрованное представление, в котором показаны непроверенные действия по улучшению для этого решения.

2. В строке предполагаемого решения в столбце **открыть решение** нажмите кнопку **Открыть**. Вы увидите решение или расположение в центрах безопасности и соответствиях Microsoft 365 и Office 365, где можно выполнить Рекомендуемые действия.

## <a name="assessments-page"></a>Страница "оценки"

На странице "оценки" перечислены все [оценки](compliance-manager-assessments.md) , настроенные для Организации. Знаменатель показателя соответствия определяется всеми отслеживаемыми оценками. По мере добавления оценок вы увидите дополнительные действия по улучшению, приведенные на странице действия по улучшению, а также увеличится делитель оценки соответствия требованиям.

На странице "оценки" собраны основные сведения о каждой оценке:

- **Оценка**: название оценки
- **Состояние**:
    - **Завершено** — все элементы управления имеют состояние "передано" или по крайней мере один из них передаются, а оставшаяся часть "вне области".
    - **Неполный** — по крайней мере один элемент управления имеет состояние "Failed"
    - **None** — все элементы управления не тестировались
    - **В ходе выполнения действия по** улучшению имеют любое другое состояние, в том числе «выполняется», «частичный кредит» или «не обнаружено».
- **Ход оценки**: процент работы, выполненной в направлении завершения, измеряемый числом успешно проверенных элементов управления
- **Действия по улучшению**: количество выполненных действий, которые удовлетворяют реализации элементов управления
- **Действия Майкрософт**: количество выполненных действий в соответствии с реализацией элементов управления Майкрософт
- **Группа**: имя группы, к которой относится оценка
- **Продукт**: связанная служба Microsoft 365
- **Нормативная информация:** нормативный стандарт, политика или закон, применимые к оценке

3. Нажмите кнопку **Apply (применить** ). Область фильтра будет закрыта, и вы увидите отфильтрованное представление.

Вы также можете изменить представление, чтобы просмотреть оценки по группам, продуктам или нормативным данным, выбрав тип группировки в раскрывающемся меню **Группа** над списком оценки.
