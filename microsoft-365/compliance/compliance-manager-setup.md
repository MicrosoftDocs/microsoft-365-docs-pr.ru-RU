# <a name="get-started-with-compliance-manager"></a>Начало работы с диспетчером соответствия требованиям

**В этой статье:** Эта статья поможет вам настроить диспетчер соответствия требованиям. Узнайте, как **получить доступ к диспетчеру** соответствия требованиям, **установить роли** и разрешения, а также настроить автоматическое тестирование **действий по улучшению.** Пройдите **информационную** панель в диспетчере соответствия требованиям и найдите основные страницы: страницу действий по улучшению, страницу решений, страницу оценок и страницу шаблонов оценки.

## <a name="who-can-access-compliance-manager"></a>Кто может получить доступ к диспетчеру соответствия требованиям

Диспетчер соответствия требованиям доступен организациям с лицензиями на Office 365 и Microsoft 365, а также клиентам с облачными службами сообщества для государственных организаций США (GCC) и клиентами GCC High. Доступность оценки и возможности управления зависят от лицензионного соглашения.  [Просмотр сведений о описании службы.](https://go.microsoft.com/fwlink/?linkid=2132371)

## <a name="before-you-begin"></a>Перед началом работы

Глобальный администратор Microsoft 365 для вашей организации, скорее всего, будет первым пользователем, который получит доступ к диспетчеру соответствия требованиям. Мы рекомендуем глобальному администратору войти и установить разрешения пользователей, как описано ниже при первом посещении диспетчера соответствия требованиям.

## <a name="sign-in"></a>Вход

1. Перейдите в [Центр соответствия требованиям Microsoft 365](https://compliance.microsoft.com/) и войдите **с** помощью учетной записи глобального администратора Microsoft 365.
2. Выберите **диспетчер соответствия требованиям** на левой области навигации. Вы будете перенаправлены на информационную [панель диспетчера соответствия требованиям.](#understand-the-compliance-manager-dashboard)

Прямая ссылка на доступ к диспетчеру соответствия [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) требованиям: .

## <a name="set-user-permissions-and-assign-roles"></a>Настройка разрешений пользователя и назначение ролей

Диспетчер соответствия требованиям использует модель разрешений управления доступом на основе ролей (RBAC). Только пользователи, которым назначена роль, могут получать доступ к диспетчеру соответствия требованиям, а действия, разрешенные каждым пользователем, ограничены [типом роли.](#role-types)

### <a name="where-to-set-permissions"></a>Где устанавливать разрешения

Пользователь с ролью глобального администратора в вашей организации может установить разрешения пользователя для диспетчера COmpliance. Разрешения можно настроить в Центре безопасности и соответствия требованиям Office 365& а также в Azure Active Directory (Azure AD).

> [!NOTE]
> Клиенты в средах с высоким уровнем сообщества для государственных организаций США (GCC) могут устанавливать только разрешения и роли пользователей для диспетчера соответствия требованиям в Azure AD. Инструкции azure AD и определения типов ролей см. ниже.

Чтобы задать разрешения и назначить роли в Центре безопасности и & соответствия требованиям Office 365, выполните следующие действия.

1. Перейдите в Центр безопасности и соответствия & [Office 365](https://protection.office.com/) и выберите **"Разрешения** для навигации слева".

2. Найдите группу ролей, в которую нужно добавить одного или несколько пользователей, и пойдите слева от имени группы. (См. [список ролей и связанных функций ниже.](#role-types) Имена групп ролей имитируют имя роли.)

3. Во flyout pane for that group, select **Edit** under the **Members** header.

4. Select **Choose members**. Появится еще одно окно.

5. Select **+ Add** to choose one or more users to add to the group.

6. Рядом с именами, которые вы хотите добавить,  выберите кнопку "Добавить" в нижней части.

7. После назначения пользователей выберите **"Готово",** затем **"Сохранить"** и **"Закрыть".**

## <a name="settings-for-automated-testing-and-user-history"></a>Параметры автоматического тестирования и истории пользователей

Параметры диспетчера соответствия требованиям в Центре соответствия требованиям Microsoft 365 позволяют включить и отключить автоматическое тестирование действий по улучшению. Эти параметры также позволяют управлять данными пользователей, связанных с действиями по улучшению, включая возможность перенаправки действий по улучшению другому пользователю.  Только люди с ролью глобального администратора или администратора диспетчера соответствия требованиям могут получить доступ к настройкам диспетчера соответствия требованиям.

> [!NOTE]
> Функция автоматического тестирования недоступна для клиентов в средах GCC High, так как в этих средах недоступна оценка безопасности. Клиентам GCC High потребуется вручную реализовать и протестировать свои действия по улучшению.

### <a name="set-up-automated-testing"></a>Настройка автоматического тестирования

Некоторые действия по улучшению в диспетчере соответствия требованиям также отслеживаются службой ["Оценка безопасности (Майкрософт)".](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score) Вы можете настроить автоматическое тестирование действий, которые совместно отслеживаются, то есть при тестировании и обновлении действия в оценке безопасности эти результаты синхронизируются с теми же действиями в диспетчере соответствия требованиям и учитываются в вашем показателе соответствия требованиям.

Автоматическое тестирование включено по умолчанию для организаций, которые впервые в диспетчере соответствия требованиям. При первом развертывании Microsoft 365 или Office 365 оценка безопасности занимает около семи дней, чтобы полностью собирать данные и учитывать их в вашем показателе соответствия требованиям.  Если автоматическое тестирование включено, дата тестирования действия не будет обновлена, но состояние проверки будет обновлено. После создания новых оценок оценки автоматически включают оценки контроля Майкрософт и интеграцию оценки безопасности.

Глобальный администратор организации может изменить параметры автоматического тестирования в любое время. Вы можете отключить автоматическое тестирование для распространенных действий по улучшению или включить его для отдельных действий. Следуйте инструкциям ниже, чтобы изменить параметры автоматического тестирования.

#### <a name="to-manage-your-automated-testing-settings"></a>Для управления настройками автоматического тестирования:

1. Выберите **параметры навигации** слева из любого места в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/)

2. На странице параметров выберите диспетчер **соответствия требованиям.**

3. Выберите **автоматическое тестирование в** левой области навигации.

4. Выберите применимую кнопку, чтобы включить автоматическое тестирование для всех действий по улучшению, отключить ее для всех действий или включить по отдельным действиям.

5. Если выбрать **"Включить для каждого действия по улучшению",** в списке будут отобрашены все доступные действия по улучшению.  Заключив этот поле рядом с любым действием, для проверки вы хотите автоматически.

6. Выберите **"Сохранить",** чтобы сохранить параметры. В верхней части экрана вы получите подтверждение того, что ваш выбор сохранен. Если вы получили уведомление о сбое, попробуйте еще раз.

**Примечание.** Только глобальный администратор может включить или отключить автоматическое обновление для всех действий. Администратор диспетчера соответствия требованиям может включить автоматические обновления для отдельных действий, но не для всех действий глобально.

### <a name="manage-user-history"></a>Управление историями пользователей

Параметры **"Управление историями** пользователей" помогают быстро определить, какие пользователи работали с действиями по улучшению в диспетчере соответствия требованиям. К идентифицируемым пользовательским данным, связанным с действиями по улучшению, относятся все действия по внедрению и тестированию, загруженные документы и все заметки, которые они ввели. Понимание и сбор данных этого типа может потребоваться для соблюдения требований вашей организации.

Параметры истории пользователей также позволяют перенамерять все действия по улучшению от одного пользователя к другому.

**Чтобы найти параметры истории пользователей:**

1. Выберите параметры навигации слева из любого места в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/)

2. На странице параметров выберите диспетчер **соответствия требованиям.**

3. Выберите **"Управление историями пользователей"** в области навигации слева.

На **странице "Управление историями** пользователей" показан список всех пользователей по адресу электронной почты, которым назначено действие по улучшению. Используйте **кнопку** "Поиск", чтобы быстро найти определенного пользователя, введя его адрес электронной почты.

Справа от адреса электронной почты  каждого пользователя в выпадаемом меню "Выбор" имеются параметры экспорта отчета, перенамерки действий по улучшению или удаления истории. Подробные сведения о каждом параметре см. в каждом разделе ниже.

#### <a name="export-a-report-of-user-history-data"></a>Экспорт отчета по данным истории пользователей

Вы можете экспортировать файл Excel, содержащий список действий по улучшению, которые в настоящее время назначены пользователю.  В отчете также перечислены все файлы свидетельств, загруженные этим пользователем. Эти сведения могут помочь в перенанамеровыванию действий по улучшению открытых данных.

Отчет отражает состояние действия по улучшению на момент его создания. Это не исторический отчет о всех предыдущих изменениях состояния или назначения (узнайте, как экспортировать отчет со страницы действий [по улучшению).](compliance-manager-improvement-actions.md#export-a-report)

**Выполните следующие действия, чтобы экспортировать отчет по пользователям:**

1. Выберите **параметры навигации** слева из любого места в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/)

2. На странице параметров выберите диспетчер **соответствия требованиям.**

3. Выберите **"Управление историями пользователей"** в области навигации слева.

4. Найдите нужного пользователя, выйдя на адрес  электронной почты списка или выбрав "Поиск" и введите адрес электронной почты пользователя.

5. В **выпадаемом** меню "Выбор" выберите **"Экспорт отчета".**

6. После того как файл Excel отчета создан, его можно открыть и сохранить на локальном компьютере.

#### <a name="reassign-improvement-actions-to-another-user"></a>Перенанамерование действий по улучшению другому пользователю

Действия по улучшению можно перенамерять от одного пользователя к другому. При переназначенном действии история отправки документа не меняется, но имя пользователя, который изначально загрузил документацию, больше не отображается в действии по улучшению.

**Выполните следующие действия, чтобы перенакомить действия по улучшению другому пользователю:**

1. Выберите **параметры навигации** слева из любого места в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/)

2. На странице параметров выберите диспетчер **соответствия требованиям.**

3. Выберите **"Управление историями пользователей"** в области навигации слева.

4. Найдите пользователя, выйдя на адрес электронной  почты списка или выбрав "Поиск" и введите адрес электронной почты этого пользователя.

5. В **выпадаемом** меню "Выбор" выберите **"Перенана себя" действий по улучшению.** Появится **flyout-pane** действий по перенанайму действий по улучшению.

6. В поле **"Пользователи** поиска" введите имя или адрес электронной почты пользователя, для которого необходимо назначить действия по *улучшению.*

7. Когда вы увидите имя нужного пользователя в действии **улучшения,** выберите пользователя, а затем выберите **"Назначить действия".**

8. После завершения перена назначения вы увидите во flyout pane подтверждение того, что все действия по улучшению предыдущего пользователя были перена назначены новому пользователю. Если вы получили уведомление о сбое перенамечания, закройте окно и попробуйте еще раз. Чтобы закрыть вылетную области, выберите **"Готово".**

Новый назначаемая получает сообщение электронной почты, которое ему назначено действие по улучшению. Сообщение электронной почты содержит прямую ссылку на страницу сведений о действии по улучшению.
 
 > [!NOTE]
> Если перенаправить действие с ожидающим обновлением, прямая ссылка на действие в сообщении электронной почты о перенайме не будет работать, если обновление будет принято после перенанесения. Это можно исправить, переназначив действие пользователю после принятия обновления. Узнайте больше об [обновлениях для действий по улучшению.](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)

#### <a name="delete-user-history"></a>Удаление истории пользователей

Удаление истории пользователя удалит его как владельца действий по улучшению и удалит его имя из всех других полей в диспетчере соответствия требованиям. При удалении истории пользователя действия по улучшению, которые  им принадлежат, не будут отображать назначенное значение, пока не будет назначен новый пользователь. В любых документах, загруженных  в действие по улучшению, пользователь будет удален, а не имя удаленного пользователя. Удаление истории пользователей не является постоянным.

Чтобы удалить историю пользователя, выполните следующие действия.

1. Выберите **параметры навигации** слева из любого места в Центре соответствия требованиям [Microsoft 365.](https://compliance.microsoft.com/)

2. На странице параметров выберите диспетчер **соответствия требованиям.**

3. Выберите **"Управление историями пользователей"** в области навигации слева.

4. Найдите пользователя, выйдя на адрес электронной  почты списка или выбрав "Поиск" и введите адрес электронной почты этого пользователя.

5. В **выпадаемом** меню "Выбор" выберите пункт **"Удалить историю".**

6. Появится окно с просьбой подтвердить окончательное удаление истории пользователя. Чтобы продолжить удаление, выберите **"Удалить историю".** Чтобы выйти, не удалив историю, выберите **"Отмена".**

7. Вы вернетсяе  на страницу "Управление истории пользователей" с сообщением подтверждения в верхней части страницы об удалении пользователя.

## <a name="understand-the-compliance-manager-dashboard"></a>Понимание информационной панели диспетчера соответствия требованиям

Информационная панель диспетчера соответствия требованиям предоставляет вам быстрое представление о текущем состоянии соответствия требованиям.

![Диспетчер соответствия требованиям — информационная панель](../media/compliance-manager-dashboard.png "Информационная панель диспетчера соответствия требованиям")

### <a name="overall-compliance-score"></a>Общий рейтинг соответствия требованиям

Рейтинг соответствия требованиям находится в верхней части. В нем показан процент на основе пунктов, достижимых для выполнения действий по улучшению, которые будут реализации ключевых стандартов и нормативов защиты данных. Баллы [от действий](compliance-manager-assessments.md#microsoft-actions-tab)Майкрософт, которыми управляет моя корпорация Майкрософт, также учитываются в вашем показателе соответствия требованиям.

Если вы впервые приходите в диспетчер соответствия требованиям, исходная оценка основывается на базовых параметрах защиты данных [Microsoft 365.](compliance-manager-assessments.md#data-protection-baseline-default-assessment) Эта базовая оценка, доступная всем организациям, является набором средств контроля, включающих общие отраслевые нормативы и стандарты. Диспетчер соответствия требованиям проверяет существующие решения Microsoft 365 и дает вам первоначальную оценку на основе текущих параметров конфиденциальности и безопасности. По мере добавления оценок, релевантных для вашей организации, оценка становится более осмысленной для вас.

**Дополнительные.** [Узнайте, как рассчитывается оценка соответствия требованиям.](compliance-score-calculation.md)

### <a name="key-improvement-actions"></a>Основные действия по улучшению

В этом разделе перечислены самые лучшие действия по улучшению, которые можно принять прямо сейчас, чтобы оказать наибольшее положительное влияние на общий рейтинг соответствия требованиям. Выберите **"Просмотреть все действия по** улучшению", чтобы перейти на страницу действий по улучшению.

### <a name="solutions-that-affect-your-score"></a>Решения, влияющие на оценку

В этом разделе выделены решения, содержащие действия по улучшению, которые могут положительно повлиять на оценку, а также количество невыполнений действий по улучшению в этих решениях. Выберите **"Просмотреть все решения",** чтобы посетить страницу решений.

### <a name="compliance-score-breakdown"></a>Разбивка по оценке соответствия требованиям

В этом разделе вы можете получить более подробное представление об оценке двумя разными способами:

- **Categories**: shows the percentage of your overall score within data protection categories, such as "protect information" or "manage devices".
- **Оценки**: показывает процент вашего прогресса в управлении оценками для определенных стандартов соответствия и защиты данных, нормативов или законов, таких как GDPR или NIST 800-53.

### <a name="filtering-your-dashboard-view"></a>Фильтрация представления панели мониторинга

Вы можете отфильтровать представление панели мониторинга, чтобы просмотреть только элементы, связанные с определенными нормами и стандартами, решениями, типом действий, группами оценки или категориями защиты данных. Фильтрация представления таким образом также отфильтрует оценку на информационной панели, показывая, сколько баллов вы получили из общего числа возможных баллов на основе критериев фильтра.

Чтобы применить фильтры:

1. Выберите **"Фильтр"** в верхней правой части панели мониторинга.
2. Выберите критерии фильтра на **области** "Фильтры" и выберите **"Применить".**

После применения фильтра показатель будет скорректирован в режиме реального времени. Процент соответствия требованиям и сведения о разбивке, а также действия по улучшению и решения, теперь относятся только к данным, на которые распространяется ваш критерий фильтрации. Если вы выходите из диспетчера соответствия требованиям, отфильтрованный вид остается при входе.

Чтобы удалить фильтры:

- В **заголовке "Примененные фильтры"** над показателем соответствия выберите **X** рядом с отдельным фильтром, который нужно удалить; или
- Выберите **"Фильтр"** в верхней правой части  панели мониторинга, а затем на панели "Фильтры" выберите **"Очистить фильтры".**

## <a name="improvement-actions-page"></a>Страница действий по улучшению

[Действия по улучшению](compliance-manager-improvement-actions.md) управляются вашей организацией. Работа с действиями по улучшению помогает централизовать действия по обеспечению соответствия требованиям и согласовывать их с нормами и стандартами защиты данных. Каждое действие по улучшению предоставляет подробные рекомендации по реализации и ссылку на соответствующее решение. Для выполнения реализации и тестирования пользователям в организации могут быть назначены действия по улучшению. Вы также можете хранить документацию, заметки и записывать обновления состояния в рамках действия по улучшению.

### <a name="view-your-improvement-actions"></a>Просмотр действий по улучшению

На информационной панели диспетчера соответствия требованиям показаны **основные действия по улучшению.** Чтобы просмотреть все действия по улучшению, выберите вкладку "Действия по улучшению" на информационной панели, на которой вы будете перенаться на страницу действий по улучшению. Вы также можете выбрать "Просмотреть все действия по улучшению" в списке действий по улучшению клавиш на информационной панели, чтобы вернуться на страницу действий по улучшению.

На странице действий по улучшению показаны все действия по улучшению, управляемые вашей организацией. Действия, управляемые корпорацией Майкрософт, можно просмотреть в рамках каждой оценки (узнайте больше о [действиях Майкрософт).](compliance-manager-assessments.md#microsoft-actions-tab)

Если на странице действий по улучшению имеется длинный список действий, может оказаться полезным отфильтровать представление. Выберите **фильтр** в правом верхнем углу списка действий. После того **как появится** flyout pane фильтров, выберите критерии на основе нормативов и стандартов, решения и группы. Вы также можете настроить представление, выбрав **группу** в правом верхнем углу. В выпадаемом меню выберите для просмотра группу, решение, категорию, тип действия или состояние.

В представлении по умолчанию для этой страницы не показываются действия по улучшению с состоянием теста **"Пройдено".** Чтобы просмотреть действия, которые прошли  тестирование, проверьте флажок "Пройдено" во flyout области фильтров. Только действия с состоянием теста **"Передано" учитываются** в вашем счете. Некоторые действия могут показывать метку **ожидающих обновлений.** Узнайте больше об [обновлениях для действий по улучшению.](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)

На странице действий по улучшению показаны следующие точки данных для каждого действия по улучшению:

- **Достигнутое количество** баллов: количество баллов, достигнутое из общего числа доступных при выполнении действия
- **Нормативы**: нормативы или стандарты, относящиеся к действию
- **Группа**: группа, которой назначено действие
- **Решения**: решение, в которое можно перейти для выполнения действия
- **Оценки**: оценки, содержащие действие
- **Категории:** связанная категория защиты данных (например, защита информации, управление устройствами и т. д.)
- **Состояние тестирования:**
    - **Нет** — обновление состояния не записано
    - **Не оценено** — тестирование не начато
    - **Пройдено** — реализация успешно протестирована
    - **Сбой с низким риском** — сбой тестирования, низкий риск
    - **Сбой среднего риска** — сбой тестирования, средний риск
    - **Сбой с высоким уровнем риска** — сбой тестирования, высокий риск
    - **Вне области** действия — действие не находится в области оценки и не влияет на оценку
    - **Для обнаружения —** для ручного тестирования указывает, что действие реализовано, но не проверено; для автоматического тестирования, означает, что действие ожидает результата автоматизации
    - **Не удалось обнаружить —** не удалось определить автоматическое состояние
    - **Частично протестированная** автоматическая опробовка, которая не является частью баллов

**Дополнительные.** [Узнайте, как назначать и выполнять работу над действиями по улучшению.](compliance-manager-improvement-actions.md)

## <a name="solutions-page"></a>Страница "Решения"

На странице решений показана доля заработаных и потенциальных точек, уорганизованная по решению. Просмотр оставшихся точек и действий по улучшению в этом представлении помогает понять, какие решения требуют более непосредственного внимания.

Найдите страницу решений, выбрав вкладку **"Решения"** на информационной панели диспетчера соответствия требованиям. Вы также можете выбрать **"Просмотреть все** решения под **решениями",** влияющие на оценку в правом верхнем разделе панели мониторинга.

### <a name="filtering-your-solutions-view"></a>Фильтрация представления решений

Чтобы отфильтровать свое представление решений:

1. Выберите **фильтр** в левом верхнем углу списка оценок.
2. На **flyout** области фильтров поместите проверку рядом с нужными критериями (стандарты и нормативы, решение, тип действия, группа диспетчера соответствия требованиям, категория).
3. Выберите **кнопку "Применить".** The filter pane will close and you'll see your filtered view.

Вы также можете изменить свое представление, чтобы просмотреть оценки по группе, продукту или нормативу, выбрав тип группировки в выпадаемом меню группы над списком оценок. 

### <a name="taking-action-from-the-solution-page"></a>Действие со страницы решения

На странице решений отображаются решения организации, подключенные к действиям по улучшению. В таблице перечислены вклад каждого решения в общую оценку, точки, которые можно получить и получить в рамках этого решения, а также оставшееся количество действий по улучшению, сгруппных в этом решении, которые могут увеличить ваш показатель.

На этом экране можно действовать двумя способами:

1. В строке вашего решения в  столбце "Оставшиеся действия" выберите номер гиперссылки. Вы увидите отфильтрованное представление экрана действий по улучшению, на котором показаны непроверенные действия по улучшению для этого решения.

2. В строке вашего решения в столбце **"Открыть решение"** выберите **"Открыть".** Вы увидите решение или расположение в Центрах безопасности и соответствия требованиям Microsoft 365 и Office 365, где вы можете принять рекомендуемое действие.

## <a name="assessments-page"></a>Страница оценок

На странице оценок перечислены все [оценки,](compliance-manager-assessments.md) которые вы настроили для организации. Ваш анализатор показателей соответствия требованиям определяется всеми отслеживаемой оценкой. По мере добавления дополнительных оценок на странице "Действия по улучшению" будут перечислены дополнительные действия по улучшению, а количество оценок соответствия требованиям увеличится.

На странице оценок ключевых сведений о каждой оценке:

- **Оценка**: имя оценки
- **Состояние:**
    - **Завершено** — все элементы управления имеют состояние "передано" или по крайней мере один передается, а остальные находятся вне области действия
    - **Неполный** — по крайней мере один из них имеет состояние "сбой"
    - **Нет** — все элементы управления не были протестированы
    - **В процессе** — действия по улучшению имеют любое другое состояние, включая "в процессе", "частичный кредит" или "незащищенный"
- **Ход оценки**: процент работы, сделанной к завершению, измеряемой количеством успешно протестных средств контроля
- **Действия по улучшению:** количество завершенных действий для реализации элементов управления
- **Действия Майкрософт:** количество завершенных действий для реализации элементов управления Майкрософт
- **Group**: name of the group the assessment belongs to
- **Продукт**: связанная служба Microsoft 365
- **Regulation**: the regulatory standard, policy, or law that applies to the assessment

### <a name="filtering-your-assessments-view"></a>Фильтрация представления оценок

Фильтрация представления оценок:

1. Выберите **фильтр** в левом верхнем углу списка оценок.
2. В области **"Фильтры"** проверьте нужные критерии.
3. Выберите **кнопку "Применить".** The filter pane will close and you will see your filtered view.

Вы также можете изменить свое представление, чтобы просмотреть оценки по группе, продукту или нормативу, выбрав тип группировки в выпадаемом меню группы над списком оценок. 

### <a name="default-assessment"></a>Оценка по умолчанию

По умолчанию на странице [](compliance-manager-assessments.md#data-protection-baseline-default-assessment) оценок вы увидите базовую оценку защиты данных.
