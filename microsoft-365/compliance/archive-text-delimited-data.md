# <a name="set-up-a-connector-to-archive-text-delimited-data"></a>Настройка соединитетеля для архива данных, делимитированные с текстом

Используйте соединители Veritas в центре Microsoft 365 для импорта и архива текстовых делимитируемых данных в почтовые ящики пользователей в Microsoft 365 организации. Veritas предоставляет [](https://globanet.com/text-delimited) текстовый соединители, настроенный для захвата элементов из стороннего источника данных (на регулярной основе) и импорта этих элементов в Microsoft 365. Соединитатель преобразует содержимое из источника данных, делимитируемого текстом, в формат сообщения электронной почты, а затем импортирует эти элементы в почтовый ящик пользователя в Microsoft 365.

После хранения текстовых данных в почтовых ящиках пользователей можно применить Microsoft 365, такие как хранение судебного разбирательства, электронные сведения, политики хранения и метки хранения. Использование соединители данных с использованием текстовых данных для импорта и архива данных в Microsoft 365 может помочь вашей организации соблюдать государственные и нормативные политики.

## <a name="overview-of-archiving-the-text-delimited-data"></a>Обзор архива данных с текстовыми данными

В следующем обзоре объясняется процесс использования соединители для архивации текстовых сведений источника в Microsoft 365.

![Архивировка рабочего процесса для данных, делимитированные с текстом](../media/TextDelimitedConnectorWorkflow.png)

1. Ваша организация работает с текстовым источником для настройки и настройки сайта с текстом.

2. Каждые 24 часа сообщения чата из текстового источника копируется на сайт Veritas Merge1. Соединителет также преобразует содержимое в формат сообщений электронной почты.

3. Создаваемая в центре соответствия требованиям Microsoft 365 подключается к сайту Veritas Merge1 каждый день и передает сообщения в безопасное расположение служба хранилища Azure в облаке Майкрософт.

4. Соединитатель импортирует преобразованные элементы сообщений в почтовые ящики определенных пользователей, используя значение свойства *Email* автоматического сопоставления пользователей, как описано в шаге 3. В почтовых ящиках пользователя создается новый поддвое в папке "Входящие" с именем **Text-Delimited,** и элементы сообщений импортированы в эту папку. Соединитатель определяет, в какой почтовый ящик импортировать элементы, используя значение свойства *Email.* Каждое сообщение содержит это свойство, которое заполняется адресом электронной почты каждого участника.

## <a name="before-you-begin"></a>Прежде чем начать

- Создайте учетную запись Veritas Merge1 для соединители Microsoft. Чтобы создать эту учетную запись, обратитесь в службу поддержки [клиентов Veritas.](https://globanet.com/ms-connectors-contact) При создании соединитетеля в шаге 1 вы вопишитесь в эту учетную запись.

- Пользователь, создававший соединители с текстовыми данными в шаге 1 (и завершивший его в шаге 3), должен быть назначен роли экспорта импорта почтовых ящиков в Exchange Online. Эта роль необходима для добавления соединители на **странице** соединители данных в центре Microsoft 365 соответствия требованиям. По умолчанию эта роль не назначена группе ролей в Exchange Online. Вы можете добавить роль экспорта импорта почтовых ящиков в группу ролей управления организацией в Exchange Online. Или вы можете создать группу ролей, назначить роль экспортировать импорт почтовых ящиков, а затем добавить соответствующих пользователей в качестве участников. Дополнительные сведения см. в разделах [Создание](/Exchange/permissions-exo/role-groups#create-role-groups) групп ролей или [изменение](/Exchange/permissions-exo/role-groups#modify-role-groups) групп ролей в статье "Управление группами ролей в Exchange Online".

## <a name="step-1-set-up-the-text-delimited-connector"></a>Шаг 1. Настройка соединитетеля с делимитированный текстом

На первом этапе необходимо  получить доступ к странице Соединители данных в центре Microsoft 365 и создать соединители для текстовых делимитных данных.

1. Перейдите [https://compliance.microsoft.com](https://compliance.microsoft.com/) к и нажмите **соединители данных**  >  **Text-Delimited**.

2. На странице описания продукта с **текстом** нажмите кнопку **Добавить соединители**.

3. На странице **Условия службы нажмите** кнопку **Принять**.

4. Введите уникальное имя, идентифицируемое соединитетелем, а затем нажмите **кнопку Далее**.

5. Впишитесь в свою учетную запись Merge1, чтобы настроить соединители.

## <a name="step-2-configure-the-text-delimited-connector-on-the-veritas-merge1-site"></a>Шаг 2. Настройка соединителя с делимитируемым текстом на сайте Veritas Merge1

Второй шаг — настройка соединитетеля с текстовыми данными на сайте Merge1. Сведения о настройке текстового соединителя на сайте Veritas Merge1 см. в руководстве по пользователю [Merge1 сторонних соединитетелей.](https://docs.ms.merge1.globanetportal.com/Merge1%20Third-Party%20Connectors%20text-delimited%20User%20Guide%20.pdf)

После нажатия **кнопки Сохранить &**  finish отображается страница сопоставления пользователей в мастере соединители в центре Microsoft 365 соответствия требованиям.

## <a name="step-3-map-users-and-complete-the-connector-setup"></a>Шаг 3. Карта пользователей и завершение установки соединитетеля

Чтобы составить карту пользователей и завершить установку соединитетеля в центре Microsoft 365 соответствия требованиям, выполните следующие действия:

1. На странице **Сопоставление внешних** пользователей для Microsoft 365 пользователей включаем автоматическое сопоставление пользователей. Исходные элементы text-delimited включают свойство *Email,* содержаное адреса электронной почты для пользователей в организации. Если соединитатель может связать этот адрес с Microsoft 365 пользователем, элементы импортируется в почтовый ящик этого пользователя.

2. Нажмите **кнопку Далее,** просмотрите параметры и перейдите на страницу соединители данных, чтобы просмотреть ход процесса импорта для нового соединитетеля. 

## <a name="step-4-monitor-the-text-delimited-connector"></a>Шаг 4. Мониторинг соединитетеля с текстовым разбором

После создания соединитетеля Text-Delimited можно просмотреть состояние соединители в центре Microsoft 365 соответствия требованиям.

1. Перейдите [https://compliance.microsoft.com](https://compliance.microsoft.com) и щелкните **соединители данных** в левом nav.

2. Щелкните **вкладку Соединители,** а затем выберите **соединители Text-Delimited,** чтобы отобразить страницу вылетов. На этой странице содержатся свойства и сведения о соединители.

3. В **состоянии Соединитель с исходным кодом** щелкните ссылку **Журнал** загрузки, чтобы открыть (или сохранить) журнал состояния соединитетеля. В этом журнале содержатся сведения о данных, импортируемых в облако Майкрософт.

## <a name="known-issues"></a>Известные проблемы

- В настоящее время мы не поддерживаем импорт вложений или элементов размером более 10 МБ. Поддержка более крупных элементов будет доступна позднее.
