# <a name="reports-in-the-security--compliance-center"></a>Отчеты в Центре безопасности и соответствия требованиям

Вы можете быстро **получить** доступ к отчетам аудита для организаций SharePoint Online и Exchange Online на странице Просмотр отчетов в Центре & безопасности. Вы также можете получить доступ к отчетам о входе пользователя Azure Active Directory (AD), отчетам о действиях пользователей и журналу аудита Azure AD со страницы **Отчеты просмотров.** Это потому, что платная подписка на Microsoft 365 включает бесплатную подписку на Microsoft Azure. При первой попытке получить доступ к этим отчетам Azure необходимо выполнить одновековую регистрацию. 
  
> [!TIP]
> Дополнительные отчеты об активности в организации см. в обзоре Отчеты об активности в центре администрирования [Microsoft 365.](../admin/activity-reports/activity-reports.md) 
  
 **Перед началом работы**
  
Для просмотра отчетов в Центре обеспечения безопасности & необходимы следующие разрешения.
  
- Для просмотра отчетов в центре администрирования Exchange (EAC) вам должна быть назначена роль читателя безопасности в центре & соответствия требованиям. По умолчанию эта роль назначена группам ролей управления организацией и читателям безопасности в EAC.
    
- Для просмотра отчетов о DLP в Центре & соответствия требованиям View-Only DLP необходимо View-Only роли управления & DLP. По умолчанию эта роль назначена группам ролей администратора соответствия требованиям, управления организацией, администратора безопасности и групп чтения & безопасности.

- Кроме того, для просмотра отчетов о DLP в EAC необходимо View-Only роли получателей в EAC. По умолчанию эта роль назначена группам по управлению соответствием требованиям, управлению организацией и View-Only организации в EAC.
  
 **Чтобы открыть страницу отчетов View в Центре & безопасности:**
  
1. Перейдите по ссылке [https://protection.office.com/#/viewreports](https://protection.office.com/#/viewreports).
    
2. Впишитесь с помощью учетных данных учетной записи пользователя в организации.
    
На странице **Просмотр отчетов** можно просмотреть следующие типы отчетов: 
  
- [Отчеты аудита](#auditing-reports)
- [Отчет о проверке надзора](#supervisory-review-report)
- [Отчеты службы защиты от потери данных](#data-loss-prevention-reports)
    
## <a name="auditing-reports"></a>Отчеты аудита

В следующей таблице описываются  отчеты в разделе Аудит на странице **Отчеты** просмотров в Центре & соответствия требованиям. 
  
|**Отчет**|**Описание**|
|:-----|:-----|
|**Отчет журнала аудита** <br/> |Вы можете искать журнал аудита для действий пользователя и администратора в организации. Отчет содержит записи пользователей и администраторов в Exchange Online, SharePoint Online, OneDrive для бизнеса и Azure Active Directory, который является службой каталогов Для Office 365. Дополнительные сведения см. [в журнале поиска аудита в Office 365.](search-the-audit-log-in-security-and-compliance.md)  <br/> |
|**Отчеты Azure AD** <br/> |Чтобы найти необычные или подозрительные действия для регистрации в организации, можно использовать отчеты о входе и действиях в Microsoft Azure. Вы также можете просматривать события в журнале аудита Azure AD. Чтобы просмотреть отчеты в Azure, нажмите **кнопку Просмотр отчетов Azure AD.** Дополнительные сведения см. в статьях: <br/><br/>[Используйте бесплатную подписку Azure Active Directory в Office 365.](use-your-free-azure-ad-subscription-in-office-365.md) <br/> [Просмотр отчетов о доступе и использовании.](/azure/active-directory/reports-monitoring/overview-reports)  <br/> |
|**Отчеты об аудите Exchange** <br/> | Функции аудита в Microsoft 365 можно использовать для отслеживания изменений, внесенных администраторами организации в конфигурацию Exchange Online. Изменения, внесенные в организацию Exchange Online администратором центра обработки данных Майкрософт или делегированным администратором, также регистрируются. Для Exchange Online журнал аудита администратора включен по умолчанию, поэтому для его включения не нужно ничего делать. Exchange Online также предоставляет журнал аудита почтовых ящиков, чтобы отслеживать доступ к почтовым ящикам кем-то, кроме владельца почтовых ящиков. Чтобы отслеживать доступ к почтовым ящикам со стороны пользователей, не являющихся их владельцами, необходимо включить функцию ведения журнала аудита для соответствующих почтовых ящиков.  <br/>  Вы можете запускать отчеты аудита для просмотра записей журналов аудита почтовых ящиков и действий администраторов. Кроме того, вы можете экспортировать журналы аудита почтовых ящиков и действий администраторов, которые в течение 24 часов будут отправлены вам в виде XML-файла, вложенного в письмо. <br/><br/>Дополнительные сведения об экспорте журналов аудита см. в указанных ниже статьях.  <br/><br/> [Экспорт журналов аудита почтовых ящиков](/exchange/security-and-compliance/exchange-auditing-reports/export-mailbox-audit-logs) <br/> [Просмотр и экспорт журнала аудита администрирования центра обработки данных](/exchange/security-and-compliance/exchange-auditing-reports/view-external-admin-audit-log) <br/> [Поиск изменений группы ролей или журналов аудита действий администраторов](/exchange/security-and-compliance/exchange-auditing-reports/search-role-group-changes) <br/>   [Отчеты о аудите Exchange](/exchange/security-and-compliance/exchange-auditing-reports/exchange-auditing-reports).  <br/> |
   
## <a name="supervisory-review-report"></a>Отчет о проверке надзора

В отчете о проверке надзора можно просмотреть состояние всех политик надзорного надзора в вашей организации. Дополнительные сведения см. в [обзоре Configure supervisory review policies for your organization.](./communication-compliance-configure.md)
  
## <a name="data-loss-prevention-reports"></a>Отчеты защиты от потери данных

Отчеты по предотвращению потери данных содержат сведения о политиках и правилах DLP, применяемых к контенту, содержащих конфиденциальные данные в организации. Кроме того, вы можете настроить отчет так, чтобы в нем отображались сведения о действиях политики защиты от потери данных, основанные на заданных вами политике и правилах защиты от потери данных. Дополнительные сведения см. в [отчете по предотвращению потери данных.](view-the-dlp-reports.md)
