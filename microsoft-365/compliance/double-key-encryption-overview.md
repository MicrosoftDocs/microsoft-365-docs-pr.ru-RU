# <a name="double-key-encryption-frequently-asked-questions"></a>Часто задаваемые вопросы о шифровании двойных ключей

У вас есть вопрос о том, как работает двойное шифрование ключей? Проверьте ответ здесь.

## <a name="what-is-double-key-encryption-for-microsoft-365-dke"></a>Что такое двойное Ключическое шифрование для Microsoft 365 (ДКЕ)?

Двойное ключевое шифрование для Microsoft 365 позволяет пользователям защищать конфиденциальные данные для удовлетворения специализированных требований. Он помогает клиентам поддерживать полный контроль над ключами шифрования. Он использует два ключа для защиты данных; один ключ в элементе управления и второй ключ, который безопасно хранится в Microsoft Azure. Для просмотра данных, защищенных с помощью двойного шифрования, требуется доступ к обоим ключам. Так как корпорация Майкрософт может получить доступ только к одному из этих разделов, защищенные данные останутся недоступными для Майкрософт, что гарантирует полный контроль за конфиденциальностью и безопасностью данных.  

Вы можете разместить службу шифрования двойных ключей, используемую для запроса ключа, в выбранном расположении (локальный сервер управления ключами или в облаке). Обслуживание службы выполняется так же, как и для любого другого приложения. Двойное шифрование позволяет управлять доступом к службе шифрования двойных ключей. Вы можете хранить конфиденциальные данные в локальной среде или переместить их в облако. Вы можете быть уверены в предотвращении доступа к сторонним программам, так как вы полностью контролируете свой ключ. Шифрование с двойным ключом позволяет хранить данные и ключ в одном расположении.

ДКЕ помогает обеспечить соблюдение нормативных требований в отношении нескольких нормативных требований и стандартов, таких как общие требования к защите данных (GDPR), функции переноса и учета страхования здоровья (HIPAA), Акт Грэмма-Лича-Блили ACT (GLBA), закона по локализации данных в России — Федеральный закон. 242 ФЗ, Федеральный закон о конфиденциальности Австралии 1988 и заявление о конфиденциальности для Новой Зеландии 1993.

## <a name="can-i-use-double-key-encryption-with-microsoft-office-built-in-sensitivity-labeling"></a>Можно ли использовать шифрование с двойным ключом с помощью встроенной метки чувствительности Microsoft Office?

Для защиты документов с помощью шифрования с двойным ключом необходимо использовать унифицированный клиент меток Azure Information Protection. В настоящее время нельзя использовать встроенные метки чувствительности Microsoft Office. 

## <a name="what-microsoft-365-apps-can-i-use-with-dke"></a>Какие приложения Microsoft 365 можно использовать с ДКЕ?

Для защиты документов с помощью настольных версий Word, Excel и PowerPoint в Windows можно использовать метки ДКЕ. Убедитесь, что вы используете *. 12711 или более поздней версии (настольные версии Word, PowerPoint и Excel) для Windows.

## <a name="how-is-double-key-encryption-different-from-the-existing-hold-your-own-key-hyok-solution"></a>Как двойное шифрование ключей отличается от существующего, в котором хранится ваше собственное ключевое решение (ХЙОК)?

Шифрование с двойным ключом шифрует данные с помощью двух ключей. Ваш ключ шифрования находится в вашем элементе управления, а второй ключ хранится в Microsoft Azure, что позволяет переместить зашифрованные данные в облако. ХЙОК защищает контент только с помощью одного ключа, а ключ всегда используется локально.  

## <a name="can-double-key-encrypted-documents-be-shared-externally"></a>Можно ли использовать двойные зашифрованные зашифрованные документы для внешнего доступа?

Вы можете совместно использовать двойные зашифрованные документы с пользователями на отдельном клиенте, если эти пользователи:

- У вас есть необходимые разрешения на доступ к ключу в службе шифрования двойных ключей.

- У вас есть необходимые разрешения на доступ к ключу в Microsoft Azure.

## <a name="what-happens-to-documents-that-are-protected-with-hyok"></a>Что происходит с документами, защищенными с помощью ХЙОК?

Развертывание двойного ключа шифрования не повлияет на существующую установку ХЙОК. Однако мы рекомендуем использовать шифрование с двойным ключом параллельно с ХЙОК.

## <a name="can-i-run-double-key-encryption-in-my-non-microsoft-air-gapped-environment"></a>Можно ли выполнять шифрование с двойным ключом в среде с отличной от Microsoft Air — гаппед?

ДКЕ не поддерживает эти среды, так как службе требуется доступ к Microsoft Azure.

## <a name="where-can-i-store-double-key-encrypted-documents"></a>Где можно хранить двойные зашифрованные документы?

Вы можете хранить зашифрованные двойным ключом документы в локальной среде или в облаке. В облаке вы можете переместить зашифрованный контент в SharePoint Online и OneDrive для бизнеса. Так как корпорация Майкрософт не имеет доступа к вашему закрытому ключу, зашифрованные данные остаются непрозрачными для Майкрософт. Это также означает, что вы не можете просматривать зашифрованные документы в Интернете в Office Web Apps.

## <a name="what-regions-and-languages-is-double-key-encryption-available-in-is-double-key-encryption-available-worldwide"></a>В каких регионах и языках доступны двойные ключи шифрования? Шифрование с двойным ключом доступно в разных странах мира?

Метки ДКЕ локализованы на те же языки, что и другие метки чувствительности в Microsoft Information Protection. Двойное ключевое шифрование доступно по всему миру.

## <a name="can-i-convert-a-non-dke-label-to-a-dke-label"></a>Можно ли преобразовать метку, не являющуюся ДКЕ, в метку ДКЕ?

Нет. После создания метки невозможно добавить ДКЕ к ней. Вместо этого необходимо выбрать параметр **использовать шифрование двойных ключей** и указать URL-адрес службы шифрования двойных ключей при создании метки.

## <a name="how-do-i-roll-my-dke-keys"></a>Как выполнить откат моих ключей ДКЕ?

В разделе [Operations for the Azure Information Protection](https://docs.microsoft.com/azure/information-protection/operations-customer-managed-tenant-key)— для получения инструкций по выбору (также называемой поворотом или назначением клавиши смены ключей), который хранится в Azure.

Для получения сведений о создании нового ключа для службы ДКЕ, ознакомьтесь со статьей [Параметры клиента и ключ](double-key-encryption.md#tenant-and-key-settings) .

При создании ключа необходимо указать имя и идентификатор GUID. Затем, если вы поверните ключ, сохраните старую запись с именем и GUID, но добавьте новую запись с тем же именем, но с другим GUID. Новый ключ задается как активный, чтобы API открытого ключа начал возвратить его для нового шифрования. Оба ключа доступны для расшифровки, чтобы можно было расшифровывать новый контент и старое содержимое.
