# <a name="create-and-track-servicenow-tickets-in-the-microsoft-365-security-center"></a>Создание и отслеживание билетов для ServiceNow в центре безопасности Майкрософт 365

>[!CAUTION]
>**Завершен период предварительной версии для соединителя ServiceNow**<br>
>Эта возможность больше не будет доступна в конце ноября 2020. Благодарим вас за отзыв и продолжение поддержки, пока мы определим дальнейшие действия.

[Центр безопасности Microsoft 365](overview-security-center.md) был усовершенствован благодаря возможности создавать и отслеживать билеты в ServiceNow. [Дополнительные сведения о ServiceNow](https://www.servicenow.com/)

В центре безопасности Администраторы безопасности могут отправить действие улучшения [оценки безопасности Майкрософт](microsoft-secure-score.md) непосредственно в ServiceNow и создать билет. Могут быть созданы как управление инцидентами, так и билеты управления изменениями. Отслеживайте билеты на домашней странице центра безопасности и ServiceNow.

## <a name="connect-microsoft-365-security-center-to-servicenow"></a>Подключение центра безопасности Microsoft 365 к ServiceNow

Перейдите на домашнюю страницу центра безопасности Microsoft 365, чтобы просмотреть карточку подключения ServiceNow.

![Использование ServiceNow](../../media/do-you-use-servicenow-250.png)

Выберите "подключиться к ServiceNow", чтобы перейти на страницу установки ServiceNow. Следуйте инструкциям для авторизации приложения Microsoft 365 Connector.

> [!NOTE]
> Прежде чем выполнять авторизацию подключения между центром безопасности Microsoft 365 и ServiceNow, убедитесь, что вы используете имя входа и пароль пользователя интеграции, которые вы создали в шагах установки. Не используйте свои личные учетные данные.

После выполнения инструкций и авторизации подключения просмотрите состояние подключения на странице Подключение к центру безопасности Microsoft 365 и в интерфейсе приложения ServiceNow Microsoft 365 с помощью соединителя. Теперь все готово для начала создания задач.

### <a name="troubleshooting"></a>Устранение неполадок

Сведения об общих ошибках, которые могут возникать в процессе подключения, а также о том, как их можно устранить в [разделе Устранение неполадок](tickets.md#troubleshooting).

## <a name="create-a-task-and-share-it-to-servicenow"></a>Создание задачи и общий доступ к ней в ServiceNow

После настройки интеграции Создайте задачи ServiceNow на основе определенных действий по улучшению [оценки безопасности Майкрософт](microsoft-secure-score.md) . Перейдите к любому действию по улучшению оценки безопасности в центре безопасности Microsoft 365 и выберите пункт **общий доступ** . Один из вариантов раскрывающегося списка — ServiceNow.

Создается задача, в которой можно задать приоритет и изменить имя, описание или срок выполнения. После заполнения всех обязательных полей отправьте задачу в ServiceNow.

Эта задача отображается в ServiceNow как запрос на изменение безопасности и конфигурации Microsoft 365.

## <a name="track-tickets"></a>Отслеживание билетов

После создания билетов управления изменениями и управления инцидентами, они отображаются на карточках на домашней странице центра безопасности Microsoft 365. Из этих карточек вы можете создать билет, просмотреть все билеты или управлять конфигурацией ServiceNow.

![Билеты управления изменениями ServiceNow](../../media/change-management-375.png)  ![Билеты управления инцидентами ServiceNow](../../media/incident-management-375.png)

Для повторной подготовки или управления интеграцией ServiceNow в центре безопасности Майкрософт 365 выберите **Управление конфигурацией servicenow** на любой из карточек. После этого удалите текущие подключения ServiceNow и настройте имена состояний билетов.

С билетами ServiceNow, которые отображаются в центре безопасности Майкрософт 365, ваши задачи находятся в том месте, где они могут отслеживаться и обрабатываться вместе с другими элементами панели мониторинга безопасности.
