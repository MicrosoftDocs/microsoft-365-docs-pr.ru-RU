# <a name="proactively-hunt-for-threats-with-advanced-hunting-in-microsoft-365-defender"></a>Профилактическое профилактическое слежение за угрозами с помощью расширенного поиска в защитнике Microsoft 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]


**Область применения:**
- Microsoft 365 Defender

> Хотите работать с защитником Microsoft 365? Вы можете [оценить его в лабораторной среде](https://aka.ms/mtp-trial-lab) или [запустить пилотный проект в рабочей](https://aka.ms/m365d-pilotplaybook)среде.
>

Расширенное выслеживание— это средство для выслеживания угроз на основе запросов, которое позволяет изучать необработанные данные за период до 30 дней. Вы можете проанализировать события в сети, чтобы определить, какие индикаторы и сущности угроз. Гибкий доступ к данным обеспечивает неограниченный поиск как известных, так и потенциальных угроз.
<p></p>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4Bp7O]

Вы можете использовать одни и те же запросы на поиск угроз для создания пользовательских правил обнаружения. Эти правила выполняются автоматически для проверки и реагирования на обнаруженные нежелательные действия, неправильно настроенные машины и другие результаты.

Эта возможность аналогична [расширенному поиску в защитнике Майкрософт для конечной точки](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-hunting-overview). В центре безопасности Microsoft 365 эта возможность поддерживает запросы, которые проверяют более широкий набор данных из:

- Microsoft Defender для конечной точки
- Microsoft Defender для Office 365
- Microsoft Cloud App Security
- Microsoft Defender для удостоверений

Чтобы использовать расширенный поиск, [включите Защитник Microsoft 365](mtp-enable.md).

## <a name="get-access"></a>Получение доступа
Чтобы использовать расширенные функции обнаружения или другие возможности [защитника Microsoft 365](microsoft-threat-protection.md) , вам потребуется соответствующая роль в Azure Active Directory. Кроме того, доступ к данным конечной точки определяется параметрами управления доступом на основе ролей (RBAC) в защитнике Майкрософт для конечной точки. [Сведения об управлении доступом к защитнику Microsoft 365](mtp-permissions.md)

## <a name="data-freshness-and-update-frequency"></a>Обновление данных и частота обновления
Расширенные данные о поиске можно классифицировать по двум разным типам, каждая из которых отличается друг от друга.

- **Данные о событиях или действиях**— заполняет таблицы, связанные с оповещениями, событиями безопасности, системными событиями и регулярной оценкой. Расширенный поиск получает эти данные почти сразу же после того, как датчики, собирающие их, успешно передают их в соответствующие облачные службы. Например, вы можете запрашивать данные о событиях из работоспособных датчиков на рабочих станциях или контроллерах доменов почти сразу же после их появления в защитнике Майкрософт для конечной точки и защитника Майкрософт для удостоверения.
- **Данные сущности**— заполняет таблицы сведениями о пользователях и устройствах. Эти данные поступают из относительно статических источников данных и динамических источников, таких как записи Active Directory и журналы событий. Для обеспечения актуальности данных таблицы обновляются с любой новой информацией каждые 15 минут, добавляя строки, которые могут быть заполнены не полностью. Каждые 24 часа данные консолидируются для вставки записи, содержащей самый последний набор данных о каждой сущности.

## <a name="time-zone"></a>Часовой пояс
Сведения о времени в расширенном поиске находятся в часовом поясе UTC.
