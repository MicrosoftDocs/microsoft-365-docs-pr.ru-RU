# <a name="device-monitoring-and-reporting-in-the-microsoft-365-security-center"></a>Мониторинг устройств и отчетность в Центре безопасности Microsoft 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]


Сохраняйте безопасность, последние и сохраняйте потенциальные угрозы на своих устройствах в Центре безопасности Microsoft 365.

## <a name="view-device-alerts"></a>Просмотр оповещений устройств

Получать оповещения о нарушении безопасности и других угрозах на устройствах из Microsoft Defender для конечной точки (доступно с лицензией E5). Центр безопасности Microsoft 365 эффективно отслеживает эти оповещения на высоком уровне с помощью предпочитаемого рабочего процесса.

### <a name="monitor-high-impact-alerts"></a>Отслеживание оповещений с высоким уровнем воздействия

Каждому оповещению Microsoft Defender для конечной точки соответствует степень серьезности (высокая, средняя, низкая или информационная). Это указывает на потенциальное влияние на сеть, если оставить ее безотвратно.  

Используйте **карточку серьезности** оповещений устройства, чтобы сосредоточиться на более строгих оповещениях, которые могут требовать немедленного реагирования. С этой карточки можно просмотреть дополнительные сведения на портале Центра безопасности Microsoft Defender.

![Карточка серьезности оповещений устройства](../../media/device-alerts-severity.png)

### <a name="understand-sources-of-alerts"></a>Понимание источников оповещений

Microsoft Defender для конечной точки использует данные из широкого диапазона датчиков безопасности и источников аналитики для создания оповещений. Например, он может использовать сведения об обнаружении из антивирусной программы "Microsoft Defender" и сторонняя защита от вирусов. Он также может использовать собственную настраиваемую аналитику угроз, предоставляемую через API веб-службы.

Карточка **источников обнаружения оповещений** устройства показывает распределение оповещений по источнику. Отслеживание действий, связанных с определенными источниками, особенно настраиваемые источники. Вы также можете использовать карточку, чтобы сосредоточиться на оповещениях от датчиков, которые не настроены для автоматического блокировки вредоносных действий или компонентов.

![Карточка источников обнаружения оповещений устройства](../../media/device-alert-detection-sources.png)

С этой карточки можно просмотреть дополнительные сведения на портале Центра безопасности Microsoft Defender.

### <a name="understand-the-types-of-threats-that-trigger-alerts"></a>Типы угроз, которые вызывают оповещения

Microsoft Defender for Endpoint sorts each alert into a category representing a certain stage in the attack chain or type of threat component. Например, обнаруженное действие с угрозой может быть классифицироваться как "другое перемещение", чтобы указать на то, что была предпринята попытка связаться с другими устройствами в сети. Действие, скорее всего, произошло после того, как злоумышленники получили начальный foothold. При обнаружении компонент угрозы может широко классифицироваться как вредоносная программа или, в частности, как определенный тип угрозы. К ним относятся программы-вымогателя, кража учетных данных и другие типы вредоносных или нежелательных программ.

На **карточке категорий угроз устройства** показано распределение оповещений по этим категориям. Используйте эту информацию для определения действий с угрозами, например попыток кражи учетных данных, которые обычно имеют большее влияние, чем попытки социальной инженерии. Вы также можете отслеживать потенциально деструктивные угрозы, такие как программы-вымогателя.

![Карточка категорий угроз устройств](../../media/device-threat-categories.png)

### <a name="monitor-active-alerts"></a>Отслеживание активных оповещений

Карточка **состояния оповещений** устройства указывает количество оповещений, которые не были разрешены и могут требовать внимания. С этой карточки можно просмотреть дополнительные сведения на портале Центра безопасности Microsoft Defender.

![Карточка состояния оповещений устройства](../../media/device-alert-status.png)

### <a name="monitor-classification-of-resolved-alerts"></a>Мониторинг классификации разрешенных оповещений

При разрешении оповещений в Microsoft Defender для конечной точки сотрудники службы безопасности могут указать, проверено ли оповещение как:

* Настоящее оповещение, которое определяет фактические действия по нарушению безопасности или компоненты угроз
* Ложное оповещение, которое неправильно обнаружило обычную активность

На **карточке классификации** оповещений устройства показано, были ли разрешенные оповещения классифицированы как истинные или ложные. С этой карточки можно просмотреть дополнительные сведения на портале Центра безопасности Microsoft Defender.

Примечание. В некоторых случаях сведения о классификации недоступны для определенных оповещений.

![Карточка классификации оповещений устройства](../../media/device-alert-classification.png)

### <a name="monitor-determination-of-resolved-alerts"></a>Отслеживание определения разрешенных оповещений

Помимо классификации того, является ли оповещение истинным или ложным при разрешении, сотрудники службы безопасности могут предоставить определение. Определение указывает тип обычной или вредоносной активности, обнаруженной при проверке оповещения.

На **карточке определения оповещений** устройства показано определение, предоставленное для каждого оповещения.

* **APT**: advanced persistent threat, indicating that the detected activity or threat component is part of a sophisticated breach designed to gain a foothold in the affected network  
* **Вредоносная** программа: вредоносный файл или код
* **Персонал безопасности:** обычная активность, выполняемая сотрудниками службы безопасности
* **Тестирование безопасности**: действия или компоненты, предназначенные для имитации фактических угроз и предназначенные для запуска датчиков безопасности и создания оповещений
* **Нежелательное программное обеспечение:** приложения и другое программное обеспечение, которые не считаются вредоносными, но в противном случае нарушают политику или допустимые стандарты использования
* **Others**: any other determination that doesn't fall under the provided types

На этой карточке можно просмотреть дополнительные сведения в Центре безопасности Microsoft Defender.

![Карточка определения оповещений устройства](../../media/device-alert-determination.png)

### <a name="understand-which-devices-are-at-risk"></a>Понять, какие устройства находятся под угрозой

**Защита устройств** показывает уровень риска для устройств. Уровень риска зависит от таких факторов, как тип и серьезность оповещений на устройстве.

![Карточка защиты устройств](../../media/device-protection.png)

## <a name="monitor-and-report-status-of-intune-managed-devices"></a>Мониторинг и отчет о состоянии устройств, управляемых Intune

В следующих отчетах содержатся данные с устройств, зарегистрированных в Intune. Данные с устройств, не включенных в нее, не включаются. Только глобальные администраторы могут просматривать эти карточки.

Данные зарегистрированных устройств в Intune включают:

* Соответствие устройства требованиям
* Устройства с активным вредоносным ПО
* Типы вредоносных программ на устройствах
* Вредоносные программы на устройствах
* Устройства с обнаружением вредоносных программ
* Пользователи с обнаружением вредоносных программ

### <a name="monitor-device-compliance"></a>Отслеживание соответствия устройств требованиям

**Соответствие устройств** требованиям показывает, сколько устройств, зарегистрированных в Intune, соответствуют политикам конфигурации.

![Карточка соответствия устройств требованиям](../../media/device-compliance.png)

### <a name="discover-devices-with-malware-detections"></a>Обнаружение устройств с обнаружением вредоносных программ

**Обнаружение вредоносных программ на** устройствах предоставляет количество зарегистрированных в Intune устройств с вредоносными программами, которые не были полностью разрешены. Отсутствие разрешения может быть из-за ожидающих действий, перезагрузки, полной проверки, действий пользователя вручную или из-за того, что действие по исправлению не было успешно завершено.

![Карта обнаружения вредоносных программ на устройстве](../../media/device-malware-detections.png)

### <a name="understand-the-types-of-malware-detected"></a>Понять, какие типы вредоносных программ обнаружены

**Типы вредоносных программ на устройствах** показывают различные виды вредоносных программ, обнаруженных на устройствах, зарегистрированных в Intune. Вы можете изучить каждый тип в Центре безопасности Microsoft 365.

![Типы вредоносных программ на карточках устройств](../../media/types-of-malware-on-devices.png)

**Вредоносные программы на устройствах** предоставляют список определенных вредоносных программ, обнаруженных на ваших устройствах.

![Карта вредоносного ПО на устройствах](../../media/malware-on-devices.png)

**Устройства с обнаружением вредоносных программ** показывают, на каких устройствах обнаружено больше всего вредоносных программ. В Центре безопасности Microsoft 365 вы можете выяснить, активна ли вредоносная программа, кто использует устройство, и состояние управления в Intune.

## <a name="monitor-and-manage-attack-surface-reduction-rule-deployment-and-detections"></a>Мониторинг и управление развертыванием и обнаружением правил уменьшения поверхности атаки

[Правила уменьшения уязвимости (ASR)](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/attack-surface-reduction) помогают предотвратить заражение устройств действиями и приложениями, которые обычно используются вредоносными программами, которые ищут эксплойт. Эти правила определяют, когда и как запускаются исполняемые файлы. Например, вы можете предотвратить запуск загруженного исполняемого файла в JavaScript или VBScript, блокировать вызовы Win32 API из макросов Office или блокировать процессы, которые выполняются с USB-накопителей.

![Карта уменьшения поверхности атаки](../../media/attack-surface-reduction-rules.png)

На карточке **Правила сокращения направлений атак** приведены общие сведения о развертывании правил на устройствах.

В верхнем столбце на этой карточке показано общее количество устройств, которые находятся в следующих режимах развертывания:

* **Режим блокировки:** устройства с по крайней мере одним правилом, настроенным для блокировки обнаруженной активности
* **Режим аудита:** устройства без правил, блокируют обнаруженную активность, но хотя бы одно правило настроено для аудита обнаруженной активности  
* **Off**: devices with all ASR rules turned off

В нижней части этой карточки приведены параметры всех устройств, отсортированные по правилам. На каждой панели указывается количество устройств, для которых настроено блокировка, обнаружение аудита или отключение правила полностью.

### <a name="view-asr-detections"></a>Просмотр обнаружений ASR

Чтобы просмотреть подробные сведения об обнаружении  правил ASR в сети, выберите "Просмотр обнаружений на карточке правил уменьшения поверхности **атаки".** Откроется **вкладка "Обнаружения"** на странице подробного отчета.

![Вкладка "Обнаружения"](../../media/detections-tab.png)

На диаграмме в верхней части страницы показаны обнаружения с течением времени стека, которые были либо заблокированы, либо были аудитом. В таблице внизу перечислены последние обнаруженные действия. Чтобы понять, в чем заключаются обнаруженные действия, используйте следующие сведения в таблице:

* **Обнаруженный файл**: файл, как правило, сценарий или документ, содержимое которого вызвало подозрительное действие атаки
* **Rule**: name describing the attack activities the rule is designed to catch. Ознакомьтесь с существующими правилами ASR
* **Source app**: the application that loaded or executed content triggering the suspected attack activity. Это может быть законное приложение, например веб-браузер, приложение Office или системное средство, например PowerShell
* **Publisher**: поставщик, который выпустил исходный приложение

### <a name="review-device-asr-rule-settings"></a>Просмотр параметров правил ASR устройства

На странице **отчета о правилах уменьшения** поверхности атаки перейдите на вкладку "Конфигурация", чтобы просмотреть параметры правил для отдельных устройств.  Выберите устройство, чтобы получить подробные сведения о том, находится ли каждое правило в режиме блокировки, режиме аудита или полностью отключено.

![Вкладка "Конфигурация"](../../media/configuration-tab.png)

Microsoft Intune предоставляет функции управления для правил ASR. Если вы хотите обновить параметры, выберите  "Начало работы" в области "Настройка устройств" на вкладке, чтобы открыть управление устройствами в Intune. 

### <a name="exclude-files-from-asr-rules"></a>Исключение файлов из правил ASR

Центр безопасности Microsoft 365 собирает [](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/enable-attack-surface-reduction#exclude-files-and-folders-from-asr-rules) имена файлов, которые можно исключить из обнаружения с помощью правил уменьшения поверхности атаки. За исключением файлов можно уменьшить количество обнаружений ложных срабатывавай и более уверенно развертывать правила уменьшения направлений атаки в режиме блокировки.

Исключениями можно управлять в Microsoft Intune, но Центр безопасности Microsoft 365 предоставляет средство анализа, помогая понять файлы. Чтобы начать сбор файлов для исключения, перейдите на вкладку **"Добавление** исключений" на странице отчета о правилах **уменьшения** поверхности атаки.

>[!NOTE]  
>Средство анализирует обнаружение всех правил уменьшения поверхности атаки, но только некоторые правила [поддерживают исключения.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/troubleshoot-asr)

![Вкладка "Добавление исключений"](../../media/add-exclusions-tab.png)

В таблице перечислены все имена файлов, обнаруженные правилами уменьшения поверхности атаки. Вы можете выбрать файлы, чтобы просмотреть влияние их исключения:

* Сколько меньше обнаружений
* Сколько устройств сообщает об обнаружении

Чтобы получить список выбранных файлов с полными путями для исключения, выберите **"Получить пути исключения".**

Журналы для правила ASR Блокируют кражу учетных данных из подсистемы локального органа безопасности **Windows (lsass.exe)** и захватывают исходный код **lsass.exe.** Это обычный системный файл, который захватывается как обнаруженный файл. В результате в созданный список путей исключения будет включен этот файл. Чтобы исключить файл, который инициирует это правило, **lsass.exe,** используйте путь к приложению-источнику вместо обнаруженного файла.

Чтобы найти исходные приложения, [](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/advanced-hunting) запустите следующий запрос на расширенный поиск для этого правила (определенного по коду правила 9e6c4e1f-7d60-472f-ba1a-a39ef669e4b2):

```kusto
DeviceEvents
| where Timestamp > ago(7d)
| where ActionType startswith "Asr"
| where AdditionalFields contains "9e6c4e1f-7d60-472f-ba1a-a39ef669e4b2"
| project InitiatingProcessFolderPath, InitiatingProcessFileName
```

#### <a name="check-files-for-exclusion"></a>Проверка файлов на исключение

Перед исключением файла из ASR рекомендуется проверить файл, чтобы определить, действительно ли он не является вредоносным.

Чтобы просмотреть файл, используйте страницу сведений о [файле](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/investigate-files) в Центре безопасности Microsoft Defender. На странице приводится информация о преобладание и коэффициент обнаружения вирусов VirusTotal. Вы также можете использовать страницу для отправки файла для глубокого анализа.

Чтобы найти обнаруженный файл в Центре безопасности Microsoft Defender, найдите все обнаружения ASR с помощью следующего запроса на расширенный поиск:

```kusto
MiscEvents
| where EventTime > ago(7d)
| where ActionType startswith "Asr"
| project FolderPath, FileName, SHA1, InitiatingProcessFolderPath, InitiatingProcessFileName, InitiatingProcessSHA1
```

Используйте **SHA1** или **InitiatingProcessSHA1** в результатах для поиска файла с помощью универсальной панели поиска в Центре безопасности Microsoft Defender.
