# <a name="recommended-microsoft-cloud-app-security-policies-for-saas-apps"></a>Рекомендуемые политики безопасности облачных приложений Майкрософт для приложений SaaS
Microsoft Cloud App Security использует политики условного доступа Azure AD для обеспечения мониторинга и управления в режиме реального времени с помощью приложений SaaS, таких как блокировка скачивания, загрузки, копирование и вклейка и печать. Эта функция добавляет безопасность сеансам, которые несут в себе риски, например, когда корпоративные ресурсы доступны с неугомонные устройства или гостевых пользователей. 

Microsoft Cloud App Security также интегрируется с Microsoft Information Protection, обеспечивая проверку контента в режиме реального времени для поиска конфиденциальных данных на основе типов конфиденциальной информации и меток конфиденциальности, а также для принятия соответствующих мер. 

В этом руководстве содержатся рекомендации по этим сценариям:
- Приведение приложений SaaS в управление ИТ-ресурсами
- Настройка защиты для определенных приложений SaaS
- Настройка предотвращения потери данных (DLP), чтобы соответствовать правилам защиты данных

## <a name="bring-saas-apps-into-it-management"></a>Приведение приложений SaaS в управление ИТ-ресурсами

Первым шагом к использованию microsoft Cloud App Security для управления приложениями SaaS является обнаружение этих приложений и добавление их в клиент Azure AD. Если вам нужна помощь в обнаружении, см. [в справке Откройте и управляйте приложениями SaaS в сети.](https://docs.microsoft.com/cloud-app-security/tutorial-shadow-it) После обнаружения приложений добавьте их в клиент [Azure AD.](https://docs.microsoft.com/azure/active-directory/manage-apps/add-application-portal)  

Вы можете начать управлять этими данными, делая следующее:
1. Сначала в Azure AD создайте новую политику условного доступа и настройте ее на "Управление приложениями условного доступа". Это перенаправляет запрос в cloud App Security. Вы можете создать одну политику и добавить все приложения SaaS в эту политику.
1. Далее в приложении Cloud App Security создайте политики сеансов. Создайте по одной политике для каждого необходимого управления. 

Разрешения для приложений SaaS обычно основаны на бизнес-потребности в доступе к приложению. Эти разрешения могут быть очень динамичными. Использование политик безопасности облачных приложений обеспечивает защиту данных приложений независимо от того, назначены ли пользователи группе Azure AD, связанной с базовой, чувствительной или строго регулируемой защитой.

Чтобы защитить данные из коллекции приложений SaaS, на следующей схеме показано, как использовать необходимую политику условного доступа Azure AD, а также предлагаемые политики, которые можно создать в cloud App Security. В этом примере политики, созданные в области безопасности облачных приложений, применяются к всем приложениям SaaS, которые вы управляете. Они предназначены для применения соответствующих элементов управления в зависимости от того, управляются ли устройствами, а также меток чувствительности, которые уже применяются к файлам.

## <a name="tune-protection-for-specific-saas-apps"></a>Настройка защиты для определенных приложений SaaS
Может потребоваться применить дополнительные средства мониторинга и управления к определенным приложениям SaaS в вашей среде. Безопасность облачных приложений позволяет выполнить это. Например, если приложение типа Box активно используется в среде, имеет смысл применить дополнительные элементы управления. Или, если ваш юридический или финансовый отдел использует определенное приложение SaaS для конфиденциальных бизнес-данных, вы можете нацелить дополнительную защиту на эти приложения. 

Например, вы можете защитить среду Box с помощью таких типов встроенных шаблонов политики обнаружения аномалий:
- Активность с анонимных IP-адресов
- Активность из редкой страны
- Активность с подозрительных IP-адресов
- Неосуществимое перемещение
- Действия, выполняемые прекращенным пользователем (требуется AAD в качестве IdP)
- Обнаружение вредоносных программ
- Несколько неудачных попыток входа
- Активность вымогателей
- Рискованное приложение Oauth
- Необычные действия для обмена файлами

Вот примеры. Регулярно добавляются дополнительные шаблоны политики. Примеры применения дополнительной защиты к определенным приложениям см. в примере [Protecting connected apps.](https://docs.microsoft.com/cloud-app-security/protect-connected-apps) 

[Как cloud App Security помогает защитить](https://docs.microsoft.com/cloud-app-security/protect-box) среду Box, демонстрирует типы элементов управления, которые могут помочь защитить бизнес-данные в Box и других приложениях с конфиденциальными данными.


## <a name="configure-data-loss-prevention-dlp-to-help-comply-with-data-protection-regulations"></a>Настройка предотвращения потери данных (DLP), чтобы соответствовать правилам защиты данных

Безопасность облачных приложений может быть ценным средством настройки защиты для нормативных актов соответствия требованиям. В этом случае вы создаете определенные политики, чтобы искать конкретные данные, к которых применяется регулирование, и настраивать каждую политику для принятия соответствующих действий. 

В следующей иллюстрации и таблице приводится несколько примеров политик, которые можно настроить, чтобы соответствовать Общему правилу защиты данных (GDPR). В этих примерах политики искать конкретные данные. В зависимости от чувствительности данных каждая политика настроена для принятия соответствующих действий.
