# <a name="configure-your-event-hub"></a>Настройка центра событий

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**Область применения:**
- [Microsoft 365 Defender](https://go.microsoft.com/fwlink/?linkid=2118804)

Узнайте, как настроить центр событий, чтобы он может гнать события из Microsoft 365 Defender.


## <a name="setup-the-required-resource-provider-in-the-event-hub-subscription"></a>Настройка необходимого поставщика ресурсов в подписке Event Hub


1. Войдите на [портал Azure](https://portal.azure.com).
1. Выберите **подписки \> { Выберите ***подписку,*** \>** в центре событий будет развернут } Поставщик ресурсов .
1. Убедитесь, что **поставщик microsoft.Аналитика** зарегистрирован. В противном случае зарегистрируйте его.

![Изображение поставщиков ресурсов в Microsoft Azure](../../media/f893db7a7b1f7aa520e8b9257cc72562.png)

## <a name="setup-azure-active-directory-app-registration"></a>Настройка Azure Active Directory регистрации приложений


>! [ПРИМЕЧАНИЕ] Необходимо установить роль администратора или Azure Active Directory (AAD), чтобы разрешить не администраторам регистрировать приложения. Кроме того, для назначения главной роли службы должна быть роль администратора доступа к пользователю или владельца.  
>Дополнительные сведения см. в статью Создание приложения Azure AD & на портале — платформа удостоверений Майкрософт [ \| Microsoft Docs.](/azure/active-directory/develop/howto-create-service-principal-portal)

1. Создайте новую регистрацию (которая по своей сути создает главную службу) в Azure Active Directory **\> регистрации Приложения \> Новая регистрация.**

1. Заполните форму только именем (URI перенаправления не требуется).

    ![Изображение регистрации приложения](../../media/336bc84e6be23900c43232b4ef0c253c.png)

    ![Изображение сведений об обзоре](../../media/06ac04c4ff713c2065cec2ef2f99a294.png)

1. Создайте секрет, щелкнув по **сертификатам & \> секреты Новый секрет клиента:**

    ![Изображение сертификатов и секретов](../../media/d2ef88d3d2310d2c60c294b569cdf02e.png)

>[!WARNING]
>Вы не сможете получить доступ к секрету **клиента снова, поэтому убедитесь, что сохранить его**.

## <a name="setup-event-hub-namespace"></a>Пространство имен центра событий установки


1. Создание пространства имен концентратора событий:

    Перейдите к концентраторам событий **\> Добавить** и выбрать уровень цен, единиц пропускной способности и Авто-Надув (требует стандартного ценообразования и под функции), соответствующие нагрузке, которая ожидается.  
    Дополнительные сведения см. в [руберсе Pricing - Event Hubs \| Microsoft Azure](https://azure.microsoft.com/en-us/pricing/details/event-hubs/)

    >[!NOTE]
    > Можно использовать существующий концентратор событий, но пропускная способность и масштабирование задаваются на уровне пространства имен, поэтому рекомендуется разместить узел событий в пространстве имен.

   ![Изображение пространства имен event Hub](../../media/ebc4ca37c342ad1da75c4aee4018e51a.png)

1. Вам также потребуется ИД ресурса этого пространства имен центра событий. Перейдите на страницу Свойства пространства имен Azure Event \> Hubs. Скопируйте текст в разделе Resource ID и зафиксируйте его для использования в разделе Конфигурация M365 ниже. 

    ![Изображение свойств](../../media/759498162a4e93cbf17c4130d704d164.png)

1. После создания пространства имен центра событий необходимо добавить главу службы регистрации приложений в качестве ридера, приемник данных Azure Event Hubs и пользователя, который будет войти в Microsoft 365 Defender в качестве участника (это также можно сделать на уровне Resource Group или Подписка).

    Это делается в центре событий **Namespace \> Access Control (IAM) \> Add** and verify under **Role assignments:**

    ![Изображение управления доступом](../../media/9c9c29137b90d5858920202d87680d16.png)

## <a name="setup-event-hub"></a>Центр событий установки


**Вариант 1:**

Вы можете создать центр событий в  пространстве имен, и все типы событий (таблицы), выбранные для экспорта, будут записаны в этом **центре** событий.

**Вариант 2:**

Вместо того чтобы экспортировать все типы событий (таблицы) в один центр событий, вы можете экспортировать каждую таблицу в другой центр событий в пространстве имен центра событий (по одному концентратору событий на тип событий).  

В этом варианте Microsoft 365 Defender для вас будут создаваться концентраторы событий.  
>[!NOTE]
> Если вы используете пространство имен центра  событий, которое не входит в кластер event Hub, вы сможете экспортировать до 10 типов событий (Таблицы) для экспорта в каждом экспортируемом Параметры, которое вы определяете, из-за ограничения Azure в 10 концентраторов событий в пространстве имен узлов событий.

Например:

![Image of example Event Hub](../../media/005c1f6c10c34420d387f594987f9ffe.png)

Если вы выберете этот параметр, вы можете перейти к разделу [Configure Microsoft 365 Defender для отправки таблиц электронной](#configure-microsoft-365-defender-to-send-email-tables) почты.

Создайте центр событий в пространстве имен, выбрав **концентраторы \> событий и концентратор событий.**

Количество разделов позволяет увеличить пропускную способность с помощью параллелизма, поэтому рекомендуется увеличить это число в зависимости от ожидаемой нагрузки.  
Рекомендуется использовать значения хранения и захвата сообщений по умолчанию 1 и off.

![Изображение создания центра событий](../../media/1db04b8ec02a6298d7cc70419ac6e6a9.png)

Для этого центра событий (а не пространства имен) необходимо настроить политику общего доступа с помощью отправки, прослушивания утверждений. Щелкните политики общего доступа к центру событий **\> + \> Добавить,** а затем дать ему имя политики (не используется в других местах) и проверить **Отправка** и **прослушивание**.

![Изображение политик общего доступа](../../media/1867d13f46dc6a0f4cdae6cf00df24db.png)

## <a name="configure-microsoft-365-defender-to-send-email-tables"></a>Настройка Microsoft 365 Defender для отправки таблиц электронной почты


### <a name="setup-microsoft-365-defender-send-email-tables-to-splunk-via-event-hub"></a>Настройка Microsoft 365 Defender таблицы электронной почты в Splunk через центр событий


1. Войти в Microsoft 365 Defender с <https://security.microsoft.com> учетной записью, которая соответствует всем следующим требованиям роли:

    - Роль участника на уровне ресурсов пространства *имен центра* событий или выше для центра событий, в который будет экспортироваться. Без этого вы получите ошибку экспорта при попытке сохранить параметры.

    - Роль глобального администратора или администратора безопасности в клиенте, связанной с Microsoft 365 Defender и Azure.

    ![Изображение портала безопасности](../../media/55d5b1c21dd58692fb12a6c1c35bd4fa.png)

1. Нажмите **кнопку Экспорт необработанных \> данных +Add**.

    Теперь вы будете использовать данные, записанные выше.

    **Имя.** Это локальное и должно быть все, что работает в вашей среде.

    **Переадранить события в центр событий.** Выберите этот почтовый ящик.

    **ИД ресурса Event-Hub.** Это ИД ресурса пространства имен event Hub, который был записан выше при настройке центра событий.

    **Имя Event-Hub.** Если вы создали центр событий в пространстве имен центра событий, вклейте имя Центра событий, записанное выше.

    Если вы хотите Microsoft 365 Defender создать концентраторы событий для типов событий (Таблицы), оставьте это поле пустым.

    **Типы событий.** Выберите таблицы advanced Hunting, которые необходимо переадружить в центр событий, а затем в настраиваемые приложения. Таблицы оповещений из Microsoft 365 Defender, таблицы Устройств — из Microsoft Defender для конечной точки (EDR), а таблицы электронной почты — из Microsoft Defender для Office 365. События электронной почты записывают все транзакции электронной почты. Url-адрес (SafeLinks), Вложения (Сейф вложения) и события после доставки (ZAP) также записываются и могут быть соединены с событиями электронной почты в поле NetworkMessageId.

    ![Изображение параметров API потоковой передачи](../../media/3b2ad64b6ef0f88cf0175f8d57ef8b97.png)

1. Убедитесь, что нажмите **кнопку Отправить**.

### <a name="verify-that-the-events-are-being-exported-to-the-event-hub"></a>Убедитесь, что события экспортируются в центр событий


Вы можете убедиться в том, что события отправляются в центр событий с помощью базового запроса advanced Hunting. Выберите **запрос \> "Охота расширенный \> запрос на охоту"** и введите следующий запрос:

```
EmailEvents
|joinkind=fullouterEmailAttachmentInfoonNetworkMessageId
|joinkind=fullouterEmailUrlInfoonNetworkMessageId
|joinkind=fullouterEmailPostDeliveryEventsonNetworkMessageId
|whereTimestamp\>ago(1h)
|count
```

Это покажет, сколько электронных писем было получено за последний час, соединив их во всех остальных таблицах. Он также покажет, видят ли вы события, которые можно экспортировать в центр событий. Если в этом подсчете показано 0, то вы не увидите никаких данных, выехав в центр событий.

![Изображение продвинутой охоты](../../media/c305e57dc6f72fa9eb035943f244738e.png)

После проверки экспорта данных можно просмотреть центр событий, чтобы убедиться, что сообщения входящие. Это может занять до одного часа. 
 
1. В Azure перейдите в **концентраторы событий Нажмите кнопку Концентраторы событий пространства имен \> \> \> Щелкните центр событий.**  
1. В **обзоре** прокрутите вниз и в графе Сообщения вы увидите входящие сообщения. Если вы не видите никаких результатов, не будет сообщений для пользовательского приложения, чтобы гнать.

    ![Изображение вкладки обзор с сообщениями](../../media/e88060e315d76e74269a3fc866df047f.png)
