## <a name="microsoft-365-defender-pillars"></a>Microsoft 365 Defender столбы
Microsoft 365 Defender состоит из четырех столпов. Хотя один из столбов уже может обеспечить безопасность вашей сетевой организации, включение четырех столбов Microsoft 365 Defender обеспечит вашей организации наибольшее значение.

![Решение of_Microsoft 365 Defender для пользователей, Microsoft Defender for Identity, для конечных точек Microsoft Defender для конечной точки, для облачных приложений, Microsoft Cloud App Security и для данных, Microsoft Defender для Office 365](../../media/mtp/m365pillars.png)

В этом разделе вы сможете настроить:

- Microsoft Defender для Office 365
- Microsoft Defender для удостоверений
- Microsoft Cloud App Security
- Microsoft Defender для конечной точки

## <a name="configure-microsoft-defender-for-office-365"></a>Настройка Microsoft Defender для Office 365

> [!NOTE]
> Пропустить этот шаг, если вы уже включили Defender для Office 365.

Существует модуль PowerShell, называемый Office 365 рекомендуемый анализатор конфигурации *(ORCA),* который помогает определить некоторые из этих параметров. При запуске в качестве администратора в клиенте get-ORCAReport поможет сгенерировать оценку параметров защиты от нежелательной почты, защиты от фишинга и других параметров гигиены сообщений. Этот модуль можно скачать из https://www.powershellgallery.com/packages/ORCA/ .

1. Перейдите [к Office 365 безопасности & центра управления](https://protection.office.com/homepage)  >    >  **угрозами.**

   ![Страница of_Office 365 & центра управления угрозами](../../media/mtp-eval-32.png)

2. Нажмите **кнопку "Анти-фишинг",** **выберите Создать** и заполнить имя и описание политики. Нажмите **Далее**.

   ![Страница of_Office 365 & центра по борьбе с фишингом, на которой можно назвать политику](../../media/mtp-eval-33.png)

   > [!NOTE]
   > Изменение политики advanced anti-phishing в Microsoft Defender для Office 365. Измените **расширенный порог фишинга** на **2 - Aggressive.**

3. Нажмите **кнопку Добавить** выпадаемое меню условия и выберите домен (s) в качестве домена получателя. Нажмите **Далее**.

   ![Страница of_Office 365 & центра защиты от фишинга, на которой можно добавить условие для его приложения](../../media/mtp-eval-34.png)

4. Просмотрите параметры. Нажмите **кнопку Создать эту политику,** чтобы подтвердить.

   ![Страница of_Office 365 & центра по борьбе с фишингом, на которой можно просмотреть параметры и нажать кнопку создать эту политику](../../media/mtp-eval-35.png)

5. Выберите **Сейф вложения** и выберите параметр Включить ATP для **SharePoint, OneDrive и Microsoft Teams.**

   ![Страница of_Office 365 & безопасности, на которой можно включить atP для SharePoint, OneDrive и Microsoft Teams](../../media/mtp-eval-36.png)

6. Щелкните значок +, чтобы создать новую политику безопасного вложения, применить ее в качестве домена получателя к доменам. Щелкните **Сохранить**.

   ![Страница of_Office 365 & безопасности, на которой можно создать новую политику безопасного вложения](../../media/mtp-eval-37.png)

7. Далее выберите Сейф **ссылки,** а затем нажмите значок карандаша, чтобы изменить политику по умолчанию.

8. Убедитесь, что не **отслеживайте,** когда пользователи щелкают параметр безопасных ссылок, а остальные параметры выбраны. Сведения [см. Сейф ссылки.](/microsoft-365/security/office-365-security/recommended-settings-for-eop-and-office365) Щелкните **Сохранить**.

   ![Страница of_Office 365 & центра соответствия требованиям, которая показывает, что параметр Не отслеживайте, когда пользователи щелкают безопасно, не выбран](../../media/mtp-eval-38.png)

9. Далее выберите **политику борьбы с вредоносными** программами, выберите значение по умолчанию и выберите значок карандаша.

10. Щелкните **Параметры** выберите **Да и используйте текст** уведомления по умолчанию, чтобы включить ответ обнаружения **вредоносных программ.** Включив фильтр **типов общих вложений.** Щелкните **Сохранить**.

    ![Страница of_Office 365 & центра соответствия требованиям, которая показывает, что ответ на обнаружение вредоносных программ включен с помощью уведомления по умолчанию и включен фильтр типов вложений](../../media/mtp-eval-39.png)

11. Перейдите [к Office 365 безопасности &](https://protection.office.com/homepage)центра поиска журнала аудита соответствия требованиям и  >    >   включив аудит.

    ![Страница of_Office 365 & центра соответствия требованиям, на которой можно включить поиск журнала аудита](../../media/mtp-eval-40.png)

12. Интеграция Microsoft Defender для Office 365 с Microsoft Defender для конечной точки. Перейдите Office 365 безопасности [&](https://protection.office.com/homepage)центр управления угрозами и выберите Microsoft Defender для конечной точки Параметры в правом верхнем углу  >    >   экрана.  В диалоговом окне Защитник для конечной точки включите Подключение **Microsoft Defender для конечной точки.**

    ![Страница of_Office 365 & безопасности, на которой можно включить подключение Microsoft Defender для конечной точки](../../media/mtp-eval-41.png)

## <a name="configure-microsoft-defender-for-identity"></a>Настройка Microsoft Defender для удостоверений

> [!NOTE]
> Пропустить этот шаг, если вы уже включили Microsoft Defender для identity

1. Перейдите [Microsoft 365 центра](https://security.microsoft.com/info) безопасности > **дополнительные** ресурсы Microsoft Defender для  >  **удостоверений.**

   ![Страница of_Microsoft 365 Центр безопасности, на которой можно открыть Microsoft Defender для удостоверений](../../media/mtp-eval-42.png)

2. Нажмите **кнопку Создать,** чтобы запустить мастер Microsoft Defender для удостоверений.

   ![Страница мастера of_Microsoft Defender for Identity, на которой необходимо нажать кнопку Создать](../../media/mtp-eval-43.png)

3. Выберите Предоставление имени пользователя и пароля для подключения **к лесу Active Directory.**

   ![Приветственный of_Microsoft Защитник удостоверений](../../media/mtp-eval-44.png)

4. Введите учетные данные Active Directory на месте. Это может быть любая учетная запись пользователя, которая имеет доступ к active Directory.

   ![Страница of_Microsoft defender for Identity Directory, на которой необходимо поместить учетные данные](../../media/mtp-eval-45.png)

5. Далее выберите **настройка датчика загрузки** и передача файла на контроллер домена.

   ![Страница of_Microsoft Defender для удостоверений, на которой можно выбрать установку датчика загрузки](../../media/mtp-eval-46.png)

6. Выполните установку датчика идентификации Microsoft Defender для идентификации и начните следовать за мастером.

   ![Страница of_Microsoft Defender для удостоверения, на которой необходимо щелкнуть далее, чтобы следовать мастеру датчика идентификации Microsoft Defender для удостоверений](../../media/mtp-eval-47.png)

7. Нажмите **кнопку Далее** в типе развертывания датчика.

   ![Страница of_Microsoft Defender для удостоверения, на которой необходимо щелкнуть кнопку далее, чтобы перейти на следующую страницу](../../media/mtp-eval-48.png)

8. Скопируйте ключ доступа, так как необходимо ввести его далее в мастере.

   ![Страница of_the датчиков, на которой необходимо скопировать ключ доступа, который необходимо ввести на следующей странице мастера установки датчика настройки сенсора Microsoft Defender для удостоверений](../../media/mtp-eval-49.png)

9. Скопируйте ключ доступа в мастер и нажмите кнопку **Установить**.

   ![Страница мастера of_Microsoft защитника для датчика удостоверений, на которой необходимо предоставить ключ доступа, а затем нажать кнопку установки](../../media/mtp-eval-50.png)

10. Поздравляем, вы успешно настроили Microsoft Defender для identity на контроллере домена.

    ![Завершение of_Microsoft защитника для мастера установки датчика удостоверений, на котором необходимо нажать кнопку готовой](../../media/mtp-eval-51.png)

11. В разделе [Защитник Майкрософт для](https://go.microsoft.com/fwlink/?linkid=2040449) параметров удостоверений выберите **Microsoft Defender для конечной точки **, а затем включив переключить. Щелкните **Сохранить**.

    ![Страница of_the защитника Майкрософт для параметров удостоверений, на которой необходимо включить переключить параметры Microsoft Defender для конечной точки](../../media/mtp-eval-52.png)

## <a name="configure-microsoft-cloud-app-security"></a>Настройка Microsoft Cloud App Security

> [!NOTE]
> Пропустить этот шаг, если вы уже включили Microsoft Cloud App Security.

1. Перейдите [Microsoft 365 Центр безопасности](https://security.microsoft.com/info)  >  **Дополнительные ресурсы**  >  **Microsoft Cloud App Security**.

   ![Страница of_Microsoft центра безопасности 365, на которой можно увидеть карту Microsoft Cloud App и нажать кнопку открыть](../../media/mtp-eval-53.png)

2. На информационном запросе для интеграции Microsoft Defender для удостоверений выберите **Включить Microsoft Defender для интеграции данных удостоверений.**

   ![Подсказка of_the для интеграции Microsoft Defender для удостоверений, где необходимо выбрать ссылку Включить microsoft Defender для интеграции данных удостоверений](../../media/mtp-eval-54.png)

   > [!NOTE]
   > Если вы не видите эту подсказку, это может означать, что интеграция данных Microsoft Defender для удостоверений уже включена. Однако, если вы не уверены, обратитесь к ИТ-администратору, чтобы подтвердить это.

3. Перейдите **Параметры,** включите переключить кнопку **Microsoft Defender для** интеграции удостоверений, а затем нажмите **кнопку Сохранить**.

   ![Страница of_the, на которой необходимо включить переключению интеграции Microsoft Defender для удостоверений, а затем нажмите кнопку сохранить](../../media/mtp-eval-55.png)

   > [!NOTE]
   > Для новых экземпляров Microsoft Defender для удостоверений эта интеграция автоматически включена. Подтвердите, что интеграция Microsoft Defender для удостоверений включена, прежде чем приступить к следующему шагу.

4. В настройках обнаружения облака выберите Microsoft Defender для интеграции **конечных** точек, а затем в этом параметре включить интеграцию. Щелкните **Сохранить**.

   ![Изображение of_the для страницы Microsoft Defender для конечной точки, на которой выбирается блок несанкционированная проверка приложений в рамках интеграции Microsoft Defender для конечной точки. Щелкните сохранить.](../../media/mtp-eval-56.png)

5. В параметрах обнаружения облака выберите обогащение **пользователей,** а затем в этом параметре включить интеграцию с Azure Active Directory.

   ![Изображение раздела Обогащение пользователей, в котором выбирается обогащение обнаруженных идентификаторов пользователей с Azure Active Directory имен пользователей](../../media/mtp-eval-57.png)

## <a name="configure-microsoft-defender-for-endpoint"></a>Настройка Microsoft Defender для конечной точки

> [!NOTE]
> Пропустить этот шаг, если вы уже включили Microsoft Defender для конечной точки.

1. Перейдите [Microsoft 365 Центр безопасности](https://security.microsoft.com/info)  >  **Дополнительные ресурсы**  >  **Центр безопасности в Microsoft Defender**. Нажмите кнопку **Open** (Открыть).

   ![Параметр of_Microsoft Центр безопасности Defender на странице Microsoft 365 Центр безопасности](../../media/mtp-eval-58.png)

2. Следуйте мастеру Microsoft Defender для конечных точек. Нажмите **Далее**.

   ![Страница of_the Центр безопасности в Microsoft Defender мастера](../../media/mtp-eval-59.png)

3. Выбор зависит от предпочтительного расположения хранилища данных, политики хранения данных, размера организации и выбора функций предварительного просмотра.

   ![Страница of_the для выбора страны хранения данных, политики хранения и размера организации. Нажмите кнопку далее, когда вы закончили выбор.](../../media/mtp-eval-60.png)

   > [!NOTE]
   > Вы не можете изменить некоторые параметры, например расположение хранилища данных.

   Нажмите **Далее**.

4. Нажмите **кнопку Продолжить,** и она будет поддерживать защитника Microsoft для клиента конечной точки.

   ![Запрос of_the страницы, нажав кнопку продолжить для создания облачного экземпляра](../../media/mtp-eval-61.png)

5. На борту конечных точек с помощью групповых политик, Microsoft Endpoint Manager или запуска локального сценария в Microsoft Defender для конечной точки. Для простоты в этом руководстве используется локальный сценарий.

6. Нажмите **кнопку Скачать** пакет и скопируйте сценарий в конечной точке (ы).

   ![Изображение of_page нажать кнопку Пакет загрузки, чтобы скопировать сценарий включения в конечную точку или конечные точки](../../media/mtp-eval-62.png)

7. На конечной точке запустите сценарий в качестве администратора и выберите Y.

   ![Командная of_the, в которой вы запустите сценарий бортовой работы и выберите Y для работы](../../media/mtp-eval-63.png)

8. Поздравляем, вы вошел в первую конечную точку.

   ![Изображение of_the, где вы получаете подтверждение того, что вы на борту вашей первой конечной точки. Нажмите любой ключ, чтобы продолжить](../../media/mtp-eval-64.png)

9. Скопируйте тест обнаружения в мастере Microsoft Defender для конечных точек.

   ![Image of_the выполнить тестовый шаг обнаружения, на котором необходимо нажать копию, чтобы скопировать сценарий тестирования обнаружения, который необходимо вклеить в командную подсказку](../../media/mtp-eval-65.png)

10. Скопируйте скрипт PowerShell в командную команду и запустите его.

    ![Подсказка of_command, где следует скопировать сценарий PowerShell в командную команду и запустить его](../../media/mtp-eval-66.png)

11. Выберите **Начните использовать Microsoft Defender для конечной точки** из мастера.

    ![Запрос of_the подтверждения от мастера, где необходимо нажать кнопку Начните использовать Microsoft Defender для конечной точки](../../media/mtp-eval-67.png)

12. Посетите [Центр безопасности в Microsoft Defender](https://securitycenter.windows.com/). Перейдите **Параметры** и выберите **расширенные функции.**

    ![Меню of_Microsoft Центр безопасности Defender Параметры, в котором необходимо выбрать расширенные функции](../../media/mtp-eval-68.png)

13. Включи интеграцию с **Microsoft Defender для удостоверений.**

    ![Расширенные of_Microsoft Центра безопасности Защитника, параметр Microsoft Defender для удостоверений, чтобы включить](../../media/mtp-eval-69.png)

14. Включи интеграцию с **Office 365 intelligence**.

    ![Расширенные of_Microsoft центра безопасности Defender, Office 365 параметру Threat Intelligence, который необходимо включить](../../media/mtp-eval-70.png)

15. Включи интеграцию **с Microsoft Cloud App Security.**

    ![Расширенные of_Microsoft центра безопасности Defender, Microsoft Cloud App Security параметра, который необходимо включить](../../media/mtp-eval-71.png)

16. Прокрутите вниз **и нажмите кнопку Сохранить параметры,** чтобы подтвердить новые интеграции.

    ![Кнопка of_Save, которую необходимо нажать](../../media/mtp-eval-72.png)

Дополнительные рекомендации см. в [Microsoft 365 Defender.](m365d-enable.md)
