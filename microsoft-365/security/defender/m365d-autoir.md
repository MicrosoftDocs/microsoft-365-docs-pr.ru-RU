# <a name="automated-investigation-and-response-in-microsoft-365-defender"></a>Автоматическое расследование и ответ в Microsoft 365 Defender

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender.md)]

**Область применения:**
- Microsoft 365 Defender

Если ваша организация использует [Microsoft 365 Defender,](microsoft-365-defender.md)группа операций безопасности получает оповещение Microsoft 365 центра безопасности при обнаружении вредоносных или подозрительных действий или артефактов. Учитывая, казалось бы, бесконечный поток угроз, которые могут поступать, группы безопасности часто сталкиваются с проблемой решения большого объема оповещений. К счастью, Microsoft 365 Defender включает в себя функции автоматического расследования и реагирования (AIR), которые помогут вашей группе операций безопасности эффективнее и эффективнее решать угрозы.

В этой статье представлен обзор air и ссылки на последующие действия и дополнительные ресурсы.

> [!TIP]
> Хотите попробовать Microsoft 365 Defender? Вы можете [оценить его в лабораторной среде](m365d-evaluation.md?ocid=cx-docs-MTPtriallab) или [запустить пилотный проект в производственной среде](m365d-pilot.md?ocid=cx-evalpilot).

## <a name="how-automated-investigation-and-self-healing-works"></a>Как работает автоматизированное исследование и самовосстановления

По мере запуска оповещений о безопасности ваша группа операций безопасности должна инициализации этих оповещений и принять меры для защиты организации. Определение приоритета и анализ оповещений могут отнимать очень много времени, особенно в условиях постоянного появления новых оповещений во время анализа. Службы обеспечения безопасности могут быть перегружены из-за огромного количества угроз, которые им нужно отслеживать и от которых необходимо защищать. Автоматизированные возможности исследования и реагирования, с самовосстановления, в Microsoft 365 Defender может помочь.

Просмотрите следующее видео, чтобы узнать, как работает самовосстановления: <p>

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE4BzwB]

В Microsoft 365 Defender автоматизированные исследования и ответы с помощью функций самовосстановления работают на ваших устройствах, & контенте и удостоверениях.
 
> [!TIP]
> В этой статье описывается, как работает автоматическое расследование и ответы. Чтобы настроить эти возможности, см. в перенастройке возможности автоматического расследования и ответа [в Microsoft 365 Defender.](m365d-configure-auto-investigation-response.md)

## <a name="your-own-virtual-analyst"></a>Собственный виртуальный аналитик

Imagine виртуального аналитика в группе операций по безопасности уровня 1 или 2 уровня. Виртуальный аналитик имитирует идеальные действия, которые должен предпринять отдел обеспечения безопасности для анализа и устранения угроз. Виртуальный аналитик может работать в 24x7 с неограниченной емкостью и взять на себя значительную нагрузку исследований и устранения угроз. Такой виртуальный аналитик может значительно сократить время реагирования, освободив команду операций безопасности для других важных угроз или стратегических проектов. Если этот сценарий звучит как научная фантастика, это не так! Такой виртуальный аналитик является частью пакета Microsoft 365 Defender, и его имя — автоматическое исследование *и ответ.*

Возможности автоматического расследования и реагирования позволяют группе операций безопасности значительно увеличить возможности организации по работе с оповещениями о безопасности и инцидентами. С помощью автоматического расследования и реагирования можно снизить затраты на работу с действиями по расследованию и реагированию и получить максимальное количество данных из пакета защиты от угроз. Автоматизированные возможности расследования и реагирования помогают вашей группе по работе с безопасностью:

1. Определение того, требуется ли угроза действий.
2. Принятие (или рекомендация) любых необходимых действий по исправлению.
3. Определение того, следует ли и какие другие исследования должны происходить.
4. повторяет весь этот процесс с другими оповещениями.

## <a name="the-automated-investigation-process"></a>Процесс автоматизированного анализа

Оповещение создает инцидент, который может начать автоматическое расследование. Автоматизированное расследование приводит к вынесению вердикта по каждому фрагменту доказательств. Вердикты могут быть:
- *Злоумышленная*
- *Подозрительные* 
- *Угрозы не найдены* 

Определены действия по исправлению вредоносных или подозрительных сущностями. Примеры действий по исправлению:

- Отправка файла на карантин
- Остановка процесса
- Изолирование устройства
- Блокировка URL-адреса 
- Другие действия

Дополнительные сведения см. в см. в Microsoft 365 [Действия по исправлению.](m365d-remediation-actions.md)

В зависимости [от](m365d-configure-auto-investigation-response.md) того, как настраиваются возможности автоматического расследования и реагирования для вашей организации, действия по исправлению будут приниматься автоматически или только после утверждения вашей командой операций безопасности. Все действия, ожидающих или завершенных, перечислены в [центре действий.](m365d-action-center.md)

Во время проведения анализа все другие связанные с ним оповещения добавляются в анализ до его завершения. Если затрагиваемая сущность видна в другом месте, автоматизированное расследование расширяет его область, включая его, и процесс расследования повторяется. 

В Microsoft 365 Defender каждое автоматическое расследование коррелирует сигналы в Microsoft Defender for Identity, Microsoft Defender для конечной точки и Microsoft Defender для Office 365, как по сумме в следующей таблице: 

|Объекты |Службы защиты от угроз  |
|:---------|:---------|
|Устройства (также именуемые конечными точками или машинами) |[Защитник для конечной точки](../defender-endpoint/automated-investigations.md) |      
|Локальное поведение пользователей Active Directory, поведение сущности и действия     |[Defender для удостоверений](/azure-advanced-threat-protection/what-is-atp) |      
|Содержимое электронной почты (сообщения электронной почты, которые могут содержать файлы и URL-адреса)     |[Defender для Office 365](../office-365-security/defender-for-office-365.md) |

> [!NOTE]
> Не каждое оповещение вызывает автоматическое расследование, и не каждое расследование приводит к автоматическим действиям по исправлению. Это зависит от настройки автоматического расследования и ответа для вашей организации. См. [в этой версии Настройка возможностей автоматического расследования и ответа.](m365d-configure-auto-investigation-response.md)
