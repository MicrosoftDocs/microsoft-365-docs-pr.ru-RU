# <a name="microsoft-defender-for-endpoint-for-us-government-customers"></a>Microsoft Defender для конечной точки для государственных служб США

**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/p/?linkid=2154037)

Microsoft Defender for Endpoint для клиентов правительства США, построенная в среде правительства США Azure, использует те же базовые технологии, что и Defender for Endpoint в Azure Commercial.

Это предложение доступно для GCC, GCC High и DOD и основано на той же профилактике, обнаружении, расследовании и исправлении, что и коммерческая версия. Тем не менее, существуют некоторые различия в доступности возможностей для этого предложения.

> [!NOTE]
> Если вы являетсяе GCC, используя Defender for Endpoint в рекламе, пожалуйста, обратитесь к страницам публичной документации.

### <a name="service-backend-ip-ranges"></a>Диапазоны IP-диапазона бэкэнда службы

Если сетевые устройства не поддерживают правила на основе DNS, вместо этого используйте IP-диапазоны.

Защитник конечной точки для клиентов правительства США построен в среде правительства США Azure, развернутой в следующих регионах:

- AzureCloud.usgovtexas
- AzureCloud.usgovvirginia

Вы можете найти IP-диапазоны Azure в [диапазонах IP Azure и тегах обслуживания - Облако правительства США](https://www.microsoft.com/download/details.aspx?id=57063).

> [!NOTE]
> В качестве облачного решения диапазоны IP-адресов могут меняться. Рекомендуется перейти на правила, основанные на DNS.
