# <a name="turn-on-block-at-first-sight"></a>Включить блокировку при первом появлении

> Эта статья предназначена для корпоративных администраторов и ИТ-специалистов, которые управляют настройками безопасности для организаций. ## <a name="what-is-block-at-first-sight"></a>Что такое "блок с первого взгляда"?

Блок на первый взгляд — это функция защиты от угроз следующего поколения, которая обнаруживает новые вредоносные программы и блокирует их в течение нескольких секунд. Блок с первого взгляда включен при включении определенных параметров безопасности. В эти параметры входят:

- Защита с облачным доставкой; 
- Указанный пример времени отправки (например, 50 секунд); и 
- Высокий уровень блокировки файлов. 

В большинстве корпоративных организаций параметры, необходимые для обеспечения блокировки на первый взгляд, настраиваются с помощью развертывания антивирусных программ Microsoft Defender. 

## <a name="how-it-works"></a>Как это работает

Когда антивирус Microsoft Defender сталкивается с подозрительным, но незащищенным файлом, он запрашивает наш запас облачной защиты. В облачной папке применяется юристика, машинное обучение и автоматический анализ файла, чтобы определить, являются ли файлы вредоносными или нет угрозой.

Антивирус Microsoft Defender использует несколько технологий обнаружения и предотвращения для обеспечения точной, интеллектуальной и защиты в режиме реального времени.

## <a name="a-few-things-to-know-about-block-at-first-sight"></a>Несколько вещей, которые нужно знать о блоке с первого взгляда

- В Windows 10 версии 1803 или более поздней версии блокировка на первый взгляд может блокировать не переносимые исполняемые файлы (такие как JS, VBS или макрос) и исполняемые файлы.

- Блок на первый взгляд использует только архивирование облачной защиты для исполняемых файлов и не переносимых исполняемых файлов, скачанных из Интернета или которые происходят из зоны Интернета. Значение hash файла .exe проверяется с помощью облачной задней части, чтобы определить, является ли файл ранее незащищенным.

- Если в облачной папке не удается определиться, антивирус Microsoft Defender блокирует файл и загружает копию в облако. Облако выполняет дополнительный анализ для достижения определения, прежде чем оно либо позволяет файлу запускать, либо блокирует его во всех будущих встречах, в зависимости от того, определяет ли он файл как вредоносный или не представляет угрозы.

- Во многих случаях этот процесс позволяет сократить время отклика на новые вредоносные программы с нескольких часов до секунд.

- Можно [указать, как](configure-cloud-block-timeout-period-microsoft-defender-antivirus.md) долго должен быть предотвращен запуск файла, пока облачная служба защиты анализирует файл. При [блокировке](/windows/security/threat-protection//windows-defender-security-center/wdsc-customize-contact-information.md) файла можно настроить сообщение, отображаемую на рабочих столах пользователей. Вы можете изменить имя компании, контактные данные и URL-адрес сообщений.

## <a name="turn-on-block-at-first-sight-with-microsoft-intune"></a>Включив блок с первого взгляда с помощью Microsoft Intune

> [!TIP]
> Microsoft Intune теперь входит в состав Microsoft Endpoint Manager.

1. В центре администрирования microsoft Endpoint Manager [https://endpoint.microsoft.com](https://endpoint.microsoft.com) (), перейдите к профилям   >  **конфигурации устройств.**

2. Выберите или создайте профиль с помощью типа **профилей ограничений** устройства.

3. В **параметрах Конфигурация** для профиля ограничений устройства установите или подтвердить следующие параметры в **антивирусе Microsoft Defender:**

   - **Защита от облачной доставки:** включена
   - **Уровень блокировки файлов:** высокий
   - **Расширение времени для сканирования файлов облаком:** 50
   - **Запрос пользователей перед отправкой** примера: Отправка всех данных без запроса

   ![Config Intune](images/defender/intune-block-at-first-sight.png)

4. Сохраните параметры.

> [!TIP]
> - Настройка уровня блокировки файлов **до уровня High** применяет высокий уровень обнаружения. В маловероятном случае, если блокировка файлов вызывает ложное срабатывающее обнаружение законных файлов, группа операций безопасности может восстановить [карантинные файлы.](./restore-quarantined-files-microsoft-defender-antivirus.md)
> - Дополнительные сведения о настройке ограничений антивирусных устройств Microsoft Defender в Intune см. в дополнительных сведениях о настройке параметров ограничения устройств [в Microsoft Intune.](/intune/device-restrictions-configure)
> - Список ограничений антивирусных устройств Microsoft Defender в Intune см. в списке ограничений устройств для [параметров Windows 10 (и более новых) в Intune.](/intune/device-restrictions-windows-10#microsoft-defender-antivirus)

## <a name="turn-on-block-at-first-sight-with-microsoft-endpoint-manager"></a>Включив блок с первого взгляда с помощью Microsoft Endpoint Manager

> [!TIP]
> Если вы ищете Microsoft Endpoint Configuration Manager, он теперь является частью Microsoft Endpoint Manager.

1. В Microsoft Endpoint Manager [https://endpoint.microsoft.com](https://endpoint.microsoft.com) () перейдите к **антивирусу безопасности**  >  **конечных точек**.

2. Выберите существующую политику или создайте новую политику с помощью типа антивирусного профиля **Microsoft Defender.**

3. Установите или подтвердит следующие параметры конфигурации:

   - **Включаем облачную защиту:** Да.
   - **Уровень защиты с облачной доставкой:** высокий
   - **Расширенное время растянутой разгона Defender** Cloud в секундах : 50

   :::image type="content" source="images/endpointmgr-antivirus-cloudprotection.png" alt-text="Блокировка параметров с первого взгляда в диспетчере конечных точек":::

4. Применить антивирусный профиль Microsoft Defender к группе, например всем **пользователям,** всем устройствам или всем пользователям **и устройствам.**

## <a name="turn-on-block-at-first-sight-with-group-policy"></a>Включив блок с первого взгляда с помощью групповой политики

> [!NOTE]
> Рекомендуется использовать Intune или Microsoft Endpoint Manager для включения блокировки с первого взгляда. 

1. На компьютере управления групповой политикой откройте консоль управления групповой политикой [правой](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731212(v=ws.11))кнопкой мыши объект групповой политики, который необходимо настроить, и выберите **Изменить**. 

2. С помощью **редактора управления групповой политикой** перейдите к **шаблонам** администрирования конфигурации  >    >  **компьютеров Windows Components**  >  **Microsoft Defender Antivirus**  >  **MAPS.** 

3. В разделе MAPS дважды щелкните Настройка функции **"Блок** с первого взгляда" и установите ее **включенной,** а затем выберите **ОК**.

    > [!IMPORTANT]
    > Параметр Always **prompt (0)** снижает состояние защиты устройства. Параметр Никогда **не отправлять (2)** означает, что блок с первого взгляда не будет работать.

4. В разделе MAPS дважды щелкните **Отправить** образцы файлов при необходимости дальнейшего анализа и установите его на **включенную**. В **статье Отправка образцов файлов при** необходимости дополнительного анализа выберите **Отправить** все образцы, а затем выберите **ОК.**

5. Передиплойте объект групповой политики в сети, как обычно.

## <a name="confirm-block-at-first-sight-is-enabled-on-individual-client-devices"></a>Подтверждение включения блока с первого взгляда на отдельных клиентских устройствах

Вы можете подтвердить, что блокировка с первого взгляда включена на отдельных клиентских устройствах с помощью приложения Windows Security. Блок с первого взгляда автоматически включен  до тех  пор, пока включена защита от облачной доставки и автоматическая отправка образцов.

1. Откройте приложение Windows Security.

2. Выберите **защиту &** вирусов, а затем в & параметров защиты от угроз выберите Управление  **настройками.**

   ![Снимок экрана метки & защиты от угроз в приложении Windows Security](images/defender/wdav-protection-settings-wdsc.png)

3. **Подтвердим, что** защита от облачной доставки и **автоматическая отправка** образцов включены.

> [!NOTE]
> - Если необходимые параметры настроены и развернуты с помощью групповой политики, описанные в этом разделе параметры будут серыми и недоступны для использования в отдельных конечных точках. 
> - Изменения, внесенные с помощью объекта групповой политики, сначала должны быть развернуты в отдельных конечных точках, прежде чем параметр будет обновлен в параметрах Windows.

## <a name="turn-off-block-at-first-sight"></a>Отключите блок с первого взгляда

> [!CAUTION]
> Отключение блока на первый взгляд снижает состояние защиты устройства и сети.

Вы можете отключить блокировку с первого взгляда, если вы хотите сохранить необходимые параметры без использования блокировки с первого взгляда. Вы можете временно отключить блокировку с первого взгляда, чтобы узнать, как эта функция влияет на сеть. Однако мы не рекомендуем отключать блок с первого взгляда навсегда.

### <a name="turn-off-block-at-first-sight-with-microsoft-endpoint-manager"></a>Отключите блок с первого взгляда с помощью Microsoft Endpoint Manager

1. Перейдите в Центр администрирования конечной точки Microsoft Manager [https://endpoint.microsoft.com](https://endpoint.microsoft.com) () и войдите.

2. Перейдите к **антивирусу безопасности**  >  **Endpoint,** а затем выберите политику антивируса Microsoft Defender.

3. В **статье Управление** выберите **свойства**.

4. Рядом с **настройками конфигурации** выберите **Изменить**.

5. Измените один или несколько следующих параметров:

   - Установите **включив облачную защиту,** **чтобы не** настроить или не **настроить.**
   - Установите **уровень защиты с облачной доставкой,** чтобы не **настраиваться.**
   - Очистка контрольного окна для расширенного растянутой растянутой минуты Defender **Cloud в секундах.**

6. Просмотр и сохранение параметров.

### <a name="turn-off-block-at-first-sight-with-group-policy"></a>Отключите блок с первого взгляда с помощью групповой политики

1. На компьютере управления групповой [](/previous-versions/windows/it-pro/windows-server-2008-R2-and-2008/cc731212(v=ws.11))политикой откройте консоль управления групповой политикой, щелкните правой кнопкой мыши объект групповой политики, который необходимо настроить, а затем выберите **Изменить**.

2. С помощью **редактора управления групповой политикой** перейдите к **конфигурации компьютера** и выберите **административные шаблоны.**

3. Расширение дерева с помощью **компонентов Windows** Microsoft  >  **Defender Antivirus**  >  **MAPS**.

4. Дважды **щелкните Настройка функции "Блок** с первого взгляда" и установите параметр **Отключен.**

    > [!NOTE]
    > Отключение блока на первый взгляд не отключит и не изменит обязательные групповые политики.

## <a name="not-an-enterprise-admin-or-it-pro"></a>Не администратор предприятия или ИТ-профессионал?

Если вы не администратор предприятия или ИТ-профессионал, но у вас есть вопросы о блокировке на первый взгляд, этот раздел для вас. Блок на первый взгляд — это функция защиты от угроз, которая обнаруживает и блокирует вредоносные программы в течение нескольких секунд. Несмотря на то, что не существует определенного параметра под названием "Блок с первого взгляда", функция включена при настройке определенных параметров на устройстве.

### <a name="how-to-manage-block-at-first-sight-on-or-off-on-your-own-device"></a>Управление блоком с первого взгляда на собственном устройстве или выключение

Если у вас есть личное устройство, которое не управляется организацией, вам может быть интересно, как включить блокировку с первого взгляда или отключить. С помощью приложения Windows Security можно управлять блоком с первого взгляда.

1. На компьютере Windows 10 откройте приложение Windows Security.

2. Выберите **защиту & вирусов.**

3. В **соответствии с & параметров** защиты от угроз выберите **Параметры Управления**.

4. Выполните одно из следующих действий:

   - Чтобы включить блок на первый взгляд, убедитесь, что **обе** системы защиты с облачной доставкой и **автоматическая** отправка образцов будут включены.

   - Чтобы отключить блокировку на первый взгляд, **отключите** облачную защиту или **автоматическую отправку образца.** <br/>
    
     > [!CAUTION]
     > Отключение блока с первого взгляда снижает уровень защиты устройства. Мы не рекомендуем навсегда отключить блок с первого взгляда.
