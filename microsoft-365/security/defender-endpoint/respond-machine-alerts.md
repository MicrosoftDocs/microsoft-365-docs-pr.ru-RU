# <a name="take-response-actions-on-a-device"></a>Выполнение действий ответов на устройстве

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/?linkid=2154037)

>Хотите испытать Defender for Endpoint? [Подпишитесь на бесплатную пробную версию.](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp?ocid=docs-wdatp-respondmachine-abovefoldlink) 

Быстро реагируйте на обнаруженные атаки, изолируя устройства или собирая пакет расследований. После принятия мер на устройствах, вы можете проверить информацию о деятельности в центре действий.

Действия реагирования проходят в верхней части страницы конкретного устройства и включают в себя:

- Управление тегами
- Инициировать автоматизированное расследование
- Инициировать сеанс live Response
- Сбор пакета исследования
- Запуск проверки на вирусы
- Ограничить выполнение приложения
- Изолировать устройство
- Обратитесь к эксперту по угрозам
- Центр уведомлений

[![Изображение действий реагирования ](images/response-actions.png)](images/response-actions.png#lightbox)

 Страницы устройств можно найти на любом из следующих представлений:

- **Панель мониторинга операций** безопасности - Выберите имя устройства из карты риска Устройств.
- **Очередь** оповещений - Выберите имя устройства рядом с иконой устройства из очереди оповещений.
- **Список устройств** - Выберите заголовок имени устройства из списка устройств.
- **Окно поиска** - Выберите устройство из выпадают из меню и введите имя устройства.

>[!IMPORTANT]
> - Эти действия реагирования доступны только для устройств на Windows 10, версии 1703 или позже. 
> - Для неправительственных Windows, возможности реагирования (такие как изоляция устройств) зависят от сторонних возможностей.

## <a name="manage-tags"></a>Управление тегами

Добавление или управление тегами для создания логической принадлежности к группе. Теги устройств поддерживают правильное отображение сети, позволяя прикреплять различные теги для захвата контекста и включения создания динамического списка в рамках инцидента.

Для получения дополнительной информации о маркировке устройств [см.](machine-tags.md)

## <a name="initiate-automated-investigation"></a>Инициировать автоматизированное расследование

При необходимости можно начать автоматическое исследование на устройстве общего назначения. Во время времени времени расследования любое другое оповещение, полученное с устройства, будет добавлено к продолжающемуся автоматизированному расследованию до завершения этого расследования. Кроме того, если такая же угроза наблюдается и на других устройствах, эти устройства добавляются к расследованию.

Для получения дополнительной информации об автоматизированных исследованиях [см.](automated-investigations.md)

## <a name="initiate-live-response-session"></a>Инициировать сеанс live Response

Live ответ является возможность, которая дает вам мгновенный доступ к устройству с помощью удаленного соединения оболочки. Это дает вам возможность проработать углубленную следственную работу и принять незамедлительные меры реагирования для оперативного сдерживания выявленных угроз – в режиме реального времени.

Реакция в реальном времени предназначена для улучшения расследований, позволяя собирать криминалистико-криминалистию, запускать сценарии, отправлять подозрительные объекты для анализа, устранения угроз и активно охотиться за возникающими угрозами.

Для получения дополнительной информации о живом ответе [см.](live-response.md)

## <a name="run-microsoft-defender-antivirus-scan-on-devices"></a>Вы запустите антивирусная программа в Microsoft Defender сканирование на устройствах

В рамках расследования или процесса ответа можно удаленно инициировать антивирусное сканирование, чтобы помочь выявить и исправить вредоносные программы, которые могут присутствовать на скомпрометированном устройстве.

>[!IMPORTANT]
>- Это действие доступно для устройств на Windows 10, версии 1709 или позже.
>- Сканирование антивирусная программа в Microsoft Defender (Microsoft Defender AV) может работать вместе с другими антивирусными решениями, независимо от того, является ли Microsoft Defender AV активным антивирусным решением или нет. Microsoft Defender AV может быть в пассивном режиме. Для получения дополнительной информации [с антивирусная программа в Microsoft Defender м.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-antivirus/microsoft-defender-antivirus-compatibility.md)

Один вы выбрали **запустить антивирусное сканирование,** выберите тип сканирования, который вы хотели бы запустить (быстрый или полный) и добавить комментарий, прежде чем подтвердить сканирование.

![Изображение уведомления, чтобы выбрать быстрое сканирование или полное сканирование и добавить комментарий](images/run-antivirus.png)

Центр действий покажет информацию сканирования, а хронология устройства будет включать в себя новое событие, отражающее, что на устройстве было представлено действие сканирования. Оповещения Microsoft Defender AV будут отражать любые обнаружения, которые всплыли во время сканирования.

>[!NOTE]
>При запуске сканирования с помощью действия Defender for Endpoint антивирус Microsoft Defender 'ScanAvgCPULoadFactor' значение по-прежнему применяется и ограничивает влияние сканирования на процессор.<br> Если ScanAvgCPULoadFactor не настроен, значение по умолчанию является ограничением 50% максимальной нагрузки процессора во время сканирования.<br>
>Для получения дополнительной информации [см.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-antivirus/configure-advanced-scan-types-microsoft-defender-antivirus)

## <a name="restrict-app-execution"></a>Ограничить выполнение приложения

В дополнение к сдерживанию атаки путем остановки вредоносных процессов, вы также можете заблокировать устройство и предотвратить последующие попытки потенциально вредоносных программ от запуска.

>[!IMPORTANT]
> - Это действие доступно для устройств на Windows 10, версии 1709 или позже.
> - Эта функция доступна, если ваша организация использует антивирусная программа в Microsoft Defender.
> - Это действие должно соответствовать Защитник Windows политики управления приложениями и требованиям к подписанию. Для получения дополнительной информации [см.](https://docs.microsoft.com/windows/device-security/device-guard/requirements-and-deployment-planning-guidelines-for-device-guard#code-integrity-policy-formats-and-signing)

Чтобы ограничить запуск приложения, применяется политика целостности кода, которая позволяет файлам работать только в том случае, если они подписаны сертификатом корпорации Майкрософт. Этот метод ограничения может помочь предотвратить злоумышленника от управления скомпрометированными устройствами и выполнения дальнейших вредоносных действий.

>[!NOTE]
>Вы сможете отменить ограничение запуска приложений в любое время. Кнопка на странице устройства будет меняться, чтобы сказать **Удалить ограничения приложения,** а затем вы принимаете те же меры, как ограничение выполнения приложения.

После того как вы выбрали **Ограничение выполнения приложения** на странице устройства, ввеми комментарий и выберите **Подтвердить**. Центр действий покажет информацию сканирования, а хронология устройства будет включать в себя новое событие.

## <a name="isolate-devices-from-the-network"></a>Изолировать устройства от сети

В зависимости от серьезности атаки и чувствительности устройства можно изолировать устройство от сети. Это действие может помочь предотвратить злоумышленника от управления скомпрометированным устройством и выполнения дальнейших действий, таких как эксфильтрация данных и боковое движение.

>[!IMPORTANT]
>- Полная изоляция доступна для устройств на Windows 10, версия 1703.
>- Селективная изоляция доступна для устройств на Windows 10, версии 1709 или позже.
>- При изоляции устройства допускаются только определенные процессы и направления. Таким образом, устройства, стоящие за полным VPN туннелем, не смогут добраться до облачного сервиса Microsoft Defender for Endpoint после того, как устройство будет изолировано. Мы рекомендуем использовать VPN с разделенным туннелированием для Microsoft Defender для Endpoint и антивирусная программа в Microsoft Defender облачного трафика, связанного с защитой.

Функция изоляции устройства отключает скомпрометированное устройство от сети, сохраняя при этом подключение к службе Defender for Endpoint, которая продолжает следить за устройством.

В Windows 10, версии 1709 или позже, вы будете иметь дополнительный контроль над уровнем изоляции сети. Вы также можете включить Outlook, Microsoft Teams и Skype для бизнеса (такую же «селективную изоляцию»).

>[!NOTE]
>Вы сможете подключить устройство к сети в любое время. Кнопка на странице устройства будет меняться, чтобы сказать **освобождение от изоляции,** а затем вы принимаете те же меры, как изоляция устройства.

После того как вы выбрали **устройство Isolate** на странице устройства, ввеми комментарий и выберите **Подтвердить**. Центр действий покажет информацию сканирования, а хронология устройства будет включать в себя новое событие.

![Изображение изолятного устройства](images/isolate-device.png)

>[!NOTE]
>Устройство будет оставаться подключенным к службе Defender for Endpoint, даже если оно изолировано от сети. Если вы выбрали для включения Outlook Skype для бизнеса связи, то вы сможете общаться с пользователем в то время как устройство изолировано.

**Уведомление пользователя устройства:**</br>
При изоляции устройства отображается следующее уведомление, информирующее пользователя о том, что устройство изолировано от сети:

![Изображение без подключения к сети](images/atp-notification-isolate.png)

## <a name="consult-a-threat-expert"></a>Обратитесь к эксперту по угрозам

Вы можете проконсультироваться с экспертом по угрозам корпорации Майкрософт для получения дополнительной информации о потенциально скомпрометированном устройстве или уже скомпрометированном. эксперты Майкрософт по угрозам могут быть вовлечены непосредственно изнутри Центр безопасности в Microsoft Defender для своевременного и точного ответа. Эксперты дают представление не только о потенциально скомпрометированном устройстве, но и для лучшего понимания сложных угроз, целевых уведомлений об атаках, которые вы получаете, или если вам нужно больше информации о оповещениях, или контексте разведки угроз, который вы видите на панели мониторинга портала.

[Подробнее об этом можно усмотреть у эксперта по](https://docs.microsoft.com/microsoft-365/security/defender-endpoint/configure-microsoft-threat-experts#consult-a-microsoft-threat-expert-about-suspicious-cybersecurity-activities-in-your-organization) угрозам корпорации Майкрософт.


## <a name="check-activity-details-in-action-center"></a>Проверка сведений о действиях в центре действий

Центр **действий предоставляет** информацию о действиях, которые были предприняты на устройстве или файле. Вы сможете просмотреть следующие детали:

- Сбор пакетов расследований
- Антивирусное сканирование
- Ограничение приложения
- Изоляция устройства

Все остальные связанные детали также отображаются, например, дата отправки/время отправки пользователя, и если действие удалось или не удалось.
