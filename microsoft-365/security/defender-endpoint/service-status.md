# <a name="check-the-microsoft-defender-for-endpoint-service-health"></a>Проверка состояния службы "Защитник Майкрософт" для службы конечных точек

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]


**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/?linkid=2154037)



>Хотите испытать Microsoft Defender для конечной точки? [Зарегистрився для бесплатной пробной.](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp?ocid=docs-wdatp-servicestatus-abovefoldlink)

**Служба health** предоставляет сведения о текущем состоянии службы Defender для конечных точек. Вы сможете проверить, является ли состояние службы здоровым или имеются ли текущие проблемы. Если есть проблемы, вы увидите сведения, например, когда проблема была обнаружена, какова предварительная корневая причина, и ожидаемое время разрешения.

Вы также увидите сведения об исторических проблемах, которые были устранены, и сведения, такие как дата и время решения проблемы. Если в службе нет проблем, вы увидите здоровый статус.

Сведения о состоянии службы можно просмотреть, щелкнув плитку из  панели мониторинга операций безопасности или выбрав меню здоровья службы из области навигации. 

На **странице сведения о** здоровье службы есть следующие вкладки:

- **Текущее состояние**
- **История состояния**

## <a name="current-status"></a>Текущее состояние
На **вкладке Текущий** статус отображает текущее состояние службы Defender для конечной точки. При бесперебойной работе службы отображается здоровое состояние службы. Если имеются проблемы, показано, что ниже показаны сведения о службе, которые помогут вам лучше понять проблему:

- Дата и время обнаружения проблемы
- Краткое описание проблемы
- Время обновления
- Сводка последствий
- Предварительная корневая причина
- Дальнейшие действия
- Ожидаемое время разрешения

Обновления о ходе выполнения проблемы отражаются на странице по мере решения проблемы. Вы увидите обновления сведений, таких как обновленное время разрешения оценки или последующие действия.

Когда проблема будет устранена, она будет записана на вкладке **История состояния.**

## <a name="status-history"></a>История состояния
Вкладка **История** состояния отражает все исторические проблемы, которые были замечены и устранены. Вы увидите сведения об разрешенных проблемах, а также другие сведения, которые были включены во время их решения.
