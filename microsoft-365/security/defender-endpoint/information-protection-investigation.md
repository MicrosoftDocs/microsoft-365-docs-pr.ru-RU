# <a name="use-sensitivity-labels-to-prioritize-incident-response"></a>Использование меток конфиденциальности для приоритизации реагирования на инциденты  

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/p/?linkid=2146631)
- [Microsoft 365 Defender](https://go.microsoft.com/fwlink/?linkid=2118804)

> Хотите испытать Defender для конечной точки? [Зарегистрився для бесплатной пробной.](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp?ocid=docs-wdatp-exposedapis-abovefoldlink) 


Типичный расширенный жизненный цикл сохраняемой угрозы включает эксфильтрация данных. При инциденте с безопасностью важно иметь возможность приоритизировать расследования, в которых конфиденциальные файлы могут быть под угрозой, чтобы корпоративные данные и сведения были защищены.

Defender for Endpoint позволяет значительно упростить приоритеты инцидентов безопасности с помощью меток конфиденциальности. Метки конфиденциальности быстро выявляют инциденты, которые могут включать устройства с конфиденциальной информацией, например конфиденциальной информацией. 

## <a name="investigate-incidents-that-involve-sensitive-data"></a>Расследование инцидентов с конфиденциальными данными
Узнайте, как использовать метки конфиденциальности данных для приоритетов расследования инцидентов.

>[!NOTE]
>Метки обнаруживаются для Windows 10, версии 1809 или более поздней версии.

1. В Центре безопасности Защитника Майкрософт выберите **Инциденты**. 

2. Прокрутите вправо, чтобы увидеть столбец **"Чувствительность к данным".** В этом столбце отражены метки чувствительности, которые были замечены на устройствах, связанных с инцидентами, что указывает на то, может ли инцидент повлиять на конфиденциальные файлы.

    ![Изображение столбца чувствительности к данным](images/data-sensitivity-column.png)

    Можно также фильтровать на основе **чувствительности к данным** 

    ![Изображение фильтра чувствительности к данным](images/data-sensitivity-filter.png)

3. Откройте страницу инцидента для дальнейшего расследования.

    ![Изображение сведений о странице инцидента](images/incident-page.png)

4. Выберите **вкладку Devices,** чтобы определить устройства, хранимые файлы с метками конфиденциальности.

    ![Изображение вкладки устройства](images/investigate-devices-tab.png)
   

5. Выберите устройства, на которых хранятся конфиденциальные данные и которые будут искать в временной шкале, чтобы определить, какие файлы могут повлиять, затем примите соответствующие меры для обеспечения защиты данных. 

   Вы можете сузить события, показанные на временной шкале устройства, ища метки чувствительности к данным. При этом будут показываться только события, связанные с файлами с именем метки.

    ![Изображение временной шкалы устройства с суженными результатами поиска на основе метки](images/machine-timeline-labels.png)


>[!TIP]
>Эти точки данных также выявляются с помощью "DeviceFileEvents" в продвинутой охоте, что позволяет расширенным запросам и обнаружению расписания учитывать метки чувствительности и состояние защиты файлов.
