# <a name="api-explorer"></a>Обозреватель API

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/?linkid=2154037)


Обозреватель API конечных точек Microsoft Defender — это средство, которое помогает вам исследовать различные API-API Defender для конечных точек в интерактивном режиме. 

Обозреватель API позволяет легко создавать и делать запросы API, тестировать и отправлять запросы для любой доступной конечной точки API Defender для конечной точки. Используйте обозреватель API, чтобы принять меры или найти данные, которые могут еще не быть доступны через пользовательский интерфейс.

Этот инструмент полезен при разработке приложения. Это позволяет выполнять запросы API, которые уважают параметры доступа пользователей, сокращая потребность в создании маркеров доступа.

С помощью этого средства можно изучить галерею примерных запросов, скопировать примеры кода результатов и создать сведения о отлаговке.

С помощью обозревателя API можно:

- Выполнить запросы для любого метода и увидеть ответы в режиме реального времени
- Быстро просмотрите примеры API и узнайте, какие параметры они поддерживают
- С легкостью делайте вызовы API; нет необходимости проверки подлинности за пределами входного знака портала управления

## <a name="access-api-explorer"></a>Обозреватель API доступа

В левом меню навигации **выберите &**  >  **API Explorer API.**

## <a name="supported-apis"></a>Поддерживаемые API

Обозреватель API поддерживает все API, предлагаемые Defender для конечной точки.
  
Список поддерживаемых API доступен в документации [API.](apis-intro.md) 

## <a name="get-started-with-the-api-explorer"></a>Начало работы с обозревателем API

1. В левой области имеется список примеров запросов, которые можно использовать. 
2. Следуйте ссылкам и нажмите **кнопку Выполнить запрос**. 

В некоторых примерах может потребоваться указать параметр в URL-адресе, например {machine-ID}.

## <a name="faq"></a>Вопросы и ответы

**Нужно ли иметь маркер API для использования обозревателя API?** <br>
Учетные данные для доступа к API не нужны. Обозреватель API использует маркер портала управления Defender для конечных точек всякий раз, когда он делает запрос.

Учетные данные проверки подлинности пользователей, зарегистрированные в журнале, используются для проверки того, что обозреватель API уполномочен получать доступ к данным от вашего имени.

Конкретные запросы API ограничены в зависимости от ваших привилегий RBAC. Например, запрос на "Отправить индикатор" ограничен ролью администратора безопасности.
