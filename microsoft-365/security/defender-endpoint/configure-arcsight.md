# <a name="configure-micro-focus-arcsight-to-pull-defender-for-endpoint-detections"></a>Настройте Micro Focus ArcSight, чтобы вытащить Defender для обнаружения конечных точек

[!INCLUDE [Microsoft 365 Defender rebranding](../../includes/microsoft-defender.md)]

**Область применения:**
- [Microsoft Defender для конечной точки](https://go.microsoft.com/fwlink/p/?linkid=2154037)
- [Microsoft 365 Defender](https://go.microsoft.com/fwlink/?linkid=2118804)


>Хотите испытать Defender для конечной точки? [Зарегистрився для бесплатной пробной.](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp?ocid=docs-wdatp-configurearcsight-abovefoldlink) 

Вам потребуется установить и настроить некоторые файлы и средства для использования Micro Focus ArcSight, чтобы он может вытащить Defender для обнаружения конечных точек.

>[!Note]
>- [Defender for Endpoint Alert](alerts.md) состоит из одного или нескольких обнаружений
>- [Defender for Endpoint Detection](api-portal-mapping.md) состоит из подозрительного события, произошедшего на устройстве, и связанных с ним сведений оповещения.

## <a name="before-you-begin"></a>Перед началом работы

Для настройки средства подключения Micro Focus ArcSight требуется несколько файлов конфигурации, чтобы он извлекает и разборирует обнаружения из приложения Azure Active Directory (AAD).

В этом разделе вы можете получить необходимую информацию, чтобы правильно настроить и использовать необходимые файлы конфигурации.

- Убедитесь, что вы включили функцию интеграции SIEM из меню **Параметры.** Дополнительные сведения см. в [дополнительных сведениях: Включение интеграции SIEM в Defender для конечной точки.](enable-siem-integration.md)

- Готовый файл, сохраненный при включительно функции интеграции SIEM. Необходимо получить следующие значения:
  - URL-адрес обновления маркера OAuth 2.0
  - ID клиента OAuth 2.0
  - Секрет клиента OAuth 2.0

- Готовы следующие файлы конфигурации:
  - WDATP-connector.properties
  - WDATP-connector.jsonparser.properties

    Вы сохранили бы файл .zip, который содержит эти два файла, когда вы выбрали Micro Focus ArcSight в качестве типа SIEM, используемого в организации.

- Убедитесь, что вы создаете следующие маркеры и готовы:
  - Маркер доступа
  - Маркер обновления

  Эти маркеры можно создавать из раздела **установки интеграции SIEM** портала.

## <a name="install-and-configure-micro-focus-arcsight-flexconnector"></a>Установка и настройка Micro Focus ArcSight FlexConnector

1. Установка последнего 32-битного установщика Windows FlexConnector. Это можно найти в центре программного обеспечения HPE. Обычно средство устанавливается в следующем расположении по умолчанию: `C:\Program Files\ArcSightFlexConnectors\current\bin` .</br></br>Вы можете выбрать, где сохранить средство, например \\ *C: folder_location*\current\bin, *folder_location* представляет расположение установки.

   - Выберите папку ярлыков
   - Сводка предварительной установки
   - Установка...

7. Соединитетелем открывается окно браузера. Войдите в систему с учетными данными приложения. После входа вам будет предложено предоставить разрешение клиенту OAuth2. Необходимо предоставить разрешение клиенту OAuth 2 для проверки подлинности конфигурации соединители.

   Если <code>redirect_uri</code> url-адрес https, вы будете перенаправлены на URL-адрес локального хоста. Вы увидите страницу, на которую требуется доверять сертификату, предоставленного соединитетелем, работающим на локальном хосте. Вам необходимо доверять этому сертификату, если redirect_uri является https.
   
   Если же вы указываете URL-адрес http для redirect_uri, вам не нужно предоставлять согласие на доверие сертификату.

8. Продолжить установку соединиттеля, возвращаясь к окну Установки соединиттеля ArcSight Micro Focus.

10. Введите ip/hostname назначения в **имени hostname** диспетчера и учетные данные в форме параметров. Все остальные значения в форме должны сохраняться со значениями по умолчанию. Нажмите кнопку **Далее**.

11. Введите имя соединитетеля в форме сведений о соединители. Все остальные значения в форме необязательны и могут быть оставлены пустыми. Нажмите кнопку **Далее**.

12. Отображается окно сертификата импорта ESM Manager. Выберите **импорт сертификата для соединитетеля из пункта назначения** и нажмите **кнопку Далее**. Отображается **окно Сводки соединителю** Добавить и импортируется сертификат.

13. Убедитесь, что сведения в окне **Сводка** добавить соединителю правильные, нажмите **кнопку Далее**.

14. Выберите **Установите в качестве службы и** нажмите **кнопку Далее**.

15. Введите имя в **поле "Внутреннее имя** службы". Все остальные значения в форме можно сохранить со значениями по умолчанию или оставить пустыми. Нажмите кнопку **Далее**.

16. Введите параметры службы и нажмите кнопку **Далее**. Отображается окно с **сводка** службы установки. Нажмите кнопку **Далее**.

17. Завершите установку, выбрав **Exit** и **Далее.**

## <a name="install-and-configure-the-micro-focus-arcsight-console"></a>Установка и настройка консоли ArcSight Micro Focus

1. Выполните следующие задачи мастера установки:
   - Введение
   - Лицензионный договор
   - Специальное уведомление
   - Выбор каталога установки ArcSight
   - Выберите папку ярлыков
   - Сводка предварительной установки

2. Нажать кнопку **Установить**. После завершения установки откроется мастер конфигурации консоли ArcSight.

3. Введите localhost в **имени хозяина диспетчера** и 8443 в **диспетчерском порту,** а затем нажмите **кнопку Далее**.

4. Выберите **Использование прямого подключения,** а затем нажмите **кнопку Далее**.

5. Выберите **проверку подлинности на основе пароля,** а затем нажмите **кнопку Далее**.

6. Выберите **Это установка одного пользователя. (Рекомендуется)** и нажмите кнопку **Далее**.

7. Щелкните **Кнопку Готово,** чтобы выйти из установки.

8. Вход на консоль Micro Focus ArcSight.

9. Перейдите к **набору active channel**  >  **New Condition**  >    >  **Device Device Product**.

10. Set **Device Product = Microsoft Defender ATP**. Когда вы убедились, что события течет в инструмент, остановите процесс снова и перейдите в Службы Windows и запустите Rest FlexConnector ArcSight.

Теперь можно запускать запросы в консоли Micro Focus ArcSight.

Обнаружение Defender для конечных точек будет отображаться в качестве дискретных событий, а "Microsoft" в качестве поставщика и "Защитник Windows ATP" в качестве имени устройства.


## <a name="troubleshooting-micro-focus-arcsight-connection"></a>Устранение неполадок подключение к Micro Focus ArcSight

**Проблема:** Не удалось обновить маркер. Вы можете найти журнал, расположенный в \\ *C: folder_location*\current\logs, folder_location представляет расположение, где установлено средство.  Откройте _agent.log_ и найдите `ERROR/FATAL/WARN` .

**Симптом:** Вы получите следующее сообщение об ошибке:

`Failed to refresh the token. Set reauthenticate to true: com.arcsight.common.al.e: Failed to refresh access token: status=HTTP/1.1 400 Bad Request FATAL EXCEPTION: Could not refresh the access token`

**Решение:**

1. Остановите процесс, щелкнув Ctrl + C в окне Соединителя. Нажмите **кнопку Y** при запросе "Завершить пакетное задание Y/N?".

2. Перейдите к папке, в которой хранится файл WDATP-connector.properties, и отредактировать его, чтобы добавить следующее значение: `reauthenticate=true` .

3. Перезапустите соединитектор, запустив следующую команду: `arcsight.bat connectors` .

   Появляется окно браузера. Разрешить его запуск, он должен исчезнуть, и соединители должны быть запущены.

> [!NOTE]
> Убедитесь, что соединитатель запущен, остановив процесс снова. Затем запустите соединители снова, и окно браузера не должно отображаться.
