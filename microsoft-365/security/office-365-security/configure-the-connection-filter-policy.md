# <a name="configure-connection-filtering"></a>Настройка фильтрации подключений

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Exchange Online Protection](exchange-online-protection-overview.md)
- [Microsoft Defender для Office 365 (план 1 и план 2)](office-365-atp.md)
- [Microsoft 365 Defender](../mtp/microsoft-threat-protection.md)


Если вы клиент Microsoft 365 с почтовыми ящиками в Exchange Online или автономный клиент Exchange Online Protection (EOP) без почтовых ящиков Exchange Online, вы используете фильтрацию подключений в EOP (в частности, политику фильтрации подключений по умолчанию) для определения хороших или плохих исходных серверов электронной почты по их IP-адресам. Основные компоненты политики фильтра подключения по умолчанию:

- **СПИСОК IP-адресов:** пропустить фильтрацию нежелательной почты для всех входящих сообщений с исходных серверов электронной почты, которые указаны в диапазоне IP-адресов или IP-адресов. Сценарии, в которых фильтрация нежелательной почты может по-прежнему происходить в сообщениях из этих источников, см. в разделе Сценарии, в которых сообщения из источников в [списке ip Allow по-прежнему](#scenarios-where-messages-from-sources-in-the-ip-allow-list-are-still-filtered) фильтруются позже в этой статье. Дополнительные сведения о том, как список IP-адресов должен вписаться в общую стратегию безопасных отправителей, см. в статью Создание списков безопасных отправителей [в EOP.](create-safe-sender-lists-in-office-365.md)

- **Список блоков** IP. Блокировать все входящие сообщения с исходных серверов электронной почты, которые указаны по IP-адресу или диапазону IP-адресов. Входящие сообщения отклоняется, не помечены как нежелательной почты, а дополнительная фильтрация не происходит. Дополнительные сведения о том, как блок ip-список должен вписаться в общую стратегию заблокированных отправителей, см. в статью Создание списков отправителей блоков [в EOP.](create-block-sender-lists-in-office-365.md)

- **Безопасный** список. Безопасный *список* — это динамический список разрешаний в центр обработки данных Майкрософт, который не требует конфигурации клиента. Корпорация Майкрософт определяет эти надежные источники электронной почты из подписок на различные сторонние списки. Включить или отключить использование безопасного списка; вы не можете настроить исходные серверы электронной почты в безопасном списке. Фильтрация нежелательной почты пропускается при входящих сообщениях с серверов электронной почты в безопасном списке.

В этом разделе описывается настройка политики фильтра подключения по умолчанию в Центре обеспечения безопасности & соответствия требованиям или в PowerShell (Exchange Online PowerShell для организаций Microsoft 365 с почтовыми ящиками в Exchange Online; автономный EOP PowerShell для организаций без почтовых ящиков Exchange Online). Дополнительные сведения о том, как EOP использует фильтрацию подключений, являются частью общих параметров защиты от нежелательной почты в организации, см. в см. в [рублях Защита от нежелательной почты.](anti-spam-protection.md)

> [!NOTE]
> Список IP-адресов, безопасный список и список блоков IP являются частью общей стратегии допуска или блокировки электронной почты в организации. Дополнительные сведения см. в [дополнительных сведениях: Создание](create-safe-sender-lists-in-office-365.md) списков безопасного отправитель и [создание заблокированных списков отправитель](create-block-sender-lists-in-office-365.md).

## <a name="use-the-security--compliance-center-to-modify-the-default-connection-filter-policy"></a>Используйте Центр & безопасности для изменения политики фильтра подключения по умолчанию

1. В Центре обеспечения & безопасности и перейдите в **центр по** борьбе со спамом политики управления \>  \> **угрозами.**

2. На странице **Параметры для** борьбы со нежелательной почтой разойдитесь  по политике фильтрации подключения, нажав значок Расширение, а затем нажмите ![ ](../../media/scc-expand-icon.png) **кнопку Изменить политику**.

3. В **вылете** по умолчанию настройте любой из следующих параметров:

   - **Описание.** Введите необязательный описательный текст.

   - **СПИСОК IP-адресов:** нажмите **кнопку Изменить**. В поле **ip Allow List,** который отображается, введите адрес IPV4 в поле **Адрес** или диапазон адресов с помощью следующего синтаксиса:

     - Один IP. Например, 192.168.1.1.

     - Диапазон IP: Например, 192.168.0.1-192.168.0.254.

     - IP CIDR. Например, 192.168.0.1/25. Допустимые значения сетевой маски от /24 до /32. Чтобы пропустить фильтрацию нежелательной почты для значений IP-маски CIDR /1 к /23, см. в статье Пропустить фильтрацию нежелательной почты для [IP-адреса CIDR](#skip-spam-filtering-for-a-cidr-ip-outside-of-the-available-range) за пределами доступного раздела диапазона.

     Чтобы добавить IP-адрес или диапазон адресов, нажмите **кнопку Добавить значок** ![ Добавить ](../../media/ITPro-EAC-AddIcon.png) . Чтобы удалить запись, выберите запись в Разрешенный **IP-адрес** и нажмите кнопку **Удалить** ![ удалить ](../../media/scc-remove-icon.png) . По завершении нажмите кнопку **Сохранить**.

   - **Список блоков IP:** нажмите **кнопку Изменить**. В **вылете** списка блокировки IP введите один IP-адрес, ip-диапазон или IP-адрес CIDR в поле Адрес или диапазон адресов, как описано ранее в параметре **список ip Allow.** 

     Чтобы добавить IP-адрес или диапазон адресов, нажмите **кнопку Добавить значок** ![ Добавить ](../../media/ITPro-EAC-AddIcon.png) . Чтобы удалить запись, выберите запись в **заблокированных** IP-адресе и нажмите **кнопку Удалить** ![ ](../../media/scc-remove-icon.png) . По завершении нажмите кнопку **Сохранить**.

   - **Включи безопасный** список. Включить или отключить использование безопасного списка для идентификации известных и хороших отправителей, которые будут пропускать фильтрацию нежелательной почты.

4. По завершении нажмите кнопку **Сохранить**.

## <a name="use-exchange-online-powershell-or-standalone-eop-powershell-to-modify-the-default-connection-filter-policy"></a>Используйте Exchange Online PowerShell или автономный EOP PowerShell для изменения политики фильтра подключения по умолчанию

Используйте указанный ниже синтаксис.

```powershell
Set-HostedConnectionFilterPolicy -Identity Default [-AdminDisplayName <"Optional Comment">] [-EnableSafeList <$true | $false>] [-IPAllowList <IPAddressOrRange1,IPAddressOrRange2...>] [-IPBlockList <IPAddressOrRange1,IPAddressOrRange2...>]
```

**Примечания**.

- Допустимые значения ip-адреса или диапазона адресов:

  - Один IP. Например, 192.168.1.1.

  - Диапазон IP: Например, 192.168.0.1-192.168.0.254.

  - IP CIDR. Например, 192.168.0.1/25. Допустимые значения сетевой маски от /24 до /32.

- Чтобы *переписать все* существующие записи со значениями, которые вы указываете, используйте следующий синтаксис: `IPAddressOrRange1,IPAddressOrRange2,...,IPAddressOrRangeN` .

- Чтобы *добавить или удалить* IP-адреса или диапазоны адресов без влияния на другие существующие записи, используйте следующий синтаксис: `@{Add="IPAddressOrRange1","IPAddressOrRange2",...,"IPAddressOrRangeN";Remove="IPAddressOrRange3","IPAddressOrRange4",...,"IPAddressOrRangeN"}` .

- Чтобы очистить список IP-адресов или список блоков IP, используйте значение `$null` .

В этом примере настраивается список IP-адресов и список блоков IP с указанными IP-адресами и диапазонами адресов.

```powershell
Set-HostedConnectionFilterPolicy -Identity Default -IPAllowList 192.168.1.10,192.168.1.23 -IPBlockList 10.10.10.0/25,172.17.17.0/24
```

В этом примере добавляются и удаляются указанные IP-адреса и диапазоны адресов из списка ip Allow.

```powershell
Set-HostedConnectionFilterPolicy -Identity Default -IPAllowList @{Add="192.168.2.10","192.169.3.0/24","192.168.4.1-192.168.4.5";Remove="192.168.1.10"}
```

Подробные сведения о синтаксисах и параметрах см. в [ссылке Set-HostedConnectionFilterPolicy.](https://docs.microsoft.com/powershell/module/exchange/set-hostedconnectionfilterpolicy)

## <a name="additional-considerations-for-the-ip-allow-list"></a>Дополнительные соображения для списка ip-адресов

В следующих разделах определяются дополнительные элементы, которые необходимо знать при настройке списка IP-адресов.

### <a name="skip-spam-filtering-for-a-cidr-ip-outside-of-the-available-range"></a>Пропустить фильтрацию нежелательной почты для IP-адреса CIDR за пределами доступного диапазона

Как описано выше в этой статье, в списке ip-адресов IP CIDR можно использовать только с сетевой маской /24 до /32 в списке IP-адресов. Чтобы пропустить фильтрацию нежелательной почты на сообщения с исходных серверов электронной почты в диапазоне /1-23, необходимо использовать правила потока почты Exchange (также известные как правила транспорта). Но мы рекомендуем не делать этого, если это вообще возможно, так как сообщения будут заблокированы, если IP-адрес в диапазоне IP-адресов CIDR от /1 до /23 появится в любом из проприетарных или сторонних списков блоков Майкрософт.

Теперь, когда вы полностью осведомлены о потенциальных проблемах, можно создать правило потока почты со следующими настройками (как минимум), чтобы сообщения с этих IP-адресов пропускали фильтрацию нежелательной почты:

- Условие **правила.** Применяйте это правило, если IP-адрес отправитель находится в любом из этих диапазонов или точно совпадает (введите IP-адрес CIDR с сетевой маской \>  \>  \> от /1 до /23).

- Действие правила. **Изменение свойств сообщений** Установите уровень доверия к нежелательной почте \> **(SCL)** Обход \> **фильтрации нежелательной почты.**

Вы можете проверить правило, проверить правило, активировать правило в течение определенного периода времени и другие выборы. Мы рекомендуем протестировать правило в течение определенного времени перед его применением. Дополнительные сведения см. в [рублях Управление правилами потока почты в Exchange Online.](https://docs.microsoft.com/Exchange/security-and-compliance/mail-flow-rules/manage-mail-flow-rules)

### <a name="skip-spam-filtering-on-selective-email-domains-from-the-same-source"></a>Пропустить фильтрацию нежелательной почты на выборочные домены электронной почты из того же источника

Обычно добавление IP-адреса или диапазона адресов в список IP-адресов означает, что вы доверяете всем входящие сообщения из этого источника электронной почты. Но что делать, если этот источник отправляет электронную почту из нескольких доменов, и вы хотите пропустить фильтрацию нежелательной почты для некоторых из этих доменов, но не для других? Для этого нельзя использовать только список IP-адресов, но список ip Allow List можно использовать в сочетании с правилом потока почты.

Например, исходный сервер электронной почты 192.168.1.25 отправляет электронную почту из доменов contoso.com, fabrikam.com и tailspintoys.com, но для отправки сообщений от отправителей в fabrikam.com необходимо пропустить фильтрацию нежелательной почты. Для этого используйте следующие действия:

1. Добавьте 192.168.1.25 в список ip-адресов.

2. Настройка правила потока почты с помощью следующих параметров (как минимум):

   - Условие **правила.** Применяйте это правило, если IP-адрес отправитель находится в любом из этих диапазонов или точно соответствует \>  \>  192.168.1.25 (тот же IP-адрес или диапазон адресов, который вы добавили в список IP-адресов на предыдущем \> шаге).

   - Действие правила: **Изменение свойств сообщений** Установите уровень доверия к нежелательной почте \> **(SCL)** \> **0**.

   - Исключение **правила:** домен отправитель fabrikam.com (только домен или домены, которые необходимо пропустить фильтрацию \>  \> нежелательной почты).

### <a name="scenarios-where-messages-from-sources-in-the-ip-allow-list-are-still-filtered"></a>Сценарии, в которых сообщения из источников в списке IP-адресов по-прежнему фильтруются

Сообщения с сервера электронной почты в списке IP-адресов по-прежнему подвергаются фильтрации нежелательной почты в следующих сценариях:

- IP-адрес в списке IP-адресов также настроен в локальном входящий соединителю на основе IP в любом клиенте в Microsoft 365 (назовем это клиентом **A),** а также в клиенте A и на сервере EOP, который первым сталкивается с сообщением, которое находится в одном лесу *Active* Directory в центрах обработки данных Майкрософт.  В этом сценарии **IPV:CAL**  добавляется в заглавные папки сообщений для борьбы со спамом (указывающее на то, что сообщение обошел фильтрацию нежелательной почты), но это сообщение [по-прежнему](anti-spam-message-headers.md) подвергается фильтрации нежелательной почты.

- Ваш клиент, содержащий список IP-адресов и сервер EOP, который впервые сталкивается с сообщением, находится в различных лесах *Active* Directory в центрах обработки данных Майкрософт. В этом сценарии **IPV:CAL**  не добавляется в главы сообщений, поэтому сообщение по-прежнему подвергается фильтрации нежелательной почты.

Если вы столкнулись с одним из этих сценариев, можно создать правило потока почты со следующими настройками (как минимум), чтобы сообщения с проблемных IP-адресов пропускали фильтрацию нежелательной почты:

- Условие правила. **Применяйте** это правило, если IP-адрес отправитель находится в любом из этих диапазонов или точно совпадает \>  \>  \> (ваш IP-адрес или адрес).
