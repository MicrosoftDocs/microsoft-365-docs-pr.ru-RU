# <a name="an-overview-of-automated-investigation-and-response-air-in-microsoft-defender-for-office-365"></a>Обзор автоматического исследования и ответа (AIR) в защитнике Майкрософт для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


По мере инициирования оповещений системы безопасности Группа управления операциями может искать эти оповещения и принимать меры по защите Организации. В некоторых случаях команды безопасности могут быть перегружены объемом триггерных оповещений. Возможности автоматического исследования и реагирования (AIR) в защитнике Майкрософт для Office 365 могут помочь. 

Воздушный поток позволяет группе управления безопасностью работать эффективнее и эффективно. Возможности воздуха включают автоматизированные процессы расследования в ответ на известные угрозы, существующие сегодня. Соответствующие действия по исправлению ожидают утверждения, позволяя группе управления операциями безопасности отвечать на обнаруженные угрозы. 

В этой статье представлен обзор воздуха. Когда вы будете готовы приступить к работе с AIR, ознакомьтесь со статьей [Автоматическое исследование и реагирование на угрозы](office-365-air.md).

## <a name="at-a-high-level"></a>На высоком уровне

При срабатывании оповещений "Playbooks" безопасности переходит в действующий. В зависимости от ситуации может начаться [процесс автоматического исследования](https://docs.microsoft.com/microsoft-365/security/office-365-security/office-365-air) . Во время и после автоматического исследования рекомендуются [действия по исправлению](air-remediation-actions.md) . В защитнике Майкрософт для Office 365 никакие действия не выполняются автоматически. Команда по обеспечению безопасности выполняет проверку, а затем [утверждает или отвергает каждое действие по исправлению](air-review-approve-pending-completed-actions.md). После утверждения или отклонения всех действий, выполняемых после проведения расследования, исследование завершается. Все эти действия отслеживаются и отображаются в центре безопасности Microsoft 365 ( [https://security.microsoft.com](https://security.microsoft.com) ). (Дополнительные сведения см. в статье [Просмотр подробных сведений о расследовании](air-view-investigation-results.md#view-details-of-an-investigation)).

В следующих разделах представлены дополнительные сведения об оповещениях, безопасности "Playbooks" и примерах воздуха в действии.

## <a name="alerts"></a>Оповещения

[Оповещения](../../compliance/alert-policies.md#viewing-alerts) представляют триггеры для рабочих процессов группы операций безопасности для реагирования на инциденты. Приоритизация правильного набора оповещений для изучения, а отсутствие неустраненных угроз не является проблематичным. При расследовании в оповещениях выполняются вручную, поэтому группы операций безопасности должны осуществлять слежение за сущностями (например, контент, устройства и пользователей) от угроз. Такие задачи и рабочие процессы могут занимать много времени и включать несколько средств и систем. В среде AIR исследование и ответ на события безопасности автоматизированы благодаря автоматическому триггеру оповещений о безопасности и управлении угрозами "Playbooks" отклика. 

В настоящее время для воздуха оповещения, созданные из следующих видов политик оповещений, автоматически изучены:  

- Обнаружен потенциально вредоносный URL-адрес.
- Сообщение электронной почты, предоставленное пользователем как фишинг`*`
- Сообщения электронной почты, содержащие вредоносную программу, удалена после доставки`*`
- Сообщения электронной почты, содержащие URL-адреса фишинга, удаленные после доставки`*`
- Обнаружены подозрительные шаблоны отправки электронной почты
- Пользователю запрещено отправлять электронную почту.
- Администратор инициировал ручное исследование электронной почты`*`

> [!NOTE]
> Оповещения, помеченные звездочкой ( `*` ), назначаются *Informational* в соответствующих политиках оповещений в центре безопасности Microsoft 365 с отключенными уведомлениями электронной почты. Уведомления по электронной почте можно включить с помощью [настройки политики оповещений](../../compliance/alert-policies.md#alert-policy-settings). 

Чтобы просмотреть оповещения, в центре безопасности & соответствия требованиям выберите **оповещения**  >  **Просмотр оповещений**. Выберите оповещение, чтобы просмотреть сведения о нем, а затем воспользуйтесь ссылкой **Просмотр расследования** , чтобы перейти к соответствующему [исследованию](air-view-investigation-results.md#investigation-graph).  

> [!NOTE]
> По умолчанию информационные оповещения скрыты в представлении оповещений. Чтобы просмотреть их, измените фильтрацию оповещений, чтобы она включала информационные оповещения.

Если ваша организация управляет оповещениями системы безопасности с помощью системы управления оповещениями, системы управления службами или системы безопасности и управления событиями (SIEM), вы можете отправлять оповещения в эту систему с помощью уведомления по электронной почте или с помощью [API действий управления Office 365](https://docs.microsoft.com/office/office-365-management-api/office-365-management-activity-api-reference). Уведомления об анализе с помощью электронной почты или API содержат ссылки для доступа к оповещениям в центре безопасности Microsoft 365, что позволяет администратору безопасности быстро перейти к расследованию.

![Оповещения, ссылающиеся на расследования](../../media/air-alerts-page-details.png) 

## <a name="security-playbooks"></a>"Playbooks" безопасности

"Playbooks" безопасности — это внутренние политики, которые находятся в сердце автоматизации в защитнике Майкрософт для Office 365 и защиты от угроз Майкрософт. "Playbooks" безопасности, предоставляемые в среде AIR, основаны на распространенных сценариях безопасности и разработаны на основе отзывов от Teams. Стратегия безопасности запускается автоматически при активации определенных оповещений в Организации. После срабатывания триггера оповещения связанный стратегия запускается системой автоматического исследования и ответа. В ходе анализа пошаговое руководство по анализу оповещений на основе стратегия, в котором просматриваются все связанные метаданные (включая сообщения электронной почты, пользователей, темы, отправители и т. д.). В зависимости от полученных результатов исследования стратегия воздух рекомендует набор действий, которые Группа безопасности Организации может предпринять для управления и устранения угроз. 

"Playbooks" по обеспечению безопасности, которые вы будете получать с помощью воздуха, предназначены для самых распространенных угроз, с которыми организации сталкиваются сегодня с помощью электронной почты. Они основываются на входных данных из операций безопасности и команд реагирования на инциденты, в том числе людей, которые помогают защищать ресурсы корпорации Майкрософт и наших клиентов.

- Сообщение о phishing-атаке, сообщенное пользователем
- URL-адрес: нажмите кнопку вредоносности Change
- Вредоносная программа обнаружила пост-Delivery (вредоносный ZAP)
- Антифишинг обнаружил, что после доставки ZAP (фишинг ZAP)
- Пользователь сообщил о нарушении 
- Ручное исследование электронной почты (инициировано администратором от вредоносных программ в Explorer, фишинга или всех почтовых представлений)

Дополнительные обновления "Playbooks" и стратегия будут выпускаться по мере их завершения. Посетите [план Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap) , чтобы увидеть, что еще запланировано и скоро.

### <a name="playbooks-include-investigation-and-recommendations"></a>"Playbooks": исследование и рекомендации

В среде AIR каждый стратегия безопасности включает: 

- корневое исследование объектов электронной почты (таких как файлы, URL-адреса, получателей, IP-адреса и т. д.);
- дальнейшее Поиск похожих сообщений электронной почты, полученных Организацией 
- действия, предпринимаемые для определения и сопоставления других потенциальных угроз, а также 
- Рекомендуемые действия по исправлению угроз.

Каждый этап высокого уровня включает в себя ряд подэтапов, которые выполняются для обеспечения детального, подробного и исчерпывающего ответа на угрозы.

## <a name="example-a-user-reported-phish-message-launches-an-investigation-playbook"></a>Пример: сообщение фишинга, зарегистрированное пользователем, запускает расследования стратегия

Предположим, что пользователь в Организации получает электронное сообщение, которое она думает — попытка фишинга. Пользователь, обученный отчету о таких сообщениях, использует [надстройку Message Report](enable-the-report-message-add-in.md) , чтобы отправить ее в корпорацию Майкрософт для анализа. Отправка также передается в систему и отображается в проводнике в представлении **отправок** (прежнее название — "представление в **отчете о пользователях** "). Кроме того, сообщение, полученное пользователем, теперь запускает системное информационное оповещение, которое автоматически запускает исследование стратегия.

Во время корневого этапа исследования различные аспекты электронного исследования оцениваются. К ним относятся следующие аспекты:

- Определение типа угрозы, который может быть таким:
- Отправитель;
- Откуда отправлено сообщение (инфраструктура отправки);
- Указывает, были ли доставляются или заблокированы другие экземпляры электронной почты;
- Оценка из наших аналитик;
- Связан ли электронная почта со всеми известными кампаниями;
- и многое другое.

По завершении корневого исследования стратегия предоставляет список рекомендуемых действий, которые необходимо выполнить для исходной электронной почты и связанных с ним сущностей.
  
После этого выполняются некоторые этапы расследования и исследования угроз:

- Аналогичные сообщения электронной почты определяются с помощью поиска в кластере электронной почты.
- Этот сигнал используется совместно с другими платформами, например с [защитником Майкрософт для конечной точки](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection).
- Определение, когда пользователи щелкают любые вредоносные ссылки в подозрительных сообщениях электронной почты.
- Проверка выполняется в Exchange Online Protection ([EOP](exchange-online-protection-overview.md)) и Office 365 Advanced Threat protection ([ATP](office-365-atp.md)), чтобы проверить наличие других подобных сообщений, о которых сообщили пользователи.
- Проверка того, был ли пользователь скомпрометирован. Эта проверка использует сигналы в Office 365, [Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security)и [Azure Active Directory](https://docs.microsoft.com/azure/active-directory), соотнесение со всеми связанными аномалиями действий пользователей.

Во время фазы Поиск риски и угрозы назначаются для различных шагов по поиску. 

Исправление является завершающим этапом стратегия. На этом этапе выполняются действия по исправлению, основанные на этапах расследования и исследования. 

## <a name="example-a-security-administrator-triggers-an-investigation-from-threat-explorer"></a>Пример: администратор безопасности инициирует исследование в обозревателе угроз

В дополнение к автоматическому расследованию, которое инициируется оповещением, Группа операций безопасности в Организации может инициировать автоматическое исследование из представления в [обозревателе угроз](threat-explorer.md).  В ходе этого исследования также создается оповещение, чтобы инциденты защитника и внешние средства SIEM могли видеть срабатывание этого исследования. 

Например, предположим, что вы используете представление **вредоносных программ** в проводнике. Используя вкладки под диаграммой, перейдите на вкладку **Электронная почта** . При выборе одного или нескольких элементов в списке активируется кнопка **+ Actions (действия** ). 

![Проводник с выбранными сообщениями](../../media/Explorer-Malware-Email-ActionsInvestigate.png)

С помощью меню **действия** можно выбрать **триггеры для расследования**.

![Меню действий для выбранных сообщений](../../media/explorer-malwareview-selectedemails-actions.jpg)

Как и в "Playbooks", вызванных оповещением, автоматическое исследование, запущенное из представления в проводнике, включает в себя корневое исследование, действия по определению и корреляции угроз, а также Рекомендуемые действия по устранению этих угроз.

## <a name="example-a-security-operations-team-integrates-air-with-their-siem-using-the-office-365-management-activity-api"></a>Пример: Группа "операции безопасности" интегрирует воздух с SIEM с помощью API действий управления Office 365.

Возможности воздуха в защитнике Microsoft для Office 365 включают [отчеты & сведения о](air-view-investigation-results.md) том, какие операции по обеспечению безопасности Teams могут использовать для отслеживания и устранения угроз. Но вы также можете интегрировать возможности AIR с другими решениями. Примеры включают сведения о безопасности и систему управления событиями (SIEM), систему управления делами или настраиваемое решение для создания отчетов. Эти виды интеграции можно выполнить с помощью [API действий управления Office 365](https://docs.microsoft.com/office/office-365-management-api/office-365-management-activity-api-reference). 

Например, в некоторых случаях организация, в которой настроена система безопасности, позволяет просматривать оповещения о phishing-атаках, которые уже были обработаны ВОЗДУХом. Их решение интегрирует соответствующие оповещения с сервером SIEM Организации и системой управления делами. Решение значительно сокращает количество ложных срабатываний, чтобы команда по обеспечению безопасности могла сосредоточиться на реальных угрозах и усилиях. Чтобы узнать больше об этом пользовательском решении, обратитесь [к блогу технического сообщества: улучшите эффективность SoC с помощью Office 365 ATP и API управления O365](https://techcommunity.microsoft.com/t5/microsoft-security-and/improve-the-effectiveness-of-your-soc-with-office-365-atp-and/ba-p/1525185).
