# <a name="an-overview-of-automated-investigation-and-response-air-in-microsoft-365"></a>Обзор автоматизированного анализа угроз и реагирования на них (AIR) в Microsoft 365

При срабатывании оповещений безопасности ваша группа операций безопасности должна просмотреть эти оповещения и принять меры для защиты вашей организации. Иногда группы операций безопасности могут быть перегружены большим количеством срабатываний оповещений. Вам могут помочь возможности автоматического анализа угроз и реагирования на них в Office 365 Advanced Threat Protection (Office 365 ATP). 

AIR позволяет вашей службе обеспечения безопасности эффективнее и эффективнее управлять. Возможности автоматизированного анализа угроз включают автоматизированные процессы расследования в ответ на известные угрозы, существующие на сегодняшний день. Соответствующие действия по исправлению в ожидании утверждаются, позволяя группе операций безопасности реагировать на обнаруженные угрозы. 

В этой статье приведен обзор AIR. Когда вы будете готовы приступить к работе с AIR, см. раздел [автоматического изучения угроз и реагирования на них.](office-365-air.md)

## <a name="at-a-high-level"></a>На высоком уровне

При срабатывании оповещений подразумевается, что схемы безопасности вступают в силу. В зависимости от ситуации может [начинаться автоматизированный процесс расследования.](https://docs.microsoft.com/microsoft-365/security/office-365-security/office-365-air) Во время и после автоматического исследования [рекомендуется выполнить](air-remediation-actions.md) действия по исправлению. В Office 365 Advanced Threat Protection никаких действий не предпринимаются автоматически. Ваша группа операций безопасности проверяет и затем [утверждает или отклоняет каждое действие по исправлению.](air-review-approve-pending-completed-actions.md) После утверждения или отклонения всех действий, следующих за проведенным исследованием, завершается анализ. Все эти действия отслеживаются и отображаются в Центре безопасности & требованиям (см. [подраздел "Просмотр сведений об исследовании").](air-view-investigation-results.md#view-details-of-an-investigation)

В следующих разделах представлены дополнительные сведения об оповещениях, планбегах безопасности и примерах AIR в действии.

## <a name="alerts"></a>Оповещения

[Оповещения представляют](../../compliance/alert-policies.md#viewing-alerts) триггеры рабочих процессов группы операций безопасности для реагирования на инциденты. Определение приоритетного набора оповещений для анализа угроз, не имеющих проблем. При выполнении расследований оповещений вручную группы безопасности должны заслуживать и коррелировать объекты (например, контент, устройства и пользователи) под угрозами. Такие задачи и рабочие процессы могут отличаться очень много временем и затрачивать несколько средств и систем. Благодаря AIR автоматизируются анализ угроз безопасности и реагирование на них, автоматически приводя к появления ключевых средств безопасности и оповещений об управлении угрозами. 

В настоящее время для автоанализировщика угроз и речи для неоднорангового анализа угроз исследований создаются предупреждения, созданные по следующим типам политик предупреждений.  

- обнаружен щелчок потенциально вредоносного URL-адреса;
- Сообщение электронной почты, сообщаемое пользователем как фишинговое письмо`*`
- Сообщения электронной почты, содержащие вредоносные программы, были удалены после доставки`*`
- Сообщения электронной почты с удаленными url-адресами после доставки`*`
- Обнаружены шаблоны подозрительных отправителей сообщений электронной почты`#`
- Пользователю запрещена отправка электронной почты`#`

> [!NOTE]
> Оповещениям, помеченным звездочкой ( ) в политике оповещений серьезность присваивается серьезность `*` в политиках оповещений в Центре безопасности & соответствия требованиям, при этом по электронной почте уведомления отключаются. *Informational* Уведомления по электронной почте могут быть включены через [конфигурацию политики оповещений.](../../compliance/alert-policies.md#alert-policy-settings) Оповещения, помеченные хэшем ( `#` ) — это общедоступные оповещения, связанные с общедоступными модулями предварительной версии.

Чтобы просмотреть оповещения, в Центре безопасности & соответствия требованиям выберите просмотр **Alerts**  >  **оповещений.** Выберите оповещение, чтобы просмотреть его сведения. Перейдите по ссылке **"Просмотр исследования",** чтобы перейти на соответствующий [анализ.](air-view-investigation-results.md#investigation-graph)  

> [!NOTE]
> Информационные оповещения по умолчанию скрыты в представлении оповещений. Чтобы их увидеть, измените фильтрацию оповещений, включаюв в нее информационные оповещения.

Если ваша организация управляет оповещениями системы безопасности с помощью системы управления оповещениями, системой управления службами или системы SIEM, вы можете отправлять оповещения в эту систему с помощью уведомления по электронной почте или с помощью API действий управления [Office 365.](https://docs.microsoft.com/office/office-365-management-api/office-365-management-activity-api-reference) Уведомления оповещений об исследовании через электронные письма или API содержат ссылки на доступ к оповещениям в Центре безопасности безопасности &, позволяя назначенным администраторам безопасности быстро переходить к исследованию.

![Оповещения, привязаные к исследованиям](../../media/air-alerts-page-details.png) 

## <a name="security-playbooks"></a>Схемы безопасности

Создание резервных конникровок системы безопасности — это внутренние политики, которые являются основой автоматизации в Office Advanced Threat Protection и Защите от угроз (Майкрософт). Сборники схем безопасности, предоставляемые в AIR, основаны на распространенных сценариях обеспечения реальных обеспечения безопасности и разработаны на основе отзывов групп операций по безопасности. Схемный конник системы безопасности запускается автоматически при срабатывании определенных оповещений в организации. После активации оповещения связанный playbook запускается системой автоматического анализа угроз и реагирования на них (AIR). Проанализ оповещения на основе инфографики оповещения, просматривая эти метаданные (включая сообщения электронной почты, пользователей, темы, отправителя и т. д.). На основе результатов анализа исследований AIR рекомендует набор действий, которые группа обеспечения безопасности в вашей организации может выполнить для контроля и устранения угроз. 

Конфликты схем безопасности, которые вы получите с AIR, призваны участвовать в наиболее частых угрозах, с которыми сталкиваются организации с письмами. Они основаны на входных данных, поступящих за безопасность, и группы реагирования на инциденты, включая людей, которые помогают защищать корпорацию Майкрософт и ресурсы наших клиентов.

### <a name="security-playbooks-are-rolling-out-in-phases"></a>Поэтапное внедрение схем безопасности

В рамках AIR схемы безопасности стали доступны поэтапно. Этап 1 теперь общедоступный и содержит несколько гиокниг, обеспечивающих рекомендации по действиям, которые администраторы безопасности могут просмотреть и утвердить:

- Сообщение фишинга, о которых было отправлено пользователем
- Изменение запроса щелчка по URL-адресу
- После доставки вредоносное ПО (Зу, несанкционированное вредоносное ПО)
- Фишинговый (фишинговый zAP) после доставки

Этап 1 включает также поддержку исследования электронной почты, запущенной администратором (при [использовании Threat Explorer).](threat-explorer.md)

Этап 2 теперь проходит с указанных ниже схемных модулей в **общедоступной предварительной версии,** предоставляет рекомендации по действиям, а администраторам безопасности при изучении проблем администраторы безопасности:

- Пользователь сообщил о скомпрометированной (общедоступная предварительная версия)

По завершении этих схем освобождаются дополнительные конники схем. В таблице Microsoft 365 Roadmap посетите сайт Microsoft [365 Roadmap,](https://www.microsoft.com/microsoft-365/roadmap) чтобы узнать о планировке и в ближайшее поле.

### <a name="playbooks-include-investigation-and-recommendations"></a>Книжки схем включают исследование и рекомендации

В AIR каждый из схем безопасности включает следующее: 

- корневое исследование объектов электронной почты (файлов, URL-адресов, получателей, IP-адресов и т. д.);
- дальнейший охват похожих электронных писем, полученных организацией 
- предпринимаемые действия по выявлению и соотнесению других потенциальных угроз; 
- Рекомендуемые действия по исправлению угроз.

Каждый высокоуровневый этап включает ряд подэтапов, которые позволяют предоставить подробный, подробный и исчерпывающий реакции на угрозы.

## <a name="example-a-user-reported-phish-message-launches-an-investigation-playbook"></a>Пример. Сообщение, о которых сообщается о фишинговом пользователе, запускает журнал исследования

Предположим, пользователь вашей организации получает письмо о том, что думается попытка фишинга. Пользователь, обученный отправке таких сообщений, с помощью [надстройки Report Message](enable-the-report-message-add-in.md) отправляет их в корпорацию Майкрософт для анализа. Отправка также отправляется в систему и отображается в проводнике **в представлении отправок** (в предыдущем называвшемся **представлением о пользователе).** Кроме того, сообщение, отправляемое пользователем, теперь вызывает системное информационное сообщение, которое автоматически запускает журнал исследования.

На этапе расследования будут проверены различные аспекты сообщения электронной почты. К этим аспектам относятся:

- Определение возможных типов угроз.
- Кто отправил его;
- откуда отправлялось сообщение (инфраструктура отправки);
- указывает, доставлено или заблокировано ли в прочатке сообщение электронной почты;
- Оценка аналитиков;
- связано ли электронное письмо с какими-либо известными кампаниями;
- и другие.

После завершения корневого исследования схема создаст список рекомендуемых действий, которые необходимо выполнить к исходному сообщению электронной почты и сущностям, которые он связан.
  
Далее выполняются несколько действий по анализу угроз и выслеживанию угроз:

- Аналогичные сообщения электронной почты определяются при поиске кластера электронной почты.
- Сигнал совместно используется с другими платформами, такими как [ATP в Microsoft Defender.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/microsoft-defender-advanced-threat-protection)
- Определение определяется тем, прошедшие ли какие-либо пользователи по вредоносным ссылкам в подозрительных сообщениях электронной почты.
- Проверка проводится в Exchange Online Protection[(EOP) и](exchange-online-protection-overview.md)Office 365 Advanced Threat Protection[(ATP), чтобы](office-365-atp.md)узнать, есть ли другие сообщения, похожие пользователям.
- Проверка выполнится, чтобы узнать, взломан ли пользователь. Эта проверка использует сигналы в Office 365, [Microsoft Cloud App Security](https://docs.microsoft.com/cloud-app-security)и Azure Active [Directory,](https://docs.microsoft.com/azure/active-directory)что сопоставляет любые связанные действия пользователей.

На этапе выслеживания угрозы и риски назначаются разным этапам выслеживания. 

Исправление — это последний этап модулей схемы. На этом этапе выполняется действия по исправлению, основанные на фазах расследования и охоты. 

## <a name="example-a-security-administrator-triggers-an-investigation-from-threat-explorer"></a>Пример. Администратор безопасности инициирует исследование из обозревателя угроз

Помимо автоматизированного исследования, которые запускаются оповещением, ваша группа операций по обеспечению безопасности в организации может запустить автоматизированное исследование [из представления в Обозревателе угроз.](threat-explorer.md)

Например, предположим, что вы используете представление вредоносных программ **в** обозревателе угроз. На вкладках под диаграммой выберите вкладку **"Электронная почта".** При выборе одного или нескольких элементов в списке **активируется кнопка "+Actions".** 

![Проводник с выбранными сообщениями](../../media/Explorer-Malware-Email-ActionsInvestigate.png)

В меню **"Действия"** можно выбрать функцию **исследования Trigger.**

![Меню действий для выбранных сообщений](../../media/explorer-malwareview-selectedemails-actions.jpg)

Как и к модулям инноваций, активированным оповещением, автоматические исследования, которые активируются из представления в проводнике, включают в себя корневой анализ угроз, действия по выявлению и корреляции угроз, а также рекомендуемые действия для устранения этих угроз.

## <a name="example-a-security-operations-team-integrates-air-with-their-siem-using-the-office-365-management-activity-api"></a>Пример. Группа операций безопасности интегрирует со своим SIEM с помощью API действий управления Office 365

Возможности AIR Office 365 ATP [включают отчеты & сведения,](air-view-investigation-results.md) которые группа операций безопасности может использовать для отслеживания и устранения угроз. Однако с другими решениями можно также интегрировать возможности одного и токена. В качестве примеров можно привелось добавить в список систему управления информационной безопасностью и событиями (SIEM), систему управления обращениями или настраиваемое решение для создания отчетов. Эти виды интеграции можно выполнить с помощью [API действий управления Office 365.](https://docs.microsoft.com/office/office-365-management-api/office-365-management-activity-api-reference) 

Например, организация настроила способ, по звонящему их группе операций безопасности, просматривать предупреждения фишинга, о которых сообщали пользователи, которые уже были обработаны AIR. Решение интегрирует соответствующие оповещения с сервером SIEM организации и системой управления обращениями. Решение значительно уменьшает количество ложных срабатываний, чтобы группа их операций по обеспечению безопасности могла сконцентрировать свое время и усилия на реальных угрозах. Дополнительные сведения об этом настраиваемом решении см. в [блоге Tech Community: повышение эффективности soC с помощью Office 365 ATP и API управления O365.](https://techcommunity.microsoft.com/t5/microsoft-security-and/improve-the-effectiveness-of-your-soc-with-office-365-atp-and/ba-p/1525185)
