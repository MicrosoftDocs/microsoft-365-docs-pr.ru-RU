# <a name="zero-hour-auto-purge-zap-in-exchange-online"></a>Автоматическая очистка (ZAP) в Exchange Online

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


## <a name="basic-features-of-zap"></a>Основные функции ZAP

В организациях Microsoft 365 с почтовыми ящиками в Exchange Online автоматическая очистка (ZAP) — это функция защиты электронной почты, которая автоматически обнаруживает и нейтрализует вредоносные фишинговые сообщения, спам или вредоносные сообщения, которые уже были доставлены в почтовые ящики Exchange Online.

ZaP не работает в автономных средах Exchange Online Protection (EOP), которые защищают почтовые ящики локальной среды Exchange.

## <a name="how-zap-works"></a>Как работает ZAP

Сигнатуры нежелательной почты и вредоносных программ обновляются в службе в режиме реального времени ежедневно. Однако пользователи по-прежнему могут получать вредоносные сообщения по различным причинам, в том числе по причине иконки контента после его доставки пользователям. ZaP решает эту проблему, постоянно отслеживая обновления сигнатур нежелательной почты и вредоносных программ в службе. ZaP может находить и удалять сообщения, которые уже находятся в почтовом ящике пользователя.

Действие ZAP не является удобным для пользователя; они не будут уведомлены об обнаружении и перемещении сообщения.

[Списки надежных отправитель,](create-safe-sender-lists-in-office-365.md)правила потока почты (также известные как правила транспорта), правила для входящих сообщений или дополнительные фильтры имеют приоритет над ZAP. Аналогично тому, что происходит в потоке почты, это означает, что даже если служба определяет, что доставленное сообщение нуждается в ZAP, это сообщение не будет действовать из-за конфигурации надежных отправителей. Это еще одна причина, по которой следует быть осторожными при настройке сообщений для обхода фильтрации.

### <a name="malware-zap"></a>ZaP вредоносных программ

Для **прочитанные или непрочитанные** сообщения, которые содержат вредоносную программу после доставки, zap занося в карантин сообщение, содержающее вложение вредоносной программы. Только администраторы могут просматривать вредоносные сообщения из карантина и управлять ими.

ZaP вредоносных программ включен по умолчанию в политиках для борьбы с вредоносными программами. Дополнительные сведения см. в подстройке "Настройка политик для борьбы с [вредоносными программами" в EOP.](configure-anti-malware-policies.md)

### <a name="phish-zap"></a>Фишинговая zap

Для  прочитанные или непрочитанные сообщения, которые определены как фишинговые после доставки, результат ZAP  зависит от действия, настроенного для решения фильтрации фишинговых сообщений электронной почты в соответствующей политике защиты от нежелательной почты. Доступные действия по решению фильтра для фишинга и их возможные результаты zaP описаны в следующем списке:

- **Добавьте X-загон,** добавьте строку темы в предзапись **текстом:** ZAP не делает никаких действий над сообщением.

- **Перемещение сообщения в** папку нежелательной почты: ZAP перемещает сообщение в папку нежелательной почты, если для почтового ящика включено правило нежелательной почты (оно включено по умолчанию). Дополнительные сведения см. в подстройке "Настройка параметров нежелательной почты" для почтовых ящиков [Exchange Online в Microsoft 365.](configure-junk-email-settings-on-exo-mailboxes.md)

- **Перенаправление сообщения на адрес электронной** почты, **удаление** сообщения , сообщение **карантина**: ZAP карантин сообщения.

По умолчанию фишинговая очистка включена в политиках защиты  от нежелательной почты, а действием по умолчанию для решения фильтрации фишинговых сообщений является сообщение карантина, которое означает, что фишинговое сообщение по умолчанию находится в карантине.

Дополнительные сведения о настройке решения фильтрации нежелательной почты см. в подстройке "Настройка политик борьбы с нежелательной [почтой" в Microsoft 365.](configure-your-spam-filter-policies.md)

### <a name="spam-zap"></a>ZaP нежелательной почты

Для **непрочитанные** сообщения, которые определены как спам после доставки, результат ZAP зависит  от действия, настроенного для решения фильтра нежелательной почты в применимой политике борьбы с нежелательной почтой. Доступные действия по решению фильтра для нежелательной почты и их возможные результаты zap описаны в следующем списке:

- **Добавьте X-загон,** добавьте строку темы в предзапись **текстом:** ZAP не делает никаких действий над сообщением.

- **Перемещение сообщения в** папку нежелательной почты: ZAP перемещает сообщение в папку нежелательной почты, если для почтового ящика включено правило нежелательной почты (оно включено по умолчанию). Дополнительные сведения см. в подстройке "Настройка параметров нежелательной почты" для почтовых ящиков [Exchange Online в Microsoft 365.](configure-junk-email-settings-on-exo-mailboxes.md)

- **Перенаправление сообщения на адрес электронной** почты, **удаление** сообщения , сообщение **карантина**: ZAP карантин сообщения. Конечные пользователи могут просматривать и управлять своими сообщениями в карантине нежелательной почты.

По умолчанию zap нежелательной почты включен в политиках борьбы  с нежелательной почтой, а действие по умолчанию для  решения фильтра нежелательной почты — переместить сообщение в папку нежелательной почты. Это означает, что эта политика перемещает непрочитанные сообщения в папку нежелательной почты по умолчанию.

Дополнительные сведения о настройке решения фильтрации нежелательной почты см. в подстройке "Настройка политик борьбы с нежелательной [почтой" в Microsoft 365.](configure-your-spam-filter-policies.md)

### <a name="zap-considerations-for-microsoft-defender-for-office-365"></a>Вопросы zap для Microsoft Defender для Office 365

ZaP не будет карантить сообщения, которые [](atp-safe-attachments.md#dynamic-delivery-in-safe-attachments-policies) находится в процессе динамической доставки при сканировании безопасных вложений, или если фильтрация вредоносных программ EOP уже заменила вложение файлом оповещения о вредоносных программах **Text.txt.** Если для этих типов сообщений получен сигнал фишинга или нежелательной почты, а решение фильтра в политике защиты от нежелательной почты настроено на какое-либо действие с сообщением (перемещение в папку нежелательной почты, перенаправление, удаление или карантин), то zap по умолчанию будет принимать действие "Переместить в нежелательное".

## <a name="how-to-see-if-zap-moved-your-message"></a>How to see if ZAP moved your message

Чтобы определить, переместила ли zap ваше [](view-email-security-reports.md#threat-protection-status-report) сообщение, можно воспользоваться отчетом о состоянии защиты от угроз или обозревателем угроз (и обнаружением в [режиме реального времени).](threat-explorer.md) Обратите внимание, что в системном действии zap не регистрируется в журналах аудита почтовых ящиков Exchange.

## <a name="zap-faq"></a>ZAP FAQ

### <a name="what-happens-if-a-legitimate-message-is-moved-to-the-junk-email-folder"></a>Что произойдет, если действительное сообщение будет перемещено в папку нежелательной почты?

Следует следовать обычному процессу создания отчетов о [ложных срабатываях.](report-junk-email-messages-to-microsoft.md) Единственная причина, по которой сообщение перемещается из папки "Входящие" в папку нежелательной почты, заключается в том, что служба определила, что сообщение нежелательное или вредоносное.

### <a name="what-if-i-use-the-quarantine-folder-instead-of-the-junk-mail-folder"></a>Что делать, если я использую папку карантина вместо папки нежелательной почты?

ZaP будет принимать меры к сообщению в зависимости от конфигурации ваших политик борьбы с нежелательной почтой, как описано ранее в этой статье.

### <a name="what-if-im-using-safe-senders-mail-flow-rules-or-allowedblocked-sender-lists"></a>Что делать, если я использую надежных отправителей, правила потока почты или списки разрешенных и заблокированных отправителей?

Приоритет имеют надежные отправитые, правила потока почты или блокировка и разрешение параметров организации. Эти сообщения исключаются из zap, так как служба делает то, что вы настроили. Это еще одна причина, по которой следует быть осторожными при настройке сообщений для обхода фильтрации.

### <a name="what-if-a-message-is-moved-to-another-folder-eg-inbox-rules"></a>Что делать, если сообщение перемещается в другую папку (например, правила для папки "Входящие")?

ZaP по-прежнему работает до тех пор, пока сообщение не было удалено или пока не применено то же или более сильное действие. Например, если политика защиты от фишинга настроена на карантин, а сообщение уже находится в нежелательной почте, то zap примет действие для карантина сообщения.

### <a name="how-does-zap-affect-mailboxes-on-hold"></a>Как ZAP влияет на почтовые ящики на удержании?

ZaP не будет карантить сообщения из почтовых ящиков на удержании. ZaP может перемещать сообщения в папку нежелательной почты на основе действия, настроенного для решения о нежелательной почте или фишинге в политиках защиты от нежелательной почты.

Дополнительные сведения о удержаниях в Exchange Online см. в сведениях об удержании на месте и удержании [для судебного разбирательства в Exchange Online.](https://docs.microsoft.com/Exchange/security-and-compliance/in-place-and-litigation-holds)
