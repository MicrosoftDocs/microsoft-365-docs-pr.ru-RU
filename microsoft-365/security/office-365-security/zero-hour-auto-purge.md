# <a name="zero-hour-auto-purge-zap-in-exchange-online"></a>Автоматическая очистка (ZAP) в Exchange Online

## <a name="basic-features-of-zap"></a>Основные возможности ZAP

В организациях Microsoft 365 с почтовыми ящиками в Exchange Online автоматическая очистка (ZAP) — это функция защиты электронной почты, которая позволяет обнаруживать и нейтрально получать фишинговые сообщения, спам или вредоносные сообщения, уже доставленные в почтовые ящики Exchange Online.

Зумне не работает в автономных средах Exchange Online Protection (EOP), защищающих локальные почтовые ящики Exchange.

## <a name="how-zap-works"></a>Как работает ZAP

Сигнатуры нежелательной почты и вредоносных программ ежедневно обновляются в режиме реального времени. При этом пользователи по-прежнему могут получать вредоносные сообщения по различным причинам, включая добавление описания вывода содержимого в оружие и их доставку пользователям. ZAP устраняет эту проблему, постоянно проследивая обновление сигнатур нежелательной почты и вредоносных программ в службе. Этот метод поддерживает поиск и удаление сообщений, уже найденных в почтовом ящике пользователя.

Действие ZAP не прозрачно для пользователя; они не будут уведомлены о обнаружении и перемещении сообщения.

[Списки надежных отправителей,](create-safe-sender-lists-in-office-365.md)правила для потока обработки почты (также называемые правилами транспорта), правила для папки "Входящие" или дополнительные фильтры имеют приоритет над ZAP. Как и в потоке обработки почты, это означает, что даже если служба определяет запрос ZAP, сообщение не будет действовать из-за конфигурации надежных отправителей. Это еще одно, по которой будьте взроскими при настройке сообщений для обхода фильтрации.

### <a name="malware-zap"></a>Malware ZAP

Для **прочитанных или непрочитанных** сообщений, которые содержат вредоносную программу после доставки, замещает сообщение, содержащее вредоносные вложения. Только администраторы могут просматривать сообщения о вредоносных программах, помещенные на карантин, и управлять ими.

Функция ZAP вредоносного ПО по умолчанию включена в политиках защиты от вредоносных программ. Дополнительные сведения см. в статье ["Настройка политик защиты от вредоносных программ" в EOP.](configure-anti-malware-policies.md)

### <a name="phish-zap"></a>Фишинговый (Фишинговый зэс

Для **прочитанных или непрочитанных** сообщений, которые определяются как фишинговые после доставки, результат ZAP зависит от действия, настроенного для решения о фильтрации фишинговых сообщений, поступающих в действующую политику защиты от нежелательной почты. **Phishing email** Доступные действия с условиями фильтрации для фишинговых сообщений и их возможные результаты zAP описаны в следующем списке:

- **Добавлять X-заголовок,** в **комплект темы текст**"Фагковая проверка подлинности" не выполняет никаких действий с сообщением.

- **Перемещение сообщения в папку "Нежелательная**почта": Пользователь Перемещает сообщение в папку "Нежелательная почта", если в почтовом ящике включено правило нежелательной почты (оно включено по умолчанию). Дополнительные сведения см. в статье ["Настройка параметров нежелательной почты для почтовых ящиков Exchange Online в Microsoft 365".](configure-junk-email-settings-on-exo-mailboxes.md)

- **Сообщение на электронный адрес, удаление**сообщения, **сообщение карантина:** Замещает сообщение в карантин. **Quarantine message**

По умолчанию в политиках защиты от нежелательной почты включена **Phishing email** функция ZAP, а действием по умолчанию для среды защиты от фишинговых сообщений по умолчанию является **сообщение "Карантин".** Это означает, что сообщение по умолчанию по умолчанию помечает сообщение в фишинговой программе фишинговой программы.

Дополнительные сведения о настройке запросов фильтров нежелательной почты см. в статье ["Настройка политик защиты от нежелательной почты" в Microsoft 365.](configure-your-spam-filter-policies.md)

### <a name="spam-zap"></a>Замечания нежелательной почты

Что **касается** нежелательных сообщений после доставки, результат ZAP зависит от действий, настроенных для решения фильтров нежелательной почты, которые включены в соответствующие политики защиты от нежелательной почты. **Spam** Доступные действия затрачивания при фильтрации для нежелательной почты и их возможные результаты выполнения описаны в следующем списке:

- **Добавлять X-заголовок,** в **комплект темы текст**"Фагковая проверка подлинности" не выполняет никаких действий с сообщением.

- **Перемещение сообщения в папку "Нежелательная**почта": Пользователь Перемещает сообщение в папку "Нежелательная почта", если в почтовом ящике включено правило нежелательной почты (оно включено по умолчанию). Дополнительные сведения см. в статье ["Настройка параметров нежелательной почты для почтовых ящиков Exchange Online в Microsoft 365".](configure-junk-email-settings-on-exo-mailboxes.md)

- **Сообщение на электронный адрес, удаление**сообщения, **сообщение карантина:** Замещает сообщение в карантин. **Quarantine message** Конечные пользователи могут просматривать собственные сообщения, помещенные на карантин нежелательной почты, и управлять ими.

По умолчанию Проверка нежелательной почты включена в политиках **Spam** защиты от нежелательной почты, и действием по умолчанию для **unread** решения фильтра нежелательной почты является перемещение **сообщений в папку нежелательной почты.** Это означает, что непрочитанные сообщения перемещаются в папку нежелательной почты по умолчанию.

Дополнительные сведения о настройке запросов фильтров нежелательной почты см. в статье ["Настройка политик защиты от нежелательной почты" в Microsoft 365.](configure-your-spam-filter-policies.md)

### <a name="zap-considerations-for-office-365-advanced-threat-protection-office-365-atp"></a>Рекомендации по ZAP для Office 365 Advanced Threat Protection (Office 365 ATP)

Этот алгоритм не будет помещать на карантин сообщения, которые надо помещать в процессе сканирования динамической доставки, или если при фильтрации вредоносных программ уже было заменено [Dynamic Delivery](dynamic-delivery-and-previewing.md) **вложение файлом Text.txtвредоносных** программ. Если для этих типов сообщений получен сигнал фишинга или спама, а для решений фильтров в политике защиты от нежелательной почты установлено выполнение каких-либо действий с сообщением (перемещение в папку "Нежелательное", "Перенаправление", "Удаление" и "Карантин"), то по умолчанию узел ZAP поумолчани использует действие "Переместить в папку нежелательной почты".

## <a name="how-to-see-if-zap-moved-your-message"></a>Как узнать, было ли перемещено ли сообщение чаше

Чтобы определить, было ли ZAP перемещено сообщение, можно использовать отчет [о](view-email-security-reports.md#threat-protection-status-report) состоянии защиты от угроз [или обозреватель угроз.](threat-explorer.md) Обратите внимание, что в качестве системного действия ZAP не заносится в журнал аудита почтовых ящиков Exchange.

## <a name="zap-faq"></a>Вопросы и ответы

### <a name="what-happens-if-a-legitimate-message-is-moved-to-the-junk-email-folder"></a>Что произойдет, если обыктические сообщения перемещаются в папку нежелательной почты?

При проверке ложных срабатываний следует выполнить [обычный процесс создания отчетов.](report-junk-email-messages-to-microsoft.md) Из папки "Входящие" в папку нежелательной почты из-за этого служба определила, что сообщение было спамом или вредоносным.

### <a name="what-if-i-use-the-quarantine-folder-instead-of-the-junk-mail-folder"></a>Что делать, если вместо папки нежелательной почты используется папка "Карантин"?

Этот сервер будет выполнять действия в зависимости от конфигурации политик защиты от нежелательной почты, как описано ранее в этом разделе.

### <a name="what-if-im-using-safe-senders-mail-flow-rules-or-allowedblocked-sender-lists"></a>Что делать, если используются надежные отправители, правила для потока обработки почты, а также списки разрешенных и заблокированных отправителей?

Надежные отправители, правила для потока обработки почты или блокировка, а параметры организации имеют приоритет. Эти сообщения исключаются из ZAP, так как служба делает настроенные ею действия. Это еще одно, по которой будьте взроскими при настройке сообщений для обхода фильтрации.

### <a name="what-if-a-message-is-moved-to-another-folder-eg-inbox-rules"></a>Что делать, если сообщение перемещается в другую папку (например, правила для папки "Входящие")?

Этот сервер работает, если сообщение не было удалено (или более корец) действие еще не применено. Например, если политика фишинга настроена на карантин, а пользователь или администратор уже спам карантине сообщения, к карантин будет выполнять действия по карантину.

### <a name="how-does-zap-affect-mailboxes-on-hold"></a>Как ZAP повлияет на почтовые ящики на хранении?

Зумма, сообщения из почтовых ящиков на хранении не будут помещаться в карантин. ZAP может перемещать сообщения в папку нежелательной почты согласно действию, настроенному для нежелательной почты или решения фишинга, заключающихся в политиках защиты от нежелательной почты.

Дополнительные сведения о режимах хранения в Exchange Online см. в статье "Хранение на месте" и "Хранение [для судебного разбирательства" в Exchange Online.](https://docs.microsoft.com/Exchange/security-and-compliance/in-place-and-litigation-holds)
