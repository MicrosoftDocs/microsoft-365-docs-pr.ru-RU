# <a name="configure-your-microsoft-365-tenant-for-increased-security"></a>Настройка клиента Microsoft 365 для повышения безопасности

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


В этом разделе описывается рекомендуемая конфигурация параметров на уровне клиента, влияющих на безопасность среды Microsoft 365. Для обеспечения безопасности может потребоваться больше или меньше безопасности. Используйте эти рекомендации в качестве отправной точки.

## <a name="check-office-365-secure-score"></a>Проверка показателя безопасности Office 365

Office 365 Secure Score анализирует безопасность Организации в соответствии с обычными действиями и параметрами безопасности и назначает оценку. Начните с заметок о текущем показателе. Настройка некоторых параметров на уровне клиента приводит к увеличению оценки. Цель не допустить максимальной оценки, но знать о возможностях защиты среды, которая не оказывает негативного влияния на производительность пользователей. Просмотр [оценки безопасности Майкрософт](../mtp/microsoft-secure-score.md).

## <a name="configure-tenant-wide-sharing-policies-in-sharepoint-admin-center"></a>Настройка политик общего доступа на уровне клиента в центре администрирования SharePoint

Корпорация Майкрософт предлагает рекомендации по настройке сайтов группы SharePoint при повышении уровня защиты, начиная с базовой защиты. Дополнительные [сведения см.](https://docs.microsoft.com/microsoft-365-enterprise/secure-sharepoint-online-sites-and-files)

Сайты групп SharePoint, настроенные на базовом уровне, позволяют предоставлять общий доступ к файлам внешним пользователям с помощью ссылок анонимного доступа. Этот подход рекомендуется вместо отправки файлов в сообщении электронной почты.

Чтобы обеспечить поддержку задач базовой защиты, настройте политики общего доступа на уровне клиента, как это рекомендовано. Параметры общего доступа к отдельным сайтам могут быть более строгими, чем эта политика на уровне клиента, но не более строгая.

****

|Область|Включает политику по умолчанию|Рекомендация|
|---|---|---|
|**Общий доступ** (SharePoint Online и OneDrive для бизнеса)|Да|Внешний общий доступ включен по умолчанию. Рекомендуется использовать следующие параметры: <br/>* Разрешить общий доступ к прошедшему проверку подлинности внешним пользователям и использовать ссылки анонимного доступа (по умолчанию). <br/> * Срок действия ссылок для анонимного доступа истечет через это количество дней. При необходимости введите число (например, 30 дней). <br/>* Тип ссылки по умолчанию — выберите internal (только для пользователей в организации). Пользователи, которые хотят поделиться с помощью анонимных ссылок, должны выбрать этот параметр в меню Общий доступ. <br/><br/> Дополнительные сведения: [Обзор внешнего общего доступа](https://docs.microsoft.com/sharepoint/external-sharing-overview)|

## <a name="configure-settings-in-azure-active-directory"></a>Настройка параметров в Azure Active Directory

Не забудьте посетить эти две области в Azure Active Directory, чтобы выполнить настройку на уровне клиента для более безопасных сред.

### <a name="configure-named-locations-under-conditional-access"></a>Настройка именованных расположений (в разделе условный доступ)

Если в Организации есть офисы с защищенным доступом к сети, добавьте диапазоны надежных IP-адресов в Azure Active Directory в качестве именованных расположений. Эта функция позволяет сократить количество ложных срабатываний ложных срабатываний для событий риска входа.

См.: [именованные расположения в Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-named-locations)

### <a name="block-apps-that-dont-support-modern-authentication"></a>Блокировка приложений, не поддерживающих современные проверки подлинности

Многофакторная проверка подлинности требует наличия приложений, поддерживающих современные проверки подлинности. Приложения, не поддерживающие современные проверки подлинности, не могут блокироваться с помощью правил условного доступа.

Для безопасных сред обязательно отключите проверку подлинности для приложений, которые не поддерживают современные проверки подлинности. Это можно сделать в Azure Active Directory с помощью элемента управления, который скоро появится.

Чтобы сделать это для SharePoint Online и OneDrive для бизнеса, воспользуйтесь одним из указанных ниже способов.

- С помощью PowerShell можно просмотреть раздел [блокировать приложения, для которых не используется современная проверка подлинности (ADAL)](https://docs.microsoft.com/mem/intune/protect/app-modern-authentication-block).

- Настройте этот параметр в центре администрирования SharePoint на странице "доступ к устройству" — "Управление доступом из приложений, для которых не используется современная проверка подлинности". Выберите пункт блокировать.
