# <a name="configure-your-microsoft-365-tenant-for-increased-security"></a>Настройка клиента Microsoft 365 для повышения безопасности

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Exchange Online Protection](exchange-online-protection-overview.md)
- [Microsoft Defender для Office 365 (план 1 и план 2)](office-365-atp.md)
- [Microsoft 365 Defender](../mtp/microsoft-threat-protection.md)

В этом разделе вы можете рекомендовать настройку параметров на всем клиенте, влияющих на безопасность среды Microsoft 365. Ваши потребности в безопасности могут требовать большей или меньшей безопасности. Используйте эти рекомендации в качестве отправной точки.

## <a name="check-office-365-secure-score"></a>Проверка оценки безопасности Office 365

Оценка безопасности Office 365 анализирует безопасность организации на основе обычных действий и параметров безопасности и назначает оценку. Начните с заметок о текущей оценке. Настройка некоторых параметров на клиенте увеличит ваш показатель. Цель состоит не в достижении максимального результата, а в том, чтобы иметь возможность защитить среду, которая не отрицательно влияет на производительность пользователей. См. ["Оценка безопасности (Майкрософт)".](../mtp/microsoft-secure-score.md)

Сайты группы SharePoint, настроенные на базовом уровне, позволяют обмениваться файлами с внешними пользователями с помощью ссылок анонимного доступа. Этот подход рекомендуется вместо отправки файлов по электронной почте.

Для поддержки целей базовой защиты настройте политики общего доступа на уровне клиента, как рекомендуется в этом разделе. Параметры общего доступа для отдельных сайтов могут быть более строгими, чем эта политика на протяжении всего клиента, но не более строгими.

## <a name="configure-settings-in-azure-active-directory"></a>Настройка параметров в Azure Active Directory

Не забудьте посетить эти две области в Azure Active Directory, чтобы завершить настройку в масштабе клиента для более безопасных сред.

### <a name="configure-named-locations-under-conditional-access"></a>Настройка именуемых местоположений (при условном доступе)

Если ваша организация включает офисы с защищенным сетевым доступом, добавьте надежные диапазоны IP-адресов в Azure Active Directory в качестве именуемого расположения. Эта функция помогает уменьшить количество ложных срабатывавай при событиях риска при входе.

See: [Named locations in Azure Active Directory](https://docs.microsoft.com/azure/active-directory/active-directory-named-locations)

### <a name="block-apps-that-dont-support-modern-authentication"></a>Блокировать приложения, которые не поддерживают современную проверку подлинности

Для многофакторной проверки подлинности требуются приложения, которые поддерживают современную проверку подлинности. Приложения, которые не поддерживают современную проверку подлинности, не могут быть заблокированы с помощью правил условного доступа.

В безопасных средах отключать проверку подлинности для приложений, которые не поддерживают современную проверку подлинности. Это можно сделать в Azure Active Directory с помощью скорого управления.

В то же время используйте один из следующих методов для выполнения этой задачи для SharePoint Online и OneDrive для бизнеса:

- Используйте PowerShell, см. блокирование приложений, которые не используют современную проверку подлинности [(ADAL).](https://docs.microsoft.com/mem/intune/protect/app-modern-authentication-block)

- Настройте его в Центре администрирования SharePoint на странице "Доступ устройств" — "Управление доступом из приложений, которые не используют современную проверку подлинности". Выберите "Блокировать".

## <a name="get-started-with-cloud-app-security-or-office-365-cloud-app-security"></a>Начало работы с Cloud App Security или Office 365 Cloud App Security

Используйте Office 365 Cloud App Security для оценки риска, оповещения о подозрительных действиях и автоматического принятия мер. Требуется план Office 365 E5.

Или используйте Microsoft Cloud App Security, чтобы получить более глубокую видимость даже после предоставления доступа, комплексных элементов управления и улучшенной защиты для всех облачных приложений, включая Office 365.

Так как это решение рекомендует план EMS E5, рекомендуем начать с Cloud App Security, чтобы использовать его с другими приложениями SaaS в вашей среде. Начните с политик и параметров по умолчанию.
