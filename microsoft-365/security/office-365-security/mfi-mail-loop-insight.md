# <a name="fix-possible-mail-loop-insight-in-the-security--compliance-center"></a>Устранение возможных статистики почтового цикла в Центре безопасности & соответствия требованиям

Почтовый цикл недопустим, поскольку он препятствует системным ресурсам, потребляет квоту на объем почтовых ресурсов организации и делает путаницу неудобств теми отправителями.

Эта **информация о возможной проблеме** с почтовым циклом в **рекомендуемой** панели мониторинга [потока обработки](mail-flow-insights-v2.md) почты в Центре безопасности & уведомляет о обнаружении почтового цикла в организации. Эта аналитика появляется только после определения условия (если у вас нет почтовых циклов, аналитика не будет отображаться).

![Устранение медленных результатов правил потока обработки почты в рекомендуемой области панели мониторинга потока обработки почты](../../media/mfi-fix-possible-mail-loop.png)

Когда вы **щелкаете** "Просмотр сведений" в мини-приложении, появится всплывающая область с дополнительной информацией:

- **Домен**
- **Количество сообщений:** можно нажать кнопку **«Просмотреть пример сообщений»,** чтобы просмотреть результаты трассировки для образца сообщений, затронутых циклом. [message trace](message-trace-scc.md)
- **Тип домена**" Например, "Доверенное" или "Незаслуживающие доверия".
- **Запись MX:** узел **(почтовый сервер)** **и значения приоритета** для записи MX для домена.
- **Причина цикла** и **рекомендации:** мы пытаемся определить наиболее распространенные сценарии почтового цикла и предоставить рекомендуемые действия (если они доступны).

![Всплывающий элемент "Подробности", который отображается после нажатия элемента "Просмотр сведений" в разделе "Исправление проблемы" из почтового цикла](../../media/mfi-fix-possible-mail-loop-details.png)
