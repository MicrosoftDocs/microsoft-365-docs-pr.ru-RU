# <a name="mail-flow-intelligence-in-eop"></a>Логика обмена корреспонденцией почты в EOP

В организациях Microsoft 365 с почтовыми ящиками в Exchange Online или в автономных организациях Exchange Online Protection (EOP) без почтовых ящиков Exchange Online, как правило, соединитель используется для маршрутизации сообщений из EOP в локальную среду электронной почты. Вы также можете использовать соединитель для маршрутизации сообщений из Microsoft 365 в партнерскую организацию. Если Microsoft 365 не удается доставить сообщение через соединитель, они добавляются в очередь в Microsoft 365. Microsoft 365 продолжает попытки доставить каждое сообщение в течение 24 часов. После 24 часов срок действия сообщения в очереди истечет, а сообщение будет возвращено исходному отправителю в отчете о недоставке.

Microsoft 365 генерирует ошибку, когда сообщение не удается доставить с помощью соединителя. Наиболее распространенные ошибки и их решения описаны в этом разделе. Помещение в очередь недоставленных сообщений, отправленных через соединители, и ошибки уведомлений о них называют _логикой потока обработки почты._

## <a name="error-code-450-44312-dns-query-failed"></a>Код ошибки: 450 4.4.312 Сбой запроса DNS

Как правило, эта ошибка означает, что Microsoft 365 пытался подключиться к промежуточному узлу, указанному в соединителе, но поиск IP-адресов промежуточного узла выполнить не удалось. Возможные причины этой ошибки:

- Проблема со службой размещения DNS домена (компанией, которая обслуживает полномочные серверы доменных имен для вашего домена).

- Срок действия домена недавно истек, поэтому запись MX невозможно восстановить.

- Запись MX домена была недавно изменена, а информация о нем на DNS-серверах остается прежней.

### <a name="how-do-i-fix-error-code-450-44312"></a>Как исправить ошибку 450 4.4.312?

- Определите и устраните проблему с доменом вместе со службой размещения DNS.

- Если ошибка возникла в партнерской организации (например, сторонним поставщиком облачных услуг), обратитесь к партнеру за исправлением проблемы.

## <a name="error-code-450-44315-connection-timed-out"></a>Код ошибки: 450 4.4.315 Время ожидания подключения истекло

Как правило, это означает, что Microsoft 365 не может подключиться к конечному почтовому серверу. Описание проблемы см. в сведениях об ошибке. Например:

- Локальный почтовый сервер не доступен.

- Ошибка в параметрах промежуточного узла соединителя, поэтому Microsoft 365 пытается подключиться к некорректный IP-адрес.

### <a name="how-do-i-fix-error-code-450-44315"></a>Как исправить ошибку 450 4.4.315?

- Выясните, какой из этих пунктов относится к вам, и внесите необходимые изменения. Например, если поток обработки почты работал правильно и вы не изменяли параметры соединителя, проверьте локальную почтовую среду, чтобы узнать, не отключен ли сервер и не было ли изменений в сетевой инфраструктуре (например, изменился интернет-провайдер, поэтому теперь у вас другие IP-адреса).

- Если ошибка возникла в партнерской организации (например, сторонним поставщиком облачных услуг), обратитесь к партнеру за исправлением проблемы.

## <a name="error-code-450-44316-connection-refused"></a>Код ошибки: 450 4.4.316 В подключении отказано

Как правило, эта ошибка означает, что при попытке подключения к целевому почтовому серверу произошла ошибка. Вероятно, эта ошибка вызвана тем, что ваш брандмауэр блокирует подключения с IP-адресов Microsoft 365. Эта ошибка касается того, что если вы полностью перенесли локальную систему электронной почты в Microsoft 365 и завершили работу локальной почтовой среды.

### <a name="how-do-i-fix-error-code-450-44316"></a>Как исправить ошибку 450 4.4.316?

- Если в локальной среде есть почтовые ящики, измените настройки брандмауэра, чтобы разрешать подключения к локальным почтовым серверам с IP-адресов Microsoft 365 на TCP-порту 25. Список IP-адресов Microsoft 365 см. в статье ["URL-адреса и диапазоны IP-адресов Microsoft 365".](https://docs.microsoft.com/office365/enterprise/urls-and-ip-address-ranges)

- Если вам больше не требуется доставить сообщения в локальную среду, щелкните **"Исправить" теперь** в оповещении, чтобы Microsoft 365 мог сразу отклонить сообщения с недопустимыми получателями. Это снижает риск превышения квоты организации для недопустимых получателей, что может повлиять на обычную доставку сообщений. Или вы можете воспорить следующие инструкции, чтобы устранить проблему вручную:

  - В [Центре администрирования Exchange отключите](https://docs.microsoft.com/Exchange/exchange-admin-center)соединитель, который доставляет электронную почту из Microsoft 365, в локальную среду электронной почты:

    1. В Центре администрирования Exchange перейдите к **соединителям** \> **потока обработки почты.**

    2. Выберите соединитель со **значением "От"** **Office 365,** значением в поле **"Кому"** почтового сервера вашей **организации** выполните одно из следующих действий:

       - Delete the connector by clicking the **Delete** ![ Remove icon](../../media/adf01106-cc79-475c-8673-065371c1897b.gif)

       - Отключите соединитель, нажав **значок** ![ ](../../media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif) правки и **снимите его.**

  - Измените обслуживаемый домен в Microsoft 365, который связан с локальной почтовой средой, с **внутренней ретрансляции** на **уполномоченный.** Инструкции см. в статье Управление [обслуживаемыми доменами в Exchange Online.](https://docs.microsoft.com/exchange/mail-flow-best-practices/manage-accepted-domains/manage-accepted-domains)

  **Примечание. Обычно**изменения вступили в силу от 30 минут до 1 часа. Через час проверьте, больше ли вы не видеть ошибку.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных служб), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-44317-cannot-connect-to-remote-server"></a>Код ошибки: 450 4.4.317 Не удается подключиться к удаленному серверу

Как правило, эта ошибка означает, что Microsoft 365 подсоединен к целевому почтовому серверу, но ответ сервера содержит ошибку или он не соответствует требованиям для подключения. Описание проблемы см. в сведениях об ошибке. Например:

- Ответ целевого почтового сервера с сообщением об ошибке "Служба недоступна", что означает, что сервер не может поддерживать связь с Microsoft 365.

- Соединитель настроен так, чтобы он требоходил протокол TLS, но конечный почтовый сервер не поддерживает TLS.

### <a name="how-do-i-fix-error-code-450-44317"></a>Как исправить ошибку 450 4.4.317?

- Проверьте настройки TLS и сертификаты на локальных почтовых серверах и параметры TLS на соединителе.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-44318-connection-was-closed-abruptly"></a>Код ошибки: 450 4.4.318 Соединение было внезапно прервано

Как правило, эта ошибка означает, что у Microsoft 365 возникли проблемы со связью с локальной почтовой средой, поэтому соединение было разорвано. Возможные причины этой ошибки:

- Брандмауэр использует правила проверки пакетов SMTP, и эти правила работают неправильно.

- Локальный почтовый сервер не работает правильно (например, зависания служб, сбои или низкие системные ресурсы), из-за чего время ожидания истекает и сервер закрывает подключение к Microsoft 365.

- Проблемы с сетью между локальной средой и Microsoft 365.

### <a name="how-do-i-fix-error-code-450-44318"></a>Как исправить ошибку 450 4.4.318?

- Выясните, какой из этих пунктов относится к вам, и внесите необходимые изменения.

- Если проблема вызвана проблемами с сетью между локальной средой и Microsoft 365, обратитесь к службе поддержки сети для ее устранения.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-47320-certificate-validation-failed"></a>Код ошибки: 450 4.7.320 Не удалось проверить сертификат

Как правило, эта ошибка означает, что при попытке проверить сертификат целевого почтового сервера произошла ошибка. Описание см. в сведениях об ошибке. Например:

- Срок действия сертификата истек

- Субъект сертификата не совпадает с именем узла

- Сертификат больше не действителен

### <a name="how-do-i-fix-error-code-450-47320"></a>Как исправить ошибку 450 4.7.320?

- Исправьте сертификат или настройки соединителя, чтобы можно было доставить сообщения в очереди Microsoft 365.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных служб), свяжитесь с ней, чтобы решить проблему.

## <a name="other-error-codes"></a>Коды других ошибок

Microsoft 365 не удается доставить сообщения на локальный или партнерский почтовый сервер. Используйте информацию **Конечный сервер** в сообщении об ошибке, чтобы устранить проблему в своей среде, или измените соединитель, если ошибка связана с конфигурацией.

Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.
