# <a name="mail-flow-intelligence-in-eop"></a>Логика обмена почтой в EOP

В организациях Microsoft 365 с почтовыми ящиками в организациях Exchange Online или автономной службе Exchange Online Protection (EOP) без почтовых ящиков Exchange Online, обычно используется соединитель для маршрутизации сообщений электронной почты из EOP в локальную среду электронной почты. Вы также можете использовать соединитель для маршрутизации сообщений от Microsoft 365 к партнерской организации. Когда Microsoft 365 не может доставить эти сообщения через соединитель, они помещаются в очередь в Microsoft 365. Microsoft 365 продолжит повторять доставку для каждого сообщения в течение 24 часов. Через 24 часа срок действия сообщения в очереди истечет, и сообщение будет возвращено исходному отправителю в отчете о недоставке (также называемом сообщением о недоставке или сообщении Bounce).

Microsoft 365 создает сообщение об ошибке, если сообщение не может быть доставлено с помощью соединителя. Наиболее распространенные ошибки и их решения описаны в этом разделе. Коллективно, ошибки очереди и уведомления о недоставленных сообщениях, отправляемых через соединители, называются _логикой обработки почтового процесса_.

## <a name="error-code-450-44312-dns-query-failed"></a>Код ошибки: 450 4.4.312 Сбой запроса DNS

Как правило, эта ошибка означает, что Microsoft 365 пытался подключиться к промежуточному узлу, указанному в соединителе, но DNS-запрос на поиск IP-адресов промежуточного узла окончился неудачей. Возможные причины этой ошибки:

- Проблема со службой размещения DNS домена (компанией, которая обслуживает полномочные серверы доменных имен для вашего домена).

- Срок действия домена недавно истек, поэтому запись MX невозможно восстановить.

- Запись MX вашего домена была недавно изменена, а DNS-серверы по-прежнему сохранили кэшированные данные DNS для вашего домена.

### <a name="how-do-i-fix-error-code-450-44312"></a>Как исправить ошибку с кодом 450 4.4.312?

- Работа со службой хостинга DNS для определения и устранения проблемы с доменом.

- Если ошибка связана с вашей партнерской организацией (например, сторонним поставщиком облачных служб), обратитесь к своему партнеру, чтобы устранить эту проблему.

## <a name="error-code-450-44315-connection-timed-out"></a>Код ошибки: 450 4.4.315 Время ожидания подключения истекло

Как правило, это означает, что Microsoft 365 не может подключиться к конечному серверу электронной почты. Описание проблемы см. в сведениях об ошибке. Например:

- Ваш локальный сервер электронной почты не работает.

- Произошла ошибка в параметрах промежуточного узла соединителя, поэтому Microsoft 365 пытается подключиться к неправильному IP-адресу.

### <a name="how-do-i-fix-error-code-450-44315"></a>Как исправить ошибку с кодом 450 4.4.315?

- Выясните, какой из этих пунктов относится к вам, и внесите необходимые изменения. Например, если почтовые потоки работают правильно, а параметры соединителя не были изменены, необходимо проверить локальную среду электронной почты, чтобы проверить, не работает ли сервер, или если вы изменили сетевую инфраструктуру (например, вы изменили поставщика услуг Интернета).

- Если ошибка связана с вашей партнерской организацией (например, сторонним поставщиком облачных служб), обратитесь к своему партнеру, чтобы устранить эту проблему.

## <a name="error-code-450-44316-connection-refused"></a>Код ошибки: 450 4.4.316 В подключении отказано

Как правило, эта ошибка означает, что при попытке подключения к конечному серверу электронной почты в Microsoft 365 возникла ошибка подключения. Вероятно, эта ошибка вызвана тем, что ваш брандмауэр блокирует подключения к IP-адресам Microsoft 365. Эта ошибка может быть вызвана тем, что вы полностью перенесли локальную почтовую систему в Microsoft 365 и закрываете локальную почтовую среду.

### <a name="how-do-i-fix-error-code-450-44316"></a>Как исправить ошибку с кодом 450 4.4.316?

- Если у вас есть почтовые ящики в локальной среде, необходимо изменить параметры брандмауэра, чтобы разрешить подключения с IP-адресов Microsoft 365 через TCP-порт 25 к локальным почтовым серверам. Список IP-адресов Microsoft 365 можно найти в разделе URL-адреса [и диапазоны IP-адресов microsoft 365](https://docs.microsoft.com/office365/enterprise/urls-and-ip-address-ranges).

- Если вы не хотите доставлять сообщения в локальную среду, нажмите кнопку **исправить сейчас** в окне оповещение, чтобы Microsoft 365 мог сразу же отклонить сообщения с недопустимыми получателями. Это снизит риск превышения квоты Организации для недопустимых получателей, что может повлиять на обычную доставку сообщений. Вы также можете исправить проблему вручную с помощью следующих инструкций:

  - В [центре администрирования Exchange](https://docs.microsoft.com/Exchange/exchange-admin-center)отключите или удалите соединитель, который доставляет электронную почту от Microsoft 365 в локальную почтовую среду:

    1. В центре администрирования Exchange перейдите в **Mail flow** раздел \> **соединители**поток обработки почты.

    2. Выберите соединитель со значением " **от** " **Office 365** и укажите значение в поле " **Кому** " **почтового сервера организации** и выполните одно из следующих действий:

       - Удаление соединителя путем нажатия кнопки " **Удалить** ![ значок удаления"](../../media/adf01106-cc79-475c-8673-065371c1897b.gif)

       - Отключите соединитель, нажав кнопку **изменить** ![ значок редактирования ](../../media/ebd260e4-3556-4fb0-b0bb-cc489773042c.gif) и сняв флажок **включить**.

  - Изменение обслуживаемого домена в Microsoft 365, связанного с локальной средой электронной почты, от **внутренней ретрансляции** к **удостоверяющему**. Инструкции см в разделе [Manage обслуживаемые домены в Exchange Online](https://docs.microsoft.com/exchange/mail-flow-best-practices/manage-accepted-domains/manage-accepted-domains).

  **Примечание**. как правило, для вступления в силу этих изменений требуется от 30 минут до одного часа. Через один час убедитесь, что сообщение об ошибке больше не отображается.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-44317-cannot-connect-to-remote-server"></a>Код ошибки: 450 4.4.317 Не удается подключиться к удаленному серверу

Как правило, эта ошибка означает, что Microsoft 365 подключен к конечному серверу электронной почты, но сервер ответил на немедленную ошибку или не соответствует требованиям к подключению. Описание проблемы см. в сведениях об ошибке. Например:

- Конечный сервер электронной почты выдал сообщение об ошибке "Служба недоступна", которая указывает на то, что сервер не может поддерживать связь с Microsoft 365.

- Для соединителя настроено требование TLS, но конечный сервер электронной почты не поддерживает TLS.

### <a name="how-do-i-fix-error-code-450-44317"></a>Как исправить ошибку с кодом 450 4.4.317?

- Проверьте параметры TLS и сертификаты на локальных почтовых серверах и параметры TLS на соединителе.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-44318-connection-was-closed-abruptly"></a>Код ошибки: 450 4.4.318 Соединение было внезапно прервано

Как правило, эта ошибка означает, что в Microsoft 365 возникают проблемы связи с локальной средой электронной почты, поэтому подключение было разорвано. Возможные причины этой ошибки:

- Брандмауэр использует правила проверки пакетов SMTP, и эти правила работают неправильно.

- Локальный сервер электронной почты работает неправильно (например, служба зависает или не хватает системных ресурсов), что приводит к превышению времени ожидания сервера и закрытию подключения к Microsoft 365.

- Существуют сетевые проблемы между локальной средой и Microsoft 365.

### <a name="how-do-i-fix-error-code-450-44318"></a>Как исправить ошибку с кодом 450 4.4.318?

- Выясните, какой из этих пунктов относится к вам, и внесите необходимые изменения.

- Если проблема возникает из-за проблем с сетью между локальной средой и Microsoft 365, обратитесь в группу сети, чтобы устранить эту проблему.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.

## <a name="error-code-450-47320-certificate-validation-failed"></a>Код ошибки: 450 4.7.320 Не удалось проверить сертификат

Как правило, эта ошибка означает, что при попытке проверки сертификата конечного сервера электронной почты в Microsoft 365 возникла ошибка. Описание см. в сведениях об ошибке. Например:

- Срок действия сертификата истек

- Субъект сертификата не совпадает с именем узла

- Сертификат больше не действителен

### <a name="how-do-i-fix-error-code-450-47320"></a>Как исправить ошибку с кодом 450 4.7.320?

- Исправьте сертификат или параметры на соединителе таким образом, чтобы сообщения в очереди в Microsoft 365 могли быть доставлены.

- Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных служб), свяжитесь с ней, чтобы решить проблему.

## <a name="other-error-codes"></a>Коды других ошибок

Microsoft 365 испытывает трудности при доставке сообщений на локальный или партнерский сервер электронной почты. Используйте информацию **Конечный сервер** в сообщении об ошибке, чтобы устранить проблему в своей среде, или измените соединитель, если ошибка связана с конфигурацией.

Если ошибка связана с партнерской организацией (например, сторонним поставщиком облачных услуг), свяжитесь с ней, чтобы решить проблему.
