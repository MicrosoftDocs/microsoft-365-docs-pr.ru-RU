# <a name="attack-simulator-in-microsoft-defender-for-office-365"></a>Симулятор атак в Microsoft Defender для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Применяется к** [Microsoft Defender для Office 365 плана 2](defender-for-office-365.md)

Если в вашей организации есть Microsoft Defender для Office 365 [](office-365-ti.md)Plan 2, который включает возможности расследования угроз и реагирования, вы можете использовать симулятор атак в Центре обеспечения безопасности & для выполнения реалистичных сценариев атак в вашей организации. Эти смоделированные атаки помогут вам определить и найти уязвимых пользователей до того, как реальная атака сказывается на вашей нижней строке. Подробнее читайте в этой статье.

> [!NOTE]
>
> Симулятор атаки, как описано в этой статье, теперь  только для  чтения и заменен обучением имитации атаки в узле совместной & электронной почты в центре безопасности [Microsoft 365.](https://security.microsoft.com) Дополнительные сведения см. в [дополнительных сведениях: Начало работы с обучением имитации атаки.](attack-simulation-training-get-started.md)
>
> Отключена возможность запуска новых имитаций из этой версии симулятора атаки. Однако с 24 января 2021 г. вы можете получать доступ к отчетам сроком до 90 дней.

## <a name="what-do-you-need-to-know-before-you-begin"></a>Что нужно знать перед началом работы

- Чтобы открыть Центр безопасности и соответствия требованиям, перейдите по ссылке <https://protection.office.com/>. Симулятор атаки доступен в **симуляторе атаки управления** \> **угрозами.** Чтобы перейти непосредственно к симулятору атаки, откройте <https://protection.office.com/attacksimulator> .

- Дополнительные сведения о доступности симулятора атак для различных Microsoft 365 см. в описании [службы Microsoft Defender для Office 365.](/office365/servicedescriptions/office-365-advanced-threat-protection-service-description)

- Необходимо быть членом групп ролей **"Управление организацией"** или **"Администратор** безопасности". Дополнительные сведения о группах ролей в Центре безопасности и соответствия требованиям см. в статье [Разрешения в Центре безопасности и соответствия требованиям](permissions-in-the-security-and-compliance-center.md).

- Ваша учетная запись должна быть настроена для многофакторной проверки подлинности (MFA) для создания и управления кампаниями в Симуляторе атак. Инструкции см. [в инструкции Настройка многофакторной проверки подлинности.](../../admin/security-and-compliance/set-up-multi-factor-authentication.md)

- Симулятор атак работает только на облачных почтовых ящиках.

- Фишинговые кампании будут собирать и обрабатывать события в течение 30 дней. Данные исторической кампании будут доступны в течение 90 дней после запуска кампании.

- Имитация атаки и связанные с обучением данные хранятся с другими данными клиентов для Microsoft 365 служб. Дополнительные сведения [см. в Microsoft 365 расположения данных.](../../enterprise/o365-data-locations.md)

- Для Симулятора атак нет соответствующих комлетов PowerShell.

## <a name="spear-phishing-campaigns"></a>Фишинговые кампании с копьем

*Фишинг —* это общий термин для атак электронной почты, которые пытаются украсть конфиденциальные сведения в сообщениях, которые, как представляется, являются законными или доверенными отправителями. *Фишинг* на копье — это целенаправленная фишинговая атака, которая использует сфокусированный и настраиваемый контент, специально адаптированный к целевым получателям (как правило, после разведки получателей злоумышленником).

В Симуляторе атак доступны два различных типа фишинговых кампаний копья:

- **Фишинг для копей (сбор учетных данных).** Атака пытается убедить получателей щелкнуть URL-адрес в сообщении. Если они щелкнуть ссылку, им будет предложено ввести свои учетные данные. Если это так, они будут доставлены в одно из следующих мест:

  - Страница по умолчанию, которая объясняет, что это был просто тест, и дает советы по распознаванию фишинговых сообщений.

    ![Что видят пользователи, щелкнув фишинговую ссылку и введите учетные данные](../../media/attack-simulator-phishing-result.png)

  - Настраиваемая страница (URL-адрес), указанная вами.

- **Фишинг (вложение)** копей: атака пытается убедить получателей открыть .docx или .pdf вложения в сообщении. Вложение содержит тот же контент из фишинговой ссылки по умолчанию, но первое предложение начинается с ", вы видите это сообщение как недавнее сообщение электронной почты, которое вы \<Display Name\> открыли...".

> [!NOTE]
> В настоящее время срок действия фишинговых кампаний в Симуляторе атак не истекает.

### <a name="create-a-spear-phishing-campaign"></a>Создание фишинговой кампании копья

Важной частью любой кампании фишинга копья является внешний вид сообщения электронной почты, отправленного целевым получателям. Чтобы создать и настроить сообщение электронной почты, у вас есть такие параметры:

- **Используйте встроенный шаблон электронной** почты: доступны два встроенных шаблона: **Prize Giveaway** и **Payroll Update.** При создании и запуске кампании можно дополнительно настроить некоторые, все или никакие свойства электронной почты из шаблона.

- **Создайте многопользоваемый** шаблон электронной почты. После создания и сохранения шаблона электронной почты вы можете использовать его снова в будущих фишинговых кампаниях. При создании и запуске кампании можно дополнительно настроить некоторые, все или никакие свойства электронной почты из шаблона.

- **Создайте сообщение электронной** почты в мастере. Вы можете создать сообщение электронной почты непосредственно в мастере при создании и запуске кампании фишинга копья.

#### <a name="step-1-optional-create-a-custom-email-template"></a>Шаг 1 (необязательный): создание настраиваемой шаблона электронной почты

Если вы собираетесь использовать один из встроенных шаблонов или создать сообщение электронной почты непосредственно в мастере, вы можете пропустить этот шаг.

1. В Центре & безопасности перейдите к симулятору **атаки управления** \> **угрозами.**

2. На странице **Имитация** атак в разделах "Фишинг на копье" (Сбор учетных **данных)** или "Фишинг для копей" **(Вложения)** щелкните **Сведения об атаке.**

   Не важно, где вы создаете шаблон. Доступные параметры в шаблоне одинаковы для обоих типов фишинговых атак.

3. На странице **Сведения о атаке,** открываемой в разделе **Шаблоны** фишинга в области **Создание** шаблонов, нажмите **кнопку Новый шаблон**.

4. Мастер **настройки шаблона фишинга** начинается с нового вылета. На **шаге Начните** введите уникальное имя отображения шаблона и нажмите кнопку **Далее**.

5. В **шаге Настройка сведений электронной почты** настройте следующие параметры:

   - **From (Name)**: имя отображения, используемая для отправитель сообщения.

   - **From (Email)**: адрес электронной почты отправитель.

   - **Фишинговый URL-адрес login Server.** Щелкните выпадение вниз и выберите один из доступных URL-адресов из списка. Это URL-адрес, который пользователи будут склонны щелкнуть. Имеются следующие варианты.

     - <http://portal.docdeliveryapp.com>
     - <http://portal.docdeliveryapp.net>
     - <http://portal.docstoreinternal.com>
     - <http://portal.docstoreinternal.net>
     - <http://portal.hardwarecheck.net>
     - <http://portal.hrsupportint.com>
     - <http://portal.payrolltooling.com>
     - <http://portal.payrolltooling.net>
     - <http://portal.prizegiveaway.net>
     - <http://portal.prizesforall.com>
     - <http://portal.salarytoolint.com>
     - <http://portal.salarytoolint.net>

     > [!NOTE]
     >
     > Служба репутации URL-адресов может определить один или несколько из этих URL-адресов как небезопасные. Проверьте доступность URL-адреса в поддерживаемых веб-браузерах перед использованием URL-адреса в фишинговой кампании.

   - **Пользовательский URL-адрес** страницы для посадки. Введите необязательный посадочный лист, на котором будут приниматься пользователи, если они щелкнуть фишинговую ссылку и введите свои учетные данные. Эта ссылка заменяет посадочную страницу по умолчанию. Например, если у вас есть обучение внутренней осведомленности, вы можете указать этот URL-адрес здесь.

   - **Категория.** В настоящее время этот параметр не используется (все, что вы вводите, игнорируется).

   - **Тема.** **Поле Subject** сообщения электронной почты.

   По завершении нажмите кнопку **Далее**.

6. В **шаге Compose email** создайте текст сообщения сообщения. Вы можете использовать вкладку **Email** (богатый редактор HTML) или вкладку **Source** (необработанные HTML-коды).

   Форматирование HTML может быть таким же простым или сложным, как и нужно. Вы можете вставить изображения и текст, чтобы повысить достоверность сообщения в клиенте электронной почты получателя.

   - `${username}` вставляет имя получателя.

   - `${loginserverurl}` вставляет **URL-адрес фишинг-сервера login Server** с предыдущего шага.

   По завершении нажмите кнопку **Далее**.

7. На **шаге Подтверждение** нажмите кнопку **Готово**.

#### <a name="step-2-create-and-launch-the-spear-phishing-campaign"></a>Шаг 2. Создание и запуск кампании фишинга копья

1. В Центре & безопасности перейдите к симулятору **атаки управления** \> **угрозами.**

2. На странице **Имитация атак** сделайте один из следующих выборов в зависимости от типа кампании, необходимой для создания:

   - В разделе **Фишинг для копей (Сбор учетных данных)** нажмите **кнопку Запуск атаки** или нажмите **кнопку Атака Сведения о** \> **запуске атаки**.

   - В разделе **Фишинг для копей (Вложения)** нажмите **кнопку Начать** атаку или нажмите **кнопку Атака Сведения о** \> **запуске атаки**.

3. Мастер **настройки фишинговых атак** запускается в новом вылете. На **шаге Начните** делать один из следующих действий:

   - В поле **Имя** введите уникальное имя отображения для кампании. Не нажимайте **кнопку Использовать шаблон,** так как сообщение электронной почты будет создаваться позже в мастере.

   - Щелкните **Использовать шаблон** и выберите встроенный или настраиваемый шаблон электронной почты. После выбора шаблона поле **Имя** автоматически заполняется на основе шаблона, но можно изменить имя.

   > [!div class="mx-imgBorder"]
   > ![Страница "Запуск фишинга"](../../media/5e93b3cc-5981-462f-8b45-bdf85d97f1b8.jpg)

   По завершении нажмите кнопку **Далее**.

4. На **шаге Целевые получатели** сделайте один из следующих действий:

   - Нажмите **адресную** книгу, чтобы выбрать получателей (пользователей или групп) для кампании. У каждого целевого получателя должен быть Exchange Online почтовый ящик. Если щелкнуть **фильтр** и **применить** без ввода критериев поиска, все получатели возвращаются и добавляются в кампанию.

   - Щелкните **импортировать импорт файлов,** чтобы импортировать разделенное запятой значение (CSV) или файл адресов электронной почты, разделенный строками.  Каждая строка должна содержать адрес электронной почты получателя.

   По завершении нажмите кнопку **Далее**.

5. В **шаге Настройка сведений электронной почты** настройте следующие параметры:

   Если вы выбрали шаблон на шаге **Начните,** большинство из этих значений уже настроены, но их можно изменить.

   - **From (Name)**: имя отображения, используемая для отправитель сообщения.

   - **From (Email)**: адрес электронной почты отправитель. Вы можете ввести реальный или поддельный адрес электронной почты из домена электронной почты организации или ввести реальный или поддельный внешний адрес электронной почты. Допустимый адрес электронной почты отправитель из организации фактически будет решаться в клиенте электронной почты получателя.

   - **Фишинговый URL-адрес login Server.** Щелкните выпадение вниз и выберите один из доступных URL-адресов из списка. Это URL-адрес, который пользователи будут склонны щелкнуть. Имеются следующие варианты.

     - <http://portal.docdeliveryapp.com>
     - <http://portal.docdeliveryapp.net>
     - <http://portal.docstoreinternal.com>
     - <http://portal.docstoreinternal.net>
     - <http://portal.hardwarecheck.net>
     - <http://portal.hrsupportint.com>
     - <http://portal.payrolltooling.com>
     - <http://portal.payrolltooling.net>
     - <http://portal.prizegiveaway.net>
     - <http://portal.prizesforall.com>
     - <http://portal.salarytoolint.com>
     - <http://portal.salarytoolint.net>

     > [!NOTE]
     >
     > - Все URL-адреса намеренно http, а не https.
     >
     > - Служба репутации URL-адресов может определить один или несколько из этих URL-адресов как небезопасные. Проверьте доступность URL-адреса в поддерживаемых веб-браузерах перед использованием URL-адреса в фишинговой кампании.
     >
     > - Необходимо выбрать URL-адрес. Для **кампаний фишинга (вложения)** можно удалить ссылку из тела сообщения на следующем шаге (в противном случае сообщение будет содержать как ссылку, так **и** вложение).

   - **Тип вложения.** Этот параметр доступен только в кампаниях **по фишингу (вложениям).** Нажмите кнопку drop down **и выберите**.DOCX **или.PDF** из списка.

   - **Имя вложения.** Этот параметр доступен только в кампаниях **по фишингу (вложениям).** Введите имя файла для .docx или .pdf вложения.

   - **Пользовательский URL-адрес** страницы для посадки. Введите необязательный посадочный лист, на котором будут приниматься пользователи, если они щелкнуть фишинговую ссылку и введите свои учетные данные.    По завершении нажмите кнопку **Далее**.

6. В **шаге Compose email** создайте текст сообщения сообщения. Если вы выбрали шаблон на шаге **Начните,** тело сообщения уже настроено, но его можно настроить. Вы можете использовать вкладку **Email** (богатый редактор HTML) или вкладку **Source** (необработанные HTML-коды).

   Форматирование HTML может быть таким же простым или сложным, как и нужно. Вы можете вставить изображения и текст, чтобы повысить достоверность сообщения в клиенте электронной почты получателя.

   - `${username}` вставляет имя получателя.

   - `${loginserverurl}`вставляет **URL-адрес фишинговых url-адресов login Server.**

   Для кампаний фишинга **(вложения)** следует удалить ссылку из тела сообщения (в противном случае сообщение будет содержать ссылку и вложение, а щелчки ссылок не отслеживаются в кампании вложения).

   По завершении нажмите кнопку **Далее**.

7. На **шаге Подтверждение** нажмите **кнопку Готово,** чтобы запустить кампанию. Фишинговое сообщение доставляется целевым получателям.

## <a name="password-attack-campaigns"></a>Кампании атаки паролей

Атака *паролей* пытается угадать пароли для учетных записей пользователей в организации, как правило, после того, как злоумышленник идентифицировал одну или несколько допустимых учетных записей пользователей.

В Симуляторе атак доступны два различных типа кампаний атаки паролей для проверки сложности паролей пользователей:

- **Грубый** пароль силы (атака словаря) :  атака грубой силы или словаря использует большой файл паролей словаря на учетной записи пользователя в надежде, что один из них будет работать (много паролей против одной учетной записи).  Неправильные блокировки паролей помогают отпугивать атаки грубого принудительного пароля.

  При атаке словаря можно указать один или несколько паролей для проверки (вручную вложенный или в загруженный файл), а также указать одного или многих пользователей.

- **Атака спрей** пароля. *Атака* спрей пароля использует тот же тщательно рассмотренный пароль против списка учетных записей пользователей (один пароль против многих учетных записей). Атаки спрей паролей труднее обнаружить, чем атаки грубого принудительного пароля (вероятность успеха увеличивается, когда злоумышленник пытается один пароль на десятках или сотнях учетных записей без риска отключения блокировки неправильного пароля пользователя).

  Для атаки спрей пароля можно указать только один пароль, чтобы попытаться, и вы можете указать одного или многих пользователей.

> [!NOTE]
> Атаки паролей в Симуляторе атак передают запросы имени пользователя и пароля Basic auth в конечную точку, поэтому они также работают с другими методами проверки подлинности (AD FS, синхронизация хеш-кодов паролей, сквозной доступ, PingFederate и т.д.). Для пользователей с включенной поддержкой MFA, даже если атака пароля пытается использовать свой фактический пароль, попытка всегда  регистрируется как сбой (другими словами, пользователи MFA никогда не будут отображаться в графе Успешные попытки кампании). Это ожидаемый результат. MFA — это основной метод защиты от атак паролей.

### <a name="create-and-launch-a-password-attack-campaign"></a>Создание и запуск кампании атаки паролей

1. В Центре & безопасности перейдите к симулятору **атаки управления** \> **угрозами.**

2. На странице **Имитация атак** сделайте один из следующих выборов в зависимости от типа кампании, необходимой для создания:

   - В разделе **Brute Force Password (Dictionary Attack)** нажмите **кнопку Запуск** атаки или нажмите **кнопку Атака Сведения о** \> **запуске атаки**.

   - в разделе **Атака спрей пароля** нажмите **кнопку Начать** атаку или нажмите **кнопку Атака Сведения о** \> **запуске атаки**.

3. Мастер **настройки атаки паролей** начинается с новой вылетной записи. На **шаге Начните** введите уникальное имя отображения для кампании и нажмите кнопку **Далее**.

4. На **шаге Целевые** пользователи сделайте один из следующих действий:

   - Нажмите **адресную** книгу, чтобы выбрать получателей (пользователей или групп) для кампании. У каждого целевого получателя должен быть Exchange Online почтовый ящик. Если щелкнуть **фильтр** и **применить** без ввода критериев поиска, все получатели возвращаются и добавляются в кампанию.

   - Щелкните **импортировать импорт файлов,** чтобы импортировать разделенное запятой значение (CSV) или файл адресов электронной почты, разделенный строками.  Каждая строка должна содержать адрес электронной почты получателя.

   По завершении нажмите кнопку **Далее**.

5. В **шаге Выбор параметров атаки** выберите, что делать в зависимости от типа кампании:

   - **Brute Force Password (Dictionary Attack)**: Do either of the following steps:

     - **Ввод паролей вручную.** Введите кнопку **Нажмите,** чтобы добавить пароль, введите пароль и нажмите кнопку ENTER. Повторите этот шаг необходимое количество раз.

     - **Upload** паролей из файла словаря: **щелкните** Upload, чтобы импортировать существующий текстовый файл, содержащий один пароль на каждой строке и пустую последнюю строку. Размер текстового файла должен быть не более 10 МБ и не может содержать более 30000 паролей.

   - **Атака спрей пароля.** **В пароль(ы)** для использования в поле атаки введите один пароль.

   По завершении нажмите кнопку **Далее**.

6. На **шаге Подтверждение** нажмите **кнопку Готово,** чтобы запустить кампанию. Указанные пароли опробуются на указанных вами пользователях.

## <a name="view-campaign-results"></a>Просмотр результатов кампании

После запуска кампании вы можете проверить ход и результаты на главной странице **Имитация атак.**

Активные кампании будут показывать планку состояния, завершенное процентное значение и количество "(завершенных пользователей) (всего пользователей)". Нажатие **кнопки "Обновление"** позволит обновить ход активных кампаний. Чтобы остановить активную **кампанию,** можно также нажать Кнопку Прекратить.

По завершению кампании состояние изменяется в **Attack.** Результаты кампании можно просмотреть, выступая с одним из следующих действий:

- На главной **странице Имитация атак** щелкните **Просмотр отчета** под именем кампании.

- На главной **странице Имитация атак** щелкните **Сведения о** атаке в разделе для типа атаки. На **открываемой странице** Сведения о атаке выберите кампанию в разделе **История атак.**

В любом из предыдущих действий вы будете перенабираться на страницу с именем **Attack details**. Сведения, доступные на этой странице для каждого типа кампании, описаны в следующих разделах.

### <a name="spear-phishing-credentials-harvest-campaign-results"></a>Результаты кампании По борьбе с фишингом (сбор учетных данных)

На странице Сведения о **атаке** для каждой кампании доступны следующие сведения:

- Продолжительность (дата начала,времени и конечной даты/времени) кампании.

- **Всего пользователей, целевых**

- **Успешные** попытки: число пользователей,  которые нажали ссылку и ввели свои учетные данные *(любое* имя пользователя и значение пароля).

- **Общая скорость успеха:** процент, который рассчитывается **по** успешным попыткам  /  **Всего целевых пользователей**.

- **Быстрый щелчок:** Сколько времени потребовалось первому пользователю, чтобы щелкнуть ссылку после запуска кампании.

- **Средний щелчок:** сумма времени, за который каждый щелкнуть ссылку, разделенную количеством пользователей, нажавших ссылку.

- **Click Success Rate**: A percentage that's calculated by (number of users who clicked the link) / **Total users targeted.**

- **Быстрые учетные** данные. Сколько времени потребовалось первому пользователю, чтобы ввести свои учетные данные после запуска кампании.

- **Средние учетные** данные: сумма времени ввода учетных данных каждого пользователя, разделенного на число пользователей, вступивших в свои учетные данные.

- **Коэффициент успешности** учетных данных: процент, который рассчитывается по (число пользователей, вступивших в свои учетные данные) / **Общее количество целевых пользователей.**

- Диаграмма панели, на которую показано **нажатие ссылки** и **предоставленные учетные данные** в день.

- Круговой график, на который показано **нажатие ссылки,** предоставленные учетные данные, и **проценты none** для кампании.

- В **разделе "Скомпрометированная** информация о пользователях" перечислены сведения пользователей, нажавших ссылку:

  - Адрес электронной почты пользователя.

  - Дата и время, когда они щелкнули ссылку.

  - IP-адрес клиента.

  - Сведения о пользовательской версии веб-Windows и веб-браузере.

  Для экспорта **результатов** в CSV-файл можно нажать кнопку Экспорт.

### <a name="spear-phishing-attachment-campaign-results"></a>Результаты кампании фишинга (вложения)

На странице Сведения о **атаке** для каждой кампании доступны следующие сведения:

- Продолжительность (дата начала,времени и конечной даты/времени) кампании.

- **Всего пользователей, целевых**

- **Успешные** попытки: количество пользователей, которые открыли или скачали и открыли вложение (предварительный просмотр не считается).

- **Общая скорость успеха:** процент, который рассчитывается **по** успешным попыткам  /  **Всего целевых пользователей**.

- **Самое быстрое время открытия** вложения: сколько времени потребовалось первому пользователю, чтобы открыть вложение после запуска кампании.

- **Среднее время открытия** вложения: сумма, за которую каждый открыл вложение, разделенное на число пользователей, открывавших вложение.

- **Показатель успешности** открытия вложения: процент, который рассчитывается по (число пользователей, открывавших вложение) / **Всего пользователей, целевых**.

### <a name="brute-force-password-dictionary-attack-campaign-results"></a>Результаты кампании Brute Force Password (Dictionary Attack)

На странице Сведения о **атаке** для каждой кампании доступны следующие сведения:

- Продолжительность (дата начала,времени и конечной даты/времени) кампании.

- **Всего пользователей, целевых**

- **Успешные** попытки: число пользователей, у которых был обнаружен один из указанных паролей.

- **Общая скорость успеха:** процент, который рассчитывается **по** успешным попыткам  /  **Всего целевых пользователей**.

- В **разделе "Скомпрометированная** пользователи" перечислены адреса электронной почты затронутых пользователей. Для экспорта **результатов** в CSV-файл можно нажать кнопку Экспорт.

### <a name="password-spray-attack-campaign-results"></a>Результаты кампании атаки спрей пароля

На странице Сведения о **атаке** для каждой кампании доступны следующие сведения:

- Продолжительность (дата начала,времени и конечной даты/времени) кампании.

- **Всего пользователей, целевых**

- **Успешные** попытки: число пользователей, у которых установлено использование указанного пароля.

- **Общая скорость успеха:** процент, который рассчитывается **по** успешным попыткам  /  **Всего целевых пользователей**.
