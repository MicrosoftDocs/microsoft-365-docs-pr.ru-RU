# <a name="attack-simulator-in-microsoft-defender-for-office-365"></a>Имитатор атак в Microsoft Defender для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

Если в вашей организации есть Microsoft Defender для Office 365 (план 2), который включает возможности исследования угроз и реагирования на [них,](office-365-ti.md)вы можете использовать имитатор атак в Центре безопасности и соответствия требованиям & для выполнения реалистичных сценариев атак в организации. Эти смоделированные атаки помогут вам определить и найти уязвимых пользователей, прежде чем реальные атаки поляв на вашу нижнюю часть. Чтобы узнать больше, ознакомьтесь с этой статьей.

> [!NOTE]
> Имитация атак и учебные данные хранятся вместе с другими данными клиентов для служб Microsoft 365. Дополнительные сведения см. в [расположениях данных Microsoft 365.](/microsoft-365/enterprise/o365-data-locations)

> [!TIP]
> Обучение моделированию атак доступно для предварительной версии в Центре безопасности Microsoft 365. Чтобы узнать больше, ознакомьтесь с имитацией фишинговой атаки с [помощью Microsoft Defender для Office 365.](attack-simulation-training.md)

## <a name="what-do-you-need-to-know-before-you-begin"></a>Что нужно знать перед началом работы

- Чтобы открыть Центр безопасности и соответствия требованиям, перейдите по ссылке <https://protection.office.com/>. Имитатор атак доступен в **имитаторе атак** \> **управления угрозами.** Перейдите непосредственно в имитатор атак, откройте <https://protection.office.com/attacksimulator> .

- Дополнительные сведения о доступности имитатора атак в различных подписках Microsoft 365 см. в описании службы [Microsoft Defender для Office 365.](https://docs.microsoft.com/office365/servicedescriptions/office-365-advanced-threat-protection-service-description)

- Необходимо быть членом группы  ролей "Управление организацией" или **"Администратор** безопасности". Дополнительные сведения о группах ролей в Центре безопасности и соответствия требованиям см. в статье [Разрешения в Центре безопасности и соответствия требованиям](permissions-in-the-security-and-compliance-center.md).

- Ваша учетная запись должна быть настроена для многофакторной проверки подлинности (MFA) для создания кампаний и управления имитатором атак. Инструкции см. в [настройках многофакторной проверки подлинности.](https://docs.microsoft.com/microsoft-365/admin/security-and-compliance/set-up-multi-factor-authentication)

- Фишинговые кампании собирают и обрабатывает события в течение 30 дней. Данные истории кампании будут доступны в течение 90 дней после запуска кампании.

- Для имитатора атак нет соответствующих cmdlets PowerShell.

## <a name="spear-phishing-campaigns"></a>Фишинговые кампании

*Фишинг —* это универсальный термин для атак электронной почты, которые пытаются украсть конфиденциальную информацию в сообщениях, которые кажутся надежными или надежными отправителями. *Целевой* фишинг — это целевая фишинговая атака, использующая сфокусированное и настроенное содержимое, специально адаптированное для целевых получателей (как правило, после погруженого применения злоумышленником к получателям).

В имитаторе атак доступны два различных типа фишинговых кампаний:

- **Фишинговое фишинговое письмо (сбор** учетных данных): атака пытается убедить получателей в том, что нужно щелкнуть URL-адрес в сообщении. Если щелкнуть ссылку, им будет предложено ввести свои учетные данные. В этом случае они будут доставлены в одно из следующих мест:

  - Страница по умолчанию, объясняя, что это всего лишь тест, и дает советы по распознаванию фишинговых сообщений.

    ![Что видят пользователи, щелкнув фишинговую ссылку и введите свои учетные данные](../../media/attack-simulator-phishing-result.png)

  - Настраиваемая страница (URL-адрес), которую вы указываете.

- **Фишинговое письмо (вложение):** атака пытается убедить получателей в том, что в сообщении открывается вложение DOCX или PDF. Вложение содержит то же содержимое из фишинговой ссылки по умолчанию, но первое предложение начинается с ", вы видите это сообщение как недавнее сообщение электронной \<Display Name\> почты, которое вы открыли...".

> [!NOTE]
> В настоящее время срок действия фишинговых кампаний в имитаторе атак не истекает.

### <a name="create-a-spear-phishing-campaign"></a>Создание антифишинговой кампании

Важной частью любой целевой фишинговой кампании является внешний вид сообщения электронной почты, которое отправляется целевым получателям. Чтобы создать и настроить сообщение электронной почты, у вас есть указанные здесь параметры.

- **Используйте встроенный шаблон электронной** почты: доступны два встроенных шаблона: **"Обновление** оплаты" и "Обновление **зарплаты".** При создании и запуске кампании вы можете дополнительно настроить некоторые, все или никакие свойства электронной почты из шаблона.

- **Создайте шаблон электронной** почты для повторного использования: после создания и сохранения шаблона электронной почты вы сможете использовать его снова в будущих фишинговых кампаниях. При создании и запуске кампании вы можете дополнительно настроить некоторые, все или никакие свойства электронной почты из шаблона.

- **Создайте сообщение электронной** почты в мастере: вы можете создать сообщение электронной почты непосредственно в мастере при создании и запуске фишинговой кампании.

#### <a name="step-1-optional-create-a-custom-email-template"></a>Шаг 1 (необязательно). Создание пользовательского шаблона электронной почты

Если вы собираетесь использовать один из встроенных шаблонов или создать сообщение электронной почты непосредственно в мастере, вы можете пропустить этот шаг.

1. В Центре безопасности & соответствия требованиям перейдите в имитатор **атак** \> **управления угрозами.**

2. На странице **"Имитация**  **атак"** щелкните "Сведения о атаке". 

   Не имеет значения, где вы создаете шаблон. Доступные варианты в шаблоне одинаковы для обоих типов фишинговых атак.

3. На **открываемой** странице сведений о атаке в  разделе "Фишинговые шаблоны" в области "Создание шаблонов" щелкните **"Создать шаблон".** 

4. Мастер **настройки фишинговых шаблонов** запускается в новом flyout. На **шаге** "Начните" введите уникальное отображаемое имя шаблона и нажмите кнопку **"Далее".**

5. На **этапе настройки сведений электронной** почты настройте следующие параметры:

   - **From (Name)**— отображаемая фамилия, используемая для отправитель сообщения.

   - **From (Email)**: адрес электронной почты отправитель.

   - **URL-адрес фишинговой учетной** записи сервера входа: щелкните в выпадаемом списке и выберите один из доступных URL-адресов в списке. Это URL-адрес, который пользователи смогут щелкнуть. Имеются следующие варианты.

     - <http://portal.docdeliveryapp.com>
     - <http://portal.docdeliveryapp.net>
     - <http://portal.docstoreinternal.com>
     - <http://portal.docstoreinternal.net>
     - <http://portal.hardwarecheck.net>
     - <http://portal.hrsupportint.com>
     - <http://portal.payrolltooling.com>
     - <http://portal.payrolltooling.net>
     - <http://portal.prizegiveaway.net>
     - <http://portal.prizesforall.com>
     - <http://portal.salarytoolint.com>
     - <http://portal.salarytoolint.net>

     > [!NOTE]
     >
     > Служба репутации URL-адресов может определить один или несколько таких URL-адресов как небезопасные. Перед использованием URL-адреса в фишинговой кампании проверьте доступность URL-адреса в поддерживаемых веб-браузерах.

   - **URL-адрес настраиваемой** страницы: введите необязательные страницы, на которых пользователи будут переходить по фишинговой ссылке и вводить свои учетные данные. Эта ссылка заменяет тесную страницу по умолчанию. Например, если у вас есть обучение внутренней осведомленности, вы можете указать этот URL-адрес здесь.

   - **Категория:** в настоящее время этот параметр не используется (все, что вы вводите, игнорируется).

   - **Тема:** поле **темы** сообщения электронной почты.

   По завершении нажмите кнопку **Далее**.

6. На **этапе создания сообщения** создайте текст сообщения. Вы можете использовать вкладку **"Электронная** почта" (редактор ФОРМАТ HTML) или вкладку **"Источник"** (необработанные HTML-коды).

   Форматирование HTML может быть простым или сложным. Вы можете вставлять изображения и текст, чтобы улучшить надежность сообщения в почтовом клиенте получателя.

   - `${username}` вставляет имя получателя.

   - `${loginserverurl}` вставляет **URL-адрес сервера фишинга для входа** в систему из предыдущего шага.

   По завершении нажмите кнопку **Далее**.

7. На **шаге "Подтверждение"** нажмите кнопку **"Готово".**

#### <a name="step-2-create-and-launch-the-spear-phishing-campaign"></a>Шаг 2. Создание и запуск первой фишинговой кампании

1. В Центре безопасности & соответствия требованиям перейдите в имитатор **атак** \> **управления угрозами.**

2. На странице **"Имитация атак"** сделайте одно из следующих выбором в зависимости от типа создаемой кампании:

   - В разделе **"Копирошение фишинга (сбор** учетных данных") щелкните **"Атака при** запуске" или **"Атака с** подробными \> **сведениями о атаке при запуске".**

   - В разделе **"Вложение" выберите**  "Атака при запуске" или **"Атака с** подробными \> **сведениями о запуске атаки".**

3. Мастер **настройки фишинговых атак** запускается в новом окле. На **шаге** "Начните" сделайте одно из следующих действий:

   - В поле **"Имя"** введите уникальное отображаемое имя кампании. Не нажимайте кнопку **"Использовать шаблон",** так как вы создадим сообщение электронной почты позже в мастере.

   - Щелкните **"Использовать шаблон"** и выберите встроенный или настраиваемый шаблон электронной почты. После выбора шаблона поле **"Имя"** заполняется автоматически на основе шаблона, но вы можете изменить имя.

   ![Начальная страница фишинга](../../media/5e93b3cc-5981-462f-8b45-bdf85d97f1b8.jpg)

   По завершении нажмите кнопку **Далее**.

4. На **шаге "Получатели"** сделайте одно из следующих действий:

   - Щелкните **адресную** книгу, чтобы выбрать получателей (пользователей или группы) для кампании. У каждого целевого получателя должен быть почтовый ящик Exchange Online. Если нажать **кнопку "Фильтр"** и **"Применить"** без ввода условия поиска, все получатели будут возвращены и добавлены в кампанию.

   - Нажмите **кнопку** **"Импорт** и импорт файла", чтобы импортировать CSV-файл или файл адресов электронной почты, разделенный запятой. Каждая строка должна содержать адрес электронной почты получателя.

   По завершении нажмите кнопку **Далее**.

5. На **этапе настройки сведений электронной** почты настройте следующие параметры:

   Если вы выбрали шаблон  на шаге "Начните", большинство из этих значений уже настроены, но их можно изменить.

   - **From (Name)**— отображаемая фамилия, используемая для отправитель сообщения.

   - **From (Email)**: адрес электронной почты отправитель. Вы можете ввести реальный или поддельный адрес электронной почты из домена электронной почты вашей организации или ввести реальный или поддельный внешний адрес электронной почты. Допустимый адрес электронной почты отправитель из вашей организации фактически будет разрешаться в почтовом клиенте получателя.

   - **URL-адрес фишинговой учетной** записи сервера входа: щелкните в выпадаемом списке и выберите один из доступных URL-адресов в списке. Это URL-адрес, который пользователи смогут щелкнуть. Имеются следующие варианты.

     - <http://portal.docdeliveryapp.com>
     - <http://portal.docdeliveryapp.net>
     - <http://portal.docstoreinternal.com>
     - <http://portal.docstoreinternal.net>
     - <http://portal.hardwarecheck.net>
     - <http://portal.hrsupportint.com>
     - <http://portal.payrolltooling.com>
     - <http://portal.payrolltooling.net>
     - <http://portal.prizegiveaway.net>
     - <http://portal.prizesforall.com>
     - <http://portal.salarytoolint.com>
     - <http://portal.salarytoolint.net>

     > [!NOTE]
     >
     > - Все URL-адреса намеренно являются http, а не https.
     >
     > - Служба репутации URL-адресов может определить один или несколько таких URL-адресов как небезопасные. Перед использованием URL-адреса в фишинговой кампании проверьте доступность URL-адреса в поддерживаемых веб-браузерах.
     >
     > - Необходимо выбрать URL-адрес. Для кампаний по борьбе с фишингом **(вложениями)** ссылку можно удалить из тела сообщения на следующем шаге (в противном случае сообщение будет содержать как ссылку, так **и** вложение).

   - **Тип вложения:** этот параметр доступен только в кампаниях **по фишингу** с вложениями. Щелкните в этом выпадаемом окте и выберите **. DOCX** или **. PDF** из списка.

   - **Имя вложения:** этот параметр доступен только в кампаниях **по фишингу** с вложениями. Введите имя файла для вложения DOCX или PDF.

   - **URL-адрес настраиваемой** страницы: введите необязательные страницы, на которых пользователи будут переходить по фишинговой ссылке и вводить свои учетные данные. Эта ссылка заменяет тесную страницу по умолчанию. Например, если у вас есть обучение внутренней осведомленности, вы можете указать этот URL-адрес здесь.

   - **Тема:** поле **темы** сообщения электронной почты.

   По завершении нажмите кнопку **Далее**.

6. На **этапе создания сообщения** создайте текст сообщения. Если вы выбрали шаблон  на шаге "Начните", текст сообщения уже настроен, но его можно настроить. Вы можете использовать вкладку **"Электронная** почта" (редактор ФОРМАТ HTML) или вкладку **"Источник"** (необработанные HTML-коды).

   Форматирование HTML может быть простым или сложным. Вы можете вставлять изображения и текст, чтобы улучшить надежность сообщения в почтовом клиенте получателя.

   - `${username}` вставляет имя получателя.

   - `${loginserverurl}` вставляет **URL-адрес сервера фишинга для** входа в систему.

   Для кампаний по борьбе с фишингом **(вложениями)** следует удалить ссылку из тела  сообщения (в противном случае сообщение будет содержать как ссылку, так и вложение, а щелчки по ссылкам не отслеживаются в кампании вложения).

   ![Тело сообщения электронной почты для составить](../../media/9bd65af4-1f9d-45c1-8c06-796d7ccfd425.jpg)

   По завершении нажмите кнопку **Далее**.

7. На **шаге "Подтверждение"** нажмите кнопку **"Готово",** чтобы запустить кампанию. Фишинговое сообщение доставляется целевым получателям.

## <a name="password-attack-campaigns"></a>Атаки с помощью паролей

Атака *с паролем* пытается угадать пароли для учетных записей пользователей в организации, как правило, после того, как злоумышленник определил одну или несколько допустимых учетных записей пользователей.

В имитаторе атак доступны два различных типа кампаний по атаке с помощью паролей для проверки сложности паролей пользователей:

- Пароль для атак с использованием  словаря,  в которых используется большой файл словаря паролей для учетной записи пользователя в расчете на то, что один из них будет работать (много паролей для одной учетной записи).  Неправильные блокировки паролей помогают предотвратить атаки с помощью атак с помощью перенастраховки.

  Для атаки с помощью словаря можно указать один или несколько паролей для проверки (вручную или в добавленный файл), а также указать одного или несколько пользователей.

- **Атака распыления пароля:** при атаке *распыления* пароля используется тот же тщательно просмотряный пароль для списка учетных записей пользователей (один пароль для многих учетных записей). Атаки с распылением пароля обнаруживать сложнее, чем атаки с помощью атак злоумышленника.

  Для атаки распыления пароля можно указать только один пароль, который необходимо попробовать, и указать одного или несколько пользователей.

> [!NOTE]
> Атаки с паролями в симуляторе атак передают запросы проверки подлинности пользователя и пароля Basic конечной точке, поэтому они также работают с другими методами проверки подлинности (AD FS, синхронизация хеш-функций паролей, сквозной доступ, PingFederate и т. д.). Для пользователей, для пользователей с включенной многофаксной многофаксной поддержкой, даже если атака с паролем пытается  использовать их фактический пароль, попытка всегда регистрируется как ошибка (другими словами, пользователи MFA никогда не будут отображаться в подсчете успешных попыток кампании). Это ожидаемый результат. MFA — это основной способ защиты от атак с помощью паролей.

### <a name="create-and-launch-a-password-attack-campaign"></a>Создание и запуск кампании по атаке с помощью паролей

1. В Центре безопасности & соответствия требованиям перейдите в имитатор **атак** \> **управления угрозами.**

2. На странице **"Имитация атак"** сделайте одно из следующих выбором в зависимости от типа создаемой кампании:

   - In the **Brute Force Password (Dictionary Attack)** section, click **Launch Attack** or click **Attack Details** Launch \> **Attack**.

   - в разделе **"Атака с распылением** пароля" щелкните **"Атака при запуске"** или **"Атака с** подробными \> **сведениями о атаке при запуске".**

3. Мастер **настройки атаки паролем** запускается в новом окле. На **шаге** "Начните" введите уникальное отображаемое имя для кампании и нажмите кнопку **"Далее".**

4. На **шаге "Целевые** пользователи" сделайте одно из следующих действий:

   - Щелкните **адресную** книгу, чтобы выбрать получателей (пользователей или группы) для кампании. У каждого целевого получателя должен быть почтовый ящик Exchange Online. Если нажать **кнопку "Фильтр"** и **"Применить"** без ввода условия поиска, все получатели будут возвращены и добавлены в кампанию.

   - Нажмите **кнопку** **"Импорт** и импорт файла", чтобы импортировать CSV-файл или файл адресов электронной почты, разделенный запятой. Каждая строка должна содержать адрес электронной почты получателя.

   По завершении нажмите кнопку **Далее**.

5. На **шаге "Выбор параметров атаки"** выберите, что делать в зависимости от типа кампании:

   - **Пароль при атаке** по словарю: сделайте один из следующих действий:

     - **Введите пароли** вручную:  введите пароль в поле "Нажмите ввод", чтобы добавить пароль, а затем нажмите ввод. Повторите этот шаг необходимое количество раз.

     - **Отправка паролей из файла словаря:** нажмите кнопку **"Отправить",** чтобы импортировать существующий текстовый файл, содержащий один пароль в каждой строке и пустую последнюю строку. Размер текстового файла не должен быть меньше 10 МБ и не может содержать более 30 000 паролей.

   - **Атака распылением пароля:** в поле атаки введите один пароль. 

   По завершении нажмите кнопку **Далее**.

6. На **шаге "Подтверждение"** нажмите кнопку **"Готово",** чтобы запустить кампанию. Указанные пароли пытаются использовать для указанных пользователей.

## <a name="view-campaign-results"></a>Просмотр результатов кампании

После запуска кампании вы можете проверить ход выполнения и результаты на главной странице **имитации атак.**

Активные кампании будут показывать планку состояния, завершенное процентное значение и количество "(завершенных пользователей) (всего пользователей)". Нажатие **кнопки "Обновить"** обножит ход выполнения всех активных кампаний. Вы также можете нажать **кнопку "Завершить",** чтобы остановить активную кампанию.

После завершения кампании состояние "Атака" изменится на **"Атака завершена".** Чтобы просмотреть результаты кампании, с помощью одного из следующих действий:

- На главной **странице "Имитация атак"** щелкните **"Просмотреть отчет"** под именем кампании.

- On the main **Simulate attacks** page, click **Attack Details** in the section for the type of attack. На **открывскойся** странице сведений о атаке выберите кампанию в разделе **"История атак".**

При любом из предыдущих действий вы будете перенабираться на страницу с именем **"Сведения о атаке".** Сведения, доступные на этой странице для каждого типа кампании, описаны в следующих разделах.

### <a name="spear-phishing-credentials-harvest-campaign-results"></a>Результаты кампании по фишингу (сбор учетных данных)

На странице сведений о атаке для каждой **кампании** доступны следующие сведения:

- Продолжительность кампании (дата и время начала, время и дата окончания).

- **Общее количество целевых пользователей**

- **Успешные попытки:** количество пользователей, которые щелкнули ссылку и **ввели** свои учетные данные *(любое* имя пользователя и пароль).

- **Общий показатель успешности**: процент, который рассчитывается по **успешным попыткам**  /  **Всего целевых пользователей.**

- **Самый быстрый щелчок:** сколько времени потребовалось первому пользователю, чтобы щелкнуть ссылку после запуска кампании.

- **Средний щелчок**: сумма времени, за который все щелкнули ссылку, разделенную на количество пользователей, щелкнув ссылку.

- **Click Success Rate**: A percentage that's calculated by (number of users who clicked the link) / **Total users targeted**.

- **Самые быстрые учетные** данные: сколько времени потребовалось первому пользователю, чтобы ввести свои учетные данные после запуска кампании.

- **Средние учетные данные**: сумма времени, за который все ввели свои учетные данные, разделенные на количество пользователей, введвших свои учетные данные.

- **Показатель успешности учетных** данных : процент, который рассчитывается на 1 процент (количество пользователей, ввели свои учетные данные) / **Общее количество целевых пользователей.**

- Диаграмма с **нажатием** ссылки и предоставленными учетными **данными** номерами в день.

- Круговой график, на который показаны щелчок ссылки, предоставленные учетные данные **и** процентные соотношения **"Нет"** для кампании.

- В **разделе "Скомпрометированная** пользователи" перечислены сведения о пользователях, щелкнув ссылку:

  - Адрес электронной почты пользователя.

  - Дата и время щелчка по ссылке.

  - IP-адрес клиента.

  - Сведения о версии Windows и веб-браузере пользователя.

  Можно нажать **кнопку "Экспорт",** чтобы экспортировать результаты в CSV-файл.

### <a name="spear-phishing-attachment-campaign-results"></a>Результаты кампании по фишингу (вложениям)

На странице сведений о атаке для каждой **кампании** доступны следующие сведения:

- Продолжительность кампании (дата и время начала, время и дата окончания).

- **Общее количество целевых пользователей**

- **Успешные** попытки: количество пользователей, открывавших или скачавших и открывавших вложение (предварительная версия не учитывается).

- **Общий показатель успешности**: процент, который рассчитывается по **успешным попыткам**  /  **Всего целевых пользователей.**

- **Самое быстрое время открытия** вложения: сколько времени потребовалось первому пользователю, чтобы открыть вложение после запуска кампании.

- **Среднее время открытия вложения:** сумма времени, за который все открывали вложение, разделив его на количество пользователей, открывавших вложение.

- **Показатель успешности** открытия вложения: процент, который рассчитывается на долю (количество пользователей, открывавших вложение) / общее количество пользователей, на **которые нацелено.**

### <a name="brute-force-password-dictionary-attack-campaign-results"></a>Результаты кампании по выбору принудительного пароля (атака по словарю)

На странице сведений о атаке для каждой **кампании** доступны следующие сведения:

- Продолжительность кампании (дата и время начала, время и дата окончания).

- **Общее количество целевых пользователей**

- **Успешные** попытки: количество пользователей, которые использовали один из указанных паролей.

- **Общий показатель успешности**: процент, который рассчитывается по **успешным попыткам**  /  **Всего целевых пользователей.**

- В **разделе "Скомпрометированная** пользователи" перечислены адреса электронной почты затронутых пользователей. Можно нажать **кнопку "Экспорт",** чтобы экспортировать результаты в CSV-файл.

### <a name="password-spray-attack-campaign-results"></a>Результаты кампании по распылку пароля

На странице сведений о атаке для каждой **кампании** доступны следующие сведения:

- Продолжительность кампании (дата и время начала, время и дата окончания).

- **Общее количество целевых пользователей**

- **Успешные** попытки: количество пользователей, которые были найдены с использованием указанного пароля.

- **Общий показатель успешности**: процент, который рассчитывается по **успешным попыткам**  /  **Всего целевых пользователей.**
