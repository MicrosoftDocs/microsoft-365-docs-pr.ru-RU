# <a name="attack-simulator-in-atp"></a>Имитатор атак в ATP

Если в вашей организации используется план 2 Office 365 Advanced Threat Protection (ATP) (план 2), который включает в себя средства анализа угроз и [реагирования на них,](office-365-ti.md)вы можете использовать эмулятор атак в Центре соответствия требованиям безопасности & для запуска реализованных атак в вашей организации. Эти смоделированные атаки помогают идентифицировать и находить уязвивых пользователей, прежде чем на снизу вы не сможете взломать на вашу снизу. Прочитайте эту статью для получения дополнительных сведений.

## <a name="what-do-you-need-to-know-before-you-begin"></a>Что нужно знать перед началом работы

- Чтобы открыть Центр безопасности и соответствия требованиям, перейдите по ссылке <https://protection.office.com/>. Имитатор атак доступен в **имитаторе атак на** \> **управлении угрозами.** Перейдите непосредственно к имитатору атак, <https://protection.office.com/attacksimulator> откройте.

- Дополнительные сведения о доступности имитатора атак в различных подписках на Microsoft 365 см. в [описании службы Office 365 Advanced Threat Protection.](https://docs.microsoft.com/office365/servicedescriptions/office-365-advanced-threat-protection-service-description)

- Вы должны быть членом группы **ролей "Управление организацией"** **или "Администратор** безопасности". Дополнительные сведения о группах ролей в Центре безопасности и соответствия требованиям см. в статье [Разрешения в Центре безопасности и соответствия требованиям](permissions-in-the-security-and-compliance-center.md).

- Для создания кампаний и управления ими в симуляторе атак ваша учетная запись должна быть настроена для использования многофакторной проверки подлинности (MFA). Инструкции см. в статье ["Настройка многофакторной проверки подлинности".](https://docs.microsoft.com/microsoft-365/admin/security-and-compliance/set-up-multi-factor-authentication)

- Фишинговые кампании собирают и обрабатывают события в течение 30 дней. Данные исторических кампаний будут доступны в течение 90 дней после запуска кампании.

- Соответствующих командлетов PowerShell для имитатора атак не существует.

## <a name="spear-phishing-campaigns"></a>Развитые фишинговые кампании

*Фишинг — это общий* термин атак, которые старались украсть конфиденциальную информацию в сообщениях, кажущихся как надежных, так и доверенных отправителей. *Зеркальные фишинг —* это целевая фишинговая атака, использующая сфокусированное и настраиваемое содержимое, специально подобранное указанным получателям (обычно после получения получателей злоумышленником).

В симуляторе атак доступны два различных типа специализированных фишинговых кампаний:

- **Земление от радозии (свяжение**с учетными данными).Атака пытается синхронизировать получателей для перехода по URL-адресу в сообщении. Если пользователь перейдет по ссылке, ему будет отображен запрос на ввод учетных данных. В этом случае нужно перенестись в одно из следующих расположений:

  - Страница по умолчанию, в которой описывается, что это всего лишь тест и ее внимание раскрыто.

    ![Что пользователь видит, щелкнут ссылку для фишинга, и вводят свои учетные данные](../../media/attack-simulator-phishing-result.png)

  - Указанная пользовательская страница (URL-адрес).

- **Проверка с ветвления фишинга (вложение).** Атака пытается сообщить получателям, чтобы открыть в сообщении вложение DOCX или PDF. Вложение содержит то же содержимое, которое содержит стандартную фишинговую ссылку, но первое предложение начинается с " , вы видите это сообщение как недавнее сообщение электронной почты, которое \<Display Name\> вы открывали...".

> [!NOTE]
> В настоящее время срок действия разностных фишинговых кампаний в имитаторе атак не ограничен.

### <a name="create-a-spear-phishing-campaign"></a>Создание спектральной фишинг-кампании

Важной частью любой из злоумышленников является внешний вид и функции сообщения, отправляемого целевым получателям. При создании и настройке электронного сообщения можно использовать следующие варианты:

- **Используйте встроенный шаблон электронной почты:** доступны два встроенных шаблона: **Prize Inaway (Высота и** **обновление зарплаты).** Вы можете дополнительно настраивать некоторые, все или не вообще ни одного из свойств электронной почты из шаблона при создании и запуске кампании.

- **Создайте повторно используемый шаблон электронной почты:** после создания и сохранения шаблона электронной почты можно снова использовать его в последующих кампаниях с фишинговыми кампаниями. Вы можете дополнительно настраивать некоторые, все или не вообще ни одного из свойств электронной почты из шаблона при создании и запуске кампании.

- **Создайте электронное сообщение с помощью мастера:** вы можете создать сообщение электронной почты прямо в мастере при создании и запуске специальной фишинга.

## <a name="password-attack-campaigns"></a>Кампании по атакам паролей

*Атака с* паролями пытается угадать пароли учетных записей пользователей в организации (как правило, после идентификации одной или нескольких допустимых учетных записей пользователя злоумышленником).

В симуляторе атак для проверки сложности паролей пользователей доступны два различных типа кампаний, указанных в специальности паролей.

- **Метод подбора пароля (атака перебором по словарю):** *атака методом* подбора или атака перебором по словарю использует большой файл словаря паролей на учетной записи пользователя с надежностью, что один из них будет работать (множество паролей для одной учетной записи). *dictionary* Неправильная блокировка пароля помогает проводить атаки методом подбора пароля.

  Для атак с словарем можно указать один или несколько паролей для проверки подлинности (вручную введенных или введенных в выгрузку файлах), а также одного или нескольких пользователей.

- **Атака распыления пароля:** *атаки распыления* пароля используют тот же пароль, что и в списке учетных записей пользователей (один пароль для многих учетных записей). Обнаружение атак методом подбора пароля усложняется обнаружение, чем подбор пароля (вероятность успешной резки при попыток перебора пароля злоумышленнику на несколько десятков или сотен учетных записей без риска неправильноблокировки пароля пользователя).

  Для атаки с распылежением пароля можно указать только один пароль, который можно попробовать, и можно указать одного или нескольких пользователей.

> [!NOTE]
> Атаки на пароли в Имитаторе атак передают запросы имени пользователя и сложную проверку подлинности пароля в конечную точку, поэтому они также работают с другими методами проверки подлинности (AD FS, синхронизация хэша паролей, сквозная проверка, PingFederate и т. д.). Для пользователей, для которых включена многофакторная идентификация, даже если атака пароля повторит попытку фактического пароля, эта попытка всегда будет регистрироваться как ошибка (другими словами, пользователи MFA никогда не будут отображаться в **подсчете** успешных попыток кампании). Это ожидаемый результат. MFA — это основной метод защиты от атак с паролем.

### <a name="create-and-launch-a-password-attack-campaign"></a>Создание и запуск кампании по атаке пароля

1. В Центре безопасности & требованиям откройте **эмулятор атак на** \> **управление угрозами.**

2. На странице **"Имитация атак"** выберите один из следующих вариантов в зависимости от типа создаваемой кампании.

   - В разделе **"Пароль перебора брюзов "Перебор пароля" (Атака перебором по словарю)** щелкните команду **Запускать атаки** или щелкните **"Сведения об** \> **атаке запуска".**

   - в разделе **атака с распылежением** пароля щелкните "Запуск **атаки"** или щелкните "Сведения **об атаке** \> **запуска".**

3. Мастер **настройки атак нановнова** запустится с новым всплывающим элементом. На **начальном этапе** введите уникальное отображаемое имя кампании и нажмите кнопку **"Далее".**

4. В **шаге "Конечные** пользователи" выполните одно из указанных ниже действий.

   - Щелкните **адресную** книгу, чтобы выбрать получателей (пользователей или группы) для кампании. У каждого получателя должен быть почтовый ящик Exchange Online. Если **щелкнуть Filter** **и применить без** ввода критериев поиска, все получатели возвращаются и добавляются в кампанию.

   - **Нажмите** **кнопку "Импорт файла",** чтобы импортировать файл с разделителями-запятыми или электронные адреса, разделенные запятыми. Каждая строка должна содержать электронный адрес получателя.

   По завершении нажмите кнопку **Далее**.

5. В **шаге "Выбор параметров атак"** выберите, что делать, в зависимости от типа кампании:

   - **Брюксор ное перебор пароля (Атака перебором по словарю):** выполните одно из следующих действий:

     - **Введите пароли вручную:** в **поле ввода пароля** введите пароль и нажмите клавишу ВВОД. Повторите этот шаг необходимое количество раз.

     - **Загрузите пароли из файла словаря:** нажмите "Отправить", чтобы импортировать существующий текстовый файл, содержащий по одному паролю в каждой строке, и пустая последняя строка. **Upload** Размер текстового файла не должен превышать 10 МБ и не может содержать более 30000 паролей.

   - **Атака с распылежением**пароля: **в паролях, которые необходимо использовать в атаке,** введите один пароль.

   По завершении нажмите кнопку **Далее**.

6. В **Confirm** шаге подтверждения **нажмите кнопку** "Готово", чтобы запустить кампанию. Указанные пароли применятся к указанным пользователям.

## <a name="view-campaign-results"></a>Просмотр результатов кампании

После запуска кампании можно проверить ход выполнения и результаты на главной странице **атак** "Смоделирование".

В активных кампаниях отображается строка состояния, выполненное значение процента и "(количество завершенных пользователей) количество пользователей)" (всего пользователей). При нажатии **кнопки** "Обновить" будет обновляться ход выполнения всех активных кампаний. Вы также можете **нажать кнопку "Завершение",** чтобы остановить активную кампанию.

После завершения кампании состояние "Атака **завершена".** Чтобы просмотреть результаты кампании, выполните одно из следующих действий.

- На главной **странице "Смоделирование атак"** нажмите кнопку **"Просмотреть отчет"** под именем кампании.

- На главной странице **моделировании атак щелкните** **"Сведения об атаке"** в разделе для типа атаки. На **открывшейся странице** "Сведения об атаке" выберите кампанию в разделе **"История атак".**

Любое из описанных выше действий позволит перенагуть страницу под названием "Сведения об **атаке".** Сведения, доступные на этой странице для каждого типа кампании, описаны в следующих разделах.

### <a name="spear-phishing-credentials-harvest-campaign-results"></a>Результаты кампании от радостивания отражения (Phishing)

Следующие сведения доступны на странице **сведений об атаке** для каждой кампании:

- Длительность кампании (дата/время начала и дата и время окончания).

- **Всего пользователей, нацеленных на**

- **Число пользователей,** которые щелкнули **and** ссылку и ввели свои учетные данные *(любое значение* имени пользователя и пароля).

- **Общая доля успешности:** процент, вычисленный успешными **попытками, идентифицированным**  /  **total users.**

- **Быстрый щелчок:** время, которое первый пользователь потребовал нажать на ссылку после запуска кампании.

- **Среднее число переходов:** сумма времени, деленная на количество пользователей, щелкнувших ссылку.

- **Показатель количества переходов:** процент, рассчитанный (количество пользователей, щелкнувших по ссылке) / **Всего пользователей, указанных.**

- **Быстрые учетные данные:** время, которое первый пользователь ввел свои учетные данные после запуска кампании.

- **Среднее количество учетных данных:** сумма времени, которое потребовалось всем ввести свои учетные данные на количество пользователей, которые ввели свои учетные данные.

- **Показатель успешности данных:** процент, рассчитанный по (числу пользователей, введенных в учетные данные) / **Общее количество пользователей, ориентированных на пользователей.**

- Линейчатая диаграмма, на которой **показаны нажатые параметры** **ссылки и указанные** числа ежедневно.

- График кругов, на котором показаны нажатая **ссылка,** **введенные**учетные данные и не **доли** этих процентов.

- В **разделе "Скомпрометированные** пользователи" перечислены сведения о пользователях, которые щелкнули ссылку:

  - Адрес электронной почты пользователя.

  - Дата и время перехода по ссылке.

  - IP-адрес клиента.

  - Сведения о версии Windows и веб-браузере пользователя.

  Вы можете **щелкнуть** "Экспорт", чтобы экспортировать результаты в CSV-файл.

### <a name="spear-phishing-attachment-campaign-results"></a>Результаты кампании от радостив (вложений)

Следующие сведения доступны на странице **сведений об атаке** для каждой кампании:

- Длительность кампании (дата/время начала и дата и время окончания).

- **Всего пользователей, нацеленных на**

- **Успешных**попыток: количество пользователей, открывавших или скачавших и открывших вложение (предварительная версия не учитывается).

- **Общая доля успешности:** процент, вычисленный успешными **попытками, идентифицированным**  /  **total users.**

- **Самое быстрое открытие вложения:** сколько времени потребовалось первому пользователю открыть вложение после запуска кампании.

- **Среднее время открытия вложения:** сумма времени, спомощьтельная продолжительность открытия вложения на количество пользователей, открывших вложение.

- **Частота успешности открытия**вложений: процент, рассчитанный (количество пользователей, открывших вложение) / **общее количество прогнозированных пользователей.**

### <a name="brute-force-password-dictionary-attack-campaign-results"></a>Результаты кампании пароля (атаки перебором по словарю)

Следующие сведения доступны на странице **сведений об атаке** для каждой кампании:

- Длительность кампании (дата/время начала и дата и время окончания).

- **Всего пользователей, нацеленных на**

- **Успешные**попытки: количество пользователей, которые были найдены по паролям.

- **Общая доля успешности:** процент, вычисленный успешными **попытками, идентифицированным**  /  **total users.**

- В **разделе "Скомпрометированные** пользователи" перечислены адреса электронной почты затронутых пользователей. Вы можете **щелкнуть** "Экспорт", чтобы экспортировать результаты в CSV-файл.

### <a name="password-spray-attack-campaign-results"></a>Результаты противодействия атакам, распыления пароля

Следующие сведения доступны на странице **сведений об атаке** для каждой кампании:

- Длительность кампании (дата/время начала и дата и время окончания).

- **Всего пользователей, нацеленных на**

- **Успешные**попытки: количество пользователей, которые обнаружили, используя указанный пароль.

- **Общая доля успешности:** процент, вычисленный успешными **попытками, идентифицированным**  /  **total users.**
