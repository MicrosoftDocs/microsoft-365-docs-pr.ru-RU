# <a name="application-guard-for-office-public-preview-for-admins"></a>Application Guard для Office (предварительная версия) для администраторов

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


**Применимо к:** Word, Excel и PowerPoint для Microsoft 365, Windows 10 Корпоративная

> [!IMPORTANT]
> Некоторые сведения относятся к предварительно выпущенного продукта, который может быть существенно изменен до его коммерческого выпуска. Microsoft makes no warranties, express or implied, with respect to the information provided here.

Application Guard в Microsoft Defender для Office (Application Guard для Office) помогает предотвратить доступ недоверенных файлов к доверенным ресурсам и защитить предприятие от новых и возникающих атак. В этой статье администраторы могут настраивать устройства для предварительного просмотра Application Guard для Office. В нем содержится информация о системных требованиях и действиях по установке, чтобы включить Application Guard для Office на устройстве.

## <a name="prerequisites"></a>Предварительные условия

### <a name="minimum-hardware-requirements"></a>Минимальные требования к оборудованию

* **ЦП:** 64-битный, 4 ядра (физический или виртуальный), расширения виртуализации (Intel VT-x ИЛИ AMD-V), эквивалент Core i5 или выше рекомендуется
* **Физическая память**: 8 ГБ ОЗУ
* **Жесткий диск**: 10 ГБ свободного места на системных дисках (рекомендуется использовать SSD)

### <a name="minimum-software-requirements"></a>Минимальные требования к программному обеспечению

* **Windows 10**: Windows 10 Корпоративная, сборка клиента версии 2004 (20H1) сборка 19041
* **Office**: office Beta Channel Build version 2008 16.0.13212 or later
* **Пакет обновления:** накопительные ежемесячные обновления для системы безопасности Windows 10 [KB4571756](https://support.microsoft.com/help/4571756/windows-10-update-KB4571756)

Подробные требования к системе можно найти в системных требованиях [для Application Guard в Microsoft Defender.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-application-guard/reqs-md-app-guard) Чтобы узнать больше о сборках Office Insider Preview, обратитесь к началу развертывания сборки [office Insider.](https://insider.office.com/business/deploy)

### <a name="licensing-requirements"></a>Требования к лицензированию

* Безопасность Microsoft 365 E5 или Microsoft 365 E5

## <a name="deploy-application-guard-for-office"></a>Развертывание Application Guard для Office

### <a name="enable-application-guard-for-office"></a>Включить Application Guard для Office

1. Скачайте и установите накопительные ежемесячные обновления для системы безопасности **Windows 10 KB4571756.**

2. Выберите **Application Guard в окнах** Windows и выберите "ОК".  Включение функции Application Guard вы запросит перезагрузку системы. Вы можете выбрать перезагрузку сейчас или после шага 3.

   ![Диалоговое окно "Функции Windows" с ag](../../media/ag03-deploy.png)

   Эту функцию также можно включить с помощью следующей команды PowerShell от администратора:

   ```powershell
   Enable-WindowsOptionalFeature -online -FeatureName Windows-Defender-ApplicationGuard
   ```

3. Найди групповую политику Application Guard в Microsoft Defender в управляемом режиме, расположенную на сайте "Административные шаблоны конфигурации компьютера" Для компонентов **Windows Application Guard в Microsoft \\ \\ \\ Defender.** Включив эту политику, заключив значение в области "Параметры" как **2** или **3,** выберите "ОК" **или** **"Применить".**

   ![Включить AG в управляемом режиме](../../media/ag04-deploy.png)

   Кроме того, можно настроить соответствующую политику CSP:

   > OMA-URI: **./Device/Vendor/MSFT/WindowsDefenderApplicationGuard/Settings/AllowWindowsDefenderApplicationGuard** <br> Тип данных: **Integer** <br> Значение: **2**

4. Перезагружает систему.

### <a name="confirm-that-application-guard-for-office-is-enabled-and-working"></a>Подтверждение включения и работы Application Guard для Office

Перед подтверждением включения Application Guard для Office запустите Word, Excel или PowerPoint на устройстве, где развернуты политики. Убедитесь, что Office активирован. Для активации продукта Office сначала может потребоваться использовать свое удостоверение.

Чтобы подтвердить, что Application Guard для Office включен, запустите Word, Excel или PowerPoint и откройте недопустимый документ. Например, вы можете открыть документ, загруженный из Интернета, или вложение электронной почты от кого-либо за пределами вашей организации.

## <a name="configure-application-guard-for-office"></a>Настройка Application Guard для Office

Office поддерживает следующие политики, позволяющие настроить возможности Application Guard для Office. Эти политики можно настроить с помощью групповых политик или службы облачных политик Office.

> [!NOTE]
> Эти политики станут доступны в ближайшее время.
> Кроме того, настройка этих политик может отключить некоторые функции для файлов, открытых в Application Guard для Office.

|Политика|Описание|
|---|---|
|Отключение Application Guard для Office|Включение этой политики принудительно приметит Word, Excel и PowerPoint к использованию контейнера изоляции защищенного просмотра вместо Application Guard для Office. Эту политику можно использовать для временного отключения Application Guard для Office при проблемах, не включаемой в Edge.|
|Отключение копирования и внесения для документов, открытых в Application Guard|Включение этой политики не позволит пользователю копировать и вкопировать содержимое из документа, открытого в Application Guard для Office, в документ, открытый за его пределами.|
|Запретить пользователям удалять защиту Application Guard для файлов|Включение этой политики удалит параметр (в приложении Office) отключить защиту Application Guard или открыть файл за пределами Application Guard. <p> **Примечание.** Пользователи по-прежнему могут обойти эту политику, вручную удалив из файла веб-свойство mark-of-web или перемещая документ в надежное расположение.|
|Ограничение печати из документов, открытых в Application Guard|Включение этой политики ограничивает принтеры, на которые пользователь может печатать, из файла, открытого в Application Guard для Office. Например, вы можете использовать эту политику, чтобы запретить пользователям печатать только в формате PDF.|
|Отключение доступа с камеры и микрофона к документам, открытым в Application Guard|Включение этой политики удалит доступ Office к камере и микрофону в Application Guard для Office.|
|

> [!NOTE]
> Чтобы в силу вступили в силу следующие политики, пользователю потребуется выйти из системы и повторно войти в Windows:
>
> * Отключение копирования и внесения для документов, открытых в Application Guard
> * Ограничение печати для документов, открытых в Application Guard
> * Отключение доступа камеры и микрофона к документам, открытым в Application Guard

## <a name="submit-feedback"></a>Отправить отзыв

### <a name="submit-feedback-via-feedback-hub"></a>Отправка отзывов через Центр отзывов

Если при запуске Application Guard для Office возникнут проблемы, рекомендуем отправить свой отзыв через Центр отзывов:

1. Откройте приложение **"Центр отзывов"** и вопишите в нее.

2. Если при запуске Application Guard вы  получите диалоговое окно ошибки, выберите "Отчет для Майкрософт" в диалоговом окте об ошибке, чтобы начать новую отправку отзывов. В противном случае выберите правильную категорию для Application Guard, а затем выберите +Добавить новую обратную связь <https://aka.ms/mdagoffice-fb> в правом верхнем направлении. 

3. Заполните поле **"Подвести итоги",** если оно еще не заполнено.

4. Заполните поле **"Объяснение"** более подробным описанием проблемы и действиями, которые вы выбрали, а затем выберите **"Далее".**

5. Выберите пузырьок рядом с проблемой. Убедитесь, что выбрана категория "Безопасность и конфиденциальность Application Guard в **\> Microsoft Defender — Office",** а затем выберите **"Далее".**

6. Выберите **"Новый отзыв"** и **"Далее".**

7. Соберйте трассировки по этой проблеме:

   1. Раз expand the **Recreate my problem** tile.

   2. Если проблема возникает во время работы Application Guard, откройте экземпляр Application Guard. Это позволяет собирать дополнительные трассировки из контейнера Application Guard.

   3. Выберите **"Начать запись"** и дождись остановки закрепления плитки и скажите *"Остановить запись".*

   4. Полностью воспроизведет проблему с помощью Application Guard. Это может включать попытку запуска экземпляра Application Guard и ожидание его сбой или повторение проблемы в запущенных экземплярах Application Guard.

   5. Выберите **плитку "Остановить запись".**

   6. Оохраняйте все запущенные экземпляры Application Guard открытыми даже через несколько минут после отправки, чтобы можно было собирать диагностику контейнера.

8. Вложите все соответствующие снимки экрана или файлы, связанные с проблемой.

9. Выберите **Отправить**.

### <a name="submit-feedback-via-office-customer-voice"></a>Отправка отзывов с помощью голосовой связи клиента Office

Вы также можете отправить отзыв из Office, если проблема происходит, когда документы Office открываются в Application Guard. Для отправки [отзывов обратитесь](https://insider.office.com/handbook) к справочнику по оценке пользователей Office.

Microsoft Defender for Endpoint — это платформа безопасности, призванная помочь корпоративным сетям предотвращать, обнаруживать, исследовать и реагировать на сложные угрозы. Дополнительные сведения об этой платформе можно и на странице ["Microsoft Defender for Endpoint".](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp) Узнайте больше о подмавке устройств на этой платформе на сайте [Onboard devices to the Microsoft Defender for Endpoint service.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/onboard-configure)

Вы также можете настроить Microsoft Defender для Office 365 для работы с Защитником для конечной точки. Ссылка на [интеграцию Защитника Office 365 с Microsoft Defender для конечной точки.](integrate-office-365-ti-with-wdatp.md)

## <a name="limitations-and-considerations"></a>Ограничения и соображения

* Application Guard для Office — это ограниченный режим, который изолирует недоверенные документы от доступа к доверенным корпоративным ресурсам, интрасети, удостоверению пользователя и произвольным файлам, присутствующим на компьютере. В результате, если пользователь попытается получить доступ к функции, которая имеет зависимость от такого доступа, например вставка изображения из локального файла на диск, он не сможет получить запрос, как по приведенной ниже. Чтобы включить недоверенный документ для доступа к надежным ресурсам, пользователи должны удалить защиту Application Guard из документа.

  ![Диалоговое окно со словами о том, что эта функция недоступна для сохраняемой безопасности](../../media/ag10-limitations.png)

  > [!NOTE]
  > Рекомендуем пользователям удалять защиту, только если они доверяют файлу и его источнику или источнику.

* Активное содержимое в таких документах, как макрос и ActiveX элементов управления, отключено в Application Guard для Office. Чтобы включить активный контент, пользователям необходимо удалить защиту Application Guard.

* Неподтверченные файлы, открытые из сетевых общих файлов или файлов, доступ к которые был открыт из OneDrive, OneDrive для бизнеса или SharePoint Online из другой организации, открываются в Application Guard только для чтения. Пользователи могут сохранить локализованную копию таких файлов, чтобы продолжить работу в контейнере, или удалить защиту для непосредственной работы с исходным файлом.

* Файлы, защищенные с помощью управления правами на управление правами на данные (IRM), продолжают открываться в защищеном представлении.

* Все настройки приложений Office в Application Guard для Office не будут сохраняться после того, как пользователь выйдите из приложения и снова войдите в систему или перезагружает устройство.

* Только средства accessibility, которые используют структуру UIA, могут предоставлять доступ к файлам, открытым в Application Guard для Office.

* Для первого запуска Application Guard после установки требуется сетевое подключение. Это необходимо, чтобы Application Guard проверил лицензию.

* В разделе информации документа свойство *Last Modified By* может отображать WDAGUtilityAccount в качестве пользователя. Это анонимный пользователь, настроенный в Application Guard, учитывая, что удостоверение пользователя рабочего стола не является общим в контейнере Application Guard.

## <a name="performance-optimizations-for-application-guard"></a>Оптимизация производительности для Application Guard

В этом разделе представлен обзор оптимизации производительности, используемой в Application Guard для Office. Эти сведения могут помочь администраторам диагностировать отчеты от пользователей, связанные с производительностью Office или всей системы при включаемом Application Guard.

Application Guard использует виртуализированный контейнер для изоляции недоверных документов от системы. Процесс создания контейнера и настройки контейнера Application Guard для открытия документов Office имеет накладные расходы на производительность, которые могут негативно повлиять на работу пользователей, когда пользователи открывают недоверный документ.

Чтобы предоставить пользователям ожидаемое открытие файлов, Application Guard использует логику для предварительного создания контейнера, если в системе соблюдены следующие действия: пользователь открыл файл в защищенном представлении или Application Guard за последние 28 дней.

При этом Office предварительно создаст контейнер Application Guard для пользователя после входа в Windows. Во время выполнения этой предварительной операции производительность системы может замедлиться. Это будет разрешаться сразу после завершения операции.

> [!NOTE]
> Подсказки, необходимые для heuristic, используемого для предварительного создания контейнера, создаются приложениями Office по мере их использования пользователем. Если пользователь устанавливает Office в новой системе, в которой включен Application Guard, Office не будет предварительно создавать контейнер до тех пор, пока пользователь впервые не откроет недоверчивый документ в системе. Пользователь будет наблюдать, что открытие первого файла в Application Guard занимает больше времени.

## <a name="known-issues-in-preview"></a>Известные проблемы в предварительной версии

* При нажатии на веб-ссылки `http` `https` (или) браузер не открывается.
* Обновления .NET приводят к сбойу открытия файлов в Application Guard. В качестве обходного решения пользователи могут перезагрузить свое устройство при этой проблеме. Узнайте больше о проблеме при получении сообщения об ошибке при попытке открыть Защитник Windows [Application Guard или песочницу Windows.](https://support.microsoft.com/help/4575917/receiving-an-error-message-when-attempting-to-open-windows-defender-ap)
