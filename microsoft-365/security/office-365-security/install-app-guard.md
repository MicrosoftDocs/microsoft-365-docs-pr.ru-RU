# <a name="application-guard-for-office-for-admins"></a>Application Guard для Office для администраторов

**Применимо к:** Word, Excel и PowerPoint для Microsoft 365, Windows 10 Корпоративная

Application Guard в Microsoft Defender для Office (Application Guard для Office) помогает предотвратить доступ недоверенных файлов к доверенным ресурсам и защитить предприятие от новых и возникающих атак. В этой статье данная статья посвящена настройке устройств для предварительного просмотра Application Guard для Office. В нем содержится информация о системных требованиях и действиях по установке, чтобы включить Application Guard для Office на устройстве.

## <a name="prerequisites"></a>Предварительные условия

### <a name="minimum-hardware-requirements"></a>Минимальные требования к оборудованию

* **ЦП:** 64-битный, 4 ядра (физический или виртуальный), расширения виртуализации (Intel VT-x ИЛИ AMD-V), эквивалент Core i5 или выше рекомендуется
* **Физическая память**: 8 ГБ ОЗУ
* **Жесткий диск**: 10 ГБ свободного места на системных дисках (рекомендуется использовать SSD)

### <a name="minimum-software-requirements"></a>Минимальные требования к программному обеспечению

* **Windows 10**: Windows 10 Корпоративная, сборка клиента версии 2004 (20H1) сборка 19041 или более поздней
* **Office**: office Current Channel Build 2011 16.0.13530.10000 или более поздней версии
* **Пакет обновления:** накопительное ежемесячное обновление для системы безопасности Windows 10 [KB4571756](https://support.microsoft.com/help/4571756/windows-10-update-KB4571756)

Подробные требования к системе можно найти в системных требованиях [для Application Guard в Microsoft Defender.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-application-guard/reqs-md-app-guard) Дополнительные сведения о каналах обновления Office см. в обзоре каналов обновления [для Microsoft 365.](https://docs.microsoft.com/deployoffice/overview-update-channels)

### <a name="licensing-requirements"></a>Требования к лицензированию

* Безопасность Microsoft 365 E5 или Microsoft 365 E5

## <a name="deploy-application-guard-for-office"></a>Развертывание Application Guard для Office

### <a name="enable-application-guard-for-office"></a>Включить Application Guard для Office

1. Скачайте и установите накопительные ежемесячные обновления для системы безопасности **Windows 10 KB4571756.**

2. Выберите **Application Guard в окнах** Windows и выберите "ОК".  Включение функции Application Guard вы запросит перезагрузку системы. Вы можете выбрать перезагрузку сейчас или после шага 3.

   ![Диалоговое окно "Функции Windows" с ag](../../media/ag03-deploy.png)

   Эту функцию также можно включить с помощью следующей команды PowerShell от администратора:

   ```powershell
   Enable-WindowsOptionalFeature -online -FeatureName Windows-Defender-ApplicationGuard
   ```

3. Поиск Application **Guard в Microsoft Defender в управляемом** режиме , групповая политика в административных шаблонах конфигурации компьютера **\\ \\ Компоненты Windows Application Guard в Microsoft \\ Defender.** Включив эту политику, заключив значение в области "Параметры" как **2** или **3,** а затем выбрав "ОК" **или** **"Применить".**

   ![Включить AG в управляемом режиме](../../media/ag04-deploy.png)

   Вместо этого можно настроить соответствующую политику CSP:

   > OMA-URI: **./Device/Vendor/MSFT/WindowsDefenderApplicationGuard/Settings/AllowWindowsDefenderApplicationGuard** <br> Тип данных: **Integer** <br> Значение: **2**

4. Перезапустите систему.

### <a name="confirm-that-application-guard-for-office-is-enabled-and-working"></a>Подтверждение включения и работы Application Guard для Office

Перед подтверждением включения Application Guard для Office запустите Word, Excel или PowerPoint на устройстве, где развернуты политики. Убедитесь, что Office активирован. Для активации продукта Office сначала может потребоваться использовать свое удостоверение.

Чтобы подтвердить, что Application Guard для Office включен, запустите Word, Excel или PowerPoint, а затем откройте недопустимый документ. Например, можно открыть документ, загруженный из Интернета, или вложение электронной почты от человека за пределами вашей организации.

При первом открываемом неподтверщенном файле может появиться экран-заставка Office, как в следующем примере. Оно может отображаться в течение некоторого времени во время активации Application Guard для Office и открытия файла. Последующие открытия недоверных файлов должны быть быстрее.

![Экран-заставка приложения Office](../../media/ag08-confirm.png)

* Вызов на ленте

  ![Doc-файл с небольшой заметкой App Guard](../../media/ag09-confirm.png)

* Значок приложения с экраном на панели задач

## <a name="configure-application-guard-for-office"></a>Настройка Application Guard для Office

Office поддерживает следующие политики, позволяющие настроить возможности Application Guard для Office. Эти политики можно настроить с помощью групповых политик или службы облачных политик Office.

> [!NOTE]
> Настройка этих политик может отключить некоторые функции для файлов, открытых в Application Guard для Office.

|Политика|Описание|
|---|---|
|Не используйте Application Guard для Office|Включение этой политики принудительно приметит Word, Excel и PowerPoint к использованию контейнера изоляции защищенного просмотра вместо Application Guard для Office. Эту политику можно использовать для временного отключения Application Guard для Office при проблемах, не включаемой в Microsoft Edge.|
|Настройка Application Guard для предварительного создания контейнера Office|Эта политика определяет, предварительно ли создан контейнер Application Guard для Office для изоляции недоверных файлов для повышения производительности во время выполнения. Если вы встроили этот параметр, вы можете указать количество дней для продолжения предварительного создания контейнера или позволить встроенной ауристической службе Office предварительно создать контейнер.
|Не разрешайте копирование и ветвь для документов Office, открытых в Application Guard для Office|Включение этой политики не позволит пользователю скопировать и вкопировать содержимое из документа, открытого в Application Guard для Office, в документ, открытый вне него.|
|Отключение аппаратного ускорения в Application Guard для Office|Эта политика контролирует, использует ли Application Guard для Office аппаратное ускорение для отрисовки графики. Если включить этот параметр, Application Guard для Office будет использовать программную отрисовку (ЦП) и не будет загружать сторонние графические драйверы или взаимодействовать с подключенным графическим оборудованием.
|Отключение защиты неподключаемой защиты типов файлов в Application Guard для Office|Эта политика контролирует, будет ли Application Guard для Office блокировать открытие неподтверченных типов файлов или будет ли она включить перенаправление в защищенный просмотр.
|Отключение доступа с камеры и микрофона для документов, открытых в Application Guard для Office|Включение этой политики удалит доступ Office к камере и микрофону в Application Guard для Office.|
|Ограничение печати из документов, открытых в Application Guard для Office|Включение этой политики ограничивает принтеры, в которые пользователь может печатать, из файла, открытого в Application Guard для Office. Например, вы можете использовать эту политику, чтобы запретить пользователям печатать только в формате PDF.|
|Запретить пользователям удалять защиту Application Guard для Office для файлов|Включение этой политики удалит параметр (в приложении Office) отключить защиту Application Guard для Office или открыть файл за пределами Application Guard для Office. <p> **Примечание.** Пользователи по-прежнему могут обходить эту политику, вручную удалив из файла веб-свойство пометки или перемещая документ в надежное расположение.|
|

> [!NOTE]
> Следующие политики требуют от пользователя выйти и снова войти в Windows, чтобы вступил в силу:
>
> * Отключение копирования и внесения для документов, открытых в Application Guard для Office
> * Ограничение печати для документов, открытых в Application Guard для Office
> * Отключение доступа камеры и микрофона к документам, открытым в Application Guard для Office

## <a name="submit-feedback"></a>Отправить отзыв

### <a name="submit-feedback-via-feedback-hub"></a>Отправка отзывов через Центр отзывов

Если при запуске Application Guard для Office возникнут проблемы, рекомендуем отправить свой отзыв через Центр отзывов:

1. Откройте приложение **"Центр отзывов" и** во sign in.

2. Если при запуске Application Guard вы  получите диалоговое окно ошибки, выберите "Отчет для Майкрософт" в диалоговом окте об ошибке, чтобы начать новую отправку отзывов. В противном случае выберите правильную категорию для Application Guard, а затем выберите "Добавить <https://aka.ms/mdagoffice-fb> **+ &nbsp; новую** обратную связь" в правом верхнем направлении.

3. Введите сводку в поле **"Сводка** по отзывам", если она еще не заполнена.

4. Введите подробное описание проблемы и действия, которые вы  выбрали в поле "Объяснение" в более подробном поле, а затем выберите **"Далее".**

5. Выберите пузырьок рядом с **проблемой.** Убедитесь, что выбрана категория "Безопасность и конфиденциальность Application Guard в **\> Microsoft Defender — Office",** а затем выберите **"Далее".**

6. Выберите **"Новый отзыв"** и **"Далее".**

7. Соберйте трассировки по этой проблеме:

   1. Раз expand the **Recreate my problem** tile.

   2. Если проблема возникает во время работы Application Guard, откройте экземпляр Application Guard. Открытие экземпляра позволяет собирать дополнительные трассировки из контейнера Application Guard.

   3. Выберите **"Начать запись"** и дождись остановки закрепления плитки и скажите *"Остановить запись".*

   4. Полностью воспроизведет проблему с помощью Application Guard. Воспроизведение может включать попытку запуска экземпляра Application Guard и ожидание его сбой или воспроизведение проблемы в запущенных экземплярах Application Guard.

   5. Выберите **плитку "Остановить запись".**

   6. Оохраняйте все запущенные экземпляры Application Guard открытыми даже в течение нескольких минут после отправки, чтобы можно было собирать диагностику контейнеров.

8. Вложите все соответствующие снимки экрана или файлы, связанные с проблемой.

9. Выберите **Отправить**.

### <a name="submit-feedback-via-office-customer-voice"></a>Отправка отзывов с помощью голосовой связи клиента Office

Вы также можете отправить отзыв из Office, если проблема произойдет, когда документы Office открываются в Application Guard. Для отправки [отзывов обратитесь](https://insider.office.com/handbook) к справочнику по оценке пользователей Office.

## <a name="integration-with-microsoft-defender-for-endpoint-and-microsoft-defender-for-office-365"></a>Интеграция с Microsoft Defender для конечной точки и Microsoft Defender для Office 365

Application Guard для Office интегрирован с Microsoft Defender для конечной точки, чтобы обеспечить мониторинг и оповещение о вредоносных действиях, которые происходят в изолированной среде.

Microsoft Defender for Endpoint — это платформа безопасности, призванная помочь корпоративным сетям предотвращать, обнаруживать, исследовать и реагировать на сложные угрозы. Дополнительные сведения об этой платформе см. в [microsoft Defender for Endpoint.](https://www.microsoft.com/microsoft-365/windows/microsoft-defender-atp) Дополнительные узнать о подмавке устройств на этой платформе см. в подмывке устройств к [службе Microsoft Defender для конечных точек.](https://docs.microsoft.com/windows/security/threat-protection/microsoft-defender-atp/onboard-configure)

Вы также можете настроить Microsoft Defender для Office 365 для работы с Защитником для конечной точки. Дополнительные сведения: "Интеграция [Защитника для Office 365 с Microsoft Defender для конечной точки".](integrate-office-365-ti-with-wdatp.md)

## <a name="limitations-and-considerations"></a>Ограничения и соображения

* Application Guard для Office — это ограниченный режим, изолирует недоверенные документы, чтобы они не могли получить доступ к доверенным корпоративным ресурсам, интрасети, удостоверению пользователя и произвольным файлам на компьютере. В результате, если пользователь попытается получить доступ к функции, которая зависит от такого доступа ( например, вставка изображения из локального файла на диск), доступ будет неудачным и будет выводить запрос, как в следующем примере. Чтобы включить недоверенный документ для доступа к надежным ресурсам, пользователи должны удалить защиту Application Guard из документа.

  ![Диалоговое окно со словами о том, что эта функция недоступна для сохраняемой безопасности](../../media/ag10-limitations.png)

  > [!NOTE]
  > Рекомендуем пользователям удалять защиту, только если они доверяют файлу и его источнику или источнику.

* Активное содержимое в таких документах, как макрос и ActiveX элементов управления, отключено в Application Guard для Office. Чтобы включить активный контент, пользователям необходимо удалить защиту Application Guard.

* Неподтверченные файлы из сетевых общих папк или файлов, к которые был общий доступ из OneDrive, OneDrive для бизнеса или SharePoint Online из другой организации, открываются в Application Guard только для чтения. Пользователи могут сохранить локализованную копию таких файлов, чтобы продолжить работу в контейнере, или удалить защиту для непосредственной работы с исходным файлом.

* Файлы, защищенные с помощью управления правами на доступ к данным (IRM), блокируются по умолчанию. Если пользователи хотят открыть такие файлы в защищенном представлении, администратор должен настроить параметры политики для неподтверганных типов файлов для организации.

* Любые настройки приложений Office в Application Guard для Office не будут сохраняться после того, как пользователь снова выходит из приложения и выходит из нее, или после перезапуска устройства.

* Только средства accessibility, которые используют структуру UIA, могут предоставлять доступ к файлам, открытым в Application Guard для Office.

* Для первого запуска Application Guard после установки требуется сетевое подключение. Для проверки лицензии Application Guard требуется подключение.

* В разделе информации документа свойство *Last Modified By* может отображать **WDAGUtilityAccount** в качестве пользователя. WDAGUtilityAccount — это анонимный пользователь, настроенный в Application Guard. Удостоверение пользователя рабочего стола не является общим в контейнере Application Guard.

## <a name="performance-optimizations-for-application-guard-for-office"></a>Оптимизация производительности для Application Guard для Office

В этом разделе представлен обзор оптимизации производительности, используемой в Application Guard для Office. Эти сведения могут помочь администраторам диагностировать отчеты от пользователей, связанные с производительностью Office или всей системы при включаемом Application Guard.

Application Guard использует виртуализированный контейнер для изоляции недоверных документов от системы. Процесс создания контейнера и настройки контейнера Application Guard для открытия документов Office имеет накладные расходы на производительность, которые могут негативно повлиять на работу пользователей, когда пользователи открывают недоверный документ.

Чтобы предоставить пользователям ожидаемое открытие файлов, Application Guard использует логику для предварительного создания контейнера, если в системе соблюдены следующие действия: пользователь открыл файл в защищенном представлении или Application Guard за последние 28 дней.

При этом Office предварительно создаст контейнер Application Guard для пользователя после его вход в Windows. Во время выполнения этой предварительной операции производительность системы может замедлиться, но эффект будет разрешаться сразу после завершения операции.

> [!NOTE]
> Подсказки, необходимые для предварительного создания контейнера, создаются приложениями Office по мере их использования пользователем. Если пользователь устанавливает Office в новой системе, в которой включен Application Guard, Office не будет предварительно создавать контейнер до тех пор, пока пользователь впервые не откроет недоверчивый документ в системе. Пользователь будет наблюдать, что открытие первого файла в Application Guard занимает больше времени.

## <a name="known-issues"></a>Известные проблемы

* При выборе веб-ссылок `http` (или) браузер `https` не открывается.
* В настоящее время не поддерживается вкладка содержимого или изображений в формате RTF в документы Office, открытые с помощью Application Guard.
* Обновления .NET приводят к сбойу открытия файлов в Application Guard. В качестве обходного решения пользователи могут перезапустить свое устройство, когда они поймят об этом сбое. Узнайте больше о проблеме при получении сообщения об ошибке при попытке открыть Защитник Windows [Application Guard или песочницу Windows.](https://support.microsoft.com/help/4575917/receiving-an-error-message-when-attempting-to-open-windows-defender-ap)
