# <a name="set-up-safe-links-policies-in-microsoft-defender-for-office-365"></a>Настройка политик Сейф ссылок в Microsoft Defender для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Microsoft Defender для Office 365 (план 1 и план 2)](defender-for-office-365.md)
- [Microsoft 365 Defender](../defender/microsoft-365-defender.md)

> [!IMPORTANT]
> Эта статья предназначена для бизнес-клиентов, использующих [Microsoft Defender для Office 365](defender-for-office-365.md). Если вы домашний пользователь, который ищет сведения о Safelinks в Outlook, см. в [Outlook.com.](https://support.microsoft.com/office/882d2243-eab9-4545-a58a-b36fee4a46e2)

Сейф Ссылки — это функция в Microsoft Defender для Office 365, которая обеспечивает сканирование [URL-адресов](defender-for-office-365.md) входящие сообщения электронной почты в потоке почты, а также время проверки URL-адресов и ссылок в сообщениях электронной почты и в других местах. Дополнительные сведения см. [в Сейф Ссылки в Microsoft Defender для Office 365.](safe-links.md)

Не существует политики встроенных или Сейф ссылок. Чтобы получить Сейф ссылки для сканирования URL-адресов, необходимо создать одну или несколько Сейф ссылок, как описано в этой статье.

> [!NOTE]
> Вы настраивает глобальные параметры защиты Сейф **ссылок** за пределами политик Сейф ссылки. Инструкции см. в перенастройке глобальных [параметров для Сейф ссылок в Microsoft Defender для Office 365.](configure-global-settings-for-safe-links.md)

Политики Сейф Ссылок можно настроить в Центре обеспечения безопасности & соответствия требованиям или в PowerShell (Exchange Online PowerShell для подходящих Microsoft 365 организаций с почтовыми ящиками в Exchange Online; автономных EOP PowerShell для организаций без Exchange Online почтовых ящиков, но с Microsoft Defender для Office 365 надстройки).

Основные элементы политики Сейф ссылки:

- Политика безопасных ссылок: включайте защиту Сейф ссылок, включайте сканирование URL-адресов в режиме реального времени, укажите, следует ли ждать завершения сканирования в режиме реального времени перед доставкой сообщения, включить сканирование внутренних сообщений, указать, следует ли отслеживать клики пользователя по URL-адресам, а также указать, следует ли разрешить пользователям щелкнуть корыто на исходный URL-адрес.
- **Правило безопасных ссылок:** указывает фильтры приоритета и получателя (к кому применяется политика).

> [!IMPORTANT]
> Администраторы должны учитывать различные параметры конфигурации для SafeLinks. Один из доступных вариантов — включить идентифицируемые сведения пользователей в SafeLinks. Эта функция позволяет *командам security Ops* исследовать потенциальный пользовательский компромисс, принимать меры по исправлению и ограничивать дорогостоящие нарушения.

Разница между этими двумя элементами не очевидна при управлении Сейф ссылками в Центре & безопасности:

- При создании политики Сейф ссылки создается правило безопасных ссылок и связанная политика безопасных ссылок одновременно с использованием одного и того же имени для обоих.
- При изменении политики Сейф ссылки параметры, связанные с именем, приоритетом, включенной или отключенной, а также фильтры получателей изменяют правило безопасных ссылок. Все остальные параметры изменяют связанную политику безопасных ссылок.
- При удалении политики Сейф ссылки правило безопасных ссылок и связанная политика безопасных ссылок удаляются.

В Exchange Online PowerShell или автономном EOP PowerShell вы управляете политикой и правилом отдельно. Дополнительные сведения см. в разделе Использование Exchange Online PowerShell или автономный [EOP PowerShell](#use-exchange-online-powershell-or-standalone-eop-powershell-to-configure-safe-links-policies) для настройки Сейф ссылок далее в этой статье.

## <a name="use-the-security--compliance-center-to-create-safe-links-policies"></a>Используйте Центр & безопасности для создания политик Сейф ссылок

Создание настраиваемой политики Сейф ссылок в Центре & безопасности создает правило безопасных ссылок и связанную политику безопасных ссылок одновременно с использованием одного и того же имени для обоих.

1. В Центре & безопасности перейдите к  политике управления угрозами \>  \> **ATP Сейф ссылки**.

2. На странице **Сейф ссылки** нажмите кнопку **Создать**.

3. Откроется **мастер политики Сейф** ссылки. На странице **Имя политики** настройте следующие параметры:

   - **Имя**. Укажите уникальное, понятное имя политики.

   - **Описание**. Укажите необязательное описание политики.

   По завершении нажмите кнопку **Далее**.

4. На **Параметры,** которая появится, настройте следующие параметры:

   - **Выберите действие для неизвестных** потенциально вредоносных URL-адресов в сообщениях: Выберите On, чтобы включить защиту Сейф ссылок для ссылок в сообщениях электронной почты. 

   - **Выберите действие для** неизвестных или потенциально вредоносных URL-адресов в Microsoft Teams : **Выберите** on, чтобы включить защиту Сейф ссылок для ссылок в Teams.

   - **Применение сканирования URL-адресов** в режиме реального времени для подозрительных ссылок и ссылок, которые указывают на файлы: Выберите этот параметр, чтобы включить сканирование ссылок в электронной почте в режиме реального времени.

   - **Дождись завершения** сканирования URL-адресов перед доставкой сообщения. Выберите этот параметр, чтобы дождаться завершения сканирования URL-адресов в режиме реального времени перед доставкой сообщения.

   - **Применяйте Сейф** ссылки на сообщения электронной почты, отправленные в организации: Выберите этот параметр, чтобы применить политику Сейф ссылок к сообщениям между внутренними отправителями и внутренними получателями.

   - **Не отслеживайте щелчки** пользователя: оставьте этот параметр незасеченным, чтобы включить щелчки пользователя отслеживания url-адресов в сообщениях электронной почты.

   - **Не позволяйте пользователям** щелкнуть исходный URL-адрес: Выберите этот параметр, чтобы заблокировать доступ пользователей к исходному URL-адресу на страницах [с предупреждением.](safe-links.md#warning-pages-from-safe-links)

   - **Не переписать** следующие URL-адреса: разрешить доступ к указанным URL-адресам, которые в противном случае будут заблокированы Сейф ссылками.

     В поле введите НУЖНЫЙ URL-адрес или значение, а затем нажмите кнопку ![Добавление значка кнопки](../../media/ITPro-EAC-AddIcon.png).

     Чтобы удалить существующую запись, выберите ее и нажмите кнопку ![Удаление значка кнопки](../../media/ITPro-EAC-DeleteIcon.png).

     Синтаксис записи см. в синтаксис Записи в списке "Не переписать следующие [URL-адреса".](safe-links.md#entry-syntax-for-the-do-not-rewrite-the-following-urls-list)

   Подробные сведения об этих параметрах см. в [Сейф ссылки](safe-links.md#safe-links-settings-for-email-messages) на сообщения электронной почты и [Сейф ссылки для Microsoft Teams.](safe-links.md#safe-links-settings-for-microsoft-teams)

   Дополнительные рекомендуемые значения для параметров стандартной и строгой политики см. в [Сейф параметров политики ссылок.](recommended-settings-for-eop-and-office365.md#safe-links-policy-settings)

   По завершении нажмите кнопку **Далее**.

5. На странице **Applied to** page, которая появится, определите внутренних получателей, к которые применяется политика.

   Условие или исключение можно использовать только один раз, но можно указать для них несколько значений. Указать несколько значений в одном условии или исключении можно с помощью оператора OR (например, _\<recipient1\>_ or _\<recipient2\>_). Между разными условиями и исключениями используется оператор AND (например, _\<recipient1\>_ and _\<member of group 1\>_).

   Нажмите **кнопку Добавить условие**. В отсеве, которое появится, выберите условие в **applied, если**:

   - **Получатель:** указывает один или несколько почтовых ящиков, пользователей почты или почтовых контактов в организации.
   - **Получатель является членом**: указывает одну или несколько групп в вашей организации.
   - **Домен получателя**: Указывает получателей в одном или нескольких настроенных принятых доменов в вашей организации.

   После выбора условия в любом из этих поле появится соответствующее **отсев.**

   - Щелкните в поле и прокрутите список значений для выбора.
   - Щелкните в поле и начните печатать, чтобы отфильтровать список и выбрать значение.
   - Чтобы добавить дополнительные значения, щелкните в пустой области в поле.
   - Чтобы удалить отдельные записи, **нажмите кнопку Удалить** ![ значок Удалить ](../../media/scc-remove-icon.png) значение.
   - Чтобы удалить все условие, нажмите **кнопку Удалить значок Удалить** ![ в ](../../media/scc-remove-icon.png) состоянии.

   Чтобы добавить дополнительное условие, нажмите **кнопку Добавить условие** и выберите оставшееся значение **в applied if**.

   Чтобы добавить исключения, нажмите **кнопку Добавить условие** и выберите исключение в соответствии **с исключением, если**. Параметры и поведение полностью идентичны условиям.

   По завершении нажмите кнопку **Далее**.

6. На странице **Обзор параметров,** которая отображается, просмотрите параметры. Чтобы изменить его, можно нажать **кнопку Изменить** каждый параметр.

   Закончив, нажмите кнопку **Готово**.

## <a name="use-the-security--compliance-center-to-modify-safe-links-policies"></a>Используйте Центр & безопасности для изменения политик Сейф ссылок

Чтобы включить или отключить политику или установить порядок приоритета политики, см. в следующих разделах.

### <a name="enable-or-disable-safe-links-policies"></a>Включить или отключить Сейф ссылки

1. В Центре & безопасности перейдите к  политике управления угрозами \>  \> **ATP Сейф ссылки**.

2. Обратите внимание на значение в **столбце Состояние:**

   - Чтобы отключить политику, переместите переключатель влево: ![Отключение политики](../../media/scc-toggle-off.png).

   - Чтобы включить политику, переместите переключатель вправо: ![Включаем политику](../../media/scc-toggle-on.png).

### <a name="set-the-priority-of-safe-links-policies"></a>Установите приоритет политик Сейф ссылки

По умолчанию политикам Сейф ссылки дается приоритет, основанный на порядке, в который они были созданы (более новые политики имеют более низкий приоритет по сравнению с более старыми политиками). Чем ниже значение, тем выше приоритет политики (0 — наивысший приоритет), а порядок обработки политик зависит от их приоритетов (политики с высоким приоритетом обрабатываются раньше, чем политики с низким приоритетом). Никакие две политики не могут иметь одинаковый приоритет, и обработка политики прекращается после применения первой политики.

Дополнительные сведения о приоритетах, а также оценке и применении нескольких политик см. в статье [Порядок и приоритет защиты электронной почты](how-policies-and-protections-are-combined.md).

Сейф Политики ссылок отображаются в порядке их обработки (первая политика имеет значение **Priority** 0).

> [!NOTE]
> В Центре & безопасности можно изменить приоритет политики Сейф ссылок после ее создания. В PowerShell можно переопредить приоритет по умолчанию при создании правила безопасных ссылок (которое может повлиять на приоритет существующих правил).

Чтобы изменить приоритет политики, переместите ее вверх или вниз в списке (в Центре безопасности и соответствия требованиям невозможно напрямую изменить значение свойства **Приоритет**).

1. В Центре & безопасности перейдите к  политике управления угрозами \>  \> **ATP Сейф ссылки**.

2. На странице **Сейф ссылки** выберите политику из списка и нажмите на нее (не выберите шажок).

3. В появились сведения о политике, щелкните доступную кнопку приоритета:

   - Политика Сейф ссылок с **значением Приоритет** **0** имеет только доступную кнопку **Приоритет уменьшения.**

   - Политика Сейф ссылок с наименьшим значением Приоритет (например, **3)** имеет только доступную кнопку  **Увеличение** приоритета.

   - Если у вас есть три или более Сейф политик ссылок, политики между самыми  высокими  и самыми низкими значениями приоритетов имеют доступные кнопки "Увеличение приоритета" и "Уменьшение приоритетов".

4. Щелкните **Увеличить приоритет или** уменьшить **приоритет,** чтобы изменить **значение Priority.**

5. Когда закончите, нажмите кнопку **Закрыть**.

## <a name="use-exchange-online-powershell-or-standalone-eop-powershell-to-configure-safe-links-policies"></a>Для настройки Сейф ссылок используйте Exchange Online PowerShell или автономный EOP PowerShell

Как описано выше, политика Сейф ссылок состоит из политики безопасных ссылок и правила безопасных ссылок.

В PowerShell очевидна разница между политиками безопасных ссылок и правилами безопасных ссылок. Вы управляете политиками безопасных ссылок с помощью **\* кодлетов -SafeLinksPolicy** и управляете правилами безопасных ссылок с помощью **\* cmdlets-SafeLinksRule.**

- В PowerShell сначала создается политика безопасных ссылок, а затем создается правило безопасных ссылок, которое определяет политику, к которую применяется правило.
- В PowerShell параметры политики безопасных ссылок и правила безопасных ссылок изменяются отдельно.
- При удалении политики безопасных ссылок из PowerShell соответствующее правило безопасных ссылок не удаляется автоматически, и наоборот.

### <a name="use-powershell-to-create-safe-links-policies"></a>Использование PowerShell для создания Сейф ссылок

Создание политики Сейф ссылок в PowerShell — это двухшаговая процедура:

1. Создание политики безопасных ссылок.
2. Создайте правило безопасных ссылок, которое указывает политику безопасных ссылок, к которую применяется правило.

> [!NOTE]
> 
> - Можно создать новое правило безопасных ссылок и назначить к ней существующую политику безопасных ссылок без связи. Правило безопасных ссылок не может быть связано с более чем одной политикой безопасных ссылок.
> 
> - Вы можете настроить следующие параметры для новых политик безопасных ссылок в PowerShell, недоступных в Центре обеспечения безопасности & до момента создания политики:
> 
>   - Создайте новую политику как отключенную _(включена_ `$false` в **комлете New-SafeLinksRule).**
>   - Задай приоритет политики во время создания _(priority)_ в _\<Number\>_ **комлете New-SafeLinksRule.**
> 
> - Новая политика безопасных ссылок, которую вы создаете в PowerShell, не отображается в Центре & безопасности, пока не назначите политику правилу безопасных ссылок.

#### <a name="step-1-use-powershell-to-create-a-safe-links-policy"></a>Шаг 1. Использование PowerShell для создания политики безопасных ссылок

Чтобы создать политику безопасных ссылок, используйте этот синтаксис:

```PowerShell
New-SafeLinksPolicy -Name "<PolicyName>" [-AdminDisplayName "<Comments>"] [-IsEnabled <$true | $false>] [-EnableSafeLinksForTeams <$true | $false>] [-ScanUrls <$true | $false>] [-DeliverMessageAfterScan <$true | $false>] [-EnableForInternalSenders <$true | $false>] [-DoNotAllowClickThrough <$true | $false>] [-DoNotTrackUserClicks <$true | $false>] [-DoNotRewriteUrls "Entry1","Entry2",..."EntryN"]
```

> [!NOTE]
> 
> - Подробные сведения о синтаксисе записи, используемом для параметра _DoNotRewriteUrls,_ см. в материале Запись синтаксиса для списка "Не переписывай следующие [URL-адреса".](safe-links.md#entry-syntax-for-the-do-not-rewrite-the-following-urls-list)
> 
> - Дополнительный синтаксис, который можно использовать для параметра _DoNotRewriteUrls_ при изменении существующих политик безопасных ссылок с помощью комлета **Set-SafeLinksPolicy,** см. в разделе [Использование PowerShell](#use-powershell-to-modify-safe-links-policies) для изменения политик безопасных ссылок в этой статье.

В этом примере создается политика безопасных ссылок с именем Contoso All со следующими значениями:

- Включайте сканирование URL-адресов и переписывание в сообщениях электронной почты.
- Включив сканирование URL Teams в режиме Teams (только для предварительного просмотра TAP).
- Включайте сканирование url-адресов в режиме реального времени, в том числе щелкнув ссылки, которые указывают на файлы.
- Дождись завершения сканирования URL-адресов перед доставкой сообщения.
- Включи сканирование URL-адресов и переописывание внутренних сообщений.
- Отслеживайте щелчки пользователей, связанные с защитой Сейф ссылок (мы не используем параметр _DoNotTrackUserClicks,_ а значение по умолчанию $false, что означает отслеживание кликов пользователей).
- Не позволяйте пользователям щелкнуть исходный URL-адрес.

```PowerShell
New-SafeLinksPolicy -Name "Contoso All" -IsEnabled $true -EnableSafeLinksForTeams $true -ScanUrls $true -DeliverMessageAfterScan $true -EnableForInternalSenders $true -DoNotAllowClickThrough $true
```

Подробные сведения о синтаксисах и параметрах см. [в обзоре New-SafeLinksPolicy.](/powershell/module/exchange/new-safelinkspolicy)

#### <a name="step-2-use-powershell-to-create-a-safe-links-rule"></a>Шаг 2. Использование PowerShell для создания правила безопасных ссылок

Чтобы создать правило безопасных ссылок, используйте этот синтаксис:

```PowerShell
New-SafeLinksRule -Name "<RuleName>" -SafeLinksPolicy "<PolicyName>" <Recipient filters> [<Recipient filter exceptions>] [-Comments "<OptionalComments>"] [-Enabled <$true | $false>]
```

В этом примере создается правило безопасных ссылок с именем Contoso All со следующими условиями:

- Правило связано с политикой безопасных ссылок с именем Contoso All.
- Правило применяется ко всем получателям в contoso.com домене.
- Так как мы не используем параметр _Priority,_ используется приоритет по умолчанию.
- Правило включено (мы не используем параметр _Включен,_ а значение по `$true` умолчанию).

```powershell
New-SafeLinksRule -Name "Contoso All" -SafeLinksPolicy "Contoso All" -RecipientDomainIs contoso.com
```

Подробные сведения о синтаксисах и параметрах см. [в обзоре New-SafeLinksRule.](/powershell/module/exchange/new-safelinksrule)

### <a name="use-powershell-to-view-safe-links-policies"></a>Использование PowerShell для просмотра политик безопасных ссылок


В этом примере возвращается сводный список всех политик безопасных ссылок.

```PowerShell
Get-SafeLinksPolicy | Format-Table Name
```

В этом примере возвращаются подробные сведения о политике безопасных ссылок с именем Contoso Executives.

```PowerShell
Get-SafeLinksPolicy -Identity "Contoso Executives"
```

Подробные сведения о синтаксисах и параметрах см. в [обзоре Get-SafeLinksPolicy.](/powershell/module/exchange/get-safelinkspolicy)

### <a name="use-powershell-to-view-safe-links-rules"></a>Использование PowerShell для просмотра правил безопасных ссылок

Чтобы просмотреть существующие правила безопасных ссылок, используйте следующий синтаксис:

```PowerShell
Get-SafeLinksRule [-Identity "<RuleIdentity>"] [-State <Enabled | Disabled] [| <Format-Table | Format-List> <Property1,Property2,...>]
```

В этом примере возвращается сводный список всех правил безопасных ссылок.

```PowerShell
Get-SafeLinksRule | Format-Table Name,State
```

```PowerShell
Get-SafeLinksRule -State Disabled
```

```PowerShell
Get-SafeLinksRule -State Enabled
```

В этом примере возвращается подробная информация для правила безопасных ссылок с именем Contoso Executives.

```PowerShell
Get-SafeLinksRule -Identity "Contoso Executives"
```

Подробные сведения о синтаксисах и параметрах см. в [обзоре Get-SafeLinksRule.](/powershell/module/exchange/get-safelinksrule)

### <a name="use-powershell-to-modify-safe-links-policies"></a>Использование PowerShell для изменения политик безопасных ссылок

В PowerShell нельзя переименовать политику безопасных ссылок (в **комлете Set-SafeLinksPolicy** нет _параметра Name)._ При переименовании политики Сейф ссылок в Центре & обеспечения безопасности переименуется только правило безопасных _ссылок._

Единственным дополнительным фактором для изменения политик безопасных ссылок в PowerShell является доступный синтаксис для параметра _DoNotRewriteUrls_ (список "Не переписать следующие [URL-адреса"):](safe-links.md#do-not-rewrite-the-following-urls-lists-in-safe-links-policies)

- Чтобы добавить значения, которые заменят существующие записи, используйте следующий синтаксис: `"Entry1","Entry2,..."EntryN"` .
- Чтобы добавить или удалить значения, не затрагивая другие существующие записи, используйте следующий синтаксис: `@{Add="Entry1","Entry2"...; Remove="Entry3","Entry4"...}`

В противном случае те же параметры доступны при создании политики безопасных ссылок, как описано в шаге [1: Использование PowerShell](#step-1-use-powershell-to-create-a-safe-links-policy) для создания раздела политики безопасных ссылок ранее в этой статье.

Чтобы изменить политику безопасных ссылок, используйте этот синтаксис:

```PowerShell
Set-SafeLinksPolicy -Identity "<PolicyName>" <Settings>
```

Подробные сведения о синтаксисах и параметрах см. в [инструкции Set-SafeLinksPolicy.](/powershell/module/exchange/set-safelinkspolicy)

### <a name="use-powershell-to-modify-safe-links-rules"></a>Использование PowerShell для изменения правил безопасных ссылок

Единственным параметром, недоступным при изменении правила безопасных ссылок  в PowerShell, является параметр Включен, который позволяет создать правило отключения. Чтобы включить или отключить существующие правила безопасных ссылок, см. в следующем разделе.

В противном случае те же параметры доступны при создании правила, описанного в шаге [2: Использование PowerShell](#step-2-use-powershell-to-create-a-safe-links-rule) для создания раздела правила безопасных ссылок в начале этой статьи.

Чтобы изменить правило безопасных ссылок, используйте этот синтаксис:

```PowerShell
Set-SafeLinksRule -Identity "<RuleName>" <Settings>
```

Подробные сведения о синтаксисах и параметрах см. в [инструкции Set-SafeLinksRule.](/powershell/module/exchange/set-safelinksrule)

### <a name="use-powershell-to-enable-or-disable-safe-links-rules"></a>Использование PowerShell для включения или отключения правил безопасных ссылок

Включение или отключение правила безопасных ссылок в PowerShell включает или отключает всю политику Сейф ссылок (правило безопасных ссылок и назначенную политику безопасных ссылок).

Чтобы включить или отключить правило безопасных ссылок в PowerShell, используйте этот синтаксис:

```PowerShell
<Enable-SafeLinksRule | Disable-SafeLinksRule> -Identity "<RuleName>"
```

В этом примере отключает правило безопасных ссылок с именем Marketing Department.

```PowerShell
Disable-SafeLinksRule -Identity "Marketing Department"
```

В этом примере включается то же правило.

```PowerShell
Enable-SafeLinksRule -Identity "Marketing Department"
```

Подробные сведения о синтаксисах и параметрах см. в [рублях Enable-SafeLinksRule](/powershell/module/exchange/enable-safelinksrule) и [Disable-SafeLinksRule.](/powershell/module/exchange/disable-safelinksrule)

### <a name="use-powershell-to-set-the-priority-of-safe-links-rules"></a>Используйте PowerShell, чтобы установить приоритет правил безопасных ссылок

Максимальный приоритет, который можно задать для правила, — 0. Минимальное значение зависит от количества правил. Например, если у вас есть пять правил, вы можете использовать значения 0-4. Изменение приоритета существующего правила оказывает каскадное влияние на другие правила. Например, если вы измените приоритет правила на 2, когда у вас есть пять настраиваемых правил (с приоритетами от 0 до 4), то для существующего правила с приоритетом 2 будет задан приоритет 3, а для правила с приоритетом 3 будет задан приоритет 4.

Чтобы установить приоритет правила безопасных ссылок в PowerShell, используйте следующий синтаксис:

```PowerShell
Set-SafeLinksRule -Identity "<RuleName>" -Priority <Number>
```

В этом примере для правила Marketing Department задается приоритет 2. Все правила с приоритетом не больше 2 понижаются на один ранг (значения приоритета увеличиваются на 1).

```PowerShell
Set-SafeLinksRule -Identity "Marketing Department" -Priority 2
```

> [!NOTE]
> Чтобы установить приоритет нового правила при его создании, используйте параметр _Priority_ в **комлете New-SafeLinksRule.**

Подробные сведения о синтаксисах и параметрах см. в [инструкции Set-SafeLinksRule.](/powershell/module/exchange/set-safelinksrule)

### <a name="use-powershell-to-remove-safe-links-policies"></a>Использование PowerShell для удаления политик безопасных ссылок

```PowerShell
Remove-SafeLinksPolicy -Identity "<PolicyName>"
```

В этом примере удаляется политика безопасных ссылок с именем Marketing Department.

```PowerShell

```

Подробные сведения о синтаксисах и параметрах см. в этой ссылке [Remove-SafeLinksPolicy.](/powershell/module/exchange/remove-safelinkspolicy)

### <a name="use-powershell-to-remove-safe-links-rules"></a>Использование PowerShell для удаления правил безопасных ссылок

При удалении правила безопасных ссылок с помощью PowerShell соответствующая политика безопасных ссылок не удаляется.

Чтобы удалить правило безопасных ссылок в PowerShell, используйте этот синтаксис:


В этом примере удаляется правило безопасных ссылок с именем Отдел маркетинга.

## <a name="how-do-you-know-these-procedures-worked"></a>Как проверить, что эти процедуры выполнены?

Чтобы убедиться, что вы успешно создали, изменили или удалили Сейф ссылки, сделайте все следующие действия:

- В Центре & безопасности перейдите к  политике управления угрозами \>  \> **ATP Сейф ссылки**. Проверка списка политик, их значений **состояния** и **их значений приоритета.** Чтобы просмотреть дополнительные сведения, выберите политику из списка и просмотреть сведения в поле вылета.

- В Exchange Online PowerShell или Exchange Online Protection PowerShell замените имя политики или правила, запустите следующую команду и проверьте \<Name\> параметры:

  ```PowerShell
  Get-SafeLinksPolicy -Identity "<Name>"
  ```

  ```PowerShell
  Get-SafeLinksRule -Identity "<Name>"
  ```
