# <a name="manage-quarantined-messages-and-files-as-an-admin-in-eop"></a>Управление сообщениями и файлами, помещенными в карантин, в качестве администратора в EOP

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Exchange Online Protection](exchange-online-protection-overview.md)
- [Microsoft Defender для Office 365 (план 1 и план 2)](office-365-atp.md)
- [Microsoft 365 Defender](../mtp/microsoft-threat-protection.md)

В организациях Microsoft 365 с почтовыми ящиками в Exchange Online или в автономных организациях Exchange Online Protection (EOP) без почтовых ящиков Exchange Online на карантине хранятся потенциально опасные или нежелательные сообщения. Дополнительные сведения см. в сообщениях электронной почты, отправленных на карантин, [в EOP.](quarantine-email-messages.md)

Администраторы могут просматривать, освободить и удалять все типы сообщений на карантине для всех пользователей. Только администраторы могут управлять сообщениями, которые были отправлены на карантин как вредоносные программы, фишинг с высокой достоверности или в результате правил потока почты (также известных как правила транспорта). Администраторы также могут сообщать о ложных срабатывах в корпорацию Майкрософт.

Администраторы организаций с Microsoft Defender для Office 365 также могут просматривать, скачивать и удалять файлы на карантине в SharePoint Online, OneDrive для бизнеса и Microsoft Teams.

Сообщения, отправленные на карантин, можно просматривать и управлять ими в Центре безопасности и & соответствия требованиям или в PowerShell (Exchange Online PowerShell для организаций Microsoft 365 с почтовыми ящиками в Exchange Online; автономный EOP PowerShell для организаций без почтовых ящиков Exchange Online).

## <a name="what-do-you-need-to-know-before-you-begin"></a>Что нужно знать перед началом работы

- Чтобы открыть Центр безопасности и соответствия требованиям, перейдите по ссылке <https://protection.office.com>. Чтобы открыть страницу Карантина напрямую, перейдите на <https://protection.office.com/quarantine>.

- Сведения о том, как подключиться к Exchange Online PowerShell, см. в статье [Подключение к Exchange Online PowerShell](https://docs.microsoft.com/powershell/exchange/connect-to-exchange-online-powershell). Чтобы подключиться к автономному EOP PowerShell, см. раздел [Подключение к PowerShell Exchange Online Protection](https://docs.microsoft.com/powershell/exchange/connect-to-exchange-online-protection-powershell).

- Для выполнения процедур, описанных в этой статье, вам должны быть назначены разрешения в Центре безопасности и соответствия требованиям:
  - Чтобы принять меры в отношении сообщений на карантине для всех пользователей, необходимо быть  членом группы ролей "Управление организацией", "Администратор безопасности" или "Администратор <sup>\*</sup> карантина".
  - Для доступа всех пользователей к сообщениям в карантине, доступных только для  чтения, необходимо быть членом группы ролей **"Глобальный** читатель" или "Читатель безопасности".

  Дополнительные сведения см. в статье [Разрешения в Центре безопасности и соответствия требованиям](permissions-in-the-security-and-compliance-center.md).

  **Примечания**.

  - Добавление пользователей в соответствующую роль Azure Active Directory в Центре безопасности Microsoft 365 предоставляет пользователям необходимые разрешения в Центре безопасности и соответствия требованиям _и_ разрешения для других функций в Microsoft 365. Дополнительные сведения см. в статье [О ролях администраторов](../../admin/add-users/about-admin-roles.md).
  - Группа ролей **Управление организацией с правами только на просмотр** в [Exchange Online](https://docs.microsoft.com/Exchange/permissions-exo/permissions-exo#role-groups) также предоставляет доступ только для чтения к этой функции.
  - <sup>\*</sup>Для процедур  карантина в Exchange Online PowerShell члены группы ролей администратора карантина также должны быть членами группы ролей "Управление санатурой" в [Exchange Online.](https://docs.microsoft.com/Exchange/permissions-exo/permissions-exo#role-groups) 

- Сообщения на карантине сохраняются в течение периода времени по умолчанию перед их автоматическим удалением:
  - 30 дней для сообщений, отправленных в карантин политиками защиты от нежелательной почты (спам, фишинг и массовая рассылка). Это значение по умолчанию и максимальное значение. Чтобы настроить (ниже) это значение, см. "Настройка политик борьбы [с нежелательной почтой".](configure-your-spam-filter-policies.md)
  - 15 дней для сообщений, содержащих вредоносные программы.
  - 15 дней для файлов, вложенных в карантин с помощью безопасных вложений для SharePoint, OneDrive и Microsoft Teams в Защитнике Office 365.

  По истечении срока действия сообщения из карантина восстановить его нельзя.

## <a name="use-the-security--compliance-center-to-manage-quarantined-email-messages"></a>Использование Центра безопасности & соответствия требованиям для управления сообщениями электронной почты, отправленными на карантин

### <a name="view-quarantined-email"></a>Просмотр электронной почты, отправленной на карантин

1. В центре безопасности и соответствия требованиям выберите **управление угрозами** \> **просмотр** \> **карантина**.

2. **Убедитесь, что для просмотра сообщения** электронной почты в карантине установлено значение по **умолчанию.**

3. Вы можете отсортировать результаты, нажав на доступный заголовок столбца. Нажмите кнопку **изменить столбцы**, чтобы отобразить не более семи столбцов. Значения по умолчанию помечены звездочкой (<sup>\*</sup>):

   - **Получено**<sup>\*</sup>
   - **Отправитель**<sup>\*</sup>
   - **Тема**<sup>\*</sup>
   - **Причина карантина**<sup>\*</sup>
   - **Выпущено?**<sup>\*</sup>
   - **Тип политики**<sup>\*</sup>
   - **Expires**
   - **Получатель**
   - **КОД сообщения**
   - **Имя политики**
   - **Размер**
   - **Направление**

   Когда все будет готово, нажмите кнопку **сохранить** или выберите пункт **назначить по умолчанию**.

4. Чтобы отфильтровать результаты, нажмите **Фильтр**. Доступные фильтры:

   - **Срок действия истекает**: Фильтрация сообщений по истечении срока действия из карантина.
     - **Сегодня**
     - **Следующие 2 дня**
     - **Следующие 7 дней**
     - **Настраиваемые**: введите **дату начала** и **конечную дату**.

   - **Время получения**: введите **дату начала** и **конечную дату**.

   - **Причина карантина**:
     - **Политика:** сообщение соответствует условиям правила потока почты (также известного как правило транспорта).
     - **Массовая рассылка**
     - **Фишинг**: решением  фильтра нежелательной почты было фишинговое сообщение или защита от фишинга, которое было отправлено на карантин сообщения [(параметры](set-up-anti-phishing-policies.md#spoof-settings) спуфинга или защита от [подмены).](set-up-anti-phishing-policies.md#impersonation-settings-in-anti-phishing-policies-in-microsoft-defender-for-office-365)
     - **Вредоносная программа**
     - **Спам**
     - **Фишинг с высокой достоверности**

   - **Тип политики**: фильтрация сообщений по типу политики:
     - **Политика борьбы с вредоносными программами**
     - **Политика безопасных вложений**
     - **Политика защиты от фишинга**
     - **Политика фильтрации размещенного содержимого** (политика защиты от нежелательной почты)
     - **Правило транспорта**

   - **Получатель электронной** почты: все пользователи или только сообщения, отправленные вам. Конечные пользователи могут управлять только отправленными на карантин сообщениями.

   Чтобы очистить фильтр, нажмите **Очистить**. Чтобы скрыть всплывающую подсказку фильтра, снова нажмите **Фильтр**.

5. Используйте **Сортировку результатов по** (кнопка **ID сообщения** по умолчанию) и соответствующее значение, чтобы найти конкретные сообщения. Подстановочные знаки не поддерживаются. Вы можете искать по следующим значениям:

   - **Идентификатор сообщения**. Глобальный уникальный идентификатор сообщения.

     Например, вы [](message-trace-scc.md) использовали трассировку сообщений для того, чтобы найти сообщение, отправленное пользователю в вашей организации, и определили, что сообщение было отправлено на карантин, а не доставлено. Не забудьте включить полное значение ИД сообщения, которое может включать угловые скобки ( \<\> ). Пример: `<79239079-d95a-483a-aacf-e954f592a0f6@XYZPR00BM0200.contoso.com>`.

   - **Адрес электронной почты отправителя**: адрес электронной почты одного отправителя.

   - **Имя политики**: используйте полное имя политики сообщения. При поиске регистр не учитывается.

   - **Адрес электронной почты получателя**: адрес электронной почты одного получателя.

   - **Тема**: Используйте всю тему сообщения. При поиске регистр не учитывается.

   - **Имя политики:** имя политики, ответственной за карантин сообщения.

   После того, как вы ввели критерии поиска, нажмите ![кнопку Обновить](../../media/scc-quarantine-refresh.png) **Обновить**, чтобы обновить результаты.

После того, как вы найдете конкретное сообщение на карантине, выберите сообщение, чтобы просмотреть сведения о нем и принять меры к нему (например, просмотреть, выпустить, загрузить или удалить сообщение).

#### <a name="view-quarantined-message-details"></a>Просмотр сведений о карантине

При выборе сообщения электронной почты в списке на всплывающей панели **Сведения** отображаются следующие сведения:

- **Идентификатор сообщения**: Глобальный уникальный идентификатор сообщения.

- **Адрес отправителя**

- **Получено**: Дата и время получения сообщения.

- **Тема**

- Причина карантина: показывает, было ли сообщение определено как нежелательное, массовое, фишинговое, соответствует правилу потока почты **(** правило транспорта) или определено как содержащее **вредоносную программу.** 

- **Количество получателей**

- **Получатели**: Если в сообщении содержится несколько получателей, вам необходимо нажать **Просмотр сообщения** или **Просмотреть заголовок сообщения**, чтобы просмотреть полный список получателей.

- **Истекает**: дата / время, когда сообщение будет автоматически и окончательно удалено из карантина.

- **Разблокировано для пользователей**. Все электронные адреса (если таковые имеются), для которых сообщение было освобождено.

- **Пока не выпустили**: все адреса электронной почты (если таковые имеются), на которые еще не было отправлено сообщение.

### <a name="take-action-on-quarantined-email"></a>Выполнение действий с электронной почтой в карантине

После выбора сообщения у вас есть несколько вариантов действий  с сообщениями во flyout области сведений:

- **Сообщение о выпуске:** в отобра следующей области выберите следующие параметры:

  - **Сообщить о сообщениях** в корпорацию Майкрософт для анализа: этот выбор выбран по умолчанию и сообщает о ошибочном срабатынии сообщения в карантине корпорации Майкрософт. Если сообщение было в карантине как нежелательное, массовое, фишинговое или содержащее вредоносную программу, сообщение также передается в службу анализа нежелательной почты Майкрософт. В зависимости от их анализа правила фильтрации нежелательной почты для всей службы могут быть настроены, чтобы разрешить сообщение.

  - Выберите один из следующих параметров:
    - **Освобождение сообщений для всех получателей**
    - **Освобождение сообщений для определенных получателей**
    - **Освобождать сообщения другим людям:** обратите внимание, что освобождение вредоносных сообщений для людей, кроме исходных получателей, не поддерживается.

  Когда все будет готово, нажмите кнопку **Выпуск сообщений**.

  Примечания об освобождении сообщений:

  - Вы не можете освободить сообщение для одного и того же получателя более одного раза.
  - В списке потенциальных получателей будут отображаться только те получатели, которые не получили сообщение.

- **Просмотр заголовка сообщения**: Выберите эту ссылку, чтобы просмотреть текст заголовка сообщения. Чтобы подробно проанализировать поля и значения заголовков, скопируйте текст заголовка сообщения в буфер обмена, а затем выберите **Microsoft Message Header Analyzer**, чтобы перейти к анализатору удаленных подключений (щелкните правой кнопкой мыши и выберите **Открыть на новой вкладке**, если вы не хотите закрывать Microsoft 365 для выполнения этой задачи). Вставьте заголовок сообщения на страницу в разделе «Анализатор заголовков сообщений» и выберите **Анализировать заголовки**:

- **Предварительный просмотр сообщения**: в появившейся всплывающей панели выберите один из следующих параметров:

  - **Исходный вид**: показывает HTML-версию тела сообщения со всеми отключенными ссылками.
  - **Текстовое представление**: показывает текст сообщения в виде простого текста.

- **Удаление из** карантина: после нажатия кнопки **"Да"** в отображемом предупреждении сообщение немедленно удаляется, не отправив его исходным получателям.

- **Загрузка сообщения**: В появившейся всплывающей панели выберите **Я понимаю риски, связанные с загрузкой этого сообщения**, чтобы сохранить локальную копию сообщения в формате .eml.

- **Отправка сообщения:** в отобра следующей области выберите следующие параметры:

  - **Тип объекта:** **электронная почта** (по умолчанию), **URL-адрес** или **вложение.**

  - **Формат отправки:** **сетевой** ИД сообщения (по  умолчанию с соответствующим значением в поле "ИД сетевого сообщения") или **"Файл"** (перейдите к локальному EML-файлу или MSG-файлу). Обратите внимание, что если **выбрать** файл и затем **выбрать** сетевой ИД сообщения, исходное значение исчезнет.

  - **Recipients**: Type at lease one original recipient of the message, or click **Select All** to identify all recipients. Можно также **щелкнуть "Выбрать все",** а затем выборочно удалить отдельных получателей.

  - **Причина отправки:** **не должна быть заблокирована (по** умолчанию) или **должна быть заблокирована.**

  По завершению нажмите кнопку **"Отправить".**

Если вы не отпустите или не удалите сообщение, оно будет удалено после истечения срока хранения по умолчанию.

#### <a name="take-action-on-multiple-quarantined-email-messages"></a>Принять меры к нескольким сообщениям в карантине

При выборе нескольких сообщений в карантине в списке (до 100) появляется всплывающая панель **Массовые действия**, в которой можно выполнить следующие действия:

- **Выпуск сообщений**: параметры те же, что и при выпуске одного сообщения, за исключением того, что вы не можете выбрать **Отпускать сообщения конкретным получателям**; Вы можете выбрать только **Выпустить сообщение всем получателям** или **Выпустить сообщения другим людям**.

  > [!NOTE]
  > Рассмотрим следующий сценарий: john@gmail.com отправляет сообщение faith@contoso.com и john@subsidiary.contoso.com. Gmail bifurcates this message into two copies that are both routed to quarantine as phishing in Microsoft. Администратор освобождает оба этих сообщения для admin@contoso.com. Доставляется первое выпущенное сообщение, которое достигает почтового ящика администратора. Второе выпущенное сообщение будет определено как дублирующееся и пропущено. Сообщения идентифицированы как дубликаты, если они имеют одинаковый ИД сообщения и получили время.

- **Удаление сообщений:** после нажатия кнопки **"Да"** в отображемом предупреждении сообщения немедленно удаляются, не отправив их исходным получателям.

Когда закончите, нажмите кнопку **Закрыть**.

## <a name="microsoft-defender-for-office-365-only-use-the-security--compliance-center-to-manage-quarantined-files"></a>Только Microsoft Defender для Office 365: управление файлами на карантине с помощью Центра безопасности & соответствия требованиям

> [!NOTE]
> Процедуры для файлов на карантине, которые находятся в этом разделе, доступны только для подписчиков Office 365 (план 1) и Плана 2 (Microsoft Defender).

В организациях с Защитником Office 365 администраторы могут управлять файлами на карантине в SharePoint Online, OneDrive для бизнеса и Microsoft Teams. Чтобы включить защиту для этих файлов, см. раздел "Включить безопасные вложения для [SharePoint, OneDrive и Microsoft Teams".](turn-on-atp-for-spo-odb-and-teams.md)

### <a name="view-quarantined-files"></a>Просмотр файлов на карантине

1. В центре безопасности и соответствия требованиям выберите **управление угрозами** \> **просмотр** \> **карантина**.

2. Измените **представление в карантине** на файлы **значений.** Можно отсортировать поле, щелкнув доступный заголок столбца.

3. Вы можете отсортировать результаты, нажав на доступный заголовок столбца. Нажмите кнопку **изменить столбцы**, чтобы отобразить не более семи столбцов. Столбцы по умолчанию помечены звездочкой ( <sup>\*</sup> ):

   - **Пользователь**<sup>\*</sup>
   - **Расположение**<sup>\*</sup>
   - **Имя файла**<sup>\*</sup>
   - **URL-адрес файла**<sup>\*</sup>
   - **Размер файла**<sup>\*</sup>
   - **Истекает**<sup>\*</sup>
   - **Выпущено?**<sup>\*</sup>
   - **Обнаружено**
   - **Изменено по времени**

4. Чтобы отфильтровать результаты, нажмите **Фильтр**. Доступные фильтры:

   - **Срок действия истекает**: Фильтрация сообщений по истечении срока действия из карантина.
     - **Сегодня**
     - **Следующие 2 дня**
     - **Следующие 7 дней**
     - Настраиваемый диапазон дат и времени.
   - **Получено время**
   - **Причина карантина:** единственное доступное значение — **вредоносная программа.**
   - **Тип политики**

После того как вы найдете определенный файл на карантине, выберите его, чтобы просмотреть сведения о нем и принять с него меры (например, просмотреть, освободить, скачать или удалить сообщение).

#### <a name="view-quarantined-file-details"></a>Просмотр сведений о файле в карантине

При выборе файла в списке в области "Сведения" отображаются следующие сведения о файле: 

- **Имя файла**
- **URL-адрес** файла: URL-адрес, который определяет расположение файла (например, в SharePoint Online).
- **Вредоносный контент, обнаруженный в** Дата и время, когда файл был карантин.
- **Истекает** срок действия: дата удаления файла из карантина.
- **Обнаружено :** Защитник Office 365 или антивредоносный механизм Майкрософт.
- **Выпущено?**
- **Имя вредоносной программы**
- **Идентификатор документа:** уникальный идентификатор документа.
- **Размер файла:** в килобайтах (КБ).
- **Организация** Уникальный ИД организации.
- **Дата последнего изменения**
- **Кем изменен**: пользователь, который последним изменил файл.
- Значение **secure Hash Algorithm 256-bit (SHA-256):** это значение можно использовать для идентификации файла в других хранилищах репутации или в других расположениях в вашей среде.

### <a name="take-action-on-quarantined-files"></a>Действие с файлами в карантине

При выборе файла в списке можно сделать следующие действия с  файлом во flyout области сведений:

- **Release files**: Select (default) or unselect **Report files to Microsoft for analysis**, and then click **Release files**.
- **Скачивание файла**
- **Удаление файла из карантина**

Если вы не освободите или не удалите файлы, они будут удалены по истечении срока хранения карантина по умолчанию.

#### <a name="actions-on-multiple-quarantined-files"></a>Действия с несколькими файлами на карантине

При выборе нескольких файлов на карантине в списке (до  100) появляется flyout pane массовых действий, где можно сделать следующие действия:

- **Выпуск файлов**
- **Удаление файлов:** после нажатия кнопки **"Да"** в от имени предупреждения файлы немедленно удаляются.
