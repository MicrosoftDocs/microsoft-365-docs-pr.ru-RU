# <a name="manually-submit-messages-to-microsoft-for-analysis"></a>Отправка сообщений корпорации Майкрософт для анализа вручную

> [!NOTE]
> Если вы являетесь администратором организации с почтовыми ящиками Exchange Online, рекомендуем использовать портал отправки в Центре безопасности & соответствия требованиям. Дополнительные сведения см. в статье ["Использование функции отправки администратора" для отправки подозрительного спама, фишинга, URL-адресов и файлов в корпорацию Майкрософт.](admin-submission.md)

Это может создавать раздражение, если пользователи в организации получают нежелательные сообщения или фишинговые сообщения в папке "Входящие" или не получает допустимые сообщения, так как они помечаются как нежелательные. Мы постоянно точно подстраиваем фильтры нежелательной почты для более точной настройки.

Вы и ваши пользователи могут помочь сделать этот процесс, отправляя ложные срабатывания (если в качестве плохого сообщения электронной почты указаны недопустимые), ложные отрицательные отрицательные (разрешены неправильные письма) и фишинговые сообщения в корпорацию Майкрософт для анализа.

> [!NOTE]
> Из-за большого объема получаемых отправленных отправок нам удалось отвечать на все запросы для анализа.

## <a name="submit-false-negatives-to-microsoft"></a>Отправка ложных отрицательных возможностей в Корпорацию Майкрософт

> [!TIP]
> Вместо использования следующих процедур для сообщения о ложных отрицательных результатах пользователи в Outlook и Outlook в Интернете (прежнее название — Outlook Web App) могут использовать надстройку Report Message для Microsoft Outlook. Сведения об установке и использовании этого средства см. в разделе ["Включение надстройки Report Message".](enable-the-report-message-add-in.md)

Если вы получаете сообщение, прошедшее через фильтрацию нежелательной почты, которая должна быть определенным как спам или фишинг, при необходимости его можно отправить в группы анализа нежелательной почты и Microsoft Phishing. Эти результаты проверяют сообщение и добавят его в фильтры на уровне службы, если оно соответствует условиям классификации.

1. Создайте пустое сообщение электронной почты одним из следующих получателей:

   - **Junk**: `junk@office365.microsoft.com`

   - **Фишинг:**`phish@office365.microsoft.com`

2. Перетащите нежелательное или фишинговое сообщение в новое сообщение. Таким образом нежелательное или фишинговое сообщение будет сохранено в виде вложения в новом сообщении. Не копируйте и не вставляйте содержимое сообщения или переслайте его (нам необходимо исходное сообщение для проверки заголовков сообщения).

   > [!NOTE]
   >
   > - В новое сообщение можно вложить несколько сообщений. Убедитесь, что все сообщения одного типа: фишинговые или нежелательные.
   >
   > - Оставьте текст нового сообщения пустым.
   >
   > - Используйте формат MSG (стандартный формат Outlook) или EML (формат Outlook в Интернете по умолчанию) для вложенных сообщений.

3. По завершении нажмите кнопку **"Отправить".**

> [!TIP]
> У администраторов есть несколько различных способов блокировки определенных сообщений, которые не выявлены как нежелательные. Дополнительные сведения см. в статье ["Создание списков заблокированных отправителей в EOP".](create-block-sender-lists-in-office-365.md)

## <a name="submit-false-positives-to-microsoft"></a>Отправка сообщений о ложных срабатываниях в Майкрософт

> [!TIP]
> Вместо использования следующих процедур для сообщения о ложных срабатываниях пользователи в Outlook и Outlook в Интернете могут использовать надстройку Report Message для Microsoft Outlook. Сведения об установке и использовании этого средства см. в разделе ["Включение надстройки Report Message".](enable-the-report-message-add-in.md)

Если сообщение было неправильно указано как спам, вы можете отправить его в группу анализа нежелательной почты Майкрософт. Аналитики оценивают сообщение и (в зависимости от результатов анализа) можно настроить фильтры для всей службы, чтобы разрешить доставку сообщения.

1. Создайте пустое сообщение с `not_junk@office365.microsoft.com` адресом:

2. Перетащите неопределенное сообщение в новое сообщение. При этом пропущенное сообщение будет сохранено в виде вложения в новом сообщении. Не копируйте и не вставляйте содержимое сообщения или переслайте его (нам необходимо исходное сообщение для проверки заголовков сообщения).

   > [!NOTE]
   >
   > - В новое сообщение можно вложить несколько сообщений. Убедитесь, что все сообщения одного типа: фишинговые или нежелательные.
   >
   > - Оставьте текст нового сообщения пустым.
   >
   > - Используйте формат MSG (стандартный формат Outlook) или EML (формат Outlook в Интернете по умолчанию) для вложенных сообщений.

3. По завершении нажмите кнопку **"Отправить".**

> [!TIP]
> Администраторам доступно несколько различных способов разрешить определенным сообщениям пропускать фильтрацию нежелательной почты. Дополнительные сведения см. в статье ["Создание списков надежных отправителей в EOP".](create-safe-sender-lists-in-office-365.md)
