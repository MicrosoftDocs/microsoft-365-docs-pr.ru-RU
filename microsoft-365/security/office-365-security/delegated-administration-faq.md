# <a name="delegated-administration-faq"></a>Вопросы и ответы по делегированному администрированию

В этой статье приведены часто задаваемые вопросы и ответы по делегированным задачам администрирования в Microsoft 365 для партнеров Майкрософт и торговых посредников. Делегированное администрирование включает в себя возможность управления параметрами Exchange Online Protection (EOP) для других клиентов (компаний).

## <a name="im-a-reseller-and-i-need-to-manage-my-customer-tenants-how-does-this-work"></a>У меня есть торговый посредник и мне нужно управлять клиентскими клиентами. Как это работает?

Если вы являетесь партнером или торговым посредником Майкрософт, и вы вошли в систему Microsoft Advisor, вы можете запрашивать _делегированные возможности администрирования_ в организации Microsoft 365.

Делегированное администрирование позволяет управлять Microsoft 365 (включая параметры EOP), как если бы вы были администратором в этой Организации. Инструкции по настройке делегированного администрирования описаны в приведенном ниже списке.

1. Зарегистрируйтесь, чтобы стать [консультантом по Microsoft Office 365](https://aka.ms/cloudbenefits).

2. Подпишитесь на делегированное администрирование. Прежде чем приступить к администрированию клиента клиента, он должен авторизовать вас в качестве делегированного администратора. Для получения утверждения сначала отправьте заказчику [предложение о делегированном администрировании](https://support.microsoft.com/office/26530dc0-ebba-415b-86b1-b55bc06b073e). Вы также можете позднее передать делегированное администрирование клиенту.

3. Создайте учетную запись делегированного администратора, выполнив действия, описанные в статье [Добавление, изменение или удаление партнера помощника по подписке](https://docs.microsoft.com/microsoft-365/admin/misc/add-partner).

Посетите страницу [партнеры: Постройте свой бизнес и администрировать подписку по партнерам,](https://support.microsoft.com/office/30dd1681-47e0-4cbc-abfe-a222cd111319) чтобы получить дополнительные сведения о настройке делегированного администрирования.

## <a name="im-a-customer-not-a-reseller-how-can-set-up-delegated-administrator-for-my-subtenants"></a>Я — клиент, а не торговый посредник. Как можно настроить делегированного администратора для моих подклиентов?

Делегированное администрирование доступно только для торговых посредников и партнеров. Тем не менее, существует пример сценария PowerShell, который поможет вам применить политики к своим подклиентам (компаниям). Дополнительные сведения см. [в статье пример скрипта для применения параметров EOP к нескольким клиентам](sample-script-for-applying-eop-settings-to-multiple-tenants.md).

## <a name="can-i-prevent-my-subtenant-admin-from-modifying-my-policy"></a>Можно ли запретить администратору моих клиентов изменять политику?

Нет. В настоящее время Microsoft 365 не поддерживает эту возможность.

## <a name="can-i-get-consolidated-reporting-across-all-of-my-subtenants"></a>Можно ли получить консолидированную отчетность по всем моим подчиненным клиентам?

Отчетность по консолидированным организациям в центрах администрирования Microsoft 365 недоступна. Тем не менее, отчеты можно получить с помощью [Microsoft Graph](https://docs.microsoft.com/graph/overview).
