# <a name="delegated-administration-faq"></a>Вопросы и ответы по делегированному администрированию

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


В этой статье данная статья содержит вопросы и ответы о делегирования задач администрирования в Microsoft 365 для партнеров и посредников Майкрософт. Делегированная администрирование включает возможность управления настройками Exchange Online Protection (EOP) для других клиентов (компаний).

## <a name="im-a-reseller-and-i-need-to-manage-my-customer-tenants-how-does-this-work"></a>Я — посредник, и мне нужно управлять клиентами. Как это работает?

Если вы партнер или reseller корпорации Майкрософт и вы подписаны на помощника  Майкрософт, вы можете запросить делегирование возможностей администрирования в организации Microsoft 365 клиента.

Делегирование администрирования позволяет управлять Microsoft 365 (включая параметры EOP), как если бы вы были администратором в этой организации. Действия по настройке делегирования администрирования описаны в следующем списке:

1. Зарегистрируйтесь, чтобы стать [консультантом по Microsoft Office 365](https://aka.ms/cloudbenefits).

2. Зарегистрироваться для делегирования администрирования. Прежде чем приступить к администрированию клиента клиента, он должен авторизировать вас в качестве полномочным администратором. Для получения утверждения сначала отправьте заказчику [предложение о делегированном администрировании](https://support.microsoft.com/office/26530dc0-ebba-415b-86b1-b55bc06b073e). Вы также можете предложить делегированную администрирование клиенту позже.

3. Создайте делегированную учетную запись администратора с помощью действий, которые необходимо предпринять при добавлении, изменении или [удалении партнера-консультанта по подписке.](../../admin/misc/add-partner.md)

Посетите [сайт партнеров: создайте свою бизнес-подписку](https://support.microsoft.com/office/30dd1681-47e0-4cbc-abfe-a222cd111319) и администрируете партнерскую подписку, чтобы получить дополнительные сведения о том, как настроить делегированную администрирование.

## <a name="im-a-customer-not-a-reseller-how-can-set-up-delegated-administrator-for-my-subtenants"></a>Я клиент, а не посредник. Как настроить делегированного администратора для подтенантов?

Делегирование администрирования доступно только для посредников и партнеров. Однако существует пример сценария PowerShell, который поможет вам применять политики к подтенантенам (компаниям). Дополнительные сведения см. в примере сценария применения параметров [EOP к нескольким арендаторам.](sample-script-for-applying-eop-settings-to-multiple-tenants.md)

## <a name="can-i-prevent-my-subtenant-admin-from-modifying-my-policy"></a>Можно ли запретить администратору подтенантного сайта изменять мою политику?

Нет. Microsoft 365 в настоящее время не имеет этой возможности.

## <a name="can-i-get-consolidated-reporting-across-all-of-my-subtenants"></a>Можно ли получить консолидированные отчеты по всем подтенантиям?

Консолидированные отчеты для всех управляемых вами компаний недоступны в отчетах Центра администрирования Microsoft 365. Однако отчеты можно получить с помощью [Microsoft Graph.](https://docs.microsoft.com/graph/overview)
