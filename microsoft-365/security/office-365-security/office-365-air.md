# <a name="automated-investigation-and-response-air-in-microsoft-defender-for-office-365"></a>Автоматическое исследование и реагирование на них (AIR) в Microsoft Defender для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Microsoft Defender для Office 365 (план 1 и план 2)](office-365-atp.md)
- [Microsoft 365 Defender](../mtp/microsoft-threat-protection.md)

[Microsoft Defender для Office 365](office-365-atp.md) включает мощные автоматизированные средства исследования и реагирования на них( AIR), которые могут сэкономить время и усилия команды по операциям безопасности. По мере инициации оповещений ваша группа по работе с безопасностью может инициировать проверку, расставить приоритеты и реагировать на них. Следите за объемом входящих оповещений. Автоматизация некоторых из этих задач может помочь.

AIR обеспечивает более эффективную и эффективную работу группы операций безопасности. Возможности AIR включают автоматизированные процессы исследования в ответ на известные угрозы, которые существуют на сегодняшний день. Соответствующие действия по исправлению ожидают утверждения, позволяя группе по работе с безопасностью эффективно реагировать на обнаруженные угрозы. С помощью AIR группа операций безопасности может сосредоточиться на задачах с более высоким приоритетом, не упускать из виду важные оповещения, которые срабатывает.

В этой статье описываются:

- Общий [поток AIR;](#the-overall-flow-of-air)
- [Как получить AIR;](#how-to-get-air) и
- Необходимые [разрешения для](#required-permissions-to-use-air-capabilities) настройки или использования возможностей AIR.
- Изменения, которые скоро будут внесены в центр безопасности

В этой статье также [содержатся дальнейшие действия](#next-steps)и ресурсы для получения дополнительных информации.

## <a name="the-overall-flow-of-air"></a>Общий поток AIR

Срабатывает оповещение, и в книге по безопасности запускается автоматизированное исследование, которое приводит к результатам и рекомендуемые действия. Ниже пошаговая по

1. Автоматизированное исследование инициаизируется одним из следующих способов:
   - Либо [оповещение вызывается](#which-alert-policies-trigger-automated-investigations) чем-то подозрительным в сообщении электронной почты (например, сообщением, вложением, URL-адресом или скомпрометированной учетной записью пользователя). Создается инцидент и начинается автоматическое исследование; или
   - Аналитик безопасности [запускает автоматизированное исследование при](automated-investigation-response-office.md#example-a-security-administrator-triggers-an-investigation-from-threat-explorer) использовании [обозревателя угроз.](threat-explorer.md)
2. Во время автоматического исследования он собирает данные о сообщении электронной почты и связанных с ним организациях. Такие сущности могут включать файлы, URL-адреса и получателей. Область исследования может увеличиваться по мере запуска новых и связанных оповещений.
3. Во время и после автоматического исследования [можно](air-view-investigation-results.md) просмотреть подробные сведения и результаты. Результаты включают [рекомендуемые действия,](air-remediation-actions.md) которые можно принять для реагирования на все найденные угрозы и устранения их последствий.
4. Группа операций безопасности проверяет результаты [исследования](air-view-investigation-results.md)и рекомендации, а также утверждает или отклоняет действия по [исправлению.](air-review-approve-pending-completed-actions.md)
5. По мере утверждения (или отклонения) ожидающих действий по исправлению автоматическое исследование завершается.

В Microsoft Defender для Office 365 действия по исправлению не принимаются автоматически. Действия по устранению угроз и их последствий предпринимаются только после их утверждения группой безопасности вашей организации. Возможности AIR экономят время группы операций безопасности, определяя действия по исправлению и предоставляя сведения, необходимые для принятия обоснованных решений.

Во время и после каждого автоматизированного исследования группа операций безопасности может:

- [Просмотр сведений об оповещении, связанном с исследованием](air-view-investigation-results.md#view-details-about-an-alert-related-to-an-investigation)
- [Просмотр сведений о результатах исследования](air-view-investigation-results.md#view-details-of-an-investigation)
- [Проверка и утверждение действий в результате исследования](air-review-approve-pending-completed-actions.md)

> [!TIP]
> Более подробный обзор см. в [описании работы AIR.](automated-investigation-response-office.md)

## <a name="which-alert-policies-trigger-automated-investigations"></a>Какие политики оповещений инициирует автоматическое исследование?

Microsoft 365 предоставляет множество встроенных политик оповещений, которые помогают выявлять злоупотребление разрешениями администратора Exchange, вредоносные программы, потенциальные внешние и внутренние угрозы, а также риски для управления информацией. Несколько политик [оповещений по умолчанию](../../compliance/alert-policies.md#default-alert-policies) могут запускать автоматические исследования. В следующей таблице описываются оповещения, запускаемые автоматическими расследованиями, их серьезность в Центре безопасности Microsoft 365, а также то, как они создаются:

|Оповещение|Severity|Как создается оповещение|
|:---|:---|:---|
|Обнаружен потенциально вредоносный URL-адрес|**Высокий**|Это оповещение создается при следующих событиях: <ul><li>Пользователь, защищенный с помощью [безопасных ссылок](atp-safe-links.md) в организации, щелкает вредоносную ссылку</li><li>Изменения в решениях для URL-адресов определены в Microsoft Defender для Office 365</li><li>Пользователи переопределяют страницы предупреждений безопасных ссылок (в зависимости от политики безопасных [ссылок организации).](set-up-atp-safe-links-policies.md)</li></ul> <p> Дополнительные сведения о событиях, которые вызывают это оповещение, см. в [настройках политик безопасных ссылок.](set-up-atp-safe-links-policies.md)|
|Сообщение электронной почты сообщается пользователем как вредоносное или фишинговое|**Информационный**|Это предупреждение создается, когда пользователи в организации сообщают о сообщениях как о фишинговых сообщениях с помощью надстройки [Report Message](enable-the-report-message-add-in.md) или надстройки [Report Phishing.](enable-the-report-phish-add-in.md)|
|Сообщения электронной почты, содержащие вредоносные программы, удаляются после доставки|**Информационный**|Это предупреждение создается, когда все сообщения электронной почты, содержащие вредоносные программы, доставляются в почтовые ящики в организации. В этом случае корпорация Майкрософт удаляет зараженные сообщения из почтовых ящиков Exchange Online с помощью автоматической [очистки.](zero-hour-auto-purge.md)|
|Сообщения электронной почты, содержащие фишинговые URL-адреса, удаляются после доставки|**Информационный**|Это предупреждение создается, когда все сообщения, содержащие фишинг, доставляются в почтовые ящики в организации. В этом случае корпорация Майкрософт удаляет зараженные сообщения из почтовых ящиков Exchange Online с помощью автоматической [очистки.](zero-hour-auto-purge.md)|
|Обнаружены подозрительные шаблоны отправки электронной почты|**Средний**|Это предупреждение создается, когда кто-то в вашей организации отправил подозрительную электронную почту и ему может быть запрещено отправлять электронную почту. Оповещение — это раннее предупреждение о поведении, которое может указывать на то, что учетная запись скомпрометирована, но не является достаточно серьезной, чтобы ограничить пользователя. <p> Хотя это редко, оповещение, сгенерированная этой политикой, может быть аномалией. Однако лучше проверить, скомпрометирована ли учетная запись [пользователя.](responding-to-a-compromised-email-account.md)|
|Пользователю запрещено отправлять электронную почту|**Высокий**|Это предупреждение создается, когда кому-либо в вашей организации запрещено отправлять исходящие сообщения. Это оповещение обычно приводит к компрометации [учетной записи электронной почты.](responding-to-a-compromised-email-account.md) <p> Дополнительные сведения об ограниченных пользователях см. в подсети "Удаление заблокированных пользователей с портала "Пользователи с ограниченным [доступом" в Microsoft 365.](removing-user-from-restricted-users-portal-after-spam.md)|
|

> [!TIP]
> Чтобы узнать больше о политиках оповещений или изменить параметры по умолчанию, см. статью "Политики оповещений" в Центре соответствия требованиям [Microsoft 365.](../../compliance/alert-policies.md)

> - Центр безопасности Microsoft Defender ( [https://securitycenter.windows.com](https://securitycenter.windows.com) )

|Элемент|Что меняется?|

|**Вкладка "Пользователи"**|Вкладка **"Пользователи"** теперь является **вкладками "Почтовые ящики".** Сведения о пользователях перечислены на **вкладке "Почтовый ящик".**|

|**Вкладка "Свидетельство"**|На новой **вкладке Evidence** показаны основные результаты, связанные с действиями. Действия, связанные с каждым элементом свидетельства, можно утвердить (или отклонить) в боковой области, которая открывается при выборе ожидающих действий.|

Дополнительные узнать см. в центре действий. (Дополнительные см. в [центре действий.)](../mtp/mtp-action-center.md)|
