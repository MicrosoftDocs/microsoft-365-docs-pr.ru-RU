# <a name="safe-attachments-in-microsoft-defender-for-office-365"></a>Безопасные вложения в Microsoft Defender для Office 365

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Microsoft Defender для Office 365 (план 1 и план 2)](defender-for-office-365.md)
- [Microsoft 365 Defender](../defender/microsoft-365-defender.md)

Безопасные вложения [в Microsoft Defender для Office 365](defender-for-office-365.md) предоставляют дополнительный уровень защиты для вложений электронной почты, которые уже отсканированы защитой от вредоносных программ в Exchange Online Protection [(EOP).](anti-malware-protection.md) В частности, безопасные вложения используют виртуальную среду для проверки вложений в сообщениях электронной почты перед их доставкой получателям (процесс, известный как _детонация)._

Защита безопасных вложений для сообщений электронной почты контролируется политиками безопасных вложений. Политика безопасных вложений по умолчанию не существует, поэтому для защиты безопасных вложений необходимо создать одну или **несколько политик безопасных вложений.** Инструкции см. в [приложении Set up Safe Attachments policies in Defender for Office 365.](set-up-safe-attachments-policies.md)

В следующей таблице описываются сценарии безопасных вложений в организациях Microsoft 365 и Office 365, включающих Microsoft Defender для Office 365 (другими словами, отсутствие лицензирования никогда не является проблемой в примерах).

## <a name="safe-attachments-policy-settings"></a>Параметры политики безопасных вложений

В этом разделе описываются параметры политик безопасных вложений:

- **Безопасные вложения неизвестный ответ** вредоносных программ. Этот параметр управляет действием для сканирования вредоносных программ для безопасных вложений в сообщениях электронной почты. Доступные параметры описаны в следующей таблице:

  ****

  |Параметр|Эффект|Используйте, когда вы хотите:|
  |---|---|---|
  |**Off**|Вложения не отсканированы для вредоносных программ с помощью безопасных вложений. Сообщения по-прежнему отсканированы для вредоносных программ с помощью защиты от вредоносных программ [в EOP](anti-malware-protection.md).|Отключите сканирование для выбранных получателей. <p> Предотвращение ненужных задержек в маршрутизах внутренней почты. <p> **Этот параметр не рекомендуется для большинства пользователей. Этот параметр следует использовать только для отключения сканирования безопасных вложений для получателей, которые получают сообщения только от надежных отправителей.**|
  |**Монитор**|Доставляет сообщения с вложениями, а затем отслеживает, что происходит с обнаруженными вредоносными программами. <p> Доставка безопасных сообщений может быть отложена из-за сканирования безопасных вложений.|Узнайте, куда отправляется обнаруженная вредоносная программа в вашей организации.|
  |**Блокировка**|Предотвращает доставку сообщений с обнаруженными вложениями вредоносных программ. <p> Сообщения находятся [на карантине,](manage-quarantined-messages-and-files.md) где только администраторы (а не конечные пользователи) могут просмотреть, освободить или удалить сообщения. <p> Автоматически блокирует будущие экземпляры сообщений и вложений. <p> Доставка безопасных сообщений может быть отложена из-за сканирования безопасных вложений.|Защищает организацию от повторных атак с помощью тех же вложений вредоносных программ. <p> Это значение по умолчанию и рекомендуемое значение в стандартных и строгих [предустановленных политиках безопасности.](preset-security-policies.md)|
  |**Замена**|Удаляет обнаруженные вредоносные вложения. <p> Сообщает получателям об удалении вложений. <p>  Сообщения находятся [на карантине,](manage-quarantined-messages-and-files.md) где только администраторы (а не конечные пользователи) могут просмотреть, освободить или удалить сообщения. <p> Доставка безопасных сообщений может быть отложена из-за сканирования безопасных вложений.|Повышение видимости для получателей, которые удалили вложения из-за обнаруженного вредоносного ПО.|
  |**Динамическая доставка**|Доставляет сообщения немедленно, но заменяет вложения заполнителями до завершения сканирования безопасных вложений. <p> Подробные сведения см. в разделе [Динамическая доставка в разделе Политики](#dynamic-delivery-in-safe-attachments-policies) безопасных вложений далее в этой статье.|Избегайте задержек сообщений, защищая получателей от вредоносных файлов. <p> Включить получателей для предварительного просмотра вложений в безопасном режиме во время сканирования происходит.|
  |

- Перенаправление вложений при обнаружении. Включить перенаправление  и отправку вложения на следующий адрес **электронной** почты: Для блокировки, мониторинга или замены действий отправьте сообщения, содержащие вложения вредоносных программ, на указанный внутренний или внешний адрес электронной почты для анализа и расследования.   

  Рекомендация для параметров стандартных и строгих политик — включить перенаправление. Дополнительные сведения см. в [приложении "Безопасные вложения".](recommended-settings-for-eop-and-office365.md#safe-attachments-settings)

- Применив указанный выше выбор, если при сканировании вредоносных программ для вложений происходит время или возникает **ошибка:** действие, указанное в безопасных вложениях, неизвестное вредоносное по ответам, будет приниматься в сообщениях даже в тех случаях, когда сканирование безопасных вложений не может завершиться.  Всегда выберите этот параметр, если вы выберите **Включить перенаправление**. В противном случае сообщения могут быть потеряны.

- **Фильтры получателей.** Необходимо указать условия и исключения получателей, которые определяют, к кому применяется политика. Для условий и исключений можно использовать следующие свойства:

  - **Получатель**
  - **Домен получателя**.
  - **Получатель входит в группу**

  Условие или исключение можно использовать только один раз, но оно может содержать несколько значений. Указать несколько значений в одном условии или исключении можно с помощью оператора OR (например, _\<recipient1\>_ or _\<recipient2\>_). Между разными условиями и исключениями используется оператор AND (например, _\<recipient1\>_ and _\<member of group 1\>_).

- **Приоритет.** При создании нескольких политик можно указать порядок, который они применяют. Никакие две политики не могут иметь одинаковый приоритет, и обработка политики прекращается после применения первой политики.

  Дополнительные сведения о приоритетах, а также оценке и применении нескольких политик см. в статье [Порядок и приоритет защиты электронной почты](how-policies-and-protections-are-combined.md).

### <a name="dynamic-delivery-in-safe-attachments-policies"></a>Динамическая доставка в политиках безопасных вложений

> [!NOTE]
> Динамическая доставка работает только для почтовых ящиков Exchange Online.

Действие Dynamic Delivery в политиках безопасных вложений стремится устранить любые задержки доставки электронной почты, которые могут быть вызваны сканированием безопасных вложений. Тело сообщения электронной почты доставляется получателю с местообналожением для каждого вложения. Местообладатель остается до тех пор, пока вложение не будет признано безопасным, а затем вложение станет доступным для открытия или скачивания.

Если вложение признано вредоносным, сообщение находится на карантине. Только администраторы (а не конечные пользователи) могут просматривать, выпускать или удалять сообщения, которые были карантинными при сканировании безопасных вложений. Дополнительные сведения см. в [статью Управление карантинными сообщениями и файлами в качестве администратора.](manage-quarantined-messages-and-files.md)

Большинство документов PDF и Office можно просмотреть в безопасном режиме во время сканирования безопасных вложений. Если вложение не совместимо с предварительной версией динамической доставки, получатели увидят заполнитель для вложения до завершения сканирования безопасных вложений.

Если вы используете мобильное устройство, а PDF-серверы не отрисовка в предварителю динамической доставки на мобильном устройстве, попробуйте открыть сообщение в Outlook в Интернете (ранее известном как Outlook Web App) с помощью мобильного браузера.

Вот некоторые соображения для динамической доставки и переададных сообщений:

- Если получатель, переададант, защищен политикой безопасных вложений, использующей параметр Dynamic Delivery, получатель видит местообладатель с возможностью предварительного просмотра совместимых файлов.

- Если переадверяемая получатель не защищена политикой безопасных вложений, сообщение и вложения будут доставлены без сканирования безопасных вложений или вложений.

Существуют сценарии, в которых динамическая доставка не может заменить вложения в сообщениях. Это происходит в описанных ниже случаях.

- Сообщения в общедоступных папках.

- Сообщения, которые будут отправлены из почтового ящика пользователя и затем обратно в него с помощью настраиваемого правила.

- Сообщения, которые перемещаются (автоматически или вручную) из облачных почтовых ящиков в другие места, включая архивные папки.

- Удаленные сообщения.

- Папка поиска почтовых ящиков пользователя находится в состоянии ошибки.

- Организации Exchange Online, в которых включена "Восклицатель". Чтобы решить эту проблему, см. [в руб. KB4014438](https://support.microsoft.com/help/4014438).

- [S/MIME)](s-mime-for-message-signing-and-encryption.md) зашифрованные сообщения.

- Вы настраивали действие Dynamic Delivery в политике безопасных вложений, но получатель не поддерживает динамическую доставку (например, получатель — почтовый ящик в локальной организации Exchange). Однако безопасные ссылки в [Microsoft Defender для Office 365](set-up-safe-links-policies.md) могут сканировать вложения [](configure-global-settings-for-safe-links.md) файлов Office, содержащие URL-адреса (в зависимости от настройки глобальных параметров безопасных ссылок).
