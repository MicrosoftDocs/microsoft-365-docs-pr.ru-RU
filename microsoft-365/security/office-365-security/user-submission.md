# <a name="user-reported-message-settings"></a>Параметры сообщений пользователя, о чем сообщалось

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Exchange Online Protection](exchange-online-protection-overview.md)
- [Microsoft Defender для Office 365 (план 1 и план 2)](defender-for-office-365.md)
- [Microsoft 365 Defender](../defender/microsoft-365-defender.md)

В Microsoft 365 организациях с Exchange Online почтовыми ящиками можно указать почтовый ящик для получения сообщений, которые пользователи сообщают как вредоносные или не вредоносные. Когда пользователи сообщают сообщения с помощью различных вариантов отчетности, вы можете использовать этот почтовый ящик для перехвата сообщений (только для отправки в настраиваемый почтовый ящик) или получения копий сообщений (отправка в настраиваемый почтовый ящик и Microsoft). Эта функция работает со следующими вариантами отчетов о сообщениях:

- [Надстройка сообщения отчета](enable-the-report-message-add-in.md)
- [Надстройка Для фишинга отчетов](enable-the-report-phish-add-in.md)
- [Средства отчетности сторонних сторон](#third-party-reporting-tools)

Доставка сообщений пользователей в настраиваемый почтовый ящик, а не непосредственно в Корпорацию Майкрософт, позволяет администраторам выборочно и вручную отправлять сообщения в Корпорацию Майкрософт с помощью отправки [администратора.](admin-submission.md) Эти параметры ранее были известны как политика отправки пользователей.

  > [!NOTE]
  > Если отчеты отключены в [Outlook в Интернете,](report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md#disable-or-enable-junk-email-reporting-in-outlook-on-the-web)включение сообщений пользователей здесь переопределит этот параметр и позволит пользователям снова сообщать Outlook в Интернете.

## <a name="custom-mailbox-prerequisites"></a>Настраиваемые предпосылки почтового ящика

Используйте следующие статьи, чтобы настроить необходимые условия, чтобы сообщения с сообщениями, отправленные пользователем, перейти в настраиваемый почтовый ящик:

- Пропустить фильтрацию нежелательной почты на настраиваемом почтовом ящике, создав правило потока почтовой почты exchange, чтобы установить уровень доверия к нежелательной почте. См. [в документе Использование EAC](/exchange/security-and-compliance/mail-flow-rules/use-rules-to-set-scl#use-the-eac-to-create-a-mail-flow-rule-that-sets-the-scl-of-a-message) для создания правила потока почты, которое задает SCL сообщения, чтобы установить SCL для обхода фильтрации **нежелательной почты.**

- [Создайте политику Сейф](set-up-safe-attachments-policies.md) вложений, которая включает настраиваемый почтовый ящик, в котором отключается Сейф вложения **(Сейф** вложения неизвестен раздел ответа на вредоносные \> **программы).**

- [Создайте политику Сейф ссылок,](set-up-safe-links-policies.md) которая включает настраиваемый почтовый ящик, в котором отключается Сейф ссылки (Выберите действие для неизвестных потенциально вредоносных **URL-адресов** в разделе Отключение \> **сообщений).**

- [](configure-your-spam-filter-policies.md#use-the-microsoft-365-defender-portal-to-create-anti-spam-policies) Создайте политику защиты от вредоносных программ, которая включает настраиваемый почтовый ящик, в котором отключается автоматическая очистка нулевого часа (ZAP) для вредоносных программ **(раздел** Параметры защиты Включить авточистку нулевого часа для вредоносных программ не \>  выбран).

- [Создайте](configure-your-spam-filter-policies.md#use-the-microsoft-365-defender-portal-to-create-anti-spam-policies) политику защиты от нежелательной почты, которая включает настраиваемый почтовый ящик, в котором отключается ZAP для нежелательной почты и ZAP для фишинга **(раздел** Авточистки нулевого часа Включенная автоматическая очистка без часов \> **(ZAP)** не выбран).

- Отключить правило нежелательной почты в настраиваемом почтовом ящике. Чтобы [отключить правило нежелательной почты, Exchange Online](configure-junk-email-settings-on-exo-mailboxes.md) настройка параметров нежелательной почты в почтовых ящиках Exchange Online нежелательной почты. После отключения EOP не может перемещать сообщения в папку нежелательной почты на основе действия по фильтрации нежелательной почты **Move message to Junk Email folder** or the safelist collection on the mailbox.

После проверки того, что ваш почтовый ящик соответствует всем применимым требованиям, можно использовать процедуры, представленные в этой статье, для настройки почтового ящика отправки пользователей.

## <a name="third-party-reporting-tools"></a>Средства отчетности сторонних сторон

Вы можете настроить средства отчетности сторонних сообщений для отправки сообщений в настраиваемый почтовый ящик. Это можно сделать, **Outlook** параметра кнопки сообщение отчетов Майкрософт  для отключения и настройки почтового ящика моей организации Office 365 почтового ящика по вашему выбору. 

Единственным требованием является то, что исходное сообщение включено в качестве . EML или . Вложение MSG (не сжато) в сообщении, которое отправляется в настраиваемый почтовый ящик (не просто переад. исходное сообщение в настраиваемый почтовый ящик).

Требования к формату форматирования сообщений описаны в следующем разделе. Форматирование является необязательным, но если он не следует заданным форматом, отчеты всегда будут отправлены в качестве фишинга.

## <a name="message-submission-format"></a>Формат отправки сообщений

Чтобы правильно определить исходные прикрепленные сообщения, сообщения, отправленные в настраиваемый почтовый ящик, требуют определенного форматирования. Если сообщения не используют этот формат, исходные прикрепленные сообщения всегда идентифицированы как фишинговые сообщения.

Для правильной идентификации исходных подключенных сообщений сообщения, отправленные в настраиваемый почтовый ящик, должны использовать следующий синтаксис для субъекта (название конверта):

`SafetyAPIAction|NetworkMessageId|SenderIp|FromAddress|(Message Subject)`

где БезопасностьAPIAction является одним из следующих значений в несколько раз:

- 1. Нежелательное
- 2. Не нежелательно
- 3. Фишинг

В этом примере используются следующие значения:

- Сообщение сообщается как фишинг.
- Сетевой ID сообщения 49871234-6dc6-43e8-abcd-08d797f20abe.
- IP-адрес отправитель 167.220.232.101.
- Адрес From test@contoso.com.
- Тема сообщения — "проверка отправки фишинга"

`3|49871234-6dc6-43e8-abcd-08d797f20abe|167.220.232.101|test@contoso.com|(test phishing submission)`

Сообщения, которые не следуют этому формату, не отображаются должным образом на портале Отправки.
