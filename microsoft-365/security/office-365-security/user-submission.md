# <a name="user-submissions-policy"></a>Политика отправки пользователей

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]


В организациях Microsoft 365 с почтовыми ящиками Exchange Online можно указать почтовый ящик для получения сообщений, которые пользователи сообщают как вредоносные или не вредоносные. Когда пользователи отправляют сообщения с помощью различных параметров отчетов, вы можете использовать этот почтовый ящик для перехвата сообщений (только для отправки в настраиваемый почтовый ящик) или получения копий сообщений (отправки в настраиваемый почтовый ящик и Корпорацию Майкрософт). Эта функция работает со следующими вариантами отчетов о сообщениях:

- [Надстройка Report Message](enable-the-report-message-add-in.md)

- [Надстройка Report Phishing](enable-the-report-phish-add-in.md)

- [Встроенные отчеты в Outlook в Интернете](report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md) (прежнее название — Outlook Web App)

- [Встроенные отчеты в Outlook для iOS и Android](report-junk-email-and-phishing-scams-in-outlook-for-iOS-and-Android.md)

  > [!NOTE]
  > Если отчеты отключены в [Outlook в Интернете,](report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md#disable-or-enable-junk-email-reporting-in-outlook-on-the-web)включение отправки от пользователей переопределит этот параметр и позволит пользователям снова сообщать о сообщениях в Outlook в Интернете.

Вы также можете настроить сторонние средства отчетов о сообщениях для пересылки сообщений в задаемый почтовый ящик.

Доставка сообщений, о которых пользователь сообщил, в пользовательский почтовый ящик, а не непосредственно в Корпорацию Майкрософт, позволяет администраторам выборочно и вручную сообщать о сообщениях в корпорацию Майкрософт с помощью отправки [администратора.](admin-submission.md)

## <a name="custom-mailbox-prerequisites"></a>Необходимые условия для настраиваемой почтовой ящики

Используйте следующие статьи, чтобы настроить необходимые условия для того, чтобы сообщения, отправленные пользователем, переходить в ваш настраиваемый почтовый ящик:

- Пропустите фильтрацию нежелательной почты в настраиваемом почтовом ящике, создав правило потока почты Exchange, чтобы установить уровень достоверности нежелательной почты. См. ["Использование EAC для создания](use-mail-flow-rules-to-set-the-spam-confidence-level-scl-in-messages.md#use-the-eac-to-create-a-mail-flow-rule-that-sets-the-scl-of-a-message) правила потока почты, которое задает SCL для сообщения, чтобы установить для SCL **-1**.

- Отключите сканирование вложений на вредоносные программы в настраиваемом почтовом ящике. Используйте политики "Настройка безопасных вложений" в [Защитнике Office 365,](set-up-atp-safe-attachments-policies.md) чтобы создать политику безопасных вложений с параметром **"Отключить** для безопасных вложений" ответ на неизвестные вредоносные **программы.**

- Отключите проверку URL-адресов для сообщений в настраиваемом почтовом ящике. Используйте политики "Настройка безопасных ссылок" в [Защитнике Office 365,](set-up-atp-safe-links-policies.md) чтобы создать политику безопасных ссылок с параметром **"Отключить"** для параметра "Выбрать действие" для неизвестных потенциально вредоносных URL-адресов в **сообщениях.**

- Создайте политику для борьбы с вредоносными программами, чтобы отключить автоматическую очистку вредоносных программ. См. ["Использование Центра безопасности & соответствия требованиям"](configure-your-spam-filter-policies.md#use-the-security--compliance-center-to-create-anti-spam-policies) для создания политик защиты от вредоносных программ, чтобы отключить автоматическую очистку **вредоносных** **программ.**

- Создайте политику фильтрации нежелательной почты, чтобы отключить автоматическую очистку нежелательной почты и фишинга в настраиваемом почтовом ящике. См. ["Использование Центра безопасности & соответствия](configure-your-spam-filter-policies.md#use-the-security--compliance-center-to-create-anti-spam-policies) требованиям" для  создания политик защиты  от нежелательной почты и с очистки установленных для них установленных для защиты от нежелательной почты и **фишинга.**

- Отключать правило нежелательной почты в настраиваемом почтовом ящике. Чтобы [отключить правило нежелательной почты,](configure-junk-email-settings-on-exo-mailboxes.md) используйте параметры настройки нежелательной почты для почтовых ящиков Exchange Online. После отключения EOP не сможет перемещать сообщения в папку нежелательной почты на основе решения фильтра нежелательной почты **"Переместить** сообщение в папку нежелательной почты" или коллекцию списков надежных адресов в почтовом ящике.

После проверки соответствия почтового ящика всем необходимым требованиям используйте Центр безопасности и соответствия требованиям & для настройки почтового ящика отправки пользователей [(в](#use-the-security--compliance-center-to-configure-the-user-submissions-mailbox) этой статье).

## <a name="what-do-you-need-to-know-before-you-begin"></a>Что нужно знать перед началом работы

- Откройте Центр безопасности и соответствия требованиям на сайте <https://protection.office.com/>. Чтобы перейти непосредственно на **страницу "Отправки пользователей",** используйте <https://protection.office.com/userSubmissionsReportMessage> .

- Чтобы изменить конфигурацию отправки пользователей, необходимо быть участником одной из следующих групп ролей:

  - **Управление организацией** или **Администратор безопасности** в [Центре безопасности и соответствия требованиям](permissions-in-the-security-and-compliance-center.md).
  - **Управление организацией** в [Exchange Online.](https://docs.microsoft.com/Exchange/permissions-exo/permissions-exo#role-groups)

## <a name="use-the-security--compliance-center-to-configure-the-user-submissions-mailbox"></a>Использование Центра безопасности & соответствия требованиям для настройки почтового ящика отправки пользователей

1. В Центре безопасности & соответствия требованиям  перейдите к пользовательской отправке политики \>  \> **управления угрозами.**

2. На странице **"Отправки** пользователей" выберите один из следующих параметров:

   1. Включить функцию сообщения отчета для **Outlook (рекомендуется):** выберите этот параметр, если вы используете надстройку Report Message, надстройку Report Phishing или встроенную отчетность в Outlook в Интернете, а затем настройте следующие параметры:

      - **Настройте сообщение с подтверждением для пользователя:** щелкните эту ссылку. В **отобраджемом окле** "Настройка сообщения подтверждения" настройте следующие параметры:

      - **Перед** отправкой  :  в полях "Название" и "Подтверждение" введите описательный текст, который пользователи видят, прежде чем сообщать о сообщении с помощью надстройки Report Message или надстройки Report Phishing. Вы можете использовать переменную %type% для внесения типа отправки (нежелательной, а не нежелательной, фишинговой и т. д.).

        Как уже было отмечено, при выборе параметра, который отправляет сообщения, которые отправляются в корпорацию Майкрософт, к уведомлению также добавляется следующий текст:

        > Ваше сообщение электронной почты будет отправлено корпорации Майкрософт для анализа. Некоторые сообщения электронной почты могут содержать личную или конфиденциальную информацию.

      - **После отправки:** ![ щелкните значок "Развернуть". ](../../media/scc-expand-icon.png) В **полях "Название"** и "Подтверждение" введите описательное сообщение, которое пользователи видят после сообщения с помощью надстройки Report Message или надстройки Report Phishing.  Чтобы включить тип отправки, можно использовать переменную %type%.

      Когда закончите, нажмите кнопку **Сохранить**. Чтобы очистить эти значения, нажмите **кнопку "Восстановить"** на странице **"Отправка пользователя".**

      - **Отправьте сообщения с сообщением**: сделайте один из следующих выборов:

        - **Майкрософт (рекомендуется):** почтовый ящик отправки пользователей не используется (все сообщения, о чем сообщается, перенайдут в корпорацию Майкрософт).

        - **Майкрософт и настраиваемый почтовый** ящик: в поле, которое отображается, введите адрес электронной почты существующего почтового ящика Exchange Online. Группы рассылки не разрешены. Отправки пользователей будут отправлены как в корпорацию Майкрософт для анализа, так и в пользовательский почтовый ящик для анализа администратора или группы операций безопасности.

        - **Настраиваемый почтовый** ящик: в поле, которое отображается, введите адрес электронной почты существующего почтового ящика Exchange Online. Группы рассылки не разрешены. Используйте этот параметр, если сначала сообщение будет отправляться только администратору или группе операций безопасности для анализа. Сообщения не будут отправляться в корпорацию Майкрософт, если администратор не переадминит их самостоятельно.

        > [!NOTE]
        > Государственные организации США (GCC, GCC-H и DoD) могут настраивать только **настраиваемый почтовый ящик.** Остальные два параметра отключены.

      По завершению нажмите кнопку **"Подтвердить".**

      > [!CAUTION]
      > Если вы отключили создание отчетов о нежелательной почте в [Outlook](report-junk-email-and-phishing-scams-in-outlook-on-the-web-eop.md#disable-or-enable-junk-email-reporting-in-outlook-on-the-web) в Интернете с помощью политик почтовых ящиков Outlook в Интернете, но вы настроили один из предыдущих параметров для отправки отчетов о сообщениях в корпорацию Майкрософт, пользователи смогут сообщать о сообщениях корпорации Майкрософт в Outlook в Интернете с помощью надстройки Report Message или надстройки Report Phishing.

   - Отключать функцию сообщения отчетов для **Outlook:** выберите этот параметр, если вы используете сторонние средства создания отчетов вместо надстройки Report Message, надстройки Report Phishing или встроенной отчетности в Outlook в Интернете, а затем настройте следующие параметры:

      Выберите **"Использовать этот настраиваемый почтовый ящик для получения отправленных пользователем сообщений".** В этом поле введите адрес электронной почты существующего почтового ящика, который уже находится в Office 365. Это должен быть существующий почтовый ящик в Exchange Online, который может получать электронную почту.

      По завершению нажмите кнопку **"Подтвердить".**

## <a name="message-submission-format"></a>Формат отправки сообщения

Сообщения, от отправленные в настраиваемые почтовые ящики, должны иметь определенный формат отправки. Тема (название конверта) отправки должна иметь такой формат:

`SafetyAPIAction|NetworkMessageId|SenderIp|FromAddress|(Message Subject)`

где SafetyAPIAction — одно из следующих значений:

- 1: нежелаая почта
- 2: не является нежелательным
- 3: фишинг

В следующем примере:

- Сообщение сообщается как фишинговое.
- ИД сетевого сообщения: 49871234-6dc6-43e8-abcd-08d797f20abe.
- IP-адрес отправитель — 167.220.232.101.
- Адрес "От" test@contoso.com.
- Строка темы сообщения — "тестовая отправка фишинга"

`3|49871234-6dc6-43e8-abcd-08d797f20abe|167.220.232.101|test@contoso.com|(test phishing submission)`

Сообщения, которые не следуют этому формату, не отображаются должным образом на портале отправки.
