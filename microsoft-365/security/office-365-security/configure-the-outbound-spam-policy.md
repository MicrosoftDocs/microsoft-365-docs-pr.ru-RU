# <a name="configure-outbound-spam-filtering-in-eop"></a>Настройка фильтрации исходящие спама в EOP

[!INCLUDE [Microsoft 365 Defender rebranding](../includes/microsoft-defender-for-office.md)]

**Область применения**
- [Exchange Online Protection](https://go.microsoft.com/fwlink/?linkid=2148611)
- [Microsoft Defender для Office 365 (план 1 и план 2)](https://go.microsoft.com/fwlink/?linkid=2148715)
- [Microsoft 365 Defender](https://go.microsoft.com/fwlink/?linkid=2118804)

В организациях Microsoft 365 с почтовыми ящиками в организациях Exchange Online или автономных организациях Exchange Online Protection (EOP) без почтовых ящиков Exchange Online исходящие сообщения электронной почты, отправляемые через EOP, автоматически проверяются на наличие нежелательной почты и необычных действий при отправке.

Исходящие нежелательные сообщения от пользователя в организации обычно указывают на компрометации учетной записи. Подозрительные исходящие сообщения помечается как нежелательная почта (независимо от уровня уверенности в нежелательной почте или SCL) и маршрутются через пул доставки сообщений с высоким уровнем риска, чтобы защитить репутацию службы (то есть оградить исходные почтовые серверы Microsoft 365 от списков заблокированных [IP-адресов).](high-risk-delivery-pool-for-outbound-messages.md) Администраторы автоматически получают уведомления о подозрительной активности исходящие сообщения электронной почты и блокируют пользователей с помощью [политик оповещений.](../../compliance/alert-policies.md)

EOP использует политики защиты от нежелательной почты исходящие сообщения в рамках общей защиты организации от нежелательной почты. Дополнительные сведения см. в статье [Защита от нежелательной почты](anti-spam-protection.md).

Администраторы могут просматривать, изменять и настраивать (но не удалять) политику нежелательной почты для исходящие сообщения по умолчанию. Для большей детализации можно также создать настраиваемые политики нежелательной почты исходящие, которые применяются к определенным пользователям, группам или доменам в организации. Настраиваемые политики всегда имеют приоритет перед политикой по умолчанию, но вы можете изменить приоритеты (порядок применения) своих настраиваемых политик.

Политики защиты от нежелательной почты для исходящие сообщений можно настроить в Центре безопасности и & соответствия требованиям или в PowerShell (Exchange Online PowerShell для организаций Microsoft 365 с почтовыми ящиками в Exchange Online; автономный EOP PowerShell для организаций без почтовых ящиков Exchange Online).

Основные элементы политики нежелательной почты исходящие в EOP:

- **Политика фильтрации исходящие нежелательной** почты : определяет действия для решения фильтрации исходящие нежелательной почты и параметры уведомлений.
- **Правило фильтрации исходящие нежелательной** почты : определяет приоритет и фильтры получателей (к кому применяется политика) для политики фильтрации исходящие нежелательной почты.

Разница между этими двумя элементами не очевидна при управлении политиками нежелательной почты исходящие сообщения в Центре безопасности & соответствия требованиям:

- При создании политики фактически создается правило фильтрации исходящие нежелательной почты и связанная с ней политика фильтрации исходящие нежелательной почты одновременно с использованием одного и того же имени для обоих этих правил.
- При изменении политики параметры, связанные с именем, приоритетом, включенным или отключенным фильтрами получателей, изменяют правило фильтрации исходящие нежелательной почты. Все остальные параметры изменяют связанную политику фильтрации исходящие спама.
- При удалении политики удаляются правило фильтрации исходящие нежелательной почты и связанная с ней политика фильтрации исходящие нежелательной почты.

В Exchange Online PowerShell или автономном EOP PowerShell вы управляете политикой и правилом отдельно. Дополнительные сведения см. в разделе "Использование Exchange Online PowerShell или автономный [EOP PowerShell](#use-exchange-online-powershell-or-standalone-eop-powershell-to-configure-outbound-spam-policies) для настройки политик исходящие нежелательной почты" далее в этой статье.

В каждой организации есть встроенная политика нежелательной почты для исходящие сообщений с именем Default, которая имеет указанные здесь свойства.

- Политика применяется к всем получателям в организации, даже если с политикой не связано ни одно правило фильтрации исходящие нежелательной почты (фильтры получателей).
- Для политики задано специальное значение приоритета **Самый низкий**, которое невозможно изменить (эта политика всегда применяется последней). Все созданные специальные политики всегда имеют более высокий приоритет, чем политика с именем "По умолчанию".
- Эта политика является стандартной (для свойства **IsDefault** задано значение `True`), и удалить стандартную политику невозможно.

Чтобы повысить эффективность фильтрации исходящие нежелательные сообщения, можно создать настраиваемые политики исходящие нежелательные сообщения с более строгими настройками, которые применяются к определенным пользователям или группам пользователей.

## <a name="use-the-security--compliance-center-to-create-outbound-spam-policies"></a>Использование Центра безопасности & соответствия требованиям для создания политик защиты от нежелательной почты для исходящие сообщения

При создании настраиваемой политики нежелательной почты исходящие сообщения в Центре безопасности & соответствия требованиям одновременно создаются правило фильтрации нежелательной почты и связанная с ней политика фильтрации нежелательной почты с одинаковым именем.

1. В Центре безопасности и соответствия требованиям выберите **Управление угрозами** \> **Политика** \> **Защита от спама**.

2. На странице **"Параметры борьбы с нежелательной** почтой" щелкните **"Создать политику исходящие сообщения".**

3. В **открываемой** политике фильтрации исходящие нежелательной почты настройте следующие параметры:

   - **Имя**. Укажите уникальное, понятное имя политики.

   - **Описание**. Укажите необязательное описание политики.

4. (Необязательно) **Расширьте раздел** "Уведомления", чтобы настроить дополнительных пользователей, которые должны получать копии и уведомления о подозрительных исходящие сообщения электронной почты:

   - **Отправьте копию подозрительных** исходящие сообщения электронной почты определенным пользователям. Этот параметр добавляет указанных пользователей в качестве получателей "СК" в подозрительные исходящие сообщения.

     > [!NOTE]
     > Этот параметр работает только в политике нежелательной почты для исходящие сообщения по умолчанию. Он не работает в настраиваемые политики исходящие нежелательные сообщения, которые вы создаете.

     Чтобы включить этот параметр, с помощью этого параметра:

     1. Чтобы включить параметр, выберите этот параметр.

     1. Нажмите **кнопку "Добавить людей".** В **окле "Добавление и удаление получателей"** отображается:

     1. Введите электронный адрес отправителя. Можно указать несколько адресов электронной почты, разделенных точками с за ;) или один получатель на строку.

     1. Щелкните ![Значок добавления](../../media/c2dd8b3a-5a22-412c-a7fa-143f5b2b5612.png) для добавления получателей.

        Повторите эти действия необходимое количество раз.

        Добавленные получатели отображаются в **разделе** списка получателей во flyout. Чтобы удалить получателя, нажмите ![ кнопку "Удалить". ](../../media/scc-remove-icon.png)

     1. По завершении нажмите кнопку **Сохранить**.

        Чтобы отключить этот параметр, необходимо установить этот параметр.

   - **Уведомлять определенных пользователей, если отправитель заблокирован из-за отправки исходящие нежелательные сообщения:**

     > [!IMPORTANT]
     >
     > - В настоящее время этот параметр не является частью политик нежелательной почты для исходящие сообщений.
     >
     > - Политика [](../../compliance/alert-policies.md) оповещений по умолчанию User **restricted from sending email** already sends email notifications to members of the **TenantAdmins** (**Global admins)** group when users are blocked due to exceeding the limits in the **Recipient Limits** section. Мы настоятельно рекомендуем использовать политику оповещений, а не этот параметр в политике нежелательной почты для исходящие сообщения для уведомления **администраторов и других пользователей.** Инструкции см. в [подсети "Проверка параметров оповещений для пользователей с ограниченным доступом".](removing-user-from-restricted-users-portal-after-spam.md#verify-the-alert-settings-for-restricted-users)

5. (Необязательно) **Разоритте** раздел "Ограничения получателей", чтобы настроить ограничения и действия для подозрительных исходящие сообщения электронной почты:

   > [!NOTE]
   > Эти параметры применимы только к облачным почтовым ящикам.

   - **Максимальное количество получателей на пользователя**

     Допустимые значения— от 0 до 10 000. Значение по умолчанию — 0, что означает, что используются значения по умолчанию для службы. Дополнительные сведения см. в [ограничениях на отправку.](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#sending-limits-1)

     - **Внешнее почасовая ограничение**: максимальное количество внешних получателей в час.

     - **Внутреннее почасовая ограничение:** максимальное количество внутренних получателей в час.

     - **Ежедневное** ограничение : максимальное общее число получателей в день.

   - **Действие, когда пользователь превышает** вышеперечисленные ограничения: настройте действие, чтобы сделать это при превышении одного из ограничений получателей.  Для всех действий получатели,  указанные в политике предупреждений пользователя,  не могут отправлять сообщения электронной почты (а в теперь избыточном уведомлении определенных пользователей, если отправитель заблокирован из-за параметра нежелательной почты для отправки исходящие сообщения в политике нежелательной почты, получают уведомления по электронной почте.

     - **Запретить пользователю отправлять почту до следующего дня:** это значение по умолчанию. Уведомления отправляются по электронной почте, и пользователь не сможет отправлять больше сообщений до следующего дня в зависимости от времени в UTC. Администратор не может переопредить этот блок.

       - Оповещение об активности **с именем "Пользователь"** с ограничением на отправку электронной почты оповещает администраторов (по электронной почте и на странице "Просмотр **оповещений").**

       - Все получатели, указанные в уведомлении определенных пользователей, если отправитель заблокирован из-за параметра исходящие нежелательные сообщения в политике также уведомляются. 

       - Пользователь не сможет отправлять больше сообщений до следующего дня в зависимости от времени в UTC. Администратор не может переопредить этот блок.

     - Запретить пользователю отправлять **почту:** отправляются уведомления по электронной почте, пользователь добавляется на портал [Пользователи с ограниченным доступом **&] <https://sip.protection.office.com/restrictedusers>** в Центре  безопасности и соответствия требованиям, и пользователь не может отправлять электронную почту, пока администратор не удалит его с портала "Пользователи с ограниченным доступом". После того как администратор удалит пользователя из списка, он не будет снова ограничен в течение этого дня. Инструкции см. в удалении пользователя с портала "Пользователи с ограниченным [доступом" после отправки нежелательной почты.](removing-user-from-restricted-users-portal-after-spam.md)

     - **Никаких действий, только оповещение:** отправка уведомлений по электронной почте.

6. (Необязательно) Раздел **"Автоматическая переадружение"** позволяет контролировать автоматическую переадружение электронной почты пользователями внешним отправителям. Дополнительные сведения см. в поддоме "Управление автоматической переадружаемой внешней электронной [почтой" в Microsoft 365.](external-email-forwarding.md)

   > [!NOTE]
   >
   > - До сентября 2020 г. эти параметры доступны, но не применяются.
   >
   > - Эти параметры применяются только к облачным почтовым ящикам.
   >
   > - Если автоматическая пересылаемая почта отключена, получатель получает отчет о неотвечаемой доставке (также известный как сообщение о возврате), если внешние отправитые отправляют электронную почту в почтовый ящик, в который была отправлена пересылаемая почта. Если сообщение отправляется внутренним отправищиком, а методом пересылки является пересылание почтовых ящиков [(также](https://docs.microsoft.com/exchange/recipients-in-exchange-online/manage-user-mailboxes/configure-email-forwarding) известное как пересылание _SMTP),_ внутренний отправитель получит этот запрос.  Внутренний отправитель не получит сообщение о задержанной отправке, если переадреание произошло из-за правила для почтового ящика.

   Ниже представлены возможные значения.

   - **Автоматическое управление системой:** позволяет фильтровать исходящие нежелательные сообщения для управления автоматической переадружаемой внешней электронной почты. Это значение используется по умолчанию.
   - **On**: Automatic external email forwarding is not disabled by the policy.
   - **Выключите:** политика отключит автоматическую переадружение внешней электронной почты.

7. (Обязательно) **Разослать раздел "Применено",** чтобы определить внутренних отправителей, к которые применяется политика.

    Условие или исключение можно использовать только один раз, но можно указать для них несколько значений. Указать несколько значений в одном условии или исключении можно с помощью оператора OR (например, _\<sender1\>_ or _\<sender2\>_). Между разными условиями и исключениями используется оператор AND (например, _\<sender1\>_ and _\<member of group 1\>_).

    Проще всего нажать кнопку **Добавить условие** три раза, чтобы увидеть все доступные условия. Чтобы удалить условия, которые не нужно настраивать, можно щелкнуть ![Кнопка удаления](../../media/scc-remove-icon.png).

    - **Домен отправителей:** указывает отправителей в одном или более настроенных принятых доменах в организации. Щелкните поле **Добавить тег**, чтобы просмотреть и выбрать домен. Еще раз щелкните поле **Добавить тег**, чтобы выбрать дополнительные домены, если доступно несколько доменов.

    - **Sender is**: Specifies one or more users in your organization. Щелкните в поле **Добавить тег** и начните вводить текст, чтобы отфильтровать список. Щелкните еще **раз в поле "Добавить тег",** чтобы выбрать дополнительных отправителей.

    - **Sender is a member of**: Specifies one or more groups in your organization. Щелкните в поле **Добавить тег** и начните вводить текст, чтобы отфильтровать список. Щелкните еще **раз в поле "Добавить тег",** чтобы выбрать дополнительных отправителей.

    - **За исключением случаев, когда**. Чтобы добавить исключение из правила, щелкните **Добавьте условие** три раза, чтобы увидеть все доступные исключения. Параметры и поведение полностью идентичны условиям.

8. По завершении нажмите кнопку **Сохранить**.

### <a name="set-the-priority-of-custom-outbound-spam-policies"></a>Настройка приоритета настраиваемой политики нежелательной почты для исходящие сообщения

Настраиваемые политики нежелательной почты для исходящие сообщений отображаются в порядке их обработки (первая политика имеет значение **приоритета** 0). Политика фильтрации исходяшей нежелательной почты по умолчанию с именем **"Политика** фильтрации исходяшей нежелательной почты" имеет значение приоритета **"Самый** низкий", и изменить его нельзя.

2. На странице **"Параметры нежелательной** почты" найдите политики, значением которых в столбце **"Тип"** является **настраиваемая политика нежелательной почты для исходящие сообщения.** Обратите внимание на значения в столбце **Приоритет**:

   - Настраиваемая политика нежелательной почты исходящие сообщения с наименьшим приоритетом имеет значение "Стрелка вверх" n (например, значок "Стрелка ![ ](../../media/ITPro-EAC-UpArrowIcon.png)  ![ ](../../media/ITPro-EAC-UpArrowIcon.png) **вверх" 3).**

   - Если у вас есть три или более пользовательских политик нежелательной почты для исходящие сообщения, политики между самым высоким и низким приоритетом имеют значения значок стрелки вверх вниз n (например, значок стрелки вверх ![ ](../../media/ITPro-EAC-UpArrowIcon.png)![ вниз ](../../media/ITPro-EAC-DownArrowIcon.png)  ![ ](../../media/ITPro-EAC-UpArrowIcon.png)![ ](../../media/ITPro-EAC-DownArrowIcon.png) **2).**

3. Щелкните ![Значок "Стрелка вверх"](../../media/ITPro-EAC-UpArrowIcon.png) или ![Значок "Стрелка вниз"](../../media/ITPro-EAC-DownArrowIcon.png) для перемещения настраиваемой политики нежелательной почты исходящие сообщения вверх или вниз в списке приоритетов.

## <a name="use-exchange-online-powershell-or-standalone-eop-powershell-to-configure-outbound-spam-policies"></a>Использование Exchange Online PowerShell или автономный EOP PowerShell для настройки политик исходящие спама

Как было описано выше, политика исходящие нежелательных сообщений состоит из политики фильтрации исходящие нежелательной почты и правила фильтрации исходящие нежелательной почты.

В Exchange Online PowerShell или в автономных EOP PowerShell разница между политиками фильтрации исходящие нежелательной почты и правилами фильтрации исходящие нежелательной почты очевидна. Вы управляете политиками фильтрации исходящей нежелательной почты с помощью cmdlets **\* -HostedOutboundSpamFilterPolicy,** а правила фильтрации исходящей нежелательной почты с помощью **\* -HostedOutboundSpamFilterRule.**

- В PowerShell сначала создается политика фильтрации исходящие нежелательной почты, а затем правило фильтрации исходящие нежелательной почты, определяя политику, к которую применяется правило.
- В PowerShell параметры политики фильтрации исходящие нежелательной почты и правила фильтрации исходящие нежелательной почты изменяются отдельно.
- При удалении политики фильтрации исходящие нежелательной почты из PowerShell соответствующее правило фильтрации исходящие нежелательной почты не удаляется автоматически и наоборот.

### <a name="use-powershell-to-create-outbound-spam-policies"></a>Создание политик нежелательной почты для исходящие сообщения с помощью PowerShell

Создание политики нежелательной почты для исходящие сообщений в PowerShell — это двухшаговая процедура:

1. Создайте политику фильтрации исходящие нежелательной почты.
2. Создайте правило фильтрации исходящие нежелательной почты, которое определяет политику фильтрации исходящие нежелательной почты, к которую оно применяется.

 **Примечания**.

- Вы можете создать новое правило фильтрации исходящие нежелательной почты и назначить ему существующую несвязаную политику фильтрации исходящие нежелательной почты. Правило фильтрации исходящие нежелательной почты не может быть связано с более чем одной политикой фильтрации исходящие нежелательной почты.

- Вы можете настроить следующие параметры для новых политик фильтрации исходящие нежелательной почты в PowerShell, которые недоступны в Центре безопасности и соответствия требованиям & до создания политики:

  - Создайте новую политику как отключенную (включена в `$false` **cmdlet New-HostedOutboundSpamFilterRule).**
  - Установите приоритет политики во время создания _(Priority)_ в _\<Number\>_ **cmdlet New-HostedOutboundSpamFilterRule).**

- Новая политика фильтрации исходящие нежелательной почты, которую вы создаете в PowerShell, не отображается в Центре безопасности & соответствия требованиям, пока вы не назначите политику правилу фильтрации нежелательной почты.

#### <a name="step-1-use-powershell-to-create-an-outbound-spam-filter-policy"></a>Шаг 1. Создание политики фильтрации исходящие нежелательной почты с помощью PowerShell

Чтобы создать политику фильтрации исходящие спама, используйте этот синтаксис:

```PowerShell
New-HostedOutboundSpamFilterPolicy -Name "<PolicyName>" [-AdminDisplayName "<Comments>"] <Additional Settings>
```

В этом примере создается новая политика фильтрации исходящие нежелательной почты с именем Contoso Executives со следующими настройками:

- Ограничения на количество получателей ограничены небольшими значениями, за которые по умолчанию. Дополнительные сведения [см. в ограничениях на отправку в параметрах Microsoft 365.](https://docs.microsoft.com/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#sending-limits-across-office-365-options)

- После того как достигнуто одно из ограничений, пользователю не будет отправлять сообщения.

```PowerShell
New-HostedOutboundSpamFilterPolicy -Name "Contoso Executives" -RecipientLimitExternalPerHour 400 -RecipientLimitInternalPerHour 800 -RecipientLimitPerDay 800 -ActionWhenThresholdReached BlockUser
```

Подробные сведения о синтаксисах и параметрах см. в описании [New-HostedOutboundSpamFilterPolicy.](https://docs.microsoft.com/powershell/module/exchange/new-hostedoutboundspamfilterpolicy)

#### <a name="step-2-use-powershell-to-create-an-outbound-spam-filter-rule"></a>Шаг 2. Создание правила фильтрации исходящие нежелательной почты с помощью PowerShell

Чтобы создать правило фильтрации исходящие нежелательные сообщения, используйте этот синтаксис:

```PowerShell
New-HostedOutboundSpamFilterRule -Name "<RuleName>" -HostedOutboundSpamFilterPolicy "<PolicyName>" <Recipient filters> [<Recipient filter exceptions>] [-Comments "<OptionalComments>"]
```

В этом примере создается новое правило фильтрации нежелательной почты для исходящие сообщений с именем Contoso Executives с такими настройками:

- Политика фильтрации исходящие нежелательной почты с именем Contoso Executives связана с правилом.

- Правило применяется к участникам группы Contoso Executives (Руководители Contoso).

```PowerShell
New-HostedOutboundSpamFilterRule -Name "Contoso Executives" -HostedOutboundSpamFilterPolicy "Contoso Executives" -SentToMemberOf "Contoso Executives Group"
```

Подробные сведения о синтаксисах и параметрах см. в описании [new-HostedOutboundSpamFilterRule.](https://docs.microsoft.com/powershell/module/exchange/new-hostedoutboundspamfilterrule)

### <a name="use-powershell-to-view-outbound-spam-filter-policies"></a>Просмотр политик фильтрации исходящие нежелательных сообщений с помощью PowerShell

Чтобы получить сводный список всех политик фильтрации исходящие нежелательной почты, запустите команду:

```PowerShell

```

Подробные сведения о синтаксисах и параметрах см. в описании [get-HostedOutboundSpamFilterPolicy.](https://docs.microsoft.com/powershell/module/exchange/get-hostedoutboundspamfilterpolicy)

### <a name="use-powershell-to-view-outbound-spam-filter-rules"></a>Просмотр правил фильтрации исходящие нежелательных сообщений с помощью PowerShell

Чтобы просмотреть существующие правила фильтрации исходящие спама, используйте следующий синтаксис:

```

Чтобы получить сводный список всех правил фильтрации исходящие нежелательной почты, запустите команду:

Get-HostedOutboundSpamFilterRule -Identity "<RuleName>" | Format-List [<Specific properties to view>]
```

В этом примере возвращаются все значения свойств правила фильтрации исходящие нежелательной почты с именем Contoso Executives.

```PowerShell
Get-HostedOutboundSpamFilterRule -Identity "Contoso Executives" | Format-List
```

Подробные сведения о синтаксисах и параметрах см. в описании [get-HostedOutboundSpamFilterRule.](https://docs.microsoft.com/powershell/module/exchange/get-hostedoutboundspamfilterrule)

### <a name="use-powershell-to-modify-outbound-spam-filter-policies"></a>Использование PowerShell для изменения политик фильтрации исходящие нежелательной почты

При изменении политики фильтрации вредоносных программ в PowerShell доступны те же параметры, что и при создании политики, как описано в шаге [1 .Use PowerShell to create an outbound spam filter policy](#step-1-use-powershell-to-create-an-outbound-spam-filter-policy) section earlier in this article.

> [!NOTE]
> Политику фильтрации исходящей нежелательной почты переименовать нельзя (у cmdlet **Set-HostedOutboundSpamFilterPolicy** нет _параметра Name)._ При переименовании политики защиты от нежелательной почты исходящие сообщения в Центре безопасности & соответствия требованиям переименовывать только правило фильтрации исходящие нежелательной _почты._

Чтобы изменить политику фильтрации исходящие спама, используйте синтаксис:

```PowerShell
Set-HostedOutboundSpamFilterPolicy -Identity "<PolicyName>" <Settings>
```

Подробные сведения о синтаксисах и параметрах см. в описании [Set-HostedOutboundSpamFilterPolicy.](https://docs.microsoft.com/powershell/module/exchange/set-hostedoutboundspamfilterpolicy)

### <a name="use-powershell-to-modify-outbound-spam-filter-rules"></a>Изменение правил фильтрации исходящие нежелательных сообщений с помощью PowerShell

Единственный параметр, который не доступен при изменении правила фильтрации исходящие нежелательной почты в PowerShell, — это параметр _Enabled,_ который позволяет создать отключенное правило. Чтобы включить или отключить существующие правила фильтрации исходящие спама, см. следующий раздел.

В противном случае при изменении правила фильтрации исходящие нежелательной почты в PowerShell дополнительные параметры недоступны. Те же параметры доступны при создании правила, как описано в разделе "Шаг 2. Создание правила фильтрации исходящие нежелательных сообщений с помощью [PowerShell"](#step-2-use-powershell-to-create-an-outbound-spam-filter-rule) ранее в этой статье.

Чтобы изменить правило фильтрации исходящие нежелательные сообщения, используйте этот синтаксис:

```PowerShell
Set-HostedOutboundSpamFilterRule -Identity "<RuleName>" <Settings>
```

Подробные сведения о синтаксисах и параметрах см. в описании [set-HostedOutboundSpamFilterRule.](https://docs.microsoft.com/powershell/module/exchange/set-hostedoutboundspamfilterrule)

### <a name="use-powershell-to-enable-or-disable-outbound-spam-filter-rules"></a>Использование PowerShell для отключения правил фильтрации исходящие нежелательной почты

Включение или отключение правила фильтрации исходящие нежелательной почты в PowerShell включает или отключает всю политику исходящие нежелательной почты (правило фильтрации исходящие нежелательной почты и назначенная политика фильтрации исходящие нежелательной почты). Вы не можете включить или отключить политику нежелательной почты для исходящие сообщения по умолчанию (она всегда применяется к всем получателям).

Чтобы включить или отключить правило фильтрации исходящие спама в PowerShell, используйте синтаксис:

```PowerShell
<Enable-HostedOutboundSpamFilterRule | Disable-HostedOutboundSpamFilterRule> -Identity "<RuleName>"
```

В этом примере отключается правило фильтрации нежелательной почты исходящие сообщения с именем Marketing Department.

```PowerShell
Disable-HostedOutboundSpamFilterRule -Identity "Marketing Department"
```

В этом примере включается то же правило.

```PowerShell
Enable-HostedOutboundSpamFilterRule -Identity "Marketing Department"
```

Подробные сведения о синтаксисах и параметрах см. в описании [Enable-HostedOutboundSpamFilterRule](https://docs.microsoft.com/powershell/module/exchange/enable-hostedoutboundspamfilterrule) и [Disable-HostedOutboundSpamFilterRule.](https://docs.microsoft.com/powershell/module/exchange/disable-hostedoutboundspamfilterrule)

### <a name="use-powershell-to-set-the-priority-of-outbound-spam-filter-rules"></a>Настройка приоритета правил фильтрации исходящие нежелательных сообщений с помощью PowerShell

Максимальный приоритет, который можно задать для правила, — 0. Минимальное значение зависит от количества правил. Например, если у вас есть пять правил, вы можете использовать значения 0–4. Изменение приоритета существующего правила оказывает каскадное влияние на другие правила. Например, если вы измените приоритет правила на 2, когда у вас есть пять настраиваемых правил (с приоритетами от 0 до 4), то для существующего правила с приоритетом 2 будет задан приоритет 3, а для правила с приоритетом 3 будет задан приоритет 4.

Чтобы установить приоритет правила фильтрации исходящие спама в PowerShell, используйте следующий синтаксис:

```PowerShell
Set-HostedOutboundSpamFilterRule -Identity "<RuleName>" -Priority <Number>
```

В этом примере для правила Marketing Department задается приоритет 2. Все правила с приоритетом не больше 2 понижаются на один ранг (значения приоритета увеличиваются на 1).

```PowerShell
Set-HostedOutboundSpamFilterRule -Identity "Marketing Department" -Priority 2
```

> [!NOTE]
>
> - Чтобы установить приоритет нового правила при его создании, используйте параметр _Priority_ в параметре **New-HostedOutboundSpamFilterRule.**
>
> - Политика фильтрации нежелательной почты по умолчанию для исходящие сообщения не имеет соответствующего правила фильтрации нежелательной почты и всегда имеет немодифимируемый приоритет **"Самый низкий".**

### <a name="use-powershell-to-remove-outbound-spam-filter-policies"></a>Удаление политик фильтрации исходящие нежелательной почты с помощью PowerShell

При удалении политики фильтрации исходящие нежелательной почты с помощью PowerShell соответствующее правило фильтрации исходящие нежелательной почты не удаляется.

Чтобы удалить политику фильтрации исходящие спама в PowerShell, используйте синтаксис:

```PowerShell
Remove-HostedOutboundSpamFilterPolicy -Identity "<PolicyName>"
```

В этом примере удаляется политика фильтрации исходящие нежелательной почты с именем Marketing Department.

```PowerShell
Remove-HostedOutboundSpamFilterPolicy -Identity "Marketing Department"
```

Подробные сведения о синтаксисах и параметрах см. в описании [remove-HostedOutboundSpamFilterPolicy.](https://docs.microsoft.com/powershell/module/exchange/remove-hostedoutboundspamfilterpolicy)

### <a name="use-powershell-to-remove-outbound-spam-filter-rules"></a>Удаление правил фильтрации исходящие нежелательных сообщений с помощью PowerShell

При удалении правила фильтрации исходящие нежелательной почты с помощью PowerShell соответствующая политика фильтрации исходящие нежелательной почты не удаляется.

Чтобы удалить правило фильтрации исходящие нежелательной почты в PowerShell, используйте синтаксис:

```PowerShell
Remove-HostedOutboundSpamFilterRule -Identity "<PolicyName>"
```

В этом примере удаляется правило фильтрации исходящие нежелательной почты с именем Marketing Department.

```PowerShell

```
