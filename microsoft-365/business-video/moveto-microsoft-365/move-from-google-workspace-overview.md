# <a name="switch-from-google-workspace---overview"></a>Переход из рабочего пространства Google — обзор

Выполните следующие действия, чтобы переместить данные, электронную почту и пользователей из Рабочей области Google в Microsoft 365 для бизнеса.


| Шаг  |Описание  |
|---------|---------|
|Шаг 1 |  [Подпишитесь](../sign-up.md) на Microsoft 365 бизнес премиум       |
|Шаг 2 |   [Настройка Microsoft 365 для миграции рабочей области Google.](set-up-microsoft-365-forgoogle.md) </br> На этом шаге вы утвердите свой домен, который используется в Рабочей области Google, добавим пользователей и настроим базовую безопасность для устройств. |
|Шаг 3 | [Настройка политик безопасности для Windows устройств.](../secure-win10-pcs.md)</br> Windows безопасности устанавливается отдельно на странице Установки центра администрирования Microsoft 365 администратора. |
|Шаг 4|[Добавьте домен Google Workspace в Microsoft 365](add-google-domain.md) </br> После проверки вашего собственного домена, используемого для электронной почты, вы и все другие пользователи можете начать использовать старую электронную почту, чтобы войти в Microsoft 365 бизнес премиум. |
|Шаг 5 | [Установка Office и Microsoft Teams](../install-office.md).</br> Все люди, у которых Microsoft 365 лицензии, должны установить Office приложения на своих устройствах.|
|Шаг 6 | [Перенос всех элементов электронной почты и календаря.](migrate-email.md)</br> На этом шаге будет Exchange Online миграция для перемещения электронной почты, календаря и контактов каждого из Рабочей области Google.  |
|Шаг 7 | [Подключение домена Microsoft 365](connect-domain-tom365.md). </br> После подключения электронной почты домена начнется Microsoft 365, и все Microsoft 365 службы будут работать.|
|Шаг 8|Использование [Mover для перемещения](mover-migrate-files.md) всех данных с диска на OneDrive и с общих дисков на сайты Team.</br> На этом этапе все данные в персональных и общих дисках копируется и перемещается в Microsoft 365.|
|Этап 9| [Прекратите использование рабочей области Google,](cancel-google.md) но храните домен. </br> Если Google управляет доменом, его можно сохранить даже после прекращения подписки на Рабочее пространство Google. Вы также можете переместить его в другой DNS-хост, если хотите.|
