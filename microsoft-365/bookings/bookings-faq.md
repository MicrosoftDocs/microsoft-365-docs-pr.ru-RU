# <a name="microsoft-bookings-frequently-asked-questions"></a>Часто задаваемые вопросы о книгах Майкрософт

## <a name="general"></a>Общие

### <a name="what-is-microsoft-bookings"></a>Что такое резервирование Майкрософт?

Microsoft Books — это приложение Microsoft 365, которое упрощает планирование и управление встречами. Резервирование включает в себя календарь резервирования на основе веб-сайта и интегрируется с Outlook для оптимизации календаря сотрудников, что позволяет заказчикам гибко заменять время, которое лучше всего подходит для их работы. Автоматическое оповещение по электронной почте. Уменьшите количество непоказов, и организациям необходимо сэкономить время с целью сокращения повторяющихся задач планирования. С помощью резервирований можно легко выполнять виртуальные встречи с помощью Skype или Microsoft Teams, а также управлять повседневным планированием с помощью приложения "книги" в Teams. Благодаря встроенной возможности настройки резервирования предназначены для удовлетворения потребностей нескольких частей любой организации.

### <a name="how-do-i-use-microsoft-bookings"></a>Как использовать резервирование Майкрософт?

Резервирования — это веб-служба, поэтому вам не нужно загружать ничего. Просто перейдите к разделу "Выбор приложений" в веб-интерфейсе Microsoft 365. Администраторы также могут использовать вспомогательное приложение "книги", чтобы оставаться актуальными сведения о клиентах и их встречах.

### <a name="who-has-access-to-microsoft-bookings"></a>У кого есть доступ к книгам Майкрософт?

По умолчанию для Microsoft 365 бизнес премиум доступны и активны книги Microsoft 365 для бизнеса Standard, Microsoft для бизнеса Standard, A3, A5, E3 и т. д. Резервирования также доступны в Office 365 под управлением 21Vianet.

### <a name="will-my-customers-see-my-personal-or-business-calendar"></a>Будут ли клиенты видеть личный или деловой календарь?

Ваши клиенты увидят только календарь резервирования, который вы публикуете в Интернете для служб, времени и сотрудников, которые вы выбираете для регистрации.

### <a name="how-do-end-users-access-the-bookings-app"></a>Как конечные пользователи получают доступ к приложению для работы с резервированиями?

Пользователи, которым назначена лицензия на Microsoft 365 бизнес премиум, Microsoft 365 бизнес Standard, A3, A5, E3,, а также добавлены как сотрудники в роли администратора или средства просмотра могут получать доступ к книгам в средстве выбора приложений в веб-интерфейсе Microsoft 365. Кроме того, доступно вспомогательное приложение для iOS и Android.

### <a name="can-i-use-bookings-in-my-own-web-site"></a>Можно ли использовать резервирования на своем веб-сайте?

Да. Мы предоставляем способ внедрить календарь резервирования на сайте с помощью iFrame. Код внедрения ссылок находится на вкладке Страница " **книги** " в приложении "книги".

Да. Мы предоставляем ссылку на страницу резервирования на вкладке **страницы резервирования** в веб-приложении. Вам нужно просто предоставить эту ссылку клиентам или клиентам, и они увидят последние сведения о доступности для вашего бизнеса. Кроме того, вы можете предоставить доступ к странице "резервирование" на социальных носителях или даже использовать функцию внедрения для размещения ее в iFrame. Вы также можете управлять тем, кто имеет доступ к странице, отключив индексирование прямой поисковой системы и ограничив доступ только теми, которые находятся в вашей организации.

Подробные [сведения](turn-bookings-on-or-off.md) см. в разделе "разрешить только для выбранных пользователей создавать календари учета".

## <a name="pricing-and-licensing"></a>Цены и лицензирование

### <a name="how-do-i-get-microsoft-bookings"></a>Как получить резервирование корпорации Майкрософт?

Книги доступны в Microsoft 365 для клиентов с помощью лицензий Microsoft 365 бизнес премиум, Microsoft 365 бизнес Standard, A3, A5, E3 и водо, а также вспомогательного приложения для iOS и Android. Резервирование недоступно в качестве автономного приложения. Outlook Web App или Outlook в Интернете должны быть включены для использования резервирований, так как они хранят данные в Outlook.

Лицензия на резервирование обеспечивает полную функциональность продукта, в том числе создание календарей и управление ими. Он также позволяет пользователям просматривать и редактировать существующие календари, когда эти пользователи добавляются как сотрудники в роли администратора или средства просмотра.

### <a name="do-i-need-to-migrate-my-email-account-from-my-current-provider-to-microsoft-365"></a>Нужно ли перенести свою учетную запись электронной почты от текущего поставщика в Microsoft 365?

Вы можете сохранить текущий поставщик, но при этом все уведомления будут отправляться в электронную почту, используемую в учетной записи Microsoft 365.

### <a name="can-my-employees-use-bookings-without-a-microsoft-365-account"></a>Могут ли мои сотрудники использовать резервирования без учетной записи Microsoft 365?

Да. Вы можете добавлять свои сотрудники с помощью любой электронной почты, а также по-прежнему получать подтверждение электронной почты и приглашение на календарь, когда кто-то запишет встречу с ними.

### <a name="can-i-create-more-than-one-bookings-calendar-under-the-same-microsoft-365-account-and-switch-between-them"></a>Можно ли создать несколько книг в одной и той же учетной записи Microsoft 365 и переключаться между ними?

Да. Вы можете создавать и управлять несколькими книгами календаря с одной учетной записью. Вы можете переключаться между ними, используя курсор рядом с названием бизнеса в веб-приложении "книги учета".

### <a name="what-if-my-organization-has-mixed-licenses-with-f1f3-e1-and-e3e5"></a>Что делать, если в Организации есть смешанные лицензии с клавишами F1/F3, E1 и E3/, то есть?

Мы понимаем, что многие организации имеют смесь Microsoft 365 Licensing. Например, у клиента могут быть лицензии M365 E3 для сотрудников в головном офисе, но M365 E1 (или F1 или F3) для своих сотрудников магазина.

В этом примере сотрудники головного офиса с лицензией M365 E3 имеют полный доступ к резервированиям, что означает, что они могут создавать новые календари, редактировать параметры, добавлять сотрудников, публиковать страницы резервирования, создавать встречи в календаре и управлять ими, а также получать отчеты.

Сотрудники магазина с лицензиями E1/F1/F3 или без лицензий могут по-прежнему добавляться в качестве персонала в календари гостевой роли, а затем запланировать их для получения сообщений с подтверждением, когда они будут зарезервированы. Они по-прежнему могут быть зарезервированы в течение их доступных и запланированных часов, как указано на вкладке персонал. Данные о доступности календаря ограничиваются временем резервируемые по заданным часам и сервисным часам.

Сотрудники магазина также будут отображаться как недоступные в книгах, если в это время в веб-приложении "книги" уже запланирована встреча. Встречи, зарезервированные с помощью учета, будут отражаться как занятые в календаре сотрудника в рамках резервирований. Сотрудники с включенной лицензией "нерезервированные" могут по-прежнему влиять на доступность в книгах, если они находятся в одном и том же клиенте.

Пользователи, участвующие в роли "гости", могут просматривать любую информацию, предоставленную клиентом в ходе создания исходной встречи. Например, если сотрудник магазина, предназначенный для встречи, должен позвонить клиенту до встречи, он будет иметь доступ к сведениям, которые предоставляет клиент во время процесса планирования. У ответственного сотрудника будет доступ ко всей информации, которая отображается в сообщении электронной почты с подтверждением, а также о событии календаря. ICS (например, при вводе номера телефона клиента, если он был введен).

Пользователи, применяющие гостевую роль, не смогут получать доступ к веб-приложению "книги", чтобы изменить параметры или просматривать встречи (добавить, отменить и перепланировать). Тем не менее, они могут делать встречи от имени клиентов, использующих страницу самообслуживания, так же, как клиент сделает встречу.

Мы рекомендуем лицензировать бизнес-менеджер или администратора каждого магазина с лицензией с подходящими резервированием для настройки и управления страницами и встречами. Остальная часть персонала будет работать с сотрудником, лицензированным с учетом резервирования, для изменения расписания или отмены резервирования.

## <a name="product-features"></a>Функции продукта

### <a name="where-do-bookings-calendars-show-up-in-my-microsoft-365-tenant"></a>Где отображаются календари учета в моем клиенте Microsoft 365? * *

Каждый новый календарь резервирования создает соответствующий почтовый ящик в Exchange, а также связанную запись в Azure Active Directory (AAD), где запись указана как нелицензированный пользователь.

### <a name="can-i-delete-a-previously-created-bookings-calendar"></a>Можно ли удалить ранее созданный календарь резервирования?

Чтобы удалить календарь резервирования, необходимо удалить связанный почтовый ящик в Exchange.

### <a name="if-i-create-a-calendar-and-someone-else-has-access-to-bookings-would-they-be-able-to-see-my-calendar"></a>Если вы создаете календарь и кто-то другой имеет доступ к резервированиям, смогут ли они видеть свой календарь?

Только те пользователи, которые имеют доступ к календарям, созданным вами (через веб-приложение), — это любой пользователь, добавленный в качестве персонала, в роли администратора или средства просмотра. Администраторы клиентов смогут просматривать список всех почтовых ящиков резервирования в Exchange и AAD.

### <a name="are-video-conferencing-meetings-integrated-into-the-bookings-app"></a>Собраны конференции видеоконференций, интегрированные в приложение для работы с книгами?

Собраний по сети доступны в рамках резервирования при использовании клиентов Skype или Microsoft Teams. Собраний по сети можно включить на уровне службы, а клиент собрания, который вы выбираете (Skype или Teams), будет по умолчанию по умолчанию для зарезервированного сотрудника. Чтобы присоединиться к собраниям по сети, каждая новая встреча будет иметь уникальный канал собрания, присоединенный к встрече, а кроме того, вы можете использовать параметры Easy JOIN в событии и в сообщениях с подтверждением.

### <a name="how-does-scheduling-policy-work"></a>Как работает политика планирования?

Параметр, который называется **Максимальное время ожидания** , определяет крайний срок (измеряется в днях), в течение которого можно выполнить резервирование. **Минимальное** время ожидания не должно превышать 24 часа (минимальное время, в часах, для резервирования и отмены) и максимальное время ожидания. Максимальный период ожидания может быть установлен на один день, если для параметра минимальное время ожидания задано значение 0 часов, что в этом случае означает, что клиент может отменить резервирование до момента запуска, а клиенты могут планировать только те книги, в которых встреча находится на следующий день.

### <a name="how-does-bookings-work-across-time-zones"></a>Как осуществляется резервирование в часовых поясах?

Все значения времени задаются в часовом поясе бизнеса (местный часовой пояс) по умолчанию. Это означает, что в этом часовом поясе будут отображаться все параметры, настроенные для календаря резервирования, например рабочие часы. Страница самообслуживания позволяет отображать все значения времени встреч в часовом поясе конечного пользователя, которые можно отключить при необходимости. Если параметр **всегда показывать интервалы времени в часовом поясе** остается неустановленным на вкладке Страница "книги", то пользователи, посещающие страницу, увидят временные слоты в своих местных часовых поясах.

:::image type="content" source="media/bookings-faq-region.png" alt-text="Параметры области резервирования и часового пояса":::

Не существует готовности указать часовой пояс для сотрудников в резервированиях. Часовой пояс персонала, в то время как рабочие часы будут находиться в часовом поясе предприятия.

Адреса электронной почты контролируются и управляются на уровне параметров Microsoft 365 и зависят от параметров конфигурации домена. Дополнительные сведения можно найти [здесь](https://docs.microsoft.com/powershell/module/exchange/get-accepteddomain).

### <a name="is-the-ability-to-send-sms-messages-dependent-on-a-different-service-or-specific-configuration"></a>Является ли возможность отправки SMS-сообщений зависимыми от другой службы или определенной конфигурации?

SMS в настоящее время доступны в Северной Америке, а для доставки в SMS будет использоваться учетная запись Skype или Твилио.

### <a name="how-can-bookings-appointments-show-up-on-a-persons-calendar"></a>Как можно заархивировать встречи в календаре пользователя?

Подтверждение резервирования отсылается как в ящики входящих сообщений поставщика услуг, так и клиента. В сообщении электронной почты с подтверждением содержится вложенный файл *. ICS, который затем можно добавить в календарь пользователя со всеми релевантными сведениями о встрече.

### <a name="what-triggers-service-provider-or-staff-and-customer-emails"></a>Что инициирует поставщики услуг или сотрудники и сообщения электронной почты клиентов?

Сообщения электронной почты инициируются на основе параметров на вкладке "службы" в веб-приложении "книги учета". При резервировании, сделанное клиентом на странице самостоятельное обслуживание или на вкладке Календарь в веб-приложении "книги", будет инициировано подтверждение и/или напоминание по электронной почте. То же самое произойдет, когда пользователь вносит изменения в резервирование с помощью кнопки " **Управление резервированием** " в сообщении электронной почты с подтверждением или в веб-приложении. Напоминание сообщения электронной почты отправляются в указанный период времени до встречи, как описано на вкладке службы в веб-приложении "книги учета".

### <a name="can-i-book-classroom-style-appointments-that-are-1many-instead-of-11"></a>Можно ли делать встречи со стилем аудиторий от 1: многие, а не 1:1?

Да, у нас есть функции групповых резервирований, которые позволяют нескольким пользователям одновременно зарезервировать одну и ту же встречу (например, для класса "Фитнес"). Эти функции подробно описаны в этой [статье](https://techcommunity.microsoft.com/t5/microsoft-bookings-blog/microsoft-bookings-now-supports-online-meetings-and-group/ba-p/1214120).

### <a name="can-calendars-remain-unpublished-not-public-facing-but-still-accessible-to-designated-users"></a>Могут ли календари оставаться неопубликованными (а не общедоступными), но доступны только назначенным пользователям?

Да. На вкладке "страница резервирования" веб-приложения должен быть установлен флажок " **требуется учетная запись Microsoft 365 или Office 365 из моей организации для книги**". Установка этого флажка запрещает самостоятельный доступ к странице только тем, которые находятся в вашем клиенте. Календарь книги, который располагается в веб-приложении для создания и управления, может быть доступен только тем пользователям, добавляемым в качестве персонала к странице с ролями администратора или средства просмотра.

Последние данные, связанные с домашней страницей, извлекаются при каждом загрузке веб-приложения. Для получения дополнительных сведений о типе информации, отслеживаемой резервированием, ознакомьтесь со [статьей support](metrics-and-activity-tracking.md).

### <a name="can-i-turn-off-the-email-notifications-for-time-off"></a>Можно ли отключить отправку уведомлений по электронной почте?

Если какие-либо сотрудники перепутаны в уведомлении, рекомендуем добавить дополнительные сведения в примечания или название уведомления о недоступности, чтобы лучше проинформировать сотрудников о том, что происходит на стороне администратора.

### <a name="can-bookings-calendars-be-cloned-or-duplicated-and-can-they-be-templatized-for-easy-scaling"></a>Могут ли быть скопированы или дублироваться календари, а также могут ли они быть темплатизед для упрощения масштабирования?

Процесс будет использовать API Graph для получения сведений о почтовом ящике, а затем использовать эти сведения для создания нового почтового ящика. Документация по API [здесь](https://docs.microsoft.com/graph/api/resources/booking-api-overview?view=graph-rest-beta).

### <a name="what-reporting-is-available-in-bookings"></a>Какие отчеты доступны в книгах?

Все сотрудники, назначенные роли администратора, могут скачивать значения, разделенные табуляцией (. TSV-файл со всеми книгами, созданными за прошедшие 120 дней. Отчеты о резервированиях можно скачать в. TSV форма на вкладке Главная на странице "книги". С помощью API резервирования можно также собирать эти данные для более настраиваемых и специальных целей.

### <a name="is-it-possible-to-share-only-one-service-on-the-self-service-page-at-a-time"></a>Можно совместно использовать только одну службу на странице самообслуживания?

Да, вы можете создать отдельные календари учета для каждой службы или перейти на вкладку службы, изменить службу, а в верхней части страницы вы увидите возможность предоставления URL-адреса только для этой конкретной службы.

### <a name="what-options-are-available-for-embedding-a-consent-form-or-disclosure-for-those-booking-an-appointment"></a>Какие параметры доступны для внедрения формы согласия или раскрытия для тех, кому назначена встреча?

Рекомендуем разрешить согласие на **использование данных о клиентах** на странице "резервирование" и настроить его, чтобы правильно передать раскрытие или требования к согласию в Организации. Кроме того, можно добавить настраиваемое поле в службу, которая включает ссылку на форму согласия, и требовать от пользователей подтверждения завершения формы до того, как они смогут продолжить выполнение встречи. Вы также можете добавить URL-адрес формы согласия в качестве дополнительных примечаний в сообщениях с подтверждениями и напоминаний, но это не запрещает пользователям заменять встречу.

### <a name="what-changes-can-be-made-to-the-staff-selection-capability-on-the-self-service-booking-page"></a>Какие изменения могут быть внесены в функцию выбора персонала на странице самостоятельное резервирование?

Возможность клиентов выбирать определенных сотрудников, когда резервирование можно полностью удалить, отменив выбор параметра **Разрешить клиентов выбрать определенного пользователя для параметра резервирование** на странице резервирования или в разделе службы. При резервировании в зависимости от доступности сотрудников автоматически назначается доступный сотрудник в произвольном порядке.

### <a name="can-bookings-support-many-concurrent-booking-requests-and-self-service-page-visits"></a>Могут резервирования поддерживать большое количество одновременных запросов на бронирование и самостоятельных посещений страниц?

Резервирование может поддерживать большое количество посетителей и резервирований за один раз. Если на странице имеется очень большой объем трафика, пользователи получат сообщение "сервер занят". Доступность встречи обновляется при загрузке страницы, а также при выполнении резервирования. Если несколько пользователей пытаются одновременно занести одну и ту же встречу, в книгах будет предоставляться только одна учетная книга, а для других потенциальных клиентов — сообщение с уведомлением о необходимости найти другое время.

## <a name="privacy"></a>Конфиденциальность

### <a name="where-is-bookings-data-stored"></a>Где хранятся данные о резервированиях?

Резервирования это приложение Microsoft 365, то есть все данные хранятся на платформе Microsoft 365 и в Exchange. Резервирования следуют всем политикам хранения данных, установленным корпорацией Майкрософт, которые являются теми же политиками, а затем всеми приложениями Office. При резервировании в Exchange используются общие почтовые ящики для хранения сведений о клиентах, персонале, службе и встрече. Политики соответствия требованиям для общих почтовых ящиков в Exchange также применяются к почтовым ящикам с резервированием. Все данные клиента (включая сведения, предоставленные клиентами при резервировании) фиксируются в пределах приложения, поэтому хранятся в Exchange. В Microsoft резервирования используются те же политики, что и в Microsoft 365, которые можно найти [здесь.](https://www.microsoft.com/online/legal/v2/?docid=22&langid=en-us)

### <a name="is-there-a-way-to-centrally-manage-bookings-calendars-for-all-users"></a>Существует ли способ централизованного управления календарями резервирования для всех пользователей?

Каждый календарь книги поддерживается независимо. Нет консолидированного представления, кроме раскрывающегося списка поиска, после нажатия на домашней странице курсора.

### <a name="how-are-users-authenticated"></a>Как пользователи проходят проверку подлинности?

Доступ к веб-приложению "книги" включает проверку подлинности с помощью обычной проверки подлинности Azure Active Directory (AAD). Страницу самообслуживания резервирования можно сделать доступной всем, используя ссылку на веб-страницу. Однако если выбран параметр **требовать учетную запись microsoft 365 или Office 365 из моей организации** в книгу, эта страница будет ограничена только для использования в клиенте Microsoft 365 (с использованием проверки подлинности AAD).

:::image type="content" source="media/bookings-faq-access-ctrl.png" alt-text="Управление доступом к страницам резервирования при резервировании":::

### <a name="does-customer-data-leave-the-production-system-for-any-reason"></a>Покидают ли клиентские данные производственной системы по какой – либо причине?

Да, поскольку резервирование является частью Exchange, мы разрешают API Graph и API веб-служб Exchange (EWS), чтобы клиенты могли скачать собственные данные, к которым у них есть доступ.

### <a name="is-there-the-ability-to-customize-email-addresses-for-customer-replies"></a>Существует ли возможность настройки адресов электронной почты для ответов клиентов?

Существует возможность определить, кому из адресов электронной почты отправляется ответ. Этот параметр находится на вкладке **Бизнес-информация** . Имя почтового ящика книги также можно изменить в Exchange, используя обычные процессы переименования почтовых ящиков Exchange.

### <a name="we-would-like-for-the-customer-to-accept-our-terms-and-conditions-during-the-booking-process-is-that-possible"></a>Мы хотим, чтобы клиент принимал наши термины и условия во время процесса резервирования. Это возможно?

Эта функция уже существует для обеспечения соответствия требованиям GDPR. Страница "резервирование" имеет поле для настраиваемого согласия, которое находится справа в разделе сведения о публикации. Просто установите флажок и напишите собственное пользовательское сообщение согласия, которое будет отображаться как обязательное поле на странице самостоятельное обслуживание. Дата и время, когда клиент предоставит согласие, будут записаны.

Этот раздел не поддерживает форматированный текст, но ссылки на документ можно добавить в раздел Бизнес-информация и отобразить сразу под полем согласие.

### <a name="how-can-bookings-associated-mailboxes-in-exchange-be-identified"></a>Как определяется способ резервирования: связанные почтовые ящики в Exchange?

Следующую команду можно использовать для получения сведений обо всех почтовых ящиках резервирования и пользователей, имеющих доступ к ним.

`Get-Mailbox -RecipientTypeDetails SchedulingMailbox -ResultSize:Unlimited | Get-MailboxPermission |Select-Object Identity,User,AccessRights | Where-Object {($\_.user -like '\*@\*')}`

## <a name="technical-questions"></a>Технические вопросы

### <a name="if-a-user-schedules-an-appointment-for-a-service-consisting-of-an-online-meeting-how-do-they-receive-the-microsoft-teams-link-to-the-meeting-if-they-leave-the-optional-email-address-field-empty"></a>Если пользователь планирует встречу для службы, состоящей из собрания по сети, как получить ссылку на собрание по Microsoft Teams, если оставить поле адрес электронной почты пустым?

Если в системе нет сведений о сообщении электронной почты клиента, не существует способа доставки почты с подтверждением клиенту, поэтому он не будет получать ссылку Teams. Решение состоит в том, чтобы задать адрес электронной почты в качестве обязательного поля, которое можно сделать на вкладке службы для резервирования. Обратите внимание, что при создании новой службы в книгах поле адрес электронной почты клиента задается как обязательное поле по умолчанию.

### <a name="how-are-staff-assigned-to-appointments-when-anyone-is-selected-or-when-customers-do-not-have-the-ability-to-choose-a-staff-member-for-the-appointment"></a>Как сотрудники назначаются для встреч при выборе пункта "все" или когда у клиентов нет возможности выбрать сотрудника для встречи?

Сотрудники назначаются случайным образом в этих сценариях, из всех сотрудников, доступных на момент зарезервированной встречи. В настоящее время невозможно обеспечить равномерное распределение случайно назначенных встреч во всех участниках сотрудников или назначить количество назначенных встреч каждому сотруднику. При наличии балансировки нагрузки, которую необходимо выполнить для сотрудников после нескольких случайных назначений, ее можно выполнить вручную на вкладке "Календарь" на странице "книги".

### <a name="how-can-users-opt-out-of-receiving-sms-messages-if-they-have-already-opted-in-previously"></a>Как пользователи могут отказаться от принятия сообщений SMS, если они уже были добавлены ранее?

При резервировании встречи с включенными текстовыми сообщениями SMS пользователи получают запрос на проверку почтовых ящиков при каждой книге. Клиенты также могут отвечать непосредственно на SMS-сообщение (поставщик SMS), указывая на то, что сообщения будут прекращены, а затем на этот номер телефона не будут отправляться никакие другие тексты. Если клиент отправит текстовые сообщения от поставщика, а не в самих книгах, они больше не будут получать SMS-сообщения, даже если отказаться от них на странице резервирования.

### <a name="we-cant-find-a-staff-member-in-our-organization"></a>Не удается найти сотрудника в Организации

Если сотрудник оставил вашу компанию, и вы удалили его из вашей организации Microsoft 365 или Office 365, вам все равно необходимо удалить этого пользователя из книг, чтобы они не отображались в календаре и не могли заменять встречи пользователями.

### <a name="theres-a-syncing-error-we-cant-identify"></a>Ошибка синхронизации не удается определить

Это временная ошибка, которая обычно возникает из-за нестабильного подключения к Интернету. Эти проблемы обычно устраняются через несколько минут.

Если вы по-прежнему видите эту ошибку по истечении длительного периода времени или часто видите, обратитесь в службу поддержки.
