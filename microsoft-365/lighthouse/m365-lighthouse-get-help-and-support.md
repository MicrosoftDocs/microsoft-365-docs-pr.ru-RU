# <a name="get-help-and-support-for-microsoft-365-lighthouse"></a>Получите помощь и поддержку для Microsoft 365 Lighthouse 

> [!NOTE]
> Функции, описанные в этой статье, подлежат изменениям и доступны только партнерам, которые соответствуют [требованиям.](m365-lighthouse-requirements.md) Если в организации нет Microsoft 365 Lighthouse, [см.](m365-lighthouse-sign-up.md)в Microsoft 365 Lighthouse.

Если вам нужна помощь, доступно несколько вариантов. Начните с проверки текущего состояния служб клиента-клиента:

1. В левой области навигации Microsoft 365 Lighthouse выберите **службу здоровья**.
2. Просмотр подробных сведений о текущих и прошлых проблемах.

Чтобы проверить текущее состояние служб Microsoft 365 Lighthouse клиентов:

1. Перейдите в Центр администрирования Microsoft 365 <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">https://admin.microsoft.com</a> .
2. В левой области навигации выберите состояние  >  **службы здравоохранения.**
3. В списке служб найдите **Microsoft 365,** а затем расширите его, чтобы показать все службы в наборе.
4. Найдите **Microsoft 365 Lighthouse** и проверьте состояние.

Если вы столкнулись с проблемой, которая не указана ни в одной из панелей мониторинга состояния службы, следуйте инструкциям в этой статье, чтобы просмотреть варианты самостоятельной помощи или создать запрос на службу.

> [!NOTE]
> Поддержка ограничена английским языком, Microsoft 365 Lighthouse в предварительном просмотре.

## <a name="before-you-begin"></a>Прежде чем начать

- Для создания и управления запросами служб необходимо иметь по крайней мере одну роль Azure Active Directory (Azure AD) со следующим набором свойств: **microsoft.office365.supportTickets/allEntities/allTasks**. Список ролей Azure AD см. в списке встроенных ролей [Azure AD.](/azure/active-directory/roles/permissions-reference) Сведения о назначении ролей см. в видеоролике Назначение ролей [Azure AD пользователям.](/azure/active-directory/roles/manage-roles-portal)

- Если требуется создать запрос на службу, соберите как можно больше сведений о проблеме, таких как журналы диагностики, ID клиента и пользовательские ИД (если затронуты конкретные пользователи).

## <a name="access-help-and-support"></a>Справка и поддержка доступа

1.  В Microsoft 365 Lighthouse выберите **?** значок в верхней части портала, чтобы открыть области **справки,** а затем сделать один из следующих:
    
    -  Если вы на странице портала, где возникла проблема, выберите **показать диагностику.**

        Это создаст JSON-файл с информацией, которая поможет агентам поддержки устранить проблему. Сохраните файл, чтобы можно было прикрепить его к запросу службы.

        > [!NOTE]
        > Файл JSON будет содержать персональные идентифицируемые сведения.

    -  Если проблема не изолирована на текущей странице портала, перейдите к следующему шагу.

2.  В области **Справка** выберите кнопку **Справка + поддержка.** Это открывает **окно How can we help?**

    > [!NOTE]
    > Если не откроется окантовка How **can we help?** необходимо обратиться к кому-либо из клиента-партнера, у которого есть разрешения глобального администратора, и попросить их помочь.

3.  В области **How can we help?** введите описание проблемы и нажмите **кнопку Ввод**. Мы рекомендуем включить полное имя продукта Microsoft 365 *маяка* в описание, чтобы обеспечить, чтобы результаты поиска включали соответствующие статьи справки.

4.  Ознакомьтесь со списком рекомендуемых статей, чтобы узнать, помогает ли любая из них решить проблему.

    Если вы вводите описание проблемы и она не возвращает список статей справки, перефразируя свое описание и попробуйте другой поиск.

5.  Если рекомендуемые статьи не помогают, выберите **службу поддержки контактов.**

6.  Заполните сведения в форме, прикрепите все скриншоты и файл JSON, сохраненный на шаге 1, если это применимо, а затем выберите &nbsp; **Свяжитесь со мной**. Ожидаемое время ожидания указывается в области.
