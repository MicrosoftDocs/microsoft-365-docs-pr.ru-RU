# <a name="create-organization-wide-signatures-and-disclaimers"></a>Создание подписей и отказов для всей организации

 Вы можете управлять подписями электронной почты, добавляя подпись электронной почты, юридическое заявление об отказе или заявление о раскрытии в сообщениях электронной почты, которые входят в вашу организацию или покидают ее. Эта подпись может применяться ко всем входящим и исходящим сообщениям (см. инструкции ниже) или выборочно, например лишь к тем сообщениям, которые содержат определенные слова или текстовые шаблоны.

 Посмотрите короткое видео о создании подписи электронной почты для всей компании. <br><br>
  
> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE1IEWf] 

Если это видео помогло вам, ознакомьтесь с [полным учебным курсом для малых предприятий и новых пользователей Microsoft 365](https://support.microsoft.com/office/6ab4bbcd-79cf-4000-a0bd-d42ce4d12816).

## <a name="create-a-signature-that-applies-to-all-messages"></a>Создание подписи,применяемой ко всем сообщениям

> [!TIP]
> Подписи на всей организации называются "заявлением об отказе" независимо от того, что они содержатся. Например, они могут быть просто подписью, а также включать ваш адрес, юридическое заявление об отказе или другую информацию, которую вы хотите получить.
    
::: moniker range="o365-worldwide"

Перейдите в Центр администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">https://admin.microsoft.com</a>.

::: moniker-end

::: moniker range="o365-germany"

Перейдите в Центр администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=848041" target="_blank">https://portal.office.de/adminportal</a>.

::: moniker-end

::: moniker range="o365-21vianet"

Перейдите в Центр администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=850627" target="_blank">https://portal.partner.microsoftonline.cn/adminportal</a>.

::: moniker-end

1. Выберите запуск приложения ![ значок запуска приложений, а затем выберите ](../../media/7502f4ec-3c9a-435d-a7b4-b9cda85189a7.png) **"Администратор".**
   
    Не можете найти приложение, которое ищете? В средстве запуска приложений выберите **все приложения,** чтобы увидеть список доступных приложений в алфавитном порядке. Оттуда вы можете искать определенное приложение. 
    
2. Выберите **центры администрирования,** а затем выберите **Exchange.**
    
3. В области потока почты выберите **"Правила".**
    
4. Выберите **+** значок "Добавить" и выберите **"Применить заявления об отказе".**
    
5. Задайте имя правила.
    
6. В **области "Применить это правило"** выберите **[Применить для всех сообщений]**.
    
    > [!TIP]
    > [Узнайте больше](https://docs.microsoft.com/Exchange/policy-and-compliance/mail-flow-rules/signatures#Scoping) о том, как настроить условия, чтобы заявление об отказе применялось не ко всем сообщениям. (Эта статья о области для Exchange Server, но также относится к Microsoft 365.) 
  
7. В списке "Выполнить следующие действия..." оставьте выбранным пункт **Добавить в сообщение заявление об отказе**. 
    
8.  Выберите **"Введите текст"** и введите заявление об отказе. 
    
    > [!TIP]
    > [Узнайте больше](https://docs.microsoft.com/Exchange/policy-and-compliance/mail-flow-rules/signatures#FormatDisclaimer) о форматировании заявлений об отказе. (Эта статья о форматирования Exchange Server, но также относится к Microsoft 365.) 

9. Выберите **"Выбрать один"** и выберите **"Перенос"** в качестве варианта отката. Затем нажмите кнопку **ОК**. Это означает, что если заявление об отказе невозможно добавить из-за шифрования или других настроек почты, оно переносится в конверт сообщения.
    
10. Оставьте установленным флажок **Проверять выполнение этого правила со степенью серьезности**. Затем выберите степень серьезности **Низкая**, **Средняя** или **Высокая**, которая будет использоваться в журнале сообщений. 
    
11. Установите переключатель **Принудительно**, чтобы включить это правило немедленно, если его не нужно предварительно протестировать. 
    
12. Щелкните ссылку **Дополнительные параметры**, чтобы добавить дополнительные условия или исключения. 
    
13. Когда все будет готово, нажмите кнопку **Сохранить**. 
    
## <a name="limitations-of-organization-wide-signatures"></a>Ограничения подписей для всей организации

При управлении подписями электронной почты в Microsoft 365 нельзя сделать следующее:
  
- Вставка подписи непосредственно под последним ответом электронной почты или переад.
    
- Отображение подписей электронной почты на стороне сервера в папках "Отправленные" пользователей
    
- Встраив изображения в подписи электронной почты
    
- Пропускать строки, содержащие переменные, которые не удалось обновить (например, из-за того, что значение не было предоставлено пользователю)
    
Чтобы получить эти и другие возможности для управления подписями электронной почты, воспользуйтесь сторонним средством. Please do an internet search for **email signature software**. Некоторые из этих поставщиков являются партнерами Microsoft Gold, и их программное обеспечение предоставляет эти возможности.
