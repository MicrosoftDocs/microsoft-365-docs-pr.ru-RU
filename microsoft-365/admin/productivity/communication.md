# <a name="communication--people-experiences"></a>Связь — взаимодействие с пользователями

Оценка продуктивности помогает организациям преобразовывать принципы работы с аналитическими сведениями о том, как люди используют Microsoft 365 и технологический технологический интерфейс, поддерживающий их. Показатель отражает эффективность вашей организации от использования сотрудников и технологий, а также сравнивает свою оценку с организациями, такими как ваш. Данные для общения являются частью показателей людей с опытом. Чтобы узнать больше, ознакомьтесь со статьей [Обзор оценки производительности](productivity-score.md) .

## <a name="why-your-organizations-communication-score-matters"></a>Почему показатель связи в Организации имеет значение 

Корпорация Майкрософт понимает, что люди имеют разные требования к общению. Чтобы быстро получить ответ на вопрос, вы можете отправить мгновенное сообщение. Если вы хотите отправлять обновления состояния руководителю, вы можете выбрать электронное сообщение. Чтобы получить доступ к более широкой аудитории, вы можете отправить сообщение сообщества. Microsoft 365 обеспечивает эту гибкость в режимах взаимодействия для удовлетворения потребностей всех пользователей. В справочных материалах, использующих средства связи в режиме реального времени, создается более Единая Организация и выполняется построение сотрудников, независимо от расположения. [Посмотрите свидетельство](https://go.microsoft.com/fwlink/?linkid=2127669).

Мы измеряем, как часто люди используют сообщения электронной почты, чата и сообщества, чтобы узнать, как пользователи в вашей организации могут находить и совместно использовать информацию.

## <a name="how-we-calculate-the-score"></a>Как вычисляется Оценка

Для общения мы предоставляем основное представление, которое содержит ключевые метрики, связанные с обменом данными в Организации, в сочетании с моделью оценки для вычисления оценки.

### <a name="primary-insight"></a>Основное представление

Microsoft 365 предоставляет пользователям гибкие возможности для соответствия стилю общения для всех пользователей, предлагая несколько режимов. Основной обзор предоставляет меру людей в Организации, которые используют несколько режимов для общения.

:::image type="content" source="../../media/primaryinsight.png" alt-text="Основное представление режимов связи.":::

1. **Верхний колонтитул:** Предоставляет ключевую метрику людей, использующих более одного режима связи, а именно
      - Отправка электронных сообщений через Exchange
      - Отправка сообщений через Teams
      - Публикация в Yammer

    Эта ключевая метрика отображается в процентах от всех пользователей, использующих два или более таких продуктов: Exchange, Teams и Yammer. 

2. **Основной текст:** В этой статье приведены дополнительные сведения о том, как гибкость в режимах взаимодействия полезна для людей в вашей организации.
3. **Визуализация текущего состояния:**
    - Горизонтальная полоса, где синяя часть представляет процент, выраженный в заголовке
    - Выделяется (числитель/знаменатель), используемый для вычисления процентного соотношения, выраженного в заголовке:
      - **Числитель**: количество пользователей, использующих более одного режима связи за последние 28 дней
      - **Знаменатель**: количество людей, помеченных как включенные для нескольких коммуникационных продуктов за последние 28 дней
    - **Оценочное значение однорангового** показателя для ключевой метрики также отображается в процентах

1. **Просмотр ресурсов взаимодействия:** Щелкнув здесь, можно прикрепить все содержимое поддержки в виде видеороликов и статей, относящихся к общению. С помощью этих средств вы можете перейти к настраиваемому списку воспроизведения для общения, а затем для всех категорий оценки производительности.

#### <a name="trend-visualization-of-primary-insight"></a>Визуализация тенденций основной информации

 На этой диаграмме приведена тенденция числителя ключевой метрики в первичной аналитике, то есть количество людей, использующих несколько режимов связи за последние 180 дней. Здесь значение ежедневно — это совокупность количества людей, использующих несколько форм связи за последние 28 дней.

:::image type="content" source="../../media/trend-communication.png" alt-text="Диаграмма, на которой показано количество людей, использующих более одного режима для общения и времени.":::

### <a name="scoring-model"></a>Модель оценки

Показатель связи определяет, будут ли пользователи постоянно обмениваться данными с несколькими режимами в сообщениях электронной почты, чатов и сообщества в течение 28 дней.

## <a name="explore-how-people-in-your-organization-communicate"></a>Узнайте, как взаимодействуют сотрудники Организации

В этом разделе представлены сведения о действиях на уровне пользователя, относящиеся ко всем аналитикам на странице. Столбцы в таблице описываются следующим образом:

- **Username**  это адрес электронной почты пользователя.
- **Отображаемое имя**  — это полное имя пользователя.
- **Дата последнего действия**  в последний раз, когда пользователь выполнил действия, относящиеся к какому бы то ни было аналитике в рамках взаимодействия за последние 28 дней.
- **Отдел:** Отдел, определенный для пользователя в Azure Active Directory (Azure AD).
- **Организация:** Компания, определенная для пользователя в Azure AD.
- **Страна:** Страна, определенная для пользователя в Azure AD.
- **Состояние:** Состояние, определенное для пользователя в Azure AD.
- **City:** Город, определенный для пользователя в Azure AD.
- **Exchange:** Имеет значение true, если для пользователя включена поддержка Exchange за последние 28 дней, в противном случае — значение false.
- **Teams:** True, если пользователь включен для Teams за последние 28 дней, в противном случае false.
- **Yammer:** Имеет значение true, если пользователю разрешено использовать Yammer в течение последних 28 дней, в противном случае false.
- **Отправленные дни электронной почты:** Количество дней, в течение которых пользователь отправил электронную почту за последние 28 дней.
- Дни, в которых **используется чат teams:** Количество дней, в течение которых пользователь отправил сообщение чата в Microsoft Teams за последние 28 дней.
- **Дни, используемые каналами teams:** Количество дней, в течение которых пользователь отправляет сообщение канала в Microsoft Teams за последние 28 дней.
- **Дни, опубликованные в Yammer:** Количество дней, в течение которых пользователь публикуется в Yammer за последние 28 дней.
- **Дни, используемые @mention в электронной почте:** Количество дней, в течение которых пользователь отправил электронное сообщение, содержащее @mention за последние 28 дней.
