# <a name="microsoft-365-usage-analytics"></a>Аналитика использования Microsoft 365

## <a name="overview-of-microsoft-365-usage-analytics"></a>Обзор аналитики использования Microsoft 365

Используйте аналитику использования Microsoft 365 в Power BI, чтобы получить представление о том, как ваша организация использует различные службы в Microsoft 365. Вы можете визуализировать и анализировать данные об использовании Microsoft 365, создавать настраиваемые отчеты и обмениваться данными в организации. Вы также можете получить представление о том, как определенные регионы или отделы используют Microsoft 365.
  
Аналитика использования Microsoft 365 предоставляет доступ к встроенной панели мониторинга, которая предоставляет представление нескольких продуктов за последние 12 месяцев и содержит ряд встроенных отчетов. Каждый отчет предоставляет конкретные сведения об использовании. Сведения о пользователях доступны за последний полный календарный месяц.
  
Модель [данных, которая](usage-analytics-data-model.md) включает в себя шаблон приложения включает атрибуты пользователя из Active Directory, что позволяет свести в определенных отчетах. Включены следующие атрибуты Active Directory: расположение, отдел и организация. 
  
Чтобы начать сбор данных см. статью [Включение аналитики использования Microsoft 365](enable-usage-analytics.md). 
  
Аналитика использования Microsoft 365 содержит ряд отчетов, подробных в следующих разделах. 

Вы можете получить доступ к подробным отчетам для каждой области, выбрав таблицы данных. Вы можете просмотреть все встроенные отчеты, выбрав вкладки в нижней части сайта. Для получения более подробных инструкций ознакомьтесь [с навигацией](navigate-and-utilize-reports.md) и использованием отчетов и [настройкой отчетов.](customize-reports.md)

## <a name="executive-summary"></a>Сводка по исполнительным руководствам

Сводка руководства — это высокоуровневый обзор внедрения, использования, мобильности, общения, совместной работы и хранения в Microsoft 365 для бизнеса, который предназначен для лиц, принимающих бизнес-решения. В нем рассматривается, как используются некоторые отдельные службы, в зависимости от всех включенных и активных пользователей. Все значения месяца, показанного в отчете, относятся к последнему полному месяцу. 

Эта сводка позволяет быстро понять шаблоны использования в Office, а также то, как и где ваши сотрудники совместно работают.

## <a name="overview"></a>Обзор

Обзорный отчет Microsoft 365 содержит следующие отчеты. Их можно просмотреть, выбрав вкладку в верхней части страницы отчета. Все значения месяца, показанные в верхней части отчета, относятся к последнему полному месяцу.

- **Внедрение** &ndash; Обзор тенденций внедрения. Используйте отчеты в этом разделе, чтобы узнать, как пользователи использовали Microsoft 365, а также как изменилось общее использование отдельных служб за месяц. Вы можете узнать, как могут быть включены пользователи, сколько людей в вашей организации активно используют Microsoft 365, сколько возвращает пользователей и сколько пользователей используют продукт в первый раз.

- **Использование** &ndash; Предлагает более простое представление об объеме активных пользователей и основных действиях для каждого продукта за последние 12 месяцев. Используйте отчеты в этом разделе, чтобы узнать, как сотрудники вашей организации используют Microsoft 365.

- **Связь** &ndash; Вы можете быстро узнать, предпочитают ли люди в вашей организации оставаться на связи с помощью звонков Teams, Yammer, электронной почты или Skype. Вы можете наблюдать, есть ли изменения в шаблонах в использовании средств связи среди сотрудников. 

- **Совместная работа** &ndash; Узнайте, как люди в вашей организации используют OneDrive и SharePoint для хранения документов и совместной работы друг с другом, а также о том, как эти тенденции меняются ежемесячно. Вы также можете узнать, сколько документов совместно используется внутри или извне, а также сколько сайтов SharePoint или учетных записей OneDrive активно используется владельцами и другими соавторами.

- **Хранилище** &ndash; Этот отчет используется для отслеживания облачного хранилища для почтовых ящиков, сайтов OneDrive и SharePoint.

- **Мобильность** &ndash; Отслеживание клиентов и устройств, которые пользователи используют для подключения к электронной почте, Teams, Skype или Yammer.

## <a name="activation-and-licensing"></a>Активация и лицензирование

На странице активации и лицензии можно получить отчеты об активации Microsoft 365; то есть, сколько пользователей скачали и активировали приложения Office, а также сколько лицензий было назначено вашей организацией. Значение месяца к верхнему относится к текущему месяцу, а метрики отражают значения, агрегированные с начала месяца до текущей даты.

- **Активация** &ndash; Отслеживание планов обслуживания (например, приложений Microsoft 365 для предприятий, Project и Visio) в организации. Каждый человек с лицензией Office может устанавливать продукты на пяти устройствах. Вы также можете использовать отчеты в этом разделе, чтобы увидеть устройства, на которых люди установили приложения Office. Обратите внимание, что для активации плана пользователь должен установить приложение и войти с помощью своей учетной записи.

- **Лицензирование** &ndash; В этом отчете содержится обзор типов лицензий, количество пользователей, которым были назначены лицензии каждого типа, а также распределение назначений лицензий за каждый месяц. Значение месяца к верхнему относится к текущему месяцу, а метрики отражают значения, агрегированные с начала месяца до текущей даты.

## <a name="product-usage"></a>Использование продукта

Этот отчет содержит отдельный отчет для каждой службы Microsoft 365, включая Exchange, группы Microsoft 365, OneDrive, SharePoint, Skype, Teams и Yammer. Каждый отчет содержит общее число включенных и активных пользовательских отчетов, количество сущностей, таких как почтовые ящики, сайты, группы и учетные записи, а также отчеты о типах активности, если это необходимо. Все значения месяца, показанные в верхней части отчета, относятся к последнему полному месяцу.

## <a name="user-activity"></a>Действия пользователей

Отчеты об активности пользователей доступны для определенных отдельных служб. Эти отчеты предоставляют данные об использовании на уровне пользователя, которые присоединяется к атрибутам Active Directory. Кроме того, отчет о внедрении в отдел позволяет резать по атрибутам Active Directory, чтобы можно было видеть активных пользователей во всех отдельных службах. Все метрики объединяются за последний полный месяц.

## <a name="faq"></a>Вопросы и ответы

### <a name="is-this-template-app-going-to-be-available-through-purchase-or-will-it-be-free"></a>Будет ли приложение этого шаблона доступным через покупку или будет бесплатным?

Это не бесплатно, вам потребуется лицензия Power BI Pro. Дополнительные сведения [см. в](https://docs.microsoft.com/power-bi/service-template-apps-install-distribute#prerequisites) предварительных условия для установки, настройки и распространения приложения шаблона.

Чтобы поделиться панелями мониторинга с другими, см. дополнительные разделы на [панелях мониторинга и в отчетах.](https://docs.microsoft.com/power-bi/service-how-to-collaborate-distribute-dashboards-reports#share-dashboards-and-reports)

### <a name="who-can-connect-to-microsoft-365-usage-analytics"></a>Кто может подключаться к аналитике использования Microsoft 365?

Чтобы установить подключение к приложению шаблона, необходимо быть глобальным  администратором,  администратором **Exchange,** администратором **Skype** для бизнеса, администратором **SharePoint,** глобальным читателем или читателем отчетов. Дополнительные [сведения см. в](../add-users/about-admin-roles.md) сведениях о ролях администраторов.

### <a name="who-can-customize-the-usage-analytics-reports"></a>Кто может настраивать отчеты анализа использования?

Настраивать отчеты или создавать новые отчеты в веб-интерфейсе Power BI может только пользователь, задавший начальное подключение к приложению шаблона. Инструкции см. в настройках отчетов в аналитике использования [Microsoft 365.](customize-reports.md)

### <a name="can-i-only-customize-the-reports-from-the-power-bi-web-interface"></a>Можно ли настраивать отчеты только из веб-интерфейса Power BI?

Помимо настройки отчетов из веб-интерфейса Power BI, пользователи также могут использовать Power BI Desktop для прямого подключения к службе отчетов Microsoft 365 для создания собственных отчетов.

### <a name="how-can-i-get-the-pbit-file-that-this-dashboard-is-associated-with"></a>Как получить PBIT-файл, с которым связана эта панель мониторинга?

Доступ к PBIT-файлу можно получить из [Центра загрузки Майкрософт.](https://download.microsoft.com/download/7/8/2/782ba8a7-8d89-4958-a315-dab04c3b620c/Microsoft%20365%20Usage%20Analytics.pbit)

### <a name="who-can-view-the-dashboards-and-reports"></a>Кто может просматривать панели мониторинга и отчеты?

Если вы подключились к приложению шаблона, вы можете поделиться им с кем угодно с помощью [функции общего доступа.](https://go.microsoft.com/fwlink/p/?linkid=845494) Для лицензирования Power BI необходимо, чтобы как общий доступ пользователей, так и пользователь, с которым совместное использование панели мониторинга, были power BI Pro или Power BI Premium.

### <a name="can-anyone-share-the-dashboard-or-does-it-have-to-be-the-person-who-connected-to-the-dashboard"></a>Может ли любой пользователь предоставить общий доступ к панели мониторинга, или это должен выполнять пользователь, подключившийся к панели мониторинга?

При совместном использовании панели мониторинга можно разрешить пользователям либо повторно делиться панелью с другими пользователями, либо нет. Этот параметр можно установить во время общего доступа.

### <a name="is-it-possible-to-work-on-and-customize-the-same-template-app-with-a-group-of-people"></a>Можно ли работать с одним и тем же приложением шаблона и настраивать его с группой людей?

Да. Чтобы позволить группе администраторов работать вместе с одним и тем же шаблонным приложением, вы можете использовать функциональные возможности рабочей области приложения Power BI. Дополнительные сведения см. в сведениях о том, как совместно работать и делиться панелями мониторинга и [отчетами?](https://go.microsoft.com/fwlink/p/?linkid=851070) 

### <a name="for-which-timeframe-is-data-available"></a>Для какого временного интервала доступны данные?

В большинстве отчетов отображаются данные за предыдущие 12 месяцев. Однако на некоторых диаграммах может быть меньше истории, так как сбор данных для разных продуктов и отчетов был запущен в разное время, поэтому данные за полные 12 месяцев могут быть недоступны. Все отчеты в конечном итоге будут создаваться до 12 месяцев истории. Отчеты с подробными сведениями об уровне пользователя показывают данные за предыдущий полный месяц.

### <a name="what-data-is-included-in-the-template-app"></a>Какие данные включены в приложение шаблона?

Данные в приложении шаблона в настоящее время охватывают тот же набор метрик активности, который доступен в отчетах [об активности.](../activity-reports/activity-reports.md) При добавлении отчетов в отчеты об активности они будут добавлены в приложение шаблона в будущем выпуске.

### <a name="how-does-the-data-in-the-template-app-differ-from-the-data-in-the-usage-reports"></a>Чем данные в приложении шаблона отличаются от данных в отчетах об использовании?

Данные, которые вы видите в приложении шаблона, совпадают с данными, которые вы видите в отчетах об активности в Центре администрирования Microsoft 365. Основные различия в том, что данные Центра администрирования доступны в течение последних 30.07.90/180 дней, в то время как приложение шаблона ежемесячно представляет данные на срок до 12 месяцев.

Кроме того, сведения об уровне пользователя в приложении шаблона доступны только за последний месяц для пользователей, которым была назначена лицензия на продукт и выполнено действие.

### <a name="when-should-i-use-the-template-app-and-when-the-usage-reports"></a>Когда следует использовать приложение шаблона и когда отчеты об использовании?

Отчеты [об активности —](../activity-reports/activity-reports.md)  это хорошая отправная точка для понимания использования и внедрения Microsoft 365. Приложение шаблона объединяет данные об использовании Microsoft 365 и данные Active Directory вашей организации и позволяет администраторам анализировать набор данных с помощью возможностей визуальной аналитики Power BI. Это позволяет администраторам не только визуализировать и анализировать данные об использовании Microsoft 365, но и резать их по свойствам Active Directory, таким как отделы, расположение и т. д. Они также могут создавать настраиваемые отчеты и обмениваться данными в своей организации. 

### <a name="how-often-is-the-data-refreshed"></a>Как часто обновляются данные? 

При первом подключении к приложению шаблона данные автоматически заполняются за предыдущие 12 месяцев. После этого данные приложения шаблона обновляются еженедельно. Клиенты могут изменить расписание обновления, если их использование этих данных требует другого времени обновления.

Тыловая служба Microsoft 365 ежедневно обновляет данные и предоставляет данные от 5 до 8 дней с текущей даты.

Столбец **даты содержимого** в каждом наборе данных представляет дату свежести данных в приложении шаблона.

### <a name="how-is-an-active-user-defined"></a>Как определяется активный пользователь?

Определение активного пользователя такое же, как и определение активного пользователя [в](../activity-reports/active-users.md) отчетах об активности.

### <a name="what-sharepoint-site-collections-are-included-in-the-sharepoint-reports"></a>Какие семейства веб-сайтов SharePoint включены в отчеты SharePoint?

Текущая версия приложения шаблона включает действия с файлами с сайтов групп SharePoint и сайтов групп SharePoint.

### <a name="which-groups-are-included-in-the-microsoft-365-groups-usage-report"></a>Какие группы включены в отчет об использовании групп Microsoft 365?

Текущая версия приложения шаблона включает использование из групп Outlook, групп Yammer и групп SharePoint. Он не включает группы, связанные с Microsoft Teams или Планировщиком.

### <a name="when-will-an-updated-version-of-the-template-app-become-available"></a>Когда станет доступна обновленная версия приложения шаблона?

Основные изменения в приложении шаблона будут выпущены дважды в год, которые могут включать новые отчеты или новые данные. Незначительные изменения в отчетах могут быть выпущены чаще.

### <a name="is-it-possible-to-integrate-the-data-from-the-template-app-into-existing-solutions"></a>Можно ли интегрировать данные из приложения шаблона в существующие решения? 

Данные в приложении шаблона можно получить с помощью API Microsoft 365 (в предварительной версии). При отгрузки в производство они будут объединены в API [отчетов Microsoft Graph.](https://go.microsoft.com/fwlink/p/?linkid=848843) 

### <a name="are-there-plans-to-expand-the-template-app-to-show-usage-data-from-other-microsoft-products"></a>Планируется ли развернуть приложение шаблона для показа данных об использовании из других продуктов Майкрософт?

Это рассматривается для будущих улучшений. Ознакомьтесь [с планом обновления Microsoft 365.](https://www.microsoft.com/microsoft-365/roadmap)

### <a name="how-can-i-pivot-by-company-information-in-active-directory"></a>Как можно свести сведения о компании в Active Directory?

Сведения о компании включены в одно из полей Active Directory в приложении шаблона, и вы можете увидеть его как встроенный фильтр в отчетах о действиях пользователей продукта.  Он доступен в качестве столбца в **таблице UserState.**

### <a name="is-it-possible-to-bring-in-additional-fields-from-active-directory"></a>Можно ли перенести дополнительные поля из Active Directory?

Дополнительная настройка этих данных возможна путем подключения к API отчетов [Microsoft Graph,](https://go.microsoft.com/fwlink/p/?linkid=848843) чтобы получить дополнительные поля из Azure Active Directory и присоединиться к набору данных. 

### <a name="is-it-possible-to-aggregate-the-information-in-the-template-app-across-multiple-subscriptions"></a>Можно ли агрегировать сведения в приложении шаблона для нескольких подписок?

В настоящее время приложение шаблона используется для одной подписки, так как оно связано с учетными данными, которые использовались для первоначального подключения к ней.

### <a name="is-it-possible-to-see-usage-by-plan-ie-e1-e3"></a>Можно ли увидеть использование по плану (например, E1, E3)?

В приложении шаблона использование представлено на уровне продукта. Предоставляются данные о различных подписках, которые назначены пользователям, однако невозможно соотнести действия пользователей с подпиской, назначенной пользователю.

### <a name="is-it-possible-to-integrate-other-data-sets-into-the-template-app"></a>Можно ли интегрировать другие наборы данных в приложение шаблона?

Power BI Desktop можно использовать для подключения к API Microsoft 365 (в предварительной версии) для объединения дополнительных источников данных с данными приложения шаблона.

Дополнительные сведения [см. в документе "Настройка".](customize-reports.md)

### <a name="is-it-possible-to-see-the-top-users-reports-for-a-specific-timeframe"></a>Можно ли увидеть отчеты "Лучшие пользователи" в течение определенного времени?
