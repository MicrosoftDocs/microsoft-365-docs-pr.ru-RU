# <a name="microsoft-365-usage-analytics"></a>Аналитика использования Microsoft 365

Используйте Microsoft 365 аналитику использования в Power BI, чтобы получить представление о том, как ваша организация принимает различные службы в Microsoft 365. Вы можете визуализировать и анализировать Microsoft 365, создавать настраиваемые отчеты и обмениваться сведениями в организации. Вы также можете получить сведения о том, как конкретные регионы или отделы используют Microsoft 365.
  
Microsoft 365 аналитики использования предоставляет доступ к предварительно построенной панели мониторинга, которая предоставляет представление по всем продуктам за последние 12 месяцев и содержит ряд предварительно построенных отчетов. Каждый отчет предоставляет конкретные сведения об использовании. Сведения о пользователях доступны для последнего полного календарного месяца.
  
Модель [данных, которая](usage-analytics-data-model.md) включает в себя атрибуты пользователя из Active Directory, позволяет делать поворот в определенных отчетах. Включены следующие атрибуты Active Directory: расположение, отдел и организация. 
  
Чтобы начать сбор данных см. статью [Включение аналитики использования Microsoft 365](enable-usage-analytics.md). 
  
Microsoft 365 использования содержит ряд отчетов, подробных в следующих разделах. 

Вы можете получить доступ к подробным отчетам для каждой области, выбрав таблицы данных. Вы можете просмотреть все предварительно построенные отчеты, выбрав вкладки в нижней части сайта. Дополнительные инструкции по [](navigate-and-utilize-reports.md) навигации и использованию отчетов и настройке [отчетов.](customize-reports.md)

## <a name="executive-summary"></a>Сводка executive

Сводка на высоком уровне— это представление отчетов о Microsoft 365 для бизнеса, использования, мобильности, связи, совместной работы и хранения данных и предназначена для лиц, принимающих решения для бизнеса. Он предоставляет представление о том, как используются отдельные службы на основе всех пользователей, которые были включены, и тех, кто активен. Все значения месяца, показанного в отчете, относятся к последнему полному месяцу. 

Это сводка позволяет быстро понять шаблоны использования в Office, а также о том, как и где работают ваши сотрудники.

## <a name="overview"></a>Обзор

Отчет Microsoft 365 содержит следующие отчеты. Их можно просмотреть, выбрав вкладку в верхней части страницы отчета. Все значения месяца, показанные в верхней части отчета, относятся к последнему полному месяцу.

- **Принятие** &ndash; Предлагает сводку тенденций принятия. Используйте отчеты в этом разделе, чтобы узнать, как пользователи приняли Microsoft 365, а также как изменилось общее использование отдельных служб месяц за месяцем. Вы можете увидеть, как могут быть включены пользователи, сколько людей в вашей организации активно используют Microsoft 365, сколько возвращает пользователей и сколько используют продукт впервые.

- **Использование** &ndash; Предлагает обзор объема активных пользователей и основных действий для каждого продукта за последние 12 месяцев. Используйте отчеты в этом разделе, чтобы узнать, как люди в вашей организации используют Microsoft 365.

- **Связь** &ndash; Сразу видно, предпочитают ли люди в организации оставаться на связи с помощью Teams, Yammer, электронной почты или Skype вызовов. Вы можете наблюдать, есть ли сдвиги в шаблонах в использовании средств связи среди сотрудников. 

- **Совместная работа** &ndash; Узнайте, как люди в вашей организации используют OneDrive и SharePoint для хранения документов и совместной работы друг с другом, а также того, как эти тенденции развиваются месяц за месяцем. Вы также можете увидеть, сколько документов используется для внутреннего или внешнего использования, а также сколько SharePoint сайтов или OneDrive активно используются владельцами и другими сотрудниками.

- **служба хранилища** &ndash; Этот отчет используется для отслеживания облачного хранилища для почтовых ящиков, OneDrive и SharePoint сайтов.

- **Мобильность** &ndash; Отслеживайте, какие клиенты и устройства используются для подключения к электронной почте, Teams, Skype или Yammer.

## <a name="activation-and-licensing"></a>Активация и лицензирование

Страница активации и лицензии предоставляет отчеты о Microsoft 365 активации; то есть, сколько пользователей скачали и активировали Office и сколько лицензий было назначено вашей организацией. Значение месяца к началу относится к текущему месяцу, а показатели отражают значения, совокупные с начала месяца до текущей даты.

- **Активация** &ndash; Отслеживание активаций службы (например, Приложения Microsoft 365 для предприятий, Project и Visio) в организации. Каждый человек с Office лицензией может устанавливать продукты на пять устройств. Вы также можете использовать отчеты в этом разделе, чтобы увидеть устройства, на которых установлены Office приложения. Обратите внимание, что для активации плана пользователю необходимо установить приложение и войти со своей учетной записью.

- **Лицензирование** &ndash; В этом отчете содержится обзор типов лицензий, количество пользователей, которым назначен каждый тип лицензии, и распределение назначений лицензий за каждый месяц. Значение месяца к началу относится к текущему месяцу, а показатели отражают значения, совокупные с начала месяца до текущей даты.

## <a name="product-usage"></a>Использование продукта

В этом отчете содержится отдельный отчет для каждой Microsoft 365 службы, включая Exchange, Microsoft 365 группы, OneDrive, SharePoint, Skype, Teams и Yammer. Каждый отчет содержит общие включенные и общие отчеты активных пользователей, количество сущностей, таких как почтовые ящики, сайты, группы и учетные записи, а также отчеты о типах действий, если это необходимо. Все значения месяца, показанные в верхней части отчета, относятся к последнему полному месяцу.

## <a name="user-activity"></a>Действия пользователей

Отчеты о действиях пользователей доступны для определенных отдельных служб. Эти отчеты предоставляют данные об использовании подробных данных на уровне пользователя, а также атрибуты Active Directory. Кроме того, отчет об усыновлении отдела позволяет нарезать атрибуты Active Directory, чтобы можно было видеть активных пользователей во всех отдельных службах. Все показатели агрегируются за последний полный месяц.

Да. Чтобы позволить группе администраторов работать вместе в одном приложении шаблона, вы можете использовать функциональные возможности рабочего пространства приложения Power BI, дополнительные сведения см. в разделЕ Как совместно работать и обмениваться панелями мониторинга и [отчетами?](/power-bi/collaborate-share/service-how-to-collaborate-distribute-dashboards-reports) 

### <a name="for-which-timeframe-is-data-available"></a>Для какого временного интервала доступны данные?

В большинстве отчетов отображаются данные за предыдущие 12 месяцев. Однако некоторые диаграммы могут показывать меньше истории, так как сбор данных для различных продуктов и отчетов был запущен в разное время, поэтому данные за 12 месяцев могут быть недоступны. Все отчеты в конечном итоге будут создаваться до 12 месяцев истории. Отчеты, в которые покажут сведения об уровне пользователя, показывают данные за предыдущий полный месяц.

### <a name="what-data-is-included-in-the-template-app"></a>Какие данные включены в приложение шаблона?

Данные в приложении шаблона в настоящее время охватывают тот же набор показателей активности, которые доступны в [отчетах о деятельности.](../activity-reports/activity-reports.md) По мере того как отчеты добавляются в отчеты о действиях, они будут добавлены в приложение шаблона в будущем выпуске.

### <a name="how-does-the-data-in-the-template-app-differ-from-the-data-in-the-usage-reports"></a>Чем данные в приложении шаблона отличаются от данных в отчетах об использовании?

Данные, которые вы видите в приложении шаблона, совпадают с данными, которые вы видите в отчетах о действиях в Центр администрирования Microsoft 365. Основные различия в том, что в центре администрирования данные доступны в течение последних 7/30/90/180 дней, а приложение шаблона ежемесячно представляет данные на срок до 12 месяцев.

Кроме того, сведения об уровне пользователя в приложении шаблона доступны только за последний полный месяц для пользователей, которым была назначена лицензия на продукт и выполнено действие.

### <a name="when-should-i-use-the-template-app-and-when-the-usage-reports"></a>Когда следует использовать приложение шаблона и когда отчеты об использовании?

Отчеты [об активности](../activity-reports/activity-reports.md) являются хорошей отправной точкой для понимания использования и принятия Microsoft 365. Приложение шаблона Microsoft 365 данные об использовании и сведения о Active Directory организации и позволяет администраторам анализировать набор данных с помощью возможностей визуальной аналитики Power BI. Это позволяет администраторам не только визуализировать и анализировать Microsoft 365, но и нарезать их свойствами Active Directory, такими как отделы, расположение и т.д. Они также могут создавать настраиваемые отчеты и обмениваться сведениями в своей организации. 

### <a name="how-often-is-the-data-refreshed"></a>Как часто обновляются данные? 

При первом подключении к приложению шаблона оно автоматически заполняется данными за предыдущие 12 месяцев. После этого данные приложения шаблона будут обновляться еженедельно. Клиенты могут изменить расписание обновления, если их использование этих данных требует другого ритма обновления.

Служба back-end Microsoft 365 ежедневно обновляет данные и предоставляет данные, которые от 5 до 8 дней от текущей даты.

Столбец **дата контента** в каждом наборе данных представляет дату свежести данных в приложении шаблона.

### <a name="how-is-an-active-user-defined"></a>Как определяется активный пользователь?

Определение активного пользователя такое же, как и определение активного пользователя [в](../activity-reports/active-users.md) отчетах о действиях.

### <a name="what-sharepoint-site-collections-are-included-in-the-sharepoint-reports"></a>Какие семейства веб-сайтов SharePoint включены в отчеты SharePoint?

Текущая версия приложения-шаблона включает действия файлов с сайтов SharePoint групп и SharePoint групп.

### <a name="which-groups-are-included-in-the-microsoft-365-groups-usage-report"></a>Какие группы включены в отчет об использовании Microsoft 365 групп?

Текущая версия приложения-шаблона включает использование Outlook групп, Yammer групп и SharePoint групп. Он не включает группы, связанные с Microsoft Teams или планировщиком.

### <a name="when-will-an-updated-version-of-the-template-app-become-available"></a>Когда будет доступна обновленная версия приложения шаблона?

Основные изменения в приложении шаблона будут выпущены два раза в год, которые могут включать новые отчеты или новые данные. Незначительные изменения в отчетах могут быть выпущены более часто.

### <a name="is-it-possible-to-integrate-the-data-from-the-template-app-into-existing-solutions"></a>Возможно ли интегрировать данные из приложения-шаблона в существующие решения? 

Данные в приложении шаблона можно получить с помощью API Microsoft 365 (в предварительном просмотре). При отгрузки в производство они будут объединены в [API Graph отчетов](https://go.microsoft.com/fwlink/p/?linkid=848843)Microsoft. 

### <a name="are-there-plans-to-expand-the-template-app-to-show-usage-data-from-other-microsoft-products"></a>Планируется ли расширение приложения-шаблона для демонстрации данных об использовании из других продуктов Майкрософт?

Это рассматривается для будущих улучшений. Проверьте [дорожную карту Microsoft 365 обновления.](https://www.microsoft.com/microsoft-365/roadmap)

### <a name="how-can-i-pivot-by-company-information-in-active-directory"></a>Как можно свести сведения о компании в Active Directory?

Сведения о компании включены в одно из полей Active Directory в приложении шаблона, и в отчетах о действиях пользователя продукта его можно увидеть в качестве предварительно **построенного** фильтра. Он доступен в качестве столбца в **таблице UserState.**

### <a name="is-it-possible-to-bring-in-additional-fields-from-active-directory"></a>Можно ли перенести дополнительные поля из Active Directory?

Дополнительная настройка этих данных возможна путем подключения к [API](https://go.microsoft.com/fwlink/p/?linkid=848843) Graph отчетов Майкрософт, чтобы вытащить дополнительные поля из Azure Active Directory и присоединиться к набору данных. 

### <a name="is-it-possible-to-aggregate-the-information-in-the-template-app-across-multiple-subscriptions"></a>Можно ли агрегировать сведения в приложении шаблона по нескольким подпискам?

В настоящее время приложение шаблона используется для одной подписки, так как оно связано с учетными данными, которые использовались для первоначального подключения к ней.

### <a name="is-it-possible-to-see-usage-by-plan-ie-e1-e3"></a>Можно ли увидеть использование по плану (например, E1, E3)?

В приложении шаблона использование представлено на уровне каждого продукта. Предоставляются данные о различных подписях, которые назначены пользователям, однако невозможно соотнести активность пользователей с подпиской, назначенной пользователю.

### <a name="is-it-possible-to-integrate-other-data-sets-into-the-template-app"></a>Возможно ли интегрировать другие наборы данных в приложение шаблона?

Вы можете использовать Power BI Desktop для подключения к API Microsoft 365 (в предварительном просмотре) для привлечения дополнительных источников данных для объединения с данными приложения шаблона.

Дополнительные сведения см. в [документе Customize.](customize-reports.md)

### <a name="is-it-possible-to-see-the-top-users-reports-for-a-specific-timeframe"></a>Можно ли видеть отчеты "Топ-пользователи" для определенного времени?

Во всех отчетах уровня пользователей представлены агрегированные данные за предыдущий месяц.

### <a name="will-the-template-app-be-localized"></a>Будет ли локализовано приложение шаблона? 

В настоящее время это не в дорожной карте.

### <a name="i-have-a-specific-question-about-the-data-im-seeing-for-my-organization-who-can-i-reach-out-to"></a>У меня есть конкретный вопрос о данных, отображаемых для моей организации. К кому я могу обратиться?

Вы можете использовать кнопку обратной связи на странице обзор действий центра администрирования или открыть случае поддержки, чтобы получить помощь в приложении шаблона. [](../../business-video/get-help-support.md) 

### <a name="how-can-partners-access-the-data"></a>Как партнеры получают доступ к данным?

Если партнер делегировал права администратора, он может подключиться к приложению шаблона от имени клиента.

### <a name="can-i-hide-identifiable-information-such-as-user-group-and-site-names-in-reports"></a>Можно ли в отчетах скрыть идентификационные данные, такие как имена пользователей, групп и сайтов?

Да, [см. в этой ленте Сделать собранные данные анонимными.](enable-usage-analytics.md#make-the-collected-data-anonymous)
