# <a name="microsoft-365-usage-analytics"></a>Аналитика использования Microsoft 365
---
Служба анализа сведений об использовании Microsoft 365 также доступна для сообщества 365 Майкрософт для государственных учреждений США.
 
## <a name="overview-of-microsoft-365-usage-analytics"></a>Обзор аналитики использования Microsoft 365

Используйте аналитику использования Microsoft 365 в Power BI, чтобы получить подробные сведения о том, как ваша организация использует различные службы в Microsoft 365 для общения и сотрудничества. Вы можете визуализировать и анализировать данные об использовании Microsoft 365, создавать настраиваемые отчеты и предоставлять общий доступ к аналитическим сведениям в вашей организации, а также получать сведения о том, как определенные регионы или отделы используют Microsoft 365.
  
Аналитика использования Microsoft 365 — это приложение-шаблон, которое предоставляет доступ к предварительно созданной панели мониторинга, предоставляющей представление в виде нескольких продуктов за последние 12 месяцев и содержащее ряд готовых отчетов. Каждый отчет предоставляет конкретные сведения об использовании. Сведения о конкретных пользователях доступны за последний полный календарный месяц.
  
[Модель данных](usage-analytics-data-model.md) , которая включает в себя шаблон приложения, включает атрибуты пользователей из Active Directory, что позволяет выполнять сведение в определенные отчеты. Включены следующие атрибуты Active Directory: расположение, отдел и организация. 
  
Чтобы начать сбор данных см. статью [Включение аналитики использования Microsoft 365](enable-usage-analytics.md). 
  
Аналитика использования Microsoft 365 содержит ряд отчетов, подробно описанных в следующих разделах. 

Вы можете получить доступ к подробным отчетам по каждой области, выбрав таблицы данных. Просмотреть все готовые отчеты можно, выбрав вкладки в нижней части сайта, просмотрев отчеты. Для получения более подробных инструкций прочитайте [о переходах и использовании отчетов в аналитике использования microsoft 365](navigate-and-utilize-reports.md) и [настройке отчетов в аналитике использования Microsoft 365](customize-reports.md).

## <a name="executive-summary"></a>Сводка для руководителей

Краткий обзор — это высокоуровневый обзор Microsoft 365 для внедрения, использования, мобильности, обмена данными, совместной работы и хранения данных, а также отчетов об использовании, а также для ответственных за принятие бизнес-решений. В этой статье представлено представление, в котором используются некоторые отдельные службы на основе всех пользователей, которые были включены и активные. Все значения месяца, показанные в отчете, относятся к последнему полному месяцу. 

Эта сводка позволяет быстро освоиться со схемами использования в Office, а также как и как сотрудники работают вместе.

## <a name="overview"></a>Обзор

Отчет Microsoft 365 Overview содержит следующие отчеты. Вы можете просмотреть их, выбрав вкладку в верхней части страницы отчета. Все значения месяца, показанные в верхнем разделе отчета, относятся к последнему полному месяцу.

- **Принятие** &ndash; предоставляет сводную информацию о тенденциях внедрения. С помощью отчетов из этого раздела вы узнаете, как ваши пользователи используют Microsoft 365, а также о том, как общее использование отдельных служб было изменено за месяц. Вы можете узнать, как могут быть включены пользователи, сколько людей в организации активно используют Microsoft 365, сколько пользователей будут возвращаться и сколько используется в первый раз.

- **Использование** &ndash; предоставляет детальное представление объема активных пользователей и основные действия для каждого продукта за последние 12 месяцев. Используйте отчеты в этом разделе, чтобы узнать, как пользователи в организации используют Microsoft 365.

- **Связь** &ndash; вы можете видеть, как понять, как сотрудники организации предпочитают поддерживать связь с помощью команд, Yammer, электронной почты или вызовов Skype. Вы можете наблюдать за сдвигом в шаблонах при использовании средств обмена данными между сотрудниками. 

- **Совместная работа** &ndash; Узнайте, как пользователи в вашей организации используют OneDrive и SharePoint для хранения документов и совместной работы друг с другом, а также того, как эти тенденции развиваются за месяц. Вы также можете узнать, сколько документов совместно используется внутренне или внешне, и сколько сайтов SharePoint или учетных записей OneDrive активно используется, с разбивкой на владельцев и других участников совместной работы.

- **Хранилище** &ndash; . Этот отчет используется для отслеживания облачного хранилища для почтовых ящиков, OneDrive и сайтов SharePoint.

- **Мобильное** &ndash; устройство, с помощью которого пользователи и устройства могут подключаться к электронной почте, Teams, Skype или Yammer.

## <a name="activation-and-licensing"></a>Активация и лицензирование

На странице Активация и лицензия предоставляются отчеты об активации Microsoft 365; то есть сколько пользователей загрузили и активировали приложения Office и сколько лицензий было назначено вашей организацией. Значение месяца в начале соответствует текущему месяцу, а метрики — значения, собранные с начала месяца до текущей даты.

- План обслуживания отслеживания **активации** &ndash; (например, активации Microsoft 365 профессиональный плюс, Project и Visio) в Организации. Каждый пользователь, имеющий лицензию на Office, может устанавливать продукты на пяти устройствах. Кроме того, вы можете использовать отчеты в этом разделе для просмотра устройств, на которых установлены приложения Office. Обратите внимание, что чтобы активировать план, пользователь должен установить приложение и войти с его учетной записью.

- **Лицензирование** &ndash; этот отчет содержит общие сведения о типах лицензий, количестве пользователей, которым назначены все типы лицензий, а также о распределении назначений лицензий за каждый месяц. Значение месяца в начале соответствует текущему месяцу, а метрики — значения, собранные с начала месяца до текущей даты.

## <a name="product-usage"></a>Использование продуктов

Этот отчет содержит отдельный отчет для каждой службы Microsoft 365, включая Exchange, Microsoft 365, OneDrive, SharePoint, Skype, Teams и Yammer. Каждый отчет содержит общее число включенных и общих активных отчетов о пользователях, количество сущностей, таких как почтовые ящики, сайты, группы и учетные записи, а также отчеты о типах действий там, где это уместно. Все значения месяца, показанные в верхнем разделе отчета, относятся к последнему полному месяцу.

## <a name="user-activity"></a>Действия пользователей

Отчеты об активности пользователей доступны для определенных отдельных служб. Эти отчеты предоставляют сведения об использовании данных на уровне пользователя, связанные с атрибутами Active Directory. Кроме того, отчет об внедрении отдела позволяет разделить атрибуты Active Directory, чтобы видеть активных пользователей для всех отдельных служб. Все показатели объединены в течение последнего полного месяца.

## <a name="faq"></a>Вопросы и ответы

### <a name="is-this-template-app-going-to-be-available-through-purchase-or-will-it-be-free"></a>Будет ли это приложение-шаблон доступно через покупку или будет бесплатным?

Это бесплатно, вам потребуется лицензия Power BI Pro. Дополнительные сведения см. в статье [Предварительные требования](https://docs.microsoft.com/power-bi/service-template-apps-install-distribute#prerequisites) для установки, настройки и распространения приложения шаблона.

Чтобы поделиться панелями мониторинга с другими пользователями, ознакомьтесь с дополнительными сведениями в разделе [Share Dashboards and Reports](https://docs.microsoft.com/power-bi/service-how-to-collaborate-distribute-dashboards-reports#share-dashboards-and-reports).

### <a name="who-can-connect-to-microsoft-365-usage-analytics"></a>Кто может подключаться к аналитике использования Microsoft 365?

Для установки подключения к приложению-шаблону необходимо быть **глобальным**администратором, администратором **Exchange**, администратором **Skype для бизнеса**, администратором **SharePoint**, **глобальным устройством чтения** или **читателями отчетов** . Более подробную информацию можно узнать в статье [сведения о ролях администратора](../add-users/about-admin-roles.md) .

### <a name="who-can-customize-the-usage-analytics-reports"></a>Кто может настраивать отчеты аналитики использования?

Только пользователь, который может выполнить начальное подключение к приложению шаблона, может настраивать отчеты или создавать новые отчеты в веб-интерфейсе Power BI. Инструкции по [настройке отчетов в аналитике использования Microsoft 365](customize-reports.md) .

### <a name="can-i-only-customize-the-reports-from-the-power-bi-web-interface"></a>Можно ли настраивать отчеты только в веб-интерфейсе Power BI?

Помимо настройки отчетов в веб-интерфейсе Power BI, пользователи могут также использовать Power BI Desktop для прямого подключения к службе отчетов Microsoft 365 для создания собственных отчетов.

### <a name="how-can-i-get-the-pbit-file-that-this-dashboard-is-associated-with"></a>Как получить PBIT-файл, с которым связана эта панель мониторинга?

Вы можете получить доступ к файлу pbit-из [центра загрузки Майкрософт](https://download.microsoft.com/download/7/8/2/782ba8a7-8d89-4958-a315-dab04c3b620c/Microsoft%20365%20Usage%20Analytics.pbit).

### <a name="who-can-view-the-dashboards-and-reports"></a>Кто может просматривать панели мониторинга и отчеты?

Если вы подключены к приложению шаблона, вы можете предоставить к нему доступ всем пользователям с помощью [функций общего доступа](https://go.microsoft.com/fwlink/p/?linkid=845494). Для лицензирования Power BI необходимо, чтобы общий доступ пользователей и пользователь, к которому предоставлена панель мониторинга, включил Power BI Pro или Power BI Premium.

### <a name="can-anyone-share-the-dashboard-or-does-it-have-to-be-the-person-who-connected-to-the-dashboard"></a>Может ли любой пользователь предоставить общий доступ к панели мониторинга, или это должен выполнять пользователь, подключившийся к панели мониторинга?

При совместном использовании панели мониторинга вы можете разрешить пользователям повторно предоставлять общий доступ к панели мониторинга другим пользователям. Этот параметр можно установить на время совместного использования.

### <a name="is-it-possible-to-work-on-and-customize-the-same-template-app-with-a-group-of-people"></a>Можно работать над одним и тем же приложением шаблона с группой людей?

Да. Чтобы группа администраторов работала совместно с одним приложением-шаблоном, вы можете использовать функциональные возможности рабочей области приложения Power BI, чтобы получить дополнительные сведения о [том, как совместно работать с панелями мониторинга и отчеты?](https://go.microsoft.com/fwlink/p/?linkid=851070) 

### <a name="for-which-timeframe-is-data-available"></a>Для какого временного интервала доступны данные?

В большинстве отчетов отображаются данные за последние 12 месяцев. Однако некоторые диаграммы могут показывать меньше журналов, так как сбор данных для разных продуктов и отчетов был начат в разное время, и поэтому данные в течение 12 месяцев могут быть недоступны. В итоге все отчеты будут создаваться до 12 месяцев истории. Отчеты, отображающие сведения о пользовательском уровне, содержат данные за предыдущий полный месяц.

### <a name="what-data-is-included-in-the-template-app"></a>Какие данные включены в приложение-шаблон?

Данные в приложении шаблона в настоящее время охватывают один и тот же набор метрик действий, доступных в [отчетах об активности](../activity-reports/activity-reports.md). По мере добавления отчетов в отчеты об активности они будут добавлены в шаблон приложения в будущем выпуске.

### <a name="how-does-the-data-in-the-template-app-differ-from-the-data-in-the-usage-reports"></a>Чем данные в приложении шаблона отличаются от данных в отчетах об использовании?

Базовые данные, которые вы видите в шаблоне приложения, совпадают с данными, которые вы видите в отчетах об активности в центре администрирования Microsoft 365. Ключевые отличия в данных центра администрирования доступны в течение последних 7/30/90/180 дней, в то время как приложение-шаблон представляет данные ежемесячно в течение 12 месяцев.

Кроме того, сведения о пользовательском уровне в приложении шаблона доступны только для последнего полного месяца для пользователей, которым назначена лицензия на продукт и которые выполнили действие.

### <a name="when-should-i-use-the-template-app-and-when-the-usage-reports"></a>Когда следует использовать приложение шаблона и когда отчеты об использовании?

[Отчеты об активности](../activity-reports/activity-reports.md) являются хорошей отправной точкой для изучения использования и внедрения Microsoft 365. Приложение-шаблон объединяет данные об использовании Microsoft 365 и сведения о Active Directory вашей организации, а также позволяет администраторам анализировать набор данных с помощью возможностей Visual Analytics в Power BI. Это позволяет администраторам не только визуализировать и анализировать данные об использовании Microsoft 365, но также разделять их свойствами Active Directory, такими как отделы, расположение и т. д. Кроме того, они могут создавать настраиваемые отчеты и предоставлять общий доступ к аналитическим сведениям в Организации. 

### <a name="how-often-is-the-data-refreshed"></a>Как часто обновляются данные? 

При первом подключении к приложению шаблона оно автоматически заполняется данными за последние 12 месяцев. После этого данные приложения шаблона будут обновляться еженедельно. Пользователи могут изменить расписание обновления, если эти данные запрашивает другое обновление ритм.

Внутренняя служба Microsoft 365 будет обновлять данные ежедневно и предоставляет данные, которые находятся в диапазоне от 5-8 дней до текущей даты.

Столбец " **Дата контента** " в каждом наборе данных представляет дату актуальности данных в приложении шаблона.

### <a name="how-is-an-active-user-defined"></a>Как определяется активный пользователь?

Определение активного пользователя это то же, что и определение [активного пользователя](../activity-reports/active-users.md) в отчетах о действиях.

### <a name="what-sharepoint-site-collections-are-included-in-the-sharepoint-reports"></a>Какие семейства веб-сайтов SharePoint включены в отчеты SharePoint?

Текущая версия приложения шаблона включает действия с файлами на сайтах групп SharePoint и на сайтах групп SharePoint.

### <a name="which-groups-are-included-in-the-microsoft-365-groups-usage-report"></a>Какие группы включены в отчет об использовании Microsoft 365 Groups?

Текущая версия приложения шаблона включает использование из групп Outlook, групп Yammer и групп SharePoint. Он не включает группы, связанные с Microsoft Teams или планировщика.

### <a name="when-will-an-updated-version-of-the-template-app-become-available"></a>Когда станет доступна обновленная версия приложения шаблона?

Значительные изменения в приложении шаблона будут вычислены дважды в год, в том числе новые отчеты или новые данные. Незначительные изменения в отчетах могут быть выпущены чаще.

### <a name="is-it-possible-to-integrate-the-data-from-the-template-app-into-existing-solutions"></a>Можно интегрировать данные из приложения-шаблона в существующие решения? 

Данные в приложении-шаблоне можно получить с помощью API Microsoft 365 (в предварительной версии). При отправке в производственную среду они будут объединены в [API отчетов Microsoft Graph](https://go.microsoft.com/fwlink/p/?linkid=848843). 

### <a name="are-there-plans-to-expand-the-template-app-to-show-usage-data-from-other-microsoft-products"></a>Планируете ли развернуть приложение шаблона для отображения данных об использовании от других продуктов Майкрософт?

Это рассматривается для будущих усовершенствований. Обратитесь к [плану обновления Microsoft 365](https://www.microsoft.com/microsoft-365/roadmap) .

### <a name="how-can-i-pivot-by-company-information-in-active-directory"></a>Как можно свести сведения о компании в Active Directory?

Информация о компании включает один из полей Active Directory в приложении-шаблоне и отображается как готовый фильтр в отчетах об **активности пользователей продукта** . Он доступен в виде столбца в таблице **userState** .

### <a name="is-it-possible-to-bring-in-additional-fields-from-active-directory"></a>Можно ли перенести дополнительные поля из Active Directory?

Дополнительные настройки этих данных можно получить, подключившись к [API отчетов Microsoft Graph](https://go.microsoft.com/fwlink/p/?linkid=848843) , чтобы получить дополнительные поля из Azure Active Directory и присоединиться к набору данных. 

### <a name="is-it-possible-to-aggregate-the-information-in-the-template-app-across-multiple-subscriptions"></a>Можно объединять информацию в приложении-шаблоне по нескольким подпискам?

В настоящее время приложение шаблона предназначено для одной подписки, так как она связана с учетными данными, которые использовались для первоначального подключения к ней.

### <a name="is-it-possible-to-see-usage-by-plan-ie-e1-e3"></a>Можно просматривать использование по плану (то есть E1, E3)?

В приложении-шаблоне использование представлено на уровне продукта. Предоставляются данные о различных подписках, назначенных пользователям, но невозможно сопоставить действия пользователя с подпиской, назначенной пользователю.

### <a name="is-it-possible-to-integrate-other-data-sets-into-the-template-app"></a>Можно интегрировать другие наборы данных в приложение-шаблон?

Вы можете использовать Power BI Desktop для подключения к API Microsoft 365 (в предварительной версии), чтобы обеспечить объединение дополнительных источников данных с данными приложения-шаблона.

Более подробную информацию вы найдете в [документе настройка](customize-reports.md).

### <a name="is-it-possible-to-see-the-top-users-reports-for-a-specific-timeframe"></a>Можно просматривать отчеты "самые популярные пользователи" за определенный период?

Все отчеты на уровне пользователей предоставляют агрегированные данные за предыдущий месяц.

### <a name="will-the-template-app-be-localized"></a>Будет ли локализовано приложение шаблона? 

В настоящее время он не включен в схему.

### <a name="i-have-a-specific-question-about-the-data-im-seeing-for-my-organization-who-can-i-reach-out-to"></a>У меня есть определенный вопрос о данных, которые я вижу для моей организации. К кому можно получить доступ?

Вы можете использовать кнопку "Обратная связь" на странице обзора действий центра администрирования или можно открыть [обращение в службу поддержки](../contact-support-for-business-products.md) для получения справки по шаблону приложения. 

### <a name="how-can-partners-access-the-data"></a>Как партнеры получают доступ к данным?

Если партнер имеет делегированные права администратора, он может подключиться к приложению-шаблону от имени клиента.

### <a name="can-i-hide-identifiable-information-such-as-user-group-and-site-names-in-reports"></a>Можно ли в отчетах скрыть идентификационные данные, такие как имена пользователей, групп и сайтов?

Да, [в статье сделать собранные данные анонимными](enable-usage-analytics.md#make-the-collected-data-anonymous).
