# <a name="set-up-basic-mobility-and-security"></a>Настройка Basic Mobility + Security

Встроенные функции Базовой мобильности и безопасности для Microsoft 365 помогают защитить мобильные устройства пользователей, такие как iPhone, iPad, Android и телефоны с Windows, а также управлять ими. Вы можете создавать политики безопасности устройств и управлять ими, удаленно очищать устройства и просматривать подробные отчеты об устройствах.

Есть вопросы? Вопросы и ответы, которые помогут вам решить распространенные вопросы, см. в часто задаваемом вопросе о базовой мобильности и [безопасности.](frequently-asked-questions.md) Следует помнить, что для управления basic Mobility and Security нельзя использовать делегированную учетную запись администратора. Дополнительные сведения см. в [сведениях о партнерах: предложение делегирования администрирования.](https://support.microsoft.com/office/partners-offer-delegated-administration-26530dc0-ebba-415b-86b1-b55bc06b073e) 

Управление устройствами является частью Центра безопасности & соответствия требованиям, поэтому вам потребуется перейти туда, чтобы приперейти к настройке Basic Mobility and Security.

## <a name="activate-the-basic-mobility-and-security-service"></a>Активация базовой службы мобильности и безопасности

1. Во sign in to Microsoft 365 with your global admin account.

2. Перейдите к [функции активации базовой мобильности и безопасности.](https://admin.microsoft.com/EAdmin/Device/IntuneInventory.aspx)

   Активация Basic Mobility and Security может занять некоторое время. По его завершению вы получите сообщение электронной почты с объяснением следующих действий.

## <a name="set-up-mobile-device-management"></a>Настройка управления мобильными устройствами

Когда служба будет готова, выполните следующие действия, чтобы завершить настройку.

### <a name="step-1-required-configure-domains-for-basic-mobility-and-security"></a>Шаг 1. (Обязательно) Настройка доменов для базовой мобильности и безопасности

Если у вас нет пользовательского домена, связанного с Microsoft 365, или если вы не управляете устройствами с Windows, этот раздел можно пропустить. В противном случае вам потребуется добавить записи DNS для домена на своем хост-компьютере DNS. Если вы уже добавили записи, при настройке домена с помощью Microsoft 365 все настроено. После добавления записей пользователи Microsoft 365 в вашей организации, которые вошел на свое устройство с Windows с помощью адреса электронной почты, который использует ваш домен, перенаправляются для регистрации в Basic Mobility and Security.

Нужна помощь в настройке записей? Найдите регистратора доменных имен и выберите имя регистратора, чтобы перейти к пошаговой справке по созданию записи DNS в списке, предоставленном в списке "Добавление записей DNS для подключения [домена".](https://docs.microsoft.com/office365/admin/get-help-with-domains/create-dns-records-at-any-dns-hosting-provider) Используйте эти инструкции для создания записей CNAME, описанных в процедуре упрощения регистрации [Windows без Azure AD Premium.](https://docs.microsoft.com/mem/intune/enrollment/windows-enroll#simplify-windows-enrollment-without-azure-ad-premium)

После добавления двух записей CNAME перейдите в Центр безопасности & соответствия требованиям и перейдите к управлению устройствами защиты от потери данных, чтобы завершить  >     следующий шаг.

### <a name="step-2-required-configure-an-apns-certificate-for-ios-devices"></a>Шаг 2. (Обязательно) Настройка сертификата APNs для устройств с iOS

Для управления устройствами iOS, например iPad и iPhone, необходимо создать сертификат APNs.

1. Во sign in to Microsoft 365 with your global admin account.

2. В браузере:  [https://protection.office.com](https://protection.office.com/) .

3. Выберите  **управление устройствами** защиты от потери   >  **данных** и выберите **сертификат APNs для устройств с iOS.**

4. На странице "Параметры сертификата push-уведомлений Apple" нажмите кнопку **"Далее".**

5. Выберите **"Скачать CSR-файл"** и сохраните запрос подписи сертификата в нужное место на вашем   компьютере. Выберите **"Далее".**

6. На странице "Создание сертификата APNs":

   - Выберите портал Apple APNS, чтобы открыть портал push-сертификатов Apple.
   - Sign in with an Apple ID.

     > [!IMPORTANT]
     > Use a company Apple ID associated with an email account that will remain with your organization even if the user who manages the account leaves. Save this ID because you'll need to use the same ID when it's time to renew the certificate.

   - Выберите "Создать сертификат" и примите условия использования.
   - Перейдите к запросу подписи сертификата, загруженного на компьютер из Microsoft 365, и выберите "SelectUpload".
   - Download the APN certificate created by the Apple Push Certificate Portal to your computer.

     > [!TIP]
     > If you're having trouble downloading the certificate, refresh your browser.

7. Вернуться в Microsoft 365 и выбрать **"Далее".**

8.  Browse to the APN certificate you downloaded from the Apple Push Certificates Portal.

9. Select  **Finish**.

### <a name="step-3-recommended-set-up-multi-factor-authentication"></a>Шаг 3. Настройка многофакторной проверки подлинности (рекомендуется)

MFA помогает защитить вход в Microsoft 365 для регистрации мобильных устройств, требуя второй формы проверки подлинности. Пользователи должны подтвердить телефонный звонок, sms-сообщение или уведомление приложения на мобильном устройстве после правильного ввода пароля своей учетной записи работы. Они могут зарегистрировать свое устройство только после завершения второй проверки подлинности. После регистрации устройств пользователей в basic Mobility and Security пользователи могут получать доступ к ресурсам Microsoft 365 только с помощью своей рабочей учетной записи.

Чтобы узнать, как включить многофакторную проверку подлинности на портале Azure AD, см. "Настройка многофакторной [проверки подлинности".](https://go.microsoft.com/fwlink/p/?LinkId=519255)

После того как вы настроили MFA, перейдите в **** Центр безопасности & соответствия требованиям и перейдите к политикам управления устройствами для защиты от потери данных, чтобы выполнить   >     >  ****   следующий шаг.

### <a name="step-4-recommended-manage-device-security-policies"></a>Шаг 4. (Рекомендуется) Управление политиками безопасности устройств

Следующим шагом является создание и развертывание политик безопасности устройств для защиты данных организации Microsoft 365. Например, вы можете предотвратить потерю данных, если пользователь потеряет свое устройство, создав политику блокировки устройств через пять минут бездействия и стирая устройства после трех сбоев при входе.

1. Во sign in to Microsoft 365 with your global admin account.

2. Выберите ["Активировать управление мобильными устройствами".](https://admin.microsoft.com/EAdmin/Device/IntuneInventory.aspx) Если служба активирована, вместо шагов активации вы увидите ссылку на управление [устройствами.](https://admin.microsoft.com/adminportal/home#/MifoDevices)  

3. Перейдите **к политикам устройств.**

   :::image type="content" source="../../media/basic-mobility-security/bms-4-policy.png" alt-text="Основные параметры политики безопасности и мобильности":::

4. Создайте и разверните политики безопасности устройств, подходящие для вашей организации, следуя шагам в области создания политик безопасности устройств [в Basic Mobility and Security.](create-device-security-policies.md)

> [!TIP]
>
> - При создании новой политики может потребоваться разрешить доступ к политике и сообщить о нарушении политики, если устройство пользователя не соответствует политике. Это позволяет узнать, сколько мобильных устройств влияет на политику, не блокируя доступ к Microsoft 365.
>
> - Перед развертыванием новой политики для всех пользователей в организации рекомендуется протестировать ее на устройствах, используемых небольшим количеством пользователей.
>
> - Кроме того, перед развертыванием политик дайте своей организации знать о потенциальных последствиях регистрации устройства в Basic Mobility and Security. В зависимости от того, как вы настроили политики, доступ к Microsoft 365 может быть заблокирован для устройств, которые не соответствуют политикам (устройства, не соответствующие требованиям). На устройствах, которые не соответствуют требованиям, также могут быть установлены приложения, фотографии и другие персональные данные, которые на зарегистрированных устройствах могут быть удалены при стираемом устройстве. Дополнительные сведения см. в [теме "Стирка мобильного устройства"](wipe-mobile-device.md)в basic Mobility and Security.

## <a name="make-sure-users-enroll-their-devices"></a>Убедитесь, что пользователи регистрют свои устройства

После создания и развертывания политики управления мобильными устройствами каждый лицензированный пользователь Microsoft 365 в вашей организации, к который применяется политика устройств, получает сообщение о регистрации при следующем входе в Microsoft 365 с мобильного устройства. Прежде чем получить доступ к электронной почте и документам Microsoft 365, им необходимо выполнить действия по регистрации и активации. Дополнительные сведения см. в записи [мобильного устройства с помощью Basic Mobility and Security.](enroll-your-mobile-device.md)

> [!IMPORTANT]
> Если предпочтительный язык пользователя не поддерживается процессом регистрации, пользователи могут получать уведомления о регистрации и действия на своих мобильных устройствах на другом языке. В настоящее время не все языки, поддерживаемые в Microsoft 365, поддерживаются в процессе регистрации на мобильных устройствах.

Пользователи с устройствами с Android или iOS должны установить приложение "Портал компании" в процессе регистрации.
