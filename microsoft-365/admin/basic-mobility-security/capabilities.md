# <a name="capabilities-of-basic-mobility-and-security"></a>Возможности Basic Mobility + Security

Basic Mobility and Security позволяет обеспечить безопасность и управление мобильными устройствами, например iPhone, iPad, Android и Windows Phones, используемыми лицензированными пользователями Microsoft 365 в вашей организации. Вы можете создавать политики управления мобильными устройствами с помощью параметров, которые помогут управлять доступом к электронной почте и документам Microsoft 365 вашей организации для поддерживаемых мобильных устройств и приложений. Если устройство потеряли или украли, можно удаленно очистить устройство, чтобы уничтожить конфиденциальную корпоративную информацию.

## <a name="supported-devices"></a>Поддерживаемые устройства

Вы можете использовать Basic Mobility и Security для защиты и управления следующими устройствами.

- версии iOS 11.0 или более поздних версий

- Android 5.0 или более поздние версии<sup>3</sup>

- Windows 8.1<sup>1</sup>

- Windows 8.1 RT<sup>1</sup>

- Windows 10<sup>2</sup>

- Windows 10 Mobile<sup>2</sup>

<sup>1</sup> Управление доступом для устройств Windows 8.1 RT ограничено Exchange ActiveSync.

<sup>2</sup> Управление доступом для устройств Windows 8.1 RT ограничено Exchange ActiveSync.
Для управления доступом для Windows 10 требуется подписка, которая включает Azure AD Premium, и устройство должно быть присоединено к Azure Active Directory.

<sup>3</sup> Управление доступом для устройств Windows 8.1 RT ограничено Exchange ActiveSync.
После июня 2020 г. версии Android позднее 9 не могут управлять настройками паролей, за исключением устройств Samsung Knox.

>[!NOTE]
>Устройства, уже зарегистрированные с более ранними версиями ОС, продолжают функционировать, хотя возможности могут изменяться без уведомления.

Если люди в организации используют мобильные устройства, не поддерживаемые базовой мобильностью и безопасностью, вам может потребоваться заблокировать доступ Exchange ActiveSync к электронной почте Microsoft 365 для этих устройств, чтобы сделать данные организации более безопасными. Действия по блокировке Exchange ActiveSync см. в руб. Управление настройками доступа к устройствам [в Basic Mobility and Security.](manage-device-access-settings.md)

## <a name="policy-settings-for-mobile-devices"></a>Параметры политики для мобильных устройств

Если создается политика блокировки доступа с включенными определенными настройками, пользователям будет заблокирован доступ к ресурсам Microsoft 365 при использовании поддерживаемого приложения, которое указано в области управления доступом для электронной почты и документов [Microsoft 365.](capabilities.md) 

Параметры, которые могут блокировать доступ пользователей к ресурсам Microsoft 365, находятся в этих разделах:

- Безопасность

- Шифрование

- Взлом устройства

- Управляемый профиль электронной почты  

Например, на следующей схеме показано, что происходит, когда пользователь с зарегистрированным устройством не соответствует параметрам безопасности в политике управления мобильным устройством, применяемой к его устройству. Пользователь впишется в приложение, которое поддерживает управление доступом с помощью Basic Mobility и Security. Им заблокирован доступ к ресурсам Microsoft 365 в приложении до тех пор, пока их устройство не будет соблюдать параметр безопасности.

## <a name="settings-supported-by-windows"></a>Параметры, поддерживаемые Windows

Вы можете управлять устройствами Windows 10, зачислив их в качестве мобильных устройств. После развертывания применимой политики пользователям с устройствами Windows 10 будет необходимо зарегистрироваться в Basic Mobility and Security при первом использовании встроенного приложения электронной почты для доступа к электронной почте Microsoft 365 (требуется премиум-подписка Azure AD).

Следующие параметры поддерживаются для устройств с Windows 10, зарегистрированных в качестве мобильных устройств. Эти параметры не будут блокировать доступ пользователей к ресурсам Microsoft 365.

### <a name="security-settings"></a>Параметры безопасности

- Запрос алфавитно-цифрового пароля

- Минимальная длина пароля

- Количество неудачных попыток входа перед очисткой устройства

- Минут простоя до блокировки устройства

- Срок действия пароля (дней)

- Журнал паролей и предотвращение повторного использования

>[!NOTE]
>Следующие параметры, регулирующие пароли, контролируют только локальные учетные записи Windows. Учетные записи Windows, предоставляемые в рамках пользования доменом или Azure Active Directory, не влияют на эти параметры.

### <a name="system-settings"></a>Параметры системы

Блокировка отправки диагностических данных с устройства.

### <a name="additional-settings"></a>Дополнительные параметры

Эти дополнительные параметры политики можно установить с помощью cmdlets PowerShell:

- AllowConvenienceLogon

- UserAccountControlStatus

- FirewallStatus

- AutoUpdateStatus

- AntiVirusStatus

- AntiVirusSignatureStatus

- SmartScreenEnabled

- WorkFoldersSyncUrl

## <a name="remotely-wipe-a-mobile-device"></a>Удаленная очистка мобильного устройства

Если устройство потеряно или украдено, вы можете удалить конфиденциальные организационные данные и предотвратить доступ к ресурсам организации Microsoft 365, стирая данные из центра обеспечения & соответствия требованиям > управления устройствами по предотвращению потери   >  данных. Можно выполнить выборочную очистку, удалив только данные организации, или полную очистку для удаления всей информации с устройства и возврата устройства к заводским установкам.

Дополнительные сведения см. в [приложении Wipe a mobile device in Basic Mobility and Security.](wipe-mobile-device.md)
