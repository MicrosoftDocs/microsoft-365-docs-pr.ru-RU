# <a name="create-device-security-policies-in-basic-mobility-and-security"></a>Создание политик безопасности устройств в basic Mobility and Security

Вы можете использовать Basic Mobility and Security для создания политик устройств, которые помогают защитить информацию организации в Microsoft 365 от несанкционированного доступа. Вы можете применить политики к любому мобильному устройству в организации, на котором пользователь устройства имеет применимую лицензию Microsoft 365 и зарегистрировал устройство в Basic Mobility and Security.

## <a name="before-you-begin"></a>Прежде чем начать

> [!IMPORTANT]
> Перед созданием политики мобильных устройств необходимо активировать и настроить Basic Mobility and Security. Дополнительные сведения см. в обзоре базовой мобильности и безопасности.

- Узнайте об устройствах, приложениях для мобильных устройств и параметрах безопасности, поддерживаемых в Basic Mobility and Security. См. ["Возможности базовой мобильности и безопасности".](capabilities.md)
- Создайте группы безопасности, которые включают пользователей Microsoft 365, для пользователей, для пользователей, которые могут быть исключены из блокировки доступа к Microsoft 365. Перед развертыванием новой политики организации рекомендуется проверить политику, развернув ее для небольшого количества пользователей. Вы можете создать и использовать группу безопасности, включаемую только себя или небольшое количество пользователей Microsoft 365, которые могут тестировать политику за вас. Дополнительные данные о группах безопасности см. в подзагонах ["Создание, изменение и удаление группы безопасности".](https://go.microsoft.com/fwlink/p/?LinkId=518555)
- Чтобы создать и развернуть основные политики мобильности и безопасности в Microsoft 365, необходимо быть глобальным администратором Microsoft 365. Дополнительные сведения [см. в центре безопасности & соответствия требованиям.](https://support.microsoft.com/office/d10608af-7934-490a-818e-e68f17d0e9c1)
- Перед развертыванием политик дайте своей организации знать о потенциальных последствиях регистрации устройства в Basic Mobility and Security. В зависимости от того, как вы настроили политики, несовершенным устройствам может быть заблокирован доступ к Microsoft 365, а также к данным, включая установленные приложения, фотографии и персональные данные на зарегистрированных устройствах, а также данные могут быть удалены.

>[!NOTE]
>Политики и правила доступа, созданные в стандарте Basic Mobility and Security для Microsoft 365 бизнес стандарт переопределяют политики почтовых ящиков Exchange ActiveSync мобильных устройств и правила доступа к устройствам, созданные в Центре администрирования Exchange. После регистрации устройства в Basic Mobility and Security for Microsoft 365 Business Standard любая политика почтовых ящиков Exchange ActiveSync мобильного устройства или правило доступа к устройству, примененные к устройству, игнорируется. Дополнительные Exchange ActiveSync см. в [Exchange ActiveSync Exchange Online.](https://go.microsoft.com/fwlink/p/?LinkId=524380)

## <a name="step-1-create-a-device-policy-and-deploy-to-a-test-group"></a>Шаг 1. Создание политики устройств и развертывание в тестовой группе

Перед началом убедитесь, что вы активировали и настроили Basic Mobility and Security. Инструкции см. в [обзоре базовой мобильности и безопасности.](overview.md)

1. В браузере введите [https://protection.office.com/devicev2](https://protection.office.com/devicev2) .

2. Выберите **"Создать политику"**.

   :::image type="content" source="../../media/basic-mobility-security/bms-4-policy.png" alt-text="Основные параметры политики мобильности и безопасности":::

3. На странице **параметров политики** укажите требования, которые необходимо применить к мобильным устройствам в организации.

4. **Требовать управления профилем** электронной почты: если этот профиль включен, устройства, не включающие профиль электронной почты, управляемый Basic Mobility and Security, считаются не соответствующими требованиям. На устройстве не может быть управляемого профиля электронной почты, если он неправильно ориентирован или пользователь вручную настроил учетную запись электронной почты на устройстве. Если оставить этот параметр **не включен** (по умолчанию), этот параметр не будет оцениваться на соответствие или несоответствие. Инструкции о том, как пользователи могут получить соответствие требованиям при выборе этого параметра, см. в существующей [учетной записи электронной почты.](https://docs.microsoft.com/intune-user-help/existing-company-email-account-found)

5. На странице "Хотите ли вы применить эту политику **сейчас?",** выберите группы, к которые вы хотите применить эту политику.

6. Выберите **"Создать эту политику".**

Политика передается на устройство каждого пользователя, к следующему входу в Microsoft 365 с помощью мобильного устройства. Если до развертывания политики пользователи не применяли политику к мобильному устройству, они получают на своем устройстве уведомление с действиями по регистрации и активации Basic Mobility and Security. Дополнительные сведения см. в записи [мобильного устройства с помощью Basic Mobility and Security.](enroll-your-mobile-device.md) Пока они не завершат регистрацию в службе Basic Mobility and Security, которая находится в службе Intune, доступ к электронной почте, OneDrive и другим службам будет ограничен. После завершения регистрации с помощью приложения "Портал компании Intune" они могут использовать службы и применить политику к устройству.

## <a name="step-2-verify-that-your-policy-works"></a>Шаг 2. Проверка работы политики

После создания политики устройств убедитесь, что она работает так, как ожидалось, прежде чем развертывать ее в организации.

1. В браузере введите [https://protection.office.com/devicev2](https://protection.office.com/devicev2) .
2. Выберите **"Просмотреть список управляемых устройств".**
3. Проверьте состояние устройств пользователя, к которым применяется политика. Необходимо управлять **состоянием** **устройств.**
4. Вы также можете полностью или выборочно сбросить  данные  на устройстве, нажав кнопку "Сброс фабрики" или "Удалить данные компании из кнопки "Управление" после выбора устройства.  Инструкции см. в [Wipe a mobile device in Microsoft 365.

## <a name="step-3-deploy-a-policy-to-your-organization"></a>Шаг 3. Развертывание политики в организации

После создания политики устройств и проверки ее работы разверните ее в организации.

1. Из браузера: [https://protection.office.com/devicev2](https://protection.office.com/devicev2) .
2. Выберите политику, к которая нужно развернуть, и выберите "Изменить **рядом** с **примененными группами".**
3. Найдите группу, которая будет добавлена, и нажмите кнопку **"Выбрать".**
4. Выберите **параметр "Закрыть"** **и "Изменить".**
5. Выберите **политику закрытия** и **редактирования.**

Политика передается на мобильное устройство каждого пользователя, к следующему входу в Microsoft 365 с мобильного устройства. Если к мобильному устройству пользователей не применена политика, они получают на своем устройстве уведомление с действиями по регистрации и активации для Базовой мобильности и безопасности. После завершения регистрации политика применяется к устройству. Дополнительные сведения см. в записи [мобильного устройства с помощью Basic Mobility and Security.](enroll-your-mobile-device.md)

## <a name="step-4-block-email-access-for-unsupported-devices"></a>Шаг 4. Блокировка доступа к электронной почте для неподтверченных устройств

Чтобы защитить сведения организации, необходимо заблокировать доступ приложений к электронной почте Microsoft 365 для мобильных устройств, которые не поддерживаются basic Mobility and Security. Список поддерживаемых устройств см. в [списке поддерживаемых устройств.](https://support.microsoft.com/office/capabilities-of-basic-mobility-and-security-a1da44e5-7475-4992-be91-9ccec25905b0#bkmk_supporteddevices)

**Чтобы заблокировать доступ к приложению:**

1. В браузере введите [https://protection.office.com/devicev2](https://protection.office.com/devicev2) .
2. Выберите **"Управление настройками доступа к устройствам для всей организации".**
3. Чтобы заблокировать неподдерговые  устройства, выберите "Блокировка" в области "Если устройство не поддерживается базовой мобильностью и безопасностью **для Microsoft 365",** а затем выберите **"Сохранить".**

   :::image type="content" source="../../media/basic-mobility-security/bms-5-block-access.png" alt-text="Параметр блокировки базовой мобильности и безопасности":::

## <a name="step-5-choose-security-groups-to-be-excluded-from-conditional-access-checks"></a>Этап 5. Выбор групп безопасности, которые будут исключены из проверок условного доступа

Если вы хотите исключить определенных пользователей из проверок условного доступа на мобильных устройствах, и для них создана одна или несколько групп безопасности, добавьте эти группы сюда. Для пользователей в этих группах не будут применяться политики для поддерживаемых мобильных устройств. Это рекомендуемый вариант, если вы больше не хотите использовать Basic Mobility and Security в своей организации.

1. В браузере введите [https://protection.office.com/devicev2](https://protection.office.com/devicev2) .

2. Выберите **"Управление настройками доступа к устройствам для всей организации".**

   :::image type="content" source="../../media/basic-mobility-security/bms-4-policy.png" alt-text="Базовая мобильность и безопасность для создания параметра политики":::

3. Выберите  "Добавить", чтобы добавить группу безопасности, в которую необходимо исключить пользователей из заблокированного доступа к Microsoft 365. Когда пользователь добавлен в этот список, он может получить доступ к электронной почте Microsoft 365, если он использует неподтверченное устройство.

4. Выберите группу безопасности, которая будет использовать, на панели **"Выбор группы".**

5. Выберите имя и **добавьте**  >  сохранение.

6. На панели **параметров доступа устройств для** всей организации выберите **"Сохранить".**

   :::image type="content" source="../../media/basic-mobility-security/bms-8-allow-access.png" alt-text="Вариант доступа к базовой мобильной сети и безопасности":::

## <a name="what-is-the-impact-of-security-policies-on-different-device-types"></a>Как политики безопасности влияют на разные типы устройств?

При применении политики к пользовательским устройствам влияние на каждое устройство несколько различается в зависимости от типа устройства. Примеры влияния политик см. в следующей таблице.

|**Политика безопасности**|**Android 4 и более поздние версии**|**Samsung KNOX**|**iOS 6 и более поздние**|**Примечания**|
|:-----|:-----|:-----|:-----|:-----|
|Требовать шифрование резервных копий|Нет|Да|Да|Требуется архивирование с шифрованием iOS.|
|Блокировать резервное копирование в облаке|Да|Да|Да|Блокировать резервное копирование Google для Android (параметр недоступен), облачное резервное копирование для iOS.|
|Блокировать синхронизацию документов|Нет|Нет|Да|iOS: блокировать документы в облаке|
|Блокировать синхронизацию фотографий |Нет|Нет|Да|iOS (собственный): блокировать Photo Stream.|
|Блокировать снимки экрана |Нет|Да|Да|Блокируется при попытке.|
|Блокировать видеоконференции |Нет|Нет|Да|FaceTime заблокирован в iOS, а не в Skype или других.|
|Блокировать отправку данных диагностики |Нет|Да|Да|Блокировать отправку отчета о сбое Google для Android.|
|Блокировать доступ к магазину приложений |Нет|Да|Да|Значок магазина приложений отсутствует на главной странице Android, отключен в Windows, отсутствует в iOS.|
|Требовать пароль для магазина приложений |Нет|Нет|Да|iOS: Пароль требуется для покупок в iTunes.|
|Блокировать подключение съемных носителей |Нет|Да|Недоступно|Android: SD-карта недоступна в параметрах, Windows сообщает пользователю, что установленные приложения недоступны|
|Блокировать подключение Bluetooth |См. примечания|См. примечания|Да|Мы не можем отключить BlueTooth в качестве параметра на Android. Вместо этого мы отключаем все транзакции, для которые требуется BlueTooth: advanced Audio Distribution, Audio/Video Remote Control, устройства без рук, гарнитура, доступ к телефонной книге и последовательный порт. При использовании этих функций в нижней части страницы появляется небольшое всплывающее сообщение.|

## <a name="what-happens-when-you-delete-a-policy-or-remove-a-user-from-the-policy"></a>Что происходит при удалении самой политики или удалении пользователя из политики?

При удалении политики или удалении пользователя из группы, в которой была развернута политика, параметры политики, профиль электронной почты Microsoft 365 и кэшные сообщения электронной почты могут быть удалены с устройства пользователя. См. следующую таблицу, чтобы узнать, что удалено для различных типов устройств.

|**Что удалено**|**iOS 6 и более поздние**|**Android 4 и более поздних версий (включая Android KNOX)**|
|:-----|:-----|:-----|
|Управляемые профили электронной почты<sup>1</sup>|Да|Нет|
|Блокировать резервное копирование в облаке|Да|Нет|

<sup>1</sup> Если политика была развернута  с параметром "Управление профилем электронной почты" выбран, управляемый профиль электронной почты и кэшные сообщения в этом профиле удаляются с устройства пользователя.

Политика удаляется с мобильного устройства для каждого пользователя, к каждому пользователю она применяется при следующей проверке устройства с помощью Basic Mobility and Security. При развертывании новой политики, применяемой к этим устройствам пользователей, им будет предложено повторно зарегистрироваться в Basic Mobility and Security.

Вы также можете полностью стереть устройство или выборочно стереть с него данные организации. Дополнительные сведения см. в [теме "Стирка мобильного устройства"](wipe-mobile-device.md)в basic Mobility and Security.
