# <a name="power-bi-in-your-organization"></a>Power BI в организации

На этой странице описывается, как пользователи в вашей организации могут использовать Power BI и как можно управлять освоением этой службы в вашей организации.

## <a name="what-is-power-bi"></a>Что такое Power BI?

С помощью Microsoft Power BI пользователи могут визуализировать данные, делиться новыми открытиями, а также использовать новые возможности для совместной работы. Дополнительные сведения см. [на сайте Power BI](https://powerbi.microsoft.com/en-us/).
  
## <a name="does-power-bi-meet-national-regional-and-industry-specific-compliance-requirements"></a>Соответствует ли Power BI национальным, региональным и отраслевым требованиям соответствия требованиям?

Дополнительные информацию о соответствии power BI см. в центре [доверия Microsoft.](https://go.microsoft.com/fwlink/?LinkId=785324)
  
## <a name="how-do-users-sign-up-for-power-bi"></a>Как пользователям зарегистрироваться в службе Power BI?

Как администратор вы можете зарегистрироваться в службе Power BI на [веб-сайте Power BI](https://powerbi.microsoft.com/en-us/). Вы также можете зарегистрироваться на странице служб покупки в центре администрирования Microsoft 365. Когда администратор регистрируется в Power BI, он может назначить пользовательские лицензии пользователям, которым требуется доступ.
  
Кроме того, отдельные пользователи в вашей организации могут зарегистрироваться в Power BI на [веб-сайте Power BI](https://powerbi.microsoft.com/en-us/). Когда пользователь вашей организации регистрируется для получения Power BI, ему автоматически назначается лицензия Power BI.
  
## <a name="how-do-individual-users-in-my-organization-sign-up"></a>Как зарегистрироваться отдельным пользователям в организации?

Для пользователей в вашей организации возможны три сценария.
  
### <a name="scenario-1-your-organization-already-has-an-existing-microsoft-365-environment-and-the-user-signing-up-for-power-bi-already-has-a-microsoft-365-account"></a>Сценарий 1. В вашей организации уже есть существующая среда Microsoft 365, а у пользователя, зарегистрировавшись в Power BI, уже есть учетная запись Microsoft 365.

В этом случае, если у пользователя уже есть учетная запись организации или учебного заведения в клиенте (например, contoso.com), но еще нет Power BI, корпорация Майкрософт просто активирует план для этой учетной записи, и пользователю автоматически отправляется уведомление с инструкциями по использованию службы Power BI.
  
### <a name="scenario-2-your-organization-has-an-existing-microsoft-365-environment-and-the-user-signing-up-for-power-bi-doesnt-have-a-microsoft-365-account"></a>Сценарий 2. Ваша организация имеет существующую среду Microsoft 365, а пользователь, подписавшись на Power BI, не имеет учетной записи Microsoft 365.

В этом случае у пользователя есть адрес электронной почты в домене организации (например, contoso.com), но учетная запись Microsoft 365 еще не установлена. В такой ситуации пользователь может зарегистрироваться в Power BI и автоматически получить учетную запись. Таким образом пользователь получает доступ к службе Power BI. Например, если сотрудник по имени Нэнси использует свой рабочий адрес электронной почты (например, Nancy@contoso.com) для регистрации, Корпорация Майкрософт автоматически добавит Нэнси в качестве пользователя в среде Contoso Microsoft 365 и активирует Power BI для этой учетной записи.
  
### <a name="scenario-3-your-organization-does-not-have-a-microsoft-365-environment-connected-to-your-email-domain"></a>Сценарий 3. В вашей организации нет среды Microsoft 365, подключенной к домену электронной почты.

Нет никаких административных действий, необходимых вашей организации для использования Power BI.
  
> [!IMPORTANT]
> Если в организации имеется несколько доменов электронной почты, и вы предпочитаете, чтобы все расширения адресов электронной почты были в одном клиенте, прежде чем пользователи создадут основного клиента, добавьте все домены адресов электронной почты в этот клиент, прежде чем пользователи создадут основного клиента. Автоматизированный механизм перемещения пользователей по клиентам после их создания не существует. Дополнительные сведения об этом процессе см. в статье Если у меня несколько доменов, могу ли я управлять клиентом, к который добавляются [пользователи?](#if-i-have-multiple-domains-can-i-control-the-tenant-that-users-are-added-to) далее в этой статье и добавьте домен в [Office 365](../setup/add-domain.md) в Интернете.
  
## <a name="how-will-this-change-the-way-i-manage-identities-for-users-in-my-organization-today"></a>Как изменится существующий в моей организации подход к управлению удостоверениями пользователей?

Если в вашей организации уже есть существующая среда Microsoft 365 и все пользователи в вашей организации имеют учетные записи Microsoft 365, управление удостоверениями не изменится.
  
Если ваша организация уже имеет существующую среду Microsoft 365, но не все пользователи в организации имеют учетные записи Microsoft 365, мы создадим пользователя в клиенте и назначим лицензии на основе работы пользователя или школьного адреса электронной почты. Это означает, что количество пользователей, которыми вы управляете на текущий момент, будет увеличиваться по мере регистрации пользователей вашей организации для получения службы.
  
Если вы управляете своим каталогом локально и используете службы федерации Active Directory, корпорация Майкрософт не будет добавлять пользователей в клиент, а все пользователи, попытавшиеся присоединиться к нему, получат сообщение с просьбой обратиться к администратору своей организации.
  
Если в вашей организации нет среды Microsoft 365, подключенной к вашему домену электронной почты, не будет изменений в управлении удостоверением. Пользователи будут добавлены в новый, полностью облачный каталог пользователей, а вы сможете при желании принять на себя обязанности администратора клиента и приступить к управлению.
  
## <a name="what-is-the-process-to-manage-a-tenant-created-by-microsoft-for-my-users"></a>В чем заключается процесс управления клиентом, созданным корпорацией Майкрософт для моих пользователей?

Если клиент был создан корпорацией Майкрософт, вы можете переключить на себя управление им, выполнив указанные ниже действия.

Присоединитесь к клиенту, [выполнив регистрацию в Power BI](https://go.microsoft.com/fwlink/?LinkId=522448) с использованием домена электронной почты, совпадающего с доменом клиента, которым вы хотите управлять. Например, если корпорация Майкрософт создала клиент contoso.com, вам необходимо присоединиться к нему, используя адрес электронной почты, который заканчивается символами @contoso.com.

1. Запросите административный контроль, подтвердив владение доменом: после присоединения к клиенту вы можете повысить свой уровень до роли администратора, подтвердив владение доменом. Для этого выполните указанные ниже действия.

::: moniker range="o365-worldwide"

3. Перейдите на веб-сайт [https://admin.microsoft.com](https://admin.microsoft.com).

::: moniker-end

::: moniker range="o365-germany"

3. Перейдите по ссылке [https://portal.office.de](https://portal.office.de).

::: moniker-end

::: moniker range="o365-21vianet"

3. Перейдите по ссылке [https://portal.partner.microsoftonline.cn](https://portal.partner.microsoftonline.cn).

::: moniker-end

4. В левом верхнем углу щелкните значок средства запуска приложений и выберите **Администратор**.

    ![Запуск приложения с выделенным приложением Admin](../../media/4eea9dbc-591b-48be-9916-322d41c6525b.png)
  
5. Прочитайте инструкции на странице **Стать администратором,** а затем выберите **Да, я хочу быть администратором**.

    > [!NOTE]
    >  Если этот параметр не появится, администратор уже на месте.
  
## <a name="if-i-have-multiple-domains-can-i-control-the-tenant-that-users-are-added-to"></a>Если у меня несколько доменов, могу ли я управлять клиентом, в который добавлены пользователи?

По умолчанию клиенты создаются для каждого домена и поддомена электронной почты пользователей.
  
Если вы хотите разместить всех пользователей в одном клиенте независимо от расширений адресов электронной почты, сделайте следующее:
  
- Заранее создайте целевой клиент (или используйте существующий) и добавьте в него все имеющиеся домены и поддомены. В этом случае все пользователи с адресами электронной почты, соответствующими этим доменам и поддоменам, при регистрации будут автоматически присоединены к целевому клиенту.

> [!IMPORTANT]
> Автоматизированная процедура перемещения уже созданных пользователей в другие клиенты не предусмотрена. Подробнее о добавлении доменов к одному клиенту Microsoft 365 см. в добавлении домена [в Office 365.](../setup/add-domain.md)

> [!IMPORTANT]
> Дополнительные сведения и рекомендации по управлению клиентами см. в инструкции [По администрированию Power BI?.](/power-bi/service-admin-administering-power-bi-in-your-organization)
  
## <a name="how-can-i-prevent-users-from-joining-my-existing-tenant"></a>Как запретить пользователям присоединяться к существующему клиенту?

Существует ряд действий, которые можно предпринять в качестве администратора, чтобы запретить пользователям присоединяться к существующему клиенту. Если вы заблокировали присоединение пользователей к клиенту, попытки пользователей войти не увенчаются ошибкой, и они будут направлены на контакт с администратором организации. Вам не нужно повторять этот процесс, если вы уже отключили автоматическое распространение лицензий до этого (например, Office 365 Education for Students, Faculty, and Staff).
  
Для выполнения этих действий требуется Windows PowerShell. Чтобы начать работу с Windows PowerShell, см. [Руководство по началу работы с PowerShell](/powershell/scripting/overview).
  
Для выполнения следующих действий необходимо установить последнюю 64-битную версию модуля [PowerShell Azure Active Directory V2.](https://www.powershellgallery.com/packages/AzureADPreview/2.0.2.5)
  
После выбора ссылки выберите **Выполнить** для запуска пакета установщика.
  
**Disable automatic tenant join** (Отключить автоматическое присоединение к клиенту). Используйте эту команду Windows PowerShell, чтобы запретить новым пользователям присоединяться к клиенту, которым вы управляете.
  
Отключение автоматического присоединения к клиенту для новых пользователей:  `Set-MsolCompanySettings -AllowEmailVerifiedUsers $false`
  
Включение автоматического присоединения к клиенту для новых пользователей:  `Set-MsolCompanySettings -AllowEmailVerifiedUsers $true`
  
> [!NOTE]
> Эта блокировка не позволяет новым пользователям в организации зарегистрироваться в Power BI. Пользователи, которые зарегистрировались в Power BI до отключения новых регистраций в организации, по-прежнему сохраняют лицензии. См. инструкции по удалению Power BI для пользователей, которые уже [подписались?](#how-do-i-remove-power-bi-for-users-that-already-signed-up) для инструкций по удалению доступа к Power BI для пользователей, которые ранее подписались на службу.
  
## <a name="how-can-i-allow-users-to-join-my-existing-tenant"></a>Как разрешить пользователям присоединяться к существующему клиенту?

To allow users to join your tenant, run the opposite command as described in the question above:  `Set-MsolCompanySettings -AllowEmailVerifiedUsers $true`
  
## <a name="how-do-i-verify-if-i-have-the-block-on-in-the-tenant"></a>Как можно проверить, заблокирован ли клиент?

Воспользуйтесь следующим сценарием PowerShell:  `Get-MsolCompanyInformation | fl allow*`
  
## <a name="how-can-i-prevent-my-existing-users-from-starting-to-use-power-bi"></a>Как запретить имеющимся пользователям начать работу с Power BI?

**Отключение автоматического распространения лицензии:** Используйте этот Windows PowerShell, чтобы отключить автоматические рассылки лицензий для существующих пользователей. Вам не нужно повторять этот процесс, если вы уже отключили автоматическое распространение лицензий до этого (например, Office 365 Education for Students, Faculty, and Staff).
  
Отключение автоматической рассылки лицензий для существующих пользователей:  `Set-MsolCompanySettings -AllowAdHocSubscriptions $false`
  
Включение автоматической рассылки лицензий для существующих пользователей:  `Set-MsolCompanySettings -AllowAdHocSubscriptions $true`
  
> [!NOTE]
> Флаг *AllowAdHocSubscriptions* используется для управления несколькими возможностями пользователей в организации, включая возможность регистрации пользователей в службе управления правами Azure. Состояние этого флажка влияет на все эти возможности.
  
## <a name="how-can-i-allow-my-existing-users-to-sign-up-for-power-bi"></a>Как разрешить существующим пользователям зарегистрироваться в Power BI?

To allow your existing users to sign up for Power BI, run the opposite command as described in the question above:  `Set-MsolCompanySettings -AllowAdHocSubscriptions $true`
  
## <a name="how-do-i-remove-power-bi-for-users-that-already-signed-up"></a>Как удалить доступ к Power BI для пользователей, которые уже зарегистрированы?

Если пользователь подписался на Power BI, но вы больше не хотите, чтобы у них был доступ к Power BI, вы можете удалить лицензию Power BI для этого пользователя.

::: moniker range="o365-worldwide"
  
1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. Найдите пользователя, для которого нужно удалить лицензию, а затем выберите его имя.

3. На **вкладке Лицензии и Приложения** очистить поле **Microsoft Power BI.**

4. Нажмите кнопку **Сохранить изменения**.

::: moniker-end

::: moniker range="o365-germany"

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=847686" target="_blank">Активные пользователи</a>.

2. Найдите пользователя, для которого нужно удалить лицензию, а затем выберите его имя.

3. Рядом с **лицензиями на продукт** выберите **Изменить**.

4. Отключение **параметра Microsoft Power BI.**

5. Нажмите **Сохранить**.

::: moniker-end

::: moniker range="o365-21vianet"

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=850628" target="_blank">Активные пользователи</a>.

2. Найдите пользователя, для которого нужно удалить лицензию, а затем выберите его имя.

3. Рядом с **лицензиями на продукт** выберите **Изменить**.

4. Отключение **параметра Microsoft Power BI.**

5. Нажмите **Сохранить**.

::: moniker-end

## <a name="how-do-i-know-when-new-users-have-joined-my-tenant"></a>Как узнать, что к моему клиенту подключились новые пользователи?

Пользователи, присоединившиеся к клиенту в рамках данной программы, получают уникальную лицензию, по которой можно отфильтровать список активных пользователей на панели администратора.
  
Чтобы создать это новое представление в центре администрирования, выполните действия в [Create a custom user view](../add-users/create-edit-or-delete-a-custom-user-view.md#create-a-custom-user-view). В **соответствии с назначенной лицензией на продукт** выберите Microsoft Power **BI**. После создания нового представления вы сможете увидеть всех пользователей в клиенте, которые зарегистрировались в этой программе.
  
## <a name="are-there-any-additional-things-i-should-be-prepared-for"></a>Есть ли какие-нибудь подводные камни, которые нужно иметь в виду?

Возможно, вы чаще будете сталкиваться с запросами на сброс пароля. Сведения об этой процедуре см. в статье [Сброс пароля пользователя](../add-users/reset-passwords.md).
  
Вы можете удалить пользователя из клиента с помощью стандартного процесса в центре администрирования. Однако если у пользователя по-прежнему имеется активный адрес электронной почты, предоставленный вашей организацией, он сможет снова присоединиться к клиенту, если только вы не заблокируете эту возможность для всех пользователей.
  
## <a name="why-did-1-million-licenses-for-microsoft-power-bi-show-up-in-my-tenant"></a>Почему в моем клиенте было 1 миллион лицензий для Microsoft Power BI?

Пользователи соответствующей организации имеют право использовать службу Microsoft Power BI, и эти лицензии обозначают доступное количество новых пользователей Power BI в вашем клиенте. Это бесплатные лицензии. Если вы решили разрешить пользователям самостоятельно зарегистрироваться в Power BI, им будет назначена одна из этих доступных бесплатных лицензий, когда они завершат процесс регистрации. Вы также можете назначить эти лицензии пользователям самостоятельно через центр администрирования.
  
## <a name="is-this-free-will-i-be-charged-for-these-licenses"></a>Это бесплатный продукт? Мне нужно платить за эти лицензии?

Эти лицензии предназначены для бесплатной версии Power BI. Если вам необходимы дополнительные возможности, обратите внимание на версию Power BI Pro.
  
## <a name="why-1-million-licenses"></a>Зачем нужен 1 миллион лицензий?

Мы выбрали номер, который был достаточно большим, чтобы у большинства организаций было достаточно лицензий, чтобы обеспечить эту выгоду без задержек для своих пользователей.
  
## <a name="what-if-i-need-more-than-1-million-licenses"></a>Что делать, если требуется больше 1 миллиона лицензий?

За сведениями о том, как получить дополнительные лицензии, обратитесь к своему представителю корпорации Майкрософт.
