## <a name="1-set-up-multi-factor-authentication"></a>1. Настройка многофакторной проверки подлинности
<a name="setup"> </a>

Использование многофакторной проверки подлинности является одним из самых простых и эффективных способов повышения безопасности организации. Это проще, чем кажется: при входе в систему многофакторная проверка подлинности означает, что вы введите код с телефона, чтобы получить доступ к Microsoft 365. Это может помешать хакерам взять на себя, если они знают пароль. Многофакторная проверка подлинности также называется 2-ступенчатой проверкой. Пользователи могут легко добавлять двухшаговую проверку в большинство учетных записей, например в учетные записи Google или Microsoft. Вот как добавить [двухшаговую проверку в личную учетную запись Майкрософт.](https://go.microsoft.com/fwlink/p/?linkid=2016403)

Для предприятий, использующих Microsoft 365, добавьте параметр, который требует от пользователей входа с помощью многофакторной проверки подлинности. При этом изменении пользователям будет предложено настроить телефон для двух факторов проверки подлинности при следующем входе.
Чтобы увидеть обучающий видеоролик о том, как настроить [MFA](../../business-video/turn-on-mfa.md) и как пользователи завершают настройка, см. в этой ссылке настройка MFA и [настройка пользователя.](../../business-video/set-up-mfa.md)

Чтобы настроить многофакторную проверку подлинности, включаем по умолчанию безопасность:

Параметры безопасности по умолчанию обеспечивают подходящий для большинства организаций уровень безопасности при входе в систему. Дополнительные сведения см. в статье [Что такое параметры безопасности по умолчанию?](/azure/active-directory/fundamentals/concept-fundamentals-security-defaults)

Если у вас новая подписка, то параметры безопасности по умолчанию могли быть включены автоматически.

Включить или отключить параметры безопасности по умолчанию можно на портале Azure в области **Свойства** для Azure Active Directory (Azure AD).

1. Войдите в [Центр администрирования Microsoft 365](https://admin.microsoft.com) с учетными данными глобального администратора.
2. На левой панели навигации нажмите **Показать все**, и в разделе **Центры администрирования** выберите **Azure Active Directory**.
3. В центре администрирования **Azure Active Directory** выберите **Azure Active Directory** > **Свойства**.
4. В нижней части страницы щелкните **Управление параметрами безопасности по умолчанию**.
5. Выберите **Да**, чтобы включить параметры безопасности по умолчанию, или **Нет**, чтобы их отключить, а затем нажмите **Сохранить**.

После настройки многофакторной проверки подлинности для организации вашим пользователям потребуется настроить двухфакторную проверку на своих устройствах. Дополнительные сведения см. [в ок. Настройка двухшаговой](https://support.microsoft.com/office/ace1d096-61e5-449b-a875-58eb3d74de14)проверки для Microsoft 365 .

Подробные сведения и рекомендации см. в материале [Настройка многофакторной](set-up-multi-factor-authentication.md)проверки подлинности для пользователей.

## <a name="3-use-dedicated-admin-accounts"></a>3. Использование выделенных учетных записей администратора
<a name="admin"> </a>

Административные учетные записи, которые вы используете для администрирования Microsoft 365 среды, включают повышенные привилегии. Это ценные цели для хакеров и киберпреступлений. Используйте учетные записи администратора только для администрирования. Администраторы должны иметь отдельную учетную запись пользователя для регулярного, неуправленного использования и использовать свою административную учетную запись только при необходимости для выполнения задачи, связанной с их функцией работы. Дополнительные рекомендации:

- Убедитесь, что учетные записи администратора также настроены для многофакторной проверки подлинности.

- Перед использованием учетных записей администратора закрой все несвязанные сеансы браузера и приложения, включая личные учетные записи электронной почты.

- После выполнения задач администратора не забудьте выйти из сеанса браузера.

## <a name="4-raise-the-level-of-protection-against-malware-in-mail"></a>4. Повышение уровня защиты от вредоносных программ в почте
<a name="malware"> </a>

Ваша Microsoft 365 среда включает защиту от вредоносных программ, но эту защиту можно увеличить, блокируя вложения с типами файлов, которые обычно используются для вредоносных программ. Чтобы натыкаться на защиту от вредоносных программ в электронной почте, просмотреть короткое обучающее [видео](../../business-video/anti-malware.md)или выполнить следующие действия:

1. Войдите <https://protection.office.com> и войдите с учетными данными учетной записи администратора.

2. В Центре & безопасности в левой области навигации под управлением **Угрозы** выберите **политику** \> **защиты от вредоносных программ.**

3. Дважды щелкните политику по умолчанию, чтобы изменить эту политику всей компании.

4. Выберите **Настройки**.

5. В **соответствии с общим фильтром типов вложений** выберите **On**. Заблокированные типы файлов перечислены в окне непосредственно под этим управлением. При необходимости можно добавить или удалить типы файлов позже.

6. Выберите **Сохранить.**

Дополнительные сведения см. в [программе Защита от вредоносных программ в EOP.](../../security/office-365-security/anti-malware-protection.md)

## <a name="5-protect-against-ransomware"></a>5. Защита от программ-вымогателей
<a name="ransomware"> </a>

Программа-вымогательство ограничивает доступ к данным путем шифрования файлов или блокировки экранов компьютеров. Затем он пытается вымогать деньги у жертв, запрашивая "выкуп", как правило, в виде криптовалют, таких как Биткоин, в обмен на доступ к данным.

Вы можете защититься от программ-вымогателей, создав одно или несколько правил потока почты, чтобы заблокировать расширения файлов, которые обычно используются для программ-вымогателей, или предупредить пользователей, которые получают эти вложения по электронной почте. Хорошей отправной точкой является создание двух правил:

- Предупредить пользователей перед открытием Office файлов, которые включают макрос. Вымогатели могут быть скрыты в макросах, поэтому мы будем предупреждать пользователей не открывать эти файлы от людей, которых они не знают.

- Блокировать типы файлов, которые могут содержать вымогателей или другой вредоносный код. Начнем с общего списка исполняемых (перечисленных в таблице ниже). Если в организации используется любой из этих исполняемых типов и вы ожидаете, что они будут отправлены по электронной почте, добавьте их к предыдущему правилу (предупредите пользователей).

Чтобы создать правило транспорта почты, просмотреть [короткое обучающее видео](../../business-video/prevent-ransom-in-email.md)или выполнить следующие действия:

1. Перейдите в [Центр администрирования Exchange](https://go.microsoft.com/fwlink/p/?linkid=2059104).

2. В категории **потока** почты выберите **правила**.

3. Выберите **+** и **создайте новое правило.**

4. Выберите **** в нижней части диалогового окна, чтобы увидеть полный набор параметров.

5. Применение параметров в следующей таблице для каждого правила. Оставьте остальные параметры по умолчанию, если вы не хотите изменить эти параметры.

6. Нажмите кнопку **Сохранить**.

## <a name="6-stop-auto-forwarding-for-email"></a>6. Остановка автоматической переададки для электронной почты
<a name="forwarding"> </a>

Хакеры, которые получают доступ к почтовому ящику пользователя, могут отключить почту, настроив почтовый ящик для автоматической пересылки электронной почты. Это может произойти даже без осведомленности пользователя. Это можно предотвратить, настроив правило потока почты.

Чтобы создать правило транспорта почты:

1. Перейдите в [Центр администрирования Exchange](https://go.microsoft.com/fwlink/p/?linkid=2059104).

2. В категории **потока** почты выберите **правила**.

3. Выберите **+** и **создайте новое правило.**

4. Выберите **дополнительные параметры** в нижней части диалогового окна, чтобы увидеть полный набор параметров.

5. Применить параметры в следующей таблице. Оставьте остальные параметры по умолчанию, если вы не хотите изменить эти параметры.

6. Нажмите кнопку **Сохранить**.

|Setting|Отклонение автоматических сообщений электронной почты на внешние домены|
|---|---|
|Имя|Предотвращение автоматической отправки электронной почты во внешние домены|

|Добавление условия|Получатель . . . внешний/внутренний . . . Вне организации|
|Добавление условия|Свойства сообщения . . . включаем тип сообщения. . . Автопродажа|
|Сделайте следующее ...|Блокировка сообщения . . . отклонить сообщение и включить объяснение.|
|Предоставление текста сообщения|Автопродажа электронной почты за пределами этой организации предотвращается из соображений безопасности.|

## <a name="7-use-office-message-encryption"></a>7. Использование Office шифрования сообщений
<a name="encryption"> </a>

Office Шифрование сообщений включено в Microsoft 365. Она уже настроена. С Office шифрования сообщений организация может отправлять и получать зашифрованные сообщения электронной почты между людьми внутри и за ее пределами. Шифрование сообщений Office 365 работает с Outlook.com, Yahoo!, Gmail и другими почтовыми службами. Шифрование сообщений электронной почты позволяет убедиться, что только предполагаемые получатели могут просматривать содержимое сообщения.

Office Шифрование сообщений предоставляет два варианта защиты при отправке почты:

- Не переад.

- Шифрование

Возможно, в организации были настроены дополнительные параметры, которые применяют метку к электронной почте, например Confidential.

### <a name="to-send-protected-email"></a>Отправка защищенной электронной почты

В Outlook для ПК выберите **Параметры** в электронной почте, а затем **выберите Разрешения**.

![Шифрование сообщений электронной почты в Outlook](../../media/08e90a7e-a2d2-41a4-bae9-0a46b4ce639a.png)

В Outlook.com выберите **Protect** в электронной почте. Защита по умолчанию не **переад.** Чтобы изменить этот код для шифрования, выберите **Шифруем разрешения на** \> **изменение.**

![Шифрование сообщений электронной почты в Outlook.com](../../media/329ccf50-f6b1-4fb8-b249-60b907a82b7e.png)

### <a name="to-receive-encrypted-email"></a>Получение зашифрованной электронной почты

Если у получателя Outlook 2013 или Outlook 2016 учетная запись электронной почты Майкрософт, он увидит предупреждение об ограничениях элемента в области чтения. После открытия сообщения получатель может просматривать сообщение так же, как и любое другое.

Если получатель использует другой клиент электронной почты или учетную запись электронной почты, например Gmail или Yahoo, он увидит ссылку, которая позволяет ему либо войти, чтобы прочитать сообщение электронной почты, либо запросить разовой пароль для просмотра сообщения в веб-браузере. Если пользователи не получают сообщение электронной почты, убедитесь, что они проверяют свою папку нежелательной почты или нежелательной почты.

Дополнительные сведения см. в [тексте Отправка, просмотр](https://support.microsoft.com/office/eaa43495-9bbb-4fca-922a-df90dee51980)и ответ на зашифрованные сообщения в Outlook для ПК .

## <a name="8-protect-your-email-from-phishing-attacks"></a>8. Защита электронной почты от фишинговых атак
<a name="phishing"> </a>

Если вы настроили один или несколько настраиваемых доменов для Microsoft 365 среды, можно настроить адресную защиту от фишинга. Защита от фишинга, которая является частью Microsoft Defender для Office 365, может помочь защитить организацию от вредоносных фишинговых атак на основе обезличения и других фишинговых атак. Если вы не настроили настраиваемый домен, вам не нужно это делать.

Рекомендуется начать работу с этой защитой, создав политику для защиты наиболее важных пользователей и настраиваемого домена.

![Создание политики защиты от фишинга в Microsoft Defender для Office 365](../../media/security-and-compliance-center.png)

Чтобы создать политику защиты от фишинга в Defender для [](../../business-video/setup-anti-phishing.md)Office 365, просмотреть короткое обучающее видео или выполнить следующие действия:

1. Перейдите на сайт <https://protection.office.com>.

2. В Центре & безопасности в левой области навигации под управлением **угрозой** выберите **Политику**.

3. На странице Политика выберите **anti-phishing**.

4. На странице Anti-phishing выберите **+Create**. Мастер запускает этот шаг, определяя политику защиты от фишинга.

5. Укажите имя, описание и параметры политики, как рекомендуется на приведенной ниже диаграмме. Дополнительные сведения см. в материале "Сведения о политике защиты от фишинга" в [Microsoft Defender Office 365 дополнительные](../../security/office-365-security/set-up-anti-phishing-policies.md) сведения.

6. После просмотра параметров выберите **Создать** эту политику или **сохранить,** если это необходимо.

|Параметр или параметр|Рекомендуемый параметр|
|---|---|
|Имя|Домен и самый ценный сотрудник кампании|
|Описание|Убедитесь, что наиболее важные сотрудники и наш домен не будут вымысированы.|
|Добавить защищаемых пользователей|Выберите **+ Добавьте условие, получатель .** Введите имена пользователей или введите адрес электронной почты кандидата, руководителя кампании и других важных сотрудников. Можно добавить до 20 внутренних и внешних адресов, которые необходимо защитить от обезличения.|
|Добавить защищаемые домены|Выберите **+ Добавьте условие, домен получателя**. Введите настраиваемый домен, связанный с Microsoft 365 подпиской, если вы его определили. Вы можете ввести несколько доменов.|
|Выберите действия|Если электронная почта отправляется обезличенным пользователем: выберите перенаправление сообщения на другой адрес электронной **почты,** а затем введите адрес электронной почты администратора безопасности; например, securityadmin@contoso.com. <br/> Если электронная почта отправляется с помощью обезличенных доменов: выберите **сообщение карантина.**|
|Аналитика почтовых ящиков|По умолчанию аналитика почтовых ящиков выбирается при создании политики защиты от фишинга. Для достижения наилучших результатов оставьте значение **Вкл.**|
|Добавить отправителей и домены, которым можно доверять|В этом примере не задавайте никаких переопределений.|
|Применяется к|Выберите **Домен получателя**. В разделе **Любой из этих вариантов** щелкните **Выбрать**. Выберите **+ Добавить**. Выберите контрольный ящик рядом с именем домена, например contoso.com, в списке, а затем выберите **Добавить**. Нажмите кнопку **Готово**.|
|

## <a name="9-protect-against-malicious-attachments-and-files-with-safe-attachments"></a>9. Защита от вредоносных вложений и файлов с помощью Сейф вложений
<a name="atp"> </a>

Люди регулярно отправляют, получают и делятся вложениями, такими как документы, презентации, электронные таблицы и другие. Не всегда легко определить, является ли вложение безопасным или вредоносным, просто глядя на сообщение электронной почты. Microsoft Defender для Office 365 включает Сейф вложения, но эта защита не включается по умолчанию. Мы рекомендуем создать новое правило, чтобы приступить к использованию этой защиты. Эта защита распространяется на файлы в SharePoint, OneDrive и Microsoft Teams.

Чтобы создать политику Сейф вложений, просмотреть короткое обучающее [видео](../../business-video/safe-attachments.md)или выполнить следующие действия:

1. Перейдите <https://protection.office.com> к учетной записи администратора и войдите в нее.

2. В Центре & безопасности в левой области навигации под управлением **угрозой** выберите **Политику**.

3. На странице Политика выберите **Сейф вложения.**

4. На странице Сейф вложения широко применяйте эту защиту, выбрав поле Включить ATP для **SharePoint, OneDrive** и Microsoft Teams.

5. Выберите **+** для создания новой политики.

6. Применить параметры в следующей таблице.

7. После просмотра параметров выберите **Создать** эту политику или **сохранить,** если это необходимо.

|Параметр или параметр|Рекомендуемый параметр|
|---|---|
|Имя|Блокировка текущих и будущих сообщений электронной почты с обнаруженными вредоносными программами.|
|Описание|Блокировка текущих и будущих сообщений электронной почты и вложений с обнаруженными вредоносными программами.|
|Сохранение вложений неизвестного ответа на вредоносные программы|Выберите Блок . Блокировать текущие и будущие электронные почты и вложения с **обнаруженными вредоносными программами.**|
|Перенаправление вложений при обнаружении|Включить перенаправление (выберите это поле) <br/> Введите учетную запись администратора или настройку почтового ящика для карантина. <br/> Применение выше выбора, если сканирование вредоносных программ для вложений раз или ошибка происходит (выберите этот поле).|
|Применяется к|Домен получателя . . . выберите домен.|
|

Дополнительные сведения см. в [сообщении Set up anti-phishing policies in Defender for Office 365.](../../security/office-365-security/configure-atp-anti-phishing-policies.md)

## <a name="10-protect-against-phishing-attacks-with-safe-links"></a>10. Защита от фишинговых атак с помощью Сейф ссылки
<a name="phishingatp"> </a>

Иногда хакеры скрывают вредоносные веб-сайты в ссылках в электронной почте или других файлах. Сейф Ссылки, которые являются частью Microsoft Defender для Office 365, могут помочь защитить организацию, предоставив проверку веб-адресов (URL-адресов) в сообщениях электронной почты и Office документах. Защита определяется с помощью Сейф ссылок.

Рекомендуется сделать следующее:

- Измените политику по умолчанию, чтобы повысить защиту.

- Добавьте новую политику, ориентированную на всех получателей в домене.

Чтобы получить Сейф ссылки, просмотреть [короткое обучающее видео](../../business-video/safe-links.md)или выполнить следующие действия:

1. Перейдите <https://protection.office.com> к учетной записи администратора и войдите в нее.

2. В Центре & безопасности в левой области навигации под управлением **угрозой** выберите **Политику**.

3. На странице Политика выберите **ссылки Сейф ссылки**.

Чтобы изменить политику по умолчанию:

1. На странице Сейф ссылки, в статье **Политики,** применимые к всей организации, дважды щелкните политику **по умолчанию.**

2. В **Параметры,** которые применяются к контенту Office 365, введите URL-адрес, который будет заблокирован, например _example.com,_ и выберите **+** .

3. В **Параметры,** которые применяются к контенту, кроме электронной почты, выберите Office 365 **приложения,** не отслеживайте, когда пользователи щелкают безопасные ссылки, и не позволяйте пользователям щелкнуть безопасные ссылки на исходный **URL-адрес.**

4. Нажмите кнопку **Сохранить**.

Создание новой политики, адресной всем получателям в вашем домене:

1. На странице Сейф ссылки в статье **Политики,** применимые к конкретным получателям, выберите **+** для создания новой политики.

2. Применение параметров, перечисленных в следующей таблице.

3. Нажмите кнопку **Сохранить**.

|Параметр или параметр|Рекомендуемый параметр|
|---|---|
|Имя|Сейф ссылки для всех получателей в домене|
|Выберите действие для неизвестных потенциально вредоносных URL-адресов в сообщениях|Выберите On — URL-адреса будут переписаны и проверены в отношении списка известных вредоносных ссылок при нажатии **пользователем ссылки.**|
|Применение сканирования URL-адресов в режиме реального времени для подозрительных ссылок и ссылок, которые указывают на файлы|Выберите это поле.|
|Применяется к|Домен получателя . . . выберите домен.|
|

Дополнительные сведения см. [в Сейф Ссылки в Microsoft Defender для Office 365.](../../security/office-365-security/atp-safe-links.md)
