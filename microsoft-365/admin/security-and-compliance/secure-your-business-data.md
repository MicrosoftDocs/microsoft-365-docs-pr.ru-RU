## <a name="1-set-up-multi-factor-authentication"></a>1. Настройка многофакторной проверки подлинности
<a name="setup"> </a>

Использование многофакторной проверки подлинности — один из самых простых и эффективных способов повышения безопасности организации. Это проще, чем звучит. При входе многофакторная проверка подлинности означает, что вы введите код с телефона, чтобы получить доступ к Microsoft 365. Это может помешать злоумышленникам взять на себя контроль, если они знают ваш пароль. Многофакторная проверка подлинности также называется двухэтапной проверкой подлинности. Пользователи могут легко добавлять двухшаговую проверку в большинство учетных записей, например в свои учетные записи Google или Майкрософт. Вот как добавить [двухшаговую проверку в личную учетную запись Майкрософт.](https://go.microsoft.com/fwlink/p/?linkid=2016403)

Для предприятий, использующих Microsoft 365, добавьте параметр, который требует, чтобы пользователи входить с помощью многофакторной проверки подлинности. После внести это изменение пользователям будет предложено настроить свой телефон для двух-факторной проверки подлинности при следующем входе в систему.
Обучающий видеоролик о том, как настроить MFA и как [](https://support.microsoft.com/office/e12187b8-216a-4490-9e3b-df34a06fb787) пользователи завершат ее, см. в этой [теме.](https://support.microsoft.com/office/a32541df-079c-420d-9395-9d59354f7225)

Чтобы настроить многофакторную проверку подлинности, необходимо включить значения по умолчанию для системы безопасности:

Параметры безопасности по умолчанию обеспечивают подходящий для большинства организаций уровень безопасности при входе в систему. Дополнительные сведения см. в статье [Что такое параметры безопасности по умолчанию?](https://docs.microsoft.com/azure/active-directory/fundamentals/concept-fundamentals-security-defaults)

Если у вас новая подписка, то параметры безопасности по умолчанию могли быть включены автоматически.

Включить или отключить параметры безопасности по умолчанию можно на портале Azure в области **Свойства** для Azure Active Directory (Azure AD).

1. Войдите в [Центр администрирования Microsoft 365](https://admin.microsoft.com) с учетными данными глобального администратора.
2. На левой панели навигации нажмите **Показать все**, и в разделе **Центры администрирования** выберите **Azure Active Directory**.
3. В центре администрирования **Azure Active Directory** выберите **Azure Active Directory** > **Свойства**.
4. В нижней части страницы щелкните **Управление параметрами безопасности по умолчанию**.
5. Выберите **Да**, чтобы включить параметры безопасности по умолчанию, или **Нет**, чтобы их отключить, а затем нажмите **Сохранить**.

После настройки многофакторной проверки подлинности для организации вашим пользователям потребуется настроить двухфакторную проверку на своих устройствах. Дополнительные сведения [см. в настройках двухшаговой проверки для Microsoft 365.](https://support.microsoft.com/office/ace1d096-61e5-449b-a875-58eb3d74de14)

Подробные сведения и полные рекомендации см. в настройках многофакторной [проверки подлинности для пользователей.](set-up-multi-factor-authentication.md)

## <a name="3-use-dedicated-admin-accounts"></a>3. Использование выделенных учетных записей администраторов
<a name="admin"> </a>

Административные учетные записи, которые вы используете для администрирования среды Microsoft 365, включают повышенные привилегии. Это ценные цели для злоумышленников и киберпреступных. Используйте учетные записи администратора только для администрирования. Администраторы должны иметь отдельную учетную запись пользователя для обычного, не административного использования и использовать административную учетную запись, только если это необходимо для выполнения задачи, связанной с их рабочими задачами. Дополнительные рекомендации:

- Убедитесь, что учетные записи администраторов также настроены для многофакторной проверки подлинности.

- Перед использованием учетных записей администратора закроем все несвязанные сеансы браузера и приложения, включая личные учетные записи электронной почты.

- После выполнения задач администрирования обязательно выйдите из сеанса браузера.

## <a name="4-raise-the-level-of-protection-against-malware-in-mail"></a>4. Повышение уровня защиты от вредоносных программ в почте
<a name="malware"> </a>

Ваша среда Microsoft 365 включает защиту от вредоносных программ, но вы можете повысить эту защиту, блокируя вложения с типами файлов, которые обычно используются для защиты от вредоносных программ. Чтобы защитить электронную почту от вредоносных программ, просмотреть [короткое](https://support.microsoft.com/office/02b5783a-eea0-42e8-8856-62440718c3f0)обучающее видео или выполнить следующие действия:

1. Войдите в учетную запись <https://protection.office.com> администратора и войдите в нее.

2. В Центре безопасности & соответствия требованиям в области навигации слева в области управления угрозами выберите **"Политика** защиты от \> **вредоносных программ".**

3. Дважды щелкните политику по умолчанию, чтобы изменить эту политику для всей компании.

4. Выберите **Настройки**.

5. В **области фильтра общих типов вложений** выберите **"В".** Заблокированные типы файлов перечислены в окне непосредственно под этим управлением. При необходимости можно добавить или удалить типы файлов позже.

6. Выберите **"Сохранить".**

Дополнительные сведения см. в сведениях о защите [от вредоносных программ в EOP.](https://docs.microsoft.com/microsoft-365/security/office-365-security/anti-malware-protection)

## <a name="5-protect-against-ransomware"></a>5. Защита от программ-вымогателей
<a name="ransomware"> </a>

Программа-вымогаатель ограничивает доступ к данным путем шифрования файлов или блокировки экранов компьютеров. Затем он пытается вымогать деньги у пострадавших, запрашивая "выкуп", как правило, в виде майлипов, таких как "Жижий", в обмен на доступ к данным.

Вы можете защититься от программ-вымогателей, создав одно или несколько правил потока почты, чтобы заблокировать расширения файлов, которые обычно используются для программ-вымогателей, или предупредить пользователей, которые получают эти вложения по электронной почте. Отправной точкой является создание двух правил:

- Предупреждать пользователей перед открытием вложений Office, которые включают макрос. Программы-вымогатели могут быть скрыты в макросах, поэтому мы предупреждаем пользователей, что не открывать эти файлы от людей, которых они не знают.

- Блокировать типы файлов, которые могут содержать программы-вымогателя или другой вредоносный код. Начнем с общего списка исполняемых исполняний (перечисленных в таблице ниже). Если ваша организация использует любой из этих типов исполняемых данных и вы ожидаете, что они будут отправлены по электронной почте, добавьте их в предыдущее правило (предупреждать пользователей).

Чтобы создать правило транспорта почты, просмотреть [короткое](https://support.microsoft.com/office/a9ecca03-42a6-4867-b9fd-38e3f6bb06ad)обучающее видео или выполнить следующие действия:

1. Перейдите в [Центр администрирования Exchange](https://go.microsoft.com/fwlink/p/?linkid=2059104).

2. В категории **потока почты** выберите **правила.**

3. Выберите **+** и **создайте новое правило.**

4. Выберите **** в нижней части диалоговых окна, чтобы увидеть полный набор параметров.

6. Нажмите кнопку **Сохранить**.

## <a name="6-stop-auto-forwarding-for-email"></a>6. Остановка автоматической переад вперед для электронной почты
<a name="forwarding"> </a>

Злоумышленники, которые получают доступ к почтовому ящику пользователя, могут похитить почту, настроив в этом почтовом ящике автоматический пересылаемый почтовый ящик. Это может произойти даже без информирования пользователя. Это можно предотвратить, настроив правило потока почты.

Создание правила транспорта почты:

1. Перейдите в [Центр администрирования Exchange](https://go.microsoft.com/fwlink/p/?linkid=2059104).

2. В категории **потока почты** выберите **правила.**

3. Выберите **+** и **создайте новое правило.**

4. Выберите **дополнительные параметры** в нижней части диалоговых окна, чтобы увидеть полный набор параметров.

5. Применив параметры в следующей таблице. Оставьте остальные параметры по умолчанию, если вы не хотите их изменить.

6. Нажмите кнопку **Сохранить**.

|---|---|

## <a name="7-use-office-message-encryption"></a>7. Использование шифрования сообщений Office
<a name="encryption"> </a>

Шифрование сообщений Office входит в состав Microsoft 365. Она уже настроена. С помощью шифрования сообщений Office ваша организация может отправлять и получать зашифрованные сообщения электронной почты между людьми внутри и за пределами организации. Шифрование сообщений Office 365 работает с Outlook.com, Yahoo!, Gmail и другими службами электронной почты. Шифрование сообщений электронной почты позволяет гарантировать, что только получатели смогут просматривать содержимое сообщения.

Шифрование сообщений Office предоставляет два варианта защиты при отправке почты:

- Не переад вперед

- Шифрование

Возможно, в вашей организации настроены дополнительные параметры, которые применяют метку к электронной почте, например "Конфиденциально".

### <a name="to-send-protected-email"></a>Отправка защищенной электронной почты

В Outlook для ПК выберите **"Параметры"** в сообщении электронной почты, а затем выберите **"Разрешения".**

![Шифрование сообщений электронной почты в Outlook](../../media/08e90a7e-a2d2-41a4-bae9-0a46b4ce639a.png)

В Outlook.com выберите **"Защитить"** в сообщении электронной почты. Защита по умолчанию — **"Не переад вперед".** Чтобы изменить этот код для шифрования, выберите **"Изменить разрешения для** \> **шифрования".**

![Шифрование сообщений электронной почты в Outlook.com](../../media/329ccf50-f6b1-4fb8-b249-60b907a82b7e.png)

### <a name="to-receive-encrypted-email"></a>Получение зашифрованной электронной почты

Если у получателя есть Outlook 2013 или Outlook 2016 и учетная запись электронной почты Майкрософт, в области чтения от него будет вывести предупреждение об ограниченных разрешениях элемента. После открытия сообщения получатель может просмотреть его так же, как и любой другой.

Если получатель использует другой почтовый клиент или учетную запись электронной почты, например Gmail или Yahoo, он увидит ссылку, которая позволяет ему войти для чтения сообщения электронной почты или запросить разовой пароль для просмотра сообщения в веб-браузере. Если пользователи не получают сообщение электронной почты, убедитесь, что они проверили свою папку нежелательной почты или нежелательной почты.

Дополнительные сведения см. в сведениях об отправке, просмотре и ответе на зашифрованные сообщения [в Outlook для ПК.](https://support.microsoft.com/office/eaa43495-9bbb-4fca-922a-df90dee51980)

Защита от фишинга , часть Microsoft Defender для Office 365, может помочь защитить вашу организацию от вредоносных фишинговых атак на основе фишинга на основе фишинга и других фишинговых атак. Если настраиваемый домен не настроен, это не требуется.

## <a name="9-protect-against-malicious-attachments-and-files-with-safe-attachments"></a>9. Защита от вредоносных вложений и файлов с помощью безопасных вложений

Люди регулярно отправляют, получают и делятся вложениями, такими как документы, презентации, электронные таблицы и другие. Не всегда легко определить, является ли вложение безопасным или вредоносным, просто посмотрев на сообщение электронной почты. Microsoft Defender для Office 365 включает защиту безопасных вложений, но эта защита не включена по умолчанию. Мы рекомендуем создать новое правило, чтобы начать использовать эту защиту. 3. На странице "Политика" выберите **"Безопасные вложения".**

4. На странице "Безопасные вложения" широко применив эту защиту, включив ATP для **SharePoint, OneDrive** и Microsoft Teams.

5. Выберите, **+** чтобы создать новую политику.

6. Применив параметры в следующей таблице.

7. После проверки параметров выберите "Создать **эту** политику" или "Сохранить".

## <a name="10-protect-against-phishing-attacks-with-safe-links"></a>10. Защита от фишинговых атак с помощью безопасных ссылок
<a name="phishingatp"> </a>

Злоумышленники иногда скрывают вредоносные веб-сайты по ссылкам в электронной почте или других файлах. Безопасные ссылки, которые являются частью Microsoft Defender для Office 365, могут помочь защитить организацию, предоставляя проверку url-адресов (URL-адресов) в электронных сообщениях и документах Office. Защита определяется с помощью политик безопасных ссылок.

Рекомендуется сделать следующее:

- Измените политику по умолчанию, чтобы повысить уровень защиты.

- Добавьте новую политику, ориентированную на всех получателей в вашем домене.

Чтобы добраться до "Безопасных ссылок", просмотреть [короткое](https://support.microsoft.com/office/61492713-53c2-47da-a6e7-fa97479e97fa)обучающее видео или выполнить следующие действия:

1. Войдите <https://protection.office.com> в учетную запись администратора и войдите в нее.

2. В Центре безопасности & соответствия требованиям в области навигации слева в **области** управления угрозами выберите **"Политика".**

3. На странице "Политика" выберите **"Безопасные ссылки".**

Изменение политики по умолчанию:

1. На странице "Безопасные ссылки" в области **"Политики",** которые применяются для всей организации, дважды щелкните политику **по** умолчанию.

2. В области "Параметры", которые применяются к содержимому **в Office 365,** введите URL-адрес для блокировки, _например_ example.com, и выберите **+** .

3. В параметрах, которые применяются к содержимому, кроме электронной почты, выберите приложения **Office 365,** не отслеживайте, когда пользователи щелкают безопасные ссылки, и не позволяйте пользователям перебирать безопасные ссылки на исходный **URL-адрес.**

4. Нажмите кнопку **Сохранить**.

Чтобы создать новую политику, ориентированную на всех получателей в вашем домене:

1. На странице "Безопасные ссылки" в области **"Политики",** которые применяются к определенным получателям, выберите **+** создание новой политики.

2. Применив параметры, перечисленные в следующей таблице.

3. Нажмите кнопку **Сохранить**.

|Параметр или параметр|Рекомендуемый параметр|
|---|---|
|Имя|Политика безопасных ссылок для всех получателей в домене|
|Выберите действие для неизвестных потенциально вредоносных URL-адресов в сообщениях|Выберите **"В сети" URL-адреса** будут перезаписаны и проверены на наличие списка известных вредоносных ссылок, когда пользователь щелкает ссылку.|
|Применение сканирования URL-адресов в режиме реального времени для подозрительных ссылок и ссылок, которые указывают на файлы|Выберите это поле.|
|Применяется к|Домен получателя: . . . выберите свой домен.|
|

Дополнительные сведения см. в [веб-сайте "Безопасные ссылки" в Microsoft Defender для Office 365.](https://docs.microsoft.com/microsoft-365/security/office-365-security/atp-safe-links)
