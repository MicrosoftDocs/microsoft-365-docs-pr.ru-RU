# <a name="add-another-email-alias-for-a-user"></a>Добавление дополнительных псевдонимов электронной почты для пользователя

::: moniker range="o365-21vianet"

> [!NOTE]
> Изменяется Центр администрирования. Если ваш интерфейс не соответствует приведенным здесь сведениям, см. раздел [О новом Центре администрирования Microsoft 365](https://docs.microsoft.com/microsoft-365/admin/microsoft-365-admin-center-preview?view=o365-21vianet&preserve-view=true).

::: moniker-end
  
Эта статья для администраторов Microsoft 365 с бизнес-подписками. Она не предназначена для домашних пользователей.
  
Основной адрес электронной почты в Microsoft 365 обычно это адрес электронной почты, который пользователь был назначен при его учетной записи. Когда пользователь отправляет сообщения другим людям, именно этот адрес, как правило, указывается в поле  *От*  . У них также может быть несколько адресов электронной почты, связанных с их учетной записью Microsoft 365 для бизнеса. Дополнительные адреса называются псевдонимами. 
  
Например, предположим, что у Йенны есть адрес электронной почты jenna@contosoco.com, но она также хочет получать электронную почту по jen@contosoco.com так как некоторые люди ссылаются на нее по этому имени. Вы можете создать псевдонимы для нее, чтобы оба адреса электронной почты были отправлены в почтовый ящик Ольга.
<br><br>  
  
Для одного пользователя можно создать до 400 псевдонимов. Дополнительные расходы и лицензии не требуются.
  
> [!Tip]
> Если вы хотите, чтобы несколько человек могли управлять электронной почтой, отправляемой на один адрес электронной почты, например info@NodPublishers.com или sales@NodPublishers.com, создайте общий почтовый ящик. Дополнительные узнать см. в под [этой теме.](create-a-shared-mailbox.md)
  
## <a name="add-email-aliases-to-a-user"></a>Добавление псевдонимов электронной почты
<a name="AddEmailPreview"> </a>

Для этого у вас [должны быть](../add-users/about-admin-roles.md) разрешения администратора. 

  
::: moniker range="o365-worldwide"

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. На странице **"Активные пользователи"** выберите пользователя, > **управление псевдонимами электронной почты.** Этот параметр не будет отозвался, если ему не назначена лицензия. 
    
3. Выберите **+Добавить псевдоним и** введите новый псевдоним для пользователя.   
    
    > [!Important] 
    > Если вы получите сообщение об ошибке "Не удается найти параметр, который соответствует имени параметра **"EmailAddresses",** это означает, что настройка клиента или пользовательского домена, если вы недавно добавили его, проходит немного больше времени. На настройку может потребоваться до 4 часов. Дождитесь ее окончания, а затем повторите попытку. Если проблема сохранится, обратитесь в службу поддержки, и ее специалисты выполнят для вас полную синхронизацию.
    
  
    > [!IMPORTANT]
    > если вы приобрели подписку у GoDaddy или другого партнера, для настройки нового псевдонима в качестве основного следует использовать консоль управления GoDaddy или партнера. 
  
    > [!TIP]
    > Псевдоним электронной почты должен заканчиваются доменом из выпадаемого списка. Чтобы добавить в список другое доменное имя, см. запись ["Добавление домена в Microsoft 365".](https://docs.microsoft.com/microsoft-365/admin/setup/add-domain) 
  
     
5. После этого выберите "Сохранить **изменения".**
    
6. Подождите 24 часа, пока новые псевдонимы не будут заполнены в Microsoft 365.
    
    Теперь у пользователя будет основной адрес и псевдоним. Например, вся почта, отправленная на основной адрес Ольги Хоффман, Eliza@NodPublishers.com и ее псевдоним Sales@NodPublishers.com, будут перенаправляться в почтовый ящик Ольги.
    
  
7. **Когда пользователь отвечает, *основным*  псевдонимом электронной почты будет адрес "От".** Например, предположим, что сообщение отправляется Sales@NodPublishers.com и поступает в почтовый ящик Елизы. Когда Елиза отвечает на сообщение, ее основной адрес электронной почты будет отображаться как отправитель, а не Sales@NodPublishers.com. 
    
::: moniker-end

## <a name="did-you-get-a-parameter-cannot-be-found-that-matches-parameter-name-emailaddresses"></a>Вы получите сообщение "Не удается найти параметр, который соответствует имени параметра EmailAddresses"?


Если вы получите сообщение об ошибке "Не удается найти параметр, который соответствует имени параметра **EmailAddresses",** это означает, что настройка клиента или пользовательского домена, если вы недавно добавили его, проходит немного больше времени. На настройку может потребоваться до 4 часов. Дождитесь ее окончания, а затем повторите попытку. Если проблема сохранится, обратитесь в службу поддержки, и ее специалисты выполнят для вас полную синхронизацию.
  
## <a name="did-you-purchase-your-subscription-from-godaddy-or-another-partner"></a>Вы приобрели подписку у GoDaddy или другого партнера?


Если вы приобрели подписку у GoDaddy или другого партнера, для настройки нового псевдонима в качестве основного следует использовать консоль управления GoDaddy или партнера.
