# <a name="about-shared-mailboxes"></a>Сведения об общих почтовых ящиках

Общие почтовые ящики используются, если нескольким людям необходим доступ к одному и тому же почтовому ящику, например к электронному адресу службы поддержки, почтовому ящику со сведениями о компании, столу регистрации или к другой функции, которую совместно используют несколько человек.

Пользователи с разрешениями для почтового ящика группы "Отправить как" или "Отправить от имени" могут отправлять сообщения с электронного адреса почтового ящика, если это разрешено администратором. Эта возможность особенно полезна для почтовых ящиков службы справки и поддержки, так как пользователи могут отправлять письма из таких групп, как "Служба поддержки Contoso" или "Создание стола регистрации".

## <a name="before-you-begin"></a>Прежде чем начать

Перед [созданием общего почтового ящика](create-a-shared-mailbox.md)вы должны знать несколько вещей:

- **Лицензии:** Общий почтовый ящик может хранить до 50 ГБ данных без назначения ему лицензии. Для хранения большего объема почтовому ящику нужно назначить лицензию. Дополнительные сведения о лицензировании общих почтовых ящиков см. в Exchange Online [Ограничения.](/office365/servicedescriptions/exchange-online-service-description/exchange-online-limits#StorageLimits) Когда размер общего почтового ящика достигнет предела, в течение некоторого времени вы сможете получать электронную почту, но не сможете отправлять новые сообщения. Затем вы перестанете получать электронную почту. Пользователи, отправившие сообщения в этот почтовый ящик, будут получать уведомление о недоставке.

- **Разрешения пользователей:** Необходимо предоставить пользователям разрешения (членство) для использования общего почтового ящика. Использовать общий почтовый ящик могут только пользователи в пределах организации.

- **Внешние пользователи:** Вы не можете предоставить доступ к общему почтовому ящику для людей, не входящих в ваш бизнес (например, людей с учетной записью Gmail). Если вы хотите сделать это, подумайте о создании группы для Outlook вместо этого. Дополнительные дополнительные возможности см. в [Microsoft 365 группы в центре администрирования.](../create-groups/create-groups.md)

- **Используйте с Outlook:** Помимо использования Outlook в Интернете браузера для доступа к общим почтовым ящикам, вы также можете использовать Outlook для приложения iOS или Outlook для android-приложения. Дополнительные дополнительные сообщения см. в раздел Добавление общего почтового ящика [в Outlook мобильный телефон.](https://support.microsoft.com/office/f866242c-81b2-472e-8776-6c49c5473c9f) Другим вариантом является создание группы для общего почтового ящика. Дополнительные дополнительные данные см. в [см. в "Сравнить группы".](../create-groups/compare-groups.md)

- **Шифрование:** Нельзя шифровать электронную почту, отправленную из общего почтового ящика. Это происходит из-за того, что общий почтовый ящик не имеет собственного контекста безопасности (имя пользователя/пароль), поэтому ему не может быть назначен ключ. Если несколько человек являются участниками, и они отправляют или получают зашифрованные ими электронные письма с помощью собственных ключей, другие пользователи могут прочитать электронную почту, а другие — нет, в зависимости от того, с какой общедоступный ключ была зашифрована электронная почта.

- **Преобразование почтовых ящиков:** Можно преобразовать почтовые ящики пользователей в общие почтовые ящики. См. [раздел Преобразование почтового ящика пользователя в общий почтовый ящик.](convert-user-mailbox-to-shared-mailbox.md)

- **Роли администратора:** Пользователи с глобальным администратором или Exchange ролей администратора могут создавать общие почтовые ящики.

- **Требования к подписке:** Чтобы создать общий почтовый ящик, необходимо подписаться на Microsoft 365 бизнес-плана, который включает электронную почту (Exchange Online службу). Подписка Приложения Microsoft 365 для бизнеса не включает электронную почту. Microsoft 365 бизнес стандарт включает электронную почту.

- **Вход в:** Общий почтовый ящик не предназначен для прямого входного сообщения в связанной учетной записи пользователя. Всегда следует заблокировать вход для учетной записи общего почтового ящика и заблокировать ее.

- **Слишком много пользователей:** Если слишком много назначенных пользователей одновременно получает доступ к общему почтовому ящику, они могут периодически не подключаться к этому почтовому ящику. В этом случае можно уменьшить число пользователей или использовать другую нагрузку, такую Microsoft 365 группу или папку Public.

- **Удаление сообщений:** К сожалению, нельзя запретить людям удалять сообщения в общем почтовом ящике. Единственный способ обойти это — создать группу Microsoft 365 вместо общего почтового ящика. Группа в Outlook как общий почтовый ящик. Для сравнения этих двух групп см. [сравнительные группы.](../create-groups/compare-groups.md) Дополнительные информацию о группах см. [в дополнительных подробной информации о группах.](https://support.microsoft.com/office/b565caa1-5c40-40ef-9915-60fdb2d97fa2)


> [!NOTE]
> Чтобы получить доступ к общему почтовому ящику, у пользователя должна быть Exchange Online лицензия, но для общего почтового ящика не требуется отдельная лицензия. У каждого общего почтового ящика есть соответствующая учетная запись пользователя. Вы заметили, что вам не потребовалось указывать пароль при создании общего почтового ящика? У этой учетной записи есть пароль, но он создается системой (неизвестен). Не следует использовать учетную запись для входа в общий почтовый ящик. Однако без лицензии общие почтовые ящики ограничены размером 50 ГБ.  Чтобы увеличить максимальный размер до 100 ГБ, необходимо назначить общему почтовому ящику лицензию Exchange Online (план 2) или лицензию Exchange Online (план 1) с дополнительной лицензией Exchange Online Archiving. Это также позволит включить архивы с автоматическим расширением для неограниченного объема архивного хранилища. Чтобы наложить на почтовый ящик общего доступа удержание для судебного разбирательства, необходимо назначить этому общему почтовому ящику лицензию Exchange Online (план 2) или лицензию Exchange Online (план 1) с дополнительной лицензией Exchange Online Archiving. Если вы хотите применить расширенные функции, такие как Microsoft Defender для Office 365, Advanced eDiscovery или политики автоматического хранения, общий почтовый ящик должен быть лицензирован для этих функций.
