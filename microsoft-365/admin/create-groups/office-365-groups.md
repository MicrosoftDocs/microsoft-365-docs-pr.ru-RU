# <a name="overview-of-microsoft-365-groups-for-administrators"></a>Обзор групп Microsoft 365 для администраторов

Группы Microsoft 365 — это служба, которая является основой для командной работы в Microsoft 365. С помощью групп Microsoft 365 вы можете предоставить группе пользователей доступ к коллекции общих ресурсов. К этим ресурсам относятся:

- Общая почта Outlook
- Общий календарь
- Библиотека документов SharePoint
- Планировщик
- Записная книжка OneNote
- Power BI
- Yammer (если группа была создана из Yammer)
- Команда (если группа была создана из Teams)
- Roadmap (if you have Project for the web)

В группе Microsoft 365 не нужно вручную назначать разрешения каждому из этих ресурсов. При добавлении людей в группу они автоматически получают необходимые разрешения.

Любой пользователь может создать группу, если только создание группы не ограничивается определенным [набором людей.](manage-creation-of-groups.md) Если ограничить создание групп, пользователи, которые не смогут создавать группы, не смогут создавать сайты SharePoint, планировщики или команды. Эти службы требуют, чтобы люди, создав их, могли создать группу. Пользователи по-прежнему могут участвовать в групповых действиях, таких как создание задач в Планировщике или использование чата Teams, если они являются участниками группы.

Группы имеют следующие роли:

- **Владельцы** — владельцы группы могут добавлять или удалять участников и иметь уникальные разрешения, например возможность удалять беседы из общей почтовой ящики или изменять различные параметры группы. Владельцы группы могут переименовать группу, обновить описание или рисунок и другие.
- **Участники** — участники могут получать доступ ко всем данным в группе, но не могут изменять параметры группы. По умолчанию участники группы могут приглашать гостей присоединиться к группе, но вы можете [управлять этим параметром.](manage-guest-access-in-groups.md)
- **Гости —** гости группы — это участники, которые находятся за пределами вашей организации.

Создавать группы и управлять ими в Центре администрирования Microsoft 365 могут только глобальные администраторы, администраторы пользователей и администраторы групп. Вы не можете быть полномочным администратором (например, консультантом, действующим от имени администратора).

Администратор может:

- [Указание пользователей, которые могут создавать группы](manage-creation-of-groups.md)
- [Создание политики именования для групп в организации](groups-naming-policy.md)
- [Выбор домена, который будет применяться при создании группы](choose-domain-to-create-groups.md)
- [Управление гостевым доступом к группам](manage-guest-access-in-groups.md)
- [Восстановление удаленной группы](restore-deleted-group.md) (в течение 30 дней после удаления)

Если вы предпочитаете более автоматизированный способ управления жизненным циклом групп Microsoft 365, вы можете использовать политики истечения срока действия групп с определенным интервалом времени. Владельцы группы получат сообщение электронной почты за 30, 15 и 1 день до истечения срока действия группы, которое позволит им обновить группу, если она по-прежнему необходима. См. политику срока действия группы [Microsoft 365.](office-365-groups-expiration-policy.md)

Вы можете администрировать группы в Центре администрирования Microsoft 365 или [с помощью PowerShell.](https://docs.microsoft.com/microsoft-365/enterprise/manage-microsoft-365-groups-with-powershell)

Если у вас много пользователей, например в крупной корпорации или организации, может быть много пользователей, которые создают группы для различных целей. Мы настоятельно рекомендуем иным образом просмотреть рекомендации по планированию управления в группах [Microsoft 365.](plan-for-groups-governance.md)

## <a name="group-limits"></a>Ограничения для групп

К группам Microsoft 365 применяются следующие ограничения:

|Максимум...|Значение|
|:---------|:----|
|Owners per group|100|
|Groups a user can create|250|
|Группы, которые может создать администратор|Не более 500 K для клиента по умолчанию|
|Количество участников|Более 1000, хотя только 1000 могут получать доступ к групповым беседам одновременно. <br>Пользователи могут заметить задержки при доступе к календарю и беседам в больших группах в Outlook.|
|Количество групп, участником которые может быть пользователь|7,000|
|Хранилище файлов|1 терабайт + 10 ГБ на подписанного пользователя и любое другое приобретенное хранилище. Вы можете приобрести неограниченное количество дополнительного хранилища.|
|Размер почтового ящика группы|50 ГБ|

Максимальное число групп Microsoft 365 по умолчанию в организации составляет 500 000. Чтобы выйти за пределы ограничения по умолчанию, обратитесь в службу поддержки Майкрософт. Дополнительные сведения об ограничениях для групп Microsoft 365 см. в справке для администраторов По группам [Microsoft 365.](https://support.microsoft.com/office/b565caa1-5c40-40ef-9915-60fdb2d97fa2)

Управление группами Microsoft 365 более эффективно при использовании групп с действиями. Центр администрирования Microsoft 365 имеет средство создания отчетов, которое позволяет вам видеть использование хранилища, количество активных групп и то, как пользователи используют эти группы. Дополнительные сведения см. в отчетах [Microsoft 365](../activity-reports/office-365-groups.md) в Центре администрирования.

## <a name="sensitivity-labels"></a>Метки конфиденциальности

Вы можете создавать метки конфиденциальности, которые пользователи в вашей организации могут устанавливать при создании группы Microsoft 365. С помощью меток конфиденциальности можно настроить: 

- Конфиденциальность (частная или частная)
- Доступ внешних пользователей
- Неугодный доступ к устройствам

Например, можно создать метку  "Строго конфиденциально" и указать, что любая группа, созданная с этой меткой, будет закрытой и не разрешит внешним пользователям. Когда пользователи в организации выбирают эту метку во время создания группы, группе будет назначено закрытое, а участникам группы будет запрещено добавлять внешних пользователей в группу.

> [!IMPORTANT]
> Если в настоящее время вы используете метки классификации, они больше не будут доступны для пользователей, создавших группы после включения меток конфиденциальности. 

Сведения о создании, управлении и использовании меток конфиденциальности см. в под названием "Использование меток конфиденциальности для защиты контента в Microsoft Teams, группах [Microsoft 365](https://docs.microsoft.com/microsoft-365/compliance/sensitivity-labels-teams-groups-sites)и на сайтах SharePoint".

## <a name="which-microsoft-365-plans-include-groups"></a>Какие планы Microsoft 365 включают группы?

Любая подписка на Microsoft 365 с Exchange Online и SharePoint Online будет поддерживать группы. К ним относятся планы Business Essentials и Business Premium, а также корпоративные планы E1, E3 и E5. Группа берет на себя лицензирование человека, создавшего группу (также известного как "организатор" группы). Если у организатора есть соответствующая лицензия на любые функции, которые вы хотите иметь в группе, эта лицензия будет передана группе.

> [!NOTE]
> Дополнительные сведения о семействах служб и планах Microsoft 365 см. в параметрах плана [Microsoft 365.](https://docs.microsoft.com/office365/servicedescriptions/office-365-platform-service-description/office-365-plan-options)

Если у вас есть план только для Exchange, вы по-прежнему можете получить общие функции почтового ящика и общего календаря групп в Outlook, но не получите библиотеку документов, Планировщик или другие возможности.

Группы Microsoft 365 работают с Azure Active Directory. Получаемые функции групп зависят от того, какая у вас подписка на Azure Active Directory и какие лицензии назначены организатору группы.

> [!IMPORTANT]
> Для всех функций групп, если у вас есть подписка на Azure AD Premium, пользователи могут присоединиться к группе независимо от того, назначена ли им лицензия AAD P1. Лицензирование не выполняется принудительно.
> Периодически создаются отчеты об использовании, которые сообщают, какие пользователи не имеют лицензии, и им нужна назначенная лицензия, чтобы они соответствовали требованиям лицензирования. Например, предположим, что у пользователя нет лицензии и он добавляется в группу, в которой применена политика именования. В отчете будет помечаться, что им нужна лицензия.
