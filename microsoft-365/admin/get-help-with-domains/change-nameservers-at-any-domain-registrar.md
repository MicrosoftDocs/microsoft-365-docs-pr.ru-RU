# <a name="change-nameservers-to-set-up-microsoft-365-with-any-domain-registrar"></a>Изменение именных имен для Microsoft 365 с любым регистратором домена

 Если вы не нашли то, что вы ищете, см. раздел **[Вопросы и ответы по доменам](../setup/domains-faq.yml)**. 
  
Следуйте этим инструкциям, чтобы добавить и настроить домен в Microsoft 365, чтобы ваши службы, такие как электронная почта и Teams, использовали собственное доменное имя. Для этого необходимо проверить домен, а затем изменить имена домена на Microsoft 365, чтобы можно было настроить правильные записи DNS. Выполните следующие действия, если в следующих утверждениях описана ваша ситуация:
  
- У вас есть собственный домен и вы хотите настроить его для работы с Microsoft 365.
    
- Вы хотите Microsoft 365 управлять записями DNS для вас. При желании вы можете [управлять своими записями DNS самостоятельно](../setup/add-domain.md).
    
## <a name="add-a-txt-or-mx-record-for-verification"></a>Добавьте запись TXT или MX для проверки
<a name="BKMK_verify"> </a>

> [!NOTE]
> Нужно создать только одну из этих записей. TXT является предпочтительным типом, но некоторые поставщики услуг размещения DNS не поддерживают его. В таком случае можно создать MX-запись. 
  
Прежде чем вы сможете использовать свой домен в Microsoft 365, мы должны убедиться в том, что вы являетесь его владельцем. Если вы войдете в свою учетную запись на сайте регистратора доменных имен и создадите запись DNS, это послужит для Microsoft 365 подтверждением того, что вы владеете доменом.
  
> [!NOTE]
> Эта запись используется исключительно для проверки принадлежности домена. При желании вы сможете удалить ее позже. 
  
### <a name="find-the-area-on-your-dns-hosting-providers-website-where-you-can-create-a-new-record"></a>Найдите область на веб-сайте поставщика хостинга DNS, в которой можно создать новую запись

1. Войдите на сайт поставщика услуг размещения DNS.
    
2.  Выберите свой домен.
    
3. Найдите страницу, на которой можно редактировать DNS-записи для домена.
    
### <a name="create-the-record"></a>Создание записи

При создании записи TXT или MX следуйте соответствующим инструкциям.
  
**Для создания TXT-записи используйте следующие значения:**
    
|||||
|:-----|:-----|:-----|:-----|
|**Record Type** (Тип записи) <br/> |**Alias** (Псевдоним) или **Host Name** (Имя узла) <br/> |**Value** (Значение) <br/> |**TTL** (Срок жизни) <br/> |
|TXT  <br/> |Выполните одно из следующих действий: Введите **@**, оставьте поле пустым или укажите имя своего домена.  <br/> > [!NOTE]> У различных узлов DNS могут быть разные требования к этому полю.           
|MS=ms *XXXXXXXX*  <br/> > [!NOTE]> Это пример. Используйте здесь собственное значение **Назначение или адрес "указывает на"** из таблицы в Microsoft 365.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)          |Задайте для этого значения **1 hour** (1 час) или эквивалент в минутах ( **60** ), секундах ( **3600** ) и т. д.  <br/> |
   
**Для создания MX-записи используйте следующие значения:**
    
||||||
|:-----|:-----|:-----|:-----|:-----|
|**Record Type** (Тип записи)|**Alias** (Псевдоним) или **Host Name** (Имя узла)|**Value** (Значение)|**Priority** (Приоритет)|**TTL** (Срок жизни)|
|MX|Введите знак **@** или имя своего домена. |MS=ms *XXXXXXXX* > [!NOTE]> Это пример. Используйте здесь собственное значение **Назначение или адрес "указывает на"** из таблицы в Microsoft 365.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)          |В поле **Priority** (Приоритет) используйте более низкое значение приоритета, чем указанное для других записей MX, во избежание конфликтов с записью MX, используемой для потока обработки почты. Дополнительные сведения о приоритете см. в статье [Что такое приоритет записей MX?](../setup/domains-faq.yml). |Задайте для этого значения **1 hour** (1 час) или эквивалент в минутах ( **60** ), секундах ( **3600** ) и т. д. |
   
### <a name="save-the-record"></a>Сохранение записи

Теперь, когда запись добавлена на сайте регистратора доменных имен, вернитесь в Microsoft 365 и подайте запрос на ее поиск.
  
Если Microsoft 365 обнаружит правильную запись TXT, это значит, что домен проверен.
  

1. В Центре администрирования перейдите на страницу **Settings** (Параметры) \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Domains</a> (Домены).
    
2. На странице **Domains** (Домены) выберите домен, который нужно проверить. 
    
  
3. На странице **Setup** (Настройка) выберите **Start setup** (Начать настройку).
 
    
  
4. На странице **Проверка домена** выберите **Проверить**.
    
    
  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="change-your-domains-nameserver-ns-records"></a>Изменение записей сервера доменных имен
<a name="BKMK_nameservers"> </a>

Когда вы доберется до последнего шага мастера настройки доменов в Microsoft 365, у вас остается одна задача. Чтобы настроить домен с Microsoft 365 службами, например электронной почтой, вы измените записи имен домена (или NS) в регистраторе домена, чтобы указать на Microsoft 365 и вторичные имена. Затем, Microsoft 365 в котором размещена DNS, для вас автоматически устанавливаются необходимые записи DNS для ваших служб. Вы можете самостоятельно обновлять записи сервера доменных имен, следуя инструкциям в разделе справки на веб-сайте регистратора доменных имен. Если вы еще не знакомы с DNS, обратитесь в службу поддержки регистратора доменных имен.

::: moniker range="o365-worldwide"
  
Чтобы самостоятельно изменить серверы доменных имен на веб-сайте регистратора доменных имен, выполните указанные ниже действия.
  
1. Найдите область на веб-сайте регистратора домена, в которой можно изменить тезки для домена или области, в которой можно использовать настраиваемые именные имена.
    
2. Создайте записи nameserver или отредактировать существующие записи nameserver, чтобы соответствовать следующим значениям:
    
|||
|:-----|:-----|
|Основной сервер доменных имен  <br/> |ns1.bdm.microsoftonline.com  <br/> |
|Второстепенный сервер доменных имен  <br/> |ns2.bdm.microsoftonline.com  <br/> |
|Третий сервер доменных имен  <br/> |ns3.bdm.microsoftonline.com  <br/> |
|Четвертый сервер доменных имен  <br/> |ns4.bdm.microsoftonline.com  <br/> |
   
   > [!TIP]
   > Лучше всего добавить все четыре записи, но если регистратор поддерживает только **две,** добавьте ns1.bdm.microsoftonline.com **и ns2.bdm.microsoftonline.com.** 
  
3. Сохраните изменения.
    
> [!CAUTION]
> При изменении записей NS домена, указываю на Microsoft 365 имен, все службы, которые в настоящее время связаны с доменом, затронуты. Если вы пропустили какие-либо этапы работы мастера, например добавление адресов электронной почты, или используете свой домен для размещения блогов, корзин для покупок или других служб, необходимо выполнить дополнительные действия. В противном случае это изменение может привести к простою служб, например к отсутствию доступа к электронной почте или веб-сайту. 

::: moniker-end

::: moniker range="o365-21vianet"
  
1. Найдите на веб-сайте регистратора домена область, в которой можно изменить серверы доменных имен для вашего домена.
    
2. Создайте две записи сервера доменных имен или измените существующие так, чтобы они соответствовали указанным ниже значениям.
    
|||
|:-----|:-----|
|Основной сервер доменных имен  <br/> |ns1.dns.partner.microsoftonline.cn  <br/> |
|Второстепенный сервер доменных имен  <br/> |ns2.dns.partner.microsoftonline.cn  <br/> |
   
   > [!TIP]
   > Необходимо использовать по крайней мере две записи nameserver. Если в списке указаны другие имена, их можно либо  удалить, либо изменить на ns3.dns.partner.microsoftonline.cn **и ns4.dns.partner.microsoftonline.cn.** 
  
3. Сохраните изменения.
    
> [!CAUTION]
> При изменении записей NS вашего домена, чтобы указать на Office 365, выполняемые 21 nameserversVianet, все службы, которые в настоящее время связаны с вашим доменом, затронуты. Если вы пропустили какие-либо этапы работы мастера, например добавление адресов электронной почты, или используете свой домен для размещения блогов, корзин для покупок или других служб, необходимо выполнить дополнительные действия. В противном случае это изменение может привести к простою служб, например к отсутствию доступа к электронной почте или веб-сайту. 

::: moniker-end
  
Например, для размещения электронной почты и веб-сайта может понадобиться выполнить дополнительные действия.
  
- Перед изменением записей NS переместите все адреса электронной почты, Microsoft 365 домена.
    
- Хотите добавить домен, который в настоящее время используется с веб-адресом, например www.fourthcoffee.com? При добавлении домена можно сделать следующие действия, чтобы сохранить веб-сайт, на котором сейчас находится сайт, чтобы люди могли по-прежнему получать доступ к веб-сайту после изменения записей NS домена, чтобы указать на Microsoft 365.

1. В Центре администрирования перейдите на страницу **Settings** (Параметры) \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Domains</a> (Домены).

2. На странице **Домены** выберите домен.

3. На странице сведения о домене выберите вкладку **записей DNS.**
 
4. Выберите **запись Добавить**.

5. В **пользовательской области записи DNS**  из списка выпаданий типа выберите **A (Address).**

6. В поле **Имя хозяина или псевдоним введите** **@** .

7. В поле **IP-адрес** введите статичный IP-адрес веб-сайта, на котором он в настоящее время находится. Например, 172.16.140.1.
    
> [!IMPORTANT]
>  Это должен быть _статичный_ IP-адрес для веб-сайта, а не _динамический_ IP-адрес. Чтобы убедиться, что вы можете получить статичный IP-адрес для общедоступных веб-сайтов, ознакомьтесь с сайтом, на котором размещен ваш веб-сайт.
   
8. Если вы хотите изменить параметр TTL для записи, выберите новое время из списка **отсевов TTL.** В противном случае продолжайте шаг 9.
    
9. Нажмите кнопку **Сохранить**. 
    
Кроме того, можно создать запись CNAME, чтобы упростить поиск веб-сайта.
  
1.  Выберите **запись Добавить**.

3.  В **пользовательской области записи DNS**  из списка выпаданий типа выберите **CNAME (Alias).**
4.  В поле **Имя хозяина или псевдоним** введите **www**.
5.  В поле **Пункты для адреса** введите полное доменное имя (FQDN) для вашего веб-сайта. Например, **contoso.com**.
6.  Если вы хотите изменить параметр TTL для записи, выберите новое время из списка **отсевов TTL.** В противном случае продолжайте шаг 6.
7.  Нажмите **Сохранить**.

После обновления записей имен, чтобы указать на Microsoft, настройка домена завершена. Электронная почта отправлена в Корпорацию Майкрософт, и трафик на ваш веб-сайт продолжает переходить на текущий веб-сайт."
    
> [!NOTE]
> Your nameserver record updates may take up to several hours to update across the Internet's DNS system. Затем ваша электронная почта Майкрософт и другие службы будут настроены на работу с доменом.
