# <a name="change-nameservers-to-set-up-microsoft-365-with-any-domain-registrar"></a>Изменение серверов доменных имен для настройки Microsoft 365 с помощью любого регистратора доменных имен

 Если вы не нашли то, что вы ищете, см. раздел **[Вопросы и ответы по доменам](../setup/domains-faq.md)**. 
  
Сначала проверьте [настройку домена (инструкции для конкретных узлов)](../get-help-with-domains/set-up-your-domain-host-specific-instructions.md) , чтобы узнать, есть ли у вас инструкции для регистратора. 
  
Следуйте этим инструкциям, чтобы добавить и настроить свой домен в Microsoft 365, чтобы ваши службы, такие как электронная почта и Teams, использовали собственное доменное имя. Для этого необходимо проверить домен, а затем изменить серверов доменных имен домена на Microsoft 365, чтобы можно было настроить соответствующие записи DNS. Выполните следующие действия, если следующие инструкции описывают ситуацию:
  
- У вас есть свой домен, и вы хотите настроить его для работы с Microsoft 365.
    
- Вы хотите, чтобы Microsoft 365 управляла своими записями DNS. При желании вы можете [управлять своими записями DNS самостоятельно](../setup/add-domain.md).
    
## <a name="add-a-txt-or-mx-record-for-verification"></a>Добавьте запись TXT или MX для проверки
<a name="BKMK_verify"> </a>

> [!NOTE]
> Нужно создать только одну из этих записей. TXT является предпочтительным типом, но некоторые поставщики услуг размещения DNS не поддерживают его. В таком случае можно создать MX-запись. 
  
Прежде чем вы сможете использовать свой домен в Microsoft 365, мы должны убедиться в том, что вы являетесь его владельцем. Если вы войдете в свою учетную запись на сайте регистратора доменных имен и создадите запись DNS, это послужит для Microsoft 365 подтверждением того, что вы владеете доменом.
  
> [!NOTE]
> Эта запись используется исключительно для проверки принадлежности домена. При желании вы сможете удалить ее позже. 
  
### <a name="find-the-area-on-your-dns-hosting-providers-website-where-you-can-create-a-new-record"></a>Найдите область на веб-сайте поставщика услуг размещения DNS, на которой можно создать новую запись.

1. Войдите на сайт поставщика услуг размещения DNS.
    
2.  Выберите свой домен.
    
3. Найдите страницу, на которой можно редактировать DNS-записи для домена.
    
### <a name="create-the-record"></a>Создание записи

При создании записи TXT или MX следуйте соответствующим инструкциям.
  
**Для создания TXT-записи используйте следующие значения:**
    
|||||
|:-----|:-----|:-----|:-----|
|**Record Type** (Тип записи) <br/> |**Alias** (Псевдоним) или **Host Name** (Имя узла) <br/> |**Value** (Значение) <br/> |**TTL** (Срок жизни) <br/> |
|TXT  <br/> |Выполните одно из следующих действий: Введите **@**, оставьте поле пустым или укажите имя своего домена.  <br/> > [!NOTE]> У различных узлов DNS могут быть разные требования к этому полю.           
|MS=ms *XXXXXXXX*  <br/> > [!NOTE]> Это пример. Используйте здесь собственное значение **Назначение или адрес "указывает на"** из таблицы в Microsoft 365.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)          |Задайте для этого значения **1 hour** (1 час) или эквивалент в минутах ( **60** ), секундах ( **3600** ) и т. д.  <br/> |
   
**Для создания MX-записи используйте следующие значения:**
    
||||||
|:-----|:-----|:-----|:-----|:-----|
|**Record Type** (Тип записи)|**Alias** (Псевдоним) или **Host Name** (Имя узла)|**Value** (Значение)|**Priority** (Приоритет)|**TTL** (Срок жизни)|
|MX|Введите знак **@** или имя своего домена. |MS=ms *XXXXXXXX* > [!NOTE]> Это пример. Используйте здесь собственное значение **Назначение или адрес "указывает на"** из таблицы в Microsoft 365.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)          |В поле **Priority** (Приоритет) используйте более низкое значение приоритета, чем указанное для других записей MX, во избежание конфликтов с записью MX, используемой для потока обработки почты. Дополнительные сведения о приоритете см. в статье [Что такое приоритет записей MX?](../setup/domains-faq.md#what-is-mx-priority). |Задайте для этого значения **1 hour** (1 час) или эквивалент в минутах ( **60** ), секундах ( **3600** ) и т. д. |
   
### <a name="save-the-record"></a>Сохранение записи

Теперь, когда запись добавлена на сайте регистратора доменных имен, вернитесь в Microsoft 365 и подайте запрос на ее поиск.
  
Если Microsoft 365 обнаружит правильную запись TXT, это значит, что домен проверен.
  

1. В Центре администрирования перейдите на страницу **Settings** (Параметры) \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Domains</a> (Домены).
    
2. На странице **Domains** (Домены) выберите домен, который нужно проверить. 
    
  
3. На странице **Setup** (Настройка) выберите **Start setup** (Начать настройку).
 
    
  
4. На странице **Проверка домена** выберите **Проверить**.
    
    
  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="change-your-domains-nameserver-ns-records"></a>Изменение записей сервера доменных имен
<a name="BKMK_nameservers"> </a>

При переходе к последнему этапу работы мастера установки доменов в Microsoft 365 остается одна из задач. Чтобы настроить свой домен с помощью служб Microsoft 365, например электронной почты, измените записи сервера доменных имен (или записи NS) в вашем регистраторе доменных имен, чтобы они ссылались на основной и дополнительный серверов доменных имен Microsoft 365. После этого, так как в Microsoft 365 размещается DNS, необходимые записи DNS для служб автоматически настраиваются. Вы можете самостоятельно обновлять записи сервера доменных имен, следуя инструкциям в разделе справки на веб-сайте регистратора доменных имен. Если вы еще не знакомы с DNS, обратитесь в службу поддержки регистратора доменных имен.

::: moniker range="o365-worldwide"
  
Чтобы самостоятельно изменить серверы доменных имен на веб-сайте регистратора доменных имен, выполните указанные ниже действия.
  
1. Найдите область на веб-сайте регистратора доменных имен, на которой можно изменить серверов доменных имен для вашего домена или области, где можно использовать пользовательские серверов доменных имен.
    
2. Создайте записи сервера доменных имен или измените существующие записи серверов доменных имен, чтобы они соотнесены со следующими значениями:
    
|||
|:-----|:-----|
|Основной сервер доменных имен  <br/> |ns1.bdm.microsoftonline.com  <br/> |
|Второстепенный сервер доменных имен  <br/> |ns2.bdm.microsoftonline.com  <br/> |
|Третий сервер доменных имен  <br/> |ns3.bdm.microsoftonline.com  <br/> |
|Четвертый сервер доменных имен  <br/> |ns4.bdm.microsoftonline.com  <br/> |
   
   > [!TIP]
   > Рекомендуется добавить все четыре записи, но если регистратор поддерживает только два, добавьте **NS1.BDM.microsoftonline.com** и **NS2.BDM.microsoftonline.com**. 
  
3. Сохраните изменения.
    
> [!CAUTION]
> Когда вы изменяете записи NS домена так, чтобы они ссылались на Microsoft 365 серверов доменных имен, затрагиваются все службы, которые в настоящее время связаны с вашим доменом. Если вы пропустили какие-либо этапы работы мастера, например добавление адресов электронной почты, или используете свой домен для размещения блогов, корзин для покупок или других служб, необходимо выполнить дополнительные действия. В противном случае это изменение может привести к простою служб, например к отсутствию доступа к электронной почте или веб-сайту. 

::: moniker-end

::: moniker range="o365-21vianet"
  
1. Найдите на веб-сайте регистратора домена область, в которой можно изменить серверы доменных имен для вашего домена.
    
2. Создайте две записи сервера доменных имен или измените существующие так, чтобы они соответствовали указанным ниже значениям.
    
|||
|:-----|:-----|
|Основной сервер доменных имен  <br/> |ns1.dns.partner.microsoftonline.cn  <br/> |
|Второстепенный сервер доменных имен  <br/> |ns2.dns.partner.microsoftonline.cn  <br/> |
   
   > [!TIP]
   > Следует использовать по крайней мере две записи серверов доменных имен. Если в списке есть другие серверов доменных имен, их можно удалить или изменить на **NS3.DNS.Partner.microsoftonline.CN** и **NS4.DNS.Partner.microsoftonline.CN**. 
  
3. Сохраните изменения.
    
> [!CAUTION]
> Когда вы изменяете записи NS домена так, чтобы они ссылались на Office 365 под управлением 21Vianet серверов доменных имен, затрагиваются все службы, которые в настоящее время связаны с вашим доменом. Если вы пропустили какие-либо этапы работы мастера, например добавление адресов электронной почты, или используете свой домен для размещения блогов, корзин для покупок или других служб, необходимо выполнить дополнительные действия. В противном случае это изменение может привести к простою служб, например к отсутствию доступа к электронной почте или веб-сайту. 

::: moniker-end
  
Например, для размещения электронной почты и веб-сайта может понадобиться выполнить дополнительные действия.
  
- Перед изменением записей NS переместите все адреса электронной почты, которые используют ваш домен, в Microsoft 365.
    
- Хотите добавить домен, который в настоящее время используется с адресом веб-сайта, например www.fourthcoffee.com? Вы можете выполнить следующие действия, а затем добавить домен для размещения веб-сайта, на котором размещен сайт, чтобы пользователи по-прежнему могли перейти на веб-сайт после того, как вы измените записи NS домена так, чтобы они ссылались на Microsoft 365.

1. В Центре администрирования перейдите на страницу **Settings** (Параметры) \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Domains</a> (Домены).

2. На странице **домены** выберите домен, а затем — **записи DNS**.

3. В разделе **параметры DNS**выберите **Настраиваемые записи**, а затем нажмите **создать настраиваемую запись**.

4. Выберите тип записи DNS, которую вы хотите добавить, и введите сведения для новой записи.

5. Нажмите **Сохранить**.
    
> [!NOTE]
> Your nameserver record updates may take up to several hours to update across the Internet's DNS system. После этого ваша электронная почта Майкрософт и другие службы будут настроены для работы с вашим доменом.
