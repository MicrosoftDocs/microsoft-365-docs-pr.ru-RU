# <a name="create-dns-records-at-cloudflare-for-microsoft"></a>Создание записей DNS в Cloudflare для Microsoft

 Если вы не нашли то, что вы ищете, см. раздел **[Вопросы и ответы по доменам](../setup/domains-faq.yml)**. 
  
Если ваш поставщик услуг размещения DNS  Cloudflare, выполните действия, описанные в этой статье, чтобы подтвердить владение доменом и настроить записи DNS для электронной почты, Skype для бизнеса Online и других служб.
  
После добавления этих записей в Cloudflare домен будет настроен для работы с службами Microsoft 365.
  
  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="change-your-domains-nameserver-ns-records"></a>Изменение записей сервера доменных имен
<a name="BKMK_Nameservers"> </a>

> [!IMPORTANT]
> Эту процедуру необходимо выполнить на сайте регистратора доменных имен, на котором вы приобрели и зарегистрировали свой домен. 
  
При регистрации в Cloudflare вы добавили домен с помощью процесса **настройки**. 
  
Добавленный домен был приобретен в Cloudflare или отдельном регистраторе домена. Чтобы проверить и создать записи DNS для вашего домена в Microsoft 365, сначала необходимо изменить тезки в регистраторе домена, чтобы они использовали тезки Cloudflare.
  
Чтобы самостоятельно изменить серверы доменных имен на веб-сайте регистратора доменных имен, выполните следующее:
  
1. Найдите на веб-сайте регистратора домена область, в которой можно изменить серверы доменных имен для вашего домена.
    
2. Создайте две записи серверов доменных имен, используя значения из приведенной ниже таблицы, либо измените существующие записи серверов доменных имен, чтобы они соответствовали следующим значениям:
    
    |||
    |:-----|:-----|
    |Основной сервер доменных имен  <br/> |Используйте значение сервера доменных имен, предоставленное Cloudflare.  <br/> |
    |Второстепенный сервер доменных имен  <br/> |Используйте значение сервера доменных имен, предоставленное Cloudflare.  <br/> |
   
    > [!TIP]
    > You should use at least two name server records. Если в списке есть другие серверы имен, их следует удалить. 
  
3. Сохраните изменения.
    
> [!NOTE]
> На распространение изменений, внесенных вами в записи серверов имен, в системе DNS по всему Интернету может потребоваться несколько часов. Затем ваша электронная почта Майкрософт и другие службы будут настроены на работу с доменом. 
  
## <a name="add-a-txt-record-for-verification"></a>Добавление записи TXT для проверки
<a name="BKMK_verify"> </a>

Прежде чем вы сможете использовать свой домен при работе с продуктами корпорации Майкрософт, мы должны убедиться в том, что вы являетесь его владельцем. Если вы войдете в свою учетную запись на сайте регистратора доменных имен и создадите запись DNS, для корпорации Майкрософт это послужит подтверждением того, что вы владеете доменом.
  
> [!NOTE]
> Эта запись используется исключительно для проверки принадлежности домена. При желании вы сможете удалить ее позже. 
  
1. Чтобы приступить к работе, откройте страницу со своими доменами на сайте Cloudflare по [этой ссылке](https://www.cloudflare.com/a/login). Сначала вам потребуется выполнить вход.
  
2. На **домашней странице** выберите домен, который необходимо обновить. 
  
3. На странице **Обзор** для домена выберите **DNS.**

  
4. На странице **управления DNS** щелкните **Добавить запись,** а затем выберите значения из следующей таблицы. 
    
    | Тип | Имя | Automatic TTL | Содержимое |
    |:-----|:-----|:-----|:----|
    |TXT  <br/> |@  <br/> |30 minutes  <br/> |MS=ms *XXXXXXXX*  <br/> **Примечание.** Это пример. Используйте здесь свое конкретное значение **Назначение или адрес "Указывает на"** из этой таблицы.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)    |
  
    
5. Нажмите **Сохранить**.
  
  
9. Подождите несколько минут, пока созданная запись не будет обновлена в Интернете.
    
Теперь, когда вы добавили запись на сайте регистратора домена, вы будете возвращаться в Microsoft и искать запись.
  
Когда продукт корпорации Майкрософт обнаружит правильную запись TXT, ваш домен будет подтвержден.
  
1. В центре администрирования Майкрософт перейдите на страницу **Настройка** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Домены</a>.

    
2. На странице **Domains** (Домены) выберите домен, который нужно проверить. 
    
    
  
3. На странице **Setup** (Настройка) выберите **Start setup** (Начать настройку).
    
    
  
4. На странице **Проверка домена** выберите **Проверить**.
    
    
  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="add-an-mx-record-so-email-for-your-domain-will-come-to-microsoft"></a>Добавьте запись MX, чтобы сообщения электронной почты для вашего домена доставлялись в продукты корпорации Майкрософт.
<a name="BKMK_add_MX"> </a>

1. Чтобы приступить к работе, откройте страницу со своими доменами на сайте Cloudflare по [этой ссылке](https://www.cloudflare.com/a/login). Сначала вам потребуется выполнить вход.
  
2. На **домашней странице** выберите домен, который необходимо обновить. 
  
3. На странице **Обзор** для домена выберите **DNS.**

  
4. На странице **управления DNS** щелкните **Добавить запись,** а затем выберите значения из следующей таблицы. 
    
    | Тип | Имя | Почтовый сервер | Priority (Приоритет) | TTL (Срок жизни) |
    |:-----|:-----|:-----|:-----|:-----|
    |MX  <br/> |@  <br/> |*\<domain-key\>*  .mail.protection.outlook.com  <br/> **Примечание:** Получите свою  *\<domain-key\>*  учетную запись Microsoft 365.   [Как его найти?](../get-help-with-domains/information-for-dns-records.md) |1  <br/> Дополнительные сведения о приоритете см. в статье [Приоритет записей MX](../setup/domains-faq.yml).    <br/>|30 минут  <br/> |
   

  
5. Нажмите **Сохранить**.
  
9. Если в разделе **MX Records** (Записи MX) есть другая запись MX, удалите ее, щелкнув значок **Delete (X)** (Удалить). 
  
10. В диалоговом окне подтверждения выберите **Удаление,** чтобы подтвердить изменения. 

  
## <a name="add-the-six-cname-records-that-are-required-for-microsoft"></a>Добавление шести записей CNAME, необходимых для Корпорации Майкрософт
<a name="BKMK_add_CNAME"> </a>

1. Чтобы приступить к работе, откройте страницу со своими доменами на сайте Cloudflare по [этой ссылке](https://www.cloudflare.com/a/login). Сначала вам потребуется выполнить вход.
    
  
2. На **домашней странице** выберите домен, который необходимо обновить. 
  
3. На странице **Обзор** для домена выберите **DNS.**

  
4. Добавьте первую из пяти записей CNAME.
    
    На странице **управления DNS** щелкните **Добавить запись,** а затем выберите значения из следующей таблицы.
    
    
    | Тип | Имя | Target | TTL |
    |:-----|:-----|:-----|:-----|
    |CNAME  <br/> |autodiscover  <br/> |autodiscover.outlook.com  <br/> |30 minutes  <br/> |
    |CNAME  <br/> |sip  <br/> |sipdir.online.lync.com  <br/> |30 minutes  <br/> |
    |CNAME  <br/> |lyncdiscover  <br/> |webdir.online.lync.com  <br/> |30 minutes  <br/> |
    |CNAME  <br/> |enterpriseregistration  <br/> |enterpriseregistration.windows.net  <br/> |30 minutes  <br/> |
    |CNAME  <br/> |enterpriseenrollment  <br/> |enterpriseenrollment-s.manage.microsoft.com  <br/> |30 minutes  <br/> |
    |CNAME  <br/> |msoid  <br/> |clientconfig.microsoftonline-p.net  <br/> |30 minutes  <br/> |
    
  
5. Выберите **значок трафика DNS** (изменение оранжевого облака на серый) для обхода серверов Cloudflare.
  
6. Нажмите **Сохранить**.
  
7. Добавьте остальные пять записей CNAME.

    
## <a name="add-a-txt-record-for-spf-to-help-prevent-email-spam"></a>Добавление записи TXT для SPF, предотвращающей рассылку спама
<a name="BKMK_add_TXT"> </a>

> [!IMPORTANT]
> Для записи инфраструктуры политики отправителей (SPF) для домена можно указать только одну запись TXT. Если у вашего домена больше одной записи SPF, это приведет к сбоям в работе почты и ошибкам классификации входящих писем и спама. Если вы уже указали запись SPF для домена, не создавайте еще одну для Microsoft 365. Вместо этого добавьте необходимые значения Microsoft 365 в текущую запись. Таким образом, в *одной* записи SPF будут указаны оба набора значений. 
  
1. Чтобы приступить к работе, откройте страницу со своими доменами на сайте Cloudflare по [этой ссылке](https://www.cloudflare.com/a/login). Сначала вам потребуется выполнить вход.
    
  
2. На **домашней странице** выберите домен, который необходимо обновить. 
  
3. На странице **Обзор** для домена выберите **DNS.**

  
4. На странице **управления DNS** щелкните **Добавить запись,** а затем выберите значения из следующей таблицы.  
    
    | Тип | Имя | TTL | Содержимое |
    |:-----|:-----|:-----|:-----|
    |TXT  <br/> |@  <br/> |30 minutes  <br/> |v=spf1 include:spf.protection.outlook.com -all  <br/> **Примечание.** Рекомендуется скопировать и вставить эту запись, чтобы сохранить все пробелы.       |

 
5. Нажмите **Сохранить**.
    

  
## <a name="add-the-two-srv-records-that-are-required-for-microsoft"></a>Добавление двух записей SRV, необходимых для продуктов корпорации Майкрософт
<a name="BKMK_add_SRV"> </a>

> [!IMPORTANT]
> Обратите внимание, что Cloudflare отвечает за обеспечение этой функции. Если вы видите несоответствия между шагами ниже и текущим графическим интерфейсом Cloudflare (Графический пользовательский интерфейс), используйте [сообщество Cloudflare.](https://community.cloudflare.com/) 

1. Чтобы приступить к работе, откройте страницу со своими доменами на сайте Cloudflare по [этой ссылке](https://www.cloudflare.com/a/login). Сначала вам потребуется выполнить вход.
      
2. На **домашней странице** выберите домен, который необходимо обновить. 
  
3. На странице **Обзор** для домена выберите **DNS.**
  
4. Добавьте первую из двух записей SRV.

    На странице **управления DNS** щелкните **Добавить** запись, а затем выберите значения из первого ряда следующей таблицы.
        
    | Тип | Служба | Протокол | Имя | TTL | Priority | Насыщенность | Порт | Target |
    |:-----|:-----|:-----|:-----|:-----|:-----|:-----|:-----|:-----|
    |SRV|_sip |TLS |Использование *domain_name;* например, contoso.com  |30 минут | 100|1 |443 |sipfed.online.lync.com  |
    |SRV|_sipfederationtls | TCP|Использование *domain_name;* например, contoso.com   |30 минут |100 |1 |5061 | sipfed.online.lync.com |

  
5. Нажмите **Сохранить**.

  
6. Добавьте другую запись SRV, выбрав значения из второго ряда таблицы. 

    
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md).
