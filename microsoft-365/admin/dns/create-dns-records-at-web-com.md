# <a name="create-dns-records-at-webcom-for-microsoft"></a>Создание записей DNS в web.com microsoft

 Если вы не нашли то, что вы ищете, см. раздел **[Вопросы и ответы по доменам](../setup/domains-faq.yml)**. 
  
Если web.com является поставщиком хостинга DNS, выполните действия в этой статье, чтобы проверить домен и настроить записи DNS для электронной почты, Skype для бизнеса Online и так далее.
  
После добавления этих записей в web.com домен будет настроен для работы с службами Майкрософт.

  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="change-your-domains-nameserver-ns-records"></a>Изменение записей сервера доменных имен
<a name="BKMK_Nameservers"> </a>

> [!IMPORTANT]
> Эту процедуру необходимо выполнить на сайте регистратора доменных имен, на котором вы приобрели и зарегистрировали свой домен. 
  
Когда вы подписались на web.com, вы добавили домен с помощью web.com **установки.** 
  
Чтобы проверить и создать записи DNS для вашего домена в Microsoft, сначала необходимо изменить имена в регистраторе домена, чтобы они использовали именные имена веб-сайта.
  
Чтобы самостоятельно изменить серверы доменных имен на веб-сайте регистратора доменных имен, выполните следующее:
  
1. Найдите на веб-сайте регистратора домена область, в которой можно изменить серверы доменных имен для вашего домена.
    
2. Создайте две записи серверов доменных имен, используя значения из приведенной ниже таблицы, либо измените существующие записи серверов доменных имен, чтобы они соответствовали следующим значениям:
    
    |||
    |:-----|:-----|
    |Основной сервер доменных имен  <br/> |Используйте значение nameserver, предоставленного web.com.  <br/> |
    |Второстепенный сервер доменных имен  <br/> |Используйте значение nameserver, предоставленного web.com.  <br/> |
   
    > [!TIP]
    > You should use at least two name server records. Если в списке есть другие серверы имен, их следует удалить. 
  
3. Сохраните изменения.
    
> [!NOTE]
> На распространение изменений, внесенных вами в записи серверов имен, в системе DNS по всему Интернету может потребоваться несколько часов. Затем ваша электронная почта Майкрософт и другие службы будут настроены на работу с доменом. 
  
## <a name="add-a-txt-record-for-verification"></a>Добавление записи TXT для проверки
<a name="BKMK_verify"> </a>

Прежде чем вы сможете использовать свой домен при работе с продуктами корпорации Майкрософт, мы должны убедиться в том, что вы являетесь его владельцем. Если вы войдете в свою учетную запись на сайте регистратора доменных имен и создадите запись DNS, для корпорации Майкрософт это послужит подтверждением того, что вы владеете доменом.
  
> [!NOTE]
> Эта запись используется исключительно для проверки принадлежности домена. При желании вы сможете удалить ее позже. 
  
1. Чтобы начать работу, перейдите на страницу доменов в web.com с помощью [этой ссылки.](https://checkout.web.com/manage-it/index.jsp) Войдите сначала.
  
2. На странице **Диспетчер учетных записей** выберите **Мои доменные имена**. 
  
3. В статье **Manage *my domain**** выберите **Изменить расширенные записи DNS.**

  
4. На странице **Имена доменов** в **статье Text (TXT Records)** нажмите **кнопку Изменить записи TXT,** а затем выберите значения из следующей таблицы. 
    
    |**Host (Узел)**|**TTL (Срок жизни)**|**Text (Текст)**|
    |:-----|:-----|:----|
    |@  <br/> |3600  <br/> |MS=ms *XXXXXXXX*  <br/> **Примечание.** Это пример. Используйте здесь свое конкретное значение **Назначение или адрес "Указывает на"** из этой таблицы.           [Как его найти?](../get-help-with-domains/information-for-dns-records.md)    |
  
    
5. Выберите **Продолжить**.
  
  
6. Подождите несколько минут, прежде чем проверить новую запись TXT, чтобы созданная запись обновлялась через Интернет.
    
Теперь, когда запись добавлена на веб-сайт регистратора доменных имен, вернитесь в продукт корпорации Майкрософт и запросите эту запись.
  
Когда продукт корпорации Майкрософт обнаружит правильную запись TXT, ваш домен будет подтвержден.
  
1. В Центре администрирования перейдите на страницу **Settings** (Параметры) \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834818" target="_blank">Domains</a> (Домены).

    
2. На странице **Domains** (Домены) выберите домен, который нужно проверить. 
    
    
  
3. На странице **Setup** (Настройка) выберите **Start setup** (Начать настройку).
    
    
  
4. На странице **Проверка домена** выберите **Проверить**.
    
    
  
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md). 
  
## <a name="add-an-mx-record-so-email-for-your-domain-will-come-to-microsoft"></a>Добавьте запись MX, чтобы сообщения электронной почты для вашего домена доставлялись в продукты корпорации Майкрософт.
<a name="BKMK_add_MX"> </a>

1. Чтобы начать работу, перейдите на страницу доменов в web.com с помощью [этой ссылки.](https://checkout.web.com/manage-it/index.jsp) Войдите сначала.
  
2. На странице **Диспетчер учетных записей** выберите **Мои доменные имена**. 
  
3. В статье **Manage *my domain**** выберите **Изменить расширенные записи DNS.**

4. В **статье Mail Servers (MX Records)** нажмите **кнопку Изменить записи MX,** а затем выберите значения из следующей таблицы. 
    
    |**Priority** (Приоритет)|**TTL** (Срок жизни)|**Mail Server (Почтовый сервер)**|
    |:-----|:-----|:-----|
    |1  <br/> Дополнительные сведения о приоритете см. в статье [Приоритет записей MX](../setup/domains-faq.yml).    <br/> |3600  <br/> |*\<domain-key\>*  .mail.protection.outlook.com  <br/> **Примечание:** Получите вашу  *\<domain-key\>*  учетную запись Майкрософт.   [Как его найти?](../get-help-with-domains/information-for-dns-records.md) |
   

5. Нажмите **Сохранить**.
  
6. Если в разделе **MX Records** указаны другие записи MX, выберите контрольный ящик рядом с записью **в разделе Delete** и выберите **Сохранить**. 
  
7. На экране подтверждения выберите **Сохранить изменения.** 

  
## <a name="add-the-six-cname-records-that-are-required-for-microsoft"></a>Добавление шести записей CNAME, необходимых для Корпорации Майкрософт
<a name="BKMK_add_CNAME"> </a>

1. Чтобы начать работу, перейдите на страницу доменов в web.com с помощью [этой ссылки.](https://checkout.web.com/manage-it/index.jsp) Сначала вам потребуется выполнить вход.
     
2. На странице **Диспетчер учетных записей** выберите **Мои доменные имена**. 
  
3. В статье **Manage *my domain**** выберите **Изменить расширенные записи DNS.**

4. Добавьте первую из шести записей CNAME.
    
    В **статье Host Aliases (CNAME Records)** нажмите **кнопку Изменить записи CNAME,** а затем выберите значения из следующей таблицы.
    
    
    |**Alias (Псевдоним)**|**TTL (Срок жизни)**|**Refers to Host Name (Указывает на имя узла)**|**Другой хост**|
    |:-----|:-----|:-----|:-----|
    |autodiscover  <br/> |3600  <br/> |@ (нет)  <br/> |autodiscover.outlook.com  <br/> |
    |sip  <br/> |3600  <br/> |@ (нет)  <br/> |sipdir.online.lync.com  <br/> |
    |lyncdiscover  <br/> |3600  <br/> |@ (нет)  <br/> | webdir.online.lync.com  <br/> |
    |enterpriseregistration  <br/> |3600  <br/> |@ (нет)  <br/> |enterpriseregistration.windows.net  <br/> |
    |enterpriseenrollment  <br/> |3600  <br/> |@ (нет)  <br/> |enterpriseenrollment-s.manage.microsoft.com  <br/> |
    |msoid  <br/> |3600  <br/> |@ (нет)  <br/> |clientconfig.microsoftonline-p.net  <br/> |
    
  
5. Выберите **Продолжить**.
  
6. Добавьте остальные пять записей CNAME.

    
## <a name="add-a-txt-record-for-spf-to-help-prevent-email-spam"></a>Добавление записи TXT для SPF, предотвращающей рассылку спама
<a name="BKMK_add_TXT"> </a>

> [!IMPORTANT]
> Для записи инфраструктуры политики отправителей (SPF) для домена можно указать только одну запись TXT. Если у вашего домена больше одной записи SPF, это приведет к сбоям в работе почты и ошибкам классификации входящих писем и спама. Если вы уже указали запись SPF для домена, не создавайте еще одну для продуктов корпорации Майкрософт. Вместо этого добавьте необходимые значения Microsoft в текущую  запись, чтобы у вас была одна запись SPF, которая включает оба набора значений. 
  
1. Чтобы начать работу, перейдите на страницу доменов в web.com с помощью [этой ссылки.](https://checkout.web.com/manage-it/index.jsp) Войдите сначала.
    
  
2. На странице **Диспетчер учетных записей** выберите **Мои доменные имена**. 
  
3. В статье **Manage *my domain**** выберите **Изменить расширенные записи DNS.**

  
4. На странице **Имена доменов** в **статье Text (TXT Records)** нажмите **кнопку Изменить записи TXT,** а затем выберите значения из следующей таблицы.   
    
    |**Host (Узел)**|**TTL (Срок жизни)**|**Text (Текст)**|
    |:-----|:-----|:-----|
    |@  <br/> |3600  <br/> |v=spf1 include:spf.protection.outlook.com -all  <br/> **Примечание.** Рекомендуется скопировать и вставить эту запись, чтобы сохранить все пробелы.       |

 
5. Выберите **Продолжить**.

6. Нажмите кнопку **Сохранить изменения**.
    

  
## <a name="add-the-two-srv-records-that-are-required-for-microsoft"></a>Добавление двух записей SRV, необходимых для продуктов корпорации Майкрософт
<a name="BKMK_add_SRV"> </a>

> [!IMPORTANT]
> Обратите внимание, что web.com несет ответственность за обеспечение этой функции. Если вы видите несоответствия между шагами ниже и текущим интерфейсом web.com (графический пользовательский интерфейс), используйте web.com [community](https://community.web.com.com/). 

1. Чтобы начать работу, перейдите на страницу доменов в web.com с помощью [этой ссылки.](https://checkout.web.com/manage-it/index.jsp) Войдите сначала.
      
2. На странице **Диспетчер учетных записей** выберите **Мои доменные имена**. 
  
3. В статье **Manage *my domain**** выберите **Изменить расширенные записи DNS.**
  
4. Добавьте первую из двух записей SRV.

    В **статье Service (SRV Records)** нажмите **кнопку Изменить записи SRV** и выберите значения из следующей таблицы. 
        
    |**Служба**|**Protocol (Протокол)**|**TTL (Срок жизни)**|**Priority** (Приоритет)|**Weight** (Вес)|**Port** (Порт)|**Target (Назначение)**|
    |:-----|:-----|:-----|:-----|:-----|:-----|:-----|
    |_sip |_tls |3600 | 100|1 |443 |sipfed.online.lync.com  |
    |_sipfederationtls |_tcp |3600 |100 |1 |5061 | sipfed.online.lync.com |

  
5. Добавьте другую запись SRV, выбрав значения из второго ряда таблицы. 
  
6. Выберите **Продолжить**.

7. Нажмите кнопку **Сохранить изменения**.

    
> [!NOTE]
>  Обычно на вступление изменений DNS в силу требуется около 15 минут. Однако иногда распространение внесенного изменения в системе DNS по всему Интернету занимает больше времени. Если после добавления записей DNS возникла проблема с потоком обработки почты или другие неполадки, см. статью [Устранение неполадок после смены имени домена или записей DNS](../get-help-with-domains/find-and-fix-issues.md).
