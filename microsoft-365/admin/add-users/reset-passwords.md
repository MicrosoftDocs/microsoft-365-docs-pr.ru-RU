# <a name="reset-passwords"></a>Сброс паролей

В этой статье объясняется, как сбросить пароли для себя и для пользователей, когда у вас есть Microsoft 365 для бизнес-подписки.

## <a name="before-you-begin"></a>Прежде чем начать

Эта статья адресована тем, кто устанавливает политику срока действия паролей в компании, учебном заведении или некоммерческой организации. Чтобы выполнить эти действия, вам нужно войти с помощью своей учетной записи администратора Microsoft 365. [Что такое учетная запись администратора?](../../business-video/admin-center-overview.md)

Вы должны быть глобальным [администратором или администратором паролей](about-admin-roles.md) для выполнения этих шагов.

Посмотрите короткое видео о сбросе паролей пользователей.<br><br>

## <a name="steps-reset-a-business-password-for-a-user"></a>Шаги: Сброс бизнес-пароля для пользователя

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. На странице **Активных пользователей** выберите пользователя, а затем выберите **пароль Сброс.**

3. Следуйте инструкциям на странице **пароля Reset,** чтобы автоматически создать новый пароль для пользователя или создать его для них, а затем выбрать **Reset.**  

4. Введите адрес электронной почты, на который пользователь может попасть, чтобы получить новый пароль, и следите за ним, чтобы убедиться, что он получил его.

## <a name="let-users-reset-their-own-passwords"></a>Предоставление пользователям прав на самостоятельный сброс пароля

Мы настоятельно рекомендуем вам настроить самостоятельный сброс пароля. В таком случае вам не придется сбрасывать пароли вручную для всех пользователей в организации. Сведения о том, как это сделать, см. в статье [Предоставление пользователям прав на самостоятельный сброс пароля в Office 365](let-users-reset-passwords.md).

## <a name="reset-my-admin-password"></a>Сброс пароля администратора

Используйте эти шаги, если вы забыли пароль, но можете войти в систему Microsoft 365, потому что, например, пароль сохраняется в браузере:

1. Выберите свое имя (икону) в правом верхнем углу > **личная информация**  >  **моего аккаунта.**

2. Под **контактными данными,** перепроверьте, что **ваша альтернативная** электронная почта является точной и что вы предоставили номер мобильного телефона. Если нет, измените их сейчас.

3. Выйти: выберите свое имя в правом верхнем \> **углу Выйти**.

4. Теперь ввись снова: ввемите имя \> **пользователя Далее,** \> а затем **выберите забытый пароль.**

5. Следуйте шагам мастера, чтобы сбросить пароль. Он использует ваши альтернативные контактные данные, чтобы убедиться, что вы тот человек, чтобы сбросить пароль.

Если вы забыли пароль и не можете войти в:

- Попросите другого глобального администратора в вашем бизнесе сбросить пароль для вас.

- Убедитесь, что вы предоставили альтернативную контактную информацию, включая номер мобильного телефона.

- Или позвоните [в службу поддержки Майкрософт.](../../business-video/get-help-support.md)

## <a name="reset-all-business-passwords-for-everyone-in-your-organization-at-the-same-time"></a>Сброс всех бизнес-паролей для всех в вашей организации в то же время
<a name="bkmk_forgot"> </a>

Эти шаги работают для бизнеса с десятками пользователей. Если у вас есть сотни или тысячи пользователей, смотрите следующий раздел о сбросе паролей оптом (максимум 40 пользователей одновременно).
  
1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. Выберите опцию рядом с **именем Дисплея,** чтобы выбрать всех в вашем бизнесе. Затем снимите флажок напротив себя. Собственный пароль невозможно сбросить одновременно со всеми остальными пользователями.

3. Выберите **пароль сброса**. 

4. Следуйте инструкциям на странице **пароля Сброс** и выберите **Reset**.  Если вы выбрали автоматическое создание паролей, новые временные пароли будут отображаться.

5. Введите адрес электронной почты, где вы можете получить временные пароли. Вы должны будете уведомить пользователей, что их временные пароли.
