# <a name="deploy-information-protection-for-data-privacy-regulations-with-microsoft-365"></a>Развертывание защиты информации для нормативов конфиденциальности данных с помощью Microsoft 365

Ваша организация может подчиняться региональным нормативным актам о конфиденциальности данных, которые требуют защиты, управления и предоставления прав и контроля над личной информацией, хранимой в вашей ИТ-инфраструктуре, включая как локальное, так и облачное. Лучшим примером норматива конфиденциальности данных является Общий регламент Ес по защите данных (GDPR). Невыполнение нормативных требований конфиденциальности данных может привести к значительным штрафам.

Примеры типов данных в Microsoft 365: сеансы чата в Microsoft Teams, сообщения электронной почты в Exchange и файлы в SharePoint и OneDrive. Это решение предоставляет рекомендации по оценке рисков и выявлению информации, защите, управлению и реагированию на инциденты конфиденциальности данных для персональных данных, хранимых в службах Microsoft 365, на которые налагаются правила конфиденциальности данных.

![Что такое защита информации в правилах конфиденциальности данных](../media/information-protection-deploy/information-protection-data-privacy-regulations-overview.png)

Кроме того, вы можете получить дополнительные сведения об использовании удостоверений, устройств и средств защиты от угроз Microsoft 365 для обеспечения конфиденциальности данных. 

Чтобы соответствовать критериям защиты информации в соответствии с правилами конфиденциальности данных, используйте эти возможности и функции Microsoft 365.

| Возможность или функция | Описание | Лицензирование |
|:-------|:-----|:-------|
| Диспетчер соответствия требованиям | Управляйте действиями по обеспечению соответствия нормативным требованиям, получите общую оценку текущей конфигурации соответствия требованиям и найдите рекомендации по улучшению этого средства оценки рисков на основе рабочего процесса в Центре соответствия требованиям Microsoft 365. | Microsoft 365 E3 и E5 |
| Microsoft Defender для Office 365 | Защитите от атак свои приложения и данные в Microsoft 365, такие как сообщения электронной почты, документы Office и средства совместной работы. | Microsoft 365 E3 и E5 | 
| Метки конфиденциальности | Классифицируйте и защищайте данные своей организации путем присвоения меток различных уровней защиты сообщениям электронной почты, файлам и сайтам, не вызывая снижения эффективности работы пользователей и не ограничивая возможностей их совместной работы. | Microsoft 365 E3 и E5 |
| Защита от потери данных (DLP) | Выявляйте, создавайте предупреждения и блокируйте рискованное, ненамеренное или нежелательное предоставление общего доступа (внутри и вне организации) к ресурсам, например к информации, содержащей персональные данные. | Microsoft 365 E3 и E5 | 
| Метки и политики хранения данных | Воспользуйтесь такими элементами управления информацией, как параметр продолжительности хранения данных и требования к хранению персональных данных о клиентах, для соблюдения политик организации и нормативных требований в отношении данных. | Microsoft 365 E3 и E5 |
| Шифрование электронной почты | Воспользуйтесь возможностью отправки и получения пользователями в организации и за ее пределами зашифрованных сообщений электронной почты, содержащих данные, на которые распространяется действие нормативных положений, например персональные данные клиентов. | Microsoft 365 E3 и E5 |
||||

## <a name="organization-of-the-guidance-in-this-solution"></a>Организация рекомендаций в этом решении

Чтобы помочь вам понять средства Microsoft 365, доступные для идентификации, управления, контроля и отслеживания персональных данных, на которые налагается один или несколько нормативных требований, связанных с конфиденциальностью, это руководство разорвется по разделам.
 
![Действия по реализации защиты информации для нормативов конфиденциальности данных](../media/information-protection-deploy/information-protection-data-privacy-regulations-steps.png)

Каждый из этих разделов соответствует отдельной статье в этом решении.

>[!Note]
>Если вы уже знакомы с обязательствами по конфиденциальности данных и выполняете их в рамках существующего плана, вам может потребоваться сосредоточиться на руководстве по предотвращению, защите, ведению и анализу.

>[!Important]
>В соответствии с этим руководством вы не обязательно будете соблюдать какие-либо нормы конфиденциальности данных, особенно учитывая количество необходимых действий, которые находятся вне контекста функций. Вы несете ответственность за обеспечение соответствия требованиям, а также за консультации с юридическими группами и группами по обеспечению соответствия требованиям, а также за консультации со сторонними лицами, специализирующимися на обеспечении соответствия требованиям.
>

## <a name="plan-assess-data-privacy-risks-and-identify-sensitive-items"></a>План: оценка рисков конфиденциальности данных и определение конфиденциальных элементов

Оценка нормативов конфиденциальности данных и рисков, на которые может лагаться ваша организация, является ключевым первым шагом перед началом реализации улучшений, включая те, которые достижимы с помощью конфигурации Microsoft 365. Это может включать общую оценку готовности или идентификацию определенных типов конфиденциальной информации, которые подотчеты нормативным требованиям вашей организации, а также их появление в среде Microsoft 365.

Дополнительные сведения см. в [подпунктов "Оценка рисков конфиденциальности данных и идентификация конфиденциальных элементов".](information-protection-deploy-assess.md)

## <a name="track-run-risk-assessments-and-check-your-compliance-score"></a>Отслеживание: запуск оценки рисков и проверка оценки соответствия требованиям

Диспетчер соответствия требованиям, доступный в Центре соответствия требованиям Microsoft 365, предоставляет встроенную возможность отслеживать и управлять действиями по улучшению в целом, а также действиями, связанными с несколькими нормативными актами конфиденциальности данных, которые применяются к вам.

Используйте встроенные шаблоны оценки, специфические для каждого норматива, где можно отслеживать элементы действий для каждого выбранного шаблона оценки, а также просматривать определенные нормативные меры и соотносят их с конкретными действиями.

Дополнительные сведения см. в диспетчере [соответствия требованиям для управления действиями по улучшению.](information-protection-deploy-compliance.md)

## <a name="protect-information-subject-to-data-privacy-regulation"></a>Защита информации, влияемой на нормы конфиденциальности данных

Нормативы конфиденциальности данных определяют ряд средств контроля защиты персональных данных, которые могут быть использованы в вашей среде, в том числе несколько правил защиты информации только в четырех нормативных актах о конфиденциальности данных в нашем примере GDPR, Закон Калифорнии о защите потребителей (CCPA), HIPAA-HITECH (закон о конфиденциальности здравоохранения США) и Закон Бразилии о защите данных (LGPD).

Дополнительные сведения [см.](information-protection-deploy-protect-information.md)в этой теме.

В этой статье данная статья посвящена основным схемам контроля, которые можно использовать для решения потребностей в защите информации для обеспечения конфиденциальности данных в организации.

## <a name="retain-govern-information-subject-to-data-privacy-regulation"></a>Сохранение: управление информацией, подавляемой нормативным актом о конфиденциальности данных

Нормативы конфиденциальности данных используются для средств контроля управления личной информацией, которые могут быть использованы в вашей среде, включая более 24 средств контроля в четырех нормативах конфиденциальности данных в нашем примере набора GDPR, CCPA, HIPAA-HITECH и LGPD.

Дополнительные сведения см. в под вопросе "Управление информацией, подавляемой [нормативным актом конфиденциальности данных в организации".](information-protection-deploy-govern.md)

Хотя правила конфиденциальности данных могут быть неуважительными относительно управления информацией, таких как намеренное хранение, удаление и архивация, в этой статье рассматриваются основные схемы контроля, которые можно использовать для решения потребностей в управлении информацией для обеспечения конфиденциальности данных в &mdash; &mdash; организации.

## <a name="investigate-monitor-investigate-and-respond-to-data-privacy-incidents"></a>Исследование: мониторинг, исследование инцидентов конфиденциальности данных и реагирование на них

Существуют функции Microsoft 365, которые помогают отслеживать, исследовать инциденты конфиденциальности данных в организации и реагировать на них при работе связанных возможностей. 

Процессы, процедуры и другая документация для каждого из них могут быть важны для демонстрации соответствия нормативным органам.

Дополнительные сведения см. в этой теме, чтобы отслеживать инциденты конфиденциальности данных в организации [и реагировать на них.](information-protection-deploy-monitor-respond.md)
