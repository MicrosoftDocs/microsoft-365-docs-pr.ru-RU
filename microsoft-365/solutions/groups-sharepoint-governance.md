# <a name="settings-interactions-between-microsoft-365-groups-and-sharepoint"></a>Взаимодействие параметров между группами Microsoft 365 и SharePoint

Некоторые параметры групп Microsoft 365 и SharePoint в Microsoft 365, в частности связанные с совместным использованием и созданием сайтов групп и групп, перекрываются друг с другом. В этой статье приводится описание этих взаимодействий и лучшие методики работы с этими настройками.

![Диаграмма Венна для функций SharePoint, Yammer и групп](../media/groups-sharepoint-venn.png)

## <a name="the-effects-of-sharepoint-settings-on-microsoft-365-groups"></a>Влияние параметров SharePoint на группы Microsoft 365

|Параметр SharePoint|Description|Влияние на группы Microsoft 365|Рекомендация|
|:-----------------|:----------|:-----------------------------|:-------------|
|Внешний общий доступ для организации и сайта|Определяет, можно ли совместно использовать сайты, файлы и папки с людьми за пределами организации.|Если параметры SharePoint и групп не совпадают, гостям в группе может быть заблокирован доступ к сайту, или внешний доступ может быть доступен на сайте, но не в группе.|При изменении параметров общего доступа проверьте параметры групп и сайтов SharePoint для сайтов групп, подключенных к группе.<br><br>См. ["Совместное взаимодействие с гостями" на сайте.](https://docs.microsoft.com/microsoft-365/solutions/collaborate-in-site)|
|Разрешение или блокировка домена|Разрешает или предотвращает обмен контентом с указанными доменами.|Группы не распознают списки допустимой или заблокированной sharePoint. Пользователи из доменов, не доступных в SharePoint, могут получить доступ к SharePoint через группу.|Совместное управление списками доменов для Azure AD и SharePoint. Создайте процесс управления для всей организации, разрешая и блокируя домены.<br><br>См. [параметры домена SharePoint](https://docs.microsoft.com/sharepoint/restricted-domains-sharing) и [домена Azure AD](https://docs.microsoft.com/azure/active-directory/b2b/allow-deny-list)|
|Разрешить только пользователям из определенных групп безопасности предоставлять внешний общий доступ|Указывает группы безопасности, которые могут делиться сайтами, папками и файлами извне.|Этот параметр не влияет на внешние группы общего доступа владельцев групп. Гости группы имеют доступ к связанному сайту SharePoint.||
|Параметры общего доступа к сайтам SharePoint|Определяет, кто может делиться сайтом непосредственно за пределами членства в группах. Это настраивается группой или владельцем сайта.|Этот параметр не влияет непосредственно на группу, но может разрешить пользователям добавляться на сайт и не иметь доступа к другим ресурсам группы.|С помощью этого параметра можно ограничить общий доступ к сайту напрямую и управлять доступом к сайту через группу.|
|Позволить пользователям создавать сайты на странице "Начните" и OneDrive в SharePoint|Указывает, могут ли пользователи создавать новые сайты SharePoint.|Если этот параметр отключен, пользователи по-прежнему могут создавать сайты групп, подключенные к группе, создавая группу.||

## <a name="the-effects-of-microsoft-365-groups-setting-on-sharepoint"></a>Влияние настройки групп Microsoft 365 на SharePoint

|Настройка групп Microsoft 365|Description|Влияние на SharePoint|Рекомендация|
|:---------------------------|:----------|:-------------------|:-------------|
|Политики именования|Указывает префиксы и суффиксы имени группы, а также заблокированные слова для создания группы|Политики применяются для пользователей, создав сайты групп, подключенные к группе, но не сайты для общения с другими шаблонами.|При необходимости создайте отдельные рекомендации по именоведению для сайтов связи.|
|Гостевой доступ группы|Указывает, могут ли люди за пределами организации добавляться в группы.|Если параметры SharePoint и групп не совпадают, гостям в группе может быть заблокирован доступ к сайту, или внешний доступ может быть доступен на сайте, но не в группе.|При изменении параметров общего доступа проверьте параметры групп и сайтов SharePoint для сайтов групп, подключенных к группе.<br><br>См. ["Совместное взаимодействие с гостями" на сайте](https://docs.microsoft.com/microsoft-365/solutions/collaborate-in-site)|
|Создание группы по группе безопасности|Группы могут создаваться только членами определенной группы безопасности.|Пользователи, которые не являются участниками группы безопасности, не смогут создать сайт группы, подключенный к группе.|Убедитесь, что процесс запроса группы содержит инструкции по запросу сайта.|
|Политика срока действия группы|Указывает период времени, после которого группы, которые не используются активно, будут автоматически удаляться.|При удалении группы также удаляется связанный сайт SharePoint. Контент, защищенный политиками хранения, сохраняется.|Используйте политики истечения срока действия, чтобы избежать спрайтов неиспользаемой группы и сайты.|
