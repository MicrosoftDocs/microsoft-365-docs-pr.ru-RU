# <a name="microsoft-365-group-expiration-policy"></a>Политика срока действия группы Microsoft 365

С увеличением использования групп Microsoft 365 и Microsoft Teams администраторам и пользователям необходим способ очистки неиспользованных групп и команд. Политика истечения срока действия групп Microsoft 365 может помочь удалить неактивные группы из системы и ухумить их.

По истечении срока действия группы все связанные с ней службы (почтовый ящик, Планировщик, сайт SharePoint, группа и т. д.) также удаляются.

По истечении срока действия группы происходит "мягкое удаление", что означает, что ее можно восстановить в течение 30 дней.

Администраторы могут указать срок действия, и любая неактивная группа, которая достигнет конца этого периода и не будет продлена, будет удалена. (К ним относятся архивные команды.) Срок действия начинается с создания группы или с даты последнего продления. Владельцы группы будут автоматически отправлены по электронной почте до истечения срока действия, что позволит им продлить группу на другой срок действия. Пользователи Teams будут видеть постоянные уведомления в Teams.

Группы, которые активно используются, обновляются автоматически. Любое из следующих действий автоматически обновляет группу:
- SharePoint — просмотр, редактирование, скачивание, перемещение, отправка или отправка файлов. (Просмотр страницы SharePoint не считается действием для автоматического продления.)
- Outlook — присоединить группу, прочитать или записать сообщение группы из группы и как сообщение (Outlook в Интернете).
- Teams — посещение канала teams.

> [!IMPORTANT]
> При изменении политики истечения срока действия служба пересчитает дату окончания срока действия для каждой группы. Он всегда начинает отсчет с даты создания группы, а затем применяет новую политику истечения срока действия.

Важно знать, что срок действия по умолчанию отключен. Администраторы должны включить его для своей организации, если они хотят использовать его.

> [!NOTE]
> Для настройки и использования политики истечения срока действия для групп Microsoft 365 необходимо обладать лицензиями Azure AD Premium, но не обязательно назначать их участникам всех групп, к которым применяется политика истечения срока действия. Дополнительные сведения [см. в сведениях о том, как начать работу с Azure Active Directory Premium.](https://docs.microsoft.com/azure/active-directory/active-directory-get-started-premium)

## <a name="who-can-configure-and-use-the-microsoft-365-groups-expiration-policy"></a>Кто может настраивать и использовать политику истечения срока действия групп Microsoft 365?

|Роль|Что они могут делать|
|---------|---------|
|Глобальный администратор Office 365 (в Azure, администратор компании), администратор пользователей|Создание, чтение, обновление или удаление параметров политики истечения срока действия групп Microsoft 365.|
|Пользователь|Обновление или [восстановление собственной](https://docs.microsoft.com/azure/active-directory/users-groups-roles/groups-restore-deleted) группы Microsoft 365|

## <a name="how-to-set-the-expiration-policy"></a>Как установить политику истечения срока действия

Как было отмечено выше, срок действия по умолчанию отключен. Администратор должен включить политику истечения срока действия и установить ее свойства, чтобы она вступила в силу. Чтобы включить его, перейдите к **истечении срока действия групп Azure Active Directory.**  >    >   Здесь можно задать время жизни группы по умолчанию и указать, как далеко заранее будут отходить первые и вторые уведомления об истечении срока действия для владельца группы.

Время жизни группы указывается в днях и может быть задано как 180, 365, так и указанное вами пользовательское значение. Пользовательское значение должно быть не менее 30 дней.

Если у группы нет владельца, сообщения с истечением срока действия будут отправлены указанному администратору.

Вы можете настроить политику для всех групп, только выбранных групп или полностью отключить ее, выбрав **"Нет".** Обратите внимание, что в настоящее время нельзя использовать разные политики для разных групп.

## <a name="how-expiry-works-with-the-retention-policy"></a>Как истечение срока действия работает с политикой хранения

Если вы настроили политику хранения для групп в Центре безопасности и соответствия требованиям, политика истечения срока действия будет работать с политикой хранения. По истечении срока действия группы беседы и файлы почтовых ящиков группы на сайте группы сохраняются в контейнере хранения в течение определенного количества дней, определенного в политике хранения. Однако пользователи не увидят группу или ее содержимое по истечении срока действия.

## <a name="how-and-when-a-group-owner-learns-if-their-groups-are-going-to-expire"></a>Как и когда владелец группы узнает, истечет ли срок действия их групп

Владельцы группы будут уведомлены только по электронной почте. Если группа была создана с помощью Планировщика, SharePoint или любого другого приложения, уведомления об истечении срока действия всегда будут отправляться по электронной почте. Если группа была создана с помощью Teams, владелец группы получит уведомление о продлении через раздел действий. Не рекомендуется включить истечение срока действия для группы, если у владельца группы нет допустимых адресов электронной почты.

За 30 дней до истечения срока действия группы владельцы группы (или адреса электронной почты, указанные для групп, у которых нет владельца) получат сообщение электронной почты, позволяя им легко обновить группу. Если они не продлиют его, они получат еще одно сообщение о продлении через 15 дней до истечения срока действия. Если они еще не обновили его, они получат еще одно уведомление по электронной почте за день до истечения срока действия.

Если по какой-либо причине ни один из владельцев или администраторов не продлевает группу до истечения ее срока действия, администратор по-прежнему может восстановить группу в течение 30 дней после истечения срока ее действия. For details see: [Restore a deleted Microsoft 365 group](https://support.office.com/article/restore-a-deleted-office-365-group-b7c66b59-657a-4e1a-8aa0-8163b1f4eb54).
