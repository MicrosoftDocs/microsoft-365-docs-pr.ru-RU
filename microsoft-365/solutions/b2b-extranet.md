# <a name="create-a-b2b-extranet-with-managed-guests"></a>Создание экстрасети B2B с управляемыми гостями

Управление правами [на использование Azure Active Directory](/azure/active-directory/governance/entitlement-management-overview) можно использовать для создания экстрасети B2B для совместной работы с организацией-партнером, использующей Azure Active Directory. Это позволяет пользователям самостоятельно зарегистрироваться на сайте экстрасети или в команде и получать доступ через рабочий процесс утверждения.

С помощью этого метода обмена ресурсами для совместной работы организация-партнер может помочь поддерживать и утверждать гостей на их конце, уменьшая нагрузку на ИТ-отдел и позволяя тем, кто наиболее знаком с соглашением о сотрудничестве, управлять доступом пользователей.

В этой статье пошаговая статья посвящена созданию пакета ресурсов (в данном случае сайта или группы), которые можно делиться с организацией-партнером с помощью модели регистрации доступа к самообслуживке. 

Перед началом создайте сайт или команду, которые вы хотите поделиться с организацией-партнером и включить его для общего доступа к гостю. Дополнительные [сведения см. в](collaborate-in-site.md) совместной работе с гостями на сайте или в совместной работе с гостями [в группе.](collaborate-as-team.md) Кроме того, рекомендуется [](create-secure-guest-sharing-environment.md) просмотреть создание безопасной среды обмена гостями для получения сведений об особенностях безопасности и соответствия требованиям, которые можно использовать для поддержания политик управления при совместном сотрудничестве с гостями.

## <a name="license-requirements"></a>Требования лицензирования

Для использования этой функции требуется лицензия Azure AD Premium P2. 

Специализированные облака, такие как Azure Germany и Azure China 21Vianet, в настоящее время недоступны для использования.

## <a name="connect-the-partner-organization"></a>Подключение организации-партнера

Чтобы пригласить гостей из организации-партнера, необходимо добавить домен партнера в качестве связанной организации в Azure Active Directory.

Добавление подключенной организации
1. В [Azure Active Directory](https://aad.portal.azure.com)нажмите **кнопку Управление удостоверением**.
2. Щелкните **Подключенные организации**.
4. Щелкните **Добавить подключенную организацию**.
5. Введите имя и описание организации, а затем нажмите **кнопку Далее: Домен Directory +**.
6. Нажмите **кнопку Добавить каталог + домен**.
7. Введите домен для организации, которую необходимо подключить, и нажмите кнопку **Добавить**.
8. Нажмите **кнопку Подключиться,** а затем **нажмите кнопку Далее: Спонсоры**.
9. Добавьте людей из вашей организации или организации, к которым вы подключаете, чтобы утвердить доступ для гостей.
10. Щелкните **Далее: проверка и создание**.
11. Просмотрите выбранные параметры и нажмите кнопку **Создать**.

    ![Снимок экрана страницы подключенных организаций в Azure Active Directory](../media/identity-governance-connected-organizations.png)

## <a name="choose-the-resources-to-share"></a>Выбор ресурсов для обмена

Первым шагом в выборе ресурсов для совместной работы с организацией-партнером является создание каталога, который будет содержать их.

Создание каталога
1. В [Azure Active Directory](https://aad.portal.azure.com)нажмите **кнопку Управление удостоверением**.
2. Щелкните **Каталоги**.
3. Нажмите **кнопку Новый каталог**.
4. Введите имя и описание для  каталога  и убедитесь, что включено и включено для внешних **пользователей,** оба за набором да .
5. Нажмите кнопку **Создать**.

   ![Снимок экрана страницы каталогов в Azure Active Directory Identity Governance](../media/identity-governance-catalogs.png)

После создания каталога добавьте сайт SharePoint или команду, которые вы хотите поделиться с организацией-партнером.

Добавление ресурсов в каталог
1. В Azure AD Identity Governance щелкните **Каталоги,** а затем щелкните каталог, в котором необходимо добавить ресурсы.
2. Щелкните **Ресурсы** и нажмите **кнопку Добавить ресурсы**.
3. Выберите группы или сайты SharePoint, которые необходимо включить в свою экстрасеть, а затем нажмите **кнопку Добавить**.

   ![Снимок экрана страницы ресурсов каталога в Azure Active Directory Identity Governance](../media/identity-governance-catalog-resource.png)

После определения ресурсов, которыми вы хотите поделиться, следующим шагом является создание пакета доступа, который определяет тип доступа, который предоставляется пользователям-партнерам, и процесс утверждения для новых пользователей-партнеров, запрашивающих доступ.

Создание пакета доступа
1. В Azure AD Identity Governance щелкните **Каталоги,** а затем щелкните каталог, в котором необходимо создать пакет доступа.
2. Щелкните **пакеты Access** и нажмите **кнопку Новый пакет доступа.**
3. Введите имя и описание пакета доступа, а затем нажмите **кнопку Далее: Роли ресурса**.
4. Выберите ресурсы из каталога, которые необходимо использовать для экстрасети.
5. Для каждого ресурса в столбце **Роль** выберите роль пользователя, которую вы хотите предоставить гостям, которые используют экстрасети.
6. Нажмите **кнопку Далее: Запросы**.
7. В **статье Пользователи, которые могут запрашивать доступ,** выберите **Для пользователей, не в каталоге**.
8. Убедитесь, что выбран параметр **"Конкретные подключенные** организации", а затем нажмите **кнопку Добавить каталоги.**
9. Выберите подключенную организацию, которую вы добавили ранее, и нажмите кнопку **Выберите**
10. В **соответствии с утверждением** выберите **Да** для **утверждения.**
11. В **первом утверждении** выберите один из спонсоров, которые вы добавили ранее, или выберите конкретного пользователя.
12. Нажмите **кнопку Добавить откат** и выберите одобрение отката.
13. В **статье Включить** выберите **Да**.
14. Нажмите **кнопку Далее: жизненный цикл**.
15. Выберите параметры проверки истечения срока действия и доступа, которые необходимо использовать, а затем нажмите **кнопку Далее: Обзор + Создание**.
16. Просмотрите параметры и нажмите кнопку **Создать**.

    ![Снимок экрана экрана пакетов доступа в Azure Active Directory Identity Governance](../media/identity-governance-access-packages.png)

Если вы в партнерстве с крупной организацией, возможно, захотите скрыть пакет доступа. Если пакет скрыт, пользователи в организации-партнере не увидят пакет на портале *My Access.* Вместо этого им необходимо отправить прямую ссылку для регистрации пакета. Сокрытие пакета доступа может уменьшить количество ненадлежащих запросов на доступ, а также помочь сохранить доступные пакеты доступа, организованные на портале организации-партнера.

Настройка пакета доступа к скрытому
1. В Azure AD Identity Governance щелкните **пакеты Access** и нажмите кнопку пакет доступа.
2. На странице **Обзор** нажмите кнопку **Изменить**.
3. В **статье Свойства** выберите **Да** для **скрытых** и нажмите кнопку **Сохранить**.

   ![Снимок экрана свойства пакета редактирования доступа](../media/identity-governance-access-package-hidden.png)

## <a name="invite-partner-users"></a>Приглашение пользователей-партнеров

Если пакет доступа установлен скрытым, необходимо отправить прямую ссылку в организацию-партнер, чтобы они могли запрашивать доступ к вашему сайту или команде.

Поиск ссылки портала доступа
1. В Azure AD Identity Governance щелкните **пакеты Access** и нажмите кнопку пакет доступа.
2. На странице **Обзор** щелкните **Скопируйте ссылку на буфер обмена** для ссылки **портала "Мой доступ".**

   ![Снимок экрана свойств пакета доступа со ссылкой на портал доступа](../media/identity-governance-access-portal-link.png)

После скопив ссылку, вы можете поделиться ссылкой со своим контактом в организации-партнере, и они могут отправить ее пользователям в их команде совместной работы.
