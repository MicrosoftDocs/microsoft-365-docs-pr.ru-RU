# <a name="create-a-b2b-extranet-with-managed-guests"></a>Создание экстрасети B2B с управляемыми гостями

Вы можете использовать [Azure Active Directory Entitlement Management](https://docs.microsoft.com/azure/active-directory/governance/entitlement-management-overview) для создания экстрасети B2B для совместной работы с партнерской организацией, использующей Azure Active Directory. Это позволяет пользователям самостоятельно зарегистрироваться на сайте экстрасети или в группе и получить доступ через рабочий процесс утверждения.

С помощью этого метода общего доступа к ресурсам для совместной работы партнерская организация может помочь поддерживать и утверждать гостей на их стороне, снижая нагрузку на ИТ-отдел и позволяя пользователям, знакомым с соглашением о совместной работе, управлять доступом пользователей.

В этой статье пошаговая статья посвящена созданию пакета ресурсов (в данном случае это сайт или группа), которые можно использовать совместно с партнерской организацией с помощью модели регистрации для самостоятельного доступа. 

Перед началом создайте сайт или команду, которые вы хотите поделиться с партнерской организацией, и в качестве гостевого общего доступа. Дополнительные [сведения см.](collaborate-in-site.md) в [](collaborate-as-team.md) совместной работе с гостями на сайте или совместной работе с гостями в команде. Также рекомендуется просмотреть [](create-secure-guest-sharing-environment.md) статью "Создание безопасной среды гостевого общего доступа" для получения сведений о функции безопасности и соответствия требованиям, которые можно использовать для поддержки политик управления при совместной работы с гостями.

## <a name="license-requirements"></a>Требования лицензирования

Для использования этой функции требуется лицензия Azure AD Premium P2. 

Специализированные облака, такие как Azure Germany и Azure China 21Vianet, в настоящее время недоступны для использования.

## <a name="connect-the-partner-organization"></a>Подключение партнерской организации

Чтобы пригласить гостей из партнерской организации, необходимо добавить домен партнера как подключенную организацию в Azure Active Directory.

Добавление подключенной организации
1. В [Azure Active Directory щелкните](https://aad.portal.azure.com) **"Управление удостоверением".**
2. Щелкните **"Подключенные организации".**
4. Нажмите **кнопку "Добавить подключенную организацию".**
5. Введите имя и описание организации, а затем нажмите кнопку **"Далее: каталог + домен".**
6. Нажмите **кнопку "Добавить каталог+ домен".**
7. Введите домен для организации, которую нужно подключить, и нажмите кнопку **"Добавить".**
8. Нажмите **кнопку "Подключиться",** а затем нажмите **кнопку "Далее: спонсоры".**
9. Добавьте людей из вашей организации или организации, к которым вы подключаете тех, к кому вы хотите утвердить доступ для гостей.
10. Нажмите **кнопку "Далее": просмотр и создание.**
11. Просмотрите выбранные параметры и нажмите кнопку **"Создать".**

    ![Снимок экрана: страница подключенных организаций в Azure Active Directory](../media/identity-governance-connected-organizations.png)

## <a name="choose-the-resources-to-share"></a>Выбор ресурсов для обмена

Первым шагом при выборе ресурсов для обмена с партнерской организацией является создание каталога, в котором они будут содержаться.

Создание каталога
1. В [Azure Active Directory щелкните](https://aad.portal.azure.com) **"Управление удостоверением".**
2. Щелкните **каталоги.**
3. Щелкните **"Новый каталог"**.
4. Введите имя и описание каталога,  а также убедитесь, что для обоих этих ок есть "Да" и "Включено" для **внешних пользователей.** 
5. Нажмите **Создать**.

   ![Снимок экрана: страница каталогов в службе управления удостоверениями Azure Active Directory](../media/identity-governance-catalogs.png)

После создания каталога добавьте сайт или команду SharePoint, которые вы хотите поделиться с партнерской организацией.

Добавление ресурсов в каталог
1. В средстве управления удостоверениями Azure AD щелкните **"Каталоги",** а затем выберите каталог, в который требуется добавить ресурсы.
2. Щелкните **"Ресурсы"** и выберите **"Добавить ресурсы".**
3. Выберите группы или сайты SharePoint, которые вы хотите включить в экстрасеть, и нажмите кнопку **"Добавить".**

   ![Снимок экрана: страница ресурсов каталога в службе управления удостоверением Azure Active Directory](../media/identity-governance-catalog-resource.png)

После определения ресурсов, к которым вы хотите предоставить доступ, необходимо создать пакет доступа, который определяет тип доступа, который предоставляется пользователям-партнерам, и процесс утверждения новых пользователей партнеров, запрашивающих доступ.

Создание пакета доступа
1. В службе управления удостоверениями Azure AD щелкните **"Каталоги",** а затем выберите каталог, в котором требуется создать пакет доступа.
2. Щелкните **пакеты Access** и выберите **"Новый пакет доступа".**
3. Введите имя и описание пакета доступа и нажмите кнопку **"Далее: роли ресурсов".**
4. Выберите ресурсы из каталога, который вы хотите использовать для экстрасети.
5. Для каждого ресурса в столбце **"Роль"** выберите роль пользователя, которую вы хотите предоставить гостям, которые используют экстрасеть.
6. Нажмите **кнопку "Далее": запросы.**
7. Under **Users who can request access**, choose For users not in your **directory**.
8. Убедитесь, что выбран **параметр "Определенные подключенные** организации" и нажмите кнопку **"Добавить каталоги".**
9. Выберите подключенную организацию, которую вы добавили ранее, и нажмите кнопку **"Выбрать"**
10. В **области "Утверждение"** выберите **"Да** для **утверждения".**
11. В **области "Первый автор"** выберите одного из спонсоров, добавленных ранее, или выберите определенного пользователя.
12. Нажмите **кнопку "Добавить откат"** и выберите утвержденного.
13. В **области "Включить"** выберите **"Да".**
14. Нажмите **кнопку "Далее": жизненный цикл.**
15. Выберите параметры срока действия и проверки доступа, которые вы хотите использовать, а затем нажмите кнопку **"Далее: Просмотр+ Создать"**.
16. Просмотрите параметры и нажмите кнопку **"Создать".**

    ![Снимок экрана: экран пакетов доступа в службе управления удостоверениями Azure Active Directory](../media/identity-governance-access-packages.png)

Если вы сотрудничают с крупной организацией, вам может потребоваться скрыть пакет доступа. Если пакет скрыт, пользователи партнерской организации не увидят пакет на своем портале *My Access.* Вместо этого им необходимо отправить прямую ссылку для регистрации в пакете. Скрытие пакета доступа может уменьшить количество недопустимых запросов на доступ, а также помочь организовать доступные пакеты доступа на портале партнерской организации.

Настройка скрытого пакета доступа
1. В службе управления удостоверениями Azure AD щелкните **пакеты Access** и выберите пакет доступа.
2. На странице **"Обзор"** нажмите кнопку **"Изменить".**
3. В **области "Свойства"** выберите **"Да** **для скрытого"** и нажмите кнопку **"Сохранить".**

   ![Снимок экрана со свойствами пакета правки доступа](../media/identity-governance-access-package-hidden.png)

## <a name="invite-partner-users"></a>Приглашение партнеров

Если пакет доступа скрыт, необходимо отправить прямую ссылку партнерской организации, чтобы они могли запросить доступ к сайту или команде.

Поиск ссылки на портал доступа
1. В службе управления удостоверениями Azure AD щелкните **пакеты Access** и выберите пакет доступа.
2. На странице **"Обзор"** **щелкните ссылку "Копировать в буфер** обмена" для ссылки **портала "Мой доступ".**

   ![Снимок экрана: свойства пакета доступа со ссылкой на портал доступа](../media/identity-governance-access-portal-link.png)

После копирования ссылки вы можете поделиться ссылкой со своим контактом в партнерской организации, и они смогут отправить ее пользователям в своей группе совместной работы.
