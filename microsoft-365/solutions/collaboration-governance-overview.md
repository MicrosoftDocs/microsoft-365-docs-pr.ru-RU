# <a name="what-is-collaboration-governance"></a>Что такое управление совместной работой?

Управление совместной работой — это управление доступом пользователей к ресурсам, соответствием бизнес-стандартам и обеспечением безопасности данных.

В настоящее время в организациях используется разнообразный набор инструментов. Группа разработчиков использует командный чат, руководители, отправляющие электронную почту, и вся организация подключается через корпоративные социальные сети. Используется несколько средств совместной работы, так как каждая группа уникальна и имеет собственные функциональные потребности и стиль работы. Одни будут использовать только электронную почту, а другие — в основном в чате. 

Если пользователи считают, что средства, предоставляемые ИТ-службами, не соответствуют их потребностям, они, скорее всего, скачат свое любимое потребительское приложение, которое поддерживает их сценарии. Хотя этот процесс позволяет пользователям быстро начать работу, он вызывает затруднения в работе пользователей в организации с несколькими учетными данными, сложностями с совместным доступом и отсутствием единого места для просмотра контента. Это понятие называется "теневой ИТ-частью" и представляет серьезный риск для организаций. Это уменьшает возможность равномерного управления доступом пользователей, обеспечения безопасности и соответствия требованиям к службам.

Такие службы, как группы Microsoft 365, Teams и Yammer, расширяют возможности пользователей и снижают риск теневых ИТ-технологий, предоставляя средства, необходимые для совместной работы. Microsoft 365 имеет богатый набор средств для реализации любых возможностей управления, которые могут потребоваться вашей организации. 

![Диаграмма с вариантами управления совместной работой в Microsoft 365](../media/collaboration-governance-overview.png)

Эта серия статей поможет вам понять, как взаимодействуют группы, команды и параметры SharePoint, какие возможности управления доступны, а также как создать и внедрить план управления для функций совместной работы в Microsoft 365.

## <a name="what-are-microsoft-365-groups"></a>Что такое группы Microsoft 365?

Группы Microsoft 365 позволяют выбрать группу людей, с которыми вы хотите сотрудничать, и легко настроить коллекцию ресурсов, к которым эти люди будут делиться. При добавлении участников в группу автоматически предоставляются необходимые разрешения для всех активов, предоставленных группой. Как Teams, так и Yammer используют группы Microsoft 365 для управления их членством.

Группы Microsoft 365 включают набор связанных ресурсов, которые пользователи могут использовать для общения и совместной работы. Группы всегда включают сайт SharePoint, Планировщик, рабочее пространство Power BI, почтовый ящик и календарь, а также Stream. В зависимости от того, как вы создаете группу, можно при желании добавить другие службы, такие как Teams, Yammer и Project.

Группы Microsoft 365 включают различные средства управления, в том числе политику истечения срока действия, соглашения об именовом имени и политику заблокированных слов, чтобы помочь вам управлять группами в организации. Поскольку группы управляют членством и доступом к этому набору ресурсов, управление группами является ключевой частью управления совместной работой в Microsoft 365.

## <a name="define-a-collaboration-strategy-for-your-organization"></a>Определение стратегии совместной работы для организации

Существует несколько мест для совместной работы и бесед в Microsoft 365. Понимание того, где пользователи могут начинать беседы, поможет вам определить стратегию для общения.

Microsoft 365 поддерживает три основных способа связи:

- Outlook: совместная работа с помощью электронной почты с общей почтовой ящиком группы и календарем
- Microsoft Teams: рабочее пространство на основе сохраняемого чата, в котором можно общаться в неформальных беседах в режиме реального времени по различным темам, организованным по определенным подгруппам
- Yammer: корпоративный социальный опыт для совместной работы

![Схема использования Teams, Yammer и Outlook](../media/inner-loop-outer-loop.png)

- Teams: рабочее пространство на основе чата (совместная работа с высокой скоростью) — внутренний цикл
  - Создан для совместной работы с пользователями, с которых пользователи работают каждый день
  - Помещает информацию на кончиках пальцев пользователей в одном оке
  - Добавление вкладок, соединителю и ботов
  - Live chat, audio/video conferencing, recorded meetings

- Yammer: подключение через организацию (корпоративная социальная сеть) — внешний цикл
  - Сообщества практики — кроссфункциональные группы людей, которые делятся общими интересами или опытом, но не всегда работают вместе каждый день
  - Подключение руководства, учебные сообщества, сообщества на основе ролей

- Почтовый ящик и календарь (совместная работа по электронной почте)
  - Используется для целевого взаимодействия с группой людей
  - Общий календарь для собраний с другими участниками группы
 
При определении способа использования функций совместной работы в Microsoft 365 учитывайте эти способы связи и то, какие пользователи, скорее всего, будут использовать в различных сценариях.

> [!NOTE]
> Когда новая группа Office 365 создается через Yammer или Teams, она не отображается в Outlook или адресной книге, так как основная связь между этими пользователями происходит в соответствующих клиентах. Группы Yammer не могут быть подключены к Teams.


## <a name="best-practices"></a>Рекомендации

При запуске процесса планирования управления необходимо помнить о указанных здесь практических методиках.

- **По взаимодействию с пользователями** определите самых крупных пользователей функций совместной работы и познайте их основные бизнес-требования и сценарии использования.

- **Балансируйте риски и преимущества** : просмотрите бизнес,нормативы, юридические и нормативные требования и спланируйте решение, оптимизировав его для всех результатов.

-  Адаптируемся к различным организациям и различным типам контента и сценариев. Учитывайте различные потребности в разных группах или отделах, а также различные типы контента, например контент интрасети, по сравнению с содержимым OneDrive пользователя.

- **Соответствие бизнес-приоритетам.** Бизнес-цели помогут вам определить, сколько времени и энергии требуется для инвестиций в управление.

- **Внедряйте решения по** управлению непосредственно в решения, которые вы создаете. Многие решения по управлению можно реализовать, включив или отключив функции в Microsoft 365.

- **Подкрепите** себя обучением— адаптируем решения, такие как пути обучения [Microsoft 365,](https://docs.microsoft.com/office365/customlearning) чтобы убедиться, что ожидания вашей организации укрепляются с помощью обучения, предоставленного корпорацией Майкрософт.

- **Создайте стратегию** для обмена политиками и рекомендациями по управлению в своей организации— создайте Центр внедрения Microsoft 365 на информационном сайте SharePoint для обмена данными с политиками и процедурами.

- **Определите роли** и обязанности: определите основную команду управления и продумайте ключевые решения по управлению подготовками, именованиями и внешним доступом, а затем продумайте оставшиеся решения.

- **Пересматривайте решения по мере изменения** бизнеса и технологий. Периодически просматривайте новые возможности и новые бизнес-ожидания.

Подробнее об этих методиках можно узнать в документе ["Создание плана управления совместной работой".](collaboration-governance-first.md)

## <a name="end-user-impact-and-change-management"></a>Влияние и управление изменениями для конечных пользователей

Так как группы и команды можно создавать несколькими способами, мы рекомендуем обуять пользователей использовать метод, который лучше всего подходит для вашей организации:

- Если ваша организация большую часть общения использует электронную почту, учите пользователей создавать группы в Outlook.
- Если ваша организация активно использует SharePoint или переносит данные из локальной точки SharePoint, учите пользователей создавать сайты группы SharePoint для совместной работы.
- Если ваша организация развернула Teams, учите пользователей создавать команду, если им требуется пространство для совместной работы.

Это помогает избежать путаницы, если пользователи не знакомы с тем, как группы связаны со своими связанными службами. Дополнительные сведения о том, как общаться с пользователями о группах, см. в поясняет группы [Microsoft 365 для пользователей.](../admin/create-groups/explain-groups-knowledge-worker.md)

## <a name="collaboration-governance-planning-step-by-step"></a>Пошаговая пошаговая работа по планированию управления совместной работой

Чтобы создать план управления, выполните следующие основные действия.

1. Продумайте ключевые бизнес-цели и процессы— [создайте план](collaboration-governance-first.md) управления в целях удовлетворения потребностей вашего бизнеса.
2. В этой теме вы можете понять параметры в службах: параметры в группах и [SharePoint](groups-sharepoint-governance.md) взаимодействуют друг с другом, как и параметры в [группах, SharePoint, Teams](groups-sharepoint-teams-governance.md) и [других службах.](groups-services-interactions.md) Не забудьте понять эти взаимодействия при планировании стратегии управления.
3. Планирование управления доступом пользователей — спланируйте уровень доступа, который вы хотите предоставить пользователям в [группах, SharePoint и Teams.](groups-teams-access-governance.md)
4. Планирование управления настройками соответствия - просмотрите доступные варианты соответствия для групп [Microsoft 365, Teams и совместной работы SharePoint.](groups-teams-compliance-governance.md)
5. Планирование управления коммуникациями . Просмотрите доступные варианты управления [коммуникациями для сценариев совместной работы.](groups-teams-communication-governance.md)
6. Планирование управления организацией и жизненным циклом — выберите политики, которые вы хотите использовать для создания групп и команд, именования, истечения срока действия и [архивации.](plan-organization-lifecycle-governance.md) Кроме того, вы можете [понять, как закончить жизненный цикл групп, команд и Yammer.](end-life-cycle-groups-teams-sites-yammer.md)
