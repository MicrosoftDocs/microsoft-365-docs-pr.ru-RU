# <a name="what-is-collaboration-governance"></a>Что такое управление совместной работой?

Управление совместной работой — это управление доступом пользователей к ресурсам, соблюдением бизнес-стандартов и обеспечением безопасности данных.

Организации сегодня используют разнообразный набор инструментов. В команде разработчиков используется командный чат, руководители, отправляющие электронную почту, и вся организация, подключаемая к корпоративной социальной сети. Используется несколько средств совместной работы, так как каждая группа уникальна и имеет свои функциональные потребности и стиль работы. Некоторые из них будут использовать только электронную почту, а другие будут жить в основном в чате. 

Если пользователи считают, что предоставляемые ИТ-средства не соответствуют их потребностям, они, скорее всего, скачат свое любимое потребительское приложение, которое поддерживает их сценарии. Хотя этот процесс позволяет пользователям быстро начать работу, он приводит к разочарованию пользователей в организации с несколькими логинами, затруднением совместного доступа и отсутствием единого места для просмотра контента. Эта концепция называется "Shadow IT" и представляет значительный риск для организаций. Это снижает возможность единообразного управления доступом пользователей, обеспечения безопасности и соответствия требованиям к службам.

Такие службы, как группы Microsoft 365, Teams и Yammer, расширяют возможности пользователей и снижают риск теневых ИТ-служб, предоставляя средства, необходимые для совместной работы. Microsoft 365 обладает богатым набором средств для реализации любых возможностей управления, которые могут потребоваться вашей организации. 

![Диаграмма, на которой показаны параметры управления совместной работой в Microsoft 365](../media/collaboration-governance-overview.png)

Эта серия статей поможет вам понять, как взаимодействуют группы, группы и параметры SharePoint, какие возможности управления доступны, а также как создать и реализовать план управления для функций совместной работы в Microsoft 365.

### <a name="setting-up-secure-collaboration-with-microsoft-365"></a>Настройка безопасной совместной работы с Microsoft 365

Существует множество вариантов развертывания групп и групп Microsoft 365 для безопасного взаимодействия в организации. Мы рекомендуем использовать этот контент управления наряду с настройками безопасной совместной работы с [Microsoft 365](setup-secure-collaboration-with-teams.md) и связанными с ними статьями для создания наилучшего решения для совместной работы в организации.

## <a name="why-microsoft-365-groups-are-important"></a>Почему важны группы Microsoft 365

Группы Microsoft 365 позволяют выбрать набор людей, с которыми вы хотите сотрудничать, и легко настроить коллекцию ресурсов для обмена этими людьми. Добавление участников в группу автоматически предоставляет необходимые разрешения всем активам, предоставляемым группой. И Teams, и Yammer используют группы Microsoft 365 для управления своим членством.

Группы Microsoft 365 включают набор связанных ресурсов, которые пользователи могут использовать для общения и совместной работы. Группы всегда включают сайт SharePoint, планировщик, рабочее пространство Power BI, почтовый ящик и календарь и Stream. В зависимости от того, как создать группу, можно добавить другие службы, такие как Teams, Yammer и Project.

Microsoft 365 Groups включает различные элементы управления, в том числе политику истечения срока действия, конвенции имен и политику заблокированных слов, чтобы помочь вам управлять группами в организации. Поскольку группы контролируют членство и доступ к этому набору ресурсов, управление группами является ключевой частью управления совместной работой в Microsoft 365.

## <a name="define-a-collaboration-strategy-for-your-organization"></a>Определение стратегии совместной работы для организации

Существует несколько мест для совместной работы и бесед в Microsoft 365. Понимание того, где пользователи могут начать беседы, поможет вам определить стратегию общения.

Существует три основных способа коммуникации, поддерживаемых Microsoft 365:

- Outlook: совместная работа по электронной почте с общим почтовым ящиком группы и календарем
- Microsoft Teams: рабочее пространство на основе сохраняемого чата, где вы можете иметь неофициальные беседы в режиме реального времени по различным темам, организованные определенными подгруппами.
- Yammer: корпоративный социальный опыт для совместной работы

![Схема, показывающая, когда использовать Teams, Yammer и Outlook](../media/inner-loop-outer-loop.png)

- Команды: рабочее пространство на основе чата (совместная работа с высокой скоростью) — внутренний цикл
  - Создан для совместной работы с пользователями, с которых пользователи работают каждый день
  - Помещает сведения в один доступ пользователей в одном ок.
  - Добавление вкладок, соединителю и ботов
  - Живая беседа, аудио- и видеоконференция, записанные собрания

- Yammer: подключение через org (корпоративный социальный) — внешний цикл
  - Сообщества практики — межфункциональные группы людей, которые разделяют общие интересы или знания, но не всегда работают вместе на ежедневной основе.
  - Подключение к лидерству, обучающие сообщества, сообщества на основе ролей

- Почтовый ящик и календарь (совместная работа на основе электронной почты)
  - Используется для целевого общения с группой людей
  - Общий календарь собраний с другими участниками группы
 
При определении того, как использовать функции совместной работы в Microsoft 365, рассмотрите эти методы связи и их использование пользователями в различных сценариях.

> [!NOTE]
> Когда новая группа Office 365 создается с помощью Yammer или Teams, группа не отображается в Outlook или адресной книге, так как основная связь между этими пользователями происходит в соответствующих клиентах. Группы Yammer не могут быть подключены к Teams.


## <a name="best-practices"></a>Рекомендации

При запуске процесса планирования управления имейте в виду эти наилучшие практики:

- **Поговорите со своими пользователями** — определите крупнейших пользователей функций совместной работы и познакомьтесь с ними, чтобы понять их основные бизнес-требования и использовать сценарии.

- **Балансируйте риски** и преимущества — просмотрите бизнес,нормативные, юридические и нормативные требования и спланируйте решение, оптимизированое для всех результатов.

- **Адаптируемся** к различным организациям и различным типам контента и сценариев — учитывайте различные потребности для различных групп или отделов, а также различные типы контента, например контента интрасети по сравнению с контентом OneDrive пользователя.

- **Согласование с бизнес-приоритетами** — бизнес-цели помогут определить, сколько времени и энергии необходимо вложить в управление.

- **Встраивая решения** по управлению непосредственно в решения, которые вы создаете, многие решения по управлению можно реализовать, включив или отключив функции в Microsoft 365.

- **Укрепляйте** обучение — адаптируем такие решения, как пути обучения [Microsoft 365,](https://docs.microsoft.com/office365/customlearning) чтобы убедиться, что ваши ожидания в организации будут усилены с помощью обучения, предоставляемого Корпорацией Майкрософт.

- **У вас есть** стратегия для обмена политиками и рекомендациями по управлению в организации — создайте Центр принятия Microsoft 365 на сайте коммуникации SharePoint для обмена политиками и процедурами.

- **Определите роли** и обязанности — определите основную команду управления и проработать ключевые решения по управлению, в первую очередь подготовка, имя и внешний доступ, а затем проработать оставшиеся решения.

- **Пересмотр ваших решений по мере изменения** бизнеса и технологий — периодически встречайтеся, чтобы просмотреть новые возможности и новые бизнес-ожидания.

Подробнее об этих практиках читайте в материале Создание плана управления [совместной работой.](collaboration-governance-first.md)

## <a name="end-user-impact-and-change-management"></a>Управление влиянием на конечных пользователей и изменением

Так как группы и группы могут создаваться несколькими способами, рекомендуется обуять пользователей использовать метод, который лучше всего подходит вашей организации:

- Если ваша организация делает большую часть общения с помощью электронной почты, поручите пользователям создавать группы в Outlook.
- Если ваша организация активно использует SharePoint или мигрирует из локальной точки SharePoint, поручите пользователям создавать сайты команд SharePoint для совместной работы.
- Если ваша организация развернула Teams, поручите пользователям создать группу, если требуется пространство для совместной работы.

Это помогает избежать путаницы, если пользователи не знакомы с отношением групп к связанным службам. Дополнительные сведения о том, как общаться с пользователями о группах, см. в поясняет [группы Microsoft 365 для пользователей.](../admin/create-groups/explain-groups-knowledge-worker.md)

1. Рассмотрите ключевые бизнес-цели и процессы — [создайте план](collaboration-governance-first.md) управления для удовлетворения потребностей вашего бизнеса.
2. Понимание параметров в службах — параметры в группах и [SharePoint](groups-sharepoint-governance.md) взаимодействуют друг с другом, как и параметры в [группах, SharePoint и Teams](groups-sharepoint-teams-governance.md) и других [службах.](groups-services-interactions.md) Не забудьте понять эти взаимодействия при планировании стратегии управления.

### <a name="microsoft-365-information-protection-and-compliance-capabilities"></a>Возможности защиты и соответствия требованиям Microsoft 365

Microsoft 365 включает широкий набор возможностей защиты и соответствия требованиям. Вместе с средствами обеспечения производительности Майкрософт эти возможности предназначены для того, чтобы помочь организациям сотрудничать в режиме реального времени, придерживаясь строгих нормативных рамок соответствия требованиям.
