# <a name="groups-services-interactions"></a>Группирует взаимодействия служб

Группы Microsoft 365 предоставляют общую структуру для ряда служб и рабочих нагрузок на платформе Microsoft 365, чтобы обеспечить подключение для конечных пользователей. В основном существует группа Microsoft 365, которая предоставляет следующие компоненты:

- Способ управления членством (Azure AD)
- Место для обмена сообщениями и беседами (почтовый ящик Exchange, Microsoft Teams, Yammer)
- Место для хранения файлов (SharePoint)
- Календарь для планирования (Exchange)
- Записная книжка для записи заметок (OneNote)

В момент создания группы также подготавливается ряд других ресурсов, но они не видны, пока не будут доступны в первый раз из службы.

- Доска для управления задачами групп (планировщик)
- Рабочая область для отчетов (Power BI)
- Область для общих видеороликов (Microsoft Stream)
- Область для общих форм (форм)

В Microsoft 365 другие службы могут взаимодействовать с группами Microsoft 365, чтобы обеспечить дополнительные функции и возможности для групп.
В число таких примеров входят:

- Приложения для опытных приложений
- Автоматизация управления для рабочих процессов
- Проект в Интернете и схема управления каскадным управлением проектами
- Teams для бесед на основе каналов
- Yammer — сообщества, представляющие интерес

## <a name="user-interactions-with-groups"></a>Взаимодействие с пользователями с помощью групп

Группы Microsoft 365 могут создаваться и управляться с помощью различных интерфейсов, как администраторов, так и конечных пользователей. 

### <a name="administrative-experiences"></a>Административные возможности

Администраторы могут создавать группы Microsoft 365 и управлять ими из нескольких центров администрирования рабочей нагрузки, интерфейсов командной строки, поддерживающих сценарии, а также пользовательских приложений, взаимодействующих с API Graph. Единственным исключением являются группы Yammer, которые необходимо создать в веб-интерфейсе Yammer.

**Связанные параметры**

В различных административных интерфейсах, которые могут управлять параметрами групп, существует несколько перекрытий, о которых следует знать.

**Центр администрирования Microsoft 365**

В центре администрирования Microsoft 365 гостевой доступ к группам включен по умолчанию, как и в случае предоставления владельцам возможности добавлять гостей. Для групп из этого центра администрирования не предусмотрены другие элементы управления уровня Организации.

**Центр администрирования Azure AD**

Центр администрирования Azure AD позволяет управлять тем, могут ли пользователи создавать группы или назначать владельцев в порталах Azure, а также параметры истечения срока действия и политики именования.

В центре администрирования также представлен ряд мер по управлению гостевыми приглашениями, которые выходят за рамки центра администрирования Microsoft 365, например возможность ограничения того, могут ли владельцы, не являющиеся владельцами, приглашать гостей.

**SharePoint**

Сайты SharePoint создаются с помощью групп безопасности "владелец", "участник" и "Посетители", первые два из которых соответствуют их аналогам группы Microsoft 365. Несмотря на то, что членство в сайтах SharePoint Online обычно управляется связанной группой Microsoft 365, это не двунаправленное отношение. Любые изменения, внесенные в членство на уровне группы Microsoft 365, отражаются в SharePoint, но при изменении членства в группе SharePoint это не отражается в группе Microsoft 365.

### <a name="user-experiences"></a>Взаимодействие с пользователем

Конечные пользователи могут создавать группы из нескольких служб в Microsoft 365, и в других случаях они могут предоставлять доступ только группе.

В следующих службах разрешается создание групп с помощью конечных пользователей:
                         
Проект планировщика Outlook для потока веб-SharePoint в Yammer Microsoft Teams

**Ограничение создания групп**

Распространенный подход к управлению перемножеством Teams заключается в ограничении того, какие пользователи могут их создавать. Это можно сделать только путем ограничения создания групп. Это влияет на возможность создания групп из других служб, где это может потребоваться для конечного пользователя. Группы Microsoft 365 не поддерживают возможность ограничения создания групп из некоторых приложений или служб, но разрешать их от других приложений.

Возможности ограничения создания групп могут различаться в зависимости от приложений и служб:


|Приложение или служба|Интерфейс|
|:-------------|:---------|
|Outlook|Параметр " **создать группу** " удален из меню "создание" на странице "Пользователи"|
|Планировщик|**Новый план** объясняет, что создание групп отключено и предлагает добавить план в существующую группу.|
|Проект для веб-сайта и плана|Меню " **создать группу** " объясняет, что создание группы ограничено и предлагает использовать существующую группу.|
|SharePoint|Все еще можно создать сайт группы, не подключенный к группе.|
|Stream|Параметр **группы** не отображается в меню " **создать**".|
|Teams|Пользователь не может создать команду с новой группой, но по-прежнему сможет создать команду, использующую существующую группу.<br><br>Кнопка " **создать** группу" заменяется на **команду Создать группу**.|
|Yammer|Параметр " **создать группу** " удаляется из главной группы или структуры навигации.|

## <a name="azure-ad"></a>Azure AD

Azure AD предоставляет базовые возможности управления удостоверениями в Microsoft 365.

**Основные функции, предоставляемые группам**

- Участие в группе
- Политика именования
- Политика прекращения действия
- Guests
- Ограничение создания групп

**Можно создать группу в Azure AD.**

Да, группы Microsoft 365 можно создавать из Azure AD с помощью веб-портала администрирования PowerShell или API Graph.

**Существует ли Azure AD без группы?**

Да, Azure AD выполняет очень много служб, не имеющих отношения к группам Microsoft 365. Каждая группа Microsoft 365 представлена в виде объекта в Azure AD.

**Может ли быть несколько экземпляров Azure AD для каждой группы?**

Нет, существует только один экземпляр Azure AD.

**Можно ли связать Azure AD с несколькими группами?**

Да, так как Azure AD — это базовая платформа, предоставляющая службу членства в группах.

**Можно связать Azure AD с изменением группы?**

Нет, Azure AD — это базовая платформа, в которой существуют группы.

**Удаляет ли экземпляр эту группу?**

Удаление группы в Azure AD приведет к удалению соответствующих служб и контента, связанных с группой.

## <a name="teams"></a>Teams

Teams — это Рабочая область для общения в чате, предназначенная для улучшения совместной работы путем предоставления единственного интерфейса для взаимодействия со множеством служб Майкрософт и сторонних производителей.

По умолчанию при создании команды почтовый ящик и календарь, связанный с группой Microsoft 365, скрыты как в глобальном списке адресов в Exchange, так и в Outlook. Это может быть вручную переопределено администратором, если пользователь хочет использовать как Outlook, так и Microsoft Teams в одной и той же группе Microsoft 365.

**Основные функции, предоставляемые группам**

- Беседы
- Вкладки каналов &
- Собрания

**Можно создать группу с помощью Teams?**

Да, при создании новой команды будет создана новая группа Microsoft 365. Кроме того, можно создать команду для существующей группы, которая в настоящее время не существует.

**Существуют Teams без группы?**

Нет, команда не может существовать без группы.

**Может ли быть несколько команд для каждой группы?**

Нет, отношение между группой и группой равно 1:1.

**Может ли группа быть связана с несколькими группами?**

Нет, сама команда может быть связана только с одной группой.

**Может связь группы с изменением группы?**

Нет, команда может быть связана только с группой, с которой она была изначально связана.

**Удаляет ли группа удаление группы?**

Да, при удалении команды в Microsoft Teams будет удалена группа, связанные с группой службы и контент.

## <a name="exchange"></a>Exchange

Exchange Online предоставляет функции обмена сообщениями, календаря, контактов и связанных с ними функций. В контексте группы связывается только один ресурс — а не весь экземпляр службы.

**Основные функции, предоставляемые группам**

- Почтовый ящик и календарь
- Возможность отправлять по электронной почте всех участников группы
- Хранение бесед по каналам Teams для целей обнаружения электронных данных, комментариев в планировщике

**Может Exchange создать группу?**

Да, можно создать группу из центра администрирования Exchange Online, а также из Outlook. Вы также можете преобразовать списки рассылки Exchange в группы Microsoft 365.

**Существует ли Exchange без группы?**

Да, Exchange Online предоставляет ряд служб, в том числе общие почтовые ящики и календари, без сопоставления групп.

**Можно ли использовать несколько экземпляров почтовых ящиков или календарей Exchange для каждой группы?**

Нет, для группы может быть только один почтовый ящик Exchange Online и календарь.

**Могут ли почтовые ящики и календари Exchange быть связаны с несколькими группами?**

Нет, почтовый ящик и календарь имеют связь 1:1 с группой. Можно предоставить общий доступ к почтовому ящику другим пользователям или группам, но при этом не будет задана любая форма сопоставления служб.

**Можно ли изменить связь почтового ящика Exchange или календаря с изменением группы?**

Нет, почтовый ящик и календарь нельзя изменить на другую группу. Однако содержимое можно переместить из одного почтового ящика в другой в Outlook или с помощью стороннего средства.

**Удаляет ли почтовый ящик удаление группы?**

Да, при удалении почтового ящика в Exchange будет удалена группа, а также связанные с группой службы и контент.

## <a name="forms"></a>Forms

С помощью форм вы найдете веб-опросы и контрольные опросы.

**Основные функции, предоставляемые группам**

- Владение формами

**Могут создаваться формы группы?**

Нет, формы не могут создавать группы.

**Существуют формы без группы?**

Да, опросы и тесты можно создавать непосредственно в учетной записи конечного пользователя.

**Допускается ли несколько форм для каждой группы?**

Да, может быть несколько форм, связанных с группой.

**Могут ли формы быть связаны с несколькими группами?**

Нет, форма может быть связана только с одной группой.

**Можно выполнить связь формы с изменением группы?**

Нет, после того как форма связана с группой (созданной непосредственно в пределах или владельцем, передаваемой из отдельного пользователя), ее невозможно переместить в другую группу.

**Удаляется ли форма удалить группу?**

Нет, невозможно удалить группу из интерфейса форм, а только отдельные формы.

## <a name="onenote"></a>OneNote

OneNote — это приложение для цифровых записных книжек. Записная книжка OneNote, созданная с группой, это файл на связанном сайте SharePoint, а не в службе, подключенной к группе.

**Основные функции, предоставляемые группам**

- Общая записная книжка (хранится в связанной с группой SharePoint библиотеке SharePoint)

**Можно создать группу в OneNote?**

Нет, приложение OneNote не может создать группу.

**Существуют записные книжки OneNote без группы.**

Да, записные книжки можно создавать непосредственно в OneDrive или в других общих папках.

**Есть ли несколько записных книжек OneNote для каждой группы?**

Да, записная книжка создается по умолчанию, и другие пользователи могут добавляться, однако все ссылки на OneNote из служб, связанных с группами, всегда заключаются в записную книжку по умолчанию.

**Можно ли связать записную книжку OneNote с несколькими группами?**

Нет, записная книжка хранится в библиотеке сайтов SharePoint, связанной с группами, и связана с различными интерфейсами. Тем не менее, они могут быть предоставлены другим группам тем же способом, что и пользователи.

**Может измениться связь записной книжки с изменением группы?**

Нет, сама Записная книжка связана с группой и доступна напрямую из других служб, подключенных к группам, однако содержимое можно переместить из одной записной книжки в другую в приложении OneNote.

**Удаляется ли Записная книжка удалить группу?**

Нет, однако если Записная книжка OneNote удалена, в некоторых службах, связанных с группами, могут быть разорваны ссылки.

## <a name="planner"></a>Планировщик

Планировщик — это облегченная служба управления групповыми задачами.

**Основные функции, предоставляемые группам**

- Доска для управления задачами групп

**Может создать группу с помощью планировщика?**

Да, при создании плана будет создана новая группа.

**Существует ли доска планировщика без группы?**

Нет, план должен быть связан с группой.

**Допускается ли несколько планов для каждой группы?**

Да, может быть несколько планов для каждой группы.

**Может ли план быть связан с несколькими группами?**

Нет, план использует только сведения о членстве в группе для определения доступа.

**Можно связать план с изменением группы?**

Нет, но при копировании плана создается новая группа.

> [!NOTE]
> Группа, созданная любым другим приложением, не будет автоматически отображаться в планировщике для пользователя. Чтобы получить доступ к доске изначально, им потребуется открыть другой интерфейс на основе группы, например Outlook.

**Удаляет ли план удаление группы?**

Да, при удалении плана удаляются группы и связанные с группами службы и контент.

## <a name="power-apps"></a>Power Apps

Power Apps — это холст для разработки приложений без кода.

**Основные функции, предоставляемые группам**

- Доступ к приложениям можно предоставить группе для запуска и изменения

**Можно переключить приложение на группу?**

Нет, Power Apps не удается создать группу Microsoft 365.

**Существуют приложения для опытных приложений без группы?**

Да, приложения можно создавать в приложениях для управления питанием и хранить их в учетной записи Creators до предоставления общего доступа или публикации.

**Может ли быть несколько приложений для каждой группы?**

Да, существует несколько приложений, которым можно предоставить доступ к группе.

**Могут ли приложения связываться с несколькими группами?**

Да, к приложению можно предоставить доступ нескольким группам.

**Может быть связь приложения с изменением группы?**

Да, так как Ассоциация между приложениями для управления питанием и группой Microsoft 365 доступна только для совместного использования — приложение по-прежнему находится у создателя.

> Для [предоставления доступа к приложениям для групп необходимо включить систему безопасности](https://docs.microsoft.com/powerapps/maker/canvas-apps/share-app#share-an-app-with-office-365-groups).

**Удаляет ли приложение группу?**

Нет, приложения не подключаются к группе, кроме предоставления общего доступа к ним.

## <a name="power-automate"></a>Power Automate

Автоматизация управления питанием (ранее известной как Microsoft Flow) предоставляет рабочие процессы и службы автоматизации.

**Основные функции, предоставляемые группам**

- Рабочие процессы можно совместно использовать для запуска и изменения группы.

**Можно автоматизировать создание группы**

Нет, Power Автоматизация не может создать группу Microsoft 365 в контексте, связанной с одной.

Однако можно создать процесс, выполняющий различные операции, такие как создание группы безопасности Azure AD или обновление сведений о членстве в группе Microsoft 365.

**Существуют потоки без группы?**

Да, потоки можно создавать в автоматизированной среде и находиться в учетной записи Creators до предоставления общего доступа или публикации.

**Может ли быть несколько потоков для каждой группы?**

Да, можно использовать несколько потоков совместно с группой.

**Может ли последовательность быть связана с несколькими группами?**

Да, потоки могут совместно использоваться несколькими группами.

**Может связь с данным процессом с изменением группы?**

Да, так как Ассоциация между автоматизированной автоматизацией и группой Microsoft 365 доступна только для совместного использования — этот процесс по-прежнему располагается у создателя.

**Удаляет ли группа удаление группы?**

Нет, как и Power Apps, потоки не подключаются к группе, отличной от общего доступа к ним.

## <a name="power-bi-classic"></a>Power BI (классическая)

Power BI предоставляет интерактивные панели мониторинга и отчеты, управляемые данными.

**Основные функции, предоставляемые группам**

- Отчеты о данных

**Можно Power BI создать группу?**

Да, при создании классической рабочей области будет создана группа Microsoft 365.

**Существует ли классическая Рабочая область Power BI без группы?**

Нет, [Классическая Рабочая область в Power BI должна быть связана с группой](https://docs.microsoft.com/power-bi/collaborate-share/service-collaborate-power-bi-workspace).

**Можно ли использовать несколько рабочих областей Power BI для каждой группы?**

Нет, отношение между классической рабочей областью и группой составляет 1:1.

**Может ли Рабочая область быть связана с несколькими группами?**

С технической точки зрения, в то время как классическая Рабочая область создается вместе с группой, доступ к содержимому может осуществляться вне группы с пользователями и группами безопасности.

**Может измениться связь рабочей области с изменением группы?**

Нет, сама классическая Рабочая область связана с группой, однако содержимое можно переместить из одной рабочей области в другую в интерфейсе Power BI или путем локального экспорта содержимого.

**Удаляется ли Рабочая область удалить группу?**

Да, удаление рабочей области в Power BI приведет к удалению групп и служб, связанных с группами, и содержимым.

## <a name="power-bi-new"></a>Power BI (Новая)

Power BI предоставляет интерактивные панели мониторинга и отчеты, управляемые данными.

При создании рабочей области в Power BI не создается группа Microsoft 365, а создание группы по-другому означает создание новой (неклассической) рабочей области в Power BI.

**Основные функции, предоставляемые группам**

- Отчеты о данных

**Можно Power BI создать группу?**

Нет, невозможно создать группу Microsoft 365 из нового интерфейса Power BI.

**Существует ли новая Рабочая область Power BI без группы?**

Да, отчеты и рабочие области можно создавать в Power BI, которые не связаны с группами Microsoft 365.

**Может ли использоваться несколько рабочих областей для каждой группы?**

Да, [несколько рабочих областей, созданных Power BI, могут совместно использоваться одной группой](https://docs.microsoft.com/power-bi/collaborate-share/service-create-the-new-workspaces#give-access-to-your-workspace).

**Может ли Рабочая область быть связана с несколькими группами?**

Нет, Рабочая область, созданная Power BI, может быть связана только с одной группой.

**Может измениться связь рабочей области с изменением группы?**

Да и нет. Рабочая область, созданная Power BI, может быть связана только с одной группой в каждый момент времени, но может изменить связь в любое время. Рабочая область, созданная в Power BI группой, будет навсегда связана с этой группой.

**Удаляется ли Рабочая область удалить группу?**

Да, удаление рабочей области в Power BI приведет к удалению групп и служб, связанных с группами, и содержимым.

## <a name="sharepoint"></a>SharePoint

SharePoint — это веб-платформа управления контентом, обеспечивающая помимо прочего, Услуги хранения для ряда служб Microsoft 365.

**Основные функции, предоставляемые группам**

- Библиотека документов
- Библиотека для хранения записной книжки OneNote
- Хранение файлов вики-сайта Teams

**Можно создать группу SharePoint?**

Да, при создании сайта группы в SharePoint по умолчанию будет создана группа Microsoft 365. Кроме того, можно создать группу и (при необходимости) команду для существующего сайта.

**Существуют сайты SharePoint без группы?**

Да, SharePoint предоставляет ряд служб и сайтов, не связанных с группами, таких как информационные и центральные сайты. 

**Может быть несколько сайтов для каждой группы?**

Нет, может быть только один сайт для каждой группы. Частные каналы в Teams используют дополнительные сайты, не подключенные к группе.

**Могут ли сайты быть связаны с несколькими группами?**

Технически нет, но пока сайт создан с помощью группы, доступ к контенту можно предоставлять другим группам.

**Может измениться связь сайта с изменением группы?**

Нет, сам сайт связан с группой, но контент можно переместить с одного сайта на другой в интерфейсе SharePoint, экспортировав содержимое локально или с помощью стороннего средства.

**Удаляет ли сайт эту группу?**

Да, удаление сайта в SharePoint приведет к удалению служб и контента, связанных с группами и группами.

## <a name="stream"></a>Stream

Microsoft Stream — это платформа для хранения и совместного использования видео.

**Основные функции, предоставляемые группам**

- Хранилище видео
- Запись собрания Teams
- Видеоканалы

**Может Потокировать создание группы?**

Да, можно создать новую группу Microsoft 365 непосредственно из потока.

**Существует ли потоковая передача без группы?**

Да, видеоканалы и видеоролики могут находиться в потоке без связывания с группой.

**Можно ли использовать несколько видеороликов и каналов для каждой группы?**

Да, в каждой группе может быть несколько видеороликов и каналов.

**Может ли видео или канал быть связаны с несколькими группами?**

Да, несмотря на то, что видео или канал создается вместе с группой, к ним можно предоставить доступ другим группам.

**Можно выполнить связь с изменением группы?**

Да и нет; видео в потоке принадлежат исходному загрузчику или средству записи собраний, поэтому их можно связать с любой группой, но видеоканалы можно связать только с группой, в которой они были созданы.

**Удаляет ли группа видеоролики или каналы?**

Нет, удаление видео или каналов не приводит к удалению группы. Однако удаление самой группы в потоке приведет к удалению связанных с группой служб и контента, за исключением реальных видеороликов.

## <a name="yammer"></a>Yammer

Yammer — это корпоративная социальная платформа, предназначенная для ИТ-взаимодействия в организациях и между организациями.

Создание сообщества (прежнее название "Группа") в Yammer создает почтовый ящик, но на данный момент он не используется.

Группа Microsoft 365, связанная с Yammer, не может использоваться совместно с командой в Microsoft Teams.

**Основные функции, предоставляемые группам**

- Область беседы

**Можно создать группу Microsoft 365 в Yammer?**

Да, при создании новой группы в Yammer будет создана новая группа Microsoft 365, если платформы подключены и у пользователя есть возможность создать группу.

Группа Yammer с соответствующей группой Microsoft 365 не может быть создана ни в одном интерфейсе или службе, отличной от Yammer.

**Существует ли группа Yammer без группы Microsoft 365?**

Да, можно создать группу Yammer без группы Microsoft 365.

Если платформа Yammer не подключена к группам Microsoft 365 или пользователи не имеют возможности создавать группу Microsoft 365, группы Yammer создаются без сопоставления групп Microsoft 365.

**Может ли быть несколько групп Yammer для каждой группы Microsoft 365?**

Нет, отношение между группой Yammer и группой Microsoft 365 — 1:1.

**Можно ли связать группу Yammer с несколькими группами Microsoft 365?**

Нет, Группа Yammer может быть связана только с одной группой Microsoft 365. Записи можно предоставлять в другие группы Yammer или перемещать в нее.

**Может связь группы Yammer с изменением группы Microsoft 365?**

Нет, Группа Yammer может быть связана только с группой Microsoft 365, к которой она была изначально связана.

**Удаляет ли группа Yammer удаление группы Microsoft 365?**

Да, удаление группы в Yammer приведет к удалению связанных групп и связанных с группами служб и контента Майкрософт.
