# <a name="groups-services-interactions"></a>Взаимодействие служб групп

Группы Microsoft 365 предоставляют общую структуру для ряда служб и рабочих нагрузок на платформе Microsoft 365, чтобы обеспечить единый опытом для конечных пользователей. По своей сути существует группа Microsoft 365, которая предоставляет:

- Способ управления членством (Azure AD)
- Место для обмена сообщениями и бесед (почтовый ящик Exchange, Microsoft Teams, Yammer)
- Место хранения файлов (SharePoint)
- Календарь для планирования (Exchange)
- Записная книжка для записи заметок (OneNote)

На момент создания группы также можно получить ряд других ресурсов, однако они будут видны только после первого доступа из службы:

- Доска для управления групповыми задачами (Планировщик)
- Рабочее пространство для отчетов (Power BI)
- Область для общих видео (Microsoft Stream)
- Область для общих форм (формы)

В Microsoft 365 другие службы могут взаимодействовать с группами Microsoft 365, чтобы предоставлять участникам группы дополнительные функции и возможности.
Примеры этого:

- Power Apps для приложений
- Power Automate для рабочего процесса
- Project в Интернете и план управления проектами на основе каскадных проектов
- Teams для бесед на основе каналов
- Yammer для интересующие сообщества

## <a name="user-interactions-with-groups"></a>Взаимодействие пользователей с группами

Группы Microsoft 365 можно создавать и управлять с помощью различных интерфейсов, как администраторов, так и конечных пользователей. 

### <a name="administrative-experiences"></a>Административные процедуры

Администраторы могут создавать группы Microsoft 365 и управлять ими из нескольких центров администрирования рабочих нагрузок, интерфейсов командной строки, которые поддерживают сценарии, а также настраиваемые приложения, взаимодействующие с API Graph. Единственным исключением являются группы Yammer, которые должны создаваться из веб-интерфейса Yammer.

**Связанные параметры**

В различных административных интерфейсах, которые могут управлять групповыми настройками, существует несколько перекрытий, о которых следует помнить.

**Центр администрирования Microsoft 365**

В Центре администрирования Microsoft 365 гостевой доступ к группам включен по умолчанию, как и возможность добавления гостей владельцами. В этом Центре администрирования нет дополнительных элементов управления на уровне организации, доступных для групп.

**Центр администрирования Azure AD**

В Центре администрирования Azure AD есть элементы управления, которые могут создавать группы или назначать владельцев на порталах Azure, а также параметры политики истечения срока действия и именования.

Центр администрирования также предоставляет ряд мер по контролю приглашений гостей, которые выходят за рамки центра администрирования Microsoft 365, например возможность ограничить возможность приглашения гостей, не внося в группу владельцев.

**SharePoint**

Сайты SharePoint создаются с помощью групп безопасности "Владелец", "Участник" и "Посетитель", первые два из них совпадают с группами Microsoft 365. Хотя членством на сайтах SharePoint Online, как правило, управляет связанная группа Microsoft 365, это не является отношением между участниками. Любые изменения членства на уровне группы Microsoft 365 отражаются в SharePoint, но если членство в группе SharePoint изменилось, это не отражается в группе Microsoft 365.

### <a name="user-experiences"></a>Пользовательские интерфейсы

Конечные пользователи могут создавать группы из нескольких служб в Microsoft 365, а в других могут делиться только с группой.

Следующие службы позволяют создавать группы конечными пользователями:
                         
Outlook Planner Project for the web SharePoint Stream Microsoft Teams Yammer

**Ограничение создания группы**

Распространенный подход к контролю спрайтов групп заключается в ограничении того, какие пользователи могут их создавать. Это можно сделать, ограничив создание групп. Это влияет на возможность создания групп из других служб, где это может потребоваться для конечных пользователей. Группы Microsoft 365 не поддерживают возможность ограничить создание групп из одних приложений или служб, разрешив их другим.

Работа с ограничениями на создание групп зависит от приложений и служб:


|Приложение или служба|Интерфейс|
|:-------------|:---------|
|Outlook|**Параметр "Новая** группа" удален из меню "Новый" на странице "Люди"|
|Планировщик|**В новом** плане объясняется, что создание группы отключено, и предлагается добавить план в существующую группу|
|Проект в Интернете и план|**В меню создания** группы объясняется, что создание группы ограничено, и предлагается использовать существующую группу.|
|SharePoint|По-прежнему можно создавать сайт группы, не подключенный к группе.|
|Stream|**Параметр** "Группа" не появляется в меню **"Создать".**|
|Teams|Пользователь не может создать команду с новой группой, но все равно может создать команду, которая использует существующую группу.<br><br>**Кнопка "Создать команду"** заменена кнопкой **"Создать команду из группы".**|
|Yammer|**Создание параметра** группы удаляется из навигации основных групп или сообществ.|

## <a name="azure-ad"></a>Azure AD

Azure AD предоставляет возможности управления удостоверениями в Microsoft 365.

**Основные функции, предоставляемые группам**

- Участие в группе
- Политика именования
- Политика прекращения действия
- Гости
- Ограничение создания группы

**Может ли Azure AD создать группу?**

Да, группы Microsoft 365 можно создавать из Azure AD на веб-портале администрирования, с помощью PowerShell или API Graph.

**Существует ли Azure AD без группы?**

Да, Azure AD выполняет большое количество служб, которые не имеют отношения к группам Microsoft 365. Каждая группа Microsoft 365 представлена в Azure AD как объект.

**Может ли быть несколько экземпляров Azure AD для одной группы?**

Нет, существует только один экземпляр Azure AD.

**Можно ли связывать Azure AD с несколькими группами?**

Да, так как Azure AD является платформой, которая предоставляет службу членства в группах.

**Можно ли изменить связь Azure AD с группой?**

Нет, Azure AD — это платформа, на которой существуют группы.

**Удаляет ли экземпляр группу?**

При удалении группы в Azure AD удаляются соответствующие службы и контент, связанные с группой.

## <a name="teams"></a>Teams

Teams — это рабочее пространство на основе чата, предназначенное для улучшения совместной работы, предоставляя единый интерфейс для взаимодействия с различными службами Майкрософт и сторонних служб.

По умолчанию при составлении команды почтовый ящик и календарь, связанные с группой Microsoft 365, скрыты как в глобальном списке адресов в Exchange, так и в Outlook. Это может быть переопределено администратором вручную, если пользователь хочет использовать Outlook и Teams в одной группе Microsoft 365.

**Основные функции, предоставляемые группам**

- Беседы
- Каналы & вкладок
- Собрания

**Можно ли создать группу в Teams?**

Да, при создании новой команды будет создаваться новая группа Microsoft 365. Кроме того, можно создать команду для существующей группы, которая в настоящее время не имеет ее.

**Существуют ли команды без группы?**

Нет, команда не может существовать без группы.

**Может ли быть несколько команд на группу?**

Нет, отношение между командой и группой — 1:1.

**Можно ли связывать команду с несколькими группами?**

Нет, сама команда может быть связана только с одной группой.

**Может ли измениться связь команды с группой?**

Нет, команда может быть связана только с группой, с которой она была изначально связана.

**Удаляет ли команда группу?**

Да, удаление команды в Microsoft Teams удалит группу, связанные с группой службы и содержимое.

## <a name="exchange"></a>Exchange

Exchange Online предоставляет функции обмена сообщениями, календаря, контактов и связанных с ними функций. В контексте группы связан только один ресурс, а не весь экземпляр службы.

**Основные функции, предоставляемые группам**

- Почтовый ящик и календарь
- Возможность отправлять сообщения всем участникам группы по электронной почте
- Хранение бесед канала Teams для целей eDiscovery, комментарии Планировщика

**Может ли Exchange создать группу?**

Да, можно создать группу в Центре администрирования Exchange Online, а также в Outlook. Списки рассылки Exchange также можно преобразовать в группы Microsoft 365.

**Существует ли Exchange без группы?**

Да, Exchange Online предоставляет ряд служб, включая общие почтовые ящики и календари, без связи с группой.

**Может ли быть несколько экземпляров почтовых ящиков Или календарей Exchange для одной группы?**

Нет, для группы может быть только один почтовый ящик и календарь Exchange Online.

**Можно ли связывать почтовые ящики и календари Exchange с несколькими группами?**

Нет, почтовый ящик и календарь имеют отношение 1:1 с группой. Можно поделиться почтовым ящиком с другими пользователями или группами, но это не устанавливает связь службы.

**Может ли измениться связь почтового ящика Или календаря Exchange с группой?**

Нет, почтовый ящик и календарь нельзя изменить на другую группу. Однако содержимое можно перемещать из одного почтового ящика в другой в Outlook или с помощью стороне средства.

**Удаляет ли почтовый ящик группу?**

Да, при удалении почтового ящика в Exchange удаляется группа, а также связанные с группой службы и содержимое.

## <a name="forms"></a>Forms

Формы предоставляют веб-опросы и тесты.

**Основные функции, предоставляемые группам**

- Владение формами

**Могут ли формы создать группу?**

Нет, формы не могут создать группу.

**Существуют ли формы без группы?**

Да, опросы и тесты можно создавать непосредственно в учетной записи конечного пользователя.

**Может ли быть несколько форм для группы?**

Да, с группой может быть связано несколько форм.

**Можно ли связывать формы с несколькими группами?**

Нет, форму можно связывать только с одной группой.

**Может ли измениться связь формы с группой?**

Нет, когда форма связана с группой (либо создается непосредственно внутри, либо владельцем, переданным от физического лица), ее нельзя перенести в другую группу.

**Удаляет ли форма группу?**

Нет, удалить группу из интерфейса Forms невозможно, только отдельные формы.

## <a name="onenote"></a>OneNote

OneNote — это приложение цифровой записной книжки. Записная книжка OneNote, созданная с помощью группы, — это файл на связанном сайте SharePoint, а не в службе, подключенной к группе.

**Основные функции, предоставляемые группам**

- Общая записная книжка (хранится в библиотеке SharePoint, связанной с группой)

**Может ли OneNote создать группу?**

Нет, приложение OneNote не может создать группу.

**Существуют ли записные книжки OneNote без группы?**

Да, записные книжки можно создавать непосредственно в OneDrive или в других общих расположениях.

**Может ли быть несколько записных книги OneNote для одной группы?**

Да, записная книжка создается по умолчанию, и можно добавлять другие записные книжки, однако любая ссылка на OneNote из связанных с группой служб всегда будет переходить к записной книжке по умолчанию.

**Можно ли связывать записную книжку OneNote с несколькими группами?**

Нет, записная книжка хранится в связанной с группой библиотеке сайтов SharePoint и связывается с различными интерфейсами. Однако он может быть общим для других групп таким же образом, как он может быть общим для отдельных пользователей.

**Может ли связь записной книжки с группой измениться?**

Нет, сама записная книжка связана с группой и может быть напрямую связана с другими службами, подключенными к группе, однако содержимое можно перемещать из одной записной книжки в другую в приложении OneNote.

**Удаляет ли записная книжка группу?**

Нет, но если записная книжка OneNote удалена, в некоторых службах, связанных с группой, могут быть нарушены ссылки.

## <a name="planner"></a>Планировщик

Планировщик — это облегченная служба управления задачами группы.

**Основные функции, предоставляемые группам**

- Доска для управления групповыми задачами

**Может ли Планировщик создать группу?**

Да, при создании плана будет создаваться новая группа.

**Существует ли доска Планировщика без группы?**

Нет, план должен быть связан с группой.

**Может ли быть несколько планов на группу?**

Да, для группы может быть несколько планов.

**Можно ли связывать план с несколькими группами?**

Нет, для определения доступа план зависит только от членства в группе.

**Может ли связь плана с группой измениться?**

Нет, однако при копировании плана создается новая группа.

> [!NOTE]
> Группа, созданная любым другим приложением, не будет автоматически отключаться в Планировщике для пользователя. Чтобы получить доступ к доске изначально, им потребуется открыть его из другого группового интерфейса, например Outlook.

**Удаляет ли план группу?**

Да, при удалении плана будут удалены службы и контент, связанные с группой.

## <a name="power-apps"></a>Power Apps

Power Apps предоставляет холст для разработки приложений без кода.

**Основные функции, предоставляемые группам**

- Приложения можно совместно использовать для запуска и изменения в группе

**Могут ли Power Apps создать группу?**

Нет, Power Apps не может создать группу Microsoft 365.

**Существуют ли приложения Power Apps без группы?**

Да, приложения можно создавать в Power Apps и находиться в учетной записи создателей до тех пор, пока не будет опубликован или опубликован общий доступ.

**Может ли быть несколько приложений на группу?**

Да, группа может использовать несколько приложений.

**Можно ли связывать приложения с несколькими группами?**

Да, приложение может совместно работать с несколькими группами.

**Может ли связь приложения с группой измениться?**

Да, так как связь между Power Apps и группой Microsoft 365 заключается только в совместном использовании — приложение по-прежнему находится у создателя.

> [!IMPORTANT]
> [Группы должны быть включены для обеспечения безопасности, прежде чем к ним можно будет получить доступ к приложениям.](https://docs.microsoft.com/powerapps/maker/canvas-apps/share-app#share-an-app-with-office-365-groups)

**Удаляет ли приложение группу?**

Нет, приложения не подключены к группе, кроме общего с ними.

## <a name="power-automate"></a>Power Automate

Power Automate (прежнее название — Microsoft Flow) предоставляет рабочий процесс и службы автоматизации.

**Основные функции, предоставляемые группам**

- Для запуска и изменения рабочего процесса можно совместно использовать группу.

**Может ли Power Automate создать группу?**

Нет, Power Automate не может создать группу Microsoft 365 в контексте ее связи.

Однако можно создать поток, который выполняет различные операции, такие как создание группы безопасности Azure AD или обновление членства в группе Microsoft 365.

**Существуют ли потоки без группы?**

Да, потоки можно создавать в Power Automate и находиться в учетной записи создателей до тех пор, пока они не будут опубликованы или опубликованы.

**Может ли быть несколько потоков на группу?**

Да, группа может использовать несколько потоков.

**Можно ли связывать поток с несколькими группами?**

Да, поток может быть общим для нескольких групп.

**Может ли измениться связь потока с группой?**

Да, так как связь между Power Automate и группой Microsoft 365 заключается только в совместном использовании — поток по-прежнему находится с создателем.

**Удаляет ли поток группу?**

Нет, как и в Power Apps, потоки не подключены к группе, кроме общего с ними.

## <a name="power-bi-classic"></a>Power BI (классическая)

Power BI предоставляет интерактивные панели мониторинга и отчеты на основе данных.

**Основные функции, предоставляемые группам**

- Отчеты о данных

**Может ли Power BI создать группу?**

Да, при создании классической рабочей области будет создаваться группа Microsoft 365.

**Существует ли классическая рабочая область Power BI без группы?**

Нет, [классическая рабочая область в Power BI должна быть связана с группой.](https://docs.microsoft.com/power-bi/collaborate-share/service-collaborate-power-bi-workspace)

**Может ли быть несколько рабочей области Power BI для группы?**

Нет, отношение между классической рабочей областью и группой — 1:1.

**Можно ли связывать рабочее пространство с несколькими группами?**

Технически нет, хотя классическая рабочая область создается вместе с группой, доступ к содержимому за пределами группы может быть общим для пользователей и групп безопасности.

**Может ли меняться связь рабочей области с группой?**

Нет, с группой связана сама классическая рабочая область, однако содержимое можно перемещать из одной рабочей области в другую в интерфейсе Power BI или экспортировать содержимое локально.

**Удаляет ли рабочая область группу?**

Да, при удалении рабочей области в Power BI удаляются службы и контент, связанные с группой.

## <a name="power-bi-new"></a>Power BI (новый)

Power BI предоставляет интерактивные панели мониторинга и отчеты на основе данных.

При создании новой рабочей области в Power BI группа Microsoft 365 не создается, а создание группы другими средствами создает новую (не классическую) область в Power BI.

**Основные функции, предоставляемые группам**

- Отчеты о данных

**Может ли Power BI создать группу?**

Нет, создать группу Microsoft 365 из нового интерфейса Power BI невозможно.

**Существует ли новая рабочая область Power BI без группы?**

Да, в Power BI можно создавать отчеты и рабочей области, не связанные с группами Microsoft 365.

**Может ли быть несколько рабочей области для группы?**

Да, [несколькими рабочей областью, созданными с](https://docs.microsoft.com/power-bi/collaborate-share/service-create-the-new-workspaces#give-access-to-your-workspace)помощью Power BI, можно совместно использовать одну группу.

**Можно ли связывать рабочее пространство с несколькими группами?**

Нет, рабочая область, созданная Power BI, может быть связана только с одной группой.

**Может ли меняться связь рабочей области с группой?**

Да и нет. Рабочая область, созданная Power BI, может быть связана только с одной группой одновременно, но может изменить связь в любое время. Рабочая область, созданная в Power BI группой, навсегда связана с этой группой.

**Удаляет ли рабочая область группу?**

Да, при удалении рабочей области в Power BI удаляются службы и контент, связанные с группой.

**Может ли связь проекта с группой измениться?**

## <a name="sharepoint"></a>SharePoint

SharePoint — это веб-платформа управления контентом, которая, помимо прочего, предоставляет службы хранения для ряда служб Microsoft 365.

**Основные функции, предоставляемые группам**

- Библиотека документов
- Библиотека для хранения записной книжки OneNote
- Хранилище вики-файлов Teams

**Может ли SharePoint создать группу?**

Да, при создании сайта группы в SharePoint по умолчанию создается группа Microsoft 365. Кроме того, можно создать группу и при желании команду для существующего сайта.

**Существуют ли сайты SharePoint без группы?**

Да, SharePoint предлагает ряд не связанных с группой служб и сайтов, таких как сайты связи и концентраторы. 

**Может ли быть несколько сайтов на группу?**

Нет, на группу может быть только один сайт. Частные каналы в Teams используют дополнительные сайты, которые не подключены к группе.

**Можно ли связывать сайты с несколькими группами?**

Технически нет, но во время создания сайта с группой контент можно совместно использовать с другими группами.

**Может ли связь сайта с группой измениться?**

Нет, сам сайт связан с группой, однако контент можно перемещать с одного сайта на другой в интерфейсе SharePoint, экспортировать контент локально или с помощью сторонного средства.

**Удаляет ли сайт группу?**

Да, при удалении сайта в SharePoint удаляются службы и контент, связанные с группой.

## <a name="stream"></a>Stream

Microsoft Stream — это платформа для размещения видео и общего доступа.

**Основные функции, предоставляемые группам**

- Хранилище видео
- Запись собрания Teams
- Видеоканалы

**Может ли Stream создать группу?**

Да, можно создать группу Microsoft 365 непосредственно из Stream.

**Существует ли Stream без группы?**

Да, видеоканалы и видео могут существовать в Stream без связи с группой.

**Может ли быть несколько видео и каналов для группы?**

Да, в каждой группе может быть несколько видео и каналов.

**Можно ли связывать видео или канал с несколькими группами?**

Да, хотя видео или канал создается с группой, его можно совместно использовать с другими группами.

**Может ли ее связь с группой измениться?**

Да и нет; видео в Stream принадлежат исходному загрузителю или записи собраний, поэтому их можно связывать с любой группой, однако каналы видео могут быть связаны только с группой, в котором они изначально были созданы.

**Удаляет ли группа видео или каналы?**

Нет, при удалении видео или каналов группа не удаляется. Однако при удалении самой группы в Stream удаляются связанные с группой службы и контент, за исключением фактических видео.

## <a name="yammer"></a>Yammer

Yammer — это корпоративная социальная платформа, призванная способствовать вовлечению сообщества в организации и между ними.

При создании сообщества (прежнее название — "группа") в Yammer создается почтовый ящик, но в настоящее время это не используется.

Группу Microsoft 365, связанную с Yammer, нельзя использовать с командой в Microsoft Teams.

**Основные функции, предоставляемые группам**

- Область беседы

**Может ли Yammer создать группу Microsoft 365?**

Да, при создании новой группы в Yammer будет создаваться новая группа Microsoft 365, если платформы подключены и пользователь может создать группу.

Группу Yammer со связанной группой Microsoft 365 нельзя создать в интерфейсе или службе, кроме самой Yammer.

**Существует ли группа Yammer без группы Microsoft 365?**

Да, можно создать группу Yammer без группы Microsoft 365.

Если платформа Yammer не подключена к группам Microsoft 365 или пользователи не могут создать группу Microsoft 365, группы Yammer создаются без связи с группами Microsoft 365.

**Может ли быть несколько групп Yammer для группы Microsoft 365?**

Нет, отношение между группой Yammer и группой Microsoft 365 составляет 1:1.

**Можно ли связывать группу Yammer с несколькими группами Microsoft 365?**

Нет, группу Yammer можно связывать только с одной группой Microsoft 365. Публикации могут быть общими или перенесены в другие группы Yammer.

**Может ли связь группы Yammer с группой Microsoft 365 измениться?**

Нет, группу Yammer можно связывать только с группой Microsoft 365, с которой она была изначально связана.

**Удаляет ли группа Yammer группу Microsoft 365?**

Да, при удалении группы в Yammer будут удалены связанные службы и контент, связанные с группой Майкрософт.
