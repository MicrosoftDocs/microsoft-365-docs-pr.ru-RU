# <a name="manage-auto-claim-policies"></a>Управление политиками автоматического утверждения

Политика автоматического утверждения позволяет пользователям автоматически требовать лицензию на продукт при первом входе в приложение. Как администратор обычно вы назначаете лицензии пользователям вручную или с помощью группового лицензирования. Используя политики автоматического утверждения, вы управляете продуктами, для которых пользователи могут автоматически претендовать на лицензии. Вы также можете контролировать, какие продукты из этих лицензий.

После создания политики автоматического утверждения можно выполнять следующие задачи для управления политикой:

- [Включите или отключите политику](#turn-a-policy-on-or-off)
- [Изменение удобного имени политики](#edit-the-policy-friendly-name)
- [Добавление или удаление продуктов резервного копирования](#add-or-remove-backup-products)
- [Управление назначением приложений и служб](#change-the-assigning-apps-and-services)
- [Изменение порядка назначения](#change-the-assigning-order-for-backup-products)
- [Просмотр отчета политики](#view-an-auto-claim-policy-report)

> [!IMPORTANT]
> Политики автоматического утверждения в настоящее время доступны только для Microsoft Teams. Дополнительные продукты будут доступны для использования в будущем.

## <a name="before-you-begin"></a>Подготовка к работе

Вы должны быть глобальным администратором, администратором биллинга или пользователя для создания и управления политиками автоматического утверждения. Дополнительные сведения см. в статье [Роли администраторов в Microsoft 365](../../admin/add-users/about-admin-roles.md).

## <a name="turn-the-auto-claim-policy-feature-on-or-off"></a>Включить или отключить функцию политики автоматического утверждения

По умолчанию функция политики автоматического утверждения отключена. Прежде чем использовать эту функцию, сначала необходимо включить ее. После включив функцию, можно создать политику автоматического утверждения.

### <a name="turn-on-auto-claim-policies"></a>Включаем политики автоматического утверждения

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. В центре страницы выберите кнопку **Включив параметр.**

### <a name="turn-off-auto-claim-policies"></a>Отключение политик автоматического утверждения

1. В центре администрирования перейдите на страницу **Параметры** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=2053743" target="_blank">Org.</a>
2. В нижней части таблицы выберите приложения и службы, которые принадлежат **пользователю.**
3. В правой области очистить поле для автоматического утверждения пользователями лицензий при **первом входе**.

Если у вас уже есть активная политика, но вы не хотите, чтобы больше пользователей претендовали на лицензии, [отключите политику.](#turn-a-policy-on-or-off) Когда вы отключите политику автоматического утверждения, больше пользователи не смогут претендовать на лицензию с этого момента. Пользователи, которые уже претендовали на лицензию, не теряют лицензию.

## <a name="create-an-auto-claim-policy"></a>Создание политики автоматического утверждения

На <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">вкладке Политика автоматического</a> утверждения перечислены политики, которые вы создаете. На этой вкладке можно увидеть: имя политики, приложение, связанное с политикой, продукт, который назначен политике, количество доступных лицензий и состояние политики.

При создании политики автоматического утверждения можно добавить в нее резервный продукт. Если основной продукт не имеет лицензий, резервный продукт используется для назначения лицензий пользователям. Можно добавить до четырех резервных продуктов и [изменить порядок](#change-the-assigning-order-for-backup-products)их использования. Дополнительные дополнительные информации см. в [добавлении или удаление продуктов резервного копирования.](#add-or-remove-backup-products)

> [!NOTE]
> В настоящее время можно создать только одну политику автоматического утверждения. Количество политик, которые можно создать, будет увеличиваться по мере того, как больше продуктов смогут использовать эту функцию.

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите **Добавить политику**.
3. На странице **Имя этой политики автоматического утверждения** введите имя политики, а затем выберите **Далее**.
4. На странице **Set an auto-claim app and product** select an app and the subscription to assign licenses from.
5. Если вы хотите добавить резервный продукт, выберите **Добавить** резервный продукт в эту политику, а затем выберите продукт из списка.
6. Нажмите кнопку **Далее**.
7. На странице **Выберите приложения,** очистить или выбрать поля для приложений, чтобы исключить или включить с лицензией, а затем выберите **Далее**.
8. Если добавлен один или несколько продуктов резервного копирования, повторите шаг 7 для каждого продукта. В противном случае перейдите к шагу 9.
9. На странице **Просмотреть и закончить** проверить новые сведения политики, внести необходимые изменения, а затем выбрать **Создать политику**.
10. Нажмите **Закрыть**.

## <a name="turn-a-policy-on-or-off"></a>Включите или отключите политику

Когда вы отключите политику, больше пользователи не смогут претендовать на лицензии в соответствии с этой политикой. Это изменение не влияет на пользователей, которые уже претендовали на лицензии в соответствии с этой политикой.

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области подробных сведений в статье **Включите** или отключите эту политику, выберите или очистите поле.
4. Выберите **Сохранить,** чтобы закрыть области сведений.

## <a name="edit-the-policy-friendly-name"></a>Изменение удобного имени политики

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области сведений в разделе **Имя политики** выберите **Изменить**.
4. Введите новое имя политики, а затем выберите **Сохранить**.
5. Выберите **Сохранить,** чтобы закрыть области сведений.

## <a name="add-or-remove-backup-products"></a>Добавление или удаление продуктов резервного копирования

При создании политики к ней добавляется продукт. Затем лицензии автоматически назначены пользователям из этого пула лицензий. Вы можете добавлять или удалять продукты для политики автоматического утверждения в любое время. Если у вас уже есть один продукт, связанный с политикой, все добавленные продукты считаются продуктами резервного копирования. Когда используется доступное количество лицензий первого продукта, политика использует следующий продукт резервного копирования в списке для назначения лицензий. Список [продуктов можно переупланить по](#change-the-assigning-apps-and-services) мере на то, что вам нравится.

Когда вы удаляете резервный продукт, он больше не используется для назначения лицензий. Пользователи с существующей лицензией по-прежнему имеют эту лицензию, но новые пользователи не могут получать лицензии для этого продукта.

> [!NOTE]
> Политика автоматического утверждения должна содержать по крайней мере один продукт. Вы не можете удалить все продукты из политики. Если вы больше не хотите назначать лицензии из определенной политики автоматического утверждения, [отключите политику.](#view-an-auto-claim-policy-report)

### <a name="add-a-backup-product"></a>Добавление резервного продукта

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области сведений в нижней части выберите Добавить резервный продукт **в эту политику.**
    > [!NOTE]
    > Если вы не видите эту ссылку, это потому, что у вас есть только один продукт, связанный с учетной записью.
4. В **области Добавить продукт используйте** выпадаю, чтобы выбрать продукт, чтобы добавить в политику, а затем **выберите Добавить**.
5. Выберите **Сохранить,** чтобы закрыть области сведений.

### <a name="remove-a-backup-product"></a>Удаление резервного продукта

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области сведений в нижней части выберите **Удаление продукта.**
4. В поле **Удаление продукта из области** политики выберите поле для политики, которую необходимо удалить, а затем выберите **Сохранить**.
5. Закрой области сведений.

## <a name="change-the-assigning-apps-and-services"></a>Изменение назначения приложений и служб

Каждый продукт имеет коллекцию приложений и служб, связанных с ним.
Для каждого продукта в политике автоматического утверждения можно указать, какие приложения и службы необходимо включить, когда пользователю автоматически назначена лицензия на этот продукт.

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области сведений в статье **Приложения и службы** выберите **Изменить**.
4. В области **Приложений и служб** выберите  один продукт или все **продукты.**
5. Проверка или очистка полей для приложений и служб, к которые пользователи должны иметь или не иметь доступа.
6. По завершению выберите **Сохранить,** а затем закроем области сведений.

## <a name="change-the-assigning-order-for-backup-products"></a>Изменение порядка назначения продуктов резервного копирования

Если для политики назначены резервные продукты, можно изменить порядок назначения лицензий при входе пользователей в приложение.

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите политику, которую необходимо изменить.
3. В области подробных сведений в разделе **Лицензии** на продукт выберите поле рядом с продуктом, который необходимо переместить, а затем выберите **Move up** or **Move down.**
4. Повторите шаг 3 для каждого продукта, который необходимо переуступить.
5. Когда вы закончите перенастройку продуктов, выберите **Сохранить,** чтобы закрыть области сведений.

## <a name="view-an-auto-claim-policy-report"></a>Просмотр отчета о политике автоматического утверждения

1. В центре администрирования перейдите на страницу **"Лицензии** на выставление счета", а затем \>  выберите вкладку <a href="https://go.microsoft.com/fwlink/p/?linkid=2134398" target="_blank">политика автоматического утверждения.</a>
2. Выберите **отчет View**. На **странице отчета о политике автоматического** утверждения перечислены все лицензии, присвоенные каждой политике за последние 90 дней. По умолчанию на странице показаны последние 90 дней.
3. Чтобы изменить показанный период времени, выберите выпадаемый список за **последние 30** дней. Отчеты можно просматривать за последние 1, 7, 30 и 90 дней.
