# <a name="allotment-basics"></a>Основы реализации

Назначения лицензий могут устанавливать ограничения лицензий и делегировать управление назначением лицензий только выбранным продуктам и лицензиям.

Для назначения лицензий пользователям используются групповые лицензии. Ограничения лицензий обеспечивают дополнительный контроль над количеством лицензий, которые назначены пользователям в группах. Поэтому даже по мере увеличения количества пользователей в группах вы можете следить за тем, чтобы вы оставались в пределах лимита лицензий, установленного для вашего лимита.

Вы также можете делегировать управление своими ресурсами. Владельцы делегирования получают доступ к Центру администрирования, но могут видеть и управлять лицензиями только в своих собственных лицензиях. Это обеспечивает более детальное делегирования управления лицензией в организации.

## <a name="prerequisites"></a>Предварительные условия

Необходимо соответствовать требованиям лицензирования для [группового лицензирования.](https://docs.microsoft.com/azure/active-directory/fundamentals/active-directory-licensing-whatis-azure-portal#licensing-requirements)

Вы можете использовать средства для любого продукта, доступного пользователям:

- Наборы Office и автономные продукты
- Корпоративные и мобильные продукты
- Продукты Dynamics 365

С назначениями нельзя использовать следующие продукты:

- Приложения Microsoft Store
- Бессрочные программы или программное обеспечение, которое напрямую назначено пользователю, если лицензия не задействована.
- Ресурсы Azure

Для начала работы с этим процессом необходимо быть глобальным администратором или администратором лицензии.
