# <a name="understand-the-proposal-workflow"></a>Понимание рабочего процесса предложения

Предложение — это формальное предложение корпорации Майкрософт для вашей организации приобрести продукты и службы Майкрософт. Вы работаете напрямую с представителем Майкрософт, чтобы определить конкретные продукты, службы и условия для вашего предложения.

Представитель Корпорации Майкрософт с черновиками предлагает предложения, которые содержат обсуждаемые вами и вашим представителем элементы. Представитель отправляет вам сообщение электронной почты со ссылкой на сайт предложений. Сайт содержит предложение, подготовленное специально для вас и вашей организации.

Получив уведомление по электронной почте, пере следуйте ссылке на сайт предложений. После того как вы войуете на сайт, вы можете начать процесс проверки предложений.

## <a name="prerequisites-for-buying-items-with-a-proposal"></a>Необходимые условия для покупки элементов с предложением

Перед покупкой элементов для предложения необходимо иметь счет и соглашение с корпорацией Майкрософт.

### <a name="billing-account"></a>Учетная запись для вычета

Для управления настройками учетной записи, счетами, профилями выставления счетов, продуктами и службами используется учетная запись выставления счетов. Если у вас еще нет учетной записи для вычета счетов, ваш представитель Майкрософт создаст ее для вас.
В противном случае они используют существующую учетную запись для вашей организации, если у вас есть разрешение на использование этой учетной записи.

Разрешения учетной записи вы выставления счетов управляются владельцем учетной записи вы выставления счетов.
Глобальные администраторы могут назначить себя роли владельца учетной записи вы выставления счетов, а затем сделать владельцем учетной записи для вы выставления счетов других людей.

Дополнительные сведения об учетных записях вы выставления счетов см. в под [управлением учетных записей для вычета счетов.](manage-billing-accounts.md)

### <a name="microsoft-customer-agreement"></a>Соглашение с клиентом Майкрософт

Соглашение с клиентом Майкрософт (MCA) позволяет организации приобретать продукты и службы Майкрософт. Дополнительные сведения см. в [соглашении с клиентом Майкрософт.](https://www.microsoft.com/en-us/Licensing/how-to-buy/microsoft-customer-agreement)

## <a name="permissions-needed-to-sign-an-agreement-or-pay-for-items"></a>Разрешения, необходимые для подписания соглашения или оплаты элементов

Если у вас нет назначенной роли в учетной записи вычета, при просмотре предложения вам будет назначена базовая роль читателя. Эта роль позволяет просматривать предложение, но не принимать никаких действий с этим предложением. Чтобы подписать соглашение или приобрести продукты и услуги, вам должна быть назначена роль владельца учетной записи вывески или участника счета. Владелец вашей учетной записи вы выставления счетов может назначить эту роль вам.

Дополнительные сведения о ролях учетных записей для вы выставления счетов см. в сведениях о [доступе к учетным записям вы выставления счетов.](manage-billing-accounts.md#understand-access-to-billing-accounts)

Если это новая учетная запись для вычета, и никто не принял соглашение, вы автоматически станете владельцем учетной записи вычета при условии, что:

- Имя человека в предложении
    ИЛИ
- Уже являются глобальным [администратором Azure Active Directory](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-assign-admin-roles) в организации

## <a name="what-is-the-overall-workflow"></a>Каков общий рабочий процесс?

Общий рабочий процесс предложений выглядит так:

- Ваш представитель Майкрософт создает предложение и отправляет вам ссылку по электронной почте.

- Используйте эту ссылку, чтобы перейти на страницу регистрации предложений.

- Просмотрите сведения о вашей организации.

- Вы просматриваете предложение, при необходимости принимаете MCA и завершаете процесс иска.

    > [!IMPORTANT]
    > У вас должны быть полномочия подписывать MCA от имени вашей организации. Если у вас нет такого полномочия, то кто-то должен сделать это.

- После завершения проверки вам будут предоставлены дополнительные ссылки на возможность настроить продукты и службы.

Профиль выставления счетов включает имя счета для выставления счетов, способы оплаты, используемые для определенного профиля выставления счетов, контактные данные, параметры счета и разрешения, которые могут изменять профиль выставления счетов, оплачивать счета и приобретать продукты и услуги. |
| Существующие соглашения | Любое соглашение, уже подписанное вашей организацией с корпорацией Майкрософт. Это может включать, но не ограничивается, соглашением Соглашение Enterprise microsoft Product & Services или Соглашением с клиентом Майкрософт. |
| Соглашение с клиентом Майкрософт (MCA) | Соглашение, которое описывает условия учетной записи, удерживаемой вашей организацией, с корпорацией Майкрософт. |

| Дополнительные термины | Внести изменения в MCA, содержащие термины, специфификовременные для вашей организации. Чтобы принять дополнительные условия, необходимо использовать DocuSign для записи электронной подписи. |

## <a name="step-1-review-organization-information"></a>Шаг 1. Просмотр сведений об организации

После этого сначала просмотрите сведения о вашей организации.

### <a name="your-organization"></a>Ваша организация

В **разделе "Организация"** отображается связанная с ней учетная запись для вы выставления счетов. Сведения об учетной записи вы выставления счетов извлекаются из существующей учетной записи или создаются для вас представителем Майкрософт. Если ваша организация является аффилированной организацией  другой организации, вы также увидите раздел "Ведущие организации" с именем и адресом этой организации.

Если это первый раз, когда ваша организация устанавливает коммерческие отношения с Корпорацией Майкрософт и вы  еще не  подписали MCA, если сведения из вашей организации или ведущие организации неправильны, обратитесь к представителю, чтобы внести за вас изменения. После принятие MCA вы можете просмотреть и изменить адрес и контактные данные организации на странице "Учетные записи вы выставления счетов" в Центре администрирования Microsoft 365. [](https://go.microsoft.com/fwlink/p/?linkid=2084771) Если имя организации изменится, откройте запрос на обслуживание, чтобы обновить его. [Узнайте, как открыть запрос на обслуживание](../admin/contact-support-for-business-products.md)

### <a name="your-information"></a>Ваши сведения

If you're a new customer, enter your name, email address, and phone number under **Your information**, then select **Save**. Если вы — существующий клиент, убедитесь, что ваши сведения верны. Чтобы внести изменения, выберите **"Изменить",**"Внести необходимые изменения" и **"Сохранить".**

Когда все будет готово, выберите **"Продолжить,** чтобы перейти к следующему шагу".

## <a name="step-2-review-proposal"></a>Шаг 2. Обзор предложения

Предложение содержит сведения о пунктах, которые вы обсуждаете с вашим представителем Майкрософт. Вы можете переадлить сообщение электронной почты со ссылкой на предложение, чтобы поделиться им с другими заинтересованными лицами в вашей организации. Любой другой, кто использует ссылку, имеет представление предложения только для чтения.

Если вы хотите внести какие-либо изменения в предложение после проверки, обратитесь к представителю Майкрософт.

### <a name="proposal-contents"></a>Содержимое предложения

Предложение содержит следующие сведения:

| Раздел | Description |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Название организации | Название организации, для которой было подготовлено предложение. |
| Действительна до даты | Дата истечения срока действия предложения. Если вы пропустите эту дату истечения срока действия, обратитесь к представителю Корпорации Майкрософт, чтобы он сообщил ему, что вы по-прежнему заинтересованы в предложении. |
| Валюта | Валюта, используемая для расчета стоимости элементов в предложении. |
| Подготовлено к | Имя, адрес, контактный адрес электронной почты и номер телефона пользователя, запросившего предложение. |
| Подготовлено | Адрес электронной почты представителя Майкрософт, который подготовил предложение. |
| Аннотация | Показывает подстановки, связанные с предложением. При необходимости в нем также будет демонстрироваться валютная ставка, используемая для расчета затрат. |
| Элементы строки предложения | В этом разделе содержится количество, цена за единицу и подстановка всех элементов, включенных в предложение. |
| Следующий этап | В этом разделе указаны необходимые действия. |

Чтобы подписать MCA, выберите кнопку в области **"Следующий шаг".** Если необходимо подписать дополнительные термины, ссылка перенальжет вас на сайт DocuSign, где вы выполните действия, чтобы подписать документ.

После подписания всех необходимых соглашений или дополнительных условий выберите **"Перейти для инауки".**

## <a name="step-3-checkout"></a>Шаг 3. Проверка

Страница инаучения содержит следующие разделы:

### <a name="sold-to"></a>Покупатель

В этом разделе показана учетная запись вы выставления счетов, используемая для предложения. Если вам нужно изменить какие-либо сведения, выберите ссылку **"Изменить".** Вы также можете использовать **ссылку** "Изменить", чтобы добавить налоговый ИД организации. Налоговый ИД должен быть связан со странаю, перечисленной в разделе **"Продано".** Если у вас есть освобождение от уплаты налога, необходимо открыть запрос в службу поддержки, чтобы запросить освобождение от уплаты налога.

Дополнительные сведения об ИД налога и о том, как подать заявку на освобождение от уплаты налога, см. в [сведениях о налогах.](billing-and-payments/tax-information.md)

### <a name="billed-to"></a>Выставлен счет

В этом разделе показан профиль выставления счетов, используемый для определения того, какие элементы включены в счет и как вы платите по счетам. Каждый цикл выставления счетов получает отдельный счет для каждого профиля выставления счетов. Вы платите за счета с помощью чека или перевода по проводной сети или предоплаты Azure. Если у вас еще нет профиля вы выставления счета, ваш представитель Майкрософт создаст его для вас. Во время инауки можно выбрать другой профиль вы выставления счета, если он у вас есть, изменить имя профиля вы выставления счета или добавить P.O. number. Вы также можете создать новый профиль вы выставления счета.

Сведения о профилях вы выставления счета см. в под [управлением профилей вы выставления счета.](billing-and-payments/manage-billing-profiles.md)

### <a name="proposal-items-in-this-order"></a>Элементы предложений в этом порядке

В этом разделе показан список всех элементов, включенных в предложение. Список может включать одну или несколько из следующих категорий:

- **Дополнительные термины** Список изменений в MCA, содержащих термины для вашей организации. Например, этот список может включать термины HIPAA или GDPR.

- **Приобрести** Список элементов, которые вы платите во время инауки в конце рабочего процесса принятия предложения.

- **Скидки (применяются к будущим платежам)** Список скидок, которые вы получаете в рамках предложения.

- **Включено** Список элементов, включенных в пакет предложений без дополнительной платы. Некоторые из этих элементов могут быть связаны с затратами в будущем.

### <a name="summary"></a>Аннотация

В этом разделе показано количество уплачиваемых элементов, итоговая сумма, предполагаемые налоги и общая сумма заказа.

Чтобы разместить заказ, выберите **place order** или Accept agreement & **place order**.

После заказа вы получите подтверждение с последующими действиями. Если вы приобрели план Azure, следующим шагом будет настройка учетной записи вычета на портале Azure.

## <a name="step-4-set-up-your-new-billing-account-azure-customers-only"></a>Шаг 4. Настройка новой учетной записи вы выставления счетов (только для клиентов Azure)

Если вы — новый клиент и приобрели продукты Azure в рамках предложения, следующим шагом будет настройка новой учетной записи вы выставления счетов. Чтобы узнать, как это сделать, см. в подстройку учетной записи вы выставления счетов [для соглашения с клиентом Майкрософт.](https://docs.microsoft.com/azure/cost-management-billing/manage/mca-setup-account)

Если у вас есть клиент Azure с Соглашение Enterprise и вы подписываете MCA в первый раз, следующим шагом будет узнать об изменениях между соглашениями и о том, как выполнять задачи с помощью новой учетной записи вы выставления счетов. Дополнительные данные см. в Соглашение Enterprise задач в учетной записи вы выставления счетов в [рамках соглашения с клиентом Майкрософт.](https://docs.microsoft.com/azure/cost-management-billing/manage/mca-enterprise-operations)

## <a name="understand-invoicing"></a>В этой теме выявить

После исмотра и завершения заказа в течение 24–48 часов отправляется первоначальный счет. После этого вы получаете счета-фактуры 5-го месяца. Ежемесячный счет содержит платежи за предыдущий месяц. Если у вас есть какие-либо кредиты для вашей учетной записи, они вычитаются из денежных четов вашего профиля выставления счетов и применяются к вашему счету. Оставшийся баланс после кредиты является сроком оплаты. У вас есть 30 дней с даты выставления счетов, чтобы оплатить счет.

Инструкции по оплате для отправки чеков или банковских переводов включены в PDF-копию счета. Чтобы просмотреть или скачать счет, см. свой [счет или счет.](billing-and-payments/view-your-bill-or-invoice.md)
