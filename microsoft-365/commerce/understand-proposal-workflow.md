# <a name="understand-the-proposal-workflow"></a>Понимание рабочего процесса предложения

Предложение — это официальное предложение корпорации Майкрософт для вашей организации о покупке продуктов и служб Майкрософт. Вы напрямую работаете с представителем Майкрософт, чтобы определить конкретные продукты, службы и условия для вашего предложения.

Представитель Корпорации Майкрософт подготовил предложение, содержа которое содержит элементы, которые вы и ваш представитель обсудили. Представитель отправляет вам сообщение электронной почты со ссылкой на сайт предложения. Сайт содержит предложение, подготовленное специально для вас и вашей организации.

После получения электронной почты уведомления выполните ссылку на сайт предложения. После регистрации на сайте можно приступить к процессу рассмотрения предложений.

## <a name="prerequisites-for-buying-items-with-a-proposal"></a>Необходимые условия для покупки элементов с предложением

Перед покупкой элементов для предложения необходимо иметь учетную запись и соглашение с Корпорацией Майкрософт.

### <a name="billing-account"></a>Учетная запись биллинга

Вы используете учетную запись для управления настройками учетной записи, счетами, профилями выставления счетов, а также продуктами и службами. Если у вас еще нет учетной записи для вы выставления счетов, ваш представитель Майкрософт создает ее для вас.
В противном случае они используют существующую учетную запись для вашей организации, если у вас есть разрешение на использование этой учетной записи.

Разрешения на выставление счетов управляются владельцем учетной записи для вы выставления счетов.
Глобальные администраторы могут назначить себя роли владельца учетной записи для вы выставления счетов, а затем сделать других владельцев счетов счетов.

Дополнительные сведения об учетных записях вы выставления счетов см. в [руб. Управление счетами.](manage-billing-accounts.md)

### <a name="microsoft-customer-agreement"></a>Соглашение о клиентах Майкрософт

Соглашение о клиентах Майкрософт (MCA) позволяет организации покупать продукты и службы Майкрософт. Дополнительные сведения см. в [сайте Microsoft Customer Agreement.](https://www.microsoft.com/en-us/Licensing/how-to-buy/microsoft-customer-agreement)

## <a name="permissions-needed-to-sign-an-agreement-or-pay-for-items"></a>Разрешения, необходимые для подписания соглашения или оплаты элементов

Если у вас нет назначенной роли в учетной записи для выставления счетов, при просмотре предложения вам назначена базовая роль читателя. Эта роль позволяет просматривать предложение, но не принимать никаких действий. Перед подписанием соглашения или покупкой продуктов и услуг необходимо на назначении владельцу учетной записи для вычета счетов или роли вкладчика счетов. Владелец учетной записи вычета может назначить вам эту роль.

Дополнительные сведения о роли учетных записей для биллинга см. в рублях ["Понимание доступа к учетным записям с выставлением счетов".](manage-billing-accounts.md#understand-access-to-billing-accounts)

Если это новая учетная запись для биллинга, и никто не принял соглашение, вы автоматически станете владельцем учетной записи для вычета при условии:

- Имя человека в предложении\
    OR
- Уже являются глобальным [администратором Azure Active Directory для](/azure/active-directory/users-groups-roles/directory-assign-admin-roles) вашей организации

## <a name="what-is-the-overall-workflow"></a>Каков общий рабочий процесс?

Общий рабочий процесс предложения выглядит так:

- Ваш представитель Майкрософт создает предложение и отправляет ссылку на вас по электронной почте.
- Вы используете ссылку, чтобы перейти на страницу регистрации предложения.
- Вы просматривайте сведения организации.
- Вы просмотрите предложение, при необходимости примите MCA и завершите процесс проверки.
    > [!IMPORTANT]
    > Вы должны иметь право подписывать MCA от имени организации. Если у вас нет такого права, то кто-то должен сделать этот шаг.
- После завершения проверки вам будут предоставлены дополнительные ссылки на настройка продуктов и служб.

## <a name="proposal-terms"></a>Условия предложения

В следующей таблице содержатся термины и определения, которые отображаются в вашем предложении и на сайте предложения.

| **Термин** | **Определение** |
|------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Учетная запись биллинга | Учетная запись, используемая для управления настройками учетной записи, счетами, методами оплаты и продуктами. |
| Профиль биллинга | Сведения о вашей организации, которая позволяет настраивать элементы, включенные в ваш счет, и оплату счетов. Профиль выставления счетов включает имя учетной записи выставления счетов, методы оплаты, используемые для определенного профиля выставления счетов, контактные данные, параметры счетов и разрешения, которые могут изменять профиль выставления счетов, оплачивать счета и покупать продукты и услуги. |
| Существующие соглашения | Любое соглашение, которое ваша организация уже заключила с Корпорацией Майкрософт. Это может включать, но не ограничивается, соглашением Соглашение Enterprise microsoft product & services или Microsoft Customer Agreement. |
| Соглашение о клиентах Майкрософт (MCA) | Соглашение, которое описывает условия и условия учетной записи, удерживаемой вашей организацией с Корпорацией Майкрософт. |
| Представитель Microsoft | Уполномоченный представитель Корпорации Майкрософт, который готовит предложение для вас и вашей организации. |
| Организация | Юридическое лицо, использующее продукты, технологии или службы Майкрософт. |
| Подготовлено | Адрес электронной почты представителя Майкрософт, который подготовил предложение. |
| Дополнительные термины | Поправки к MCA, содержащие термины, специфические для организации. Чтобы принять дополнительные термины, необходимо использовать DocuSign для записи электронной подписи. |

## <a name="step-1-review-organization-information"></a>Шаг 1. Просмотр сведений об организации

После регистрации первое, что необходимо сделать, это просмотреть сведения организации.

### <a name="your-organization"></a>Ваша организация

В **разделе Ваша организация** отображается связанная с ней учетная запись с выставлением счетов. Сведения об учетной записи для выставления счетов либо извлекаются из существующей учетной записи счетов, либо создаются для вас представителем Корпорации Майкрософт. Если ваша организация является филиалом другой организации, вы также увидите раздел **Lead organization** с именем и адресом этой организации.

Если ваша организация впервые устанавливает коммерческие отношения с Корпорацией Майкрософт, и вы еще не подписали  MCA,  если сведения в организации или организации Lead неверны, обратитесь к представителю для внесения изменений. После того как вы приняли MCA, вы можете просмотреть и изменить [](https://go.microsoft.com/fwlink/p/?linkid=2084771) адрес организации и контактные данные на странице учетных записей биллинга в центре администрирования Microsoft 365. Если имя организации меняется, откройте запрос на обновление службы. [Узнайте, как открыть запрос на службу](../business-video/get-help-support.md)

### <a name="your-information"></a>Ваши сведения

Если вы новый клиент, введите свое имя, адрес электронной почты и номер телефона в соответствии с вашими сведениями, а затем выберите **Сохранить**. Если вы уже существующий клиент, убедитесь, что ваша информация верна. Чтобы внести какие-либо исправления, **выберите Изменить,** внести необходимые изменения, а затем выберите **Сохранить**.

Когда вы будете готовы, выберите **Продолжить** переход к следующему шагу.

## <a name="step-2-review-proposal"></a>Шаг 2. Обзор предложения

В предложении содержатся сведения о предметах, которые вы обсудили с вашим представителем Майкрософт. Вы можете переадлить сообщение со ссылкой на предложение, чтобы поделиться им с другими заинтересованными сторонами в вашей организации. Любой другой, кто использует ссылку, имеет только для чтения представление предложения.

Если вы хотите внести какие-либо изменения в предложение после проверки, обратитесь к представителю Майкрософт.

### <a name="proposal-contents"></a>Содержимое предложения

Предложение содержит следующие сведения:

| Section | Описание |
|---------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Название организации | Имя организации, для которой было подготовлено предложение. |
| Допустимо до даты | Дата истечения срока действия предложения. Если вы пропустили этот срок действия, обратитесь к представителю Майкрософт, чтобы дать им знать, что вы по-прежнему заинтересованы в предложении. |
| Денежный | Валюта, используемая для расчета стоимости элементов в предложении. |
| Подготовлено для | Имя учетной записи, адрес, контактный адрес электронной почты и номер телефона человека, запросившего предложение. |
| Подготовлено | Адрес электронной почты представителя Майкрософт, который подготовил предложение. |
| Сводка | Отображает подсказок, связанный с предложением. При необходимости он также будет показывать курс иностранной валюты (FX), используемый для расчета затрат. |
| Элементы строки предложения | В этом разделе содержится количество, цена единицы и subtotal всех элементов, включенных в предложение. |
| Следующий шаг | В этом разделе указаны необходимые действия, которые необходимо принять. |

Чтобы подписать MCA, выберите кнопку в **статье Next Step**. Если вы должны подписать дополнительные условия, ссылка отвещает вас к сайту DocuSign, где вы выполните действия по подписанию документа.

После подписания необходимых соглашений или дополнительных условий выберите **Go to checkout.**

## <a name="step-3-checkout"></a>Шаг 3. Проверка

Страница проверки содержит следующие разделы:

### <a name="sold-to"></a>Покупатель

В этом разделе показана учетная запись для выставления счетов, используемая для предложения. Если вам нужно изменить любую информацию, выберите ссылку **Изменить.** Вы также можете использовать ссылку **Изменить,** чтобы добавить налоговый ID организации. Налоговый ID должен быть связан со страной, перечисленной в разделе **Продано в** раздел. Если у вас есть освобождение от уплаты налогов, необходимо открыть билет на поддержку, чтобы запросить статус, освобождаемый от налогов.

Дополнительные сведения о налоговых ИД и о том, как подать заявление на освобождение от налогообложения, см. [в сведениях о налогах.](billing-and-payments/tax-information.md)

### <a name="billed-to"></a>Выставлен счет

В этом разделе показан профиль выставления счетов, используемый для определения элементов, включенных в счет-фактуру, и оплаты счетов. Каждый цикл выставления счетов получает отдельный счет для каждого профиля выставления счетов. Вы оплачиваете счета с помощью чека или переноса провода или предоплаты Azure. Если у вас еще нет профиля биллинга, ваш представитель Майкрософт создает его для вас. Во время проверки можно выбрать другой профиль биллинга, если он у вас есть, изменить имя профиля биллинга или добавить P.O. номер. Вы также можете создать новый профиль вы выставления счета.

Сведения о профилях вы выставления счета см. в [странице Управление профилями биллинга.](billing-and-payments/manage-billing-profiles.md)

### <a name="proposal-items-in-this-order"></a>Элементы предложения в этом порядке

В этом разделе показан список всех элементов, включенных в предложение. Список может включать одну или несколько следующих категорий:

- **Дополнительные термины** Список любых поправок к MCA, содержащих термины для организации. Например, этот список может включать термины HIPAA или GDPR.
- **Покупка сейчас** Список элементов, за которые вы платите во время проверки в конце рабочего процесса принятия предложения.
- **Скидки (применимые к будущим платежам)** Список скидок, которые вы получаете в рамках предложения.
- **Включено** Список элементов, включенных в пакет предложений без дополнительной платы. Некоторые из этих элементов могут иметь затраты, связанные с ними в будущем.

### <a name="summary"></a>Сводка

В этом разделе показано количество уплачиваемых элементов, подсобные, сметные налоги и общая сумма для заказа.

Чтобы разместить заказ, выберите **порядок "Место"** или **"Принять" &amp; порядок места соглашения.**

После на месте заказа вы получите подтверждение с последующими действиями. Если вы приобрели план Azure, следующим шагом будет настройка учетной записи счетов на портале Azure.

## <a name="step-4-set-up-your-new-billing-account-azure-customers-only"></a>Шаг 4. Настройка новой учетной записи вы выставления счетов (только для клиентов Azure)

Если вы новый клиент и приобрели продукты Azure в рамках предложения, следующим шагом будет настройка новой учетной записи счетов. Подробнее о том, как настроить учетную запись вы выставления счетов [для соглашения с клиентами Майкрософт.](/azure/cost-management-billing/manage/mca-setup-account)

Если вы существующий клиент Azure с Соглашение Enterprise, и вы подписываете MCA впервые, следующим шагом будет узнать об изменениях между соглашениями и о том, как выполнять задачи с помощью новой учетной записи счетов. Дополнительные дополнительные данные см. в Соглашение Enterprise задач в учетной записи вы выставления счетов [для соглашения о клиентах Майкрософт.](/azure/cost-management-billing/manage/mca-enterprise-operations)

## <a name="understand-invoicing"></a>Понимание взимаемой

После проверки и выполнения заказа начальный счет отправляется в течение 24-48 часов. После этого вы получаете счета-фактуры примерно 5-го месяца. Ежемесячный счет содержит сборы за предыдущий месяц. Если у вас есть какие-либо кредиты для вашей учетной записи, они вычитаются из денежных кредитов вашего профиля выставления счетов и применяются к балансу счетов. Оставшийся баланс после того, как будут применены кредиты, — это должен быть баланс. У вас есть 30 дней с даты выставления счетов для оплаты счета.

Инструкции по оплате для отправки чеков или переводов с провода включены в копию PDF-счета. Чтобы просмотреть или скачать счет, см. в этой ссылке [Просмотр счета или счета.](billing-and-payments/view-your-bill-or-invoice.md)
