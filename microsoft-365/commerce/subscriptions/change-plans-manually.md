# <a name="change-plans-manually"></a>Изменение планов вручную

## <a name="step-1-decide-how-to-change-plans"></a>Шаг 1. Решение об изменении планов

Лучший способ изменить всех пользователей из одного плана в другой — [использовать вкладку Upgrade.](upgrade-to-different-plan.md) Иногда это невозможно. Изменение планов вручную:

- Если **вкладка Upgrade** указывает, что обновление текущего плана не возможно.

- Если при выборе вкладки **Обновление** план, который вы хотите, не указан.

- Если вы не хотите обновить всех пользователей таким же образом. Некоторым организациям необходимо, чтобы разные пользователи были подписаны на разные планы. Для этого используйте ручное изменение.

Чтобы продолжить вручную, ознакомьтесь с [разделом Шаг 2. Купить новую подписку](#step-2-buy-a-new-subscription) в этом разделе.

> [!IMPORTANT]
> Если вы изменяете в план с меньшем количеством служб, связанных с данными, чем текущий план (понижение), необходимо вручную сохранять все данные, которые вы хотите сохранить. Дополнительные сведения см. в [дополнительных данных перед изменением планов.](back-up-data-before-switching-plans.md)

## <a name="step-2-buy-a-new-subscription"></a>Шаг 2. Покупка новой подписки

**Уже приобрели?** Если у вас уже есть подписка, в которая вы хотите переместить пользователей, пропустите этот шаг и перейдите к шагу [3.](#step-3-check-your-new-subscription-and-licenses) Проверьте новую подписку и лицензии в этом разделе.

ИЛИ

**Приобретение новой подписки и лицензий:** Следуйте шагам в [Microsoft 365 подписки для](../try-or-buy-microsoft-365.md) бизнеса, чтобы купить новую подписку.

Убедитесь в том, что покупаете подписку для той же организации, в которой сейчас находятся пользователи. Например, проверьте адреса электронной почты для пользователей, которые необходимо переместить. Если их адреса электронной почты включают contoso.com, необходимо приобрести новую подписку \@ для contoso.com.
Добавьте лицензию для каждого пользователя, которого хотите переместить.

## <a name="step-3-check-your-new-subscription-and-licenses"></a>Шаг 3. Проверка новой подписки и лицензий

1. В Центре администрирования перейдите на страницу **Выставление счетов** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=842054" target="_blank">Ваши продукты</a>.

2. **Убедитесь, что обе подписки перечислены и активны** Подписка, из которую перемещаются пользователи, и подписка, на которую перемещаются пользователи, должны быть указаны вместе. Если при первой проверке новой подписки нет, попробуйте еще раз позже. Убедитесь, что обе подписки активны. [Новая подписка не указана или неактивна](#the-new-subscription-isnt-listed-or-isnt-active).

3. **Убедитесь, что у вас достаточно лицензий для каждого пользователя** Каждому пользователю нужна лицензия, которая соответствует его подписке. Поэтому для перемещения десяти пользователей в Microsoft 365 бизнес премиум необходимо убедиться, что доступны десять лицензий.

4. **Нужно больше лицензий для новой подписки?**
   Перейдите на **страницу Ваши продукты** [и купите больше лицензий.](../licenses/buy-licenses.md)

> [Что насчет старых лицензий?](#what-about-the-old-licenses)

### <a name="the-new-subscription-isnt-listed-or-isnt-active"></a>Новая подписка не указана или неактивна

- **Если вы приобрели две подписки, и они обе не указаны**, возможно, они были приобретены для разных организаций (разных доменов). Подписки не распространяются за пределы организаций.

- **Если вы знаете, что у вас** есть дополнительная подписка, и она не указана здесь или не активна, позвоните [в службу поддержки Майкрософт.](../../business-video/get-help-support.md)

### <a name="what-about-the-old-licenses"></a>Что насчет старых лицензий?

Лицензии для текущей подписки будут удалены позже. С этого момента вы будете платить только за новые пользовательские лицензии.

## <a name="step-4-reassign-licenses"></a>Шаг 4. Повторное лицензирование

При обновлении с Office 365 до Microsoft 365 плана необходимо изменить назначения лицензии для всех пользователей. При изменении планов вручную лицензии не назначены автоматически.

### <a name="reassign-a-license-for-one-user"></a>Переназначение лицензии для одного пользователя

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. На странице **Активные пользователи** выберите пользователя, которому необходимо назначить лицензию.

3. На **вкладке Лицензии** и Приложения разоберите лицензии, выберите поля для лицензий, которые необходимо назначить, а затем **выберите Сохранить изменения.**

### <a name="reassign-licenses-for-multiple-users-at-once"></a>Переназначение лицензий сразу для нескольких пользователей

1. В Центре администрирования откройте страницу **Пользователи** \> <a href="https://go.microsoft.com/fwlink/p/?linkid=834822" target="_blank">Активные пользователи</a>.

2. Выберите круги рядом с именами пользователей, для которых необходимо заменить существующие лицензии.

3. В верхней части выберите три точки (больше действий), а затем выберите **Управление лицензиями продукта.**

4. Select **Replace existing product license assignments** \> **Next**.

5. Установите выключатель для продукта, лицензию на который вы хотите назначить, в положение **Вкл.**

    > [!TIP]
    > - Чтобы ограничить доступ пользователей к службам, переключите на переключатели в положение **Off** для служб, которые необходимо удалить для этого пользователя. Например, чтобы пользователь имел доступ ко всем имеющимся службам, кроме Skype для бизнеса online, установите выключатель для службы Skype для бизнеса online в положение **Выкл.**
    > - Все предыдущие назначения лицензий для выбранных пользователей будут удалены.

6. В нижней части области **Заменить имеющиеся продукты** нажмите **Заменить** \> **Закрыть**.

## <a name="step-5-cancel-subscriptions-or-remove-licenses-that-you-no-longer-need-optional"></a>Шаг 5. Отмена подписки или удаление лицензий, которые больше не нужны (необязательный)

Если вы переместили всех пользователей с одной подписки на другую и вам больше не нужна исходная подписка, вы можете [отменить ее](cancel-your-subscription.md).

Если вы переместили в другую подписку только часть пользователей, [удалите лицензии](../licenses/buy-licenses.md), которые вам больше не нужны.
