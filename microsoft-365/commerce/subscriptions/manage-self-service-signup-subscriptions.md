# <a name="manage-self-service-sign-up-subscriptions"></a>Управление подписками для самостоятельной регистрации

## <a name="what-are-self-service-sign-up-subscriptions"></a>Что такое подписки для самостоятельной регистрации?

Существует ограниченное количество бесплатных подписок для самостоятельной регистрации, доступных пользователям в вашей организации для регистрации. Пользователь может зарегистрироваться и использовать для себя только самообслуживаемую подписку. Вы управляете подписками для самостоятельной регистрации, блокируя регистрацию пользователей и удаляя бесплатные подписки, на которые зарегистрировались пользователи. Дополнительные сведения о самостоятельной регистрации и доступных подписках см. в документе "Использование самостоятельной [регистрации в организации".](../../admin/misc/self-service-sign-up.md)

## <a name="view-a-list-of-self-service-sign-up-subscriptions"></a>Просмотр списка подписок для самостоятельной регистрации

1. В Центре администрирования перейдите на страницу **Выставление счетов** > <a href="https://go.microsoft.com/fwlink/p/?linkid=842054" target="_blank">Ваши продукты</a>.
2. На **вкладке** "Продукты" выберите значок фильтра, а затем выберите **"Бесплатно".** Отображается список всех подписок для самостоятельной регистрации.

## <a name="how-are-these-subscriptions-different-from-self-service-purchase-subscriptions"></a>Чем эти подписки отличаются от подписок на самостоятельное приобретение?

Подписки для самостоятельной регистрации бесплатны и доступны для большего списка продуктов, чем подписки на самостоятельное приобретение. Когда пользователь подписывается на подписку на самообслуживающие покупки, он несет ответственность за ее оплату. Подписки на самостоятельное приобретение доступны только для продуктов Power Platform (Power BI, Power Apps и Power Automate), Project и Visio. Дополнительные сведения см. в [самостоятельной покупке: faq](self-service-purchase-faq.md).

## <a name="block-users-from-signing-up"></a>Блокировка регистрации для пользователей

Чтобы определить, могут ли пользователи зарегистрироваться для самостоятельной регистрации, используйте с параметром **AllowAdHocSubscriptions** cmdlet [**Set-MsolCompanySettings.**](https://docs.microsoft.com/powershell/module/msonline/set-msolcompanysettings?view=azureadps-1.0&preserve-view=true) Дополнительные сведения см. в сведениях о том, [как управлять настройками самообслуживки?](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-self-service-signup#how-do-i-control-self-service-settings)

## <a name="delete-a-self-service-sign-up-subscription"></a>Удаление подписки для самостоятельной регистрации

> [!IMPORTANT]
> При удалении подписки для самостоятельной регистрации вы блокируете доступ всех пользователей к своим данным и электронной почте, а также удаляете все данные и электронную почту.

1. В Центре администрирования перейдите на страницу **Выставление счетов** > <a href="https://go.microsoft.com/fwlink/p/?linkid=842054" target="_blank">Ваши продукты</a>.
2. На **вкладке** "Продукты" выберите значок фильтра, а затем выберите **"Бесплатно".**
3. Выберите подписку для самостоятельной регистрации, которую нужно удалить. 
4. На странице сведений о подписке в разделе **"Подписки** и параметры оплаты" выберите **"Удалить подписку".**
5. В области **"Удалить подписку"** повысьте его, а затем выберите **"Удалить подписку".**

## <a name="i-have-a-self-service-sign-up-subscription-that-blocks-directory-deletion"></a>У меня есть подписка для самостоятельной регистрации, которая блокирует удаление каталогов

Средства самостоятельной регистрации, для регистрации в которые могут зарегистрироваться отдельные пользователи, также создают гостевых пользователей для проверки подлинности в каталоге Azure AD. Чтобы избежать потери данных, эти продукты самообслуживания блокируют удаление каталогов, пока они не будут полностью удалены из каталога. Удалить их может только администратор Azure AD. Дополнительные сведения см. в подстановок "Удаление [каталога" в Azure Active Directory.](https://docs.microsoft.com/azure/active-directory/users-groups-roles/directory-delete-howto)
