# <a name="self-service-purchase-faq"></a>Вопросы и ответы по самостоятельной покупке

Самостоятельная Покупка дает пользователям возможность испытать новые технологии и разрабатывать решения, которые в конечном итоге повышают эффективность своих организаций. Центральные отделы снабжения и ИТ могут видеть все пользователи, которые приобрели и разворачивают самостоятельные решения для приобретения через <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центр администрирования Microsoft 365</a>. Администраторы могут отключать самостоятельные покупки для каждого продукта с помощью PowerShell. Чтобы узнать больше, ознакомьтесь [со статьей использование алловселфсервицепурчасе для модуля PowerShell мскоммерце](allowselfservicepurchase-powershell.md).

Самостоятельная Покупка доступна для платформы управления питанием (Power BI, Power Apps и Автоматизация управления питанием), Project и Visio.

> [!NOTE]
> Самостоятельная Покупка недоступна в Индии и недоступна для правительственных организаций и клиентов для образовательных учреждений.

## <a name="making-a-self-service-purchase"></a>Самостоятельная Покупка

### <a name="how-does-a-customer-make-a-self-service-purchase"></a>Как клиент делает самостоятельную покупку?

Пользователи могут выполнять самостоятельную покупку через Интернет на веб-сайтах продуктов или в приглашении на покупку в приложении. Сначала пользователям предлагается ввести адрес электронной почты, чтобы убедиться, что они являются пользователями существующего клиента Azure Active Directory (AD). Затем они направляются для входа с использованием учетных данных Azure AD. После входа клиент должен выбрать количество подписок, которые они хотят купить, и предоставить платеж по кредитной карте. По завершении покупки они могут начать использовать свою подписку. Покупатель имеет доступ к ограниченному представлению <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a> , где пользователи могут назначать лицензии на продукт другим пользователям в Организации.

### <a name="what-are-the-payment-options-for-self-service-purchases"></a>Каковы варианты оплаты при самостоятельных покупках?

В настоящее время кредитная карта — единственный доступный способ оплаты. Оплата через выставление счетов не поддерживается.

### <a name="who-can-buy-through-self-service-purchase"></a>Кто может покупать через самостоятельную покупку?

Любой пользователь с пользовательской учетной записью, не являющейся гостевым, в управляемом клиенте Azure AD может выполнить самостоятельную покупку. Самостоятельное приобретение недоступно для клиентов, которые являются правительственными или образовательными организациями. Если это применимо к вашей организации, то для управления самостоятельными покупками дополнительных действий не требуется.

Пользователи в организациях или рынках, которые не подлежат самостоятельному приобретению, видят сообщение с вопросом о том, как связаться с ИТ ИТ admin.

### <a name="can-guest-users-buy-through-self-service-purchase"></a>Могут гостевые пользователи покупать самостоятельную закупку?

Нет, гостевые пользователи не могут выполнять самостоятельную покупку в клиенте, в котором они являются гостями.

### <a name="can-users-synced-from-an-on-premises-active-directory-buy-through-self-service-purchase"></a>Можно синхронизировать пользователей из локальной службы Active Directory с помощью самостоятельного приобретения?

Если у пользователя есть активная учетная запись пользователя в подходящего клиента Azure AD, она может выполнить самостоятельную покупку.

### <a name="who-can-self-service-purchasers-assign-licenses-to"></a>Кому могут самостоятельно предоставлять покупатели лицензии на самостоятельное обслуживание?

Самостоятельные покупатели могут назначать лицензии только пользователям в одном клиенте Azure AD. У покупателя есть доступ к ограниченному представлению <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a> для назначения лицензий. Покупатели могут назначать лицензии тем продуктам, которые они приобрели с помощью самостоятельного приобретения, и могут назначать эти лицензии пользователям в том же клиенте Azure AD.

### <a name="where-does-the-self-service-purchaser-see-and-manage-their-purchases"></a>Где самостоятельный покупатель видит и управляет покупками?

Самостоятельные покупатели могут управлять покупками в ограниченном представлении <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центра администрирования Microsoft 365</a>. Покупатели всегда могут получить доступ к центру администрирования, используя плитку **"Администратор** " в средстве запуска приложений, встроенных во все приложения Microsoft 365 и Dynamics Online. Покупатели могут просматривать сделанные покупки, покупать дополнительные подписки на одну и ту же службу и назначать лицензии для этих подписок другим пользователям в Организации. Кроме того, покупатели могут просмотреть и оплатить свой счет, обновить способ оплаты и отказаться от их подписки.

## <a name="pricing"></a>Цены

### <a name="what-is-the-pricing-for-self-service-purchases"></a>Каковы цены для самостоятельных покупок?

Цены на продукты для самостоятельной покупки доступны на веб-сайте Майкрософт. Цены также отображаются в рамках функции извлечения, когда пользователи выполняют самостоятельную покупку. Эти цены могут отличаться от цен, которые Организация оплачивает при создании центральных покупок или цен, предлагаемых через партнера.

### <a name="who-is-responsible-for-payment"></a>Кто отвечает за оплату?

Сотрудник, который покупает подписку через самостоятельную покупку, — это лицо, которому выставлен счет и который отвечает за оплату на основе условий и ценообразования покупки.

## <a name="admin-capabilities"></a>Возможности администрирования

### <a name="what-capabilities-does-an-admin-have-for-self-service-purchases"></a>Какие возможности имеет администратор для самостоятельного приобретения?

Администраторы могут просматривать все самостоятельные покупки, сделанные в Организации, в <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a>. Они могут видеть название продукта, покупателя, подписки на покупку, дату истечения срока действия, историю заказов, покупную цену и пользователей, назначенных для каждой самостоятельной покупки. В центре управления питанием администраторы также могут просматривать возможности самостоятельного приобретения. Если это необходимо для Организации, администраторы могут отключать самостоятельные покупки для каждого продукта с помощью PowerShell. Администраторы имеют одинаковые политики управления данными и доступа к продуктам, приобретаемым через самостоятельную покупку или централизованно.

Администраторы также могут управлять самостоятельными покупками пользователей в Организации. Для получения дополнительных сведений обратитесь к разделу [use алловселфсервицепурчасе для модуля PowerShell мскоммерце](allowselfservicepurchase-powershell.md).

### <a name="how-is-microsoft-respecting-data-governance-and-compliance-by-enabling-self-service-purchase"></a>Как корпорация Майкрософт уважает управление данными и обеспечивает соответствие требованиям, разрешая самостоятельную покупку?

Администраторы поддерживают контроль над тем, какие службы и продукты доступны в их клиенте в соответствии с требованиями к управлению данными и соответствия требованиям. Все политики управления данными и доступа, включенные в Организации, применяются к доступным самостоятельным службам.

### <a name="who-owns-the-product-data-created-from-self-service-purchases"></a>Кто владеет данными продукции, созданными с помощью самостоятельных покупок?

Данные, созданные на основе продуктов, приобретенных через самостоятельную закупку, принадлежать Организации и управляются им.

### <a name="how-do-i-centralize-the-purchases-made-through-self-service-purchase"></a>Как централизовать покупки, совершенные с помощью самостоятельной покупки?

Администраторы могут назначать существующие лицензии или покупать дополнительные подписки на самостоятельные продукты для самостоятельного приобретения через существующие соглашения и цены для пользователей, которым назначена самостоятельная покупка. После назначения этих централизованно приобретенных лицензий администраторы могут запросить, чтобы покупатели отменили существующие подписки.

### <a name="where-does-the-admin-see-self-service-purchases"></a>Где администратор видит самостоятельные приобретения услуг?

Глобальные администраторы и администраторы выставления счетов могут просматривать подписки, приобретаемые с помощью самостоятельной **покупки в**  >  **Your products** <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a>. Они могут отфильтровать список продуктов, чтобы отображались только подписки, приобретенные через центральную торговую закупку, или включить подписки, приобретаемые с помощью самостоятельной покупки.

Администраторы могут видеть продукт, название покупателя, приобретенную подписку, дату истечения срока действия, историю заказов, покупную цену и назначенные пользователи.

## <a name="support-and-training"></a>Поддержка и обучение

### <a name="are-customers-it-departments-or-partners-expected-to-support-products-bought-through-self-service-purchase"></a>Являются ли ИТ отделы или партнеры, которые должны поддерживать продукты, приобретаемые через самостоятельную покупку услуг?

ИТ отделы и партнеры не должны предоставлять поддержку продуктов, приобретенных с помощью самостоятельной покупки. Корпорация Майкрософт предоставляет стандартную поддержку для самостоятельных покупателей.

### <a name="if-a-self-service-purchaser-calls-support-does-that-use-the-customers-premier-support-incidents"></a>Если самостоятельный абонент звонит в службу поддержки, то в этом случае используются обращения пользователей в службу поддержки Premier Support?

Самостоятельные покупатели не будут использовать службу поддержки Premier Support для получения поддержки по покупкам самостоятельных служб.

### <a name="how-are-users-expected-to-receive-training-on-the-products-they-buy-through-self-service-purchase"></a>Как пользователи должны будут получать обучающие материалы по продуктам, которые они приобрели через самостоятельную покупку?

Подробные учебные курсы для пользователей представлены на веб-сайтах продукции. У продуктов есть обучающие материалы, документация, примеры и надежные сообщества для получения ответов и советов непосредственно от других пользователей.

### <a name="what-happens-to-a-self-service-purchase-if-a-user-leaves-the-organization"></a>Что происходит с самостоятельным приобретением, если пользователь покидает организацию?

Если пользователь, который изначально купил продукт приобретения самообслуживания, оставляет организацию, действительные пользователи продолжают использовать продукт в течение всего срока подписки. Подписка остается активной до тех пор, пока покупатель не отменяет ее или не запрашивает отмену подписки через службу поддержки клиентов. Администраторы также могут назначить централизованно приобретенную лицензию пользователям отмененной подписки.

## <a name="partners"></a>Партнеры

### <a name="whats-the-role-of-microsofts-partners-in-self-service-purchases"></a>Какова роль партнеров корпорации Майкрософт в самостоятельных покупках?

Партнеры, у которых есть делегированные права на администрирование, могут видеть самостоятельные приобретения в <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">центре администрирования Microsoft 365</a>, как и администратор. Партнеры могут поддерживать организацию, которая хочет централизовать продукты, приобретаемые с помощью самостоятельных покупок. Кроме того, партнеры могут предоставить решения для расширения возможностей самостоятельных покупок.
