# <a name="self-service-purchase-faq"></a>Самостоятельное приобретение: faq

Самообслуживка дает пользователям возможность опробовать новые технологии и разрабатывать решения, которые в конечном итоге будут пользоваться преимуществами более крупных организаций. Центральные отделы закупок и ИТ-отделы могут быть всем пользователям, которые приобретают и развертывают решения для самостоятельного приобретения через Центр администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Администраторы могут отключить самообслуживающие покупки для каждого продукта с помощью PowerShell. Чтобы узнать больше, [см. использование AllowSelfServicePurchase для модуля MSCommerce PowerShell.](allowselfservicepurchase-powershell.md)

Самообслуживка доступна для Power Platform (Power BI, Power Apps и Power Automate), Project и Visio.

> [!NOTE]
> Самообслуживка недоступна в Индии или для государственных или образовательных клиентов. Project и Visio недоступны для самостоятельной покупки в Бразилии и Конго.

## <a name="making-a-self-service-purchase"></a>Самостоятельное приобретение

### <a name="how-does-a-customer-make-a-self-service-purchase"></a>Как клиент делает самообслуживаемую покупку?

Клиенты могут самостоятельно приобрести продукт через Интернет на веб-сайтах продуктов или в запросах на покупку из приложения. Сначала клиентам будет предложено ввести адрес электронной почты, чтобы убедиться, что они являются пользователем в существующем клиенте Azure Active Directory (AD). Затем они будут направлены на вход с использованием своих учетных данных Azure AD. После входов клиенту будет предложено выбрать, сколько подписок он хочет приобрести, и предоставить платеж по кредитной карте. После завершения покупки они могут начать использовать свою подписку. Покупатель имеет доступ к ограниченному представлению Центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365,</a> где он может назначать лицензии продукту другим людям в своей организации.

### <a name="what-are-the-payment-options-for-self-service-purchases"></a>Какие варианты оплаты доступны при самостоятельном приобретении?

В настоящее время единственным доступным способом оплаты является кредитная карта. Оплата через счет не поддерживается.

### <a name="who-can-buy-through-self-service-purchase"></a>Кто может приобрести услуги самостоятельной покупки?

Любой пользователь с не гостевой учетной записью в управляемом клиенте Azure AD может самостоятельно приобрести клиент. Самообслуживка недоступна для клиентов, которые являются государственными или образовательными организациями. Если это относится к вашей организации, то для управления самостоятельной покупкой не требуется никаких дополнительных действий.

Пользователи в организациях или рынках, не имеющие права на самообслуживку, видят сообщение с просьбой связаться с ИТ-администратором.

### <a name="can-guest-users-buy-through-self-service-purchase"></a>Могут ли гостовые пользователи приобретать услуги самостоятельно?

Нет, гостевых пользователей не может выполнить самостоятельное приобретение в клиенте, в котором они гостевых.

### <a name="can-users-synced-from-an-on-premises-active-directory-buy-through-self-service-purchase"></a>Могут ли пользователи, синхронизированные с локальной службой Active Directory, приобретать покупки самостоятельно?

Если у пользователя есть активная учетная запись пользователя в клиенте Azure AD, соответствующий требованиям, он может выполнить самостоятренную покупку.

### <a name="who-can-self-service-purchasers-assign-licenses-to"></a>Кому могут назначать лицензии самообслуживатели?

Самостоятельно приобретатели могут назначать лицензии только пользователям в одном клиенте Azure AD. Покупатель имеет доступ к ограниченному представлению Центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365</a> для назначения лицензий. Покупатели могут назначать лицензии тем продуктам, которые они приобрели самостоятельно, и назначать их только пользователям в том же клиенте Azure AD.

### <a name="where-does-the-self-service-purchaser-see-and-manage-their-purchases"></a>Где покупатель самообслуживий видит покупки и управляет ими?

Самостоятельно приобретатели могут управлять покупками в ограниченном представлении Центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Покупатели всегда могут получить доступ  к Центру администрирования с помощью плитки "Администратор" в приложении, встроенном во все приложения Microsoft 365 и Dynamics Online. Покупатели могут просматривать сделанные покупки, приобретать дополнительные подписки на ту же службу и назначать лицензии на эти подписки другим пользователям в своей организации. Кроме того, покупатели могут просматривать и оплачивать свой счет, обновлять способ оплаты и отменять подписку.

## <a name="pricing"></a>Цены

### <a name="what-is-the-pricing-for-self-service-purchases"></a>Какова стоимость покупок в самообслужии?

Цены на каждый продукт для самостоятельной покупки доступны на веб-сайте Майкрософт. Цены также отображаются при самостоятельной покупке. Эти цены могут отличаться от цен, которые организация платит при центральных покупках или ценах, предлагаемых партнером.

### <a name="who-is-responsible-for-payment"></a>Кто несет ответственность за оплату?

Лицо, приобретавший подписку с помощью самостоятельной покупки, является лицом, которому выставлен счет и который несет ответственность за оплату на основе условий и цен покупки.

## <a name="admin-capabilities"></a>Возможности администратора

### <a name="what-capabilities-does-an-admin-have-for-self-service-purchases"></a>Какие возможности есть у администратора для самостоятельного приобретения?

Администраторы могут просматривать все самостоятельно сделанные покупки в своей организации в Центре администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Они могут видеть продукт, имя покупателей, приобретенные подписки, дату окончания срока действия, журнал заказов, цену покупки и назначенного пользователя для каждой самостоятельной покупки. В Центре администрирования Power Platform администраторы также могут просматривать возможности самостоятельного приобретения. При необходимости администраторы могут отключить самообслуживающие покупки для каждого продукта с помощью PowerShell. Администраторы имеют одинаковые политики управления данными и доступа к продуктам, приобретаемой посредством самостоятельной покупки или централизованно.

Администраторы также могут управлять возможностью самостоятельной покупки пользователями в организации. Дополнительные сведения см. в использовании [AllowSelfServicePurchase для модуля MSCommerce PowerShell.](allowselfservicepurchase-powershell.md)

### <a name="how-is-microsoft-respecting-data-governance-and-compliance-by-enabling-self-service-purchase"></a>Как Майкрософт соблюдает управление данными и соответствие требованиям, разрешая самообслуживающие покупки?

Администраторы контролируют, какие службы и продукты доступны в клиенте на основе требований к управлению данными и обеспечению соответствия требованиям. Все политики управления данными и доступа, которые ваша организация включила, применяются к доступным самостоятельно приобретенным службам.

### <a name="who-owns-the-product-data-created-from-self-service-purchases"></a>Кому принадлежат данные о продукте, созданные при самостоятельном приобретении?

Данные, созданные на основе продуктов, приобретаемого посредством самостоятельной покупки, принадлежат организации и контролируются этой организацией.

### <a name="how-do-i-centralize-the-purchases-made-through-self-service-purchase"></a>Как централизовать покупки, сделанные с помощью самостоятельной покупки?

Администраторы могут назначить существующие лицензии или приобрести дополнительные подписки на продукты самообслуживки с помощью существующих соглашений и цен для пользователей, имеющих право на самостоятельное приобретение. После назначения этих централизованно приобретенных лицензий администраторы могут попросить покупателей отменить существующие подписки.

### <a name="where-does-the-admin-see-self-service-purchases"></a>Где администратор видит покупки с самообслуживом?

Глобальные администраторы и администраторы вы выставления счета могут видеть подписки, купленные через самообслуживающие покупки в Центре администрирования  >   <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Они могут фильтровать список продуктов, чтобы показывать только подписки, приобретенные через централизованные закупки, или включать подписки, приобретенные посредством самостоятельной покупки.

Администраторы могут видеть продукт, имя покупателей, приобретенную подписку, дату окончания срока действия, журнал заказов, цену покупки и назначенного пользователя.

## <a name="support-and-training"></a>Поддержка и обучение

### <a name="are-customers-it-departments-or-partners-expected-to-support-products-bought-through-self-service-purchase"></a>Ожидают ли ИТ-отделы или партнеры клиентов поддержку продуктов, приобретаемой посредством самостоятельной покупки?

ИТ-отделы и партнеры не должны предоставлять поддержку продуктов, приобретаемой посредством самостоятельной покупки. Корпорация Майкрософт предоставляет стандартную поддержку для самообслуживателей.

### <a name="if-a-self-service-purchaser-calls-support-does-that-use-the-customers-premier-support-incidents"></a>Если покупатель самостоятельно вызывает службу поддержки, использует ли он инциденты поддержки Premier?

Самостоятельно приобретатели не будут использовать инциденты поддержки Premier клиента для получения поддержки по самостоятельной покупке.

### <a name="how-are-users-expected-to-receive-training-on-the-products-they-buy-through-self-service-purchase"></a>Как предполагается, что пользователи будут проходить обучение по продуктам, приобретаемой с помощью самостоятельной покупки?

На веб-сайтах продуктов предоставляются широкие учебные курсы для пользователей. Продукты имеют руководство по обучению, документации, примерам и сильному сообществу, чтобы получать ответы и советы непосредственно от других пользователей.

### <a name="what-happens-to-a-self-service-purchase-if-a-user-leaves-the-organization"></a>Что произойдет с самостоятельной покупкой, если пользователь уедет из организации?

Если человек, который изначально приобрел продукт самостоятельной покупки, покидает организацию, допустимые пользователи продолжают использовать продукт в течение срока действия подписки. Подписка остается активной, пока покупатель не отменит ее напрямую или администратор не запросит отмену подписки через службу поддержки клиентов. Администраторы также могут назначить централизованно приобретенную лицензию пользователям отмененной подписки.

## <a name="partners"></a>Партнеры

### <a name="whats-the-role-of-microsofts-partners-in-self-service-purchases"></a>Какова роль партнеров Корпорации Майкрософт в самостоятельном приобретении?

Партнеры, которым делегированы права администратора, могут видеть покупки самостоятельно в Центре администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365,</a>как и администратор. Партнеры могут помочь в поддержке организации, которая хочет централизовать продукты, приобретаемые посредством самостоятельного приобретения. Кроме того, партнеры могут предлагать решения для расширения возможностей самостоятельной покупки.
