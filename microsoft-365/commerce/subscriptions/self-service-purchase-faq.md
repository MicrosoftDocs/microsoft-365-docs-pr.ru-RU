# <a name="self-service-purchase-faq"></a>FaQ покупки самообслужи

Самообслуживка позволяет пользователям опробовать новые технологии и разработать решения, которые в конечном итоге будут выгоднее для крупных организаций. Центральные группы закупок и ИТ-отделы имеют видимость для всех пользователей, которые покупают и развертывают решения по покупке самообслужива через центр администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Администраторы могут отключить самообслуживающие покупки на основе каждого продукта с помощью PowerShell. Дополнительные дополнительные новости см. в см. в обзоре [Использование AllowSelfServicePurchase для модуля MSCommerce PowerShell.](allowselfservicepurchase-powershell.md)

Самообслуживка доступна для Power Platform (Power BI, Power Apps и Power Automate), Project и Visio.

> [!NOTE]
> Самообслуживка недоступна в Индии или для государственных или образовательных клиентов. Project и Visio недоступны для самостоятельной покупки в Бразилии и Демократической Республике Конго.

## <a name="making-a-self-service-purchase"></a>Покупка самообслужи

### <a name="how-does-a-customer-make-a-self-service-purchase"></a>Как клиент делает покупку с самообслуживаем?

Клиенты могут сделать покупку самостоятельной службы в Интернете с веб-сайтов продуктов или из подсказок о покупке в приложении. Сначала клиентам требуется ввести адрес электронной почты, чтобы убедиться, что они являются пользователем существующего клиента Azure Active Directory (AD). Далее они должны войти с помощью учетных данных Azure AD. После регистрации клиенту будет предложено выбрать количество подписок, которые он хочет купить, и предоставить оплату по кредитной карте. После завершения покупки они могут начать использовать подписку. Покупатель имеет доступ к ограниченному представлению центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365,</a> где он может назначить лицензии продукту другим людям в своей организации.

### <a name="what-are-the-payment-options-for-self-service-purchases"></a>Какие варианты оплаты для покупок с самообслуживом?

В настоящее время кредитная карта является единственным доступным способом оплаты. Оплата посредством взимания не поддерживается.

### <a name="who-can-buy-through-self-service-purchase"></a>Кто может покупать с помощью самообслужива?

Любой пользователь с не гостевой учетной записью в управляемом клиенте Azure AD может сделать самообслуживаемую покупку. Покупка самообслуживателя недоступна для клиентов, которые являются государственными или образовательными организациями. Если это относится к организации, то для контроля покупки самообслужива не требуется никаких дополнительных действий.

Пользователи в организациях или на рынках, не имеющие права на покупку самообслуживки, видят сообщение с просьбой связаться со своим ИТ-администратором.

### <a name="can-guest-users-buy-through-self-service-purchase"></a>Могут ли гостевых пользователей покупать с помощью самообслужива?

Нет, гостевых пользователей не может завершить самообслуживную покупку в клиенте, в котором они гости.

### <a name="can-users-synced-from-an-on-premises-active-directory-buy-through-self-service-purchase"></a>Могут ли пользователи синхронизироваться с локальной покупкой Active Directory с помощью самостоятельной покупки?

Если у пользователя есть активная учетная запись в клиенте Azure AD, он может выполнить покупку самостоятельно.

### <a name="who-can-self-service-purchasers-assign-licenses-to"></a>Кому могут назначать лицензии покупатели самообслуживателей?

Покупатели самообслуживателей могут назначать лицензии только пользователям в том же клиенте Azure AD. Покупатель имеет доступ к ограниченному представлению центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365</a> для назначения лицензий. Покупатели могут назначать лицензии тем продуктам, которые они приобрели с помощью самообслуживателей, и могут назначать их только пользователям в том же клиенте Azure AD.

### <a name="where-does-the-self-service-purchaser-see-and-manage-their-purchases"></a>Где покупатель самообслуживки видит и управляет своими покупками?

Покупатели самообслуживателей могут управлять покупками в ограниченном представлении центра администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Покупатели всегда могут добраться до центра администрирования из плитки **администрирования** в пусковой установке приложения, встроенной во все онлайн-приложения Microsoft 365 и Dynamics. Покупатели могут просматривать сделанные ими покупки, покупать дополнительные подписки на ту же службу и назначать лицензии на эти подписки другим пользователям в их организации. Кроме того, покупатели могут просматривать и оплачивать счет, обновлять метод оплаты и отменять подписку.

## <a name="pricing"></a>Цены

### <a name="what-is-the-pricing-for-self-service-purchases"></a>Какова цена для покупок самообслужива?

Цены на каждый из продуктов для самостоятельной покупки доступны на веб-сайте Майкрософт. Цены также отображаются в ходе проверки при покупке самостоятельной службы. Эти цены могут отличаться от цен, которые организация платит при центральных покупках или ценах, предлагаемых через партнера.

### <a name="who-is-responsible-for-payment"></a>Кто несет ответственность за оплату?

Человек, который покупает подписку с помощью самообслуживной покупки, является лицом, которому выставлен счет и который несет ответственность за оплату в зависимости от условий и цен покупки.

## <a name="admin-capabilities"></a>Возможности администрирования

### <a name="what-capabilities-does-an-admin-have-for-self-service-purchases"></a>Какие возможности есть у администратора для покупок с самообслуживаем?

Администраторы могут просматривать все покупки самообслуживки, сделанные в их организации, в центре администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Они могут видеть продукт, имя покупателей, приобретенные подписки, дату истечения срока действия, историю заказов, цену покупки и назначенного пользователя для каждой покупки самообслуживки. В Центре администрирования power Platform администраторы также могут просматривать возможности покупок самообслужива. Если это необходимо для организации, администраторы могут отключить покупку самообслуживки на основе каждого продукта с помощью PowerShell. Администраторы имеют одинаковые политики управления данными и доступа к продуктам, купленным с помощью самостоятельной покупки или централизованно.

Администраторы также могут контролировать, могут ли пользователи в своей организации делать покупки с самообслуживаем. Дополнительные сведения см. [в обзоре Использование AllowSelfServicePurchase для модуля MSCommerce PowerShell.](allowselfservicepurchase-powershell.md)

### <a name="how-is-microsoft-respecting-data-governance-and-compliance-by-enabling-self-service-purchase"></a>Как Корпорация Майкрософт соблюдает управление данными и соблюдение требований, включив покупку с помощью самообслужива?

Администраторы поддерживают контроль над тем, какие службы и продукты доступны в клиенте, исходя из их требований к управлению данными и соответствия требованиям. Все политики управления данными и доступа, которые включила ваша организация, применяются к доступным приобретаемой службе самообслуживки.

### <a name="who-owns-the-product-data-created-from-self-service-purchases"></a>Кому принадлежат данные о продукте, созданные при покупках самообслужива?

Данные, созданные из продуктов, купленных при покупке самообслужива, принадлежат и контролируются организацией.

### <a name="how-do-i-centralize-the-purchases-made-through-self-service-purchase"></a>Как централизовать покупки, сделанные с помощью самообслужива?

Администраторы могут назначать существующие лицензии или покупать дополнительные подписки на продукты самообслуживки через существующие соглашения и цены для пользователей, назначаемой для покупок самообслужива. После назначения этих централизованно приобретенных лицензий администраторы могут попросить покупателей отменить существующие подписки.

### <a name="where-does-the-admin-see-self-service-purchases"></a>Где администратор видит покупки самообслужива?

Администраторы глобальной и платежной систем могут видеть подписки, купленные с помощью самообслуживной покупки в центре администрирования  >   <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365.</a> Они могут фильтровать список продуктов, чтобы показывать только подписки, приобретенные через центр закупок, или включать подписки, приобретенные через покупку самообслуживаемых.

Администраторы могут видеть продукт, имя покупателей, приобретенную подписку, дату истечения срока действия, историю заказов, цену покупки и назначенного пользователя.

## <a name="support-and-training"></a>Поддержка и обучение

### <a name="are-customers-it-departments-or-partners-expected-to-support-products-bought-through-self-service-purchase"></a>Ожидается ли, что ИТ-отделы или партнеры клиентов будут поддерживать продукты, приобретаемые с помощью самообслужива?

Ожидается, что ИТ-отделы и партнеры не будут оказывать поддержку продуктам, приобретаемой с помощью самообслужива. Корпорация Майкрософт предоставляет стандартную поддержку для покупателей самообслуживателей.

### <a name="if-a-self-service-purchaser-calls-support-does-that-use-the-customers-premier-support-incidents"></a>Если покупатель самообслуживки вызывает поддержку, используется ли в этом случае инциденты службы premier support клиента?

Покупатели самообслуживателей не будут использовать инциденты с премьер-поддержкой клиента для получения поддержки для покупок самообслуживателей.

### <a name="how-are-users-expected-to-receive-training-on-the-products-they-buy-through-self-service-purchase"></a>Как ожидается, что пользователи будут получать обучение по продуктам, которые они покупают с помощью самостоятельной покупки?

На веб-сайтах продукта предоставляется обширная подготовка для пользователей. Продукты имеют руководство обучением, документацией, примерами и сильными сообществами, чтобы получать ответы и советы непосредственно от других пользователей.

### <a name="what-happens-to-a-self-service-purchase-if-a-user-leaves-the-organization"></a>Что происходит с самообслуживаемой покупкой, если пользователь покидает организацию?

Если человек, который первоначально приобрел продукт самообслуживки, покидает организацию, допустимые пользователи по-прежнему будут использовать продукт в полном объеме на время действия подписки. Подписка остается активной до тех пор, пока покупатель не отменит ее напрямую или администратор запросит отмену подписки с помощью поддержки клиентов. Администраторы также могут назначить централизованную приобретенную лицензию пользователям отмененной подписки.

## <a name="partners"></a>Партнеры

### <a name="whats-the-role-of-microsofts-partners-in-self-service-purchases"></a>Какова роль партнеров Корпорации Майкрософт в покупках самообслужива?

Партнеры, которым делегирована привилегия администрирования, могут видеть покупки самообслуживки в центре администрирования <a href="https://go.microsoft.com/fwlink/p/?linkid=2024339" target="_blank">Microsoft 365,</a>как и администратор. Партнеры могут помочь поддерживать организацию, которая хочет централизовать продукты, купленные с помощью закупок самообслужива. Кроме того, партнеры могут предлагать решения для расширения возможностей самостоятельной покупки.
